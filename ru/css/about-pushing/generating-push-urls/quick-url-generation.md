# Быстрое создание URL

## 1. Как быстро получить URL

Если вы хотите создать группу URL для тестирования, вам нужно открыть [CSS Console > CSS Code Access > Push Generator](https://console.tencentcloud.com/live/livecodemanage) и нажать кнопку **Generate Push URL**, чтобы создать URL потока и три URL воспроизведения с различными протоколами воспроизведения.

![img](https://staticintl.cloudcachetci.com/cms/backend-cms/b37a0172d44f11eea2b0525400bb593a.png)

## 2. Как реализовать автоматическое создание на серверной части

### 2.1 URL потока

В реальных сценариях приложений невозможно вручную создавать URL потоков и воспроизведения для каждого вещателя. Вместо этого эти URL создаются автоматически вашим сервером. В качестве URL потока можно использовать любой URL, соответствующий спецификациям Tencent Cloud. Ниже приведен пример стандартного URL потока, состоящего из трех частей:

![url](https://staticintl.cloudcachetci.com/cms/backend-cms/b36b789ad44f11eea2b0525400bb593a.png)

- **CSS Code**
Также называется ID комнаты. Рекомендуется использовать случайные числа или ID пользователя. Валидный CSS code должен содержать префикс BIZID.
- **txTime**
Это время истечения срока действия URL. Формат — UNIX timestamp в шестнадцатеричной системе счисления, например, 5867D600 означает, что URL истечет в 00:00:00 1 января 2017 года. Как правило, txTime устанавливается на время, которое на 24 часа позже текущего времени. Не рекомендуется устанавливать слишком короткий период действия, чтобы избежать невозможности восстановления потока вещателем в случае временного разрыва сетевого соединения во время трансляции.
- **txSecret**
Это сигнатура защиты от горячих ссылок, которая используется для предотвращения имитации вашего серверного приложения злоумышленниками при создании URL потока. Дополнительные сведения о методе вычисления см. в разделе [Computing of Hotlink Protection](https://intl.cloud.tencent.com/document/product/267/7977).
- **Sample Code**
Перейдите в [CSS Console > CSS Code Access > Push Generator](https://console.tencentcloud.com/live/livecodemanage). В нижней части страницы приведены примеры кода (PHP и Java), показывающие, как создать URL с защитой от горячих ссылок.

## 3. Как рассчитать защиту от горячих ссылок

Защита от горячих ссылок безопасности относится к полю **txSecret** в URL потоков и воспроизведения. Она используется для предотвращения имитации вашего серверного приложения злоумышленниками при создании URL потока или перехвата вашего URL воспроизведения в своих целях.

### 3.1 Как это работает

Чтобы предотвратить имитацию вашего сервера при создании URL потока, вам необходимо настроить в консоли CSS

**ключ шифрования защиты от горячих ссылок**

, который маловероятно будет получен злоумышленниками для фальсификации действительного URL потока. На рисунке ниже показано, как это работает.

![img](https://staticintl.cloudcachetci.com/cms/backend-cms/b37188a5d44f11ee9409525400c26da5.png)

### 3.2 Процедура вычисления

- **Шаг 1: Обмен ключами**
Вам необходимо согласовать **ключ шифрования** на консоли официального сайта. Этот ключ шифрования используется для создания сигнатуры защиты от горячих ссылок на вашем сервере. Поскольку Tencent Cloud имеет такой же ключ, что и вы, она может расшифровать и проверить сигнатуру защиты от горячих ссылок, созданную вашим сервером.

Ключи шифрования классифицируются на **ключи защиты от горячих ссылок потока** и **ключи защиты от горячих ссылок воспроизведения**. Первые используются для создания URL защиты от горячих ссылок потока, вторые — для создания URL защиты от горячих ссылок воспроизведения. Вы можете настроить ключи защиты от горячих ссылок потока в [CSS Console](https://console.tencentcloud.com/live), как показано ниже:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b36c4fe1d44f11ee9409525400c26da5.png)

> **Защита от горячих ссылок воспроизведения отключена по умолчанию:** Поскольку синхронизация ключа защиты от горячих ссылок воспроизведения должна быть проведена на тысячах кластеров CDN, ключ не может часто изменяться на этапе отладки из-за длительного периода синхронизации. Если вам необходимо настроить защиту от горячих ссылок воспроизведения, обратитесь к нам, позвонив в нашу службу поддержки. Синхронизация во всех кластерах обычно занимает от 1 до 3 дней.

- **Шаг 2: Создание txTime**
В сигнатуре открытый текст — это txTime, который указывает период действия URL. Например, если текущее время — 2016-07-29 11:13:45, а ожидается, что созданный URL истечет через 24 часа, txTime можно установить на 2016-07-30 11:13:45.

Такая длинная строка времени займет слишком много места в URL. В реальном сценарии 2016-07-30 11:13:45 преобразуется в UNIX timestamp, т. е. 1469848425 (различные языки программирования серверной части обрабатываются напрямую доступными функциями времени при преобразовании). Затем timestamp преобразуется в шестнадцатеричную строку для дальнейшего сокращения длины строки, то есть txTime = 1469848425 (десятичная) = 579C1B69 (шестнадцатеричная).

Как правило, txTime устанавливается на время, которое на 24 часа позже текущего времени. Не рекомендуется устанавливать слишком короткий период действия, чтобы избежать невозможности восстановления потока вещателем в случае временного разрыва сетевого соединения во время трансляции.

- **Шаг 3: Создание txSecret**
txSecret создается следующим образом: txSecret = **MD5 (KEY + stream_id + txTime)**. Здесь **KEY** — это ключ шифрования, который вы настроили на шаге 1. В этом примере stream_id — это 8888_test001, txTime — это 579C1B69, рассчитанный выше, и **MD5** — это стандартный односторонний необратимый алгоритм хеширования.
- **Шаг 4: Создание URL с защитой от горячих ссылок**
Объедините URL потока (или воспроизведения), txTime, указывающий время истечения URL, и txSecret, который может быть расшифрован и проверен только Tencent Cloud, для создания безопасного URL с защитой от горячих ссылок.

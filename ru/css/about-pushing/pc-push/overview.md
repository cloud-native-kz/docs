# Обзор

## Обзор CSS для ПК

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c3256d5cd45011ee89c5525400170219.png)

Tencent Cloud CSS для ПК используется для передачи сжатых и закодированных изображений (таких как **прямые трансляции событий**, **обучение**, **проекция** или **игры**) на **URL отправки** Tencent **Video Cloud** с использованием ПО отправки (**OBS (рекомендуется)** или **XSplit**), установленного на ПК (**Windows/Mac**). Параллельно зрители могут видеть **изображения в реальном времени**, используя URL воспроизведения, соответствующий URL отправки.

## Процедура CSS для ПК

Вы можете реализовать CSS для ПК легко, следуя приведенным ниже шагам:

- **Куда отправляются потоки**: Получите **URL отправки** и 3 URL воспроизведения из консоли Tencent Cloud CSS.
- **Что отправляется**: Установите источники аудио/видео для отправки и параметры кодирования в стороннем ПО отправки.
- Зритель может смотреть CSS, используя наше RTMP DEMO для установки URL воспроизведения. Это помогает **доставить содержимое зрителям**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c324853cd45011ee9409525400c26da5.png)

## 1. До начала CSS

Активируйте сервис облачного CSS на Tencent Cloud. Если вы не активировали сервис Tencent Cloud CSS, нажмите здесь для [подачи заявки на сервис](https://console.tencentcloud.com/live).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c310a55ed45011eea2b0525400bb593a.png)

### 1.2 Генерирование URL отправки

Если у вас нет **URL отправки**, вы можете сгенерировать **URL отправки** и 3 URL воспроизведения, нажав [Управление доступом > CSS код доступа > Конфигурация доступа](https://console.tencentcloud.com/live).

URL, чье доменное имя **livepush.myqcloud.com**, является URL отправки:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c3102e50d45011ee89c5525400170219.png)

### 1.3 Выбор сети для CSS

- **Выбор сети**

| Тип сети | Доступность | Стабильность |
| --- | --- | --- |
| Проводная сеть | Низкая | Высокая |
| WIFI | Высокая | Низкая |

Рекомендуется использовать кабельную сеть, если это возможно, которая более стабильна, чем WIFI, и обеспечивает менее помехоустойчивые сигналы. Для трансляции событий рекомендуется использовать WIFI для удобства.

- **Проверка пропускной способности восходящего канала**

Требование к пропускной способности восходящего канала зависит от качества видео и разрешения. Как правило, лучшее качество видео с более высоким разрешением требует более высокой пропускной способности восходящего канала. Пропускная способность восходящего канала не должна быть менее 1 Мбит/с. Чтобы проверить состояние пропускной способности восходящего канала, вы можете провести проверку с использованием [speedtest](http://www.speedtest.net/).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f5443b27d45011ee9409525400c26da5.png)

### 1.4 Установка ПО отправки

- **Установка OBS**
Вы можете загрузить пакет установки на [официальном сайте OBS](https://obsproject.com/download) и установить согласно параметрам по умолчанию. OBS поддерживается в таких системах как Windows/Mac/Linux. Убедитесь, что это Open Broadcaster Software. OBS также предоставляет OBS Studio, который не рассматривается в этом документе.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/0595e7fdd45111eea2b0525400bb593a.png)

- **Установка XSplit**

Вы можете загрузить пакет установки на официальном сайте XSplit и установить его согласно параметрам по умолчанию.
XSplit не является бесплатным ПО. В качестве альтернативы вы можете использовать OBS (**бесплатно**). Для игровой трансляции XSplit имеет отдельный пакет установки. Для не-игровой трансляции рекомендуется использовать BroadCaster.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c314dd66d45011eeb0d9525400461a83.png)

## 2. Конфигурация параметров ПО

### 2.1 Установка URL отправки

Если у вас есть следующий URL отправки:

> rtmp://3891.livepush.myqcloud.com/live/3891_test?bizid=3891&txSecret=xxx&txTime=58540F7F

Вам необходимо установить две части отдельно:

> Первая часть URL отправки **"rtmp://3891.livepush.myqcloud.com/live/"** называется **FMS URL**. Вторая часть URL отправки **"3891_test?bizid=3891&txSecret=xxx&txTime=58540F7F"** называется **кодом потока**.

- **Конфигурация URL отправки OBS**

Нажмите **Settings**, выберите **Broadcasting Settings**, затем установите **Mode** на **CSS Stream**, **Streaming Service** на **Custom**, **FMS URL** на первую часть URL отправки и **Playback Path/Streaming Code** на вторую часть. Рекомендуется установить флажок **Automatic Reconnection**, что означает автоматическое переподключение отправки при обнаружении OBS исключения (например, разрыва сетевого соединения).

- **Параметры URL отправки OBS Studio**

Нажмите **Settings** в нижнем правом углу страницы, выберите **Stream**, затем установите **Streaming Type** на **Custom Streaming Media Server**, **URL** на первую часть URL отправки и **Stream Key** на вторую часть.

### 2.2 Установка источников аудио и видео

Источники аудио и видео подобны содержимому отправляемого пакета. Существует три формата:

- От устройств захвата видео, таких как камера или видеозаписывающее устройство.
- От окон ПК или источников игр;
- От файлов мультимедиа, хранящихся на ПК, таких как видео или изображения.
- **Параметры источников аудио и видео OBS. Примечание**: **Щелкните правой кнопкой мыши** в поле Source. Левая кнопка не применяется. Появляется меню Add, за которым следует **Acquire from Window**, Acquire from Screen, Image Source, Slide Show, Text Source, CLR Browser, **Video Capturing Device**, Game Source и т.д. Обычно мы используем Acquire from Window и Video Capturing Device. Другие источники конфигурируются по-другому. Далее мы расскажем, как конфигурировать **Video Capturing Device**.
- **Параметры источников аудио/видео OBS Studio**
Для получения дополнительной информации см. **Параметры источников аудио и видео OBS**.

### 2.3 Установка форматов аудио и видео

После конфигурации источника видео, хотя вы можете получить сигналы аудио и видео, исходные не подходят для распространения в сети, поскольку требуют большей пропускной способности. Поэтому важно конфигурировать параметры кодирования аудио и видео перед началом CSS.

| Элемент конфигурации | Характеристика |
| --- | --- |
| **x264** | Кодер h264 наиболее часто используется в отрасли с более высоким коэффициентом сжатия видео при одинаковом качестве изображения. **Рекомендуется выбрать этот вариант**. |
| Nvidia NVENC | Выполняет кодирование, используя ядро видеообработки, выделенное для видеокарты nv. Требуется видеокарта Nvidia. |
| Quick Sync | Использует технологию Intel Quick Sync Video для поддержки аппаратного кодирования с высокой скоростью кодирования и качеством изображения. Но имеет плохую совместимость и высокий битрейт. |
| **CBR** | Это один из режимов управления битрейтом кодирования видео, называется контролем постоянного битрейта. При постоянном битрейте он более подходит для передачи по сети. **Рекомендуется выбрать этот вариант**. |
| **AAC** | Это наиболее широко используемый формат кодирования аудио для прямой трансляции. **Рекомендуется выбрать этот вариант**. |

## 3. Проверка воспроизведения

### 3.1 Подтверждение URL воспроизведения

Если URL отправки (livepush):

> rtmp://3891.livepush.myqcloud.com/live/3891_test?bizid=3891&txSecret=xxx&txTime=58540F7F

то URL воспроизведения (liveplay):

| Протокол воспроизведения | URL воспроизведения |
| --- | --- |
| FLV | `rtmp://3891.liveplay.myqcloud.com/live/3891_test` |
| RTMP | `http://3891.liveplay.myqcloud.com/live/3891_test.flv` |
| HLS(m3u8) | `http://3891.liveplay.myqcloud.com/live/3891_test.m3u8` |

### 3.2 Проверка воспроизведения VLC

Нажмите здесь для [загрузки VLC](http://www.videolan.org/vlc/). Вы можете установить согласно параметрам по умолчанию. Откройте ПО, нажмите **Media Menu**, выберите **Open Network Stream**, введите URL воспроизведения и нажмите **Play**.

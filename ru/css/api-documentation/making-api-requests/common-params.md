# Общие параметры

Общие параметры используются для всех API, которые аутентифицируют запросители. Общие параметры должны быть включены во все запросы API и не будут описаны в отдельных документах API.

Точное содержание общих параметров будет варьироваться в зависимости от версии используемого метода подписи.

## Общие параметры для алгоритма подписи v3

При использовании алгоритма TC3-HMAC-SHA256 общие параметры должны быть равномерно размещены в заголовке HTTP-запроса, как показано ниже:

| Имя параметра | Тип | Обязателен | Описание |
| --- | --- | --- | --- |
| X-TC-Action | String | Да | Имя API для требуемой операции. Для получения конкретного значения см. описание общего параметра `Action` в входных параметрах в связанной документации API. Например, API для запроса списка экземпляров CVM — это `DescribeInstances`. |
| X-TC-Region | String | Да | Параметр региона, который используется для идентификации региона, к которому относятся данные, с которыми вы хотите работать. Для значений, поддерживаемых API, см. описание общего параметра `Region` в входных параметрах в связанной документации API. Примечание: этот параметр не требуется для некоторых API (что будет указано в связанной документации API) и не будет иметь никакого эффекта, даже если он передан. |
| X-TC-Timestamp | Integer | Да | Текущая временная метка UNIX, которая записывает время инициирования запроса API, например 1529223702. Примечание: если разница между временной меткой UNIX и временем сервера больше 5 минут, может возникнуть ошибка истечения подписи. |
| X-TC-Version | String | Да | Версия API для действия. Для допустимых значений см. описание общего входного параметра `Version` в документации API. Например, версия CVM — 2017-03-12. |
| Authorization | String | Да | Заголовок аутентификации HTTP-запроса, например: TC3-HMAC-SHA256 Credential=AKID*******/Date/service/tc3_request, SignedHeaders=content-type;host, Signature=fe5f80f77d5fa3beca038a248ff027d0445342fe2855ddc963176630326f1024  Здесь: - TC3-HMAC-SHA256: метод подписи, в настоящее время зафиксирован как это значение; - Credential: учетные данные подписи; AKID******* — это SecretId; Date — это дата в формате UTC, и это значение должно совпадать со значением X-TC-Timestamp (общий параметр) в формате UTC; service — имя продукта/услуги и обычно является префиксом имени домена. Например, имя домена cvm.tencentcloudapi.com относится к продукту CVM, и значение будет cvm; - SignedHeaders: заголовки, которые содержат информацию аутентификации; content-type и host — это обязательные заголовки; - Signature: дайджест подписи. |
| X-TC-Token | String | Нет | Токен, используемый для временного сертификата. Его необходимо использовать с временным ключом. Вы можете получить временный ключ и токен, вызвав API CAM. Для долгосрочного ключа токен не требуется. |

Предположим, вы хотите запросить список экземпляров облачной виртуальной машины в регионе Гуанчжоу. Структура запроса в виде URL запроса, заголовка запроса и тела запроса может быть следующей:

Пример структуры HTTP GET-запроса:

```
https://cvm.tencentcloudapi.com/?Limit=10&Offset=0

Authorization: TC3-HMAC-SHA256 Credential=AKID********************************/2018-10-09/cvm/tc3_request, SignedHeaders=content-type;host, Signature=5da7a33f6993f0614b047e5df4582db9e9bf4672ba50567dba16c6ccf174c474
Content-Type: application/x-www-form-urlencoded
Host: cvm.tencentcloudapi.com
X-TC-Action: DescribeInstances
X-TC-Version: 2017-03-12
X-TC-Timestamp: 1539084154
X-TC-Region: ap-guangzhou
```

Следующий пример показывает, как структурировать HTTP POST-запрос (application/json):

```
https://cvm.tencentcloudapi.com/

Authorization: TC3-HMAC-SHA256 Credential=AKID********************************/2018-05-30/cvm/tc3_request, SignedHeaders=content-type;host, Signature=582c400e06b5924a6f2b5d7d672d79c15b13162d9279b0855cfba6789a8edb4c
Content-Type: application/json
Host: cvm.tencentcloudapi.com
X-TC-Action: DescribeInstances
X-TC-Version: 2017-03-12
X-TC-Timestamp: 1527672334
X-TC-Region: ap-guangzhou

{"Offset":0,"Limit":10}
```

Пример структуры HTTP POST-запроса (multipart/form-data) (поддерживается только конкретными API):

```
https://cvm.tencentcloudapi.com/

Authorization: TC3-HMAC-SHA256 Credential=AKID********************************/2018-05-30/cvm/tc3_request, SignedHeaders=content-type;host, Signature=582c400e06b5924a6f2b5d7d672d79c15b13162d9279b0855cfba6789a8edb4c
Content-Type: multipart/form-data; boundary=58731222010402
Host: cvm.tencentcloudapi.com
X-TC-Action: DescribeInstances
X-TC-Version: 2017-03-12
X-TC-Timestamp: 1527672334
X-TC-Region: ap-guangzhou

--58731222010402
Content-Disposition: form-data; name="Offset"

0
--58731222010402
Content-Disposition: form-data; name="Limit"

10
--58731222010402--
```

## Общие параметры для алгоритма подписи v1

Для использования методов подписи HmacSHA1 и HmacSHA256 общие параметры должны быть помещены в строку запроса, как показано ниже:

| Имя параметра | Тип | Обязателен | Описание |
| --- | --- | --- | --- |
| Action | String | Да | Имя API для требуемой операции. Для получения конкретного значения см. описание общего параметра `Action` в входных параметрах в связанной документации API. Например, API для запроса списка экземпляров CVM — это `DescribeInstances`. |
| Region | String | Да | Параметр региона, который используется для идентификации региона, к которому относятся данные, с которыми вы хотите работать. Для значений, поддерживаемых API, см. описание общего параметра `Region` в входных параметрах в связанной документации API. Примечание: этот параметр не требуется для некоторых API (что будет указано в связанной документации API) и не будет иметь никакого эффекта, даже если он передан. |
| Timestamp | Integer | Да | Текущая временная метка UNIX, которая записывает время инициирования запроса API, например 1529223702. Если разница между значением и текущим системным временем слишком велика, может возникнуть ошибка истечения подписи. |
| Nonce | Integer | Да | Случайное положительное целое число, используемое вместе с `Timestamp` для предотвращения атак воспроизведения. |
| SecretId | String | Да | Идентифицирующий SecretId, полученный на странице [Cloud API Key](https://console.cloud.tencent.com/capi). SecretId соответствует уникальному SecretKey, который используется для создания подписи запроса (Signature). |
| Signature | String | Да | Подпись запроса, используемая для проверки действительности этого запроса. Она рассчитывается на основе фактических входных параметров. Для получения дополнительной информации о том, как это рассчитывается, см. документацию по аутентификации API. |
| Version | String | Да | Версия API для действия. Для допустимых значений см. описание общего входного параметра `Version` в документации API. Например, версия CVM — 2017-03-12. |
| SignatureMethod | String | Нет | Метод подписи. В настоящее время поддерживаются только HmacSHA256 и HmacSHA1. Алгоритм HmacSHA256 используется для проверки подписи только при указании этого параметра как HmacSHA256. В других случаях подпись проверяется с помощью HmacSHA1. |
| Token | String | Нет | Токен, используемый для временного сертификата. Его необходимо использовать с временным ключом. Вы можете получить временный ключ и токен, вызвав API CAM. Для долгосрочного ключа токен не требуется. |

Предположим, вы хотите запросить список экземпляров облачной виртуальной машины в регионе Гуанчжоу. Структура запроса в виде URL запроса, заголовка запроса и тела запроса может быть следующей:

Пример структуры HTTP GET-запроса:

```
https://cvm.tencentcloudapi.com/?Action=DescribeInstances&Version=2017-03-12&SignatureMethod=HmacSHA256&Timestamp=1527672334&Signature=37ac2f4fde00b0ac9bd9eadeb459b1bbee224158d66e7ae5fcadb70b2d181d02&Region=ap-guangzhou&Nonce=23823223&SecretId=AKID********************************

Host: cvm.tencentcloudapi.com
Content-Type: application/x-www-form-urlencoded
```

Пример структуры HTTP POST-запроса:

```
https://cvm.tencentcloudapi.com/

Host: cvm.tencentcloudapi.com
Content-Type: application/x-www-form-urlencoded

Action=DescribeInstances&Version=2017-03-12&SignatureMethod=HmacSHA256&Timestamp=1527672334&Signature=37ac2f4fde00b0ac9bd9eadeb459b1bbee224158d66e7ae5fcadb70b2d181d02&Region=ap-guangzhou&Nonce=23823223&SecretId=AKID********************************
```

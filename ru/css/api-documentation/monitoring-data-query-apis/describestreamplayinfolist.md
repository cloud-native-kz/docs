# DescribeStreamPlayInfoList

## 1. Описание API

Доменное имя для запроса API: live.intl.tencentcloudapi.com.

Этот API используется для запроса данных воспроизведения. Он поддерживает запрос деталей воспроизведения по имени потока и агрегированные данные по доменному имени воспроизведения. Данные за последние 4 минуты или около того не могут быть запрошены из-за задержки.
Примечание: для запроса по `AppName` необходимо сначала отправить тикет. После успешного подтверждения заявки потребуется примерно 5 рабочих дней (время указывается в ответе) для вступления конфигурации в силу.

Максимально 500 запросов могут быть инициированы в секунду для этого API.

## 2. Входные параметры

Следующий список параметров запроса предоставляет только параметры запроса API и некоторые общие параметры. Для полного списка общих параметров см. [Общие параметры запроса](https://www.tencentcloud.com/document/api/267/30763).

| Имя параметра | Обязательный | Тип | Описание |
| --- | --- | --- | --- |
| Action | Да | String | [Общие параметры](https://www.tencentcloud.com/document/api/267/30763). Значение, используемое для этого API: DescribeStreamPlayInfoList. |
| Version | Да | String | [Общие параметры](https://www.tencentcloud.com/document/api/267/30763). Значение, используемое для этого API: 2018-08-01. |
| Region | Нет | String | [Общие параметры](https://www.tencentcloud.com/document/api/267/30763). Этот параметр не требуется для этого API. |
| StartTime | Да | String | Время начала запроса, поддерживает запрос данных за последний месяц, разница между временем начала и временем окончания не может превышать двадцать четыре часа. Интерфейс запроса поддерживает два формата времени: |
| EndTime | Да | String | Время окончания запроса, поддерживает запрос данных за последний месяц, разница между временем начала и временем окончания не может превышать двадцать четыре часа. Интерфейс запроса поддерживает два формата времени: 1) YYYY-MM-DDThh:mm:ssZ: формат времени ISO, дополнительные сведения см. в [Описание формата даты ISO](https://intl.cloud.tencent.com/document/product/267/32941) 2) YYYY-MM-DD hh:mm:ss: При использовании этого формата по умолчанию представляет время Пекина. |
| PlayDomain | Нет | String | Доменное имя воспроизведения. Если этот параметр пуст, будут запрошены данные всех доменных имен воспроизведения. |
| StreamName | Нет | String | Имя потока (точное совпадение). Если этот параметр пуст, будут запрошены полные данные воспроизведения. |
| AppName | Нет | String | Адрес трансляции. Его значение совпадает с `AppName` в адресе воспроизведения. Поддерживает точное совпадение и действует только при одновременной передаче `StreamName`. Если оставить пусто, будут запрошены полные данные воспроизведения. Примечание: для запроса по `AppName` необходимо сначала отправить тикет. После успешного подтверждения заявки потребуется примерно 5 рабочих дней (время указывается в ответе) для вступления конфигурации в силу. |
| ServiceName | Нет | String | Имя сервиса. Допустимые значения: LVB, LEB. Если этот параметр пуст, будут запрошены все данные LVB и LEB. |

## 3. Выходные параметры

| Имя параметра | Тип | Описание |

# CreateLivePullStreamTask

## 1. Описание API

Доменное имя для запроса API: live.intl.tencentcloudapi.com.

Этот API используется для создания задачи по извлечению потоков из видеофайлов или внешнего источника прямой трансляции и публикации их на указанный URL-адрес назначения.
Примечания:

По умолчанию одновременно можно иметь максимум 20 задач по извлечению потоков. Вы можете отправить заявку, чтобы увеличить этот лимит.
Для видео поддерживаются только H.264 и H.265. Если исходное видео имеет другой формат, сначала его необходимо перекодировать.
Для аудио поддерживается только AAC. Если исходное аудио имеет другой формат, сначала его необходимо перекодировать.
Вы можете включить автоматическое удаление в консоли, чтобы автоматически удалять истекшие задачи.
Функция извлечения и трансляции является платной функцией. Сведения о её тарификации см. на странице
Relay
.
CSS отвечает только за извлечение и трансляцию содержимого. Убедитесь, что ваше содержимое авторизовано и соответствует действующему законодательству. В случае нарушения авторских прав или нарушения законодательства CSS приостановит для вас обслуживание и оставляет за собой право на судебное преследование.

Для этого API можно инициировать максимум 50 запросов в секунду.

## 2. Входные параметры

Следующий список параметров запроса содержит только параметры запроса API и некоторые часто используемые параметры. Полный список обычных параметров см. в разделе [Общие параметры запроса](https://www.tencentcloud.com/document/api/267/30763).

| Имя параметра | Обязательный | Тип | Описание |
| --- | --- | --- | --- |
| Action | Да | String | [Общие параметры](https://www.tencentcloud.com/document/api/267/30763). Значение, используемое для этого API: CreateLivePullStreamTask. |
| Version | Да | String | [Общие параметры](https://www.tencentcloud.com/document/api/267/30763). Значение, используемое для этого API: 2018-08-01. |
| Region | Да | String | [Общие параметры](https://www.tencentcloud.com/document/api/267/30763). Дополнительную информацию см. в [списке регионов](https://www.tencentcloud.com/document/api/267/30763#region-list), поддерживаемых продуктом. Этот API поддерживает только: ap-bangkok, ap-beijing, ap-guangzhou, ap-hongkong, ap-mumbai, ap-seoul, ap-shanghai, ap-singapore, ap-tokyo, eu-frankfurt, na-ashburn, na-siliconvalley. |
| SourceType | Да | String | Тип источника. Допустимые значения: |
| SourceUrls.N | Да | Array of String | URL-адрес(а) источника. Если `SourceType` имеет значение `PullLivePushLive`, вы можете указать только один URL-адрес источника. Если `SourceType` имеет значение `PullVodPushLive`, вы можете указать максимум 30 URL-адресов источника. Поддерживаемые форматы файлов: FLV, MP4, HLS. Поддерживаемые протоколы: HTTP, HTTPS, RTMP, RTMPS, RTSP, SRT. Примечания: 1. Рекомендуется использовать файлы FLV в качестве источника. Плохо чередующиеся файлы MP4 могут привести к прерывистому воспроизведению. Вы также можете перечередовать файлы MP4 перед добавлением их в качестве источника. 2. Не используйте домены приватной сети или вредоносные URL-адреса. CSS будет блокировать такие аккаунты. 3. Чтобы избежать проблем с отправкой и воспроизведением, убедитесь, что исходные файлы правильно чередуются. 4. Поддерживаемые форматы видеокодирования: H.264, H.265. 5. Поддерживаемый формат аудиокодирования: AAC. 6. Используйте небольшие видеофайлы, предпочтительно не длиннее одного часа. Загрузка больших файлов может занять много времени или восстановление после паузы. Трансляция может не удаться, если затраченное время превышает 15 секунд. |
| DomainName | Да | String | Доменное имя для отправки. Извлеченный поток отправляется на этот домен. Примечание: Если назначение не является адресом CSS и его формат отличается от формата адресов CSS, передайте полный адрес в `ToUrl`. Подробнее см. описание параметра `ToUrl`. |
| AppName | Да | String | Приложение для отправки. Извлеченный поток отправляется в это приложение. |
| StreamName | Да | String | Имя потока. Извлеченный поток отправляется под этим именем. |
| StartTime | Да | String | Время начала. Должно быть в формате UTC. Пример: 2019-01-08T10:00:00Z. Примечание: Пекинское время на 8 часов впереди UTC. Используется [формат ISO 8601](https://intl.cloud.tencent.com/document/product/266/11732#iso-date-format). |
| EndTime | Да | String | Время окончания. Примечания: 1. Время окончания должно быть позже времени начала. 2. Время окончания и время начала должны быть позже текущего времени. 3. Время окончания и время начала должны быть разделены менее чем на семь дней. Должно быть в формате UTC. Пример: 2019-01-08T10:00:00Z. Примечание: Пекинское время на 8 часов впереди UTC. Используется [формат ISO 8601](https://intl.cloud.tencent.com/document/product/266/11732#iso-date-format). |
| Operator | Да | String | Оператор. |
| PushArgs | Нет | String | Параметр отправки. Это пользовательский параметр, передаваемый во время отправки. Пример: bak=1&test=2 |
| CallbackEvents.N | Нет | Array of String | События для прослушивания. Если вы не передаете этот параметр, будут прослушиваться все события. TaskStart: Обратный вызов при запуске задачи TaskExit: Обратный вызов при завершении задачи VodSourceFileStart: Обратный вызов при начале извлечения из видеофайлов VodSourceFileFinish: Обратный вызов при остановке извлечения из видеофайлов ResetTaskConfig: Обратный вызов при изменении задачи  `TaskAlarm` указывает на событие предупреждения. Примеры `AlarmType`: PullFileUnstable: Извлечение из видеофайлов нестабильно. PushStreamUnstable: Отправка нестабильна. PullFileFailed: Ошибка извлечения из видеофайлов. PushStreamFailed: Ошибка отправки. FileEndEarly: Видеофайл закончился преждевременно. |
| VodLoopTimes | Нет | String | Количество раз для повтора видеофайлов. Значение по умолчанию: -1. -1: Повторять бесконечно 0: Не повторять > 0: Количество повторов. Задача завершится либо при повторе видео указанное количество раз, либо в указанное время окончания задачи, в зависимости от того, что произойдет раньше. Этот параметр действует только тогда, когда источником являются видеофайлы. |
| VodRefreshType | Нет | String | Поведение после изменения исходных видеофайлов (`SourceUrls`). ImmediateNewSource: Немедленно воспроизвести новые видео ContinueBreakPoint: Воспроизвести новые видео после завершения текущего видео (оставшиеся видео из старого списка воспроизведения не будут воспроизведены).  Этот параметр действует только если источник перед изменением является видеофайлом. |
| CallbackUrl | Нет | String | Пользовательский URL-адрес обратного вызова. Обратные вызовы о событиях извлечения и трансляции будут отправлены на этот URL-адрес. |
| ExtraCmd | Нет | String | Другие параметры. Например, вы можете использовать `ignore_region` для игнорирования переданного региона и назначения региона на основе распределения нагрузки. |
| Comment | Нет | String | Примечания к задаче, не превышающие 512 байт. |
| ToUrl | Нет | String | Полный URL-адрес назначения. Если вы указываете этот параметр, убедитесь, что передали пустую строку для `DomainName`, `AppName` и `StreamName`.  Примечание: Убедитесь, что время истечения подписи позже времени окончания задачи. |
| BackupSourceType | Нет | String | Тип резервного источника. PullLivePushLive: Прямая трансляция PullVodPushLive: Видеофайлы Примечания: 1. Резервные источники поддерживаются только если тип первичного источника — прямая трансляция. 2. Если извлечение из первичного источника прерывается, система будет извлекать из резервного источника. 3. Если резервный источник является видеофайлом, каждый раз при завершении видео система проверяет, восстановился ли первичный источник, и переключается обратно при восстановлении. |
| BackupSourceUrl | Нет | String | URL-адрес резервного источника. Вы можете указать только один URL-адрес резервного источника. |
| WatermarkList.N | Нет | Array of [PullPushWatermarkInfo](https://www.tencentcloud.com/document/api/267/30767#PullPushWatermarkInfo) | Информация о водяных знаках для добавления. Примечания: 1. Вы можете добавить до четырех водяных знаков в разные места видео. 2. Убедитесь, что используете общедоступные URL-адреса для изображений водяных знаков. 3. Поддерживаемые форматы изображений включают PNG, JPG и GIF. |
| VodLocalMode | Нет | Integer | Использовать ли локальный режим, когда тип источника — видеофайлы. По умолчанию имеет значение `0`. 0: Не использовать локальный режим 1: Использовать локальный режим Примечание: Если вы включите локальный режим, файлы MP4 будут загружены в локальное хранилище, и локальные файлы будут использованы для отправки. Это обеспечивает более надежную отправку. Отправка локального файла будет взиматься дополнительная плата. |

### Пример 1. Образец запроса

#### Пример входных данных

```
POST / HTTP/1.1
Host: live.intl.tencentcloudapi.com
Content-Type: application/json
X-TC-Action: CreateLivePullStreamTask
<Common request parameters>

{
    "SourceType": "PullLivePushLive",
    "DomainName": "5000.livepush.myqcloud.com",
    "SourceUrls": [
        "rtmp://5000.liveplay.myqcloud.com/live/stream1"

    "StreamName": "test",
    "AppName": "live",
    "StartTime": "2019-12-16T11:02:00Z",
    "Operator": "vinson",
    "EndTime": "2019-12-17T12:02:00Z"
}
```

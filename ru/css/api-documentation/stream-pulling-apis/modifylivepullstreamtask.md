# ModifyLivePullStreamTask

## 1. Описание API

Доменное имя для запроса API: live.intl.tencentcloudapi.com.

Этот API используется для изменения задачи извлечения потока.

Вы не можете изменить URL-адрес назначения. Для публикации на новый адрес, пожалуйста, создайте новую задачу.
Вы не можете изменить тип источника. Для использования другого типа источника, пожалуйста, создайте новую задачу.

Максимум 20 запросов могут быть инициированы в секунду для этого API.

## 2. Входные параметры

В следующем списке параметров запроса приведены только параметры API запроса и некоторые общие параметры. Полный список общих параметров см. в разделе [Общие параметры запроса](https://www.tencentcloud.com/document/api/267/30763).

| Имя параметра | Обязательный | Тип | Описание |
| --- | --- | --- | --- |
| Action | Да | String | [Общие параметры](https://www.tencentcloud.com/document/api/267/30763). Значение, используемое для этого API: ModifyLivePullStreamTask. |
| Version | Да | String | [Общие параметры](https://www.tencentcloud.com/document/api/267/30763). Значение, используемое для этого API: 2018-08-01. |
| Region | Да | String | [Общие параметры](https://www.tencentcloud.com/document/api/267/30763). Дополнительные сведения см. в разделе [Список поддерживаемых регионов](https://www.tencentcloud.com/document/api/267/30763#region-list). Этот API поддерживает только: ap-bangkok, ap-beijing, ap-guangzhou, ap-hongkong, ap-mumbai, ap-seoul, ap-shanghai, ap-singapore, ap-tokyo, eu-frankfurt, na-ashburn, na-siliconvalley. |
| TaskId | Да | String | Идентификатор задачи. |
| Operator | Да | String | Оператор. |
| SourceUrls.N | Нет | Array of String | URL-адрес(а) источника. Если `SourceType` это `PullLivePushLive`, вы можете указать только один URL источника. Если `SourceType` это `PullVodPushLive`, вы можете указать до 30 URL-адресов источника. |
| StartTime | Нет | String | Время начала. Должно быть в формате UTC. Пример: 2019-01-08T10:00:00Z. Примечание: время Пекина на 8 часов впереди UTC. Используется [формат ISO 8601](https://intl.cloud.tencent.com/document/product/266/11732#iso-date-format). |
| EndTime | Нет | String | Время окончания. Примечания: 1. Время окончания должно быть позже времени начала. 2. Время окончания и время начала должны быть позже текущего времени. 3. Время окончания и время начала должны быть менее чем на семь дней друг от друга. Должно быть в формате UTC. Пример: 2019-01-08T10:00:00Z. Примечание: время Пекина на 8 часов впереди UTC. Используется [формат ISO 8601](https://intl.cloud.tencent.com/document/product/266/11732#iso-date-format). |
| VodLoopTimes | Нет | Integer | Количество повторений видеофайлов. -1: Повторять неограниченно 0: Не повторять > 0: Количество повторений. Задача завершится либо когда видео повторены указанное количество раз, либо в указанное время окончания задачи, в зависимости от того, что произойдет раньше. Этот параметр действителен только если источником являются видеофайлы. |
| VodRefreshType | Нет | String | Поведение после изменения видеофайлов источника (`SourceUrls`). ImmediateNewSource: Воспроизводить новые видео немедленно ContinueBreakPoint: Завершить текущее видео, а затем извлечь из нового источника. Этот параметр действителен только если источником являются видеофайлы. |
| Status | Нет | String | Включить или приостановить задачу. Допустимые значения: enable pause |
| CallbackEvents.N | Нет | Array of String | События для прослушивания. Если вы не передадите этот параметр, будут прослушиваться все события. TaskStart: Обратный вызов для запуска задачи TaskExit: Обратный вызов для завершения задачи VodSourceFileStart: Обратный вызов для начала извлечения из видеофайлов VodSourceFileFinish: Обратный вызов для остановки извлечения из видеофайлов ResetTaskConfig: Обратный вызов для изменения задачи |
| CallbackUrl | Нет | String | Пользовательский URL обратного вызова. Обратные вызовы будут отправлены на этот URL. |
| FileIndex | Нет | Integer | Индекс видео для начала воспроизведения. Значение этого параметра не может быть меньше 1 или больше количества элементов в `SourceUrls`. |
| OffsetTime | Нет | Integer | Смещение воспроизведения (в секундах). Примечания: 1. Этот параметр должен использоваться вместе с `FileIndex`. |
| Comment | Нет | String | Примечания для задачи. |
| BackupSourceType | Нет | String | Тип резервного источника. PullLivePushLive: Прямая трансляция PullVodPushLive: Видеофайлы Примечания: 1. Резервные источники поддерживаются только если тип основного источника — прямая трансляция. 2. Оставление этого параметра пустым сбросит резервный источник. 3. Когда извлечение из основного источника прерывается, система будет извлекать из резервного источника. 4. Если резервный источник является видеофайлом, каждый раз, когда видео заканчивается, система проверит, восстановлен ли основной источник, и переключится обратно, если да. |
| BackupSourceUrl | Нет | String | URL-адрес резервного источника. Вы можете указать только один URL резервного источника. |
| WatermarkList.N | Нет | Array of [PullPushWatermarkInfo](https://www.tencentcloud.com/document/api/267/30767#PullPushWatermarkInfo) | Информация о водяных знаках для добавления. Примечания: 1. Вы можете добавить до четырех водяных знаков в разные места видео. 2. Убедитесь, что вы используете общедоступные URL-адреса для изображений водяных знаков. 3. Поддерживаемые форматы изображений включают PNG и JPG. 4. Если вы измените конфигурацию водяного знака задачи, источником которой является список видеофайлов, новая конфигурация вступит в силу для следующего файла в списке. 5. Если вы измените конфигурацию водяного знака задачи, источником которой является прямой поток, новая конфигурация вступит в силу немедленно. 6. Если вы хотите прекратить использование водяных знаков, передайте пустой массив. 7. В настоящее время анимированные водяные знаки не поддерживаются. |
| VodLocalMode | Нет | Integer | Использовать ли локальный режим, когда тип источника — видеофайлы. По умолчанию `0`. 0: Не использовать локальный режим 1: Использовать локальный режим Примечание: Если вы включите локальный режим, файлы MP4 будут загружены в локальное хранилище, и локальные файлы будут использованы для отправки. Это обеспечивает более надежную отправку. Отправка локального файла будет влечь дополнительные расходы. |

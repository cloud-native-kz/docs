# Контроль доступа на основе CAM

CSS поддерживает управление разрешениями через CAM, позволяя вам управлять доступом к вашим доменам CSS, параметрам и другим данным. Вы можете создавать, управлять или прекращать работу пользователей или групп пользователей, а также предоставлять им разрешения на доступ к API для реализации управления идентификацией и политиками.
Вы можете использовать CAM для привязки пользователя или группы пользователей к политике, которая позволяет или запрещает им доступ к указанным ресурсам для выполнения определённых задач.

## Концепции

- Корневой аккаунт: аккаунт Tencent Cloud
- Под-пользователь: пользователь, созданный корневым аккаунтом и полностью им управляемый.
- Сотрудник: вы можете добавить другой корневой аккаунт в качестве сотрудника к своему аккаунту. Добавленный аккаунт становится под-аккаунтом вашего аккаунта.
- Группа пользователей: пользователи, выполняющие одинаковые функции и могущие быть привязанными к политике разрешений для централизованного управления доступом.

> **Примечание:** дополнительную информацию о концепциях и разрешениях см. в разделе [Типы пользователей](https://tencentcloud.com/document/product/598/32633).

## Инструкции

### Шаг 1. Создание под-пользователя или группы пользователей

Под каждым корневым аккаунтом можно создать одного или нескольких под-пользователей, которые могут быть связаны с определёнными ролями и политиками. Под-пользователь имеет уникальный ID и учётные данные, которые можно использовать для входа в консоль Tencent Cloud. Он также имеет доступ к API. Вы можете войти в [консоль CAM](https://console.tencentcloud.com/cam/) для создания под-пользователя.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/933d84158b8711efac345254002693fd.png)

> **Примечание:** подробные инструкции см. в разделе [Создание под-пользователя](https://intl.cloud.tencent.com/document/product/598/13674) и [Создание группы пользователей](https://intl.cloud.tencent.com/document/product/598/33380).

### Шаг 2. Добавление политики под-пользователю или группе пользователей

Вы можете связать политики на странице управления пользователями/группами пользователей или на странице управления политиками. Подробные инструкции см. в разделе [Управление авторизацией](https://intl.cloud.tencent.com/document/product/598/10602).

Способ 1. Добавление политики под-пользователю или группе пользователей

Способ 2. Связывание политики с пользователем/группой пользователей

Перейдите на страницу пользователей/групп пользователей и выберите пользователя/группу, к которому вы хотите добавить политику.

- Выберите **Users > User List** или **User Groups** на левой боковой панели консоли CAM. Найдите пользователя/группу, к которому вы хотите добавить политику, нажмите **Authorize** справа, выберите политику CSS и нажмите **OK**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d12085a48b8811ef9ba5525400f69702.png)

- Выберите **Users > User List** или **User Groups** на левой боковой панели и нажмите на имя пользователя/группы, к которому вы хотите добавить политику. Нажмите **Associate Policy**, выберите политику CSS и нажмите **OK**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/2de5af908b8911ef9ed652540075b605.png)

Выберите **Policies** на левой боковой панели консоли CAM, найдите политику, которую вы хотите связать, и нажмите **Associate User/User Group/Role** в столбце **Operation**. Выберите пользователя/группу, с которым вы хотите связать политику, и нажмите **OK**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/889c49f48b8911efa11a525400a9236a.png)

#### Доступные для добавления политики

- **Предустановленные политики**: вы можете просмотреть все предустановленные политики на странице **Policies**.
  - Предустановленные политики CSS включают [QcloudLIVEFullAccess](https://console.tencentcloud.com/cam/policy/detail/9545933&QcloudLIVEFullAccess&2) (политика чтения и записи) и [QcloudLIVEReadOnlyAccess](https://console.tencentcloud.com/cam/policy/detail/13346800&QcloudLIVEReadOnlyAccess&2) (политика только для чтения).
  - Для использования тегов пользователем необходимо связать [QcloudTAGFullAccess](https://console.tencentcloud.com/cam/policy/detail/1592575&QcloudTAGFullAccess&2) (полный доступ на чтение и запись по тегам).
  - Для использования журналов в реальном времени свяжите [QcloudCamFullAccess](https://console.tencentcloud.com/cam/policy/detail/596169&QcloudCamFullAccess&2) (полный доступ на чтение/запись к CAM).
  - Для использования функции скриншотов и обнаружения порнографии свяжите [QcloudAccessFoLVBRoleInSaveLiveScreenshottoCOS](https://console.tencentcloud.com/cam/policy/detail/115867019&QcloudAccessFoLVBRoleInSaveLiveScreenshottoCOS&2) с вашей ролью сервиса CSS, чтобы предоставить ей доступ к COS.
- **Пользовательская политика**: перейдите на страницу **Policies**, нажмите **Create Custom Policy** и выберите **Create by Policy Generator**. Дополнительную информацию см. в разделе [Пользовательская политика](https://intl.cloud.tencent.com/document/product/598/10601).

> **Примечание:** в настоящее время некоторые API CSS поддерживают авторизацию на уровне ресурсов.

**Пример:** если вы хотите разрешить под-пользователю использовать API **DescribeLiveDomains**, выполните следующие действия для предоставления разрешения.

  1.1. Создайте политику уровня домена, разрешающую доступ к API: перейдите на страницу **Create by Policy Generator** и выполните следующие параметры:

| Элемент | Обязательный | Параметр |
| --- | --- | --- |
| Effect | Да | Выберите Allow |
| Service | Да | Выберите Cloud Streaming Services |
| Action | Да | Выберите DescribeLiveDomains |
| Resource | Да | Выберите все ресурсы или конкретные ресурсы. Сервисы Tencent Cloud, для которых гранулярность авторизации является операцией или сервисом, не поддерживают описания ресурсов из шести частей; для них выберите "All resources". Для сервисов Tencent Cloud, поддерживающих авторизацию на уровне ресурсов, вы можете выбрать конкретные ресурсы. Сведения о методе описания ресурсов и гранулярности авторизации сервисов Tencent Cloud см. в разделе [Продукты с поддержкой CAM](https://intl.cloud.tencent.com/document/product/598/10588). |
| Condition | Нет | Установите условие вступления авторизации в силу. Если вы введёте IP-адреса, API будет доступен только в том случае, если запрос исходит из указанного диапазона IP. Вы также можете добавить другие условия. Дополнительную информацию см. в разделе [Условия](https://www.tencentcloud.com/document/product/598/58420). |

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9f52151e8b8a11ef9ba5525400f69702.png)

> **Примечание:** если вы хотите авторизовать несколько сервисов, нажмите **Add Permissions**.

  1.2. Нажмите **Next** для создания политики. Затем свяжите её, используя один из двух приведённых выше способов.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/eccf15518b8a11efa11a525400a9236a.png)

### Шаг 3. Использование под-аккаунта

Теперь вы можете использовать аккаунт под-пользователя (ID аккаунта и пароль) для вызова авторизованного API (такого как `DescribeLiveDomains`) и получения соответствующих данных CSS (таких как все домены под текущим аккаунтом).

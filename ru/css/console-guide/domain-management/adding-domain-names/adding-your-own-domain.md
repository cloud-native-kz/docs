# Добавление собственного домена

Для использования CSS требуется как минимум два доменных имени: одно в качестве домена push и одно в качестве домена воспроизведения. Домен push и домен воспроизведения не могут быть одинаковыми. Однако они могут различаться по доменам второго уровня, не ограничиваясь двумя поддоменами. Например, `a.example.com` можно использовать в качестве домена push, а `b.example.com` можно использовать в качестве домена воспроизведения для услуг push и pull.

## Примечание

CSS по умолчанию предоставляет тестовое доменное имя `xxxx.livepush.myqcloud.com`. Вы можете использовать его для тестирования push, но мы не рекомендуем использовать его в качестве домена push для вашего реального бизнеса. Это доменное имя нельзя удалить.

## Предварительные требования

Вы активировали [CSS](https://www.tencentcloud.com/products/css).

## Добавление собственного домена

1. Войдите в [консоль CSS](https://console.tencentcloud.com/live) и выберите **Domain Management** на левой боковой панели.
2. Нажмите **Add Domain** и выполните следующие настройки во всплывающем окне:
  2.1. Если вам нужно добавить **домен push**: введите доменное имя и выберите тип домена как **Push Domain**.
  2.2. Если вам нужно добавить **домен воспроизведения**: введите доменное имя, выберите тип домена как **Playback Domain** и выберите регион ускорения, по умолчанию — **Chinese mainland**.
  2.3. Теги используются для классификации и управления ресурсами в разных измерениях. Если существующий тег не соответствует вашим требованиям, вы также можете перейти в [Tag Console](https://console.tencentcloud.com/tag/taglist) для унифицированного управления тегами.
  2.4. Нажмите **Add domain**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e144d1d70fa911f09fca52540099c741.png)

> **Примечание:** Доменное имя может содержать до **45** символов и не может содержать прописные буквы. По умолчанию вы можете добавить до 100 доменов в каждой учетной записи Tencent Cloud. Если вам нужно добавить более 100 доменов, [отправьте заявку](https://console.tencentcloud.com/workorder/category) для увеличения лимита. Вы можете изменить регион ускорения добавленного домена. На странице **Domain Management** нажмите на имя домена или нажмите **Manage** справа. Выберите вкладку **Advanced Configuration**, нажмите **Edit** в области **Region configuration**, выберите регион ускорения еще раз во всплывающем окне и нажмите **Save**.

## Проверка домена

Чтобы убедиться, что домен может быть добавлен только его владельцем, вам необходимо проверить права собственности на домен перед его добавлением в консоль CSS. Например, чтобы добавить `a.test.com`, вам нужно проверить права собственности на `test.com`. При добавлении доменов с одним и тем же родительским доменом, таких как `b.test.com`, повторная проверка не требуется. Вы можете проверить домен либо добавив запись DNS, либо загрузив HTML-файл. Если ранее добавленный домен не проверен, при добавлении домена с одинаковым родительским доменом все равно требуется проверка.

### Запись DNS

1. Войдите в [консоль DNSPod](https://console.dnspod.com/).
2. Выберите **DNS** > **My Domains** на левой боковой панели и нажмите родительский домен домена, который вы хотите добавить.
3. На странице **Record Management** нажмите **Add Record**.
4. Введите следующую информацию:

| Параметр | Описание |
| --- | --- |
| Host | Введите "cssauth". |
| Record Type | Выберите "TXT". |
| Record Value | CSS присваивает уникальное значение записи для каждого домена. Вы можете просмотреть его в консоли CSS при добавлении домена. |

5. Нажмите **Confirm**. Запись TXT вступит в силу примерно за пять минут.
6. Нажмите **Verify and add domain**. Если проверка пройдена успешно, вы можете перейти к следующему шагу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3f14d3fb0faa11f08c275254001c06ec.png)

### HTML-файл

Вы также можете проверить домен, загрузив HTML-файл.

1. Когда вам будет предложено проверить домен в консоли CSS, выберите **HTML file**.
2. Загрузите файл.
3. Загрузите файл в корневой каталог домена второго уровня.
4. Убедитесь, что файл доступен по адресу `http://second-level domain/second-level domain_cssauth.html`.
5. Нажмите **Verify and add domain**. Если проверка пройдена успешно, вы можете перейти к следующему шагу.

> **Примечание:** После завершения **Basic settings** вы можете перейти к шагу **CNAME configuration**. Дополнительную информацию о конфигурации CNAME см. в разделе [Configuring CNAME for Domain Name](https://intl.cloud.tencent.com/document/product/267/31057).

# Настройка CNAME

После подключения домена к CSS система автоматически назначит вам CNAME домен (суффикс CNAME для домена push — `.tlivepush.com`, суффикс CNAME для домена playback — `.txlivecdn.com`). Вы можете просмотреть его в разделе [Domain Management](https://console.tencentcloud.com/live/domainmanage). Чтобы сделать ваш домен доступным, вам необходимо добавить запись CNAME у вашего поставщика услуг DNS. Вы можете использовать CSS только после того, как конфигурация вступит в силу.

## Примечания

- Разрешение CNAME требуется как для доменов playback, так и для доменов push.
- Для получения подробных инструкций по добавлению записи CNAME обратитесь к вашему поставщику услуг DNS.
- Конфигурация CNAME обычно вступает в силу примерно за 15 минут. Если вы настроите несколько уровней CNAME, CSS не сможет отследить результат разрешения. Если ваш домен доступен, то конфигурация CNAME выполнена успешно.
- Если конфигурация CNAME не вступает в силу в течение длительного времени, см. раздел [Domain Configuration](https://intl.cloud.tencent.com/document/product/267/32478) для устранения проблемы.

## Предварительные условия

- Вы зарегистрировали домен.
- Вы проверили домен и [добавили его](https://intl.cloud.tencent.com/document/product/267/35970) в раздел [Domain Management](https://console.tencentcloud.com/live/domainmanage) консоли CSS. Вы не добавили запись CNAME для домена (значок в столбце **CNAME** — ![](https://staticintl.cloudcachetci.com/cms/backend-cms/7c7c953d530411ee94c3525400d793d0.png)).

## Инструкции

В этом документе описано, как установить CNAME DNS с поставщиками услуг, отличными от Tencent Cloud. Этот метод приведен только в справочных целях; если он не соответствует фактической конфигурации, следуйте информации от вашего поставщика услуг DNS. После настройки CNAME для вашего доменного имени вы можете проверить, успешно ли конфигурирован CNAME для вашего доменного имени, используя метод, описанный в разделе [Проверка эффективности CNAME](#check).

### Метод конфигурации без Tencent Cloud

**Общие этапы настройки CNAME DNS с поставщиками услуг, отличными от Tencent Cloud, следующие:**

1. Войдите в консоль управления вашего поставщика услуг DNS.
2. Найдите функцию управления доменным именем или управления DNS и перейдите на страницу параметров DNS для доменного имени, где необходимо настроить CNAME.
3. Добавьте новую запись DNS. Выберите тип записи `CNAME`.
4. Установите запись хоста, маршрут DNS, значение записи и другие соответствующие параметры.
5. Сохраните запись DNS.

**При настройке записи CNAME обычно необходимо учитывать следующие параметры:**

  - Тип записи: выберите `CNAME`.
  - Запись хоста: введите имя поддомена или префикс доменного имени. Если доменное имя playback — `play.myqcloud.com`, то добавьте `play`; если вам нужно непосредственно разрешить основное доменное имя `myqloud.com`, то введите `@`; если вам нужно разрешить подстановочный домен, то введите `\\*`.
  - Маршрут DNS: рекомендуется выбрать тип "`Default`" для адаптации к различным сетевым окружениям.
  - Значение записи: (введите целевое доменное имя, на которое вы хотите указать поддомен) значение CNAME соответствующего доменного имени на странице управления доменным именем консоли CSS, форматированное как `domain.txlivecdn.com`.
  - TTL: установите время кеша DNS. Чем меньше значение, тем быстрее запись вступает в силу глобально. Как правило, вы можете использовать значение по умолчанию или установить короткое время, например 600 (10 минут).

> **Примечание:** Указанный выше метод приведен только в справочных целях для поставщиков услуг, отличных от Tencent Cloud. Для фактической конфигурации см. документацию и рекомендации, предоставленные вашим поставщиком услуг DNS.

## Проверка записей CNAME

- **Способ 1:** перейдите в раздел [Domain Management](https://console.tencentcloud.com/live/domainmanage) консоли CSS. Если значок в столбце **CNAME** — ![](https://staticintl.cloudcachetci.com/cms/backend-cms/7c7bdede530411ee974d5254005f490f.png), то конфигурация CNAME успешна.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7c7e0619530411ee974d5254005f490f.png)

- **Способ 2:** когда вы [добавляете ваш домен](https://console.tencentcloud.com/live/domainmanage) в консоль CSS, после завершения основных параметров на этапе **CNAME configuration** вы можете просмотреть статус CNAME доменного имени.
- **Способ 3:** в Linux/macOS выполните команду `dig` (`dig your domain`). Если в первой строке отображается целевой домен, предоставленный CSS, конфигурация CNAME успешна.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b5ff083282c511ee88f85254009a298a.png)

- **Способ 4**: в Windows откройте окно командной строки и введите `nslookup your domain`. Если отображается целевое доменное имя, предоставленное CSS, конфигурация CNAME успешна.
![](https://staticintl.cloudcachetci.com/cms/backend-cms/7c7bb189530411ee84f2525400494e51.png)

> **Примечание:** если конфигурация CNAME не вступает в силу в течение длительного времени, см. раздел [Domain Configuration](https://intl.cloud.tencent.com/document/product/267/32478) для устранения проблемы.

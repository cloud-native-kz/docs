# Управление сертификатами

Обычно доменные имена потокового вещания используют протокол Hypertext Transfer Protocol (HTTP). HTTP можно преобразовать в Hyper Text Transfer Protocol over Secure Socket Layer (HTTPS) с помощью протокола SSL/TLS для зашифрованной передачи данных. Вы можете перейти в **Управление сертификатами**, чтобы запросить и настроить SSL-сертификаты для доменных имен.

## Как настроить

Целью настройки SSL-сертификата для доменного имени является шифрование ключевых данных пользователя для безопасной передачи. Сертификат Secure Sockets Layer (SSL) позволяет сайту переключиться с HTTP (HyperText Transfer Protocol) на его зашифрованную версию на основе SSL — HTTPS (HyperText Transfer Protocol over Secure Socket Layer). В настоящее время только доменное имя воспроизведения поддерживает настройку SSL-сертификатов.

## Настройка сертификата

1. Перейдите на страницу [Управление доменами](https://console.tencentcloud.com/live/domainmanage) в консоли CSS и нажмите **Управление сертификатами**, чтобы перейти на страницу управления сертификатами.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/db44debf3f6311ef91ee525400d5f8ef.png)

2. Нажмите **Настроить сертификат**, чтобы добавить конфигурацию сертификата.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/fd044f8c3f6311efa180525400f69702.png)

3. В окне конфигурации сертификата выберите источник сертификата:
  - **Собственный сертификат**: введите замечания, содержимое и ключ этого сертификата. После сохранения конфигурации информация о сертификате будет синхронизирована с разделом [Управление сертификатами](https://console.tencentcloud.com/ssl) в консоли SSL Certificate Service. Подробнее о том, как установить содержимое сертификата и ключ, см. в разделе [Конфигурация HTTPS](https://intl.cloud.tencent.com/document/product/267/31066).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/453b1da33f6411efa917525400fdb830.png)

  - **Сертификат, размещенный в Tencent Cloud**: выберите сертификат, который вы приобрели в консоли SSL Certificate Service.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/189a76fe3f6511efa917525400fdb830.png)

4. После подтверждения того, что сертификат доступен, нажмите **Далее**, чтобы перейти на страницу конфигурации доменного имени.
5. В разделе **Привязать доменные имена** выберите одно или несколько доменных имен воспроизведения, которые соответствуют сертификату. Если выбранное доменное имя уже привязано к сертификату, применится новый сертификат.
6. В разделе **Выбранные** вы можете просмотреть выбранные доменные имена и включена ли для них конфигурация HTTPS.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9a3d6479520c11ee84f2525400494e51.png)

7. Выберите, следует ли включить конфигурацию HTTPS для выбранных доменных имен:

> **Примечание:** Включение параметра **Включить конфигурацию HTTPS** активирует конфигурацию HTTPS для доменных имен. **Включить конфигурацию HTTPS** включено по умолчанию. Если вы отключите эту кнопку, статус конфигурации HTTPS доменных имен не изменится после привязки, будет обновлен только сертификат.

8. Нажмите **Подтвердить**.

## Просмотр конфигурации сертификата

После [настройки сертификата](https://www.tencentcloud.com/document/product/267/41317#configuring-certificate) вы можете перейти в раздел [Управление сертификатами](https://console.tencentcloud.com/live/domainmanage/certificate), чтобы просмотреть его конфигурацию, включая доменное имя, замечания, источник, конфигурацию HTTPS и время истечения.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9a4626a5520c11ee84f2525400494e51.png)

## Обновление конфигурации сертификата

1. Перейдите в раздел [Управление сертификатами](https://console.tencentcloud.com/live/domainmanage/certificate), найдите целевую конфигурацию сертификата в списке и нажмите **Обновить** справа от нее.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9a486d8e520c11eeabd75254005810a4.png)

2. На странице конфигурации сертификата снова настройте сертификат.
3. Нажмите **Подтвердить**.

## Удаление конфигурации сертификата

1. Перейдите в раздел [Управление сертификатами](https://console.tencentcloud.com/live/domainmanage/certificate), найдите целевую конфигурацию сертификата в списке и нажмите **Удалить** справа от нее.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9a45a918520c11ee94c3525400d793d0.png)

2. В окне подтверждения нажмите **Подтвердить**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9a3b6839520c11ee974d5254005f490f.png)

> **Примечание:** После отмены привязки сертификата доменные имена не смогут использовать конфигурацию HTTPS.

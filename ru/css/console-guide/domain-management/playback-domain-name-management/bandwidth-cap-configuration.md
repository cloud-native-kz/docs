# Конфигурация ограничения полосы пропускания

CSS позволяет установить ограничение полосы пропускания для вашего домена воспроизведения. В регионе ускорения вашего домена, если пиковая исходящая полоса пропускания в опорном периоде достигнет установленного вами ограничения, запросам на воспроизведение будет возвращена ошибка 403. Эта функция отключена по умолчанию.

> **Примечание:** Если конфигурация ограничения полосы пропускания включена, грануляция сканирования составляет 5 минут. Если в течение короткого периода времени произойдет внезапный скачок использования, предыдущее сканирование может не вызвать порог, а следующее сканирование может напрямую превысить порог. В этом сценарии будет определенная задержка (примерно 5 минут) в операции перехвата доступа, а потребление в течение этого периода будет выставлено в обычном режиме.

## Предварительные требования

- Вы вошли в [консоль CSS](https://console.tencentcloud.com/live).
- Вы добавили [доменное имя воспроизведения](https://intl.cloud.tencent.com/document/product/267/35970).

## Ограничения

| Регион ускорения | Регион ограничения полосы пропускания по умолчанию | Примечания |
| --- | --- | --- |
| Материковая часть Китая | Материковая часть Китая | Вы можете установить ограничение только для материковой части Китая. |
| За пределами материковой части Китая | За пределами материковой части Китая | Вы можете установить ограничение только для областей за пределами материковой части Китая. |
| Глобальное ускорение | Глобальное ускорение | Вы можете установить различные ограничения для материковой части Китая и областей за её пределами. Вы также можете установить глобальное ограничение. |

## Конфигурация ограничения полосы пропускания

1. Перейдите в [Domain Management](https://console.tencentcloud.com/live/domainmanage). Нажмите на имя вашего домена воспроизведения или **Manage** справа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/2c38eea611d711ef9af4525400720cb5.png)

2. Выберите **Advanced Configuration** > **Bandwidth cap configuration** для просмотра вкладки **Bandwidth cap configuration**.
3. Нажмите **Edit** в верхнем правом углу вкладки для входа на страницу конфигурации ограничения полосы пропускания.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/960b7da7cc0611f091ab5254007c27c5.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a5f17474cc0611f084a45254005ef0f7.png)

4. Нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/98f2a38011d711ef89cc5254002fd0a8.png) для включения **Bandwidth Cap**.

> **Примечание:** После конфигурации порога полосы пропускания для глобального ускорения он будет применяться приоритетно.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d15f742acc0611f08e74525400bf7822.png)

  4.1. После включения ограничения полосы пропускания выполните следующую конфигурацию:
  - **Bandwidth Threshold**
    - **Valid Region** определяется на основе типа региона ускорения доменного имени воспроизведения. Для связанных правил конфигурации см. [Ограничения](https://www.tencentcloud.com/document/product/267/36543#limits).
    - Заполните порог полосы пропускания в соответствии с вашими реальными потребностями бизнеса. Выберите единицу измерения порога: Mbps, Gbps или Tbps.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ecfb4158cc0611f096d1525400e889b2.png)

5. Нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/cbc2785811d711efa2935254005ac0ca.png) для включения **Alert threshold**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/063daa17cc0711f0afdc52540044a08e.png)

  5.1. После включения порога оповещения установите **alert threshold percentage** в соответствии с вашими реальными потребностями бизнеса. Когда доступная полоса пропускания/ограничение полосы пропускания достигает порога оповещения, система отправит вам оповещение через Центр сообщений и другие способы.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/cbbb26fb11d711efbf645254007bbd8c.png)

6. Нажмите **Save**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3307a562cc0711f091ab5254007c27c5.png)

> **Примечание:** Коэффициент преобразования между единицами полосы пропускания составляет 1000: 1 Tbps = 1000 Gbps; 1 Gbps = 1000 Mbps. Если вы измените регион ускорения для вашего домена, вам потребуется переконфигурировать ограничение полосы пропускания. Порог оповещения по умолчанию составляет 80% ограничения полосы пропускания. Диапазон значений: 0-100.

## Отключение ограничения полосы пропускания

1. Перейдите в [Domain Management](https://console.tencentcloud.com/live/domainmanage). Нажмите на имя вашего домена воспроизведения или **Manage** справа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3fcc91c411d811efaa1c525400f65c2a.png)

2. Выберите вкладку **Advanced Configuration** для просмотра вкладки **Bandwidth cap configuration**.
3. Нажмите **Edit** в верхнем правом углу вкладки для входа на страницу конфигурации ограничения полосы пропускания.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/837de35dcc0711f093295254001c06ec.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/96953994cc0711f0ae4f52540099c741.png)

4. Нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/a98a83b011d811ef9fa952540019e87e.png) для отключения **Bandwidth Cap**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/aed98a8ccc0711f091ab5254007c27c5.png)

5. Нажмите **Save**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c42fcd62cc0711f084a45254005ef0f7.png)

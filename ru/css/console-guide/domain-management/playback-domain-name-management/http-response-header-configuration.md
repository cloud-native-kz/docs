# Конфигурация HTTP заголовков ответа

Вы можете использовать HTTP заголовки для определения полей, которые содержат информацию о HTTP транзакциях. Поля заголовков работают на уровне домена. Это означает, что настроенные вами поля заголовков будут действовать для всех ответов в вашем домене.

## Предварительные требования

- Вы вошли в [консоль CSS](https://console.tencentcloud.com/live).
- Вы добавили [доменное имя воспроизведения](https://intl.cloud.tencent.com/document/product/267/35970).

## Ограничения использования

- Вы не можете добавить два поля с одинаковым именем. Чтобы указать несколько значений для поля, используйте следующий формат: `значение 1, значение 2, значение 3`.
- Если настраиваемое вами поле совпадает с полем, используемым серверной частью CSS, вам будет предложено его изменить.
- Пользовательское поле может быть длиной от 1 до 100 символов и может содержать буквы, цифры и дефисы (-).
- Значение поля не может быть пустым. Оно может быть длиной от 1 до 1000 символов и не может содержать китайские символы.

## Конфигурирование HTTP заголовков ответа

1. Перейдите на страницу [Управление доменами](https://console.tencentcloud.com/live/domainmanage). Нажмите на имя вашего **доменного имени воспроизведения** или нажмите **Управление** справа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/599633fb142811ef83585254007bbd8c.png)

2. Выберите вкладку **Расширенная конфигурация**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/15ccff3bcc0d11f08e74525400bf7822.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/25569110cc0d11f084a45254005ef0f7.png)

3. В области **HTTP заголовки ответа** нажмите **Редактировать** для добавления новых полей заголовков или изменения/удаления существующих полей заголовков.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ba2c49f1cc0d11f093295254001c06ec.png)

Чтобы добавить поле заголовка, нажмите **Новое**:

  - Выберите **Предустановленное** для добавления предустановленного поля: `Access-Control-Allow-Methods`, `Access-Control-Max-Age` или `Access-Control-Expose-Headers`.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e3205a7ccc0d11f08e74525400bf7822.png)

| Поле | Описание |
| --- | --- |
| Access-Control-Allow-Methods | Указывает, какие HTTP методы разрешены для запросов между доменами. Вы можете указать несколько методов одновременно: Access-Control-Allow-Methods: POST, GET, OPTIONS. |
| Access-Control-Max-Age | Указывает, как долго (в секундах) результаты предварительного запроса могут быть кэшированы |
| Access-Control-Expose-Headers | Указывает, какие заголовки могут быть открыты для клиентов как часть ответа |

  - Выберите **Пользовательское** для добавления пользовательского поля.

> **Примечание:** Имя пользовательского поля может быть длиной от 1 до 100 символов и может содержать буквы, цифры и дефисы. Значение пользовательского поля может быть длиной от 1 до 1000 символов и не может содержать китайские символы. Система имеет встроенную поддержку параметра заголовка Access-Control-Allow-Origin, который используется для включения запросов между доменами без необходимости настройки. Существует два конкретных сценария: Когда заголовок запроса не включает Origin, возвращаемый заголовок будет 'Access-Control-Allow-Origin: *'. Когда заголовок запроса включает 'Origin: ${Origin}', возвращаемый заголовок будет 'Access-Control-Allow-Origin: ${Origin}'. Например, когда заголовок запроса имеет Origin: https://cloud.tencent.com, возвращаемый заголовок будет Access-Control-Allow-Origin: https://cloud.tencent.com.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/84a75d96cc0e11f0ae4f52540099c741.png)

4. После завершения нажмите **OK**. Ваша конфигурация может иметь один из трех статусов: ещё не вступила в силу, ошибка или активна.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ca4f8494cc0e11f096d1525400e889b2.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e463ac72cc0e11f0afdc52540044a08e.png)

# Конфигурация HTTPS

## Обзор

Протокол HTTPS — это сетевой протокол, основанный на протоколах SSL и HTTP, предназначенный для зашифрованной передачи данных и аутентификации, что делает его более безопасным, чем протокол HTTP. Если вы хотите включить ускорение HTTPS, вы можете сделать это, включив функцию HTTPS для доменного имени воспроизведения и настроив правильный и действительный сертификат. Вы можете приобрести сертификат в Tencent Cloud [SSL Certificate Service](https://intl.cloud.tencent.com/product/ssl). Если у вас уже есть сертификат, вы можете загрузить его в консоль CSS для настройки. В настоящее время CSS поддерживает только формат PEM. Если ваш сертификат имеет другой формат, сначала необходимо преобразовать его в формат PEM. Требования к формату и метод настройки сертификата приведены ниже:

## Предварительные условия

- Вы вошли в [консоль CSS](https://console.tencentcloud.com/live).
- Вы [добавили доменное имя воспроизведения](https://intl.cloud.tencent.com/document/product/267/35970).

## Инструкции

### Шаг 1. Редактирование конфигурации HTTPS

1. Откройте [**Domain Management**](https://console.tencentcloud.com/live/domainmanage) и нажмите на **доменное имя воспроизведения**, который необходимо настроить, или нажмите **Manage** справа, чтобы открыть страницу сведений о доменном имени.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b8a43a9d2a2b11efa01d5254005235d8.png)

2. Выберите **Advanced Configuration** > **HTTPS Configuration**, затем нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/c2b8a2e82a2b11ef97da5254007d9c55.png), чтобы включить сервис HTTPS.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9d1ca38fcb9511f084a45254005ef0f7.png)

3. После включения сервиса HTTPS откроется страница конфигурации HTTPS.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/aec80bc4cb9511f0ae4f52540099c741.png)

4. Выберите источник сертификата, который необходимо настроить, введите соответствующую информацию и нажмите **Save**.

| Источник сертификата | Требуемые элементы конфигурации |
| --- | --- |
| Собственный сертификат | Имя сертификата: введите пользовательское имя для идентификации сертификата.Содержимое сертификата: введите содержимое файла .crt для Nginx. Для получения дополнительной информации см. [Содержимое сертификата](#content).Содержимое приватного ключа: введите содержимое файла .key для Nginx. Для получения дополнительной информации см. [Ключ сертификата](#private_key). |
| Сертификат, размещенный в Tencent Cloud | Список сертификатов: выберите загруженный сертификат в [SSL Certificate Service](https://console.tencentcloud.com/ssl). |

> **Примечание:** Функция HTTPS вступит в силу приблизительно через 2 часа после завершения конфигурации. Пожалуйста, будьте терпеливы.

#### Описание сертификата

Сертификат, предоставленный [CA](https://intl.cloud.tencent.com/document/product/1007/30192#354), включает файлы для Apache, IIS, Nginx и Tomcat. **Служба шифрования CSS использует Nginx, поэтому вы должны выбрать содержимое файлов Nginx для конфигурации.**

Перейдите в **консоль SSL Certificate Service** > [**Certificate Management**](https://console.tencentcloud.com/ssl), выберите целевой сертификат, нажмите **Download** в столбце "Operation" и распакуйте загруженный пакет, чтобы получить следующие файлы:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/55402e92df7911eea462525400bb593a.png)

##### **Содержимое****сертификата**:

  - Выберите файл `.crt` в Nginx и заполните поле ввода всем содержимым, включая `-----BEGIN CERTIFICATE-----` и `-----END CERTIFICATE-----`.
  - **Пример содержимого:**

****

![](https://staticintl.cloudcachetci.com/cms/backend-cms/5540a5c1df7911ee9f745254008eb8a8.png)

> **Примечание:** Если ваш сертификат выдан промежуточным центром сертификации и содержит несколько сертификатов, содержимое сертификата должно быть объединено следующим образом:
> -----BEGIN CERTIFICATE-----
> -----END CERTIFICATE-----
> -----BEGIN CERTIFICATE-----
> -----END CERTIFICATE-----

  - Введите все содержимое между `-----BEGIN RSA PRIVATE KEY-----` и `-----END RSA PRIVATE KEY-----` в файле `.key` для Nginx.

### Шаг 2. Проверка конфигурации

Конфигурация HTTPS вступит в силу примерно через 2 часа. Пожалуйста, посетите доменное имя примерно через 2 часа после отправки сертификата. Если в адресной строке браузера отображается HTTPS, конфигурация выполнена успешно.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/55403e9ddf7911ee9f745254008eb8a8.png)

### Шаг 3. Изменение конфигурации

Конфигурацию можно изменить, если функция HTTPS включена. Если эта функция отключена, редактирование будет невозможно. После отключения CSS больше не будет предоставлять услугу HTTPS для доменного имени. Если сертификат истек, его следует заменить на новый действительный сертификат.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/dffc8ce1cb9511f0a5fa525400454e06.png)

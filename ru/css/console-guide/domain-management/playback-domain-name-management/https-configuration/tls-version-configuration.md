# Конфигурация версии TLS

## Справочная информация

Протокол Transport Layer Security (TLS) предназначен для обеспечения безопасности и конфиденциальности данных, обмениваемых между двумя приложениями. В настоящее время доступны четыре версии протокола TLS, включая TLS 1.0, TLS 1.1, TLS 1.2 и TLS 1.3. Более ранние версии имеют лучшую совместимость, но меньшую безопасность, а более поздние версии имеют повышенную безопасность, но меньшую совместимость.

#### Использование более безопасной функции шифрования TLS обновленной версии для шифрования сетевых соединений на уровне транспорта

| Версия | Описание |
| --- | --- |
| TLS 1.3 (Рекомендуется) | RFC 8446, опубликована в 2018 году. TLS 1.3 быстрее и безопаснее, чем TLS 1.2. |
| TLS 1.2 (Рекомендуется) | RFC 5246, опубликована в 2008 году. Использует надежную технологию шифрования для обеспечения более высокой защиты безопасности. |
| TLS 1.1 | RFC 4346, опубликована в 2006 году. Устранила несколько уязвимостей в TLS 1.0. |
| TLS 1.0 | RFC 2246, опубликована в 1999 году на основе SSL v3.0. Эта версия подвержена различным атакам, таким как BEAST и POODLE. |

## Обзор

Когда вы включили конфигурацию HTTPS, Cloud Streaming Services (CSS) по умолчанию поддерживает несколько версий TLS для удовлетворения потребностей доступа различных пользовательских устройств. Обычно нет необходимости изменять эту конфигурацию. Если вы имеете повышенные требования к безопасности вашего сайта и вам необходимо предотвратить доступ пользователей с использованием версий TLS с низкой безопасностью, вы можете пользовательски настроить версии SSL/TLS. CSS по умолчанию поддерживает TLS 1.0, TLS 1.1, TLS 1.2 и TLS 1.3. Вы можете отключить/включить конкретные версии TLS в зависимости от ваших потребностей в бизнесе.

## Предварительные условия

- Вы вошли в [консоль CSS](https://console.tencentcloud.com/live).
- Вы [добавили имя домена для воспроизведения](https://www.tencentcloud.com/document/product/267/35970).
- **Для изменения конфигурации версии TLS необходимо сначала убедиться, что HTTPS правильно настроен и включен, так как конфигурация версии TLS зависит от HTTPS. Перед установкой версии TLS завершите конфигурацию сертификатов SSL и включите HTTPS. Методы операций см. в** [Конфигурация HTTPS](https://www.tencentcloud.com/document/product/267/31066).

## Примечания

- Конфигурация версии TLS вступит в силу примерно через 15 минут после завершения.
- Понижение версии TLS (например, с TLS 1.2 на TLS 1.1 или TLS 1.0) или отключение конфигурации версии TLS может привести к проблемам безопасности и соответствия. Действуйте с осторожностью.
- TLS 1.3 включена по умолчанию и не может быть отключена.
- После отключения конфигурации HTTPS консоль скроет конфигурацию версии TLS.
- Отключение конфигурации HTTPS приведет к сбою конфигурации версии TLS.
- Если вы измените версию TLS после включения конфигурации HTTPS, а затем отключите конфигурацию HTTPS, версия TLS останется прежней при повторном включении HTTPS в следующий раз.

## Руководство по конфигурации

### Просмотр конфигурации версии TLS

1. Перейдите в [Управление доменами](https://console.tencentcloud.com/live/domainmanage) и нажмите на **домен воспроизведения**, который вы хотите настроить, или **Управление** справа, чтобы войти на страницу деталей домена.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d9186135ae1e11efbfb3525400bdab9d.png)

2. Перейдите на вкладку **Расширенная конфигурация** и найдите **Конфигурация версии TLS**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8bb4dbb0cb9911f0afdc52540044a08e.png)

### Изменение конфигурации версии TLS

1. Перейдите в [Управление доменами](https://console.tencentcloud.com/live/domainmanage) и нажмите на **домен воспроизведения**, который вы хотите настроить, или **Управление** справа, чтобы войти на страницу деталей домена.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/dc9d6a6aae1e11ef8c01525400fdb830.png)

2. Выберите **Расширенная конфигурация** > **Конфигурация версии TLS** и нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/5cfa1514b21711ef970f525400d5f8ef.png), чтобы войти на страницу изменения конфигурации версии TLS.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/05ee231acb9a11f093295254001c06ec.png)

3. В области управления версией TLS вы можете включить или отключить соответствующую версию TLS в зависимости от ваших потребностей в бизнесе.

> **Примечание:** Вы можете включить одну версию или несколько последовательных версий. Например, вы можете одновременно включить версии 1.0, 1.1 и 1.2, но не версии 1.0 и 1.2. Вы не можете отключить все версии.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ae80cceeae1f11ef96e55254002693fd.png)

4. Нажмите **Подтвердить**, чтобы сохранить конфигурацию. Отредактированная конфигурация версии TLS вступит в силу примерно через 15 минут. Пожалуйста, подождите.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6904e62dcb9a11f093295254001c06ec.png)

# Конфигурация пакета Live Stream

CSS предоставляет услугу пакета Live Stream, поддерживающую протоколы упаковки, такие как HLS - CMAF, DASH - ISO, DASH - CMAF, LL-HLS - TS и LL-HLS - CMAF.

| **Протокол пакета** | **Формат срезов** | **Технические принципы** | **Поддерживаемые форматы кодирования** |
| --- | --- | --- | --- |
| **HLS - CMAF** | CMAF | Объединение протокола HLS с контейнером CMAF, использование формата упаковки fMP4 в качестве замены традиционному MPEG-TS. | Форматы видеокодирования: H.264, H.265, AV1 |
|  |  |  | Форматы аудиокодирования: AAC, MP3, OPUS |
| **DASH - ISO** | M4S (ISO BMFF) | На основе стандарта MPEG-DASH видеопоток разделяется на несколько коротких фрагментов, каждый из которых независимо инкапсулируется в формате ISO BMFF. | Форматы видеокодирования: H.264, H.265, AV1 |
|  |  |  | Форматы аудиокодирования: AAC, MP3, OPUS |
| **DASH - CMAF** | CMAF | Протокол DASH беспрепятственно интегрируется с контейнером CMAF, использующим механизм фрагментированной передачи CMAF для достижения низкой латентности. | Форматы видеокодирования: H.264, H.265, AV1 |
|  |  |  | Форматы аудиокодирования: AAC, MP3, OPUS |
| **LL-HLS - TS** | TS | Решение низколатентного HLS продолжает использовать формат инкапсуляции MPEG-TS, эффективно снижая латентность за счет сокращения длительности сегмента и предварительной загрузки сегментов. | Форматы видеокодирования: H.264, H.265, AV1, H.266 |
|  |  |  | Форматы аудиокодирования: AAC, MP3, OPUS |
| **LL-HLS - CMAF** | CMAF | Низколатентный HLS в сочетании с CMAF достигает сверхнизкой латентности посредством сегментированного кодирования. | Форматы видеокодирования: H.264, H.265, AV1 |
|  |  |  | Форматы аудиокодирования: AAC, MP3, OPUS |

## Предварительные требования

- Вы активировали CSS и вошли в [консоль CSS](https://console.tencentcloud.com/live/livestat).
- Вы добавили **домен воспроизведения**. Для получения дополнительной информации обратитесь к разделу [Добавление собственного домена](https://www.tencentcloud.com/document/product/267/35970?lang=en&pg=).

## Важно знать

- Первая конфигурация пакета должна вступить в силу в течение 15 минут. Использование функции пакета live stream будет сопровождаться начислением сборов за пакет live stream.
- Из-за ограничений браузера Safari для воспроизведения низколатентного HLS в окружении Safari и iOS необходимо включить HTTPS 2.0. В противном случае поток не может быть воспроизведен.
- Рекомендуется поддерживать стабильное значение GOP для прямого потока во время трансляции. Рекомендуется устанавливать значение GOP на стороне отправки или стороне перекодирования на 1 или 2 секунды.
- Использование параметра txPackageType при извлечении может обеспечить воспроизведение прямых потоков в соответствующем формате контейнера. Параметр указывает формат контейнера. Допустимые значения: hls-cmaf, dash-iso, dash-cmaf, ll-hls-ts и ll-hls-cmaf.

> **Примечание:** **Пример конструкции URL-адреса воспроизведения:**Пример URL-адреса воспроизведения HLS - CMAF`http(s)://${your_domain_name}/${path}/${streamname}.m3u8?txPackageType=hls-cmaf`Пример URL-адреса воспроизведения DASH - ISO`http(s)://${your_domain_name}/${path}/${streamname}.mpd?txPackageType=dash-iso`Пример URL-адреса воспроизведения DASH - CMAF`http(s)://${your_domain_name}/${path}/${streamname}.mpd?txPackageType=dash-cmaf`Пример URL-адреса воспроизведения LL-HLS - TS`http(s)://${your_domain_name}/${path}/${streamname}.m3u8?txPackageType=ll-hls-ts`Пример URL-адреса воспроизведения LL-HLS - CMAF`http(s)://${your_domain_name}/${path}/${streamname}.m3u8?txPackageType=ll-hls-cmaf`

## Создание конфигурации пакета Live Stream

1. Войдите в консоль CSS, выберите [Управление доменами](https://console.tencentcloud.com/live/domainmanage) и нажмите на **домен воспроизведения**, для которого вы хотите настроить пакет Live Stream, или нажмите кнопку **Управление** справа для доступа к странице управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/259f9b5c769311f09e39525400454e06.png)

2. Выберите **Пакет Live Stream**. Если вам нужно добавить новый пакет, вы можете нажать кнопку **Добавить** или **Новый** справа для настройки **нового пакета Live Stream**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/dba267bfcc3511f0afdc52540044a08e.png)

3. В открывшемся окне вы можете настроить параметры в соответствии с вашими конкретными бизнес-требованиями.

Вариант 1

Вариант 2

| Элемент конфигурации | Обязательный элемент | Описание |
| --- | --- | --- |
| AppName | Да | Поддерживает только буквы, цифры и символы. * служит подстановочным символом. |
| StreamName | Да | Поддерживает только буквы, цифры и символы. * служит подстановочным символом. |
| Протокол пакета | Да | Поддерживает множественный выбор; вы можете выбрать из следующих параметров:HLS-CMAFDASH-ISODASH-CMAFLL-HLS-TSLL-HLS-CMAF |
| Количество сегментов | Да | Значение по умолчанию — 3, допустимый диапазон целых чисел от 3 до 5. |
| Длительность сегмента | Да | Когда **протокол пакета** установлен на HLS - CMAF, DASH - ISO или DASH - CMAF, значение по умолчанию для длительности сегмента — 5, допустимый диапазон целых чисел от 1 до 10. Рекомендуется настроить этот параметр как целое кратное значению GOP. |
|  |  | Когда **протокол пакета** установлен на LL-HLS - TS или LL-HLS - CMAF, значение по умолчанию — 1, допустимый диапазон целых чисел от 1 до 2. Рекомендуется настроить этот параметр как целое кратное значению GOP. |
| Длительность сегмента части | Да | Длительность сегмента части должна быть настроена только при выборе **протокола пакета** LL-HLS - TS или LL-HLS - CMAF. Значение по умолчанию — 350, допустимый диапазон целых чисел между 200 и 1000, измеряется в миллисекундах (мс). Рекомендуется установить это значение немного больше одной трети длительности сегмента. |

4. Нажмите кнопку **Подтвердить**, чтобы сохранить конфигурацию.
5. На основе ваших конкретных бизнес-требований нажмите кнопку **Новый** справа для добавления конфигурации пакета Live Stream.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/065dc849769611f096685254001c06ec.png)

## **Просмотр** конфигурации пакета Live Stream

1. Войдите в консоль CSS, выберите [Управление доменами](https://console.tencentcloud.com/live/domainmanage) и нажмите на **домен воспроизведения** для пакета Live Stream, который вы хотите просмотреть, или нажмите кнопку **Управление** справа для доступа к странице управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/2eb62c9f769311f096685254001c06ec.png)

2. Выберите **Пакет Live Stream** и нажмите на треугольный значок слева, чтобы раскрыть параметры расширенного протокола пакета.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b72c411dcc3611f091ab5254007c27c5.png)

## Изменение конфигурации пакета Live Stream

1. Войдите в консоль CSS, выберите [Управление доменами](https://console.tencentcloud.com/live/domainmanage) и нажмите на **домен воспроизведения** пакета Live Stream, который вы хотите изменить, или нажмите кнопку **Управление** справа для доступа к странице управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/36f8c5d6769311f081ce52540044a08e.png)

2. Выберите **Пакет Live Stream** и нажмите кнопку **Изменить**, чтобы перейти на страницу конфигурации упаковки.
3. Измените параметры конфигурации в соответствии с вашими требованиями, затем нажмите кнопку **Подтвердить**, чтобы завершить изменения.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/df5ef7fecc3611f084a45254005ef0f7.png)

## **Удаление** конфигурации пакета Live Stream

1. Войдите в консоль CSS, выберите [Управление доменами](https://console.tencentcloud.com/live/domainmanage) и нажмите на **домен воспроизведения** пакета Live Stream, который вы хотите удалить, или нажмите кнопку **Управление** справа для доступа к странице управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3e2af387769311f087e15254005ef0f7.png)

2. Выберите **Пакет Live Stream**, выберите успешно настроенный пакет Live Stream и нажмите кнопку **Удалить** в верхнем правом углу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/fa27aebfcc3611f08658525400454e06.png)

3. Подтвердите, хотите ли вы удалить текущую конфигурацию упаковки. Нажмите кнопку **ОК**, чтобы успешно выполнить удаление.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/cec4dcc3769711f09f3d52540099c741.png)

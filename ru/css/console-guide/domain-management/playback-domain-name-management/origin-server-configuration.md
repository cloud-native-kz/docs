# Конфигурация сервера-источника

Если у вас есть самостоятельно созданный сервер-источник и источник потокового вещания, CSS может получать потоки с вашего сервера-источника и распространять контент. Этот документ описывает, как настроить информацию сервера-источника для домена воспроизведения в консоли CSS.

## Ограничения

- Конфигурация сервера-источника вступает в силу примерно через один час после завершения конфигурации.
- После настройки сервера-источника для домена воспроизведения вы больше не сможете привязать домен отправки к домену воспроизведения, указав `StreamName`. Кроме того, вы не сможете настроить задачи водяного знака, транскодирования, записи, снимка экрана или обнаружения порнографии для домена воспроизведения.
- После завершения конфигурации origin-pull в консоли, если вам требуется дополнительно установить список разрешённых IP-адресов служб во время процесса origin-pull Tencent Cloud Live, пожалуйста, [отправьте тикет](https://console.tencentcloud.com/workorder/category), чтобы получить список диапазонов IP-адресов origin-pull. Кроме того, предоставьте соответствующие доменные имена (предполагаемое использование) для оценки и конфигурации на бэкэнде.

## Предварительные условия

- Вы вошли в [консоль CSS](https://console.tencentcloud.com/live).
- Вы создали сервер-источник потокового вещания.
- Вы добавили **доменное имя воспроизведения**.

## Конфигурация Origin-Pull

Вы можете редактировать информацию сервера-источника домена, включая основную информацию, протокол и хост в консоли.

1. В консоли CSS выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели. Щёлкните имя вашей **Playback Configuration** или щёлкните **Manage** справа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/559d32c8c8ed11ef81865254005ef0f7.png)

2. Рядом с **Domain Management**/**Domain** щёлкните ![](https://staticintl.cloudcachetci.com/cms/backend-cms/c4a96139d21d11eeb0d9525400461a83.png), чтобы включить или отключить **Origin server mode**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/abd73782558011ef8357525400bdab9d.png)

3. Когда режим сервера-источника включен, вы можете выполнить конфигурацию origin-pull на основе требований вашего бизнеса.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/5ecb155ac8ee11efa2ff52540044a08e.png)

### Описание базовой конфигурации

#### Информация сервера-источника

| Информация сервера-источника | Описание |
| --- | --- |
| Тип сервера-источника | Поддерживает два типа: сервер-источник потокового вещания и StreamPackage. |
| Протокол маршрутизации | Поддерживает протоколы RTMP, HTTP-FLV и HLS. |
| HTTPS | Если используется протокол FLV или HLS, вы можете включить HTTPS. Если вы включите HTTPS, будет использован порт 443. Также поддерживается перенаправление после HTTPS. Нет ограничений на порты. |
| Основной сервер-источник | Адрес основного сервера-источника, который может быть IP-адресом или доменом. Вы также можете настроить резервный сервер-источник. Адреса будут опрашиваться. |
| Резервный сервер-источник | Адрес резервного сервера-источника (необязательно). |
| Заголовок хоста сервера-источника | По умолчанию адрес сервера-источника используется как заголовок Host, если он не настроен. |

![](https://staticintl.cloudcachetci.com/cms/backend-cms/bcfaadc6c8ee11ef81865254005ef0f7.png)

##### Заголовок хоста

Если используется протокол FLV или HLS, вы можете настроить HTTP-заголовок хоста, который указывает точный домен, к которому CSS получает доступ при извлечении с сервера-источника. Если не настроено, адрес источника используется как заголовок Host по умолчанию.

##### Примечания

Разница между адресом сервера-источника и заголовком хоста заключается в следующем:

- Адрес сервера-источника — это IP-адрес, на который отправляется запрос origin-pull.
- Заголовок хоста указывает домен адреса сервера-источника, на который отправляется запрос.

##### Пример конфигурации

1. Сервер-источник настроен для домена воспроизведения `xx001.elementtest.org` следующим образом:
![](https://staticintl.cloudcachetci.com/cms/backend-cms/f8748b43558011efb927525400fdb830.png)
2. Процесс получения с сервера-источника будет следующим:
Когда пользователь получает доступ к ресурсу, открывая `http://xx001.elementtest.org/index.m3u8`, потому что ресурс ещё не кэширован в Tencent Cloud, CSS разрешит домен `test001.com` для получения адреса сервера источника. Предположим, это `1.1.1.1`. CSS получит доступ к серверу `1.1.1.1`, найдёт файл `index.m3u8` в веб-сервере `test002.com` и затем вернёт ресурс пользователю.

### Переупаковка

Если используется протокол RTMP или HTTP-FLV, вы можете включить переупаковку HLS. Ниже приведены форматы адресов RTMP, HTTP-FLV и HLS.

- RTMP: `rtmp://Playback domain/AppName/StreamName`
- FLV: `http://Playback domain/AppName/StreamName.flv`
- M3U8: `http://Playback domain/AppName/StreamName.m3u8`

#### Примечания

- Количество HLS-сегментов: три по умолчанию. Диапазон значений: 3-10.
- Размер HLS-сегмента: три секунды по умолчанию. Диапазон значений: 3-10. Фактические генерируемые сегменты не будут меньше размера GOP.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/04bb06e2c8ef11efb1a552540099c741.png)

### Конфигурация HTTP

Когда протокол origin-pull — это HLS, вы можете настроить

**HTTP settings**

. На основе требований вашего бизнеса щёлкните

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3fec55a2558111ef8f105254002693fd.png)

, чтобы включить соответствующие функции.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7f61d07bc8ef11ef9d3a5254001c06ec.png)

| Элемент | Описание |
| --- | --- |
| Перенаправление | Если вы включите это, Tencent Cloud не будет кэшировать код состояния 301 или 302. Когда сервер-источник возвращает 301 или 302, Tencent Cloud автоматически перенаправляет, пока не получит запрашиваемый ресурс (максимум 10 перенаправлений) и вернёт ресурс пользователю. Перенаправления на конце пользователя не требуются. Если вы отключите перенаправление, Tencent Cloud вернёт код состояния 301 или 302 конечному пользователю, который перенаправит для получения ресурса. |
| Пропуск параметров URL сервера-источника | По умолчанию параметры URL не пропускаются. Если вы включите это, параметры могут быть добавлены в URL без выполнения кодирования или декодирования URL. |
| Пропуск HTTP-заголовков запроса | По умолчанию HTTP-заголовки запроса не пропускаются. Вы можете включить это для пропуска заголовков. Дублирующиеся заголовки (без учёта регистра) в настоящее время не поддерживаются. |
| Пропуск HTTP-заголовков ответа | По умолчанию HTTP-заголовки ответа не пропускаются. Вы можете включить это для пропуска заголовков. Дублирующиеся заголовки (с учётом регистра) поддерживаются в настоящее время. |
| Запрос OPTIONS | По умолчанию поддерживается запрос GET, и запрос Option поддерживается после включения. |
| [Пользовательский заголовок запроса Origin-pull](https://www.tencentcloud.com/document/product/267/35070#bf7c2c67-4483-4b83-b9c8-298aa78760e2) | При выполнении запроса origin-pull добавьте необходимые заголовки для передачи IP-адреса клиента, порта, метки и т. д. По умолчанию для элемента конфигурации выбран заголовок запроса индекса с поддержкой переключения на заголовок запроса среза. Параметр заголовка: состоит из прописных и строчных букв, цифр и дефисов (-), с поддерживаемой длиной 1-100 символов и без пробелов. Значение заголовка: китайские символы не поддерживаются, и оно не может начинаться с $, с поддерживаемой длиной 1-100 символов и без пробелов. По умолчанию система включает опции синхронизации и автоматически синхронизирует конфигурации заголовков запросов индекса и среза при одном добавлении. Поддерживается добавление нескольких записей, максимум 10 пользовательских заголовков запроса origin-pull для индекса и среза соответственно. |

#### Конфигурация пользовательского заголовка запроса Origin-pull

1. **Добавление пользовательского заголовка запроса Origin-pull**
  1.1. Щёлкните ![](https://staticintl.cloudcachetci.com/cms/backend-cms/fb26d140558111efb927525400fdb830.png), чтобы включить переключатель пользовательского заголовка запроса origin-pull и добавить конфигурацию пользовательского заголовка запроса origin-pull.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/fb27030a558111efb66652540055f650.png)

  1.2. После завершения конфигурации щёлкните  **Confirm**  , чтобы завершить создание.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/fb28660d558111ef9bf1525400a9236a.png)

  1.3. После создания щёлкните  **Save** . Конфигурация займёт некоторое время, чтобы вступить в силу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/fb1613e2558111ef8f105254002693fd.png)

2. **Изменение пользовательского заголовка запроса Origin-pull**

> **Примечание:** После удаления пользовательского заголовка запроса origin-pull конфигурация больше не будет действовать. Выполняйте с осторожностью.

  2.1. На основе требований вашего бизнеса вы можете щёлкнуть  **Edit** , чтобы изменить, добавить или удалить настроенный пользовательский заголовок запроса origin-pull.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/fb26bcce558111ef8357525400bdab9d.png)

  2.2. Щёлкните  **Add** , чтобы продолжить добавление нескольких записей. Можно настроить максимум 10 пользовательских заголовков запроса origin-pull.
  2.3. Щёлкните  **Modify** , чтобы изменить настроенный пользовательский заголовок запроса origin-pull.
  2.4. Щёлкните  **Delete** , чтобы удалить настроенный пользовательский заголовок запроса origin-pull. После удаления всех конфигураций пользовательского заголовка запроса origin-pull щёлкните  **Save** . Система автоматически отключит функцию конфигурации пользовательского заголовка запроса origin-pull.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/fb20ddba558111ef9bf1525400a9236a.png)

### Конфигурация кэша

Если используется протокол HLS, вы можете настроить время кэширования ресурсов. После того как Tencent Cloud успешно получает запрашиваемый ресурс с сервера-источника (код состояния 200), она будет кэшировать файл индекса и сегменты в соответствии с конфигурацией.

| Элемент | Описание |
| --- | --- |
| Время кэширования файла индекса | Время кэширования файла индекса при возврате сервером-источником кода состояния 200. Время кэширования по умолчанию — 1000 мс. Максимальное время, которое можно установить, — 60000 мс. |
| Время кэширования сегмента | Время кэширования TS/M4S/MP4-сегментов при возврате сервером-источником кода состояния 200. Время кэширования по умолчанию — 1000 мс. Максимальное время, которое можно установить, — 60000 мс. |
| Время кэширования по коду состояния | В соответствии с кодом состояния, соответствующим конфигурированному кэшу, если один и тот же запрос получен в течение времени кэширования, нет необходимости посещать сервер-источник, и код состояния может быть возвращён непосредственно. Время кэширования по умолчанию — 1 с. Когда сервер-источник возвращает код состояния отличный от 200, если вы не можете обработать это немедленно и не хотите пропускать все последующие запросы на сервер-источник, вы можете кэшировать код состояния и вернуть его пользователю напрямую. Это может снизить нагрузку на сервер-источник. В настоящее время следующие коды состояния могут быть кэшированы, независимо от типа файла: 4XX: 400, 403, 404, 405 5XX: 500, 503, 504 |
| Конфигурация правила ключа кэша | Для конфигурации правила ключа кэша сохраняйте параметры, которые влияют на содержимое ресурса, как ключ кэша, преобразуйте категорию запросов для одного и того же ресурса в единый ключ кэша и попадайте в один и тот же кэш, чтобы улучшить коэффициент попадания. **Тип файла** Выберите типы файлов. Есть варианты индекса или среза, индекс выбирается по умолчанию. **Сохранить указанные параметры** Можно вводить только английские буквы и цифры. Несколько параметров разделяются ";". Поддерживается до 30 групп параметров. Имя каждого параметра не должно превышать 20 символов. Если параметр указан, даже если параметр не имеет значения, он будет кэширован отдельно от параметров с тем же именем, которые имеют значение. **Примечание:** Вы должны включить пропуск параметров URL origin-pull в [HTTP settings](https://www.tencentcloud.com/document/product/267/35070#http-configuration) перед конфигурацией правил ключа кэша. |

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ecab6c5ad21f11eea2b0525400bb593a.png)

### Переписание URL

Если используется протокол HLS, вы можете настроить переписание URL.

Tencent Cloud позволяет вам переписать фактический URL, который CSS получает, на URL, который лучше соответствует вашему серверу-источнику. В настоящее время вы можете переписать только путь URL.

#### Примечания

- **Исходный URL**: Запросы сопоставляются по префиксу. Например, если вы введёте `/test01`, правило переписывания будет применено ко всем запросам под `/test01`. Регулярные выражения в настоящее время не поддерживаются.
- **Целевой URL**: Запросы сопоставляются по префиксу. Например, если вы введёте `/test01/test02`, все запросы под `/test` будут переписаны на `/test01/test02`. Регулярные выражения в настоящее время не поддерживаются.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/0cfffe2ad22011ee89c5525400170219.png)

#### Ограничения

- Вы можете настроить максимум 10 правил переписания для каждого домена воспроизведения.
- Пробелы и следующие специальные символы не поддерживаются: < > " # { } | \\ ^ ~ [ ] `
- Вы можете переупорядочить правила, чтобы отрегулировать их приоритеты. Правила в верхней части имеют более высокие приоритеты.

### Прочее

| Элемент | Описание |
| --- | --- |
| Время ожидания соединения | Период ожидания установления TCP-соединения. Время по умолчанию — 10000 мс, диапазон значений — 2000-60000 (мс). Установите период ожидания в соответствии с условиями вашего сервера-источника и сетевыми условиями. Если период ожидания слишком короткий, при неудаче запроса на получение из-за проблем с сетью CSS может слишком часто переключаться между серверами-источниками. Если период ожидания слишком долгий, CSS может долго ждать перед попыткой другого сервера-источника, что вызывает ошибку воспроизведения на конце клиента. |
| Максимальное количество повторных попыток | Максимальное количество попыток повторного подключения. Если настроено несколько адресов сервера-источника, при неудаче запроса CSS попытается использовать другой адрес. Диапазон значений: 1-10. |

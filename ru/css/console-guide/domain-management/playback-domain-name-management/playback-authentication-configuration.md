# Конфигурация проверки подлинности при воспроизведении

## Обзор

По умолчанию контент Облачных услуг потокового вещания (CSS) является общедоступным, что позволяет просматривать содержимое потокового вещания сразу после получения адреса воспроизведения. Если вам требуется контроль доступа к содержимому потокового вещания при использовании Трансляции живого видео (LVB) или Трансляции живых событий (LEB), это можно достичь через параметры проверки подлинности для защиты содержимого ресурсов потокового вещания. Проверка подлинности ключа используется для создания поля проверки подлинности txSecret в адресе потокового вещания, что может предотвратить несанкционированное использование потокового вещания из-за утечек доменных имен. Рекомендуется включить функцию проверки подлинности ключа для повышения безопасности содержимого вашего потокового вещания.

## Как настроить

Для включения проверки подлинности URL пользователю CSS необходимо создать зашифрованный URL и предоставить его другим пользователям. Когда пользователь запрашивает содержимое с помощью зашифрованного URL с узла ускорения CSS, узел проверит информацию проверки подлинности запроса, чтобы определить, является ли запрос действительным. Если это так, узел вернет содержимое в обычном режиме; в противном случае узел отклонит запрос, защищая содержимое вашего потокового вещания.

## Предварительные условия

- CSS была активирована, и вы вошли в [консоль CSS](https://console.tencentcloud.com/live).
- Вы добавили [доменное имя воспроизведения](https://www.tencentcloud.com/document/product/267/35970).

## Включение **проверки подлинности ключа**

1. Выберите [**Управление доменными именами**](https://console.tencentcloud.com/live/domainmanage) и нажмите на **доменное имя воспроизведения**, для которого вы хотите включить проверку подлинности, или нажмите **Управление**, чтобы перейти на страницу управления доменным именем.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/98c093f2293211efac39525400560de4.png)

2. В разделе **Контроль доступа** > **Проверка подлинности ключа** нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/d478446e293211ef8b7b52540096e81f.png), чтобы включить проверку подлинности ключа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/040794c7ab2811f096c2525400454e06.png)

3. Завершите следующие настройки на странице конфигурации проверки подлинности:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f91763d9293211efac39525400560de4.png)

| Элемент конфигурации | Описание |
| --- | --- |
| Основной ключ | При конфигурации основного ключа для проверки подлинности, в зависимости от ваших фактических потребностей и политики безопасности, вы можете выбрать основной ключ, **случайно сгенерированный** системой, или ввести **пользовательский** основной ключ, например, test. Поддерживает только прописные буквы, строчные буквы и цифры, максимальная длина 256 символов. |
| Резервный ключ | При конфигурации резервного ключа для проверки подлинности, в зависимости от ваших фактических потребностей и политики безопасности, вы можете выбрать резервный ключ, случайно сгенерированный системой, или ввести пользовательский резервный ключ, например, testing. Поддерживает только прописные буквы, строчные буквы и цифры, максимальная длина 256 символов. |
| Период действия (с) | Поддерживает только целые числа. Введите период действия подписи, например 20. |

> **Примечание:** Проверка подлинности при воспроизведении доменного имени воспроизведения **отключена** по умолчанию. **Ключ проверки подлинности**: включает основной ключ (обязательный) и резервный ключ (дополнительный). Вы можете плавно переключиться на резервный ключ, если ваш основной ключ раскрыт.

4. Нажмите **Сохранить**, чтобы сохранить конфигурацию.

## Изменение **проверки подлинности ключа**

1. Выберите [**Управление доменными именами**](https://console.tencentcloud.com/live/domainmanage) и нажмите на **доменное имя воспроизведения**, которому требуется конфигурация проверки подлинности, или нажмите **Управление**, чтобы перейти на страницу управления доменным именем.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c49bb6f1293311ef9dd5525400441de3.png)

2. В разделе **Контроль доступа** > **Проверка подлинности ключа** нажмите **Изменить**, чтобы перейти на страницу конфигурации проверки подлинности ключа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7f0c0fbbab2811f096c2525400454e06.png)

3. Измените информацию элемента конфигурации в соответствии с вашими фактическими потребностями и нажмите **Сохранить**, чтобы завершить изменение.

## Отключение **проверки подлинности ключа**

> **Примечание:** При выборе отключения проверки подлинности ключа убедитесь, что вы понимаете связанные с этим риски. Отключение проверки подлинности ключа может подвергнуть вашу услугу потокового вещания пиратству, что приведет к дополнительным расходам на услуги. Поэтому рекомендуется сохранить проверку подлинности ключа включенной для защиты содержимого вашего потокового вещания.

После [включения проверки подлинности ключа](https://www.tencentcloud.com/document/product/267/31060#directions), если вам нужно отключить эту функцию, выполните следующие действия:

1. Выберите [**Управление доменными именами**](https://console.tencentcloud.com/live/domainmanage) и нажмите на **доменное имя воспроизведения**, которому требуется конфигурация проверки подлинности, или нажмите **Управление**, чтобы перейти на страницу управления доменным именем.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c4950d97293311efa45a5254008fe934.png)

2. В разделе **Контроль доступа** > **Проверка подлинности ключа** нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/c4a3f10b293311ef8b7b52540096e81f.png), чтобы отключить проверку подлинности ключа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/97c192b9ab2811f096c2525400454e06.png)

3. Перед отключением проверки подлинности ключа еще раз подтвердите, действительно ли вы хотите продолжить. Если вы понимаете и принимаете риски, нажмите **Подтвердить**, чтобы продолжить.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c497b860293311ef9dd5525400441de3.png)

> **Примечание:** После включения проверки подлинности для доменного имени воспроизведения исходный URL воспроизведения станет недоступным и будет возвращена ошибка 403. Перед включением этой функции убедитесь, что ваша платформа потокового вещания совместима со следующим алгоритмом проверки подлинности, чтобы ваши услуги потокового вещания не были затронуты.

## Пример

Исходный URL воспроизведения:

```
http://www.test.com/live/test01.flv
```

Параметры проверки подлинности настроены следующим образом:

```
Основной ключ: ngoeiq03Резервный ключ: -Период действия: 12495 секунд
```

> **Примечание:** Если вы включили проверку подлинности, фактическое время истечения URL будет `txTime` плюс период действия ключа. Для удобства время, которое вы установили в консоли, является фактическим временем истечения. **Если вы включили проверку подлинности, система рассчитает** `txTime` **при создании URL-адресов воспроизведения.** Если вы используете методы FLV или RTMP для начала получения потока до времени истечения, поток будет поддерживаться нормально, пока соединение не прервется или не остановится, даже если время истечения истекло. Если вы используете метод HLS для начала получения потока до времени истечения, поток будет остановлен при наступлении времени истечения.

Расчет временной метки:

```
Время установки: 2018.12.01 08:30:00Десятичная временная метка Unix: 1543624200Шестнадцатеричная временная метка Unix: 5C01D608 (без учета регистра). CSS использует шестнадцатеричные временные метки для проверки подлинности.
```

Расчет подписи проверки подлинности:

```
txSecret = MD5(key+StreamName+txTime) StreamName — это имя потока, которое совпадает с StreamIDtxTime — это временная метка key — это ключ проверки подлинности txSecret = MD5(ngoeiq03+test01+5C01D608) txSecret = MD5(ngoeiq03test015C01D608) txSecret = ce797dc6238156d548ef945e6ad1ea20
```

Новый URL воспроизведения:

```
http://www.test.com/live/test01.flv?txSecret=ce797dc6238156d548ef945e6ad1ea20&txTime=5C01D608
```

Время истечения этого URL — 2018.12.01 08:30:00 + 12495 секунд, т.е. 2018.12.01 11:58:15 по пекинскому времени.
Если проверка подлинности не пройдена или срок действия URL истек, CSS вернет 403.

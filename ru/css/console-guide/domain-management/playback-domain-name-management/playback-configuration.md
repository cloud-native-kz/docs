# Конфигурация воспроизведения

После успешной отправки потока вы можете использовать генератор адресов консоли CSS для создания URL воспроизведения (необходимо ввести `StreamName`, который должен совпадать с ID потока в URL отправки).

## Предварительные условия

- Вы вошли в [консоль CSS](https://console.tencentcloud.com/live).
- Вы добавили **домен воспроизведения**. Инструкции по добавлению домена см. в разделе [Adding Your Own Domain Name](https://intl.cloud.tencent.com/document/product/267/35970).

## Инструкции

1. Выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели и нажмите на имя домена воспроизведения или нажмите **Manage** справа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/29d8288411cb11ef9fa952540019e87e.png)

2. Выберите **Playback Configuration** и в **Playback Address Generator** выполните следующие настройки:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8627bcbeab2611f09710525400e889b2.png)

  2.1. Требуется выбрать **тип шифрования**. Выбирайте на основе ваших требований безопасности и соображений производительности. Доступные типы шифрования включают MD5 и SHA256, MD5 используется по умолчанию.
  2.2. Выберите тип потока для воспроизведения, который может быть **исходным потоком**, **транскодированным потоком** или **потоками с адаптивной скоростью**. Если вы выберите **Transcoded stream**, вам нужно указать шаблон транскодирования. Если вы выберите **Adaptive bitrate streams**, вам нужно указать шаблон адаптивной скорости.
  2.3. Введите **StreamName**.
  - Поддерживает только английские буквы, цифры и символы.
  - например **liveteststream**. Убедитесь, что он совпадает с **StreamName** в вашем URL отправки.
  2.4. После выбора шаблона адаптивной скорости имена потоков в шаблоне будут отображены в убывающем порядке по битрейту.
  2.5. Выберите **URL Expiration Time** (время истечения URL), например `2025-10-18 14:55:40`.

> **Примечание:** Время истечения URL относится к hex(time) в информации аутентификации, и его изменение изменит URL воспроизведения. Фактический период действия — это сумма времени истечения URL и периода действия аутентификации. По истечении этого фактического периода действия новые запросы будут отклонены, что предотвратит дальнейшую передачу потока через этот адрес.

3. Нажмите **Generate Address**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/4792fc5dab2711f09710525400e889b2.png)

4. Если вы не включили аутентификацию для домена воспроизведения, в области **Playback URL** на той же странице вы можете найти URL воспроизведения для RTMP, FLV, HLS и UDP. Замените `StreamName` в URL на ID потока в вашем URL отправки, и вы сможете использовать URL для воспроизведения потока.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6c1ee8f2ab2711f0b5345254005ef0f7.png)

> **Примечание:** Для получения дополнительной информации см. [Live Playback](https://intl.cloud.tencent.com/document/product/267/31559).

## Формат URL воспроизведения

### URL воспроизведения исходного потока

```
RTMP format: `rtmp://domain/AppName/StreamName?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)`FLV format: `http://domain/AppName/StreamName.flv?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)`M3U8 format: `http://domain/AppName/StreamName.m3u8?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)`UDP format: webrtc://domain/AppName/StreamName?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)
```

- `domain`: Ваше имя домена воспроизведения.
- `AppName`: Имя приложения потоковой трансляции, по умолчанию это `live` и может быть настроено.
- `StreamName`: ID потока, который уникально идентифицирует поток и может быть настроен.
- `txSecret`: Строка аутентификации, созданная после включения аутентификации воспроизведения.
- `txTime`: Временная метка истечения URL воспроизведения, настроенная в консоли.

> **Примечание:** Если вы включили аутентификацию, фактическое время истечения URL будет `txTime` плюс период действия ключа аутентификации. Для удобства консоль позволяет вам указать время истечения URL в читаемом формате. Если вы включите аутентификацию, при создании URL воспроизведения система преобразует его в шестнадцатеричную временную метку (значение `txTime`). Если вы начнете отправку или воспроизведение до истечения срока действия и поток не будет прерван, отправка или воспроизведение могут продолжаться даже после истечения URL.

### URL воспроизведения транскодированного потока

Если шаблон транскодирования привязан к вашему домену воспроизведения и вы хотите воспроизвести транскодированный поток, вам нужно добавить имя шаблона (`_transcoding template name`) к исходному URL воспроизведения.

Например, если исходный URL воспроизведения — `http://domain/AppName/StreamName.flv?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)`, а имя привязанного шаблона транскодирования — `hd`, URL воспроизведения транскодированного потока будет `http://domain/AppName/StreamName_hd.flv?txSecret=Md5(key+StreamName_hd+hex(time))&txTime=hex(time)`.

### URL воспроизведения с адаптивной скоростью

Для воспроизведения с адаптивной скоростью поддерживаются только HLS и WebRTC. Форматы URL для двух протоколов различаются.

- Чтобы получить **URL адаптивной скорости HLS**, добавьте имя шаблона (`_adaptive bitrate template name`) после `StreamName` исходного URL воспроизведения.
Например, предположим, что исходный URL воспроизведения — `http://domain/AppName/StreamName.m3u8?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)`, а имя привязанного шаблона адаптивной скорости — `autobitrate`.
URL адаптивной скорости HLS будет `http://domain/AppName/StreamName_autobitrate.m3u8?txSecret=Md5(key+StreamName_autobitrate+hex(time))&txTime=hex(time)`.
- Формат **URL адаптивной скорости WebRTC**:

`Домен воспроизведения (domain) + Имя приложения (AppName, по умолчанию live), ID потока (StreamName) + Информация аутентификации + Имена потоков адаптивной скорости + Имя первоначально воспроизводимого потока + Режим управления битрейтом.`

> **Примечание:** Имена адаптивных потоков отображаются в убывающем порядке по битрейту.
> Предположим, что привязанный шаблон адаптивной скорости имеет три потока. Их имена — "test 1", "test 2" и "test 3", а их битрейты — 200 Кбит/с, 300 Кбит/с и 400 Кбит/с соответственно. URL адаптивной скорости WebRTC будет: webrtc://domain/AppName/StreamName?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)&tabr_bitrates=test3,test2,test1&tabr_start_bitrate=test1&tabr_control=auto

### URL воспроизведения H.265

CSS поддерживает отправку и воспроизведение потоков H.265. Если исходный поток — это поток H.264, вы можете настроить шаблон транскодирования для его транскодирования в поток H.265. Подробнее о том, как это сделать в консоли или путем вызова API, см. [Live Remuxing and Transcoding](https://intl.cloud.tencent.com/document/product/267/31561).

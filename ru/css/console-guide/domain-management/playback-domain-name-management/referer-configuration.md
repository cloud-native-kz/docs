# Конфигурация Referer

Вы можете установить черный/белый список referer и правила для блокирования/разрешения запросов воспроизведения с целью защиты контента прямой трансляции. Вы также можете выбрать, разрешить ли пустой referer.

## Способ конфигурации

URL-адрес Referer основан на протоколе HTTP. CSS использует поле referer в HTTP-запросе для определения источника и проверки запроса, а затем определяет, принять или отклонить запрос.

## Примечания

- Информация о Referer включена в HTTP. После конфигурации RTMP и не веб-WebRTC не будут проверять конфигурацию Referer и будут по-прежнему воспроизводиться нормально. Если вам нужно настроить Referer, рекомендуется использовать FLV или HLS для воспроизведения.
- Включение, отключение или изменение referer вступает в силу через 15-20 минут после конфигурации. Вам не нужно повторно отправлять потоки.
- Функция защиты от hotlink-ссылок Referer проверяет информацию о referer в заголовке HTTP-запроса для проверки действительности запроса и разрешения или отклонения прямой трансляции. Однако могут быть случаи, когда поддельный referer обходит проверку и получает горячую ссылку на сервис. Поэтому мы рекомендуем не полагаться сильно на referer для защиты контента.

## Предварительные условия

- Вы активировали сервис CSS и вошли в [консоль CSS](https://console.tencentcloud.com/live/livestat).
- Вы [добавили домен воспроизведения](https://intl.cloud.tencent.com/document/product/267/35970).

## Включение Referer

1. Выберите [**Управление доменом**](https://console.tencentcloud.com/live/domainmanage) и нажмите на целевой **домен воспроизведения** или нажмите **Управление** справа, чтобы войти на страницу управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d4c60ab309ee11f0990f52540099c741.png)

2. В разделе **Управление доступом** > **Конфигурация Referer** нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/568af26447f711eeb917525400b31cf9.png), чтобы включить защиту от hotlink-ссылок Referer.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/4af0aa5eab2911f096c2525400454e06.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6af68babab2911f0a0a052540099c741.png)

3. И конфигурируйте следующим образом:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/811ede53ab2911f0a0a052540099c741.png)

| Элемент конфигурации | Описание |
| --- | --- |
| Тип Referer | Выберите **Черный список** или **Белый список** в качестве типа referer. Вы не можете выбрать оба одновременно. Когда настроен белый список referer, источники запросов в списке будут разрешены для доступа к контенту прямой трансляции, а те, которые не в списке, будут заблокированы. Когда настроен черный список referer, источники запросов в списке будут заблокированы от доступа к контенту прямой трансляции, а те, которые не в списке, будут разрешены. |
| Разрешить пустой Referer | Если эта функция включена, доступ будет разрешен для HTTP-запросов с пустым или отсутствующим полем referer. Пользователи могут получить прямой доступ к URL-адресу прямой трансляции через браузеры. Если эта функция отключена, запросы с пустым referer будут отклонены. |
| Шаблоны Referer | Общее количество символов для правил не может превышать 4000. (Рекомендуется, чтобы количество правил не превышало 200.) Разделяйте правила разрывами строк. Пустые строки и точки с запятой (；) не допускаются. Для обычных правил строки в этих правилах могут быть сопоставлены, и поддерживается подстановочный символ * для нечеткого сопоставления. Например, `https://*.domain.com`. Для правил регулярных выражений они должны быть заключены в скобки（）. Например, вы можете использовать `(^https?://www.domain.com($/)` для сопоставления `www.domain.com` и использовать `(https?://[^/?]*domain.com($\|/)` для сопоставления `*.domain.com`. |

4. Нажмите **Сохранить**, чтобы сохранить конфигурацию.

## Изменение Referer

1. Выберите [**Управление доменом**](https://console.tencentcloud.com/live/domainmanage) и нажмите на целевой **домен воспроизведения** или нажмите **Управление** справа, чтобы войти на страницу управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a4f0848409f111f09d28525400bf7822.png)

2. В разделе **Управление доступом** > **Конфигурация Referer** нажмите **Изменить**, чтобы перейти на страницу конфигурации referer.
3. Измените элементы конфигурации и нажмите **Сохранить**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a582b4f3ab2911f0b08552540044a08e.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/bc709151ab2911f096c2525400454e06.png)

## Отключение Referer

После [включения referer](#enabling-referer) вы можете отключить его, выполнив следующие действия:

1. Выберите [**Управление доменом**](https://console.tencentcloud.com/live/domainmanage) и нажмите на целевой **домен воспроизведения** или нажмите **Управление** справа, чтобы войти на страницу управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c7b670e309f111f09ecc52540044a08e.png)

2. В разделе **Управление доступом** > **Конфигурация Referer** нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/c2384be847f811ee9f16525400a32c73.png), чтобы отключить referer.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c7a92ee7ab2911f09b75525400bf7822.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e97a5b3bab2911f09b75525400bf7822.png)

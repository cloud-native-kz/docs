# Конфигурация удаленной аутентификации

При удаленной аутентификации после проверки запроса push/playback для защиты от hotlink CSS вызовет API вашего сервера, чтобы отправить запрос на ваш сервер. Таким образом вы сможете определить, является ли запрос законным. На основе результата, возвращаемого вашим сервером, CSS одобрит или отклонит запрос push/playback. Это обеспечивает более точную аутентификацию и повышает безопасность. Однако вам потребуется разработать собственный сервер аутентификации.

## Рабочий процесс

Удаленная аутентификация работает следующим образом:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e6459340df7a11eea462525400bb593a.png)

| № | Описание |
| --- | --- |
| 1 | Запрос отправляется в CSS. |
| 2 | Если удаленная аутентификация включена для домена, CSS будет обрабатывать запрос согласно указанным спецификациям, а затем отправит его на ваш сервер аутентификации. |
| 3 | Ваш сервер аутентификации возвращает результат. Код состояния HTTP 200 указывает на то, что запрос должен быть одобрен, а код 403 указывает на то, что запрос должен быть отклонен. |
| 4 | CSS одобряет или отклоняет запрос на основе результата. |

## Предварительные требования

- Вы активировали CSS и вошли в [консоль CSS](https://console.tencentcloud.com/live/livestat).
- Вы добавили [доменное имя для воспроизведения](https://www.tencentcloud.com/document/product/267/35970).

## Конфигурирование удаленной аутентификации

1. Выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели. Нажмите на имя целевого доменного имени для воспроизведения или нажмите **Manage** справа, чтобы войти на страницу управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6887017d013011efbf03525400ae4d13.png)

2. На вкладке **Access Control** найдите **Remote authentication**.
3. Нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/18160fb8cc2311f091ab5254007c27c5.png), чтобы включить удаленную аутентификацию и выполните следующие настройки:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e68ab9a9cc2211f0afdc52540044a08e.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/0cccb5dbcc2311f093295254001c06ec.png)

| Элемент конфигурации |  | Описание |
| --- | --- | --- |
| Адрес сервера аутентификации |  | Адрес вашего сервера аутентификации (обязательно). Формат: http(s)://+Доменное имя или IP адрес+Порт+Путь. |
| Метод запроса |  | По умолчанию выбран POST. Вы также можете использовать HEAD или GET. |
| Аутентификация URL | Параметры для сохранения | По умолчанию сохраняются все параметры URL. Вы также можете указать параметры для сохранения или удалить все параметры. Если вы выберете "Сохранить указанный параметр", заполните поле параметрами, которые вы хотите сохранить. Разделяйте их символом \|, например value1\|value2. Параметры чувствительны к регистру ("key" и "KEY" — это разные параметры). |
|  | Пользовательские параметры для добавления | Нажмите "Add" для добавления параметров аутентификации (максимум 50). Вы можете выбрать параметр для добавления или добавить пользовательский параметр. Доступные параметры включают "host", "uri", "client_ip" и "cdn_ip", которые обозначают доменное имя для воспроизведения, исходный URL запроса, IP адрес клиента и IP адрес CDN соответственно. Если вы выберете "Custom", обязательны "Parameter" и "Value". Имена и значения чувствительны к регистру ("key" и "KEY" — это разные параметры). Китайские символы не допускаются. |
| Аутентификация заголовка запроса | Заголовок запроса для сохранения | По умолчанию сохраняются все параметры URL. Вы также можете указать параметры для сохранения или удалить все параметры. Если вы выберете "Сохранить указанный параметр", заполните поле параметрами, которые вы хотите сохранить. Разделяйте их символом \|, например value1\|value2. Если вы выберете "Сохранить все", узел CDN удалит заголовок host. Если вы хотите его сохранить, выберите "Сохранить указанный параметр" или добавьте пользовательский параметр. Параметры не чувствительны к регистру. |
|  | Пользовательские параметры для добавления | Нажмите "Add" для добавления параметров аутентификации (максимум 50). Вы можете выбрать параметр для добавления или добавить пользовательский параметр. Доступные параметры включают "User-Agent", "Referer" и "X-Forwarded-For", которые обозначают информацию о системе и браузере пользователя, источник URL и маскировку URL соответственно. Если вы выберете "Custom", обязательны "Parameter" и "Value". Имена и значения не чувствительны к регистру. Китайские символы не допускаются. |
| Период ожидания (мс) за попытку |  | Это поле обязательно. Введите значение от 500 до 3000. По умолчанию 3000. |
| Максимальное количество повторов |  | Введите значение от 0 до 3. По умолчанию 1. |
| Поведение при истечении времени ожидания |  | По умолчанию установлено значение "Approve". Вы также можете установить значение "Reject". Одобрить или отклонить запрос, если система не получит ответ (код состояния HTTP 200 или 403) после истечения общего периода ожидания (Общее время ожидания = Период ожидания за попытку x (Максимальное количество повторов + 1)). |
| Асинхронная аутентификация |  | По умолчанию асинхронная аутентификация отключена. Вы можете включить эту функцию вручную в зависимости от ваших конкретных бизнес-требований. После включения воспроизведение начнется без ожидания результата удаленной аутентификации, что позволит немедленно просматривать контент. Если удаленная аутентификация впоследствии не пройдет, воспроизведение будет прервано. Такой подход избегает проблемы увеличения времени загрузки первого кадра из-за задержки удаленной аутентификации. В режиме асинхронной аутентификации аутентификация файлов ts и m3u8 в протоколе HLS не будет эффективна, и синхронная аутентификация будет продолжать поддерживаться. |

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3d23eb69cc2311f091ab5254007c27c5.png)

4. Нажмите **Save**.

> **Примечание:** После конфигурирования функции удаленной аутентификации для ее вступления в силу потребуется примерно 10 минут. Спасибо за ваше терпение.![](https://staticintl.cloudcachetci.com/cms/backend-cms/55167161013111efbba352540095b445.png)

## Изменение параметров удаленной аутентификации

1. Выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели. Нажмите на имя целевого доменного имени для воспроизведения или нажмите **Manage** справа, чтобы войти на страницу управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d9be9dac013011efbf03525400ae4d13.png)

2. На вкладке **Access Control** найдите **Remote authentication** и нажмите **Edit**.
3. Измените параметры и нажмите **Save**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e0e7a338013011efbf03525400ae4d13.png)

## Отключение удаленной аутентификации

1. Выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели. Нажмите на имя целевого доменного имени для воспроизведения или нажмите **Manage** справа, чтобы войти на страницу управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ea426cbc013011efbf03525400ae4d13.png)

2. На вкладке **Access Control** найдите **Remote authentication** и нажмите ![img](https://staticintl.cloudcachetci.com/cms/backend-cms/e6470b93df7a11eeb419525400ea3514.png), чтобы отключить удаленную аутентификацию.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f3598bd1013011ef9125525400275b90.png)

3. Чтобы отключить конфигурацию удаленной аутентификации, предполагается, что это займет примерно 10 минут. Нажмите **Confirm**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/0f03a165013111efb881525400a7e516.png)

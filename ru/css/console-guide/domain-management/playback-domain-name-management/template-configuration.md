# Конфигурация шаблона

В CSS по умолчанию для воспроизведения используется исходный битрейт. Для использования другого битрейта воспроизведения вы можете привязать свой домен к шаблону транскодирования или адаптивного битрейта. В этом документе показано, как привязать шаблон к домену воспроизведения и отвязать его.

## Примечания

- Шаблон вступает в силу примерно через 5-10 минут после привязки к домену.
- После указания шаблона транскодирования backend будет генерировать URL-адреса воспроизведения различных форматов для трансодированного потока. Чтобы избежать искажения изображения, отправляйте поток с разрешением, максимально близким к исходному разрешению.
- H.265 поддерживается меньшим числом плееров, чем H.264. Воспроизведение может не сработать, если плеер не поддерживает H.265. Чтобы решить эту проблему, вы можете [настроить шаблон транскодирования](https://intl.cloud.tencent.com/document/product/267/31071) для транскодирования видео в H.264.
- Первому пользователю, обращающемуся к URL-адресу с другим битрейтом воспроизведения, может потребоваться время на загрузку.
- Один домен может быть привязан к нескольким шаблонам транскодирования. После привязки шаблона видео будет трансодировано в соответствии со спецификациями шаблона.
- Вы можете создать до **50** шаблонов транскодирования.

## Предварительные условия

- Вы вошли в [консоль CSS](https://console.tencentcloud.com/live) и добавили [домен воспроизведения](https://intl.cloud.tencent.com/document/product/267/35970).
- Вы создали [шаблон транскодирования](https://intl.cloud.tencent.com/document/product/267/31071) или [шаблон адаптивного битрейта](https://www.tencentcloud.com/document/product/267/50271#creating-an-adaptive-bitrate-template).

## Шаблон транскодирования

### Привязка шаблона транскодирования

1. Перейдите в [Управление доменами](https://console.tencentcloud.com/live/domainmanage). Нажмите на имя домена воспроизведения или **Управление** справа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7240bc1111ce11ef89cc5254002fd0a8.png)

2. Выберите **Конфигурация шаблона > Конфигурация транскодирования** и нажмите **Редактировать** в верхнем правом углу вкладки **Конфигурация транскодирования**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7d91ccbbcb9311f08e74525400bf7822.png)

3. В зависимости от ваших фактических потребностей выберите различные шаблоны конфигурации транскодирования.

> **Примечание:** Выбор различных шаблонов конфигурации транскодирования определит метод кодирования и параметры битрейта, установленные шаблоном транскодирования для URL-адреса воспроизведения в этом домене.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ca861dd8cb9311f096d1525400e889b2.png)

4. Нажмите **Подтвердить**.

### Формат URL-адреса для трансодированных потоков

Предположим, имя привязанного шаблона транскодирования — **hd**, а исходный URL-адрес воспроизведения выглядит следующим образом:

```
http://domain/AppName/StreamName.flv?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)
```

Для воспроизведения трансодированного потока необходимо использовать следующий URL-адрес:

```
http://domain/AppName/StreamName_hd.flv?txSecret=Md5(key+StreamName_hd+hex(time))&txTime=hex(time)
```

### Отвязка шаблона транскодирования

1. Перейдите в [Управление доменами](https://console.tencentcloud.com/live/domainmanage). Нажмите на имя домена воспроизведения или **Управление** справа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/4f58dd4211cf11ef9fa952540019e87e.png)

2. Выберите **Конфигурация шаблона > Конфигурация транскодирования** и нажмите **Редактировать** в верхнем правом углу вкладки **Конфигурация транскодирования**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/4076d7fdcb9411f08e74525400bf7822.png)

3. В зависимости от ваших фактических потребностей снимите выделение с соответствующих шаблонов.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/68ddb3eecb9411f091ab5254007c27c5.png)

4. Нажмите **Подтвердить**.

> **Примечание:** Чтобы удалить шаблон, сначала отвяжите его, а затем перейдите в **Конфигурация функций** > [Прямое транскодирование](https://console.tencentcloud.com/live/config/transcode) для его удаления. Для получения подробной информации см. [Удаление шаблона](https://intl.cloud.tencent.com/document/product/267/31071#delect).

## Шаблон адаптивного битрейта

### Привязка шаблона адаптивного битрейта

1. Перейдите в [Управление доменами](https://console.tencentcloud.com/live/domainmanage). Нажмите на имя домена воспроизведения или **Управление** справа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/cd87817411cf11ef9af4525400720cb5.png)

2. Выберите **Конфигурация шаблона > Конфигурация адаптивного битрейта** и нажмите **Редактировать** в верхнем правом углу вкладки **Конфигурация адаптивного битрейта**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ac978c18cb9411f0ae4f52540099c741.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/bbca0a75cb9411f093295254001c06ec.png)

3. В зависимости от ваших фактических потребностей выберите подходящий шаблон конфигурации адаптивного битрейта.

> **Примечание:** Выбор различных шаблонов конфигурации адаптивного битрейта определит информацию подпотока, установленную шаблоном адаптивного битрейта для URL-адреса воспроизведения в этом домене.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e292e377cb9411f084a45254005ef0f7.png)

4. Нажмите **Подтвердить**.

### Формат URL-адреса адаптивного битрейта

Для воспроизведения адаптивного битрейта поддерживаются только HLS и WebRTC. Форматы URL-адресов для двух протоколов отличаются. Для получения подробной информации см. [Конфигурация воспроизведения](https://intl.cloud.tencent.com/document/product/267/31058).

**URL-адрес HLS**:

**Предположим**, имя привязанного шаблона адаптивного битрейта — **autobitrate**, а исходный URL-адрес воспроизведения выглядит следующим образом:

```
http://domain/AppName/StreamName.m3u8?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)
```

Для воспроизведения трансодированного потока необходимо использовать следующий URL-адрес:

```
http://domain/AppName/StreamName_autobitrate.m3u8?txSecret=Md5(key+StreamName_autobitrate+hex(time))&txTime=hex(time)
```

**URL-адрес WebRTC**:

**Предположим**, привязанный шаблон адаптивного битрейта имеет три потока. Их имена — "test 1", "test 2" и "test 3", а их битрейты составляют 200 Кбит/с, 300 Кбит/с и 400 Кбит/с соответственно.

URL-адрес для воспроизведения адаптивного битрейта будет следующим:

```
webrtc://domain/AppName/StreamName?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)&tabr_bitrates=test3,test2,test1&tabr_start_bitrate=test1&tabr_control=auto.
```

### Отвязка шаблона адаптивного битрейта

1. Перейдите в [Управление доменами](https://console.tencentcloud.com/live/domainmanage). Нажмите на имя домена воспроизведения или **Управление** справа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/519fdadf11d011efa2935254005ac0ca.png)

2. Выберите **Конфигурация шаблона > Конфигурация адаптивного битрейта** и нажмите **Редактировать** в верхнем правом углу вкладки **Конфигурация адаптивного битрейта**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/09fd4c21cb9511f0ae4f52540099c741.png)

3. В зависимости от ваших фактических потребностей снимите выделение с соответствующих шаблонов.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/1ffa3442cb9511f0ae4f52540099c741.png)

4. Нажмите **Подтвердить**.

> **Примечание:** Чтобы удалить шаблон, сначала отвяжите его, а затем перейдите в **Конфигурация функций** > [Прямое транскодирование](https://console.tencentcloud.com/live/config/transcode) для его удаления. Для получения подробной информации см. [Удаление шаблона](https://intl.cloud.tencent.com/document/product/267/31071#delect).

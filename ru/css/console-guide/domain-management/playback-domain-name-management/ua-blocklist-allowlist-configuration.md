# Конфигурация списка блокировки/разрешения UA

Cloud Streaming Services (CSS) поддерживает контроль доступа путем настройки правил чёрного и белого списков User-Agent. Этот метод принимает решение о правилах на основе информации User-Agent в заголовке HTTP-запроса пользователя, чтобы разрешить или запретить доступ пользователя по мере необходимости.

## Как это работает

- **Настройка списка разрешения UA**: только настроенный контент UA может получить доступ к текущему контенту прямого эфира.
- **Настройка списка блокировки UA**: только настроенный контент UA не может получить доступ к текущему контенту прямого эфира.

## Что нужно знать

- Включение/отключение списка блокировки/разрешения UA должно вступить в силу в течение 15-20 минут.
- Если аутентификация UA и другие методы аутентификации настроены одновременно, порядок приоритета следующий: Протокол > IP > Регион > UA. Система сначала проводит аутентификацию по протоколу, затем по IP, региону и, наконец, по UA.

## Предварительные требования

- Вы активировали CSS и вошли в [консоль CSS](https://console.tencentcloud.com/live/livestat).
- Вы добавили [домен воспроизведения](https://www.tencentcloud.com/document/product/267/35970).

## Включение списка блокировки/разрешения UA

1. Выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage), нажмите на **Playback Domain**, для которого требуется настройка списка блокировки/разрешения UA, или нажмите кнопку **Manage** справа и перейдите на страницу управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6abeb062959a11ef9dcd525400f69702.png)

2. В разделе **Access Control** > **UA Blocklist/Allowlist** нажмите на ![](https://staticintl.cloudcachetci.com/cms/backend-cms/6ab51ea8959a11efb488525400a9236a.png), чтобы включить список блокировки/разрешения UA.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/85a512b3cc3211f0afdc52540044a08e.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6bf680f7cc3311f08e74525400bf7822.png)

3. После включения **списка блокировки/разрешения UA** перейдите на страницу конфигурации **списка блокировки/разрешения UA** и выполните следующую настройку:

Список блокировки

Список разрешения

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d0d9b52f95cc11efb488525400a9236a.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/df6d901a95cc11ef8bbd525400fdb830.png)

| Элемент конфигурации | Описание |
| --- | --- |
| Authenticate By | Список разрешения или блокировки: вы не можете выбрать оба одновременно. |
| Empty UA | Функция пустого User-Agent по умолчанию отключена и может быть включена вручную. После включения пустого User-Agent: в сценарии чёрного списка запросы разрешены, если значение UA пусто или поле UA не существует. В сценарии белого списка запросы отклоняются, если значение UA пусто, и запросы разрешены, если поле UA не существует. |
| Authentication Content | Поддерживаются подстановочные знаки * и несколько значений, по одному значению в строке. Для нескольких значений заполните каждое значение на отдельной строке. Английские точки с запятой (;) не поддерживаются. Например: curl**IE**Chrome**Firefox*. Учитывается регистр символов, поддерживается до 100 символов. |

4. Нажмите **Confirm**, чтобы сохранить конфигурацию.

> **Примечание:** список блокировки/разрешения UA вступит в силу примерно через 10 минут после конфигурации. Пожалуйста, подождите.

## Изменение списка блокировки/разрешения UA

1. Выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели и нажмите на целевой **домен воспроизведения** или нажмите кнопку **Manage** справа, чтобы перейти на страницу управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6aede7ea959a11ef9dcd525400f69702.png)

2. Нажмите на Access Control и в области **UA Blocklist/Allowlist** нажмите кнопку **Edit**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8c8c060695d011efa50f52540075b605.png)

3. Измените конфигурацию и нажмите **Confirm**.

## Отключение списка блокировки/разрешения UA

Выполните следующие шаги для отключения списка блокировки/разрешения UA:

1. Выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели и нажмите на целевой **домен воспроизведения** или нажмите кнопку **Manage** справа, чтобы перейти на страницу управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6abfd58e959a11efafd5525400d5f8ef.png)

2. Выберите вкладку **Access Control**. В области **UA Blocklist/Allowlist** нажмите на ![img](https://staticintl.cloudcachetci.com/cms/backend-cms/6abbebf5959a11efafd5525400d5f8ef.png), чтобы отключить список блокировки/разрешения UA.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a6d38ff595d111efb488525400a9236a.png)

3. При закрытии списка блокировки/разрешения UA система выведет окно подтверждения. Нажмите **Confirm**, чтобы отключить функцию. Обратите внимание, что после отключения вступление в силу может занять 15-20 минут.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/5888f8fb95d311efa50f52540075b605.png)

# Контроль задержки

Возможно контролировать задержку при воспроизведении HLS путём изменения размера и количества сегментов HLS. Обратите внимание, что установка слишком низкой задержки может привести к прерывистому воспроизведению.

## Предварительные требования

- Вы активировали CSS и вошли в [консоль CSS](https://console.tencentcloud.com/live/livestat).
- Вы добавили [доменное имя для трансляции](https://intl.cloud.tencent.com/document/product/267/35970).

## Контроль задержки

1. Выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели и нажмите на целевое **доменное имя для трансляции** или нажмите **Manage** справа для входа на страницу деталей.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/06fae34811ca11ef9fa952540019e87e.png)

2. В разделе **Advanced Configuration** > **Playback latency configuration** вы можете настроить задержку для HLS.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/225f28d911ca11efaa1c525400f65c2a.png)

3. На основе ваших фактических требований к бизнесу выберите подходящие параметры задержки.

Выберите различные конфигурации, соответствующие конфигурациям сегментов HLS: высокая — 5 секунд × 4 сегмента, средняя — 4 секунды × 3 сегмента, низкая — 2 секунды × 3 сегмента. Фактическая длительность слайса связана с длиной GOP. Каждый TS должен содержать по крайней мере один GOP. Рекомендуется установить GOP потока трансляции на 1–2 секунды.

4. Когда GOP установлен на две секунды, задержка выглядит следующим образом:

| Параметр | Высокая | Средняя | Низкая |
| --- | --- | --- | --- |
| Предполагаемая задержка | 20–25с | 10–15с | 6–8с |

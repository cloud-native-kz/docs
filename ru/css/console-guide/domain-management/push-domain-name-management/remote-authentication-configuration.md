# Конфигурация удаленной аутентификации

При удаленной аутентификации после аутентификации запроса push/playback для защиты от hotlink CSS вызовет ваш серверный API для отправки запроса на ваш сервер, чтобы вы могли определить, является ли запрос законным. На основе результата, который возвращает ваш сервер, CSS одобрит или отклонит запрос push/playback. Это обеспечивает более точную аутентификацию и повышает безопасность. Однако вам необходимо разработать собственный сервер аутентификации.

## Рабочий процесс

Удаленная аутентификация работает следующим образом:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7d79906564f711ee94c3525400d793d0.png)

| № | Описание |
| --- | --- |
| 1 | Запрос отправляется в CSS. |
| 2 | Если удаленная аутентификация включена для домена, CSS обработает запрос как указано, а затем отправит его на ваш сервер аутентификации. |
| 3 | Ваш сервер аутентификации возвращает результат. HTTP-код состояния 200 указывает на то, что запрос должен быть одобрен, а код 403 указывает на то, что запрос должен быть отклонен. Чтобы обеспечить непрерывность основных бизнес-операций и предотвратить неправильное отклонение запросов пользователей из-за ошибок службы аутентификации, система разрешит все запросы, если сервер аутентификации вернет код состояния, отличный от 200 или 403. |
| 4 | CSS одобряет или отклоняет запрос на основе результата. |

## Предварительные условия

- Вы активировали CSS и вошли в [консоль CSS](https://console.tencentcloud.com/live/livestat).
- Вы добавили домен push.

## Конфигурирование удаленной аутентификации

1. Войдите в консоль CSS и выберите [Domain Management](https://console.intl.cloud.tencent.com/live/domainmanage) на левой боковой панели. Нажмите на **домен push**, для которого вы хотите настроить удаленную аутентификацию, или нажмите на **Manage** справа, чтобы перейти на страницу управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/0c93ddba2c8c11efa4f552540077de32.png)

2. На вкладке **Advanced Configuration** найдите **Remote authentication**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9e9616b1ab2511f0b0045254007c27c5.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b2bb315aab2511f09710525400e889b2.png)

3. Нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/cf3c26bc64fc11ee94c3525400d793d0.png), чтобы включить удаленную аутентификацию, и выполните следующие настройки:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/22432a622c8c11ef97da5254007d9c55.png)

| Элемент конфигурации |  | Описание |
| --- | --- | --- |
| Authentication server address |  | Адрес вашего сервера аутентификации (обязательно). Формат: http(s)://+Домен или IP-адрес+Порт+Путь. |
| Request method |  | По умолчанию выбран POST. Вы также можете использовать HEAD или GET. |
| URL authentication | Parameters to keep | По умолчанию сохраняются все параметры URL. Вы также можете указать параметры для сохранения или удалить все параметры.Если вы выберете **Keep specified parameters**, вам нужно ввести параметры, которые будут сохранены, в поле ввода. Китайские символы и пробелы не поддерживаются. Разделяйте несколько параметров символом "\|", например: `value1\|value2`.Параметры аутентификации чувствительны к регистру; "key" и "KEY" считаются двумя разными параметрами. |
|  | Custom parameters to add | Нажмите **Add**, тип параметра может быть либо **Select Parameter**, либо **Custom**. (можно добавить до 50 параметров)**Select Parameter** поддерживает выбор параметров host, uri, query, client_ip и cdn_ip.**host**: Домен push.**uri**: Исходный URL запроса.**client_ip**: IP-адрес клиента запроса.**cdn_ip**: IP-адрес CDN-стороны запроса.При выборе **Custom** вам нужно заполнить поля параметра и значения. Китайские символы и пробелы не поддерживаются. Параметры аутентификации чувствительны к регистру; "key" и "KEY" считаются двумя разными параметрами. |
| Timeout period (ms) per try |  | Обязательно. Введите значение между 500 и 3000. По умолчанию 3000. |
| Max retries |  | Введите значение между 0 и 3. По умолчанию 1. |
| Behavior upon timeout |  | Это указывает, следует ли одобрить или отклонить запрос, если система не получает ответ (HTTP код состояния 200 или 403) после истечения общего периода ожидания (Общий период ожидания = Период ожидания на попытку × (Максимальное количество повторов + 1)). По умолчанию установлено значение **Approve**. Вы также можете установить его на **Reject**. |

4. Нажмите **Save**.

> **Примечание:** Конфигурация удаленной аутентификации вступит в силу примерно через 10 минут после завершения.

## Изменение параметров удаленной аутентификации

1. Выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели. Нажмите на домен push, для которого вы хотите настроить удаленную аутентификацию, или нажмите на **Manage** справа, чтобы перейти на страницу управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/37c1a75c2c8c11ef97da5254007d9c55.png)

2. На вкладке **Advanced Configuration** найдите **Remote authentication** и нажмите Edit.
3. Измените параметры и нажмите **Save**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3dea25f12c8c11efb0275254006c0558.png)

## Отключение удаленной аутентификации

1. Выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели. Нажмите на **домен push**, для которого вы хотите отключить удаленную аутентификацию, или нажмите **Manage** справа, чтобы перейти на страницу управления доменом.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/5974bb7e2c8c11ef918f52540005b090.png)

2. На вкладке **Advanced Configuration** найдите **Remote authentication** и нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/de123ae42c8a11ef9130525400bf8054.png), чтобы отключить удаленную аутентификацию.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/5ffd93e62c8c11efb8c45254005a8b94.png)

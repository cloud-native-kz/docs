# Стирание аудиовизуального контента

Стирание аудиовизуального контента—Live stream moderation 2.0 может точно идентифицировать несоответствующий аудиоконтент и изображения, связанные с приватностью, такие как лица и номерные знаки, во время прямой трансляции, а затем автоматически выполнять стирание в реальном времени. Это обеспечивает полностью автоматизированное управление рисками контента на протяжении всего процесса и гарантирует безопасность контента и соответствие требованиям платформ прямых трансляций.

В этом документе описано, как создавать, изменять и удалять шаблоны стирания аудиовизуального контента через консоль.

## Примечания

- После создания шаблона его можно привязать к домену трансляции. Дополнительные сведения см. в разделе [Конфигурация стирания аудиовизуального контента](https://www.tencentcloud.com/document/product/267/70285). Привязка шаблона обычно вступает в силу в течение 5-10 минут.
- Привязка, изменение или отвязка шаблона влияют только на новые прямые трансляции и не влияют на текущие. Чтобы применить новые правила к текущим прямым трансляциям, необходимо остановить их и запустить снова.
- Стирание аудиовизуального контента—Live stream moderation 2.0 может точно идентифицировать несоответствующий аудиоконтент во время прямой трансляции и автоматически выполнять отключение звука в реальном времени. Это обеспечивает полностью автоматизированное управление рисками контента на протяжении всего процесса и гарантирует безопасность контента и соответствие требованиям платформ прямых трансляций.
- Функция стирания аудиовизуального контента является платной дополнительной услугой. Использование этой функции во время прямой трансляции повлечет взимание сборов за дополнительные услуги и сборов за интеллектуальную идентификацию MPS. Правила выставления счетов см. в разделе [Live Stream Moderation](https://www.tencentcloud.com/document/product/267/58267).
- Когда Smart Erasing и функция субтитров используются одновременно, они не могут быть активны одновременно (только стирание звука может быть синхронизировано с субтитрами). Если пользовательский шаблон настроен как с стиранием изображения, так и со стиранием аудио, и функция субтитров включена, будут действовать только субтитры и стирание аудио.

## Предварительные условия использования

Услуга CSS активирована.

## Создание шаблона стирания аудиовизуального контента

1. Войдите в [консоль CSS](https://console.tencentcloud.com/live/config/examine/erasure). Выберите **Feature Configuration** > **AI Features** > **Audio-Video Content Erasing**.

> **Примечание:** Для использования **Audio-Video Content Erasing** в **AI Features** при первом создании необходимо создать роль сервиса и авторизовать роль текущей учетной записи для использования продуктов **Media Processing Service** (MPS) и **Cloud Object Storage** (COS).

2. Нажмите **Authorize**, чтобы перейти на страницу управления ролями CAM.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b4efd5d2c9da11f08942525400e889b2.png)

3. На странице управления ролями нажмите **Authorize**. После прохождения проверки личности вы можете авторизовать Media Processing Service и Cloud Object Storage (COS) и использовать услуги Media Processing Service и Cloud Object Storage в обычном режиме.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d5d7d47700b511f18e41525400ecee81.png)

4. После успешной авторизации выберите соглашение об услугах и нажмите **Start**. Система автоматически активирует продукт Media Processing Service и Cloud Object Storage (COS) и перенаправит вас на страницу управления Intelligent Summary.
5. Нажмите **Create template**, чтобы перейти на страницу создания шаблона стирания аудиовизуального контента и настроить шаблон следующим образом:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/79126a9c00ca11f1a1eb52540073fd3b.png)

| Элемент конфигурации |  | Описание |
| --- | --- | --- |
| Template Name |  | Максимум 30 символов; поддерживаются буквы, цифры, подчеркивания и дефисы. |
| Template Description |  | Максимум 100 символов; поддерживаются китайские символы, буквы, цифры, пробелы, _ и  -. |
|   Recognition Scheme | Erasing Type | Non-compliant audio — распознавание несоответствующего текстового контента в аудио прямой трансляции и отключение звука соответствующих аудиосегментов для эффективной защиты безопасности контента прямой трансляции. Разработка типов стирания для несоответствующих изображений, логотипов и контента защиты приватности находится в процессе. Следите за обновлениями. |
|  | Non-compliant Audio Recognition Scheme | Привязать шаблон в модуле модерации прямой трансляции с выбранной аудио-текстовой идентификацией для конфигурации контента модерации. Вы можете настроить неподходящий текстовый контент, который хотите стереть, в форме пользовательского словаря в [политике распознавания аудиотекста](https://www.tencentcloud.com/document/product/267/58265#084db3fe-6220-4f02-8a0e-702efbc95940). |
| Output Information |  | Output Content — аудио после стирания |

## Привязка доменов

1. Войдите в [консоль CSS](https://console.tencentcloud.com/live/livestat). Выберите **Feature Configuration** > **AI Feature** > **Audio-Video Content Erasing**.
2. Вы можете привязать шаблон модерации к домену трансляции следующими способами:
  - **Прямая привязка домена**: нажмите **Bind Domain Name** в верхнем левом углу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e4a107f500ca11f183755254001d6acc.png)

  - **Привязка домена после создания нового шаблона стирания аудиовизуального контента**: после успешного создания шаблона стирания аудиовизуального контента нажмите **Bind Domain Name** во всплывающем окне.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b3ce377400ca11f18080525400074c32.png)

3. В окне привязки доменов выберите **шаблон стирания аудиовизуального контента** и **домен трансляции** (можно одновременно привязать несколько доменов трансляции), которые вы хотите привязать вместе, и нажмите **Confirm**, чтобы привязать их.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/001edc4500cb11f1a616525400a31896.png)

## Отвязка

1. Войдите в [консоль CSS](https://console.tencentcloud.com/live/livestat). Выберите **Feature Configuration** > **AI Feature** > **Audio-Video Content Erasing**.
2. Выберите шаблон стирания аудиовизуального контента, от которого вы хотите отвязать домен трансляции, и нажмите **Unbind**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/18d270f600cb11f18080525400074c32.png)

3. Во всплывающем окне нажмите **Confirm**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e38fbaecef6311f0a94d52540073fd3b.png)

## Изменение шаблона

1. В левой панели навигации выберите **Feature Configuration** > **AI Features** > **Audio-Video Content Erasing**.
2. Выберите успешно созданный шаблон стирания аудиовизуального контента, нажмите **Edit** справа, чтобы перейти к изменению информации о шаблоне.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/5833cd8d00cb11f198c8525400370dda.png)

## Удаление шаблона

> **Примечание:** Если шаблон уже привязан к домену трансляции, необходимо сначала [отвязать](#cee4a59a-cba1-40f9-a4f5-1efecb3f35fe) его перед удалением шаблона. Удаленный шаблон не может быть восстановлен. Будьте осторожны при удалении шаблонов.

1. В левой панели навигации выберите **Feature Configuration** > **AI Feature** > **Audio-Video Content Erasing**.
2. Выберите ранее созданный шаблон стирания аудиовизуального контента и нажмите **Delete**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6c08852200cb11f1a1eb52540073fd3b.png)

3. Нажмите **Confirm**, чтобы окончательно удалить шаблон.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b199d6b300cc11f18ab45254001d6acc.png)

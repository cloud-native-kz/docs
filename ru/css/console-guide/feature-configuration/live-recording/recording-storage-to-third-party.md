# Сохранение записей в хранилище третьих сторон

Облачные потоковые услуги (CSS) предоставляют функцию записи прямых трансляций и сохранения файлов записей на платформы третьих сторон (Amazon S3 и Google Storage). В этом документе описано, как сохранять файлы записей на платформы третьих сторон.

## Примечания

- Функция прямой записи является **платной дополнительной** услугой. Перед использованием необходимо активировать услугу хранилища объектов третьей стороны. Использование функции прямой записи будет влечь за собой взимание платежей за максимальное количество каналов записи, плату за доставку записей в сервисы третьих сторон и плату за хранилище после записи. Подробные правила выставления счетов см. в [документации по выставлению счетов](https://www.tencentcloud.com/document/product/267/39605).
- Во время прямой трансляции соответствующий файл можно получить примерно через 5 минут после завершения записи. Например, если запись прямой трансляции начинается в 12:00 и заканчивается в 12:30, вы можете получить соответствующий клип с 12:00 до 12:30 примерно в 12:35.
- После успешного создания шаблона записи его можно привязать к домену отправки. Дополнительные сведения см. в разделе [Конфигурация записи](https://www.tencentcloud.com/document/product/267/34224). Шаблон вступит в силу примерно через 5–10 минут после успешной привязки.
- Если при инициировании задачи записи не указан шаблон записи, файл записи будет сохранен в Video on Demand (VOD) по умолчанию.

## Предварительные требования

- Вы активировали CSS и добавили [домен отправки](https://www.tencentcloud.com/document/product/267/35970).
- Вы активировали услугу хранилища объектов третьей стороны (Amazon S3 и Google Storage).

## Создание шаблона записи

1. Войдите в консоль CSS и перейдите в **Feature Configuration** > [Live Recording](https://console.tencentcloud.com/live/config/record/other).
2. В разделе Live Recording выберите **Recording Storage to Third Party**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d7cd00d3c4f811f0b285525400bf7822.png)

3. Нажмите **Create Template**, чтобы установить информацию шаблона и выполнить следующие конфигурации:
  - [Инструкции базовой конфигурации записи](https://www.tencentcloud.com/document/product/267/66854#f5092f1a-ad38-4076-b5a3-1d1f3409e1f5): включая имя шаблона, содержание записи, формат записи и другие элементы конфигурации.
  - [Инструкции по конфигурации формата записи](https://www.tencentcloud.com/document/product/267/66854#9058675d-7979-4e8d-ac53-8037acc24331): включая сегментирование файлов HLS, максимальное время записи на файл, время ожидания возобновления и другие элементы конфигурации.
4. После завершения нажмите **Save**.

### Инструкции базовой конфигурации записи

> **Примечание:** Когда исходный поток отправляется и записывается через WebRTC, звук может нормально записываться и воспроизводиться для форматов HLS и MP4, тогда как для формата FLV звук будет потерян. Рекомендуется выбрать формат HLS или MP4. Если во время записи указанного перекодированного потока выбран шаблон перекодирования только аудио, первые 2 секунды содержимого записи в форматах HLS/FLV/MP4 будут потеряны из-за преобразования формата. Пожалуйста, разумно спланируйте время отправки и записи потока. Инициирование задачи перекодирования требуется для записи перекодированных потоков, что будет влечь дополнительные расходы на перекодирование. Однако, если тот же шаблон перекодирования используется для воспроизведения, плата не будет взиматься повторно.

Если он не включен, файл записи не будет включать содержимое резервного потока. Для операций, связанных с резервными потоками, см. [Резервные потоки](https://cloud.tencent.com/login/subAccount?s_url=https%3A%2F%2Fconsole.tencentcloud.com%2Flive%2Fconfig%2Fpad). **Примечание:** При использовании водяного потока содержимое резервного потока может подвергаться сегментированной записи из-за различий в разрешении и методах кодирования прямого потока. Чтобы избежать этой проблемы, рекомендуется назначить перекодированный поток для записи. Это обеспечивает перекодирование содержимого резервного потока, эффективно предотвращая проблемы сегментирования и пикселяции. |
| Time zone | | Вы можете выбрать UTC+8 или UTC. Если этому параметру установлено значение UTC+8, время в имени файла будет в UTC+8. Если этому параметру установлено значение UTC, время в имени файла будет в UTC. |
| Recording Format | | Видео можно выводить в форматах HLS, FLV, MP4 и AAC (для записи только аудио). |

#### Инструкции по конфигурации формата записи

> **Примечание:** Поскольку файл записи загружается во время записи, невозможно получить время окончания до загрузки, поэтому время окончания не может быть добавлено в имя файла. Включение загрузки во время записи обеспечивает загрузку файлов сразу после завершения записи. Один файл записи поддерживает длительность до 12 часов и повышает надежность записи FLV при сбое. При воспроизведении файлы могут заикаться при перемещении при просмотре в Интернете, но это не влияет на локальное воспроизведение.

1. После проверки желаемого формата записи в содержании записи откроется интерфейс параметров для формата. Вы можете выбрать один или несколько форматов записи для одновременной настройки. Завершите следующие параметры:

Вы можете переключаться между различными вкладками для просмотра требований конфигурации для Audio/Video - HLS, Audio/Video - FLV, Audio/Video - MP4 и Audio-only - AAC.

Если требуется услуга постобработки, рекомендуется включить сегментирование файлов HLS и установить продолжительность отдельных файлов записи в формате HLS. Если сегментирование файлов HLS включено, продолжительность отдельных файлов записи в формате HLS может быть установлена для определения продолжительности файлов, создаваемых при постобработке записи. Если сегментирование файлов HLS отключено, запись в формате HLS будет продолжаться до завершения прямой трансляции. Если интервал прерывания потока не превышает установленное время ожидания возобновления, из одной прямой трансляции будет создан только один файл. Однако файл записи будет получен только после истечения времени ожидания возобновления, и расходы на запись будут понесены в течение времени ожидания возобновления. Разумно установите время ожидания возобновления. **Только формат HLS** поддерживает возобновление записи после прерывания потока, период времени ожидания возобновления можно настроить от 1 до 1800 секунд. |

Её переменные следующие: `{RecordSource}`: содержание записи, которое является origin для исходного потока и ID шаблона перекодирования для перекодированного потока. `{StartYear}`: время начала - год `{StartMonth}`: время начала - месяц `{StartDay}`: время начала - день `{StartMinute}`: время начала - минута `{StartSecond}`: время начала - секунда `{Domain}`: домен отправки `{AppName}`: путь отправки `{StreamID}`: ID потока `{RecordId}`: ID записи, который различает правила записи/задачи записи. При воспроизведении файлы могут заикаться при перемещении при просмотре в Интернете, но это не влияет на локальное воспроизведение. |

## Привязка домена

1. Войдите в консоль CSS и перейдите в **Feature Configuration** > [Live Recording](https://cloud.tencent.com/login/subAccount?s_url=https%3A%2F%2Fconsole.tencentcloud.com%2Flive%2Fconfig%2Frecord) > Recording Storage to Third Party.
  - **Прямая привязка домена**: Нажмите **Bind Domain Name** в верхнем левом углу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/86fdc3e0c4f911f086d9525400e889b2.png)

  - **Привязка домена после создания шаблона записи**: После успешного создания шаблона записи нажмите **Bind Domain Name** в поле напоминания.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/33102f66c4f911f0a62b5254007c27c5.png)

2. В окне привязки домена выберите **Recording Template** и **Push Domain Name** (можно одновременно привязать несколько доменов отправки), которые необходимо привязать, и нажмите **Confirm**, чтобы завершить успешную привязку.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/4742f3f3c4f911f099d952540099c741.png)

## Отвязка домена

1. Войдите в консоль CSS и перейдите в **Feature Configuration** > [Live Recording](https://cloud.tencent.com/login/subAccount?s_url=https%3A%2F%2Fconsole.tencentcloud.com%2Flive%2Fconfig%2Frecord) > Recording Storage to Third Party.
2. Выберите шаблон записи, привязанный с доменами, выберите домен для отвязки и нажмите **Unbind** справа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c054a0380adc11f0b3015254001c06ec.png)

3. Подтвердите, хотите ли вы отвязать текущий привязанный домен. Нажмите **Confirm**, чтобы отвязать его.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/599d25e4abe511ef804852540055f650.png)

> **Примечание:** Отвязка шаблона записи не повлияет на текущие прямые трансляции. Чтобы отвязка вступила в силу, необходимо остановить прямые трансляции и отправить их снова, и из новых прямых трансляций не будет создано ни одного файла записи.

## Изменение шаблона

1. Перейдите в **Feature Configuration** > [Live Recording](https://cloud.tencent.com/login/subAccount?s_url=https%3A%2F%2Fconsole.tencentcloud.com%2Flive%2Fconfig%2Frecord) > Recording Storage to Third Party.
2. Выберите успешно созданный шаблон записи и нажмите **Edit** справа, чтобы изменить информацию шаблона.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/148ff19b0add11f0ae6a525400454e06.png)

3. Просто нажмите **Save**.

## Удаление шаблона

> **Примечание:** Если шаблон привязан к домену, его необходимо отвязать перед удалением. После удаления шаблона его невозможно восстановить. Пожалуйста, будьте осторожны. Управление шаблонами записи в консоли находится на уровне домена. В настоящее время невозможно отменить правила, созданные привязанными API. Если вы привязали определённый поток через API управления записью, вам необходимо вызвать [DeleteLiveRecordRule](https://www.tencentcloud.com/document/product/267/30843) для отвязки.

1. Войдите в консоль CSS и перейдите в **Feature Configuration** > [Live Recording](https://cloud.tencent.com/login/subAccount?s_url=https%3A%2F%2Fconsole.tencentcloud.com%2Flive%2Fconfig%2Frecord) > Recording Storage to Third Party.
2. Выберите успешно созданный шаблон записи и нажмите **Delete** в верхнем правом углу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/49192f300add11f0a6d15254007c27c5.png)

3. Подтвердите, хотите ли вы удалить текущий шаблон записи. Нажмите **Confirm**, чтобы удалить его.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8a55928babe511ef897b52540075b605.png)

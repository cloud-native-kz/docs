# Запись в COS

С помощью CSS можно записывать прямые трансляции и сохранять файлы записей в VOD или COS. В этом документе показано, как записывать в облачное хранилище объектов (COS).

## Примечания

- Для записи в COS необходимо предварительно активировать COS. Рекомендуется заранее приобрести пакет хранилища, чтобы избежать приостановки обслуживания из-за задолженности. Дополнительные сведения см. в разделе [COS > Начало работы](https://www.tencentcloud.com/document/product/436/32955).
- После включения функции записи убедитесь, что ваш сервис COS находится в нормальном состоянии. Если COS не активирован или приостановлен из-за задолженности, запись прямой трансляции не удастся. Файлы записей создаваться не будут. Плата также взиматься не будет.
- Файл записи становится доступным примерно через пять минут после окончания записи. Например, если вы начали запись прямой трансляции в 12:00 и остановили в 12:30, вы сможете получить записанное видео примерно в 12:35.
- После создания шаблона записи его необходимо привязать к домену отправки. Подробные инструкции см. в разделе «Конфигурация записи». Шаблон вступает в силу через 5–10 минут после привязки.
- Запись смешанного потока не поддерживает смешивание потоков внутри китайского материка с потоками снаружи. Это приведет к ошибке и воспроизведение файла записи не удастся.
- CSS требуется разрешение на сохранение файлов записей в COS. Перед использованием функции записи в COS убедитесь, что вы предоставили необходимое разрешение. Если запись в COS не удалась из-за недостаточных разрешений, видео невозможно восстановить. Инструкции по предоставлению разрешения см. в разделе «Авторизация CSS для сохранения файлов записей в COS».
- Из-за ограничений трафика и QPS по умолчанию для бакетов хранилища COS, если ваше предполагаемое количество одновременных потоков отправки превышает 5000, пожалуйста, [подайте заявку](https://console.tencentcloud.com/workorder/category), чтобы отрегулировать лимит QPS и избежать влияния на нормальное использование вашего бизнеса.
- Если вы не укажете шаблон записи при инициировании задачи записи, файл записи будет сохранен в VOD.
- При сохранении записанного контента в COS, если папка хранилища включает ID потока, убедитесь, что ID потока соответствует соглашениям об именовании папок/файлов COS, чтобы предотвратить сбои при сохранении файлов. Дополнительные сведения о соглашениях об именовании объектов COS см. в разделе [Документация по соглашениям об именовании](https://www.tencentcloud.com/document/product/436/13324).

## Предварительные требования

- Вы активировали CSS и добавили [домен отправки](https://www.tencentcloud.com/document/product/267/35970).
- Вы активировали [COS](https://www.tencentcloud.com/products/cos).

## Создание шаблона записи

1. Войдите в консоль CSS и выберите **Feature Configuration** > [Live Recording](https://console.tencentcloud.com/live/config/record) на левой боковой панели.
2. Выберите **Save to COS**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d4b8fecb768111f087e15254005ef0f7.png)

3. Нажмите **Create template**, чтобы установить информацию шаблона и выполнить следующие конфигурации:
  - Базовая конфигурация записи: включает имя шаблона, содержание записи, формат записи и другие элементы конфигурации. Дополнительные сведения см. в разделе [Инструкции по конфигурации базовой записи](#5769e56e-2fcd-425b-a91d-a61ef3cfac5f).
  - Конфигурация формата базовой записи: включает сегментирование файлов HLS, максимальное время записи на файл, время ожидания возобновления и другие элементы конфигурации. Дополнительные сведения см. в разделе [Инструкции по конфигурации формата базовой записи](#5869c3a3-5513-4a3a-bc3b-9c7558ebe5bc).
  - (Дополнительно) Расширенная конфигурация формата записи: нажав на кнопку **Advanced Configuration**, вы можете получить доступ и выбрать дополнительные конфигурации. Дополнительные сведения см. в разделе Инструкции по расширенной конфигурации формата записи.
4. По завершении нажмите **Save**.

### Инструкции по конфигурации базовой записи

> **Примечание:** При записи исходного потока через потоковую передачу WebRTC оба формата HLS и MP4 могут записывать и воспроизводить аудио нормально, но формат FLV потеряет аудио. Рекомендуется выбрать формат HLS или MP4. Когда во время записи указанного перекодированного потока выбран шаблон только для аудио транскодирования, файл записи HLS/FLV/MP4 потеряет первые 2 секунды содержимого из-за преобразования формата. Пожалуйста, соответственно спланируйте расписание отправки и записи. Инициирование задачи транскодирования требуется для записи перекодированных потоков, что повлечет дополнительные расходы на транскодирование. Однако, если один и тот же шаблон транскодирования используется для воспроизведения, плата не будет дублироваться.

По умолчанию этот переключатель отключен. Если он включен, файлы записей будут содержать содержание резервного потока. Только при записи потока с водяным знаком и перекодированного потока с водяным знаком записанные файлы могут содержать содержание резервного потока. Если он не включен, файл записи не будет включать содержание резервного потока. Для операций, связанных с резервными потоками, см. раздел [Резервные потоки](https://cloud.tencent.com/login/subAccount?s_url=https%3A%2F%2Fconsole.tencentcloud.com%2Flive%2Fconfig%2Fpad). **Примечание:** При использовании потока с водяным знаком содержание резервного потока может пройти сегментированную запись из-за различий в разрешении и методах кодирования от прямой трансляции. Чтобы избежать этой проблемы, рекомендуется назначить перекодированный поток для записи. Это гарантирует, что содержание резервного потока будет перекодировано, что эффективно предотвращает сегментацию и проблемы с пикселизацией. |

| Формат записи |  | Видео могут быть выведены в форматах HLS, MP4, FLV и AAC (для записи только аудио). Включив постобработку мультимедиа после записи, вы можете использовать оркестрацию MPS для выполнения ряда услуг постобработки, таких как транскодирование и упаковка исходных файлов записей. Параметры хранилища для файлов постобработки определяются параметрами задачи оркестрации MPS. Исходные файлы записей остаются сохраненными. В настоящее время постобработка не поддерживается, когда содержание записи имеет адаптивные битрейты. Совместимое решение находится в разработке. Следите за обновлениями. |

### Инструкции по конфигурации формата базовой записи

> **Примечание:** Поскольку файл записи загружается во время записи, невозможно определить время окончания до загрузки, что предотвращает включение времени окончания в имя файла. Включение одновременной записи и загрузки гарантирует, что файлы загружаются сразу после окончания записи. Один файл записи поддерживает продолжительность до 12 часов и повышает возможность восстановления при сбое для записи FLV. Файлы воспроизведения могут испытывать задержки при перетаскивании для онлайн-воспроизведения, но это не влияет на локальное воспроизведение.

1. Выберите содержание и форматы записи и завершите следующие параметры:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/67e3e01b768211f096685254001c06ec.png)

| Элемент конфигурации формата базовой записи | Описание |
| --- | --- |
| Сегментирование файлов HLS | Функция сегментирования файлов HLS отключена по умолчанию. Если требуются услуги постобработки, рекомендуется включить сегментирование файлов HLS и установить продолжительность отдельных файлов записей HLS. Если сегментирование файлов HLS включено, можно настроить продолжительность отдельных файлов записей HLS, что позволяет определить продолжительность файлов, создаваемых постобработкой. Если сегментирование файлов HLS отключено, запись будет продолжаться без перерыва до окончания прямой трансляции. Если была настроена постобработка, она будет инициирована после завершения записи. |
| Максимальное время записи на файл (мин) | Аудио/видео - формат HLS. Нет верхнего лимита на продолжительность записи файла в формате HLS. Если время ожидания продолжения записи превышено, будет создан новый файл записи для продолжения записи. Когда файл записи HLS сохраняется в COS, продолжительность одного TS-файла по умолчанию установлена на 10 секунд. Когда сегментирование файлов HLS включено, продолжительность одного файла записи HLS может варьироваться от 1 до 720 минут. Аудио/видео - формат FLV. Продолжительность одного файла, записанного в формате FLV, ограничена от 1 до 720 минут. Аудио/видео - формат MP4. Продолжительность одного файла, записанного в формате MP4, ограничена от 1 до 720 минут. Только аудио - формат AAC. Продолжительность одного файла, записанного в формате AAC, ограничена от 1 до 120 минут. |
| Время ожидания возобновления (сек) | Время ожидания возобновления напрямую влияет на время создания файла записи. Когда интервал прерывания потока не превышает установленное время ожидания возобновления, одна прямая трансляция создаст только один файл. Однако файл записи будет получен после истечения времени ожидания возобновления, и плата за запись будет взиматься в течение этого периода. Пожалуйста, установите разумное время ожидания возобновления. **Только** **формат HLS** поддерживает возобновление записи после прерывания потока, с временем ожидания возобновления, настраиваемым от 1 до 1800 секунд. |
| Путь хранилища | Вы можете выбрать бакет COS из бакетов, которые вы создали и авторизовали в **Cloud Object Storage**. Регион — это регион упомянутого выше бакета, который нельзя изменить. |
| Путь резервного хранилища | Функция пути резервного хранилища отключена по умолчанию. Вы можете вручную включить эту функцию в соответствии с потребностями вашего бизнеса. Когда сетевые помехи препятствуют сохранению файла записи в основном пути хранилища, система автоматически сохранит его в пути резервного хранилища, чтобы предотвратить потерю файла. После восстановления основного пути хранилища файл записи в пути резервного хранилища будет автоматически синхронизирован с основным путем хранилища. Основной и вторичный регионы не могут совпадать. |
| Папка | Папка хранилища по умолчанию: `{RecordSource}/{Domain}/{AppName}/{StreamID}/{RecordId}/{StartYear}-{StartMonth}-{StartDay}-{StartHour}-{StartMinute}-{StartSecond}`. `{RecordSource}` указывает тип содержимого. Если записан исходный поток, это «origin». Если записан перекодированный поток, это ID шаблона транскодирования. `{StartYear}` указывает начальный год. `{StartMonth}` указывает начальный месяц. `{StartDay}` указывает начальный день. `{StartMinute}` указывает начальную минуту. `{StartSecond}` указывает начальную секунду. `{Domain}` указывает домен отправки. `{AppName}` указывает путь отправки. `{StreamID}` указывает ID потока. `{RecordId}` указывает ID задачи записи, который возвращается API CreateRecordTask. (/) указывает уровни папок. (-) является обычным символом. `{RandomID}`: случайное число |

2. Нажмите **Save**.

### Инструкции по расширенной конфигурации формата записи

1. Войдите в консоль CSS и выберите **Feature Configuration** > **Live recording** > [Save to COS](https://console.tencentcloud.com/live/config/record/cos) > Create template > Recording Format > Advanced Configuration.

> **Примечание:** ![](https://staticintl.cloudcachetci.com/cms/backend-cms/c37c841ef70d11eea865525400eeaf97.png)
> Предварительное условие: Поскольку это **первый раз активации функции постобработки MPS**, консоль Cloud Streaming Services должна вызвать Media Processing Service (MPS), что требует создания роли сервиса и [авторизации](https://console.tencentcloud.com/cam/role/grant?roleName=LVB_QCSRole&policyName=QcloudAccessForLVBRoleInMPS&principal=eyJzZXJ2aWNlIjoibHZiLnFjbG91ZC5jb20ifQ%3D%3D&serviceType=record&s_url=https://console.tencentcloud.com/live/config/record/cos) текущей учетной записи для использования продукта MPS.

2. Нажмите **authorize**, чтобы перейти на страницу управления ролями CAM.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c37af468f70d11ee8ebc52540085f884.png)

3. На странице управления ролями нажмите **Grant**, чтобы завершить проверку личности и завершить авторизацию MPS, что позволит нормально использовать MPS.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c34f85ebf70d11eea3bf525400fe11be.png)

4. После успешной авторизации система автоматически активирует продукт MPS и отобразит страницу Save to COS.
5. Выберите **Create template > Recording Format > Advanced Configuration** и выполните следующую конфигурацию:
  5.1. Вы можете переключаться между различными вкладками, чтобы просмотреть требования конфигурации для формата Audio/Video - HLS, формата Audio/Video - FLV, формата Audio/Video - MP4 и формата Audio-only - AAC.

Вы можете вручную включить эту функцию на основе потребностей вашего бизнеса. **В настоящее время только формат FLV поддерживает функцию загрузки во время записи**. После включения это позволяет немедленно загружать файлы после окончания записи, поддерживает продолжительность файла записи до 12 часов и повышает возможность восстановления при сбое для записи FLV. Файлы воспроизведения могут испытывать задержки при перетаскивании для онлайн-воспроизведения, но это не влияет на локальное воспроизведение. |

## Привязка доменного имени

1. Войдите в консоль CSS, выберите **Feature Configuration** > [**Live Recording**](https://console.tencentcloud.com/live/config/record) на левой боковой панели и нажмите **Save to COS**.
  - **Привязка домена к существующему шаблону**: нажмите **Bind Domain Name** в левом верхнем углу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/1a3dca52c4f811f0a1dc52540044a08e.png)

  - **Привязка домена после создания шаблона:** После создания шаблона нажмите **Bind Domain Name** в появившемся диалоговом окне.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8232536ec4f711f0a1dc52540044a08e.png)

2. Во всплывающем окне выберите **Recording template** и **Push domain name** (можно одновременно привязать несколько доменов отправки) и затем нажмите **Confirm**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e42a4f61c4f711f099d952540099c741.png)

## Отвязка доменного имени

1. Войдите в консоль CSS, выберите **Feature Configuration** > [**Live Recording**](https://console.tencentcloud.com/live/config/record) на левой боковой панели и нажмите **Save to COS**.
2. Выберите шаблон записи с привязанными доменами, найдите целевой домен и нажмите **Unbind**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9e0e7622782d11f08c4552540099c741.png)

3. Во всплывающем окне нажмите **Confirm**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/28e4f91e0ad911f0990f52540099c741.png)

> **Примечание:** Отвязка шаблона записи не повлияет на текущие прямые трансляции. Чтобы отменить запись текущих трансляций, остановите трансляции и начните их заново.

## Изменение шаблона

1. Перейдите в **Feature Configuration** > [**Live Recording**](https://console.tencentcloud.com/live/config/record) и выберите **Save to COS**.
2. Выберите целевой шаблон записи и нажмите **Edit** справа, чтобы изменить информацию шаблона.
3. Нажмите **Save**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/bfcf5a4b782d11f09a9a5254001c06ec.png)

## Удаление шаблона

> **Примечание:** Если шаблон уже связан, необходимо сначала [отвязать его](https://www.tencentcloud.com/document/product/267/53402#bbe9130f-7653-4f13-b3c6-194bef2f54de), прежде чем его можно будет удалить. После удаления шаблона его невозможно восстановить. Пожалуйста, действуйте с осторожностью. Управление шаблонами записей в консоли осуществляется на уровне доменного имени, и в настоящее время невозможно отменить правила, созданные связанным интерфейсом. Если вы связали конкретный поток через интерфейс управления записью, вам потребуется вызвать [Delete Recording Rule](https://www.tencentcloud.com/document/product/267/30843), чтобы отвязать его.

1. Войдите в консоль CSS, выберите **Feature Configuration** > [**Live Recording**](https://console.tencentcloud.com/live/config/record) на левой боковой панели и нажмите **Save to COS**.
2. Выберите целевой шаблон записи и нажмите **Delete** в правом верхнем углу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/db44d85f782d11f080fb5254005ef0f7.png)

3. Во всплывающем окне нажмите **Confirm**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ea41fef8782d11f080fb5254005ef0f7.png)

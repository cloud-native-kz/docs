# Запись в VOD

CSS поддерживает запись трансляций и сохранение файлов записей в VOD для скачивания и предпросмотра. В этом документе описывается, как создавать, привязывать, отвязывать, изменять и удалять шаблоны записей.
Создать шаблон записи можно двумя способами:

- В консоли CSS: подробные инструкции см. в разделе [Создание шаблона записи](#C_record).
- С помощью API: параметры и примеры API см. в разделе [CreateLiveRecordTemplate](https://intl.cloud.tencent.com/document/product/267/30845).

## Примечания

- Файлы записей сохраняются в [VOD](https://console.tencentcloud.com/vod/overview) по умолчанию. Сначала активируйте VOD. Чтобы избежать приостановки обслуживания из-за просроченных платежей, вы также можете заранее приобрести пакеты хранилища VOD. Дополнительные сведения см. в разделе [Начало работы с VOD](https://www.tencentcloud.com/document/product/266/8757).
- После включения функции записи убедитесь, что ваша служба VOD находится в нормальном состоянии. Если она не активирована или приостановлена из-за просроченных платежей, запись трансляции не будет выполнена. Файлы записей не будут созданы. Плата также не будет взиматься.
- Файл записи становится доступным примерно через пять минут после окончания записи. Например, если вы начали запись трансляции в 12:00 и остановили в 12:30, вы можете получить запись примерно в 12:35.
- Ограничено поддержкой форматов аудио- и видеофайлов (FLV/MP4/HLS) для типов кодеков, вы можете использовать только видеокодеки H.264 и H.265 и аудиокодек AAC.
- После создания шаблона записи вы можете привязать его к доменам отправки. Подробные инструкции см. в разделе [Конфигурация записи](https://intl.cloud.tencent.com/document/product/267/34224). Привязка вступает в силу примерно через 5-10 минут.
- Правила наименования создаваемых файлов записей см. в разделе [VodFileName](https://intl.cloud.tencent.com/document/product/267/30767#RecordParam).
- Привязка, отвязка или изменение шаблона влияют только на новые трансляции, а не на текущие. Чтобы применить изменение к текущим трансляциям, необходимо остановить их и отправить снова.
- Запись смешанного потока не поддерживает смешивание потоков внутри материковой части Китая с потоками за пределами. Это вызовет ошибку и воспроизведение не будет работать.

## Предварительные требования

- Вы активировали CSS и добавили [домен отправки](https://intl.cloud.tencent.com/document/product/267/35970).
- Вы активировали [сервис VOD](https://intl.cloud.tencent.com/document/product/266/8757#.E6.AD.A5.E9.AA.A41.EF.BC.9A.E5.BC.80.E9.80.9A.E4.BA.91.E7.82.B9.E6.92.AD).

## Создание шаблона записи

1. Войдите в консоль CSS и выберите **Конфигурация функций** > [Запись трансляции](https://console.tencentcloud.com/live/config/record) на левой боковой панели.
2. В параметрах записи трансляции выберите **Сохранить в VOD** .

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e7500372c4f511f0a7ca5254001c06ec.png)

3. Нажмите **Создать шаблон** для установки информации о шаблоне и выполните следующие конфигурации:
  - Базовая конфигурация записи: это включает имя шаблона, содержание записи, формат записи и другие элементы конфигурации. Подробные сведения см. в разделе [Инструкции по базовой конфигурации записи](https://www.tencentcloud.com/document/product/267/34223#a9fb1c47-c9fd-4033-b603-6536d57cce40).
  - Базовая конфигурация формата записи: это включает сегментацию файла HLS, максимальное время записи на файл, время ожидания возобновления и другие элементы конфигурации. Подробные сведения см. в разделе [Инструкции по базовой конфигурации формата записи](https://www.tencentcloud.com/document/product/267/34223#880da2a2-947b-4fee-9423-79e51e9fd843).
  - (Опционально) Расширенная конфигурация формата записи: нажав **Дополнительная конфигурация**, вы можете получить доступ к дополнительным конфигурациям и выбрать их. Подробные сведения см. в разделе [Инструкции по расширенной конфигурации формата записи](https://www.tencentcloud.com/document/product/267/34223#a3a869a0-9854-4f6c-92ae-22f54b8a7b9b).
- По завершении нажмите **Сохранить** .

### Инструкции по базовой конфигурации записи

> **Примечание:**При записи исходного потока через трансляцию WebRTC оба формата HLS и MP4 могут нормально записывать и воспроизводить звук, но формат FLV потеряет звук. Рекомендуется выбрать формат HLS или MP4.Когда во время записи указанного перекодированного потока выбирается шаблон перекодирования только для аудио, файл записи HLS/FLV/MP4 потеряет первые 2 секунды содержимого из-за преобразования формата. Пожалуйста, соответствующим образом спланируйте расписание отправки и записи.Инициирование задачи перекодирования требуется для записи перекодированных потоков, что будет влечь дополнительные затраты на перекодирование. Однако если для воспроизведения используется тот же шаблон перекодирования, плата не будет взиматься дважды.

| Записать содержание резервного потока |  | Переключатель "Записать содержание резервного потока" отображается только когда содержание записи включает поток с водяным знаком или перекодированный поток с водяным знаком. По умолчанию этот переключатель отключен.Если он включен, файлы записей будут содержать содержание резервного потока. Операции, связанные с резервными потоками, см. в разделе [Резервные потоки](https://cloud.tencent.com/login/subAccount?s_url=https%3A%2F%2Fconsole.tencentcloud.com%2Flive%2Fconfig%2Fpad). Только если вы записываете поток с водяным знаком и перекодированный поток с водяным знаком, записанные файлы могут содержать содержание резервного потока.Если он отключен, файл записи не будет включать содержание резервного потока. Операции, связанные с резервными потоками, см. в разделе [Резервные потоки](https://cloud.tencent.com/login/subAccount?s_url=https%3A%2F%2Fconsole.tencentcloud.com%2Flive%2Fconfig%2Fpad).**Примечание:**При использовании потока с водяным знаком содержание резервного потока может подвергаться сегментированной записи из-за различий в разрешении и методах кодирования от трансляции. Чтобы избежать этой проблемы, рекомендуется назначить перекодированный поток для записи. Это гарантирует, что содержание резервного потока подвергается перекодированию, эффективно предотвращая проблемы сегментации и пикселизации. |

Включите сервис обработки потока задач VOD и привяжите потоки задач VOD. После создания файла записи выполняется шаблон потока задач VOD. В настоящее время постобработка не поддерживается, когда содержание записи находится на адаптивных битрейтах. Совместимое решение находится в разработке. Оставайтесь в курсе. |

#### Инструкции по базовой конфигурации формата записи

> **Примечание:**Поскольку файл записи загружается по мере его записи, невозможно определить время окончания перед загрузкой, что предотвращает включение времени окончания в имя файла.Включение одновременной записи и загрузки обеспечивает немедленную загрузку файлов после окончания записи. Один файл записи поддерживает продолжительность до 12 часов и повышает возможность восстановления при сбое записи FLV. Файлы воспроизведения могут испытывать задержку при перетаскивании для воспроизведения в интернете, но это не влияет на локальное воспроизведение.

1. Выберите содержание и форматы записи и выполните следующие параметры:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7a633690766611f0b9a25254007c27c5.png)

| Элемент базовой конфигурации формата записи |  | Описание |
| --- | --- | --- |
| Сегментация файла HLS |  | Функция сегментации файла HLS отключена по умолчанию. Если требуются услуги постобработки, рекомендуется включить сегментацию файла HLS и установить продолжительность отдельных файлов записей HLS.Если сегментация файла HLS включена, можно настроить продолжительность отдельных файлов записей HLS, что позволяет определить продолжительность файлов, создаваемых постобработкой записи.Если сегментация файла HLS отключена, запись будет продолжаться без перерыва до окончания трансляции. Если постобработка записи была настроена, она будет инициирована после завершения записи. |
| Максимальное время записи на файл (мин) |  | Аудио/видео - формат HLSНет верхнего предела для продолжительности записи файла в формате HLS. Если время ожидания продолжения записи превышено, будет создан новый файл записи для продолжения записи.Когда файл записи HLS сохраняется в VOD, продолжительность одного файла TS устанавливается на 60 секунд по умолчанию.Если сегментация файла HLS включена, продолжительность одного файла записи HLS может варьироваться от 1 до 720 минут.Аудио/видео - формат FLVПродолжительность одного файла, записанного в формате FLV, ограничена от 1 до 720 минут.Аудио/видео - формат MP4Продолжительность одного файла, записанного в формате MP4, ограничена от 1 до 720 минут.Только аудио - формат AACПродолжительность одного файла, записанного в формате AAC, ограничена от 1 до 120 минут. |
| Время ожидания возобновления (сек) |  | Период времени ожидания возобновления напрямую влияет на время создания файла записи.Когда интервал прерывания потока не превышает установленный период времени ожидания возобновления, одна трансляция создаст только один файл. Однако файл записи будет получен после истечения периода времени ожидания возобновления, и затраты на запись будут понесены в течение периода времени ожидания возобновления. Пожалуйста, установите разумный период времени ожидания возобновления. **Только формат HLS** поддерживает возобновление записи после прерывания потока, при этом период времени ожидания возобновления настраивается от 1 до 1 800 секунд. |
| Период хранения (дни) |  | Вы можете выбрать "Постоянно" для постоянного сохранения файла записи или "Пользовательское" для указания периода хранения (до 1 500 дней). Если установить период на 0, файлы записей будут сохраняться постоянно.Если выбран указанный период, в соответствии с национальными нормами операторы должны записывать содержание видеотрансляции и обеспечивать резервное копирование хранилища. Рекомендуется хранить файлы записей от 60 дней до 3 лет. |
| Подприложение VOD/Категория |  | Поддерживается запись в указанную категорию VOD в VOD [приложении](https://console.tencentcloud.com/vod/app-manage). По умолчанию запись сохраняется в главном приложении учетной записи, и поддерживаются только приложения с открытым статусом записи. |

## Привязка имени домена

  - **Привязать домен к существующему шаблону**: нажмите **Привязать имя домена** в верхнем левом углу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9205a19dc4f611f0a1dc52540044a08e.png)

  - **Привязать домен после создания шаблона**: После [создания шаблона](#creating-a-recording-template) нажмите **Привязать имя домена** в появившемся диалоговом окне.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/2af40955c4f611f0a84e5254005ef0f7.png)

2. В всплывающем окне выберите **шаблон записи** и **домен отправки** (одновременно можно привязать несколько доменов отправки) и нажмите **Подтвердить**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/43a0d717c4f611f085c7525400454e06.png)

## Отвязка имени домена

2. Выберите шаблон записи, привязанный к именам доменов, найдите целевой домен и нажмите **Отвязать**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/138afea0766911f096685254001c06ec.png)

3. В всплывающем окне нажмите **Подтвердить**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9fb779c9f4b011ee941952540085f884.png)

> **Примечание:**Отвязка шаблона записи не повлияет на текущие трансляции.Чтобы отменить запись для текущих потоков, остановите потоки и отправьте их снова.

## Удаление шаблона

> **Примечание:**Если к шаблону привязаны имена доменов, вам необходимо [отвязать](#unbinding-a-domain-name) их перед удалением шаблона.После удаления шаблона его невозможно восстановить. Пожалуйста, будьте осторожны.В консоли шаблоны записей управляются на уровне домена. Чтобы отвязать правила записей, привязанные к потокам API, вызовите [DeleteLiveRecordRule](https://intl.cloud.tencent.com/document/product/267/30843).

2. Выберите целевой шаблон записи и нажмите **Удалить** в верхнем правом углу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/83ca74ec766911f09f3d52540099c741.png)

3. В всплывающем окне нажмите **Подтвердить**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/98efb5c8766911f0b9a25254007c27c5.png)

# Снимок экрана трансляции в реальном времени

Cloud Streaming Services (CSS) предоставляет функцию создания снимков экрана, поддерживающую использование шаблонов снимков, настроенных через консоль. После привязки шаблона к домену push-вещания система захватывает экран потока в процессе вещания и сохраняет данные снимков в Tencent Cloud Object Storage (COS). Если домен push-вещания уже привязан к конфигурации обратного вызова, Tencent Cloud отправит запрос на сервер клиента во время прямой трансляции при возникновении события обратного вызова, и сервер клиента отвечает за обработку запроса. После проверки клиент может получить пакет JSON, содержащий информацию обратного вызова снимка экрана.
В этом документе описывается создание, привязка, отвязка, изменение и удаление шаблонов снимков экрана через консоль.

Есть два способа создания шаблона снимка экрана:

- Для создания шаблона через консоль CSS см. пошаговые инструкции в разделе [Создание шаблона снимка экрана](https://www.tencentcloud.com/document/product/267/31072#.E5.88.9B.E5.BB.BA.E6.88.AA.E5.9B.BE.E9.89.B4.E9.BB.84.E6.A8.A1.E6.9D.BF).
- Вызовите API CreateLiveSnapshotTemplate для создания шаблона. Информацию о параметрах и примерах запросов см. в разделе [CreateLiveSnapshotTemplate](https://intl.cloud.tencent.com/document/product/267/30834).

## Важная информация

- Функция создания снимков экрана может использоваться независимо, однако функция обнаружения порнографического контента может быть включена только после включения функции создания снимков экрана и не может использоваться независимо. Обнаружение порнографического контента в прямом потоке полностью обновлено до функции модерации прямых трансляций, которая больше не зависит от возможности создания снимков экрана прямых трансляций. Для лучшего пользовательского опыта рекомендуется использовать функцию модерации прямых трансляций. Подробности см. в разделе [Модерация прямых трансляций](https://www.tencentcloud.com/document/product/267/58265).
- Функции создания снимков экрана и обнаружения порнографического контента стоят 0,0176 USD и 0,2294 USD за 1000 снимков соответственно. Подробности см. в разделе [Интеллектуальное обнаружение порнографического контента](https://intl.cloud.tencent.com/document/product/267/39607).
- Снимки экрана и результаты обнаружения порнографического контента хранятся в вашем бакете COS, за что взимается плата за хранилище COS. Дополнительную информацию см. в разделе [Цены на COS](https://intl.cloud.tencent.com/pricing/cos).
- Создание снимков экрана будет неудачным для потоков только с аудио, в этом случае расходы на создание снимков экрана не будут начислены.
- Если вы хотите хранить данные в баке COS **другой учетной записи**, вам необходимо сначала предоставить CSS разрешение на запись в этот бак COS. Дополнительную информацию см. в разделе [Авторизация CSS для сохранения снимков экрана в баке COS](https://intl.cloud.tencent.com/document/product/267/33384).
- Если ваш бак COS разрешает открытый доступ на чтение и содержит политически чувствительный, порнографический или другой неприемлемый контент, во избежание блокировки бака, пожалуйста, сначала удалите контент.
- После создания шаблона необходимо привязать его к домену push-вещания. Дополнительную информацию см. в разделе [Конфигурация создания снимков экрана и обнаружения порнографического контента](https://www.tencentcloud.com/document/product/267/31063). Конфигурация вступает в силу примерно за 5-10 минут.
- Управление шаблонами снимков экрана в консоли осуществляется на уровне доменного имени, и в настоящее время невозможно отменить правила, созданные через API. Для правил создания снимков экрана, привязанных к конкретным потокам через API, необходимо вызвать [DeleteLiveSnapshotRule](https://intl.cloud.tencent.com/document/product/267/30833) для их отвязки.
- Привязка, отвязка или изменение шаблона влияет только на новые потоки трансляции, но не на текущие потоки. Чтобы применить новые правила к текущим потокам, необходимо остановить их и начать вещание заново.

## Предварительные требования

- Вы активировали CSS и добавили [домен push-вещания](https://intl.cloud.tencent.com/document/product/267/35970).
- Вы создали бак COS. Подробные инструкции см. в разделе [Создание бака](https://www.tencentcloud.com/document/product/436/13309).

## Создание шаблона снимка экрана

1. Войдите в консоль CSS и выберите **Feature Configuration** > [Live Screencapture](https://console.tencentcloud.com/live/config/jtjh).

> **Примечание:** Так как сервис создания снимков экрана CSS требует сохранения снимков в баки COS, при создании шаблона снимка экрана **в первый раз** необходимо создать роль сервиса и авторизовать CSS с разрешениями на чтение и запись в COS.

2. Нажмите **Create template**.
3. Консоль откроет окно для запроса авторизации ресурсов. Нажмите **Authorize Now**, чтобы перейти на страницу управления ролями.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d0194f3a2a2211ef97da5254007d9c55.png)

4. На странице управления ролями нажмите **Grant**. После завершения проверки идентификации вы сможете завершить авторизацию ресурсов COS и обычно использовать сервис снимков экрана прямых трансляций. После успешной авторизации вы будете перенаправлены на страницу создания шаблонов снимков экрана.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d970fc192a2211ef918f52540005b090.png)

5. Нажмите **Create template**, чтобы перейти на страницу создания шаблона снимка экрана и выполнить следующую конфигурацию:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/70ddb76fc50e11f0a7ca5254001c06ec.png)

| Элемент конфигурации |  | Обязательно | Описание |
| --- | --- | --- | --- |
| Имя шаблона |  | Да | Имя шаблона создания снимков экрана и обнаружения порнографического контента, может содержать до 30 китайских символов, букв, цифр, подчеркиваний (_) и дефисов (-). |
| Описание шаблона |  | Нет | Описание шаблона снимка экрана, может содержать до 100 китайских символов, букв, цифр, пробелов, подчеркиваний (_) и дефисов (-). |
| Интервал создания снимков экрана |  | Да | Интервал создания снимков экрана, по умолчанию составляет 2 секунды. Диапазон значений: 2-300 секунд. |
| Разрешение видео |  | Да | По умолчанию сохраняется исходное разрешение, но вы можете установить высоту снимка экрана. Диапазон входной высоты составляет от 1 пикселя до 2000 пикселей, другая сторона будет автоматически масштабироваться пропорционально разрешению по умолчанию. |
| Интеллектуальное обнаружение порнографического контента |  | Нет | Функция интеллектуального обнаружения порнографического контента отключена по умолчанию и может быть включена вручную. После включения интеллектуального обнаружения порнографического контента необходимо настроить обратный вызов для получения результатов. |
| Аккаунт COS |  | Да | **Current Account** или **Other Account**. |
| AppId |  | Да | Требуется только при выборе **Other Account**. Вы можете просмотреть `APPID` аккаунта на странице [Account Information](https://console.tencentcloud.com/developer) консоли. Чтобы сохранить данные в баке COS другой учетной записи, сначала необходимо предоставить CSS доступ на чтение и запись к этому баку. Подробности см. в разделе [Авторизация CSS для сохранения снимков экрана в COS](https://www.tencentcloud.com/en/document/product/267/33384#authorizing-css-to-store-screenshots-in-cos). |
| Путь хранилища | Бак | Да | Вы можете выбрать бак COS в поле Bucket, который вы уже создали и авторизовали в **COS**. |
|  | Регион | Да | Регион соответствует региональной информации упомянутого выше бака и не может быть изменен. |
|  | Папка | Нет | Нажмите на поле для выбора папки COS. По умолчанию: {Year}-{Month}-{Day}/. Примечание: Имена папок COS могут содержать только [a-z, A-Z, 0-9] и символы -, !, _, ., * а также заполнители. |
| Путь резервного хранилища |  | Нет | Функция пути резервного хранилища отключена по умолчанию. Вы можете включить эту функцию вручную в зависимости от потребностей вашего бизнеса. Когда сетевые помехи препятствуют сохранению снимка экрана в основной путь хранилища, система автоматически сохранит файл в путь резервного хранилища для предотвращения потери файла. После восстановления основного пути хранилища снимки экрана в пути резервного хранилища будут автоматически синхронизированы с основным путем хранилища. | Имя файла |  | Нет | Формат имен файлов снимков экрана. Вы можете установить свой собственный формат. По умолчанию используется `{StreamID}-screenshot-{Hour}-{Minute}-{Second}-{Width}x{Height}{Ext}`. {AppName}: Имя приложения push. {PushDomain}: Домен push-вещания. {StreamID}: Идентификатор потока. {Year}: Время снимка экрана (год). {Month}: Время снимка экрана (месяц). {Day}: Время снимка экрана (день). {Hour}: Время снимка экрана (час). {Minute}: Время снимка экрана (минута). {Second}: Время снимка экрана (секунда). {Width}: Ширина снимка экрана. {Height}: Высота снимка экрана. {Ext}: Расширение файла (.jpg). **Примечание:** Имя файла может содержать только буквы, цифры, заполнители и символы (-, !, _, ., *). **Пример:** Если формат имени файла — `{Year}-{Month}-{Day}- {Hour}-{Ext}`, снимок экрана, захваченный в 14:00:00 1 января 2020 года, будет назван `2020010114.jpg` в COS. |

6. Нажмите **Save**, чтобы сохранить текущий шаблон.

## Привязка доменного имени

1. Войдите в консоль CSS и выберите **Feature Configuration** > [Live Screencapture](https://console.tencentcloud.com/live/config/jtjh).
2. Привяжите доменное имя одним из двух способов:
  - **Привязать домен к существующему шаблону**: Нажмите **Bind Domain Name** в верхнем левом углу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/1ab0f825c50f11f085c7525400454e06.png)

  - **Привязать доменное имя после успешного создания шаблона снимка экрана**: После успешного [создания шаблона снимка экрана](#.E5.88.9B.E5.BB.BA.E6.88.AA.E5.9B.BE.E9.89.B4.E9.BB.84.E6.A8.A1.E6.9D.BF) нажмите **Bind Domain Name** в поле напоминания.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d1b1f99dc50e11f0a7ca5254001c06ec.png)

3. В окне привязки домена выберите **шаблон снимка экрана** и **домен push-вещания** (могут быть привязаны несколько доменов push-вещания одновременно), которые необходимо привязать, и нажмите **Confirm**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ea79e32dc50e11f0a1dc52540044a08e.png)

## Отвязка доменного имени

1. Войдите в консоль CSS и выберите **Feature Configuration** > [Live Screencapture](https://console.tencentcloud.com/live/config/jtjh).
2. Выберите целевой шаблон создания снимков экрана и обнаружения порнографического контента, найдите домен, который необходимо отвязать, и нажмите **Unbind**.
![](https://staticintl.cloudcachetci.com/cms/backend-cms/203bc9f52a2311efa4f552540077de32.png)
3. В открывшемся окне нажмите **Confirm**.
![](https://staticintl.cloudcachetci.com/cms/backend-cms/248007932a2311efb8c45254005a8b94.png)

## Изменение шаблона

1. Войдите в консоль CSS и выберите **Feature Configuration** > [Live Screencapture](https://console.tencentcloud.com/live/config/jtjh).
2. Выберите шаблон создания снимков экрана и обнаружения порнографического контента, нажмите **Edit** справа и измените его информацию.
3. Нажмите **Save**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/0ebe9926df7c11eea462525400bb593a.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/2abd65162a2311ef9130525400bf8054.png)

## Удаление шаблона

> **Примечание:** Если шаблон привязан к доменам, необходимо [отвязать](#отвязка-доменного-имени) их перед удалением шаблона. Обратите внимание, что удаленный шаблон не может быть восстановлен. Действуйте с осторожностью.

1. Войдите в консоль CSS и выберите **Feature Configuration** > [Live Screencapture](https://console.tencentcloud.com/live/config/jtjh).
2. Выберите успешно созданный шаблон снимка экрана и нажмите **Delete** выше.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/30d29f6b2a2311ef9bb3525400ab9413.png)

3. Подтвердите, что вы хотите удалить текущий шаблон снимка экрана, и нажмите **Confirm** для его удаления.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/38461bed2a2311ef918f52540005b090.png)

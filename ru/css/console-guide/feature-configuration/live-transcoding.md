# Живое транскодирование

Живое транскодирование (включая видеотранскодирование и аудиотранскодирование) — это процесс, при котором исходный поток, отправляемый с сайта живого потокового вещания, преобразуется в облаке в потоки с различными кодеками, разрешениями и битрейтами перед отправкой зрителям. Это позволяет удовлетворить потребности воспроизведения в различных сетевых окружениях на разных устройствах. В этом документе описано, как создавать, привязывать, отвязывать, изменять и удалять шаблон транскодирования через консоль CSS.

**Вы можете создать шаблон транскодирования двумя способами:**

- Создать шаблон транскодирования в консоли CSS. Подробные инструкции см. в разделах [Создание стандартного шаблона транскодирования](#creating-a-standard-transcoding-template), [Создание шаблона транскодирования TSC](#C_topspeed) и [Создание шаблона транскодирования только аудио](#C_audio).
- Создать шаблон транскодирования для живых потоков с использованием API. Параметры API и примеры см. в разделе [CreateLiveTranscodeTemplate](https://www.tencentcloud.com/document/product/267/30790).

## Важная информация

- CSS поддерживает стандартное транскодирование, транскодирование Top Speed Codec (TSC) и транскодирование только аудио. Перед использованием услуг прочитайте документы по выставлению счётов.
  - Стандартное транскодирование: [Пакеты стандартного транскодирования](https://www.tencentcloud.com/document/product/267/52220#standard_pag), [Стандартное транскодирование (оплата по мере использования)](https://intl.cloud.tencent.com/document/product/267/39604)
  - Транскодирование TSC: [Пакеты транскодирования TSC](https://www.tencentcloud.com/document/product/267/52220#.E6.9E.81.E9.80.9F.E9.AB.98.E6.B8.85.E8.BD.AC.E7.A0.81.E5.8C.85), [Транскодирование TSC (оплата по мере использования)](https://intl.cloud.tencent.com/document/product/267/39604)
- По сравнению со **стандартным транскодированием**, **транскодирование TSC** обеспечивает более высокое качество видео при меньшем битрейте. Используя технологии, включая интеллектуальное распознавание сцен, динамическое кодирование и управление битрейтом на уровне CTU/строк/кадров, транскодирование TSC позволяет предоставлять услуги потокового вещания с более высоким разрешением при более низких битрейтах (в среднем на 50% ниже). Оно широко используется для потокового вещания игр, трансляций из торговых залов и трансляций событий.
- После создания шаблона вы можете привязать его к имени домена воспроизведения. Привязка вступает в силу в течение 5–10 минут.
- После привязки шаблона вы можете добавить название шаблона (`_название шаблона`) после `StreamName` в исходный URL для создания URL выходных данных транскодирования. Обратите внимание, что **название шаблона** и **название потока** не могут иметь одинаковый суффикс. Например, если название шаблона — `hd`, а `StreamName` — `test_a1_hd`, система будет использовать `test_a1` в качестве названия потока и `hd` в качестве шаблона транскодирования, и воспроизведение не удастся.
- Если вы указали высоту и ширину или короткую и длинную стороны выходных данных транскодирования, чтобы предотвратить искажение изображения, старайтесь максимально приблизить разрешение публикуемых потоков к заданным значениям.
- На странице **Live Transcoding** консоли вы можете просмотреть домен, к которому привязан шаблон, а также более точные привязки, выполненные через API. Вы также можете [отвязать](#unbinding-a-domain-name) шаблон отсюда.
- Вы можете привязать одно имя домена воспроизведения с **несколькими шаблонами транскодирования** или привязать один шаблон транскодирования с **несколькими именами доменов воспроизведения**.
- Вы можете создать до **50** шаблонов транскодирования.
- Шаблон конфигурации транскодирования для живого потокового вещания поддерживает конфигурацию функции **размытия лиц**, которая может достичь размытия лиц и специальных объектов. Чтобы использовать эту функцию, необходимо [подать тикет](https://console.tencentcloud.com/workorder/category) для запроса поддержки. Включение этой услуги повлечёт [расходы на живое транскодирование](https://www.tencentcloud.com/document/product/267/39604) и расходы на [интеллектуальное распознавание](https://www.tencentcloud.com/document/product/1041/49204) Media Processing Service (MPS).

## Создание шаблона транскодирования

### Создание стандартного шаблона транскодирования

1. Войдите в консоль CSS и выберите **Feature Configuration** > [**Live Transcoding**](https://console.tencentcloud.com/live/config/transcode).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ff4e06ecc46111f0aa02525400e889b2.png)

2. Нажмите **Create Template**, выберите **Standard Transcoding** для типа транскодирования и завершите следующую конфигурацию:
  - Базовая конфигурация: название шаблона, видеобитрейт, видеоразрешение и прочее. Подробнее см. в разделе [Базовая конфигурация стандартного транскодирования](https://www.tencentcloud.com/document/product/267/31071#e0336bf4-96b8-4b92-975d-7fbcf342bf67).
  - Расширенная конфигурация (опционально): нажмите **Advanced Configuration**, чтобы показать расширенные параметры. Подробнее см. в разделе [Расширенная конфигурация стандартного транскодирования](https://www.tencentcloud.com/document/product/267/31071#910fd1af-3f0b-473b-b737-4376216e2b8f).
3. Нажмите **Save**.

#### Базовая конфигурация стандартного транскодирования

| Базовая конфигурация стандартного транскодирования | Обязательно | Описание |
| --- | --- | --- |
| Тип транскодирования | Да | Доступные типы транскодирования включают стандартное транскодирование, транскодирование TSC или транскодирование только аудио. |
| Название шаблона | Да | Пожалуйста, введите 1–10 символов. Название шаблона живого транскодирования. Поддерживает только буквы и буквенно-цифровые комбинации и не поддерживает чистые цифры. Название шаблона не должно дублировать существующие названия шаблонов транскодирования, названия шаблонов адаптивного битрейта или названия подпотоков. |
| Описание шаблона | Нет | Описание шаблона живого транскодирования. Может содержать только китайские иероглифы, буквы, цифры, пробелы, подчёркивания (_) и дефисы (-). |
| Качество видео | Нет | Вы можете выбрать Smooth, SD, HD или FHD. После выбора значения система автоматически введёт рекомендуемый видеобитрейт и высоту, которые можно изменить. |
| Видеобитрейт (в Кбит/с) | Да | Вы можете выбрать сохранение исходного битрейта, установить видеобитрейт или использовать стандартный битрейт транскодирования. Установите скорость кодирования транскодирования, диапазон значений: 101 Кбит/с – 8000 Кбит/с. Если вы вводите значение не более 1000, оно должно быть кратно 100. Если вы вводите значение более 1000, оно должно быть кратно 500. |
| Видеоразрешение (пиксели) | Да | Выберите сохранение исходного разрешения, установку по ширине и высоте или установку по длине. По умолчанию устанавливается по длинной и короткой сторонам. Введённое значение — это значение короткой стороны, которое можно переключить на параметры ширины и высоты, и введённое значение — это значение высоты. Диапазон значений: 0–3000. Значение должно быть кратно 2. Другая сторона будет масштабироваться автоматически. |
| Шифрование DRM | Нет | По умолчанию отключено и может быть включено вручную. Чтобы включить шифрование DRM, сначала необходимо получить информацию о ключе в DRM Management. Шифрование DRM Widevine, FairPlay и NormalAES поддерживается для HLS. Для шифрования FairPlay вам необходимо загрузить сертификат, полученный от Apple, в ваш плеер. Типы шифрования: стандартное Widevine, опциональное Fairplay и NormalAES. |

Вы можете в любое время отрегулировать шаблон субтитров в соответствии с бизнес-требованиями. |
| Ограничение параметров | Нет | Ограничения параметров отключены по умолчанию. После включения ограничения, если вы введёте значение выше исходного, будет использовано исходное значение. Это может избежать проблем с качеством видео, вызванных использованием высоких параметров качества видео для транскодирования видео низкого качества. |

### Создание шаблона транскодирования TSC

2. Нажмите **Create Template**, выберите **Top Speed Codec Transcoding** для типа транскодирования и завершите следующую конфигурацию:

  - Расширенная конфигурация (опционально): нажмите **Advanced Configuration**, чтобы показать расширенные параметры. Подробнее см. в разделе [Расширенная конфигурация транскодирования Top Speed Codec](https://www.tencentcloud.com/document/product/267/31071#fa43b38d-4392-46c0-8297-47f7441fe167).
3. Нажмите **Save**.

Типы шифрования: стандартное **Widevine**, опциональное Fairplay и NormalAES. |

![](https://staticintl.cloudcachetci.com/cms/backend-cms/2cf9eac3241a11f0b44b5254007c27c5.png)

## Привязка имени домена

2. Привяжите имя домена одним из двух способов:
  - **Привязать домен к существующему шаблону**: нажмите **Bind Domain Name** в левом верхнем углу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9f4ae583c46311f0b35752540099c741.png)

  - **Привязать имя домена после создания шаблона транскодирования**: после [создания шаблона](#creating-a-transcoding-template) нажмите **Bind Domain Name** в появившемся окне.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/85e026ddc46211f0b4a7525400454e06.png)

3. В окне привязки домена выберите **шаблон транскодирования** и **имя домена воспроизведения** (можно одновременно привязать несколько имён доменов воспроизведения) и нажмите **Confirm**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a1312a83c46211f0a0935254007c27c5.png)

## Отвязка имени домена

2. Выберите целевой шаблон и нажмите **Unbind**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/358638ea241c11f09e67525400bf7822.png)

3. В появившемся окне нажмите **Confirm**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/fe9a8d83b8d011eea201525400170219.png)

## Изменение шаблона

2. Выберите целевой шаблон транскодирования и нажмите **Edit** справа для его изменения.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/57436eee241c11f0b47352540044a08e.png)

3. После изменения нажмите **Save**.

## Удаление шаблона

> **Примечание:** Если шаблон был привязан к доменам, вам необходимо [отвязать](#unbinding-a-domain-name) их перед удалением шаблона. Вы не можете удалить шаблон транскодирования, для которого включена функция живых субтитров.

1. Войдите в консоль CSS и выберите **Feature Configuration** > [Live Transcoding](https://console.tencentcloud.com/live/config/transcode).
2. Выберите целевой шаблон (убедитесь, что он не привязан к домену) и нажмите **Delete**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8775ddc9241c11f0948f52540099c741.png)

3. В появившемся окне нажмите **Confirm**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a8147c31241c11f09e67525400bf7822.png)

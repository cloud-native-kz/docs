# Детали функции сдвига во времени

Трансляция Tencent Cloud Live Broadcast поддерживает управление сдвигом во времени через информацию об индексе, включая функции просмотра деталей сдвига во времени, настройки воспроизведения со сдвигом во времени и редактирования трансляции.

## Предварительные требования

- Вы вошли в консоль CSS.
- Вы создали [шаблон сдвига во времени](https://www.tencentcloud.com/document/product/267/53312), привязали его к домену отправки и успешно отправили поток.
- Для использования возможности редактирования и сохранения видео убедитесь, что вы активировали Tencent Cloud [MPS](https://console.intl.cloud.tencent.com/mps). Сохраненный контент будет храниться в сервисе COS, а активация сервиса MPS автоматически активирует сервис COS.

## Руководство по операциям с информацией об индексе

## Инструкции

1. Нажмите **Feature Configuration** > **Time Shifting** на левой боковой панели и выберите вкладку [Time shifting details](https://console.tencentcloud.com/live/config/time-shift/meta).
2. Выберите домен или введите ID потока, укажите период времени (не более 24 часов) и нажмите **Query**.
3. Нажмите **Details**, чтобы перейти на страницу деталей.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/897796e4742111f0b9a25254007c27c5.png)

4. Просмотрите URL отправки и тип потока в области **Basic Info**.
5. Вы можете навести мышь на шкалу времени на странице **Index Details**, чтобы просмотреть позицию и время. Нажимая на шкалу времени, вы можете отметить время.
6. Нажмите на шкалу времени, чтобы просмотреть видео в определенный момент времени и отметить эту точку.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/285c0c6f742211f096685254001c06ec.png)

> **Примечание:** Для просмотра контента со сдвигом во времени домен, используемый для воспроизведения, должен иметь сертификат HTTPS. Если ваш домен воспроизведения еще не имеет сертификата HTTPS, добавьте его в **Domain Management** > [Certificate Management](https://console.tencentcloud.com/live/domainmanage/certificate). Использование функции сдвига во времени будет сопровождаться взиманием платежей за трафик/пропускную способность воспроизведения.

7. **Time Shifting and Live Clipping** можно настроить следующим образом**:**

| Элемент |  | Описание |
| --- | --- | --- |
| Выбор функции | Time shifting | Создать URL со сдвигом во времени, который задерживает воспроизведение трансляции на определенное время |
|  | Live clipping | Создать фрагмент из трансляции в указанное время начала и окончания и сохранить его постоянно |
| Домен воспроизведения |  | Выберите домен воспроизведения, добавленный в CSS. |
| Выбрать поток ABR: （Адаптивная потоковая передача） |  | По умолчанию выбран основной поток, дополнительный поток можно выбрать через раскрывающееся меню. |
| Создать URL воспроизведения со сдвигом во времени |  | Нажмите **Generate Address** для создания URL воспроизведения со сдвигом во времени и скопируйте его. Если вы выбрали поток ABR как основной поток, адрес будет создан на основе основного потока. Если вы выбрали дополнительный поток, адрес будет создан на основе дополнительного потока. |
| Переход в MPS для сохраненного контента со сдвигом во времени |  | Формат инкапсуляции можно выбрать как **MP4 type** или **HLS type**. Когда **Live clipping** выбран в качестве режима функции, формат инкапсуляции основного потока устанавливается на HLS type по умолчанию, а формат инкапсуляции дополнительного потока устанавливается на MP4 type по умолчанию. |

# Relay

Если ваш источник прямой трансляции не поддерживает возможность отправки потоков или если вы хотите транслировать видео по запросу, вы можете использовать функцию relay для быстрого получения контента из существующего источника прямой трансляции или видео с последующей доставкой в пункт назначения. Вам не нужно отправлять потоки самостоятельно.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8ea628892a0911ee909c525400cea498.png)

## Предварительные требования

- Вы активировали [CSS](https://intl.cloud.tencent.com/product/css) и вошли в [консоль CSS](https://console.intl.cloud.tencent.com/live/livestat).
- Вы добавили [доменное имя отправки потока](https://intl.cloud.tencent.com/document/product/267/35970) в консоль.

## Ограничения

- Вы можете создать до **200** задач relay. Если объем вашего бизнеса relay существенный и вам требуется больший лимит задач, обратитесь к нам, [отправив заявку в поддержку](https://intl.cloud.tencent.com/en/account/login?s_url=https%3A%2F%2Fconsole.intl.cloud.tencent.com%2Fworkorder%2Fcategory), или обратитесь к вашему менеджеру.
- Функция relay тарифицируется на основе **продолжительности задач relay**. Дополнительную информацию см. в разделе [Relay](https://intl.cloud.tencent.com/document/product/267/41059).
- CSS отвечает только за получение и пересылку контента. **Убедитесь, что ваш контент авторизован и соответствует действующему законодательству. В случае нарушения авторских прав или нарушения законов и правил CSS приостановит для вас услугу и оставляет право на судебное разбирательство.**
- Режим локального relay стал платной функцией с **00:00 23 ноября 2022 г.** Дополнительную информацию см. в разделе [Дополнительные функции](https://www.tencentcloud.com/document/product/267/51555).
- Для relay при получении или отправке глобальных потоков необходимо установить регион задачи вне материковой части Китая.
- Вы можете использовать функцию записи relay, привязав шаблон записи облачной трансляции, что будет генерировать [расходы на запись](https://www.tencentcloud.com/document/product/267/39605?has_map=1&lang=en&pg=). Шаблоны записи, привязанные к функции relay, поддерживают только шаблоны, которые записывают исходный контент потока. Для изменения шаблона записи во время задачи relay необходимо остановить текущую задачу на 30 секунд перед перезапуском, только после этого новый шаблон записи вступит в силу.
- После привязки relay к шаблону транскодирования поток сначала будет отправлен в CSS для транскодирования, а затем перенаправлен на целевой адрес задачи. Стиль ID потока для отправляемого потока, используемого для транскодирования, — это "pp_relay task id", пример: pp_12345678. Могут быть начислены дополнительные расходы на передачу потока вверх по течению. Для правил тарификации передачи потока вверх по течению см. [Инструкции по тарификации передачи потока вверх по течению](https://www.tencentcloud.com/document/product/267/2818#cfac37fd-ceab-4431-bcc1-cdd283719c91).

## Создание задачи

1. Перейдите на страницу [Relay](https://console.tencentcloud.com/live/tools/relay) и нажмите **Create Task**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/58d2d3d118e111f0b04252540044a08e.png)

2. Введите основную информацию:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8f05415818e211f0abe05254005ef0f7.png)

| Элемент конфигурации | Описание |
| --- | --- |
| Task Description | Описание задачи. |
| Execution Time | По умолчанию установлено значение от `Current Time` до `Current Time + 24 Hours`. Допустимый диапазон времени выполнения — любое время в течение года от текущего момента, но продолжительность не может превышать 30 дней. Предположим, текущее время — 11:34:28 14 апреля 2025 г., тогда: допустимое время — с 11:34:28 14 апреля 2025 г. по 11:34:28 14 мая 2025 г. Время окончания не может превышать 11:34:28 14 мая 2025 г. |
| Event Callback Notification | Введите URL обратного вызова для получения уведомлений о событиях relay. |

3. Предоставьте информацию об источнике.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/beaa8ddebdfd11ef928f525400d5f8ef.png)

  - Region: Random (китайский материк), North China (Пекин), East China (Шанхай), South China (Гуанчжоу), Southeast Asia (Сингапур), Southeast Asia (Бангкок), Northeast Asia (Токио), Northeast Asia (Сеул), Hong Kong/Macao/Taiwan (Китай) Hong Kong (Китай), West US (Кремниевая долина), East US (Вирджиния), Europe (Франкфурт).
  - Если вы выберите **Random (китайский материк)**, система назначит ближайший регион.
4. Для **Content Type** вы можете выбрать **Live streaming**, **Custom video path** или **Image**.
  4.1. **Live streaming**:
  - Введите URL прямой трансляции (**только один** допускается).
  - Вы можете выбрать **Enable backup**.
    - В случае отказа основного источника входного потока произойдет автоматическое переключение на резервный источник входного потока для получения контента. Если тип контента резервного источника входного потока — прямая трансляция, требуется ручное переключение обратно на основной источник после его восстановления. Однако если тип контента резервного источника — пользовательское видео, система автоматически вернется к основному источнику после завершения текущего цикла воспроизведения пользовательского видео. Резервный источник поддерживает только непрерывное циклическое воспроизведение одного видео.
    - Сценарии использования: функция резервного источника идеальна для долгосрочных задач прямой трансляции, предотвращающих черный экран при прерывании потока и обеспечивающих резервные источники и дополнение. Для ручного переключения в реальном времени в сценариях прямой трансляции рекомендуется использовать функцию облачной станции режиссера.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/cf031a1ebdfd11ef96d352540075b605.png)

  4.2. **Custom video path**:
  - Вы можете ввести **несколько** (максимум 30) URL источников.
  - Выберите **Repeat** для неопределенного повторного воспроизведения или **Specified** для указания количества воспроизведений (1-100).
  - Если вы включите локальный режим, источники в формате MP4 будут кэшированы на **локальном узле** перед пересылкой. Это обеспечивает более плавное и надежное воспроизведение.

> **Примечание:**Система остановит задачу relay либо когда количество воспроизведений достигнет указанного значения, либо когда задача достигнет времени окончания. В случае изменения задачи: если вы измените только количество воспроизведений, после применения нового значения счетчик начнет с 2. Если вы измените как URL источника, так и количество воспроизведений, после того как новая конфигурация вступит в силу (сразу или после завершения текущего воспроизведения), счетчик начнет с 1. Если вы измените целевой URL, количество воспроизведений будет сброшено. Пересылка локально кэшированного файла MP4 повлечет дополнительные расходы, которые рассчитываются на основе продолжительности пересылаемого файла.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/eb6c6cb5bdfd11ef96d352540075b605.png)

  4.3. **Image**:
  - Загрузите изображение или введите URL изображения. Вы можете нажать **Preview** для предпросмотра изображения.
  - Поддерживаются изображения в формате JPEG, JPG, PNG или BMP. Если вы вводите URL изображения, размер изображения не ограничен. Если вы загружаете изображение, оно не должно превышать 2 МБ.
  - Имена файлов загружаемых изображений поддерживают только буквы, цифры и следующие символы: `- ! _ . *`

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e572fd06190a11ef942e525400720cb5.png)

5. Вы можете выбрать **Watermark Configuration**, **Transcoding Configuration** и **Recording Configuration**. Методы конфигурации следующие:
  5.1. **Watermark Configuration:**

Нажмите

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8a028c82779e11ee8fb05254000e2caf.png)

для включения конфигурации водяного знака. Поддерживаются форматы изображений водяного знака PNG, JPG и GIF.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e426e8d4bdff11efb86b525400f69702.png)

    - Установите **Watermark Type**. Вы можете выбрать **Custom watermark URL** и **Upload image**.
    - Для оптимального визуального эффекта водяной знак должен быть прозрачным изображением PNG размером менее 2 МБ.
    - Имена файлов загружаемых изображений поддерживают только буквы, цифры и следующие символы: `- ! _ . *`

Custom watermark URL

Upload image

Выберите **Custom watermark URL**, введите URL изображения водяного знака в поле ввода адреса изображения. Нажав на **Preview**, вы сможете просмотреть водяной знак в разделе предпросмотра.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/4c414d58190b11ef8a48525400762795.png)

Выберите **Upload image**, нажмите **Select Image** для загрузки изображения водяного знака. Размер изображения водяного знака поддерживает растяжение на весь размер окна.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/038ed0e5be0011ef934f52540055f650.png)

    - Настройте размер окна предпросмотра для изображений водяного знака:
      - Размеры по умолчанию: ширина 1920px, высота 1080px.
      - Диапазон размеров: от 360px до 4096px.
      - Вы можете нажать кнопку **Update** справа для выполнения автоматической проверки и синхронизации обновления окна предпросмотра водяного знака.
    - Установите позицию отображения изображения водяного знака следующими способами:
      - Перетащите позицию изображения на панели конфигурации водяного знака.
      - Установите позицию отображения в направлениях осей X и Y.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/28b2854ebe0111efa28d525400329841.png)

> **Примечание:** Введите исходную информацию контента. Когда тип контента — изображение, конфигурация водяного знака не может быть включена. Включение функции водяного знака будет генерировать [расходы на транскодирование](https://www.tencentcloud.com/document/product/267/39604?lang=en&pg=). При изменении водяного знака для задач с источником прямой трансляции изменения вступают в силу немедленно, а для задач с источником видео по запросу изменения вступают в силу начиная со следующего файла. Изменение водяного знака может привести к задержкам при воспроизведении. Сценарий использования: рекомендуется использовать при пересылке на сервер-источник третьей стороны, который не имеет функции водяного знака. Например, если вы пересылаете на CSS, вы можете использовать функцию живого водяного знака CSS.

  5.2. **Transcoding Configuration**:
    - Нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/e217948b190b11efa1975254005ac0ca.png), выберите для включения конфигурации транскодирования, выберите шаблон транскодирования и нажмите **Confirm**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/4f9ad3e1be0111efae1f525400bdab9d.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6d2a532bbe0111efad135254002693fd.png)

> **Примечание:** Включение транскодирования будет генерировать [расходы на живое транскодирование](https://www.tencentcloud.com/document/product/267/39604). После привязки relay к шаблону транскодирования поток сначала будет отправлен в CSS для транскодирования, а затем перенаправлен на целевой адрес задачи. Стиль ID потока для отправляемого потока, используемого для транскодирования, — это "pp_relay task id", пример: pp_12345678. Могут быть начислены дополнительные расходы на передачу потока вверх по течению. Для правил тарификации передачи потока вверх по течению см. [Инструкции по тарификации передачи потока вверх по течению](https://www.tencentcloud.com/document/product/267/2818#cfac37fd-ceab-4431-bcc1-cdd283719c91). Включение, отключение или изменение шаблона транскодирования вступает в силу немедленно. Модификации текущего привязанного шаблона транскодирования в задаче получения вступают в силу только после перезапуска задачи.

  5.3. **Recording Configuration**
    - Нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/5ad725d277b911eeb3675254006b5335.png) для включения конфигурации записи, выберите шаблон конфигурации записи и нажмите **Confirm**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/84eabbe3be0111ef934f52540055f650.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a4d010c5be0111efb0b5525400d5f8ef.png)

    - Выберите режим записи. Вы можете выбрать **Record and relay** или **Record only**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c8b4760fbe0111efae1f525400bdab9d.png)

> **Примечание:** Включение записи будет генерировать [расходы на запись](https://www.tencentcloud.com/document/product/267/39605). Шаблоны записи, привязанные к функции relay, поддерживают только шаблоны, которые записывают исходный контент потока. Шаблоны для потоков с водяными знаками и потоков транскодирования не поддерживаются. Для изменения шаблона записи во время задачи relay необходимо остановить текущую задачу на 30 секунд перед перезапуском, только после этого новый шаблон записи вступит в силу. Если шаблон записи уже был связан, он должен быть сначала отсоединен перед удалением. Для процедур отсоединения см. раздел [отсоединение конфигурации записи](https://www.tencentcloud.com/document/product/267/34224).

6. Введите целевой URL.
  6.1. Нажмите **Address Generator** для входа на страницу генерирования URL.
    - Нажмите **Add** для добавления целевого адреса 2.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/438702c0190c11efac7f5254007bbd8c.png)

  6.2. Выберите существующий домен отправки, введите `Appname`, `StreamName` и время истечения срока действия, затем нажмите **Confirm** для генерирования URL отправки, который будет автоматически заполнен как **Destination Address**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e041ccffbe0111efad135254002693fd.png)

> **Примечание:** Время истечения срока действия URL должно быть позже времени окончания задачи. Если вы измените целевой URL после начала задачи, она остановится и перезапустится.

7. После завершения всех деталей конфигурации просто нажмите **Save**.

## Управление задачами

### Просмотр деталей задачи

В [списке задач](https://console.tencentcloud.com/live/tools/relay) найдите вашу задачу и нажмите на ее описание/ID, чтобы просмотреть детали задачи во всплывающем окне.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7149abdebe0211efb0b5525400d5f8ef.png)

> **Примечание:** Вы можете нажать кнопки в нижней части всплывающего окна для **редактирования задачи**, **переключения источников**, **перезапуска задачи** или **отключения задачи**.

### Просмотр статуса задачи

В [списке задач](https://console.tencentcloud.com/live/tools/relay) найдите вашу задачу и нажмите на ее описание/ID, чтобы просмотреть статус выполнения во всплывающем окне.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a5133bbabe0211efbb7252540075b605.png)

| Статус задачи | Значение поля | Описание |
| --- | --- | --- |
| Not started | Inactive | Задача еще не начата. |
| Valid | Active | Задача начата и выполняется как ожидается. |
|  | Inactive | Задача начата, но не выполняется как ожидается. |
| Disabled | Inactive | Задача отключена. |
| Expired | Inactive | Срок действия задачи истек. |

### Запрос данных потока

В списке задач выберите успешно созданную задачу relay и нажмите **Streaming data** в столбце **Operation** справа, чтобы запросить данные потока задачи relay. Информация об отображении события включает тип события, тип контента, время, исходный IP, целевой IP и детальную информацию.

### ![](https://staticintl.cloudcachetci.com/cms/backend-cms/dcdc6ee0be0211efbb7252540075b605.png)

- Страница запроса данных потока задачи relay позволяет запрашивать данные одного потока за последние семь дней, включая частоту кадров видео, битрейт видео, частоту кадров аудио и битрейт аудио.
- Интервал между временем начала и окончания запроса не должен превышать 3 часа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3139eb71be0311efb86b525400f69702.png)

### Изменение задачи

1. В [списке задач](https://console.tencentcloud.com/live/tools/relay) найдите задачу, которую вы хотите изменить, и нажмите **Edit**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/06aed0cfbe0311ef8fa3525400fdb830.png)

2. Измените информацию задачи, затем нажмите **Save**.
  - Вы не можете изменить регион или тип контента.
  - При изменении времени окончания задачи убедитесь, что целевой URL остается действительным до окончания задачи. Изменение целевого URL приведет к остановке и перезапуску задачи.
  - Если вы измените параметры водяного знака для источника прямой трансляции, изменения вступят в силу немедленно. Для источника видео по запросу изменения параметров водяного знака вступят в силу начиная со следующего видео. Изменение параметров водяного знака может привести к задержкам при воспроизведении. Мы рекомендуем использовать функцию водяного знака для relay только если вы пересылаете на сайты третьих сторон, которые не имеют возможности добавления водяных знаков. Если вы пересылаете на CSS, вы можете использовать функцию живого водяного знака CSS.
  - Шаблоны записи, привязанные к функции relay, поддерживают только шаблоны, которые записывают исходный контент потока. Для изменения шаблона записи во время задачи relay необходимо остановить текущую задачу на 30 секунд перед перезапуском, только после этого новый шаблон записи вступит в силу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b2efb278190c11ef8bfe5254002fd0a8.png)

3. Во всплывающем окне проверьте информацию:
  - Предположим, вы изменили **время начала, время окончания и количество воспроизведений** задачи. Вы увидите следующую информацию во всплывающем окне:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e0a1eb25be0311efad135254002693fd.png)

  - Если URL источников для **Custom video path** изменены, вам нужно выбрать, применять ли изменение **After the current video ends** (по умолчанию) или **Now**. После того как изменения вступят в силу, relay перезапустится с первого URL источника.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8ea615c62a0911eea27e525400c56988.png)

  - Если **Destination Address** изменен, система напомнит вам, что после нажатия **Confirm** текущая задача relay **остановится и перезапустится**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8ead0af22a0911ee909c525400cea498.png)

4. После проверки нажмите **Confirm**.

### Копирование задачи

1. В [списке задач](https://console.tencentcloud.com/live/tools/relay) найдите задачу relay, которую вы хотите скопировать, и нажмите **Copy**. Вы будете перенаправлены на страницу создания задачи.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/175a7f74be0411efbb7252540075b605.png)

2. Информация скопированной задачи будет автоматически заполнена. Вы можете **изменить** ее по мере необходимости.
3. Нажмите **Save** для создания новой задачи relay.

### Перезапуск задачи

Перезапуск задачи **не изменит ее статус**. Текущая задача будет **перезапущена с начала**. Выполните следующее для перезапуска задачи:

1. В [списке задач](https://console.tencentcloud.com/live/tools/relay) найдите задачу relay, которую вы хотите перезапустить, и нажмите **Restart**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3eef5d99be0411efa28d525400329841.png)

2. Во всплывающем окне нажмите **Restart**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/4f7528fabe0411efa28d525400329841.png)

### Отключение задачи

Если вы отключите задачу, **задача остановится**. Вы можете нажать **Enable** для ее перезапуска. Выполните следующее для отключения задачи:

1. В [списке задач](https://console.tencentcloud.com/live/tools/relay) найдите задачу relay, которую вы хотите отключить, и нажмите **Disable**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7922398ebe0411efae1f525400bdab9d.png)

2. Во всплывающем окне нажмите **Disable**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/86bff3e6be0411efb0b5525400d5f8ef.png)

### Включение задачи

Если вы включите задачу, **задача начнется с начала**. Выполните следующее для включения задачи:

1. В [списке задач](https://console.tencentcloud.com/live/tools/relay) найдите задачу relay, которую вы хотите включить, и нажмите **Enable**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b0cbc8cfbe0411efb0b5525400d5f8ef.png)

2. Во всплывающем окне нажмите **OK**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c0807dc8be0411efbb7252540075b605.png)

### Удаление задачи

Удаленные задачи **не могут быть восстановлены**. Выполните следующее для удаления задачи:

1. В [списке задач](https://console.tencentcloud.com/live/tools/relay) найдите задачу relay, которую вы хотите удалить, и нажмите **Delete**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e81e4e96be0411efb86b525400f69702.png)

2. Во всплывающем окне нажмите **Delete**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f941c07dbe0411efae1f525400bdab9d.png)

### Массовые операции

# Генератор адресов

Консоль CSS предоставляет генератор адресов, который можно использовать для быстрого создания URL-адресов отправки/воспроизведения. Основные части URL-адреса прямой трансляции включают имя домена (`domain`), имя приложения (`AppName`), имя потока (`StreamName`) и ключ аутентификации (`Key`).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8ead88913e7511eeacb1525400cea498.jpeg)

После создания URL-адресов вы можете **выбрать и скопировать** нужный или скопировать его **нажав на значок копирования**. Вы также можете получить URL-адрес **отсканировав QR-код**.

## Поддержка и ограничения

- Если вам нужно создать несколько URL-адресов прямой трансляции, рекомендуется объединить их в соответствии с инструкциями [Объединение URL-адресов прямой трансляции](https://intl.cloud.tencent.com/document/product/267/38393).
- CSS предоставляет тестовое имя домена `xxxx.livepush.myqcloud.com`. Вы можете использовать его для тестирования отправки, но мы не рекомендуем использовать его в коммерческих целях.
- При воспроизведении транскодированного потока StreamName в адресе воспроизведения должен быть дополнен суффиксом "_TranscodingTemplateName" для работы. Рекомендуется не включать "_" в StreamName. Если строка после "_" совпадает с именем шаблона транскодирования, часть после "_" будет распознана как имя шаблона транскодирования, что может привести к отказу при извлечении потока.
- Вы можете получить URL-адрес, созданный путем сканирования QR-кода с помощью [TCToolkit](https://intl.cloud.tencent.com/document/product/1071/38147).
- Недавние записи генератора адресов хранятся в локальном кэше браузера. Очистка кэша также удалит эти записи.

## Предварительные условия

Вы вошли в [консоль CSS](https://console.tencentcloud.com/live/livestat) и добавили [имя домена отправки/воспроизведения](https://intl.cloud.tencent.com/document/product/267/35970).

## Описание параметров

| Параметр | Описание |
| --- | --- |
| Тип URL | Вы можете выбрать один из следующих трех типов адресов для конфигурации: Адрес отправки, Адрес воспроизведения, URL-адреса отправки и воспроизведения |
| Выбрать имя домена | Вы можете выбрать: Имя домена отправки, Имя домена воспроизведения, Выбрать оба — Имя домена отправки и Имя домена воспроизведения |
| AppName | Имя приложения, которое используется для определения пути файла потока. Значение по умолчанию: `live`. Разрешены только буквы, цифры и символы. |
| StreamName | Пользовательское имя потока — это уникальный идентификатор для каждой прямой трансляции. Поддерживает только английские буквы, цифры и символы. Длина StreamName ограничена 255 символами. |
| Type | Тип шифрования по умолчанию: MD5, альтернативный вариант: SHA256 |
| Время истечения | Фактическое время истечения URL-адреса воспроизведения — это указанное время плюс период действия ключа аутентификации. Время истечения URL-адреса отправки — это указанное время. |
| Шаблон транскодирования | Этот параметр используется только при выборе типа адреса как Адрес воспроизведения и выборе имени домена как Имя домена воспроизведения. Если вы выберете [Шаблон транскодирования](https://www.tencentcloud.com/document/product/267/31071), созданный адрес воспроизведения будет для транскодированного потока прямой трансляции. Если вам нужно воспроизвести исходный поток прямой трансляции, нет необходимости выбирать шаблон транскодирования для создания адреса. Если вы выберете [Шаблон адаптивной битовой скорости](https://www.tencentcloud.com/document/product/267/50271), созданный адрес воспроизведения будет для потока прямой трансляции с адаптивной битовой скоростью. Если вам нужно воспроизвести исходный поток прямой трансляции, нет необходимости выбирать шаблон транскодирования для создания адреса. |

## Создание URL-адресов отправки

### Инструкции

1. Войдите в консоль CSS и выберите [Генератор адресов](https://console.tencentcloud.com/live/addrgenerator/addrgenerator) на левой боковой панели.
2. Выберите тип адреса как **Адрес отправки**.
3. Выберите имя домена отправки, которое вы добавили в Управление доменами.
4. Введите **AppName**. По умолчанию это `live`.
5. Введите **StreamName**, например `liveteststream`.
6. Выберите ваш **Тип шифрования**.
7. Выберите время истечения URL-адреса, например `2024-10-01 14:55:52`.
8. Нажмите **Создать адрес**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/25b75d8d7efa11efb9d8525400f69702.png)

### Формат URL-адреса отправки

CSS поддерживает RTMP, WebRTC, SRT, RTMP over SRT и RTMP over QUIC для отправки. В результате созданные URL-адреса отправки начинаются с `rtmp://`, `webrtc://`, `srt://` или `rtmp://`.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/75acc1a57efa11efa87e52540055f650.png)

## Создание URL-адресов воспроизведения

### Инструкции

2. Выберите тип адреса как **Адрес воспроизведения** и выберите имя домена воспроизведения, которое вы добавили в Управление доменами.
3. Введите **AppName**. По умолчанию это `live`.
4. Введите **StreamName**, например `liveteststream`.
5. Выберите ваш **Тип шифрования**.
6. Выберите время истечения URL-адреса, например `2024-10-01 14:55:52`.
7. Выберите существующий шаблон транскодирования (опционально).
8. Нажмите **Создать адрес**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c9b15f497efa11efb9d8525400f69702.png)

### Формат URL-адреса воспроизведения

Если используется шаблон транскодирования, созданный адрес воспроизведения будет для транскодированного потока прямой трансляции. Воспроизведение поддерживает протоколы RTMP, FLV, HLS и WebRTC. Вы можете создавать адреса воспроизведения с префиксами, такими как `rtmp://`, `http://` и `webrtc://`, используя генератор адресов. По сравнению с исходным потоком прямой трансляции необходимо добавить "_TranscodingTemplateName" после StreamName.

> **Примечание:** URL-адреса воспроизведения UDP предназначены для [LEB](https://intl.cloud.tencent.com/document/product/267/41030). Чтобы узнать о выставлении счетов за LEB, см. [Обзор выставления счетов](https://intl.cloud.tencent.com/document/product/267/2819).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/dbb582287efa11efa87e52540055f650.png)

## Создание группы URL-адресов отправки и воспроизведения

### Инструкции

1. Войдите в консоль CSS и выберите [Генератор адресов](https://console.tencentcloud.com/live/addrgenerator/addrgenerator) на левой боковой панели.
2. Выберите тип адреса как **URL-адреса отправки и воспроизведения**.
3. Выберите **Имя домена отправки** и **Имя домена воспроизведения**, которые вы добавили в Управление доменами.
4. Введите **AppName**. По умолчанию это `live`.
5. Введите StreamName, например `liveteststream`.
6. Выберите ваш **Тип шифрования**.
7. Выберите время истечения URL-адреса, например `2024-10-01 14:55:52`.
8. Выберите существующий шаблон транскодирования (опционально).
9. Нажмите **Создать адрес**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f320aac37efa11ef82535254002693fd.png)

### Формат URL адаптивной битовой скорости

- Если вы используете Шаблон адаптивной битовой скорости, созданный адрес воспроизведения будет для воспроизведения с адаптивной битовой скоростью.
- Этот адрес воспроизведения поддерживает протоколы HLS и WebRTC. Вы можете создавать адреса воспроизведения с префиксами, такими как `http://` и `webrtc://`, используя генератор адресов.
- Для адресов получения адаптивной битовой скорости с использованием протокола HLS метод обработки аналогичен методу для обычных транскодированных адресов.
- Для адресов получения адаптивной битовой скорости с использованием протокола WebRTC нет необходимости добавлять имя шаблона транскодирования после StreamName. Вместо этого необходимо добавить `"&tabr_bitrates=AdaptiveBitrateSubTemplateNameList&tabr_start_bitrate=StartingPlaybackBitrateSubTemplateName&tabr_control=auto"` после исходного URL потока прямой трансляции`. Имена подшаблонов адаптивной битовой скорости в списке должны быть расположены в порядке убывания битовой скорости, разделены запятыми.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a65cea683e8e11ee96d3525400088f3a.png)

## Объяснение недавних записей

В соответствии с вашими потребностями вы можете просматривать или удалять записи недавно созданных адресов. Эти записи включают адреса отправки, адреса воспроизведения, группы адресов и другую информацию.

### Просмотр записей

1. Войдите в консоль CSS и выберите [Генератор адресов](https://console.tencentcloud.com/live/addrgenerator/addrgenerator) на левой боковой панели. Нажмите **История** для входа на страницу управления недавними записями.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/62b3beb17efb11efa87a525400bdab9d.png)

2. Выберите информацию о записи адреса прямой трансляции, которую вы хотите просмотреть, и нажмите **Просмотр**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/bdf31cf67efb11ef80ff525400d5f8ef.png)

### Удаление записи адреса прямой трансляции

Если вам нужно удалить запись адреса прямой трансляции, сначала выберите тип адреса, который вы хотите удалить, а затем нажмите **Удалить** на правой стороне страницы. Далее вы будете перенаправлены в окно подтверждения удаления; нажмите **Подтвердить** еще раз для удаления соответствующей записи адреса прямой трансляции.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d763190c7efb11efb9d8525400f69702.png)

> **Примечание:** URL-адреса автоматически сохраняются в кэш браузера и будут удалены при очистке кэша![](https://staticintl.cloudcachetci.com/cms/backend-cms/0e010b683e9911eeacb1525400cea498.png)

# Совместимость с Apple ATS

Apple Inc. объявила на WWDC 2016, что по умолчанию все новые приложения, отправленные начиная с 1 января 2017 года, не смогут использовать `NSAllowsArbitraryLoads=YES` для обхода ограничений ATS. Tencent Cloud официально поддерживает HTTPS начиная с 12 декабря. К тому времени вам просто нужно использовать новую версию SDK (API остается прежним) и изменить префикс видео-URL с `http://` на `https://`. Новый SDK может автоматически адаптироваться к этому изменению.

Обратите внимание, что по сравнению с HTTP, HTTPS обеспечивает более высокий уровень безопасности, который не является абсолютно необходимым для видео, но приводит к снижению скорости соединения и увеличению использования CPU. Если ваше приложение по-прежнему нуждается в использовании HTTP в соответствии с новой политикой Apple, вам необходимо изменить Info.plist, добавив `myqcloud.com` в `NSExceptionDomains`, как показано ниже.

![myqcloud](http://qzonestyle.gtimg.cn/qzone/vas/opensns/res/img/myqcloud.png)

Отключение ATS для конкретных доменов может быть одобрено командой проверки Apple, но вам может потребоваться указать, что `myqcloud.com` является доменом для воспроизведения видео.

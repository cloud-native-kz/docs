# Конфигурация домена

### Как добавить доменное имя в CSS?

1. Войдите в [CSS Console](https://console.tencentcloud.com/live) и перейдите в **Domain Management**.
2. Добавьте свое доменное имя для публикации или воспроизведения. Дополнительную информацию см. в разделе [Adding Domain Name](https://intl.cloud.tencent.com/document/product/267/35970).
3. Сконфигурируйте CNAME. Дополнительную информацию см. в разделе [CNAME Configuration](https://intl.cloud.tencent.com/document/product/267/31057).
4. После успешной конфигурации вы можете публиковать и воспроизводить содержимое со своим доменным именем.

### Почему CNAME по-прежнему отображается как некфигурированный после конфигурации?

После конфигурации CNAME в соответствии с инструкциями в разделе [CNAME Configuration](https://intl.cloud.tencent.com/document/product/267/31057) подождите, так как вступление конфигурации в силу занимает 15–30 минут. Кроме того, вы можете проверить успешность конфигурации CNAME, выполнив действия, описанные в разделе [Verifying the Effect of CNAME Record](https://intl.cloud.tencent.com/document/product/267/31057).

> **Примечание:** DNS-разрешение должно выполняться через общественную сеть на Linux/macOS/Windows. Если проблема с конфигурацией CNAME сохраняется, обратитесь к регистратору вашего доменного имени.

### Что делать, если я не добавлю свое доменное имя?

Если вы активировали услугу CSS после 17 октября 2018 г., вам необходимо добавить свое доменное имя для воспроизведения; в противном случае вы не сможете воспроизводить содержимое трансляции.

Если вы активировали услугу до этого, CSS предоставил вам доменное имя по умолчанию, но мы рекомендуем заменить его на свое собственное. Tencent Cloud начала поэтапное отключение доменных имен по умолчанию с 31 декабря 2018 г.

> **Примечание:** Доменные имена по умолчанию — это системные доменные имена, назначенные CSS в формате `bizid.liveplay.com` и `bizid.tlivecdn.com`.

### Я сконфигурировал специальные элементы для доменного имени по умолчанию. Может ли мое доменное имя разрешаться по умолчанию?

Чтобы использовать новое доменное имя, подключите его к CSS с нуля. Рекомендуется добавить и сконфигурировать свое доменное имя в CSS Console.

### Требуется ли добавление собственного зарегистрированного доменного имени для транслирования с передачей в интерактивной трансляции?

Если вы используете услугу транслирования с передачей, вам необходимо добавить свое собственное зарегистрированное доменное имя для нижестоящего воспроизведения транслирования с передачей.

### Какой формат сертификата следует заполнить для конфигурации HTTPS для трансляции?

Трансляция в настоящее время поддерживает доступ только к сертификатам в формате PEM. Если ваш сертификат в другом формате, преобразуйте его в формат PEM.

### Как определить, находится ли сертификат в формате PEM или DER?

- **Формат PEM**: текст начинается с `-----BEGIN CERTIFICATE-----` и заканчивается на `-----END CERTIFICATE-----`, содержимое представляет собой файл ASCII-кода, кодированный в Base64. Типичные расширения: `.pem, .crt, .cer, и .key.`

> **Примечание:** Серверы Apache и Nginx предпочитают этот формат кодирования.

- **Формат DER:** файл находится в двоичном формате и нечитаем. Типичные расширения: `.der` и `.cer`.

> **Примечание:** Серверы Java и Windows предпочитают этот формат кодирования.

### Какой период действия трансляции Tencent Cloud?

Аутентификация воспроизведения позволяет сконфигурировать параметр времени действия, который в основном используется, когда адрес воспроизведения распределения был предоставлен платформе, но время окончания конференции может измениться. В этом случае, чтобы обеспечить отсутствие влияния на воспроизведение нижестоящим потоком, период действия аутентификации доменного имени воспроизведения можно изменить для достижения отложенного времени истечения адреса.

Период действия по умолчанию — 0. В этом случае время истечения — это время, указанное txTime. Фактическое время окончания — это значение txTime плюс период действия аутентификации.

# Глобальный сервис CSS

### Как настроить ускорение прямой трансляции вне материковой части Китая?

Перейдите в консоль CSS > [Управление доменами](https://console.tencentcloud.com/live/domainmanage), где по умолчанию предварительно настроен домен прямой трансляции с ускорением вне материковой части Китая. Если вы хотите использовать свой домен прямой трансляции, добавьте его и выполните операции, связанные с CNAME, в соответствии с инструкциями, чтобы настроить ускорение вне материковой части Китая для него.

### Как ускорить прямую трансляцию для домена воспроизведения вне материковой части Китая?

Перейдите в консоль CSS > [Управление доменами](https://console.tencentcloud.com/live/domainmanage) > Добавить домен воспроизведения, где вы можете выбрать один из двух типов ускорения в соответствии с вашими потребностями: **Глобальное ускорение**, которое требует, чтобы домен получил регистрацию ICP на материковой части Китая; в противном случае конфигурация не будет выполнена; **Ускорение в Гонконге/Макао/Тайване (регион Китая) и других регионах**, которое можно настроить в соответствии с инструкциями, но воспроизведение будет недоступно на материковой части Китая в этом случае.

### Как проверить, было ли настроено ускорение вне материковой части Китая для моего домена?

Вы можете использовать сервер вне материковой части Китая для выполнения команды `ping` на доменах прямой трансляции и воспроизведения; если IP-адрес сервера — это IP-адрес ближайшего узла, то ускорение вне материковой части Китая настроено. Вы также можете использовать [этот инструмент](https://tools.ipip.net/dns.php) для тестирования вашего домена и проверки соответствия разрешённого IP-адреса с регионом ускорения.

### Как устранить проблемы с отказом при воспроизведении прямой трансляции вне материковой части Китая?

В настоящее время воспроизведение вне материковой части Китая поддерживает HTTP-FLV, HLS и RTMP. Вы можете устранить проблемы воспроизведения следующим образом:

1. **Доступен ли домен для ping?**
Если нет, проверьте текущую сетевую среду.
2. **Получен ли код состояния HTTP 200?**
Если это не 200, существует несколько возможных причин отказа. Как правило, если код состояния — 403, что указывает на ошибку аутентификации, вам необходимо проверить, соответствует ли формат расчёта защиты от hotlink требованиям. Если код состояния — 404, что указывает на то, что воспроизводимый поток отсутствует на платформе, вам необходимо проверить, работает ли прямая трансляция нормально. Для других кодов ошибок [отправьте заявку](https://console.tencentcloud.com/workorder/category) для получения помощи.

### Какие протоколы поддерживает прямая трансляция вне материковой части Китая?

В настоящее время поддерживается только RTMP.

### Поддерживает ли воспроизведение прямой трансляции вне материковой части Китая HTTPS?

Да. Перейдите в консоль CSS > [Управление доменами](https://console.tencentcloud.com/live/domainmanage) > Домен воспроизведения > Управление > Расширенная конфигурация > Конфигурация HTTPS, где вы можете добавить HTTPS-сертификат к домену, для которого вы хотите настроить загрузку HTTPS.

### Могу ли я изменить регион ускорения воспроизведения прямой трансляции вне материковой части Китая?

Да. Вы можете сделать это в консоли CSS > Управление доменами. Применение изменения займёт примерно 15 минут.

### Поддерживает ли ускорение прямой трансляции вне материковой части Китая маршрутизацию HttpDNS?

Да, но вам необходимо [отправить заявку](https://console.tencentcloud.com/workorder/category) для настройки на сервере.

# Прямая запись

### Как работает прямая запись?

![img](https://staticintl.cloudcachetci.com/cms/backend-cms/14f97f0152f311ee974d5254005f490f.jpeg)

После включения записи для прямой трансляции каждый аудио/видео кадр, опубликованный с мобильного телефона хоста, будет передан в систему записи и записан в файл записи.

Когда прямая трансляция прерывается, уровень доступа немедленно уведомляет сервер записи о прекращении записи в файл, передает его в систему Video on Demand (VOD) или Cloud Object Storage (COS) и генерирует для него индекс. Затем вы сможете увидеть только что созданный файл записи в системе VOD или COS. Одновременно, если вы настроили уведомления о событиях записи, система записи уведомит ранее настроенный сервер об **ID индекса** файла и **URL-адресе воспроизведения в онлайне**.

### Почему я не могу использовать функцию прямой записи?

Функция прямой записи зависит от [сервиса VOD](https://console.intl.cloud.tencent.com/vod/overview) или [сервиса COS](https://console.intl.cloud.tencent.com/cos) Tencent Cloud. Если вам необходимо использовать функцию записи, убедитесь, что сервис VOD или COS работает нормально. Невозможность включить сервис VOD или COS либо его приостановка из-за задолженности по счету приведет к невозможности записи прямых трансляций, и во время этого периода не будут создаваться файлы записи или начисляться плата за запись.

### Когда файл записи будет готов после завершения прямой трансляции?

Файл записи будет готов, когда вы получите обратный вызов записи (примерно через 5 минут после завершения прямой трансляции). Дополнительные сведения см. в разделе [Обратный вызов прямой трансляции](https://intl.cloud.tencent.com/document/product/267/31074).

### Как получить файлы записи?

После создания файла записи он будет сохранен на платформе VOD или в Cloud Object Storage (COS). Клиентам необходимо активировать [сервис VOD](https://console.tencentcloud.com/vod/overview) и [Cloud Object Storage](https://www.tencentcloud.com/products/cos?lang=en&pg=) для успешного сохранения. Вы можете получить записанный файл следующими способами:

- Сохранено на платформе Tencent Cloud Video on Demand (VOD)
  - [Консоль VOD](https://intl.cloud.tencent.com/document/product/267/31563)
  - [Запись в VOD](https://intl.cloud.tencent.com/document/product/267/31563)
  - [Запрос через API VOD](https://intl.cloud.tencent.com/document/product/267/31563)
- Сохранено в Cloud Object Storage (COS)
  - [Консоль Cloud Object Storage (COS)](https://console.tencentcloud.com/cos)
  - [Запись в COS](https://www.tencentcloud.com/document/product/267/57044#)
  - [Запрос через API COS](https://www.tencentcloud.com/document/product/267/57044#)

### Могу ли я перенести свои видео?

Для переноса видео необходимо получить адреса загрузки и загрузить видео.

### Как установить период хранения видео?

CSS не устанавливает ограничение на период хранения видео. Вы можете управлять файлами видео в консоли или через API-интерфейсы RESTful.

### Сколько файлов записи создается для одной прямой трансляции?

- **Запись в формате HLS**: нет верхнего предела для продолжительности файлов записи. Однако, если прямая трансляция прерывается и истекает период ожидания возобновления с точки разрыва (который можно установить на 0s до 1800s), будет создан новый файл записи. Если включена сегментация файлов HLS, продолжительность одного файла записи HLS может составлять от 1 до 720 минут.
- **Запись в формате FLV**: продолжительность одного файла ограничена от 1 до 720 минут.
- **Запись в формате MP4 или AAC**: продолжительность одного файла ограничена от 1 до 120 минут.
- **Запись в формате MP4, FLV или AAC**: продолжительность одного файла ограничена от 1 до 120 минут.
- Вы можете установить более короткую максимальную продолжительность, задав параметр RecordInterval в API [CreateLiveRecordTemplate](https://intl.cloud.tencent.com/document/product/267/30845).
  - Если прямая трансляция настолько коротка, что завершается до запуска модуля записи, файлы записи не будут созданы.
  - Если прямая трансляция короче `RecordInterval` и не прерывается, будет создан только один файл записи.
  - Если прямая трансляция длится дольше `RecordInterval`, поток будет записан в несколько файлов. Это сделано для уменьшения неопределенности времени передачи файла в распределенной системе.
  - Если прямая трансляция прерывается и возобновляется, новый файл записи будет создан каждый раз при каждом прерывании.

### Как узнать, какой прямой трансляции принадлежит файл записи?

В большинстве сценариев мобильного прямого вещания период от момента начала прямого вещания хостом до его завершения рассматривается как одна прямая трансляция.

Если вы используете приведенный выше стандарт, чтобы определить, какой прямой трансляции принадлежит файл записи, просто выполните поиск уведомлений о записи по коду потока и времени. Каждое уведомление о записи содержит информацию, включая **ID потока**, **время начала** и **время окончания**.

### Как мне объединить файлы записи прямой трансляции?

Вы можете использовать облачный API для объединения файлов записи.

### Почему при настройке только одного шаблона записи существует два канала записи?

1. Проверьте в консоли, выбрали ли вы два формата записи.
  - Если вы используете новую консоль, выберите [Управление доменом](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели, найдите ваш домен отправки, нажмите **Управление** справа, чтобы перейти на страницу сведений, и выберите **Конфигурация шаблона**, чтобы просмотреть конфигурацию записи.
  - Если вы используете старую консоль, перейдите в [**Доступ к коду прямой трансляции**](https://console.tencentcloud.com/live/livecodemanage) > **Конфигурация доступа**, чтобы просмотреть конфигурацию записи.
2. Убедитесь, что вы одновременно не вызываете API 3.0 [CreateRecordTask](https://intl.cloud.tencent.com/document/product/267/37309) или API 2.0 Live_Tape_Start для запуска задачи записи. Вы можете либо вызвать API [CreateRecordTask](https://intl.cloud.tencent.com/document/product/267/37309), либо [создать шаблон записи](https://intl.cloud.tencent.com/document/product/267/34223) в консоли для запуска записи. Если вы создадите задачу записи и одновременно настроите шаблон записи для прямой трансляции, она будет записана дважды.

> **Примечание:** Если вы включили прямую запись в старой консоли и хотите отключить её в новой консоли, пожалуйста, [отправьте заявку](https://console.tencentcloud.com/workorder/category). Если проблема сохраняется, пожалуйста, [отправьте заявку](https://console.tencentcloud.com/workorder/category).

### Как мне записать только аудио?

- **Только аудио**: `record_type=audio`
- **Видео**: `record_type=video`

Если вы хотите как видео, так и файлы только с аудио, вы можете сначала записать видео, а затем перекодировать видеофайлы в аудиофайлы в VOD.

### Как сохранить свои видео постоянно?

Установите период хранения в шаблоне записи на 0. Подробнее см. в разделе [Прямая запись](https://intl.cloud.tencent.com/document/product/267/34223).

### Может ли CSS автоматически удалять начало и конец файла записи?

- Обрезать записанные видео. Подробнее см. в разделе [EditMedia](https://intl.cloud.tencent.com/document/product/266/34126).
- Отрегулировать прогресс воспроизведения.

# Микширование потоков в облаке

### Что делать, если после отправки потока возвращается код ошибки -505 при микшировании потока?

Подождите примерно 5 секунд после начала отправки потока перед началом микширования потока.

### Как устранить неполадки, если API микширования потока вернул код -505?

Если API микширования потока возвращает -505, это означает, что идентификатор потока не имеет соответствующих данных на серверной части CSS.

1. Вы можете извлечь поток, чтобы проверить, успешно ли он был отправлен. Если извлечение успешно, это означает, что отправка прошла успешно.
2. Если поток можно извлечь, но API возвращает -505, пожалуйста, проверьте, правильный ли `AppID` введен в параметрах микширования потока.

### Что делать, если голос вспомогательного хоста не слышен после микширования потока аудио?

Пожалуйста, проверьте, установлен ли `input_type` аудиопотока в значение 4.

### Что произойдет, если микширование потока не отменить после его применения?

Микширование потока будет продолжаться до получения команды на его отмену.

### Что произойдет, если входящий поток внезапно прервется во время микширования потока?

Если прерывается поток, не являющийся фоновым, его изображение зафиксируется на последнем кадре. Если прерывается фоновый поток, все видеоизображение зафиксируется. Если прерванный поток успешно отправлен снова с тем же идентификатором потока в течение 15 минут, микширование потока возобновится автоматически.

### Как получить результат микширования потока записи?

Подробные инструкции см. в разделе [Создание задачи записи](https://intl.cloud.tencent.com/document/product/267/37309).

### Почему на видеоизображении появляются черные полосы после микширования потока?

Возможные причины:

1. Исходный поток содержит черные полосы.
2. Выходное соотношение сторон в параметрах микширования потока отличается от соотношения сторон исходного потока. Например, если целевое соотношение сторон микширования потока составляет 16:9, а соотношение сторон исходного видео — 4:3, серверная часть микширования потока добавит черные полосы к исходному видео, чтобы изменить соотношение сторон на 16:9 для вывода.

Вы можете предотвратить появление черных полос двумя способами:

1. Используйте соотношение сторон входящего видео для выходящего видео.
2. Используйте параметры обрезки в соответствии с инструкциями функции обрезки.

### Почему видеоизображение вспомогательного хоста в смешанном потоке находится не в ожидаемой позиции?

Обычно это вызвано изменением разрешения входящего потока во время микширования потока. Например, если разрешение входящего потока составляет 1280x720 при применении микширования потока, но затем становится 2560x1440, выходящее видеоизображение изменится и будет в другой позиции.

> **Примечание:** Рекомендуется не изменять разрешение входящего потока во время микширования. При необходимости следует пересчитать параметры позиции и повторно подать заявку на микширование потока.

### Может ли вывод микширования потока кодироваться в H.265?

В настоящее время микширование потока поддерживает только кодирование H.264 для выходящего потока. Даже если входящий поток кодируется в H.265, выходящий поток может кодироваться только в H.264.

### Почему возвращается код ошибки -30300 после двойной отмены микширования потока?

API для отмены микширования потока необходимо вызвать только один раз и не следует вызывать снова после успешной отмены.

### Когда микширование потока будет автоматически отменено после прерывания входящего потока во время микширования потока?

На примере микширования двух потоков: если один поток прерывается, микширование потока не будет отменено автоматически. Если включена запись, запись также будет продолжаться. Если оба потока прерваны, микширование потока будет отменено автоматически через 15 минут.

### Почему видеоизображение немного меняется при вызове микширования потока?

Механизм реализации микширования потока будет максимально согласовывать видеоизображения различных потоков; поэтому во время обработки может произойти небольшое изменение. Чтобы это не повлияло на вашу деятельность, по возможности не вызывайте API микширования потока часто.

### Если хост отключит микрофон во время микширования потока, будет ли макет микширования потока автоматически изменяться?

Нет. Планировщик микширования потока не будет изменять ваши параметры макета. Если хост отключит микрофон, вам потребуется пересчитать параметры макета и повторно инициировать микширование потока.

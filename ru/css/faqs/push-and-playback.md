# Отправка и воспроизведение

### Существует ли верхний предел для количества одновременных зрителей?

По умолчанию CSS не ограничивает количество онлайн-зрителей для прямой трансляции, пока это позволяют сетевые и другие условия. Однако, если вы установили ограничение пропускной способности, новые зрители не смогут смотреть прямую трансляцию, если существующих зрителей так много, что превышено ограничение пропускной способности. В этом случае количество онлайн-зрителей ограничено.

### Как использовать трансляционное транскодирование?

Вы можете перейти на страницу [Live Transcoding](https://console.tencentcloud.com/live/config/transcode) и настроить шаблоны транскодирования с различными битрейтами и разрешениями для разных сетевых условий. Дополнительные сведения о транскодировании см. в разделе [Live Remuxing and Transcoding](https://intl.cloud.tencent.com/document/product/267/31561).

#### Исходное качество, HD и SD

- Для исходного качества битрейт воспроизведения совпадает с битрейтом, используемым при отправке.
- Для HD рекомендуемые битрейт и разрешение воспроизведения составляют 2000 Кбит/с и 1080p.
- Для SD рекомендуемые битрейт и разрешение составляют 1000 Кбит/с и 720p.

### Как использовать функцию сдвига по времени?

Функция сдвига по времени позволяет перемотать видеопоток и воспроизводить его с более ранних моментов времени. В настоящее время функция поддерживает только протокол HLS. Дополнительные сведения о сдвиге по времени и способе его активации см. в разделе [Time Shifting](https://intl.cloud.tencent.com/document/product/267/31565).

### Как использовать HTTPS для воспроизведения?

Чтобы ваш домен воспроизведения поддерживал HTTPS, убедитесь, что у вас есть действительный сертификат и закрытый ключ, перейдите на страницу [Domain Management](https://console.tencentcloud.com/live/domainmanage), найдите целевой домен воспроизведения, нажмите кнопку **Manage** и выберите **Advanced Configuration** > **HTTPS Configuration**, чтобы добавить конфигурацию HTTPS. Конфигурация вступит в силу примерно через 2 часа.

### Как использовать узел кэша вне материка Китая для воспроизведения?

CSS предоставляет стабильные узлы CDN по всему материку Китая и по всему миру. Если ваши конечные пользователи расположены вне материка Китая, вы можете выбрать **Global** или **Outside Chinese mainland** для параметра **Acceleration region** при добавлении доменов на странице [Domain Management](https://console.tencentcloud.com/live/domainmanage).

### Как включить защиту от горячих ссылок?

| Параметр защиты от горячих ссылок | Описание | Замечания |
| --- | --- | --- |
| txTime | Время истечения срока действия URL воспроизведения. | Должна быть шестнадцатеричная метка времени Unix. Воспроизведение разрешено только если `txTime` позже времени отправки запроса. |
| key | Ключ MD5. | Этот параметр настраивается. Вы можете установить основной и резервный ключ. Если основной ключ скомпрометирован, вы можете использовать резервный ключ для создания URL воспроизведения при изменении основного ключа. |
| txSecret | Параметр шифрования в URL воспроизведения. | Значение этого параметра вычисляется на основе `key`, `StreamName` и `txTime` с использованием алгоритма MD5. `txSecret` = MD5 (key+StreamName+txTime) |
| Validity period | Период действия аутентификации. | Должен быть больше 0. Если вы установите `txTime` на текущее время и период действия на 300 с, время истечения URL будет текущим временем плюс 300 с. |

#### Вычисление URL защиты от горячих ссылок

```
txSecret = MD5(somestringtest5c2acacc) = b77e812107e1d8b8f247885a46e1bd34http://domain/live/test.flv?txTime=5c2acacc&txSecret=b77e812107e1d8b8f247885a46e1bd34http://domain/live/test.m3u8?txTime=5c2acacc&txSecret=b77e812107e1d8b8f247885a46e1bd34
```

URL воспроизведения потока HD:

```
txSecret = MD5(somestringtest_9005c2acacc) = 4beae959b16c77da6a65c7edda1dfefehttp://domain/live/test_900.flv?txTime=5c2acacc&txSecret=4beae959b16c77da6a65c7edda1dfefehttp://domain/live/test_900.m3u8?txTime=5c2acacc&txSecret=4beae959b16c77da6a65c7edda1dfefe
```

#### Включение защиты от горячих ссылок

2. Выберите вкладку **Access Control**. В разделе **Key Authentication** включите опцию **Playback Authentication**.
3. Завершите настройки во всплывающем окне.
4. Нажмите кнопку **Save**, чтобы сохранить конфигурацию.

> **Примечание:** Конфигурация вступит в силу в течение **30 минут**. HTTP-FLV: текущее воспроизведение может продолжаться даже после `txTime`, но новые запросы на воспроизведение будут отклонены. HLS: при использовании HLS поток разбивается на короткие фрагменты, и сегменты TS запрашиваются по мере продолжения воспроизведения. Поэтому, если вы установите `txTime` на 10 минут после текущего времени, воспроизведение будет отклонено через 10 минут. Чтобы избежать этого, вы можете динамически обновлять URL запроса HLS на сервере или установить более длительный период действия.

### Каковы требования к формату основного ключа для аутентификации воспроизведения? Есть ли ограничение на его период действия?

Основной ключ может содержать только буквы и цифры, максимум 256 символов. Дополнительные сведения см. в разделе [Playback Authentication Configuration](https://intl.cloud.tencent.com/document/product/267/31060).
Рекомендуется установить период его действия равным продолжительности сеанса прямой трансляции.

### Могу ли я создать URL отправки, действительный длительное время? Каков верхний предел периода действия?

> **Примечание:** Не рекомендуется устанавливать очень длительный период действия. Это может привести к ошибкам и сбоям аутентификации.

### Будет ли логотип Tencent Cloud отображаться на изображении прямой трансляции?

Нет.

### Какова задержка при прямой трансляции?

Обычно задержка при воспроизведении потока FLV, отправленного через RTMP, составляет около 2-3 секунд. Если ваша задержка воспроизведения исключительно высока, см. раздел [Troubleshooting High Latency](https://intl.cloud.tencent.com/document/product/267/7971), чтобы устранить проблему.

### Могу ли я изменить максимальный битрейт во время прямой трансляции?

Нет. Вы можете установить его только перед отправкой потока. Битрейт зависит от скорости вашей загрузки. Установка слишком высокого значения может привести к потере кадров и зависаниям. Кроме того, если битрейт трансляции превышает 5 Мбит/с, не рекомендуется использовать протокол WebRTC. Вместо этого рекомендуется использовать протокол RTMP.

### Как удалить комнату прямой трансляции, которая больше не используется?

> **Примечание:** Режим канала — это устаревший режим, который больше не обновляется и не поддерживается.

### Когда следует использовать API включения/отключения отправки?

Вы можете использовать API для отключения прямой трансляции при обнаружении порнографического или другого неподобающего контента. Дополнительные сведения см. в разделе [ForbidLiveStream](https://intl.cloud.tencent.com/document/product/267/30794).

### Как реализовать воспроизведение в фоновом режиме?

Воспроизведение в фоновом режиме позволяет аудио прямой трансляции продолжать воспроизводиться даже когда приложение работает в фоне. Эта функция зависит от устройства воспроизведения и должна быть реализована в зависимости от вашей фактической бизнес-логики.

### Что делать, если при изменении конфигурации HTTPS возникает ошибка "incorrect certificate"?

CSS использует Nginx для шифрования, поэтому убедитесь, что тип сертификата — Nginx.

### Почему URL воспроизведения перестают работать после отключения аутентификации воспроизведения?

URL воспроизведения с включенной аутентификацией станет недействительным после отключения аутентификации, даже если период действия еще не истек.

### Какие ограничения на частоту запросов API?

CSS устанавливает верхний предел на общее количество запросов, отправленных всеми `SecretId` под аккаунтом. После достижения предела новые запросы не будут обработаны.
Например, верхний предел в 200 запросов в секунду означает, что сервер Tencent Cloud может получить до 200 запросов, отправленных всеми `SecretId` под вашим аккаунтом в течение одной секунды. Не имеет значения, отправляют ли запросы один или несколько пользователей или они запрашивают один или несколько потоков.

### При отправке потоков возникает ошибка "RTMP close", но в журнале указано, что отправка успешна. Что делать?

Проблема может быть в вашем URL отправки. Вы можете использовать наше приложение [TCToolkit app](https://intl.cloud.tencent.com/document/product/1071/38147) для проверки нормальной работы URL. Дополнительные решения см. в разделе [Troubleshooting Push Failure](https://intl.cloud.tencent.com/document/product/267/33383).

### После изменения частоты кадров я не смог отправлять потоки, пришлось несколько раз перезагружать сервис и часто отключаться. Что делать?

Возможно, вы установили слишком высокую частоту кадров. Частота кадров выше 15 кадров в секунду достаточна для обеспечения плавного воспроизведения. Попробуйте снизить частоту кадров.

### Если поток долгое время не содержит данные, когда он будет отключен?

Поток может долгое время не содержать данные при проблемах с отправкой, например, если приложение упало или телефон выключен.

В этих случаях, если бэкэнд не получит никаких данных в течение 70 секунд, он отключит поток.

### При использовании Web Push SDK для отправки контента почему изображение размыто и не достигает ожидаемого результата?

Если установлено высокое разрешение, но видео остается размытым, это обычно означает, что браузер автоматически снижает качество трансляции в зависимости от условий сети. Вы можете установить разрешение, установив атрибут contentHint видеопотока. Для подробной конфигурации см. [документацию](https://www.tencentcloud.com/document/product/267/58159#821096b5-7a51-4cd7-8f08-3f068c7e9093) и вызовите команду setVideoContentHint('detail') после захвата видеопотока.

### Как настроить ключ API в новой консоли?

Ключи API используются для аутентификации устаревших API. Мы обновили наши API до версии 3.0. Для использования новых API перейдите на страницу [Manage API Key](https://console.tencentcloud.com/cam/capi) и получите `SecretId` и `SecretKey`.

### Почему я не могу воспроизводить видео в кодировке H.265?

H.265 поддерживается меньшим количеством браузеров, чем H.264, и воспроизведение может не сработать, если ваш браузер не поддерживает H.265. Вы можете [настроить шаблон транскодирования](https://intl.cloud.tencent.com/document/product/267/31071), чтобы перекодировать видео в H.264.

### Могут ли имена файлов M3U8 содержать китайские символы?

Файлы M3U8 именуются автоматически на основе имен потоков. Имена потоков **не поддерживают** китайские символы.

### Как просмотреть количество онлайн-зрителей?

Вы можете получить количество онлайн-зрителей, вызвав API [DescribeStreamPlayInfoList](https://intl.cloud.tencent.com/document/product/267/37297), но результат может быть неточным на 100%. Например, если три пользователя используют один и тот же IP-адрес для воспроизведения одновременно, они будут учтены как один зритель. Обратите внимание, что данные, возвращаемые этим API, актуальны только если ваш протокол воспроизведения — RTMP или FLV, а не HLS.

### Поддерживает ли CSS резервные потоки?

Резервные потоки включены по умолчанию в CSS. Если вы одновременно отправляете два потока с одинаковым именем, при воспроизведении будет видно только первый поток. Если этот поток прерывается, становится видно второй поток.

### Могу ли я добавить разные водяные знаки для одного домена отправки?

Нет. К каждому домену отправки можно привязать только один шаблон водяного знака.

### Как просмотреть продолжительность воспроизведения пользователей?

В настоящее время вы не можете просмотреть продолжительность воспроизведения пользователей.

### Могут ли пользователи смотреть мои прямые трансляции, если я не транскодирую потоки?

Воспроизведение основано на URL воспроизведения и возможно даже без транскодирования потоков (при условии, что URL действителен).

### Какие факторы влияют на время первого кадра?

Время первого кадра главным образом зависит от количества зрителей. Чем больше зрителей, тем выше коэффициент попадания в кэш и тем короче время первого кадра.

### Могу ли я создать чёрный список/белый список для воспроизведения?

- **IP белый список**: Только IP-адреса из списка могут получить доступ к вашему потоковому контенту.
- **IP чёрный список**: IP-адреса из списка не могут получить доступ к вашему потоковому контенту.

### Сколько снимков берется для определения порнографии?

CSS определяет порнографический контент на основе снимков. Количество снимков, сделанных для определения порнографии, зависит от интервала снимков, который вы можете установить в разделе [Live Screencapture](https://console.tencentcloud.com/live/config/jtjh) консоли CSS.

> **Примечание:** Интервал снимков по умолчанию во время отправки составляет две секунды. Диапазон значений: 2–300 (секунд).

### Как запросить мою оплачиваемую пропускную способность и трафик?

Вы можете использовать API [DescribeBillBandwidthAndFluxList](https://intl.cloud.tencent.com/document/product/267/36098) для запроса данных.

### Как узнать, была ли отправка успешной?

- Вы также можете вызвать API [DescribeLiveStreamState](https://intl.cloud.tencent.com/document/product/267/30796) для запроса состояния потока.

> **Примечание:** При сбое отправки или воспроизведения вы можете использовать [инструмент самодиагностики](https://console.tencentcloud.com/live/tools/selfcheck) консоли CSS для устранения неполадок. Дополнительные сведения см. в разделе [Self-Diagnosis](https://intl.cloud.tencent.com/document/product/267/39467).

### Могу ли я отправлять потоки только с аудио?

- [Pushing audio-only streams with OBS](https://intl.cloud.tencent.com/document/product/267/31569).
- [Creating a transcoding template](https://intl.cloud.tencent.com/document/product/267/30790)

### Как запросить продолжительность прямой трансляции?

Вы можете вызвать RESTful API для запроса данных. Дополнительные сведения см. в разделе [DescribeStreamPlayInfoList](https://intl.cloud.tencent.com/document/product/267/37297).

### Как запросить продолжительность активного потока?

CSS не предоставляет API для запроса продолжительности активного потока. Вы можете рассчитать продолжительность на основе обратных вызовов для отправки и прерывания потока.

### Как запросить зрителей прямой трансляции?

- Перейдите в консоль CSS, выберите **Data Center** > **Stream Data Query** на левой боковой панели и щелкните **Playback Data**, чтобы просмотреть количество одновременных подключений.

> **Примечание:** Количество одновременных подключений равно количеству онлайн-зрителей только если протокол воспроизведения — RTMP или FLV.

- Вызовите API [DescribeStreamPlayInfoList](https://intl.cloud.tencent.com/document/product/267/37297), чтобы получить количество онлайн-зрителей.

### Если я отключусь после отправки потока, получу ли я обратный вызов для прерывания потока?

Сначала необходимо настроить обратный вызов для прерывания потока. Если поток прерывается из-за разрыва сетевого соединения, система попытается автоматически возобновить поток. Если поток возобновляется в течение 70 секунд, вы не получите обратный вызов для прерывания потока. Если система не сможет возобновить поток в течение 70 секунд, она отключит поток, и вы получите обратный вызов прерывания потока.

### Должен ли я использовать один и тот же протокол для отправки и воспроизведения?

Нет. Вы можете использовать RTMP для отправки и RTMP, FLV, HLS или UDP для воспроизведения.

### Почему я не могу воспроизводить видео из CDN при использовании беспроводной сети?

- Проверьте, не заблокировала ли ваша беспроводная сеть IP-адреса Tencent Cloud.
- Если вы используете iPhone, убедитесь, что вы разрешили приложению использовать Wi-Fi.

Если вышеприведенное не решает вашу проблему, обратитесь в техническую поддержку Tencent Cloud.

### Я настроил HTTPS для своего домена воспроизведения, поэтому почему воспроизведение не сработало?

- Убедитесь, что вы загрузили сертификат.
- Проверьте время загрузки сертификата. После настройки HTTPS подождите около двух часов перед доступом к домену.

> **Примечание:** Если вышеприведенное не решает вашу проблему, попробуйте использовать [инструмент самодиагностики](https://console.tencentcloud.com/live/tools/selfcheck) консоли CSS для выявления проблемы. Дополнительные указания см. в разделе [Self-Diagnosis](https://intl.cloud.tencent.com/document/product/267/39467).

### При использовании LEB, как изменить разрешение хостов во время прямой трансляции?

LEB — это решение для воспроизведения, а хосты находятся на конце отправки. Чтобы изменить разрешение хостов, вам нужно создать [шаблон транскодирования](https://intl.cloud.tencent.com/document/product/267/31071) и привязать его к своему домену перед воспроизведением.

### Поддерживает ли трансляция WebRTC форматы H.265, VP8 или VP9?

По умолчанию трансляция WebRTC поддерживает формат H.264. Если вы хотите использовать форматы H.265, VP8 или VP9, вы можете включить их, добавив определенные параметры к URL трансляции: `&tx_h265=true` (для включения H.265), `&tx_vp8=true` (для включения VP8) или `&tx_vp9=true` (для включения VP9).

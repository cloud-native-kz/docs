# Уведомление об аномальном событии записи

Callback аномального события записи в основном используется для обратного вызова конкретной информации об аномальных событиях, произошедших во время процесса записи. Вам необходимо настроить URL для получения уведомлений callback аномального события записи. Тенцентский бэкенд CSS будет отправлять результаты на установленный вами целевой сервер.

Этот документ описывает поля в уведомлениях callback, отправляемых Tencent CSS пользователям после срабатывания callback аномального события записи.

## Важная информация

Перед прочтением этого документа вы должны знать, как настроить функцию callback Tencent CSS и как получать уведомления callback. Дополнительные сведения см. в разделе [Как получать уведомления о событиях](https://www.tencentcloud.com/document/product/267/38080).

## Параметры аномального события записи

### Тип события

| Тип события | Значение |
| --- | --- |
| Событие аномальной записи | event_type = 341 |

### Общий параметр callback

| Параметр | Тип | Описание |
| --- | --- | --- |
| t | int64 | Время истечения, которое является временем Unix, когда истекает подпись уведомления о событии. Стандартный период действия уведомления callback от Tencent Cloud составляет 10 минут. Если время, указанное значением `t` в уведомлении, прошло, это уведомление считается недействительным. Это предотвращает атаки повторного воспроизведения сети. Значение `t` — это десятичная временная метка Unix, то есть количество секунд, прошедших с 00:00:00 (время UTC/GMT), 1 января 1970 года. |
| sign | string | Подпись безопасности. **sign = MD5(key + t).** Примечание: Tencent Cloud объединяет ключ шифрования **key** и `t`, создает хеш MD5 объединенной строки и встраивает его в сообщения callback. Ваш бэкенд-сервер может выполнить те же вычисления при получении сообщения callback. Если подпись совпадает, это указывает на то, что сообщение поступило от Tencent Cloud. |

> **Примечание:** Вы можете установить ключ callback в разделе **Конфигурация функций** > [Live Stream Callback](https://console.tencentcloud.com/live/config/callback), который используется для аутентификации. Рекомендуем вам установить это поле для обеспечения безопасности данных. ![](https://staticintl.cloudcachetci.com/cms/backend-cms/a5f3c96ccb6711f08942525400e889b2.png)

### Параметры callback

| Параметр | Тип | Описание |
| --- | --- | --- |
| appid | int | [APPID](https://console.tencentcloud.com/developer) пользователя. |
| stream_id | string | Имя потока живого вещания. |
| session_id | string | ID сеанса записи живого вещания. Используется для различия различных задач записи. |
| file_format | string | Формат файла записи, включая FLV, TS, MP4 и AAC. |
| timestamp | int | Время callback события аномальной записи (в секундах). |
| exception_level | string | Уровень исключения, включая info, warning и error. |
| exception | json | Подробное описание события аномалии. |

### Параметры исключения

| Параметр | Тип | Описание |
| --- | --- | --- |
| type | int | [Тип события аномалии](#b1208801-ad7a-4e79-8c36-b2ca1fce9e5a). |
| occurs | int | Количество возникновений в пределах цикла статистики аномальных событий. |
| desc | string | Описание события аномалии. |

#### Тип события аномалии

| Тип | Описание |
| --- | --- |
| -8 | Временная метка аудио значительно изменилась. |
| -9 | Временная метка видео значительно изменилась. |
| -10 | Временные метки аудио и видео значительно отличаются. |
| -11 | Временная метка видео недействительна. |
| -16 | Ошибка при извлечении потока. |
| -17 | Аудио не отправляется, когда для потока установлена запись аудио. |
| -18 | Видео не отправляется, когда для потока установлена запись видео. |
| -22 | Временная метка аудио не изменяется. |
| -23 | Временная метка видео не изменяется. |
| -24 | Временная метка аудио недействительна. |
| -100 | Разрешение для отправки потока изменилось. |
| -104 | Произошла ошибка при записи кадров аудио/видео в контейнеры (файлы). |
| -120 | Файл записи усечен из-за изменения способа кодирования видео. |
| -130 | Аномальные кадры видео удалены. |
| -131 | Аномальные кадры аудио удалены. |
| -132 | Длина GOP аномальна. |
| -200 | Временная метка аудио не увеличивается равномерно. |
| -201 | Временная метка видео не увеличивается равномерно. |

> **Примечание:** В настоящее время callback события аномальной записи не поддерживает конфигурацию для отдельных событий и выполняет callback только для происходящих в настоящее время аномальных событий. Если аномальное событие не произойдет, callback не выполняется. Callback события аномальной записи выполняет статистику только по текущим событиям аномальной записи. Чтобы избежать частых callback, одинаковые аномальные события, произошедшие в короткий промежуток времени, будут объединены для обработки. Поэтому callback событий аномалии могут иметь задержку (не превышающую 2 минут). Никакие другие операции не будут выполняться в бэкенде записи, кроме обнаружения аномального события и callback события аномалии.

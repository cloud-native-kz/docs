# Как получить уведомление о событии

Если событие, настроенное в шаблоне, вызывает обратный вызов во время прямой трансляции, Tencent Cloud отправит запрос на сервер пользователя, который отвечает за ответ. После прохождения проверки сервер получит JSON-пакет обратного вызова.

В настоящее время следующие события могут вызвать уведомление во время прямой трансляции: отправка потока, перерыв потока, запись прямой трансляции, снимок экрана прямой трансляции, аудит изображений прямой трансляции, аудит аудио прямой трансляции, события ошибок отправки и события трансляции.

## Общий процесс

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a2b8dc855c4611ee974d5254005f490f.png)

**Описание процесса:**

1. Пользователь настраивает URL-адреса уведомлений о событиях и функции, такие как запись и снимок экрана, в консоли или путем вызова API TencentCloud.
2. Пользователь начинает и прерывает потоковую передачу.
3. При возникновении события сообщение будет отправлено в бэкэнд пользователя через сервис уведомлений о событиях.

## Протокол уведомления о событиях

### Сетевой протокол

- Ответ: HTTP код состояния = 200. Сервер игнорирует конкретное содержимое пакета ответа. Для совместимости протоколов рекомендуется добавить `JSON: `{"code":0}`` в ответ.

### Надежность уведомлений

Сервис уведомления о событиях имеет механизм повтора. Для события снимка экрана будет сделано до 5 повторных попыток с интервалом 2 минуты. Для событий отправки потока, прерывания потока, записи и обнаружения порнографии будет сделано до 12 повторных попыток с интервалом 1 минута.
Чтобы предотвратить перегрузку сервера и пропускной способности из-за частых повторов, убедитесь, что пакеты ответа возвращаются должным образом. Повтор запускается в следующих случаях:

- Пакет ответа не возвращается в течение длительного времени (не менее 20 секунд).
- HTTP код состояния в ответе не равен `200`.

## Как настроить обратные вызовы события

Вы можете настроить обратные вызовы через [консоль CSS](https://www.tencentcloud.com/document/product/267/38080#css-console) или [API сервера](https://www.tencentcloud.com/document/product/267/38080#server-apis).

> **Примечание:** CSS позволяет настроить URL-адреса обратных вызовов отдельно для событий отправки потока, прерывания потока, записи, снимка экрана и обнаружения порнографии.

### Консоль CSS

1. Войдите в консоль CSS и нажмите **Конфигурация функций** > [**Обратный вызов прямой трансляции**](https://console.tencentcloud.com/live/config/callback), чтобы создать шаблон обратного вызова. Подробные инструкции см. в [Создание шаблона обратного вызова](https://www.tencentcloud.com/document/product/267/31074).
2. Нажмите [Управление доменами](https://console.tencentcloud.com/live/domainmanage), найдите целевое имя домена отправки и нажмите **Управление** > **Конфигурация шаблона**, чтобы привязать его к шаблону обратного вызова. Подробные инструкции см. в [Конфигурация обратного вызова](https://www.tencentcloud.com/document/product/267/31065).

### API сервера

1. Вызовите API [CreateLiveCallbackTemplate](https://www.tencentcloud.com/document/product/267/30815), чтобы создать шаблон обратного вызова и установить параметры обратного вызова.
2. Вызовите API [CreateLiveCallbackRule](https://www.tencentcloud.com/document/product/267/30816), чтобы установить параметры `DomainName` (имя домена отправки) и `TemplateId` (возвращено на шаге 1). Введите `AppName` в URL-адреса отправки и воспроизведения, чтобы включить обратный вызов для определенных прямых трансляций.

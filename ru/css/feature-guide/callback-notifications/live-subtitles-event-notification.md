# Уведомление о событии живых субтитров

Этот документ в основном описывает поля уведомления обратного вызова, отправляемые пользователям при возникновении события живых субтитров в сервисе потоковой передачи Tencent Cloud (CSS).

## Примечание

- В этом руководстве предполагается, что вы знаете, как настроить обратные вызовы и получать уведомления обратного вызова от CSS. Подробности см. в разделе [Как получить уведомление о событии](https://intl.cloud.tencent.com/document/product/267/38080).
- Информация обратного вызова возвращается только при извлечении потоков субтитров.

## Протокол уведомления события сообщения

**Сетевые протоколы**

- **Запрос**: используется HTTP POST запрос с содержимым тела в формате JSON. Конкретное содержимое тела для каждого типа сообщения подробно описано в последующих разделах.
- **Ответ**: код состояния HTTP — 200. Хотя сервер не учитывает конкретное содержимое пакета ответа, для совместимости протокола рекомендуется, чтобы клиент включал JSON полезную нагрузку в ответ: `{"code":0}`.

**Надежность сообщения**

Служба уведомления о событиях включает механизм повтора с общим количеством попыток 3. Чтобы предотвратить потенциальное влияние повторных попыток на производительность вашего сервера и пропускную способность сети, убедитесь, что возвращаются надлежащие пакеты ответов. Условия, вызывающие повторные попытки, следующие:

- Пакет ответа не получен в течение длительного времени.
- Возвращаемый код состояния HTTP не равен 200.

## Параметры события живых субтитров

### Тип события

| Тип события | Значение |
| --- | --- |
| Живые субтитры | event_type = 338 |

### Общие параметры обратного вызова

| Параметр | Тип | Описание |
| --- | --- | --- |
| t | int64 | Время истечения, которое является временем Unix, когда истекает срок действия подписи уведомления о событии. Период действия уведомления обратного вызова от Tencent Cloud по умолчанию составляет 10 минут. Если время, указанное значением `t` в уведомлении, истекло, то это уведомление считается недействительным. Это предотвращает атаки повтора в сети. Значение `t` — это десятичная временная метка Unix, то есть количество секунд, прошедших с 00:00:00 (время UTC/GMT) 1 января 1970 года. |
| sign | string | Подпись безопасности. sign = MD5(key + t). Tencent Cloud объединяет ключ шифрования **key** и `t`, генерирует хеш MD5 объединенной строки и встраивает его в сообщения обратного вызова. Ваш внутренний сервер может выполнить такой же расчет при получении сообщения обратного вызова. Если подпись совпадает, это указывает на то, что сообщение поступило от Tencent Cloud. |

> **Примечание:** Вы можете установить ключ обратного вызова в разделе **Конфигурация функций** > [Обратный вызов потока в реальном времени](https://console.tencentcloud.com/live/config/callback), который используется для аутентификации. Рекомендуется установить это поле для обеспечения безопасности данных. ![](https://staticintl.cloudcachetci.com/cms/backend-cms/9884fd08cb6711f084a45254005ef0f7.png)

### Параметры обратного вызова

| Параметр | Тип | Описание |
| --- | --- | --- |
| appid | int | [APPID](https://console.tencentcloud.com/developer) пользователя. |
| stream_id | string | Имя потока в реальном времени. |
| channel_id | string | Совпадает с именем потока в реальном времени. |
| task_id | string | ID задачи субтитров. |
| data | [Структура data](#data) | Результаты обработки. |

| Параметр | Тип | Описание |
| --- | --- | --- |
| subtitle_tmp_res | [Массив subtitle_tmp_res](#subtitle_tmp_res) | Результаты текста субтитров. |

| Параметр | Тип | Описание |
| --- | --- | --- |
| src_txt | string | Исходный текст распознавания речи. |
| dst_txt | string | Переведенный текст. Если перевод не настроен, это значение не возвращается. |
| start_pts | int64 | Начальная точка временной метки потока аудио, соответствующая src_txt, в миллисекундах (мс). |
| end_pts | int64 | Конечная точка временной метки потока аудио, соответствующая src_txt, в миллисекундах (мс). |
| start_unix_time | int64 | Начальная точка времени UNIX трансляции в реальном времени, соответствующая src_txt. |
| end_unix_time | int64 | Конечная точка времени UNIX трансляции в реальном времени, соответствующая src_txt. |
| steady_state | bool | Флаг установившегося состояния: если значение true, это означает, что предложение достигло стабильного состояния и текст больше не будет пересмотрен. |

### Пример обратного вызова

```
{  "event_type": 338,  "stream_id": "teststream",  "channel_id": "teststream",  "task_id": "7735333",  "data": {    "subtitle_tmp_res": [      {        "src_txt": "Today we shall delve into the study of linear equations in one variable.",        "start_pts": 123000,        "end_pts": 126333,		"stead_state": bool      }    ]  },  "sign": "ca3e25e5dc17a6f9909a9ae7281e300d",  "t": 1754623810}
```

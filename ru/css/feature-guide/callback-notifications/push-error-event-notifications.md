# Уведомления о событиях ошибок отправки

Обратные вызовы ошибок отправки уведомляют вас об информации об ошибках отправки. Необходимо настроить адрес обратного вызова в консоли CSS, и CSS будет отправлять обратные вызовы об ошибках отправки на настроенный вами сервер.
В этом документе описаны поля в уведомлении обратного вызова, отправленном CSS после возникновения ошибки отправки.

## Обязательные знания

В этом руководстве предполагается, что вы знаете, как настроить обратные вызовы и получать уведомления о событиях обратного вызова от CSS. Подробные сведения см. в разделе [Как получать уведомления о событиях](https://intl.cloud.tencent.com/document/product/267/38080).

## Параметры обратного вызова ошибки отправки

### Тип события

| Тип события | Значение параметра |
| --- | --- |
| Ошибки отправки | event_type = 321 |

### Общие параметры обратного вызова

| Параметр | Тип | Описание |
| --- | --- | --- |
| t | int64 | Время истечения, представляющее собой временную метку Unix, когда истекает подпись уведомления о событии. Период действия обратного вызова от Tencent Cloud по умолчанию составляет 10 минут. Если истекло время, указанное значением `t` в уведомлении, то это уведомление считается недействительным. Это предотвращает атаки воспроизведения сети. Значение `t` — это десятичная временная метка Unix, то есть количество секунд, прошедших с 00:00:00 (время UTC/GMT) 1 января 1970 года. |
| sign | string | Подпись безопасности. **sign = MD5(key + t)**. Примечание: Tencent Cloud объединяет ключ шифрования **key** и `t`, генерирует хеш MD5 объединенной строки и встраивает его в сообщения обратного вызова. Сервер вашего бэкэнда может выполнить то же самое вычисление при получении сообщения обратного вызова. Если подпись совпадает, это указывает на то, что сообщение поступило от Tencent Cloud. |

> **Примечание:** Вы можете установить ключ обратного вызова в **Конфигурация функций** > [Обратный вызов потока в реальном времени](https://console.tencentcloud.com/live/config/callback), который используется для аутентификации. Рекомендуется установить это поле для обеспечения безопасности данных. ![](https://staticintl.cloudcachetci.com/cms/backend-cms/9f60336dcb6711f0a4a55254001c06ec.png)

### Параметры обратного вызова

| Параметр | Тип | Описание |
| --- | --- | --- |
| appid | int | [APPID](https://console.tencentcloud.com/developer) пользователя. |
| user_ip | string | IP-адрес отправки пользователя. |
| data_time | int | Время обратного вызова события отправки (единица: мс). |
| domain | string | Доменное имя отправки. |
| path | string | Путь отправки. |
| report_interval | int | Интервал отчета (в мс) при возникновении ненормального события. |
| sequence | string | Номер последовательности сообщения, идентифицирующий деятельность потоковой передачи. |
| stream_id | string | Идентификатор потока. |
| stream_param | string | Параметры URL отправки. |
| timeout | int | Время ожидания запроса обратного вызова (в мс). |
| abnormal_event | json | [Группа детальных ненормальных событий](https://www.tencentcloud.com/document/product/267/53310#.60abnormal_event.60-parameters). |

#### Параметры `abnormal_event`

| Параметр | Тип | Описание |
| --- | --- | --- |
| count | int | Количество раз, когда ошибка произошла между двумя отчетами (в пределах интервала отчета). |
| customer_visible | string | Видимо ли событие исключения для клиента, фиксировано как "true". |
| detail | json | desc: Описание ошибки. occur_time: UNIX временная метка события исключения. |
| event_key | string | Дополнительные китайские описания ненормальных событий. |
| type | int | [Типы ошибок](#2d78c98a-ab47-4f29-b4f8-4511dedf2ffa). |
| type_desc_cn | string | Описание ошибки на китайском языке. |
| type_desc_en | string | Описание ошибки на английском языке. |

#### Типы ошибок

| Тип | Описание |
| --- | --- |
| 1 | Временная метка видео переместилась назад. |
| 2 | Временная метка аудио переместилась назад. |
| 3 | Временная метка видео заметно увеличилась (более чем на 1000 миллисекунд). |
| 4 | Временная метка аудио заметно увеличилась (более чем на 1000 миллисекунд). |
| 5 | Размер блока слишком большой (больше 8192). |
| 6 | Два последовательных видеокадра поступили с опозданием (более чем на 3000 миллисекунд). |
| 7 | Два последовательных аудиокадра поступили с опозданием (более чем на 1000 миллисекунд). |
| 8 | Видеокодек изменился. |
| 9 | Аудиокодек изменился. |
| 10 | Отсутствует заголовок кодека перед поступлением видеокадра. |
| 11 | Отсутствует заголовок кодека перед поступлением аудиокадра. |
| 12 | Интервал между временными метками аудиокадров слишком велик. |
| 13 | Форматы аудио/видеокодеков необычны. |
| 14 | Невозможно разобрать заголовок видео. |
| 15 | Частота кадров видео слишком низкая. |
| 16 | Размер GOP слишком велик. В настоящее время, если он превышает 10 с, считается слишком большим. |

> **Примечание:** В настоящее время вы не можете настроить обратные вызовы для определенного типа ошибки отправки. Обратный вызов об ошибке отправки включает информацию обо всех ошибках отправки, которые произошли в течение интервала отчета. Если ошибок отправки не происходит, обратные вызовы отправляться не будут. Обратный вызов об ошибке отправки собирает данные только об ошибках отправки в текущем цикле отчета. Система не будет обрабатывать ошибки.

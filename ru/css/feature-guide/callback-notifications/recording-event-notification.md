# Уведомление о событии записи

Функция живой записи записывает потоки в реальном времени в соответствии с шаблоном записи, привязанным к доменному имени push, а затем сохраняет файлы записи в VOD. Обратный вызов записи уведомляет вас об информации файла записи, включая время начала и конца записи, ID файла записи, размер файла и URL загрузки. Чтобы получать обратные вызовы записи, необходимо настроить шаблон обратного вызова, указать адрес сервера для обратного вызова и привязать шаблон к доменному имени push. При возникновении события записи бэкэнд CSS отправит информацию о файле записи на указанный сервер.

В этом документе описаны поля в уведомлении обратного вызова, отправляемом CSS после события файла записи.

## Примечания

- В этом руководстве предполагается, что вы знаете, как настраивать обратные вызовы и получать уведомления об обратных вызовах от CSS. Подробные сведения см. в разделе [How to Receive Event Notification](https://intl.cloud.tencent.com/document/product/267/38080).
- Записанные видеофайлы сохраняются в консоли [VOD](https://console.tencentcloud.com/vod) по умолчанию с возможностью сохранения в продукты [Cloud Object Storage (COS)](https://console.tencentcloud.com/cos). Рекомендуется заранее активировать сервис VOD или COS, чтобы избежать приостановки сервиса из-за задолженности.
- Если задача записи создана API [CreateRecordTask](https://intl.cloud.tencent.com/document/product/267/37309), обратный вызов записи не будет включать параметры `stream_param` URL push. Они будут включены, если задача создана другим способом.
- Если включено возобновление записи HLS, обратный вызов будет срабатывать только для окончательного файла записи. Обратные вызовы не будут отправляться при прерывании push.

## Параметры события записи

### Тип события

| Тип события | Значение |
| --- | --- |
| Живая запись (файлы) | event_type = 100 |

### Общие параметры обратного вызова

| Параметр | Тип | Описание |
| --- | --- | --- |
| t | int64 | Время (временная метка Unix), когда истекает срок действия сигнатуры уведомления. Период действия обратного вызова от Tencent Cloud по умолчанию составляет 10 минут. После истечения времени, указанного значением **t**, уведомление будет считаться недействительным. Это предотвращает атаки на повтор сетевых данных. Значение **t** представляет собой десятичную временную метку Unix, которая является количеством секунд, прошедших с 00:00:00 (время UTC/GMT), 1 января 1970 года. |
| sign | string | Сигнатура безопасности, **sign = MD5(key + t)**. Tencent Cloud объединяет ключ шифрования **key** и **t**, генерирует хэш MD5 объединенной строки и встраивает его в уведомления обратного вызова. Ваш бэкэнд-сервер выполняет тот же расчет при получении обратного вызова, и если сигнатура совпадает, это указывает на то, что уведомление поступает из Tencent Cloud. |

> **Примечание:** Вы можете установить ключ обратного вызова в **Feature Configuration** > [Live Stream Callback](https://console.tencentcloud.com/live/config/callback), который используется для аутентификации. Рекомендуется установить это поле для обеспечения безопасности данных. ![](https://staticintl.cloudcachetci.com/cms/backend-cms/40fba85ccb6711f0a93d52540044a08e.png)

### Параметры обратного вызова записи

| Параметр | Тип | Описание |
| --- | --- | --- |
| appid | int | [APPID](https://console.tencentcloud.com/developer) пользователя. |
| app | string | Доменное имя push. |
| appname | string | Путь push. |
| stream_id | string | Имя потока в прямом эфире. |
| channel_id | string | Совпадает с ID потока. |
| file_id | string | ID файла VOD, который однозначно идентифицирует файл в [VOD](https://intl.cloud.tencent.com/document/product/266/33895). |
| record_file_id | string | ID файла записи. |
| record_temp_id | string | ID шаблона задачи записи указывает, что файл был создан задачей шаблона, если он существует и не равен null; в противном случае это означает, что файл был создан глобальной задачей. |
| file_format | string | Формат файла. Допустимые значения: `flv`, `hls`, `mp4`, `aac`. |
| task_id | string | ID задачи записи, возвращаемый API [CreateRecordTask](https://intl.cloud.tencent.com/document/product/267/37309) и действительный только если задача создана API. |
| start_time | int64 | Время начала записи. |
| end_time | int64 | Время конца записи. |
| start_time_usec | int | Время начала записи (микросекунды). |
| end_time_usec | int | Время конца записи (микросекунды). |
| duration | int64 | Продолжительность файла записи в секундах. Разница между start_time и end_time может отличаться от значения duration при нормальных обстоятельствах, особенно в случаях слабого сетевого соединения и аномалий при передаче потока. |
| file_size | uint64 | Размер файла записи в байтах. |
| stream_param | string | Параметры URL push (пользовательские). |
| video_url | string | URL загрузки файла записи. |
| media_start_time | int | PTS при первом извлечении потока для записи. Это не обязательно PTS первого кадра файла записи. |
| record_bps | int | Битрейт (кбит/с) записанного выходного потока после перекодирования. |
| callback_ext | Строка JSON-объекта. | JSON-объект включает несколько полей: `video_codec` указывает видеокодек. `resolution` указывает разрешение переданного потока. `session_id` указывает ID задачи записи. Это все дополнительные поля обратного вызова записи. Рекомендуется не полагаться на них слишком сильно в своей бизнес-логике. |

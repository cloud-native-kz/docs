# Уведомление о событии захвата экрана

Захват живого экрана делает снимки экрана в реальном времени из потока в прямом эфире с указанным интервалом и сохраняет их в COS. Обратный вызов захвата экрана возвращает информацию о сохраненных снимках экрана, включая время создания снимка, размер изображения, путь файла и ссылку для скачивания. Для получения обратных вызовов захвата экрана необходимо настроить адрес вашего сервера в шаблоне обратного вызова и привязать шаблон к вашему домену отправки. Когда происходит событие захвата живого экрана, серверная часть CSS отправляет информацию о снимке экрана на сервер, указанный в конфигурации.

Этот документ описывает поля в сообщении обратного вызова захвата живого экрана.

## Примечания

- В этом руководстве предполагается, что вы знаете, как настроить обратные вызовы и получать уведомления обратного вызова от CSS. Подробнее см. в разделе [Как получать уведомления о событиях](https://intl.cloud.tencent.com/document/product/267/38080).
- Информация, возвращаемая обратным вызовом захвата экрана, может использоваться для обнаружения порнографического контента, создания миниатюр видео в прямом эфире и других сценариев.

## Параметры события захвата экрана

### Тип события

| Тип события | Значение |
| --- | --- |
| Захват живого экрана | event_type = 200 |

### Общие параметры обратного вызова

| Параметр | Тип | Описание |
| --- | --- | --- |
| t | int64 | Время истечения, которое является временной меткой Unix, когда истекает срок действия подписи уведомления о событии. Период действия по умолчанию для уведомления сообщения от Tencent Cloud составляет 10 минут. Если время, указанное значением `t` в уведомлении сообщения, истекло, такое уведомление считается недействительным, что предотвращает атаки воспроизведения в сети. Значение `t` — это десятичная временная метка Unix, то есть количество секунд, прошедших с 00:00:00 (время UTC/GMT) 1 января 1970 г. |
| sign | string | Подпись безопасности. sign = MD5(key + t). Tencent Cloud объединяет ключ шифрования **key** и `t`, генерирует хеш MD5 объединенной строки и встраивает его в сообщения обратного вызова. Ваш серверный сервер может выполнить тот же расчет при получении сообщения обратного вызова. Если подпись совпадает, это указывает на то, что сообщение поступает от Tencent Cloud. |

> **Примечание:** Ключ используется для аутентификации. Вы можете установить его в **Feature Configuration** > [Live Stream Callback](https://console.tencentcloud.com/live/config/callback). Мы рекомендуем вам установить его, чтобы обеспечить безопасность данных.![](https://staticintl.cloudcachetci.com/cms/backend-cms/4edc50c4cb6711f084a45254005ef0f7.png)

### Параметры сообщения обратного вызова

| Параметр | Тип | Описание |
| --- | --- | --- |
| app | string | Имя домена отправки |
| appname | string | Путь отправки |
| stream_param | string | Параметры URL отправки |
| stream_id | string | Имя потока в прямом эфире |
| channel_id | string | Совпадает с именем потока |
| create_time | int64 | Временная метка Unix, когда создается снимок экрана |
| file_size | int | Размер файла снимка экрана в байтах |
| width | int | Ширина снимка экрана в пикселях |
| height | int | Высота снимка экрана в пикселях |
| pic_url | string | Путь файла снимка экрана (`/path/name.jpg`) |
| pic_full_url | string | URL для скачивания снимка экрана |
| task_id | string | ID задачи захвата экрана |

# Уведомление о событии интеллектуального стирания

Этот документ в основном описывает поля обратного вызова, передаваемые сервисом потоковой передачи Tencent Cloud (CSS) пользователям при срабатывании события интеллектуального стирания.

## Примечание

Перед прочтением этого документа вам необходимо разобраться в том, как настроить обратные вызовы и как вы будете получать сообщения через CSS Tencent Cloud. Дополнительные сведения см. в разделе [Как получать уведомления о событиях](https://intl.cloud.tencent.com/document/product/267/38080).

## Протокол уведомления о событии сообщения

**Сетевые протоколы**

- **Запрос**: использует HTTP POST запрос с содержимым тела в формате JSON. Конкретное содержимое тела для каждого типа сообщения подробно описано в последующих разделах.
- **Ответ**: код состояния HTTP — 200. Хотя сервер игнорирует конкретное содержимое пакета ответа, для совместимости протокола рекомендуется, чтобы клиент включал полезную нагрузку JSON в ответ: `{"code":0}`.

**Надежность сообщений**

Служба уведомления о событиях включает механизм повтора с общим количеством 3 попыток. Чтобы предотвратить возможное влияние повторов на производительность вашего сервера и пропускную способность сети, пожалуйста, убедитесь, что возвращаются надлежащие пакеты ответов. Условия, вызывающие повторы, следующие:

- Пакет ответа не получен в течение длительного времени.
- Возвращаемый код состояния HTTP не равен 200.

## Параметры события интеллектуального стирания

### Параметр типа события

| Тип события | Описание значения поля |
| --- | --- |
| Интеллектуальное стирание | event_type = 347 |

### Общие параметры обратного вызова

| Параметр | Тип | Описание |
| --- | --- | --- |
| t | int64 | Время истечения, это UNIX временная метка, когда истекает подпись уведомления о событии. Период действия уведомления о сообщении от Tencent Cloud по умолчанию составляет 10 минут. Если время, указанное значением `t` в уведомлении о сообщении, истекло, то это уведомление считается недействительным, что предотвращает атаки повторного воспроизведения сети. Формат `t` — это десятичная UNIX временная метка, т. е. количество секунд, прошедших с 00:00:00 (время UTC/GMT), 1 января 1970 года. |
| sign | string | Подпись безопасности уведомления о событии sign = MD5(key + t). Примечание: Tencent Cloud объединяет ключ шифрования **key** и `t`, вычисляет значение `sign` через MD5 и помещает его в сообщение уведомления. Когда ваш серверный сервер получает сообщение уведомления, он может подтвердить, правильный ли `sign` на основе того же алгоритма, и затем определить, действительно ли сообщение поступает с серверного сервера Tencent Cloud. |

> **Примечание:** Вы можете установить ключ обратного вызова в **Event Center** > [**Live Stream Callback**](https://console.tencentcloud.com/live/config/callback), который используется для аутентификации. Рекомендуем вам установить это поле для обеспечения безопасности данных. ### Параметры сообщения обратного вызова

| Параметр | Тип | Описание |
| --- | --- | --- |
| appid | int | [APPID](https://console.tencentcloud.com/developer) пользователя. |
| stream_id | string | Имя потока трансляции. |
| channel_id | string | Совпадает с именем потока трансляции. |
| task_id | string | ID задачи интеллектуального стирания. |
| data | [Структура data](#256eda27-74c0-4c1a-ad2c-79b78212b426) | Результаты обработки. |

#### **data**

| Параметр | Тип | Описание |
| --- | --- | --- |
| subtitle_tmp_res | [Массив subtitle_tmp_res](#7a55029a-406c-48a4-b967-09ece9df22ad) | Результаты текста. |

#### **subtitle_tmp_res**

| Параметр | Тип | Описание |
| --- | --- | --- |
| src_txt | string | Исходный текст распознавания аудио. |
| start_pts | int64 | Начальная точка временной метки аудиопотока, соответствующая src_txt, в миллисекундах (мс). |
| end_pts | int64 | Конечная точка временной метки аудиопотока, соответствующая src_txt, в миллисекундах (мс). |
| start_unix_time | int64 | Начальная точка UNIX времени трансляции, соответствующая src_txt. |
| end_unix_time | int64 | Конечная точка UNIX времени трансляции, соответствующая src_txt. |
| keywords_info | [Массив keywords_info](#c52f8023-60ce-4059-9e68-8fcbc275d7a0) | Список ключевых слов, при которых срабатывает подавление звука. |

#### **keywords_info**

| Параметр | Тип | Описание |
| --- | --- | --- |
| word | string | Ключевое слово, на которое сработало срабатывание. |
| begin_time | int64 | Начальная точка временной метки аудиопотока, соответствующая ключевому слову, в миллисекундах (мс). |
| end_time | int64 | Конечная точка временной метки аудиопотока, соответствующая ключевому слову, в миллисекундах (мс). |

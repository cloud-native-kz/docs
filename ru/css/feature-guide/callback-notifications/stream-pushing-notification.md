# Уведомление об обратном вызове при передаче потока

Обратный вызов при передаче потока информирует вас об успешности или прерывании передачи потока. Вам необходимо настроить адрес сервера для обратного вызова в шаблоне обратного вызова и связать шаблон с доменом передачи. После начала передачи через URL, созданный в этом домене, бэкенд Tencent Cloud отправит обратный вызов на указанный вами сервер.

Этот документ описывает параметры уведомления об обратном вызове при передаче потока, отправляемого вам CSS.

## Примечание

В данном руководстве предполагается, что вы знаете, как настроить обратные вызовы и получать уведомления о событиях от CSS. Подробнее см. [How to Receive Event Notification](https://intl.cloud.tencent.com/document/product/267/38080).

## Параметры событий передачи потока

### Тип события

| Тип события | Значение |
| --- | --- |
| Успешная передача | event_type = 1 |
| Прерывание передачи | event_type = 0 |

### Общие параметры обратного вызова

| Параметр | Тип | Описание |
| --- | --- | --- |
| t | int64 | Время истечения, которое является Unix-временем, когда срок действия подписи уведомления о событии истекает. Период действия уведомления об обратном вызове от Tencent Cloud составляет 10 минут по умолчанию. Если время, указанное значением `t` в уведомлении, прошло, это уведомление считается недействительным. Это предотвращает атаки воспроизведения сети. Значение `t` — это десятичный Unix-timestamp, то есть количество секунд, прошедших с 00:00:00 (UTC/GMT время) 1 января 1970 года. |
| sign | string | Подпись безопасности. sign = MD5(key + t). Примечание: Tencent Cloud объединяет ключ шифрования **key** и `t`, генерирует хеш MD5 объединённой строки и встраивает его в сообщения об обратном вызове. Ваш бэкенд-сервер может выполнить тот же расчёт при получении сообщения об обратном вызове. Если подпись совпадает, это означает, что сообщение поступило от Tencent Cloud. |

> **Примечание:** Вы можете установить ключ обратного вызова в разделе **Feature Configuration** > [Live Stream Callback](https://console.tencentcloud.com/live/config/callback), который используется для аутентификации. Рекомендуется установить это поле для обеспечения безопасности данных.![](https://staticintl.cloudcachetci.com/cms/backend-cms/35a19253cb6711f0a93d52540044a08e.png)

### Параметры обратного вызова

| Параметр | Тип | Описание |
| --- | --- | --- |
| appid | int | [APPID](https://console.tencentcloud.com/developer) пользователя |
| app | string | Доменное имя передачи |
| appname | string | Путь передачи |
| stream_id | string | Имя прямого потока |
| channel_id | string | Совпадает с названием прямого потока |
| event_time | int64 | Unix-время создания сообщения о событии |
| sequence | string | Номер последовательности сообщения, который идентифицирует передачу. Уведомления для передачи, независимо от того, касаются ли они успешной передачи или прерывания потока, имеют одинаковый номер последовательности. |
| node | string | IP точки доступа прямого потока |
| user_ip | string | IP пользователя при передаче |
| stream_param | string | Параметры URL при передаче пользователя |
| push_duration | string | Длительность передачи прерванного потока в миллисекундах |
| errcode | int | Код ошибки при передаче потока |
| errmsg | string | Сообщение об ошибке при передаче потока |
| set_id | int | Поступает ли передача с территории Китайской Народной Республики. 1-6: да; 7-200: нет. |
| width | int | Ширина видео. Значение этого параметра может быть `0`, если информация заголовка видео отсутствует в начале передачи. |
| height | int | Высота видео. Значение этого параметра может быть `0`, если информация заголовка видео отсутствует в начале передачи. |

### Причины прерывания потока

Список причин прерывания потока см. в разделе [Stream Interruption Records](https://intl.cloud.tencent.com/document/product/267/31083).

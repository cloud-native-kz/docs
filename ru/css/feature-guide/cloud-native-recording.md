# Облачная нативная запись

Сервис облачной нативной записи основан на [Cloud Streaming Services (CSS)](https://www.tencentcloud.com/document/product/267/37453) и [Cloud Application Rendering (CAR)](https://www.tencentcloud.com/document/product/1158).

Традиционные функции записи могут записывать только чистые живые изображения и не могут полностью записывать аудио и видеоконтент клиента с точки зрения зрителя. Для достижения эффекта "что видишь, то записываешь" нативный контент на клиенте может использоваться в качестве источника записи, а функциональный модуль "локальной записи экрана" на пользовательских терминалах может быть перенесен на экземпляр Windows CAR или контейнер Android. После отправки потока в Tencent Cloud CSS файлы записи могут быть созданы модулем записи. Это функция "облачной нативной записи". Она записывает содержимое экрана терминалов в облаке, снижая затраты на трансформацию и поддержку клиента. Кроме того, клиент APK/EXE/WEB может быть настроен для реализации сложных функций, которые невозможно реализовать на терминалах. Это расширяет возможности клиента APK/EXE/WEB в облаке и снижает сложность операций на терминалах.

## Сценарии применения

### Запись интерактивных живых трансляций и голосовых чат-комнат

Традиционные функции записи могут записывать только чистые живые изображения и не могут полностью записывать аудио и видеоконтент для пользователей с точки зрения зрителя. **Облачная нативная запись может записывать не только изображение живой трансляции, но и спецэффекты, такие как интерактивное совместное вещание и сообщения в чате, которые видят аудитория во время живой трансляции**. Это может комплексно улучшить иммерсивный опыт участия в записанном видео. Записанный контент может служить материалами UGC для вторичного распространения и основой для воспроизведения и просмотра.

### Запись онлайн-занятий

Онлайн-занятия являются расширением режима онлайн-обучения. Они действительно воспроизводят опыт взаимодействия оффлайн-занятий и предоставляют функции онлайн-вопросов и ответов, голосований и взаимодействия с доской. **Облачная нативная запись может записывать полный процесс обучения, анимированные учебные материалы, документы PowerPoint, содержание вопросов и ответов, содержание голосований и содержание взаимодействия с доской**. Записанные видео можно использовать как ресурсы для последующего совместного использования и вторичного распространения. По сравнению с простыми учебными видео видео, содержащие реальное содержание взаимодействия учащихся, более популярны и привлекательны.

### Распознавание и запись страниц приложений

Облачная нативная запись может записывать звук и изображение веб-страниц и страниц приложений клиента и создавать файлы записи, содержащие контент сообщений на доске объявлений, эффектов подарков и интерактивного совместного вещания с точки зрения зрителя. Функция интеллектуального определения [Tencent Cloud Media Processing Service (MPS)](https://www.tencentcloud.com/products/mps) может распознавать и извлекать текстовое содержание и аудио интерактивного совместного вещания с точки зрения зрителя. Такой контент может быть использован в качестве материалов для обучения LLM и анализа сценариев для повышения коэффициента конверсии сценариев.

## Архитектура и процесс

Облачная нативная запись использует контент APK/EXE/WEB в качестве источника записи для достижения эффекта записи "что видишь, то получаешь". Пользователям нужно только запустить экземпляр облачного телефона с параллелизмом или экземпляр CAR, отправить поток в CSS и использовать модуль записи для создания файлов записи. Они могут напрямую загружать файлы в VOD или COS или использовать их в качестве источника живой трансляции для распространения контента среди аудитории.

## Преимущества функции

| Преимущества | Описание |
| --- | --- |
| Широкий спектр применимых сценариев | Облачная нативная запись может использоваться в сценариях бизнеса, связанных с аудио/видео, таких как живые трансляции и запись. |
| Отличный эффект воспроизведения | Вся внешняя информация на клиенте, такая как спецэффекты в играх и обучении, может быть записана на 100% для воспроизведения фактического состояния. |
| Низкая стоимость интеграции | Пользователям не нужно интегрировать SDK на клиенте. Вместо этого они могут напрямую вызвать API на серверной части бизнеса. |
| Низкая стоимость трансформации | Стоимость трансформации клиента APK/EXE/WEB, работающего в облаке, низка. Исходный клиент APK/EXE/WEB может быть развернут непосредственно. |
| Мощные расширенные функции | Клиент APK/EXE/WEB, работающий в облаке, может быть настроен для реализации сложных функций, которые невозможно реализовать для клиента APK/EXE/WEB на терминалах. |

## Примеры использования клиентами

| Справочная информация о требованиях | Популярное во всей стране приложение для живых трансляций должно накапливать видеоконтент функций взаимодействия, таких как "якорь PK" и "голосовая чат-комната", в качестве материалов UGC для вторичного распространения. |
| --- | --- |
| Исходное решение | Сотрудники операций вручную записывают содержимое экрана с помощью мобильных телефонов во время живой трансляции. |
| Болевые точки | Видеопоток, полученный традиционным методом видеозаписи, представляет собой исходное изображение с точки зрения отдельного ведущего. Даже при записи смешанного потока эффекты анимации сообщений в чате не могут быть записаны. В определенных сценариях (таких как "голосовые чат-комнаты") содержимое экрана может быть записано только сотрудниками операций с помощью мобильных телефонов. В результате невозможно достичь автоматизированного и крупномасштабного накопления материалов. |
| Текущее решение | Используется функция облачной нативной записи Tencent Cloud. Настроенное приложение без входа работает в облаке. Когда ведущий начинает живую трансляцию, серверная часть бизнеса клиента автоматически отправляет сообщение облачному приложению. После получения сигнала облачное приложение входит в определенную комнату и воспроизводит видео. Функция живой записи Tencent Cloud используется для накопления видеоконтента в качестве материалов. |

## Предварительные условия

- Вы активировали CSS и добавили [доменное имя отправки потока](https://www.tencentcloud.com/document/product/267/35970).
- Вы активировали [CAR](https://www.tencentcloud.com/document/product/1158).
- Если требуется сервис записи, заранее активируйте [Video on Demand (VOD)](https://www.tencentcloud.com/document/product/266) или [Cloud Object Storage (COS)](https://www.tencentcloud.com/document/product/436).

## **Инструкции**

1. Если клиент является приложением EXE, загрузите файл клиента EXE в [консоль CAR](https://www.tencentcloud.com/zh/document/product/1158/49618). Если клиент является веб-браузером, создайте веб-приложение в [консоли CAR](https://console.tencentcloud.com/car) вместо загрузки веб-браузера. Если клиент является приложением APK, свяжитесь с сотрудниками бизнеса Tencent, чтобы включить функцию создания приложений APK в серверной части.
2. См. [Руководство пользователя CAR](https://www.tencentcloud.com/document/product/1158/49612) для создания проекта, покупки пакета параллелизма CAR, тестирования и выпуска проекта.
3. Вызовите API [StartPublishStream](https://www.tencentcloud.com/document/product/1158/56682) для отправки полного потока клиента в экземпляре CAR в [CSS](https://www.tencentcloud.com/document/product/267/37453).
4. [Запустите запись в консоли CSS](https://www.tencentcloud.com/document/product/267/34223) и сохраните полный поток клиента в качестве файла записи. Живой поток может быть использован для воспроизведения и распространения. Пользователи также могут использовать функцию интеллектуального определения [MPS](https://www.tencentcloud.com/document/product/1041/54517) для распознавания контента, такого как текст и аудио.

## Описание сборок

1. Клиент работает в облачном экземпляре, что будет включать [плату за параллелизм CAR](https://www.tencentcloud.com/document/product/1158/49606).
2. Отправка потока в CSS будет включать [плату за восходящий поток CSS](https://www.tencentcloud.com/document/product/267/2818). По умолчанию взимается только плата за нисходящее воспроизведение. Для сценариев с несбалансированным восходящим и нисходящим трафиком (воспроизведение нисходящего потока:восходящий поток < 10:1) дополнительная плата за отправку потока будет взиматься в соответствии с фактическим трафиком отправки потока, если ежедневная пиковая пропускная способность отправки потока превышает 100 Мбит/с.
3. Использование функции живой записи CSS будет включать [плату за живую запись](https://www.tencentcloud.com/document/product/267/39605).
4. Записанные видеофайлы могут быть сохранены в [VOD](https://console.tencentcloud.com/vod/overview) или [COS](https://console.tencentcloud.com/cos), что будет включать [плату за VOD](https://www.tencentcloud.com/document/product/266/2838) или [плату за COS](https://www.tencentcloud.com/document/product/436/16871).

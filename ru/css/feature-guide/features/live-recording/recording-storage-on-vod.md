# Хранение записей на платформе VOD

Запись живого потока сохраняет файлы, созданные путём мультиплексирования исходных потоков (без изменения информации, такой как аудио и видео данные и соответствующие временные метки), на платформу VOD.

## Примечания

- Вы можете использовать один из следующих методов для записи: [создать задачу записи](https://intl.cloud.tencent.com/document/product/267/37309) или [создать шаблон записи](https://intl.cloud.tencent.com/document/product/267/34223). Если вы создадите одновременно и шаблон записи, и задачу записи для одного и того же живого потока, он будет записан повторно.
- Поскольку существует небольшая задержка при запуске задачи записи после отправки потока, очень короткая отправка не может создать файлы записи. Рекомендуется, чтобы продолжительность каждой отправки для записи была более 10 секунд.

## Хранилище записей

Поскольку файлы записи хранятся на платформе VOD, сначала необходимо активировать [сервис VOD](https://intl.cloud.tencent.com/product/vod).

> **Примечание:** Правила именования созданных файлов записи см. в разделе [VodFileName](https://intl.cloud.tencent.com/document/product/267/30767#RecordParam).

## Поддерживаемые форматы записи

Поддерживаемые форматы записанных файлов: FLV/HLS/MP4/AAC, где AAC — это запись только звука.

**Потоковая передача определённых методов кодирования аудио и видео не поддерживается для записи. Статус поддержки различных методов кодирования аудио и видео для функции записи выглядит следующим образом** :

| Метод кодирования | Формат записи |  |  |
| --- | --- | --- | --- |
|  | FLV | HLS | MP4 |
| H.264 | Поддерживается | Поддерживается | Поддерживается |
| H.265 | Расширенная поддержка | Поддерживается | Поддерживается |
| H.266 | Расширенная поддержка | Поддерживается | Поддерживается |
| AV1 | Расширенная поддержка | Не поддерживается | Поддерживается |
| AAC | Поддерживается | Поддерживается | Поддерживается |
| MP3 | Поддерживается | Поддерживается | Поддерживается |
| OPUS | Расширенная поддержка | Поддерживается | Поддерживается |

> **Примечание:** "Расширенная поддержка" означает приватное расширение для потоковой передачи (как проигрыватель, так и инструмент ffmpeg требуют настройки), и это отличается от метода реализации ffmpeg. Рекомендуется скорректировать любой метод кодирования на потоки с кодированием H.264 или H.265, если метод кодирования не поддерживает или имеет только расширенную поддержку для формата записи.

## Сценарии использования записи

| Сценарий использования | Описание |
| --- | --- |
| Многоуровневая запись по доменному имени отправки и имени потока | Вы можете настроить, будет ли записываться поток на уровне доменного имени отправки и имени потока. |
| Запись в течение указанного периода времени | Вы можете вызвать API для установки времени начала и окончания записи потока в течение указанного периода времени. |
| Запись в реальном времени | Вы можете вызвать API для записи любого кадра потока в реальном времени. |
| Запись только звука | Вы можете использовать формат .aac для записи потоков только со звуком. |

## Включение записи для всех живых потоков под указанным доменным именем отправки

Параметры записи управляются шаблонами. Вы можете создавать шаблоны записи для различных сценариев и гибко управлять конфигурациями записи путём привязки шаблонов к различным доменным именам и именам потоков отправки.
После активации VOD вы можете записывать живые потоки под указанным доменным именем отправки двумя способами:

### Консоль CSS

1. Перейдите к разделу **Feature Configuration** > [**Live Recording**](https://console.tencentcloud.com/live/config/record) для создания шаблона записи.
2. Выберите [**Domain Management**](https://console.tencentcloud.com/live/domainmanage), чтобы добавить доменное имя, и нажмите **Manage** для привязки его к шаблону записи. Для получения дополнительной информации см. [Recording Configuration](https://intl.cloud.tencent.com/document/product/267/34224).

### API

1. Вызовите API [CreateLiveRecordTemplate](https://intl.cloud.tencent.com/document/product/267/30845) для установки как минимум одного формата записи, например `FlvParam`.
2. Вызовите API [CreateLiveRecordRule](https://intl.cloud.tencent.com/document/product/267/30846) для установки `DomainName` (доменное имя отправки) и `TemplateId` (возвращается на шаге 1). Вы можете оставить `AppName` и `StreamName` пустыми для записи всех потоков под доменным именем. После успешной обработки параметры вступят в силу примерно за 5–10 минут.

Вы также можете указать конкретный поток для записи.

Шаблон может быть привязан к различным доменным именам, приложениям и потокам отправки, но одно и то же доменное имя, приложение или поток отправки не может быть привязано к нескольким шаблонам. Если вы привяжете один и тот же поток к нескольким шаблонам (в редких случаях), вступит в силу только тот с наивысшим приоритетом. Приоритет шаблона определяется следующим образом.

| Приоритет | DomainName | AppName | StreamName |
| --- | --- | --- | --- |
| 1 | ✓ | ✓ | ✓ |
| 2 | ✓ | × | ✓ |
| 3 | ✓ | ✓ | × |
| 4 | ✓ | × | × |

✓ означает, что значение параметра не пусто, а × означает, что оно пусто.

## Отключение записи для конкретных потоков под доменным именем отправки

Когда вы уже настроили запись для доменного имени отправки, но не нуждаетесь в записи некоторых потоков под ним:

1. Вызовите API [CreateLiveRecordTemplate](https://intl.cloud.tencent.com/document/product/267/30845) без указания какого-либо формата записи.

```
https://live.tencentcloudapi.com/?Action=CreateLiveRecordTemplate&TemplateName=norecord&Description=test&<Common request parameters>
```

2. Перейдите в [консоль CSS](https://intl.cloud.tencent.com/document/product/267/34223#conect) или используйте API [CreateLiveRecordRule](https://intl.cloud.tencent.com/document/product/267/30846) для привязки приведённого выше шаблона записи к конкретным `DomainName` и `StreamName`.

> **Примечание:** Этот метод применим к сценариям, когда только несколько потоков не нуждаются в записи. Если есть слишком много потоков, рекомендуется использовать другое доменное имя отправки для управления ими, потому что: максимальное допустимое количество шаблонов записи или правил записи составляет 50. Управление по доменному имени отправки более гибко, так как шаблоны и правила записи не будут затронуты даже при изменении вашего бизнеса.

## Запись в течение указанного периода времени

Вы можете использовать API для указания времени начала, окончания и других параметров записи для некоторых потоков. Это отличается от использования предустановленного шаблона записи с указанными параметрами. Обычно API используются, когда шаблон записи не создан.

### API

Вызовите API [CreateRecordTask](https://intl.cloud.tencent.com/document/product/267/37309).

### Пример записи

- В простых сценариях вам нужно указать только `StreamName`, `DomainName`, `AppName` и `EndTime`.
Следующий пример кода создаёт задачу видеозаписи в формате .flv с 8:00 до 10:00 10 августа 2020 года с сегментами по 30 минут, и файлы записи будут сохранены навсегда.
**Пример входного кода:**

```
https://live.tencentcloudapi.com/?Action=CreateRecordTask&AppName=live&DomainName=mytest.live.push.com&StreamName=livetest&StartTime=1597017600&EndTime=1597024800&TemplateId=0&<Common request parameters>
```

- Вы также можете указать формат записи, тип записи и параметры хранения.
Следующий пример кода создаёт задачу записи в формате .mp4 с 8:00 до 10:00 10 августа 2020 года с часовыми сегментами, и файлы записи будут сохранены навсегда.
  1.1. Вызовите API [CreateLiveRecordTemplate](https://intl.cloud.tencent.com/document/product/267/30845) для создания шаблона записи.
**Пример входного кода:**

```
https://live.tencentcloudapi.com/?Action=CreateLiveRecordTemplate&TemplateName=templat&Description=test&Mp4Param.Enable=1&Mp4Param.RecordInterval=3600&Mp4Param.StorageTime=0&<Common request parameters>
```

**Пример выходного кода:**

```
{"Response": {  "RequestId": "839d12da-95a9-43b2-a9a0-03366d01b532",  "TemplateId": 17016}}
```

  1.2. Вызовите API [CreateRecordTask](https://intl.cloud.tencent.com/document/product/267/37309) для создания задачи записи.
**Пример входного кода:**

```
https://live.tencentcloudapi.com/?Action=CreateRecordTask&StreamName=livetest&AppName=live&DomainName=mytest.live.push.com&StartTime=1597017600&EndTime=1597024800&TemplateId=17016&<Common request parameters>
```

> **Примечание:** Для одного и того же живого потока не существует конфликта между запланированными задачами или между запланированной задачей и задачей записи другого типа. Другими словами, периоды времени нескольких запланированных задач могут перекрываться, и вы можете вызвать API для создания задачи записи в дополнение к включению конфигурации записи. Рекомендуется создать задачу записи заранее (например, за 1 час до или рано утром, если ваше событие происходит в течение дня), и установить время начала задачи немного раньше времени начала события.

## Запись в реальном времени

Если вы хотите записать любые кадры немедленно в процессе потоковой трансляции для создания фрагментов с выделением, вы можете вызвать API для включения записи в реальном времени.

```
https://live.tencentcloudapi.com/?Action=CreateRecordTask&StreamName=test&AppName=live&DomainName=mytest.live.push.com&EndTime=1597024800&<Common request parameters>
```

Примечания к записи в реальном времени:

- Убедитесь, что отправка продолжается при создании задачи записи.
- Вы можете вызвать API [StopRecordTask](https://intl.cloud.tencent.com/document/product/267/37307) для остановки задачи заранее.
- Это также поддерживается для потоков вне континентального Китая.

## Запись смешанного потока

При использовании функции записи смешанного потока обратитесь к документации [Live Stream Mixing](https://www.tencentcloud.com/document/product/267/37665) для понимания связанных знаний и этапов выполнения сервиса смешанного потока.

Для сценариев, использующих службы Live Cloud Mixing Services, сторона записи классифицирует смешанные потоки в две категории в зависимости от параметра `OutputStreamType` (тип выходного потока):

- Если `OutputStreamType` установлена на `0`, выходной поток находится в списке входных потоков, что означает, что не будет создан новый поток.
- Если `OutputStreamType` установлена на `1`, выходной поток не находится в списке входных потоков, что означает, что будет создан новый поток.

Предположим, что отправленные потоки — это A и B, а смешанный поток — это выходной поток C:

- Для случая, когда `OutputStreamType` равна `0`, если поток C является потоком A (с тем же именем потока, но со смешанным изображением), включение конфигурации записи по умолчанию создаст файлы записи для потока A (смешанное изображение) и потока B. Поскольку один и тот же идентификатор потока переиспользуется, исходная отправка потока A не будет создавать запись. Для случая, когда `OutputStreamType` равна `1`, включение конфигурации записи по умолчанию создаст файлы записи для потоков A, B и C (смешанное изображение).
- Если вы хотите записать только смешанный поток, вы можете вызвать API [CreateRecordTask](https://intl.cloud.tencent.com/document/product/267/37309). Обратите внимание, что если `OutputStreamType` установлена на `1`, параметр `StreamType` должен быть установлен на `1` при вызове этого API.

> **Примечание:** Запись смешанного потока не поддерживает смешивание потоков внутри и вне континентального Китая, так как это приведёт к ошибкам файла записи и повлияет на нормальное воспроизведение.

## Автоматическое сцепленное восстановление записи (запись с несколькими отправками)

Для решения проблемы прерываний передачи в потоках отправки, вызванных сетевыми помехами и другими факторами на стороне отправки, служба записи предоставляет функцию автоматического сцепления. Эта функция может записывать несколько прерванных потоков отправки в один файл, что удобно для просмотра прямой трансляции.

Эта функция разделяет аудио и видео данные по тегам `#EXT-X-DISCONTINUITY` в записи HLS. Из-за прерываний потока временные метки аудио и видео данных, видеокодек, аудиокодек и частоты дискретизации до и после маркировки могут отличаться. Проигрывателю необходимо обновить декодер для достижения бесшовного воспроизведения. Для использования этой функции проигрыватель должен поддерживать тег `#EXT-X-DISCONTINUITY`. В настоящее время тег поддерживается на встроенном проигрывателе и Safari на iOS, ExoPlayer на Android и проигрывателе HLS.js в сети, но не поддерживается на проигрывателе VLC.

После включения этой функции вам необходимо установить период истечения времени автоматического сцепления. Этот период составляет до 30 минут, что означает, что файлы записи между прерываниями продолжительностью до 30 минут могут быть объединены в один файл HLS после завершения последней отправки.

В настоящее время автоматическое сцепленное восстановление поддерживается только для формата HLS. Вы можете установить период истечения времени автоматического сцепления в разделе [Live Recording](https://intl.cloud.tencent.com/document/product/267/34223).

> **Примечание:** Этот режим не поддерживает живые потоки без аудиоданных. API `ComposeMedia` VOD можно использовать для составления видеофайлов. Для получения дополнительной информации см. [ComposeMedia](https://intl.cloud.tencent.com/document/product/266/34127). После включения возобновления записи HLS обратный вызов будет запущен только при создании файла записи, а не при прерывании потока.

## Получение файлов записи

Файлы записи автоматически сохраняются в системе VOD после создания и могут быть найдены через:

### Консоль VOD

Войдите в [консоль VOD](https://console.tencentcloud.com/vod/media), перейдите к разделу **Application Management** > **Enter Application** > **Audio/Video Management** для просмотра всех файлов, созданных записью.

### Уведомление о событии записи

Адрес обратного вызова записи может быть установлен в консоли или путём вызова API. Уведомление будет отправлено на адрес обратного вызова после создания файлов записи. После этого вы можете обратиться к уведомлению о событии записи [callback event message notification](https://intl.cloud.tencent.com/document/product/267/38080), чтобы сделать следующий шаг.

Поскольку обратные вызовы уведомления о событии эффективны, надёжны и происходят в реальном времени, рекомендуется использовать их для получения информации о файлах записи.

### Запрос через API VOD

Вы можете вызвать API [SearchMedia](https://intl.cloud.tencent.com/document/product/266/34179) VOD для фильтрации и запроса файлов записи.

> **Примечание:** Когда вы вызываете API [CreateRecordTask](https://intl.cloud.tencent.com/document/product/267/37309), параметры `stream_param`, содержащиеся в URL отправки, не будут возвращены в обратном вызове записи. Однако если вы используете другие методы записи, такие параметры будут возвращены в обратном вызове записи.

## Примечания по изменению конфигурации

Рекомендуется перезапустить отправку и проверить конфигурацию записи, если вы её изменили. Конфигурация вступает в силу в соответствии со следующими правилами:

- По умолчанию конфигурация вступает в силу в течение 10 минут.
- Конфигурация вступает в силу при начале прямой трансляции и не будет обновлена в процессе записи.
- В сценариях, где отправка продолжается длительное время (например, наблюдение за записью), вам необходимо прервать и перезапустить отправку для вступления конфигурации в силу.

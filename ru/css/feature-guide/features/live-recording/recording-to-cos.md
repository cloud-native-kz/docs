# Запись в COS

Живая запись — это сервис, который сохраняет исходные файлы живого потока, прошедшие инкапсуляцию аудио и видео (без изменения данных аудио, видео и соответствующих временных меток и т. д.) на платформу Cloud Object Storage (COS).

## Примечания

- Для записи в COS можно [создать задачу записи](https://intl.cloud.tencent.com/document/product/267/37309) или [создать шаблон записи](https://intl.cloud.tencent.com/document/product/267/34223). Если вы создадите и шаблон записи, и задачу записи для одного и того же потока, запись будет выполнена повторно.
- Поскольку при запуске задачи записи после передачи потока возникает небольшая задержка, очень короткий поток не может создать файл записи. Рекомендуется, чтобы длительность каждой передачи для записи была больше 10 секунд.

## Хранилище записей

Так как живая запись сохраняет файлы в [COS](https://www.tencentcloud.com/products/cos?lang=en&pg=), необходимо сначала активировать COS перед использованием функции живой записи. Для управления сроком хранения записанных файлов можно [настроить жизненный цикл](https://www.tencentcloud.com/document/product/436/14605?lang=en&pg=) для файлов, сохраненных в COS.

## Поддерживаемые форматы записи

Поддерживаемые форматы записанных файлов: FLV/HLS/MP4/AAC, где AAC — запись только аудио.

**Запись некоторых кодеков аудио и видео не поддерживается в живом потоке. Статус поддержки различных методов кодирования аудио и видео для функции записи следующий:**

| Метод кодирования | Формат записи |  |  |
| --- | --- | --- | --- |
|  | FLV | HLS | MP4 |
| H.264 | Поддерживается | Поддерживается | Поддерживается |
| H.265 | Расширенная поддержка | Поддерживается | Поддерживается |
| H.266 | Расширенная поддержка | Поддерживается | Поддерживается |
| AV1 | Расширенная поддержка | Не поддерживается | Поддерживается |
| AAC | Поддерживается | Поддерживается | Поддерживается |
| MP3 | Поддерживается | Поддерживается | Поддерживается |
| OPUS | Расширенная поддержка | Поддерживается | Поддерживается |

> **Примечание:** «Расширенная поддержка» означает частное расширение для живого потока (как плеер, так и инструмент ffmpeg требуют настройки) и отличается от метода реализации ffmpeg. Рекомендуется скорректировать любой метод кодирования на потоки H.264 или H.265 с перекодированием, если метод кодирования не поддерживает или имеет только расширенную поддержку для формата записи.

## Сценарии использования записи

| Сценарий использования | Описание |
| --- | --- |
| Многоуровневая запись по доменам передачи и именам потоков | Вы можете настроить запись потока на уровне домена передачи и имени потока. |
| Запись в течение указанного периода времени | Вы можете использовать API для установки времени начала и окончания для записи потока в течение указанного периода. |
| Запись в реальном времени | Вы можете использовать API для записи любой части потока в реальном времени. |
| Запись только аудио | Вы можете использовать формат .aac для записи только аудио потоков. |

## Включение записи для всех живых потоков в указанном домене передачи

Параметры записи управляются с помощью шаблонов. Вы можете создавать шаблоны записи для различных сценариев и гибко управлять конфигурациями записи, связывая шаблоны с различными доменами передачи и именами потоков.
После активации COS, если требуется записать потоки в прямом эфире в определенном домене передачи, используйте следующие методы:

### Консоль CSS

1. Перейдите в раздел **Feature Configuration** > [Live Recording](https://console.tencentcloud.com/live/config/record), нажмите на [Save to COS](https://console.intl.cloud.tencent.com/live/config/record/cos) и добавьте шаблон конфигурации записи.
2. Перейдите в раздел [Domain Management](https://console.tencentcloud.com/live/domainmanage), добавьте домен и нажмите **Manage**, чтобы связать его с шаблоном записи. Дополнительные сведения см. в разделе [Recording Configuration](https://intl.cloud.tencent.com/document/product/267/34224).

### API

1. Вызовите API [CreateLiveRecordTemplate](https://intl.cloud.tencent.com/document/product/267/30845), чтобы установить по меньшей мере один формат записи, например `FlvParam`.
2. Вызовите API [CreateLiveRecordRule](https://intl.cloud.tencent.com/document/product/267/30846), чтобы установить `DomainName` (домен передачи) и `TemplateId` (возвращается на шаге 1). Вы можете оставить `AppName` и `StreamName` пустыми для записи всех потоков в домене. После успешной обработки параметры вступят в силу примерно через 5–10 минут.

Вы также можете указать конкретный поток для записи.

Шаблон может быть привязан к различным доменам передачи, приложениям и потокам, но один и тот же домен передачи, приложение или поток не может быть привязан к нескольким шаблонам. Если вы привяжете один и тот же поток к нескольким шаблонам (в редких случаях), вступит в силу только тот, у которого наивысший приоритет. Приоритет шаблона определяется следующим образом.

| Приоритет | DomainName | AppName | StreamName |
| --- | --- | --- | --- |
| 1 | ✓ | ✓ | ✓ |
| 2 | ✓ | × | ✓ |
| 3 | ✓ | ✓ | × |
| 4 | ✓ | × | × |

✓ означает, что значение параметра не пусто, × означает, что оно пусто.

## Отключение записи для конкретных потоков в домене передачи

Если вы уже настроили запись для домена передачи, но не хотите записывать некоторые потоки в домене, выполните следующие действия:

1. Вызовите API [CreateLiveRecordTemplate](https://intl.cloud.tencent.com/document/product/267/30845) без указания какого-либо формата записи.

```
https://live.tencentcloudapi.com/?Action=CreateLiveRecordTemplate&TemplateName=norecord&Description=test&<Common request parameters>
```

2. Перейдите в [консоль CSS](https://intl.cloud.tencent.com/document/product/267/34223#conect) или используйте API [CreateLiveRecordRule](https://intl.cloud.tencent.com/document/product/267/30846), чтобы связать приведенный выше шаблон записи с конкретными `DomainName` и `StreamName`.

> **Примечание:** Этот метод применяется в сценариях, где только несколько потоков не требуют записи. Если потоков слишком много, рекомендуется использовать другой домен передачи для их управления, потому что: максимально допустимое количество шаблонов записи или правил записи составляет 50. Управление по доменам передачи более гибкое, так как шаблоны записи и правила записи не будут затронуты при изменении вашего бизнеса.

## Запись в течение указанного периода времени

Вы можете использовать API для указания времени начала, окончания и других параметров записи для некоторых потоков. Это отличается от использования предварительно заданного шаблона записи с указанными параметрами. Обычно API используются, когда шаблон записи не создан.

### API

Вызовите API [CreateRecordTask](https://intl.cloud.tencent.com/document/product/267/37309).

### Пример записи

- В простых сценариях необходимо указать только `StreamName`, `DomainName`, `AppName` и `EndTime`.
Следующий пример кода создает задачу записи видео в формате .flv с 8:00 до 10:00 10 августа 2020 г., с сегментами продолжительностью 30 минут, и файлы записи будут сохранены постоянно.
**Пример кода ввода:**

```
https://live.tencentcloudapi.com/?Action=CreateRecordTask&AppName=live&DomainName=mytest.live.push.com&StreamName=livetest&StartTime=1597017600&EndTime=1597024800&TemplateId=0&<Common request parameters>
```

- Вы также можете указать формат записи, тип записи и параметры хранилища.
Следующий пример кода создает задачу записи в формате .mp4 с 8:00 до 10:00 10 августа 2020 г., с сегментами продолжительностью 1 час, и файлы записи будут сохранены постоянно.
  1.1. Вызовите API [CreateLiveRecordTemplate](https://intl.cloud.tencent.com/document/product/267/30845), чтобы создать шаблон записи.
**Пример кода ввода:**

```
https://live.tencentcloudapi.com/?Action=CreateLiveRecordTemplate&TemplateName=templat&Description=test&Mp4Param.Enable=1&Mp4Param.RecordInterval=3600&<Common request parameters>
```

**Пример кода вывода:**

```
{"Response": {  "RequestId": "839d12da-95a9-43b2-a9a0-03366d01b532",  "TemplateId": 17016}}
```

  1.2. Вызовите API [CreateRecordTask](https://intl.cloud.tencent.com/document/product/267/37309), чтобы создать задачу записи.
**Пример кода ввода:**

```
https://live.tencentcloudapi.com/?Action=CreateRecordTask&StreamName=livetest&AppName=live&DomainName=mytest.live.push.com&StartTime=1597017600&EndTime=1597024800&TemplateId=17016&<Common request parameters>
```

> **Примечание:** Для одного потока в прямом эфире запланированные задачи и другие типы задач записи не будут конфликтовать друг с другом. Другими словами, периоды времени нескольких запланированных задач могут перекрываться, и вы можете вызвать API для создания задачи записи в дополнение к включению конфигурации записи. Рекомендуется создавать задачу записи заранее (например, за 1 час до события или в начале дня, если ваше событие происходит днем) и устанавливать время начала задачи немного раньше времени начала события.

## Запись в реальном времени

Если вы хотите записать части потока в прямом эфире в реальном времени для создания отрывков с лучшими моментами, вы можете вызвать API для включения записи в реальном времени.

```
https://live.tencentcloudapi.com/?Action=CreateRecordTask&StreamName=test&AppName=live&DomainName=mytest.live.push.com&EndTime=1597024800&<Common request parameters>
```

Примечания к записи в реальном времени:

- Убедитесь, что передача идет при создании задачи записи.
- Вы можете вызвать API [StopRecordTask](https://intl.cloud.tencent.com/document/product/267/37307), чтобы остановить задачу раньше.
- Это также поддерживается для потоков вне китайского материка.

## Запись смешанного потока

При использовании функции записи смешанного потока см. раздел [Live Stream Mixing](https://www.tencentcloud.com/document/product/267/37665), чтобы ознакомиться с соответствующими концепциями и операциями сервиса смешивания потоков.

Для сценариев, в которых используется смешивание потоков в облаке CSS, сторона записи классифицирует смешанные потоки на две категории на основе параметра `OutputStreamType`:

- Если `OutputStreamType` установлен в `0`, выходной поток находится в списке входных потоков, что означает, что новый поток не будет создан.
- Если `OutputStreamType` установлен в `1`, выходной поток не находится в списке входных потоков, что означает, что будет создан новый поток.

Предположим, что переданные потоки — это A и B, а смешанный поток — это выходной поток C:

- Для случая, когда `OutputStreamType` равен `0`, если поток C — это поток A (с тем же именем потока, но смешанным изображением потока), включение конфигурации записи будет создавать файлы записи для потока A (смешанное изображение потока) и потока B по умолчанию. Поскольку один и тот же ID потока повторно используется, исходная передача потока A не будет создавать запись. Для случая, когда `OutputStreamType` равен `1`, включение конфигурации записи будет создавать файлы записи для потоков A, B и C (смешанное изображение потока) по умолчанию.
- Если вы хотите записать только смешанный поток, вы можете вызвать API [CreateRecordTask](https://intl.cloud.tencent.com/document/product/267/37309). Обратите внимание, что если `OutputStreamType` установлен в `1`, при вызове этого API параметр `StreamType` должен быть установлен в `1`.

> **Примечание:** Запись смешанного потока не поддерживает смешивание потоков в материковом Китае и вне его, так как это приведет к ошибкам файлов записи и повлияет на нормальное воспроизведение.

## Автоматическая сшиваемая запись (запись с несколькими передачами)

Для решения проблемы внезапных прерываний передачи потока, вызванных сетевыми помехами и другими факторами на стороне потока, сервис живой записи предоставляет функцию автоматического сшивания, при которой несколько прерванных переданных потоков записываются в один файл, что удобно для воспроизведения потока в прямом эфире.

Эта функция разделяет данные аудио и видео по тегам **#EXT-X-DISCONTINUITY** в записи HLS. Из-за прерываний потока временные метки данных аудио и видео, видеокодека, аудиокодека и частоты дискретизации до и после тегирования могут быть различными. Плеер должен обновить декодер для достижения безупречного воспроизведения. Для использования этой функции плеер должен поддерживать тег **#EXT-X-DISCONTINUITY**. В настоящее время тег поддерживается встроенным плеером и Safari на iOS, ExoPlayer на Android и плеером HLS.js на веб-сайте, но не поддерживается на плеере VLC.

После включения этой функции необходимо установить период тайм-аута автоматического сшивания. Этот период может составлять до 30 минут, что означает, что при наличии прерываний продолжительностью до 30 минут во время записи потока в прямом эфире файлы записи будут сшиты в один файл HLS после завершения последней передачи.

В настоящее время автоматическая сшиваемая запись поддерживается только для формата HLS. Вы можете установить период тайм-аута автоматического сшивания в [Live Recording](https://intl.cloud.tencent.com/document/product/267/34223).

> **Примечание:** Этот режим не поддерживает потоки в прямом эфире без данных аудио. API `ComposeMedia` VOD можно использовать для композиции видеофайлов. Дополнительные сведения см. в разделе [ComposeMedia](https://intl.cloud.tencent.com/document/product/266/34127). После включения возобновления записи HLS обратный вызов будет срабатывать только при создании файла записи, а не при прерывании потока.

## Получение файлов записи

Файлы записи автоматически сохраняются в COS после создания и могут быть найдены следующими способами:

### В консоли COS

Войдите в [консоль COS](https://console.intl.cloud.tencent.com/cos/bucket) и нажмите **имя бакета** из списка для просмотра и поиска всех файлов, созданных записью.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/efa9ff7856e011ee84f2525400494e51.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f505f6e056e011ee84f2525400494e51.png)

### Запрос через API COS

Вы можете вызвать API COS [List Objects](https://www.tencentcloud.com/document/product/436/30614?lang=en&pg=), чтобы отфильтровать и запросить файлы записи.

### Использование уведомления о событии записи

Адрес обратного вызова записи можно установить в консоли или путем вызова API. Уведомление будет отправлено на адрес обратного вызова после создания файлов записи. После этого вы можете обратиться к [Recording Event Notification](https://www.tencentcloud.com/document/product/267/38082?lang=en&pg=) для выполнения следующих шагов.

Рекомендуется использовать обратные вызовы уведомления о событиях для получения файлов записи, поскольку они эффективны, надежны и отправляются в реальном времени.

## Примечания при изменении конфигурации

Если вы изменяете конфигурацию живой записи, рекомендуется перезапустить передачу и проверить конфигурацию записи. Конфигурация вступает в силу по следующим правилам:

- По умолчанию конфигурация вступает в силу через 10 минут.
- Конфигурация вступает в силу при начале передачи в прямом эфире и не будет обновлена, если запись уже выполняется.
- В сценариях, когда передача продолжается длительное время (например, запись с видеонаблюдения), необходимо прервать и перезапустить передачу для вступления конфигурации в силу.

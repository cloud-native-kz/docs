# Живая переупаковка и трансляция

## Живая переупаковка

Живая переупаковка — это процесс преобразования исходного потока, передаваемого с сайта живого потокового вещания (обычно использующего протокол RTMP), в различные форматы контейнеров в облаке перед отправкой зрителям.

### Поддерживаемые форматы выходного контейнера

- RTMP
- FLV
- HLS
- DASH
- TS stream

### Поддерживаемые типы выходных данных

- Выходные данные только с аудио: удаляет видеофайлы и создает выходные данные только с аудио. Форматы контейнеров описаны выше.
- Выходные данные только с видео: удаляет аудиофайлы и создает выходные данные только с видео. Форматы контейнеров описаны выше.

### Поддерживаемые схемы шифрования медиа

- **FairPlay**
Переупаковка HLS поддерживает решение DRM Apple FairPlay.
- **Widevine**
Переупаковка DASH поддерживает решение DRM Google Widevine.
- **Универсальное шифрование AES-128 для HLS**
Переупаковка HLS поддерживает универсальные схемы шифрования AES-128.

## Живая трансляция

Живая трансляция (включая как видеотранскодирование, так и аудиотранскодирование) — это процесс трансляции исходного потока, передаваемого с сайта живого потокового вещания, в потоки с различными кодеками, разрешениями и битрейтами в облаке перед отправкой зрителям. Это помогает удовлетворить потребности воспроизведения в различных сетевых условиях и на различных устройствах.

### Типичные варианты использования

- Исходный видеопоток может быть транскодирован в потоки с различным качеством. Зрители могут выбирать видеопотоки различных битрейтов в соответствии с условиями их сети для обеспечения плавного воспроизведения.
- Вы можете добавить пользовательский водяной знак к исходному видеопотоку в целях авторского права и маркетинга.
- Видеопоток может быть транскодирован в видеокодек с более высоким коэффициентом сжатия. Например, при наличии большого количества зрителей вы можете преобразовать видеопоток H.264 в поток H.265, который имеет более высокий коэффициент сжатия, тем самым снижая использование пропускной способности и затраты.
- Исходный видеопоток может быть транскодирован в различные кодеки, подходящие для воспроизведения на специальных устройствах. Например, если видеопоток H.264 не может быть воспроизведен в реальном времени из-за проблем с производительностью, вы можете транскодировать его в формат .mpeg для декодирования и воспроизведения в реальном времени.

### Параметры видеотранскодирования

| Тип параметра | Описание |
| --- | --- |
| Видеокодек | Поддерживаемые видеокодеки: H.264, H.265, AV1 |
| Видеопрофиль | Поддерживаемые видеопрофили: Baseline, Main, High |
| Видеобитрейт кодирования | Поддерживаемый диапазон выходного видеобитрейта: 101 Кбит/с — 8000 Кбит/с. Исходный битрейт будет выходным битрейтом, если вы укажете выходной битрейт выше исходного. Например, если указанный выходной битрейт составляет 3000 Кбит/с, но исходный битрейт входного потока составляет только 2000 Кбит/с, то выходной битрейт будет 2000 Кбит/с. |
| Видеочастота кадров кодирования | Поддерживаемый диапазон выходной частоты кадров видео: 1–60 кадров/с. Исходная частота кадров будет выходной частотой кадров, если вы укажете выходную частоту кадров выше исходной. Например, если указанная выходная частота кадров составляет 30 кадров/с, но исходная частота кадров входного потока составляет только 20 кадров/с, то выходная частота кадров будет 20 кадров/с. |
| Видеоразрешение | Поддерживаемый диапазон ширины: 0–3000. Поддерживаемый диапазон высоты: 0–3000. Вы можете указать только ширину, и высота будет масштабироваться пропорционально. Вы можете указать только высоту, и ширина будет масштабироваться пропорционально. |
| Длина видеокадра GOP | Поддерживаемый диапазон длины видеокадра GOP: 1–10 с; рекомендуемый диапазон: 2–4 с. |
| Метод контроля видеобитрейта | Поддерживаемые методы контроля видеобитрейта: фиксированный битрейт (CBR), динамический битрейт (VBR) |
| Поворот видеоизображения | Исходное видео можно повернуть по часовой стрелке на: 90 градусов, 180 градусов, 270 градусов |

## Транскодирование с использованием Top Speed Codec

Функция транскодирования Top Speed Codec (TSC) основана на многолетнем опыте аудио- и видеокодирования, интеллектуальном распознавании сценариев, динамическом кодировании, трехуровневой (CTU/line/frame) модели точного контроля битрейта и других технологиях, обеспечивая потоковую передачу более высокой четкости при более низких битрейтах (в среднем на 50% меньше) для живого вещания и видео по требованию.

### Варианты использования

Если битрейт прямой трансляции высокий и изображение сложное, вы можете использовать интеллектуальную технологию динамического кодирования и точную модель контроля битрейта, чтобы сохранить высокую четкость при низком битрейте, обеспечивая, что качество видеоизображения, просматриваемого зрителем, соответствует исходному качеству.

## Преимущества

Поскольку пользователи различных видеоплатформ предъявляют все возрастающие требования к высокому качеству видеоисточника и плавному опыту просмотра, в современной индустрии живого вещания разрешение 1080p и битрейт 3–10 Мбит/с постепенно стали основной конфигурацией, а расходы на пропускную способность составляют значительную часть общих затрат видеоплатформы. В этом случае снижение видеобитрейта может эффективно снизить расходы на пропускную способность.
**Пример:**
Предположим, вы провели живую сессию с битрейтом 3 Мбит/с в течение 4 часов с 200 зрителями. Кодек — H.264, и транскодирование TSC не используется. Пиковая пропускная способность составляет 600 Мбит/с. Стоимость пропускной способности для этой живой сессии составляет 600 x 0,2118 = 127,08 USD.

- Если используется транскодирование TSC для снижения битрейта, понесенные платежи за пропускную способность составят примерно 127,08 x (100% — 30%) = 88,956 USD.
- Платежи за транскодирование TSC: 0,0443 x 240 = 10,632 USD (опубликованная цена без скидок).
- Общие платежи: 88,956 + 10,632 = 99,588 USD.

Таким образом, транскодирование TSC может эффективно снизить расходы платформы на пропускную способность при обеспечении лучшего опыта просмотра.

### Ключевые параметры

Параметры транскодирования TSC настраиваются в основном так же, как параметры стандартного транскодирования живого вещания. Для получения дополнительной информации см. [Параметры видеотранскодирования](https://www.tencentcloud.com/document/product/267/31561#video-transcoding-parameters).

## Живой водяной знак

Вы можете использовать живой водяной знак для добавления предустановленного логотипа к исходному видеопотоку в целях авторского права и маркетинга.

### Параметры водяного знака

Основные параметры водяного знака включают положение водяного знака и размер водяного знака, которые определяются параметрами `XPosition`, `YPosition`, `Width` и `Height`, как подробно описано ниже:

- **XPosition**: смещение по оси X, которое указывает процентное расстояние от левого края водяного знака до левого края видео.
- **YPosition**: смещение по оси Y, которое указывает процентное расстояние от верхнего края водяного знака до верхнего края видео.
- **Width**: ширина водяного знака или его процент от ширины видео живого вещания.
- **Height**: высота водяного знака или его процент от высоты видео живого вещания.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/db198d105d0a11ee9ff8525400d917da.png)

> **Примечание:** Если вы включите многобитрейтное транскодирование для потока (т. е. один исходный поток транскодируется в потоки различных разрешений) и хотите добавить водяной знак, вы можете установить его процентное положение по осям X и Y в [консоли CSS](https://console.tencentcloud.com/live) или через соответствующий [API](https://www.tencentcloud.com/document/product/267/30826), и система автоматически определит положение водяного знака.

### Пример параметров водяного знака

Предположим, что разрешение выходного изображения составляет 1920 x 1080, разрешение водяного знака составляет 320 x 240, XPosition = 5, YPosition = 5 и Width = 10 (единица: процент).
Абсолютное положение и размер водяного знака на выходном видео показаны ниже:

```
XPosition_pixel = 1920 x 5% = 96YPosition_pixel = 1080 x 5% = 54Width_pixel = 1920 x 10% = 192Height_pixel = 192 x 240/320 = 144
```

Водяной знак находится в 96 пикселях от левого края выходного видеоизображения и в 54 пикселях от верхнего края изображения. Размер водяного знака составляет 192 x 144.

### Способ использования

Вы можете добавить водяной знак в [консоли CSS](https://console.tencentcloud.com/live) или через [серверный API](https://www.tencentcloud.com/document/product/267/30760) в зависимости от ваших потребностей в бизнесе.

#### Консоль CSS

1. Перейдите в **Feature Configuration** > [Live Watermarking](https://console.tencentcloud.com/live/config/watermark) для добавления шаблона конфигурации водяного знака, установки параметров водяного знака и создания соответствующего ID шаблона водяного знака. Подробные инструкции см. в [Watermark Template Configuration](https://intl.cloud.tencent.com/document/product/267/31073).
2. Выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage) для добавления доменного имена и нажмите **Manage** > **Template Configuration** для привязки его к шаблону водяного знака. Для получения дополнительной информации см. [Watermark Configuration](https://intl.cloud.tencent.com/document/product/267/31064).

#### Вызов API

1. Вызовите [AddLiveWatermark](https://intl.cloud.tencent.com/document/product/267/30826) API для добавления водяного знака путем установки имени водяного знака и других параметров.
2. Вызовите [CreateLiveWatermarkRule](https://intl.cloud.tencent.com/document/product/267/30825) API для создания правила водяного знака. Установите `DomainName` (доменное имя прямой трансляции) и `WatermarkId` (возвращено на шаге 1). Используйте тот же `AppName`, что и в адресах прямой трансляции и воспроизведения, который по умолчанию составляет `live`.

Примечание: Использование функции водяного знака будет взимать стандартные платежи за транскодирование.

## Настройка параметров транскодирования

### Способ использования

Вы можете установить параметры транскодирования через [консоль CSS](https://console.tencentcloud.com/live) или [серверные API](https://www.tencentcloud.com/document/product/267/30760). В любом случае вы будете использовать в основном шаблоны водяных знаков, шаблоны транскодирования и правила транскодирования для настройки.

#### Консоль CSS

1. Перейдите в **Feature Configuration** > [Live Transcoding](https://console.tencentcloud.com/live/config/transcode) для добавления шаблона конфигурации транскодирования. Вы можете добавить шаблон [standard transcoding](https://intl.cloud.tencent.com/document/product/267/31071) или [TSC transcoding](https://intl.cloud.tencent.com/document/product/267/31071).
2. Создайте соответствующий тип транскодирования и установите параметры транскодирования по мере необходимости. Вы можете использовать параметры по умолчанию системы, и будет создан соответствующий ID шаблона транскодирования.
3. Выберите [Domain Management](https://console.tencentcloud.com/live/domainmanage), найдите целевое доменное имя воспроизведения и нажмите **Manage** > **Template Configuration** для привязки его к шаблону транскодирования. Для получения дополнительной информации см. [Transcoding Configuration](https://intl.cloud.tencent.com/document/product/267/31062).

#### Вызов API

1. Вызовите [CreateLiveTranscodeTemplate](https://intl.cloud.tencent.com/document/product/267/30790) API для установки параметров типа транскодирования.
2. Вызовите [CreateLiveTranscodeRule](https://intl.cloud.tencent.com/document/product/267/30791) API для установки параметров `DomainName` (доменное имя воспроизведения) и `TemplateId` (возвращено на шаге 1). Введите пустую строку в `AppName` и `StreamName` в качестве подстановочного символа для согласования всех потоков в доменном имени. Вы также можете привязать шаблон транскодирования к различным названиям потоков для включения транскодирования для этих живых потоков.
3. Каждый шаблон транскодирования имеет **уникальное имя шаблона транскодирования**, которое используется как уникальный ID для воспроизведения выходного потока. Вы можете поместить имя шаблона транскодирования после ID потока в адрес воспроизведения для получения выходного потока, соответствующего шаблону транскодирования.

> **Примечание:** Правило транскодирования используется для установки, должно ли быть включено указанное шаблон транскодирования для указанного доменного имени или потока. Доменное имя воспроизведения может быть использовано для получения шаблона транскодирования только после создания соответствующего правила транскодирования. Если правило транскодирования не было создано, адрес воспроизведения, составленный с использованием имени шаблона транскодирования, недействителен.

### Пример

```
**Адрес воспроизведения = доменное имя воспроизведения + путь воспроизведения + ID потока_имя шаблона транскодирования + строка проверки подлинности**
```

Для прямой трансляции с ID потока `1234_test` исходный поток и потоки с водяными знаками различных битрейтов могут быть воспроизведены через следующие адреса:

- **Исходный поток:** `http://liveplay.tcloud.com/live/1234_test.flv?authentication string`
- **Поток стандартного транскодирования (с водяным знаком):** `http://liveplay.tcloud.com/live/1234_test_sd.flv?authentication string`
- **Поток транскодирования TSC (с водяным знаком):** `http://liveplay.tcloud.com/live/1234_test_hd.flv?authentication string`

> **Примечание:** Для воспроизведения потока с водяным знаком необходимо привязать соответствующее доменное имя прямой трансляции к созданному шаблону водяного знака.

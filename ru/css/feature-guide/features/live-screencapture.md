# Захват снимков в реальном времени

Функция захвата снимков в реальном времени делает снимки экрана трансляции в реальном времени через регулярные промежутки и генерирует изображения. Вы можете получить информацию о снимке экрана через уведомление обратного вызова. Эти снимки имеют различные применения, такие как обнаружение порнографии и миниатюры.

## Процесс захвата снимков в реальном времени

![](https://staticintl.cloudcachetci.com/cms/backend-cms/82f933b0508511ee94c3525400d793d0.png)

Общий процесс:

1. Настройте функцию захвата снимков в консоли или через API TencentCloud.
2. Запустите живую трансляцию.
3. Служба захвата снимков генерирует данные снимков согласно конфигурации и сохраняет их в COS.
4. Информация о созданном снимке возвращается в обратном вызове.

### Конфигурация интервала захвата снимков

Вы можете указать частоту захвата снимков в зависимости от потребностей вашего бизнеса, т. е. интервал захвата снимков (SnapshotInterval). Доступный диапазон составляет 2–300 секунд с интервалом по умолчанию 2 секунды.

### Конфигурация ширины и высоты снимка

Служба захвата снимков поддерживает захват снимков с указанной шириной и высотой:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/82e02e16508511eeabd75254005810a4.png)

> **Примечание:** Если вам не нужно указывать ширину и высоту, ширина и высота снимка по умолчанию (установленные на 0) будут соответствовать ширине и высоте переданного видеопотока, и вы можете пропустить приведённые ниже инструкции конфигурации и перейти к следующему разделу.

Сначала рассмотрите следующие три концепции ширины и высоты:

- Ширина и высота трансляции, т. е. ширина и высота видео прямой трансляции, которые в этом документе обозначены как (X, Y).
- Настроенная ширина и высота, т. е. ширина и высота, настроенные в консоли/через API TencentCloud, которые в этом документе обозначены как (W, H).
- Ширина и высота снимка, т. е. ширина и высота снимка, созданного службой захвата снимков, которые в этом документе обозначены как (N, M).

**Служба захвата снимков поддерживает следующие конфигурации:**

- Если ширина и высота не установлены, то по умолчанию используется (W, H) = (0, 0). Ширина и высота снимка совпадают с шириной и высотой трансляции, т. е. (N, M) = (X, Y).
- Если установлена только ширина W, то ширина снимка будет N = W, а высота снимка масштабируется пропорционально, т. е. M = N / X * Y.
- Если установлена только высота H, то высота снимка будет M = H, а ширина снимка масштабируется пропорционально, т. е. N = M / Y * X.
- Если (W, H) установлены одновременно, то ширина и высота снимка совпадают с настроенной шириной и высотой, т. е. (N, M) = (W, H).

**Автоматический обмен настроенной ширины и высоты подходит для следующего сценария:**

- Если W меньше, чем H, оба W и H больше 0, и X больше, чем Y при трансляции, то настроенная ширина меньше высоты, но ширина трансляции больше высоты.

> **Примечание:** В этом случае при прямом захвате снимка он будет искажён. Чтобы избежать искажения, бэкэнд службы захвата снимков в реальном времени автоматически обменяет значения W и H, чтобы гарантировать, что настроенное соотношение сторон совпадает с соотношением сторон прямой трансляции.

## Уведомление о событии для захвата снимков в реальном времени

Для конфигурации уведомления о событии см. раздел [Уведомление о событии](https://www.tencentcloud.com/document/product/267/38080). Уведомление обратного вызова захвата снимков отправляется на предварительно настроенный сервер получения через протокол HTTP POST в формате JSON.

### Поля обратного вызова захвата снимков

| Имя поля | Тип | Описание |
| --- | --- | --- |
| event_type | int | Тип информации обратного вызова, всегда 200 для обратного вызова захвата снимков |
| stream_id | string | Имя потока |
| channel_id | string | То же, что и имя потока |
| create_time | int64 | Временная метка UNIX при создании снимка |
| file_size | int | Размер файла снимка в байтах |
| width | int | Ширина снимка в пикселях |
| height | int | Высота снимка в пикселях |
| pic_url | string | Путь к файлу снимка /path/name.jpg. Дополнительную информацию см. в разделе [Детали поля](#.3Cspan-id.3D.22jump.22.3Efield-details.3C.2Fspan.3E) ниже |
| pic_full_url | string | Полный URL снимка. Дополнительную информацию см. в разделе [Детали поля](#.3Cspan-id.3D.22jump.22.3Efield-details.3C.2Fspan.3E) ниже |
| sign | string | Подпись обратного вызова. Дополнительную информацию см. в разделе [Уведомление о событии](https://www.tencentcloud.com/document/product/267/38080) |
| t | int64 | Временная метка UNIX при истечении подписи обратного вызова. Дополнительную информацию см. в разделе [Уведомление о событии](https://www.tencentcloud.com/document/product/267/38080) |

### Детали поля

- `pic_url`
  - детали:
    - path: year-month-day
    - name: live stream name-screenshot-hour-minute-second-widthxheight.jpg
  - Пример:

```
 /2018-12-17/stream_name-screenshot-19-06-59-640x352.jpg
```

Это поле можно использовать для составления пользовательского доменного имени COS CDN. Если вам не нужно доменное имя CDN, используйте непосредственно pic_full_url.

- `pic_full_url`
  - детали:
    - `http://COS`` доменное имя+pic_url`
  - Пример:

```
 http://testbucket-1234567890.cos.region.myqcloud.com/2018-12-17/stream_name-screenshot-19-06-59-640x352.jpg
```

# Редактирование основных моментов прямой трансляции

Редактирование основных моментов прямой трансляции зависит от возможностей сдвига времени, что означает, что во время или после прямой трансляции пользователь может выбрать захватывающий сегмент из прошлого контента прямой трансляции, создать адрес воспроизведения со сдвигом времени и удобно перераспределить основные моменты. Кроме того, благодаря возможностям обработки мультимедиа основные моменты прямой трансляции можно консолидировать в хранилище объектов для долгосрочного сохранения.

В этой статье рассказывается о том, как использовать сдвиг времени прямой трансляции для создания основных моментов из контента прямой трансляции, а также как использовать возможности обработки мультимедиа для консолидации и хранения клипов основных моментов прямой трансляции.

## Предварительные требования

- Для использования функции вырезания основных моментов прямой трансляции убедитесь, что вы создали [шаблон сдвига времени](https://www.tencentcloud.com/document/product/267/53312), привязали домен трансляции и успешно выполнили трансляцию.
- Для использования возможности редактирования и консолидации прямой трансляции убедитесь, что вы активировали сервис Tencent Cloud [MPS](https://console.intl.cloud.tencent.com/mps). Консолидированный контент будет сохранен в [сервисе COS](https://www.tencentcloud.com/document/product/436/32955?has_map=1&lang=zh&pg=), и активирование сервиса MPS автоматически активирует сервис COS.

## Редактирование основных моментов прямой трансляции

### Руководство по работе с консолью

На основе ваших потребностей в бизнесе вы можете настроить воспроизведение со сдвигом времени. Дополнительную информацию см. в документе [Информация об индексе сдвига времени прямой трансляции](https://www.tencentcloud.com/document/product/267/54302?has_map=1).

#### Инструкции

1. После трансляции с помощью домена, привязанного к шаблону сдвига времени, выберите **Конфигурация функций** > **Сдвиг времени** > [Детали сдвига времени](https://console.intl.cloud.tencent.com/live/config/time-shift/meta) в левой строке меню для входа на страницу информации об индексе.
2. Выберите прямую трансляцию, которую вы хотите вырезать, и нажмите на **Детали** справа, чтобы войти на страницу деталей сдвига времени.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e4a5312c742211f087e15254005ef0f7.png)

3. Вы можете просмотреть адрес отправки и контент со сдвигом времени в **Основная информация**.
4. Вы можете переместить мышь по временной шкале в **Детали индекса**, чтобы просмотреть время позиции. Нажмите на **временную шкалу**, чтобы **отметить время**.
5. Нажмите на временную шкалу, чтобы отметить время и предпросмотреть контент со сдвигом времени.
6. Настройте домен воспроизведения: выберите соответствующий домен воспроизведения со сдвигом времени.
7. Выберите поток ABR: по умолчанию выбирается основной поток, подпоток можно выбрать через раскрывающееся меню.
8. Для создания адреса воспроизведения со сдвигом времени: нажмите **Создать адрес**, чтобы создать адрес воспроизведения со сдвигом времени, который поддерживает копирование адреса в один клик.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/11534b07742311f084fc525400bf7822.png)

### Самостоятельное склеивание адреса

Вы можете обратиться к документу [Практика функции сдвига времени](https://www.tencentcloud.com/document/product/267/31565), чтобы склеить адреса клипов основных моментов прямой трансляции в соответствии с правилами. Вы также можете обратиться к следующим правилам склеивания, чтобы самостоятельно собрать адреса клипов основных моментов прямой трансляции.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f18c48e36bfa11eebb14525400c5de2a.png)

## Консолидация видео клипа

### Создание задачи консолидации видео клипа

1. **Перейдите к консолидации MPS контента со сдвигом времени**: выберите формат консолидированного файла, либо **тип MP4**, либо **тип HLS**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/42fbdada742311f087e15254005ef0f7.png)

2. Когда вы нажмете на **Настроить хранилище**, система автоматически перенаправит вас на страницу **MPS** > **Создать задачу**. На этой странице система автоматически заполнит адрес клипа основных моментов со сдвигом времени и соответствующий шаблон оркестрации сервиса транскодирования. Вы можете настроить задачу в соответствии с потребностями вашего бизнеса.
3. Выберите путь вывода и нажмите **Создать**. Дополнительную информацию см. в документе [Управление задачами](https://www.tencentcloud.com/document/product/1041/48778?has_map=1&lang=en&pg=).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8f62e539742311f087e15254005ef0f7.png)

4. После завершения создания задачи вы можете просмотреть статус выполнения и результаты задачи редактирования видео. Вы можете войти в соответствующий каталог в [COS](https://console.intl.cloud.tencent.com/cos/bucket) и в этом каталоге найти отредактированные и консолидированные файлы. Как правило, имя и путь файла будут определены в соответствии с параметрами вывода, которые вы установили при создании задачи.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/68e21444742411f0bcac525400e889b2.png)

### Создание задачи редактирования видео и консолидации через API

Вы можете создать задачу редактирования видео и консолидации, инициировав запрос API Media Processing Service (MPS):

Введите входные данные и ID расписания для инициирования задачи, затем нажмите [Просмотр информации об API](https://www.tencentcloud.com/document/product/1041/33640?has_map=1&lang=zh&pg=).

> **Примечание:** Tencent Cloud предоставляет вам шаблоны расписания переупаковки редактирования и консолидации видео по умолчанию. ID шаблона расписания переупаковки MP4 по умолчанию — 10101, а ID шаблона расписания переупаковки HLS по умолчанию — 10100. Вы можете создавать и настраивать шаблоны транскодирования видео в соответствии с потребностями вашего бизнеса. Нажмите [Создать шаблон транскодирования видео](https://console.intl.cloud.tencent.com/mps/templates/avs?tab=video), чтобы перейти к настройкам настройки шаблона. Дополнительную информацию см. в документации [Шаблоны транскодирования аудио и видео](https://www.tencentcloud.com/document/product/1041/48784?has_map=1&lang=en&pg=). MPS будет взимать с вас плату в зависимости от типа вашей задачи. Дополнительную информацию см. в документации [Биллинг Media Processing Service](https://www.tencentcloud.com/document/product/1041/49204?has_map=1&lang=en&pg=).

Пример:

```
{    "InputInfo": {        "Type": "URL",     "UrlInputInfo": {      "Url": "http://domain/AppName/StreamName.m3u8?txTimeshift=on&tsFormat=human_s_8&tsStart=20230202095635&tsEnd=20230202095705&tsCodecname=transcode"    }  },  "OutputStorage": {    "Type": "COS",    "CosOutputStorage": {      "Bucket": "test-130000000",      "Region": "ap-nanjing"    },  },  "OutputDir": "/Liveclip/",  "ScheduleId": 10101,    "Action": "ProcessMedia",     "Version": "2019-06-12"}
```

Если будет настроен URL обратного вызова, обратитесь к документации для пакета ответов: [Анализ уведомлений о событиях](https://www.tencentcloud.com/document/product/1041/33679?has_map=1).

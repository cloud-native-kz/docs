# Сдвиг времени

Мы недавно обновили функцию сдвига времени. При создании шаблона сдвига времени в консоли вы теперь будете включать новую функцию сдвига времени. Создайте URL в требуемом формате, и вы сможете использовать этот URL для воспроизведения контента с более ранней временной точки. API 3.0 также теперь доступен для функции сдвига времени. Дополнительные сведения см. в разделе [API сдвига времени](https://intl.cloud.tencent.com/document/product/267/30760). В этом документе показано, как работает функция сдвига времени и как выполнять запрос на воспроизведение.

## Важно знать

- Новая функция сдвига времени в настоящее время поддерживает 30 000 одновременных зрителей. Если вам требуется более высокая пропускная способность, пожалуйста, [отправьте заявку](https://console.tencentcloud.com/workorder/category).
- Если вы включили аутентификацию для вашего домена воспроизведения и установили время истечения, URL сдвига времени истечет по истечении указанного времени.
- Для использования [старой функции сдвига времени](https://www.tencentcloud.com/document/product/266/39564?lang=en&pg=) (которая извлекает контент из домена VOD) необходимо отправить заявку. Для улучшения работы мы рекомендуем использовать новую функцию сдвига времени.

## Принцип работы

CSS включает сдвиг времени путем сохранения прямых трансляций в виде TS-сегментов и информации о времени воспроизведения каждого TS-сегмента в облаке. Эта функция часто используется для повтора телепередач или выделения основных моментов спортивных событий. Контент распространяется на клиентов через HLS. Вы можете указать точное время воспроизведения, установив параметры запроса M3U8 (дополнительные сведения см. в разделе [Запрос на воспроизведение](https://www.tencentcloud.com/document/product/267/31565#5b9e1a89-7b22-4a20-b574-bf7b53dd728f)).

## Запрос на воспроизведение

Формат URL сдвига времени: `http://domain/appname/stream.m3u8`. Существует два типа сдвига времени:

- Воспроизведение определенной продолжительности. Это подходит для повтора основных моментов спортивных событий.
- Воспроизведение с определенного момента времени в прошлом. Это подходит, если вы хотите задержать воспроизведение прямой трансляции.

### Параметры запроса для воспроизведения определенной продолжительности

| Параметр | Описание | Обязательно | Пример |
| --- | --- | --- | --- |
| txTimeshift | Включить ли новую функцию сдвига времени (`on`: Включить). | Да | txTimeshift=on |
| tsStart | Время начала сдвига времени должно быть как минимум на 30 секунд раньше текущего времени, а интервал между tsStart и tsEnd не может быть меньше длительности одного TS-сегмента и не может превышать 6 часов. | Да | tsStart=20121010010101 |
| tsEnd | Время окончания сдвига времени, интервал между tsStart и tsEnd не может быть меньше длительности одного TS-сегмента и не может быть больше 6 часов. | Да | tsEnd=20121010010102 |
| tsFormat | Формат tsStart и tsEnd: `{timeformat}_{unit}_{zone}` Допустимые значения timeformat: unix - временная метка unix. Если вы используете этот формат, вам не нужно указывать zone; human - Читаемое человеком время, например "20121010010101". Допустимые значения unit: s. s означает секунду. zone: Часовые пояса делятся на восточный, западный и центральный (нулевой часовой пояс). Диапазон значений для восточного часового пояса: от 1 до 12. Диапазон значений для западного часового пояса: от -12 до -1. | Да | tsFormat=unix_s; tsFormat=human_s_8 |
| tsCodecname | Для транскодированного потока установите этот параметр в ID шаблона транскодирования. Для исходных потоков или потоков с водяным знаком опустите этот параметр. | Нет | tsCodecname=hd |

#### Пример запроса 1 (временная метка Unix)

```
http://example.domain.com/live/stream.m3u8?txTimeshift=on&tsFormat=unix_s&tsStart=1675302995&tsEnd=1675303025&tsCodecname=test
```

#### Пример запроса 2 (читаемое человеком время)

```
http://example.domain.com/live/stream.m3u8?txTimeshift=on&tsFormat=human_s_8&tsStart=20230202095635&tsEnd=20230202095705&tsCodecname=test
```

### Параметры запроса для воспроизведения с определенного момента времени в прошлом

| Параметр | Описание | Обязательно | Пример |
| --- | --- | --- | --- |
| txTimeshift | Включить ли новую функцию сдвига времени (`on`: Включить). | Да | txTimeshift=on |
| tsDelay | Количество секунд, на которое следует задержать воспроизведение. | Да | `tsDelay=30` указывает, что воспроизведение начнется с 30 секунд назад. |
| tsCodecname | Для транскодированного потока установите этот параметр в ID шаблона транскодирования. | Нет | tsCodecname=2000 |

#### Пример запроса

```
http:://example.domain.com/live/stream.m3u8?txTimeshift=on&tsDelay=30&tsCodecname=test
```

### Параметры аутентификации

Параметры аутентификации для сдвига времени совпадают с параметрами для воспроизведения. Дополнительные сведения см. в разделе [Конфигурация аутентификации при воспроизведении](https://intl.cloud.tencent.com/document/product/267/31060) (URL-адреса HLS, созданные в консоли, действительны только один день).

### Запрос потоков со сдвигом времени

На странице **Сдвиг времени > Детали сдвига времени** консоли отображается список потоков со сдвигом времени. Вы можете нажать кнопку **Детали**, чтобы просмотреть сведения о потоке со сдвигом времени.
Вы также можете использовать API для запроса потоков со сдвигом времени и сведений о конкретном потоке. Дополнительные сведения см. в следующих документах:

- [DescribeTimeShiftStreamList](https://www.tencentcloud.com/document/product/267/53719)
- [DescribeTimeShiftRecordDetail](https://www.tencentcloud.com/document/product/267/53720)

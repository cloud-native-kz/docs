# Расчет URL-адреса для защиты от горячих ссылок

Защита от горячих ссылок достигается путем использования поля `txSecret` в URL-адресе для отправки или воспроизведения. Это может предотвратить попытки злоумышленников подделать ваши URL-адреса для отправки или использовать ваши URL-адреса для воспроизведения в целях получения прибыли без авторизации.

## Как это работает

Вы можете настроить ключ шифрования в консоли CSS (не раскрывайте этот ключ), чтобы предотвратить подделку URL-адресов для отправки и воспроизведения:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3721e3f251de11eeabd75254005810a4.jpeg)

## Инструкции

### Шаг 1. Настройте ключ

Сначала необходимо настроить ключ шифрования в консоли CSS. Это ключ, используемый для создания подписей защиты от горячих ссылок на вашем сервере. Поскольку Tencent Cloud имеет этот ключ, она сможет расшифровать созданные подписи.

Ключ защиты от горячих ссылок для отправки используется для URL-адресов отправки, а ключ защиты от горячих ссылок для воспроизведения используется для URL-адресов воспроизведения. Чтобы настроить ключ для URL-адресов отправки, перейдите в консоль CSS, нажмите [Domain Management](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели и выберите **Push Configuration**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6eb3d500cb8b11f093295254001c06ec.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/00305e82cb8c11f084a45254005ef0f7.png)

Информацию о настройке ключа защиты от горячих ссылок для воспроизведения см. в разделе [How can I enable hotlink protection](https://www.tencentcloud.com/document/product/267/35598#how-can-i-enable-hotlink-protection.3F).

### Шаг 2. Создайте `txTime`

Открытый текст в подписи — это `txTime`, который представляет период действия URL-адреса. Например, если текущее время — `2025-11-28 20:19:50`, а вы хотите, чтобы ваш URL-адрес отправки истекал через три часа, `txTime` должен быть `2025-11-28 23:19:50`.

Для упрощения созданного URL-адреса строка времени преобразуется в метку времени Unix (это можно выполнить, вызвав API времени), а для дальнейшего сокращения строки метка времени преобразуется в шестнадцатеричную или десятичную строку. Таким образом, в приведенном выше примере `txTime` должен быть `1694669601` (десятичный) или `65029b21` (шестнадцатеричный).

> **Примечание:** Фактическое конечное время действия адреса воспроизведения — это `txTime` плюс период действия ключа аутентификации. Изменение периода действия аутентификации не влияет на создание URL-адреса, но может продлить время действия аутентификации адреса. Убедитесь, что вы указали соответствующий период действия для URL-адреса: если период действия слишком короткий, когда хост отключится во время прямой трансляции, он может быть не в состоянии возобновить публикацию из-за истечения срока действия URL-адреса отправки. Если период действия слишком длительный, ваш URL-адрес может быть использован в качестве горячей ссылки.

### Шаг 3. Создайте `txSecret`

`txSecret` создается с помощью `MD5(KEY + StreamName + txTime)`. MD5 — это односторонний алгоритм хеширования (в настоящее время также добавлен алгоритм шифрования SHA256). `KEY` — это ключ шифрования, настроенный на шаге 1. `StreamName` — это ID потока. Рекомендуется установить его в случайное число или идентификатор пользователя. В приведенном ниже примере `StreamName` установлен на `test`, а `txTime` — шестнадцатеричная строка, рассчитанная на шаге 2.
Пример:

```
Предположим, что `KEY` — это `e12c46f2612d5106e2034781ab261ca3`. txSecret = MD5(e12c46f2612d5106e2034781ab261ca3test5C271099) = f85a2ab363fe4deaffef9754d79da6fe
```

### Шаг 4. Создайте URL-адрес для защиты от горячих ссылок

URL-адрес отправки, соответствующий стандарту Tencent Cloud, состоит из следующих четырех частей:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3722c7e151de11ee974d5254005f490f.jpeg)

Теперь, когда у нас есть время истечения URL-адреса

`txTime`

, подпись

`txSecret`

(которую может расшифровать только Tencent Cloud), ID потока

`StreamName`

и домен отправки (предположим, это

`livepush.tcloud.com`

), мы можем создать защищенный от горячих ссылок URL-адрес отправки:

```
rtmp://livepush.tcloud.com/live/test?txSecret=f85a2ab363fe4deaffef9754d79da6fe&txTime=5C271099
```

## Пример кода

Мы предоставляем пример кода для создания URL-адресов для защиты от горячих ссылок. В консоли CSS нажмите [Domain Management](https://console.tencentcloud.com/live/domainmanage) на левой боковой панели, выберите домен отправки и нажмите **Push Configuration**. Прокрутите вниз, и вы найдете пример кода для PHP, Java и Go. Дополнительные сведения см. в разделе [Push Configuration](https://www.tencentcloud.com/document/product/267/31059).

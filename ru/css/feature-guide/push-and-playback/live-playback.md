# Прямая трансляция воспроизведения

## Подготовка

1. Активируйте [службу CSS](https://console.tencentcloud.com/live?from=product-banner-use-lvb) и завершите [Руководство по проверке личности](https://www.tencentcloud.com/document/product/378/3629).
2. Войдите в [консоль CSS](https://console.tencentcloud.com/live/livestat), чтобы получить URL для прямой трансляции. Подробные инструкции см. в разделе [Прямая трансляция](https://intl.cloud.tencent.com/document/product/267/31558).
3. Выберите [Управление доменами](https://console.tencentcloud.com/live/domainmanage), нажмите **Добавить домен**, введите имя вашего домена, выберите тип **Домен воспроизведения** и нажмите **Сохранить**.

> **Примечание:** Если у вас нет домена воспроизведения, вы можете перейти в раздел [Регистрация доменов](https://buy.intl.cloud.tencent.com/domain/buy) для покупки домена. Вы также можете приобрести домен через других поставщиков услуг доменов.

4. Войдите в [консоль сервиса доменов Tencent Cloud](https://console.tencentcloud.com/domain) и настройте CNAME для успешно добавленного домена воспроизведения. Подробные инструкции см. в разделе [Конфигурация CNAME имени домена](https://intl.cloud.tencent.com/document/product/267/31057).

## Получение URL воспроизведения

Выберите **CSS Toolkit** > [**Address Generator**](https://console.tencentcloud.com/live/addrgenerator/addrgenerator), чтобы получить URL воспроизведения, и настройте следующее:

- Выберите тип URL как **Домен воспроизведения**.
- Выберите домен воспроизведения, который вы добавили в **Управление доменами**.
- Введите то же значение `StreamName`, что и в URL прямой трансляции. Значение `StreamName` в URL воспроизведения должно совпадать с `StreamName` в URL прямой трансляции, чтобы воспроизвести соответствующий поток.
- Вам необходимо выбрать тип шифрования в зависимости от требований безопасности и соображений производительности. Тип шифрования может быть MD5 или SHA256, по умолчанию используется MD5.
- Выберите время истечения URL, например `2023-09-13  15:24:50`.
- Нажмите **Создать адрес**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/89050dfa515811eeabd75254005810a4.png)

> **Примечание:** Вы также можете выбрать домен воспроизведения в разделе [**Управление доменами**](https://console.tencentcloud.com/live/domainmanage) в консоли CSS, нажать **Управление**, выбрать **Конфигурация воспроизведения**, введите время истечения URL воспроизведения и `StreamName`, совпадающий с параметром в URL прямой трансляции, и нажмите **Создать адрес воспроизведения**. В некоторых сценариях требуется предпросмотр резервного потока. Основной/Резервный потоки — это два независимых потока с одним и тем же именем потока (StreamName), связанные их идентификаторами потока. Основной поток — это главный поток, который приоритизируется при отправке, а резервный поток — это дополнительный поток, используемый для быстрого переключения в случае отказа основного потока. По умолчанию система воспроизводит содержимое основного потока. Для предпросмотра резервного потока выполните следующие два шага: Получение идентификатора резервного потока: вызовите интерфейс DescribeBackupStreamList, передав имя потока (StreamName), чтобы получить значение UpstreamSequence резервного потока (например, "1234"). Изменение адреса воспроизведения: добавьте два параметра потока после исходного адреса воспроизведения основного потока: UpstreamSequence=xxx&type=xxx (заполните значение UpstreamSequence, полученное на предыдущем шаге, например, "1234".). Например: основной адрес (воспроизведение по умолчанию): https://domain/AppName/StreamName.flv?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time) Адрес резервного потока (предпросмотр резервного потока с параметрами): https://domain/AppName/StreamName.flv?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)&UpstreamSequence=1234&type=1234

## Воспроизведение прямой трансляции

Перед тем как смотреть поток через URL воспроизведения, должна быть успешной [прямая трансляция](https://intl.cloud.tencent.com/document/product/267/31558). Вы можете использовать следующие методы для тестирования прямой трансляции в зависимости от вашего сценария бизнеса:

### Сценарий 1. Воспроизведение на клиенте ПК

Вы можете использовать инструменты, такие как [VLC](https://intl.cloud.tencent.com/document/product/267/32483), FFmpeg и [TCPlayerDemo](https://tcplayer.vcube.tencent.com) для воспроизведения.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/749ea617513e11ee84f2525400494e51.png)

### Сценарий 2. Воспроизведение на мобильном клиенте

1. Загрузите и установите [Tencent Cloud Toolkit](https://intl.cloud.tencent.com/document/product/1016/30285).
2. Выберите **MLVB** > **LVB Playback** или **LEB Playback**.
3. Введите URL воспроизведения в поле ввода или отсканируйте QR-код URL воспроизведения.
4. Нажмите кнопку воспроизведения в нижнем левом углу, чтобы начать воспроизведение.

> **Примечание:** Если вам нужно отправлять/воспроизводить потоки в приложении, вы можете интегрировать [Mobile Live Video Broadcasting (MLVB) SDK](https://www.tencentcloud.com/products/mlvb) с CSS. MLVB SDK поддерживает протоколы воспроизведения RTMP, HTTP-FLV, HLS и WebRTC.

### Сценарий 3. Воспроизведение в веб-браузере

Рекомендуется выбрать TCPlayer из SDK плеера для воспроизведения. На основе мощной внутренней функциональности и технологии искусственного интеллекта Tencent Cloud, TCPlayerLite обеспечивает отличные возможности воспроизведения прямых трансляций и видео по требованию. Глубоко интегрированный с сервисами Tencent Cloud LVB и VOD, Player+ отличается плавным и стабильным воспроизведением, размещением рекламы и мониторингом данных.

> **Примечание:** В настоящее время большинство мобильных браузеров на рынке не поддерживают воспроизведение HTTP-FLV. Поэтому для веб-воспроизведения рекомендуется выбрать протокол воспроизведения HTTP-FLV для браузеров ПК и HLS для мобильных браузеров.

# Прямая трансляция

Суть CSS — это потоковый процесс, аналогичный прямому телевещанию телеканалов, отправляемому аудитории через кабельные сети. Для завершения этого процесса CSS требует устройство захвата и отправки (аналогично камере), облачный сервис прямой трансляции (аналогично кабельной сети) и устройство воспроизведения (аналогично телевизору). Эти устройства могут быть интеллектуальными устройствами, такими как мобильные телефоны, ПК и планшеты, а также веб-браузеры. Мы предоставляем полные демонстрации программного обеспечения для устройств различных типов.

## Получение адреса отправки

1. Войдите в консоль CSS, выберите **Набор инструментов** > [**Генератор адресов**](https://console.tencentcloud.com/live/addrgenerator/addrgenerator) для создания адреса отправки и настройки следующим образом:
  - Выберите **Адрес отправки** в качестве типа домена.
  - Выберите домен отправки, который вы добавили в управление доменами.
  - Введите `AppName` (по умолчанию `live`). Это используется для различения путей различных приложений в одном домене.
  - Введите пользовательское имя `StreamName`, например `liveteststream`.
  - Выберите тип шифрования в зависимости от требований безопасности и соображений производительности. Доступные типы шифрования — **MD5** или **SHA256**, по умолчанию используется **MD5**.
  - Выберите время истечения адреса, например `2024-03-21 11:49:04`.
2. Нажмите кнопку **Создать адрес**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/84b1a2e2e77a11eebbb2525400564496.png)

> **Примечание:** Для обеспечения безопасности ваших потоков прямой трансляции система автоматически включит аутентификацию отправки. Вы также можете выбрать домен отправки для изменения в [**Управление доменами**](https://console.tencentcloud.com/live/domainmanage) и нажать кнопку **Управление** справа, чтобы перейти на страницу сведений о домене, и настроить информацию аутентификации в **Конфигурация отправки**. Адрес отправки имеет следующий формат:
> `rtmp://domain/AppName/StreamName?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)`Кроме описанного выше метода, вы также можете выбрать домен отправки в [**Управление доменами**](https://console.tencentcloud.com/live/domainmanage) в консоли CSS, нажать кнопку **Управление**, выбрать **Конфигурация отправки**, ввести время истечения адреса отправки и пользовательское имя `StreamName`, а затем нажать кнопку **Создать адрес отправки** для создания адреса отправки.
> - Если вам требуется **постоянный адрес отправки**, вы можете перейти в [**Управление доменами**](https://console.tencentcloud.com/live/domainmanage), выбрать домен отправки, нажать кнопку **Управление**, выбрать **Конфигурация отправки** и произвести расчет и создание, ссылаясь на образец кода в **Образец кода адреса отправки**. Дополнительную информацию см. в разделе [Как просмотреть образец кода отправки?](https://intl.cloud.tencent.com/document/product/267/31059).

## Прямая трансляция

Вы можете использовать следующие методы для реализации прямой трансляции в зависимости от вашего сценария бизнеса:

### Сценарий 1. Отправка с ПК

Для ПК (Windows/macOS) вы можете выбрать установку [OBS](https://obsproject.com/download) или [XSplit](https://www.xsplit.com) для отправки. Первый — это бесплатная программа с открытым исходным кодом для записи и потокового воспроизведения видео, которая поддерживает операционные системы, такие как Windows, macOS и Linux, а второй — платная программа, которая предлагает автономный установщик для прямой трансляции игр. Для неигровой прямой трансляции рекомендуем использовать BroadCaster.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a6e96a5f515a11eeabd75254005810a4.png)

В этом документе в качестве примера описываются шаги отправки с помощью OBS. Предположим, что подготовленный адрес отправки:

```
rtmp://xxxx.livepush.myqcloud.com/live/xxxx_test?bizid=xxxx&txSecret=xxx&txTime=58540F7F
```

1. Перейдите на [официальный веб-сайт OBS](https://obsproject.com/download), загрузите и установите инструмент отправки.
2. Откройте OBS и нажмите кнопку **Управление** > **Параметры** внизу, чтобы перейти на страницу параметров.
3. Нажмите кнопку **Поток** для входа на страницу конфигурации отправки и настройте следующим образом:
  3.1. Выберите "Пользовательский" в качестве типа сервиса.
  3.2. Введите первую половину адреса отправки в качестве сервера, например `rtmp://xxxx.livepush.myqcloud.com/live/`.
  3.3. Введите вторую половину адреса отправки в качестве ключа потока, например `xxxx?bizid=xxxx&txSecret=xxx&txTime=58540F7F`.
  3.4. Нажмите кнопку **ОК** в правом нижнем углу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8c6a912ee77b11ee91ba525400aa857d.png)

4. Нажмите кнопку **Управление** > **Начать трансляцию** для тестирования трансляции. Дополнительную информацию об использовании OBS см. в разделе [Отправка через OBS](https://intl.cloud.tencent.com/document/product/267/31569).

### Сценарий 2. Веб-отправка

2. Выберите **Набор инструментов** > [**Веб-отправка**](https://console.tencentcloud.com/live/tools/webpush).
3. Выполните следующие параметры на странице веб-отправки:

Вы можете выбрать один поток или несколько потоков. Подробные процедуры см. в разделе [Веб-отправка](https://intl.cloud.tencent.com/document/product/267/35968).

  - После выбора метода сбора, конфигурации и параметров отправки.
  - Нажмите кнопку **Создать** для перехода на страницу конфигурации генератора адресов.
  - Выберите домен отправки.
  - Введите `AppName` (по умолчанию `live`). Это используется для различения путей различных приложений в одном домене.
  - Введите пользовательское имя `StreamName`, например `liveteststream`.
  - Выберите время истечения, например `2024-03-21 11:55:59`.
4. Нажмите кнопку **Начать отправку** и предоставьте разрешение камеры для начала отправки.

> **Примечание:** Функция веб-отправки требует, чтобы на вашем устройстве была установлена камера и браузер поддерживал плагин Flash для вызова разрешения камеры.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/46f6aeece77d11ee91ba525400aa857d.png)

### Сценарий 3. Отправка с помощью Live SDK

1. Загрузите MLVB SDK.
2. Завершите интеграцию согласно инструкциям в документации интеграции для iOS или Android.

Live SDK — это набор услуг мобильной прямой трансляции. Он демонстрирует в виде бесплатного исходного кода, как использовать Tencent Cloud CSS, VOD, IM и COS для создания наиболее подходящего для вашего бизнеса решения для прямой трансляции. Дополнительные сведения см. в разделе [Mobile Live Video Broadcasting (MLVB) SDK](https://intl.cloud.tencent.com/products/mlvb).

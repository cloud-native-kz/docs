# RTMP поддерживает потоковую передачу и воспроизведение видео с кодированием AV1

Традиционный протокол RTMP, определённый Adobe в 1996 году, предлагает ограниченную поддержку видеокодирования, изначально поддерживая форматы, такие как H.264 (AVC), VP6 и Sorenson Spark, но исключая спецификации инкапсуляции для AV1. Для обеспечения поддержки кодеков следующего поколения, таких как AV1, организация VSO при участии Adobe, Google и других предложила Enhanced-RTMP — расширение протокола RTMP, которое вводит спецификации передачи видео AV1, включая определение форматов последовательных заголовков AV1.

Сервис потоковой трансляции Tencent Cloud лидирует в внедрении Enhanced-RTMP по всей цепочке прямой трансляции, обеспечивая приём и распространение потоков AV1.

## Примечания

- Источник трансляции: OBS Studio версии 29.1 или более поздней версии (доступна для скачивания через [OBS Archived Versions](https://github.com/obsproject/obs-studio/tags)). Благодаря коду, предоставленному VSO, поддерживается потоковая передача AV1 через RTMP.
- Требования к производительности: кодирование AV1 обеспечивает высокую эффективность сжатия, но предъявляет значительные требования к аппаратному обеспечению и пропускной способности сети. Необходимо убедиться, что конечная точка трансляции обладает адекватными возможностями кодирования.
- Воспроизведение: последние версии FFmpeg и VLC обе поддерживают анализ потоков AV1.
- Риски совместимости: Enhanced-RTMP — это совершенно новый протокол, и некоторые устаревшие устройства или плеи­ры могут быть неспособны декодировать потоки AV1. Рекомендуется протестировать целевую среду на совместимость.

## Отправка потока прямой трансляции

1. Откройте **OBS** и перейдите в интерфейс параметров через **Параметры** в панели инструментов Controls.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f1a9e74d70df11f096685254001c06ec.png)

2. В разделе **Параметры** OBS > **Выход** выберите **Advanced** для режима выхода и **AOM AV1** для видеокодека.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f1a8650c70df11f087e15254005ef0f7.png)

3. Существует два способа создания **адреса отправки**:
  - Самостоятельное создание путём применения правил стыковки. Подробные инструкции см. в [документации](https://www.tencentcloud.com/document/product/267/38393#splicing-push-urls).
  - В консоли CSS перейдите в раздел **Инструменты** > [Генератор адресов](https://console.tencentcloud.com/live/addrgenerator/addrgenerator)**, выберите тип адреса «Адрес отправки» и дополнительно выберите домен отправки. Подробные инструкции см. в [документации генератора адресов](https://www.tencentcloud.com/document/product/267/31084#generating-push-urls).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b07028c770e011f081ce52540044a08e.png)

4. Введите созданный адрес потоковой передачи RTMP в OBS для трансляции. Подробные инструкции см. в [документации](https://www.tencentcloud.com/document/product/267/31569).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f1b227d070df11f0bcac525400e889b2.png)

# Потоковая передача RTMPS

Tencent Video Cloud постоянно совершенствует передачу потоковых медиа, чтобы соответствовать различным требованиям. Помимо распространённого протокола потоковой передачи RTMP, теперь также поддерживается протокол RTMPS, который специально разработан для пользователей с потребностями в шифровании, особенно для тех, кто работает за рубежом. Данная статья в основном посвящена реализации потоковой передачи RTMPS.

## Сравнение преимуществ

Стандартный протокол потоковой передачи RTMP полностью полагается на добавление соответствующих параметров в URL для аутентификации, при этом сервер RTMP выполняет проверку на основе этих параметров. Однако это не шифрует передаваемые пакеты аудио и видеоданных, поэтому захваченные и декодированные RTMP-пакеты могут быть воспроизведены.

Протокол RTMPS эффективно решает эти проблемы безопасности RTMP. Это зашифрованная по SSL версия протокола RTMP, которая повышает безопасность передачи данных и обеспечивает безопасную передачу потока между зашифрованным кодировщиком и CDN.

**Сравнительная таблица различных протоколов потоковой передачи:**

| Протокол | Тип протокола | Способ передачи | Задержка | Характеристика протокола | Сценарий применения | Оптимизационное решение Tencent |
| --- | --- | --- | --- | --- | --- | --- |
| RTMPS | Протокол потоковой передачи | TCP | - | Шифрование | Сценарии с шифрованием | Поддержка нескольких доменов и сертификатов |
| SRT | Протокол потоковой передачи | UDP | 500 мс–1 с | Низкая задержка, устойчивость к потерям пакетов | OTT, трансрегиональная передача | - |
| WebRTC | Протокол потоковой передачи | RTP | 200 мс–1 с | Низкая задержка | Аудио- и видеозвонки | Оптимизация быстрой загрузки и устранение заиканий при трансляции живых событий |
| QUIC | Протокол потоковой передачи | UDP | - | Устойчивость к потерям пакетов, 0rtt | Доступ через браузер | Оптимизация передачи первого кадра |

## Примечания

Протокол потоковой передачи RTMPS, использующий SSL-шифрование, требует конфигурации сертификата для доменного имени push-сервера; в настоящее время стандартное push-доменное имя 'push.tlivecloud.com' сервиса облачной потоковой передачи уже оснащено общим сертификатом. Для других доменных имён вам необходимо [подать запрос в поддержку](https://console.tencentcloud.com/workorder/category), предоставив доменное имя и соответствующий сертификат для конфигурации.

> **Примечание:** Если вы хотите протестировать потоковую передачу RTMPS, вы можете использовать стандартное push-доменное имя сервиса облачной потоковой передачи. Если вам нужно использовать потоковую передачу RTMPS через ваше собственное доменное имя, вам необходимо [подать запрос в поддержку](https://console.tencentcloud.com/workorder/category), предоставив доменное имя и соответствующий сертификат.

## Потоковая передача RTMPS

1. Создание URL-адресов прямых трансляций можно выполнить следующими двумя способами:
  - Самостоятельное создание с помощью правил объединения. Подробные инструкции см. в разделе [Объединение URL-адресов прямых трансляций](https://www.tencentcloud.com/document/product/267/38393).
  1.1. Перейдите в консоль CSS в раздел **Инструменты >** [Генератор адресов](https://console.tencentcloud.com/live/addrgenerator/addrgenerator), выберите тип URL-адреса как адрес push-сервера и при необходимости выберите доменное имя. Подробные инструкции см. в документации Генератора адресов > [Генератор адресов](https://www.tencentcloud.com/document/product/267/31084).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e57752629ee011eebd87525400c40977.png)

2. Преобразуйте созданный URL-адрес потоковой передачи RTMP в RTMPS, введите его в OBS, чтобы начать потоковую передачу RTMPS. Подробные инструкции см. в разделе [Передача потока через OBS](https://www.tencentcloud.com/document/product/267/31569).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e56cda079ee011eea869525400c26cb9.png)

Выполняйте действия в соответствии с обычным процессом воспроизведения прямой трансляции. Дополнительные сведения см. в разделе [Воспроизведение прямой трансляции](https://www.tencentcloud.com/document/product/267/31559).

# Составление URL потоковой передачи в прямом эфире

## Примечания

- Данная статья в основном объясняет правила составления URL-адресов потоковой передачи в прямом эфире. Если вы хотите быстро сгенерировать адреса передачи и воспроизведения, вы можете перейти в консоль для их создания. Для получения дополнительной информации см. документацию по [генератору адресов](https://www.tencentcloud.com/document/product/267/31084).
- После создания [шаблона транскодирования](https://www.tencentcloud.com/document/product/267/31071#creating-a-transcoding-template) и [привязки](https://www.tencentcloud.com/document/product/267/31071#unbinding-a-domain-name) его с доменом воспроизведения необходимо добавить имя шаблона транскодирования после `StreamName` потока в прямом эфире с конфигурацией транскодирования в формате `StreamName_имя шаблона транскодирования`. Подробнее см. [Конфигурация воспроизведения](https://intl.cloud.tencent.com/document/product/267/31058).

## Предварительные условия

- Вы зарегистрировались в аккаунте Tencent Cloud и активировали [услугу CSS](https://console.intl.cloud.tencent.com/live/livestat).
- Вы зарегистрировали доменное имя через [Tencent Cloud Domain Service](https://buy.intl.cloud.tencent.com/domain/buy).
- Вы добавили домены передачи/воспроизведения в [**Управление доменами**](https://console.tencentcloud.com/live/domainmanage) **консоли CSS** и успешно настроили запись CNAME. Подробные инструкции см. в разделе [Добавление доменов](https://intl.cloud.tencent.com/document/product/267/35970).

## Составление URL передачи

Если вы управляете большим количеством комнат прямого эфира, невозможно вручную генерировать URL передачи и воспроизведения для каждого вещателя. В таких случаях вы можете использовать сервер для автоматического **составления** адресов. Любой URL, соответствующий стандартам Tencent Cloud, может использоваться для передачи. Стандартный URL передачи состоит из четырех частей, как показано ниже:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/55d4a3bdd77611eea2b0525400bb593a.png)

- **domain**

Домен передачи: Вы можете использовать домен передачи по умолчанию, предоставленный Tencent Cloud Live Streaming, или использовать собственный домен передачи, который был зарегистрирован и успешно настроен с помощью CNAME.

- **AppName**

Имя приложения потоковой передачи в прямом эфире, по умолчанию это "live" и может быть настроено.

- **StreamName（ID потока）**

Пользовательское имя потока: уникальный идентификатор для каждого потока в прямом эфире, рекомендуется использовать случайные числа или комбинацию чисел и букв.

- **Ключ аутентификации (необязательно)**

URL состоит из двух частей: txSecret и txTime, где `txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)`.

После включения аутентификации передачи необходимо использовать URL с ключом аутентификации для передачи. Если аутентификация передачи не включена, нет необходимости в "?" и последующем содержимом в адресе передачи.

  - **txTime**（Истечение URL）

Указывает, когда срок действия URL истечет, формат поддерживает шестнадцатеричные временные метки UNIX (единица времени: секунды).

> **Примечание:** Например, `5867D600` означает истечение в 0:00:00 1 января 2017 года. Наши клиенты обычно устанавливают txTime на истечение через 24 часа после текущего времени. Время истечения не должно быть слишком коротким или слишком длинным. Когда вещатель столкнется с прерыванием сетевого соединения во время прямого эфира, он возобновит передачу потока. Если время истечения слишком короткое, вещатель не сможет возобновить передачу потока из-за истечения URL передачи.

  - **txSecret (подпись против утечки)**

Используется для предотвращения подделки URL передачи, созданного вашей серверной частью злоумышленниками. Способ расчета можно найти в [Best Practices - Anti-leakage Calculation](https://intl.cloud.tencent.com/document/product/267/31560).

## Составление URL воспроизведения

URL воспроизведения состоит из префикса протокола воспроизведения, доменного имени ( `domain`), имени приложения ( `AppName`), имени потока ( `StreamName`), суффикса протокола воспроизведения, ключа аутентификации и других пользовательских параметров. Ниже приведены несколько примеров.

Примеры адресов выглядят следующим образом:

```
webrtc://domain/AppName/StreamName?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)http://domain/AppName/StreamName.flv?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)rtmp://domain/AppName/StreamName?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)http://domain/AppName/StreamName.m3u8?txSecret=Md5(key+StreamName+hex(time))&txTime=hex(time)
```

- Префикс воспроизведения

| Протокол воспроизведения | Префикс воспроизведения | Примечания |
| --- | --- | --- |
| WebRTC | webrtc:// | Мы рекомендуем WebRTC как наиболее оптимальный вариант с лучшей производительностью потоковой передачи и поддержкой сверхвысокого параллелизма. |
| HTTP-FLV | http:// или https:// | Мы рекомендуем HTTP-FLV благодаря хорошей производительности потоковой передачи и поддержке высокого параллелизма. |
| RTMP | rtmp:// | Мы не рекомендуем RTMP, так как он имеет плохую производительность потоковой передачи и не поддерживает высокий параллелизм. |
| HLS (M3U8) | http:// или https:// | Мы рекомендуем HLS для мобильных клиентов и браузера Safari на macOS. |

- **domain**

Домен передачи: Вы можете использовать домен передачи по умолчанию, предоставленный Tencent Cloud Live Streaming, или использовать собственный домен передачи, который был зарегистрирован и успешно настроен с помощью CNAME.

- **AppName**

Имя приложения потоковой передачи в прямом эфире, используется для различения пути хранения файлов мультимедиа потоковой передачи в прямом эфире, по умолчанию это "live" и может быть настроено.

- **StreamName（ID потока）**

Пользовательское имя потока: уникальный идентификатор для каждого потока в прямом эфире. Рекомендуется использовать случайные числа или комбинацию чисел и букв. Не рекомендуется включать "_". Если строка после "_" совпадает с именем шаблона транскодирования, строка будет распознана как имя шаблона транскодирования, а строка перед "_" будет распознана как StreamName, что может привести к аномалиям при воспроизведении. Например: `test_a1_hd1`, `test_a1` будет распознано как StreamName, а "hd1" будет распознано как имя шаблона транскодирования.

- **Имя шаблона транскодирования**

Добавляя суффикс "_" к StreamName, программа будет извлекать транскодированный поток согласно этому шаблону транскодирования. Если вам нужно только извлечь исходный поток, добавлять имя шаблона транскодирования необязательно.

- **Ключ аутентификации (необязательно)**

URL состоит из двух частей: txSecret и txTime, где

`txSecret=Md5(key+StreamName_Имя шаблона транскодирования+hex(time))&txTime=hex(time)`.

После включения аутентификации передачи необходимо использовать URL с ключом аутентификации для передачи. Если аутентификация передачи не включена, нет необходимости в "?" и последующем содержимом в адресе передачи.

  - **txTime（Истечение URL）**

Указывает, когда срок действия URL истечет, формат поддерживает шестнадцатеричные временные метки UNIX (единица времени: секунды).

  - **txSecret (подпись против утечки)**

Используется для предотвращения подделки URL передачи, созданного вашей серверной частью злоумышленниками. Способ расчета можно найти в [Best Practices - Anti-leakage Calculation](https://intl.cloud.tencent.com/document/product/267/31560).

## Просмотр примеров кода передачи

Перейдите в [**Управление доменами**](https://console.tencentcloud.com/live/domainmanage) консоли CSS, выберите предварительно настроенный домен передачи и нажмите **Управление** > **Конфигурация передачи**, чтобы отобразить **Пример кода адреса передачи** (для PHP и Java), который демонстрирует способ создания адреса защиты от горячих ссылок. Подробные инструкции см. в разделе [Конфигурация передачи](https://intl.cloud.tencent.com/document/product/267/31059).

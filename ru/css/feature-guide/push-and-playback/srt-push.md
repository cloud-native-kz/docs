# SRT Push

TS over SRT напрямую передает потоки TS, содержащие данные аудио/видео, используя **протокол SRT**. Существующая система потоковой трансляции используется для воспроизведения. TS over SRT используется как стандартный формат отправки для оборудования Haivision и OBS.
В этом режиме SRT-сервер анализирует потоки TS, переформатирует их в потоки RTMP и перенаправляет на бэкэнд RTMP-сервер.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/5e9694da520f11ee974d5254005f490f.png)

> **Примечание:** Использование SRT для отправки потоков не увеличивает стоимость.

## Сравнение предотвращения потери пакетов

По сравнению с QUIC, SRT снижает потерю пакетов на уровне приложения при одинаковом коэффициенте потери пакетов благодаря более быстрому, более точному управлению переданачей и механизму регулирования темпа для сценариев потоковой трансляции. Когда коэффициент потери пакетов составляет 50%, SRT по-прежнему может гарантировать стабильную передачу.

При одной и той же линии связи и одном файле потока на стороне отправки коэффициент потери пакетов снижается на 5% каждые пять минут при использовании SRT. На следующем рисунке показано, что частота кадров отправки SRT более стабильна.

## Трансляция в реальном времени

### Метод доступа

Трансляция в реальном времени поддерживает отправку потоков по SRT с использованием **порта 9000**. Вы можете [создать адрес отправки](https://intl.cloud.tencent.com/document/product/267/31084) с помощью [генератора адресов](https://console.tencentcloud.com/live/addrgenerator/addrgenerator) в консоли CSS и составить адрес по следующим правилам.

Адрес отправки SRT Tencent Cloud:

```
srt://${rtmp-push-domain}:9000?streamid=#!::h=${rtmp-push-domain},r=${app}/${stream},txSecret=${txSecret},txTime=${txTime}
```

> **Примечание:** `${app}` — это переменная, которая должна быть заменена на фактическое значение. Обратите внимание, что символы `$`, `{` и `}` не требуются.

### Метод реализации

SRT-сервер переформатирует потоки TS в потоки RTMP и передает их на доменное имя `${rtmp-push-domain}`.

Пример отправки контента OBS через протокол SRT:

Введите следующее в Server: `srt://${rtmp-push-domain}:9000?streamid=#!::h=${rtmp-push-domain},r=${app}/${stream},txSecret=${txSecret},txTime=${txTime}`

Введите следующее в Stream Key: `r=${app}/${stream},txSecret=${txSecret},txTime=${txTime}`

![](https://staticintl.cloudcachetci.com/cms/backend-cms/612ebd9c638e11ef8357525400bdab9d.png)

> **Примечание:** Если вы хотите отправлять потоки по SRT, версия OBS не может быть ниже v25.0.

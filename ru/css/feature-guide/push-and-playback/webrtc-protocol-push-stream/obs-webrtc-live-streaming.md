# Прямая трансляция OBS WebRTC

OBS (Open Broadcaster Software) поддерживает протокол WebRTC для прямой трансляции. Это означает, что вы можете легко и быстро передавать прямые потоки в Tencent Cloud Streaming Services, используя протокол WebRTC на ПК (Windows/Mac/Ubuntu), так же как при использовании протокола RTMP.

В этой статье мы рассмотрим пример OBS v30.0 на Windows, чтобы продемонстрировать, как использовать OBS для прямой трансляции по протоколу WebRTC на ПК.

Вы также можете использовать протокол WebRTC для прямой трансляции в веб-среде. Подробные шаги для Web Push см. в руководстве [Web Push](https://www.tencentcloud.com/document/product/267/57043).

## Поддерживаемые кодеки

### Видео

- **H.264** (Лучшая совместимость. Почти все клиенты могут воспроизводить его напрямую.)
- **AV1** (По сравнению с H.264 степень сжатия увеличена более чем на 40%, что позволяет сэкономить более 40% пропускной способности и затрат на хранилище при одинаковом качестве. Большинство браузеров могут воспроизводить его напрямую.)
- **HEVC** (Зависит от кодеков устройства трансляции. По сравнению с H.264 имеет более высокую степень сжатия и слабую поддержку браузером. [SDK MLVB](https://www.tencentcloud.com/document/product/1071) может воспроизводить его напрямую.)

### **Аудио**

- **Opus** (Браузер WebRTC может воспроизводить его напрямую.)

## Подготовка

- Вы можете загрузить [OBS](https://obsproject.com/download) с официального веб-сайта. Убедитесь, что вы установили версию OBS, которая поддерживает прямую трансляцию WebRTC ([v30.0](https://github.com/obsproject/obs-studio/releases) или выше; кодирование AV1 или HEVC требует [v30.2](https://github.com/obsproject/obs-studio/releases) или выше. **Пожалуйста, не используйте v30.1.x, так как эта версия содержит ошибку, вызывающую искажение видеоизображения при плохих сетевых условиях**).
- Если вы не можете обновить версию OBS, обратитесь к разделу [Использование плагина OBS для прямой трансляции WebRTC](https://www.tencentcloud.com/document/product/267/57042#a1ac1dab-cad0-4ebf-ad2f-9a20d82a0057) или рассмотрите возможность использования RTMP для трансляции.
- Вы активировали [CSS](https://www.tencentcloud.com/products/css) и подготовили зарегистрированный домен для добавления в качестве [домена отправки](https://www.tencentcloud.com/document/product/267/35970?lang=zh&pg=) (вы можете использовать домен отправки по умолчанию, предоставляемый системой, или добавить пользовательский домен для трансляции).

## Примечания

- **Пожалуйста, не используйте OBS v30.1.x, так как эта версия содержит ошибку, вызывающую искажение видеоизображения при плохих сетевых условиях.**
- CSS по умолчанию предоставляет тестовый домен `xxxx.livepush.myqcloud.com`, который вы можете использовать для тестирования прямой трансляции. Однако не рекомендуется использовать этот домен как домен отправки в производственной среде.
- При использовании протокола WebRTC для прямой трансляции каждый домен отправки по умолчанию имеет ограничение в 1000 одновременных потоков. Если вам нужно превысить это ограничение, вы можете связаться с нами, [отправив заявку](https://console.tencentcloud.com/workorder) на увеличение лимита.

## Получение адреса отправки WebRTC

1. Войдите в [консоль CSS](https://console.tencentcloud.com/live/addrgenerator/addrgenerator), выберите **CSS Toolkit** > **Address Generator** и выполните следующие конфигурации:
  1.1. Выберите тип URL: **Push Address**.
  1.2. Выберите домен, который вы уже добавили в раздел управления доменами.
  1.3. AppName используется для различия путей адресов нескольких приложений в одном домене, значение по умолчанию — "live".
  1.4. Введите пользовательское имя `StreamName`, например: `Stream_01`.
  1.5. Вам нужно выбрать тип шифрования, пожалуйста, учитывайте требования безопасности и компромиссы производительности. Для **Type** вы можете выбрать между **MD5** или **SHA256**, причем **MD5** является опцией по умолчанию.
  1.6. Выберите время истечения, например: `2024-10-10 11:35:28`.
2. Нажмите **Generate Address**, чтобы получить адрес отправки WebRTC.

## Прямая трансляция OBS в Интернете

### Шаг 1: Установка адреса сервера WHIP и адреса отправки WebRTC

1. Откройте **OBS**, и вы можете получить доступ к интерфейсу параметров, нажав кнопку **Settings** в панели управления в нижней части инструментов.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c919dcff5cee11eeabd75254005810a4.png)

2. Нажмите на **Stream** для входа в интерфейс параметров адреса прямой трансляции.
  - Выберите тип **Service** как: **WHIP**
  - В поле "**Server**" введите адрес сервера WHIP Tencent Cloud Live:

Адрес по умолчанию: `https://webrtcpush.tlivewebrtcpush.com/webrtc/v2/whip`

Резервный адрес: `https://webrtcpush.tlivewebrtcpush2.com/webrtc/v2/whip`

  - В поле "**Bearer Token**" введите [адрес отправки WebRTC](https://www.tencentcloud.com/document/product/267/57042#671acf59-54b6-4b99-93e7-98015cc6d404), который вы получили, например: `webrtc://domain/AppName/StreamName?txSecret=xxxxx&txTime=xxxxx`

![](https://staticintl.cloudcachetci.com/cms/backend-cms/313b879c378b11ef9b60525400bdab9d.png)

### Шаг 2: Установка параметров трансляции

1. Перейдите в интерфейс параметров трансляции, нажав Control > **Settings** > **Output**. Выберите режим вывода как **Advanced**.
2. Выберите опцию **Streaming** и настройте параметры, такие как кодек, битрейт, интервал ключевых кадров и другие.
3. Если вы используете решение **LEB WebRTC** на стороне воспроизведения, выполните настройку отправки согласно следующей конфигурации в зависимости от различных протоколов кодирования:
  - **Использование видеокодека H.264**

Для видеокодека выберите **x264** или другой поддерживаемый вашим устройством кодек H.264.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/5f5d3230fdfe11ee8cb552540095b445.png)

  - **Использование видеокодека AV1**

Для видеокодека выберите **AOM AV1** или другой поддерживаемый вашим устройством кодек AV1.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d6ec6178fdfe11ee8cb552540095b445.png)

  - **Использование видеокодека HEVC**

Для видеокодека выберите кодек HEVC, поддерживаемый вашим устройством, например Apple VT HEVC Hardware/Software Encoder.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b5b221f5378b11efb89a52540075b605.png)

> **Примечание:** Для потоковой передачи WebRTC необходимо добавить параметр `tx_h265=true` для включения поддержки. Формат кодирования аудио по умолчанию для отправки OBS WebRTC — Opus. При использовании LEB для воспроизведения в Интернете, поскольку Интернет по умолчанию поддерживает формат аудио Opus, нет необходимости в облачном перекодировании аудио (из AAC в Opus), как требуется при отправке по протоколу RTMP, и воспроизведение можно выполнять напрямую. Дополнительные сведения о параметре **x264 option** см. в разделе: [параметры кодирования x264 с несколькими срезами](https://www.tencentcloud.com/document/product/267/57042#6dfea2dd-491c-4c9b-a7be-a9ab9decb92f).

4. Нажмите на **Stream** для входа в интерфейс параметров адреса прямой трансляции.

### Шаг 3: Начало прямой трансляции

Нажмите на Control > **Start Streaming** в панели управления в нижней части OBS для отправки медиапотока на адрес WebRTC, который вы установили.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/181ffec05cf211ee9ff8525400d917da.png)

> **Примечание:** OBS использует WHIP (WebRTC-HTTP Ingestion Protocol) для прямой трансляции WebRTC. WHIP — это стандартный протокол на основе HTTP, который позволяет отправлять и получать потоки WebRTC в реальном времени на серверы потоковой передачи или CDN с использованием HTML5 и различных клиентов. Если вам нужна дополнительная информация об использовании функции трансляции OBS, вы можете обратиться к [Push via OBS](https://www.tencentcloud.com/document/product/267/31569).

## Сравнение сквозной задержки между прямой трансляцией OBS WebRTC и прямой трансляцией по протоколу RTMP:

Сквозная задержка зависит от нескольких факторов, таких как производительность устройства, параметры кодирования, передача сетевых данных и кэш плеера, и может колебаться в определенном диапазоне во время процесса прямой трансляции. В этом сценарии мы сравним разницу в сквозной задержке между кодированием x264 с несколькими срезами и кодированием с одним срезом. Сторона отправки использует инструмент OBS, а сторона воспроизведения использует Web LEB.

### Параметры кодирования x264 с несколькими срезами:

- Когда вы настраиваете режим zerolatency (нулевая задержка) в опциях Tune OBS, OBS автоматически включит **кодирование с несколькими срезами** для повышения скорости кодирования и снижения задержки.
- Если вы используете решение LEB Web на стороне воспроизведения, учтите, что у некоторых более старых версий браузеров WebRTC могут быть проблемы совместимости с **кодированием с несколькими срезами**. В этом случае включение **кодирования с несколькими срезами** может вызвать эффект мозаичного экрана на стороне воспроизведения при слабой сетевой потери пакетов. Чтобы избежать этой проблемы, вы можете настроить `sliced_threads=0` в параметрах x264 для отключения **кодирования с несколькими срезами.** Однако отключение **кодирования с несколькими срезами** может привести к дополнительной задержке кодирования в несколько сотен миллисекунд. Поэтому при настройке балансируйте совместимость и задержку в соответствии с вашими фактическими потребностями.

| Метод отправки | Описание |
| --- | --- |
| Потоковая передача WHIP | Примерно 300~500ms. ![](https://staticintl.cloudcachetci.com/cms/backend-cms/b132dd8e516811ee94c3525400d793d0.png) |
| Потоковая передача RTMP | Примерно 450~650ms. ![](https://staticintl.cloudcachetci.com/cms/backend-cms/a9b2c887516811eeabd75254005810a4.png) |

### Кодирование с одним срезом

При использовании кодирования с одним срезом сквозная задержка значительно зависит от производительности устройства. Следующие данные предоставляются только в справочных целях, и фактическая задержка может различаться в зависимости от производительности устройства и других факторов:

| Метод отправки | Описание |
| --- | --- |
| Потоковая передача WHIP | Примерно 700~850ms. ![](https://staticintl.cloudcachetci.com/cms/backend-cms/c6ec3e05516811ee94c3525400d793d0.png) |
| Потоковая передача RTMP | Примерно 850~1000ms. ![](https://staticintl.cloudcachetci.com/cms/backend-cms/d48f528d516811ee84f2525400494e51.png) |

> **Примечание:** Эти данные задержки могут колебаться в зависимости от таких факторов, как сетевые условия, параметры кодирования и буферизация плеера. В практических приложениях вы можете настраивать параметры кодирования и протоколы потоковой передачи в соответствии с вашими требованиями и производительностью устройства для достижения желаемой задержки и производительности качества изображения.

## Использование плагина OBS для прямой трансляции WebRTC

Версии OBS ниже v30.0 Beta 1 не поддерживают прямую трансляцию WebRTC напрямую. Tencent Cloud Streaming Services предоставляют встроенное решение плагина OBS для прямой трансляции WebRTC.

### Примечания

- Текущие требования к версиям OBS: 26.0 ≤ версия OBS ≤ 29.0.2. Вы можете загрузить и установить необходимую версию со страницы [Архивных версий OBS](https://github.com/obsproject/obs-studio/tags).
- Плагин прямой трансляции WebRTC в настоящее время поддерживает только платформу Windows. Если вы хотите реализовать прямую трансляцию WebRTC на Mac/Linux, вы можете использовать [Web Push](https://www.tencentcloud.com/document/product/267/42131).

### Настройка плагина OBS

1. **Настройка данных плагина**.
  1.1. Загрузите [плагин OBS](https://monitor-1258344699.cos.ap-guangzhou.myqcloud.com/tencent_webrtc_plugin_20230214.zip) и, в зависимости от локальной версии OBS, переместите два файла `services.json` и `package.json` из папки данных соответствующей версии в каталог **data > obs-plugins > rtmp-services**, заменив существующие файлы. (По умолчанию `obs-studio` установлен на диск C, соответствующий каталог — `C:\\obs-studio\\data\\obs-plugins\\rtmp-services`. Пожалуйста, настройте в соответствии с вашей ситуацией.)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8127e390516a11ee84f2525400494e51.png)

  1.2. Скопируйте два упомянутых выше файла JSON в каталог `C:\\Users<имя_компьютера>\\AppData\\Roaming\\obs-studio\\plugin_config\\rtmp-services` и перезапишите существующие файлы. (Замените <`имя_компьютера`> на имя вашего компьютера.)
2. **Настройка динамической библиотеки плагина.**

Переместите файл .dll из папки `obs-plugins\\64bit` в соответствующий каталог **obs-studio > obs-plugins > 64bit**. (По умолчанию `obs-studio` установлен на диск C, соответствующий каталог — `C:\\obs-studio\\obs-plugins\\64bit`. Пожалуйста, настройте в соответствии с вашей ситуацией.)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/abef3135516a11ee94c3525400d793d0.png)

### Настройка ссылки отправки

1. Создание адреса отправки WebRTC.

Войдите в консоль CSS, выберите **CSS Toolkit** > [Address Generator](https://console.tencentcloud.com/live/addrgenerator/addrgenerator), чтобы создать адрес отправки. Подробные операции см. в руководстве [Address Generator](https://www.tencentcloud.com/document/product/267/31084).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e8df24ad44d811efa24b525400a9236a.png)

2. **Настройка сервиса потоковой передачи OBS.**
  2.1. Откройте OBS, и вы можете получить доступ к интерфейсу параметров, нажав кнопку **Settings** в панели управления в нижней части инструментов.
  2.2. Нажмите на **Stream** для входа на вкладку Stream Settings, выберите тип сервиса как Tencent WebRTC, установите сервер как Default, и введите ранее созданный [адрес отправки WebRTC](https://www.tencentcloud.com/document/product/267/57042#c738b1ba-9820-4e1c-9dca-106feb82368d) в поле Stream Key.
  2.3. Текущий плагин OBS поддерживает версию OBS 29. Чтобы начать прямую трансляцию, нажмите на **Streaming** для входа на вкладку Stream Settings, выберите тип сервиса как `Tencent WebRTC`, установите сервер как `Default,` и введите ранее созданный [адрес отправки WebRTC](https://www.tencentcloud.com/document/product/267/42131) в поле Stream Key.

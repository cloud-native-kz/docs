# Трансляция через OBS

## Обзор

Open Broadcaster Software (OBS) — это стороннее программное обеспечение с открытым исходным кодом для трансляции в реальном времени. Оно просто в использовании, бесплатно и поддерживает OS X, Windows и Linux. OBS можно использовать в различных сценариях для удовлетворения большинства потребностей в прямых трансляциях без необходимости установки дополнительных плагинов. Вы можете загрузить последнюю версию на [сайте OBS](https://obsproject.com/download?spm=a2c4g.11186623.2.15.6aac1445JPlKR8).

## Требования

- Вы установили [OBS Studio](https://obsproject.com/download?spm=a2c4g.11186623.2.15.6aac1445JPlKR8).
- Вы активировали [CSS](https://console.tencentcloud.com/live) и [добавили домен воспроизведения](https://intl.cloud.tencent.com/document/product/267/35970) с номером регистрации ICP в консоли (для трансляции вы можете использовать предоставленный домен по умолчанию или добавить свой собственный).

## Получение URL трансляции

1. Войдите в консоль CSS, нажмите [Address Generator](https://console.tencentcloud.com/live/addrgenerator/addrgenerator) на левой боковой панели.
2. Войдите на страницу Address Generator и выполните следующие настройки:
  2.1. Выберите тип URL: **Push Address**.
  2.2. Выберите домен, который вы добавили в **Domain Management**.
  2.3. Введите имя приложения (`AppName`), которое используется для различения приложений в одном домене. Значение по умолчанию — `live`.
  2.4. Введите пользовательское имя потока (`StreamName`), например `liveteststream`.
  2.5. Вы должны выбрать тип шифрования в соответствии с вашими требованиями безопасности и соображениями производительности. Вы можете выбрать в качестве типа шифрования **MD5** или **SHA256**, по умолчанию используется **MD5**.
  2.6. Выберите время истечения адреса, например `2024-10-10 11:09:00`.
3. Нажмите **Generate Address**, чтобы получить URL трансляции OBS.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/5a327c7e85ec11efaee3525400bdab9d.png)

## Настройка OBS для трансляции

### Шаг 1. Настройка URL трансляции

1. Откройте OBS и нажмите **Controls** > **Settings** внизу, чтобы войти на страницу настроек.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/61aff4ed8fac11f0bdaa525400bf7822.png)

2. Нажмите **Stream** и выберите **Custom** для **Service**.
3. Заполните поля **Server** и **Stream Key** информацией, полученной в разделе [Получение URL трансляции](#step0).
  - **Server**: Введите адрес трансляции OBS (`rtmp://domain/AppName/`).
  - **Streaming Key**: Введите имя трансляции OBS (`StreamName?txSecret=xxxxx&txTime=liveteststream`).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f52cfd07d13111ee9409525400c26da5.png)

4. Нажмите **OK**, чтобы сохранить информацию.

### Шаг 2. Настройка источника

> **Примечание:** Для настроек битрейта, записи и других параметров нажмите **Tools > Auto-Configuration Wizard** в верхней строке меню и следуйте инструкциям OBS для завершения настроек.

1. Найдите **Sources** в меню внизу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6f3d39b38fac11f089d25254007c27c5.png)

2. Нажмите **+** и выберите источник, подходящий для ваших нужд, например **Display Capture**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7bbd48538fac11f0bdaa525400bf7822.png)

**Распространённые источники прямых трансляций**

| Входной источник | Описание |
| --- | --- |
| VLC Video Source | Установите VLC для включения этого источника |
| Image | Трансляция одного изображения |
| Image Slide Show | Трансляция нескольких изображений (вы можете определить порядок воспроизведения и циклическое воспроизведение) |
| Scene | Вставка целой сцены в качестве источника для получения различных эффектов трансляции |
| Media Source | Трансляция локального файла |
| Text | Добавление текста в реальном времени в вашу трансляцию |
| Display Capture | Захват и трансляция вашего монитора в реальном времени |
| Browser | Отображение содержимого URL в браузере или открытие локального файла html |
| Game Capture | Трансляция игры из определённого источника в реальном времени |
| Window Capture | Захват и трансляция выбранного вами окна в реальном времени |
| Color Source | Добавление сплошного цвета к вашей сцене. Вы можете использовать этот источник для цветов фона или глобального цветового оттенка с помощью альфа-канала. |
| Video Capture Device | Захват и трансляция изображений с камеры в реальном времени |
| Audio Input Capture | Трансляция звука (устройство звукового ввода) |
| Audio Output Capture | Трансляция звука (устройство звукового вывода) |
| Group | Размещение источников в одной группе для изменения видимости и статуса блокировки партиями |

### Шаг 3. Использование режима студии

В режиме студии вы можете редактировать вашу текущую трансляцию в реальном времени и настраивать переходы для переключения сцен, минимизируя влияние на впечатление пользователя.

1. Нажмите **Controls > Studio Mode** в меню внизу.
2. После редактирования нажмите **Transition**, чтобы переключить представления редактирования и трансляции.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c07ec13a8fab11f087025254001c06ec.png)

### Шаг 4. Начало трансляции

1. Найдите **Controls** в меню внизу.
2. Нажмите **Start Streaming**, чтобы отправить видео на настроенный URL трансляции.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d50a918e8fab11f090a8525400e889b2.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e9a6e3158fab11f097255254005ef0f7.png)

> **Примечание:** Когда внизу появляется ![](https://staticintl.cloudcachetci.com/cms/backend-cms/f53ad514d13111ee89c5525400170219.png) зелёный индикатор, это означает успешную отправку потока. Если вы хотите остановить трансляцию, просто нажмите **Stop Streaming**.

## Другие настройки трансляции

### Задержка трансляции

1. Перейдите в **Controls > Settings > Output**.
2. Выберите **Output Mode** как **Advanced** для настройки параметров, таких как **Keyframe Interval**. **Чрезмерный интервал ключевых кадров (GOP)** может повлиять на опыт прямого трансляционного вещания. Рекомендуется установить размер 2 с. Процедура настройки показана на рисунке ниже:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f53f8221d13111eea2b0525400bb593a.png)

### Удаление B-кадров в LEB

Решение LEB для веб-**не поддерживает декодирование или воспроизведение B-кадров**. Если поток содержит B-кадры, серверная часть удалит их при перекодировании, что увеличит задержку и **повлечёт за собой комиссию за перекодирование**. Избегайте отправки потоков с B-кадрами или используйте программное обеспечение для трансляции, такое как OBS, для их удаления путём настройки параметров видеокодирования. На рисунке ниже показано, как удалить B-кадры с помощью OBS:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f556b565d13111eeb0d9525400461a83.png)

### Локальная запись трансляции

Для записи прямых трансляций в локальное хранилище выполните следующие шаги:

1. Перейдите в **Controls > Settings > Output**.
2. Заполните настройки в разделе **Recording** и нажмите **OK**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f5481daed13111eea2b0525400bb593a.png)

3. Нажмите **Video** на левой боковой панели, чтобы установить разрешение и частоту кадров.

> **Примечание:** Разрешение определяет чёткость видео, отображаемого зрителям. Чем выше разрешение, тем чётче видео. Частота кадров (кадры в секунду) определяет плавность воспроизведения. Типичная частота кадров находится в диапазоне от 24 кадров/сек до 30 кадров/сек. Воспроизведение может прерываться, если частота кадров ниже 16 кадров/сек. Видеоигры требуют более высокой частоты кадров и имеют тенденцию к прерываниям при частоте кадров ниже 30 кадров/сек.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f54fbb10d13111eeb0d9525400461a83.png)

### Перекодирование

Чтобы изменить битрейт видео во время трансляции, выполните следующие шаги:

1. Нажмите **Controls > Settings** в меню внизу.
2. Нажмите **Output** на левой боковой панели и выберите **Simple** для **Output Mode**.
3. Введите необходимый битрейт и нажмите **OK**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f550ba07d13111eeb0d9525400461a83.png)

## Дополнительно

### Трансляция только звука

Согласно форумам OBS, OBS Studio версии 23.2.1 и более ранние версии не поддерживают трансляцию только звука.
Вы можете выполнить следующие шаги для реализации аналогичной функции. Этот метод использует статический холст (пустой экран или изображение) для видеоконтента. Это означает, что в прямой трансляции всё равно будут видеоданные. Чтобы снизить использование полосы пропускания, вы можете установить частоту кадров видео и битрейт на минимальные значения.

1. Как указано в разделе [Настройка источника](#adaa72d4-c5b4-44ef-a040-2ccd38f16fb5), выберите **Audio Input Capture** в качестве источника. Не используйте видео или источник изображения.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/07c50fe78fac11f087025254001c06ec.png)

2. Перейдите в **Controls > Settings > Video**.
3. Установите **Base (Canvas) Resolution** и **Common FPS Values** на минимальные значения и нажмите **OK**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f55d8c32d13111eeb0d9525400461a83.png)

4. Нажмите **Output** на левой боковой панели, настройте вывод как показано ниже (установите **Bitrate** на минимальное значение) и нажмите **OK**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f559bd8ed13111eea2b0525400bb593a.png)

5. Начните трансляцию, как указано в разделе [Настройка OBS для трансляции](#normal). Аудитория услышит звук, а видео будет пустым экраном или изображением. Поскольку битрейт видео установлен на минимальное значение, использование полосы пропускания значительно ниже, чем при видеотрансляции.

### Циклическое воспроизведение видео

1. Нажмите **+** в **Sources** и выберите **Media Source**. В появившемся окне выберите локальный файл для трансляции, выберите **Loop** и нажмите **OK**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/1dd9457f8fac11f087025254001c06ec.png)

2. Установите **Server** и **Stream Key** как указано в разделе [Настройка URL трансляции](#step1).

## Воспроизведение

После завершения трансляции вы можете создать адрес воспроизведения с тем же StreamName, что и адрес трансляции. Вы можете проверить, была ли трансляция успешной, используя следующий метод воспроизведения:

- ПК: Поддерживает использование [VLC Player](https://www.tencentcloud.com/document/product/267/32483) для извлечения потока.
- Мобильные устройства: Поддерживает воспроизведение путём интеграции [Mobile Live Video Broadcasting](https://www.tencentcloud.com/document/product/1071).

> **Примечание:** Mobile Live Video Broadcasting (MLVB) SDK — это расширение Cloud Streaming Services (CSS) в мобильных сценариях. По сравнению с сервисами CSS, в основном предназначенными для облачной интеграции, MLVB SDK не только предлагает **быстрое решение для интеграции** на основе RTMP SDK, но также предоставляет **комплексное профессиональное решение**, включающее несколько облачных сервисов, включая Live Video Broadcasting (LVB), Live Event Broadcasting (LEB), Video on Demand (VOD), чат и Cloud Object Storage (COS).
> Фактически [Live Event Broadcasting](https://www.tencentcloud.com/document/product/1071/41875) (LEB) является расширением Live Video Broadcasting (LVB) в сценариях воспроизведения с ультранизкой задержкой. Он отличается более низкой задержкой, чем традиционные протоколы потоковой передачи, и обеспечивает превосходный опыт воспроизведения с задержкой в миллисекунды. Подходит для сценариев с высокими требованиями к задержке, таких как онлайн-образование, трансляция спорта и онлайн-викторины.

# Начало работы

Этот документ показывает, как начать работу с LEB. Перед использованием LEB прочитайте [Обзор цен](https://intl.cloud.tencent.com/document/product/267/2819), чтобы узнать о его **платежных элементах** и **ценах**.

## Предварительные требования

1. Вы [зарегистрировали учетную запись Tencent Cloud](https://intl.cloud.tencent.com/account/register?s_url=https%3A%2F%2Fcloud.tencent.com%2Fproduct%2Flvb).
2. Перейдите в [консоль CSS](https://console.tencentcloud.com/live?from=product-banner-use-lvb), согласитесь с **Соглашением об услугах Tencent Cloud** и нажмите **Применить для активации**, чтобы активировать CSS.

> **Примечание:** После активации CSS вы получите 20 ГБ трафика воспроизведения для китайского материка бесплатно. Шаги по настройке доменов для LEB аналогичны шагам для LVB. Если вы уже используете LVB, вы можете перейти к [Шагу 4. Получение URL-адреса воспроизведения](#step-4.-получение-url-адреса-воспроизведения).

## Этапы выполнения

### Шаг 1. Добавление доменов

1. Зарегистрируйте ваше доменное имя. ICP-регистрация требуется, если вы хотите использовать домен в китайском материке.
2. Войдите в консоль CSS, перейдите в [Управление доменами](https://console.intl.cloud.tencent.com/live/domainmanage), нажмите **Добавить домен**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/72fc5785522911eeabd75254005810a4.png)

3. Перейдите на страницу добавления пользовательского домена, заполните доменное имя, которое было заполнено для регистрации, и выберите **Тип**.
4. Теги используются для категоризации и управления ресурсами из разных измерений. Если существующие теги не соответствуют вашим требованиям, вы также можете перейти в [Консоль тегов](https://console.intl.cloud.tencent.com/tag/taglist), чтобы управлять тегами единообразно.
5. Нажмите **Добавить домен**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8435bda7144911f0a63e5254005ef0f7.png)

> **Примечание:** CSS предоставляет тестовое доменное имя `xxxx.livepush.myqcloud.com`. Вы можете использовать его для тестирования трансляции, но мы не рекомендуем использовать его в коммерческих целях. После успешного добавления доменного имени вы можете просмотреть его информацию в списке доменов в **Управлении доменами**. Информацию о том, как управлять своими доменами, см. в [Управлении доменами](https://intl.cloud.tencent.com/document/product/267/31056). Дополнительную информацию о доменах прямой трансляции см. в [Основы прямой трансляции](https://intl.cloud.tencent.com/document/product/267/7968).

6. После добавления вашего доменного имени система присвоит ему каноническое имя (с суффиксом `.txlivecdn.com` или `.tlivepush.com`), которое недоступно до завершения конфигурации CNAME у вашего поставщика услуг DNS. Следующий пример показывает, как добавить запись CNAME, если вы используете DNS-сервис Tencent Cloud:
  6.1. Войдите в [консоль DNS](https://console.tencentcloud.com/domain).
  6.2. Найдите ваше доменное имя и нажмите **Разрешить**.
  6.3. На странице разрешения доменного имени нажмите **Добавить запись**.
  6.4. Введите префикс вашего доменного имени для **Записи хоста**, выберите CNAME для **Типа записи** и введите каноническое имя для **Значения записи**.
  6.5. Нажмите **Сохранить**.

> **Примечание:** Конфигурация CNAME требует некоторого времени для вступления в силу. Вы можете использовать CSS только после того, как это произойдет. После того как конфигурация CNAME вступает в силу, вы увидите значок ![](https://staticintl.cloudcachetci.com/cms/backend-cms/8912bf5a2c4511ee818e525400088f3a.png) перед адресом CNAME в [Управлении доменами](https://console.tencentcloud.com/live/domainmanage). Если конфигурация CNAME не вступает в силу, обратитесь к вашему поставщику DNS. О том, как добавить записи CNAME у других поставщиков DNS, см. в [Конфигурация CNAME для доменного имени](https://intl.cloud.tencent.com/document/product/267/31057).

### Шаг 2. Получение URL-адреса трансляции

1. Перейдите в **CSS Toolkit** > [Генератор адресов](https://console.intl.cloud.tencent.com/live/addrgenerator/addrgenerator).
2. Завершите следующие настройки:
  2.1. Выберите тип адреса как **Адрес трансляции**.
  2.2. Выберите домен трансляции, который вы добавили в **Управление доменами**.
  2.3. Введите AppName. По умолчанию это live.
  2.4. Введите пользовательское `StreamName`, например `liveteststream`.
  2.5. Вам нужно выбрать тип шифрования на основе ваших требований безопасности и соображений производительности. Тип шифрования может быть либо **MD5**, либо **SHA256**, с **MD5** в качестве опции по умолчанию.
  2.6. Выберите время истечения URL-адреса, например `2024-07-31 12:08:42`.
3. Нажмите **Создать адрес**, чтобы создать адрес трансляции.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7ecbeb9e4e2911efb66652540055f650.png)

> **Примечание:** Значение по умолчанию `AppName` - `live`. `txSecret` - это подпись для воспроизведения, а `txTime` - время истечения URL-адреса. Вот еще один способ создания URL-адреса трансляции: В [Управлении доменами](https://console.tencentcloud.com/live/domainmanage) найдите домен, который вы хотите использовать для трансляции, и нажмите **Управление**. В разделе **Конфигурация трансляции** выберите время истечения для URL-адреса, введите пользовательское `StreamName` и нажмите **Создать адрес трансляции**. Перед созданием URL-адреса трансляции вы можете создать [шаблоны](https://intl.cloud.tencent.com/document/product/267/31069) и привязать их к вашему домену трансляции. Для цен на дополнительные услуги CSS см. [Обзор цен](https://intl.cloud.tencent.com/document/product/267/2819).

### Шаг 3. Начало трансляции

Чтобы начать трансляцию, предоставьте созданный URL-адрес трансляции программному обеспечению, которое вы используете для трансляции.

- Для трансляции на ПК рекомендуется использовать [OBS WebRTC трансляция.](https://www.tencentcloud.com/document/product/267/57042)
- Для трансляции в интернете рекомендуется использовать [Web Push](https://console.intl.cloud.tencent.com/live/tools/webpush): Нажмите **Создать**. В появившемся окне выберите домен трансляции, введите пользовательское `StreamName`, выберите время истечения URL-адреса и нажмите **Подтвердить**. Включите камеру и нажмите **Начать трансляцию**.
- Для трансляции с мобильных устройств загрузите приложение TCToolkit, откройте его, перейдите на страницу прямой трансляции и введите URL-адрес трансляции вручную или отсканируйте QR-код, созданный на предыдущем шаге, чтобы автоматически заполнить URL-адрес. Коснитесь **Начать трансляцию**.

> **Примечание:** Вы можете интегрировать MLVB SDK в ваше приложение, чтобы реализовать функцию трансляции. Решение LEB для веб-сайтов не поддерживает декодирование или воспроизведение B-фреймов. Подробнее см. [B-фреймы](https://www.tencentcloud.com/document/product/267/41030#b-frames).

### Шаг 4. Получение URL-адреса воспроизведения

2. Перейдите в **CSS Toolkit** > [Генератор адресов](https://console.tencentcloud.com/live/addrgenerator/addrgenerator) и завершите следующие настройки:
  2.1. Выберите тип адреса: **Адрес воспроизведения**.
  2.2. Выберите тип адреса как Адрес воспроизведения и выберите домен воспроизведения, который вы добавили в Управление доменами.
  2.3. Введите AppName. По умолчанию это live.
  2.4. Введите AppName. По умолчанию это live. Введите то же самое **StreamName**, что и адрес трансляции. StreamName адреса воспроизведения должен совпадать с **StreamName** адреса трансляции, чтобы воспроизвести соответствующий поток.
  2.5. Вам нужно выбрать тип шифрования на основе ваших требований безопасности и соображений производительности. Тип шифрования может быть либо **MD5**, либо **SHA256**, с **MD5** в качестве опции по умолчанию.
  2.6. Выберите время истечения URL-адреса, например `2024-07-31 12:08:42`.
  2.7. Выберите шаблон перекодирования, если вы хотите перекодировать поток и получить URL-адрес воспроизведения перекодированного потока. Этот шаг не требуется, если вы воспроизводите исходный поток.
  2.8. Нажмите **Создать адрес**, чтобы создать URL-адрес воспроизведения LEB в формате `webrtc://domain/path/stream_id`.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e94fb3dd4e2911ef8f105254002693fd.png)

3. Вы можете использовать следующие методы для тестирования воспроизведения в различных сценариях:
  - **Воспроизведение в интернете**: Рекомендуется использовать [демо TCPlayer](https://tcplayer.vcube.tencent.com/intl/index.html) для тестирования воспроизведения.

> **Примечание:** Демо поддерживает изменение качества видео во время воспроизведения. Вы можете создать шаблон перекодирования для вывода видео в HD и SD в **Конфигурация функций** > [Прямое перекодирование](https://console.tencentcloud.com/live/config/transcode), введите в демо URL-адрес WebRTC, содержащий шаблон перекодирования, и воспроизведите его. Если вам не нужно тестировать эту функцию, введите исходный URL-адрес WebRTC. Дополнительную информацию о прямом перекодировании и его выставлении счетов см. в [Прямое перекодирование](https://intl.cloud.tencent.com/document/product/267/31071).

  - **Воспроизведение на мобильных устройствах**: Рекомендуется использовать [приложение TCToolkit](https://www.tencentcloud.com/document/product/1071/38147?has_map=1) для тестирования воспроизведения на мобильных устройствах. Откройте приложение, выберите **Прямая трансляция > LEB Player**, введите URL-адрес воспроизведения вручную или отсканируйте QR-код, созданный на предыдущем шаге, чтобы автоматически заполнить URL-адрес, и затем коснитесь **Начать воспроизведение**.

> **Примечание:** Если вы хотите воспроизвести поток в своем приложении, вы можете интегрировать MLVB SDK. Если у вас есть вопросы, см. [Часто задаваемые вопросы](#часто-задаваемые-вопросы).

### Шаг 5. Создание группы адресов трансляции и воспроизведения

1. Перейдите в **CSS Toolkit** > [Генератор адресов](https://console.intl.cloud.tencent.com/live/addrgenerator/addrgenerator).
2. Выберите тип адреса как **URL-адреса трансляции и воспроизведения**.
3. Выберите **Домен трансляции** и **Домен воспроизведения**, которые вы добавили в Управление доменами.
4. Введите **AppName**. По умолчанию это ` live`.
5. Введите StreamName, например ` liveteststream`.
6. Вам нужно выбрать тип шифрования на основе ваших требований безопасности и соображений производительности. Тип шифрования может быть либо **MD5**, либо **SHA256**, с **MD5** в качестве опции по умолчанию.
7. Выберите время истечения URL-адреса, например `2024-07-31 12:08:42`.
8. Выберите существующий шаблон перекодирования (опционально).
9. Нажмите **Создать адрес**.

### Шаг 6. Использование LEB

- Наше решение LEB для **мобильных устройств** поддерживает декодирование B-фреймов и воспроизведение файлов AAC. Оно интегрировано в MLVB SDK, пожалуйста, обратитесь к [LEB](https://www.tencentcloud.com/document/product/1071/41875?lang=zh&pg=).
- **Веб-решение**: Интегрировано в плеер TCPlayer. Для интеграции обратитесь к [Веб-интеграция](https://www.tencentcloud.com/document/product/266/51742?lang=en&pg=).

## Часто задаваемые вопросы

### Создание URL-адресов воспроизведения

Формат URL-адреса воспроизведения LEB: `webrtc://domain/path/stream_id` или `webrtc://domain/path/stream_id?txSecret=xxx&txTime=xxx` ([защита от хотлинков](https://intl.cloud.tencent.com/document/product/267/31560) включена). О том, как создать URL-адрес воспроизведения, см. в [Получение URL-адреса воспроизведения](#step-4-получение-url-адреса-воспроизведения).

> **Примечание:** Вы можете использовать URL-адреса перекодированных потоков для воспроизведения с разными разрешениями и битрейтами. О том, как создать URL-адрес воспроизведения для перекодированного потока, см. в [Прямая переупаковка и перекодирование](https://intl.cloud.tencent.com/document/product/267/31561).

### B-фреймы

Решение Fast Live Web **не поддерживает декодирование и воспроизведение B-фреймов**. Поэтому, если исходный поток содержит B-фреймы, бэкэнд автоматически перекодирует поток, чтобы удалить B-фреймы. Однако этот процесс вводит дополнительную задержку перекодирования и влечет затраты на перекодирование.

Рекомендуется избегать трансляции потоков, содержащих B-фреймы. Пользователи могут удалить B-фреймы, отрегулировав параметры кодирования видео программного обеспечения трансляции (например, OBS). Если вы используете OBS для прямой трансляции, вы можете отключить B-фреймы через настройки.

Большой **интервал ключевого кадра (GOP)** может повлиять на опыт Fast Live, рекомендуется установить интервал на 2 секунды.

Как показано ниже:

### Перекодирование аудио

Воспроизведение с использованием браузеров поддерживает только стандартный протокол WebRTC и не поддерживает AAC. Если поток, содержащий аудиоданные в формате AAC, система перекодирует аудио в формат Opus, что повлечет затраты на [перекодирование аудио](https://intl.cloud.tencent.com/document/product/267/39604).

### Пояснение относительно проблем черного экрана при прямой трансляции

Live Event Broadcasting (LEB) требует привязки сертификата HTTPS, чтобы предотвратить сбои воспроизведения или пустые экраны, которые могут возникнуть, когда устройства переходят на воспроизведение HLS из-за несовместимости с LEB, когда отсутствие сертификата HTTPS иначе вызывает такие проблемы.

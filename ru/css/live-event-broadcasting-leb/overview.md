# Обзор

Live Event Broadcasting (LEB) — это версия CSS с ультранизкой задержкой. Она обеспечивает превосходное качество воспроизведения с задержкой в миллисекунды и подходит для сценариев с высокими требованиями к задержке, таких как онлайн-образование, трансляция спортивных событий и онлайн-викторины.

## Особенности

#### Воспроизведение с задержкой в миллисекунды

LEB использует протокол UDP для поддержания задержки в пределах 1 с, что намного ниже, чем 3-5 с при традиционном прямом эфире. Это, наряду с отличной производительностью мгновенной потоковой передачи и низкой частотой перебоев, гарантирует превосходный опыт потоковой передачи.

#### Разнообразные функции и плавная миграция

LEB интегрирует широкий спектр функций LVB, включая прямой эфир, транскодирование, запись, снятие скриншотов, обнаружение порнографии и воспроизведение. Также возможна плавная миграция с LVB.

#### Удобство использования, безопасность и надежность

Вы можете легко интегрировать LEB, так как он использует стандартные протоколы. Вы можете использовать его для воспроизведения в Chrome и Safari без установки каких-либо плагинов. Кроме того, его протоколы по умолчанию шифруют потоки для повышения безопасности и надежности.

## Цены

Платежные элементы LEB включают базовые услуги и дополнительные услуги. Базовые услуги оплачиваются по входящему и исходящему трафику/полосе пропускания, а дополнительные услуги, такие как транскодирование в прямом эфире, запись, снятие скриншотов и обнаружение порнографии, оплачиваются по использованию ресурсов. Дополнительные сведения см. в разделе [Обзор выставления счетов](https://intl.cloud.tencent.com/document/product/267/2819).

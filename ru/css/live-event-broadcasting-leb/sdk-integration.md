# Интеграция SDK

LEB является низколатентной версией LVB и обеспечивает превосходный опыт **потокового вещания** с задержкой воспроизведения в миллисекунды, намного ниже, чем при использовании традиционных протоколов потокового вещания.
Перед использованием LEB прочитайте [Обзор выставления счетов за LEB](https://intl.cloud.tencent.com/document/product/267/39969), чтобы узнать о подлежащих выставлению счетам элементах и ценах.

> **Примечание:** LEB использует протокол WebRTC для обеспечения низкой задержки. Он использует кодек Opus и не поддерживает B-кадры. Если исходный поток содержит B-кадры или кодек не является Opus, серверная часть CSS удалит B-кадры и перекодирует поток в формат Opus, что будет взимать [стандартные сборы за перекодирование](https://intl.cloud.tencent.com/document/product/267/39604).

## Интеграция в приложение

### Направления

- **Прямая трансляция**: Захват с камеры или экрана телефона и передача потока на CSS с использованием протокола RTMP. Подробнее см. [Публикация (Камера)](https://www.tencentcloud.com/document/product/1071/38157) и [Публикация (Запись экрана)](https://www.tencentcloud.com/document/product/1071/41878).
- **Воспроизведение в прямом эфире**: Воспроизведение потоков с использованием WebRTC с ультра-низкой задержкой. Подробнее см. [Воспроизведение > LEB](https://www.tencentcloud.com/document/product/1071/41875).

> **Примечание:** SDK MLVB использует возможности CSS, IM, TRTC и других сервисов для реализации низколатентного аудио- и видеообмена между несколькими участниками, позволяя участникам взаимодействовать друг с другом, пока другие наблюдают. Подробнее см. [Подключение микрофона](https://www.tencentcloud.com/document/product/1071/42210).

## Интеграция на веб-сайт

### Направления

- **Трансляция на веб**: Компонент трансляции разработан и упакован в соответствии со стандартом WebRTC, который поддерживается большинством браузеров. Вы можете включить функцию трансляции в реальном времени, просто импортировав предоставленный нами код. Подробнее см. [WebRTC Push](https://intl.cloud.tencent.com/document/product/267/41620).

> **Примечание:** Для трансляции потоков WebRTC используется аудиокодек Opus. Поэтому, если вы используете протокол LVB (RTMP, HTTP-FLV или HLS) для воспроизведения, серверная часть CSS автоматически преобразует звук в формат AAC для обеспечения успешного воспроизведения, что будет взимать сборы за перекодирование звука. Подробнее см. [Прямое перекодирование > Перекодирование аудио](https://intl.cloud.tencent.com/document/product/267/39604). Если вы публикуете и воспроизводите потоки с использованием протокола LEB, CSS не будет перекодировать звук. С WebRTC каждый домен трансляции может использоваться для до **1000 одновременных потоков** по умолчанию. Если вы хотите передавать больше потоков, пожалуйста, [отправьте запрос](https://console.tencentcloud.com/workorder/category).

- **Воспроизведение на веб**: Мы рекомендуем использовать наш SDK веб-проигрывателя [TCPlayerLite](https://www.tencentcloud.com/document/product/266/33977), который поддерживает воспроизведение **потоков WebRTC** на мобильных и настольных браузерах и обеспечивает превосходный опыт потокового вещания с задержкой в миллисекунды, намного ниже, чем при использовании традиционных протоколов потокового вещания.

> **Примечание:** Если браузер не поддерживает WebRTC, URL-адрес WebRTC, переданный в проигрыватель, будет преобразован для лучшей поддержки воспроизведения. По умолчанию WebRTC преобразуется в HLS на мобильных браузерах и в HTTP-FLV на настольных браузерах.

### Бесплатная демонстрация

- **Трансляция на веб**: Вы можете протестировать функцию веб-трансляции в [Web Push](https://console.tencentcloud.com/live/tools/webpush) консоли CSS.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/4fc6582e52ad11eeabd75254005810a4.png)

- **Воспроизведение на веб**: Вы можете использовать [демонстрацию TCPlayer](https://www.tencentcloud.com/document/product/266/33977) для проверки воспроизведения на веб.

> **Примечание:** Как трансляция, так и воспроизведение на веб используют стандартный протокол WebRTC. Веб-трансляция не создает B-кадры, а звук кодируется в формат Opus, поэтому не будет никаких затрат на перекодирование звука или удаление B-кадров.

## Реализация WebRTC Push с использованием OBS

LEB (сверхнизколатентное потоковое вещание) использует WebRTC для передачи прямого аудио- и видеоконтента или видеофайлов на сервер CSS. Далее описывается, как использовать OBS для передачи потоков WebRTC.

OBS поддерживает трансляцию в прямом эфире по протоколу WebRTC, что означает, что вы можете легко и быстро передавать потоки в прямом эфире на Tencent Cloud CSS с использованием протокола WebRTC на ПК (Windows/Mac/Ubuntu), точно так же, как при использовании протокола RTMP. Следующее содержание в основном описывает, как использовать инструмент OBS для реализации функции трансляции по протоколу WebRTC.

> **Примечание:** **При интеграции OBS с трансляцией по протоколу WebRTC доступны две схемы — новая и старая**: Новая схема - OBS WebRTC трансляция в прямом эфире (OBS v30.0 Beta 1 или выше): Эта схема **не требует плагина**, что делает процесс интеграции более удобным. Для получения конкретного руководства по работе обратитесь к: [OBS WebRTC трансляция в прямом эфире](https://www.tencentcloud.com/document/product/267/57042#). Старая схема - использование плагина OBS для трансляции WebRTC: Если вы используете версию OBS ниже v30.0 Beta 1 и не можете напрямую использовать протокол WebRTC для трансляции в прямом эфире, Tencent Cloud CSS предоставляет встроенное решение плагина OBS для трансляции WebRTC. Для получения конкретного руководства по работе обратитесь к: [Плагин OBS для трансляции WebRTC](https://www.tencentcloud.com/document/product/267/57042#a1ac1dab-cad0-4ebf-ad2f-9a20d82a0057).

На основе ваших конкретных требований и версии OBS вы можете выбрать подходящее решение для трансляции WebRTC. Пожалуйста, учтите, что фактическая производительность трансляции может быть затронута такими факторами, как производительность устройства, сетевые условия и буферизация проигрывателя. Во время процесса использования вы можете настраивать параметры трансляции и инструменты в соответствии с вашими потребностями для оптимизации опыта прямого вещания.

# Начало работы

Этот документ описывает, как начать работу с CSS. Перед тем как приступить к работе с CSS, рекомендуется ознакомиться с [Обзором цен](https://intl.cloud.tencent.com/document/product/267/2819) CSS, чтобы понять его **оплачиваемые элементы** и **цены**.

## Подготовка

2. Перейдите на [страницу активации сервиса CSS](https://console.tencentcloud.com/live?from=product-banner-use-lvb), подтвердите согласие с "Условиями обслуживания Tencent Cloud" и нажмите **Применить для активации**, чтобы активировать сервис CSS.

## Шаг 1. Добавление доменных имён

Для использования CSS необходимо иметь как минимум одно **доменное имя для трансляции** и одно **доменное имя для воспроизведения**. Одно доменное имя невозможно использовать одновременно для трансляции и воспроизведения.

1. Подготовьте собственные доменные имена. Для доменных имён в регионах материковой части Китая требуется ICP регистрация.
2. Войдите в консоль CSS, откройте [**Управление доменными именами**](https://console.tencentcloud.com/live/domainmanage) и нажмите **Добавить доменное имя**.![](https://staticintl.cloudcachetci.com/cms/backend-cms/884df034537c11ee974d5254005f490f.png)
  2.1. Теги используются для категоризации и управления ресурсами по разным измерениям. Если существующие теги не соответствуют вашим требованиям, вы также можете перейти в [Консоль управления тегами](https://console.intl.cloud.tencent.com/tag/taglist), чтобы управлять тегами централизованно.
  2.2. Нажмите **Добавить доменное имя**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/650cac86144911f09b3252540044a08e.png)

> **Примечание:** CSS предоставляет тестовое доменное имя `xxxx.livepush.myqcloud.com`. Вы можете использовать его для тестирования прямой трансляции, но не рекомендуется использовать его в качестве доменного имени для трансляции в производственных целях. После успешного добавления доменного имени вы сможете просматривать его информацию в списке доменных имён в **Управлении доменными именами**. Для получения информации об управлении см. [Управление доменными именами](https://www.tencentcloud.com/document/product/267/64153). Дополнительную информацию о доменных именах для прямой трансляции см. в [Основных возможностях CSS](https://intl.cloud.tencent.com/document/product/267/7968).

3. После добавления доменного имени система присвоит ему доменное имя CNAME (с суффиксом `.txlivecdn.com` или `.tlivepush.com`), которое невозможно использовать до завершения настройки CNAME у вашего поставщика услуг DNS. После вступления в силу конфигурации CNAME вы сможете использовать CSS. В следующем примере показано, как добавить запись CNAME, если Tencent Cloud является вашим поставщиком услуг DNS:
  3.1. Войдите в [консоль сервиса доменных имён Tencent Cloud](https://console.tencentcloud.com/domain).
  3.2. Найдите целевое доменное имя и нажмите **Разрешить**.
  3.3. На странице разрешения доменного имени нажмите **Добавить запись**.
  3.4. В новой строке введите префикс доменного имени в качестве записи хоста, выберите CNAME в качестве типа записи и введите доменное имя CNAME в качестве значения записи.
  3.5. Нажмите **Сохранить**.

> **Примечание:** После успешного добавления записи CNAME конфигурация CNAME вступает в силу через некоторое время. Если конфигурация не сработает, вы не сможете использовать CSS. После успешной конфигурации CNAME вы сможете видеть, что статус адреса CNAME в [**Управлении доменными именами**](https://console.tencentcloud.com/live/domainmanage) консоли CSS изменился на ![](https://staticintl.cloudcachetci.com/cms/backend-cms/bff81031537811ee94c3525400d793d0.png). Если сбой конфигурации CNAME сохраняется, обратитесь к вашему поставщику услуг DNS. Дополнительную информацию о настройке с помощью других поставщиков услуг DNS см. в [Настройка CNAME для доменного имени](https://intl.cloud.tencent.com/document/product/267/31057).

## Шаг 2. Получение URL трансляции

2. На странице **Генератор адресов** настройте следующее:
  2.1. Выберите **Доменное имя трансляции** для **Типа доменного имени**.
  2.2. Выберите доменное имя для трансляции, которое вы добавили в **Управлении доменными именами**.
  2.3. Введите **AppName**. По умолчанию это `live`.
  2.4. Введите пользовательское имя потока `StreamName`, например `liveteststream`.
  2.5. Необходимо выбрать тип шифрования на основе требований безопасности и соображений производительности. Тип шифрования может быть либо **MD5**, либо **SHA256**, причём **MD5** является опцией по умолчанию.
  2.6. Выберите время истечения URL, например `2024-07-30 16:59:07`.
3. Нажмите **Генерировать адрес**, чтобы создать адрес трансляции.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6872cd0f4d8b11efbaba525400d5f8ef.png)

> **Примечание:** В URL трансляции `live` — это `AppName` по умолчанию, `txSecret` — это подпись для воспроизведения потока, а `txTime` — это время истечения URL. Вот ещё один способ создания URL трансляции: В [**Управлении доменными именами**](https://console.tencentcloud.com/live/domainmanage) найдите доменное имя для трансляции, которое вы хотите использовать для создания URL трансляции, нажмите **Управлять**, выберите **Конфигурация трансляции**, введите время истечения URL и пользовательское имя потока `StreamName`, затем нажмите **Генерировать адрес трансляции**. Перед созданием URL трансляции вы можете создать [шаблоны](https://intl.cloud.tencent.com/document/product/267/31069) и привязать их к доменному имени для трансляции. Цены на дополнительные услуги CSS см. в [Обзоре цен](https://intl.cloud.tencent.com/document/product/267/2819).

## Шаг 3. Трансляция прямого потока

- Для трансляции на ПК рекомендуется использовать OBS. Подробные инструкции см. в [Трансляция через OBS](https://intl.cloud.tencent.com/document/product/267/31569).
- Для трансляции в веб-браузере рекомендуется использовать [**Веб-трансляция**](https://console.tencentcloud.com/live/tools/webpush): выберите доменное имя для трансляции, введите пользовательское имя потока `StreamName`, выберите время истечения URL, включите камеру и нажмите **Начать трансляцию**.
- Для трансляции на мобильных устройствах скачайте и установите Tencent Video Cloud Demo, откройте его, выберите **Мобильное вещание видео** > **Трансляция (Камера)**, введите URL трансляции в адресное поле вручную или отсканировав QR-код, а затем нажмите значок запуска в левом нижнем углу, чтобы начать трансляцию.

> **Примечание:** Вы можете интегрировать [MLVB SDK](https://www.tencentcloud.com/document/product/1071?lang=en&pg=) Tencent Cloud в пользовательские приложения для трансляции в прямом эфире.

## Шаг 4. Получение URL воспроизведения

2. Выберите **Набор инструментов CSS** > [**Генератор адресов**](https://console.tencentcloud.com/live/addrgenerator/addrgenerator), чтобы получить адрес воспроизведения и настроить следующее:
  2.1. Выберите **Доменное имя воспроизведения** для **Типа доменного имени**.
  2.2. Выберите доменное имя для воспроизведения, которое вы добавили в **Управлении доменными именами**.
  2.3. Введите **AppName**. По умолчанию это `live`.
  2.4. Введите имя потока `StreamName` из URL трансляции для воспроизведения соответствующего потока.
  2.5. Выберите время истечения URL, например `2024-07-30 16:59:07`.
  2.6. Необходимо выбрать тип шифрования на основе требований безопасности и соображений производительности. Тип шифрования может быть либо **MD5**, либо **SHA256**, причём **MD5** является опцией по умолчанию.
  2.7. Для создания URL воспроизведения транскодированного потока выберите шаблон транскодирования, который должен быть привязан к доменному имени в URL воспроизведения. Инструкции по привязке шаблона транскодирования см. в [Транскодирование прямой трансляции > Привязка доменного имени](https://intl.cloud.tencent.com/document/product/267/31071).
  2.8. Нажмите **Генерировать адрес**, чтобы создать адрес воспроизведения.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/d2c175154d8b11ef8f105254002693fd.png)

  - Для тестирования воспроизведения на ПК рекомендуется использовать инструменты, такие как [VLC](https://intl.cloud.tencent.com/document/product/267/32483). Дополнительную информацию см. в [Воспроизведение прямой трансляции](https://intl.cloud.tencent.com/document/product/267/31559).
  - Для тестирования воспроизведения на мобильных устройствах рекомендуется скачать и установить приложение Tencent Video Cloud Toolkit, выбрать **Мобильное вещание видео** > **Воспроизведение прямой трансляции**, введите URL воспроизведения в адресное поле вручную или отсканировав QR-код, и нажмите значок воспроизведения в левом нижнем углу.

> **Примечание:** Для трансляции/воспроизведения потока в приложении вы можете интегрировать [MLVB SDK](https://www.tencentcloud.com/products/mlvb) в приложение, чтобы дополнить сервис CSS. Если у вас возникли проблемы в период испытания, см. [Часто задаваемые вопросы](https://intl.cloud.tencent.com/document/product/267/7968).

## Шаг 5. Создание группы адресов трансляции и воспроизведения

1. Перейдите в **Набор инструментов CSS** > [Генератор адресов](https://console.intl.cloud.tencent.com/live/addrgenerator/addrgenerator).
2. Выберите тип адреса **URL-адреса трансляции и воспроизведения**.
3. Выберите **Доменное имя трансляции** и **Доменное имя воспроизведения**, которые вы добавили в управление доменными именами.
4. Введите **AppName**. По умолчанию это `live`.
5. Введите StreamName, например `liveteststream`.
6. Необходимо выбрать тип шифрования на основе требований безопасности и соображений производительности. Тип шифрования может быть либо **MD5**, либо **SHA256**, причём **MD5** является опцией по умолчанию.
7. Выберите время истечения URL, например `2024-07-30 16:59:07`.
8. Выберите существующий шаблон транскодирования (необязательно).
9. Нажмите **Генерировать адрес**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/19b1d6f54d8c11ef8357525400bdab9d.png)

## Операции

- Чтобы включить **запись прямой трансляции**, сначала создайте шаблон записи и привяжите его к доменному имени. Дополнительную информацию см. в [Создание шаблона записи](https://intl.cloud.tencent.com/document/product/267/34223).
- Чтобы включить **транскодирование прямой трансляции**, сначала создайте шаблон транскодирования и привяжите его к доменному имени. Дополнительную информацию см. в [Создание шаблона транскодирования](https://intl.cloud.tencent.com/document/product/267/31071).
- Чтобы включить **водяной знак прямой трансляции**, сначала создайте шаблон водяного знака и привяжите его к доменному имени. Дополнительную информацию см. в [Создание шаблона записи](https://intl.cloud.tencent.com/document/product/267/31073).
- Чтобы включить **захват скриншота и обнаружение контента для взрослых прямой трансляции**, сначала создайте шаблон захвата скриншота и обнаружения контента для взрослых и привяжите его к доменному имени. Дополнительную информацию см. в [Создание шаблона захвата скриншота и обнаружения контента для взрослых](https://intl.cloud.tencent.com/document/product/267/31072).
- Чтобы включить **Временный сдвиг прямой трансляции**, вы можете создать шаблон временного сдвига и привязать его к доменному имени. Дополнительную информацию см. в [Создание шаблона временного сдвига](https://www.tencentcloud.com/document/product/267/53312).
- Чтобы включить **обратные вызовы прямой трансляции**, вы можете создать шаблон обратного вызова и связать его с доменным именем для конфигурации. Связанную документацию см. в [Создание шаблона обратного вызова](https://www.tencentcloud.com/document/product/267/31074).
- Чтобы включить **Проверку прямой трансляции**, вы можете создать шаблон проверки и привязать его к доменному имени. Дополнительную информацию см. в [Создание шаблона проверки](https://www.tencentcloud.com/document/product/267/58265).
- Чтобы включить **Субтитры прямой трансляции**, вы можете создать шаблон субтитров и привязать его к доменному имени. Дополнительную информацию см. в [Создание шаблона субтитров](https://www.tencentcloud.com/document/product/267/58144).
- Чтобы включить **содержимое резервной копии прямой трансляции**, вы можете создать шаблон содержимого резервной копии и связать его с доменным именем для конфигурации. Связанную документацию см. в [Создание шаблона содержимого резервной копии](https://www.tencentcloud.com/document/product/267/53400?lang=en&pg=).
- Чтобы включить **микширование потоков прямой трансляции**, вызовите API микширования потоков [CreateCommonMixStream](https://intl.cloud.tencent.com/document/product/267/35997).
- Чтобы включить **Адаптивный битрейт прямой трансляции**, вы можете создать шаблон адаптивного битрейта и привязать его к доменному имени. Дополнительную информацию см. в [Создание шаблона адаптивного битрейта](https://www.tencentcloud.com/document/product/267/50271).

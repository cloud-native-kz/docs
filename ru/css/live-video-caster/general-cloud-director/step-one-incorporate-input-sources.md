# Шаг первый: добавление источников входного сигнала

Консоль Cloud Streaming Services предоставляет систему Live Video Caster (LVC). В этом документе описывается, как использовать LVC для онлайн-трансляции после активации функции LVC.

> **Примечание:** К оплачиваемым элементам LVC относятся продолжительность трансляции и сторонняя ретрансляция. Выставление счетов основано на времени использования. Дополнительную информацию см. в разделе [Обзор ценообразования Live Video Caster](https://www.tencentcloud.com/document/product/267/58538). Чтобы избежать начисления ненужных платежей, когда вы не используете кастер, нажмите **Stop** для кастера на [странице списка Live Video Caster](https://console.tencentcloud.com/live/caster?rid=5). Подробные инструкции см. в разделе [Управление Live Video Caster](https://www.tencentcloud.com/document/product/267/58533). LVC несовместим с браузерами Internet Explorer и Firefox. Рекомендуется использовать Chrome.

## Ограничения использования

- Каждый аккаунт может создать до **пяти** экземпляров LVC. Если в вашем аккаунте уже есть пять экземпляров, необходимо удалить существующий экземпляр перед добавлением нового. Для добавления большего количества экземпляров [отправьте заявку](https://console.tencentcloud.com/workorder/category).
- Вы можете добавить до **пяти** файлов VOD в список воспроизведения входного сигнала VOD.
- Сторонняя ретрансляция поддерживает до трех потоков, один из которых по умолчанию ретранслируется на текущий аккаунт CSS, а два других могут быть ретранслированы третьим лицам. Дополнительную информацию см. в разделе [Ретрансляция](https://www.tencentcloud.com/document/product/267/58530#.E6.AD.A5.E9.AA.A44.EF.BC.9A.E7.9B.B4.E6.92.AD.E8.BD.AC.E6.8E.A8).

## Подготовка

1. Убедитесь, что вы уже активировали [CSS](https://console.tencentcloud.com/live/livestat).
2. Убедитесь, что вы добавили домен **Push** и домен **Playback** в разделе [Управление доменами](https://console.tencentcloud.com/live/domainmanage) и завершили конфигурацию CNAME для обоих доменов.
3. Перейдите на страницу [Live Video Caster](https://console.tencentcloud.com/live/caster?rid=5) в консоли CSS, чтобы открыть страницу активации LVC. Установите флажок согласия с [Условиями обслуживания Tencent Cloud](https://www.tencentcloud.com/document/product/301/9248?has_map=1) и [Обзором выставления счетов LVC](https://www.tencentcloud.com/document/product/267/58538?has_map=1). Нажмите **Activate**, чтобы активировать LVC.
4. Убедитесь, что вы [создали Live Video Caster](https://www.tencentcloud.com/document/product/267/58533#.E6.96.B0.E5.BB.BA.E5.AF.BC.E6.92.AD.E5.8F.B0) на странице **Live Video Caster**.

## Операционные шаги

1. Выполните вход на [Live Video Caster](https://console.tencentcloud.com/live/caster?rid=5) и просмотрите список Live Video Caster.
2. Найдите кастер, который вы хотите отредактировать, и нажмите его **ID** или нажмите **Open** справа, чтобы открыть страницу редактирования кастера.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c6d75097dc8211f08a7a52540099c741.png)

3. В области источника входного сигнала нажмите **Add input**, чтобы добавить видео входной сигнал.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a490ed47dc8511f09143525400bf7822.png)

4. В появившемся окне установите тип входного сигнала и введите URL видео источника. Доступны следующие четыре типа входного сигнала:

Live

On demand (поддерживает редактирование)

Image

Publish local stream

Dynamic Overlays

| Live | Название входного сигнала: CustomТип входного сигнала: **Live****Pull from URL**: Прямой поток извлекается из URL в LVC. Поддерживаются протоколы RTMP, HLS, SRT и HTTP-FLV.**Publish**: Медиафайлы отправляются в LVC с использованием протокола RTMP. |
| On demand | Название входного сигнала: CustomТип входного сигнала: **On demand URL**. Нажмите **Add URL**.Поддерживает медиафайлы, хранящиеся в Tencent Cloud COS, и медиафайлы, хранящиеся у других поставщиков.Поддерживает форматы MP4, HLS и FLV.Вы можете ввести несколько URL-адресов файлов по запросу, разделенных точкой с запятой (;). Консоль автоматически циклирует через файлы в списке. |
| Image | Название входного сигнала: CustomТип входного сигнала: **Image****Image URL**: Поддерживает изображения JPEG, JPG, PNG и BMP размером не более 1920x1080 пикселей.**Local image**: Поддерживает загрузку изображений PNG, JPG и JPEG размером не более 5 МБ. |
| Publish local stream | Название входного сигнала: CustomТип входного сигнала: **Publish local stream****Screen Sharing**: Входной сигнал может быть общим экраном (окном приложения или рабочим столом). Поддерживаемые разрешения включают 1920x1080, 1280x720, 640x480 и 640x360.**Publish from camera**: Поддерживает использование локальной камеры в качестве источника входного сигнала с поддержкой разрешений 1920x1080, 1280x720, 640x480 и 640x360.В сценарии локальной потоковой передачи с камеры функция красоты отключена по умолчанию. В зависимости от ваших потребностей в бизнесе вы можете выбрать ручное включение или отключение этой функции.Настройка параметров: Вы можете настроить параметры эффекта красоты, чтобы улучшить свой внешний вид.Предоставляются различные персонализированные параметры для удовлетворения ваших особых потребностей, включая:Beauty Effects and Makeup: Могут помочь избавиться от недостатков кожи, улучшить текстуру кожи и добавить различные эффекты макияжа.Filters and Stickers: Вы можете добавить различные забавные фильтры и наклейки.Background and Emoji: Вы можете изменить видеофон или добавить эмодзи.Virtual Avatars: Вы можете преобразовать свой внешний вид в различные виртуальные аватары. |
| Dynamic Overlays | Название входного сигнала: CustomТип входного сигнала: Dynamic OverlaysНа основе ваших потребностей в бизнесе вы можете наложить Dynamic Overlays на прямую трансляцию со стороны вещания. > **Примечание:** Убедитесь, что каждый введенный вами URL-адрес по запросу/в прямом эфире доступен. В противном случае входной сигнал не будет воспроизводиться. Если источник входного сигнала прерывается, выход будет отображать черный экран.

5. Нажмите **Confirm**, чтобы закончить добавление источника входного сигнала. Система автоматически будет воспроизводить видео.

> **Примечание:** Чтобы **изменить источник входного сигнала**, нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/c332cbfba30b11eea869525400c26cb9.png) внизу источника входного сигнала, чтобы открыть страницу редактирования источника входного сигнала, заполните информацию, которую вам нужно изменить, и нажмите **Confirm**, чтобы сохранить изменение. ![](https://staticintl.cloudcachetci.com/cms/backend-cms/f97c39676ceb11f096685254001c06ec.png) После изменения нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/c337a770a30b11ee8964525400321be2.png) в нижнем левом углу видео источника, чтобы обновить. После обновления вы сможете увидеть отображаемый измененный источник входного сигнала. ![](https://staticintl.cloudcachetci.com/cms/backend-cms/1092d7c66cec11f09e39525400454e06.png) Чтобы **удалить источник входного сигнала**, нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/c33e921da30b11eeb6c6525400e46040.png) в верхней части источника входного сигнала. Появится диалоговое окно с запросом подтверждения. Нажмите **Confirm**, чтобы удалить источник входного сигнала. ![](https://staticintl.cloudcachetci.com/cms/backend-cms/2c8856ce6cec11f096685254001c06ec.png) При изменении или удалении источника входного сигнала источник входного сигнала не может использоваться в макете предпросмотра (PVW) или главного монитора (PGM).

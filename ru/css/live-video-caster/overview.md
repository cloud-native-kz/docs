# Обзор

Консоль Cloud Streaming Services предлагает сервис Live Video Caster (LVC). В этом руководстве описаны возможности и основные сценарии применения LVC.

## Возможности Live Video Caster

| Категория | Функция | Описание |
| --- | --- | --- |
| Источники входа, поддержка до 24 входов | Прямая трансляция | **Потоковая трансляция pull:** Получение из URL-адресов прямых трансляций: Вы можете добавить URL-адрес активной прямой трансляции в консоль LVC для получения потоков по URL. Поддерживаемые протоколы: RTMP, HLS и HTTP-FLV. Что касается форматов кодирования, для видео поддерживается только кодирование H.264, а для аудио — AAC. |
|  |  | **Прямой push:** RTMP push: Это означает отправку медиафайлов в систему LVC с использованием протокола RTMP. Что касается форматов кодирования, для видео поддерживается только кодирование H.264, а для аудио — AAC. |
|  | VOD | **VOD URL:** Поддерживает как медиафайлы, хранящиеся в Tencent Cloud COS, так и файлы, хранящиеся у других поставщиков. Поддерживаются форматы MP4, HLS и FLV (рекомендуется формат FLV). Консоль будет автоматически воспроизводить файлы в списке в цикле. Что касается форматов кодирования, для видео поддерживается только кодирование H.264, а для аудио — AAC. |
|  | Изображение | **URL изображения:** Поддерживаются форматы изображений JPEG, JPG, PNG и BMP. URL-адрес настраивается в LVC, размер изображения не превышает 1920*1080 пикселей. |
|  |  | **Локальные изображения:** Можно загружать изображения PNG, JPG и JPEG размером не более 5 МБ. |
|  | Локальный push потока | **Локальная камера:** Локальная камера может использоваться в качестве источника входа с поддержкой разрешений 1920 x 1080, 1280 x 720, 640 x 480 и 640 x 360. |
|  |  | **Общий доступ к экрану:** Входом может быть общий экран (окно приложения или рабочий стол). Поддерживаемые разрешения включают 1920 x 1080, 1280 x 720, 640 x 480 и 640 x 360. |
|  | Динамические наложения | Система поддерживает наложение динамических наложений на прямые трансляции, обеспечивая эффекты, такие как добавление объявлений, табло и представления персонажей к визуальному содержимому прямой трансляции. |
| Выход | Вывод в виде прямой трансляции | Поддерживаются протоколы выходного потока RTMP, HLS и HTTP-FLV. |
|  | Трансляция | Выходные потоки можно отправлять другим поставщикам, используя протокол RTMP. Адрес push должен быть в следующем формате: `rtmp://domain/app/stream?arg1=xxx.` |
|  | Запись | Поддерживаются все возможности записи прямых трансляций (запись в формате MP4, HLS и FLV, пользовательские длительности записи и так далее). |
| Обработка аудио/видео | Видеолayout | Вы можете создать пользовательский видеолayout, комбинируя любые входы. |
|  | Компоненты функций | Можно добавлять имена водяных знаков и точно настраивать их позиции. Поддерживается наложение водяных знаков и текста. |
|  | Резервный контент | LVC может автоматически переходить на резервное видео или изображение, которое вы указали, когда основной источник входа недоступен. |
|  | Обработка аудио | Громкость для каждого входного потока в выходном потоке можно отрегулировать. Поддерживается разделение аудио и видео. Например, когда воспроизводится видеоисточник 1, вы можете выбрать воспроизведение аудио видеоисточника 2. |
| Директорская обработка | Предварительный мониторинг, основной мониторинг | Поддерживается разметка изображений предварительного мониторинга и переключение с предварительного мониторинга на основной. |
|  | Отложенное воспроизведение | Поддерживается воспроизведение с задержкой до 300 секунд. То есть выходной поток может отставать от основного монитора на срок до 300 секунд. |

## Документация

- LVC в основном используется в сценариях, таких как игры и киберспорт, прямые трансляции электронной коммерции, онлайн-образование, прямые события и другие. Дополнительную информацию см. в [сценариях применения](https://www.tencentcloud.com/document/product/267/58535#).
- LVC обладает мощными и экономичными облачными возможностями, включая механизмы резервного копирования, смешивание потоков и многое другое. Дополнительную информацию см. в [Возможностях продукта](https://www.tencentcloud.com/document/product/267/58539#).

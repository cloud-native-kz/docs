# Устранение неполадок при воспроизведении

Если вы не можете смотреть CSS и не знаете, что именно вызвало проблему, вы можете быстро определить причину, выполнив следующие шаги:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/31873e93e10f11eeb419525400ea3514.png)

## Шаг 1. Проверьте URL воспроизведения

Прежде всего убедитесь, что URL воспроизведения правильный. Неправильный URL — наиболее вероятная причина большинства проблем. URL-адреса CSS Tencent Cloud включают URL для передачи и URL для воспроизведения. Сначала убедитесь, что **URL для передачи случайно не используется как URL для воспроизведения CSS**.

![diff](https://staticintl.cloudcachetci.com/cms/backend-cms/31799b14e10f11eeb1eb525400b5f95f.png)

> **URL воспроизведения для Mini CSS:**
> URL воспроизведения для Mini CSS можно получить путем отладки. Вы можете выполнить поиск по ключевому слову startPlay в глобальном поиске, затем установить точку останова отладки, где RTMP SDK вызывается Mini CSS. Параметр startPlay — это URL воспроизведения.

## Шаг 2. Проверьте видеопоток

Правильный URL воспроизведения не всегда гарантирует нормальное воспроизведение. Далее необходимо проверить, является ли видеопоток нормальным:

- В **CSS** URL CSS становится недоступным после остановки передачи VJ.
- В **VOD**, если видеофайлы были удалены, просмотр видео также невозможен.

Часто используемое решение — проверка с помощью VLC, бесплатного плеера на ПК, поддерживающего множество протоколов.

![](https://mc.qcloudimg.com/static/img/7923a14be5525bd37719c18d54243403/image.png)

## Шаг 3. Проверьте плеер

Если видеопоток в норме, то необходимо проверить, работает ли плеер правильно, в зависимости от конкретной ситуации:

### 3.1 Веб-браузер (A)

- **Формат**: мобильные браузеры поддерживают только URL воспроизведения в форматах **HLS (m3u8) и MP4**.
- **HLS (m3u8)**: протокол HLS Tencent Cloud основан на принципе «ленивого запуска». Кратко говоря, Tencent Cloud запускает трансформацию в формат HLS только когда зритель запрашивает URL воспроизведения в формате HLS. Цель — предотвратить расходование ресурсов. Но это также создает проблему: **URL воспроизведения в формате HLS не может воспроизводиться в течение 30 секунд после первого запроса от любого пользователя в мире**.

### 3.2 RTMP SDK (B)

Если RTMP SDK DEMO работает нормально при воспроизведении, рекомендуется проверить, не содержит ли логика интеграции ошибок, обратившись к документации по воспроизведению RTMP SDK [iOS] и [Android].

## Шаг 4. Проверьте блокировку брандмауэром (C)

Часто в корпоративных сетях многих клиентов воспроизведение видео ограничивается брандмауэрами, которые определяют, являются ли ресурсы, запрашиваемые через HTTP, потоковыми медиаресурсами (в конце концов, ни один босс не хочет, чтобы его сотрудники смотрели видео во время работы). Если вы можете нормально смотреть CSS через сеть 4G, но не можете через Wi-Fi сеть компании, это указывает на то, что в компании установлены ограничения на политику сети. В этом случае обратитесь к администратору для специальной обработки вашего IP-адреса.

## Шаг 5. Проверьте передатчик (D)

Если URL CSS не работает и исключена возможность блокировки брандмауэром, описанная на шаге 4, вероятно, передача была неудачной. Перейдите к разделу «Почему передача неудачна» для дальнейшего устранения неполадок.

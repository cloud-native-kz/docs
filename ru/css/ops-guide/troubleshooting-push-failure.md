# Устранение неполадок при неудачной трансляции

Если вы следуете инструкциям в документе [Лучшие практики — живая трансляция](https://intl.cloud.tencent.com/document/product/267/31558), но трансляция по-прежнему не удается, выполните действия по устранению типичных проблем с трансляцией, описанные в данном документе.

### 1. Проверьте, настроена ли для вашего доменного имени запись CNAME, которая указывает на адрес Tencent Cloud.

Трансляция может быть успешной только в том случае, если для вашего доменного имени настроена запись CNAME, указывающая на адрес Tencent Cloud. Вы можете проверить, настроена ли запись CNAME для добавленного доменного имени трансляции в столбце **CNAME** раздела [**Управление доменными именами**](https://console.tencentcloud.com/live/domainmanage). Если запись CNAME настроена, вы увидите зелёную галочку, как показано ниже:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f298895b911611efa50f52540075b605.png)

Если для вашего доменного имени не настроена запись CNAME, вы можете [её настроить](https://intl.cloud.tencent.com/zh/document/product/267/31057).

### 2. Проверьте, что сеть работает правильно.

Трансляция RTMP по умолчанию использует **порт 1935**. Если этот порт не открыт в брандмауэре тестовой сети, вы не сможете подключиться к серверу. Вы можете проверить, вызвала ли эта проблема сбой трансляции, переключившись на другую сеть (например, сеть 4G).

### 3. Проверьте, не слишком ли короткий период действия URL трансляции.

Чтобы предотвратить кража трафика, некоторые клиенты могут установить `txTime` (период действия URL трансляции) на очень короткий период, например 5 минут от текущего времени. Это необязательно, так как подпись `txSercet` гарантирует безопасность. Если период действия слишком короткий, URL трансляции может истечь после прерывания прямой трансляции из-за разрыва сетевого соединения, и впоследствии трансляция не может быть возобновлена.
Рекомендуется установить `txTime` на 12 или 24 часа от текущего времени, чтобы она была длиннее обычной сессии прямой трансляции.

### 4. Проверьте правильность значения `txSecret`.

Чтобы обеспечить безопасность, Tencent Cloud требует настройки защиты от hotlink для всех URL трансляции и **отклоняет** все URL защиты от hotlink, которые истекли или содержат ошибку в расчётах. При отклонении трансляции CSS SDK выбросит событие **PUSH_WARNING_SERVER_DISCONNECT**. Поведение [демонстрационного приложения Mobile Live Video Broadcasting](https://www.tencentcloud.com/document/product/1071/38147) в этом случае выглядит следующим образом:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/161ce3b7e10f11eeb419525400ea3514.jpeg)

Информацию о получении надёжных URL трансляции см. в разделе

Лучшие практики > Живая трансляция

### 5. Вызов startPush с использованием V2TXLivePusher возвращает ошибку -2?

В настоящий момент ошибка -2 будет выводиться в следующих сценариях:

- Используется V2TXLivePusher версии LiteAVSDK_Smart для трансляции протокола TRTC, который начинается с `trtc://`, потому что версия Smart не поддерживает протокол TRTC и требует профессиональной версии и версии Enterprise для его поддержки.
- Адрес трансляции не содержит необходимые параметры. При вызове startPush для трансляции потока обратитесь к документу [URL публикации/воспроизведения](https://www.tencentcloud.com/document/product/1071/39359?lang=en&pg=), чтобы создать правильный адрес потока.
- Режим инициализации проигрывателя — V2TXLiveMode_RTC, используется адрес протокола RTMP, который начинается с `rtmp://`.

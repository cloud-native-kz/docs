### Часто задаваемые вопросы

- **Почему изображение видео искажено и растянуто при воспроизведении в H5?**
 При воспроизведении видео в H5 растяжение видео не поддерживается. Убедитесь, что контейнер плеера имеет правильную конфигурацию ширины/высоты.
- **Почему я получил сообщение о том, что «CSS завершен. Повторите попытку позже»?**
 Если при попытке подключиться к адресу CSS вы не получаете ответ и не можете подключиться даже после пяти попыток, появляется это сообщение. В этом случае необходимо проверить, отправляется ли видеопоток и нормальна ли сетевая связь.
- **Видео не может быть скрыто в браузере QQ.**
 Браузер QQ берет на себя функцию воспроизведения видео из H5, а ядро X5 использует самостоятельно разработанный плеер для воспроизведения видео. Браузеры QQ используют унифицированный интерфейс воспроизведения для обеспечения хорошего пользовательского опыта. Дополнительную информацию см. в [документации браузера QQ](https://x5.tencent.com/tbs/guide.html).
- **Видео автоматически воспроизводится в полноэкранном режиме на iOS.**
 По умолчанию видео воспроизводится в полноэкранном режиме в системе iOS из-за параметра webkit. Для достижения встроенного воспроизведения в приложении можно установить атрибут webkit-playsinline. Любой браузер Safari в iOS ниже версии 10 не может отключить автоматический переход в полноэкранный режим.
- **Почему Flash-плеер имеет две кнопки воспроизведения в Chrome на ПК?**
 Начиная с Chrome 42, Flash больше не воспроизводится автоматически. Chrome автоматически воспроизводит только основное содержимое Flash CSS, а другое содержимое Flash CSS приостанавливается, пока пользователи не включат его вручную.
- **Почему я могу воспроизводить видео CSS в браузерах на ПК, но не на мобильных устройствах?**
 В браузерах на мобильных устройствах поддерживаются только видео HLS. Убедитесь, что адрес получения CSS содержит URL-адрес получения HLS.

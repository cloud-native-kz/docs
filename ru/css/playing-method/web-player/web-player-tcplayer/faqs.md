# Часто задаваемые вопросы

### Какие браузеры поддерживаются TcPlayer?

TcPlayer был протестирован и поддерживает следующие браузеры: ПК: IE 10+, Edge, Chrome, Firefox, QQ Browser, MAC Safari. Мобильные устройства: Android 4.4+, iOS 8.0+, WeChat, Mobile QQ, QQ Browser, Chrome, Safari.
Для поддержки IE 8 и IE 9 необходимо подключить скрипт Polyfill перед подключением скрипта плеера, как показано ниже:

```
    <!--[if lt IE 9]>    <script src="//imgcache.qq.com/open/qcloud/video/vcplayer/libs/es5-shim.js" charset="utf-8"></script>    <script src="//imgcache.qq.com/open/qcloud/video/vcplayer/libs/es5-sham.js" charset="utf-8"></script>    <![endif]-->    <script src="//imgcache.qq.com/open/qcloud/video/vcplayer/TcPlayer-2.2.1.js" charset="utf-8"></script>;
```

### На мобильных устройствах видео не переходит в полноэкранный режим при переключении TcPlayer в полноэкранный режим, и интерфейс браузера все еще отображается. Почему?

Сначала следует знать, что решение полноэкранного режима, предоставляемое TcPlayer, использует Fullscreen API + псевдо-полноэкранный режим. Если браузер поддерживает Fullscreen API, контейнер видео заполнит весь экран при переходе плеера в полноэкранный режим, а панель управления предоставляется TcPlayer, как показано на рисунке:

![](https://mc.qcloudimg.com/static/img/df40b2b49390f8fc314fd040ba026156/image.png)

(Android Chrome)

Если браузер не поддерживает Fullscreen API, будет использован псевдо-полноэкранный режим, как показано на рисунке:

![](https://mc.qcloudimg.com/static/img/d5746d9bef48b411c3bac576fe6925b1/image.png)

![](https://main.qcloudimg.com/raw/ae4930644e249d0536b937d7de6d6289.png)

(Слева: WeChat на Android. Справа: WeChat на iOS)
Панели управления, отображаемые в этих двух полноэкранных режимах, все предоставляются TcPlayer, и вы входите в эти режимы, нажимая на кнопку полноэкранного режима на панели управления или используя метод, предоставленный TcPlayer. Однако панель управления TcPlayer может не всегда отображаться на мобильных устройствах, так как их webview захватывает воспроизведение видео в большинстве случаев и использует панель управления, предоставляемую webview. В этом случае панель управления TcPlayer не будет отображаться, и вы не сможете использовать полноэкранное решение, предоставляемое TcPlayer, как показано на рисунке:

![](https://main.qcloudimg.com/raw/684ab0223ead48aa053f0828ed323351.png)

![](https://main.qcloudimg.com/raw/f2aca48c35b86ce8695ea213123ae5d7.png)

(Слева: WeChat на Android, где воспроизведение видео захвачено TBS. Справа: iOS, где воспроизведение видео захвачено QQ Browser)
При входе в полноэкранный режим:

![](https://main.qcloudimg.com/raw/71c3c9eff736ca84090d70c10f487cd0.png)

![](https://main.qcloudimg.com/raw/9d941f7f72326dab581b99bbce89ef8d.png)

(Слева: WeChat на Android. Справа: QQ Browser на iOS)
Как видите, панель управления отличается от панели TcPlayer. В этом полноэкранном режиме API для JavaScript не предоставляются, поэтому TcPlayer не может реализовать этот режим. Мы обычно называем полноэкранный режим, в котором видео занимает весь экран, System Full Screen, а режим, в котором видео занимает всю область страницы браузера, — Pseudo-Full Screen. Следовательно, если вы видите интерфейс браузера после перехода в полноэкранный режим, то это псевдо-полноэкранный режим. Вы можете использовать только панель управления, предоставляемую системой, если хотите войти в системный полноэкранный режим на мобильных устройствах. Вы можете выбрать тип панели управления, используя атрибут "controls" TcPlayer.

### Почему экран растягивается при воспроизведении видео в H5?

При воспроизведении видео в H5 растяжение видео не поддерживается. Проверьте, правильно ли настроена конфигурация ширины/высоты контейнера плеера.

### Почему я не могу разместить свой div поверх видео?

Разные браузеры по-разному реализуют тег `<video>`. Например, если вы откроете веб-страницу в QQ или WeChat (на системе Android), то велика вероятность, что используется ядро браузера X5 (ядро, связанное с QQ или WeChat, то есть ядро браузера QQ). По определенным причинам команда QQ Browser реализовала решение, при котором "видео `<video>` должно быть размещено на верхнем слое" (дополнительную информацию см. в документации QQ Browser). Однако благодаря недавней координации между командами компании команда QQ Browser постепенно меняет эту политику. Эта проблема, возможно, уже была решена в последнем ядре X5 браузера, который вы читаете в этом документе.

### Почему настроенное изображение обложки не работает?

Это та же проблема, что и в предыдущем случае ("невозможно разместить div поверх видео"). Изображение обложки не может быть отображено до тех пор, пока браузер не позволит элементам перекрывать тег `<video>`.

### Почему видео отображается в полноэкранном режиме по умолчанию в определенных мобильных браузерах?

Если видео воспроизводится посредством встроенного воспроизведения внутри приложения (то есть ваше собственное приложение инкапсулирует элемент управления iOS webview, который используется для отображения веб-страниц. В этом режиме вы можете настроить детали webview, которые могут отличаться от стандартного браузера Safari), вы можете настроить атрибут "webkit-playsinline" для тега `<video>` в HTML (или настроить атрибут "playsinline", если вы используете iOS 10), а затем настроить "allowsInlineMediaPlayback" для webview. Таким образом, видео воспроизводятся в неполноэкранном режиме (встроенное воспроизведение) при открытии страниц в приложении.
Если вы откроете страницу в Safari на iOS 10, вы можете реализовать неполноэкранное воспроизведение (встроенное воспроизведение), используя описанный выше метод (настройте атрибут "playsinline" для тега `<video>`). Вы не можете отключить автоматическое полноэкранное воспроизведение для Safari на системах ниже iOS 10. Этот атрибут уже добавлен для нашего браузера. Остается только его поддержка устройствами.
Для устройств Android известно, что существует множество различных настроенных версий операционных систем Android, и каждая система по-разному реализует тег `<video>` без универсального стандарта. По этой причине возможности воспроизведения видео на Android имеют гораздо меньшую согласованность по сравнению с iOS. Атрибут "webkit-playsinline playsinline" уже добавлен для плеера согласно текущему универсальному методу. Остается только его поддержка устройствами.

### Почему я не могу реализовать автоматическое воспроизведение видео в мобильных браузерах?

Существует только два способа реализации автоматического воспроизведения видео на мобильной веб-странице: путем настройки атрибута "autoplay" для тега `<video>` или путем вызова метода "play()", предоставляемого тегом `<video>`. Однако автоматическое воспроизведение видео всегда было запрещено на мобильной веб-странице, поэтому универсальный метод теперь заключается в запуске воспроизведения видео по действиям пользователя. Например, отслеживайте события ontap пользователей и вызывайте метод "play()". Возможно, что определенные настроенные webviews поддерживают атрибут "autoplay" тега `<video>` или могут реализовать автоматическое воспроизведение видео, вызывая другие специальные функции. Видео могут воспроизводиться автоматически при открытии страниц в этих webviews. Наш плеер добавит атрибут "autoplay" для тега `<video>`, если "autoplay" настроен как "true". Теперь нам остается только поддержка этого атрибута устройствами.

### Почему Flash-плеер имеет две кнопки воспроизведения в Chrome на ПК?

Начиная с Chrome 42, Flash больше не воспроизводится автоматически (Google приобрел WebRTC и сделал его открытым кодом). Chrome автоматически воспроизводит только основное содержимое Flash, в то время как другое содержимое Flash приостанавливается до тех пор, пока пользователи не включат его вручную.

### Почему видео CSS можно воспроизводить в браузерах на ПК, но не на мобильных устройствах?

При воспроизведении видео потоковой трансляции в мобильных браузерах поддерживается только протокол HLS (M3U8). Таким образом, вы должны убедиться, что адреса потоковой трансляции содержат URL для извлечения потока HLS (M3U8). Видео не может быть воспроизведено на мобильных телефонах, если вы предоставляете нашему плееру только адрес flv или rtmp.

# Устранение неисправностей

## Описание функции

Этот документ описывает, как использовать Web Player (Web SDK) сервиса Tencent Cloud VOD, который позволяет пользователям непосредственно использовать проверенные возможности воспроизведения видео. Благодаря гибким API-интерфейсам SDK можно легко интегрировать в самостоятельно разработанные веб-приложения для достижения воспроизведения, сравнимого с десктопными приложениями. При этом Web SDK обеспечивает возможность загрузки видео с веб-сайта.

Форматы файлов, поддерживаемые SDK, ограничены функцией глобальной защиты от hotlink-а. Дополнительную информацию см. в разделе часто задаваемых вопросов на официальном сайте. Этот документ предназначен для разработчиков, которые хотят использовать Web SDK проигрывателя Tencent Cloud VOD для разработки и имеют базовые знания JavaScript.

## Поддерживаемые функции

### Формат воспроизведения

В следующей таблице приведены форматы видео, поддерживаемые Web SDK:

| Формат воспроизведения | Браузер ПК | Мобильный браузер |
| --- | --- | --- |
| HLS (m3u8) | Да | Да |
| MP4 | Да | Да |
| FLV | Нет | Нет |

**Совместимость с системой Android.** В отличие от iPhone, которые используют только браузер Safari, на телефонах Android может быть установлено множество различных встроенных браузеров. По этой причине совместимость браузеров Android стала проблемой, которая беспокоит отрасль. Поэтому приведенная выше таблица не применима ко всем телефонам Android.

### Формат загрузки

Форматы видео, поддерживаемые SDK при загрузке, следующие:

| Стандарт | Подробный формат |
| --- | --- |
| Microsoft | WMV, WM, ASF, ASX |
| REAL | RM, RMVB, RA, RAM |
| MPEG | MPG, MPEG, MPE, VOB, DAT |
| Другое | MOV, 3GP, MP4, MP4V, M4V, MKV, AVI, FLV, F4V |

**Сервис транскодирования платформы VOD.** Поскольку форматы MP4 и HLS (m3u8) широко поддерживаются как ПК, так и мобильными браузерами, платформа Tencent Cloud VOD всегда публикует загруженные видео в этих форматах.

### Совместимость платформ

Создание двух наборов кода для мобильного браузера и браузера ПК требует чрезмерных усилий. Однако, используя этот проигрыватель, один и тот же код будет адаптироваться к браузеру ПК/мобильному браузеру, что означает, что проигрыватель автоматически выберет оптимальный метод воспроизведения в зависимости от платформы, на которой он работает. Например, на ПК сервис будет использовать Flash-плеер в качестве приоритета, чтобы справиться с различными форматами видео. На смартфоне сервис будет использовать технологию HTML5 для воспроизведения видео.

## Подготовка

### Шаг 1. Активация сервиса

Зарегистрируйте учетную запись Tencent Cloud на [официальном сайте Tencent Cloud](https://intl.cloud.tencent.com/) и активируйте сервис **VOD**.

### Шаг 2. Загрузка файлов

После активации сервиса VOD перейдите на страницу [Управление видео VOD](https://console.tencentcloud.com/video/videolist) для загрузки новых видеофайлов. Поскольку документ предназначен для введения в использование проигрывателя, эта операция поможет вам получить собственный адрес онлайн-видео. Вы не сможете открыть эту страницу, если не активировали сервис VOD.

### Шаг 3. Получение ID

После загрузки видео вы можете найти идентификатор файла (наиболее важную информацию для воспроизведения видео проигрывателем) на странице управления видео. Проигрыватель также включает функцию статистики качества. Перед использованием проигрывателя необходимо проверить свой APPID, который можно найти на странице управления видео. На рисунке ниже идентификатор слева — это ID видеофайла, а другой — это ваш APPID.

### Шаг 4. Подготовка страниц

Введите скрипт инициализации на страницу, на которой вы хотите воспроизводить видео (включая ПК или H5):

```
<script src="//qzonestyle.gtimg.cn/open/qcloud/video/h5/h5connect.js" charset="utf-8"></script>;
```

**Ограничение локальных файлов.** Вы не можете воспроизводить локальные файлы с помощью проигрывателя из-за проблем кроссдомена. Вы должны получать доступ к проигрывателю через IP-адрес или домен — вот почему мы попросили вас сначала загрузить видеофайл и получить его адрес онлайн-воспроизведения.

## Добавление проигрывателя

Вы можете добавить видеопроигрыватель на страницу, выполнив два простых шага ниже.

### Шаг 1. Добавление контейнера проигрывателя

Поместите контейнер проигрывателя на страницу, где вы хотите отобразить проигрыватель. Например, добавьте следующий код в index.html. ID контейнера, ширина и высота могут быть настроены.

```
<div id="id_video_container" style="width:100%; height:auto;"></div>;
```

### Шаг 2. Создание объекта проигрывателя

Затем создайте объект проигрывателя во введенном JavaScript с помощью конструктора проигрывателя.

```
var player = new qcVideo.Player("id_video_container", {    "file_id": "1465197896261041838",    "app_id": "125132611",    "width":640,    "height":480});
```

Конструктор используется для создания объекта проигрывателя и поиска соответствующего видео для воспроизведения на основе file_id и app_id. Вы можете управлять проигрывателем с помощью объекта проигрывателя. Дополнительную информацию см. в разделе [Обзор методов API](#third_video) для параметров объекта проигрывателя.

### Полный пример кода

```
<!DOCTYPE html><html><head>    <meta http-equiv="X-UA-Compatible" content="IE=Edge,chrome=1" />    <meta http-equiv="Content-Type" content="text/html; charset=utf-8"/>    <meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=0, minimum-scale=1.0, maximum-scale=1.0"/>    <title>VOD</title></head><body><div id="id_video_container" style="width:100%; height:auto;"></div><script src="//qzonestyle.gtimg.cn/open/qcloud/video/h5/h5connect.js" charset="utf-8"></script><script type="text/javascript">    (function () {        var player = new qcVideo.Player("id_video_container", {            "file_id": "1465197896261041838",            "app_id": "125132611",            "width":640,            "height":480        });    })()</script></body></html>
```

## Другие случаи

### Случай 1. Как воспроизвести видео, если у меня есть адрес видео, но нет file_id и app_id?

Необходимо передать адрес воспроизведения видео, в этом случае file_id и app_id не требуются. Пример на JS:

```
    var option = {    "width": 640,    "height": 480,    //...Вы можете использовать другие пользовательские атрибуты    "third_video": {    "urls":{            20 : "http://208.vod.myqcloud.com/204.mp4"//Это только пример. Пожалуйста, замените его фактическим адресом видео        }    }};var player = new qcVideo.Player("id_video_container", option);
```

Атрибут "urls" параметра "third_video" является объектом, в котором вы можете передать несколько адресов видео с разными качеством видео. Подробное описание параметров можно найти в разделе [Обзор методов API](#third_video).

> **Примечание.** "urls" должен включать по крайней мере один адрес видео

### Случай 2. Как использовать «Комментарии на экране»?

После инициализации проигрывателя вы можете добавить комментарии на экран для видео, вызвав метод addBarrage(barrage) объекта проигрывателя. Дополнительную информацию о параметрах см. в разделе [Обзор методов API](#barrage). Например, добавьте два комментария на экран для воспроизводимого видео:

```
var barrage = [{"type":"content", "content":"hello world", "time":"1"},{"type":"content", "content":"Center", "time":"1", "style":"C64B03;30","position":"center"}];player.addBarrage(barrage);
```

> **Примечание.** **Комментарии на экране реализованы только на фронтенде. Поддержка бэкенда должна разрабатываться самостоятельно. Эта функция применима только к Flash-плеерам на ПК, но не поддерживается в H5.**

### Случай 3. Как выполнить некоторые операции в конце видео, такие как рекомендация видео?

Используйте параметр listener и передайте функцию обратного вызова для события playStatus. Эта функция будет вызвана при изменении статуса воспроизведения. Описание конкретных параметров функции обратного вызова см. в разделе [Обзор API](#playstatus).

Например:

```
var option ={    "file_id":"1465197896261041838",    "app_id":"125132611",    "width":800,    "height":720    //...Вы можете использовать другие пользовательские атрибуты};var listener = {    playStatus: function (status){        //TODO        console.log(status);    }};var player = new qcVideo.Player("id_video_container", option, listener);
```

### Случай 4. Как сделать, чтобы проигрыватель запомнил прогресс воспроизведения видео и начал воспроизведение с этой же точки в следующий раз?

В "option" установите параметр "remember" в значение 1. Проигрыватель записывает временную точку, когда видео было остановлено при предыдущем воспроизведении, и продолжает с этой точки при следующем воспроизведении. Например:

```
var option ={    "file_id":"1465197896261041838",    "app_id":"125132611",    "width":800,    "height":720,    "remember":1    //...Вы можете использовать другие пользовательские атрибуты};var player = new qcVideo.Player("id_video_container", option);
```

### Случай 5. Как сделать, чтобы проигрыватель автоматически изменял свой размер вместе с веб-страницей?

Используйте метод объекта проигрывателя "resize(width, height)" для динамического изменения размера проигрывателя.

```
player.resize(640, 480);
```

### Случай 6. Как воспроизвести видео, которое было настроено с паролями из Cloud Video Management?

Так же, как воспроизведение обычных видео, SDK автоматически отображает окно ввода пароля. Воспроизведение начинается после ввода пароля.

> **Примечание.** Функция пароля доступна только при воспроизведении видео путем передачи идентификаторов видео.

### Случай 7. Как сгенерировать ссылку, которая передается с использованием QR-кода или ссылки?

Пример (пожалуйста, замените appid и fileid в ссылке на фактические ID):

```
http://play.video.qcloud.com/qrplayer.html?appid=1251769111&fileid=14651978969211156176147&autoplay=0&sw=640&sh=426&$def=20&wmode=transparent
```

```
http://play.video.qcloud.com/iplayer.html?appid=1251769111&fileid=14651978969211156176147&autoplay=0&sw=1800&sh=1200&def=20&wmode=transparent
```

### Случай 8. Как указать определение воспроизведения видео?

Используйте параметр "definition" для указания качества, с которым будет воспроизводиться видео (при условии, что это качество доступно для этого видео). Это доступно для двух методов воспроизведения: воспроизведение с ID видео и воспроизведение с адресом. Дополнительную информацию см. в ссылке [Описание параметров](#definition). Например:

```
var option ={    "file_id":"14651978969261415426",    "app_id":"1251606588",    "definition":30,    "width":800,    "height":700};var player = new qcVideo.Player("id_video_container", option);
```

## Обзор методов API

### Конструктор

```
qcVideo.Player(id, option, listener);
```

**ID**

: String;

**Обязательно**

; этот параметр указывает ID контейнера, в котором находится проигрыватель, и может быть настроен.

**option**

: Object ;

**Обязательно**

.
Этот параметр указывает параметры, которые можно установить для параметров проигрывателя. Параметры следующие:

| Параметр | Тип | Значение по умолчанию | Описание |
| --- | --- | --- | --- |
| file_id | String | Нет | Уникальный ID файла VOD. Это **обязательно** при воспроизведении видео по ID видео |
| app_id | String | Нет | Параметр **обязателен**, если видео прямой трансляции воспроизводится с использованием ID видео. Для видео под одной учетной записью этот параметр остается неизменным. |
| width | Number | Нет | **Обязательно**, используется для установки ширины проигрывателя (в пиксельях). Пример: 640 |
| height | Number | Нет | **Обязательно**, используется для установки высоты проигрывателя (в пиксельях). Пример: 480 |
| auto_play | Number | 0 | Разрешено ли автоматическое воспроизведение. 0: отключить; 1: включить **Примечание. Этот параметр применяется только к Flash-плеерам на ПК.** |
| disable_full_screen | Number | 0 | Разрешен ли полноэкранный режим. 0: включить; 1: отключить **Примечание. Этот параметр применяется только к Flash-плеерам на ПК.** |
| disable_drag | Number | 0 | Разрешено ли пользователям перетаскивать полосу прогресса видео. 0: разрешить; 1: запретить**Примечание. Этот параметр применяется только к Flash-плеерам на ПК.** |
| stretch_full | Number | 0 | Должно ли видео быть масштабировано до размера проигрывателя. 0: не масштабировать. 1: масштабировать на весь экран**Примечание. Этот параметр применяется только к Flash-плеерам на ПК.** |
| stop_time | Number | Нет | Пробный режим. Например, если вы установите значение "60", воспроизведение видео остановится через 60 секунд, и при этом будет запущено событие "playStatus" |
| remember | Number | 0 | Запомнить ли последнюю позицию воспроизведения. 0: нет. 1: да. Если включено, временная точка, когда видео было остановлено при предыдущем воспроизведении, будет записана, и следующее воспроизведение начнется с этой позиции. **Примечание. Этот параметр применяется только к Flash-плеерам на ПК.** |
| playbackRate | Number | 1 | Скорость воспроизведения. Например, "2" означает, что видео будет воспроизводиться с двойной скоростью, а "0,5" означает половинную скорость. **Примечание. Этот параметр применяется только к H5-плеерам** |
| hide_h5_setting | Boolean | false | Скрывать ли кнопку параметров H5. true: скрыть. false: не скрывать |
| hide_h5_error | Boolean | false | Скрывать ли сообщения об ошибках H5. **Примечание. Этот параметр применяется только к H5-плеерам.** |
| WMode | String | window | В оконном режиме вы не можете размещать другие элементы страницы поверх Flash-плеера. Вы можете изменить это на opaque или значения параметров для другого flash wmode, если требуется. **Примечание. Этот параметр применяется только к Flash-плеерам на ПК.** |
| stretch_patch | Boolean | false | Если настроено на "true", видеообъявление, которое отображается при запуске, завершении или паузе видео, будет увеличено, чтобы закрыть весь проигрыватель. |
| definition | Number | Нет | Используется для указания качества, с которым будет воспроизводиться видео. Настроенное качество должно быть доступно для видео. Доступные значения: 10, 20, 30, 40, 210, 220, 230, 240. Дополнительную информацию о взаимосвязи между этими значениями и типами видео см. в описании параметров для [third_video](#third_video). |

### Получение параметров и статуса

Здесь приведены методы для получения параметров и статусов объекта проигрывателя, возвращаемого конструктором

| Метод | Возвращаемое значение | Описание |
| --- | --- | --- |
| getVolume | Number. Диапазон значений: 0-1 | Получить текущий уровень громкости |
| getDuration | Number (в сек) | Получить общую длительность текущего видео |
| getCurrentTime | Number (в сек) | Получить текущую позицию воспроизведения |

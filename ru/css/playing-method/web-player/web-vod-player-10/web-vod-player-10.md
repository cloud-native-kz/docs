# Web VOD Player 1.0

## Описание функций

Данный документ описывает использование Web Player (Web SDK) сервиса Tencent Cloud VOD, который позволяет пользователям непосредственно использовать проверенные возможности воспроизведения видео. Благодаря гибким API, SDK можно легко интегрировать с собственными Web Apps для достижения воспроизведения, аналогичного настольным приложениям. Одновременно Web SDK предоставляет возможность загрузки видео с веб-конца.

Форматы файлов, поддерживаемые SDK, ограничены функцией глобальной защиты от горячих ссылок. Дополнительную информацию см. в разделе часто задаваемых вопросов на официальном веб-сайте. Данный документ предназначен для разработчиков, которые хотят использовать Web SDK плеера Tencent Cloud VOD для разработки и обладают базовыми знаниями JavaScript.

## Поддерживаемые функции

### Формат воспроизведения

В следующей таблице перечислены форматы видео, поддерживаемые Web SDK:

| Формат воспроизведения | Браузер ПК | Мобильный браузер |
| --- | --- | --- |
| HLS (m3u8) | Да | Да |
| MP4 | Да | Да |
| FLV | Нет | Нет |

**Совместимость с системой Android:** В отличие от iPhone, которые используют только браузер Safari, на телефонах Android может быть встроено множество различных браузеров. По этой причине совместимость браузеров Android является проблемой, которая беспокоит индустрию. Поэтому приведённая выше таблица не применима ко всем телефонам Android.

### Формат загрузки

Форматы видео, поддерживаемые SDK для загрузки, выглядят следующим образом:

| Стандарт | Подробный формат |
| --- | --- |
| Microsoft | WMV, WM, ASF, ASX |
| REAL | RM, RMVB, RA, RAM |
| MPEG | MPG, MPEG, MPE, VOB, DAT |
| Прочие | MOV, 3GP, MP4, MP4V, M4V, MKV, AVI, FLV, F4V |

**Сервис транскодирования платформы VOD:** Поскольку форматы MP4 и HLS (m3u8) широко поддерживаются как в браузерах ПК, так и в мобильных браузерах, платформа Tencent Cloud VOD всегда публикует загруженные видео в этих форматах.

### Совместимость платформ

Создание двух наборов кода для браузера смартфона и браузера ПК требует чрезмерных усилий. Однако, используя этот плеер, один и тот же код будет автоматически адаптироваться к браузеру ПК/мобильному браузеру, что означает, что плеер автоматически выберет оптимальный метод воспроизведения в зависимости от платформы, на которой он работает. Например: на ПК сервис будет использовать Flash-плеер в качестве первого приоритета, чтобы справиться с различными форматами видео. На смартфоне сервис будет использовать технологию HTML5 для воспроизведения видео.

## Подготовка

### Шаг 1: Активация сервиса

Зарегистрируйте учётную запись Tencent Cloud на [официальном веб-сайте Tencent Cloud](https://intl.cloud.tencent.com/) и активируйте сервис **VOD**.

### Шаг 2: Загрузка файлов

После активации сервиса VOD перейдите в раздел [Управление видео VOD](http://console.tencentcloud.com/video/videolist) для загрузки новых файлов видео. Поскольку данный документ предназначен для представления использования плеера, эта операция поможет вам получить собственный адрес онлайн видео. Вы не сможете открыть эту страницу, если не активировали сервис VOD.

### Шаг 3: Получение идентификатора

После загрузки видео вы можете найти идентификатор файла (наиболее базовая информация, необходимая плеру для воспроизведения видео) на странице управления видео. Плеер также включает функцию статистики качества. Перед использованием плеера необходимо проверить ваш APPID, который можно найти на странице управления видео. На рисунке ниже идентификатор слева — это идентификатор видеофайла, а другой — это ваш APPID.

### Шаг 4: Подготовка страниц

Представьте скрипт инициализации на странице, на которой вы хотите воспроизводить видео (включая ПК или H5):

```
<script src="//qzonestyle.gtimg.cn/open/qcloud/video/h5/h5connect.js" charset="utf-8"></script>;
```

**Ограничение на локальные файлы:** Вы не можете воспроизводить локальные файлы с помощью плеера из-за проблем с кроссдоменом. Вы должны получить доступ к плееру через IP-адрес или домен — именно поэтому мы попросили вас сначала загрузить видеофайл и получить его адрес онлайн-воспроизведения.

## Добавление плеера

Вы можете добавить видеоплеер на вашу страницу, выполнив два простых шага ниже.

### Шаг 1: Добавление контейнера плеера

Поместите контейнер плеера на странице, на которой вы хотите отобразить плеер. Например, добавьте следующий код в index.html. Идентификатор контейнера, ширина и высота могут быть настроены.

```
<div id="id_video_container" style="width:100%; height:auto;"></div>;
```

### Шаг 2: Создание объекта плеера

Далее создайте объект плеера в представленном JavaScript, используя конструктор плеера.

```
var player = new qcVideo.Player("id_video_container", {    "file_id": "1465197896261041838",    "app_id": "125132611",    "width":640,    "height":480});
```

Конструктор используется для создания объекта плеера и поиска соответствующего видео для воспроизведения на основе file_id и app_id. Вы можете управлять плеером с помощью объекта плеера. Дополнительную информацию см. в разделе [Обзор методов API](#third_video) для параметров объекта плеера.

### Полный пример кода

```
<!DOCTYPE html><html><head>    <meta http-equiv="X-UA-Compatible" content="IE=Edge,chrome=1" />    <meta http-equiv="Content-Type" content="text/html; charset=utf-8"/>    <meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=0, minimum-scale=1.0, maximum-scale=1.0"/>    <title>VOD</title></head><body><div id="id_video_container" style="width:100%; height:auto;"></div><script src="//qzonestyle.gtimg.cn/open/qcloud/video/h5/h5connect.js" charset="utf-8"></script><script type="text/javascript">    (function () {        var player = new qcVideo.Player("id_video_container", {            "file_id": "1465197896261041838",            "app_id": "125132611",            "width":640,            "height":480        });    })()</script></body></html>
```

## Другие случаи

### Случай 1: Как воспроизвести видео, если у меня есть адрес видео, но нет file_id и app_id?

Необходимо передать адрес видео, в этом случае file_id и app_id не требуются. Пример JS:

```
    var option = {    "width": 640,    "height": 480,    //...Вы можете использовать другие пользовательские атрибуты    "third_video": {    "urls":{            20 : "http://208.vod.myqcloud.com/204.mp4"//Это только пример. Пожалуйста, замените это на фактический адрес видео        }    }};var player = new qcVideo.Player("id_video_container", option);
```

Атрибут "urls" параметра "third_video" — это объект, в который можно передать несколько адресов видео с разными определениями видео. Подробное описание параметров см. в разделе [Обзор методов API](#third_video).

> **Примечание:** "urls" должен содержать по крайней мере один адрес видео

### Случай 2: Как использовать "On-screen Comment"?

После инициализации плеера вы можете добавлять на видео встроенные комментарии, вызывая метод addBarrage(barrage) объекта плеера. Дополнительную информацию о параметрах см. в разделе [Обзор методов API](#barrage). Например, добавьте два встроенных комментария к воспроизводящемуся видео:

```
var barrage = [{"type":"content", "content":"hello world", "time":"1"},{"type":"content", "content":"Center", "time":"1", "style":"C64B03;30","position":"center"}];player.addBarrage(barrage);
```

> **Примечание:** **Встроенный комментарий реализуется только на фронтенде. Поддержка бэкенда должна быть разработана самостоятельно. Эта функция применяется только к Flash-плеерам на ПК, но не поддерживается в H5.**

### Случай 3: Как выполнить некоторые операции в конце видео, такие как рекомендация видео?

Используйте параметр listener и передайте функцию обратного вызова для события playStatus. Эта функция будет вызвана при изменении статуса воспроизведения. Описание конкретных параметров функции обратного вызова см. в разделе [Обзор API](#playstatus).

Например:

```
var option ={    "file_id":"1465197896261041838",    "app_id":"125132611",    "width":800,    "height":720    //...Вы можете использовать другие пользовательские атрибуты};var listener = {    playStatus: function (status){        //TODO        console.log(status);    }};var player = new qcVideo.Player("id_video_container", option, listener);
```

### Случай 4: Как заставить плеер запомнить прогресс воспроизведения видео и начать воспроизведение видео с того же момента в следующий раз?

В "option" установите параметр "remember" в значение 1. Плеер записывает точку времени, когда видео было остановлено при предыдущем воспроизведении, и продолжает с этого момента при следующем воспроизведении. Например:

```
var option ={    "file_id":"1465197896261041838",    "app_id":"125132611",    "width":800,    "height":720,    "remember":1    //...Вы можете использовать другие пользовательские атрибуты};var player = new qcVideo.Player("id_video_container", option);
```

### Случай 5: Как заставить плеер автоматически изменять размер вместе с веб-страницей?

Используйте метод объекта плеера "resize(width, height)" для динамического изменения размера плеера.

```
player.resize(640, 480);
```

### Случай 6: Как воспроизводить видео, которые были настроены с паролями в управлении облачным видео?

Так же, как воспроизведение обычных видео, SDK автоматически отображает окно ввода пароля. Воспроизведение начинается после ввода пароля.

> **Примечание:** Функция пароля доступна только при воспроизведении видео путём передачи идентификаторов видео.

### Случай 7: Как создать ссылку, совместно используемую с помощью QR-кода или ссылки?

Пример (пожалуйста, замените appid и fileid в ссылке на фактические идентификаторы):

```
http://play.video.qcloud.com/qrplayer.html?appid=1251769111&fileid=14651978969211156176147&autoplay=0&sw=640&sh=426&$def=20&wmode=transparent
```

```
http://play.video.qcloud.com/iplayer.html?appid=1251769111&fileid=14651978969211156176147&autoplay=0&sw=1800&sh=1200&def=20&wmode=transparent
```

### Случай 8: Как указать определение воспроизведения видео?

Используйте параметр "definition" для указания определения, с помощью которого будет воспроизводиться видео (при условии, что это определение доступно для этого видео). Это доступно для двух методов воспроизведения: воспроизведение с идентификатором видео и воспроизведение с адресом. Дополнительную информацию см. в ссылке [Описание параметров](#definition). Например:

```
var option ={    "file_id":"14651978969261415426",    "app_id":"1251606588",    "definition":30,    "width":800,    "height":700};var player = new qcVideo.Player("id_video_container", option);
```


: String;

**Обязательно**

; Этот параметр указывает идентификатор контейнера, в котором находится плеер в

**option**

: Object ;

**Обязательно**

.
Этот параметр указывает параметры, которые можно установить для параметров плеера. Параметры следующие:

| Параметр | Тип | Значение по умолчанию | Описание |
| --- | --- | --- | --- |

### Получение параметра и статуса

Здесь приведены методы получения параметров и статусов для объекта плеера, возвращаемого конструктором

| Метод | Возвращаемое значение | Описание |
| --- | --- | --- |
| getVolume | Number. Диапазон значений: 0-1 | Получить текущий уровень громкости |
| getDuration | Number (в сек) | Получить общую продолжительность текущего видео |

# Концепции

### Push

Процесс, при котором хост отправляет локальные видео и аудиоисточники на серверы Tencent Video Cloud. В некоторых случаях также известен как "RTMP push".

### Playback

Прямая трансляция, при которой видео и аудио извлекаются с серверов Tencent Video Cloud и воспроизводятся по указанному адресу после выполнения прямой отправки потока. Видеоисточник генерируется в реальном времени. Имеет смысл только если кто-то отправляет прямой поток. После того как хост прекращает трансляцию, URL прямого потока становится недействительным. Поскольку прямые потоки воспроизводятся в реальном времени, полоса прогресса не отображается в плеере во время воспроизведения.

### Push domain name

Доменное имя, используемое для отправки прямых потоков, которое является обязательной настройкой. Перед использованием доменного имени для прямой трансляции необходимо его зарегистрировать. После настройки доменного имени push CSS генерирует соответствующий URL для отправки. Дополнительные сведения см. в разделе [Splicing Live Stream URLs](https://intl.cloud.tencent.com/document/product/267/38393).

### Playback domain name

Доменное имя, используемое для воспроизведения прямых потоков, которое является обязательной настройкой. Перед использованием доменного имени для прямой трансляции необходимо его зарегистрировать. После настройки доменного имени playback CSS генерирует соответствующий URL для воспроизведения. Дополнительные сведения см. в разделе [Splicing Live Stream URLs](https://intl.cloud.tencent.com/document/product/267/38393).

### Domain CNAME

Доменное имя с суффиксом `.liveplay.myqcloud.com`, назначаемое системой доменному имени ускорения, настроенному в консоли CSS. Необходимо настроить запись CNAME у своего поставщика услуг доменных имен. После вступления записи в силу CSS будет отвечать за разрешение доменного имени, и все запросы, поступающие на это доменное имя, будут перенаправлены на граничные серверы CSS.

### AppName

### Transcoding

- Повышение совместимости: перекодировка исходного видео в форматы, совместимые с несколькими типами устройств для плавного воспроизведения.
- Повышение адаптируемости к пропускной способности: перекодировка исходного видео в выходные форматы Smooth, SD, HD и UHD. Конечные пользователи могут выбирать битрейты в зависимости от своих условий сети.
- Снижение использования пропускной способности: использование продвинутых кодеков для перекодировки с целью значительного снижения битрейта видео с сохранением исходного качества, что снижает использование пропускной способности на восходящем канале.

### H.264

- Позволяет передавать SD цифровое изображение (разрешением ниже 1280 x 720) со скоростью ниже 1 Мбит/с.
- Обеспечивает лучшее качество изображения по сравнению с другими видеокодеками при той же пропускной способности.

### H.265

- Позволяет передавать общее HD аудио/видео (720p с разрешением 1280 x 720) со скоростью 1-2 Мбит/с.
- Достигает оптимального баланса между битовым потоком, качеством кодирования, задержкой и сложностью алгоритма.

### Event message notification

Когда во время прямой отправки потока срабатывает уведомление о событии, Tencent Cloud отправляет запрос на ваш сервер в соответствии с настроенным шаблоном сообщения, и ваш сервер проводит аутентификацию и отправляет ответ на запрос. Дополнительные сведения о протоколе ответа см. в разделе [Event Message Notification Protocol](https://intl.cloud.tencent.com/document/product/267/38080). После прохождения аутентификации ваш сервер получит пакет JSON, содержащий информацию обратного вызова для анализа и записи.

### Referer

Поле `txSecret` в URL-адресах отправки и воспроизведения. Referer используется для предотвращения подделки URL-адресов отправки злоумышленниками через ваш бэкенд и кража URL-адресов воспроизведения.

### Live recording

Во время отправки потока видеофайлы, созданные путем мультиплексирования исходных потоков (без изменения информации, такой как данные аудио и видео и соответствующие временные метки), могут храниться на платформе VOD. Для использования этой функции необходимо предварительно активировать [VOD](https://www.tencentcloud.com/products/vod).

### Watermark

Чтобы защитить авторские права на видео во время прямой отправки потока, вы можете добавить водяной знак к видеопотоку во время перекодировки. Водяной знак может быть текстовым или изображением.

### Screencapture

Эта функция захватывает видеоизображения отправленных прямых потоков через указанные интервалы и затем сохраняет созданные файлы изображений в COS. Чтобы включить захват скриншотов, необходимо предварительно предоставить CSS разрешение на запись в ваш корзину COS. Дополнительные сведения см. в разделе [Authorizing CSS to Store Screenshots in a COS Bucket](https://intl.cloud.tencent.com/document/product/267/33384).

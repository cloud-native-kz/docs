# LEB

Live Event Broadcasting (LEB) — это ультранизкозадержное решение на основе CSS. Обеспечивает превосходное качество воспроизведения с задержкой в миллисекунды, что делает его идеальным для сценариев с высокими требованиями к задержке, таких как онлайн-образование, потоковая трансляция спортивных событий и онлайн-викторины.

## Возможности

#### Воспроизведение с задержкой в миллисекунды

Реализация возможностей прямой трансляции с задержкой на уровне миллисекунд в сценариях с высокой параллелизацией с использованием протокола UDP, что устраняет недостатки задержки в 3–5 секунд при традиционной прямой трансляции, одновременно учитывая ключевые показатели, такие как мгновенный запуск и частота зависаний, обеспечивая пользователям опыт прямой трансляции с ультранизкой задержкой.

#### Разнообразные возможности и гладкая миграция

LEB интегрирует широкий спектр возможностей LVB, включая прямую отправку, трансокодирование, запись, снимок экрана, обнаружение порнографии и воспроизведение. Также обеспечивает гладкую миграцию из LVB.

#### Удобство использования, безопасность и надежность

Вы можете легко интегрировать LEB, так как он использует стандартные протоколы. Вы можете использовать его для воспроизведения в Chrome и Safari без установки дополнительных плагинов. Кроме того, его протоколы по умолчанию шифруют потоки для повышения безопасности и надежности.

## Ценообразование

Подлежащие оплате услуги LEB включают базовые услуги и услуги с добавленной стоимостью. Базовые услуги выставляются счетом по потребленному входящему и исходящему трафику/полосе пропускания. Услуги с добавленной стоимостью, такие как прямое трансокодирование, запись, снимок экрана и обнаружение порнографии, выставляются счетом по использованию ресурсов. Дополнительные сведения см. в [Обзор выставления счетов](https://intl.cloud.tencent.com/document/product/267/2819).

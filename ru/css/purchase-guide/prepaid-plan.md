## Важно знать

- Коэффициент преобразования для единиц трафика/полосы пропускания составляет 1000. Например, 1 ТБ = 1000 ГБ.
- Необходимо сначала купить план перед их использованием для вычета.
- Вы можете использовать ваучеры CSS для покупки плана, но общие ваучеры неприменимы.
- План становится действительным сразу после покупки. Период действия составляет один год. Использование каждый день вычитается на следующий день (фактическое время выставления счета может варьироваться). План, срок действия которого истекает раньше всего, будет использован для вычета в первую очередь. Если все ваши планы исчерпаны, дополнительное использование будет выставлено ежедневно по тарифам оплаты по мере использования.
- Вы можете приобрести несколько планов, но периоды действия планов не будут добавляться.
- План не может вычитать использование, которое произошло в день до покупки плана.

> Предположим, вы покупаете план 25 декабря 2022 года. 26 декабря его можно будет использовать для вычета использования 25 декабря (выставлено 26 декабря), но он не может вычитать использование 24 декабря (выставлено 25 декабря).

- **Если вы используете режим ежемесячного выставления счетов, ваш план будет заморожен, и его действие не будет продлено. Для изменения на режим ежедневного выставления счетов, пожалуйста, обратитесь к вашему представителю отдела продаж. Изменение вступит в силу в следующем месяце.**
- Вы можете вернуть план в течение пяти дней с момента покупки, если он не был использован. **Независимо от того, исчерпан ли план, он считается использованным, если он был использован для вычета использования**. ### Детали выставления счета

- План трафика может вычитать использование трафика только в режиме ежедневного счета по трафику. Если вы переключитесь на ежемесячное выставление счетов или ежедневное выставление счетов по полосе пропускания, план будет заморожен, и их периоды действия не будут продлены.
- Восходящий и нисходящий трафик, потребленный за пределами материковой части Китая, вычитается в различных соотношениях в зависимости от региона.
- Использование нисходящего трафика сначала вычитается из плана трафика. Если ваши планы исчерпаны, дополнительное использование будет выставлено ежедневно по [тарифам оплаты по мере использования](https://www.tencentcloud.com/document/product/267/2818).

### Примеры выставления счета

#### Пример 1

Предположим, пользователь A, использующий режим ежедневного счета по трафику, купил план трафика CSS объемом 10 ТБ 4 декабря 2022 года и потребил 11 ТБ нисходящего трафика LVB на материковой части Китая 4 декабря 2022 года. 5 декабря счет за трафик для пользователя A будет следующим:

- 10 ТБ будет вычтено из плана трафика.
- Оставшиеся 1 ТБ будут выставлены по тарифам оплаты по мере использования, что составляет 0.0423 (USD/ГБ) x 1 x 1000 = 42.3 USD.

#### Пример 2

Предположим, пользователь B потребил 100 ГБ нисходящего трафика LEB на материковой части Китая, 50 ГБ нисходящего трафика LEB в Европе, 50 ГБ трафика LEB в Азиатско-Тихоокеанском регионе 3, 100 ГБ нисходящего трафика LVB на материковой части Китая, 100 ГБ нисходящего трафика LVB в Северной Америке, 50 ГБ восходящего трафика на материковой части Китая и 50 ГБ восходящего трафика в Азиатско-Тихоокеанском регионе 1 4 декабря 2022 года, и использовался режим ежедневного счета по трафику для LVB и LEB внутри и за пределами материковой части Китая. 5 декабря 2022 года счет за трафик для пользователя B будет следующим:

- Вычитаемое использование трафика = 100 ГБ x 2 + 50 ГБ x 5.3846 + 50 ГБ x 3.3846 + 100 ГБ + 100 ГБ x 1.6923 + 50 ГБ + 50 ГБ x 1.7692 = 1,046.15 ГБ
- Если у пользователя B был план трафика объемом 1 ТБ, использование будет вычтено в следующем порядке:
  - Нисходящий трафик LEB на материковой части Китая: 100 ГБ x 2 = 200 ГБ (осталось 800 ГБ)
  - Нисходящий трафик LEB в Азиатско-Тихоокеанском регионе 3: 50 ГБ x 5.3846 = 269.23 ГБ (осталось 530.77 ГБ)
  - Нисходящий трафик LEB в Европе: 50 ГБ x 3.3846 = 169.23 ГБ (осталось 361.54 ГБ)
  - Нисходящий трафик LVB на материковой части Китая: 100 ГБ (осталось 261.54 ГБ)
  - Нисходящий трафик LVB в Северной Америке: 100 ГБ x 1.6923 = 169.23 ГБ (осталось 92.31 ГБ)
  - Восходящий трафик на материковой части Китая: 50 ГБ (осталось 42.31 ГБ)
  - Восходящий трафик в Азиатско-Тихоокеанском регионе 1 = 50 ГБ x 1.7692 = 88.46 ГБ (план исчерпан). Оставшийся трафик восходящего направления объемом 46.15 ГБ, потребленный в Азиатско-Тихоокеанском регионе 1, будет выставлен по тарифам оплаты по мере использования (которые совпадают с [тарифами оплаты по мере использования для нисходящего трафика LVB](https://www.tencentcloud.com/document/product/267/2818)): 0.0748 (USD/ГБ) x 46.15 ГБ = 3.45202 USD.

### Детали выставления счета

- План стандартного транскодирования может вычитать продолжительность стандартного транскодирования и аудиотранскодирования только в режиме ежедневного выставления счета. Если вы переключитесь на ежемесячное выставление счетов, план будет заморожен, и их периоды действия не будут продлены.

### Как работает вычет

План стандартного транскодирования вычитает продолжительность транскодирования H.264_480P в соотношении 1:1. Продолжительность других разрешений и продолжительность аудиотранскодирования вычитаются в различных соотношениях на основе их тарифов оплаты по мере использования.

- Вычитаемая продолжительность для конкретного разрешения транскодирования = продолжительность плана x Тариф оплаты по мере использования для H.264_480P ÷ [Тариф оплаты по мере использования для разрешения](https://www.tencentcloud.com/document/product/267/39604)
- Вычитаемая продолжительность для аудиотранскодирования = продолжительность плана x Тариф оплаты по мере использования для H.264_480P ÷ [Тариф оплаты по мере использования для аудиотранскодирования](https://www.tencentcloud.com/document/product/267/39604) = продолжительность плана x 0.0028 ÷ 0.00099

### Примеры вычета

Предположим, пользователь A купил план стандартного транскодирования на 100 часов за 13.43 USD.

- Вычитаемая продолжительность для транскодирования H.264_720P = 100 (часов) x 0.0028 ÷ 0.0057 ≈ 49.122807‬ ≈ 49.12 часов
- Вычитаемая продолжительность для аудиотранскодирования = 100 (часов) x 0.0028 / 0.00099 ≈ 282.828283 ≈ 282.83 часов

### Как работает вычет

План транскодирования TSC вычитает продолжительность транскодирования TSC 480p в соотношении 1:1. Продолжительность других разрешений вычитается в различных соотношениях на основе их тарифов оплаты по мере использования.
Вычитаемая продолжительность для конкретного разрешения транскодирования = Продолжительность плана x Тариф оплаты по мере использования для транскодирования TSC H.264_480P ÷ [Тариф оплаты по мере использования для разрешения](https://www.tencentcloud.com/zh/document/product/267/39604)

### Примеры вычета

Предположим, пользователь A купил план транскодирования TSC на 50 часов (22.24 USD). Продолжительность транскодирования H.264_720P, которую может вычесть план, будет следующей:
50 (часов) x 0.0116 ÷ 0.0222 ≈ 26.1261261 ≈ 26.13 часов

### Примеры выставления счета

Предположим, пользователь B провел сеанс потоковой трансляции LVB в Азиатско-Тихоокеанском регионе 1. Трансляция длилась четыре часа и наблюдалась 200 людьми. Битрейт составлял 3 Мбит/с, использовался кодек H.264, разрешение составляло 1080p.

- Если бы TSC транскодирование не использовалось, стоимость полосы пропускания была бы: 3 Мбит/с x 200 x 0.1854 = 111.24 USD.
- Если бы использовалось TSC транскодирование для снижения битрейта, стоимость полосы пропускания была бы: 111.24 x (100% - 30%) = 77.868 USD.
- Стоимость транскодирования TSC была бы: 0.0443 x 240 = 10.632 USD (используется цена списка).
- Общая стоимость была бы: 77.868 + 10.632 = 88.5 USD.

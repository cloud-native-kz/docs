# Live Quality Control и Live Assessment

## Обзор

CSS предоставляет возможности **Live Quality Control** (Контроль качества в реальном времени) и **Live Assessment** (Оценка в реальном времени), позволяющие вам всесторонне выявлять проблемы качества в живых потоках и оценивать показатели производительности во время трансляций, а также предоставлять соответствующие рекомендации.

#### Live Quality Control

Предоставляет возможности **Format Diagnosis** (Диагностика формата) и **Content Quality Inspection** (Проверка качества контента):

| Возможность |  | Описание |
| --- | --- | --- |
| **Format Diagnosis** |  | Облачный сервис обработки аудио и видео, предназначенный для диагностики живых потоков в реальном времени. Способен анализировать такие проблемы, как аномалии информации потока, нарушения синхронизации временных меток, ненормальное состояние потока, ошибки инкапсуляции контейнера и отказы декодирования. |
| Content Quality Inspection | Common Image Issue | Поддерживает обнаружение различных проблем качества живой трансляции, включая артефакты мозаики, сбои экрана, размытие изображения, дрожание экрана и визуальные шумы. |
|  | Recognition Code Detection | Поддерживает возможности обнаружения QR-кодов, штрих-кодов, кодов приложений и других идентификационных кодов. |
|  | Solid Color Screen and Brightness Detection | Поддерживает обнаружение сценариев в живой трансляции, таких как черные/белые края, одноцветные экраны, условия низкой освещенности и переэкспозиция. |
|  | Common Sound Issue | Поддерживает различные функции обнаружения качества живого аудио, включая Silence (Тишина), Low Voice (Тихий голос) и High Voice (Громкий голос). |

#### Live Assessment

Включает **Scoring Without Reference** (Оценка без опорного сравнения) и **Scoring With Reference** (Оценка с опорным сравнением):

- **Scoring Without Reference** (Оценка без опорного сравнения): Алгоритм объективно оценивает качество живых потоков без опоры на сравнительные ссылки, непосредственно предоставляя оценки.
- **Scoring With Reference** (Оценка с опорным сравнением): Как правило, кодированный повторно поток оценивается и получает баллы путем сравнения с исходным потоком или потоком с водяным знаком. Возможности представлены в таблице ниже:

| Возможность | Описание |
| --- | --- |
| PSNR | Peak Signal-to-Noise Ratio (Пиковое отношение сигнала к шуму): Проводит оценку с опорным сравнением качества переформатированного потока путем расчета среднеквадратической ошибки между исходным/потоком с водяным знаком и переформатированным потоком для измерения качества сжатия или восстановления изображения живой трансляции. |
| SSIM | Structural Similarity Index (Индекс структурного сходства): Проводит оценку с опорным сравнением качества переформатированного потока путем сравнения яркости, контраста и структурной информации между переформатированным потоком и исходным/потоком с водяным знаком, имитируя восприятие качества изображения человеческой зрительной системой. |
| VMAF | Video Multimethod Assessment Fusion (Мультиметодная оценка и объединение видео): Проводит объективную оценку с опорным сравнением качества переформатированного потока на основе нескольких основных метрик и моделей машинного обучения. |

## Важная информация

- Если исходный поток, поток с водяным знаком и переформатированный поток одновременно сконфигурированы как содержимое обнаружения в шаблоне Live Quality Control и Live Assessment, Live Quality Control и Scoring Without Reference инициируют задачи контроля качества и оценки для исходного потока, потока с водяным знаком (если применимо) и переформатированного потока. Scoring With Reference же инициирует задачи оценки только для переформатированного потока.
- Live Quality Control и Live Assessment запускаются путем отправки потока.
- Минимальная единица задачи для контроля качества и оценки составляет одну минуту. Задачи продолжительностью менее одной минуты будут тарифицированы как одна минута.

## Детали расчета платежей

- Объект расходов: Продолжительность Live Quality Control
- Способ расчета: Оплата по факту
- Цикл расчета: Ежедневное выставление счетов. Почасовое выставление счетов. Счета выставляются и комиссии вычитаются почасово. Детальное выставление счетов и время выставления подлежат фактическому счету-фактуре.

## Примеры расчета платежей

Если вы используете услуги Quality Control и Assessment 1 июня 2025 года с содержимым обнаружения, сконфигурированным в шаблоне как исходный поток и переформатированный поток (шаблон одного переформатирования), обе разрешения живых потоков установлены на 1080P и включены возможности контроля качества, включая Format Diagnosis, Crash Screen Detection, QR Code Detection, Barcode Detection, Silence Detection, High Voice Detection, а также возможности оценки такие как PSNR и SSIM, при общей продолжительности живой трансляции 1 час с постоянным включением обнаружения, сумма счета будет:

- Платежи за контроль качества:
  - Платежи за диагностику формата = 0.008 (USD/мин) × 60 (мин) (исходный поток) + 0.008 (USD/мин) × 60 (мин) (переформатированный поток) = 0.96 USD
  - Платежи за обнаружение сбоев экрана = 0.008 (USD/мин) × 60 (мин) (исходный поток) + 0.008 (USD/мин) × 60 (мин) (переформатированный поток) = 0.96 USD
  - Платежи за обнаружение идентификационных кодов (включая обнаружение QR-кода и штрих-кода) = 0.008 (USD/мин) × 60 (мин) (исходный поток) + 0.008 (USD/мин) × 60 (мин) (переформатированный поток) = 0.96 USD
  - Платежи за обнаружение проблем со звуком (включая обнаружение тишины и громкого голоса) = 0.008 (USD/мин) × 60 (мин) (исходный поток) + 0.008 (USD/мин) × 60 (мин) (переформатированный поток) = 0.96 USD
- Платежи за оценку: Платежи за оценку с опорным сравнением (включая PSNR и SSIM) = 0.016 (USD/мин) × 60 (мин) (переформатированный поток) = 0.96 USD

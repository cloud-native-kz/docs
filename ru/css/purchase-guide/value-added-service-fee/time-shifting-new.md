# Сдвиг во времени (новое)

Сдвиг во времени обеспечивается возможностью записи CSS. Это позволяет пользователям перематывать и воспроизводить более ранние части прямой трансляции. Обычно это используется для воспроизведения лучших моментов прямых трансляций спортивных событий.

## Примечания

- Сдвиг во времени оплачивается на основе трафика прямых трансляций, для которых включен сдвиг во времени. Использование этой функции также приведет к [затратам на трафик/пропускную способность воспроизведения](https://intl.cloud.tencent.com/document/product/267/2819).
- Сдвиг во времени отключен по умолчанию. Вы можете включить его в консоли или используя API TencentCloud.

## Тарифы

| Дни сдвига во времени | Цена (USD/GB/день) |
| --- | --- |
| 1 день | 0.035 |
| 3 дня | 0.052 |
| 7 дней | 0.069 |
| 15 дней | 0.12 |
| 30 дней | 0.21 |

### Детали расчета

- Платежный элемент: трафик прямой трансляции
- Способ оплаты: оплата по факту
- Цикл выставления счетов: ежедневно. Комиссия за сдвиг во времени, сгенерированная в каждый день, будет снята с вашего счета в следующий день (фактическое время снятия средств и создания счета могут отличаться).

### Формула расчета

Комиссия за сдвиг во времени = Трафик прямой трансляции в день (GB) x Цена (USD/GB/день)

### Примеры расчета

Предположим, пользователь создал шаблон сдвига во времени в консоли CSS, привязал его к домену отправки и начал прямую трансляцию в 12:00 5 января. Прямая трансляция длилась **пять** часов. Битрейт видео составлял 1 Мбит/с. Максимальное количество дней, доступных для сдвига во времени, составляет семь дней.

Трафик прямой трансляции = 1 Мбит/с x 18000 секунд ÷ 8 = 2.25 GB
Комиссия за сдвиг во времени, возникшая 5 января = 2.25 GB x 0.069 USD/GB = 0.15525 USD. Комиссия будет снята 6 января.

Сдвиг во времени прямой трансляции разрешен с 12:00 5 января по 17:00 12 января.

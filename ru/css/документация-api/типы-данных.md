## BandwidthInfo

Информация о пропускной способности

Используется действиями: DescribeDeliverBandwidthList.

| Имя | Тип | Описание |
| --- | --- | --- |
| Time | String | Формат возвращаемого значения: yyyy-mm-dd HH:MM:SS Точность времени соответствует детализации запроса. |
| Bandwidth | Float | Пропускная способность. |

## BatchDomainOperateErrors

Информация об ошибках для доменов, операции с которыми не удалась выполнить API пакетной обработки доменов.

Используется действиями: ModifyLiveDomainCertBindings.

| Имя | Тип | Описание |
| --- | --- | --- |
| DomainName | String | Домен, операция с которым не удалась API. |
| Code | String | Код ошибки API 3.0. |
| Message | String | Сообщение об ошибке API 3.0. |

## BillDataInfo

Информация о пропускной способности и трафике.

Используется действиями: DescribeBillBandwidthAndFluxList.

| Имя | Тип | Описание |
| --- | --- | --- |
| Time | String | Момент времени в формате `yyyy-mm-dd HH:MM:SS`. |
| Bandwidth | Float | Пропускная способность в Мбит/с. |
| Flux | Float | Трафик в МБ. |
| PeakTime | String | Момент времени пиковой нагрузки в формате `yyyy-mm-dd HH:MM:SS`. Так как исходные данные имеют детализацию 5 минут, при запросе данных с детализацией 1 час или 1 день будет возвращен момент времени пиковой пропускной способности на соответствующем уровне детализации. |

(2) Этот параметр также требуется для смешивания потока, содержащего только аудио. Обратите внимание, что два входа не могут иметь одинаковое значение `ImageLayer`. |
| InputType | Integer | Нет | Тип входа. Диапазон значений: [0,5]. Если этот параметр не задан, по умолчанию используется 0. 0: входной поток содержит аудио и видео. 2: входной поток содержит изображение. 3: входной поток содержит холст.  4: входной поток содержит аудио. 5: входной поток содержит чистое видео. |
| ImageHeight | Float | Нет | Выходная высота входного видеоизображения. Диапазон значений: Пиксель: [0,2000] Процент: [0.01,0.99] Если этот параметр не задан, по умолчанию будет использоваться высота входного потока. Если используется процент, ожидаемый результат — (процент * высота фона). |
| ImageWidth | Float | Нет | Выходная ширина входного видеоизображения. Диапазон значений: Пиксель: [0,2000] Процент: [0.01,0.99] Если этот параметр не задан, по умолчанию будет использоваться ширина входного потока. Если используется процент, ожидаемый результат — (процент * ширина фона). |
| LocationX | Float | Нет | Смещение входа по оси X в выходном видеоизображении. Диапазон значений: Пиксель: [0,2000] Процент: [0.01,0.99] Если этот параметр не задан, по умолчанию используется 0. Горизонтальное смещение от верхнего левого угла фонового видеоизображения основного хоста.  Если используется процент, ожидаемый результат — (процент * ширина фона). |
| LocationY | Float | Нет | Смещение входа по оси Y в выходном видеоизображении. Диапазон значений: Пиксель: [0,2000] Процент: [0.01,0.99] Если этот параметр не задан, по умолчанию используется 0. Вертикальное смещение от верхнего левого угла фонового видеоизображения основного хоста.  Если используется процент, ожидаемый результат — (процент * ширина фона) |
| Color | String | Нет | Когда `InputType` имеет значение 3 (холст), это значение указывает цвет холста. Обычно используемые цвета включают: Красный: 0xcc0033. Желтый: 0xcc9900. Зеленый: 0xcccc33. Синий: 0x99CCFF. Черный: 0x000000. Белый: 0xFFFFFF. Серый: 0x999999 |
| WatermarkId | Integer | Нет | Когда `InputType` имеет значение 2 (изображение), это значение — ID водяного знака. |

# TXLivePusher

Pusher для Tencent Cloud CSS, в основном используется для веб-трансляций в браузере. Он захватывает изображение и звук пользователя через браузер и передает видео и аудиопотоки на сервер Tencent Cloud через WebRTC. Если вам нужно включить функцию локального смешивания, вызовите метод TXVideoEffectManager [enableMixing()](https://www.tencentcloud.com/document/product/267/58159#04f540f5-f365-41b7-b950-7bbf65eda92c) для включения.

Сначала создайте объект экземпляра для всех последующих операций.

```
const livePusher = new TXLivePusher();
```

## checkSupport

Статическая функция для проверки совместимости браузера.

```
static checkSupport(): Promise<TXSupportResult>;
```

Возвращаемое значение:

Возвращает объект Promise, где структура данных результата указана как [TXSupportResult](https://www.tencentcloud.com/document/product/267/58159#726c01ad-7629-49ad-ab89-ad6ebe6fc0e4).

## setRenderView

Установка контейнера предпросмотра локального видеоэкрана, предоставление узла div, и локально захватываемое видео будет отрендерено в контейнере. Если включена функция локального смешивания, контейнер будет отрендеровать смешанное аудио и видео после обработки.

```
setRenderView(container: string | HTMLDivElement): void;
```

Параметр:

| Поле | Тип | Описание |
| --- | --- | --- |
| container | string \| HTMLDivElement | ID контейнера или узел DOM. |

## setVideoQuality

Установка качества видео при трансляции. SDK имеет встроенные шаблоны качества видео, позволяя установить качество видео при трансляции непосредственно через предопределенные шаблоны.

```
setVideoQuality(quality: string): void;
```

Параметр:

| Поле | Тип | Описание |
| --- | --- | --- |
| quality | string | Названия предопределенных шаблонов качества видео. |

Встроенные шаблоны качества видео следующие:

| Название шаблона | Разрешение (ширина x высота) | Частота кадров (fps) | Битрейт (kbps) |
| --- | --- | --- | --- |
| 120p | 160 x 120 | 15 | 200 |
| 180p | 320 x 180 | 15 | 350 |
| 240p | 320 x 240 | 15 | 400 |
| 360p | 640 x 360 | 15 | 800 |
| 480p | 640 x 480 | 15 | 900 |
| 720p | 1280 x 720 | 15 | 1500 |
| 1080p | 1920 x 1080 | 15 | 2000 |
| 2K | 2560 x 1440 | 30 | 4860 |
| 4K | 3840 x 2160 | 30 | 9000 |

> **Примечание:** Из-за ограничений устройства и браузера видеоразрешение может быть неточно сопоставлено. В этом случае браузер автоматически отрегулирует разрешение, приближая его к соответствующему разрешению. Если параметры качества видео (разрешение, частота кадров и битрейт) не соответствуют вашим требованиям, вы можете установить пользовательские значения отдельно, используя [setProperty()](https://www.tencentcloud.com/document/product/267/58159#cfba56db-c7ee-46b5-8251-3756c5068265). Видеоразрешение здесь в основном относится к разрешению локально захватываемого видео. Разрешение во время трансляции может быть ниже разрешения захвата, и браузер автоматически отрегулирует разрешение трансляции на основе полосы пропускания сети и других факторов. По умолчанию это 720p, т.е. `setVideoQuality('720p')`.

## setAudioQuality

Установка качества аудио при трансляции. SDK имеет встроенный шаблон качества аудио. Качество аудио при трансляции можно установить напрямую, используя предопределенные шаблоны.

```
setAudioQuality(quality: string): void;
```

Параметры описаны следующим образом:

| Поле | Тип | Описание |
| --- | --- | --- |
| quality | string | Предопределенное название шаблона качества аудио. |

Встроенные шаблоны качества аудио следующие:

| Название шаблона | Частота дискретизации | Битрейт (Kbps) |
| --- | --- | --- |
| standard | 48000 | 40 |
| high | 48000 | 128 |

> **Примечание:** Если параметры качества аудио (частота дискретизации и битрейт) не соответствуют вашим требованиям, вы можете отдельно установить пользовательские значения через [setProperty()](https://www.tencentcloud.com/document/product/267/58159#cfba56db-c7ee-46b5-8251-3756c5068265). По умолчанию используется 'standard', то есть `setAudioQuality('standard')`.

## setProperty

В основном используется для вызова расширенных функций, таких как установка разрешения, частоты кадров и битрейта видео, а также установка частоты дискретизации и битрейта аудио и других.

```
setProperty(key: string, value: any): void;
```

Параметры описаны следующим образом:

| Поле | Тип | Описание |
| --- | --- | --- |
| key | string | Ключ, соответствующий расширенному API. |
| value | * | Параметры, необходимые при вызове расширенного API. |

В настоящее время поддерживаются следующие расширенные функции:

| Ключ | Значение | Описание | Пример кода |
| --- | --- | --- | --- |
| setVideoResolution | { width: number; height:number; } | Установка разрешения видео | setProperty('setVideoResolution', { width: 1920, height: 1080 }) |
| setVideoFPS | number | Конфигурация частоты кадров видео | setProperty('setVideoFPS', 25) |
| setVideoBitrate | number | Установка битрейта видео | setProperty('setVideoBitrate', 2000) |
| setAudioSampleRate | number | Конфигурация частоты дискретизации аудио | setProperty('setAudioSampleRate', 44100) |
| setAudioBitrate | number | Установка битрейта аудио | setProperty('setAudioBitrate', 200) |
| setConnectRetryCount | number | Установка количества попыток переподключения, по умолчанию: 3; диапазон: 0 - 10. Когда SDK неожиданно отключается от сервера, он попытается переподключиться. | setProperty('setConnectRetryCount', 5) |
| setConnectRetryDelay | number | Установка задержки повторного подключения, значение по умолчанию: 1, в секундах; в диапазоне: 0 - 10. Когда SDK неожиданно отключается от сервера, он попытается переподключиться. | setProperty('setConnectRetryDelay', 2) |
| enableAudioAEC | boolean | Включение подавления эха | setProperty('enableAudioAEC', true) |
| enableAudioAGC | boolean | Включение автоматического усиления | setProperty('enableAudioAGC', true) |
| enableAudioANS | boolean | Включение подавления шума | setProperty('enableAudioANS', true) |
| enableLog | boolean | Должны ли логи печататься в консоли | setProperty('enableLog', true) |

> **Примечание:** Подавление эха, автоматическое усиление и подавление шума включены по умолчанию, их эффективность в конечном итоге зависит от устройства и браузера. Рекомендуется либо включать, либо отключать эти три функции вместе. Установите настройки перед захватом и трансляцией потока.

## startCamera

Инициирование устройства камеры. Требуется авторизация пользователя, чтобы позволить браузеру доступ к камере. Если авторизация или доступ к устройству не удается, возвращенный объект Promise выбросит ошибку.

```
startCamera(deviceId?: string): Promise<string>;
```

Параметры описаны следующим образом:

| Поле | Тип | Описание |
| --- | --- | --- |
| deviceId | string | ID устройства камеры, опциональный параметр, определяет устройство камеры, которое нужно открыть. ID устройства может быть получен с помощью метода [getDevicesList](https://www.tencentcloud.com/document/product/267/58159#d0e3c8fa-6dfe-4bca-b593-77df628ea679) в `TXDeviceManager`. На мобильных устройствах передние и задние камеры можно указать, введя 'user' и 'environment'. |

Возвращаемое значение:

Возвращает объект Promise. При успехе возвращается ID потока, служащий уникальным идентификатором в SDK. В случае ошибки выбрасывается соответствующее сообщение об ошибке.

> **Примечание:** Этот интерфейс не поддерживает использование по протоколу HTTP; пожалуйста, разверните ваш веб-сайт с использованием протокола HTTPS. При возникновении сбоя при открытии камеры обратитесь к возвращенному сообщению об ошибке, при необходимости проконсультировавшись с [исключениями getUserMedia](https://developer.mozilla.org/en-US/docs/Web/API/MediaDevices/getUserMedia#exceptions).

## stopCamera

Операция отключения устройства камеры.

```
stopCamera(streamId?: string): void;
```

Параметры описаны следующим образом:

| Поле | Тип | Описание |
| --- | --- | --- |
| streamId | string | ID потока, опциональный параметр, обозначает конкретный поток камеры, который нужно деактивировать. После включения локального смешивания потока, если были собраны несколько потоков камеры, вы можете отключить указанный поток камеры через ID потока; в противном случае будут закрыты все потоки камеры. |

| --- | --- | --- |
| audio | boolean | Определяет, нужно ли захватывать системный звук или звук страницы: true - захват звука, false - без захвата звука, по умолчанию установлено false. |

> **Примечание:** Этот интерфейс не поддерживает использование по протоколу HTTP; пожалуйста, разверните ваш веб-сайт с использованием протокола HTTPS.

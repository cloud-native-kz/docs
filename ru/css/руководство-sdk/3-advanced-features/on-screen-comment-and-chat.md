# Комментарии и чат на экране

## Обзор

В бизнесе прямых трансляций хостам и зрителям обычно необходимо взаимодействовать в реальном времени различными способами, такими как комментарии на экране и чат. Однако интеграция таких функций сложна. В этом документе используется IM в качестве примера для описания того, как реализовать требования, такие как комментарии на экране, чат и рекомендации товаров во время прямой трансляции, а также возможные проблемы и соображения, дающие представление о бизнесе прямых трансляций и требованиях.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/1fadd85de10e11ee9ca3525400bb593a.jpeg)

## Описание ключевых функций

| Функция | Описание |
| --- | --- |
| Комментарии на экране, подарки и лайки в прямом эфире | Может поддерживать сотни миллионов одновременных сообщений для создания дружелюбного опыта взаимодействия. |
| Различные режимы чата, такие как один-на-один и групповой чат в прямом эфире | Пользователи могут отправлять и получать сообщения от других членов в той же комнате чата. Разнообразные типы сообщений, такие как текст, изображения, аудио и короткие видео, могут отправляться в реальном времени, что стимулирует большую активность пользователей. |
| Отправка товаров при прямой покупке | В сценариях прямой покупки, когда хост рекомендует товар, его необходимо немедленно отобразить в ячейке товара внизу экрана и уведомить всех зрителей. Уведомления о новых товарах, как правило, инициируются виртуальным ассистентом. |
| Трансляция в комнате прямого эфира | Функция трансляции похожа на системное уведомление, отправляемое в комнаты прямого эфира. Когда системный администратор отправляет сообщение трансляции, все комнаты прямого эфира под `SDKAppID` его получат. |

## Метод интеграции

### Шаг 1. Создание приложения

Чтобы установить комнату прямого эфира в Tencent Cloud, вам необходимо создать приложение IM в [консоли](https://console.tencentcloud.com/im), как показано ниже:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/491cd9e3498711ef8357525400bdab9d.png)

### Шаг 2. Завершение соответствующей конфигурации

Приложение, созданное на шаге 1, является выпуском Free, который применяется только для разработки. В производственной среде вам необходимо активировать выпуск Pro или Ultimate по мере необходимости. Дополнительную информацию о различиях между различными выпусками см. в разделе [Ценообразование](https://www.tencentcloud.com/document/product/1047/34350).
В сценариях прямых трансляций требуется дополнительная конфигурация после создания приложения.

- **Расчет `UserSig` с использованием ключа**

В системе учетных записей IM пароль, требуемый для входа пользователя, рассчитывается сервером с использованием ключа, предоставленного IM. Дополнительную информацию см. в разделе [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). На этапе разработки, чтобы избежать задержки разработки на клиенте, вы также можете рассчитать `UserSig` в [консоли](https://console.tencentcloud.com/im/tool-usersig), как показано ниже:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/81e77873498711ef8357525400bdab9d.png)

- **Конфигурация учетной записи администратора**

Во время прямой трансляции администратор может понадобиться отправлять сообщения в комнату прямого эфира и отключать (удалять) несоответствующих пользователей, что можно сделать через API на сервере IM, как описано в [Список REST API](https://www.tencentcloud.com/document/product/1047/34621). Для вызова этих API вам необходимо [создать учетную запись администратора IM](https://console.tencentcloud.com/im/account-management). По умолчанию IM предоставляет учетную запись с `UserID`, равным `administrator`. Вы также можете создать несколько учетных записей администратора по мере необходимости. Обратите внимание, что вы можете создать максимум пять учетных записей администратора.

- **Конфигурация адреса обратного вызова и включение обратного вызова**

Для реализации розыгрышей на основе комментариев на экране, сбора статистики сообщений, обнаружения чувствительного контента и других требований вам необходимо использовать модуль обратного вызова IM, где серверная часть IM выполняет обратный вызов деловой серверной части в определенных сценариях. Вам нужно только предоставить HTTP API и настроить его в модуле [Конфигурация Webhook](https://console.tencentcloud.com/im/callback-setting) в консоли, как показано ниже:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/61d2693c498811ef818752540055f650.png)

### Шаг 3. Интеграция клиентского SDK

После завершения подготовительной работы вам необходимо интегрировать клиентские SDK IM и TRTC в ваш проект. Вы можете выбрать различные варианты интеграции по мере необходимости. Подробные указания см. в разделе [Начало работы](https://www.tencentcloud.com/document/product/1047/60367).
Ниже описаны распространенные функции в комнате прямого эфира и приведены лучшие практики с кодом реализации.

### Шаг 4. Разработка основных функций комнаты прямого эфира

1. **Выбор типа группы**

Раздел чата пользователя в сценариях прямых трансляций имеет следующие характеристики:

  - Пользователи часто присоединяются и покидают группу, информация о групповых разговорах (непрочитанное количество и `lastMessage`) не требует управления.
  - Пользователи могут присоединиться к группе без одобрения.
  - Пользователи отправляют сообщения, не беспокоясь об истории чата.
  - Обычно большое количество членов группы.
  - Информация членов группы не требует хранения.

Поэтому вы можете выбрать **AVChatRoom** в качестве **типа группы** для комнаты прямого эфира на основе характеристик группы IM, описанных в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529). Аудиовизуальные группы IM (`AVChatRoom`) имеют следующие характеристики:

  - **Поддерживают интерактивные сценарии прямых трансляций с неограниченным количеством членов группы.**
  - Сообщения (системные уведомления группы) могут быть отправлены всем онлайн-членам.
  - Пользователи могут входить в группу напрямую без одобрения администратора.

> **Примечание:** SDK IM для веб-сайта позволяет пользователям присоединяться только к одной аудиовизуальной группе (`AVChatRoom`) одновременно. Если пользователь входит в клиент и входит в комнату прямого эфира A, многоклиентский вход включен в [консоли](https://console.tencentcloud.com/im/login-message), и пользователь входит в другой клиент и входит в комнату прямого эфира B, пользователь будет удален из комнаты прямого эфира A.

2. **Конфигурация комментариев на экране, подарков и лайков для комнаты прямого эфира**
  - **Комментарии на экране**

Аудиовизуальные группы (`AVChatRoom`) поддерживают комментарии на экране, подарки и сообщения о лайках для создания дружелюбного опыта взаимодействия.

Чтобы создать комментарий на экране, вы можете использовать API IM для создания текстового или пользовательского сообщения. После успешной отправки сообщения вам необходимо получить его текст или пользовательские атрибуты в комнате прямого эфира, получив обратный вызов [OnRecvNewMessage()](https://im.sdk.qcloud.com/doc/zh-cn/classV2TIMAdvancedMsgListener.html#acbd1dba3526377d88f14c2183901511c), и затем отобразить его в нужном интерфейсе.

  - **Подарок**
    - Неперсистентные запросы подключения от клиента отправляются на деловой сервер, что включает логику выставления счетов.
    - После возникновения сборов отправитель может видеть, что XXX отправил подарок XXX (чтобы отправитель мог видеть подарок, отправленный им самим; когда сообщений много, может быть инициирована политика отбрасывания сообщений).
    - После урегулирования платежей вы можете вызвать API сервера для отправки пользовательского сообщения (подарка).
    - Если подарки отправляются несколько раз подряд, вам нужно объединить сообщения.
      - Если количество подарков выбирается заранее, например 99 подарков, вы можете отправить сообщение с включенным `99` в параметр.
      - Если подарки отправляются несколько раз и общее количество неопределенно, вы можете отправить сообщение за каждые 20 подарков (значение можно настроить) или клики в течение секунды. Например, если 99 подарков кликается подряд, после оптимизации нужно отправить только пять сообщений.
  - **Лайк**
    - В отличие от сообщения о подарке, сообщение о лайке не выставляется в счет и отправляется непосредственно на клиенте.
    - Для сообщений о лайках, которые требуют подсчета на сервере, после применения регулирования трафика на клиенте, лайки на клиенте подсчитываются, и сообщения о лайках в течение короткого периода времени объединяются в одно. Деловой сервер получает количество лайков в обратном вызове перед отправкой сообщения.
    - Для сообщений о лайках, которые не требуют подсчета, используется логика шага 2, где деловой сервер отправляет сообщение после применения регулирования трафика на клиенте и не требует получения количества в обратном вызове перед отправкой сообщения.

3. **Отправка товаров при прямой покупке**

Когда хост рекомендует товар, его необходимо немедленно отобразить в ячейке товара внизу экрана и уведомить всех зрителей. Уведомления о новых товарах, как правило, инициируются виртуальным ассистентом. Мы рекомендуем вам реализовать уведомления о новых товарах, разрешив администратору изменять пользовательское поле группы следующим образом:

  3.1. **Добавление пользовательского поля группы**

    3.1.1. Войдите в [консоль IM](https://console.tencentcloud.com/im), щелкните карточку целевого приложения IM и выберите **Конфигурация** > **Конфигурация группы** > **Пользовательское поле группы** на левой боковой панели.

    3.1.2. На странице **Пользовательское поле группы** щелкните **Добавить** в верхнем правом углу.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/680babe6498911ef998b525400f69702.png)

    3.1.3. В появившемся диалоговом окне введите имя поля и установите типы групп и разрешения на чтение/запись.

> **Примечание:** Имя поля может содержать до 16 символов, поддерживая буквы, цифры и подчеркивания (_). Оно не может начинаться с цифры. Пользовательское поле группы и пользовательское поле члена группы не могут иметь одно и то же имя.![](https://staticintl.cloudcachetci.com/cms/backend-cms/c6db5e93498911ef998b525400f69702.png)

    3.1.4. Выберите **Я понимаю, что после добавления пользовательского поля группы можно изменять только разрешения на чтение-запись добавленного типа группы; тип группы не может быть переизбран или удален; поле не может быть удалено.** и щелкните **Подтвердить**.

> **Примечание:** Пользовательское поле группы вступит в силу примерно через десять минут после конфигурации.

  3.2. **Использование пользовательского поля группы**

Виртуальный ассистент вызывает [REST API для изменения профиля группы](https://www.tencentcloud.com/document/product/1047/34962) в качестве администратора группы в соответствующее время для обновления пользовательского поля группы, чтобы отправлять уведомления о новых товарах и уведомления об изменении статуса прямой трансляции в комнате прямого эфира.

4. **Трансляция в комнате прямого эфира**

Функция трансляции похожа на функцию системного уведомления в комнате прямого эфира, но отличается от нее тем, что относится к обмену сообщениями. Когда системный администратор отправляет сообщение трансляции, все комнаты прямого эфира под `SDKAppID` его получат.
Функция трансляции в настоящее время доступна только для выпуска Ultimate и должна быть включена в консоли. Дополнительную информацию о том, как отправить сообщение трансляции на деловой сервер, см. в разделе [Сообщение трансляции аудиовизуальной группы](https://www.tencentcloud.com/document/product/1047/49440).

> **Примечание:** Если вы не являетесь пользователем выпуска Ultimate, вы можете реализовать функцию, отправив пользовательское групповое сообщение на сервере.

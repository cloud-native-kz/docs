# Введение

## Обзор

Сервис обработки медиа (MPS) предоставляет интеллектуальные и мощные возможности обработки мультимедийных данных, поддерживая наиболее полный набор стандартов кодирования аудио/видео в отрасли. Благодаря самостоятельно разработанному ядру кодирования и моделям компьютерного зрения, MPS предоставляет функции трансграции аудио/видео, улучшения качества, медиа AI и проверки качества. Это помогает повысить качество медиа, снизить затраты и удовлетворить разнообразные потребности в обработке аудио/видео. Все API-интерфейсы MPS, описанные в этой главе, соответствуют спецификации OpenAPI 3.0.
Вы можете вызывать API-интерфейсы для выполнения операций, связанных с обработкой медиа, таких как инициирование задачи обработки медиа, создание шаблона или запрос сведений о задаче.
Сведения обо всех API-интерфейсах, поддерживаемых MPS, см. в разделе [Категории API](https://intl.cloud.tencent.com/document/product/1041/33625).

## Глоссарий

Общие термины API MPS см. в таблице ниже.

| Термин | Описание |
| --- | --- |
| Инициирование задачи обработки | Это означает инициирование задачи обработки медиа для автономных файлов или прямых трансляций. |
| Уведомление о событии | Вы можете настроить адрес уведомления о событии при инициировании задачи обработки, чтобы позволить MPS синхронизировать прогресс задачи, статус и информацию о выходных данных в реальном времени во время обработки. |
| Шаблон | Шаблоны используются для настройки определенных параметров функций для последующего использования, таких как трансграция аудио/видео, улучшение качества аудио/видео, водяной знак, снимок экрана, медиа AI и проверка качества медиа. При инициировании задачи обработки вы можете связать задачу с ID шаблона. |
| Оркестрировка сервиса | Оркестрировка используется для объединения различных функций в автоматический рабочий процесс обработки. Кроме того, она поддерживает автоматическое запуск. Когда новый файл медиа загружается в указанную директорию триггера COS, он будет автоматически обработан в соответствии с конфигурациями оркестрировки. |

## Начало работы с API-интерфейсами

Вы можете использовать инструмент API Explorer для вызова API-интерфейсов в режиме онлайн.
Данный документ использует инициирование задачи обработки в качестве примера для описания шагов вызова API-интерфейса через инструмент API Explorer.

Перейдите на страницу инструмента
API Explorer
. Дополнительную информацию об использовании инструмента API Explorer см. в разделе
Использование API Explorer
.
Создайте шаблон интеллектуальных субтитров через консоль (рекомендуется) или путем вызова API
CreateSmartSubtitleTemplate
и получите ID шаблона.
Вызовите API
Process Media
, и укажите путь входного файла, ID шаблона интеллектуальных субтитров, путь выходных данных и другую необходимую информацию. Затем инициируйте задачу обработки интеллектуальных субтитров и получите ID задачи.
Запросите результат задачи через консоль или путем вызова API
DescribeTaskDetail
.

# Создание задач

## Обзор

Media Processing Service (MPS) поддерживает три способа инициирования задач:

- **Быстрое создание задач в консоли**: Задачи обработки инициируются вручную путем выбора файлов или ввода потоков в реальном времени в консоли.
- **Автоматическое триггирование задач**: Задачи обработки инициируются автоматически после загрузки файлов в ведра COS/AWS S3 без необходимости ручного создания задач в консоли. В настоящее время поддерживаются только файлы Video on Demand (VOD).
- **Инициирование задач через API**: Задачи инициируются путем вызова API, подходит для сценариев пакетной обработки.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ecf763e2230911efa0b8525400db4520.png)

## Подробно

### Быстрое создание задач в консоли

- Для обработки аудио- и видеофайлов, хранящихся в COS и AWS S3, вы можете нажать **Create VOD Processing Task**. Если ваши видеофайлы хранятся в других продуктах облачного хранилища, вы также можете предоставить URL-адрес загрузки файла в качестве источника ввода, но выходные данные должны использовать продукт Tencent COS.
- Для обработки потоков в реальном времени вы можете нажать **Create Live Stream Processing Task**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/0fe8239b230a11ef90c35254006d3582.png)

#### Указание входных файлов

- Выберите аудио- и видеофайлы/потоки, которые необходимо обработать в качестве источника ввода, и настройте путь сохранения обработанных выходных файлов.
- Для обработки медиа VOD вы можете выбирать аудио- и видеофайлы из ведер COS или AWS S3 или предоставить адрес URL-загрузки файла.

> **Примечание:** Если вы выбираете ввод из COS или URL, вам необходимо выполнить шаги [Prerequisites > COS Authorization](https://www.tencentcloud.com/document/product/1041/33482#.E6.AD.A5.E9.AA.A42.EF.BC.9A.E6.8E.88.E6.9D.83.E7.AE.A1.E7.90.86) для создания роли сервиса, позволяющей MPS выполнять операции чтения и записи, такие как загрузка, транскодирование и загрузка файлов в ваше ведро COS. Если вы выбираете ввод из AWS S3, вам не требуется выполнять авторизацию COS, но вам необходимо обратиться к документу [Using Amazon S3 Buckets with MPS](https://www.tencentcloud.com/document/product/1041/54516) для создания подучетной записи AWS, входных и выходных ведер S3, SQS и т. д.

- Для обработки медиа потока в реальном времени вам необходимо ввести адрес URL потока в реальном времени (это должен быть адрес в реальном времени, поддерживающий RTMP, HLS, FLV и т. д.).

> **Примечание:** Вам необходимо убедиться, что адрес потока в реальном времени введен правильно. Если не удается извлечь поток в реальном времени, будет повторена попытка 3 раза. Если по-прежнему не удается получить поток в реальном времени, задача будет отмечена как не выполненная.

#### Обработка входных файлов

##### Создание оркестраций

- Оркестрация — это комбинация различных функций MPS, служащая автоматизированным рабочим процессом обработки. Нажав на узел ![](https://staticintl.cloudcachetci.com/cms/backend-cms/fc36ffa1190011ef8bfe5254002fd0a8.png), вы можете добавить необходимые функции.
- Необходимо добавить по крайней мере один узел функции. Вы также можете, в соответствии с вашими потребностями, расположить богатые функции MPS в последовательном или параллельном порядке. Например, если вам необходимо выполнить операции улучшения, такие как улучшение качества изображения и удаление артефактов, на входном файле, а затем снизить битрейт через транскодирование и сжатие, вы можете сначала добавить узел улучшения аудио/видео, а затем узел транскодирования аудио/видео.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e6cc81ea230a11ef95b8525400e64ebc.png)

- Если вам необходимо одновременно выполнить транскодирование и интеллектуальный анализ источника ввода, вы можете расположить рабочий процесс параллельной задачи, как показано ниже.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/5552f8d2230b11ef860b52540049c929.png)

- Каждый раз, когда вы добавляете узел функции, вам необходимо установить конкретные параметры этой функции в слое плавающего ящика.
- На примере узла транскодирования аудио/видео вы можете выбрать системные предустановленные параметры или ранее сохраненные пользовательские параметры, нажав **Select template**. Вы также можете переключиться на **Custom** для прямого редактирования параметров. Для подробной конфигурации параметров см. [Audio/Video Transcoding Template](https://www.tencentcloud.com/document/product/1041/48784).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/fa8b2578231711ef91395254000a29ac.png)

> **Примечание:** В настоящее время только узлы транскодирования и улучшения аудио/видео предлагают возможность переключиться на **Custom** для прямого редактирования параметров без необходимости сохранения их в качестве шаблонов заранее. Если вам необходимо настроить параметры для других узлов функций, таких как интеллектуальный анализ, идентификация, снимок экрана и т. д., вы можете нажать **Create Custom Template** для входа на страницу **Templates** для создания шаблона. ![](https://staticintl.cloudcachetci.com/cms/backend-cms/180487d2231811ef90c35254006d3582.png)

##### Сохранение оркестраций

- Вы можете сохранить настроенные оркестрации для последующего повторного использования. Сохраненные оркестрации можно просмотреть в **Orchestrations**.
- При сохранении вам необходимо настроить ведро триггера, директорию триггера и другие входные и выходные пути для облегчения последующего автоматического триггирования задач. Подробнее см. в разделе [Automatically Triggering Tasks](#89c16fb9-90fe-43d8-92c9-e2fdcc20d929) ниже в этом документе.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/2a2dac07231811ef95b8525400e64ebc.png)

##### Выбор существующих оркестраций

То есть использование ранее сохраненного потока оркестрации и параметров узла функции.

##### Уведомление о событиях

Уведомление о событиях может синхронизировать прогресс и статус ваших задач в реальном времени во время потока обработки задач.

- Если Tencent Cloud COS/URL используется в качестве источника ввода, поддерживаются три механизма уведомления о событиях: обратный вызов TDMQ-CMQ, обратный вызов HTTP и обратный вызов SCF. Для подробного описания см. [Configure Event Notification](https://www.tencentcloud.com/document/product/1041/58242#.E4.BA.8B.E4.BB.B6.E9.80.9A.E7.9F.A5.E9.85.8D.E7.BD.AE).
- Если AWS S3 используется в качестве источника ввода, поддерживаются два механизма уведомления о событиях: обратный вызов Amazon SQS и обратный вызов HTTP. Для подробного описания см. [Creating an SQS queue for transcoding callbacks](https://www.tencentcloud.com/document/product/1041/54516#4707e8a4-0e97-4922-a5aa-cb79022f8faf).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/49e6508b231811efbef6525400a8a0fb.png)

#### Указание пути вывода

Укажите путь сохранения по умолчанию для выходного файла этой задачи.

### ![](https://staticintl.cloudcachetci.com/cms/backend-cms/933acd29231811ef90c35254006d3582.png)

Если вам необходимо установить путь вывода конкретного узла функции в оркестрации отдельно, например, вы добавили функции транскодирования, улучшения и снимка экрана в оркестрацию, и вы ожидаете, что выходной файл снимка экрана будет сохранен в другом пути, вы можете нажать узел снимка экрана на этапе [Create Orchestrations](#bb57e8b3-7f91-4633-8677-b0c073b6a6e3), упомянутом выше, и настроить его в **More settings - Customize Screenshot Output Path**. Вы также можете отрегулировать соглашение об именовании выходных файлов различных функций через эту настройку. Подробнее см. в разделе [Filename Variable](https://www.tencentcloud.com/document/product/1041/33495).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b30fbb48231811ef860b52540049c929.png)

> **Примечание:** Порядок приоритета путей вывода: **custom output path for each feature node in the orchestration** > **output path configured in the task** > **output path configured in the orchestration**.
> Например: При создании задачи была выбрана существующая оркестрация с путем вывода, установленным как `[ap-guangzhou]test/output1/` в оркестрации. В оркестрации были настроены узел транскодирования аудио/видео и узел улучшения аудио/видео, где узел транскодирования имел настроенный пользовательский путь вывода транскодирования как `[ap-guangzhou]test/output2/`. Указанный путь сохранения выходного файла для этой задачи — `[ap-guangzhou]test/output3/`. Таким образом, в этой задаче выходной результат улучшения будет сохранен в `[ap-guangzhou]test/output3/`, а выходной результат транскодирования будет сохранен в `[ap-guangzhou]test/output2/`.

### Автоматическое триггирование задач

Задачи обработки инициируются автоматически после загрузки файлов в ведра COS/AWS S3 без необходимости ручного создания задач в консоли. В настоящее время поддерживаются только файлы VOD.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/01d0a24a231b11efa62352540003d080.png)

1. Нажмите **Go to Configure** для входа на страницу [Orchestrations > VOD Orchestration](https://console.tencentcloud.com/mps/workflows/vod), а затем нажмите **Create VOD Orchestration**.
2. Настройте ведро триггера и директорию, выходное ведро и директорию, конкретный рабочий процесс задачи и т. д. Для подробных инструкций по конфигурации см. [VOD Orchestration Configuration Instructions](https://www.tencentcloud.com/document/product/1041/58242).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3b9a02ee231b11efbef6525400a8a0fb.png)

3. По умолчанию автотриггирование отключено для оркестрации. Чтобы включить его, вернитесь на страницу **Orchestrations > VOD Orchestration** и нажмите ![](https://staticintl.cloudcachetci.com/cms/backend-cms/fc5e5bca190011ef88ad5254002977b6.png) для включения функции автотриггирования.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/984b31e6231b11efbef6525400a8a0fb.png)

4. Загрузив видеофайл, который необходимо обработать, в ведро триггера, настроенное в оркестрации, вновь загруженное видео будет обработано автоматически в соответствии с конфигурацией задачи в оркестрации без необходимости ручного создания задач в консоли.

На странице **Orchestrations > COS Bucket** вы можете найти **trigger bucket** и **output bucket**, настроенные в оркестрации, где вы можете удобно выполнять операции с файлами, такие как загрузка, предпросмотр и загрузка файлов:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/eeac61f0231b11efbef6525400a8a0fb.png)

> **Примечание:** После включения автотриггирования для оркестрации он будет влиять только на новые видеофайлы, загруженные в ведро триггера; файлы, ранее хранящиеся в ведре триггера, не будут обработаны автоматически.

### Инициирование задач через API

#### Обработка медиа VOD

Инициируйте задачи через API [ProcessMedia](https://www.tencentcloud.com/document/product/1041/33640). Следующие новые функции еще не были запущены в консоли, но могут быть опробованы через API:

- Проверка качества медиа: Поддерживает диагностику формата видеофайла, обнаружение содержимого аудио и видео (дрожание, размытие, слабое освещение, переэкспозиция, черные и белые границы, черные и белые экраны, разрывы экрана, шум, мозаика, QR-код и т. д.) и оценку без эталона.

#### Обработка медиа потока в реальном времени

Инициируйте единую задачу обработки потока в реальном времени через API [ProcessLiveStream](https://www.tencentcloud.com/document/product/1041/33641). Он поддерживает следующие функции:

- Интеллектуальный аудит: Поддерживает обнаружение порнографии изображений, обнаружение конфиденциальной информации и обнаружение порнографии в аудио.
- Интеллектуальная идентификация: Поддерживает идентификацию лиц, объектов, текста и речи. Автоматическое распознавание речи (ASR) также поддерживает интеллектуальный перевод и преобразование подписей в реальном времени. Он включает функции, такие как тегирование игр.
- Интеллектуальный анализ: Поддерживает разделение новостей в реальном времени и многое другое.
- Проверка качества: Поддерживает диагностику формата потока в реальном времени, обнаружение содержимого аудио и видео (дрожание, размытие, слабое освещение, переэкспозиция, черные и белые границы, черные и белые экраны, разрывы экрана, шум, мозаика, QR-код и т. д.) и оценку без эталона среди других функций.
- Запись потока в реальном времени.

# Живая оркестрация

## Сценарии использования

Если вы хотите обрабатывать прямые трансляции с помощью MPS, вам необходимо создать **живую оркестрацию**, настроить задачу, например [запись прямой трансляции](https://www.tencentcloud.com/document/product/1041/56732), и указать выходной bucket и каталог.

## Создание схемы

### Переход на страницу создания

1. Войдите в [консоль MPS](https://console.tencentcloud.com/mps) и нажмите [Оркестрации > Живая оркестрация > Создать живую оркестрацию](https://console.tencentcloud.com/mps/workflows/live/add).
2. На странице **Создать живую оркестрацию** вы можете создать процесс оркестрации и информацию о конфигурации, соответствующие вашему конкретному сценарию бизнеса.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e8c063aa933411ee8964525400321be2.png)

### Настройка основной информации

| Элемент конфигурации | Обязательный | Описание конфигурации |
| --- | --- | --- |
| Имя схемы | Да | Вы можете ввести комбинацию букв, цифр, подчеркиваний (_) и дефисов (-). Длина не может превышать 128 символов. Пример: "MPS". |
| Выходной bucket | Да | По умолчанию выходной bucket совпадает с триггерным bucket. Вы также можете выбрать другой bucket в том же регионе, что и триггерный bucket, соответствующий указанному `APPID`, в качестве выходного bucket. Вновь созданные видеофайлы будут сохранены в выбранном вами bucket после завершения выполнения схемы. |
| Выходной каталог | Нет | Каталог должен начинаться и заканчиваться прямой косой чертой (/). |
| Включить уведомления о событиях | — | Подробнее см. [Конфигурация уведомления о событиях](https://www.tencentcloud.com/document/product/1041/58242#). После включения процесс транскодирования и данные результатов будут отражены через выбранный метод уведомления. |
| Действия | — | См. раздел [Настройка действий](https://www.tencentcloud.com/document/product/1041/58243#.E4.BB.BB.E5.8A.A1.E6.B5.81.E7.A8.8B.E9.85.8D.E7.BD.AE) ниже. Вы можете быстро построить бизнес-процесс, определив узлы процесса и шаблоны оркестрации. |

### Настройка действий

В этой области вы можете определить процесс для всего сервиса, добавить различные узлы обслуживания (такие как [запись прямой трансляции](https://www.tencentcloud.com/document/product/1041/56732)) и настроить различные шаблоны для каждого узла. Подробные инструкции по конфигурации приведены ниже:

1. Нажмите кнопку **+**, чтобы выбрать необходимое действие из раскрывающегося списка и добавить его.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3fdc44552f0511f0948f52540099c741.png)

2. Нажмите кнопку редактирования справа от добавленного действия для подробной конфигурации. Заполните информацию на странице подробной конфигурации и нажмите Сохранить.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c1da177b2faa11f0a62e525400454e06.png)

3. Кроме того, для узла типа живой записи в **Дополнительные параметры** вы также можете **настроить путь вывода записи**. В настоящее время поддерживается сохранение файлов записи в Tencent Cloud Object Storage (COS) и AWS S3. Здесь настроенный путь вывода записи имеет более высокий приоритет, чем путь вывода задачи прямой трансляции. Если параметр **Настроить путь вывода записи** не включен, путь вывода файлов записи по умолчанию будет совпадать с путем вывода задачи прямой трансляции.
  - Tencent Cloud COS

| Элемент конфигурации | Обзор |
| --- | --- |
| Выходной Bucket | Выберите регион. Выберите Bucket. |
| Путь вывода | Если вам нужно настроить путь вывода или имя файла, заполните конкретный путь вывода и имя файла вывода. Формат должен быть: `/custom path/filename_{variable name}.{format}`. Путь вывода по умолчанию: `/{taskId}/{rand}/{streamId}_record_{definition}.{format}`. {rand} здесь является переменной случайного числа. Если вы удалите {rand}, при одинаковых значениях других переменных параметров это может привести к перекрытию нескольких файлов результатов записи. Осторожно изменяйте или удаляйте {rand}. Если вам не требуется определять путь вывода файла и имя файла, вы также можете установить значение в этом поле ввода на пусто. Файлы записи будут сохранены в корневом каталоге выходного Bucket. |

  - AWS S3

| Элемент конфигурации | Обзор |
| --- | --- |
| Информация подучетной записи | Информация подучетной записи AWS: secretID, secretKey. |
| Выходной S3 Bucket | Регион S3: Заполните регион S3. Обратитесь к спецификации на веб-сайте AWS для формата заполнения. Например: ap-southeast-1. Имя S3: Заполните имя S3, которым вы управляете в AWS. |
| Путь выходного файла | Если вы выбираете тип пути вывода как AWS S3, этот путь выходного файла является обязательной информацией. Заполните конкретный путь вывода и имя файла вывода. Формат должен быть: `/custom path/filename_{variable name}.{format}`. Путь вывода по умолчанию: `/{taskId}/{rand}/{streamId}_record_{definition}.{format}`. {rand} здесь является переменной случайного числа. Если вы удалите {rand}, при одинаковых значениях других переменных параметров это может привести к перекрытию нескольких файлов результатов записи. Осторожно изменяйте или удаляйте {rand}. |

4. После настройки вышеупомянутой информации нажмите **Создать**, чтобы создать **оркестрацию**.

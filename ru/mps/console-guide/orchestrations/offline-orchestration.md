# Автономная оркестрация

## Сценарии использования

Функция обработки мультимедиа поддерживает оркестрацию, которая предоставляет пользователям интуитивно понятный визуальный интерфейс для бесперебойного связывания нескольких шаблонов обработки мультимедиа. Эти шаблоны могут выполняться последовательно или на основе определенных условий. Гибко комбинируя элементы оркестрации, пользователи могут достичь разнообразных эффектов обработки мультимедиа. После настройки **автономной оркестрации** видео, загруженные в указанный каталог хранилища, автоматически активируют рабочий процесс оркестрации, а выходные файлы записываются в указанный каталог хранилища. В процессе оркестрации можно настроить задачи, такие как преобразование кодека аудио и видео, улучшение аудио и видео, создание скриншотов, мультимедийный ИИ и проверка качества мультимедиа.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/42bf96d4b3a911f0b96752540044a08e.png)

## Создание автономной оркестрации

### Переход на страницу создания

1. Войдите в [консоль MPS](https://console.tencentcloud.com/mps) и нажмите **Orchestrations** > **Offline Orchestration** > [Create Offline Service Orchestration](https://console.tencentcloud.com/mps/workflows/vod/add).
2. На отобразившейся странице создайте оркестрацию в соответствии с требованиями деловых сценариев и настройте соответствующую информацию.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/15bc5c5871cc11f09e39525400454e06.png)

### Настройка оркестрации

| Элемент конфигурации | Обязательный | Описание конфигурации |
| --- | --- | --- |
| Тип триггера | - | По умолчанию выбирается "Tencent Cloud COS", что указывает на то, что хранилище, которое активирует оркестрацию, принадлежит Tencent COS. Если выбрано "AWS", хранилище триггера принадлежит AWS S3. Этот вариант можно выбрать, если хранилище настроено на услугах AWS. Для получения подробной информации см. раздел [Using Amazon S3 Buckets with MPS](https://www.tencentcloud.com/document/product/1041/54516). |
| Имя оркестрации | Да | Вы можете ввести комбинацию букв, цифр, подчеркиваний и дефисов (_). Длина не может превышать 128 символов. Пример: "MPS". |
| Хранилище триггера | Да | Вы можете выбрать хранилище, созданное под этим `APPID`, в качестве хранилища триггера. После включения оркестрации загрузка видеофайла в это хранилище автоматически активирует выполнение оркестрации. |
| Каталог триггера | Нет | Каталог должен заканчиваться прямой косой чертой (/). Если не указано, все каталоги хранилища триггера могут активировать выполнение оркестрации. |
| Форматы триггера | - | Для вновь загруженных файлов в каталог триггера задача оркестрации будет активирована только в том случае, если они соответствуют указанным расширениям файлов, что предотвращает непреднамеренные активации. По умолчанию выбран параметр **Preset**. Сведения о поддерживаемых форматах триггера при использовании предустановок см. в разделе [Форматы триггера предустановок](#formats) ниже. Вы также можете переключиться на **Custom** для определения собственного списка расширений. Если выбран параметр **No Extension Restrictions**, все вновь загруженные файлы в каталог триггера — независимо от их расширений или даже их отсутствия — будут активировать задачу оркестрации. |
| Выходное хранилище | Да | По умолчанию выходное хранилище совпадает с хранилищем триггера. Вы также можете выбрать другое хранилище в том же регионе, соответствующее указанному `APPID`, в качестве выходного хранилища. Вновь созданные видеофайлы будут сохранены в выбранном хранилище после завершения оркестрации услуг. |
| Выходной каталог | Нет | Каталог должен заканчиваться прямой косой чертой (/). Если не указано, каталог триггера будет принят в качестве выходного каталога. |
| Включить уведомления о событиях | - | См. раздел [Configure Event Notification](https://www.tencentcloud.com/document/product/1041/58242#.E4.BA.8B.E4.BB.B6.E9.80.9A.E7.9F.A5.E9.85.8D.E7.BD.AE) ниже для получения подробной информации. После включения данные процесса транскодирования и результаты будут отправлены выбранным методом уведомления. |
| Actions | - | См. раздел [Configure Actions](https://www.tencentcloud.com/document/product/1041/58242#.E4.BB.BB.E5.8A.A1.E6.B5.81.E7.A8.8B.E9.85.8D.E7.BD.AE) ниже для получения подробной информации. Вы можете быстро построить процесс, определив узлы процесса и шаблоны оркестрации услуг. |
| Associate Resource | Нет | Этот параметр конфигурации появится только в том случае, если распределение затрат было включено в разделе **General Management > General Settings**. После связывания ресурсов распределение затрат можно выполнить для счета, связанного с этой оркестрацией, на основе тегов, привязанных к ресурсам. Если вам нужно изменить ресурсы, перейдите в раздел [Cost Allocation Management](https://console.tencentcloud.com/mps/general/splitting). |

#### Форматы триггера предустановок

Когда **Trigger Format** установлен на **Preset**, система автоматически инициирует обработку вновь загруженных файлов (включая видео, аудио или файлы субтитров) в указанном каталоге, при условии, что их расширения входят в приведенный ниже список:

| Категория | Расширения файлов |
| --- | --- |
| Расширения видеофайлов | .mp4, .avi, .mov, .wmv, .flv, .mkv, .mpg, .mpeg, .rm, .rmvb, .asf, .3gp, .webm, .ts, .m4v |
| Расширения аудиофайлов | .mp3, .wav, .aac, .flac, .ogg, .m4a, .amr |
| Расширения файлов субтитров | .vtt, .srt |

> **Примечание:** Успешно обработаны могут быть только форматы файлов, поддерживаемые функциональностью MPS; в противном случае задача оркестрации завершится ошибкой. **Пример:** Если оркестрация настроена с узлом преобразования кодека аудио/видео и файл субтитров .vtt загружается в каталог триггера, оркестрация будет активирована. Однако, поскольку функция преобразования кодека аудио/видео поддерживает только обработку аудио- и видеофайлов, задача оркестрации завершится ошибкой и плата не будет взиматься.

#### Настройка уведомлений о событиях

Эта функция может предоставлять обновления в реальном времени о ходе выполнения и статусе во время выполнения задачи. Вы можете включить и настроить эту функцию для получения уведомлений. В настоящее время Tencent Cloud предоставляет два механизма уведомления о событиях: обратные вызовы TDMQ-CMQ, HTTP обратные вызовы и обратные вызовы SCF. Подробная информация приведена ниже:

| Тип обратного вызова | Описание конфигурации |
| --- | --- |
| Обратные вызовы TDMQ-CMQ | Чтобы включить обратный вызов TDMQ-CMQ, перейдите в [консоль TDMQ](https://console.tencentcloud.com/tdmq/cmq-queue?rid=1) для активации услуги CMQ и создания модели. После включения указанная услуга CMQ будет получать уведомления о событиях от MPS. Вам необходимо заполнить следующую информацию: Модель TDMQ-CMQ: по умолчанию выбирается модель очереди. Регион TDMQ-CMQ: вы можете выбрать Гуанчжоу, Шанхай, Пекин, финансовый Шанхай, финансовый Шэньчжэнь, Гонконг (Китай), Чэнду, Северная Америка или Западные США. Имя очереди: пользовательское. |
| HTTP обратные вызовы | При вызове API конфигурации уведомлений о событиях [TaskNotifyConfig](https://www.tencentcloud.com/document/product/1041/33690) укажите параметр NotifyType как URL и заполните адрес обратного вызова HTTP в параметре NotifyUrl. |

#### Настройка Actions

Вы можете определить весь процесс услуги, добавить различные узлы услуг (такие как **преобразование кодека аудио и видео, улучшение аудио и видео, мультимедийный ИИ, проверка качества мультимедиа и обработка скриншотов**, охватывающие логические узлы) и применить различные шаблоны к каждому узлу. Подробная конфигурация описана ниже:

1. Нажмите кнопку **+** для выбора требуемого действия из раскрывающегося списка и его добавления.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3b2a9062713111f096685254001c06ec.png)

2. После добавления действия отображается узел действия. Затем вы можете настроить подробную информацию на узле.

> **Примечание:** Страницы конфигурации различаются для разных действий. На следующей фигуре показана страница действия **преобразование кодека аудио/видео**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/e3eb07d771cc11f0b9a25254007c27c5.png)

#### Условия выполнения

В процессе оркестрации сначала добавьте узел проверки качества мультимедиа. Затем вы можете включить узел "IF condition", который служит логическим узлом. Этот узел не создает никаких подзадач обработки мультимедиа, но определяет, следует ли выполнять последующие узлы, на основе того, соответствует ли медиаактив условиям, установленным в узле.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a4c88fb2713111f0b9a25254007c27c5.png)

> **Примечание:** Узел оценивает, выполняются ли условия; если да, он выполняет соответствующую ветвь рабочего процесса. В противном случае он переходит к ветви рабочего процесса Else if. Каждая ветвь условия поддерживает добавление нескольких критериев оценки (AND/OR) и позволяет настроить несколько условных ветвей. Приоритет условий ветви можно установить путем перетаскивания и регулировки панели конфигурации условия ветви.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c97da441713111f087e15254005ef0f7.png)

## Включение **оркестрации** для активации автоматической активации задач

После создания оркестрация по умолчанию отключена.

- После включения оркестрации активируется автоматическая активация задач. Когда файлы загружаются в хранилище триггера, настроенное для оркестрации, система автоматически инициирует задачи обработки без необходимости вручную создавать задачи в консоли.
- Если оркестрация отключена, задачи MPS не будут выполняться для видеофайлов, загруженных в хранилище триггера.

> **Примечание:** После включения оркестрация вступает в силу только для видеофайлов, вновь загруженных в хранилище триггера. Файлы, ранее сохраненные в хранилище триггера, не будут обработаны автоматически.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/242fb7d7233011efa0b8525400db4520.png)

Вы можете найти **хранилище триггера** и **выходное хранилище**, настроенные для оркестрации, в меню **Orchestration > COS Bucket**, чтобы удобно выполнять такие операции, как загрузка, предпросмотр, загрузка файлов и т. д.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/a43d8ea7b3a911f097b152540099c741.png)

## Редактирование и удаление **оркестрации**

**Редактирование автономной оркестрации**

Нажмите **Edit** в столбце операций целевой оркестрации, чтобы перейти на страницу **Orchestration Editing**. Затем вы можете изменить имя оркестрации, хранилище триггера, каталог триггера, выходное хранилище, выходной каталог, уведомления о событиях и элементы конфигурации.

**Удаление автономной оркестрации**

- Нажмите **Delete** в столбце операций целевой оркестрации для ее удаления.
- После удаления оркестрации задачи MPS не будут выполняться для видеофайлов, загруженных в хранилище триггера.

> **Примечание:** Когда оркестрация включена, она не может быть отредактирована или удалена.

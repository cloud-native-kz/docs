# Рабочие процессы

> **Примечание:** Раздел **Workflows** (Рабочие процессы) консоли MPS заменен на **Schemes** (Схемы), которые предлагают более простые и гибкие параметры. Для настройки схемы перейдите в раздел **Schemes**.

## Обзор

После настройки рабочего процесса файлы мультимедиа, загруженные в указанный бакет и каталог, будут автоматически обрабатываться, а результаты будут загружены в указанный бакет и каталог. Рабочие процессы могут включать такие задачи, как кодирование, снятие скриншотов, создание анимированных скриншотов, модерация, распознавание содержимого, анализ содержимого и добавление водяных знаков.

## Создание рабочего процесса

2. Нажмите **Create Workflow**, чтобы перейти на страницу создания рабочего процесса и установить имя рабочего процесса, бакет-триггер, каталог-триггер, выходной бакет, выходной каталог, уведомления о событиях и задачи.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/baa60102d6d811eea2b0525400bb593a.png)

В таблице ниже указана информация, необходимая для настройки рабочего процесса.

| Элемент | Обязательно | Описание |
| --- | --- | --- |
| Имя рабочего процесса | Да | Максимум 128 символов; поддерживает китайские иероглифы, буквы, цифры, подчеркивания и дефисы. Пример: "MPS" |
| Бакет-триггер | Да | Выберите бакет, созданный под текущим `APPID`. После включения рабочего процесса видео, загруженные в этот бакет, будут автоматически обрабатываться. |
| Каталог-триггер | Нет | Строка, заканчивающаяся на (/). Если оставить пусто, рабочий процесс будет применяться ко всем каталогам в выбранном бакете-триггере. |
| Выходной бакет | Да | По умолчанию выходной бакет совпадает с бакетом-триггером. Вы также можете выбрать бакет в том же регионе под тем же `APPID`. После выполнения рабочего процесса обработанные видео будут сохранены в этом бакете. |
| Выходной каталог | Нет | Строка, заканчивающаяся на (/). Если оставить пусто, выходной каталог будет совпадать с каталогом-триггером. |
| Уведомления о событиях | Нет | Отключено по умолчанию. Для получения подробных инструкций по настройке уведомлений о событиях см. раздел [конфигурация обратного вызова](https://www.tencentcloud.com/document/product/1041/33485?lang=en#recall) ниже. Для включения уведомлений о событиях TDMQ-CMQ необходимо активировать [Tencent Distributed Message Queue](https://console.tencentcloud.com/tdmq/cmq-queue?rid=1) и создать модель. После включения уведомлений о событиях TDMQ-CMQ указанная очередь сообщений будет получать уведомления о событиях обработки видео. |
| Элементы конфигурации | Да | Из кодирования, снятия скриншотов, создания анимированных изображений, модерации, распознавания содержимого и анализа содержимого выберите как минимум одну задачу для настройки. Подробнее см. раздел [конфигурация задач](https://www.tencentcloud.com/document/product/1041/33485) ниже. |

| Метод обратного вызова | Конфигурация |
| --- | --- |
| Обратные вызовы TDMQ-CMQ | Модель TDMQ-CMQ: выберите "Queue model" (Модель очереди) или "Topic model" (Модель темы). Регион TDMQ-CMQ: выберите Гуанчжоу, Шанхай, Пекин, Финансовый центр Шанхая, Финансовый центр Шэньчжэня, Гонконг (Китай), Чэнду, Северную Америку или западное побережье США. Имя очереди/имя темы: введите пользовательское имя. |
| HTTP обратные вызовы | При вызове API конфигурации уведомлений [TaskNotifyConfig](https://www.tencentcloud.com/document/product/1041/33690#TaskNotifyConfig) установите `NotifyType` на `URL` и `NotifyUrl` на адрес HTTP обратного вызова. |
| Обратные вызовы SCF | Вы можете нажать [Go to SCF console](https://console.tencentcloud.com/scf/list-create?rid=1&ns=default&keyword=mps), чтобы настроить функцию в консоли SCF. Для получения подробных инструкций см. [MPS Task Callback Notification](https://www.tencentcloud.com/document/product/1041/40337). Конфигурация обратного вызова SCF применяется ко всем рабочим процессам и не сохраняется специально для текущего рабочего процесса. |

## Уведомления о событиях

### Получение уведомлений о событиях через CMQ

- Вы можете получать уведомления о событиях через CMQ только после активации службы CMQ и создания модели очереди или темы. Дополнительную информацию см. в разделе [CMQ > Getting Started](https://intl.cloud.tencent.com/document/product/406/8435).

### Получение уведомлений о событиях через SCF

SCF позволяет быстро обрабатывать уведомления о событиях, генерируемые MPS. На рисунке ниже показан поток данных.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ba8988d2d6d811ee85225254004842fa.png)

События отправляются в SCF триггером MPS и обрабатываются бессерверными функциями.

#### Сценарии использования

CLS может передавать данные из тем журналов в SCF через триггер журналов MPS для выполнения операций, таких как отправка уведомлений, мониторинг статуса и обработка тревог.

| Сценарий обработки функции | Описание |
| --- | --- |
| Резервная копия задачи видео в COS | Своевременное резервное копирование задач MPS, обработанных через обратный вызов, в COS через SCF |
| Уведомления обратного вызова задачи видео | Получение сообщений данных MPS в реальном времени и отправка сообщений пользователям через WeCom или электронную почту. |

> **Примечание:** Отправка данных в SCF будет сопровождаться взиманием платежей. Подробнее см. раздел [Billing Overview](https://intl.cloud.tencent.com/document/product/583/17299).

## Управление рабочими процессами

2. Список рабочих процессов отображает информацию, включая имя рабочего процесса, бакет-триггер, регион, каталог-триггер, время создания и статус. Вы можете сортировать рабочие процессы по времени создания, выполнять поиск рабочего процесса по имени, а также просматривать, редактировать или удалять рабочий процесс.
  - **Включение рабочего процесса**
    - Рабочие процессы отключены по умолчанию. Чтобы включить рабочий процесс, нажмите переключатель в столбце **Enable** (Включить).
    - После включения рабочего процесса он будет автоматически выполняться для видео, загруженных в бакет-триггер.
  - **Отключение рабочего процесса**
    - Чтобы отключить рабочий процесс, нажмите переключатель в столбце **Enable** (Включить).
    - После отключения рабочего процесса он больше не будет автоматически выполняться для видео, загруженных в бакет-триггер.
  - **Редактирование рабочего процесса**
    - Нажмите **Edit** в столбце **Operation** (Операция) целевого рабочего процесса, чтобы изменить его имя, бакет-триггер, каталог-триггер, выходной бакет, выходной каталог, параметры уведомлений о событиях и задачи.
    - Вы не можете редактировать или удалять включенный рабочий процесс.
  - **Удаление рабочего процесса**
    - Нажмите **Delete** в столбце **Operation** (Операция) целевого рабочего процесса, чтобы удалить его.
    - После удаления рабочего процесса он больше не будет автоматически выполняться для видео, загруженных в бакет-триггер.
    - Вы не можете редактировать или удалять включенный рабочий процесс.

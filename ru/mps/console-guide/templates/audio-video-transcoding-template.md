# Шаблон транскодирования аудио/видео

## Обзор

Вы можете предварительно установить различные параметры обработки для разных сценариев приложений, настроив шаблоны транскодирования аудио/видео, чтобы они могли быть переиспользованы позже.

- Если вам необходимо обрабатывать видеофайлы, вы можете настроить [шаблоны видеотранскодирования](#видео), включая шаблоны общего видеотранскодирования, шаблоны транскодирования Top Speed Codec (TSC), шаблоны адаптивного потокового видео и шаблоны переупаковки.
- Если вам необходимо обрабатывать файлы только с аудио, вы можете настроить [шаблоны аудиотранскодирования](#аудио), включая шаблоны общего аудиотранскодирования, шаблоны аудиотранскодирования TSC и шаблоны адаптивного потокового аудио.

Краткое введение в каждую функцию см. в таблице ниже:

| Применимый источник входных данных | Функция | Описание |
| --- | --- | --- |
| Видео | Общее транскодирование | Позволяет снизить битрейт видео и изменить стандарт кодирования, разрешение, частоту кадров и другие параметры исходного битового потока, адаптируя видео для воспроизведения на разных терминалах и в различных сетевых окружениях. |
|  | Транскодирование TSC | Транскодирование TSC — это «обновление» общего транскодирования. Оно может адаптивно оптимизировать видео различных типов, обеспечивая пользователям более четкое изображение при меньшей пропускной способности. Сохраняя или даже улучшая качество изображения, оно может сэкономить затраты пропускной способности на 50%+. |
|  | Адаптивное потоковое видео | Позволяет преобразовать исходный видеофайл в несколько потоков, подходящих для воспроизведения в различных сценариях, чтобы пользователи могли выбрать видео с соответствующим битрейтом для воспроизведения в зависимости от их сети, тем самым улучшая пользовательский опыт. |
|  | Переупаковка | Позволяет изменить формат контейнера исходного видеофайла. |
| Аудио | Общее аудиотранскодирование | Позволяет снизить битрейт аудио и изменить стандарт кодирования, частоту дискретизации, канал и другие параметры, адаптируя аудио для воспроизведения на разных терминалах и в различных сетевых окружениях. |
|  | Аудиотранскодирование TSC | Аудиотранскодирование TSC — это «обновление» общего аудиотранскодирования. Оно может адаптивно оптимизировать аудио различных типов, обеспечивая пользователям лучший аудиоопыт при меньшей пропускной способности. |
|  | Адаптивное потоковое аудио | Позволяет преобразовать исходный аудиофайл в несколько потоков, подходящих для воспроизведения в различных сценариях, адаптируя аудио к различным сетевым условиям и улучшая пользовательский опыт. |

|  | Разрешение и битрейт | Режим управления битрейтом | Поддерживаются следующие четыре режима управления битрейтом: VBR (по умолчанию): переменный битрейт. Битрейт выходного сигнала динамически регулируется в зависимости от сложности видеоизображения, обеспечивая более высокое качество изображения. Подходит для сценариев хранения и приложений, требующих высокого качества изображения. ABR: средний битрейт. Средний битрейт выходного видео поддерживается максимально стабильным, но допускаются кратковременные колебания битрейта. Подходит для сценариев, где требуется минимизировать общий битрейт при сохранении определенного качества изображения. CBR: постоянный битрейт. Битрейт выходного сигнала остается постоянным во время процесса кодирования видео без учета изменений сложности изображения. Подходит для сценариев со строгими требованиями к пропускной способности сети. CRF: коэффициент постоянного качества. Качество видео контролируется путем установки коэффициента качества для достижения кодирования видео с постоянным качеством, с автоматической регулировкой битрейта в зависимости от сложности содержимого. Подходит для сценариев, требующих стабильного качества изображения. |

Пользовательский: диапазон значения CRF составляет 0–51, где 0 означает сжатие без потерь, а 51 означает максимальное сжатие. Обычно рекомендуемый диапазон значений составляет 18–28, где 18 представляет низкий коэффициент сжатия с высоким качеством, а 28 представляет высокий коэффициент сжатия с низким качеством. **Примечание:** Когда режим управления битрейтом установлен на VBR и настроено значение CRF, MPS будет обрабатывать видео в режиме VBR в соответствии со значением CRF и верхним пределом среднего битрейта, чтобы уравновесить качество видео, битрейт, эффективность транскодирования и размер файла. Когда режим управления битрейтом установлен на CRF, настройка верхнего предела среднего битрейта будет недействительна, и кодирование будет выполняться в соответствии со значением CRF. Когда режим управления битрейтом установлен на ABR или CBR, CRF не требуется устанавливать. |

|  |  | Средний битрейт | Поддерживаются следующие два режима: ограничение битрейта видео не установлено. Пользовательский: настраивается в диапазоне [128, 35000] (в Кбит/с). |
|  |  | Разрешение | Поддерживаются следующие три режима: Сохранить исходное: разрешение соответствует разрешению исходного видео. Масштабирование: изображение масштабируется пропорционально в зависимости от длинной или короткой стороны (ширины или высоты) с диапазоном размера [128, 4096] (в пиксел.). Пользовательский: как длинная, так и короткая стороны (ширина и высота) можно масштабировать в пользовательской пропорции с диапазоном размера [128, 4096] (в пиксел.). |
|  |  | Соотношение сторон дисплея (SAR) | Используется, когда разрешение установлено на «Пользовательский» и соотношение сторон видео отличается от исходного видео. Белые полосы: аналогично режиму черных полос, но белые полосы добавляются вверху и внизу/слева и справа от видео. Размытие по Гауссу: применяет эффект размытия по Гауссу к растянутым или заполненным областям, чтобы уменьшить визуальный дискомфорт. Интеллектуальное кадрирование: видеоизображение, которое должно быть сохранено, интеллектуально выбирается во время кадрирования видео. |

|  | Применимые видеосценарии |  | MPS предоставляет несколько предустановленных видеосценариев с различными режимами обработки, настроенными для адаптации к различным характеристикам сценариев: Сценарии общего транскодирования: сценарии сжатия общего транскодирования. PGC видеоконтент и телепередачи высокой четкости: акцент делается на улучшение опыта просмотра фильмов и телепередач во время сжатия. Кодирование на основе ROI применяется с учетом характеристик фильмов и телепередач, при этом сохраняются высококачественные видео и аудиоматериалы. Материал высокого разрешения: этот сценарий включает категории ресурсов материала с чрезвычайно высокими требованиями к качеству изображения и содержит много элементов прозрачного изображения. Сжатие достигает почти визуально безлосевого качества. Пользовательский контент (UGC): подходит для широкого спектра сценариев UGC/коротких видео. Оптимизирует битрейт кодирования и качество изображения на основе характеристик коротких видео и улучшает метрики QoS/QoE бизнеса. Витрина/электронная торговля: уделяет особое внимание четкости деталей и улучшению области ROI во время сжатия, с особым вниманием к поддержанию качества изображения в областях лиц. Образование: акцент делается на четкость и читаемость текста и изображений во время сжатия, чтобы помочь студентам лучше понять содержание, обеспечивая четкое передачу учебного материала. |
|  | Стратегия транскодирования |  | Транскодирование TSC поддерживает стратегии **стандартного сжатия** и **экстремального сжатия**. Различные стратегии имеют различные базовые модели транскодирования, которые влияют на битрейт, качество изображения и другие результаты. Стандартное сжатие: стандартное транскодирование TSC, которое предоставляет три варианта: Низкое сжатие — качество изображения предпочтительнее, Среднее сжатие — комплексная оптимизация, Высокое сжатие — битрейт предпочтительнее. От верхнего к нижнему, чем выше уровень сжатия, тем меньше размер выходного видеофайла и тем больше потеря качества изображения. Использование любого варианта этой стратегии **взимает только плату за транскодирование TSC.** Экстремальное сжатие: «обновленная версия» стандартного сжатия, **с наивысшим уровнем сжатия**, который максимизирует битрейт сжатия при обеспечении определенного уровня качества изображения, значительно экономя затраты на пропускную способность и хранилище. Использование этой стратегии **взимает как плату за транскодирование TSC, так и плату за удаление артефактов улучшения аудио/видео.** Для получения подробной информации см. [Описание выставления счетов](https://www.tencentcloud.com/document/product/1041/49204#tsc-transcoding). **Примечание:** Если выходное видео предназначено для просмотра на телевизоре, не рекомендуется использовать стратегию экстремального сжатия. |

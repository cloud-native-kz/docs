# Шаблон проверки качества медиа

## Обзор

Сервис проверки качества медиа может комплексно выявлять проблемы в медиа-активах, проверять соответствие содержимого требованиям и предлагать рекомендации. Пользователи могут напрямую использовать предустановленные шаблоны или создавать шаблоны проверки качества медиа в консоли для настройки параметров проверки качества.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7bb1f3816f2011ef92db52540055f650.png)

Содержание проверки качества включает:

- **Диагностика формата**: диагностика медиа — это облачный сервис обработки аудио и видео, предназначенный для диагностики потоков медиа в реальном времени. Его можно использовать для анализа исключений в информации потока, временной информации, статусе потока, инкапсуляции контейнера и декодировании.
- **Оценка без эталона**: это относится к оценке качества видео без эталона с баллами в диапазоне от 0 до 100.
- **Качество изображения**: предоставляет множество функций проверки качества, включая, помимо прочего, обнаружение сбоев экрана, чёрного экрана, дрожания, шума, мозаики, низкой освещённости, QR-кода, штрихкода и кода мини-приложения.
- **Обнаружение звука**: предоставляет множество функций проверки качества, включая, помимо прочего, обнаружение отсутствия голоса, низкого голоса и высокого голоса.

## Описание параметров шаблона проверки качества

В шаблоне проверки качества вы можете включить элементы проверки и настроить их параметры. Подробные сведения о выставлении счетов за проверку качества медиа см. в [инструкциях по выставлению счетов](https://www.tencentcloud.com/document/product/1041/49204?lang=en&pg=#2bc42389-7247-4b15-a8c7-d4e4aa997853).

### Параметры обнаружения

Параметры обнаружения предоставляют настраиваемый контроль над областью обнаружения, а также методом записи обнаруженных проблем.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/87e3128d712f11f096685254001c06ec.png)

#### Способ записи проблем проверки качества

По умолчанию система регистрирует временные интервалы, в течение которых возникают проблемы с проверкой качества. Если выбран вариант «Снимки экрана начального и конечного кадров», отчёт о проверке качества дополнительно отобразит два снимка экрана, захватывающих начальную и конечную точки выявленной проблемы.

> **Примечание:** включение функции «Снимки экрана начального и конечного кадров» будет нести дополнительные расходы на снимки экрана и хранилище изображений помимо сборов за проверку качества.

#### Область обнаружения

По умолчанию проверка качества охватывает всё видео. Однако она также поддерживает конфигурацию пользовательской области обнаружения. После включения пользовательской области она будет отображаться, как показано ниже.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9b297afb712f11f0bcac525400e889b2.png)

- **Пропустить intro:** при инициировании каждой задачи проверки качества система будет пропускать указанную продолжительность перед началом обнаружения. Эта функция особенно подходит для сценариев, в которых видеосодержимое включает фиксированный intro, что эффективно предотвращает избыточное обнаружение повторяющихся видеосегментов. Диапазон для пропуска продолжительности intro составляет от 1 до 1800 секунд.
- **Продолжительность обнаружения:** продолжительность обнаружения определяет длительность каждого действия проверки качества и служит критическим параметром для выборочной проверки качества видео. Для больших объёмов видеосодержимого установка продолжительности обнаружения может значительно снизить стоимость проверки. Например, указав продолжительность обнаружения 300 секунд, задача проверки качества будет анализировать только 300 секунд видео при выполнении, устраняя необходимость проверки всей длины видео. (Допустимый диапазон параметра продолжительности обнаружения составляет 10–86 400 секунд.)
- **Интервал обнаружения:** путём настройки интервала обнаружения наряду с продолжительностью обнаружения проверка качества может достичь таких эффектов, как выборка 5 минут содержимого для проверки в течение каждого часа видео. (Диапазон для продолжительности интервала обнаружения составляет 10–3600 секунд.)
- **Количество циклов:** этот параметр используется для ограничения количества циклов повторных проверок. По умолчанию повторяющиеся проверки будут непрерывно выполнять операцию «обнаружить-интервал ожидания-обнаружить» до проверки всего видеосодержимого. Когда установлено количество циклов, проверка качества прекратится после достижения указанного количества циклов.

# Авторизация учетной записи

Если разрешения вашей учетной записи недостаточны, вы можете не суметь нормально использовать MPS, что более вероятно при использовании вспомогательных учетных записей. В этом документе описаны распространенные сценарии недостаточных разрешений и соответствующие решения.

## Сценарий 1 (обязательно): служба MPS не включена для учетной записи

Если служба MPS не включена для вашей учетной записи, вы не сможете нормально использовать MPS. В этом случае необходимо сначала включить службу MPS перед использованием консоли MPS или вызовом API. **Эта операция является обязательной**.

### Решение

- Свяжитесь с владельцем **корневой учетной записи**, войдите в корневую учетную запись, перейдите в [консоль MPS](https://console.tencentcloud.com/mps/index) и нажмите **Activate For Free**.
- Если вспомогательная учетная запись имеет полные права на чтение/запись для MPS (см. [Сценарий 2](#mpsfullaccess) для связи с предустановленной политикой QcloudMPSFullAccess), вы можете напрямую включить MPS на вспомогательной учетной записи.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/474c1881190311f09bd7525400e889b2.png)

## Сценарий 2 (обязательно): вспомогательная учетная запись не имеет разрешений на операции MPS

Если корневая учетная запись не предоставляет вспомогательной учетной записи разрешений на операции чтения/записи для MPS, вспомогательная учетная запись не сможет нормально использовать MPS.

- В этом случае при доступе к консоли MPS с вспомогательной учетной записи будет выдана следующая ошибка:

![Сообщение во всплывающем окне указывает, что вспомогательная учетная запись не имеет разрешения для API "DescribeUserInfo" MPS.](https://staticintl.cloudcachetci.com/cms/backend-cms/a45e7ec4190811f09240525400bf7822.png)

- При вызове API MPS с вспомогательной учетной записи будет выдана следующая ошибка:

![Сообщение об ошибке указывает, что вспомогательная учетная запись не имеет разрешений на операции.](https://staticintl.cloudcachetci.com/cms/backend-cms/15d4a4f3190911f0b5c65254001c06ec.png)

Для решения этой проблемы вы можете следовать любому из приведенных ниже решений, чтобы предоставить вспомогательной учетной записи разрешения на операции MPS. **Эта операция является обязательной.**

### Решение

#### Решение 1: связать вспомогательную учетную запись с предустановленной политикой QcloudMPSFullAccess (рекомендуется)

1. Свяжитесь с владельцем корневой учетной записи и войдите в корневую учетную запись.
2. Перейдите на страницу [Cloud Access Management](https://console.tencentcloud.com/cam) в корневой учетной записи, найдите вспомогательную учетную запись, которая должна использовать MPS в **списке пользователей**, и нажмите **Authorize**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7d4a9582160711f09bd7525400e889b2.png)

3. Найдите "QcloudMPSFullAccess" и предоставьте вспомогательной учетной записи разрешение предустановленной политики QcloudMPSFullAccess.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/51712b08160711f08ac55254007c27c5.png)

#### Решение 2: связать вспомогательную учетную запись с пользовательской политикой

> **Примечание:** В настоящее время QcloudMPSFullAccess является единственной предустановленной политикой MPS. Эта полная политика чтения/записи позволяет вспомогательным учетным записям получать доступ ко всем API MPS. Если корневая учетная запись хочет ограничить разрешения доступа к API для вспомогательных учетных записей в целях безопасности, она может связать пользовательскую политику с вспомогательными учетными записями.

1. Перейдите в [Cloud Access Management](https://console.tencentcloud.com/cam/policy) в корневой учетной записи и нажмите **Create Custom Policy** на странице **Policies**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/af4ff670160711f09240525400bf7822.png)

2. Выберите **Create by policy builder**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/2d90e714190911f08ac55254007c27c5.png)

3. В разделе "Service" выберите **Media Processing Service (mps)**. Затем в разделе "Action" выберите API MPS, разрешенные для вспомогательных учетных записей. За один раз можно выбрать несколько API.

> **Примечание:** Если вы используете SDK, необходимо выбрать API RequestFromSdk.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/98ddb736190911f09240525400bf7822.png)

4. В разделе "Resource" выберите **All resources** и нажмите **Next**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6327f446190a11f09bd7525400e889b2.png)

5. Введите имя политики, свяжите политику с вспомогательной учетной записью в поле "Grant this permission to users" и нажмите **Done**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/8aa807db190a11f0b5c65254001c06ec.png)

6. Вернитесь на страницу **Policies**. Вы можете просмотреть все пользовательские политики на этой странице и позже нажать на имя политики, чтобы изменить соответствующую политику.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/9cac8bc1190a11f0b5c65254001c06ec.png)

## Сценарий 3 (опционально): MPS не авторизирована для выполнения необходимых операций чтения/записи файлов в ваших сегментах

В настоящее время служба обработки мультимедиа поддерживает четыре типа источников входных файлов: Tencent Cloud Object Storage (COS), Tencent Cloud [VOD Pro](https://www.tencentcloud.com/document/product/266/67978), AWS S3 и загружаемые URL-адреса.

- **Для использования COS в качестве источника файлов необходимо предварительно завершить авторизацию COS.** Создайте роль сервиса `MPS_QcsRole`, чтобы предоставить службе обработки мультимедиа разрешения на чтение и запись, включая загрузку, перекодирование и загрузку файлов из вашего сегмента COS.
- **Для использования** [VOD Pro](https://www.tencentcloud.com/document/product/266/67978) **в качестве источника файлов необходимо предварительно завершить авторизацию VOD.** Создайте роль сервиса `MPS_QCSLinkedRoleInVOD`, чтобы предоставить службе обработки мультимедиа разрешения на чтение и запись, включая загрузку, перекодирование и загрузку файлов из вашего сегмента VOD Pro.
- Если вы планируете использовать AWS S3 или URL-адрес в качестве источника файлов, авторизацию COS можно пропустить.

Если вы инициируете задачу обработки через API MPS без завершения требуемой авторизации для сегментов COS или VOD Pro, задача завершится ошибкой. Пример приведен ниже:

![Сначала вызовите API ProcessMedia для обработки файлов в сегменте COS:](https://staticintl.cloudcachetci.com/cms/backend-cms/54a1eef6190f11f0b04252540044a08e.png)

![Затем используйте 'TaskId' для проверки статуса задачи, где вы встретите ошибку:](https://staticintl.cloudcachetci.com/cms/backend-cms/a3170ce9191011f09bd7525400e889b2.png)

### Решение

Для использования COS войдите в **корневую учетную запись**, откройте страницу обзора [консоли MPS](https://console.tencentcloud.com/mps/index), нажмите **COS Authorization** и нажмите **Authorize** во всплывающем окне. После завершения авторизации роль сервиса `MPS_QcsRole` будет автоматически создана в [Cloud Access Management > Role](https://console.tencentcloud.com/cam/role).

Аналогично, если вы хотите использовать VOD Pro, просто нажмите **VOD Authorization** для создания роли сервиса `MPS_QCSLinkedRoleInVOD`.

> **Примечание:** Эта операция авторизации **не требует включения COS или VOD и не повлечет расходы на COS или VOD**. Она просто позволяет MPS получить необходимые разрешения на чтение/запись API для сегментов COS или VOD.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/33ea794bc8e711f09e745254007c27c5.png)

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3e5f6088c8e711f08942525400e889b2.png)

## Сценарий 4 (опционально): вспомогательная учетная запись не имеет разрешения cam:GetRole

После завершения авторизации разрешений в трех приведенных выше сценариях при входе вспомогательной учетной записи в консоль MPS может появиться следующее сообщение об ошибке:

![Сообщение во всплывающем окне указывает, что вспомогательная учетная запись не имеет разрешения cam:GetRole.](https://staticintl.cloudcachetci.com/cms/backend-cms/5ef0287c191111f091eb525400454e06.png)

Это происходит потому, что интерфейс консоли MPS будет вызывать API GetRole управления доступом Tencent Cloud (CAM) для проверки наличия у пользователя роли сервиса MPS_QcsRole. Интерфейс определяет, нужно ли выводить подсказку авторизации, на основе результата запроса. Если вспомогательная учетная запись не имеет разрешения cam:GetRole, интерфейс консоли MPS не может вызвать API GetRole для запроса роли. В результате появляется окно с указанной ошибкой.

> **Примечание:** GetRole относится к операции интерфейса консоли MPS. Вспомогательная учетная запись не может нормально использовать консоль MPS, если у нее нет разрешения cam:GetRole. API по-прежнему можно вызывать. Консоль помогает выполнять операции MPS и просматривать результаты задач. Поэтому рекомендуется следовать приведенному ниже решению и предоставить вспомогательной учетной записи разрешение cam:GetRole.

### Решение

1. Свяжитесь с владельцем корневой учетной записи и войдите в корневую учетную запись.
2. Перейдите в [Cloud Access Management > Policy](https://console.tencentcloud.com/cam/policy) в корневой учетной записи, нажмите **Create Custom Policy** и выберите **Create by policy generator** для создания пользовательской политики.

Выберите Cloud Access Management (CAM) в качестве сервиса, GetRole в качестве операции чтения и All resources, как показано в красном прямоугольнике ниже. Затем нажмите **Next**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/98039f08191111f09bd7525400e889b2.png)

3. Введите имя политики, предоставьте это разрешение вспомогательной учетной записи, которой требуется использовать MPS, и нажмите **Complete**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/aab7638b191111f0897952540099c741.png)

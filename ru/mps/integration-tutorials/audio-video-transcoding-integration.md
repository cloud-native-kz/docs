# Интеграция транскодирования аудио/видео

## I. Обзор функции транскодирования аудио/видео

### Обзор

Транскодирование аудио/видео — это процесс преобразования исходного потока битрейта аудио/видео в другой поток битрейта аудио/видео, и это задача автономной обработки. Через транскодирование можно изменять параметры, такие как формат кодирования, разрешение и битрейт исходного потока, чтобы адаптировать воспроизведение на различных устройствах и в разных сетевых средах. Объяснение различных типов транскодирования см. в таблице ниже.

| Применимый источник входных данных | Тип транскодирования | Описание |
| --- | --- | --- |
| Видео | Стандартное видеотранскодирование | Может уменьшить битрейт видео и изменить параметры исходного потока, такие как стандарт кодирования, разрешение и частота кадров, позволяя адаптировать воспроизведение на различных устройствах и в разных сетевых средах. |
| | TSC транскодирование | Транскодирование Top Speed Codec (TSC) — это «улучшенная» функция стандартного видеотранскодирования. Может выполнять адаптивную оптимизацию различных типов видео, обеспечивая пользователям более четкий просмотр с меньшим потреблением полосы пропускания. При сохранении или даже улучшении качества изображения уменьшает затраты на полосу пропускания более чем на 50%. |
| | Адаптивный поток битрейта для видео | Преобразует входной видеофайл в несколько потоков с разными битрейтами, подходящих для воспроизведения в различных сценариях, позволяя пользователям выбирать видео с подходящим битрейтом на основе состояния сети и улучшая пользовательский опыт. |
| | Remux | Может изменять формат контейнера исходного видеофайла. |
| Аудио | Стандартное аудиотранскодирование | Может уменьшить битрейт аудио и изменить параметры, такие как стандарт кодирования, частота дискретизации и звуковой канал, позволяя адаптировать воспроизведение на различных устройствах и в разных сетевых средах. |
| | TSC транскодирование для аудио | TSC транскодирование для аудио — это «улучшенная» функция стандартного аудиотранскодирования. Может выполнять адаптивную оптимизацию различных типов аудио, обеспечивая пользователям более четкое прослушивание с меньшим потреблением полосы пропускания. |
| | Адаптивный поток битрейта для аудио | Может преобразовать входной аудиофайл в несколько потоков с разными битрейтами, подходящих для воспроизведения в различных сценариях, позволяя адаптировать воспроизведение к различным сетевым условиям и улучшить пользовательский опыт. |

#### **Технические преимущества**

В отрасли Tencent Cloud MPS поддерживает наиболее полный стандарт сжатия кодирования аудио/видео. Предоставляет несколько методов транскодирования, таких как TSC транскодирование, адаптивное транскодирование потока битрейта, Remux и аудиотранскодирование. С поддержкой распределенного транскодирования до 30+ раз ускорения помогает уменьшить потребление полосы пропускания при воспроизведении и снизить затраты.

##### **Полный метод кодирования, снижение битрейта более чем на 50%**

Предоставляет полный набор методов кодирования (таких как VP8, H.264, VP9, H.265, AV1, AVS3 и H.266), которые могут значительно снизить битрейт без потери качества, экономя полосу пропускания при воспроизведении и затраты на хранение.

##### **Высокое качество изображения и кристально четкий просмотр**

Использование интеллектуальной технологии динамического кодирования, TSC обеспечивает превосходный эффект просмотра при одновременном снижении битрейта. По сравнению с открытыми кодировщиками оценка VMAF увеличивается на 3–5 пунктов. Поддерживает кодирование видео в реальном времени с разрешением до 8K, интегрируя передовые технологии, такие как супер-разрешение, HDR и расширенный цветовой охват. Кроме того, был одним из первых в Китае, поддерживающих Dolby Vision/Dolby Atmos, обеспечивая исключительно четкий просмотр.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/4c172f7e4a5a11f08548525400454e06.png)

> **Примечание:** Откройте [MPS Demo](https://mps.live/demo/encoding/video), где вы можете быстро испытать эффекты функции транскодирования аудио/видео.

Tencent Cloud MPS предоставляет три способа инициирования задач: быстрое создание задач через консоль, инициирование задач через API и автоматическое срабатывание задач. На следующей блок-схеме показан общий процесс операций для каждого способа инициирования задач. ## II. Создание задач транскодирования аудио/видео

### Способ 1. Инициирование задачи без кода из консоли

1. Перейдите в консоль [Media Processing Service](https://console.tencentcloud.com/mps/index) и нажмите **Create Task > Create VOD Processing Task** (Создать задачу > Создать задачу обработки VOD).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/eba402964a5b11f0914c52540044a08e.png)

2. Сначала укажите входной видеофайл. Вы можете выбрать аудио/видеофайлы из ведра COS или AWS S3 или предоставить URL загрузки файла.
3. Затем на шаге **Select Input File** (Выбрать входной файл) добавьте узел **Audio/Video Transcoding** (Транскодирование аудио/видео).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/93c2112b4ce411f08957525400e889b2.png)

4. В окне всплывающего меню **Audio/Video Enhancement Settings** (Параметры улучшения аудио/видео) выберите требуемый шаблон транскодирования аудио/видео или пользовательские параметры транскодирования. Подробные инструкции по конфигурации параметров шаблона см. в [Шаблон транскодирования аудио/видео](https://www.tencentcloud.com/document/product/1041/48784).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f1bae5d94a5f11f092f25254007c27c5.png)

5. И наконец, после указания пути сохранения выходного видео нажмите **Create** (Создать) для инициирования задачи.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/624d8bc14a6011f08548525400454e06.png)

### Способ 2. Инициирование задачи через API

- Способ (1): Вызовите API [ProcessMedia](https://www.tencentcloud.com/document/product/1041/33640?lang=en&pg=) для инициирования задачи путем указания **Template ID** (ID шаблона). Пример:

```
{  "InputInfo": {    "Type": "URL",    "UrlInputInfo": {      "Url": "https://test-1234567.cos.ap-guangzhou.myqcloud.com/video/test.mp4"// Замените URL видео, которое требуется обработать.    }  },  "OutputStorage": {    "Type": "cos",    "CosOutputStorage": {      "Bucket": "test-1234567",      "Region": "ap-guangzhou"    }  },  "MediaProcessTask": {    "TranscodeTaskSet": [      {        "Definition": 100800//100800 — это ID предустановленного шаблона для видеотранскодирования. Может быть заменен на ID вашего пользовательского шаблона транскодирования аудио/видео.      }    ]  }}
```

- Способ (2): Вызовите API [ProcessMedia](https://www.tencentcloud.com/document/product/1041/33640?lang=en&pg=) для инициирования задачи путем указания **Orchestration ID** (ID оркестрации) (инструкции по созданию оркестрации см. в разделе [Автоматическое срабатывание](https://www.tencentcloud.com/document/product/1041/70464#fcce02ee-4dba-4a2d-870a-f136220a6a8e) ниже или в документации [VOD Orchestration](https://www.tencentcloud.com/document/product/1041/58242)). Пример:

```
{  "InputInfo": {    "Type": "COS",    "CosInputInfo": {      "Bucket": "test-1234567",      "Region": "ap-guangzhou",      "Object": "/video/123.mp4"    }  },  "OutputStorage": {    "Type": "cos",    "CosOutputStorage": {      "Bucket": "test-1234567",      "Region": "ap-guangzhou"    }  },  "ScheduleId": 12345  //Замените на ID пользовательской оркестрации. 12345 — это образец кода и не имеет практического значения.}
```

> **Рекомендуется использовать** **API Explorer для быстрой отладки и проверки** **:**Откройте страницу онлайн-отладки MPS [API Explorer](https://console.intl.cloud.tencent.com/api/explorer?Product=mps&Version=2019-06-12&Action=ProcessMedia) и выберите API [ProcessMedia](https://console.intl.cloud.tencent.com/api/explorer?Product=mps&Version=2019-06-12&Action=ProcessMedia) из списка API слева. См. рисунок ниже для заполнения параметров, таких как входной путь, выходной путь, ID шаблона и ID оркестрации, затем вы сможете инициировать онлайн-вызов API.![](https://staticintl.cloudcachetci.com/cms/backend-cms/a8953fab4a6411f085275254001c06ec.png)

#### Приложение: Дополнительные параметры для API-запросов, связанных с транскодированием

Некоторые продвинутые функции недоступны для конфигурации на странице создания шаблона транскодирования через консоль. Вы можете использовать API для [CreateTranscodeTemplate](https://www.tencentcloud.com/document/product/1041/33671) и передать дополнительные параметры через `StdExtInfo`.

#### Добавление звуковых потоков с пакетами тишины

| Название параметра | Требование | Тип | Примечания |
| --- | --- | --- | --- |
| enable_mute_audio | Необязательно. | Int | Если включено, пакеты тишины будут добавлены. Значение по умолчанию — 0.0: Не включать.1: Заполнять пакетами тишины при разрывах в аудиокадрах.2: На основе 1, добавлять пакеты тишины для случаев отсутствия аудио в конце. |

Пример JSON:

```
"StdExtInfo" : "{\\"audio_info\\":{\\"enable_mute_audio\\":1}}"
```

#### Метод хранения выходного файла транскодирования

Консоль поддерживает конфигурацию COS и AWS S3 в качестве путей вывода. Вы также можете использовать дополнительные параметры для загрузки выходных файлов транскодирования в другие облачные хранилища третьих сторон.

| Название параметра | Требование | Тип | Примечания |
| --- | --- | --- | --- |
| appid | Необязательно. | String | ID облачного аккаунта. |
| bucket | Обязательно | String | Имя ведра. |
| region | Обязательно | String | Имя региона ведра. |
| headers | Необязательно. | JSON структура | Тип JSON структура, указывающая заголовки, передаваемые при загрузке файлов. Пример:"headers": {  "agent" : "tencent",  "used" : "for-test"} |
| token | Необязательно. | String | Загрузка и скачивание с использованием метода временного токена. |
| id | Необязательно. | String | SecretID пользователя. |
| key | Необязательно. | String | SecretKey пользователя. |
| host | Необязательно. | String | host |
| access_point | Необязательно. | string | Требуется для типа aws_vpc. |
| storage_type | Необязательно. | String | "aws_s3": AWS S3."aws_vpc": Приватное облако AWS."oss": Alibaba Cloud OSS. |

Пример JSON следующий:

```
"StdExtInfo" : "{\\"cos_info\\":{\\"storage_type\\":\\"oss\\",\\"bucket\\":\\"test-test\\",\\"region\\":\\"cn-shenzhen\\",\\"id\\":\\"abcdefghijk\\",\\"key\\":\\"lmnopqrstuvwxyz\\"}}"
```

### Способ 3. Автоматическое срабатывание задачи MPS после загрузки файла в COS

Если вы хотите загрузить видеофайл в ведро COS и достичь автоматического транскодирования аудио/видео согласно предустановленным параметрам, вы можете:

1. Нажмите **Save the Orchestration** (Сохранить оркестрацию) при создании задачи и сконфигурируйте параметры, такие как срабатываемое ведро и срабатываемый каталог в окне всплывающего меню.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3a85d3974aa511f0914c52540044a08e.png)

2. Затем перейдите в список **VOD Orchestration** (Оркестрация VOD), найдите новую оркестрацию и включите переключатель в **Enable** (Включить). После этого любые видеофайлы, добавленные в срабатываемый каталог, автоматически инициируют задачи согласно предустановленному процессу и параметрам оркестрации, и обработанные видеофайлы будут сохранены в путь вывода, сконфигурированный в оркестрации.

> **Примечание:** Оркестрация вступает в силу в течение 3–5 минут после включения.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/92768f314a6511f0b25352540099c741.png)

## III. Запрос результатов задачи

### 1. Обратный вызов задачи

При инициировании задачи MPS с помощью [ProcessMedia](https://www.tencentcloud.com/document/product/1041/33640) вы можете установить информацию обратного вызова через параметр `TaskNotifyConfig`. После завершения обработки задачи результат задачи будет переправлен обратным вызовом через сконфигурированную информацию обратного вызова. Вы можете проанализировать результат уведомления о событии через [ParseNotification](https://www.tencentcloud.com/document/product/1041/33679).

### 2. Запрос результатов задачи

#### Способ 1: Запрос результатов задачи путем вызова API [DescribeTaskDetail](https://www.tencentcloud.com/document/product/1041/33644):

Введите `TaskId`, возвращенный [ProcessMedia](https://www.tencentcloud.com/document/product/1041/33640) (например, 24000022-WorkflowTask-b20a8exxxxxxx1tt110253) для запроса результатов задачи.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c6834f3f4a7911f08957525400e889b2.png)

#### Способ 2: Запрос результатов задачи в консоли:

1. Перейдите в консоль [VOD Processing Tasks](https://console.tencentcloud.com/mps/tasks/vod-list), и только что инициированная задача будет указана в списке задач.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/4226c02d4ce511f092f25254007c27c5.png)

2. Когда подзадача находится в статусе «Successful» (Успешно), вы можете перейти в **COS Bucket > Output Bucket** (Ведро COS > Выходное ведро), найти ваше место вывода и затем просмотреть или загрузить выходные файлы транскодирования аудио/видео.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/5e9eec7d4ce511f085275254001c06ec.png)

### Встраивание субтитров (жесткие субтитры) при транскодировании

> **Примечание:** Использование функции встраивания субтитров повлечет плату за транскодирование. Полное объяснение тарификации см. в [Тарификация транскодирования аудио/видео](https://www.tencentcloud.com/document/product/1041/49204#bb410a03-3f4a-4e55-8952-4780fa8a2752).

#### **Способ 1: Инициирование задач встраивания субтитров в консоли**

При создании задачи транскодирования вы можете встраивать субтитры в видеоизображение как жесткие субтитры. Вы можете выбрать источник субтитров двумя способами:

- Файл субтитров: Поддерживает выбор файла субтитров, сохраненного в COS, или ввод адреса URL файла субтитров. Поддерживаемые форматы: SRT, WebVTT, ASS, SSA, PGS и SUP.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ff148ce283de11f093f45254005ef0f7.png)

- Дорожка субтитров исходного видео: Поддерживает указание дорожки субтитров (с порядковым номером, являющимся положительным целым числом в диапазоне от 1 до 100) в исходном видео для встраивания субтитров в видеоизображение.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/41f8854983d911f0b321525400e889b2.png)

Вы можете настроить стиль отображения субтитров перед встраиванием субтитров.

- Стиль по умолчанию: Используется встроенный стиль файла субтитров. Если встроенный стиль не существует, применяется стиль по умолчанию MPS.
- Предустановленный стиль: Предоставляются четыре опции предустановленного стиля: **маленькие субтитры без фона, большие субтитры без фона, маленькие субтитры с фоном и большие субтитры с фоном**. Вы можете щелкнуть соответствующее изображение предпросмотра для просмотра подробных конфигураций параметров, которые можно регулировать при необходимости.
- Пользовательский стиль: Вы можете перейти на страницу пользовательской конфигурации для гибкой регулировки всех параметров стиля субтитров.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/785f5a8583df11f093f45254005ef0f7.png)

# Руководство по трансформации видео из горизонтального в вертикальное формат

## Обзор

### О трансформации видео из горизонтального в вертикальное формат

Трансформация видео из горизонтального в вертикальное формат — это не просто поворот, а определение области интереса (ROI) и обрезка видео до пропорции, подходящей для воспроизведения на мобильных устройствах. Трансформация видео из горизонтального в вертикальное формат позволяет массово генерировать короткие видео и преобразовывать существующие горизонтальные видеоресурсы в вертикальные видеоресурсы.

![Интеллектуальное обрезание выполняется путём отслеживания ROI (обычно расположение мяча в спортивных видео) при преобразовании видео в вертикальный формат.](https://staticintl.cloudcachetci.com/cms/backend-cms/dd33d330ad3511ef9b4c525400bdab9d.png)

### Демонстрация

1. Перейдите на [Experience Center](https://mps.live/demo/ai/htv) на страницу "Horizontal to vertical". На правой стороне выберите локальный видеофайл, укажите тип сценария и соотношение сторон, затем нажмите **One-Click Processing**.
2. После завершения обработки вы сможете просмотреть результаты.

> **Примечание:** Функция демо MPS относительно простая и предназначена только для ознакомления с базовым эффектом. Для полного тестирования используйте доступ через API.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b3ec6b18564b11f09a935254007c27c5.png)

### Форматы входных и выходных данных

Входные видеофайлы поддерживают следующие форматы:

- Стандарты кодирования: MPEG, H.264 и H.265.
- Форматы контейнеров: .mp4, .avi, .mkv, .mov и .mpg.

Выходные видеофайлы кодируются единообразно в H.264 и форматируются в .mp4. Вы можете просматривать видео в браузере Google Chrome. Если требуется трансформирование и переупаковка форматов, вы можете обработать их локально или использовать функцию аудио- и видеотранскодирования, предоставляемую Media Processing Service (MPS).

## Способ интеграции 1: инициирование задачи через API

### 1. Вызов API

Вы можете напрямую отправить POST-запрос в Tencent Cloud. Доменное имя запроса API — mps.tencentcloudapi.com. Вызовите [MPS API](https://www.tencentcloud.com/document/product/1041/33640), выберите **AiAnalysisTask** и установите Definition на **28 (предустановленный шаблон трансформации видео из горизонтального в вертикальное формат).** ExtendedParameter — это расширенный параметр, значение которого — это **экранированная JSON-строка**. Для получения информации о конкретных значениях параметров см. раздел [Описание расширенного параметра](#note) ниже.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/fe38bf01ad3511ef897b52540075b605.png)

#### **Пример запроса**

> **Примечание:** В настоящее время функция трансформации видео из горизонтального в вертикальное формат поддерживает два источника входных данных: [Tencent Cloud Object Storage (COS)](https://www.tencentcloud.com/products/cos) и адреса URL для загрузки. AWS S3 в настоящее время не поддерживается.

```
{    "Action": "ProcessMedia",    "Version": "2019-06-12",    "InputInfo": {        "Type": "URL",        "UrlInputInfo": {            "Url": "https://..." // Замените на URL видео, которое необходимо трансформировать из горизонтального в вертикальное формат, или обратитесь к документации API "ProcessMedia".        }    },    "OutputStorage": {        "CosOutputStorage": {            "Bucket": "BucketName",            "Region": "BucketRegion"        },        "Type": "COS"    },    "OutputDir": "/mycos/htv_test/result/",    "AiAnalysisTask": {        "Definition": 28, // ID предустановленного шаблона трансформации видео из горизонтального в вертикальное формат. ### 2. Быстрая проверка через API Explorer
```

Вы можете выполнить быструю проверку через [API Explorer](https://console.intl.cloud.tencent.com/api/explorer?Product=mps&Version=2019-06-12&Action=ProcessMedia). После заполнения соответствующей информации параметров на странице вы можете инициировать онлайн-вызов API.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/052fca29a64911efbd5752540055f650.png)

> **Примечание:** API Explorer автоматически преобразует формат. Вам нужно только ввести соответствующий ExtendedParameter в формате JSON без преобразования в строку. При прямом вызове API необходимо экранировать JSON-строку.

### 3. Описание расширенного параметра

Расширенные параметры позволяют включить определённые возможности. Так как MPS API не может анализировать расширенные параметры, значение ExtendedParameter — это **сериализованная JSON-строка**. ExtendedParameter для трансформации видео из горизонтального в вертикальное формат должен располагаться под htv. Ниже представлен пример JSON перед сериализацией. При использовании API введите экранированные параметры:

```
{    "htv": {      "AlgorithmType": 2,	      "SmoothWeight":  0.75,      "Ratio": "9:16"    }}// После экранирования (требуется при прямом вызове API): {\\"htv\\": {\\"AlgorithmType\\": 2, \\"SmoothWeight\\": 0.75, \\"Ratio\\": \\"9:16\\" }}
```

| Имя | Тип | Описание |
| --- | --- | --- |
| AlgorithmType | Integer | Указанные категории алгоритмов:1: общая модель с относительно быстрой скоростью обработки.2: поддерживает различные модели и оптимизирована под специальные задачи.3: использует точный алгоритм обнаружения лиц; при обнаружении двух лиц они отображаются в режиме разделённого экрана с попыткой центрирования лиц максимально возможно.5: масштабирование видео напрямую и его центрирование на портретном экране. Использует изображение, обработанное с эффектом матового стекла, в качестве фона. |
| Ratio | String | Соотношение сторон видео, которое является строкой и анализируется внутренне. При ошибке анализа используется значение по умолчанию 9**:**16 для обрезания (например, "9**:**16" указывает на преобразование в видео с соотношением сторон 9**:**16, используя высоту исходного видео). |
| FaceDetectConfig | Object | Конфигурации обнаружения лиц. |
| OutputPattern | String | Для настройки имени файла `timestamp` и `sessionId` служат доступными параметрами подстановки. Например, `"htv-{sessionId}-{timestamp}"` приведёт к имени выходного файла "htv-xxxx-202412250000", где "xxxx" представляет фактический sessionId задачи. Если не указано, имя выходного файла по умолчанию будет `"htv-{sessionId}"`. |
| BlurWeight | Integer | Параметр размытия: чем выше значение, тем более выраженный эффект размытия. **Чрезмерно высокие значения параметра могут замедлить скорость обработки.** |
| SmoothWeight | Float | Число с плавающей точкой в диапазоне от 0 до 1, которое контролирует параметр сглаживания скорости. Чем меньше значение, тем быстрее движение камеры. |

##### FaceDetectConfig

| Имя | Тип | Описание |
| --- | --- | --- |
| FaceScoreThd | Integer | Порог распознавания для алгоритма обнаружения лиц. Лицо считается допустимым только в том случае, если оценка распознавания превышает этот порог. |
| FaceAccuracy | String | Необязательное количество выполнений алгоритма обнаружения лиц, по умолчанию "Balance". Другие доступные варианты включают "Efficiency" и "Precision". |
| FallbackConfig | Object | Стратегия отката для обнаружения лиц включает сценарии с "NoFaceDetect" (лица не обнаружены) и "DoubleFace" (обнаружены два лица). |

##### FallbackConfig

| Имя | Тип | Описание |
| --- | --- | --- |
| NoFaceDetect | String | Варианты: "Scale", "ScaleWithoutBlur" (по умолчанию). |
| DoubleFace | String | Варианты: "Scale", "ScaleWithoutBlur", "SplitScreenVertical" (по умолчанию). |

- Scale: центрирование кадра путём масштабирования и замена фона изображением с эффектом матового стекла.
- ScaleWithoutBlur: центрирование кадра путём масштабирования и замена фона на чёрный цвет.
- SplitScreenVertical: логика обработки по умолчанию для двух лиц предполагает разделение экрана вертикально, с центрированием каждого лица в верхней и нижней областях соответственно.

### 4. Получение результатов задачи

- Callbacks задач: при инициировании задачи MPS с использованием [ProcessMedia](https://www.tencentcloud.com/document/product/1041/33640) вы можете установить информацию callback через параметр TaskNotifyConfig. После завершения задачи результаты задачи будут переданы через настроенную информацию callback. Вы можете анализировать результаты уведомлений о событиях через [ParseNotification](https://www.tencentcloud.com/document/product/1041/33679). Приведённые ниже [связанные структуры данных](#Data) предоставлены для справки.
- Запрос через API [DescribeTaskDetail](https://www.tencentcloud.com/document/product/1041/33644):
  - Для задач, запущенных с помощью API и шаблона, как описано в **Способе интеграции 1** выше, используйте `TaskId` из [ProcessMedia](https://www.tencentcloud.com/document/product/1041/33640) (например: 24000022-WorkflowTask-b20a8exxxxxxx1tt110253) для анализа `AiAnalysisResultSet` в `WorkflowTask`. Приведённые ниже [связанные структуры данных](#Data) предоставлены для справки.
  - Для задач, запущенных через [ProcessMedia](https://www.tencentcloud.com/document/product/1041/33640) без шаблона, но с `ScheduleId` (в последующем разделе об [автоматическом запуске задач](https://www.tencentcloud.com/document/product/1041/66045#13338cae-b5e9-416d-84d5-2117b39ef602) объясняется создание расписания), возвращённый `TaskId` будет содержать "ScheduleTask" (например 24000022-ScheduleTask-774f101xxxxxxx1tt110253). В этом сценарии используйте `TaskId` для анализа `ActivityResultSet` в `ScheduleTask`.
  - Для задач, инициированных с консоли, как описано в **Способе интеграции 2** ниже, перейдите на [Tasks -> VOD](https://console.tencentcloud.com/mps/tasks/vod-list) для ID задачи и результатов. Вы также можете анализировать `ActivityResultSet` в `ScheduleTask` в API [DescribeTaskDetail](https://www.tencentcloud.com/document/product/1041/33644) для получения результатов задачи.
- Запрос через консоль: войдите в консоль и перейдите на страницу [VOD Processing Tasks](https://console.tencentcloud.com/mps/tasks/vod-list), где новые инициированные задачи отображаются в списке задач.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/b63e2174a65b11ef9ec3525400f69702.png)

Когда статус подзадачи — "Successful", вы можете перейти на **COS Bucket > Output Bucket**, найти ваш выходной каталог и найти файлы, начинающиеся с htv- в каталоге, которые являются выходными видео после трансформации из горизонтального в вертикальное формат.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c7340c1ea65b11efba3e5254002693fd.png)

## Способ интеграции 2: инициирование задачи с консоли (без кода)

### 1. Создание задачи

  1.1. Войдите в [консоль MPS](https://console.tencentcloud.com/mps) и нажмите **Create Task > Create VOD Processing Task**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/f00042d4a65011efae7d5254002693fd.png)

  1.2. Укажите входной видеофайл. В настоящее время функция трансформации видео из горизонтального в вертикальное формат поддерживает два источника входных данных: [Tencent Cloud COS](https://www.tencentcloud.com/products/cos) и адреса URL для загрузки. AWS S3 в настоящее время не поддерживается.
  1.3. На этапе "Process Input File" добавьте узел **Intelligent Analysis**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/ffc9cd6da65011efae7d5254002693fd.png)

В открывшемся ящике параметров интеллектуального анализа выберите **предустановленный шаблон трансформации видео из горизонтального в вертикальное формат (ID шаблона: 28)**.

> Примечание: если требуется включить функцию трансформации видео из горизонтального в вертикальное формат для пользовательского шаблона интеллектуального анализа, вы можете связаться с нами и предоставить **ID шаблона**, и разработчики Tencent Cloud MPS настроят и включат функцию трансформации видео из горизонтального в вертикальное формат для вас.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/1a00bd8dad3711ef804852540055f650.png)

  1.4. После указания пути сохранения для выходного видео нажмите **Create** для инициирования задачи.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/1c5e41a9a65111ef888a52540075b605.png)

### **2. Получение результатов задачи**

Обратитесь к разделу [Получение результатов задачи](#result) выше.

### 3. Автоматическое запускание задачи (дополнительная возможность)

Если требуется автоматически выполнять трансформацию видео из горизонтального в вертикальное формат в соответствии с предустановленными параметрами после загрузки видеофайла в корзину COS, вы можете:

3.1 При создании задачи нажмите **Save The Orchestration** и настройте параметры, такие как Trigger Bucket и Trigger Directory в открывшемся окне.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/23d29e7ba65111efb7b7525400bdab9d.png)

3.2 Перейдите на список **VOD Orchestration**, найдите новую оркестровку и включите переключатель в **Enable**. Впоследствии любые новые видеофайлы, добавленные в каталог триггера, автоматически инициируют задачи в соответствии с предустановленным процессом и параметрами оркестровки, а обработанные видеофайлы будут сохранены в выходной путь, настроенный в оркестровке.

> **Примечание:** для вступления в силу оркестровки после включения требуется 3-5 минут.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/47a1b6d9ad3711ef804852540055f650.png)

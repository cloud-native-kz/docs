# Резервное копирование обратного вызова задач MPS в COS

Резервное копирование обратных вызовов задач MPS в COS через SCF является стандартной практикой. MPS содержит шаблон в SCF, который можно использовать для включения этой функции. MPS выполняет задачи обработки видео; SCF обрабатывает сообщения обратного вызова, а COS предоставляет постоянное хранилище.

## Инструкции

### Шаг 1. Создание функции

1. Войдите в [консоль SCF](https://console.tencentcloud.com/scf/list) и нажмите [**Function Service**](https://console.tencentcloud.com/scf/list) на левой боковой панели.
![](https://staticintl.cloudcachetci.com/cms/backend-cms/c733e6a2d6dc11eea2b0525400bb593a.png)
2. В верхней части страницы **Function Service** выберите **Beijing** и нажмите **Create**, чтобы перейти на страницу создания функции.
3. Установите следующие параметры:
  - **Create Method**: выберите **Template**.
  - **Fuzzy search**: выполните поиск **CLSSCFCOS**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c73361c2d6dc11ee9409525400c26da5.png)

> **Примечание:** Вы можете нажать **Learn More** в шаблоне, чтобы просмотреть его детали или загрузить шаблон.

4. Нажмите **Next**.
![](https://staticintl.cloudcachetci.com/cms/backend-cms/c74864c8d6dc11eea2b0525400bb593a.png)
5. Сохраните конфигурацию по умолчанию и нажмите **Complete**, чтобы завершить создание.

### Шаг 2. Конфигурирование триггера MPS

1. В [**консоли SCF**](https://console.tencentcloud.com/scf) нажмите [**Function Service**](https://console.tencentcloud.com/scf/list) на левой боковой панели и нажмите созданную функцию, чтобы перейти на страницу деталей.
2. Нажмите **Trigger Management** > **Create a Trigger**. Появится окно создания триггера. Выберите **MPS trigger** для метода триггера.

Основная информация о параметрах указана ниже. Для остальных элементов конфигурации используйте значения по умолчанию.

- **Event Type**: триггер MPS отправляет события на уровне учетной записи. В настоящее время поддерживаются два типа событий триггера: Workflow Task (`WorkflowTask`) и Video Edit Task (`EditMediaTask`).

> **Примечание:** При первом создании триггера MPS появится сообщение об ошибке роли сервиса. Нажмите **SCF_QcsRole** и **MPS_QcsRole**, чтобы предоставить необходимые разрешения в соответствии с подсказками. Триггер MPS использует события, созданные на уровне сервиса, в качестве источников событий, независимо от таких атрибутов, как регион и ресурсы. Каждый тип события можно привязать только к одной функции для каждой учетной записи. Если вам нужно, чтобы несколько функций обрабатывали задачу, см. [Node.js SDK](https://intl.cloud.tencent.com/document/product/583/32747).

3. Нажмите **Submit** для завершения конфигурации.

### Шаг 3. Тестирование функции

1. Запустите рабочий процесс обработки видео MPS в [**консоли MPS**](https://console.tencentcloud.com/mps).
2. Перейдите в [**консоль SCF**](https://console.tencentcloud.com/scf/list?rid=8&ns=default), чтобы просмотреть результат выполнения.
Выберите вкладку **Log Query** на странице деталей функции для просмотра информации о выводимом журнале.
3. Войдите в [консоль COS](https://console.tencentcloud.com/cos5), чтобы просмотреть результаты сохранения и обработки данных.

> **Примечание:** При необходимости вы можете написать свои собственные методы обработки данных.

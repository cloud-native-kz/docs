# Уведомление обратного вызова задачи MPS

После обработки видео MPS стандартной практикой является отправка уведомления о завершении задачи обработки видео. MPS имеет шаблон в SCF, который вы можете использовать для включения этой функции.

## Обзор

Пример в этом документе использует MPS и SCF. MPS выполняет задачи обработки видео, а SCF обрабатывает сообщения обратного вызова.

## Инструкции

### Шаг 1. Создание функции

1. Войдите в [консоль SCF](https://console.tencentcloud.com/scf/list) и нажмите [**Function Service**](https://console.tencentcloud.com/scf/list) на левой боковой панели.
2. В верхней части страницы **Function Service** выберите **Beijing** и нажмите **Create**, чтобы перейти на страницу создания функции.
3. Установите следующие параметры:
  - **Function name**: введите имя, например `MPSAnalysis`.
  - **Runtime**: шаблоны обратного вызова задач в настоящее время поддерживают только Nodejs 8.9.
  - **Create Method**: выберите **Function Template**.
  - **Fuzzy search**: введите "MPS Webhook template" и выполните поиск.

> **Примечание:** Нажмите **Learn More** в шаблоне, чтобы просмотреть детали в окне **Template Details**, которое можно загрузить.

4. Нажмите **Next**, чтобы перейти на страницу конфигурации функции.
5. Сохраните конфигурацию по умолчанию и нажмите **Finish**, чтобы завершить создание.

### Шаг 2. Настройка триггера MPS

1. В [консоли SCF](https://console.tencentcloud.com/scf) нажмите [**Function Service**](https://console.tencentcloud.com/scf/list) на левой боковой панели и нажмите созданную функцию, чтобы перейти на страницу сведений.
2. Нажмите **Trigger Management** > **Create a Trigger**. Откроется окно создания триггера. Выберите **MPS trigger** для метода триггера.

Информация об основных параметрах приведена ниже. Сохраните параметры по умолчанию для других параметров.

- **Event Type**: триггер MPS отправляет события на уровне учетной записи. В настоящее время поддерживаются два типа событий триггера: Workflow Task (`WorkflowTask`) и Video Edit Task (`EditMediaTask`).

```
>?>- При первом создании триггера MPS появится сообщение об исключении роли сервиса. Нажмите **SCF_QcsRole** и **MPS_QcsRole** согласно подсказкам, чтобы предоставить необходимые разрешения.>- Триггер MPS использует события, созданные на уровне сервиса, в качестве источников событий, независимо от таких атрибутов, как регион и ресурсы. Каждый тип события может быть привязан только к одной функции для каждой учетной записи. Если вам нужно несколько функций для обработки задачи, см. [Node.js SDK](https://intl.cloud.tencent.com/document/product/583/32747).
```

3. Нажмите **Submit**, чтобы завершить конфигурацию.

### Шаг 3. Тестирование функции

1. Войдите в [консоль MPS](https://intl.cloud.tencent.com/login) и начните рабочий процесс обработки видео.
2. Перейдите в [**консоль SCF**](https://console.tencentcloud.com/scf/list?rid=8&ns=default), чтобы просмотреть результат выполнения.
На странице сведений о функции откройте вкладку **Log Query**, чтобы просмотреть напечатанную информацию журнала.
3. Войдите в [консоль COS](https://console.tencentcloud.com/cos5), чтобы просмотреть результат выгрузки и обработки данных.

> **Примечание:** При необходимости вы можете написать свои собственные методы обработки данных.

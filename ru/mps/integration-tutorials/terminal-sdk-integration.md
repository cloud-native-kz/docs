# Интеграция Terminal SDK

Terminal SDK — это набор возможностей аудио- и видеотерминальных продуктов, запущенных Tencent Cloud. Он включает три типа SDK для видеокодирования, улучшения звука и улучшения видео. Адаптированный для удовлетворения разнообразных потребностей клиентов, он поддерживает доступ с нескольких терминалов, таких как мобильные устройства, веб и ПК.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/27a550c42ed011efa01d5254005235d8.png)

## Terminal Video Codec SDK

Кодировщик видео Tencent Top Speed Codec (TSC) для терминалов разработан для сценариев, требующих низкой вычислительной мощности, низкой задержки и высокого качества изображения на стороне терминала. По сравнению с аппаратным кодированием его преимущества включают:

1. Стабильная, надежная и быстрая загрузка.
2. При одинаковом уровне качества экономит битрейт, повышает стабильность передачи, снижает полосу пропускания распределения нисходящего канала и снижает затраты на хранение.
3. При одинаковом битрейте повышает качество изображения и улучшает пользовательский опыт.
4. Богатый набор функций удовлетворяет разнообразные деловые потребности, например использование кодирования областей интереса (ROI) для улучшения качества изображения в области лица и динамическая регулировка конфигурации кодирования для адаптации к колебаниям сети.

Для получения подробной информации см. [TSC Terminal Video Codec SDK](#294b81fb-0ba2-420c-99fc-40805708d0e2).

## Terminal Audio SDK

Клиентский аудио SDK предоставляет возможности кодирования и улучшения звука. Он достигает эффектов, включая адаптивное подавление шума, акустическое подавление эха и автоматический контроль усиления, значительно улучшая качество звука путем устранения эха и шума.

Для получения подробной информации см. [TSC Terminal Audio SDK](#47de165a-7753-4985-9415-539b9f772a74).

## Terminal Enhancement SDK

Клиентский SDK для улучшения, основанный на эффективных алгоритмах обработки изображений и возможностях вывода модели искусственного интеллекта, достигает сверхразрешения видео терминала, улучшения качества изображения, интерполяции кадров и других функций.

Для получения подробной информации см. [TSC Terminal Enhancement SDK](#84157eba-650d-42ea-a51e-4d6541bab97a).

### Интеграция SDK

Видео SDK кодека реализован на C/C++/Assembly, предоставляя унифицированный интерфейс C для различных платформ систем.

#### Android

● Предоставляет динамические библиотеки версий ARMv7 и ARMv8, приложение интегрируется через NDK.

● Предоставляет инкапсуляцию интерфейса Java. Интерфейс в основном согласуется с аппаратным кодированием Android MediaCodec, облегчая параллельную замену MediaCodec.

#### iOS

Предоставляет XCFramework версии ARMv8 и x86_64.

#### macOS

Предоставляет framework версии ARMv8 и x86_64.

#### Windows

Предоставляет динамические библиотеки версии x86 и x86_64.

### Аудио 3A для коммуникации в реальном времени

Технология Audio 3A является набором базовых функций в обработке звуковых сигналов, обычно используется в системах реального времени коммуникации, таких как видеоконференции, звонки и прямые микрофонные соединения, для обеспечения высокого качества передачи аудиосигнала и обеспечения лучшего качества связи и звукового опыта слушания. 3A обозначает адаптивное подавление шума (ANS), акустическое подавление эха (AEC) и автоматический контроль усиления (AGC).

![Канал коммуникации звука в реальном времени](https://staticintl.cloudcachetci.com/cms/backend-cms/4ebbddc72d4711efa01d5254005235d8.png)

- ANS

Основной функцией ANS является устранение компонентов фонового шума в голосовом сигнале, снижение помех и, следовательно, улучшение разборчивости речи и качества восприятия. На основе предположения аддитивной модели шума аудиосигнал, захватываемый микрофоном, можно рассматривать как суперпозицию чистого голосового сигнала и шумовых помех. Отслеживая и оценивая шум в немой части аудио, а затем вычитая энергию компонента шума в голосовых сегментах, можно получить более чистый голосовой сигнал.

- AEC

AEC в основном решает проблему эха в звуковой связи. Во время звонка звук, воспроизводимый динамиком, напрямую захватывается микрофоном или захватывается после отражения, что вызывает, что удаленный пользователь слышит свой собственный голос. Это может серьезно повлиять на качество звонка. Технология AEC может обрабатывать сигнал близкого конца на основе сигнала дальнего эталона, эффективно устраняя или снижая это явление эха, тем самым улучшая опыт звонка.

- AGC

AGC отвечает за регулировку громкости при передаче аудиосигналов. Когда громкость источника звука слишком низкая или слишком высокая, это может значительно повлиять на опыт звонка. AGC может автоматически обнаруживать громкость аудиопотока и динамически регулировать уровень громкости, чтобы он остался в комфортном диапазоне. AGC может облегчить нестабильность громкости, вызванную такими факторами, как различия в сборке устройства записи, громкости динамика и расстояния.

- Преимущество стандартного издания — производительность, наши алгоритмы достигают хороших эффектов сверхразрешения при минимальном времени и потреблении энергии. Он совместим с почти всеми мобильными телефонами различной производительности.

Профессиональное издание требует вычислительной мощности устройства и рекомендуется для использования на мобильных телефонах среднего и высокого уровня.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/3d8040652ed411ef97da5254007d9c55.png)

Например, в облачных игровых сценариях возможность сверхразрешения видео в реальном времени на терминале может снизить вычислительную мощность облачного рендеринга и кодирования, сэкономить полосу пропускания передачи и снизить затраты. В следующем примере сцена игры, передаваемая из облака с разрешением 720P (5,6 Mbps), увеличивается в реальном времени до 1080P на терминале. Эффект просмотра близок к сцене, передаваемой непосредственно с разрешением 1080P (8,2 Mbps) из облака, экономя 30% полосы пропускания.

### Интеграция SDK

#### Совместимость

- Платформа Android: применима к Android 5.0 и более поздним версиям (API 21, OpenGL ES 3.1).
- Платформа iOS: применима к устройствам iPhone 5s и более поздним версиям с минимальной версией системы iOS 12.

#### Размер пакета

- Стандартное издание: AAR для Android составляет примерно 0,3 МБ (arm64-v8a), Framework для iOS составляет примерно 0,4 МБ.
- Профессиональное издание: AAR для Android составляет примерно 2,1 МБ (единая архитектура arm64-v8a), Framework для iOS составляет примерно 1,9 МБ.

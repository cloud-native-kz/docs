# Функции

MPS транскодирует аудио- и видеофайлы на разные битрейты и разрешения для плавного воспроизведения на различных устройствах с разными вариантами полосы пропускания. Она имеет следующие функции:

## Трансcodирование аудио/видео

![](https://staticintl.cloudcachetci.com/cms/backend-cms/0b89b973da1f11ef840e52540044a08e.png)

Трансcodирование — это офлайновая задача, которая изменяет кодек, разрешение, битрейт и другие характеристики аудио- или видеопотока в соответствии с различными устройствами воспроизведения и сетевыми условиями. Преимущества трансcodирования включают:

| Функция | Описание |
| --- | --- |
| Повышенная совместимость | Исходное видео может быть перекодировано в форматы (такие как MP4), совместимые с большим количеством типов устройств для плавного воспроизведения. |
| Повышенная адаптивность к полосе пропускания | Исходное видео может быть перекодировано для вывода в несколько определений, такие как LD, SD, HD и UHD. Конечные пользователи могут выбрать наиболее подходящий битрейт в зависимости от условий сети. |
| Улучшенная эффективность воспроизведения | Атом moov может быть перемещен с конца файла MP4 в начало файла, что позволяет воспроизводить видео до полной загрузки. |
| Снижение потребления полосы пропускания | С помощью более продвинутого кодека (такого как H.265) битрейт видео может быть существенно снижен при сохранении исходного качества, что помогает снизить потребление полосы пропускания. |

Параметры, которые можно указать для трансcodирования, включают кодек, разрешение, битрейт и т.д. Подробнее см. таблицу ниже.

| Категория | Параметр | Описание |
| --- | --- | --- |
| Вход | Формат контейнера | 3GP, AVI, FLV, MP4, M3U8, MPG, ASF, WMV, MKV, MOV, TS, WebM, MXF. |
|  | Видеокодек | AV1, AVS2, H.264/AVC, H.264 intra, H.263, H.263+, H.265/HEVC, MV-HEVC, H.266/VVC, MPEG-1, MPEG-2, MPEG-4, MJPEG, VP8, VP9, RealVideo, Windows Media Video, WMV2, Quicktime. |
|  | Аудиокодек | AAC, ADPCM, AMR, DSD, MP1, MP2, MP3, PCM, RealAudio, Windows Media Audio, VORBIS, AC-3。 |
| Выход | Формат контейнера | Видео: FLV, MP4, HLS (M3U8 + TS), MXF. |
|  |  | Аудио: MP3, MP4, OGG, FLAC, M4A, PCM。 |
|  |  | Изображение: GIF, WebP. |
|  | Видеокодек | AV1, H.264/AVC, H.264 intra, H.265/HEVC, MV-HEVC, H.266/VVC, MPEG-2, MPEG-4, MJPEG, VP8, VP9, RealVideo, Windows Media Video, WMV2. |
|  | Аудиокодек | MP3, AAC, FLAC, MP2, Vorbis. |
| Упаковка | Удаление видеопотоков | Если это включено, результат трансcodирования будет содержать только аудиопотоки. |
|  | Удаление аудиопотоков | Если это включено, результат трансcodирования будет содержать только видеопотоки. |

## Водяные знаки

Водяной знак — это офлайновая задача, которая добавляет изображение в указанное положение видео во время трансcodирования видео или захвата экрана. MPS поддерживает следующие типы водяных знаков:

- **Статический водяной знак**: неанимированный водяной знак в формате PNG, который может быть логотипом правообладателя или телевизионной станции и обычно используется для заявления об авторских правах.
- **Анимированный водяной знак**: анимированный водяной знак в формате APNG

MPS позволяет добавлять несколько водяных знаков к видео или снимку экрана и указывать размер и позицию каждого водяного знака в видео или снимке экрана.

Параметры, которые можно указать для водяных знаков, включают тип водяного знака, соотношение сторон, позицию и т.д. Подробнее см. таблицу ниже.

| Параметр | Описание |
| --- | --- |
| Type | Тип водяного знака. Водяные знаки могут быть статическими или анимированными. |
| Position | Относительное положение водяного знака в видео. |
| ImageSize | Размер водяного знака в видео. |
| ImageContent | Двоичные данные водяного знака. |

## Захват скриншотов видео

Захват скриншотов — это офлайновая задача, которая захватывает скриншот видео в определенный момент времени. MPS предоставляет следующие типы скриншотов:

- **Скриншот в точке времени**: скриншоты, снятые в указанные точки времени
- **Выборочный скриншот**: скриншоты, снятые через равные интервалы
- **Спрайт изображений**: MPS может захватывать набор скриншотов видео (подизображений) в указанный интервал времени и объединять их вместе для создания большого изображения.

Параметры, которые можно указать для захвата скриншотов, включают формат скриншота, соотношение сторон и т.д. Подробнее см. таблицу ниже.

### Скриншоты в точках времени

| Параметр | Описание |
| --- | --- |
| Format | Формат скриншота (в настоящее время поддерживается только JPG) |
| Width | Ширина скриншота (px). Диапазон значений: 128-4096 |
| Height | Высота скриншота (px). Диапазон значений: 128-4096 |
| FillType | Режим заполнения ( `FillType`) указывает, как обрабатывается исходное видеоизображение, когда соотношение сторон не совпадает с указанным соотношением сторон скриншота. Поддерживаются следующие режимы заполнения:Масштабирование: исходные видеоизображения растягиваются в соответствии с соотношением сторон скриншотов. Это может привести к искажению изображений.Черные полосы: соотношение сторон исходных видеоизображений сохраняется, а пустые области окрашиваются в черный цвет.Белые полосы: соотношение сторон исходных видеоизображений сохраняется, а пустые области окрашиваются в белый цвет.Размытие Гаусса: соотношение сторон исходных видеоизображений сохраняется, и размытие Гаусса применяется к пустым областям. |

| FillType | Режим заполнения ( `FillType`) указывает, как обрабатывается исходное видеоизображение, когда соотношение сторон не совпадает с указанным соотношением сторон скриншота. Поддерживаются следующие режимы заполнения:Масштабирование: исходные видеоизображения растягиваются в соответствии с соотношением сторон скриншотов. Это может привести к искажению изображений.Черные полосы: соотношение сторон исходных видеоизображений сохраняется, а пустые области окрашиваются в черный цвет.Белые полосы: соотношение сторон исходных видеоизображений сохраняется, а пустые области окрашиваются в белый цвет.Размытие Гаусса: соотношение сторон исходных видеоизображений сохраняется, и размытие Гаусса применяется к пустым областям. |

### Спрайты изображений

| Параметр | Описание |
| --- | --- |
| Format | Формат спрайта изображения (в настоящее время поддерживается только JPG). |
| Width | Ширина подизображения в спрайте изображения. |
| Height | Высота подизображения в спрайте изображения. |
| Rows | Количество строк изображения в спрайте. |
| Columns | Количество столбцов изображения в спрайте. |
| SampleType | Способ измерения интервалов выборки. В настоящее время поддерживается только выборка по времени. |

> **Примечание:** Результат умножения `Width` x `Columns` (т.е. ширина спрайта) должен быть в диапазоне 128-4096. Результат умножения `Height` x `Rows` (т.е. высота спрайта) должен быть в диапазоне 128-4096.

### Анимированные скриншоты

Создание анимированного скриншота — это офлайновая задача, которая преобразует видеосегмент в анимированный скриншот, такой как формат GIF или WebP. Анимированный скриншот — это непрерывный цикл последовательных кадров, который может обеспечить эффект анимации с небольшим размером файла.

Параметры, которые можно установить для создания анимированного изображения, включают формат, ширину, высоту, частоту кадров и т.д. Подробнее см. таблицу ниже.

| Параметр | Описание |
| --- | --- |
| Format | Формат анимированного изображения (в настоящее время поддерживаются только GIF и WebP). |
| Width | Ширина анимированного изображения. Диапазон значений: 128–4096 px. |
| Height | Высота анимированного скриншота. Диапазон значений: 128–4096 px. |
| FPS | Частота кадров. Диапазон значений: 1–60 fps. |

## Media AI

Media AI предлагает возможности, такие как интеллектуальные субтитры, интеллектуальное стирание, резюме LLM и т.д. Он используется в трансляции, развлечениях, образовании для задач, таких как распространение фильмов, видеоконференции и потоковая трансляция, повышая эффективность производства контента.

### Интеллектуальные субтитры и переводы

Функция Smart Captions and Subtitles распознает голоса из видео или прямых трансляций в реальном времени, преобразует речь в субтитры и переводит их на различные языки, такие как китайский, английский и японско-корейский. Он полезен для прямых субтитров, видеотранскрипции для глобального распространения и извлечения субтитров из старых фильмов для восстановления. Функция поддерживает конфигурирование HotWord для более точного перевода. Дополнительную информацию см. в [Создание и перевод субтитров](https://www.tencentcloud.com/document/product/1041/54517).

##### Технические преимущества

- Комплексная поддержка платформ: способна обрабатывать файлы по требованию, прямые трансляции и интерактивное аудио-видео. Прямые трансляции с субтитрами в реальном времени поддерживают стабильный и градиентный режимы, с низким барьером для входа и без необходимости изменений на конце воспроизведения.
- Высокая точность: использует крупномасштабные модели и поддерживает библиотеки горячих слов и терминологии, лидируя в отрасли по точности.
- Богатая поддержка языков: предлагает поддержку сотен языков, включая различные диалекты и распознавание смешанного языка, такие как комбинации китайского и английского.
- Настраиваемые стили: позволяет встраивать субтитры непосредственно в видео с настраиваемыми стилями субтитров (шрифт, размер, цвет, фон, позиция и т.д.) и поддерживает пользовательскую визуализацию страницы.

Эта функция широко используется в различных областях, включая платформы коротких драм, платформы коротких видео, трансграничную электронную коммерцию и студии самостоятельных медиа и т.д. Это позволяет извлекать привлекательные сегменты из различных типов видео, таких как фильмы, спорт и игры, с опциями настройки. Дополнительную информацию см. в [Учебнике по интеллектуальным высоким точкам](https://www.tencentcloud.com/document/product/1041/66044).

##### Технологические преимущества

- Поддерживает как живой, так и записанный контент, создавая высокие точки из прошлых трансляций.
- Работает с фильмами, спортом, играми и новостями.
- Высокая настраиваемость для конкретных видеосценариев.

### Интеллектуальное разделение видео

Функция интеллектуального разделения видео разделяет видео на меньшие части, полезна для конференций и образовательных курсов. Например, она может разделить новостной репортаж на отдельные клипы, улучшив качество сегментации и экономя расходы. Дополнительную информацию см. в [Учебнике по разделению видео (длинные видео на короткие видео)](https://www.tencentcloud.com/document/product/1041/66044).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/02e23e7dda2311efaef452540099c741.png)

### Резюме LLM

Функция резюме LLM использует NLP для резюмирования содержимого из видео, таких как образовательные курсы и новости, на основе результатов ASR и OCR. Она предлагает быстрое понимание содержимого, поддерживает сегментацию офлайн и прямых видео и помогает пользователям легко находить видеосегменты. Работает с несколькими языками и как с офлайн, так и с прямыми видео. Дополнительную информацию см. в [Учебнике по резюме LLM](https://www.tencentcloud.com/document/product/1041/66042).

![](https://staticintl.cloudcachetci.com/cms/backend-cms/315c3de1da2111ef9e13525400454e06.png)

### Преобразование видео из горизонтального в вертикальное

Преобразование видео из горизонтальной в вертикальную ориентацию — это не просто вращение. Это включает идентификацию интересующих областей (ROI) и обрезку видео в пропорцию, подходящую для воспроизведения на мобильных устройствах. Этот процесс позволяет массовому производству коротких видео, преобразуя существующий горизонтальный видеоконтент в вертикальный формат. Дополнительную информацию см. в [Учебнике по преобразованию видео из горизонтального в вертикальное](https://www.tencentcloud.com/document/product/1041/66045).

##### Технические преимущества

- Поддержка сцен с несколькими людьми: в сценариях, где в видео появляются два человека, технология может автоматически разделить экран вертикально, отображая каждого человека отдельно в формате вертикального видео.
- Высокая настраиваемость: предлагает возможность адаптации параметров модели специально разработанных для ваших видеосценариев.

### Интеллектуальное распознавание ROI

Распознавание ROI (интересующий регион) может идентифицировать в реальном времени положения важных визуальных элементов в видео, таких как лица, персонажи игр или хосты потоков, и отправить эту информацию вместе с видео на устройство воспроизведения. Используя информацию ROI, плеер может делать такие вещи, как размытие фона в сцене и предотвращение закрытия экранных комментариев важными элементами видео. Дополнительную информацию см. в [Интеллектуальное распознавание ROI CSS](https://www.tencentcloud.com/document/product/267/60145?from_search=1)。

![](https://staticintl.cloudcachetci.com/cms/backend-cms/31214bfeda2111ef840e52540044a08e.png)

### Интеллектуальное распознавание

На основе работы исследовательских лабораторий Tencent, интеллектуальное распознавание распознает различные формы видеоконтента, такие как люди, речь, текст и теги кадров, и выполняет многомерный структурированный анализ.

| Тип распознавания | Описание |
| --- | --- |
| Распознавание лиц | Быстро распознает информацию о лицах в видео на основе глубокого обучения и находит кадры, в которых присутствует человек, а также положение лица человека. Вы можете использовать пользовательские библиотеки людей или вызвать публичные библиотеки людей с поддержкой видео AI для распознавания лиц. |
| Распознавание речи | Быстро распознает речь в видео и преобразует ее в текст на основе глубокого обучения. Вы можете указать пользовательские ключевые слова и найти точки времени в видео, в которых произносятся ключевые слова. |
| Распознавание текста | Распознает текст в видео, включая вертикально ориентированный текст, и автоматически извлекает ключевые слова из текста. |
| Распознавание тегов кадров | Использует глубокое обучение для автоматического распознавания тегов в видеокадрах, захватываемых на пользовательском интервале захвата кадров, и определения местоположения тегов в видео. Теги кадров разделены на девять категорий, таких как люди, пейзаж, искусственный объект, здание, растение, животное и пища, охватывая различные аспекты повседневной жизни. Вы можете использовать пользовательские теги на основе системы тегов. Она имеет возможности трансферного обучения, поэтому вы можете настроить классификаторы, просто предоставив исходные данные пользователя. Таким образом, она отвечает требованиям различных типов пользователей и делает систему тегов более гибкой. |

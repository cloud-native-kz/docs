# Пакеты предоплаченных ресурсов

## Обзор

Пакеты ресурсов MPS — это предоплаченные пакеты, специфичные для ресурсов, которые можно использовать для **вычитания соответствующих используемых вами ресурсов**.

## Примечания

- Пакет ресурсов становится действительным сразу после покупки и истекает через год. Истекшие пакеты ресурсов нельзя использовать для вычитания.
- Только учетные записи с ежедневной выставлением счетов могут покупать пакеты ресурсов. Если вы переключитесь на ежемесячное выставление счетов, ваши существующие пакеты ресурсов будут заморожены и больше не смогут использоваться.
- Используемые вами ресурсы каждый день будут вычитаться в первую очередь из ваших пакетов ресурсов. **Оставшееся использование ресурсов** будет выставляться по [тарифам с оплатой по мере использования](https://www.tencentcloud.com/document/product/1041/49204). Чтобы избежать приостановки сервиса из-за просроченных платежей, пожалуйста, обращайте внимание на оставшиеся минуты в ваших пакетах ресурсов и своевременно покупайте новые пакеты или пополняйте счет.
- Вы можете купить несколько пакетов ресурсов. Сначала будет использован пакет, срок действия которого истекает раньше всех.
- Каждый пакет ресурсов имеет свой собственный период действия. Если вы купите несколько пакетов, их периоды действия не будут складываться.
- Каждый пакет ресурсов MPS может вычитать только определенный тип ресурсов MPS.
- Пакеты ресурсов подлежат возврату в течение пяти дней с момента покупки (при условии выполнения определенных условий). Подробнее см. [Возврат средств](https://intl.cloud.tencent.com/document/product/1041/33481).
- Ваше использование каждый день выставляется на следующий день. Пакет ресурсов не может быть использован для вычитания ресурсов, выставленных в тот же день, когда был куплен пакет ресурсов (использование за предыдущий день).
- Для просмотра ваших пакетов ресурсов и их использования войдите в консоль MPS как администратор и перейдите в [Управление пакетами ресурсов](https://console.intl.cloud.tencent.com/mps/resources?tab=package).

## Пакет ресурсов общего транскодирования

Пакет ресурсов общего транскодирования может вычитать ваше использование функций **общего транскодирования, адаптивной потоковой передачи (подпотоки общего транскодирования) и редактирования видео** ([связанные документы API](https://www.tencentcloud.com/document/product/1041/37460)) MPS. Если вы купили несколько пакетов, сначала будет использован пакет, срок действия которого истекает раньше всех. Система будет рассчитывать ваше общее использование общего транскодирования и адаптивной потоковой передачи каждый день и будет вычитать его полностью или частично (в зависимости от оставшихся минут в ваших пакетах ресурсов) из применимых пакетов ресурсов. Использование, не вычтенное пакетами ресурсов, будет выставляться по [тарифам с оплатой по мере использования](https://www.tencentcloud.com/document/product/1041/49204).

**Правила вычитания**: одинаковые коэффициенты вычитания применяются к использованию транскодирования внутри и вне континентальной части Китая.

| H.264 | 4K (короткая сторона ≤ 2160 px) | 1 **:** 16 |

| H.265 | HD (короткая сторона ≤ 720 px) | 1 **:** 10 |
| H.265 | FHD (короткая сторона ≤ 1080 px) | 1 **:** 20 |
| H.265 | 2K (короткая сторона ≤ 1440 px) | 1 **:** 40 |
| H.265 | 4K (короткая сторона ≤ 2160 px) | 1 **:** 80 |

| MV-HEVC | 8K (короткая сторона ≤ 4320 px) | 1 **:** 880 |

### Примеры

- Предположим, что функция общего транскодирования использовалась для транскодирования видео продолжительностью одну минуту. Использовался кодек H.264, а разрешение выходного видео составляло 640 x 480 px. Из пакета ресурсов общего транскодирования будет вычтена одна минута.
- Предположим, что функция общего транскодирования использовалась для транскодирования видео продолжительностью одну минуту. Использовался кодек H.264, а разрешение выходного видео составляло 1280 x 720 px. Из пакета ресурсов общего транскодирования будут вычтены две минуты.
- Для адаптивной потоковой передачи вычитание для каждого выходного потока рассчитывается отдельно в соответствии с кодеком и разрешением выхода, а затем вычитания суммируются.

Чем выше разрешение выходного видео, тем быстрее будет использован ваш пакет ресурсов.

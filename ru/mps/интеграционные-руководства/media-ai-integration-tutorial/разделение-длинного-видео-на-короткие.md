# Руководство по разделению видео (длинные видео на короткие видео)

Функция интеллектуального разделения видео интегрирует передовые технологии, такие как понимание видео на основе больших моделей, автоматическое распознавание речи (ASR), извлечение текста, а также распознавание людей и объектов для разделения длинных видео и точной разметки точек разделения. Система выводит разделенные видеоклипы вместе с информацией, такой как эскизы, время начала и окончания, названия и краткие описания содержания для каждого клипа. Например, она может разделить полное видео новостного выпуска на несколько отдельных видео с новостными событиями, что значительно повышает качество разделения видео, связанного с новостями и спортивным контентом. Эта функция эффективно способствует вторичному созданию контента, одновременно значительно снижая затраты на рабочую силу и оборудование.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/514f6468cb3c11f0a4a55254001c06ec.png)

Интеллектуальное разделение видео поддерживает обработку как автономных видео, так и прямых трансляций. Подробнее см. [Обработка автономных видео](#videofile) и [Обработка прямых трансляций](#live) ниже.

## Обработка автономных видео

### Метод инициирования задачи 1: через консоль

1. Войдите в консоль [Службы обработки медиа (MPS)](https://console.tencentcloud.com/mps/index), а затем нажмите **Создать задачу > Создать задачу обработки автономного файла**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/25a5f7d2cc0b11f084a45254005ef0f7.png)

2. Последовательно выберите путь входного файла, конфигурацию обработки потока оркестрации и путь вывода. В конфигурации оркестрации выберите узел **Медиа AI - Интеллектуальный анализ**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/aa61f0d3cb3c11f09e745254007c27c5.png)

3. На странице параметров интеллектуального анализа, которая появляется справа, выберите **Предустановленный шаблон интеллектуального разделения видео (ID: 27)**. Вы можете включить **Расширенные параметры** в разделе Дополнительные параметры и импортировать [расширенные параметры](#extendedpara) в соответствии с этим документом, чтобы указать сценарий разделения видео и добиться лучшего эффекта разделения. Чтобы обеспечить эффект обработки, рекомендуется [связаться с нами](https://www.tencentcloud.com/document/product/1041/33503) для автономного подтверждения конкретных конфигураций.

> **Примечание:** Консоль MPS автоматически экранирует данные JSON. Не вводите экранированные строки. В противном случае задача не будет выполнена. Если расширенные параметры не заполнены, значение по умолчанию — **Сценарий разделения видео новостей**.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/c345e7d6cb3c11f09e745254007c27c5.png)

4. Наконец, нажмите **Создать**, чтобы инициировать задачу.

### Метод инициирования задачи 2: Автоматическое срабатывание

Если вы хотите загрузить видеофайл в корзину Cloud Object Storage (COS) и добиться автоматической обработки видео с использованием функции интеллектуального разделения видео в соответствии с предустановленными параметрами, вы можете нажать **Сохранить эту оркестрацию** при создании задачи и настроить параметры, такие как корзина срабатывания и каталог в появившемся окне.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/6d2f7255cc0b11f096d1525400e889b2.png)

Затем перейдите в список **Автономной оркестрации**, найдите вновь созданную оркестрацию и включите переключатель в столбце **Включение**. Любые видеофайлы, добавленные в отслеживаемый каталог впоследствии, автоматически инициируют задачи в соответствии с предустановленным процессом и параметрами оркестрации, а обработанные видеофайлы будут сохранены в путь вывода, установленный в конфигурации оркестрации.

> **Примечание:** Оркестрации требуется 3–5 минут, чтобы вступить в силу после включения.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/7fe90998cc0b11f084a45254005ef0f7.png)

### **Описание расширенного параметра (используется для указанных сценариев разделения видео)**

Введите расширенные параметры, чтобы указать различные сценарии разделения видео и получить лучшие результаты.

#### Сценарий 1: Разделение видео на основе LLM

##### Описание функции

После распознавания содержимого речи и текста на экране видео извлекается текст и видео разделяется на основе LLM. Выходное содержимое включает разделенные видеоклипы вместе с информацией, такой как эскизы, время начала и окончания, названия и краткие описания для каждого клипа.

##### Параметры

Заполните следующие параметры в поле ExtendedParameter. Рекомендуется подтвердить конкретные детали параметров автономно.

```
{    "des": {        "split": {            "method": "llm",            "model": "deepseek-v3",            "max_split_time_sec": 100,            "extend_prompt": "This video is for a medical scenario and needs to be segmented according to the medical knowledge points."        },        "need_ocr": true,        "text_requirement": "The summary cannot exceed 40 characters.",        "dstlang": "zh"    },    "strip": {        "type": "content"    }}
```

Дополнительные параметры части "des" см. в таблице ниже.

| Параметр | Требуется | Тип | Описание |
| --- | --- | --- | --- |
| split.method | Нет | string | Метод сегментации видео. llm указывает сегментацию на основе LLM (Large Language Model); nlp указывает традиционную сегментацию на основе NLP. Значение по умолчанию — llm. |
| split.model | Нет | string | LLM сегментации. Допустимые значения: hunyuan; deepseek-v3; deepseek-r1. Значение по умолчанию — deepseek-v3. |
| split.max_split_time_sec | Нет | int | Принудительно указывает максимальное время для сегментации в единицах секунд. Рекомендуется использовать этот параметр при необходимости, так как это может повлиять на эффект сегментации. Значение по умолчанию — 3600. |
| split.extend_prompt | Нет | string | Добавляет подсказки для задачи разделения видео на основе LLM, такие как «это видео предназначено для онлайн-образования и должно быть разделено в соответствии с соответствующими точками знаний». Рекомендуется сначала оставить этот параметр неуказанным для тестирования, а затем добавить подсказки, если результаты не соответствуют ожиданиям. |
| need_ocr | Нет |  bool | Использовать ли оптическое распознавание символов (OCR) для помощи в сегментации. true указывает на включение. Значение по умолчанию — false. Когда это не включено, система распознает только содержимое речи видео для помощи в сегментации. Если включено, система также распознает текстовое содержимое в видеоизображениях для помощи в сегментации. |
| text_requirement | Нет | string | Добавляет подсказки для задачи суммирования на основе LLM. Например, краткое описание не может превышать 40 символов. |
| dstlang | Нет | string | Язык видео. Этот параметр используется для ASR и указания языка результатов, связанных с суммированием. Значение по умолчанию — "zh". "zh": китайский. "en": английский. |

#### Сценарий 4: Разделение видео на основе целевого объекта

##### Описание функции


## Запрос результата задачи

Задача интеллектуального разделения видео выведет информацию, такую как обработанные разделенные видеофайлы и эскизы, которые сохраняются в путь вывода, настроенный для задачи.

#### Запрос результата в консоли

1. Вы можете проверить статус задачи на странице [управления задачами](https://console.tencentcloud.com/mps/tasks/vod-list) в консоли. Когда статус подзадачи — «успешно».

![](https://staticintl.cloudcachetci.com/cms/backend-cms/fe1fab1ecc0b11f0afdc52540044a08e.png)

2. Нажмите **Callback JSON**, и вы найдете выходной результат и путь выходного файла в выходном сообщении:

![](https://staticintl.cloudcachetci.com/cms/backend-cms/04eb0860cc0c11f091ab5254007c27c5.png)

3. Если вы используете COS в качестве пути вывода, вы можете войти в консоль MPS, перейти на [страницу корзины COS](https://console.tencentcloud.com/mps/workflows/buckets) и нажать вкладку **Выходная корзина**, чтобы найти ваш выходной каталог. Файлы, начинающиеся с `strip-` в каталоге, являются выходными файлами (разделенные видео и эскизы) интеллектуального разделения видео.

> **Примечание:** Текстовое содержимое, такое как названия и краткие описания, не выводится в корзину. Оно должно быть запрошено через обратный вызов события или API.

![](https://staticintl.cloudcachetci.com/cms/backend-cms/208d70d5cc0c11f084a45254005ef0f7.png)

#### Обратный вызов уведомления о событии

Когда вы используете [ProcessMedia](https://www.tencentcloud.com/document/product/1041/33640) для инициирования задачи MPS, вы можете настроить обратный вызов события через параметр `TaskNotifyConfig`. После завершения обработки задачи результат задачи будет вызван через настроенную информацию обратного вызова. Вы можете проанализировать результат уведомления о событии через [ParseNotification](https://www.tencentcloud.com/document/product/1041/33679).

#### Запрос результата задачи путем вызова API

Когда вы используете [ProcessMedia](https://www.tencentcloud.com/document/product/1041/33640) для инициирования задачи MPS, будет возвращен ID задачи (`TaskId`), такой как 24000022-WorkflowTask-b20a8exxxxxxx1tt110253 и 24000022-ScheduleTask-774f101xxxxxxx1tt110253. Вызовите API [DescribeTaskDetail](https://www.tencentcloud.com/document/product/1041/33644) и введите ID задачи, чтобы получить результат задачи. Вам нужно проанализировать поле `WorkflowTask ->AiAnalysisResultSet`, чтобы получить результат задачи.

**ExtendedParameter** используется для указания сценария разделения видео и достижения лучших результатов. Значения параметров см. в разделе [Описание расширенного параметра](#extendedpara) выше. Пример входных параметров в формате JSON:

> **Примечание:** Прямая трансляция в настоящее время поддерживает сценарии разделения новостного видео и разделения видео на основе NLP. Сценарий разделения на основе целевого объекта в настоящее время не поддерживается.

## Запрос результата задачи

Получение обратного вызова задачи: При использовании [ProcessLiveStream](https://www.tencentcloud.com/document/product/1041/33641) для инициирования задачи MPS вы можете настроить информацию обратного вызова через параметр `TaskNotifyConfig`. Во время обработки прямых трансляций результаты задач сообщаются в режиме реального времени через настроенную информацию обратного вызова. Вы можете посмотреть [Результаты обработки анализа прямых трансляций](https://www.tencentcloud.com/document/product/1041/33680), чтобы проанализировать поле `AiAnalysisResultInfo` и получить результат задачи.

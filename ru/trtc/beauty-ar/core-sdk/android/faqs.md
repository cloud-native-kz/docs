### При сборке Android приложения для выпуска произошла ошибка, что некоторые методы не найдены. Что мне делать?

- Чтобы исправить проблему, добавьте следующие правила `keep` для предотвращения удаления кода:

```
-keep class com.tencent.xmagic.** { *;}-keep class org.light.** { *;}-keep class org.libpag.** { *;}-keep class org.extra.** { *;}-keep class com.gyailib.**{ *;}-keep class com.tencent.cloud.iai.lib.** { *;}-keep class com.tencent.beacon.** { *;}-keep class com.tencent.qimei.** { *;}-keep class androidx.exifinterface.** { *;}
```

### Что мне делать, если при интеграции SDK в хост-проект Android возникает ошибка конфликта библиотеки GSON?

```
Android{  configurations {    all*.exclude group: 'com.google.code.gson'  }}
```

### Что мне делать, если SO библиотеки не загружаются, когда `Android targetSdkVersion` равен 31 или больше?

```
  <uses-native-library            android:name="libOpenCL.so"            android:required="true" />        //`true` указывает, что библиотека необходима для работы приложения. Приложение не может быть установлено в системе без библиотеки.        //`false` указывает, что приложение может использовать библиотеку, если она доступна, но может работать и без неё. Её можно установить в системе без библиотеки. Если вы установите параметр `false`, вам потребуется обработать проблемы, которые могут возникнуть из-за отсутствия библиотеки.        //Введение веб-сайта Android: %!s(<nil>)
```

Дополнительную информацию см. в разделе [<uses-native-library>](https://developer.android.google.cn/guide/topics/manifest/uses-native-library-element?hl=zh-cn).

### Если текстура, переданная при использовании фильтров красоты, является горизонтальной текстурой, как это решить?

Вы можете использовать метод 'convert' в вспомогательном классе TextureConverter.java в демонстрации для поворота текстуры и преобразования её в портретный режим, а затем передать её в SDK красоты.

```
    /**     * Этот метод используется для поворота и зеркалирования RGBA текстуры. Процесс заключается в первом повороте текстуры по часовой стрелке на 'rotation' градусов (которые могут принимать значения 0, 90, 180, 270), а затем горизонтальное отражение (flipHorizontal) и вертикальное отражение (flipVertical)     * Вариант использования: некоторые потоковые SDK возвращают горизонтальные текстуры или персонажи на изображении повёрнуты в неправильном направлении, в то время как SDK требует, чтобы персонажи на текстуре были повёрнуты вперёд. Поэтому этот метод можно использовать для преобразования текстуры.     *      * @param srcID: RGBA текстура.     * @param width: ширина текстуры.     * @param height: высота текстуры.     * @param rotation: угол поворота.     * @Returns: повёрнутая текстура. Примечание: при повороте на 90 или 270 градусов ширину необходимо поменять местами.     */    public int convert(int srcID, int width, int height, @RotationDegreesValue int rotation, boolean flipVertical, boolean flipHorizontal)
```

### Если текстура, переданная при использовании фильтров красоты, является OES текстурой, как это решить?

Вы можете использовать метод 'oes2Rgba' в вспомогательном классе TextureConverter.java в демонстрации для преобразования текстуры в RGBA текстуру, а затем передать её в SDK красоты.

```
   /**     * Этот метод используется для преобразования OES текстуры в RGBA текстуру     *     * @param srcID: OES текстура.     * @param width: ширина текстуры.     * @param height: высота текстуры.     * @Returns: идентификатор RGBA текстуры.     */    public int oes2Rgba(int srcID, int width, int height)
```

### Что если я хочу использовать другую версию PAG? Поддерживается с V3.5.0

**Когда клиент интегрирует SDK фильтра красоты:**

Если интегрировано через Maven, sdk можно подключить через implement. Если вы не хотите использовать эту версию pag, её можно исключить следующим образом:

```
implementation ('com.tencent.mediacloud:TencentEffect_S1-04:version'){               exclude group: "com.tencent.tav", module: "libpag"}
```

Если SDK красоты интегрирован вручную путём загрузки, зависимость на aar не содержит pag. вам всё равно потребуется добавить строку в build.gradle вашего приложения для подключения pag для использования.

```
implementation 'com.tencent.tav:libpag:4.3.33-noffavc'
```

Для динамической загрузки библиотек .so вам необходимо получить их со страницы официальной загрузки и интегрировать только `.aar`.

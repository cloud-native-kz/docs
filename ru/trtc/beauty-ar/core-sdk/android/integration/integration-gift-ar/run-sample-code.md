# Запуск примера кода

Tencent Gift AR предоставляет демонстрационные проекты, подходящие для различных платформ разработки. В этой статье описывается, как быстро запустить демонстрационный проект Android версии Gift AR SDK.

## Загрузка

[Нажмите для загрузки](https://mediacloud-76607.gzc.vod.tencent-cloud.com/MediaX/download/demo/TCEffectPlayerDemo_Android.zip) демонстрационного проекта Gift AR для Android.

## Требования к окружению

- Android Studio 2.0 или выше.
- Android SDK API Level 19 или выше.
- Android 4.4 или выше, мобильные устройства с поддержкой архитектур armeabi-v7a, arm-v8a.

## Импорт проекта

Сначала откройте Android Studio, выберите Open an Existing Project, затем найдите каталог загруженного и распакованного демонстрационного проекта, выберите TCEffectPlayerDemo, нажмите **Open** и завершите импорт проекта.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/205b338004a311f0ae895254005ef0f7.png)

## Изменение зависимостей и конфигураций

Дождитесь завершения импорта проекта в Android Studio. Затем необходимо внести некоторые изменения в демонстрационный проект для обеспечения его последующей нормальной работы:

1. Найдите каталог app/libs, удалите старый файл AAR примера SDK и замените его на последний файл AAR Gift AR SDK:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/205d7c8c04a311f0aa8e525400bf7822.png)

2. Замените информацию о лицензии в проекте и замените значение на информацию о лицензии Gift AR SDK, которую вы получили:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/20b6ec2404a311f08c4452540044a08e.png)

3. Измените ID приложения проекта, то есть замените значение applicationId на рисунке ниже на имя пакета Android, соответствующее лицензии Gift AR SDK, которую вы получили:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/20abe8ff04a311f08bc5525400454e06.png)

4. После завершения указанных выше операций модификации рекомендуется нажать Build/Clean Project, чтобы очистить файлы кэша старого AAR в предыдущем проекте.

## Конфигурация ресурсов анимации

> **Примечание:** Если вам не нужно воспроизводить свои собственные файлы анимации, вы можете пропустить этот шаг.

Измените файл анимации для воспроизведения. Вы можете добавить файл анимации для опыта воспроизведения в app/main/assets. Затем откройте файл MainActivity.java и измените свойство FILE_NAME_TEP (большая анимация) или FILE_NAME_TEPGS (анимация маленького аватара) в верхней части. Демонстрационный проект автоматически завершит копирование файла анимации и другие работы. Затем вы можете напрямую запустить демонстрационный проект, чтобы испытать воспроизведение анимации.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/217c43e5519211f0a63e525400e889b2.png)

> **Примечание:** Файлы анимации, встроенные в демонстрационный проект, не привязаны к лицензии. Поэтому вы можете использовать свою собственную лицензию для опыта воспроизведения. Если вы хотите заменить свою собственную анимацию, обратите внимание, что информация о лицензии анимации должна быть согласована с информацией о лицензии, установленной в проекте. В противном случае воспроизведение не удастся из-за несовпадения информации о лицензии.

## Компиляция и запуск

1. Убедитесь, что тестовое устройство уже подключено к Android Studio.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/20930d3104a311f08016525400e889b2.png)

2. Нажмите Run для развертывания демонстрационного проекта на реальное устройство для опыта.
3. После запуска демонстрационного проекта на реальном устройстве результат следующий:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2061c2ca04a311f0aa645254007c27c5.png)

На этом этапе консоль сначала выведет:

```
TCMediaX license result: errCode: 0, msg: Success
```

Эта информация в журнале указывает на то, что проверка подлинности лицензии SDK уже прошла успешно.

Затем вы можете нажать кнопку **MP4** или **TCMP4** внизу, чтобы начать опыт воспроизведения анимации.

> **Примечание:** Gift AR SDK требует сеть при выполнении первой проверки подлинности лицензии. Поэтому при запуске демонстрационного проекта при запуске требуется сеть.

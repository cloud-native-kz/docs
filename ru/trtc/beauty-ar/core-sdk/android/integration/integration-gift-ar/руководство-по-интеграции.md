# Руководство по интеграции

Данный документ описывает, как быстро интегрировать Tencent Gift AR SDK в ваш проект. Следуя приведенным ниже шагам, вы сможете завершить работу по интеграции SDK.

## Требования к среде разработки

- Android Studio 2.0+.
- Android 4.4 (SDK API 19) или выше.

## Интеграция SDK

### Ручная интеграция

1. Загрузите [MediaX_Android_SDK_Latest.zip](https://mediacloud-76607.gzc.vod.tencent-cloud.com/MediaX/download/latest/MediaX_Android_SDK_Latest.zip). После завершения загрузки распакуйте архив, чтобы получить файл aar SDK.
2. Распакуйте загруженный файл. Затем скопируйте файл aar из директории SDK в директорию app/libs вашего проекта:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/9f400390095911f0b3015254001c06ec.png)

3. В файле build.gradle в корневой директории проекта добавьте flatDir и укажите путь локального репозитория.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/9f5d1827095911f0b3015254001c06ec.png)

4. Добавьте зависимость Gift AR SDK. В app/build.gradle добавьте код для подключения пакета aar.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/9f5e7f0a095911f0a6d15254007c27c5.png)

```
implementation(name: "TCEffectPlayer_x.x.x", ext: "aar")implementation(name: "TCMediaX_x.x.x", ext: "aar")implementation(name: "xmagic-auth_x.x.x", ext: "aar")
```

> **Примечание:** Как показано в примере выше, x.x.x — это номер версии, который должен совпадать с номером версии последней загруженной версии SDK; кроме того, обратите внимание, что номера версий трех файлов aar должны быть согласованы.

5. В разделе defaultConfig файла app/build.gradle укажите архитектуру CPU, используемую приложением (в настоящее время поддерживаются armeabi-v7a и arm64-v8a).

```
defaultConfig { ndk {     abiFilters "armeabi-v7a", "arm64-v8a" }}
```

### Интеграция через Gradle

1. Откройте соответствующий файл build.gradle и добавьте зависимость Gift AR SDK в тег dependencies:

Получите версию Gift AR SDK, которая была выпущена. Пожалуйста, посетите [MavenCentral](https://central.sonatype.com/artifact/com.tencent.mediacloud/TCEffectPlayer).

```
 dependencies {    // Не рекомендуется интегрировать последнюю версию. Рекомендуется интегрировать конкретный номер версии.    //implementation "com.tencent.mediacloud:TCEffectPlayer:latest.release"    implementation "com.tencent.mediacloud:TCEffectPlayer:version number"}
```

2. В разделе defaultConfig файла app/build.gradle укажите архитектуру CPU, используемую приложением (в настоящее время поддерживаются armeabi-v7a и arm64-v8a).

```
defaultConfig { ndk {     abiFilters "armeabi-v7a", "arm64-v8a" }}
```

Наконец, нажмите **Sync Now**, чтобы синхронизировать SDK и завершить работу по интеграции Gift AR SDK.

### Настройка правил обфускации

В файле proguard-rules.pro добавьте соответствующие классы в список необфусцируемых:

```
-keep class com.tcmediax.** { *; }
```

### Руководство по интеграции поддержки программного декодирования

#### Преимущества

Из-за серьезной фрагментации системы Android на очень небольшом количестве мобильных телефонов могут возникнуть неизбежные проблемы отказа аппаратного декодирования. Поэтому необходимо использовать возможность программного декодирования для повышения успешности воспроизведения (успешность можно повысить до 99,99%). Поэтому настоятельно рекомендуется одновременно интегрировать библиотеку программного декодирования для достижения лучшего опыта воспроизведения спецэффектов.

#### Этапы интеграции

##### **Интеграция Tencent Cloud RT Cube SDK (LiteAVSDK_TRTC)**

Если вы ранее интегрировали Tencent Cloud RT Cube SDK, такой как LiteAVSDK_TRTC и LiteAVSDK_Live, в ваш проект, вам необходимо интегрировать упрощенный SDK Tencent Cloud Player (LiteAVSDK_Player_Mini). Поместите соответствующий файл LiteAVSDK_Player_Mini.aar в директорию libs проекта, а затем добавьте соответствующую зависимость в dependencies.

```
dependencies {   implementation (name:'LiteAVSDK_Player_Mini', ext:'aar')}
```

##### Интеграция связанных SDK LiteAVSDK_Professional

Если вы ранее интегрировали Tencent Cloud RT Cube SDK, такой как LiteAVSDK_Player, LiteAVSDK_Player_Premium, LiteAVSDK_Professional и LiteAVSDK_UGC, в ваш проект, поддержка программного декодирования будет автоматически доступна без дополнительной настройки.

> **Примечание:** Если Tencent Cloud RT-Cube SDK уже интегрирован в ваш проект, проверьте, содержит ли он SDK плеера, и убедитесь, что лицензия плеера была запрошена и настроена.

##### Без интеграции связанных SDK Tencent Cloud

Если вы ранее не интегрировали Tencent Cloud RT Cube SDK в ваш проект, рекомендуется интегрировать упрощенный SDK Tencent Cloud Player (LiteAVSDK_Player_Mini). Поместите соответствующий файл LiteAVSDK_Player_Mini.aar в директорию libs проекта, а затем добавьте соответствующую зависимость в dependencies:

```
dependencies {   implementation (name:'LiteAVSDK_Player_Mini', ext:'aar')}
```

На последнем этапе необходимо настроить файл so: Загрузите файл so по ссылке [jniLibs_txffmpeg_so.zip](https://mediacloud-76607.gzc.vod.tencent-cloud.com/MediaX/Res/jniLibs_txffmpeg_so.zip) локально, затем распакуйте его. Скопируйте файл so из распакованной директории jniLibs в директорию app/src/main и настройте соответствующий путь в app/build.gradle:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/9f4fc8ef095911f0b3015254001c06ec.png)

```
sourceSets {    main {        jniLibs.srcDirs = ['src/main/jniLibs']    }}
```

## Часто задаваемые вопросы

### Как разрешить конфликты Xmagic?

Если вы уже интегрировали Beauty AR SDK в ваш проект, а поскольку как Beauty AR SDK, так и Gift AR SDK используют библиотеку авторизации лицензии xmagic-auth, при интеграции возникнут конфликты классов (например: Duplicate class com.tencent.xmagic.auth.Auth) и конфликты файлов so (например: конфликт libYTCommonXMagic.so). Способы решения следующие:

1. **Локальная интеграция AAR**

Если вы интегрировали его локально через AAR, при интеграции Gift AR SDK на этом этапе нет необходимости дополнительно интегрировать xmagic-auth.aar в проект. Удалите введение зависимости: xmagic-auth-x.x.x.aar и используйте одну копию библиотеки авторизации лицензии Beauty AR SDK в проекте.

2. **Интеграция через Gradle**

Если вы интегрируете через Gradle, обратитесь к следующему коду для удаления библиотеки авторизации лицензии:

```
implementation("com.tencent.mediacloud:TCEffectPlayer:version number") {	exclude group: "com.tencent.mediacloud", module: "TCXMagicAuth"}
```

### Как обработать конфигурацию зашифрованного видео?

Если при преобразовании анимации с помощью инструмента TepTools вы отметили переключатель **Anim Encrypt**, преобразованная анимация будет зашифрована. В этом случае при воспроизведении с использованием плеера эффектов, если targetSDKVersion вашего приложения больше или равен 28, вам необходимо произвести следующую отдельную настройку:

1. Создайте файл res/xml/network_security_config.xml в проекте и установите конфигурацию сетевой безопасности:

```
  <?xml version="1.0" encoding="utf-8"?> <network-security-config>    <domain-config cleartextTrafficPermitted="true">        <domain includeSubdomains="true">localhost</domain>        <domain includeSubdomains="true">127.0.0.1</domain>    </domain-config> </network-security-config>
```

2. Добавьте следующие свойства в тег application в файле AndroidManifest.xml:

```
  <?xml version="1.0" encoding="utf-8"?> <manifest ... >    <application android:networkSecurityConfig="@xml/network_security_config"                   ... >     ...    </application></manifest>
```

### Ошибка отсутствия класса при упаковке

Если SDK введен и при упаковке возникает следующая ошибка:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/6f7f79aca5c811f0b1565254001c06ec.png)

Добавьте следующую конфигурацию в файл proguard-rules.pro проекта:

```
-dontwarn com.tencent.rtmp.ITXVodPlayListener$ITXVodAudioFrameDataListener-dontwarn com.tencent.rtmp.TXVodDef$TXVodAudioFrameData-dontwarn com.tencent.xmagic.XmagicApi$XmagicLightGameListener-dontwarn com.tencent.xmagic.XmagicApi
```

# Инструмент конвертации Gift AR

Этот документ в основном представляет Tencent Gift AR и методы использования соответствующего инструмента конвертации Gift AR. Прочитав этот документ, вы узнаете, как использовать этот инструмент.

## Введение

Инструмент конвертации Gift AR (TEP Tools) — это инструмент для создания и преобразования материалов, созданный специально для Gift AR. Он поддерживает преобразование последовательностей кадров PNG, SVGA, WebP анимации, файлов PAG анимации и Lottie в форматы .mp4 или .tcmp4, поддерживаемые проигрывателем эффектов.

> **Примечание:** Инструмент конвертации Gift AR в настоящее время поддерживает только системы MacOS 14 и выше, а также архитектуры x86 и arm64. Пожалуйста, используйте соответствующую систему для дальнейшей установки и использования.

## Загрузка

Вы можете загрузить инструмент конвертации Gift AR в вашу локальную систему с [TEP Tools](https://mediacloud-76607.gzc.vod.tencent-cloud.com/MediaX/TEP/download/TEP_Tool_Latest.zip). После завершения загрузки:

1. Распакуйте пакет TEP_Tool_Latest.zip, чтобы получить инструмент конвертации Gift AR. После распаковки, как показано на рисунке ниже:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/33154cf2d15a11ef82a5525400e889b2.png)

2. Если процессор вашего Mac имеет архитектуру Intel, дважды щелкните `x86_app/teptool` для открытия; если процессор вашего Mac имеет архитектуру Apple Silicon, дважды щелкните `arm64_app/teptool` для открытия.

Вы можете проверить архитектуру процессора вашего Mac, запустив `uname -m` в терминале.

3. При первом запуске требуется авторизация компьютера (появится всплывающее сообщение о том, что файл не может быть открыт). Шаги авторизации: System Preferences > Security & Privacy > вкладка General > нажмите "Open Anyway".
4. После авторизации повторно откройте его для нормальной работы.

> **Примечание:** При открытии teptool появляется сообщение: "teptool" повреждён и не может быть открыт. Его следует переместить в корзину. В этом случае выполните файл init.sh в папке после распаковки. Когда в консоли будет выведено: `Init executed successfully!`, повторно откройте его для нормальной работы.

## Основной интерфейс инструмента

Основной интерфейс можно разделить на область операций функций и область вывода журналов, как показано ниже:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/3084dabdfe2611efbf88525400e889b2.png)

Перед использованием проигрывателя эффектов вы получите выданную лицензию проигрывателя эффектов от Tencent Cloud. Поэтому на данный момент вы можете заполнить полученные appId и информацию LicenseKey в соответствующие столбцы в инструменте. Эта информация будет использоваться для шифрования и создания файла эффектов, повышая безопасность файла эффектов.

> **Примечание:** Если у вас несколько packageNames или bundleIds, пожалуйста, объедините их английскими запятыми, например: `com.packagName1,com.packagName2,com.packagName3`.

## Описание элементов базовой конфигурации

- **fps:** Определите частоту кадров для воспроизведения материала.
- **alpha scale:** Следует ли масштабировать область альфа-канала видео (по умолчанию масштабирование составляет 0,5). В настоящее время доступны варианты: масштабирование 0,5; без масштабирования 1. Масштабирование видео может в конечном итоге уменьшить разрешение видео и улучшить совместимость.
- **resource:** формат исходного файла.

### Анимация слияния

нажмите **add**, чтобы добавить информацию маски анимации слияния. Значения конфигураций следующие:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2cdd063efe2711ef8c825254001c06ec.png)

- id: Флаг анимации слияния, эквивалентен целочисленному флагу текущего ресурса. При воспроизведении анимации слияния в зависимости от разных значений id возвращаются разные растровые или текстовые изображения для отображения. Это значение не поддерживает настройку и автоматически увеличивается.
- source tag: Флаг анимации слияния, эквивалент строковому флагу текущего ресурса. При воспроизведении анимации слияния в зависимости от разных тегов возвращаются разные растровые изображения для отображения.
- source type: Относится к типу атрибута анимации слияния. В настоящее время поддерживаются два метода. Image указывает на анимацию слияния типа изображения; text указывает на анимацию слияния типа текста.
- fit type: Способ отображения изображения. В настоящее время поддерживаются два метода: fitXY (мозаичное изображение, по умолчанию) и centerCrop (пропорциональное масштабирование). Эти концепции аналогичны режиму выравнивания изображения в Android.
- text color: Если это текстовый тип, этот конфигурационный атрибут существует. Он указывает цвет текста. Формат: #RRGGBB.
- alignment: Если это текстовый тип, этот конфигурационный атрибут существует. Он указывает способ центрирования текста. Допустимые значения: NONE (используйте метод центрирования SDK по умолчанию, то есть в конечном итоге отображается в центре); LEFT (отображается слева); CENTER (отображается в центре); RIGHT (отображается справа).
- font size (px): Если это текстовый тип, этот конфигурационный атрибут существует. Он указывает размер текста, единица измерения — пиксель px. Если введено 0, это означает, что SDK может адаптивно рассчитать размер текста; если введено значение больше 0, текст будет отображаться на основе введённого значения размера текста.
- text Bold: Если это текстовый тип, этот конфигурационный атрибут существует. Установите, является ли текст полужирным.
- mask path: путь к изображению маски. Нажмите кнопку выбора, чтобы выбрать соответствующий путь к изображению маски.

#### Описание изображения маски

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/090aaef9d17911ef82a5525400e889b2.png)

(1) на рисунке обозначает группу масок изображений. Область чёрной рамки используется для указания области, где отображается анимация слияния. (2) представляет содержимое соответствующего видеокадра.

> **Примечание:** Правила именования файлов маски совпадают с правилами для видеокадров, например: 000.png, 001.png... 099.png. Имя файла указывает, к какому кадру принадлежит текущая маска. Размер кадра маски должен совпадать с размером видеокадра. Содержимое маски: чёрная область указывает позицию, где необходимо отображать изображение (текст). Прозрачность других областей должна быть 0 (красный цвет в чёрной области указывает область препятствия). Используйте прямоугольник для области маски. Рекомендуется, чтобы ширина и высота области маски не превышали 1/2 ширины и высоты кадра маски.

## Часто задаваемые вопросы

### Примечание: разрешение видео превышает 1504. Что нужно делать?

Во время процесса переключения в журналах может быть выведено следующее: `[Warning] Output video width: xxx or height: xxx is over 1504. Some devices will display exception. For example green screen!`

Дано напоминание о том, что окончательное разрешение видео (ширина или высота) превышает 1504, и не рекомендуется превышать 1504. При превышении 1504 могут возникнуть сбои экрана и эффекты мозаичного экрана. Это эмпирическое значение, и это значение может работать нормально на большинстве моделей. Это просто напоминание и не влияет на нормальное создание видео.

Этот инструмент поддерживает совместимый режим для создания видеофайлов, то есть контролирует выходное измерение mp4 не более чем 1080p. Как показано ниже:

Когда вы отметьте режим совместимости, переключитесь в режим совместимости. Включено по умолчанию.

Когда вы его отключите, выходной размер может превышать 1080p, что может удовлетворить требование высокого разрешения. Но могут возникнуть проблемы при воспроизведении на некоторых устройствах низкого уровня.

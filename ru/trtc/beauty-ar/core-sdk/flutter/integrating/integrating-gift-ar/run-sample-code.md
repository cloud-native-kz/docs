# Запуск примера кода

Tencent Gift AR предоставляет демонстрационные приложения, подходящие для различных платформ разработки. Этот документ в основном описывает, как быстро запустить Flutter Demo SDK Gift AR.

Flutter версия SDK зависит от Gift AR SDK для Android/iOS. Через метод Plugin, предоставляемый Flutter, функции нативного конца могут быть доступны на стороне Flutter.

## Подготовка окружения

- Поддержка Flutter 3.16.0 или позже.
- Разработка для Android:
  - Android Studio 3.5 или позже.
  - Android SDK API Level 19 или выше.
  - Android 4.4 или выше, поддерживает мобильные устройства с архитектурой armeabi-v7a и arm-v8a.
- Разработка для iOS:
  - Xcode 11.0 или позже.
  - iPhone или iPad с iOS 9.0 или позже.
  - Проект имеет сконфигурированную подпись разработчика.

## Загрузка

Нажмите, чтобы загрузить Flutter [Demo проект](https://github.com/Tencent-RTC/EffectPlayer_Flutter) для Gift AR.

## Импорт проекта

1. Откройте пример Flutter Demo в редакторе кода (здесь мы используем AndroidStudio в качестве демонстрации, конечно, вы можете использовать другие редакторы, которые вам знакомы, такие как Visual Studio Code):

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/139ba3829d0611f0b8b9525400454e06.png)

2. После открытия проекта вы можете получить зависимости Flutter через встроенный интерфейс редактора кода, например найти файл `pubspec.yaml` и нажать кнопку в правом верхнем углу для получения зависимостей:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1372dcc89d0611f0b1565254001c06ec.png)

Вы также можете непосредственно выполнить следующую команду:

```
// Получение зависимостейflutter pub get// Во время использования следующую команду можно использовать для обновления существующих зависимостей Flutterflutter pub upgrade
```

Дождитесь завершения обновления зависимостей.

## Конфигурация лицензии

### Конфигурирование стороны Flutter

Использование Tencent Gift AR SDK требует применения соответствующей лицензии. Для подробного процесса применения см. [Руководство по лицензии](https://trtc.io/document/60219?platform=android&product=beautyar).

1. Поэтому перед запуском необходимо сконфигурировать информацию о лицензии на стороне Flutter. Вы можете открыть файл example/lib/common/demo_config.dart и заполнить информацию LicenseUrl и LicenseKey, которую вы применили, как показано ниже:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1391a7569d0611f0b1565254001c06ec.png)

2. После заполнения информации о лицензии при запуске будет вызван метод `FTCMediaXBase.instance.setLicense` для инициализации. Для конкретной логики см. файл: example/lib/main.dart.

### Конфигурация Android

После завершения описанных выше операций, если вам необходимо скомпилировать и запустить на платформе Android для пробы, выполните следующие операции для конфигурирования платформы Android.

1. Откройте файл example/android/app/build.gradle.kts инструментом редактирования кода, найдите определённый в нём applicationId и измените его на Android Package, соответствующий ранее заполненной лицензии, как показано ниже:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1379bf939d0611f0872c525400bf7822.png)

2. После заполнения просто убедитесь в безошибочной инициализации лицензии на платформе Android во время выполнения.

### Конфигурация iOS

После завершения описанных выше операций, если вам необходимо скомпилировать и запустить на платформе iOS для пробы, выполните следующие операции для конфигурирования платформы iOS.

1. В директории example/ios выполните команду: `pod install`. Дождитесь завершения выполнения, затем используйте инструмент разработки XCode для открытия файла Runner.xcworkspace в этой директории для запуска проекта. Далее измените Bundle Identifier проекта на iOS BundleId, соответствующий ранее заполненной лицензии, как показано ниже:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1383dcd59d0611f0930a5254007c27c5.png)

2. После заполнения просто убедитесь в безошибочной инициализации лицензии на платформе iOS во время выполнения.

## Конфигурация ресурсов анимации

> **Примечание:** Если вам не нужно воспроизводить собственный файл анимации, вы можете пропустить этот шаг.

Для изменения файла анимации для воспроизведения вы можете добавить файл анимации для воспроизведения в директорию example/asset/anim. Затем откройте файл example/lib/tools/demo_asset_helper.dart и измените свойство _kNormalAnimNames (крупная анимация) или _kTCMP4AnimNames (анимация маленького аватара) в начале. Demo автоматически завершит задачи, такие как копирование файла анимации, как показано ниже:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/13a766a89d0611f0b8b9525400454e06.png)

Наконец, вам нужно включить файл конфигурации example/pubspec.yaml, добавить новый файл анимации в список конфигураций. Впоследствии вы сможете напрямую запустить проект Demo для демонстрации воспроизведения анимации.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/13a7ec739d0611f0930a5254007c27c5.png)

> **Примечание:** Встроенные файлы анимации в Demo не привязаны к лицензии, поэтому вы можете использовать свою собственную лицензию для демонстрации воспроизведения. Примечание: Если вы хотите заменить свою собственную анимацию, обратите внимание, что информация о лицензии должна совпадать с установленной в проекте информацией о лицензии, в противном случае может произойти сбой воспроизведения из-за несовпадения информации о лицензии.

## Запуск проекта

После завершения описанных выше операций вы можете начать запуск проекта Demo. Вы можете запустить проект Demo следующим образом (доступно множество методов, здесь мы приводим один):

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1370a05a9d0611f0b1565254001c06ec.png)

1. Сначала выберите устройство для запуска, затем нажмите ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/82a18346a80a11f0b8b9525400454e06.png) Запустить и дождитесь компиляции. После завершения установки вы можете приступить к пробе.
2. После запуска проекта представление домашней страницы выглядит следующим образом:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/92538c6ba80a11f0b8b9525400454e06.png)

Среди них:

  - EffectAnim Demo После нажатия кнопки вы сможете приступить к демонстрации анимации, включая некоторые распространённые примеры.
  - EffectAnim List После нажатия кнопки вы сможете приступить к демонстрации маленькой анимации.

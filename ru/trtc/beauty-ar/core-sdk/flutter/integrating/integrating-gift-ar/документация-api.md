### setLicense

**Описание**

Настройка лицензии.

**API**

```
Future<void> setLicense(String url, String key, FTCMediaLicenseListener listener) async
```

**Описание параметров**

| Имя параметра | Тип | Описание |
| --- | --- | --- |
| url | String | URL лицензии. |
| key | String | Ключ лицензии. |
| callback | ILicenseCallback | Обратный вызов |

Объявление обратного вызова:

```
typedef FTCMediaLicenseListener = void Function(int errCode, String msg);
```

Общие коды ошибок для проверки лицензии

| Код ошибки | Описание |
| --- | --- |
| 0 | Успех: Выполнено |
| -1 | Входной параметр недействителен, например пустой URL или KEY. |
| -3 | Загрузка не удалась. Проверьте параметры сети. |
| -4 | Локально прочитанная информация авторизации TE пуста, что может быть вызвано сбоем операции ввода-вывода. |
| -5 | Содержимое локального файла VCUBE TEMP License пусто, может быть вызвано сбоем операции ввода-вывода. |
| -6 | Поле JSON в файле v_cube.license некорректно. Свяжитесь с командой Tencent Cloud для исправления. |
| -7 | Проверка подписи не удалась. Свяжитесь с командой Tencent Cloud для исправления. |
| -8 | Расшифровка не удалась. Свяжитесь с командой Tencent Cloud для исправления. |
| -9 | Поле JSON в поле TELicense некорректно. Свяжитесь с командой Tencent Cloud для исправления. |
| -10 | Информация авторизации TE из сетевого разрешения пуста. Свяжитесь с командой Tencent Cloud для исправления. |
| -11 | Ошибка записи информации авторизации TE в локальный файл, что может быть вызвано сбоем операции ввода-вывода. |
| -12 | Загрузка не удалась. Ошибка при разборе локального ресурса. |
| -13 | Ошибка аутентификации. Проверьте, находится ли файл so в пакете или правильно ли установлен путь so. |
| 3004/3005 | Недействительная авторизация. Свяжитесь с командой Tencent Cloud для исправления. |
| 3015 | Несоответствие Bundle Id / Package Name. Проверьте, совпадает ли Bundle Id / Package Name, используемый вашим приложением, с указанным, и убедитесь, что используется правильный файл лицензии. |
| 3018 | Срок действия файла авторизации истёк. Подайте заявку на продление в Tencent Cloud. |
| Другие | Свяжитесь с командой Tencent Cloud для исправления. |

### setLogEnable

**Описание**

Включение вывода журнала. По умолчанию включено.

> **Примечание:** На Android сохраняется в директории /sdcard/Android/data/packagename/files/TCMediaX. На iOS сохраняется в директории Documents/TCMediaX в песочнице. Вы можете загружать журналы из этой директории на серверную часть согласно требованиям бизнеса для выявления проблем у онлайн-пользователей.

**API**

```
Future<void> setLogEnable(bool enable) async
```

### setVideoMode

**Описание**

Установить режим выравнивания области альфа и rgb анимации tep.

**API**

```

```

**Описание параметров**

videoMode поддерживает следующие форматы:

| Значение перечисления | Описание |
| --- | --- |
| FVideoMode.VIDEO_MODE_NONE | Обычный файл mp4 |
| FVideoMode.EVIDEO_MODE_SPLIT_HORIZONTAL | выравнивание слева направо (альфа слева, rgb справа) |
| FVideoMode.VIDEO_MODE_SPLIT_VERTICAL | выравнивание сверху вниз (альфа сверху, rgb снизу) |
| FVideoMode.VIDEO_MODE_SPLIT_HORIZONTAL_REVERSE | выравнивание слева направо (rgb слева, альфа справа) |
| FVideoMode.VIDEO_MODE_SPLIT_VERTICAL_REVERSE | выравнивание сверху вниз (rgb сверху, альфа снизу) |

### setLoop

**Описание**

Установить циклическое воспроизведение.

- true: означает циклическое воспроизведение.
- false: означает отключение циклического воспроизведения.

**API**

```
Future<void> setLoop(bool isLoop) async
```

### setLoopCount

**Описание**

Установить количество циклических воспроизведений.

loopCount: указывает количество циклических воспроизведений. Когда loopCount <= 0, это означает бесконечное циклическое воспроизведение; когда loopCount=n(n>=1), это означает воспроизведение от начала до конца воспроизведения всего n раз.

> **Примечание:** Значение по умолчанию для loopCount — 1, что означает, что анимация воспроизводится только один раз и заканчивается, если метод не вызывается активно снаружи. Метод setLoop(boolean isLoop) внутри вызывает текущий метод. Когда isLoop = true, это эквивалентно вызову setLoopCount(-1). Когда isLoop = false, это эквивалентно вызову setLoopCount(1). Поэтому эти два метода влияют друг на друга, и более поздний вызов переопределит предыдущий.

**API**

```
Future<void> setLoopCount(int loopCount) async
```

### setDuration

**Описание**

Сколько времени занимает завершение воспроизведения анимации. После установки скорость воспроизведения анимации будет автоматически регулироваться для обеспечения остановки анимации на установленную длительность.

Если установленная длительность превышает исходную длительность анимации, анимация будет воспроизводиться в замедленном режиме; если она меньше исходной длительности, она будет быстро перемотана.

durationInMilliSec: Длительность для установки в миллисекундах.

> **Примечание:** В настоящее время применимо только к анимациям в формате tcmp4. Текущий метод и метод setRate регулировки скорости являются взаимоисключающими. Более поздний вызов переопределит более ранний.

**API**

```
Future<void> setDuration(int durationInMilliSec) async
```

## FTCEffectConfig

**Описание**

Построить конфигурацию проигрывателя эффектов.

**Свойство codecType**

```
// Установить CodecTypeFCodecType? codecType;
```

**Описание параметров**

Имеет три значения:

- FCodecType.TC_MPLAYER (модуль воспроизведения MPLAYER)
- FCodecType.TC_MCODEC (модуль воспроизведения MCODEC, действует только на платформе Android)
- FCodecType.TX_LITEAV_SDK (SDK плеера Tencent Cloud)

> **Примечание:** В настоящее время поддерживается только вызов метода setConfig() для установки конфигурации воспроизведения перед запуском проигрывателя. Изменение конфигурации после начала воспроизведения не поддерживается. В настоящее время поддерживаются три CodecType, применимо только к анимации TEP. Если CodecType установлен на FCodecType.TX_LITEAV_SDK, вам необходимо отдельно импортировать SDK плеера Tencent Cloud и подать заявку и зарегистрировать соответствующую лицензию.

```
// Установить индекс кадра замерзанияFFreezeFrame? freezeFrame;
```

**Описание параметров**

Входной параметр frame указывает кадр, на котором оставаться во время воспроизведения анимации. Диапазон текущего значения:

- FFreezeFrame.FREEZE_FRAME_NONE: Отключить функцию freezeFrame. Проигрыватель будет воспроизводиться, паузироваться и исчезать нормально.
- FFreezeFrame.FREEZE_FRAME_LAST: После первого воспроизведения кадр остаётся на последнем кадре.

**Свойство animType (только для Android)**

```
// Установить формат анимации для воспроизведенияFAnimType? animType;
```

**Описание параметров**

Установить формат анимации для воспроизведения, подходит для сценариев, где суффикс файла анимации для воспроизведения изменён. Входной параметр type указывает указанный формат анимации. Текущие доступные значения:

- FAnimType.AUTO (Политика по умолчанию SDK определяет формат анимации по суффиксу файла, такому как tcmp4, mp4, tep или tepg.)
- FAnimType.MP4 (Формат анимации MP4. Впоследствии файлы анимации будут воспроизводиться как тип MP4, игнорируя суффикс файла.)
- FAnimType.TCMP4 (Формат анимации TCMP4. Впоследствии файлы анимации будут воспроизводиться как тип TCMP4, игнорируя суффикс файла.)

## FTCEffectText

**Описание**

Данные стиля текста замены анимации слияния.

**Описание атрибутов**

| Имя атрибута | Тип | Описание |
| --- | --- | --- |
| text | String | Окончательное содержимое отображаемого текста для замены. |
| color | int | Цвет текста, требуемый формат: ARGB, например 0xFFFFFFFF. |
| fontStyle | String | Стиль отображения текста, допустимые значения: "bold" означает полужирный текст, размер по умолчанию, если не указано. |
| alignment | int | Режим выравнивания текста, допустимые значения: TEXT_ALIGNMENT_NONE (значение по умолчанию, сохранить выравнивание по умолчанию SDK), TEXT_ALIGNMENT_LEFT (слева), TEXT_ALIGNMENT_CENTER (по центру), TEXT_ALIGNMENT_RIGHT (справа). |
| fontSize | double | Размер текста, единица px; если установлен размер текста (значение больше 0), внутренняя политика автомасштабирования будет отключена и установленный размер текста будет применен принудительно, что может привести к тому, что текст большого размера не отобразится полностью. |

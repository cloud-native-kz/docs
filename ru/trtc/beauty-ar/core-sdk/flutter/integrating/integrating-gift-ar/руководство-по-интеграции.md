# Руководство по интеграции

## Подготовка среды разработки

- Поддерживает Flutter 3.16.0 или более позднюю версию.
- Разработка для Android:
  - Android Studio 3.5 или более позднюю версию.
  - Android SDK API Level 19 или выше.
  - Android 4.4 или выше, поддерживает мобильные устройства с архитектурой armeabi-v7a и arm-v8a.
- Разработка для iOS:
  - Xcode 11.0 или более позднюю версию.
  - iPhone или iPad с iOS 9.0 или более поздней версией.
  - Проект имеет настроенную действительную подпись разработчика.

## Загрузка SDK

## Быстрая интеграция

### Добавление зависимостей в pubspec.yaml

1. Интегрируйте последнюю версию EffectPlayer_Player, которая также интегрирована по умолчанию. Добавьте следующую конфигурацию в `pubspec.yaml`:

```
  flutter_effect_player:
    git:
      url: https://github.com/Tencent-RTC/EffectPlayer_Flutter.git
```

При необходимости вы можете указать тег ref зависимости для использования определённой версии, как показано ниже:

```
flutter_effect_player:
  git:
    url: https://github.com/Tencent-RTC/EffectPlayer_Flutter
    ref: release_example_tag
```

Дополнительные архивированные теги см. в [списке релизов](https://github.com/Tencent-RTC/EffectPlayer_Flutter/releases).

2. После интеграции вы можете получить зависимость Flutter через встроенный интерфейс редактора кода или непосредственно использовать следующую команду:

```
flutter pub get
```

3. При использовании следующая команда может быть использована для обновления существующей зависимости Flutter:

```
flutter pub upgrade
```

## Добавление конфигурации нативной части

### Конфигурация Android

#### Конфигурация воспроизведения зашифрованного видео

Если вы отметили опцию **Anim Encrypt** при преобразовании анимаций с помощью TepTools, выходная анимация будет зашифрована. В этом случае при воспроизведении с помощью effect player, если значение targetSDKVersion вашего приложения не менее 28, настройте следующим образом:

1. Создайте файл xml в res/xml/network_security_config.xml и установите конфигурацию сетевой безопасности:

```
  <?xml version="1.0" encoding="utf-8"?>
  <network-security-config>
    <domain-config cleartextTrafficPermitted="true">
      <domain includeSubdomains="true">localhost</domain>
      <domain includeSubdomains="true">127.0.0.1</domain>
    </domain-config>
  </network-security-config>
```

2. Добавьте следующие свойства в тег application в файле AndroidManifest.xml:

```
  <?xml version="1.0" encoding="utf-8"?>
  <manifest ... >
    <application android:networkSecurityConfig="@xml/network_security_config"
                   ... >
      ...
    </application>
  </manifest>
```

### Конфигурация iOS

> **Примечание:** iOS в настоящее время не поддерживает работу и отладку в симуляторе. Рекомендуется выполнять разработку и отладку на реальном устройстве.

#### Конфигурация воспроизведения зашифрованного видео

Если вы отметили опцию **Anim Encrypt** при преобразовании анимаций с помощью TepTools, выходная анимация будет зашифрована. В этом случае при воспроизведении с помощью effect player настройте следующим образом, добавив следующую конфигурацию в `Info.plist` iOS:

```
<key>NSAppTransportSecurity</key>
<dict>
    <key>NSAllowsArbitraryLoads</key>
    <true/>
</dict>
```

# Руководство по использованию

Этот документ подробно описывает, как быстро использовать Tencent Gift AR SDK в проекте. Просто следуйте приведенным ниже шагам для завершения конфигурации и использования SDK.

## Инициализация и регистрация лицензии

Для официального использования Gift AR SDK необходимо сначала настроить лицензию. После успешной настройки SDK можно использовать в дальнейшем.

Способ конфигурации лицензии следующий:

```
import 'package:flutter_effect_player/ftceffect_player.dart';const LICENSE_URL = "${licenseUrl}";const LICENSE_KEY = "${licenseKey}";FTCMediaXBase.instance.setLicense(LICENSE_URL, LICENSE_KEY, (code, msg) {  print("TCMediaX license result: errCode:${code}, msg:${msg}");});
```

> **Примечание:** Лицензия реализует строгую логику онлайн-проверки. При вызове FTCMediaXBase.instance.setLicense после первого запуска приложения должна быть доступна сеть. При первоначальном запуске приложения, если разрешение на сетевой доступ еще не получено, необходимо дождаться предоставления разрешения и снова вызвать этот метод. Следите за результатом обратного вызова FTCMediaXBase.instance.setLicense. При ошибке повторите попытку и предоставьте руководство в соответствии с конкретной ситуацией. После многократных ошибок ограничьте частоту и дополните рекомендациями о продукте и другими подсказками, чтобы позволить пользователям проверить условия сети.

## Управление логами

Tencent Gift AR SDK по умолчанию поддерживает сохранение журналов выполнения. Если во время тестирования возникают проблемы, вы можете извлечь логи и отправить их в службу поддержки Tencent Cloud. Вы можете загружать логи из этого каталога в серверную часть в соответствии с потребностями бизнеса для локализации проблем пользователей.

- На Android логи хранятся в каталоге: `/sdcard/Android/data/${your_packagename}/files/TCMediaLog`. Файлы логов названы по датам. Экспортируйте папку TCMediaLog.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/cd3be2079d0411f0b1565254001c06ec.png)

- На iOS логи хранятся в папке песочницы Documents/TCMedialog с файлами логов, названными по датам. Для получения подробного руководства по экспорту логов [см. здесь](https://www.tencentcloud.com/document/product/1143/70544#59b6468e-c7a0-48da-9b96-94c85e49e48c).

> **Примечание:** **Нет файла логов?** Проверьте, передано ли FTCMediaXBase.instance.setLogEnable значение false для отключения логов. По умолчанию логирование в файл включено.

## Использование плеера

### Введение FTCEffectAnimView

```
FTCEffectViewController? controller;Widget _getFTCEffectAnimView() {  return FTCEffectAnimView(      controllerCallback: (controller){        // controller.setLoop(true);        // controller.startPlay("xxxxxxx");      }  );}
```

### Прослушивание воспроизведения

Перед началом воспроизведения можно установить прослушивание статуса воспроизведения анимации, вызвав метод `setPlayListener`:

```
controller?.setPlayListener(FAnimPlayListener(onPlayStart: () {  // on anim play start}, onPlayEnd: () {  // on anim play end}, onPlayEvent: (int event, Map params) {  // on anim play events}, onPlayError: (code) {  // on anim play error}));
```

Если вам нужно получить информацию о текущей воспроизводимой анимации, вы можете вызвать метод `getTCAnimInfo()` в методе `onPlayStart()` (или после его выполнения) для получения экземпляра объекта `FTCEffectAnimInfo`, а затем получить тип, продолжительность, ширину, высоту, анимацию слияния и т. д. текущей воспроизводимой анимации.

### Конфигурация воспроизведения

```
// Set playback configuration, not a necessary procedurefinal FTCEffectConfig _curConfig = FTCEffectConfig()..codecType = FCodecType.TC_MPLAYER;controller?.setConfig(_curConfig);
```

FTCEffectConfig в настоящее время поддерживает:

1. FCodecType codecType: имеет три значения:
  - FCodecType.TC_MPLAYER (механизм воспроизведения MPLAYER)
  - FCodecType.TC_MCODEC (механизм воспроизведения MCODEC) (поддерживается только на Android)
  - FCodecType.TX_LITEAV_SDK (Tencent Cloud Player SDK)

> **Примечание:** В настоящее время поддерживается только вызов метода setConfig() для установки конфигурации воспроизведения до запуска плеера. Изменение конфигурации после начала воспроизведения не поддерживается. В настоящее время вышеуказанные параметры работают только для MP4-анимаций. Если установлено значение FCodecType.TX_LITEAV_SDK, вам также необходимо отдельно импортировать Tencent Cloud Player SDK, а также подать заявку и зарегистрировать соответствующую лицензию. Если не установлено, используется CodecType = FCodecType.TC_MPLAYER по умолчанию.

2. FFreezeFrame freezeFrame: используется для установки замороженного кадра анимации воспроизведения. Подробное объяснение см. в [документации API](https://www.tencentcloud.com/document/product/1143/73977#freezeFrame).
3. FAnimType animType (поддерживается только на Android): используется для указания формата анимации для последующего воспроизведения, подходит для сценариев, где суффикс файла анимации для воспроизведения изменен. Значения:
  - FAnimType.AUTO (политика по умолчанию SDK, которая определяет формат анимации на основе суффикса файла, например tcmp4/mp4/tep/tepg).
  - FAnimType.MP4 (формат анимации MP4, впоследствии воспроизводить файлы анимации как тип MP4, игнорировать суффикс файла)
  - FAnimType.TCMP4 (формат анимации TCMP4, впоследствии воспроизводить файлы анимации как тип TCMP4, игнорировать суффикс файла).

### Конфигурация анимации слияния

Для воспроизведения анимации слияния mp4 или tcmp4 необходимо реализовать FResourceFetcher.

- Если это анимация слияния типа изображения, необходимо реализовать FResImgResultFetcher для возврата соответствующего Uint8List изображения для замены элемента.
- Если это анимация слияния текстового типа, необходимо реализовать FResTextResultFetcher для возврата соответствующего экземпляра объекта FTCEffectText и указания параметров отображения текста.

```
controller?.setFetchResource(FResourceFetcher(textFetcher: (res) {   // Return the FTCEffectText instance, you can specify text, color inside   return FTCEffectText()     ..text = "FTCEffect${res.id}"     ..color = 0xff0000ff; }, imgFetcher: (res) {   // Prepare the image data to be replaced in advance, simply return directly here.   // final data = await rootBundle.load('asset/image/head1.png');   // _samplePng = data.buffer.asUint8List();   return _samplePng; }));
```

### Управление воспроизведением

#### Начало воспроизведения

Поддерживает воспроизведение ресурсов в формате файлов .mp4 и .tcmp4.

> **Примечание:** Поддерживает воспроизведение только локальных видеоресурсов на sdcard. Если вы используете сетевые видеоресурсы, **сначала загрузите их локально, а затем воспроизведите**. **Сохраняйте суффикс файла тем же, что и исходный файл после загрузки**, например .mp4 или .tcmp4. Удаление или изменение суффикса приведет к ошибке воспроизведения. Ресурсы анимации можно создать через [инструмент преобразования Gift AR](https://www.tencentcloud.com/document/product/1143/70541). Кроме того, поддерживается воспроизведение анимации формата VAP.

```
String localPath = /path/files/tep_cool_ss.mp4controller?.startPlay("xxxx")
```

#### Пауза воспроизведения

```
controller?.pause()
```

#### Продолжение воспроизведения

```
controller?.resume()
```

#### Циклическое воспроизведение

```
controller?.setLoop(true)
```

#### Отключение звука при воспроизведении

```
controller?.setMute(true)
```

#### Остановка воспроизведения

Когда плеер больше не требуется, остановите воспроизведение и освободите занятые ресурсы.

```
controller?.stopPlay()
```

#### Как обрабатывать информацию логов при воспроизведении


**Описание**

```

```

```

```

Общие коды ошибок для проверки лицензии

| Код ошибки | Описание |
| --- | --- |
| 0 | Успешно. |
| -1 | Входные параметры неверны, например URL или KEY пусты. |
| -3 | Загрузка не удалась, проверьте параметры сети. |
| -4 | Информация об авторизации TE, прочитанная с локального хранилища, пуста, что может быть вызвано сбоем ввода-вывода. |
| -5 | Содержимое файла лицензии VCUBE TEMP пусто, что может быть вызвано сбоем ввода-вывода. |
| -6 | Поле JSON файла v_cube.license неправильно. Свяжитесь с командой Tencent Cloud для обработки. |
| -7 | Проверка подписи не удалась. Свяжитесь с командой Tencent Cloud для обработки. |
| -8 | Расшифровка не удалась. Свяжитесь с командой Tencent Cloud для обработки. |
| -9 | Поле JSON в поле TELicense неправильно. Свяжитесь с командой Tencent Cloud для обработки. |
| -10 | Информация об авторизации TE, полученная из сети, пуста. Свяжитесь с командой Tencent Cloud для обработки. |
| -11 | Не удалось записать информацию об авторизации TE в локальный файл, что может быть вызвано сбоем ввода-вывода. |
| -12 | Загрузка не удалась, анализ локального ресурса также не удался. |
| -13 | Аутентификация не удалась. |
| Другое | Свяжитесь с командой Tencent Cloud для обработки. |

**Описание**

### setLoopCount

**Описание**

Установить количество циклических воспроизведений.

loopCount: Указывает количество циклических воспроизведений. Когда loopCount <= 0, это означает бесконечное циклическое воспроизведение; когда loopCount = n (n >= 1), это означает воспроизведение от начала до конца n раз.

> **Примечание:** Значение loopCount по умолчанию равно 1, то есть когда внешний элемент не активно вызывает этот метод, анимация воспроизводится только один раз, а затем заканчивается. Метод setLoop будет внутренне вызывать текущий метод. То есть, когда isLoop = true, это эквивалентно вызову setLoopCount(-1); когда isLoop = false, это эквивалентно вызову setLoopCount(1). Поэтому эти два метода влияют друг на друга, а более поздний вызов переопределит предыдущий.

**API**

```
- (void)setLoopCount:(int)loopCount;
```

### setDuration

**Описание**

Сколько времени требуется для завершения набора анимации? После установки скорость воспроизведения анимации будет автоматически отрегулирована во время последующего воспроизведения анимации, чтобы гарантировать, что анимация остановится в установленную длительность.

То есть, если установленная длительность превышает исходную длительность анимации, анимация будет воспроизводиться в замедленном режиме; если она меньше исходной длительности анимации, она будет ускорена.

> **Примечание:** В настоящее время применимо только к анимациям в формате tcmp4. Текущий метод и метод установки скорости воспроизведения через setRate являются взаимоисключающими. Позже вызванный переопределит ранее вызванный. durationInMilliSec: Длительность, которую следует установить, в миллисекундах.

**API**

```
- (void)setDuration:(long)durationInMilliSec;
```

## TCEffectConfig

**Описание**

Конструкция конфигурации проигрывателя специальных эффектов. Поддерживаемые атрибуты:

1. vapEngineType(CodecType type): Имеет два значения параметра, которые являются:
  - TCEPCodecTypeAVPlayer, проигрыватель по умолчанию для анимации подарков специальных эффектов.
  - TCEPCodecTypeVODPlayer, проигрыватель SDK Tencent Cloud Player.

```
// Установить конфигурацию воспроизведения, необязательный шаг.TCEffectConfig *config = [[TCEffectConfig alloc] init];[self.alphaAnimView setEffectPlayerConfig:config];
```

> **Примечание:** В настоящее время вы можете вызывать метод setEffectPlayerConfig только перед запуском проигрывателя для установки конфигурации воспроизведения. Изменение конфигурации после начала воспроизведения не поддерживается. Поддерживаемые в настоящее время 2 vapEngineType применимы только к анимациям MP4. Если вы установите vapEngineType на TCEPCodecTypeVODPlayer, вам также необходимо отдельно подключить SDK Tencent Cloud Player и применить и зарегистрировать соответствующую лицензию.

2. freezeFrame используются для установки замороженного кадра воспроизведения анимации. Текущие возможные значения:
  - FRAME_NONE: Значение по умолчанию. Отключить возможность freezeFrame. Проигрыватель воспроизводит, паузирует и исчезает нормально.
  - FRAME_FIRST: После завершения воспроизведения перезапустить воспроизведение и приостановить, когда появится первый кадр в следующий раз.
  - FRAME_LAST: После первого завершения воспроизведения зрительный вид остается на последнем кадре.
3. enableAudioFrameCallback включить переключатель обратного вызова для воспроизведения аудио, подходящий для сценариев, где требуется обработка прослушивания событий для аудиодорожек анимации подарков.

> **Примечание:** В настоящее время поддерживается только вызов метода setEffectPlayerConfig() для установки конфигурации воспроизведения перед запуском проигрывателя. Изменение конфигурации после начала воспроизведения не поддерживается. Этот параметр действителен только когда CodecType — это TCEPCodecTypeVODPlayer и тип события — MP4/TEP. После включения переключателя можно установить мониторинг через метод TCEPAnimViewDelegate#tcePlayerAudioFrame.

4. enableProgressCallback включить обратный вызов воспроизведения анимации
5. progressCallbackIntervalMs интервал обратного вызова прогресса для воспроизведения анимаций в миллисекундах. Значение по умолчанию: 200 мс.

> **Примечание:** Обратите внимание, что эта конфигурация действует только при включении переключателя enableProgressCallback. После включения переключателя вы можете прослушивать событие TCEPConstant.TCEP_PLAYER_EVT_PROGRESS в обратном вызове TCEPAnimViewDelegate#onPlayEvent, чтобы получить текущий прогресс воспроизведения. Используйте TCEPEventProgressMSKey и TCEPEventDurationMSKey, чтобы соответственно получить текущую длительность воспроизведения и общую длительность (в миллисекундах) из параметра. Установите разумный временной интервал. Слишком малый интервал может повлиять на производительность.

## TCEffectText

**Описание**

Класс данных стиля текста замены анимации слияния.

**{Описание атрибутов}**

| Имя атрибута | Тип | Описание |
| --- | --- | --- |
| text | NSString | Содержимое текста, которое будет отображаться, в конце концов будет заменено. |
| color | UIColor | Цвет текста, формат требования: ARGB, например, 0xFFFFFFFF. |
| fontStyle | NSString | Стиль отображения текста. Допустимые значения: "bold" указывает полужирный текст. Если не указано, будет использован размер по умолчанию. |
| alignment | int | Режим выравнивания текста. Допустимые значения: TCEPTextAlignmentNone (значение по умолчанию, то есть сохранить режим выравнивания по умолчанию SDK), TCEPTextAlignmentLeft (выровнять влево), TCEPTextAlignmentCenter (по центру), TCEPTextAlignmentRight (выровнять вправо). |
| fontSize | CGFloat | Размер текста, единица: px; если размер текста установлен (значение больше 0), политика автоматического масштабирования будет отключена, и установленный размер текста будет принудительно использован как стандарт, что может вызвать проблему, когда текст слишком больших размеров не отображается полностью. |

## TCEffectAudioFrameData

**Описание**

Класс данных аудиоданных обратного вызова аудиодорожки

**Описание атрибутов**

| Имя атрибута | Тип | Описание |
| --- | --- | --- |
| data |  uint8_t** | Данные аудиопробы. Когда данные не являются плоскими, data[0] хранит данные; когда данные являются плоскими, data[0] к data[channels-1] хранят каждый звуковой канал. |
| size | int* | Размер данных аудиопробы. Когда данные не являются плоскими, size[0] хранит длину данных; когда данные являются плоскими, size[0] к size[channels-1] хранят размер каждого звукового канала. |
| format | int | Формат аудио |
| channelLayout | uint64_t | Расположение канала аудио |
| sampleRate | unsigned int | Частота дискретизации аудио |
| channels | int | Количество звуковых каналов |
| nbSamples | int | Количество выборок каждого звукового канала |
| ptsMs | int64_t | Текущий PTS выборки |

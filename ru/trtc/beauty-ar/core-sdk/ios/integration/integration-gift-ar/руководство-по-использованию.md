# Руководство по использованию

Этот документ подробно описывает, как быстро использовать Tencent Gift AR SDK в проекте. Просто следуйте приведенным ниже шагам для завершения конфигурации и использования SDK.

## Инициализация и регистрация лицензии

При официальном использовании Tencent Gift AR SDK необходимо сначала настроить лицензию. После успешной настройки лицензии вы можете использовать SDK.

Методы настройки лицензии следующие:

> **Примечание:** Пример кода здесь устанавливает лицензию при запуске приложения. Однако не рекомендуется вызывать это здесь в вашем проекте, поскольку в это время может не быть разрешения на сетевой доступ или частота отказов сетевого соединения может быть относительно высокой (особенно при первом запуске приложения). Пожалуйста, выберите более подходящий момент для установки лицензии.

```
#import <TCMediaX/TCMediaX.h>@interface AppDelegate ()<TCMediaXBaseDelegate>@end  @implementation AppDelegate// Replace LICENCE_URL with your own LICENCE_URL when in usestatic NSString *LICENCE_URL = @"";static NSString *LICENCE_KEY = @"";- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {    // Override point for customization after application launch.   [[TCMediaXBase getInstance] setLicenceURL:LICENCE_URL key:LICENCE_KEY];   [[TCMediaXBase getInstance] setDelegate:self];    return YES;}#pragma mark - TCMediaXBaseDelegate- (void)onLicenseCheckCallback:(int)errcode withParam:(NSDictionary *)param {    // Note: This callback may not be on the calling thread.    NSLog(@"onLicenseCheckCallback:%d",errcode);    if (errcode == TMXLicenseCheckOk) {        // Authentication successful    } else {        // Authentication failure    }}@end
```

> **Примечание:** Лицензия имеет мощную логику онлайн-проверки. При вызове TCMediaXBase#setLicense после первого запуска приложения должна быть доступна сеть. При первом запуске приложения разрешение на доступ к сети может быть еще не авторизовано. Необходимо дождаться предоставления разрешения и затем снова вызвать TCMediaXBase#setLicense. Слушайте результат загрузки TCMediaXBase#setLicence: API ILicenseCallback#onResult. В случае отказа повторите попытку и выполните рекомендации в зависимости от ситуации. Если возникает несколько отказов, ограничьте частоту и дополните всплывающими подсказками продукта и другими указаниями, чтобы пользователи могли проверить условия сети. Для приложений с несколькими процессами убедитесь, что TCMediaXBase#setLicense вызывается при запуске каждого процесса, использующего проигрыватель Gift AR. Например, для приложений, использующих отдельный процесс для воспроизведения эффектов на стороне Android, TCMediaXBase#setLicense также должен быть вызван при убийстве процесса и перезагрузке системой во время фонового воспроизведения.

## Управление логами

Tencent Effect Player SDK по умолчанию поддерживает сохранение логов запуска. Если во время тестирования возникают проблемы, вы можете извлечь логи и отправить отзыв в службу поддержки Tencent Cloud. Вы можете загрузить логи из этого каталога на backend в соответствии с требованиями бизнеса для локализации проблем онлайн-пользователей.

Логи версии iOS Tencent Effect Player SDK сохраняются в папке Documents/TCMedialog в песочнице, а файл логов названи по дате. Подробное руководство по экспорту логов см. [здесь](https://www.tencentcloud.com/document/product/1143/70544#59b6468e-c7a0-48da-9b96-94c85e49e48c).

Логирование воспроизведения включено для сохранения по умолчанию. Если вам нужно отключить его, вы можете передать NO через `TCMediaXBase#setLogEnable`, чтобы отключить его.

## Использование проигрывателя

### Инициализация TCEffectAnimView

```
- (TCEffectAnimView *)alphaAnimView {    if (!_alphaAnimView) {        _alphaAnimView = [[TCEffectAnimView alloc] init];        _alphaAnimView.backgroundColor = [UIColor clearColor];        _alphaAnimView.effectPlayerDelegate = self;        _alphaAnimView.loop = YES;        [_alphaAnimView setRenderMode:TCEPVPViewContentModeScaleToFill];    }    return _alphaAnimView;}
```

### Добавление AlphaAnimView и разметка

```
// Add alphaAnimView[self.view addSubview:self.alphaAnimView];// Layoutself.alphaAnimView.frame = self.view.bounds;
```

### Воспроизведение и прослушивание

Вы можете прослушивать статус воспроизведения анимации, установив метод прокси `effectPlayerDelegate` перед началом воспроизведения:

```
self.alphaAnimView..effectPlayerDelegate = self;#pragma mark - TCEPAnimViewDelegate // Animation starts playing- (void)tcePlayerStart:(ITCEffectPlayer *)player {    NSLog(@"player = %p, start..", player);}// Animation stops playback- (void)tcePlayerEnd:(ITCEffectPlayer *)player {    NSLog(@"player = %p, end..", player);}// Animation playback failure-  (void)tcePlayerError:(ITCEffectPlayer *)player error:(NSError *)error {    NSLog(@"player = %p, error..", player);}// Animation playback event- (void)onPlayEvent:(ITCEffectPlayer *)player              event:(int)EvtID          withParam:(NSDictionary *)param {    NSLog(@"player = %p, EvtID = %@", player, @(EvtID));}
```

### Получение видеоресурсов и запуск воспроизведения

> **Примечание:** Поддерживается только воспроизведение локальных видеоресурсов. Если вы используете онлайн-видеоресурсы, загрузите их локально и повторите воспроизведение. Анимированные ресурсы можно создавать через [инструмент преобразования Gift AR](https://www.tencentcloud.com/document/product/1143/70541). Кроме того, также могут воспроизводиться анимированные ресурсы в формате VAP.

```
 [self.alphaAnimView startPlay:localPathUrl];
```

### Конфигурация слияния анимации

Для реализации слияния анимации требуется реализовать метод прокси TCEffectAnimViewDelegate.

#### Замена текста при слиянии анимации

Замените текст и укажите режим выравнивания, цвет и поддержку ли жирного текста для указанного текста, используя API loadTextForPlayer.

```
// Replace text, and specify the alignment mode, color, and whether it is bold of the text. Use the loadTextForPlayer API. It is recommended to use this API.- (TCEffectText *)loadTextForPlayer:(ITCEffectPlayer *)player                            withTag:(NSString *)tag {    if (tag != nil && [tag isEqualToString:@"name"]) {        TCEffectText *effectText = [[TCEffectText alloc] init];        effectText.text = @"reText";        // TCEPTextAlignmentLeft = 0,     ///< Text aligns left        // TCEPTextAlignmentCenter = 1,   ///< Text aligns center        // TCEPTextAlignmentRight = 2     ///< Text aligns right        effectText.alignment = TCEPTextAlignmentLeft;         //effectText.fontStyle = @"bold";        //effectText.color = [UIColor blackColor];        return effectText;;    }    return nil;}
```

#### Замена изображения при слиянии анимации

Получите UIImage, получив значение тега. Проигрыватель слияния анимации не обрабатывает процесс преобразования внутри. Обработайте это самостоятельно в этом методе.

```
- (void)loadImageForPlayer:(ITCEffectPlayer *)player                   context:(NSDictionary *)context                completion:(void(^)(UIImage *image,                                    NSError *error))completionBlock {    dispatch_async(dispatch_get_main_queue(), ^{        // Obtain the current tag of the image to be replaced, then return the image according to the tag        NSString *tag = context[TCEPContextSourceTypeImageIndex];           ...        // Download and cache the image        if (image) {            completionBlock(image, nil);        } else {            completionBlock(nil, error);        }        }];    });}
```

Прослушивайте событие, когда на пользовательский ресурс анимации нажимается, и различайте разные области слияния анимации по тегу.

```
- (void)tcePlayerTagTouchBegan:(ITCEffectPlayer *)player tag:(NSString *)tag {    NSString *content = [NSString stringWithFormat:@"Clicked: %@", tag];    [self.view tuiad_alert:content];}
```

#### Получение свойств анимации

Для получения информации о ширине, высоте, длительности и свойств слияния анимации перед воспроизведением можно использовать следующее решение:

```
TCEffectAnimInfo* animInfo = [TCEffectAnimView preloadTCAnimInfo:playPath config:config];
```

- playPath: адрес пути анимации для получения информации, аналогично переданным параметрам при вызове метода TCEffectAnimView#startPlay().
- config: экземпляр TCEffectConfig, используемый для указания дополнительной конфигурации для анимации

> **Примечание:** При воспроизведении анимации, если вы вызвали метод TCEffectAnimView#setEffectPlayerConfig() и установили экземпляр TCEffectConfig, содержащий extendMapParams, то при вызове текущего метода preloadTCAnimInfo передайте этот экземпляр TCEffectConfig в качестве второго входного параметра для обеспечения корректного разбора, иначе просто передайте null. Для более ранних версий анимаций vap, исключающих поле vapc, текущий метод не может разобрать сообщение конфигурации анимации и вернет null. Текущий метод включает операции ввода-вывода. Обратите внимание на вызовы потоков.

### Конфигурация воспроизведения

```
// Set playback configuration, optional stepsTCEffectConfig *config = [[TCEffectConfig alloc] init];[self.alphaAnimView setEffectPlayerConfig:config];
```

TCEffectConfig в настоящее время поддерживает:

1. vapEngineType(CodecType type): имеет два значения параметров:
  - TCEPCodecTypeAVPlayer, механизм воспроизведения по умолчанию для специальных эффектов подарков анимации.
  - TCEPCodecTypeVODPlayer, механизм воспроизведения Tencent Cloud Player SDK.

> **Примечание:** В настоящее время вы можете вызвать метод setEffectPlayerConfig только перед запуском проигрывателя для установки конфигурации воспроизведения. Изменение конфигурации после начала воспроизведения не поддерживается. Два поддерживаемых в настоящее время vapEngineType применимы только к MP4-анимациям. Если вы установите vapEngineType на TCEPCodecTypeVODPlayer, вам также нужно отдельно импортировать Tencent Cloud Player SDK, а также подать заявку и зарегистрировать соответствующую лицензию.

2. freezeFrame используется для установки кадра замораживания при воспроизведении анимаций. Текущие доступные значения:
  - FRAME_NONE: значение по умолчанию. Отключить возможность freezeFrame. Проигрыватель будет воспроизводить нормально, а пауза исчезнет.
  - FRAME_FIRST: После завершения воспроизведения перезапустить воспроизведение и приостановить, когда в следующий раз появится первый кадр.
  - FRAME_LAST: когда воспроизведение завершается в первый раз, визуальное окно остается на последнем кадре.
3. enableAudioFrameCallback: включить переключатель обратного вызова аудио во время воспроизведения, подходит для сценариев, где требуется прослушивание и обработка аудиодорожек анимации подарка.

> **Примечание:** В настоящее время поддерживается только вызов метода setEffectPlayerConfig() для установки конфигурации воспроизведения перед запуском проигрывателя. Изменение конфигурации после начала воспроизведения не поддерживается. Этот параметр действует только при CodecType равном TCEPCodecTypeVODPlayer и тип анимации — MP4/TEP. После включения переключателя вы можете использовать метод TCEPAnimViewDelegate#tcePlayerAudioFrame для прослушивания обратных вызовов

4. enableProgressCallback: включить обратный вызов прогресса воспроизведения анимации.
5. progressCallbackIntervalMs: интервал обратного вызова прогресса для воспроизведения анимаций в миллисекундах, значение по умолчанию — 200 мс.

> **Примечание:** Примечание: эта конфигурация действует только при включении переключателя enableProgressCallback. После включения переключателя вы можете прослушать событие TCEPConstant.TCEP_PLAYER_EVT_PROGRESS в обратном вызове TCEPAnimViewDelegate#onPlayEvent, чтобы получить текущий прогресс воспроизведения. Не устанавливайте интервал времени слишком низким. Слишком малый интервал может повлиять на производительность.

### Управление воспроизведением

#### Начало воспроизведения

TCEffectPlayer поддерживает воспроизведение форматов файлов ресурсов .mp4, .tcmp4.

> **Примечание:** Поддерживается только воспроизведение локальных видеоресурсов. Если вы используете онлайн-видеоресурсы, загрузите их локально и повторите воспроизведение. Анимированные ресурсы можно создавать через [инструмент преобразования Gift AR](https://www.tencentcloud.com/document/product/1143/70541). Кроме того, также могут воспроизводиться анимированные ресурсы в формате VAP.

```
NSString *mp4Url = @"xxx/xxx/tuiad_vapx_cool_sss.mp4"];[self.alphaAnimView startPlay:mp4Url];
```

#### Пауза воспроизведения

```
[self.alphaAnimView pause];
```

#### Продолжение воспроизведения

```
[self.alphaAnimView resume];
```

#### Остановка воспроизведения

Когда проигрыватель больше не требуется, остановите воспроизведение и освободите занятые ресурсы.

```
[self.alphaAnimView stopPlay];
```

#### Отключение звука воспроизведения

```
[self.alphaAnimView setMute:YES];
```

#### Циклическое воспроизведение

```
[self.alphaAnimView setLoop:YES];
```

## Часто задаваемые вопросы

### Что делать, если во время процесса воспроизведения появляется следующая информация журнала?

```
License checked failed! tceffect player license required!
```

Пожалуйста, проверьте, применили ли вы для лицензии Tencent Gift AR проигрывателя и завершили ли инициализацию регистрации.

### Как извлечь логи запуска?

Извлечение логов запуска проигрывателя Gift AR. См. [Управление логами](https://www.tencentcloud.com/document/product/1143/70538#5f064c39-0316-4160-b900-ebbbdc3285ab).

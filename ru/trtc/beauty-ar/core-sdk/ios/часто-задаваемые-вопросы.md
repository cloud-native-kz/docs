# Часто задаваемые вопросы

### При запуске проекта iOS в Xcode 12.x после импорта ресурсов возникла ошибка «Building for iOS Simulator, but the linked and embedded framework '.framework'...». Что мне делать?

> **Примечание:** Если вернуть значение **Validate Workspace** в **No** после компиляции, проект всё ещё можно успешно запустить.

### Что мне делать, если при компиляции демо-проекта iOS возникает ошибка генерации `dSYM`?

```
PhaseScriptExecution CMake\\ PostBuild\\ Rules build/XMagicDemo.build/Debug-iphoneos/XMagicDemo.build/Script-81731F743E244CF2B089C1BF.sh  cd /Users/zhenli/Downloads/xmagic_s106  /bin/sh -c /Users/zhenli/Downloads/xmagic_s106/build/XMagicDemo.build/Debug-iphoneos/XMagicDemo.build/Script-81731F743E244CF2B089C1BF.shCommand /bin/sh failed with exit code 1
```

- **Причина**: Ошибка повторного подписания `libpag.framework` и `Masonary.framework`.
- **Решение**:
  1.1. Откройте файл **demo/copy_framework.sh**.
  1.2. Измените `$(which cmake)` на абсолютный путь локального CMake.
  1.3. Измените `Apple Development: ......` на вашу собственную подпись учётной записи.

### Что мне делать, если при создании демо-проекта iOS возникает ошибка?

```
unexpected service error: build aborted due to an internal error: unable to write manifest to-xxxx-manifest.xcbuild': mkdir(/data, S_IRWXU | S_IRWXG | S_IRWXO): Read-only file system (30):
```

- **Решение**:
  1.1. Перейдите в **File** > **Project settings** > **Build System** и выберите **Legacy Build System**.
  1.2. Для Xcode 13.0++, необходимо выбрать **File** > **Workspace Settings** > **Do not show a diagnostic issue about build system deprecation**.

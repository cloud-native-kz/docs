# Часто задаваемые вопросы

В этом документе ответы на вопросы, которые могут возникнуть при использовании Beauty AR Web.

### Макияж, стикеры и другие настройки не работают, а в консоли постоянно отображается сообщение "waiting for AI init".

Для уменьшения размера пакета SDK реализована модульная загрузка. При инициализации SDK настройте загружаемые модули в соответствии с параметром **module**. Макияж и стикеры зависят от **модуля beautify**, который необходимо установить на true, чтобы их включить. Для получения дополнительной информации см. [документацию API](https://www.tencentcloud.com/document/product/1143/50106).

### Что делать, если при запуске демонстрации в Chrome изображение перевернуто и есть задержки?

Так как SDK использует GPU для ускорения, необходимо включить **Use hardware acceleration when available** в параметрах браузера.

### В консоли отображается ошибка "Failed to execute 'getContext' on 'OffscreenCanvas': The provided value 'experimental-webgl' is not a valid enum value of type OffscreenRenderingContextType"

Обычно это происходит, если аппаратное ускорение в браузере не включено.

### Страница зависает при использовании SDK в 32-разрядном браузере.

SDK потребляет определенное количество памяти во время работы, обычно не превышая 500 МБ. Рекомендуется использовать 64-разрядный браузер. Лимит памяти для одной вкладки в 32-разрядном браузере обычно не превышает 1 ГБ и совместно используется с другими вкладками. Это может привести к недостатку памяти, особенно при открытии нескольких вкладок или если само приложение требует большого объема памяти, что вызывает сбой страницы браузера.

### Можно ли использовать Beauty AR Web SDK для улучшения прямых трансляций, опубликованных в веб-приложении потокового видео?

Да. SDK может работать в качестве промежуточного обработчика отрисовки для потокового видео. Он поддерживает несколько источников ввода/вывода. Информацию о том, как легко расширить веб-приложение и быстро реализовать фильтры красоты и эффекты, см. в разделах [Publishing over WebRTC](https://www.tencentcloud.com/document/product/1143/53886) и [Publishing with TRTC](https://www.tencentcloud.com/document/product/1143/53885).

### Как использовать `localhost` для локальной разработки?

Веб-лицензия будет проверять информацию о домене во время использования, поэтому необходимо убедиться, что фактический запущенный домен совпадает с доменом, указанным при подаче заявки на лицензию; в противном случае произойдет ошибка отказа при проверке домена.

Однако это не относится к адресам localhost, так как localhost автоматически валидируется. Поэтому вы можете использовать localhost для локальной разработки без каких-либо ограничений портов.

### Почему веб-сайт не может получить доступ к камере?

- Пользователь не авторизован: При первом обращении браузер выведет запрос на авторизацию. Если пользователь отказывает, ему необходимо вручную включить разрешения. Решение: Направьте пользователя на повторную авторизацию в параметрах браузера (Chrome: Параметры > Конфиденциальность и безопасность > Камера > Разрешить для указанных сайтов).
- Доступ из небезопасного окружения: Браузеры требуют HTTPS или локального окружения (localhost://127.0.0.1); доступ обычно заблокирован по HTTP. Решение: Разверните HTTPS или используйте localhost при разработке; во время отладки временно разрешите HTTP-адреса, посетив chrome://flags/#unsafely-treat-insecure-origin-as-secure.

### Почему выводится ошибка при вызове `getEffectList` для загрузки ресурсов материалов?

Обычно это происходит, если время вызова неправильно. Обязательно **вызывайте `getEffectList` когда или после обратного вызова API `created` SDK**. В этот момент логика взаимодействия с бизнесом может быть реализована на основе данных материала. Конкретные примеры использования см. в разделе [Best Practices](https://www.tencentcloud.com/document/product/1143/53886).

### Почему SDK сообщает "This project does not exist"?

Ошибка вызвана неправильной лицензией, предоставленной при инициализации SDK. Убедитесь, что лицензия заполнена правильно и находится в период действия. Кроме того, если параметры token и license поменялись местами, эта ошибка также может возникнуть.

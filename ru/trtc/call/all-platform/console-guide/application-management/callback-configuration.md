# Конфигурация обратного вызова

Сервис обратного вызова Event уведомляет ваш сервер о событиях сервиса аудио и видео в реальном времени в виде HTTP/HTTPS запросов. Сервис обратного вызова Event интегрирует некоторые события из групп Room Event и Media Event. Вы можете заполнить информацию конфигурации обратного вызова в [консоли Tencent RTC](https://console.trtc.io), следуя приведённым ниже инструкциям. После завершения конфигурации вы сможете получать уведомления о событиях обратного вызова.

## Предварительные условия

1. Перейдите в [консоль Tencent RTC > Расширенные функции](https://console.trtc.io/features), выберите целевое приложение в левом верхнем углу. Кроме того, вы можете нажать кнопку **Create**, чтобы быстро создать новое приложение здесь.
2. Перейдите на страницу конфигурации Расширенных функций целевого приложения и установите конфигурацию обратного вызова в разделе **Callback Configuration**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/bcd8ef6bcb6511f087ad52540099c741.png)

## Установка ключа и URL обратного вызова

1. Нажмите в верхнем правом углу вкладки Callback Configuration на кнопку **edit**.
2. Установите ключ обратного вызова (опционально) в зависимости от требований сервиса.
3. Введите URL обратного вызова в зависимости от требований сервиса (обязательно):
  - Обратный вызов события комнаты: уведомление о событиях создания/растворения комнат, входа/выхода из комнат и т. д. [Дополнительные описания событий обратного вызова.](https://www.tencentcloud.com/document/product/647/39558)
  - Обратный вызов события медиа: уведомления о событиях запуска/остановки передачи видеоданных, запуска/остановки передачи аудиоданных, запуска/остановки передачи данных дополнительного канала и другие. [Дополнительные описания событий обратного вызова.](https://www.tencentcloud.com/document/product/647/39558)
  - Обратный вызов записи: используется для обратного вызова событий, связанных с функцией [облачной записи](https://trtc.io/document/45169), и поддерживает уведомление о событиях запуска и выхода модуля облачной записи, запуска модуля загрузки облачной записи и завершения загрузки. [Дополнительные описания событий обратного вызова.](https://www.tencentcloud.com/document/product/647/54914)
  - Обратный вызов трансляции в CDN: обратные вызовы событий, используемые для включения функции [Трансляция в CDN](https://trtc.io/document/48245). [Дополнительные описания событий обратного вызова.](https://www.tencentcloud.com/document/product/647/54913)

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/aa4bf65ecb6511f08942525400e889b2.png)

> **Примечание:** Заголовок протокола URL обратного вызова: HTTP, HTTPS, может содержать только следующие символы: a-z, A-Z, 0-9, -, _, ?, %, =, #, ., /, и +, максимум 2083 символа.

4. Нажмите **Confirm**, чтобы успешно установить URL обратного вызова.

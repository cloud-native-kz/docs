# Создание приложения

Tencent RTC управляет различными бизнесами или проектами через приложения. Разные приложения можно отдельно создавать для различных бизнесов или проектов в консоли TRTC, что упрощает изоляцию данных между бизнесами или проектами.

## Важные моменты

Каждый аккаунт Tencent Cloud может содержать максимум 300 приложений Tencent RTC.

## Создание приложения

1. Войдите в [консоль Tencent RTC](https://console.trtc.io) и нажмите **Create Application**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/168f92fbcb5b11f08942525400e889b2.png)

2. Введите имя приложения в всплывающее окно создания, выберите продукт на основе фактических потребностей бизнеса, выберите подходящий **Deployment Region** и нажмите **Create**.

> **Примечание:** По умолчанию хранилище данных услуги передачи аудио и видео в реальном времени находится в Сингапуре, а хранилище данных услуги обмена сообщениями находится в выбранном вами центре данных.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1d2eaf50cb5b11f09e745254007c27c5.png)

3. После создания приложения по умолчанию вы войдете на выбранную страницу сведений о продукте. Вы можете нажать **Developer Docs** для быстрого ознакомления с примером проекта. Просмотрите `SDKAppID` и `SDKSecretKey` в разделе **Basic Information**, так как они будут использоваться на следующих этапах.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/23a1838ccb5b11f0aedb525400454e06.png)

## Просмотр списка приложений

После успешного создания приложения информация созданного приложения будет представлена на панели обзора. Нажмите кнопку **Manage** справа от приложения, которое вы хотите просмотреть, чтобы проверить его основную информацию.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/3568cd57cb5b11f0aedb525400454e06.png)

Отображаемая основная информация включает имя приложения, SDKAppID, SDKSecretKey, описание приложения, TAG, статус приложения, ключ, время создания приложения и область обслуживания. Кроме того, вы можете проверить, включены ли расширенные функции и какие продукты использовались.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/39c5bae7cb5b11f087ad52540099c741.png)

| **Пункт информации** | **Описание** |
| --- | --- |
| Application Name | Установите пользовательское имя приложения при создании приложения. |
| SDKAppID | SDKAppID, автоматически сгенерированный после создания приложения, служит уникальным идентификатором приложения. Этот параметр должен быть предоставлен при вызове интерфейса VOICE API. |
| SDKSecretKey | Ключевая информация для инициализации файла конфигурации SDK. |
| Application Description | У вас есть возможность создать пользовательское описание приложения. |
| Application Version | По умолчанию двигатель RTC установлен на базовую версию. Вы можете обновить на более высокую версию, чтобы разблокировать дополнительные функции в соответствии с потребностями вашего бизнеса. Дополнительную информацию см. в разделе [RTC-Engine Packages](https://www.tencentcloud.com/document/product/647/56025). Для использования приложения Call необходимо получить бесплатную пробную версию. Вы можете обновить версию на более высокую в соответствии с потребностями вашего бизнеса, чтобы разблокировать соответствующие функции добавленной стоимости. [TRTC Call Monthly Packages](https://www.tencentcloud.com/document/product/647/54632) |
| Service Status | Текущий статус обслуживания приложения состоит из двух состояний: **Normal и Disabled**. Если статус обслуживания указывает на "Disabled" и это не было приостановлено вручную, убедитесь, является ли [баланс пакета](https://console.trtc.io/statistics) нулевым, и имеется ли задолженность [учетной записи Tencent Cloud](https://console.tencentcloud.com/expense). |
| Creation Time | Время, когда приложение было успешно создано. |
| Deployment Region | Область хранилища развертывания сервиса. Данные сервиса Tencent Real-Time Communication (TRTC) по умолчанию хранятся в Сингапуре, а данные сервиса Chat хранятся в выбранном вами центре данных. |

## Связанные документы

Для ознакомления с фундаментальной подробной информацией о приложении обратитесь к Application Information.

Если вам нужно организовать или просмотреть детали конфигурации функций приложения, обратитесь к Function Configuration.

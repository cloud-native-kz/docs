# Конфигурация функций

После создания приложения вы можете включить функции [Трансляция на CDN](https://www.tencentcloud.com/document/product/647/39080#bypass), [Облачная запись](#record) и [расширенное управление правами доступа](#advanced_permission) для вашего приложения через конфигурацию функций. Изменение вступает в силу примерно через 5 минут после его внесения.

## Трансляция на CDN

### Примечание

- Сервис TRTC на основе протокола транспорта UDP подключает потоки аудио и видео к [CSS](https://www.tencentcloud.com/document/product/267) через преобразование протокола, это называется "Трансляция на CDN".
- Трансляция на CDN отключена по умолчанию. CSS должна быть включена перед включением функции трансляции на CDN.
- При использовании трансляции на CDN для просмотра CDN прямой трансляции CSS будет взимать связанные сборы на основе нисходящего трафика/полосы пропускания, созданной прямой трансляцией. Дополнительные сведения см. в разделе [CSS > Описание биллинга трафика и полосы пропускания.](https://www.tencentcloud.com/document/product/267/2819)
- Вы можете использовать трансляцию на CDN для отправки аудио и видео на CSS. Если вы используете возможность записи CSS для завершения записи, будут взиматься сборы за запись и хранение файлов. Дополнительные сведения см. в разделе [Биллинг облачной записи и доставки записей](https://www.tencentcloud.com/document/product/647/45176).
- Если вы в консоли выполняете трансляцию на CDN переадресацию имени домена (xxxx.livepush.myqcloud.com) с привязанной записью, транскодированием, идентификацией порнографии, водяным знаком и другими функциями шаблона тарификации, трансляция на CDN будет создавать дополнительные затраты, соответствующие шаблону.

### Включение функции трансляции на CDN

1. Перейдите в [консоль Tencent RTC > Расширенные функции](https://console.trtc.io/features), выберите целевое приложение в верхнем левом углу. Вы также можете нажать **Create**, чтобы быстро создать новое приложение здесь.
2. Введите страницу конфигурации расширенных функций целевого приложения, нажмите боковую кнопку рядом с **Bypass Redirect Configuration** в разделе **Global Auto Relay**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/88e1c503cb6211f0a7775254005ef0f7.png)

3. В появившемся диалоговом окне внимательно прочитайте описание рисков. Если да, нажмите **Save**, чтобы включить трансляцию на CDN.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/daeb7075509711efb66652540055f650.png)

### Переключение имени домена трансляции на CDN

1. Ваше текущее имя домена по умолчанию называется xxxx.livepush.myqcloud.com/xxx.tlivecloud.com, поддерживает переключение вашего имени домена в консоли CSS. Если вам нужно переключиться на другие имена доменов, сначала перейдите в консоль CSS для привязки.

> **Примечание:** После изменения имени домена по умолчанию шаблон записи, шаблон обратного вызова, шаблон желтой аутентификации и шаблон водяного знака, привязанные к текущему имени домена, станут недействительными. Привяжите эти шаблоны к новому имени домена, а затем завершите переключение. После вступления в силу переключения имени домена аудио и видео, созданные в новой комнате, будут отправлены на основе нового имени домена.

2. Перейдите в [консоль Tencent RTC > Приложения](https://console.trtc.io/app), выберите приложение, для которого вы хотите изменить конфигурацию функций, и нажмите **Manage**. Выберите **Advanced Features** из колонки проекта слева, а затем нажмите **View Details** на трансляции на CDN и переключите имя домена трансляции на CDN на странице конфигурации.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/f1fe8476509711ef8357525400bdab9d.png)

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e9d86f76509811ef8357525400bdab9d.png)

### Отключение трансляции на CDN

Для отключения трансляции на CDN выполните следующие действия:

1. Войдите в [консоль Tencent RTC > Приложения](https://console.trtc.io/app), выберите приложение, для которого вы хотите изменить конфигурацию функций, и нажмите **Manage**.
2. Выберите **Advanced Features** из колонки проекта слева, а затем нажмите кнопку справа от **Global auto relay** в разделе **Relay to CDN**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1c79c1bc509911efb927525400fdb830.png)

3. В появившемся диалоговом окне **disable global auto relay** внимательно прочитайте описание рисков. Если да, нажмите Confirm.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/4174836d509911ef9bf1525400a9236a.png)

## Конфигурация облачной записи

### Примечание

- Функция облачной записи использует возможности TRTC. Сборка за запись взимается со стороны TRTC. Дополнительные сведения о правилах тарификации см. в разделе [Описание биллинга облачной записи](https://trtc.io/document/45176).
- Если записанный файл сохраняется на платформе VOD, сборка за хранилище рассчитывается на основе емкости хранилища записанного файла. Дополнительные сведения о правилах тарификации см. в разделе [Описание цены хранилища видео VOD (ежедневный расчет)](https://www.tencentcloud.com/document/product/266/14666#.E8.AE.A1.E8.B4.B9.E4.BB.B7.E6.A0.BC:~:text=%E8%B4%AE%E5%AD%98-,%E5%AE%9A%E4%BB%B7,-%E8%B4%A6%E5%8D%95%E8%AF%A6%E6%83%85) или [Описание цены пакета ресурсов хранилища видео VOD](https://www.tencentcloud.com/document/product/266/52806).
- Если записанный видеофайл необходимо воспроизвести или загрузить, сервис VOD взимает плату за ускорение видео. Плата рассчитывается на основе нисходящего ускоренного трафика. Дополнительные сведения о правилах тарификации см. в разделе [Описание цены ускорения видео VOD (ежедневный расчет)](https://www.tencentcloud.com/document/product/266/14666#.E5.8A.A0.E9.80.9F.E6.9C.8D.E5.8A.A1.5B.5D(id.3Aspeed)) или [Описание цены пакета ресурсов ускорения видео VOD](https://www.tencentcloud.com/document/product/266/52806#.E6.B5.81.E9.87.8F.E8.B5.84.E6.BA.90.E5.8C.85).
- Функция облачной записи отключена по умолчанию. Для включения функции облачной записи необходимо предварительно включить CSS и VOD.

### Глобальная автоматическая запись

Облачная запись предоставляет автоматический метод записи, который не требует ручного инициирования и управления задачами записи. Для использования этого метода необходимо настроить глобальный шаблон автоматической записи и включить глобальную автоматическую запись. Вам не нужно разрабатывать или управлять инициированием и прекращением задач записи. Включите глобальную автоматическую запись. После вступления в силу в течение 5-10 минут задача записи будет запущена, когда трансляторы в комнате TRTC загружают аудио и видео, и задача записи будет остановлена, когда трансляторы в комнате выходят и превышают установленное время ожидания. Перед использованием облачной записи перейдите в [консоль Tencent RTC > Расширенные функции](https://console.trtc.io/features), выберите целевое приложение в верхнем левом углу **Select Target Application** (или нажмите **Create**, чтобы быстро создать новое приложение здесь) и введите страницу конфигурации расширенных функций целевого приложения, чтобы включить облачную запись, как показано ниже:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/d43ba017cb6211f0a4a55254001c06ec.png)

Облачная запись поддерживает запись аудио и видео каждого трансляторе в комнате в отдельный файл (запись с одним потоком), а также поддерживает смешивание аудио и видео нескольких трансляторов в комнате в один файл (запись со смешанным потоком). Вы можете одновременно выбрать режимы одного потока и слияния для записи, действительные только для вновь создаваемых комнат после включения. Эта функция не применяется к комнатам, созданным перед включением автоматической записи. Подробную реализацию см. в разделе [Реализация облачной записи и воспроизведения](https://trtc.io/document/45169).

#### **Глобальная запись с одним потоком**

Формат глобальной записи с одним потоком поддерживает запись аудио и видео, запись только аудио, запись только видео, файлы записи поддерживают MP4, HLS и AAC (формат только для аудио записи), подробные сведения о политике нарезки файла записи см. в разделе [Описание нарезки файла записи](https://trtc.io/document/45169#.E5.BD.95.E5.88.B6.E6.96.87.E4.BB.B6.E5.90.8D.E5.91.BD.E5.90.8D.E8.A7.84.E5.88.99).

| Функции | Описание |
| --- | --- |
| Режим записи | Запись с одним потоком: видеозаписи каждого трансляторе в комнате сохраняются в отдельный файл. Для записи смешанного видео нескольких трансляторов используйте [Ручное слияние записей](https://www.tencentcloud.com/document/product/647/45169#.3Cu.3Emanual-recording-process.3C.2Fu.3E). |
| Формат записи | Формат аудио и видео: записывайте потоки аудио и видео в комнате, подходит для видеовызовов и сценариев интерактивной прямой трансляции. Только формат аудио: записывайте только потоки аудио в комнате. |
| Формат файла | Поддерживаются MP4, HLS и AAC (только формат для аудио). |
| Длительность записи одного файла | Значение, которое можно использовать для **указания длительности нарезки записи**, варьируется от 1 минуты до 1440 минут. Значение по умолчанию составляет 1440 минут. |
| Продолжительность непрерывного ожидания | Если интервал перерыва не превышает указанный период ожидания, для одного вызова (или прямой трансляции) создается только один файл, но записанный файл может быть получен только после истечения периода ожидания. Значение варьируется от 5 до 86400 в секундах. (Плата за аудио будет взиматься в течение периода продолжения, пожалуйста, установите продолжительность продолжения разумно.) Примечание: в режиме со смешанным потоком черные кадры и немые кадры продолжают записываться в течение периода ожидания. В режиме одного потока черные кадры не записываются. В течение периода ожидания продолжения записи с одним потоком и со смешанным потоком взимаются платежи за длительность аудио. |
| Хранилище записанного файла | Поддерживает хранилище в [VOD](https://console.tencentcloud.com/vod/overview) VOD: поддерживает указанное приложение VOD, время хранения записанных файлов в VOD и привязку рабочего процесса задачи VOD. |

> **Примечание:** В режиме записи с одним потоком потоки аудио и видео в комнате будут записаны отдельно в соответствии с параметрами отправки без необходимости установки транскодирования. Робот записи будет продолжать ждать, пока трансляторе поднимется в комнате, чтобы завершить запись, и не закончится сразу после того, как трансляторе выйдет. Пожалуйста, установите его правильно.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/5e29499e509911efb927525400fdb830.png)

> **Примечание:** Запись одного потока может записать максимум 25 трансляторов в комнате. Если записано более 25 трансляторов, первые 25 трансляторов будут записаны в соответствии с временем входа. (Если вам нужно записать более 25 трансляторов в один поток, см. API [Ручная запись](https://trtc.io/document/46960).) Время ожидания продолжения по умолчанию составляет 5 сек. Если время без трансляторе в комнате превышает установленное время продолжения, оно будет разделено на несколько задач записи, а соответствующие файлы записи также независимы. Если время без трансляторе в комнате не превышает время продолжения, задачи записи будут сохранены и файлы записи не будут разделены.

### Ручная пользовательская запись

Если вам нужно использовать только определенные комнаты для инициирования задач записи, вы можете сделать это вручную через [CreateCloudRecording](https://trtc.io/document/46960).

### Отключение облачной записи

После включения функции облачной записи "Ручная пользовательская запись" включена по умолчанию и не может быть отключена. "Ручная пользовательская запись" генерирует использование записи на основе вашего инициирования. Если вам нужно использовать только "Ручную пользовательскую запись", отключите "Глобальную автоматическую запись".

### Управление файлами записи

После записи вы можете просмотреть записанный файл на интерфейсе [управления аудио и видео VOD](https://console.tencentcloud.com/vod/media). Мы загрузим файл в указанное вами место хранения, и вы сможете просмотреть его после завершения загрузки.

Фильтруя указанные приложения VOD, вы можете просмотреть соответствующие файлы результатов записи.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/740a05e5454a11ee9f16525400a32c73.png)

Нажав на имя файла, вы можете просмотреть подробную информацию о файле, включая предварительный просмотр видео, получить URL-адреса файлов и загрузить файлы.

> **Примечание:** В зависимости от размера записанного файла время загрузки может быть задержано на 30 секунд до нескольких минут. Вы можете подписаться на событие обратного вызова записи, чтобы получить успешное время загрузки файла.

## Расширенное управление правами доступа

Если вы хотите добавить ограничения доступа к комнате или ограничения доступа к некоторым комнатам, то есть разрешить входить в комнату или получать доступ к микрофону только указанным пользователям, и вы беспокоитесь, что суждение разрешений клиента легко столкнется с атаками на взлом, тогда вы можете рассмотреть включение [расширенного управления правами доступа](https://trtc.io/document/35157).

### Примечание

После включения расширенного управления правами доступа все пользователи текущего SDKAppID должны правильно передать параметр privateMapKey в TRTCParams, чтобы успешно войти в комнату. Если есть пользователь, использующий этот SDKAppID в Интернете, не включайте эту функцию.

### Включение расширенного управления правами доступа

1. Перейдите в [консоль Tencent RTC > Расширенные функции](https://console.trtc.io/features), выберите целевое приложение в верхнем левом углу. Вы также можете нажать **Create**, чтобы быстро создать новое приложение здесь.
2. Введите страницу конфигурации расширенных функций целевого приложения, нажмите кнопку справа от **Enable Advanced Permission Control**, а затем нажмите **Confirm** во всплывающем окне, чтобы завершить включение.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/ee271362cb6211f0a93d52540044a08e.png)

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/ee347ec2cb6211f0aedb525400454e06.png)

### Отключение расширенного управления правами доступа

1. Перейдите в [консоль Tencent RTC > Расширенные функции](https://console.trtc.io/features), выберите целевое приложение в верхнем левом углу.
2. Введите страницу конфигурации расширенных функций целевого приложения, нажмите кнопку справа от **Enable Advanced Permission Control**, а затем нажмите **Confirm** во всплывающем окне, чтобы завершить отключение.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/f70f8188cb6211f0a7775254005ef0f7.png)

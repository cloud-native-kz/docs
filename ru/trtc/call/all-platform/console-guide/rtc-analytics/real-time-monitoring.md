# Мониторинг в реальном времени

Возможность мониторинга данных в реальном времени обеспечивает отображение производительности приложений TRTC в реальном времени, в многомерном виде и с визуализацией, позволяя вам отслеживать приложения в реальном времени и своевременно обнаруживать и устранять проблемы.

> **Примечание:** Мониторинг данных стал платной услугой с 1 ноября 2022 года. По умолчанию предоставляется бесплатная версия. Вы можете приобрести [платную версию](https://console.trtc.io/subscription/buy/dashboard?packType=standard) для разблокировки дополнительных функций.

## Функции

Мониторинг в реальном времени:

- Статистика, включая количество активных комнат и пользователей, собирается и анализируется автоматически с интервалом обновления 10 секунд.
- Данные мониторинга в реальном времени связаны со статистикой по деталям вызовов, что облегчает устранение проблем, связанных с пользователями.
- Данные отображаются визуально в многомерном виде. Вы можете просматривать статистику в определённый момент времени.

## Инструкции

1. Войдите в [Консоль Tencent RTC > Мониторинг данных > Мониторинг в реальном времени](https://console.trtc.io/realtime-monitor) на левой боковой панели. Если вы ещё не активировали сервис мониторинга данных, сначала активируйте его.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/4f955f45d4d911f0a73852540099c741.png)

2. Выберите приложение (`SDKAppID`) для мониторинга. Обратите внимание, что вам нужно подождать примерно 10 минут после создания приложения, прежде чем сможете просматривать его данные.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/df3bd8ea9a4811eea869525400c26cb9.png)

### Обзор

- Данные на странице мониторинга в реальном времени обновляются каждые 20 секунд (вы можете найти время последнего обновления рядом с **Последние данные**).
- **Выберите приложение**: выберите приложение для просмотра данных всех комнат под этим приложением.
- **Приостановить данные**: нажмите **Приостановить данные**, чтобы зафиксировать данные на текущий момент времени, и нажмите **Просмотреть последние данные**, чтобы разблокировать их.

### Последние данные

В разделе **Последние данные** показаны активные пользователи, аномальные пользователи, активные комнаты, частота зависаний видео и частота зависаний аудио за последние 20 секунд. Подробные сведения см. в разделе [Показатели мониторинга](#index). Данные обновляются каждые 20 секунд.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/09f272ea9a4911ee8964525400321be2.png)

### Детали

В разделе **Детали** показаны данные по пяти различным показателям масштаба и качества. Вы можете приостановить данные на текущий момент времени.

- **Состояние времени**

Пользователи могут переключаться между приостановленным состоянием и последним состоянием, нажимая кнопку **Приостановить данные** / **Просмотреть последние данные**, или переключаться в приостановленное состояние двойным щелчком по узлу на изображении. В заблокированном состоянии данные фиксируются на определённый момент времени; в последнем состоянии они обновляются каждые 20 секунд.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/c389c5219a4911ee8964525400321be2.png)

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/c71cc7989a4911ee8964525400321be2.png)

- **Показатели масштаба**
В этой области используются линейчатые диаграммы для отображения количества активных пользователей (аномальных пользователей) и комнат выбранного приложения за последние 64 минуты. Данные обновляются каждые 10 секунд. Вы можете навести указатель мыши на любое место на диаграммах для просмотра данных в определённый момент времени и дважды щелкнуть диаграмму, чтобы приостановить данные на текущий момент времени.
- **Показатели качества**
В этой области используются линейчатые диаграммы для отображения средней частоты зависаний видео и аудио, а также количества аномальных пользователей за последние 64 минуты во всех комнатах выбранного приложения. Данные обновляются каждые 10 секунд. Вы можете навести указатель мыши на любое место на диаграммах для просмотра данных в определённый момент времени и дважды щелкнуть диаграмму, чтобы приостановить данные на текущий момент времени.

Пример: на следующем изображении показана часть страницы в приостановленном состоянии.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/53d04cb29a4a11eea869525400c26cb9.png)

### Данные комнаты

В этой области отображаются идентификатор комнаты, активные пользователи, аномальные пользователи, частота зависаний видео, частота зависаний аудио и операции активных комнат.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/989d4549a61d11ee9fd6525400bb593a.png)

- **Приостановить данные:** поддерживается приостановка данных для наблюдения деталей комнаты в определённый момент времени. Пользователи могут нажать кнопку **Приостановить данные** для переключения в приостановленное состояние и нажать ещё раз, чтобы вернуться в последнее состояние.
- **Загрузить данные комнаты**: нажмите **Загрузить данные комнаты**, чтобы загрузить файл в формате xlsx с данными комнаты на текущий момент времени.
- **Данные аномальной комнаты**: нажмите опцию **Exception** в поле «Операция», чтобы просмотреть данные аномальной комнаты, включающие 8 полей: идентификатор комнаты, идентификатор пользователя, исключение, регион, сеть, устройство, версия SDK, поставщик услуг интернета. Вы можете фильтровать 8 измерений для поиска аномальных данных.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1cb9de759a5811ee8964525400321be2.png)

- **Детали вызова**: нажмите опцию **Call Details** в поле «Операция» для просмотра данных качества вызова пользователей в комнате. Подробное описание см. в разделе [Детали вызова](https://www.tencentcloud.com/document/product/647/39070).

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b0e28a95a61d11eeae9a525400c26da5.png)

При выполнении вышеприведённых действий (загрузка данных комнаты, данные аномальной комнаты и детали вызова) данные будут приостановлены на текущий момент времени. Чтобы возобновить просмотр последних данных, нажмите **Просмотреть последние данные** рядом с **Детали**.

### Многомерные данные

- В этой области показаны данные из различных измерений, таких как регион, сеть, устройство, версия SDK и поставщик услуг интернета.
- Поддерживается приостановка данных для наблюдения информации в определённый момент времени.
- Вы также можете загрузить данные из этой области.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/085efb289a5911eeb6c6525400e46040.png)

## Ключевые показатели

### Показатели масштаба

| Показатель | Описание |
| --- | --- |
| Активные пользователи | Общее количество пользователей, вошедших в комнаты выбранного приложения за последние 10 секунд. Пользователь, вошедший в несколько комнат, учитывается несколько раз. Пользователь, вошедший в одну и ту же комнату несколько раз, учитывается только один раз. |
| Активные комнаты | Общее количество комнат, в которые вошли пользователи за последние 10 секунд |
| Аномальные пользователи | Общее количество аномальных пользователей (учитанных по идентификаторам пользователей) в комнатах выбранного приложения за последние 10 секунд. Аномальные пользователи — это пользователи, столкнувшиеся с исключением. Исключения: отсутствие аудио, аномальная частота кадров видео, высокое использование ЦП. |

### Показатели качества

| Показатель | Описание |
| --- | --- |
| Частота зависаний видео | Длительность зависания (любое зависание, продолжающееся 600 мс или дольше) / Общая длительность видео × 100%, обновляется каждые 10 секунд |
| Частота зависаний аудио | Длительность зависания (любое зависание, продолжающееся 200 мс или дольше, учитывается) / Общая длительность аудио × 100%, обновляется каждые 10 секунд |
| Тяжесть зависания видео | Низкая: [0,00% - 5,00%) (зелёный). Средне-высокая: [5,00% - 10,00%) (жёлтый). Высокая: ≥ 10,00% (красный). |
| Тяжесть зависания аудио | Низкая: [0,00% - 3,00%) (зелёный). Средне-высокая: [3,00% - 5,00%) (жёлтый). Высокая: ≥ 5,00% (красный). |

## RESTful APIs

Вы также можете использовать предоставленные RESTful API для запроса данных мониторинга в реальном времени.

> **Примечание:** Только пользователи, активировавшие версию **Standard** панели управления, могут использовать RESTful API.

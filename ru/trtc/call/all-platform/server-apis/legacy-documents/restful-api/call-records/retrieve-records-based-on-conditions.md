# Получение записей на основе условий

Администраторы могут получать доступ к записям вызовов на основе указанных условий через этот интерфейс.

> **Примечание:** RESTful API в настоящее время находится в бета-версии. Вы можете запрашивать данные вызовов, созданные в течение последних семи дней.

## Описание API

### Пример URL запроса

```
https://xxxxxx/v1/records/get_records_by_filter?sdkappid=88888888&identifier=administrator&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже указаны только параметры, которые изменяются при вызове этого API, и их описание. Для получения дополнительной информации обратитесь к [Обзору REST API](https://www.tencentcloud.com/document/product/647/60138#).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Зарезервированный домен для страны/региона, где расположен SDKAppID: `callkit-intl.trtc.tencent-cloud.com` |
| v1/records/get_records_by_filter | API запроса |
| sdkappid | SDKAppID, назначенный консолью при создании приложения |
| identifier | Использование учетной записи администратора Chat |
| usersig | Подпись, созданная учетной записью администратора приложения. Для получения подробной информации обратитесь к [Создание UserSig](https://www.tencentcloud.com/document/product/647/39074#) |
| random | Введите случайное 32-битное целое число без знака, диапазон 0-4294967295 |
| contenttype | Формат запроса фиксированное значение `json` |

### Максимальная частота вызовов

200 запросов/сек.

### Пример пакетов запроса

**Базовая форма**
Создать комнату

```
{    "startTimestamp": 1618705638,     "endTimestamp": 1618705738,     "callResult": "normal_end",     "callType": "single",     "numberPerPage": 20,     "page": 2 }
```

### Поля пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| startTimestamp | Integer | Обязательное | Временная метка начала вызова (в секундах) |
| endTimestamp | Integer | Необязательное | Временная метка окончания вызова (в секундах). Если не указано, значение по умолчанию — 7 дней |
| callResult | String | Необязательное | Результат вызова. Если не указано, значение по умолчанию — все результаты |
| callType | String | Необязательное | Тип вызова. Если не указано, значение по умолчанию — все типы |
| numberPerPage | Integer | Необязательное | Количество запросов на странице, значение по умолчанию: 100 |
| page | Integer | Необязательное | Номер страницы запроса. Если не указано, значение по умолчанию — первая страница |

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| errorCode | Integer | Код ошибки, 0 указывает на успех |
| errorMessage | String | Сообщение об ошибке |
| requestId | String | Уникальный ID запроса |
| totalNum | Integer | Общее количество для этого запроса |
| page | Integer | Когда page > 0, это указывает, что есть больше данных. Увеличение page на 1 запросит последующие данные. |
| callRecordList | Array | Обратитесь к: Описание одной записи вызова |

### Описание одной записи вызова

| Поле | Тип | Описание |
| --- | --- | --- |
| callId | String | Уникальный ID вызова |
| sdkAppId | Integer | Ваш sdkAppId |
| mediaType | String | Тип мультимедиа`video` Видеовызов`audio` Голосовой вызов |
| roomId | String | ID комнаты вызова |
| startCallTs | Integer | Временная метка начала вызова (в секундах) |
| acceptTs | Integer | Временная метка подключения вызова (в секундах) |
| endTs | Integer | Временная метка окончания вызова (в секундах) |
| caller | String | userId звонящего |
| totalUserNumber | Integer | Общее количество участников в вызове |
| callType | String | Тип вызова`single` Один-на-один вызов`group` Групповой вызов |
| callResult | String | Результат вызова:`cancel` Отмена: звонящий отменил вызов до подключения.`reject` Отклоненный вызов: вызываемый отклонил.`not_answer` Не ответили: вызываемый не ответил вовремя.`normal_end` Завершено: вызов был подключен и завершен нормально.`call_busy` Занято: линия была занята во время вызова.`interrupt` Прерывания: вызов был прерван по причине сетевых или других проблем. |
| callees | Array | Список ID пользователей вызываемых, участвовавших в вызове |

> **Примечание:** Поле callResult может отображать все типы только для один-на-один вызовов, в то время как групповые чаты имеют только `normal_end`.

## Коды ошибок

Если нет ошибки сетевого типа (например, ошибка 502), код возврата HTTP для этого интерфейса всегда 200. Фактический код ошибки и сообщение об ошибке передаются через errorCode и errorMessage в теле ответа.
Для общих кодов ошибок (70000–79999) см. [Код ошибки](https://www.tencentcloud.com/document/product/647/60138#rest-api-.E5.85.AC.E5.85.B1.E9.94.99.E8.AF.AF.E7.A0.81).

| Код ошибки | Описание |
| --- | --- |
| 0 | Запрос выполнен успешно |
| 50001 | Текущее приложение должно приобрести пакет версии группового вызова TUICallKit для использования |
| 70011 | Записи не найдены |
| Unknown error code | Неизвестная ошибка. Пожалуйста, [отправьте заявку](https://console.intl.cloud.tencent.com/workorder/category) для связи с технической поддержкой |

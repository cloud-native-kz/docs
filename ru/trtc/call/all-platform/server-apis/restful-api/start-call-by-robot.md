# Инициирование вызова (с помощью робота)

## Описание функции

Администратор приложения может использовать этот API для инициирования вызова робота.

## Описание вызова API

### Пример URL запроса

```
https://console.tim.qq.com/v4/call_engine_http_srv/start_call_by_robot?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице указаны только параметры, участвующие в изменении при вызове этого API, и их описания. Для получения дополнительной информации о параметрах см. [Введение в REST API](https://www.tencentcloud.com/document/product/647/68926).

| Параметры | Описание |
| --- | --- |
| xxxxxx | SDKAppID находится в стране/регионе, где расположено выделенное доменное имя. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/call_engine_http_srv/start_call_by_robot | API запроса. |
| sdkappid | SDKAppID, назначенный консолью IM при создании приложения. |
| identifier | Должна быть учетная запись администратора приложения. Для получения дополнительной информации см. раздел администратор приложения. |
| usersig | Созданная подпись учетной записи администратора приложения. Для конкретных операций см. [генерирование UserSig](https://www.tencentcloud.com/document/product/647/35166). |
| random | Введите случайное 32-битное целое число без знака, значение варьируется от 0 до 4294967295. |
| contenttype | Формат запроса зафиксирован как `json`. |

### Максимальная частота вызовов

200 раз/секунду.

### Пример пакета запроса

**Базовая форма**

```
{    "Robot_Account":"user1",    "CalleeList_Account":["user5"],    "Timeout":300000,    "UserData":"userdata-12345687",    "CallInfo":{        "MediaType": "Audio",        "RoomIdType":2,        "RoomId":"roomid-1234"    },    "OfflinePushInfo": {        "PushFlag": 0,        "Title":"This is the push title",        "Desc": "This is the offline push content"        "Ext": "This is the transmitted content"    }}
```

### Поля пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| Robot_Account | String | Обязательно | UserID робота-инициатора вызова. Серверная часть не выполняет проверку сердцебиения или проверку занятости для этого UserID. |
| CalleeList_Account | Array | Обязательно | Список UserID получателей вызова. |
| Timeout | Integer | Обязательно | Период ожидания (единица: мс), диапазон: 10000 - 600000 мс. |
| UserData | String | Опционально | Пользовательские данные будут переданы стороне получателя через уведомление об инициировании вызова, максимум 1000 байт. |
| CallInfo | Object | Обязательно | Информация о вызове. |
| MediaType | String | Обязательно | Тип медиа: Видеовызов Аудиовызов |
| RoomId | String | Опционально | ID комнаты TRTC. |
| RoomIdType | Integer | Опционально | Тип RoomId: `1`: числовой ID комнаты. `2`: строковый номер комнаты. |
| OfflinePushInfo | Object | Опционально | Конфигурация уведомления об автономной отправке. См. [Описание формата сообщения](https://www.tencentcloud.com/document/product/1047/33527). |

### Пример полезной нагрузки ответа

**Базовая форма**

```
{    "ErrorCode": 0,    "ErrorInfo": "",    "ActionStatus": "OK",    "RequestId": "Id-431454f25a44462d8155bdff4fed38cc-O-Seq-19029769"}
```

### Поля пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ErrorCode | Integer | Код ошибки: 0 указывает на успех, ненулевое значение указывает на отказ. |
| ErrorInfo | String | Сообщение об ошибке. |
| ActionStatus | String | Результат обработки запроса. OK: обработка успешна; FAIL: обработка не выполнена. |
| RequestId | String | Уникальный ID запроса, возвращаемый для каждого запроса. Для устранения неполадок запроса следует предоставить его RequestId. |

## Описание кодов ошибок

Если не происходит сетевая ошибка (например, 502), код возврата HTTP этого API равен 200. Фактический код ошибки и информация об ошибке находятся в ErrorCode и ErrorInfo полезной нагрузки ответа.

Общие коды ошибок (60000 до 79999), см. документ [Код ошибки](https://www.tencentcloud.com/document/product/647/54901).

Приватные коды ошибок этого API выглядят следующим образом:

| Код ошибки | Описание |
| --- | --- |
| 101001 | Внутренняя ошибка сервера, пожалуйста, повторите попытку. |
| 101002 | Недействительный параметр. Проверьте, правильный ли запрос, согласно описанию ошибки. |
| 101004 | Вызов не существует, или он мог существовать, но уже завершился. |

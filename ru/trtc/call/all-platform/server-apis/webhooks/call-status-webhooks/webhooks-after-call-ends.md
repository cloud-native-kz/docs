# Вебхуки после окончания вызова

## Описание функции

Серверная часть приложения может отслеживать в реальном времени динамику окончания вызова через этот обратный вызов. Через пять секунд после завершения вызова детали вызова можно запросить через API запроса записи вызовов.

## Важные сведения

- Чтобы включить обратный вызов, необходимо настроить URL обратного вызова через REST API и включить переключатель, соответствующий этому протоколу обратного вызова. Метод конфигурации см. в разделе [Установка конфигурации обратного вызова](https://www.tencentcloud.com/document/product/647/68938).
- Направление обратного вызова: серверная часть Call инициирует HTTP POST запрос к серверной части приложения.
- После получения запроса обратного вызова серверная часть приложения должна проверить, что значение параметра SDKAppID в URL запроса соответствует её собственному SDKAppID.

## Сценарии срабатывания обратного вызова

Завершение вызова.

## Время срабатывания обратного вызова

Когда количество участников вызова меньше или равно 1, срабатывает завершение вызова и освобождение ресурсов.

## Описание API

### Пример URL запроса

В следующем примере URL обратного вызова, настроенный для приложения, — это `https://www.example.com`.

**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST |
| www.example.com | URL обратного вызова |
| SdkAppid | Назначенный SDKAppID в консоли Chat при создании приложения |
| CallbackCommand | Зафиксировано как `Call.CallbackAfterEndCall` |
| contenttype | Значение зафиксировано как `json` |
| ClientIP | IP-адрес клиента в формате `127.0.0.1` |
| OptPlatform | Платформа клиента. Значения параметров см. в описании параметра OptPlatform в разделе [Введение в обратный вызов третьей стороны: протокол обратного вызова](https://www.tencentcloud.com/zh/document/product/1047/34354#.E5.9B.9E.E8.B0.83.E5.8D.8F.E8.AE.AE) |

### Пример пакета запроса

```
{    "CallbackCommand":"Call.CallbackAfterEndCall",    "CallRecord": {		"CallId": "055662e1-bc8a-469c-a334-1126c8c17d58",		"Caller_Account": "10001",		"MediaType": "Audio",		"CallType": "SingleCall",		"StartTime": 1741231146,		"EndTime": 1741231296,		"AcceptTime": 0,		"CallResult": "NotAnswer",		"CalleeList_Account": ["10001", "user2"],		"RoomId": "roomid-1434",		"RoomIdType": 2	}    "EventTime":1703589922780}
```

### Поля пакета запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда обратного вызова |
| CallRecord | Struct | Информация о записи вызова |
| CallId | String | ID вызова |
| Caller_Account | String | ID вызывающей стороны |
| MediaType | String | Тип медиа: `Video` видеовызов, `Audio` аудиовызов |
| CallType | String | Тип вызова: `SingleCall` один-на-один вызов, `MultiCall` групповой вызов |
| StartTime | Integer | Временная метка инициации вызова (в секундах) |
| EndTime | Integer | Временная метка окончания вызова (в секундах) |
| AcceptTime | Integer | Временная метка подключения вызова (в секундах) |
| CallResult | String | Результат вызова: `Cancel` вызывающая сторона отменила вызов до подключения, `Reject` получатель отклонил вызов, `NotAnswer` получатель не ответил до истечения времени ожидания, `NormalEnd` вызов подключен и закончился нормально, `CallBusy` занято, `Interrupt` вызов прерван из-за проблем с сетью, `Offline` сервер обнаружил истечение сердцебиения, что привело к окончанию вызова. |
| CalleeList_Account | Array | Список участников вызова |
| RoomId | String | ID комнаты TRTC |
| RoomIdType | Integer | Тип RoomId: `1` номер цифровой комнаты, `2` номер строковой комнаты |
| EventTime | Integer | Временная метка срабатывания события в миллисекундах |

### Пример пакета ответа

Пакет ответа возвращается после синхронизации данных серверной частью приложения.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // Игнорировать результат обратного вызова.}
```

### Поля пакета ответа

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательно | Результат обработки запроса. OK: успешная обработка; FAIL: ошибка обработки |
| ErrorInfo | String | Обязательно | Сообщение об ошибке |
| ErrorCode | Integer | Обязательно | Код ошибки. Введите здесь 0, чтобы игнорировать результат обратного вызова |

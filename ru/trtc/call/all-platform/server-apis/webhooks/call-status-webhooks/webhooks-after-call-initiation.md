# Вебхуки после инициирования вызова

## Описание функции

Серверная часть приложения может просматривать динамику начала вызова в реальном времени через этот обратный вызов.

## Важная информация

- Чтобы включить обратный вызов, необходимо настроить URL обратного вызова через REST API и включить переключатель, соответствующий этому протоколу обратного вызова. Метод настройки см. в разделе [Установка конфигурации обратного вызова](https://www.tencentcloud.com/document/product/647/68938#).
- Направление обратного вызова: серверная часть Call инициирует HTTP POST запрос к серверной части приложения.
- После получения запроса обратного вызова серверная часть приложения должна проверить, является ли значение параметра SDKAppID в URL запроса её собственным SDKAppID.

## Сценарии срабатывания обратного вызова

Пользователь приложения инициирует успешный вызов через клиент.

## Время срабатывания обратного вызова

После инициирования вызова.

## Описание API

### Пример URL запроса

В следующем примере URL обратного вызова, настроенный для приложения: `https://www.example.com`.

**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST |
| www.example.com | URL обратного вызова |
| SdkAppid | Назначенный SDKAppID в консоли Chat при создании приложения |
| CallbackCommand | Зафиксирован как `Call.CallbackAfterStartCall` |
| contenttype | Зафиксированное значение `json` |
| ClientIP | IP-адрес клиента в формате `127.0.0.1` |
| OptPlatform | Платформа клиента. Значение параметра см. в OptPlatform в разделе [Введение в обратные вызовы третьих сторон: Протокол обратного вызова](https://www.tencentcloud.com/zh/document/product/1047/34354#.E5.9B.9E.E8.B0.83.E5.8D.8F.E8.AE.AE) |

### Пример пакета запроса

```
{    "CallbackCommand":"Call.CallbackAfterStartCall",    "CallInfo":{        "CallId":"055662e1-bc8a-469c-a334-1126c8c17d58",        "CallMediaType":"Audio",        "ChatGroupId":"",        "RoomId":"",        "RoomIdType":1    },    "UserList":[        {            "User_Account":144115212830834855,            "Status":"Calling"        },        {            "User_Account":144115212948889925,            "Status":"Waiting"        }    ],    "EventTime":1740464128807}
```

### Поля пакета запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда обратного вызова |
| CallId | String | ID вызова |
| MediaType | String | Тип медиа: `Video` видеовызов, `Audio` аудиовызов |
| ChatGroupId | String | ID группы чата |
| RoomId | String | ID комнаты TRTC |
| RoomIdType | Integer | Тип ID комнаты TRTC: `1` номер комнаты цифрой, `2` строковый номер комнаты |
| UserList | Array | Список участников вызова |
| User_Account | String | ID пользователя, инициирующего вызов |
| Status | String | Статус вызова: `Calling` в вызове, `Waiting` ожидание подключения |
| EventTime | Integer | Временная метка события в миллисекундах |

### Пример пакета ответа

Пакет ответа возвращается после синхронизации данных серверной частью приложения.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // Игнорировать результат обратного вызова}
```

### Поля пакета ответа

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательно | Результат обработки запроса. OK указывает на успешную обработку; FAIL указывает на ошибку |
| ErrorInfo | String | Обязательно | Сообщение об ошибке |
| ErrorCode | Integer | Обязательно | Код ошибки. Введите здесь 0, чтобы игнорировать результат обратного вызова |

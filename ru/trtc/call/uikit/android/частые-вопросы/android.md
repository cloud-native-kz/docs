# Android

### Сообщение об ошибке "The package you purchased does not support this ability"?

Если вы встречаете указанное выше сообщение об ошибке, это означает, что пакет возможностей аудио- и видеовызовов вашего текущего приложения истёк или не был активирован. Вы можете обратиться к [руководству по активации сервиса](https://www.tencentcloud.com/document/product/647/59832#), чтобы получить или активировать возможность аудио- и видеовызовов, а затем продолжить использование компонента TUICallKit.

### Когда приложение находится в автономном режиме, может ли оно отобразить представление вызова?

При одиночном вызове, если вы появитесь в сети в течение периода ожидания, будет запущено представление входящего вызова; для групповых вызовов, если вы появитесь в сети в течение периода ожидания, будут получены последние 20 необработанных групповых сообщений, и если существует приглашение на звонок, оно запустит входящий звонок. Политика отображения для входящих вызовов см.: [Политика отображения входящего вызова для вызываемого абонента](https://www.tencentcloud.com/document/product/647/51022#bfe2ed33-0611-4ca2-9aa8-f75b2a443e4a)).

### Когда приложение находится в фоновом режиме, может ли оно отобразить представление вызова?

1. TUICallKit имеет две различные политики отображения для вызываемого абонента, см. ниже: [Политика отображения входящего вызова для вызываемого абонента](https://www.tencentcloud.com/document/product/647/51022#bfe2ed33-0611-4ca2-9aa8-f75b2a443e4a).
2. До версии 2.3 TUICallKit для автоматического перемещения приложения из фонового режима на передний план было необходимо проверить, включены ли разрешения "Background self-start" или "Floating window" в приложении.

Разные производители или даже один и тот же производитель с разными версиями Android предлагают различные разрешения и названия разрешений для приложений. Например, Xiaomi 6 требует только разрешения "Background pop-ups", а Redmi требует как "Background pop-ups", так и "Display over other applications".

Как включить разрешения, см. ниже: [Соответствующие включённые разрешения](https://www.tencentcloud.com/document/product/647/51022#652ee36c-bf12-4e41-bd66-82b6dd94e42f).

3. Если вы встречаете следующие сценарии, в которых интерфейс вызова не может быть открыт, причина заключается в том, что изменения в стеке запуска приложения привели к тому, что интерфейс CallKitActivity был скрыт или удалён.

Сценарий первый: После принятия вызова, если приложение переходит в фоновый режим, а затем вы нажимаете на значок рабочего стола для входа на передний план, CallKitActivity исчезает.

Сценарий второй: Если приложение находится в фоновом режиме и при получении IncomingNotificationView вы нажимаете на значок рабочего стола для входа в приложение, CallKitActivity не отображается и IncomingNotificationView исчезает.

Сценарий третий: Когда приложение находится в фоновом режиме и получено уведомление об автономной рассылке, входящие в приложение через значок рабочего стола без нажатия на уведомление рассылки либо не смогут отобразить CallKitActivity, либо вызовут краткое мерцание CallKitActivity.

В вышеуказанных сценариях вам необходимо добавить следующий код в Activity приложения по умолчанию. Activity по умолчанию каждого приложения может отличаться. Обратитесь к конфигурации AndroidManifest.xml для получения дополнительной информации. Взяв в качестве примера страницу запуска SplashActivity для большинства приложений:

Код

Расположение

```
if (!isTaskRoot() && getIntent() != null && getIntent().hasCategory(Intent.CATEGORY_LAUNCHER) 
        && Intent.ACTION_MAIN.equals(getIntent().getAction())) {
    finish();
    return;
}
```

### Политика отображения входящего вызова для вызываемого абонента

Чтобы сделать TUICallKit адаптивным к различным бизнес-потребностям, добавить функции продукта и улучшить пользовательский опыт, TUICallKit **версии 2.3 и выше** ([Журнал выпусков](https://www.tencentcloud.com/document/product/647/51020#)) оптимизировал стратегию всплывающего отображения страницы вызова после получения вызова, как детально показано ниже:

Если вы хотите, чтобы вызываемый абонент открывал полноэкранное представление вызова при получении вызова, вы можете интегрировать код [tuicallkit-kt](https://github.com/Tencent-RTC/TUICallKit/tree/main/Android/tuicallkit-kt). По умолчанию политика отображения входящего вызова для вызываемого абонента выглядит следующим образом:

```
  TUICallKit.createInstance(context).enableIncomingBanner(true);
```

Если интерфейс входящего вызова не отображается согласно вышеуказанной стратегии, отфильтруйте журнал `onCallReceived`, чтобы проверить, было ли получено приглашение на звонок.

### **Соответствующие включённые разрешения**

Чтобы обеспечить хороший опыт звонков, рекомендуется включить разрешения "notification", "Display over other apps(Floating window)" и "Background pop-ups" в вашем приложении. Конкретные методы приведены ниже:

Включение вручную

Рекомендации по коду

После установки приложения вы можете долго нажать на значок приложения, выбрать "Application Information", затем включить разрешения "notification", "Display over other applications" и "Background pop-ups". Альтернативно, вы можете перейти в `Mobile Phone > System Settings > Application Management > Application`, чтобы вручную включить эти разрешения.

- **Разрешения на уведомления**: Для отображения push-уведомлений: обратитесь к [Разрешениям уведомлений во время выполнения](https://developer.android.com/develop/ui/views/notifications/notification-permission) и [Запросу разрешений во время выполнения](https://developer.android.com/training/permissions/requesting#request-permission) и реализуйте в соответствии с бизнес-потребностями.

- **Background pop-ups:** Используется для всплывающего отображения интерфейса, когда приложение находится в фоновом режиме (например, на телефоне VIVO).

```
fun requestPermission(context: Context?) {    //In TUICallKit,Please open both OverlayWindows and Background pop-ups permission.    PermissionRequester.newInstance(         PermissionRequester.FLOAT_PERMISSION, PermissionRequester.BG_START_PERMISSION)        .request()}
```

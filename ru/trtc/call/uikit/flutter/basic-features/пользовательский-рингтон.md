# Пользовательский рингтон

В этой статье объясняется, как заменить рингтон входящего вызова TUICallKit, который делится на **рингтон приложения** и **рингтон автономной доставки**.

## Установка рингтона приложения

Существует два способа установки рингтона приложения:

### 1. Замена аудиофайла

Если вы интегрируете компонент TUICallKit через зависимость исходного кода, вы можете заменить аудиофайлы в папке **assets\\audios**, чтобы достичь цели замены рингтона:

| Имя файла | Использование |
| --- | --- |
| phone_dialing.mp3 | Рингтон при инициировании вызова |
| phone_ringing.mp3 | Рингтон при получении вызова |

### 2. Вызов интерфейса рингтона

Вы также можете установить рингтон входящего вызова через интерфейс [setCallingBell](https://www.tencentcloud.com/document/product/647/54906#setCallingBell).

```
import 'package:tencent_calls_uikit/tencent_calls_uikit.dart';void setCallingBell() {    TUICallKit.instance.setCallingBell('flie path');}
```

## Установка режима беззвучности

Если вам не нужен звон, вы можете включить режим безмолвия через интерфейс [enableMuteMode](https://www.tencentcloud.com/document/product/647/54906#enableMuteMode).

```
import 'package:tencent_calls_uikit/tencent_calls_uikit.dart';void enableMuteMode() {    TUICallKit.instance.enableMuteMode(true);}
```

## Установка рингтона автономной доставки

### 1. iOS

VoIP push не поддерживает пользовательские рингтоны определения push. APNs push позволяет изменять параметры `call` и `groupcall` в интерфейсе `params`, включая `TUIOfflinePushInfo.iOSSound`, установку на платформе `iOS` для рингтонов автономных сообщений.

### 2. Android

> **Примечание:** Интерфейс поддерживает Huawei, Xiaomi, FCM.

Рингтон push FCM устанавливается как рингтон приложения.

Для параметров рингтона push Huawei, Xiaomi и APNs, установите поля `iOSSound` и `androidSound` `TUIOfflinePushInfo.iOSSound` при вызове Call и GroupCall.

# Виртуальный фон

TUICallKit запустил новую функцию виртуального фона, позволяющую пользователям устанавливать размытый или изображение фона во время видеозвонков. Это скрывает реальную среду вызова, защищает конфиденциальность и делает звонок более интересным. Далее в этой статье подробно описано, как использовать эту функцию в компоненте TUICallKit.

## Требования подготовки

1. Перед использованием функции виртуального фона от Tencent Cloud необходимо перейти в Консоль и активировать услуги аудио и видео для вашего приложения, а также приобрести пакет `Group Call`. Подробные инструкции см. в разделе [Активировать услугу.](https://www.tencentcloud.com/document/product/647/59832#)
2. Загрузите файл [модели виртуального фона](https://liteav.sdk.qcloud.com/sdkres/feature/virtual_background/model/versions/2.0/person_segmention_2.0.zip), распакуйте его и перетащите файл `LiteavSegmentModel.bundle` в ваш проект, убедившись, что он находится в `MainBundle`.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/cc6b4b8718f911ef88ad5254002977b6.png)

3. Измените зависимости в вашем Podfile, заменив их версией `Professional`, и укажите версию TXLiteAVSDK_Professional как `11.8.15669`.

```
pod 'TUICallKit_Swift/Professional'pod 'TXLiteAVSDK_Professional', '11.8.15669'
```

> **Примечание:** Существует соответствие между версией TRTC SDK и файлом модели, убедитесь, что номер версии совпадает с моделью, см. ниже: [Соответствие файла модели и SDK](https://www.tencentcloud.com/document/product/647/60491#8e1c4015-d0f9-4b57-8f81-4c6c23820ca7).

## Включение размытого фона

Дизайн пользовательского интерфейса TUICallKit поддерживает установку размытого фона. Вызвав следующий интерфейс, вы можете отобразить кнопку функции размытого фона на пользовательском интерфейсе. Нажатие на кнопку напрямую включит функцию размытого фона.

```
import TUICallKitTUICallKit.createInstance().enableVirtualBackground(enable: true)
```

## Установка фона изображения (необязательно)

Реализация фона изображения требует, чтобы пользователи сохранили изображение локально. После сохранения вызовите следующий интерфейс для установки (в настоящее время поддерживаются только изображения с локальным путём, сетевые изображения пока не поддерживаются).

```
import TUICallEngineTUICallEngine.createInstance().setVirtualBackground("imagePath") { code, message in}
```

## Часто задаваемые вопросы

### Включение размытого фона не реагирует или задерживается?

- Убедитесь, что вы приобрели пакет `Group Call`, подробнее см. в разделе [Активировать услугу](https://www.tencentcloud.com/document/product/647/59832#).
- Убедитесь, что файл модели загружен локально.

Если файл модели не добавлен в локальный путь, при включении функции размытого фона SDK загрузит файл модели. При нормальных условиях сети загрузка занимает 1–3 секунды; чем хуже сеть, тем дольше это займет.

- Проверьте, совпадают ли файл модели и SDK.

### Совпадение файлов модели с SDK?

TUICallKit — это сценарий видео- и аудиозвонков, реализованный на основе C SDK и TRTC SDK. Виртуальный фон — это отличительная функция, предоставляемая TRTC SDK. Важно отметить, что файл модели виртуального фона должен соответствовать версии TRTC SDK; в противном случае функция размытого фона может работать неправильно. Таблица ниже указывает отношения между файлами модели и версиями TRTC SDK:

| Версия SDK | Адрес загрузки файла модели виртуального фона |
| --- | --- |
| pod 'TXLiteAVSDK_Professional', '11.8.15669' | [Загрузить version_2.0](https://liteav.sdk.qcloud.com/sdkres/feature/virtual_background/model/versions/2.0/person_segmention_2.0.zip) |
| pod 'TXLiteAVSDK_Professional', '11.7.12001' | [Загрузить version_1.0](https://liteav.sdk.qcloud.com/sdkres/feature/virtual_background/model/versions/1.0/segmention_model_1.0.zip) |

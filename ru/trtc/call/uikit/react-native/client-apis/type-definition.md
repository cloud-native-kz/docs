# Определение типов

### callParams

В [консоли Tencent RTC](https://console.trtc.io/) параметры расширения вызова, такие как номер комнаты, время ожидания приглашения вызова, пользовательское содержимое автономной отправки и т. д.

| Параметр | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| roomId | [RoomId](https://www.tencentcloud.com/document/product/647/66840#280a991e-d6a9-4194-9c98-ab703173227f) | Нет | ID комнаты. Подробности см. в RoomId. |
| timeout | Number | Нет | Время ожидания вызова, по умолчанию: 40 секунд, единица: секунды. timeout = 0 означает отсутствие времени ожидания |
| userData | String | Нет | Пользовательские поля расширения при инициировании вызова |
| offlinePushInfo | [OfflinePushInfo](https://www.tencentcloud.com/document/product/647/66840#999e53e6-d49d-48d9-b058-d5b690019c49) | Нет | Параметры пользовательской отправки автономного сообщения |
| chatGroupId | String | Нет | ID группы чата. |

### RoomId

ID комнаты, разделенный на тип String и тип Number.

| Параметр | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| strRoomId | String | Нет | ID комнаты, тип String |
| intRoomId | Number | Нет | ID комнаты, тип String |

### OfflinePushInfo

| Параметр | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| title | String | Нет | Название уведомления автономной отправки |
| desc | String | Нет | Описание уведомления автономной отправки |
| ignoreIOSBadge | boolean | Нет | Игнорировать счетчик значков для автономной отправки (только для iOS). Если установлено значение true, сообщение не будет увеличивать количество непрочитанных сообщений в значке приложения на стороне получателя iOS. |
| iOSSound | String | Нет | Параметр звука автономной отправки (только для iOS). Когда `sound = IOS_OFFLINE_PUSH_NO_SOUND`, звука не будет при получении сообщения. Когда `sound = IOS_OFFLINE_PUSH_DEFAULT_SOUND`, системный звук будет воспроизведен при получении сообщения. Если вы хотите настроить пользовательский iOSSound, вам сначала необходимо связать аудиофайл с проектом Xcode, а затем установить имя аудиофайла (с расширением) на iOSSound. |
| androidSound | String | Нет | Параметр звука автономной отправки (только для Android, поддерживается IMSDK 6.1 и выше). Установку звуковых подсказок поддерживают только телефоны Huawei и Google. Для телефонов Xiaomi см.: [Пользовательские мелодии Xiaomi](https://dev.mi.com/console/doc/detail?pId=1278%23_3_0#_3_0). Кроме того, для телефонов Google, чтобы установить звуковые подсказки для FCM-отправки на системах Android 8.0 и выше, вы должны вызвать `setAndroidFCMChannelID` для установки channelID, чтобы это вступило в силу. |
| androidOPPOChannelID | String | Нет | Установите ID канала для телефонов OPPO с системами Android 8.0 и выше. |
| androidVIVOClassification | Number | Нет | Классификация сообщений VIVO-отправки (устаревший интерфейс, сервис VIVO-отправки оптимизирует правила классификации сообщений 3 апреля 2023 г. Рекомендуется использовать setAndroidVIVOCategory для установки категории сообщения). 0: Операционные сообщения, 1: Системные сообщения. Значение по умолчанию: 1. |
| androidXiaoMiChannelID | String | Нет | Установите ID канала для телефонов Xiaomi с системами Android 8.0 и выше. |
| androidFCMChannelID | String | Нет | Установите ID канала для телефонов Google с системами Android 8.0 и выше. |
| androidHuaWeiCategory | String | Нет | Классификация сообщений Huawei-отправки, см.:[Стандарт классификации сообщений Huawei.](https://developer.huawei.com/consumer/cn/doc/development/HMSCore-Guides/message-classification-0000001149358835) |
| isDisablePush | boolean | Нет | Отключить ли уведомления отправки (по умолчанию включено). |
| iOSPushType | [IOSOfflinePushType](https://www.tencentcloud.com/document/product/647/66840#9031c0d2-69a0-4e33-bda1-aea049930293) | Нет | Тип автономной отправки iOS, по умолчанию APNs |

### FillMode

Если соотношение сторон области отображения видео не равно соотношению сторон видеоизображения, необходимо указать режим заполнения:

| Параметр | Описание |
| --- | --- |
| Fill | Режим заполнения: видеоизображение будет центрировано и масштабировано для заполнения всей области отображения, части, превышающие площадь, будут обрезаны. Отображаемое изображение может быть неполным в этом режиме. |
| Fit | Режим подгонки: видеоизображение будет масштабировано в зависимости от его длинной стороны, чтобы подогнать область отображения, короткая сторона будет заполнена черными полосами. Отображаемое изображение полное в этом режиме, но могут быть черные полосы. |

### Rotation

Мы предоставляем API-интерфейсы для установки угла поворота локальных и удаленных изображений. Следующие углы поворота все по часовой стрелке.

| Параметр | Описание |
| --- | --- |
| Rotation_0 | Без поворота |
| Rotation_90 | Поворот по часовой стрелке на 90 градусов |
| Rotation_180 | Поворот по часовой стрелке на 180 градусов |
| Rotation_270 | Поворот по часовой стрелке на 0 градусов |

### Resolution

Разрешение видео

| Параметр | Описание |
| --- | --- |
| Resolution_640_360 | Соотношение сторон: 16:9, разрешение: 640x360, рекомендуемый битрейт: 500 кбит/с |
| Resolution_640_480 | Соотношение сторон: 4:3, разрешение: 640x480, рекомендуемый битрейт: 600 кбит/с |
| Resolution_960_540 | Соотношение сторон: 16:9, разрешение: 960x540, рекомендуемый битрейт: 850 кбит/с |
| Resolution_960_720 | Соотношение сторон: 4:3, разрешение: 960x720, рекомендуемый битрейт: 1000 кбит/с |
| Resolution_1280_720 | Соотношение сторон: 16:9, разрешение: 1280x720, рекомендуемый битрейт: 1200 кбит/с |
| Resolution_1920_1080 | Соотношение сторон: 16:9, разрешение: 1920x1080, рекомендуемый битрейт: 2000 кбит/с |

### ResolutionMode

Режим соотношения сторон видео

| Параметр | Описание |
| --- | --- |
| Landscape | Разрешение в альбомной ориентации, например Resolution.Resolution_640_360 + ResolutionMode.Landscape = 640 × 360. |
| Portrait | Разрешение в портретной ориентации, например Resolution.Resolution_640_360 + ResolutionMode.Portrait = 360 × 640. |

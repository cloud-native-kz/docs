# Настройка разрешения и режима заполнения

Введение в настройку разрешения и режима заполнения.

## Установка разрешения и режима заполнения

Компонент TUICallKit предоставляет множество переключателей функций, позволяющих выборочно включать или отключать их по необходимости. Дополнительные сведения см. в разделе [Введение в атрибуты TUICallKit](https://www.tencentcloud.com/document/product/647/51015).

- `VideoDisplayMode`: режим отображения кадра.
- `VideoResolution`: разрешение вызова.

React

Vue

```
import { VideoDisplayMode, VideoResolution } from "@trtc/calls-uikit-react";<TUICallKit    videoDisplayMode={VideoDisplayMode.CONTAIN}    videoResolution={VideoResolution.RESOLUTION_1080P} />
```

```
import { VideoDisplayMode, VideoResolution } from "@trtc/calls-uikit-vue";<TUICallKit    :videoDisplayMode="VideoDisplayMode.CONTAIN"    :videoResolution="VideoResolution.RESOLUTION_1080P" />
```

### videoDisplayMode

Режим отображения `videoDisplayMode` имеет три возможных значения:

- `VideoDisplayMode.CONTAIN`
- `VideoDisplayMode.COVER`
- `VideoDisplayMode.FILL`, значение по умолчанию: `VideoDisplayMode.COVER`.

| **Атрибут** | **Значение** | **Описание** |
| --- | --- | --- |
| videoDisplayMode | VideoDisplayMode.CONTAIN | Приоритет отдается полному отображению содержимого видео. Размеры видео масштабируются пропорционально до тех пор, пока одна сторона не совпадает с границей окна просмотра. Если размеры видео и окна отличаются, видео масштабируется с сохранением соотношения сторон для заполнения окна, в результате чего вокруг масштабированного видео появляется черная полоса. |
|  | VideoDisplayMode.COVER | Приоритет отдается заполнению окна просмотра. Размер видео масштабируется пропорционально до полного заполнения окна. Если размеры видео отличаются от размеров окна отображения, видеопоток будет обрезан или растянут в соответствии с соотношением окна. |
|  | VideoDisplayMode.FILL | Гарантируется отображение всего содержимого видео при заполнении окна, но не гарантируется сохранение исходного соотношения сторон видео. Размеры видео будут растянуты в соответствии с размерами окна. |

### videoResolution

Разрешение `videoResolution` имеет три возможных значения:

- `VideoResolution.RESOLUTION_480P`
- `VideoResolution.RESOLUTION_720P`
- `VideoResolution.RESOLUTION_1080P`, значение по умолчанию: `VideoResolution.RESOLUTION_480P`.

**Пояснение к разрешению:**

| **Видеопрофиль** | **Разрешение (Ш x В)** | **Частота кадров (fps)** | **Битрейт (Кбит/с)** |
| --- | --- | --- | --- |
| 480p | 640 × 480 | 15 | 900 |
| 720p | 1280 × 720 | 15 | 1500 |
| 1080p | 1920 × 1080 | 15 | 2000 |

**Часто задаваемые вопросы:**

- iOS 13 и 14 не поддерживают кодирование видео выше 720P. Рекомендуется ограничить максимальное разрешение сбора до 720P в этих двух версиях системы. см. раздел [Известная проблема iOS Safari, случай 12](https://web.sdk.qcloud.com/trtc/webrtc/doc/en/tutorial-02-info-webrtc-issues.html#h2-4).
- Firefox не позволяет настраивать частоту кадров видео (по умолчанию установлено 30 кадров в секунду).
- Из-за влияния использования производительности системы, возможностей сбора камеры, ограничений браузера и других факторов, фактические значения разрешения видео, частоты кадров и битрейта могут не совпадать с установленными значениями. В таких случаях браузер автоматически настроит профиль, чтобы максимально приблизиться к установленным значениям.

# Объяснение биллинга для подписки на пакет длительности

Длительность пакета относится к бесплатным минутам, включенным в месячный пакет, который вы приобретаете, и может использоваться для вычитания вашего использования TRTC. Если длительность пакета исчерпана, а пакет еще не истек, дополнительное использование будет выставляться по [ценам по требованию](https://www.tencentcloud.com/document/product/647/42734).

## Обзор биллинга

- **Способ оплаты:** Предоплата. Вам необходимо приобрести пакеты до того, как они смогут использоваться для вычитания.
- **Сроки действия:** Пожалуйста, обратитесь к документации на момент покупки.
- **Время начала действия:** Пакет вступает в силу примерно через пять минут после успешной покупки. После вступления в силу он будет вычитать использование, которое не было вычтено другими пакетами с 00:00:00 дня покупки. Он не может вычитать использование, которое произошло в предыдущий день.
- **Область применения:** Минуты месячного пакета могут вычитать использование только, сгенерированное приложением (SDKAppID), к которому привязан пакет. Пакет действителен в течение одного месяца (с 00:00:00 дня покупки до 23:59:59 дня, предшествующего первому дню следующего месяца).
- **Тип вычитания:** Минуты месячного пакета могут вычитать только длительность медиа, на которые подписаны участники аудио/видеокоммуникации. О том, как рассчитывается длительность, см. в разделе [Оплата по требованию](https://www.tencentcloud.com/document/product/647/42734#).
- **Порядок вычитания:** 10 000 [бесплатных минут](https://www.tencentcloud.com/document/product/647/42735#) бесплатной длительности, предоставляемых ежемесячно, будут вычтены в первую очередь, затем следуют бесплатные минуты, включенные в оплачиваемый месячный пакет. Несколько покупок одного и того же месячного пакета не могут складываться в течение срока его действия. Если у вашего приложения есть пакет, который все еще в пределах срока действия, и вы приобретаете новый пакет, новый пакет не вступит в силу до истечения существующего. На основе фактического времени использования для каждого типа пакеты длительности с более узкой областью вычитания будут приоритизированы для вычитания. Для пакетов с одинаковой областью вычитания в первую очередь будет вычтен пакет с более ранним сроком истечения.

> **Примечание:** **Инструкции по истечению:** После истечения срока действия пакета длительности неиспользованные минуты будут автоматически удалены и не могут быть восстановлены. **Уведомление напоминание:** Мы уведомим вас до того, как пакет длительности будет исчерпан или истечет. Однако в наших уведомлениях могут быть задержки из-за калибровки данных, задержек вычитания, колебаний сети и т. д. Пожалуйста, всегда следите за использованием вашего пакета. **Избыточная оплата по требованию:** Если месячный пакет исчерпан, но все еще в пределах срока действия, дополнительное использование, включая, но не ограничиваясь, длительность видео и аудио, облачную запись и облачное микширование и транскодирование, будет выставляться по ценам по требованию. После истечения срока действия пакета услуга будет приостановлена. **Инструкции по возврату:** Вы можете вернуть неиспользованный месячный пакет или пакет длительности в течение 5 дней с момента покупки. Возврат не поддерживается в других ситуациях. Подробнее см. в разделе [Инструкции по возврату](https://www.tencentcloud.com/document/product/647/58588#).

## Коэффициенты вычитания

[Длительность аудио и видео](https://www.tencentcloud.com/document/product/647/42734#) вычитается в разных коэффициентах в зависимости от типа медиа и совокупного разрешения видео.

| Элемент |  | Фактическое использование (минуты) | Вычтено минут |
| --- | --- | --- | --- |
| Длительность аудио и видео RTC Engine/Call/ConferenceДлительность аудио и видео Live's Audio RoomДлительность Live's Video Live Anchor | Аудио | 1 | 1 |
|  | HD | 1 | 4 |
|  | FHD | 1 | 9 |
|  | 2K | 1 | 16 |
|  | 4K | 1 | 36 |
| Длительность Live's Video Live Audience | Аудио | 1 | 0.5 |
|  | HD | 1 | 2 |
|  | FHD | 1 | 4.5 |
|  | 2K | 1 | 8 |
|  | 4K | 1 | 18 |

> **Примечание:** **Live's Audio Room Audio and Video Duration:** Длительность аудио и видео, сгенерированная при использовании компонента Live Voice [Room core control component (SeatGridView)](https://trtc.io/document/60334?product=live) или при интеграции с использованием [RTCRoomEngine API](https://trtc.io/document/61639?product=live). **Live's Video Live Anchor Duration:** Длительность транслятора, сгенерированная при использовании компонента Live Video [Live core control component (LiveStreamCoreView)](https://trtc.io/document/60036?product=live) для интеграции. **Live's Video Live Audience Duration:** Приложения, созданные после 18 декабря 2024 г. (SDKAppID), которые интегрируют [основной контроль для Video Live (LiveStreamCoreView)](https://trtc.io/document/60036?product=live) с использованием версии 2.8.0 или позднее Live, будут иметь их использование аудио и видео, отнесенное к удостоверению аудитории. Длительности, не соответствующие приведенным выше критериям, по-прежнему будут рассматриваться как длительность live's video live anchor.

# Примечания к выпускам

> **Примечания к выпускам****⚠️**Начиная с **версии v4.1.0**, все пакеты перенесены в пространство имён `@trtc` с обновлением API. В таблице ниже показано соответствие между новыми и устаревшими названиями пакетов:**Фреймворк****Новый пакет (v4.1.0+)****Устаревший пакет (ниже v4.0.x)****Описание****Vue3****@trtc/calls-uikit-vue****@tencentcloud/call-uikit-vue****v4.1.0+ активно обновляется и поддерживается (рекомендуется для использования).****Для устаревших версий (v4.0.x и ниже) предоставляются только исправления ошибок.****v4.1.0+ заменяет устаревшие методы**`call`**и**`groupCall`**методом**`calls`**.****Vue2.7****@trtc/calls-uikit-vue2****@tencentcloud/call-uikit-vue2**
> **Vue2.6****@trtc/calls-uikit-vue2.6****@tencentcloud/call-uikit-vue2.6**
> **React****@trtc/calls-uikit-react****@tencentcloud/call-uikit-react**
> **Перед обновлением до версии v4.1.0+, обратите внимание:**При обновлении для использования нового метода `calls` убедитесь, что все участники вызова используют совместимые версии. **Вызовы, инициированные версией v4.1.0+, не будут получены пользователями на версии v4.0.x и ниже**, так как они не поддерживают новый интерфейс `calls`.

Исправлена проблема отображения сообщения, интегрированного в TUIKit.

## Версия 3.0.5 @2023.09.15

### Оптимизация функций

Оптимизированы взаимные ссылки в TUICallKit, чтобы избежать проблемы переполнения стека, которая возникает при упаковке мини-программ в uniapp.

### Новые возможности

Новые возможности подсказка для устройств рабочего стола при отсутствии разрешения, руководство клиентов по авторизации устройств.

### Исправления ошибок

- Исправлена проблема, при которой setCallingBell переопределял звонок вызываемого абонента звонком инициатора, приводя к проблеме повторения звонка.
- Исправлены проблемы со стилем на мобильных устройствах.

## Версия 3.0.4 @2023.09.01

### Исправления ошибок

- Исправлен setCallingBell, нацеленный на звонок входящего вызова (звонок вызываемого абонента).
- Исправлена проблема с сообщением об ошибке destroyed.
- Исправлено отсутствие китайского и английского языков в подсказке всплывающего окна ошибки.
- Исправлена проблема, при которой невозможно переключаться между поддержкой нескольких экранов после отключения камеры во время одноразового вызова.

## Версия 3.0.3 @2023.8.25

### Новые возможности

Добавлен [@tencentcloud/call-uikit-vue2.6](https://www.npmjs.com/package/@tencentcloud/call-uikit-vue2.6) совместимый с версией Vue 2.6.

### Оптимизация функций

- Оптимизируйте язык компонента по умолчанию на язык системы по умолчанию.
- Оптимизирована информация логов печати.
- Оптимизировано сообщение об ошибке, выброшенное [tuicall-engine-webrtc](https://www.npmjs.com/package/tuicall-engine-webrtc).
- Оптимизирована очистка ресурсов после уничтожения компонента.

### Исправления ошибок

- Исправлена проблема, при которой videoDisplayMode, videoResolution не вступали в силу при повторном вызове после завешивания.
- Исправлена проблема, при которой statusChanged не был запущен во время вызова.
- Исправлена проблема многократного вызова init.
- Исправлена проблема невозможности переключения между полным и разделённым экранами при отключении камеры во время вызова.

## Версия 3.0.0 @2023.8.4

### Критическое изменение

Обновлена базовая зависимость [tuicall-engine-webrtc](https://www.npmjs.com/package/tuicall-engine-webrtc) до ^2.0.0. Больше не поддерживается создание экземпляра tim с помощью [tim-js-sdk](https://www.npmjs.com/package/tim-js-sdk). Если вам нужно создать экземпляр tim, используйте [@tencentcloud/chat](https://www.npmjs.com/package/@tencentcloud/chat).

### **Добавить**

Добавлена функция пользовательского звонка setCallingBell.

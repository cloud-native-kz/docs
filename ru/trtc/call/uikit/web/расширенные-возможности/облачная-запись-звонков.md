# Облачная запись звонков

В этом документе описано, как включить облачную запись `TUICallKit` для архивирования и просмотра важных звонков. Здесь предоставлены два способа: [автоматическая запись](#scheme1) и [запись на основе RESTful API](#scheme2).

> **Примечание:** `TUICallKit` интегрирует несколько базовых сервисов Tencent Cloud PaaS, а его возможности аудио/видео зависят от [TRTC](https://www.tencentcloud.com/document/product/647/35078). Чтобы использовать функцию облачной записи `TUICallKit`, необходимо настроить её в [консоли TRTC](https://console.tencentcloud.com/trtc/app).

## Способ 1. Автоматическая запись (рекомендуется)

Мы рекомендуем использовать схему автоматической записи. В этом схеме запись не запускается и не останавливается вручную. Вместо этого задачи записи управляются на серверной части TRTC, и звонок будет записываться автоматически при наличии загружаемого аудио/видеопотока. Вы можете интегрировать его быстро и легко следующим образом:

1. Найдите приложение с целевым `SDKAppId` в разделе [**Управление приложениями**](https://console.tencentcloud.com/trtc/app) в консоли TRTC и перейдите на страницу **Дополнительные функции**.
2. На странице конфигурации **Дополнительные функции** вы можете увидеть опцию конфигурации **Облачная запись**. Нажмите на **Глобальная автозапись** для входа в окно конфигурации.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/6e211b30b5a711eeb2a1525400170219.png)

3. Мы рекомендуем следующие параметры конфигурации для сценариев бизнеса аудио/видео звонков, таких как индивидуальные и групповые звонки. Вы также можете настроить шаблон записи по мере необходимости.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/6f98b1f4b5a811ee9fd6525400bb593a.png)

> **Примечание:** Глобальная запись может смешивать потоки до восьми пользователей. Если звонок включает более восьми пользователей (включая локального пользователя), поток последнего пользователя не может быть записан.

После включения функции глобальной автоматической записи, когда звонок принят и происходит загрузка аудио/видео, задача записи будет запущена автоматически, и запись остановится автоматически по завершении звонка. Если пользователь покидает комнату из-за плохих условий сети или других исключений, серверная часть записи автоматически остановит задачу записи на основе настроенного значения `MaxIdleTime` (максимальное время ожидания, по умолчанию пять секунд), чтобы избежать ненужных платежей.

4. После создания шаблона выберите **Глобальная автозапись**.

## Способ 2. Запись на основе RESTful API

Если схема автоматической записи не соответствует вашим требованиям, вы также можете использовать более гибкую схему записи на основе RESTful API. В этой схеме вы можете записывать и подписываться на указанного якоря в комнате, настраивать макет смешанных потоков и обновлять макет и подписку во время записи. Однако **использование её функций требует использования её вместе с сервисом бизнес-бэкенда и выполнения сложных операций интеграции**:

1. Найдите приложение с целевым `SDKAppId` в разделе [**Управление приложениями**](https://console.tencentcloud.com/trtc/app) в консоли TRTC и перейдите на страницу **Дополнительные функции**.
2. На странице конфигурации **Дополнительные функции** вы можете увидеть опцию конфигурации **Облачная запись**. Нажмите на **Глобальная автозапись** для входа в окно конфигурации. **Пользовательская ручная запись**, то есть режим записи на основе RESTful API, выбран по умолчанию.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/55a0caa2b5a911eeb2a1525400170219.png)

3. Затем вы можете вызвать RESTful API [CreateCloudRecording](https://www.tencentcloud.com/document/product/647/46960) для запуска облачной записи. Здесь мы рекомендуем вам слушать событие уведомления [TUICallObserver](https://www.tencentcloud.com/document/product/647/51007) для запуска записи при начале аудио/видео звонка. Ниже приведён пример кода на Java:

```
TUICallEngine.createInstance(context).addObserver(new TUICallObserver() {    @Override    public void onCallBegin(TUICommonDefine.RoomId roomId, TUICallDefine.MediaType callMediaType, TUICallDefine.Role callRole) {        // Tell your business backend to start a recording task by using the relevant RESTful API.    }});
```

4. Поскольку звонок может быть завершён на клиенте из-за исключения, такого как плохие условия сети или завершение процесса, чтобы остановить запись, мы рекомендуем вам подписаться на обратный вызов состояния комнаты TRTC (дополнительную информацию см. в разделе [Обратные вызовы событий](https://www.tencentcloud.com/document/product/647/39558)) и вызвать RESTful API [DeleteCloudRecording](https://www.tencentcloud.com/document/product/647/46959) для остановки задачи облачной записи при получении обратного вызова отклонения комнаты TRTC.

## Часто задаваемые вопросы

### 1. Как просмотреть подробные длительности записей?

Вы можете просмотреть подробные длительности записей на странице [Облачная запись](https://console.tencentcloud.com/trtc/statistics/cloud-record) в консоли TRTC.

### 2. Как просмотреть записанные файлы?

Войдите в [консоль VOD](https://console.tencentcloud.com/vod), выберите **Управление видео/аудио** на левой боковой панели, нажмите **Поиск по префиксу** над списком, выберите **Поиск по префиксу** и введите ключевое слово в поле поиска. Имена файлов записей имеют следующие форматы:

- Формат имени файла записи одного потока MP4: `<SdkAppId>_<RoomId>_UserId_s_<UserId>_UserId_e_<MediaId>_<Index>.mp4`
- Формат имени файла записи смешанного потока MP4: `<SdkAppId>_<RoomId>_<Index>.mp4`

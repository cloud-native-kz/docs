# Data Insights

**Data Insights** на платформе RTC Analytics предоставляет функции для просмотра и анализа данных масштаба и качества операционной деятельности бизнес-панели из измерения приложения. Это помогает вам быстро понять данные масштаба вызовов и тенденции под приложением (sdkappid), а также предоставляет аналитику качества вызовов по нескольким измерениям.

> **Примечание:** Data Dashboard — это платная функция, для которой требуется подписка на платную версию для разблокирования дополнительных функций. Дополнительную информацию см. в разделе [Version Features and Billing Information](https://www.tencentcloud.com/document/product/647/54481) в Data Monitoring.

## Обзор функций

Функции Data Dashboard:

- **Dashboard Data**: поддерживает просмотр количества пользователей, количества комнат, количества пользователей в часы пик и количества комнат в часы пик за указанный период, предоставляя быстрый обзор масштаба операций.
- **Multi-dimensional Quality Analysis**: поддерживает просмотр качества вызовов в комнате по различным измерениям за указанный период, включая метрики такие как коэффициент подтормаживания видео и коэффициент подтормаживания аудио, позволяя своевременно получать аналитику качества вызовов по различным измерениям в операциях бизнеса.

## Инструкции

1. Войдите в [Tencent RTC Console > RTC Analytics > Data Insights](https://console.trtc.io/data-dashboard), перейдите в Data Monitoring и активируйте сервис. Если он уже активирован, вы можете пропустить этот шаг.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/56a210d3cb6a11f08e74525400bf7822.png)

> **Примечание:** Для использования этой функции активируйте сервис Data Monitoring заранее. Диапазон запросов данных для Data Dashboard начинается со следующего дня после активации сервиса. Исторические данные до активации сервиса не включены в статистику и не могут быть получены.

2. Отфильтруйте нужное приложение (sdkappid) и выберите временной диапазон для просмотра. **В настоящее время бесплатная версия (по умолчанию) позволяет фильтровать данные за последние 7 дней**, с максимальным диапазоном в 60 дней, доступным при разблокировке [RTC Analytics Monthly Packages](https://trtc.tencentcloud.com/buy/dashboard).

## Описание функций

### Метрики панели инструментов

- **Statistical Metrics:** метрики панели инструментов предоставляют четыре ключевые метрики масштаба операций: количество пользователей, количество комнат, количество пользователей в часы пик и количество комнат в часы пик. Эти метрики помогают вам быстро понять масштаб операций текущего приложения в выбранном временном диапазоне.

| Метрика | Описание |
| --- | --- |
| Количество пользователей | Общее количество пользователей, которые присоединились к комнате в течение выбранного временного диапазона (дедублицировано по userID и комбинации create_time и roomID). |
| Количество комнат | Общее количество комнат, созданных в течение выбранного временного диапазона (дедублицировано по комбинации create_time и roomID). |
| Количество пользователей в часы пик | Максимальное количество пользователей в одной комнате одновременно в течение выбранного временного диапазона, рассчитанное с детализацией в 1 минуту. |
| Количество комнат в часы пик | Максимальное количество комнат, существующих одновременно в течение выбранного временного диапазона, рассчитанное с детализацией в 1 минуту. |

> **Примечание:** Для количества пользователей, если один и тот же пользователь (userID) присоединяется к разным roomID, он будет учтён как два пользователя; аналогично, если один и тот же пользователь (userID) присоединяется к одному и тому же roomID, созданному в разное время, это также будет учтено как два пользователя.

- **Graphs:** они предоставляют графики линий тренда для вышеуказанных метрик, поддерживая как ежедневный, так и ежечасный просмотр:
  - Если вы просматриваете по **дням**, вы можете увидеть метрики масштаба для этого дня в каждой точке координат.
  - Если вы просматриваете по **часам**, поддерживаются только данные с ежечасной детализацией для одного дня с использованием [Indicator Statistical Rules](https://www.tencentcloud.com/document/product/647/67828#a8caefa7-cc1f-46e1-8819-62c34c506480) как описано выше. Когда вы переключаетесь на **ежечасную** детализацию, отображение автоматически переключается на самый свежий день в вашем выбранном диапазоне дат.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/59370562d25811efa4a3525400bf7822.png)

> **Примечание:** Поскольку могут быть комнаты вызовов, охватывающие несколько часов, количество пользователей в одной и той же комнате может быть учтено отдельно в двух соседних часах. В результате суммирование данных с ежечасной детализацией за день может дать более высокое общее количество пользователей и комнат, чем статистика по дневному измерению.

Вы можете щёлкнуть **View Details**, чтобы получить доступ к соответствующим деталям данных, с опцией экспорта данных.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/60e64752d25811efa2ff52540044a08e.png)

### Данные по измерениям

- **Quality:** качество в данных по измерениям предоставляет функцию для запроса метрик качества вызовов по различным измерениям (таким как регион, версия SDK и тип сети). Вы можете просмотреть нужные метрики качества в соответствующем модуле измерения и щёлкнуть **View Details**, чтобы получить доступ ко всем детальным данным, как показано ниже.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/71cd3197d25811ef81865254005ef0f7.png)

- Вы можете щёлкнуть **View Details**, чтобы отфильтровать по соответствующим измерениям и получить доступ к детальным данным.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/9c7bff9dd25811ef8c8a525400454e06.png)

- **Метрики качества**

| Метрика | Описание |
| --- | --- |
| Коэффициент подтормаживания видео | Продолжительность подтормаживаний видео / общая продолжительность видео x 100%. Подтормаживание видео продолжительностью более 600 мс включается в продолжительность подтормаживания. |
| Коэффициент подтормаживания аудио | Продолжительность подтормаживаний аудио / общая продолжительность аудио x 100%. Подтормаживание аудио продолжительностью более 200 мс включается в продолжительность подтормаживания. |
| Коэффициент задержки | Продолжительность сквозной задержки сети / общая продолжительность аудио и видео x 100%. Сквозная задержка сети, превышающая 400 мс, включается в продолжительность задержки. |
| Успешный вход в комнату | Количество пользователей, успешно присоединившихся к комнате / общее количество пользователей, попытавшихся присоединиться x 100%. |
| Вход в комнату в течение 5 сек | Количество пользователей, успешно присоединившихся к комнате в течение 5 секунд / общее количество пользователей, попытавшихся присоединиться x 100%. |

- **Измерения**

| Измерение | Описание |
| --- | --- |
| Распределение по регионам | Провинция, где находится пользователь, с различием между регионами китайского материка и регионами за пределами китайского материка. |
| Тип устройства | Тип устройства, используемого пользователем. |
| Версия SDK | Версия SDK, используемая пользователем во время вызова. |
| Оператор сети | Оператор сети, используемый пользователем. |
| Тип сети | Тип сети, используемый во время вызова, такой как проводная, Wi-Fi и 4G. |

Функция Data Dashboard предоставляет RESTful API, позволяя вам получать соответствующие данные через запросы API.

> **Примечание:** RESTful API Data Dashboard доступен только для клиентов, подписавшихся на **Basic Edition** или **Advanced Edition** сервиса Data Monitoring. Детали по включению сервиса Data Monitoring см. в [Version Features and Billing Instructions](https://www.tencentcloud.com/document/product/647/54481#0489f61c-2a38-4373-acf7-a3004cd4560d).

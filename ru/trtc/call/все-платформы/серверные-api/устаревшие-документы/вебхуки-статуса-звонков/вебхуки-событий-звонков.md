# Вебхуки событий звонков

Для обеспечения точного контроля над вашими сервисами звонков TRTC Call (TUICallKit) предоставляет обратные вызовы статуса звонков. Серверная часть вашего приложения может использовать эти обратные вызовы для отслеживания результатов звонков пользователей в реальном времени, например пропущенные вызовы, отклонения и т. д., и на основе этого выполнять анализ данных в реальном времени и другие операции. Методы конфигурации вызовов см. в разделе [Список API конфигурации обратных вызовов](https://www.tencentcloud.com/document/product/647/60110#).

## Условия использования

- Только приложения (SDKAppId), которые активировали **Групповую версию** TRTC Call, могут использовать обратный вызов статуса звонков. Вы также можете активировать пробную версию для бесплатного использования. Описания версий и инструкции по активации см. в разделе [Активация сервиса](https://www.tencentcloud.com/document/product/647/59832#).
- В настоящее время обратные вызовы статуса звонков доступны только в определённых версиях TUICallKit на различных платформах, как указано в таблице ниже:

| Платформа/Фреймворк | Номер версии |
| --- | --- |
| Android/iOS/Flutter/uni-app (клиент) | ≥ 1.7.1 |
| Web | ≥ 1.4.6 |
| WeChat Mini Program | ≥ 1.5.1 |

> **Примечание:** Информация о звонках в консоли будет доступна только после обновления всех участвующих платформ/фреймворков до указанных выше версий.

## Примечания

- Для включения обратного вызова необходимо настроить URL-адрес обратного вызова и включить переключатель для этой команды. Пожалуйста, обратитесь к разделу [Создание конфигурации обратного вызова](https://www.tencentcloud.com/document/product/647/60111#).
- Направление обратного вызова — от серверной части Callkit к серверной части приложения через HTTP POST запрос.
- После получения запроса обратного вызова серверная часть приложения должна проверить, совпадает ли SDKAppID, содержащийся в URL-адресе запроса, с собственным SDKAppID.

## Сценарии, которые могут запустить этот обратный вызов

Действия, выполняемые пользователем приложения во время звонка через клиент, такие как ответ и завершение вызова.

## Время срабатывания обратного вызова

После операций пользователя, таких как ответ и завершение вызова.

## Описание API

### URL-адрес запроса

В следующем примере URL-адрес обратного вызова, настроенный в приложении, это `https://www.example.com`.
**Пример:**

```
$http://www.example.com?sdkappid=$sdkappid&command=$command&contenttype=json&clientip=$clientip&optplatform=$optplatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| http | Протокол запроса: HTTPS или HTTP, метод запроса — POST |
| www.example.com | URL-адрес обратного вызова |
| sdkappid | SDKAppID, назначенный консолью обмена мгновенными сообщениями при создании приложения |
| command | Пожалуйста, см.: [Список команд обратного вызова](https://www.tencentcloud.com/document/product/647/60130#0ef77eb4-15d0-4bac-9448-57533360b554) |
| contenttype | Фиксированное значение: json |
| clientip | IP-адрес клиента, например 127.0.0.1 |
| optplatform | Платформы клиентов могут включать iOS, Android, Web, miniProgram |

Конкретное содержимое обратного вызова включено в тело HTTP запроса. Подробности см. в примерах обратного вызова ниже.

### Пример обратного вызова

Пример запроса вебхука:

```
POST /?sdkappid=8888888&command=caller_start_call&contenttype=json&clientip=127.0.0.1&optplatform=iOS HTTP/1.1Host: www.example.comContent-Length: 337{    "UserId": "Alice",	"RoomId": "Alice's Room",    "TotalNum": 2,    "MediaType": "audio",    "CallType": "single",    "CallId": "aheahfo-eqwnknoihfsd-qweqf",    "Role": "caller",    "Event": "start_call",    "CallResult": "",	"EventTime": 1704695566,    "StartCallTs": 1704856873,    "AcceptTs": 1704856876,    "EndTs": 1704856885}
```

### Поля пакета запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| UserId | String | ID пользователя, выполняющего операцию |
| RoomId | String | ID комнаты, в которой выполняется операция |
| TotalNum | Integer | Количество участников звонка |
| CallType | String | Тип звонка: `Single` Звонок только с аудио `Group` Видеовызов |
| CallId | String | Уникальный ID звонка |
| Role | String | Роль: `Caller` ID пользователя звонящего `Callee` ID пользователя вызываемого |
| Event | String | Событие звонка: `start_call` Звонящий инициирует звонок `call_accepted` Звонящий ответил на звонок `call_missed` Звонящий пропустил звонок `call_rejected` Звонящий отклонил звонок `call_busy` Линия звонящего занята `cancel_call` Звонящий отменил звонок `call_failed` Звонящему не удалось инициировать звонок `call_end` Звонок звонящего завершился нормально `call_interrupted` Звонок звонящего прерван `receive_call` Вызываемый получает звонок `accept_call` Вызываемый ответил на звонок `not_answer_call` Вызываемый не ответил `reject_call` Вызываемый отклонил звонок `ignore_call` Вызываемый игнорирует звонок `call_canceled` Вызываемый отменил звонок `call_end` Нормальное завершение звонка вызываемым `call_interrupted` Звонок вызываемого аномально прерван `invite_user` Приглашение пользователя посередине `join_in_group_call` Присоединение к звонку посередине |
| StartCallTs | Integer | Временная метка при инициировании звонка (в секундах), возвращается только при normal_end |
| AcceptTs | Integer | Временная метка при ответе на звонок (в секундах), возвращается только при normal_end |
| EndTs | Integer | Временная метка при окончании звонка (в секундах), возвращается только при normal_end |

> **Примечание:** Поле CallResult показывает все типы только для односторонних звонков, групповой чат имеет только `normal_end`.

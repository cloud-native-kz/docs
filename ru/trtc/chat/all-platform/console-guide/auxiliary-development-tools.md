# Вспомогательные инструменты разработки

## Проверка офлайн-доставки

### Средство поиска проблем офлайн-доставки

Этот инструмент позволяет запрашивать проблемы, связанные с отказом в получении офлайн-сообщений.

1. Войдите в [консоль Chat](https://console.trtc.io), выберите **Chat > Push >**[**Access test**](https://console.trtc.io/chat/push-plugin-push-check) на левой боковой панели и выберите целевое приложение в **верхней части**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2b4ca36dcb5811f0a7775254005ef0f7.png)

2. Выберите **тип доставки**, опция **Online push** или **Offline Push**.

Online Push

Offline Push

1. Выберите **online push**, вручную введите UserID.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/9e7e8e60cb5811f0a4a55254001c06ec.png)

2. Нажмите **Send a push test**, просмотрите **Query Result**, включая ID сертификата, токен устройства и т. д., которые в настоящий момент сообщены UserID.
1. Выберите **Offline Push**, вручную введите UserID, нажмите **get token binding status**, просмотрите **detection result**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/9e8a3ff7cb5811f0a4a55254001c06ec.png)

2. Выберите любой из ID сертификатов, сообщенных UserID, нажмите **Send a push test**, просмотрите **Query Result**, включая ID сертификата, токен устройства и т. д., которые в настоящий момент сообщены UserID.
  - Если доставка указывает на успех, это означает, что информация сертификата, заполненная в консоли, верна, и вызов SDK API для отправки токена выполнен успешно. Вы можете использовать [инструмент проверки статуса пользователя](#status) для дальнейшего расследования.
  - Если уведомление указывает на ошибку, вы можете проверить конкретную причину сбоя и решение.

> **Примечание:** Если UserID не сообщил ни один ID сертификата, токен устройства и т. д., вы не сможете перейти к следующему шагу.

### Инструмент проверки статуса пользователя

Этот инструмент автоматически получает статус клиента пользователя и проверяет, готов ли пользователь получать офлайн-сообщения.

1. Войдите в [консоль Chat](https://console.trtc.io), выберите **Chat > Push >**[**Access test**](https://console.trtc.io/chat/push-plugin-push-check) на левой боковой панели и выберите целевое приложение в **верхней части**.
2. На левой боковой панели выберите **Chat > Push > Access test**.
3. В области **User offline status checking tool** введите UserID.
4. Нажмите **Get Status** для просмотра информации, такой как текущий статус и тип клиента UserID.
 Если вам сообщено, что UserID готов получать офлайн-сообщения, вы можете войти с другим UserID на другом устройстве и отправить индивидуальное текстовое сообщение текущему UserID, чтобы проверить, может ли он получать сообщения.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/20486cb77b1511ef8631525400a9236a.png)

## Генерация и проверка UserSig

### Генератор подписи (UserSig)

Система автоматически получает ключ текущего приложения. После введения UserID вы можете использовать этот инструмент для быстрого создания подписи (UserSig) для запуска демонстраций и отладки функций локально. Если вам нужно создать UserSig для онлайн-сервисов, см. [Генерация UserSig на сервере](https://intl.cloud.tencent.com/document/product/1047/34385).

1. Войдите в [консоль Chat > Development Tools > UserSig Tools](https://console.trtc.io/usersig).
2. В области Signature (UserSig) Generator **выберите приложение**, **вручную введите UserID**.
3. Нажмите **Generate UserSig** для создания подписи, срок действия которой истекает через 180 дней.
4. Нажмите **Copy UserSig** для копирования подписи, а затем вставьте и сохраните подпись.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/8235cbaacb5911f0a4a55254001c06ec.png)

### Верификатор подписи (UserSig)

Система автоматически получает ключ текущего приложения. После введения UserID и UserSig вы можете использовать инструмент для быстрой проверки действительности UserSig.

1. Войдите в [консоль Chat > Development Tools > UserSig Tools](https://console.trtc.io/usersig).
2. В области Signature (UserSig) Verifier **выберите приложение**, **вручную введите UserID** и **UserSig**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/ce7e2388cb5911f0aedb525400454e06.png)

3. Нажмите **Verify** для просмотра результата проверки.
  - Если проверка пройдена успешно, вы можете просмотреть SDKAppID, UserID, время создания, время обслуживания и время истечения UserSig в результатах проверки.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e7819f55cb5911f0a7775254005ef0f7.png)

  - Если проверка не пройдена, вы можете просмотреть причину ошибки и решение в результатах проверки.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/f02450b9cb5911f0aedb525400454e06.png)

## Журналы самостоятельной диагностики

Консоль Tencent Cloud Chat предоставляет функцию самостоятельной диагностики, которая позволяет разработчикам запрашивать информацию из журнала сервера Chat за последние три дня для быстрого определения и решения проблем.

1. Войдите в консоль Chat, нажмите раздел целевого приложения Chat.
2. На левой боковой панели выберите **Auxiliary Tools** > **Self-Troubleshooting Logs**.
3. Установите следующие фильтры для запроса журналов:
  - Имя события (необязательно): выберите события для запроса.
  - UserID (необязательно): введите имя пользователя (UserID), что является UserID отправителя сообщения.
  - Receiver/Group ID (необязательно): введите ID целевого разговора. Для индивидуального чата это UserID получателя сообщения. Для группового чата это GroupID группы.
  - Коды ошибок (необязательно): введите коды ошибок. Описание кодов ошибок см. в [Коды ошибок](https://www.tencentcloud.com/document/product/1047/34348).
  - Диапазон времени (обязательно): выберите диапазон времени журналов для запроса. Можно запрашивать журналы за последние три дня.
4. Нажмите **Query** для просмотра отфильтрованных журналов.

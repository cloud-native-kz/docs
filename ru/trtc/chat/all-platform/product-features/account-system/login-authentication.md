# Аутентификация входа

## Обзор аутентификации входа

Система мгновенного обмена сообщениями (Chat) развивалась на основе системы мгновенного обмена сообщениями QQ. Мы извлекли общие модули QQ и интегрировали их для создания удобного для интеграции Chat SDK и серверной службы.

Chat SDK можно рассматривать как QQ без пользовательского интерфейса. Интеграция Chat SDK в ваше приложение аналогична интеграции ядра QQ.

Перед использованием Chat для обмена сообщениями необходимо войти в систему. Хотя вы входите в QQ с использованием своего QQ ID и пароля, вы должны войти в Chat SDK с указанным именем пользователя (`UserID`) и паролем (`UserSig`).

- **UserID**: Ранее известен как Identifier. Это имя пользователя для входа пользователей в Chat. На самом деле это ID пользователя в вашем приложении.
- Например, если пользователь в вашем приложении имеет ID 27149, вы можете использовать 27149 в качестве UserID для входа в Chat.
- **UserSig**: Это пароль, с помощью которого пользователь входит в Chat. Это данные, полученные после шифрования сервером приложения с использованием ключа информации, такой как UserID. Для получения дополнительной информации см. [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385).

> **Осторожно:** Серверная часть Chat полностью доверяет UserSig. Чтобы избежать влияния на ваши данные и бизнес, необходимо обеспечить безопасность приватного ключа. UserSig, созданные стандартным API Chat backend SDK, действительны в течение 180 дней. Разработчики могут использовать API с параметром действительности для настройки периода действия. Разработчики должны получить новый UserSig от серверной части разработчика перед истечением срока действия исходного UserSig. Для получения дополнительной информации о Chat backend SDK, используемом для генерирования UserSig, см. [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385).

## Администратор приложения

Администратор приложения — это особая роль пользователя в Chat, которая имеет права администратора, например, вызов API RESTful, роспуск группы и отправка сообщений всем пользователям. По сравнению с обычными учетными записями роль администратора приложения имеет наивысший уровень привилегий:

- Имеет повышенные права на чтение. Например, может получить все группы в приложении и любой профиль группы любой группы.
- Имеет повышенные права на операции. Например, может отправлять сообщения любому пользователю и добавлять или удалять участников в любой группе.

Вы можете установить администраторов приложения только в консоли. Для получения дополнительной информации о процедуре см. [Настройка администраторов учетных записей](https://intl.cloud.tencent.com/document/product/1047/34540).

# Управление статусом онлайн

## Концепции

Существует три статуса пользователя:

- Работа на переднем плане (Online)
- Работа на фоне (PushOnline)
- Не вошли в систему (Offline)

> **Внимание:** Статус PushOnline существует только на мобильных клиентах (Android, iOS, iPad) и не существует на ПК, macOS, Linux или веб-клиентах.

### Работа на переднем плане (Online)

Статус Online означает, что между клиентом и сервером Chat поддерживается стабильное TCP-соединение. В этом состоянии клиент может отправлять сообщения на сервер Chat и получать сообщения от него.

**Когда пользователь открывает приложение, статус — Online.**

При запуске приложения между клиентом и сервером Chat устанавливается постоянное TCP-соединение. Сервер Chat сохраняет информацию об онлайн-статусе клиентов, такую как сетевые каналы, платформы и версии. Во время работы приложения Chat SDK регулярно отправляет пакеты heartbeat для подтверждения статуса онлайн пользователя.

> **Примечание:** Heartbeat: каждые две минуты Chat SDK отправляет на сервер пакет heartbeat для подтверждения статуса онлайн пользователя.

### Работа на фоне (PushOnline)

Когда клиент находится в статусе PushOnline, постоянное TCP-соединение между клиентом и сервером Chat разрывается. В этом случае могут быть получены офлайн-уведомления об отправке.

В следующих сценариях статус пользователя — PushOnline:

- После использования приложения пользователь переводит приложение на фон, а затем процесс приложения завершается операционной системой (ОС) мобильного телефона или пользователь напрямую завершает процесс приложения.
- Если приложение находится в списке разрешённых для сохранения работоспособности ОС мобильного телефона, ОС не завершит процесс приложения после того, как пользователь переведёт приложение на фон. В этом случае статус остаётся Online. Одним из критериев различия между Online и PushOnline является то, завершён ли процесс приложения, то есть разорвано ли постоянное TCP-соединение между клиентом и сервером Chat.
- Пользователь отключает клиент от сети (например, включив режим полёта на мобильном телефоне) или сеть клиента полностью недоступна (например, когда пользователь входит в туннель).
- В таких обстоятельствах клиент не может даже отправить пакеты TCP FIN или RST, и потребуется 400 секунд перед истечением времени пакета heartbeat. После этого статус пользователя изменится на PushOnline.

> **Внимание:** Статус PushOnline существует только на мобильных клиентах (Android, iOS, iPad) и не существует на ПК, macOS, Linux, мини-программах и веб-клиентах.

### Не вошли в систему (Offline)

Offline — это статус до входа пользователя с использованием имени пользователя и пароля. В этом случае пользователь не может получать уведомления об отправке онлайн- или офлайн-сообщений.

В следующих сценариях статус пользователя — Offline:

- Пользователь выходит или не входил в приложение с момента его загрузки.
- Если пользователь не входит снова в течение 7 дней после изменения статуса на PushOnline, статус изменяется на Offline.

## Запрос статуса онлайн пользователей

Бэкенд приложения может запрашивать статус онлайн нескольких пользователей через RESTful API [v4/openim/querystate](https://intl.cloud.tencent.com/document/product/1047/35477).
В настоящее время Chat SDK не может получить статус онлайн пользователей.

## Уведомления об изменении статуса онлайн пользователя

Chat может уведомлять бэкенд приложения о событиях входа и выхода пользователя. Дополнительные сведения см. в разделе [Callback изменения статуса.](https://www.tencentcloud.com/document/product/1047/34520#a7c1f6fd-f459-47ce-b031-7bc521e6983d)

## Обнаружение изменения статуса в реальном времени

### Android/iOS/iPad/PC/macOS/Linux

В большинстве случаев изменения статуса пользователя можно воспринять в реальном времени. Например:

- Когда пользователь входит, статус пользователя изменяется на Online.
- Когда пользователь выходит, статус пользователя изменяется на Offline.
- Когда пользователь завершает процесс клиента или когда пользователь переводит приложение на фон и процесс клиента завершается ОС мобильного телефона, статус пользователя изменяется на PushOnline.

В следующем особом случае потребуется 400 секунд перед истечением времени пакета heartbeat. После этого экземпляр Chat CVM может обнаружить изменение статуса:

Когда сеть полностью недоступна и клиент не может даже отправить пакеты TCP FIN или RST, потребуется 400 секунд перед истечением времени пакета heartbeat. После этого экземпляр Chat CVM может определить, что статус пользователя изменяется на PushOnline. Это обычно происходит, когда пользователь отключает клиент от сети (например, включив режим полёта на мобильном телефоне) или когда пользователь входит в туннель без сигнала сети.

### Web

Когда пользователь входит на веб-клиент, Chat CVM может определить в реальном времени, что статус пользователя изменяется на Online.

Характер реального времени изменения статуса в различных сценариях выхода/отключения следующий:

- Закрытие страницы можно воспринять в реальном времени, и статус пользователя изменится на Offline.
- Отключение сети можно будет обнаружить через 60 секунд, если страница не закрыта, и статус пользователя изменится на Offline.
- Активный вызов API `destroy` можно воспринять в реальном времени, и статус пользователя изменится на Offline.

### Клиент мини-программы

Когда пользователь входит на клиент мини-программы, Chat CVM может определить в реальном времени, что статус пользователя изменяется на Online.

Время отклика для изменения статуса в различных сценариях выхода или отключения следующее:

- Когда пользователь выходит, нажав в правом верхнем углу, изменение статуса на Offline обнаруживается в течение 5 секунд.
- Когда клиент отключается от интернета (например, включив режим полёта), изменение статуса на Offline обнаруживается через 60 секунд.
- Активный вызов API destroy приводит к обнаружению в реальном времени и изменению статуса на Offline.

## Вход с нескольких устройств

### Принудительный выход

По умолчанию Chat SDK не позволяет входить с нескольких устройств (например, одновременный вход на ПК и устройстве Android). Вместо этого он принудительно переводит предыдущее онлайн-устройство в режим офлайн и позволяет оставаться онлайн только последнему вошедшему устройству. Дополнительные сведения о логике принудительного выхода см. в разделе: [Multi-Client Login and Force Logout](https://intl.cloud.tencent.com/document/product/1047/47971).

### Вход с нескольких устройств в сети

Chat позволяет изменять политику входа с нескольких клиентов в [консоли](https://console.trtc.io/). В настоящее время поддерживаются следующие политики входа с нескольких клиентов:

- Вход с одной платформы: пользователь может быть онлайн только на платформе Android, iPhone, iPad, Windows, macOS или веб-платформе.
- Вход с двух платформ: пользователь может одновременно быть онлайн на платформе Android, iPhone, iPad, Windows или macOS и на веб-платформе.
- Вход с трёх платформ: пользователь может одновременно быть онлайн на платформе Android, iPhone или iPad, платформе Windows или macOS и на веб-платформе.
- Вход с нескольких платформ: пользователь может одновременно быть онлайн на платформах Android, iPhone, iPad, Windows, macOS и веб-платформе.

По умолчанию пользователь может быть онлайн только на одном устройстве для каждой платформы (например, один клиент Android принудительно выведет в офлайн другой клиент Android). Для пользователей с выпуском Pro, Pro Plus или Enterprise вы можете настроить максимальное количество экземпляров, на которые они могут входить на платформе Android, iPhone, iPad, Windows или macOS. Кроме того, вы можете настроить максимальное количество экземпляров, на которые все пользователи могут входить на веб-платформе.

Одновременные входы на одной платформе — это ключевой инструмент для улучшения взаимодействия с пользователем. Вы можете настроить максимальное количество одновременных входов для определённой платформы, чтобы при входе пользователя на несколько устройств на одной платформе не происходило принудительного выхода.

> **Примечание:** Функция "одновременные входы с нескольких устройств на одной платформе" доступна только в выпусках Chat Pro, Pro Plus и Enterprise. Для её использования [приобретите выпуск Pro, Pro Plus или Enterprise](https://www.tencentcloud.com/document/product/1047/34349). Дополнительные сведения см. в разделе [Pricing](https://www.tencentcloud.com/zh/document/product/1047/34349#.E5.9F.BA.E7.A1.80.E6.9C.8D.E5.8A.A1.E8.AF.A6.E6.83.85).

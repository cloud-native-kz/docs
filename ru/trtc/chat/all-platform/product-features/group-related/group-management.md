# Управление группами

На основе типичных вариантов использования Chat предоставляет следующие типы групп по умолчанию: рабочая группа (Work), открытая группа (Public), группа встреч (Meeting), AVChatRoom и Community. Дополнительные сведения см. в разделе [Типы групп](https://intl.cloud.tencent.com/document/product/1047/33529).

Над группами можно выполнять следующие операции:

| Операция над группой | Описание | Примечания |
| --- | --- | --- |
| Создание группы | Эта операция создает новую группу. Можно указать тип группы, имя и список пользователей для добавления в группу. После создания группы возвращается ID группы, который является уникальным идентификатором группы и может использоваться для выполнения других операций с группой, таких как отправка и получение сообщений. | Ежедневный лимит чистого потребления групп составляет 10 000 на приложение. |
| Передача группы | Эта операция передает группу и изменяет владельца группы на другого пользователя. | Администратор приложения может передать группу через RESTful API. Единственная другая роль, которая может передать группу, — это владелец группы. |
| Распуск группы | Эта операция распускает группу, созданную в приложении. При распуске группы все члены группы получают системное сообщение о том, что группа распущена. | Администратор приложения может вызвать RESTful API для распуска любой группы. Разрешение на распуск группы в приложении зависит от роли участника: для открытой группы (Public), группы встреч (Meeting), AVChatRoom (AVChatRoom) или группы сообщества (Community) только владелец группы может распустить группу. Для рабочей группы (Work) никто в группе не может распустить группу. |

> **Внимание** При создании группы Chat назначает ID группы по умолчанию, который начинается с @TGS#. Можно также вручную указать ID группы. Дополнительные сведения см. в разделе [Пользовательские ID групп](https://intl.cloud.tencent.com/document/product/1047/33529). После создания группы системное сообщение о создании группы отправляется на устройство владельца группы для обеспечения синхронизации на нескольких устройствах (после создания группы на одном устройстве все устройства мгновенно обнаруживают созданную группу).

### Управление профилем группы

Профиль группы включает свойства, специфичные для отдельной группы, такие как имя группы, описание, объявление и владелец группы, а также пользовательские поля группы.

| Управление профилем группы | Описание | Примечания |
| --- | --- | --- |
| Получение профиля группы | Извлечение основной информации о группе. Если вы хотите извлечь пользовательскую информацию, настройте API для пользовательских полей, которые вы хотите извлечь. | **Получение профиля группы членами группы**: члены группы могут получить профиль группы. **Получение профиля группы не-членами**: не-члены могут получить профиль группы только если он открыт. **Получение профиля пользователя в группе**: пользователи могут получить свои собственные профили пользователя во всех группах или в одной группе. **Получение профиля пользователя члена группы**: AVChatRoom не хранит информацию о членах группы и не поддерживает получение профиля пользователя члена группы. |
| Изменение профиля группы | Можно изменить имя группы, описание, объявление, аватар группы, визитную карточку группы, параметры присоединения к группе, пользовательские поля уровня группы, роли членов группы, пользовательские поля уровня участника и параметры получения сообщений группы. | В настоящее время можно настроить обратные вызовы для изменений имени группы, описания, объявления и URL-адреса аватара группы для приложения в консоли. Чтобы включить обратные вызовы для изменений другой информации о группе, включая пользовательские поля уровня группы, пожалуйста, [отправьте запрос в техническую поддержку](https://console.tencentcloud.com/workorder/category?level1_id=29&level2_id=40&source=0&data_title=%E4%BA%91%E9%80%9A%E4%BF%A1%20%20IM&step=1). |

### Управление членами группы и группой

Управление членами группы включает два аспекта:

- Доступ и изменение профиля пользователя в группе. Эта информация может быть доступна и изменена только самими пользователями, например параметры получения сообщений.
- Доступ и изменение информации других членов группы, включая роль, время присоединения, время последнего сообщения, профиль группы и пользовательские поля группы на уровне участника.

| Управление членами группы | Описание | Примечания |
| --- | --- | --- |
| Получение информации о членах группы | Получение профиля пользователя или информации о других членах группы | Доступная информация включает роль, время присоединения, время последнего сообщения, ник группы и пользовательские поля уровня участника. |
| Изменение профиля члена группы | Владелец группы, администраторы группы и участники могут изменять профили членов группы. | Владелец группы и администраторы группы могут изменять роли в группе (назначать или отменять администраторов), отключать членов на время, а также изменять ник группы и пользовательские поля уровня участника. Члены группы могут изменять свои личные профили в группе, включая параметры получения сообщений, ник группы и пользовательские поля уровня участника. |
| Приглашение в группу | Пригласить не-членов в группу. | Для рабочей группы любой член группы может приглашать не-членов в группу, и приглашенные добавляются в группу без подтверждения. Для открытой группы или группы встреч по умолчанию только администратор приложения может приглашать не-членов в группу. Для AVChatRoom ни один участник не может приглашать пользователя в группу. Для группы сообщества любой член группы может приглашать не-членов в группу, и приглашенные добавляются в группу без подтверждения. |
| Запрос на присоединение к группе | Пользователь активно запрашивает присоединение к группе через Chat SDK. | Рабочие группы не позволяют не-членам присоединяться, и будет возвращена ошибка. Группы сообщества не требуют одобрения для прямого присоединения по умолчанию. Если требуется одобрение, это можно контролировать через поле `ApplyJoinOption` в профиле группы. Для других встроенных типов групп результат запроса на членство зависит от поля `ApplyJoinOption` в профиле группы. |
| Удаление члена группы | Владелец группы или администратор группы удаляет члена группы из группы. | Когда член группы удаляется из группы владельцем или администратором группы, удаленный участник получает системное уведомление об удалении, а другие члены группы также получают уведомление события об удалении. |
| Активный выход | Член группы инициирует операцию активного выхода. | Когда член группы активно выходит из группы, участник, который уходит, получает системное сообщение о том, что он вышел из группы, а другие члены группы также получают событийное сообщение о выходе участника из группы. |
| Получение списка групп, в которых участвует пользователь | Извлечение списка групп, в которых участвует текущий пользователь. Возвращаемый результат содержит только часть основной информации. | Для получения подробной информации о группе используйте функцию **профиль группы**. |
| Список ожидающих сообщений о группе | Ожидающие операции над группой — это все связанные с группой операции, требующие одобрения. | Можно извлечь список ожидающих операций над группой, сообщить статус прочтения ожидающих операций и обработать ожидающие операции (одобрить или отклонить). Для одного пользователя список может содержать до 50 ожидающих операций. |

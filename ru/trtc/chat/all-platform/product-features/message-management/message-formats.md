# Форматы сообщений

## MsgBody

Содержимое сообщения вводится в поля MsgBody. Tencent Cloud Chat поддерживает несколько элементов сообщения в одном сообщении, например, сообщение может содержать как текст, так и эмодзи. Поэтому MsgBody определяется как массив, который может включать столько элементов сообщения, сколько необходимо. Элемент сообщения называется TIMMsgElement. Примеры TIMMsgElements, составляющих MsgBody, см. в разделе [Примеры содержимого сообщения MsgBody](https://intl.cloud.tencent.com/document/product/1047/33527).

Формат TIMMsgElement определяется следующим образом:

```
{    "MsgType": "",    "MsgContent": {}}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| MsgType | String | Тип элемента сообщения. В настоящее время поддерживаются следующие объекты сообщений: TIMTextElem (текстовое сообщение), TIMLocationElem (сообщение о местоположении), TIMFaceElem (сообщение-эмодзи), TIMCustomElem (пользовательское сообщение), TIMSoundElem (аудиосообщение), TIMImageElem (сообщение с изображением), TIMFileElem (сообщение с файлом) и TIMVideoFileElem (видеосообщение). |
| MsgContent | Object | Содержимое элемента сообщения. Каждый MsgType имеет свой формат MsgContent. Подробную информацию см. ниже. |

В следующей таблице перечислены типы сообщений (MsgType), которые в настоящее время поддерживаются:

| Значение MsgType | Тип |
| --- | --- |
| TIMTextElem | Текстовое сообщение |
| TIMLocationElem | Географическое местоположение |
| TIMFaceElem | Эмодзи |
| TIMCustomElem | Пользовательское сообщение. Когда получателем является устройство iOS и приложение работает в фоновом режиме, этот тип сообщения предоставляет в APNs поля, отличные от текста. Комбинированное сообщение может содержать только один элемент пользовательского сообщения TIMCustomElem. |
| TIMSoundElem | Голосовое сообщение |
| TIMImageElem | Сообщение с изображением |
| TIMFileElem | Сообщение с файлом |
| TIMVideoFileElem | Видеосообщение |

> **Внимание:** Сообщения указанных выше типов могут быть отправлены серверными интегрированными REST API.

## TIMMsgElement

### Элемент текстового сообщения

```
{    "MsgType": "TIMTextElem",    "MsgContent": {        "Text": "hello world"    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Text | String | Содержимое сообщения. Когда получателем является устройство iOS или Android и приложение работает в фоновом режиме, содержимое сообщения отображается как текст автономной отправки. |

Когда получателем является устройство iOS или Android и приложение работает в фоновом режиме, поле Text в теле JSON-запроса отображается как текст автономной отправки.

### Элемент сообщения о местоположении

```
{    "MsgType": "TIMLocationElem",     "MsgContent": {        "Desc": "someinfo",         "Latitude": 29.340656774469956,         "Longitude": 116.77497920478824    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Desc | String | Описание местоположения |
| Latitude | Number | Широта |
| Longitude | Number | Долгота |

Когда получателем является устройство iOS или Android и приложение работает в фоновом режиме, текст автономной отправки — это [Position] для китайской версии и [Location] для английской версии.

### Элемент сообщения-эмодзи

```
{    "MsgType": "TIMFaceElem",     "MsgContent": {        "Index": 1,         "Data": "content"    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Index | Number | Индекс эмодзи, определяемый пользователем |
| Data | String | Дополнительные данные |

Когда получателем является устройство iOS или Android и приложение находится в фоновом режиме, текст автономной отправки — [Expression] для английской версии.

> **Описание:** Когда сообщение содержит только один элемент пользовательского сообщения `TIMCustomElem`, если не введены оба поля `Desc` и `OfflinePushInfo.Desc`, автономная отправка сообщения не будет получена, если только не будет введено поле `OfflinePushInfo.Desc`.

### Элемент пользовательского сообщения

> **Примечание:** Когда сообщение содержит только один элемент пользовательского сообщения TIMCustomElem, поле OfflinePushInfo.Desc требует заполнения для получения автономной отправки сообщения.

```
{    "MsgType": "TIMCustomElem",     "MsgContent": {        "Data": "message"    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Data | String | Данные пользовательского сообщения. |

### Элемент голосового сообщения

> **Внимание:** Для отправки голосового сообщения через серверный REST API необходимо ввести поля `Url`, `UUID` и `Download_Flag`. Убедитесь, что голосовое сообщение можно загрузить через `URL`, а `UUID` — это глобально уникальное строковое значение, обычно значение MD5 голосового файла. Получатель сообщения может получить указанный `UUID` через `V2TIMSoundElem.getUUID()`, и приложение может использовать `UUID` для идентификации голоса. `Download_Flag` должен быть установлен на `2`.

Версии 4.X и выше Chat SDK (для Android, iOS, macOS и Windows) отправляют элементы голосового сообщения в следующем формате:

```
{    "MsgType": "TIMSoundElem",    "MsgContent": {        "Url": "https://1234-5678187359-1253735226.cos.ap-shanghai.myqcloud.com/abc123/c9be9d32c05bfb77b3edafa4312c6c7d",        "UUID": "1053D4B3D61040894AC3DE44CDF28B3EC7EB7C0F",        "Size": 62351,        "Second": 1,        "Download_Flag": 2    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Url | String | URL для загрузки голоса, через который содержимое голоса можно загрузить напрямую. |
| UUID | String | Уникальный идентификатор голоса, значение ключа, используемое клиентом для индексации голоса. |
| Size | Number | Размер данных голоса в байтах. |
| Second | Number | Продолжительность голоса в секундах. |
| Download_Flag | Number | Флаг метода загрузки голоса. В настоящее время значение `Download_Flag` должно быть `2`, что означает, что содержимое голоса можно загрузить с URL-адреса, указанного в поле `Url`. |

> **Примечание:** Версии 2.X и 3.X Chat SDK (для Android, iOS, macOS и Windows) отправляют элементы голосового сообщения в следующем формате:

```
{    "MsgType": "TIMSoundElem",    "MsgContent": {        "UUID": "305c0201", //Уникальный идентификатор голоса строкового типа. Это значение ключа, используемое клиентом для индексации голоса. Голос не может быть загружен через это поле. Для получения голоса обновите Chat SDK до версии 4.X.        "Size": 62351,      //Размер данных голоса в байтах, числовой тип.        "Second": 1         //Продолжительность голоса в секундах, числовой тип.    }}
```

### Элемент сообщения с изображением

> **Внимание:** Для отправки сообщения с изображением через серверный REST API необходимо ввести поля `URL`, `UUID`, `Width` и `Height`. Убедитесь, что изображение можно загрузить через `URL`, чтобы вы могли [получить основную информацию об изображении](https://intl.cloud.tencent.com/document/product/1045/33722) и [обработать изображение](https://intl.cloud.tencent.com/document/product/1045/33726). `Width` и `Height` — это ширина и высота изображения (единица измерения: пиксели). `UUID` должен быть установлен на глобально уникальное строковое значение, обычно значение MD5 изображения. Получатель сообщения может получить объект `V2TIMImage`, вызвав `V2TIMImageElem.getImageList()`, а затем получить указанный `UUID`, вызвав `V2TIMImage.getUUID()`. Приложение может использовать `UUID` для идентификации изображения.

```
{    "MsgType": "TIMImageElem",    "MsgContent": {        "UUID": "1853095_D61040894AC3DE44CDFFFB3EC7EB720F",        "ImageFormat": 1,        "ImageInfoArray": [            {                "Type": 1,           //Оригинальное изображение                "Size": 1853095,                "Width": 2448,                "Height": 3264,                "URL": "http://xxx/3200490432214177468_144115198371610486_D61040894AC3DE44CDFFFB3EC7EB720F/0"            },            {                "Type": 2,      //Большое изображение                "Size": 2565240,                "Width": 0,                "Height": 0,                "URL": "http://xxx/3200490432214177468_144115198371610486_D61040894AC3DE44CDFFFB3EC7EB720F/720"            },            {                "Type": 3,   //Миниатюра                "Size": 12535,                "Width": 0,                "Height": 0,                "URL": "http://xxx/3200490432214177468_144115198371610486_D61040894AC3DE44CDFFFB3EC7EB720F/198"            }        ]    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| UUID | String | Уникальный идентификатор изображения, значение ключа, используемое клиентом для индексации изображения |
| ImageFormat | Number | Формат изображения. JPG = 1, GIF = 2, PNG = 3, BMP = 4, Others = 255. |
| ImageInfoArray | Array | Информация загрузки оригинального изображения, миниатюры или большого изображения. |
| Type | Number | Тип изображения. 1: оригинальное изображение, 2: большое изображение, 3: миниатюра. |
| Size | Number | Размер данных изображения в байтах. |
| Width | Number | Ширина изображения в пикселях. |
| Height | Number | Высота изображения в пикселях. |
| URL | String | URL загрузки изображения. |

### Элемент сообщения с файлом

> **Внимание:** Для отправки сообщения с файлом через серверный REST API необходимо ввести поля `URL`, `UUID` и `Download_Flag`. Убедитесь, что файл можно загрузить через `URL`, а `UUID` — это глобально уникальное строковое значение, обычно значение MD5 файла. Получатель сообщения может получить указанный `UUID`, вызвав `V2TIMFileElem.getUUID()`, и бизнес-приложение может использовать `UUID` для идентификации файла. `Download_Flag` должен быть установлен на `2`.

Версии 4.X и выше Chat SDK (для Android, iOS, macOS и Windows) отправляют элементы сообщения с файлом в следующем формате:

```
{    "MsgType": "TIMFileElem",    "MsgContent": {        "Url": "https://7492-5678539059-1253735326.cos.ap-shanghai.myqcloud.com/abc123/49be9d32c0fbfba7b31dafa4312c6c7d",        "UUID": "1053D4B3D61040894AC3DE44CDF28B3EC7EB7C0F",        "FileSize": 1773552,        "FileName": "file:///private/var/Application/tmp/trim.B75D5F9B-1426-4913-8845-90DD46797FCD.MOV",        "Download_Flag": 2    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Url | String | URL загрузки файла, через который файл можно загрузить напрямую. |
| UUID | String | Уникальный идентификатор файла, значение ключа, используемое клиентом для индексации файла. |
| FileSize | Number | Размер данных файла в байтах. |
| fileName | String | Имя файла. |
| Download_Flag | Number | Флаг метода загрузки файла. В настоящее время значение `Download_Flag` должно быть 2, что означает, что файл можно загрузить с URL-адреса, указанного в поле `Url`. |

> **Примечание:** Версии 2.X и 3.X Chat SDK (для Android, iOS, macOS и Windows) отправляют элементы сообщения с файлом в следующем формате:{    "MsgType": "TIMFileElem",    "MsgContent": {        "UUID": "305c02010", //Уникальный идентификатор файла строкового типа. Это значение ключа, используемое клиентом для индексации файла. Файл не может быть загружен через это поле. Для получения файла обновите Chat SDK до версии 4.X.        "FileSize": 1773552, //Размер данных файла в байтах, числовой тип.        "FileName": "file:///private/var/Application/tmp/trim.B75D5F9B-1426-4913-8845-90DD46797FCD.MOV" //Имя файла, строковый тип.    }}

### Элемент видеосообщения

> **Внимание:** Для отправки видеосообщения через серверный REST API необходимо ввести поля `VideoUrl`, `VideoUUID`, `ThumbUrl`, `ThumbUUID`, `ThumbWidth`, `ThumbHeight`, `VideoDownloadFlag` и `ThumbDownloadFlag`. Убедитесь, что видео и миниатюра видео могут быть загружены через `VideoUrl` и `ThumbUrl` соответственно. `VideoUUID` и `ThumbUUID` должны быть глобально уникальными строковыми значениями, обычно значениями MD5 видео и миниатюры видео. Получатель сообщения может получить указанные `VideoUUID` и `ThumbUUID`, вызвав `V2TIMVideoElem.getVideoUUID()` и `V2TIMVideoElem.getSnapshotUUID()` соответственно. Приложение может использовать `VideoUUID` для идентификации видео. `VideoDownloadFlag` и `ThumbDownloadFlag` должны быть установлены на `2`.

Версии 4.X и выше Chat SDK (для Android, iOS, macOS и Windows) отправляют элементы видеосообщения в следующем формате:

```
{    "MsgType": "TIMVideoFileElem",    "MsgContent": {        "VideoUrl": "https://0345-1400187352-1256635546.cos.ap-shanghai.myqcloud.com/abcd/f7c6ad3c50af7d83e23efe0a208b90c9",        "VideoUUID": "5da38ba89d6521011e1f6f3fd6692e35",        "VideoSize": 1194603,        "VideoSecond": 5,        "VideoFormat": "mp4",        "VideoDownloadFlag":2,        "ThumbUrl": "https://0345-1400187352-1256635546.cos.ap-shanghai.myqcloud.com/abcd/a6c170c9c599280cb06e0523d7a1f37b",        "ThumbUUID": "6edaffedef5150684510cf97957b7bc8",        "ThumbSize": 13907,        "ThumbWidth": 720,        "ThumbHeight": 1280,        "ThumbFormat": "JPG",        "ThumbDownloadFlag":2    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| VideoUrl | String | URL загрузки видео, через который видео можно загрузить напрямую. |
| VideoUUID | String | Уникальный идентификатор видео, значение ключа, используемое клиентом для индексации видео. |
| VideoSize | Number | Размер данных видео в байтах. |
| VideoSecond | Number | Продолжительность видео в секундах. Для веб-клиентов это поле недоступно и по умолчанию установлено на 0. |
| VideoFormat | String | Формат видео, например MP4. |
| VideoDownloadFlag | Number | Флаг, указывающий метод загрузки видео. В настоящее время значение `VideoDownloadFlag` должно быть 2, что означает, что видео можно загрузить с URL-адреса, указанного в поле `VideoUrl`. |
| ThumbUrl | String | URL загрузки миниатюры видео, через который миниатюра видео можно загрузить напрямую. |
| ThumbUUID | String | Уникальный идентификатор миниатюры видео, значение ключа, используемое клиентом для индексации миниатюры видео. |
| ThumbSize | Number | Размер данных миниатюры в байтах. |
| ThumbWidth | Number | Ширина миниатюры в пикселях. |
| ThumbHeight | Number | Высота миниатюры в пикселях. |
| ThumbFormat | String | Формат миниатюры видео, например JPG или BMP. |
| ThumbDownloadFlag | Number | Флаг метода загрузки миниатюры видео. В настоящее время значение `ThumbDownloadFlag` должно быть 2, что означает, что миниатюра видео можно загрузить с URL-адреса, указанного в поле `ThumbUrl`. |

> **Примечание:** Версии 2.X и 3.X Chat SDK (для Android, iOS, macOS и Windows) отправляют элементы видеосообщения в следующем формате:{    "MsgType": "TIMVideoFileElem",    "MsgContent": {        "VideoUUID": "1400123456_dramon_34ca36be7dd214dc50a49238ef80a6b5",//Уникальный идентификатор видео строкового типа. Это значение ключа, используемое клиентом для индексации видео. Видео не может быть загружено через это поле. Для получения видео обновите Chat SDK до версии 4.X. "VideoFormat": "mp4", //Формат видео, строковый тип, например MP4.        "ThumbUUID": "1400123456_dramon_893f5a7a4872676ae142c08acd49c18a",//Уникальный идентификатор миниатюры видео строкового типа. ### Элементы комбинированного сообщения

> **Внимание:** Отправку и получение комбинированных сообщений поддерживают только собственный SDK версии 5.2.210 и выше, а также веб-SDK версии 2.10.1 и выше.

```
{  "MsgType": "TIMRelayElem",  "MsgContent": {    "Title": "Group chat history",    "MsgNum": 2,    "CompatibleText": "The SDK version does not support combined messages. Please upgrade to the latest version.",    "AbstractList": [      "A: What do you think of this?",      "B: I think it's great."    ],    "MsgList": [      {        "From_Account": "A",        "GroupId": "group1",        "MsgSeq": 85,        "MsgRandom": 3998651049,        "MsgTimeStamp": 1664437702,        "MsgBody": [          {            "MsgContent": {              "Text": " What do you think of this?"            },            "MsgType": "TIMTextElem"          }        ]      },      {        "From_Account": "B",        "GroupId": "group1",        "MsgSeq": 86,        "MsgRandom": 965790,        "MsgTimeStamp": 1664437703,        "MsgBody": [          {            "MsgContent": {              "Text": "I think it's great."            },            "MsgType": "TIMTextElem"          }        ]      }    ]  }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Title | String | Название комбинированного сообщения. |
| MsgNum | Integer | Количество переадресованных сообщений. |
| CompatibleText | String | Совместимый текст. Когда SDK более ранней версии, который не поддерживает комбинированные сообщения, получает комбинированное сообщение, бэкэнд Chat преобразует сообщение в совместимый текст и доставляет его. |
| AbstractList | Array | Массив строк, содержащих краткие описания комбинированного сообщения. |
| MsgList | Array | Список сообщений. Это поле доступно только, если общая длина комбинированных сообщений меньше или равна 12 КБ, в этом случае поле `JsonMsgKey` недоступно. |
| JsonMsgKey | String | Ключ списка комбинированных сообщений. Это поле доступно только, если общая длина комбинированных сообщений больше 12 КБ, в этом случае поле `MsgList` недоступно. |

Структура каждого сообщения в `MsgList` выглядит следующим образом:

| Поле | Тип | Описание |
| --- | --- | --- |
| From_Account | String | UserID отправителя сообщения. Это поле доступно, когда переадресованное сообщение является сообщением C2C или группы. |
| To_Account | String | UserID получателя сообщения. Это поле доступно только, когда переадресованное сообщение является сообщением C2C. |
| GroupId | String | Идентификатор группы. Это поле доступно только, когда переадресованное сообщение является сообщением группы. |
| MsgSeq | Integer | Порядковый номер сообщения (32-битное беззнаковое целое число). |
| MsgRandom | Integer | Случайное число сообщения (32-битное беззнаковое целое число). |
| MsgTimeStamp | Integer | Временная метка сообщения в секундах. |
| MsgBody | Array | Тело сообщения. Для получения сведений о форматах см. раздел [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). (Примечание: сообщение может содержать несколько элементов сообщения, в этом случае `MsgBody` является массивом.) |
| CloudCustomData | String | Данные пользовательского сообщения. Они сохраняются в облаке и будут отправлены в другую сторону. Такие данные можно получить после удаления и переустановки приложения. |

## Примеры MsgBody

### Сообщение с одним элементом текста

Одиночное сообщение содержит только один элемент текстового сообщения. Текстовое содержимое — **hello world**.

```
{    "MsgBody": [        {            "MsgType": "TIMTextElem",             "MsgContent": {                "Text": "hello world"            }        }    ]}
```

### Комбинированные сообщения

Следующее одиночное сообщение содержит два элемента текстового сообщения и один элемент сообщения-эмодзи в последовательности текст + эмодзи + текст.

```
{    "MsgBody": [        {            "MsgType": "TIMTextElem",             "MsgContent": {                "Text": "hello"            }        },         {            "MsgType": "TIMFaceElem",             "MsgContent": {                "Index": 1,                 "Data": "content"            }        },         {            "MsgType": "TIMTextElem",             "MsgContent": {                "Text": "world"            }        }    ]}
```

> **Внимание:** Комбинированное сообщение может содержать только один элемент пользовательского сообщения `TIMCustomElem` и неограниченное количество других элементов сообщения.

## Описание данных пользовательского сообщения CloudCustomData

Каждое сообщение может содержать пользовательские данные `CloudCustomData`.

`CloudCustomData` сохраняется в облаке вместе с `MsgBody` сообщения. Они будут отправлены получателю и могут быть получены даже после удаления и переустановки приложения.

Следующий пример показывает форматы `CloudCustomData` и `MsgBody`:

```
{    "MsgBody": [        {            "MsgType": "TIMTextElem",             "MsgContent": {                "Text": "hello"            }        }    ],    "CloudCustomData": "your cloud custom data"}
```

## [Общий параметр (описание OfflinePushInfo)](https://www.tencentcloud.com/document/product/1047/77715)

OfflinePushInfo — это JSON Object, предназначенный для конфигурации автономной отправки, позволяющий настроить отключение отправки для сообщения, текст отправки, строку сквозной передачи и т. д. Если вы заполните соответствующее поле в OfflinePushInfo, конфигурация, связанная с автономной отправкой в TIMCustomElem, будет игнорирована. Подробную информацию см. в разделе [Общий параметр (описание OfflinePushInfo)](https://www.tencentcloud.com/document/product/1047/77715).

Форматы примеров для MsgBody и OfflinePushInfo выглядят следующим образом:

```
{    "MsgBody": [...], // Related description here    "OfflinePushInfo": {        "PushFlag": 0,        "Title":"Offline Push Title"        "Desc": "Offline push content"        "Ext": "{\\"entity\\":{\\"key1\\":\\"value1\\",\\"key2\\":\\"value2\\"}}",  // passthrough field, push uses string in json format        "AndroidInfo": {             // ...        },        "ApnsInfo": {            // ...        }    }}
```

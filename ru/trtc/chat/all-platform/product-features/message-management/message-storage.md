# Хранение сообщений

## Роуминг сообщений

Chat поддерживает роуминг сообщений, что означает, что когда пользователи входят на разные устройства, они по-прежнему имеют доступ к истории чата.

По умолчанию сообщения C2C-чата и групповые сообщения хранятся в течение 7 дней, после чего удаляются. Подробнее о выставлении счетов см. в разделе [Цены](https://intl.cloud.tencent.com/document/product/1047/34350).
Различные версии SDK позволяют продлить время хранения исторических сообщений для разных типов сообщений.

## Хранение непрочитанных сообщений

Chat поддерживает хранение непрочитанных сообщений; то есть сообщения, отправленные во время автономного режима пользователя, будут сохранены и получены при следующем входе.

Для C2C Chat, непрочитанные сообщения в до 100 диалогов сохраняются для каждого пользователя в течение 7 дней по умолчанию, с до 250 непрочитанными сообщениями в каждом диалоге. Сообщения, превышающие лимиты, не будут считаться непрочитанными сообщениями, но они по-прежнему будут сохранены в роуминге сообщений. Для группового чата таких ограничений нет.

## Недавние контакты

Подобно списку недавних контактов в QQ, недавние контакты отображают пользователей, которые недавно связались с текущим пользователем, а также их последние сообщения.

По умолчанию клиент будет получать недавние контакты через SDK при входе для отображения списка диалогов, и последние 100 контактов сохраняются по умолчанию в течение того же периода, что и последние сообщения; например, если контакт не отправлял и не получал сообщения в течение 7 дней, этот контакт не может быть получен в недавних контактах после истечения последнего сообщения.

## Локальное хранилище приложения

Пользователям не нужно хранить сообщения, так как SDK по умолчанию сохраняет полученные сообщения. Пользователи могут вызвать API для получения локальных сообщений (не требуется сетевое соединение). Кроме того, локальные сообщения можно получить через API getMessage. Когда в локальной истории сообщений есть пробелы, они заполняются роуминг-сообщениями.

По умолчанию SDK не удаляет сообщения пользователей, но мы предоставляем возможность удалять локальные сообщения.

# Личная переписка (One-to-One Message)

## Варианты использования

- **Чат C2C в приложении**: Сообщения C2C применимы к чатам C2C в приложении.
- **Администратор приложения отправляет сообщения**: Сообщения C2C могут быть отправлены администратором приложения из серверной части или администратором приложения, имитирующим других пользователей.
- **Администратор приложения имитирует системные сообщения**: Администратор приложения может имитировать отправку системных сообщений из серверной части. Эти имитируемые сообщения представляют собой уведомления, доставляемые пользователям, и пользовательские сообщения от администратора приложения, полученные приложением, обрабатываются специальным образом.

Система обмена мгновенными сообщениями (Chat) предоставляет комплексные возможности обмена сообщениями C2C. При этом контроль разрешений и возможности расширения обмена сообщениями C2C предоставляют функции, такие как получение истории сообщений, синхронизация между устройствами, отправка автономных сообщений и передача профилей отправителя в сообщениях.

## Типы сообщений C2C

| Тип сообщения | Описание |
| --- | --- |
| Текстовое сообщение | Содержимое сообщения — простой текст. |
| Сообщение с эмодзи | Сообщения с эмодзи настраиваются разработчиками. |
| Сообщение о местоположении | Содержимое сообщения включает название, долготу и широту местоположения. |
| Сообщение с изображением | Содержимое сообщения включает URL-адрес, размеры и размер файла изображения. Максимальный поддерживаемый размер файла изображения составляет 28 МБ. |
| Голосовое сообщение | Содержимое сообщения включает URL-адрес, размер и длительность голоса. Максимальный поддерживаемый размер файла голоса составляет 28 МБ. |
| Сообщение с файлом | Содержимое сообщения включает URL-адрес, размер и формат файла. Поддерживаются все форматы файлов, максимальный поддерживаемый размер файла составляет 100 МБ. |
| Сообщение с коротким видео | Содержимое сообщения включает URL-адрес, длительность, размер и формат короткого видео. Максимальный поддерживаемый размер файла короткого видео составляет 100 МБ. |
| Пользовательское сообщение | Типы сообщений, настраиваемые разработчиками, такие как красный конверт и камень-ножницы-бумага. |
| Сообщение системного уведомления | Этот тип сообщений делится на встроенные сообщения системного уведомления и сообщения системного уведомления, настраиваемые разработчиками. |

| --- | --- | --- |

| Получение сообщений C2C | Сообщения C2C можно получать через SDK. | Получение сообщений онлайн. Получение автономных сообщений. Запрос исторических сообщений. |

## Контроль разрешений для обмена сообщениями C2C

| Контроль разрешений для обмена сообщениями C2C | Описание | Варианты использования |
| --- | --- | --- |
| Два пользователя приложения могут отправлять сообщения C2C. | Любые два незнакомца могут отправлять сообщения C2C друг другу. | Незнакомцы отправляют сообщения C2C друг другу. |
| Администратор приложения отправляет сообщения C2C. | Администратор приложения может отправлять сообщения C2C любому пользователю. | Администратор приложения выдаёт себя за других пользователей при отправке сообщений. Администратор приложения отправляет системные сообщения. |
| Только друзья могут отправлять сообщения C2C друг другу. | Только друзья могут отправлять сообщения C2C друг другу. | Друзья отправляют сообщения C2C друг другу. |
| Блокировка сообщений от кого-либо. | Вы можете добавить пользователя в чёрный список, чтобы заблокировать его сообщения. | Удаление из друзей. Блокировка сообщений от кого-либо. |

## Обработка автономных сообщений C2C

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/d3ea4ea552d111ee974d5254005f490f.png)

#### Кэширование в автономном режиме и роуминг сообщений C2C:

1. Пользователь A вызывает `sendMessage` для отправки сообщений пользователю B, который находится в автономном режиме.
- Пользователь A добавляется в недавние контакты пользователя B с кэшированием до 100 сообщений.
- Сообщения сохраняются в кэш автономного режима на 7 дней.
- Сообщения сохраняются на сервере роуминга на 7 дней.
2. Пользователь B вызывает API `login` для входа в Chat.
3. SDK автоматически извлекает сообщения из кэша автономного режима и доставляет через обратный вызов `OnNewMessage`.
4. SDK автоматически извлекает недавние контакты и доставляет через обратный вызов `OnNewMessage`.
5. Пользователь получает уведомление через обратный вызов `OnRefresh` при завершении синхронизации сообщений.
6. Пользователь вызывает `getMessage`. Если локальные сообщения неполные, SDK автоматически извлекает их с сервера роуминга.

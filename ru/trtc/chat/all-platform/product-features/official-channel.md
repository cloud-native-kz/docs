# Официальный канал 

Функции официального аккаунта аналогичны функциям официальных аккаунтов WeChat, предоставляя клиентам возможность подписки и публикации. Через соответствующие интерфейсы официальных аккаунтов пользователи могут создавать и управлять официальными аккаунтами, публиковать статьи и мультимедийные сообщения, взаимодействовать с подписчиками, а подписчики также могут отправлять личные сообщения на официальный аккаунт. При этом серверная часть приложения может статистически управлять и обрабатывать соответствующие данные через RestAPI и сторонние обратные вызовы.

## Введение в структуру данных официального аккаунта

### Основная информация официального аккаунта

| Имя поля | Тип | Описание | Замечания |
| --- | --- | --- | --- |
| Official_Account | String | Уникальный идентификатор официального аккаунта | Только для чтения. ID официального аккаунта, гарантированно уникальный в пределах приложения, его префикс формата @TOA#_. Кроме того, приложение также может самостоятельно определить ID официального аккаунта. |
| Owner_Account | String | ID создателя официального аккаунта | Только для чтения. |
| Name | String | Имя официального аккаунта | Читаемо и записываемо. Максимальная длина 150 байт, не может быть изменено. |
| Introduction | String | Описание официального аккаунта | Читаемо и записываемо. Максимальная длина 400 байт, не может быть изменено. |
| FaceUrl | String | URL аватара официального аккаунта | Читаемо и записываемо. Максимальная длина 500 байт, не может быть изменено. |
| MaxSubscriberNum | Integer | Максимальное количество подписчиков официального аккаунта | Читаемо и записываемо. Текущее значение по умолчанию — 100000. |
| SubscriberNum | Integer | Текущее количество людей, подписанных на официальный аккаунт | Только для чтения. |
| LastMsgTime | Integer | Время последнего сообщения, отправленного между официальным аккаунтом и подписчиком | Только для чтения. |
| CreateTime | Integer | Время создания официального аккаунта | Только для чтения. |
| Organization | String | Групповая организация, к которой принадлежит официальный аккаунт | Читаемо и записываемо. Максимальная длина 500 байт, не может быть изменено. |
| CustomString | String | Поле пользовательского определения для информационных измерений официального аккаунта | Читаемо и записываемо. Максимальная длина 3000 байт, не может быть изменено. |

### Основная информация подписчика

| Имя поля | Тип | Описание | Замечания |
| --- | --- | --- | --- |
| Subscriber_Account | String | ID пользователя подписчика | Только для чтения. |
| SubscribeTime | Integer | Период подписки | Только для чтения. |
| MsgFlag | String | Опция получения сообщений | Опции получения сообщений, включая: AcceptAndNotify означает получать и уведомлять; AcceptNotNotify означает получать без уведомления (не будет запускать офлайн-push); Discard означает блокировать сообщения официального аккаунта (сообщения не будут переданы клиенту) |
| CustomString | String | Поле пользовательского определения измерения пользователя подписчика | Читаемо и записываемо. Максимальная длина 3000 байт, не может быть изменено. |

## Пользовательское определение ID официального аккаунта

По умолчанию при создании приложением официального аккаунта Chat присваивает новому официальному аккаунту ID по умолчанию. Этот ID будет начинаться с @TOA#_ и гарантированно уникален в пределах приложения.
Чтобы сделать ID официального аккаунта проще, легче запомнить и распространять, Chat поддерживает определение пользовательского ID приложениями при создании официальных аккаунтов через REST API. Пользовательский ID должен состоять из печатных символов ASCII (0x20-0x7e), быть не длиннее 48 байт, должен начинаться с @TOA#_, но не может быть @TOA#_@TOA# (чтобы избежать путаницы с назначенным по умолчанию ID официального аккаунта).

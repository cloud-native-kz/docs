# Управление профилем

## Обзор системы профилей

Мессенджер (Chat) предоставляет полный набор решений для управления профилями через свою систему хранения профилей пользователей. Сервис хостинга профилей Chat позволяет вашим пользователям легко устанавливать и получать профили.

- Chat может хранить информацию профиля, обеспечивая, что ваши данные имеют удаленное аварийное восстановление, развертывание в нескольких регионах и возможности автоматического масштабирования. Таким образом, вы полностью избавлены от сложных процессов обработки, таких как простой сервера, репликация первичного-вторичного с несколькими копиями и масштабирование емкости.
- Chat предоставляет часто используемые в отрасли бизнес-процессы, о которых вам не нужно беспокоиться касательно логики бизнеса профилей пользователей.
- Chat предоставляет профессиональные процессы эксплуатации и команды, обеспечивая 99,99% качество обслуживания в год и помогая вам предоставлять услуги, известные своей стабильностью.
- Chat предоставляет удобные в использовании API сервиса и легкодоступные руководства с премиум-услугами на протяжении всего процесса.

Используя сервис хостинга профилей Chat, вы сможете:

- Сохранять, читать и записывать стандартные поля профиля.
- Сохранять, читать и записывать пользовательские поля профиля.

## Поля профиля

Профиль — это набор данных, описывающих свойства пользователя. Система профилей Chat поддерживает стандартные и пользовательские поля профиля. Поля профиля имеют следующие характеристики:

- Поля профиля выражаются в формате ключ-значение.
- Ключ имеет строковый формат, и его имя может содержать только прописные и строчные буквы, цифры и подчеркивания.
- Значение имеет следующие типы:
 a. Целое число типа uint32_t (не поддерживается для пользовательских полей)
 b. Целое число типа uint64_t (не поддерживается для пользовательских полей)
 c. Строка типа string (длина строки не может превышать 500 байт.)

## Пользовательские поля профиля

Пользовательские поля профиля — это пользовательские данные, установленные каждым приложением в соответствии с его собственными потребностями бизнеса. Используя пользовательские поля профиля, приложение может добавлять дополнительные данные к профилям пользователей и выполнять операции чтения и записи через существующие API.

### Подача заявки на пользовательские поля профиля

Для подачи заявки на пользовательские поля профиля войдите в [консоль Chat](https://console.trtc.io/) и выберите **Конфигурация приложения** > **Конфигурация функций**. После отправки вашего приложения пользовательские поля профиля вступят в силу в течение пяти минут.
При подаче заявки на пользовательские поля профиля необходимо предоставить следующую информацию для каждого поля:

- Имя пользовательского поля профиля (Key). Дополнительную информацию см. в разделе [Правила именования пользовательских полей профиля](https://www.tencentcloud.com/document/product/1047/33520#naming-rules-for-custom-profile-fields).
- Тип пользовательского поля профиля (Value). Дополнительную информацию см. в разделе [Поля профиля](#.E8.B5.84.E6.96.99.E5.AD.97.E6.AE.B5).

### Правила именования пользовательских полей профиля

Правила для именования пользовательских полей профиля следующие:

- Имя пользовательского поля профиля содержит части префикса и ключевого слова.
- Префикс пользовательского поля профиля: Tag_Profile_Custom.
- Ключевое слово: ключевое слово должно быть строкой букв длиной не более 8 байт. Рекомендуется использовать английское слово или его аббревиатуру в качестве ключевого слова.
- Пример: если пользовательское поле, запрашиваемое приложением, имеет ключевое слово **Test**, то имя пользовательского поля профиля: Tag_Profile_Custom_Test.

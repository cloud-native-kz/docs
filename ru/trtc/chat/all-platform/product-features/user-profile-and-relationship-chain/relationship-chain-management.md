# Управление цепочкой отношений

## Обзор системы контактов

Chat может размещать контакты пользователей и предлагает полный набор решений для работы с контактами. Если вы не хотите разрабатывать или поддерживать функции отношений дружбы для пользователей приложения, но вам нужны функции, такие как добавление и удаление друзей, вам следует использовать сервис размещения контактов Chat.

- Chat может хранить контакты, обеспечивая восстановление после сбоев, развертывание в несколько регионов и возможность автоматического масштабирования. Это избавляет вас от сложных процессов обработки, таких как простои серверов, репликация первичного-вторичного типа с несколькими копиями и масштабирование.
- Chat предоставляет процессы обработки бизнеса, которые широко используются в отрасли, благодаря чему вам не нужно беспокоиться о логике контактов.
- Chat предоставляет профессиональные процессы и команды операций, обеспечивая стабильность качества обслуживания 99,99% в год и помогая вам предоставлять стабильные услуги.
- Chat предоставляет удобные API услуг и простые руководства доступа с премиум-услугами на протяжении всего процесса.

Контакты — это набор данных, используемых для описания отношений между одним пользователем и другими пользователями. Контакты, поддерживаемые Chat, включают списки друзей и черные списки.

## Поля цепочки отношений

Система контактов Chat поддерживает стандартные и пользовательские поля контактов. Поля контактов имеют следующие характеристики:

- Поля контактов отображаются в формате ключ-значение.
- Ключ имеет строковый формат, и его имя может содержать только прописные и строчные буквы, цифры и подчеркивания.
- Значение имеет следующие типы:
  a. Целое число типа uint64_t (не поддерживается для пользовательских полей контактов)
  b. Строка типа string (длина строки не может превышать 500 байт)
  c. Буфер типа bytes (длина буфера не может превышать 500 байт)
  d. Массив строк типа string (длина каждой строки не может превышать 500 байт, и этот тип используется только для поля `Tag_SNS_IM_Group` списка друзей)

## Контакты

Пользователи могут добавить до 3000 друзей в свои контакты в Chat.
Контакты поддерживают стандартные поля друзей и пользовательские поля друзей.

### Стандартные поля друзей

В настоящее время Chat поддерживает следующие стандартные поля друзей:

| Поле | Тип | Описание |
| --- | --- | --- |
| Tag_SNS_IM_Group | Array | Список друзей: 1. Поддерживается максимум 32 списка. 2. Имя списка не может быть пустым. 3. Длина имени списка не может превышать 30 байт. 4. Один друг может быть добавлен в несколько списков. |
| Tag_SNS_IM_Remark | String | Заметка: Длина заметки не может превышать 96 байт. |
| Tag_SNS_IM_AddSource | String | Источник, из которого добавлен друг: 1. Поле источника содержит префикс и ключевое слово. 2. Префикс поля источника — `AddSource_Type_`. 3. Ключевое слово: Это должна быть комбинация букв не более 8 байт. Мы рекомендуем использовать английское слово или его аббревиатуру. 4. Пример: Если ключевое слово источника — Android, то поле источника — `AddSource_Type_Android` |
| Tag_SNS_IM_AddWording | String | Содержание запроса: Длина запроса не может превышать 256 байт. |
| Tag_SNS_IM_AddTime | Integer | Временная метка добавления друзей. |

### Пользовательские поля друзей

Пользовательские поля друзей — это набор данных о друзьях, устанавливаемый каждым приложением в соответствии с его собственными требованиями бизнеса. Используя пользовательские поля друзей, приложение может добавлять дополнительные данные к друзьям и выполнять операции чтения и записи через существующие API.
Чтобы подать заявку на пользовательские поля друзей, администратор приложения может войти в [консоль Chat](https://console.tencentcloud.com/im) и выбрать **App Configuration** > **Feature Configuration**. После отправки заявки пользовательские поля друзей вступят в силу в течение 5 минут.

Требования к именованию пользовательских полей друзей следующие:

- Имя пользовательского поля друга состоит из двух частей: префикса и ключевого слова.
- Префикс пользовательского поля друга — `Tag_SNS_Custom`.
- Ключевое слово: Ключевое слово должно быть строкой букв длиной не более 8 байт. Мы рекомендуем использовать английское слово или его аббревиатуру в качестве ключевого слова.
- Пример: Если пользовательское поле друга, запрашиваемое приложением, имеет ключевое слово `Test`, то имя пользовательского поля друга — `Tag_SNS_Custom_Test`.

При подаче заявки на пользовательские поля друзей вам необходимо предоставить следующую информацию для каждого пользовательского поля друга:

- Имя пользовательского поля друга (Key).
- Тип пользовательского поля друга (Value). Дополнительные сведения см. в разделе [Поля цепочки отношений](#relationship).

### Добавление друзей

Chat поддерживает следующие режимы добавления друзей: добавление друзей партиями, без одобрения и требующее одобрения. Дополнительные сведения см. в разделе [Добавление друзей](https://intl.cloud.tencent.com/document/product/1047/34902).

Двусторонние друзья: пользователь A является другом пользователя B, и пользователь B является другом пользователя A.
Односторонний друг: пользователь A является другом пользователя B, но пользователь B не является другом пользователя A.
Метод проверки при добавлении друзей: Каждый пользователь может выбрать способ, которым его добавляют в друзья другой пользователь. Дополнительные сведения см. в поле метода проверки при добавлении друзей в разделе [Стандартные поля профиля](https://intl.cloud.tencent.com/document/product/1047/33520).
Без одобрения: Если метод одобрения запросов дружбы, установленный аккаунтом A, — `AllowType_Type_AllowAny`, то любой, кто хочет добавить аккаунт A как друга, может напрямую добавить аккаунт A. В этом сценарии есть один этап в процессе запроса и принятия дружбы.
Требующее одобрения: Если метод одобрения запросов дружбы, установленный аккаунтом A, — `AllowType_Type_NeedConfirm`, то для любого, кто хочет добавить аккаунт A как друга, аккаунт A получит сообщение с запросом на одобрение нового запроса дружбы. Затем аккаунт A принимает или отклоняет запрос для завершения процесса. В этом сценарии есть два этапа в процессе запроса и принятия дружбы.

### Удаление друзей

Chat поддерживает два режима удаления друзей: одностороннее удаление и двустороннее удаление.

| Режим удаления | DeleteType | Описание |
| --- | --- | --- |
| Одностороннее удаление | Delete_Type_Single | `To_Account` удаляется из контактов `From_Account`, но `From_Account` не удаляется из контактов `To_Account`. |
| Двустороннее удаление | Delete_Type_Both | `To_Account` удаляется из контактов `From_Account`, и `From_Account` удаляется из контактов `To_Account`. |

Chat также поддерживает удаление друзей партиями. Дополнительные сведения см. в разделе [Удаление друзей](https://intl.cloud.tencent.com/document/product/1047/34905).

### Проверка друзей

Chat поддерживает два режима проверки друзей: одностороннюю проверку и двустороннюю проверку.

| Режим проверки | CheckType | Описание |
| --- | --- | --- |
| Одностороннее одобрение | CheckResult_Type_Single | Используется для проверки, находится ли `To_Account` в контактах `From_Account`, но не проверяет, находится ли `From_Account` в контактах `To_Account`. |
| Двустороннее одобрение | CheckResult_Type_Both | Используется для проверки, находится ли `To_Account` в контактах `From_Account`, и также проверяет, находится ли `From_Account` в контактах `To_Account`. |

Возможные результаты односторонней проверки дружбы:

| Отношение | Описание |
| --- | --- |
| CheckResult_Type_NoRelation | `To_Account` не находится в контактах `From_Account`, но невозможно определить, находится ли `From_Account` в контактах `To_Account`. |
| CheckResult_Type_AWithB | `To_Account` находится в контактах `From_Account`, но невозможно определить, находится ли `From_Account` в контактах `To_Account`. |

Возможные результаты двусторонней проверки дружбы:

| Отношение | Описание |
| --- | --- |
| CheckResult_Type_BothWay | `To_Account` находится в контактах `From_Account`, и `From_Account` находится в контактах `To_Account`. |
| CheckResult_Type_AWithB | `To_Account` находится в контактах `From_Account`, но `From_Account` не находится в контактах `To_Account`. |
| CheckResult_Type_BWithA | `To_Account` не находится в контактах `From_Account`, но `From_Account` находится в контактах `To_Account`. |
| CheckResult_Type_NoRelation | `To_Account` не находится в контактах `From_Account`, и `From_Account` не находится в контактах `To_Account`. |

Дополнительные сведения о проверке дружбы см. в разделе [Проверка друзей](https://intl.cloud.tencent.com/document/product/1047/34907).

## Черные списки

У каждого пользователя есть черный список, который используется для хранения аккаунтов, заблокированных этим пользователем.
После того, как пользователь A добавляет пользователя B в черный список, пользователь A удаляет пользователя B из друзей (если они были друзьями), и пользователи A и B не могут отправлять друг другу запросы дружбы в будущем.
Каждый пользователь может добавить до 1000 аккаунтов в свой черный список Chat.

### Добавление пользователей в черный список

Chat позволяет добавлять несколько пользователей в черный список одновременно. Дополнительные сведения см. в разделе [Добавление пользователей в черный список](https://intl.cloud.tencent.com/document/product/1047/34911).

### Удаление пользователей из черного списка

Chat позволяет удалять несколько пользователей из черного списка одновременно. Дополнительные сведения см. в разделе [Удаление пользователей из черного списка](https://intl.cloud.tencent.com/document/product/1047/34912).

### Получение черного списка

Chat поддерживает получение полного черного списка по страницам. Дополнительные сведения см. в разделе [Получение черного списка](https://intl.cloud.tencent.com/document/product/1047/34914).

### Проверка черного списка

Chat поддерживает два режима проверки черного списка: одностороннюю проверку и двустороннюю проверку.

| Режим проверки | CheckType | Описание |
| --- | --- | --- |
| Одностороннее одобрение | BlackCheckResult_Type_Single | Используется для проверки, находится ли `To_Account` в черном списке `From_Account`, но не проверяет, находится ли `From_Account` в черном списке `To_Account`. |
| Двустороннее одобрение | BlackCheckResult_Type_Both | Используется не только для проверки, находится ли `To_Account` в черном списке `From_Account`, но также для проверки, находится ли `From_Account` в черном списке `To_Account`. |

Возможные результаты односторонней проверки отношения с черным списком:

| Отношение | Описание |
| --- | --- |
| BlackCheckResult_Type_AWithB | `To_Account` находится в черном списке `From_Account`, но невозможно определить, находится ли `From_Account` в черном списке `To_Account`. |
| BlackCheckResult_Type_NO | `To_Account` не находится в черном списке `From_Account`, но невозможно определить, находится ли `From_Account` в черном списке `To_Account`. |

Возможные результаты двусторонней проверки отношения с черным списком:

| Отношение | Описание |
| --- | --- |
| BlackCheckResult_Type_BothWay | `To_Account` находится в черном списке `From_Account`, и `From_Account` также находится в черном списке `To_Account`. |
| BlackCheckResult_Type_AWithB | `To_Account` находится в черном списке `From_Account`, но `From_Account` не находится в черном списке `To_Account`. |
| BlackCheckResult_Type_BWithA | `To_Account` не находится в черном списке `From_Account`, но `From_Account` находится в черном списке `To_Account`. |
| BlackCheckResult_Type_NO | `To_Account` не находится в черном списке `From_Account`, и `From_Account` не находится в черном списке `To_Account`. |

Дополнительные сведения о проверке черного списка см. в разделе [Проверка черного списка](https://intl.cloud.tencent.com/document/product/1047/34913).

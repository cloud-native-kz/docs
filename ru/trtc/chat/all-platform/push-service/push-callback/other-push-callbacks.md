# Другие callback-функции push-уведомлений

#### Описание функции

После включения плагина push уведомлений результаты отправки можно перенаправлять на бэкенд приложения, настроив базовый callback.

#### Примечания

- Этот callback относится к callback-функциям событий после срабатывания нижестоящего уведомления. Чтобы предотвратить перегрузку бэкенда приложения большим количеством запросов callback, **QPS по умолчанию: 1000 раз/сек**. Слишком частые callback-функции будут повторены с задержкой. Если вам необходимо изменить QPS из-за требований бизнеса, вы можете отправить [запрос](https://trtc.io/zh/login/redirect?s_url=https://console.tencentcloud.com/workorder/category) на изменение.
- Чтобы включить этот callback, необходимо настроить URL-адрес callback и включить соответствующий переключатель для этого callback. Дополнительные сведения о методе конфигурации см. в документации [Webhooks](https://intl.cloud.tencent.com/document/product/1047/34520).
- Во время этого callback бэкенд Chat инициирует HTTP POST запрос на бэкенд приложения.
- После получения запроса callback бэкенд приложения должен проверить, соответствует ли SDKAppID, содержащийся в URL-адресе запроса, своему собственному SDKAppID.
- Для дополнительной информации о безопасности обратитесь к документу [Обзор callback-функций третьих сторон: Вопросы безопасности](https://intl.cloud.tencent.com/document/product/1047/34354).

### Описание API

#### Пример URL запроса:

В следующем примере URL-адрес callback, настроенный в конфигурации приложения, равен `https://www.example.com`

Пример:

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json
```

#### Параметры запроса

| Поле | Описание |
| --- | --- |
| https | Протокол запроса — HTTPSМетод запроса — POST |
| www.example.com | URL-адрес callback |
| SdkAppid | SDKAppID, выделенный консолью Instant Messaging при создании приложения |
| CallbackCommand | Фиксировано как: Push.OfflinePush |
| contenttype | Полезная нагрузка запроса фиксирована как JSON |

#### Описание полей пакета запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| Events | Array [ Event Object ] | Содержимое batch callback, содержит данные до 100 событий callback (Event Object) |

##### Структура Event Object

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда callback |
| EventType | Integer | Тип события:EventType = 1 указывает на чат один-на-одинEventType = 2 указывает на групповой чат |
| EventTime | Integer | Временная метка события в секундах |
| From_Account | String | UserID отправителя сообщения |
| To_Account | String | UserID получателя |
| PushPlatform | Integer | Поставщик:PushPlatform = 0 указывает на неизвестного поставщикаPushPlatform = 1 указывает на Apple APNS pushPushPlatform = 2 указывает на Xiaomi pushPushPlatform = 3 указывает на Huawei pushPushPlatform = 4 указывает на Google FCM pushPushPlatform = 5 указывает на Meizu pushPushPlatform = 6 указывает на OPPO pushPushPlatform = 7 указывает на vivo pushPushPlatform = 8 указывает на Honor push |
| PushStage | Integer | Этап отправки:PushStage = 1 указывает на этап отправки pushPushStage = 2 указывает на этап доставки pushPushStage = 3 указывает на этап нажатия push |
| MsgKey | String | Уникальный идентификатор сообщения один-на-один, действителен только при EventType = 1. Это поле пусто для групповых чатов. |
| GroupID | String | ID группы для групповых чатов, действителен только при EventType = 2. Это поле пусто для чатов один-на-один. |
| MsgSeq | Integer | Порядковый номер сообщения, используется для обозначения соответствующего сообщения (32-битное целое число без знака) |
| MsgRandom | Integer | Случайное число сообщения, используемое для обозначения сообщения (32-битное целое число без знака). Это поле пусто для групповых чатов. |
| MsgTime | Integer | Временная метка отправки сообщения в секундах |
| PushID | String | Уникальный идентификатор отправки, может использоваться для запроса статуса доставки push в консоли Chat с помощью PushID |
| ErrCode | Integer | Результат события отправки:ErrCode = 0 указывает на успехErrCode, отличный от нуля, указывает на ошибку |
| ErrInfo | String | Описание результата события отправки, может быть пусто |

#### Пример пакета ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // 0 означает успешный callback, 1 означает ошибку callback}
```

#### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработанного запроса:OK указывает на успешную обработкуFAILURE указывает на неудачное выполнение |
| ErrorCode | Integer | Код ошибки |
| ErrorInfo | String | Описание ошибки |

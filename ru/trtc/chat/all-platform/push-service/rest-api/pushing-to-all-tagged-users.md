# Отправка всем/помеченным пользователям

All-user/Tag Push поддерживает отправку конкретного контента и может также отправлять персонализированный контент определённым группам пользователей на основе тегов и атрибутов, например, деятельность членства, региональные уведомления и т.д. Это помогает в привлечении пользователей, конверсии, активизации и других операционных этапах, а также поддерживает отчёты о доставке и инструменты самостоятельного устранения неполадок при отправке. Подробнее см. [Отображение эффектов](https://www.tencentcloud.com/zh/document/product/1047/60537).

## Обзор функций

- Поддержка отправки всем пользователям.
- Поддержка отправки по атрибутам пользователя.
- Поддержка отправки по тегам пользователя.
- Поддержка онлайн-каналов и каналов поставщиков (APNS, Huawei, Honor, OPPO, vivo, Xiaomi, Meizu, Google).
- Из-за большого количества учётных записей, которые необходимо доставить для All-user/Tag Push, требуется определённое время для доставки всем учётным записям (в зависимости от общего количества учётных записей).
- По умолчанию не увеличивает счётчик непрочитанных сообщений.

> **Примечание:** Этот API поддерживает как онлайн, так и оффлайн-отправку. Онлайн-отправка поддерживается только в версии SDK >= 8.2.6325.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/timpush/push?usersig=xxx&identifier=admin&sdkappid=88888888&random=99999999&contenttype=json
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса: HTTPSМетод запроса: POST |
| xxxxxx | Страна/регион, где находится ваш SDKAppID.Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Сеул: `adminapikr.im.qcloud.com`Франкфурт: `adminapiger.im.qcloud.com`Кремниевая долина: `adminapiusa.im.qcloud.com`Джакарта: `adminapiidn.im.qcloud.com` |
| v4/timpush/push | API запроса |
| usersig | Подпись, созданная учётной записью администратора приложения. Подробности процедуры см. в разделе [Генерирование UserSig](https://www.tencentcloud.com/document/product/1047/34385) |
| identifier | Должна быть учётная запись администратора приложения. Дополнительные сведения см. в разделе [Администратор приложения](https://www.tencentcloud.com/document/product/1047/33517) |
| sdkappid | SdkAppid, назначенный консолью Chat при создании приложения |
| random | Случайное 32-битное беззнаковое целое число в диапазоне от 0 до 4294967295 |
| contenttype | Фиксированное значение: `json` |

### Ограничение частоты вызовов

Этот API включает отправку всем пользователям, атрибутам и Tag Push. Бесплатный выпуск позволяет по умолчанию 10 отправок в день, в то время как стандартный выпуск и расширенный выпуск позволяют 100 отправок в день. Интервал отправки между двумя отправками должен быть больше 1 секунды.

### Примеры пакетов запросов

> **Примечание:** Этот API поддерживает как онлайн, так и оффлайн-отправку. Онлайн-отправка поддерживается только в версии SDK >= 8.2.6325.

#### **Примеры All-user Push**

- Администратор отправляет всем пользователям:

```
{  "From_Account": "administrator",  "MsgRandom": 3674128,  "OfflinePushInfo": {      "PushFlag": 0,  // 0 означает включённую оффлайн-отправку, 1 означает отключённую оффлайн-отправку      "Title": "Offline Push Title",      "Desc": "Offline Push Content"  }}
```

- Отправка только онлайн-пользователям (Online Push):

```
{  "From_Account": "administrator",  "MsgRandom": 3674128,  "OfflinePushInfo": {      "PushFlag": 1,  // 0 означает включённую оффлайн-отправку, 1 означает отключённую оффлайн-отправку      "Ext": "Transparent content"  }}
```

#### **Примеры Push по тегу пользователя**

- Администратор отправляет пользователям с тегами "Stock A" и "Stock B":

```
{  "From_Account": "administrator",  "MsgRandom": 124032,  "Condition": {      "TagsAnd": ["Stock A","Stock B"]  },  "OfflinePushInfo": {      "PushFlag": 0, // 0 означает включённую оффлайн-отправку, 1 означает отключённую оффлайн-отправку      "Title": "Offline Push Title",      "Desc": "Offline Push Content"  }}
```

- Администратор отправляет пользователям с тегами "Stock A" или "Stock B":

```
{  "From_Account": "administrator",  "MsgRandom": 124032,  "Condition": {      "TagsOr": ["Stock A","Stock B"]  },  "OfflinePushInfo": {      "PushFlag": 0, // 0 означает включённую оффлайн-отправку, 1 означает отключённую оффлайн-отправку      "Title": "Offline Push Title",      "Desc": "Offline Push Content"  }}
```

- Администратор отправляет сообщения пользователям, следящим за "Stock A" или "stock B" и не следящим за "Stock C" или "stock D":

```
{  "From_Account": "administrator",  "MsgRandom": 124032,  "Condition": {      "TagsOr": ["Stock A","stock B"],      "TagsNot": ["Stock C","Stock D"] // Область отправки: ("Stock A" или "stock B") и не ("Stock C" и "Stock D")  },  "OfflinePushInfo": {      "PushFlag": 0, // 0 означает выполнить оффлайн-отправку, 1 означает не выполнять оффлайн-отправку      "Title": "Offline Push Title",      "Desc": "Offline push content"  }}
```

#### **Push по атрибуту пользователя**

- Администратор отправляет сообщения члену ультра-платины в Шэньчжэне:

```
{  "From_Account": "administrator",  "MsgRandom": 389475,  "Condition": {      "AttrsAnd": {          "Membership Level": "Ultra-Platinum Member",          "city": " Shenzhen "      }  },  "OfflinePushInfo": {      "PushFlag": 0, // 0 означает включённую оффлайн-отправку, 1 означает отключённую оффлайн-отправку      "Title": "Offline Push Title",      "Desc": "Offline Push Content"  }}
```

- Администратор отправляет сообщения пользователю ультра-платины в Шэньчжэне:

```
{  "From_Account": "administrator",  "MsgRandom": 389475,  "Condition": {      "AttrsAnd": {          "Membership Level": "Ultra-Platinum User",          "city": " Shenzhen "      }  },  "OfflinePushInfo": {      "PushFlag": 0, // 0 означает включённую оффлайн-отправку, 1 означает отключённую оффлайн-отправку      "Title": "Offline Push Title",      "Desc": "Offline Push Content"  }}
```

### Описание полей пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| From_Account | String | Необязательное | Учётная запись отправителя сообщения (поддерживает указание любой существующей учётной записи)**Если отправитель не указан или указанный отправитель не существует, по умолчанию будет учётная запись вызывающей стороны.** |
| MsgRandom | Integer | Обязательное | 32-битное беззнаковое целое число случайного числа, диапазон значений от 0 до 4294967295. Бэкенд использует дедупликацию сообщений в течение одной секунды. **Пожалуйста, убедитесь, что это случайное число**. |
| Condition | Object | Необязательное | Критерии фильтрации отправки для текущей аудитории. Если Condition не заполнен, отправляет всем пользователям. Подробности см. в разделе [Описание поля Condition](#8eb4836e-16e7-4f62-afbf-8f0736668711). |
| OfflinePushInfo | Object | Необязательное | Конфигурация уведомлений оффлайн-отправки, см. [Описание OfflinePushInfo](https://www.tencentcloud.com/document/product/1047/77715) |
| DataId | String | Необязательное | Пользовательский идентификатор бизнеса клиента. Это поле будет передано в бэкенд приложения при обратном вызове. Поле ограничено максимум 64 байтами. |

#### Описание поля Condition

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| TagsOr | String Array | Необязательное | Объединение условий по тегам, длина массива не превышает 10. |
| TagsAnd | String Array | Необязательное | Пересечение условий по тегам, длина массива не превышает 10. |
| TagsNot | String Array | Необязательное | После взятия объединения нескольких тегов, отрицание условия: исключение объектов, содержащих любой из указанных тегов (NOT (tag = A OR tag = B OR ...)). Длина массива тегов не должна превышать 10. |
| AttrsOr | Object | Необязательное | Объединение условий по атрибутам, длина массива не превышает 10. |
| AttrsAnd | Object | Необязательное | Пересечение условий по атрибутам, длина массива не превышает 10. |

> **Примечание: Рекомендуется не смешивать теги и атрибуты, так как область целевых объектов отправки может стать неясной.** Различные условия по тегам и различные условия по атрибутам можно использовать вместе."Condition": {"TagAnd": ["tag1", "tag2"], "TagOr": ["tag3", "tag4"], "TagNot": ["tag5", "tag6"]}Сначала вычислите результат поля "TagAnd": tag1 и tag2=A. Затем вычислите результат поля "TagOr": tag3 или tag4=B.Затем вычислите результат поля "TagNot": не (tag5 или tag6)=C.Примечание: окончательный результат "Condition" - это A и B и C.

### Примеры пакетов ответов

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0,    "TaskId": "53743040_144115212910570789_4155518400_15723514"}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса:OK: указывает на успешную обработкуFAIL: указывает на сбой |
| ErrorCode | Integer | Код ошибки |
| ErrorInfo | String | Сообщение об ошибке |
| TaskId | String | ID задачи отправки |

## Коды ошибок

Если не происходит сетевая ошибка (например, ошибка 502), код возврата HTTP для этого интерфейса всегда равен 200.**Фактические коды ошибок и сообщения об ошибках указаны в ErrorCode и ErrorInfo в теле ответа.** Распространённые коды ошибок (60000–79999) можно найти в документации по [кодам ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).

Коды ошибок, специфичные для этого API:

| Код ошибки | Описание значения |
| --- | --- |
| 90001 | Не удалось проанализировать строку JSON, проверьте, соответствует ли пакет запроса формату JSON. |
| 90005 | Поле MsgRandom отсутствует в теле запроса в формате JSON или поле MsgRandom не имеет тип Integer. |
| 90009 | Запрос требует доступа администратора приложения. |
| 90020 | Длина тега превышает лимит (не может превышать 50 байт). |
| 90022 | Дублирующийся тег в TagsOr и TagsAnd в условиях отправки. |
| 90024 | Чрезмерная частота отправки, интервал между двумя отправками должен быть больше 1 секунды. |
| 90026 | Ошибка времени оффлайн-хранения сообщения (не должно превышать 7 дней). |
| 90032 | Количество тегов в условиях отправки превышает 10, или количество тегов в запросе на добавление тега превышает 10. |
| 90033 | Недопустимый атрибут. |
| 90039 | Отправка по атрибутам и отправка по тегам не могут сосуществовать. |
| 90040 | Один из тегов в условиях отправки пуст. |
| 90045 | Функция all-staff/Tag Push не активирована. |
| 90047 | Количество отправок превысило дневной лимит (по умолчанию 100 раз). |
| 90056 | Тело запроса для универсальной отправки уведомлений слишком велико. В настоящее время поддерживается максимальная длина 10K. |
| 90057 | Предоставленный DataId превышает максимальную допустимую длину 64 байтов. |
| 91000 | Внутренняя ошибка сервиса. Пожалуйста, попробуйте снова. |

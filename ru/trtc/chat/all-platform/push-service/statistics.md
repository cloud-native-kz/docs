# Статистика

Данная статья введен различные страницы статистики для данных статистики отправок, позволяя пользователям быстро запрашивать статистические данные.

> **Примечание:** Функция «Статистика» будет записывать только детали данных отправок последнего устройства, вошедшего в систему, и не поддерживает сценарии с несколькими устройствами, вошедшими в систему.

## Обычная отправка

### Записи отправок

Вы можете запрашивать все данные записей отправок для пользователей этого приложения. Необходимо указать временное окно, ID отправителя и ID получателя. Запрос данных отправок пользователя включает время отправки, ID отправки, заголовок отправки и содержание отправки; также поддерживает поиск определенных записей отправок по заголовку отправки или содержанию отправки.

#### Условия запроса

Временное окно: определенный период времени в рамках указанной даты, максимум один час, точное до минут и секунд, обязательно.

ID отправителя: обязательно.

ID получателя: обязательно.

Заголовок отправки: опционально.

Содержание отправки: опционально.

#### Результаты запроса (записаны за последние 7 дней)

Время отправки: точное время достижения отправки.

ID отправки: уникальный ID сообщения отправки, который можно использовать для поиска полной ситуации с ссылкой отправки в инструменте устранения неполадок.

Заголовок отправки: отображаемый заголовок отправки.

Содержание отправки: отображаемое содержание отправки.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/223cbbbbbc5a11ef928f525400d5f8ef.png)

### Данные отправки

Статистика отображает различные метрики данных отправок приложения за последние дни, включая обзор вчерашней отправки, воронку преобразования данных отправок для интервалов времени нескольких типов, поддерживает просмотр по размерности поставщика и детали ежедневных метрик данных отправок.

#### Метрические данные обзора вчерашней отправки

Включает количество, которое можно отправить, количество отправленное, количество достигнутое, количество кликов, фактический коэффициент доставки, коэффициент охвата, коэффициент кликабельности.

#### Диапазон времени

Поддерживает вчера, последние 7 дней, последние 30 дней и типы указанного временного окна.

#### Воронка преобразования данных отправки

Включает размеры отправляемого количества, отправленного количества, достигнутого количества и кликнутого количества.

#### Просмотр по размерности поставщика ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/5459b9dabc5a11ef928f525400d5f8ef.png)

Поддерживает просмотр по всем категориям поставщиков.

## Отправка всем/по тегам/указанному UserID

### Записи отправок

Вы можете запрашивать все данные записей отправок, отправленных пользователям приложения, указав временное окно для запроса. Данные отправок пользователя включают время отправки, ID задачи и содержание запроса отправки. Также поддерживает поиск и поиск определенных записей отправок по содержанию отправки.

#### Условия запроса

Временное окно: определенный период времени в рамках указанной даты, до 7 дней, точное до минут и секунд, обязательно.

Содержание отправки: опционально.

#### Результаты запроса (записаны за последние 7 дней)

Время отправки: точное время достижения отправки.

ID задачи: уникальный ID сообщения отправки, который можно использовать для поиска полного статуса ссылки уведомления отправки в инструменте устранения неполадок.

Содержание отправки: детальные данные в формате JSON отправки.

#### Метрические данные обзора вчерашней отправки

Включает размеры отправляемого количества, отправленного количества, достигнутого количества и кликнутого количества.

## Метод расчета метрических данных

**Отправляемое количество:** общее количество действительных устройств, которые могут быть достигнуты путем фильтрации целевой аудитории, выбранной для задачи отправки (когда устройство находится в сети и переключается в фоновый режим, отправки как в сети, так и в автономном режиме будут отправлены одновременно и будут учитываться как два устройства, без дедупликации).

**Отправленное количество:** общее количество действительных устройств, которые были успешно распределены через канал мгновенного обмена сообщениями (чат) и канал производителя из действительных устройств, которые можно отправить (когда устройство находится в сети и переключается в фоновый режим, отправки как в сети, так и в автономном режиме будут отправлены одновременно и будут учитываться как два устройства, без дедупликации).

**Достигнутое количество:** общее количество квитанций, успешно полученных терминалом устройства через канал мгновенного обмена сообщениями (чат) и канал производителя (когда устройство находится в сети и переключается в фоновый режим, отправки как в сети, так и в автономном режиме будут отправлены одновременно и будут учитываться как два устройства, без дедупликации).

**Кликнутое количество:** общее количество квитанций кликов после успешного отображения отправки.

**Фактический коэффициент доставки:** Отправленное количество / Отправляемое количество * 100%.

**Коэффициент охвата:** Достигнутое количество / Отправленное количество * 100%.

**Коэффициент кликабельности:** Кликнутое количество / Достигнутое количество * 100%.

# Инструмент устранения неполадок

Данная статья предназначена для ознакомления с различными страницами и особенностями использования инструмента устранения неполадок с push-уведомлениями, помогая пользователям диагностировать полную цепь push-уведомлений при недоставке автономного push-сообщения.

## Обычная отправка (Regular Push)

Инструмент устранения неполадок обычной отправки предназначен в основном для разработчиков, позволяя устранять неполадки с получением конкретных push-уведомлений. Найдя запись о отправке и скопировав уникальный идентификатор Push ID этой отправки, можно запросить подробную информацию о цепи доставки push-уведомления.

### Поля запроса

Push ID: Уникальный идентификатор для определения обычной отправки, который можно найти в записях отправки. Обязательное поле.

### Результат запроса

Основная информация: Базовая информация о текущем устройстве на момент доставки push-уведомления, включая модель, операционную систему, версию SDK и номер версии плагина и т. д.

Статус устройства: Статус переключателя панели уведомлений push-уведомления и статус привязки токена устройства на момент отправки.

Статус отправки: Сквозная информация о доставке push-уведомления, включая полную цепь от Chat server > Manufacturer Server > Terminal Device > User Click.

## Отправка всем сотрудникам / По меткам (All-staff/Tag Push)

Инструмент устранения неполадок отправки всем сотрудникам / по меткам предназначен в основном для администраторов, позволяя исследовать статус получения конкретного пользователя при отправке всем сотрудникам / по меткам. Найдя запись о отправке и скопировав уникальный идентификатор Task ID этой отправки, а также введя UserID получателя, можно запросить подробную информацию о цепи доставки push-уведомления для этого UserID.

### Поля запроса

Task ID: Уникальный идентификатор для определения отправки всем сотрудникам / по меткам, который можно найти в записях отправки. Обязательное поле.

UserID: Идентификатор пользователя, для которого предназначена отправка всем сотрудникам / по меткам, а также получатель push-уведомления. Обязательное поле.

### Результат запроса

Основная информация: На момент доставки push-уведомления для указанного UserID базовая информация о текущем устройстве включает модель, операционную систему, версию SDK и номер версии плагина и т. д.

Статус устройства: На момент доставки push-уведомления для указанного UserID статус переключателя панели уведомлений устройства, а также статус привязки токена устройства.

Статус отправки: Сквозная информация о push-уведомлении, доставленном указанному UserID, включая полную ситуацию цепи от Chat server > Manufacturer Server > Terminal Device > User Click.

# Отправка всем пользователям/пользователям по тегам

Функция отправки всем пользователям/по тегам поддерживает отправку специального контента, а также позволяет доставлять персонализированный контент конкретным группам пользователей на основе тега, атрибута, таких как действия членства, региональные уведомления и т. д. Это помогает в получении пользователей, преобразовании, активации продвижения и других рабочих фазах, а также поддерживает отчеты о доставке пушей и самостоятельный инструмент устранения неполадок при доставке. Для получения более подробной информации см. [Отображение эффекта](https://www.tencentcloud.com/document/product/1047/60537).

> **Примечание:** Учетная запись должна войти или быть вручную импортирована, чтобы получать сообщения от функции отправки для всех сотрудников/по тегам. Универсальное push-уведомление не поддерживает изменение сообщений.

## Обзор функций

- Поддерживает отправку пушей всем пользователям.
- Поддерживает отправку пушей на основе атрибутов пользователя.
- Поддерживает отправку пушей по тегу пользователя.
- Когда администратор отправляет сообщение, получатель видит отправителя сообщения как администратора.
- Когда администратор указывает учетную запись для отправки сообщений другим учетным записям, получатель видит отправителя не как администратора, а как учетную запись, указанную администратором.
- Поддерживает роуминг, продолжительность хранения при роуминге соответствует продолжительности хранения обычных сообщений.
- Из-за большого количества учетных записей, которые необходимо выпустить для функции отправки всем пользователям/по тегам, выпуск всех учетных записей занимает некоторое время (в зависимости от общего количества учетных записей).
- Установив флаг OnlineOnlyFlag на 1, можно отправлять push-уведомления без сохранения сеанса, роуминга и непрочитанных сообщений.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/timpush/push?usersig=xxx&identifier=admin&sdkappid=88888888&random=99999999&contenttype=json
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса: HTTPSМетод запроса: POST |
| xxxxxx | Выделенный домен, соответствующий стране/региону вашего SDKAppID.Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Сеул: `adminapikr.im.qcloud.com`Токио: `adminapijpn.im.qcloud.com`Франкфурт: `adminapiger.im.qcloud.com`Кремниевая долина: `adminapiusa.im.qcloud.com`Джакарта: `adminapiidn.im.qcloud.com` |
| v4/timpush/push | API запроса. |
| usersig | Подпись, созданная учетной записью администратора приложения. Для получения подробной информации см. [Создание UserSig](https://www.tencentcloud.com/document/product/1047/34385). |
| identifier | Вы должны быть учетной записью администратора приложения. Для получения более подробной информации см. [Администраторы приложения](https://www.tencentcloud.com/document/product/1047/33517#app-login-process). |

| contenttype | Фиксированное значение: `json`. |

### Ограничение частоты вызовов

Этот API включает отправку всем пользователям, по атрибутам и по тегам. По умолчанию он может вызываться до 100 раз в день с интервалом более 1 секунды между каждой отправкой.

> **Примечание:** Ограничение частоты API. По умолчанию вы можете сделать 100 бесплатных вызовов в день. Каждый дополнительный вызов будет стоить дополнительно $10 в день. Если вам нужно отрегулировать частоту API, перейдите в [консоль Chat](https://console.trtc.io/chat/push-plugin-push-identifier).

### Пример пакетов запроса

#### **Пример массовой отправки**

- Администратор проводит массовую отправку:

```
{  "From_Account": "admin",  "MsgRandom": 3674128,  "OnlineOnlyFlag": 0, // 0 означает сохранение роуминга и непрочитанных сообщений с ограничением параллелизма отправки 200 пользователей/сек; 1 означает отсутствие сохранения роуминга и непрочитанных сообщений  "MsgBody": [      {          "MsgType": "TIMTextElem",          "MsgContent": {              "Text": "hi, beauty"          }      }  ],  "OfflinePushInfo": {      "PushFlag": 0,  // 0 означает включение автономной отправки, 1 означает отключение автономной отправки      "Title": "Offline Push Title",      "Desc": "Offline Push Content"  }}
```

- Администратор направляет конкретную учетную запись для проведения массовой отправки (в примере учетная запись отправителя — xiaoming):

```
{  "From_Account": "xiaoming",  "MsgRandom": 3674128,  "OnlineOnlyFlag": 0, // 0 означает сохранение роуминга и непрочитанных сообщений с ограничением параллелизма отправки 200 пользователей/сек; 1 означает отсутствие сохранения роуминга и непрочитанных сообщений  "MsgBody": [      {          "MsgType": "TIMTextElem",          "MsgContent": {              "Text": "hi, beauty"          }      }  ],  "OfflinePushInfo": {      "PushFlag": 0,  // 0 означает включение автономной отправки, 1 означает отключение автономной отправки      "Title": "Offline Push Title",      "Desc": "Offline Push Content"  }}
```

> **Примечание:** From_Account — это учетная запись отправителя сообщения, которая поддерживает указание любой существующей учетной записи. Если отправитель не указан или не существует, по умолчанию будет использована учетная запись вызывающей стороны.

- Отправка только онлайн-пользователям (онлайн-отправка):

```
{  "From_Account": "xiaoming",  "MsgRandom": 3674128,  "OnlineOnlyFlag": 1, // 0 означает сохранение роуминга и непрочитанных сообщений с ограничением параллелизма отправки 200 пользователей/сек; 1 означает отсутствие сохранения роуминга и непрочитанных сообщений  "MsgBody": [      {          "MsgType": "TIMTextElem",          "MsgContent": {              "Text": "hi, beauty"          }      }  ],  "OfflinePushInfo": {      "PushFlag": 1  // 0 означает включение автономной отправки, 1 означает отключение автономной отправки  }}
```

> **Примечание:** Установка OnlineOnlyFlag на 1 указывает на операции отправки как для онлайн, так и для оффлайн пользователей, но установка OfflinePushInfo.PushFlag на 1 означает отсутствие автономной отправки. Таким образом, приведенный выше пример означает, что отправка будет выполняться только онлайн-пользователям.

#### **Пример отправки по тегу пользователя**

- Администратор отправляет сообщения пользователям с обоими тегами «Stock A» и «Stock B»:

```
{  "From_Account": "admin",  "MsgRandom": 124032,  "OnlineOnlyFlag": 0, // 0 означает сохранение роуминга и непрочитанных сообщений с ограничением параллелизма отправки 200 пользователей/сек; 1 означает отсутствие сохранения роуминга и непрочитанных сообщений  "Condition": {      "TagsAnd": ["Stock A","Stock B"]  },  "MsgBody": [      {          "MsgType": "TIMTextElem",          "MsgContent": {              "Text": "hi, beauty"          }      }  ],  "OfflinePushInfo": {      "PushFlag": 0, // 0 означает включение автономной отправки, 1 означает отключение автономной отправки      "Title": "Offline Push Title",      "Desc": "Offline Push Content"  }}
```

> **Примечание:** From_Account — это учетная запись отправителя сообщения, которая поддерживает указание любой существующей учетной записи. Если отправитель не указан или не существует, по умолчанию будет использована учетная запись вызывающей стороны.

- Администратор отправляет сообщения пользователям, следящим за «Stock A» или «Stock B»:

```
{  "From_Account": "admin",  "MsgRandom": 124032,  "OnlineOnlyFlag": 0, // 0 означает сохранение роуминга и непрочитанных сообщений с ограничением параллелизма отправки 200 пользователей/сек; 1 означает отсутствие сохранения роуминга и непрочитанных сообщений  "Condition": {      "TagsOr": ["Stock A","Stock B"]  },  "MsgBody": [      {          "MsgType": "TIMTextElem",          "MsgContent": {              "Text": "hi, beauty"          }      }  ],  "OfflinePushInfo": {      "PushFlag": 0, // 0 означает включение автономной отправки, 1 означает отключение автономной отправки      "Title": "Offline Push Title",      "Desc": "Offline Push Content"  }}
```

- Администратор отправляет сообщения пользователям, следящим за «Stock A» или «Stock B» и не следящим за «Stock C» или «Stock D»:

```
{  "From_Account": "admin",  "MsgRandom": 124032,  "OnlineOnlyFlag": 0, // 0 означает сохранение роуминга и непрочитанных сообщений с ограничением параллелизма отправки 200 пользователей/сек; 1 означает отсутствие сохранения роуминга и непрочитанных сообщений  "Condition": {      "TagsOr": ["Stock A","Stock B"],      "TagsNot": ["Stock C","Stock D"] // Область отправки: ("Stock A" или "Stock B") и не ("Stock C" и "Stock D")  },  "MsgBody": [      {          "MsgType": "TIMTextElem",          "MsgContent": {              "Text": "hi, beauty"          }      }  ],  "OfflinePushInfo": {      "PushFlag": 0, // 0 означает включение автономной отправки, 1 означает отключение автономной отправки      "Title": "Offline Push Title",      "Desc": "Offline Push Content"  }}
```

### Описание полей пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| From_Account | String | Опционально | Учетная запись отправителя сообщения (поддерживает указание любой существующей учетной записи)**Если отправитель не указан или указанный отправитель не существует, по умолчанию будет использована учетная запись вызывающей стороны.** |
| MsgRandom | Integer | Обязательно | 32-битное целое число без знака, диапазон значений от 0 до 4294967295. Бэкенд использует дедупликацию сообщений в течение одной секунды. **Пожалуйста, убедитесь, что это случайное число**. |
| OnlineOnlyFlag | Integer | Опционально | По умолчанию 0, означает сохранение роуминга и непрочитанных сообщений с ограничением параллелизма отправки 200 человек/сек.1 означает отсутствие сохранения истории сообщений и отсутствие подсчета непрочитанных сообщений**Примечание:** Если вам нужно отрегулировать ограничение параллелизма отправки, свяжитесь с [службой поддержки Tencent Cloud](https://intl.cloud.tencent.com/contact-us) или с сотрудниками по бизнесу. |
| Condition | Object | Опционально | Критерии фильтра отправки для текущей аудитории. Если Condition не заполнен, отправляется всем пользователям. Для получения подробной информации см. [Описание поля Condition](#8eb4836e-16e7-4f62-afbf-8f0736668711). |
| MsgBody | Array | Обязательно | Содержание сообщения, см. [Описание содержания сообщения MsgBody](https://www.tencentcloud.com/document/product/1047/33527) (сообщение может содержать несколько элементов сообщения, поэтому MsgBody имеет тип Array) |
| MsgType | String | Обязательно | Типы объектов сообщений TIM, которые в настоящее время поддерживаются:TIMTextElem (Текстовое сообщение).TIMLocationElem (Сообщение о местоположении).TIMFaceElem (Сообщение с выражением).TIMCustomElem (Пользовательское сообщение).TIMSoundElem (ВМС).TIMImageElem (Сообщение с изображением).TIMFileElem (Сообщение с файлом).TIMVideoFileElem (Видеосообщение). |
| MsgContent | Object | Обязательно | Для каждого MsgType используются разные форматы MsgContent, см. [Определение объекта TIMMsgElement](https://www.tencentcloud.com/document/product/1047/33527#timmsgelement). |
| OfflinePushInfo | Object | Опционально | Конфигурация уведомления об автономной отправке. Для получения подробной информации см. [Описание OfflinePushInfo](https://www.tencentcloud.com/document/product/1047/77715). |

#### Описание поля Condition

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| TagsOr | String Array | Опционально | Объединение условий по тегам, длина массива не превышает 10. |
| TagsAnd | String Array | Опционально | Пересечение условий по тегам, длина массива не превышает 10. |
| TagsNot | String Array | Опционально | Отрицание условий по тегам (сначала берется объединение нескольких тегов, затем дополнение результата), длина массива не превышает 10. |
| AttrsOr | Object | Опционально | Объединение условий по атрибутам, длина массива не превышает 10. |
| AttrsAnd | Object | Опционально | Пересечение условий по атрибутам, длина массива не превышает 10. |

> **Примечание: рекомендуется не смешивать теги и атрибуты, так как область целевых объектов отправки может стать неясной.** Различные условия по тегам и различные условия по атрибутам могут использоваться вместе."Condition": {"TagAnd": ["tag1", "tag2"], "TagOr": ["tag3", "tag4"], "TagNot": ["tag5", "tag6"]}Сначала вычислите результат поля "TagAnd": tag1 и tag2=A. Затем вычислите результат поля "TagOr": tag3 или tag4=B.Затем вычислите результат поля "TagNot": не (tag5 или tag6)=C.Примечание: окончательный результат "Condition" — это A и B и C.

## Коды ошибок

Если не происходит ошибка сети (например, ошибка 502), код состояния HTTP для этого интерфейса всегда будет 200. **Реальные коды ошибок и сообщения указаны в теле ответа параметрами ErrorCode и ErrorInfo**. Для общих кодов ошибок (60000–79999) см. документ [Код ошибки](https://www.tencentcloud.com/document/product/1047/34348).

Частные коды ошибок для этого API следующие:

| Код ошибки | Описание |
| --- | --- |
| 90001 | Ошибка при разборе формата JSON. Пожалуйста, убедитесь, что пакет запроса соответствует спецификации JSON. |
| 90002 | MsgBody в пакете запроса в формате JSON не соответствует описанию формата сообщения, или MsgBody не является типом Array. Пожалуйста, обратитесь к определению [объекта TIMMsgElement](https://www.tencentcloud.com/document/product/1047/33527#timmsgelement). |
| 90005 | В теле пакета запроса в формате JSON отсутствует поле MsgRandom, или поле MsgRandom не является типом Integer. |
| 90007 | Тип MsgBody в теле пакета запроса в формате JSON не является типом Array. Пожалуйста, измените на тип Array. |
| 90009 | Запрос требует разрешений администратора приложения. |
| 90010 | Пакет запроса в формате JSON не соответствует описанию формата сообщения. Пожалуйста, обратитесь к определению [объекта TIMMsgElement](https://www.tencentcloud.com/document/product/1047/33527#timmsgelement). |
| 90020 | Длина тега превышает лимит (не может превышать 50 байт). |

| 90024 | Отправки слишком частые; интервал между двумя отправками должен быть более 1 секунды. |

| 90032 | Количество тегов в условиях отправки превышает 10, или количество тегов в запросе на добавление тега превышает 10. |
| 90033 | Недопустимый атрибут. |
| 90039 | Отправка по атрибутам и отправка по тегам не могут сосуществовать. |

| 90045 | Функция отправки для всех сотрудников/по тегам не активирована. |
| 90047 | Попытки отправки превышают дневной лимит (по умолчанию 100 раз). |

| 91000 | Внутренняя ошибка сервиса, пожалуйста, попробуйте позже. |

# Отзыв push-уведомлений

Если содержимое, отправленное всему персоналу/тегам, неправильно, это может отрицательно сказаться на продукте при просмотре или клике конечным пользователем. Своевременная обработка необходима. Вы можете выбрать отзыв push-уведомления.

## Обзор функций

- Прекращение: выдача push-задач требует времени, и учетные записи, которые еще не были выданы, будут прекращены.
- Отзыв: для учетных записей, которые уже были выданы, поддерживается отзыв непрочитанных/бродячих сообщений.
- Перезапись: если выданные учетные записи получили офлайн push, поддерживается перезапись push.
- Этот интерфейс поддерживает прекращение/отзыв/перезапись push-задач для всех пользователей/тегов. В следующем тексте прекращение/отзыв/перезапись по умолчанию называется отзывом.
- Срок действия отзыва составляет 24 часа с момента инициирования задачи. Push-задачи старше 24 часов не могут быть отозваны.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/timpush/revoke?usersig=xxx&identifier=admin&sdkappid=88888888&random=99999999&contenttype=json
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса: HTTPSМетод запроса: POST |
| xxxxxx | Выделенный домен, соответствующий стране/региону вашего SDKAppID.Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Сеул: `adminapikr.im.qcloud.com`Токио: `adminapijpn.im.qcloud.com`Франкфурт: `adminapiger.im.qcloud.com`Кремниевая долина: `adminapiusa.im.qcloud.com`Джакарта: `adminapiidn.im.qcloud.com` |
| v4/timpush/revoke | Запрос API. |
| usersig | Подпись, сгенерированная учетной записью администратора приложения. Подробности см. в разделе [Генерирование UserSig](https://www.tencentcloud.com/document/product/1047/34385). |
| identifier | Должна быть учетная запись администратора приложения. Дополнительные сведения см. в разделе [Администраторы приложений](https://www.tencentcloud.com/document/product/1047/33517#app-admin). |
| sdkappid | SdkAppid, назначенный консолью Chat при создании приложения. |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Фиксированное значение: `json` |

### Частота вызовов

Вызов этого интерфейса ограничен 1 разом в секунду.

### Пример пакета запроса

```
{    "TaskId": "660cc447_537ed82a_200000cd7ee17f5_84035729_bc614e",  // Push taskId отправленный в течение 24 часов     "OfflinePushInfo": { // Если при отправке не указаны roaming/unread (OnlineOnlyFlag=0), OfflinePushInfo должен быть включен при отзыве        "Title": "Recall Title",        "Desc": "The other party has recalled a message",        "Ext": "{\\"entity\\":{\\"key1\\":\\"value1\\",\\"key2\\":\\"value2\\"}}" // Поле сквозной передачи, строка в формате JSON для push    }}
```

> **Примечание:** Поддерживаемые поставщики для перезаписи автономной отправки: APNS/Google FCM/Huawei/Honor. Перезапись автономной отправки не поддерживается для других поставщиков. (Режим уведомлений Google FCM поддерживает перезапись, режим данных еще не поддерживает перезапись)При отзыве, если получатель находится на переднем плане и автоматическое автономное push-уведомление (сообщение в панели уведомлений) прочитано, это автономное push-уведомление не будет перезаписано.

### Описание полей пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| TaskId | String | Обязательное | Идентификатор задачи Push для всех сотрудников/тегов. |
| OfflinePushInfo | Object | Необязательное | Конфигурация уведомлений автономной отправки, см. [Описание OfflinePushInfo](https://www.tencentcloud.com/document/product/1047/77715#offlinepushinfo).**Примечание:** Если установлено OfflinePushinfo.PushFlag=1 или OfflinePushInfo не установлен, автономная отправка не будет перезаписана. |

### Пример пакета ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса:OK: указывает на успешную обработку.FAIL: указывает на сбой. |
| ErrorCode | Integer | Идентификатор ошибки:0 указывает на успех.Ненулевое значение указывает на сбой. |
| ErrorInfo | String | Сообщение об ошибке. |

## Коды ошибок

Если не происходит ошибка сети (например, ошибка 502), код возврата HTTP для этого интерфейса всегда равен 200. Фактический код ошибки и информация об ошибке представлены через ErrorCode и ErrorInfo в полезной нагрузке ответа. Для общих кодов ошибок (60000–79999) см. документ [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).

| Код ошибки | Описание |
| --- | --- |
| 90001 | Не удалось разобрать формат JSON. Убедитесь, что пакет запроса соответствует спецификации JSON. |
| 90009 | Запрос требует разрешений администратора приложения. |
| 90049 | Недействительный TaskId для отзыва, нет записей отправки. Используйте интерфейс [Push для всех пользователей/тегов](https://www.tencentcloud.com/document/product/1047/67814), и только возвращаемый TaskId может быть использован для отзыва. |
| 90050 | Дублирование отзыва, push-задача, которая уже была отозвана, не может быть отозвана снова. |
| 90051 | Отзыв слишком частый, предел частоты отзыва составляет 1 раз в секунду. |
| 90052 | Превышен срок действия отзыва, отзыв должен быть выполнен в течение 24 часов. Push-задачи старше 24 часов не могут быть отозваны. |
| 90053 | Отзыв недействителен. Push-задача не указала roaming/unread (OnlineOnlyFlag=0), но OfflinePushInfo не был включен при отзыве. |
| 91000 | Ошибка внутреннего сервиса, пожалуйста, повторите попытку. |

# Отправка потокового сообщения с ответом

## Обзор функции

- Отправляйте потоковые сообщения с ответом роботу через этот API.
- Для полного текста может потребоваться вызвать этот API несколько раз для последовательного вывода частей клиенту.
- При первом вызове этого API бэкенд генерирует MsgKey для представления сообщения в этот раз. Последующие вызовы требуют передачи MsgKey в качестве параметра на сервер IM.
- Если приложение использует TUIKit версии 7.7.5282 или выше, оно может автоматически отображать эффект потокового вывода сообщения.
- Если приложение не использует TUIKit, оно должно вручную отображать эффект потокового вывода в соответствии с протоколом [потокового текстового вывода](#.E8.AF.B7.E6.B1.82.E5.8C.85.E7.A4.BA.E4.BE.8B).

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/openim_robot_http_svc/send_stream_msg?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице перечислены только параметры, задействованные при изменении при вызове этого API, и их описания. Дополнительные сведения о параметрах см. в [введении REST API](https://trtc.io/document/34620?product=chat&menulabel=serverapi).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Специализированное доменное имя для страны/региона, в котором находится SDKAppID:Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Сеул: `adminapikr.im.qcloud.com`Токио: `adminapijpn.im.qcloud.com`Франкфурт: `adminapiger.im.qcloud.com`Силиконовая долина: `adminapiusa.im.qcloud.com`Джакарта: `adminapiidn.im.qcloud.com` |
| v4/openim_robot_http_svc/send_stream_msg | API запроса. |
| sdkappid | SDKAppID, назначенный консолью IM при создании приложения. |
| identifier | Должна быть учетная запись администратора приложения. Дополнительные сведения см. в [администраторе приложения](https://www.tencentcloud.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробные сведения см. в [создание UserSig](https://trtc.io/document/34385). |
| random | Введите случайное 32-битное целое число без знака в диапазоне [0, 4294967295]. |
| contenttype | Формат запроса зафиксирован как `json`. |

### Максимальная частота вызовов

200 раз/сек.

### Пример запроса

Первая отправка потокового текста.

```
{    "From_Account": "@RBT#my_bot",    "To_Account": "user01",    "Chunk": "hello, "}
```

Последующие вызовы API продолжают вывод текста.

```
{    "From_Account": "@RBT#my_bot",    "To_Account": "user01",    "Chunk": "world",    "MsgKey": "606781340_3986845397_1706000918",    "Finish": 1}
```

### Поля запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| From_Account | String | Обязательно | UserID отправителя (используется для указания учетной записи отправителя сообщения). |
| To_Account | String | Обязательно | UserID получателя сообщения. |
| Chunk | String | Обязательно | Сегмент текста в потоковом текстовом сообщении. |
| MsgKey | String | Необязательно. | Оставьте пусто при первом вызове API. Начиная со второго вызова, это поле должно быть включено. |
| Finish | Integer | Необязательно. | При последнем вызове отметьте потоковый вывод как завершенный. |

### Пример ответа

- обычный ответ

```
{    "ActionStatus":"OK",    "ErrorInfo":"",    "ErrorCode":0,    "MsgKey":"606781340_3986845397_1706000918"}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса:OK: обработка выполнена успешно.FAIL: обработка не выполнена. |
| ErrorCode | Integer | Код ошибки:0: успех.Ненулевое значение: указывает на сбой. |
| ErrorInfo | String | Сообщение об ошибке. |
| MsgKey | String | Уникальный идентификатор сообщения, возвращаемый бэкенд IM, используется для последующего вывода текста. |

## Коды ошибок

Если не происходит сетевая ошибка (такая как 502), HTTP код возврата этого API равен 200. Фактический код ошибки и информация об ошибке находятся в полях ErrorCode и ErrorInfo полезной нагрузки ответа.

Общие коды ошибок (60000-79999) см. в документе [Код ошибки](https://trtc.io/document/34348).

Приватные коды ошибок этого API следующие:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя системная ошибка, повторите попытку. |
| 10004 | Некорректный параметр. Пожалуйста, проверьте параметры запроса. |

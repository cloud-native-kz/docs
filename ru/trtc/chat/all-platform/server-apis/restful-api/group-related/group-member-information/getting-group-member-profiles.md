# Получение профилей членов группы

## Обзор

Этот API используется администратором приложения для получения профилей членов группы на основе ID группы.

## Описание вызова API

### Применимые типы групп

| ID типа группы | Поддержка RESTful API |
| --- | --- |
| Private | Да. Аналогично рабочей группе (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. Аналогично группе встреч (Meeting) в новой версии. |
| AVChatRoom | Нет |
| Community | Да. Получение профилей членов группы с использованием поля `Next`. |

Это предустановленные типы групп в Chat. Дополнительную информацию см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

> **Примечание** Из-за большого количества членов в группах Community метод постраничного получения членов был обновлен для использования пакетного подхода `Next`. Для запроса статуса онлайн членов группы в группах Private, Public, ChatRoom и Community необходимо использовать [издание Pro, Pro Plus или Enterprise](https://trtc.io/document/67650), а затем перейти в консоль, нажать **Configuration** > **Group Feature Configuration** и включить **Group Member Online Status**.

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/get_group_member_info?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Базовый формат**
Базовый запрос используется для получения подробной информации о членах группы, включая профили членов группы и пользовательские поля членов группы. Запрос требует только ID группы.

```
{  "GroupId":"@TGS#1NVTZEAE4"  // ID группы (обязательно)}
```

- **Постраничное отображение**
Вы можете использовать поля `Limit` и `Offset` для управления режимом постраничного отображения:
- `Limit`: указывает максимальное количество членов в массиве `MemberList` в ответе. Максимальное значение: 200; рекомендуемое значение: 100.
- `Offset`: указывает, с какого члена группы начинать получение информации. Если нумерация страниц начинается с `1`, значение `Offset` для каждой страницы должно быть `(номер страницы - 1) × количество членов группы для отображения на каждой странице`.
 Например, для отображения 20 членов группы на каждой странице параметры запроса для первой страницы должны быть `{"Limit": 20, "Offset": 0}`, параметры запроса для второй страницы должны быть `{"Limit": 20, "Offset": 20}` и так далее.

```
{    "GroupId":"@TGS#1NVTZEAE4", // ID группы (обязательно)    "Limit": 100, // Максимальное количество членов для получения информации    "Offset": 0 // Номер последовательности члена, с которого начинать получение информации}
```

> **Примечание** Сообщество в настоящее время не поддерживает постраничное получение сведений о членах группы.

- **Пакетный режим**
Вы можете использовать поля `Limit` и `Next` для управления пакетным режимом:
- `Limit`: указывает максимальное количество членов в массиве `MemberList` в ответе. Максимальное значение: 100
- `Next`: указывает позицию члена, с которой необходимо получить последующую информацию. Для первого запроса параметр запроса клиента `Next` должен передать "". Для последнего запроса сервер возвращает "" для параметра `Next`, указывая, что получение информации завершено.
 Для промежуточных запросов параметр запроса клиента `Next` использует последнее значение `Next`, возвращенное сервером. Это аналогично запросам курсора сканирования Redis.

Например, если вы выполняете пакетное получение, параметры в первом запросе должны быть `{"Limit" : 20, "Next" : ""}`, и сервер возвращает следующее:

```
{   "ActionStatus": "OK",   "ErrorInfo": "",   "ErrorCode": 0,   "Next": "144115265295492787",   "MemberList": [           ....
```

Параметры во втором запросе должны быть `{"Limit" : 20, "Next" : "144115265295492787"}`.
 И так далее, пока сервер не ответит с `Next` равным "", что указывает на отсутствие последующей информации о членах, и клиент должен завершить запрос.

```
{    "GroupId":"@TGS#_@TGS#cAVQXXXXXX", // ID группы (обязательно)    "Limit": 100, // Максимальное количество членов для получения информации    "Next": "" // Начать получение с позиции, где закончилось последнее получение}
```

> **Примечание** Только сообщество поддерживает получение сведений о членах группы в пакетном режиме.

- **Указание информации для получения**
Вы можете использовать поле фильтра `MemberInfoFilter` для указания полей для получения. Поля, не указанные в нем, не будут получены.

```
{  "GroupId":"@TGS#1NVTZEAE4", // ID группы (обязательно)  "MemberInfoFilter": [ // Информация для получения, где `Member_Account` включен по умолчанию. Если это поле не указано, будет получена вся информация о членах группы.      "Role",      "JoinTime",      "MsgSeq",      "MsgFlag",      "LastSendMsgTime",      "MuteUntil",      "NameCard"  ]}
```

- **Получение информации о членах в указанной роли**
Вы можете использовать поле фильтра `MemberRoleFilter` для указания роли членов для получения информации. Если это поле не указано, будет получена информация о членах всех ролей.

```
{  "GroupId":"@TGS#37AB3PAEC", // ID группы (обязательно)  "MemberRoleFilter":[ // Фильтр роли членов      "Owner",      "Member"  ]}
```

- **Получение пользовательских полей членов группы**
Вы можете использовать поле фильтра `AppDefinedDataFilter_GroupMember` для указания пользовательских полей членов группы для получения. Поля, не указанные в нем, не будут получены.

```
{  "GroupId":"@TGS#37AB3PAEC", // ID группы (обязательно)  "AppDefinedDataFilter_GroupMember": [ // Фильтр для пользовательских полей членов группы      "MemberDefined2" // Ключ пользовательского поля члена группы  ]}
```

- **Ответ на комплексный запрос**

```
{  "GroupId":"@TGS#1NVTZEAE4", // ID группы (обязательно)  "MemberInfoFilter": [ // Информация для получения. Если это поле не указано, будет получена вся информация о членах группы.      "Role",      "JoinTime",      "MsgSeq",      "MsgFlag",      "LastSendMsgTime",      "MuteUntil",      "NameCard"  ], "MemberRoleFilter":[ // Фильтр роли членов      "Owner",      "Member"  ], "AppDefinedDataFilter_GroupMember": [ // Фильтр для пользовательских полей членов группы      "MemberDefined2", // Ключ пользовательского поля члена группы      "MemberDefined1"  ],  "Limit": 100, // Максимальное количество членов для получения информации  "Offset": 0 // Номер последовательности члена, с которого начинать получение информации}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы для получения информации о членах |
| MemberInfoFilter | Array | Нет | Информация для получения. Если это поле не указано, будет получена вся информация о членах группы. Подробную информацию о полях информации о членах группы см. в разделе [Профиль члена группы](https://www.tencentcloud.com/document/product/1047/33529#Group-Member-Profile). |
| MemberRoleFilter | Array | Нет | Роль членов группы для получения информации. Если это поле не указано, будет получена информация о членах всех ролей. Роль члена может быть `Owner`, `Admin` или `Member`. |
| AppDefinedDataFilter_GroupMember | Array | Нет | Это поле по умолчанию опускается. Оно указывает пользовательские поля членов группы для получения. Дополнительную информацию см. в разделе **Custom Fields** в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529#.E8.87.AA.E5.AE.9A.E4.B9.89.E5.AD.97.E6.AE.B5). |
| Limit | Integer | Нет | Максимальное количество членов для получения информации одновременно. Значение не может превышать 6000. Если это поле не указано, будет получена информация о всех членах группы. |
| Offset | Integer | Нет | Номер последовательности члена, с которого начинать получение информации. Если это поле установлено в `0`, информация получается начиная с первого члена. |
| Next | String | Нет | Позиция последнего полученного члена. Это поле требуется для группы сообщества. Группа сообщества не поддерживает поле `Offset`. Вместо этого она использует поле `Next`. Для первого вызова для `Next` необходимо передать "". Для последующих вызовов необходимо передать последнее значение `Next`. |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда равен 200, если только не возникает сетевая ошибка (например, ошибка 502). Конкретные код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для публичных кодов ошибок (60000–79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10003 | Неверное слово команды. |
| 10004 | Неверный параметр. Проверьте описание ошибки и устраните проблему. |
| 10007 | Нет разрешения на операцию. Проверьте, является ли оператор администратором приложения или имеет ли оператор разрешение на чтение полей в запросе. |
| 10010 | Группа не существует или была удалена. |
| 10015 | Неверный ID группы. Используйте правильный ID группы. |
| 10018 | Ответ превышает максимальный допустимый размер (1 МБ), поскольку объем данных членов группы слишком велик. Попробуйте использовать `Limit` и `Offset` для получения данных членов группы по страницам. |

# Изменение профиля члена группы

## Обзор функции

Этот API используется администратором приложения для изменения профиля члена группы.

## Описание вызова API

### Применимые типы групп

| Тип группы ID | Поддержка RESTful API |
| --- | --- |
| Private | Да. Аналогично рабочим группам (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. Аналогично группе встреч (Meeting) в новой версии. |
| AVChatRoom | Нет |
| Community | Да |

Это предустановленные типы групп в Chat. Дополнительную информацию см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

> **Примечание** Аудиовидео группы (AVChatRoom) не хранят профили членов группы и поэтому не допускают изменение профиля члена.

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/modify_group_member_info?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. раздел [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Силиконовая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/group_open_http_svc/modify_group_member_info | API запроса. |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Дополнительную информацию см. в разделе **App Admin** (Администратор приложения) в документе [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, сгенерированная учетной записью администратора приложения. Дополнительные сведения см. в разделе [Генерация UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака, колеблющееся от 0 до 4294967295. |
| contenttype | Формат запроса, который всегда должен быть `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Установка администратора**
Установка указанного члена группы в качестве администратора.

```
{  "GroupId": "@TGS#2CLUZEAEJ",  // Целевая группа (обязательно)  "Member_Account": "bob",  // Целевой член группы (обязательно)  "Role": "Admin" // Установить как администратор}
```

- **Отмена роли администратора**
Отозвать роль администратора у указанного члена группы.

```
{  "GroupId": "@TGS#2CLUZEAEJ",  // Целевая группа (обязательно)  "Member_Account": "bob",  // Целевой член группы (обязательно)  "Role": "Member" // Отменить роль администратора}
```

- **Установка типа блокировки сообщений члена**
Установка типа блокировки сообщений для указанного члена. Возможные значения следующие: AcceptAndNotify: принимать и уведомлять. Discard: ни принимать, ни уведомлять. AcceptNotNotify: принимать, но не уведомлять.

```
{  "GroupId": "@TGS#2CLUZEAEJ",  // Целевая группа (обязательно)  "Member_Account": "bob",  // Целевой член группы (обязательно)  "MsgFlag": "AcceptAndNotify" // Тип блокировки сообщений, может быть AcceptAndNotify, Discard или AcceptNotNotify}
```

- **Установка карточки имени члена в группе**
Установка карточки имени в группе для указанного пользователя. Максимальная длина составляет 50 байт.

```
{  "GroupId": "@TGS#2CLUZEAEJ",  // Целевая группа (обязательно)  "Member_Account": "bob",  // Целевой член группы (обязательно)  "NameCard": "bob" // Карточка имени в группе (опционально)}
```

- **Установка пользовательских полей члена группы**
Установка пользовательских полей члена группы. По умолчанию AppMemberDefinedData недоступна и требует включения в [консоли Chat](https://console.trtc.io/chat) перед использованием. Дополнительные сведения см. в таблице описания полей запроса.

```
{  "GroupId": "@TGS#2CLUZEAEJ",  // Целевая группа (обязательно)  "Member_Account": "bob",  // Целевой член группы (обязательно)  "AppMemberDefinedData": [ // Пользовательское поле целевого члена (опционально)      {          "Key":"MemberDefined1", // Ключ пользовательского поля целевого члена группы          "Value":"ModifyData1" // Значение ключа      },      {          "Key":"MemberDefined3",          "Value":"ModifyData3"      }  ]}
```

- **Установка периода молчания члена группы**
Установка периода молчания для указанного члена группы.

```
{  "GroupId": "@TGS#2CLUZEAEJ",  // Целевая группа (обязательно)  "Member_Account": "bob",  // Целевой член группы (обязательно)  "MuteTime":86400 // Период молчания для указанного пользователя в секундах}
```

> **Примечание** Закрытые группы (аналогично группам Work в новой версии) не поддерживают молчание пользователей в группе.

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы, профиль члена которой будет изменен |
| Member_Account | String | Обязательно | Целевой член группы. |
| Role | String | Опционально | Роль члена. `Admin`: установить в качестве администратора. `Member`: отменить роль администратора. (Личность владельца группы не может быть изменена.) |
| MsgFlag | String | Опционально | Тип блокировки сообщений. Доступны следующие варианты: AcceptAndNotify: принимать и уведомлять. AcceptNotNotify означает принимать без уведомления (не срабатывает автономная отправка). Discard: блокировать сообщения группы (сообщения не отправляются на клиенты) AcceptNotNotifyExceptAt означает принимать и уведомлять для сообщений '@' (только сообщения '@' срабатывают автономной отправкой, другие сообщения не срабатывают) |
| NameCard | String | Опционально | Карточка имени в группе (максимальная длина 50 байт). |
| AppMemberDefinedData | Array | Опционально | Пользовательское поле члена группы. По умолчанию это поле недоступно и требует включения в [консоли Chat](https://console.trtc.io/chat). Дополнительные сведения см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529). |
| MuteTime | Integer | Опционально | Период молчания в секундах. `0`: Отменить молчание |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно. `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки. `0`: Успешно. Другие значения: Ошибка |
| ErrorInfo | String | Информация об ошибке |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда равен 200, если только не произойдет ошибка сети (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для открытых кодов ошибок (60000–79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Ошибка внутреннего сервера. Попробуйте снова. |
| 10003 | Неверное слово команды. |
| 10004 | Неверный параметр. Проверьте описание ошибки и устраните проблему. |
| 10007 | Нет разрешения на операцию. Например, обычный член в открытой группе пытается удалить других пользователей из группы, но это может сделать только администратор приложения. |
| 10010 | Группа не существует или была удалена. |
| 10015 | Неверный ID группы. Используйте правильный ID группы. |

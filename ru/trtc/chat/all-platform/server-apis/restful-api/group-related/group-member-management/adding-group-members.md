# Добавление участников группы

## Обзор функции

Этот API позволяет администратору приложения добавлять новых участников в группу.

## Описание вызова API

### Поддерживаемые типы групп

| Тип группы | Поддержка этого RESTful API |
| --- | --- |
| Private | Да. Аналогично рабочим группам (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. Аналогично группам встреч (Meeting) в новой версии. |
| AVChatRoom | Нет |
| Community | Да |

Выше перечислены встроенные типы групп Tencent Cloud Chat. Дополнительные сведения см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

> **Примечание:** Аудиовизуальные группы (AVChatRoom) не поддерживают этот API. Если вы используете этот API для аудиовизуальной группы, будет возвращена ошибка 10007. Единственный способ для пользователей присоединиться к этому типу группы — подать заявку на присоединение.

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/add_group_member?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где расположен ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/group_open_http_svc/add_group_member | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Дополнительные сведения см. в разделе **Администратор приложения** в [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробнее см. в разделе [Создание UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса. Значение фиксировано как `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Базовый формат**
Используется для приглашения пользователей в группу. Один запрос поддерживает добавление до 300 участников. По умолчанию бэкенд отправляет системные уведомления группы всем участникам группы, за исключением приватных групп (или рабочих групп в новой версии), которые не активированы.

```
{  "GroupId": "@TGS#2J4SZEAEL", // (Обязательно) Целевая группа  "MemberList": [ // За один раз можно добавить до 300 участников.  {      "Member_Account": "tommy" // ID добавляемого участника (обязательно)  },  {      "Member_Account": "jared"  }]}
```

- **Добавление участников без уведомления**
Когда `Silence` установлена на `1`, система не уведомляет никого после успешного добавления участников.

```
{  "GroupId": "@TGS#2J4SZEAEL", // (Обязательно) Целевая группа  "Silence": 1, // Добавлять ли участников без уведомления (опционально)  "MemberList": [ // За один раз можно добавить до 300 участников.  {      "Member_Account": "tommy" // ID добавляемого участника (обязательно)  },  {      "Member_Account": "jared"  }]}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID целевой группы |
| Silence | Integer | Да | Добавлять ли участников без уведомления. `0`: Нет. `1`: Да. Значение по умолчанию — `0`. |
| MemberList | Array | Да | Список добавляемых участников |
| Member_Account | String | Да | UserID добавляемого участника |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0,    "MemberList": [    {         "Member_Account": "tommy",         "Result": 1 // Результат добавления участника. 0: не удалось. 1: успешно. 2: уже в группе.    },    {         "Member_Account": "jared",         "Result": 1    }]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: не удалось |
| ErrorCode | Integer | Код ошибки. `0`: Успешно; другие значения: Не удалось |
| ErrorInfo | String | Информация об ошибке |
| MemberList | Array | Результат добавления участников |
| Member_Account | String | UserID участника |
| Result | Integer | Результат добавления участника. `0`: Не удалось. `1`: Успешно. `2`: Уже в группе. `3`: Ожидание одобрения приглашаемым лицом или администратором. |

## Коды ошибок

Возвращаемый код состояния HTTP для этого API всегда равен 200, если только не возникла ошибка сети (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10003 | Неверная команда. |
| 10004 | Неверный параметр. Проверьте описание ошибки и устраните проблему. |
| 10005 | Количество учетных записей участников в пакете запроса превышает 300. |
| 10007 | Отсутствуют разрешения на операцию. В этом случае проверьте, поддерживает ли тип группы приглашение пользователей. Например, группы AVChatRoom и BChatRoom не позволяют никому приглашать других в группы. |
| 10014 | Пользователи в запросе не могут быть добавлены в группу, так как группа уже полна. В этом случае попробуйте удалить некоторые `Member_Account` в запросе или измените значение поля `MaxMemberNum` в информации базовой группы. Для получения информации см. раздел **Информация базовой группы** в [Базовая информация группы](https://intl.cloud.tencent.com/document/product/1047/33529#GroupBaseInfoFilter). |
| 10010 | Группа не существует или была удалена. |
| 10015 | Неверный ID группы. Используйте правильный ID группы. |
| 10016 | Бэкенд разработчика отклонил эту операцию через webhook. |
| 10019 | UserID не существует. Убедитесь, что все учетные записи, указанные в `Member_Account`, верны. |
| 10026 | Команда запроса `SDKAppID` отключена. Обратитесь в службу поддержки. |
| 10037 | Количество групп, в которых участвует приглашенный пользователь, превышает лимит. В этом случае проверьте и удалите `Member_Account`, который присоединился к избыточному количеству групп, или приобретите обновление в соответствии с вашими потребностями. Информацию о планах Chat см. в разделе [Цены](https://intl.cloud.tencent.com/document/product/1047/34350). |

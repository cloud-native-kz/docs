# Блокировка членов группы

## Обзор функции

Этот API используется для блокировки члена группы. Заблокированный член не может получать сообщения и повторно присоединяться к аудио-видео группе в течение периода блокировки.

> **Примечание:** Блокировка членов аудио-видео группы поддерживается SDK Enhanced edition v6.6 и выше, а также web SDK v2.22.0 и выше. Для использования этой функции необходимо [приобрести издание Pro, Pro Plus или Enterprise](https://www.tencentcloud.com/document/product/1047/34577), перейти в [консоль](https://console.trtc.io/chat/qun-setting), выбрать **Конфигурация группы** > **Конфигурация функций группы** и включить **Блокировка членов аудио-видео группы**.

## Описание вызова API

### Поддерживаемые типы групп

| ID типа группы | Поддержка RESTful API |
| --- | --- |
| Private | Да. Аналогично рабочим группам (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. Аналогично группам встреч (Meeting) в новой версии. |
| AVChatRoom | Да |
| Community | Да |

Это предварительно установленные типы групп в Chat. Дополнительные сведения см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/ban_group_member?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, в котором расположен ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/group_open_http_svc/ban_group_member | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Аккаунт администратора приложения. Дополнительные сведения см. в разделе **Администратор приложения** в [Аутентификации входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная аккаунтом администратора приложения. Дополнительные сведения см. в разделе [Создание UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Базовый формат**

Блокировка членов

```
{    "GroupId": "@TGS#aJRGC4MH6",  // ID группы, члены которой должны быть заблокированы    "Members_Account":["brennanli3", "brennanli12"],  // ID аккаунтов членов для блокировки. До 20 ID аккаунтов поддерживается на один запрос.    "Duration":3600,  // Длительность блокировки в секундах    "Description": "you are banned because of irregularities"  // Информация о блокировке, длина которой не превышает 1000 байт}
```

### Поля запроса

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы, члены которой должны быть заблокированы |
| Members_Account | Array | Да | ID аккаунтов членов для блокировки. До 20 ID аккаунтов поддерживается на один запрос. |
| Duration | Integer | Да | Длительность блокировки в секундах. 32-битное целое число без знака. |
| Description | String | Нет | Информация о блокировке, длина которой не превышает 1000 байт |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode":0}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: неудачно |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: неудачно |
| ErrorInfo | String | Информация об ошибке |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда равен 200, за исключением сетевых ошибок (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В таблице ниже описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10003 | Неверное имя команды. |
| 10004 | Неверный параметр. Проверьте описание ошибки и устраните неполадку. |
| 10005 | Количество членов для блокировки превышает верхний предел 20 членов на запрос. |
| 10007 | Недостаточно прав на выполнение операции. Например, переключатель не включен в консоли или операционный аккаунт не является корневым аккаунтом. |

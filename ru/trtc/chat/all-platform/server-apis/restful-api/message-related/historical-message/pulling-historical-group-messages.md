# Получение исторических сообщений группы

## Справочная информация

- Сообщения группы Tencent Chat сортируются по seq, и seq выделяется в порядке получения сообщений группы сервером. Seq больше для сообщений группы, отправленных ранее, и меньше для сообщений группы, отправленных позже.
- Чтобы получить все сообщения группы, вам не нужно указывать seq для начального получения. Вместо этого сервер автоматически возвращает самые последние сообщения. Для последующих получений введите значение наименьшего seq, возвращенного ранее, минус 1.
- Если значение `IsPlaceMsg` в возвращаемом сообщении равно `1`, это означает, что сообщение с этим seq истекло, не было сохранено или было удалено.

## Обзор функции

Этот API позволяет администратору приложения получать исторические сообщения группы.

## Описание вызова API

### Применимые типы групп

| ID типа группы | Поддержка этого RESTful API |
| --- | --- |
| Private | Да. То же самое, что рабочие группы (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. То же самое, что группы встреч (Meeting) в новой версии. |
| AVChatRoom | Нет |
| Community | Да |

Выше приведены встроенные группы Chat. Для получения дополнительной информации см. [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

> **Примечание**: Аудио-видео группы (AVChatRoom) не поддерживают этот API, так как исторические сообщения этого типа группы не могут быть сохранены.

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/group_msg_get_simple?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

| Параметр | Описание |

| v4/group_open_http_svc/group_msg_get_simple | API запроса |

| usersig | Подпись, созданная учетной записью администратора приложения. Дополнительные сведения см. в разделе [Создание UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Базовый формат**
Получает исторические сообщения группы. Будут возвращены самые последние `ReqMsgNumber` сообщений группы.

```
{  "GroupId": "@TGS#15ERQPAER",    // ID группы, сообщения которой должны быть получены  "ReqMsgNumber": 2      // Количество получаемых сообщений}
```

- **Получение по seq**
Получает исторические сообщения группы на основе указанного seq.
Seq возвращаемых сообщений меньше или равен `ReqMsgSeq`.

```
{  "GroupId": "@TGS#15ERQPAER",  "ReqMsgSeq": 7803321,       // Максимальный seq запрашиваемых сообщений. Будут возвращены сообщения с seq меньше или равным `ReqMsgSeq`.  "ReqMsgNumber": 2}
```

### Поля запроса

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы, сообщения которой должны быть получены |
| ReqMsgNumber | Integer | Да | Количество исторических сообщений для получения. В настоящее время за один запрос можно вернуть максимум 20 исторических сообщений. Поэтому установите значение этого поля на 20 или менее. |
| ReqMsgSeq | Integer | Нет | Максимальный seq получаемых сообщений |
| WithRecalledMsg | Integer | Опционально | Получать ли отозванные сообщения. `1`: получить отозванные сообщения. По умолчанию отозванные сообщения не получаются. |
| TopicId | String | Нет | ID темы, для которой получаются сообщения. Это поле применимо только к сообществам с поддержкой темы. |


### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorInfo | String | Информация об ошибке |
| ErrorCode | Integer | Код ошибки. `0`: Успешно; другие значения: Ошибка |
| `groupID` | `String` | ID группы в запросе |
| IsFinished | Integer | Возвращены ли все запрашиваемые сообщения. `1`: Все запрашиваемые сообщения возвращены. `0`: Не все запрашиваемые сообщения возвращены, так как сообщения слишком длинные или количество сообщений превышает 20. `2`: Запрашиваемые сообщения слишком длинные или количество сообщений превышает 20, и все сообщения истекли. |
| MsgList | Array | Список возвращаемых сообщений |
| From_Account | String | UserID отправителя сообщения |
| IsPlaceMsg | Integer | Является ли сообщение пустым. Если сообщение было удалено или истекло, `MsgBody` пусто и значение этого поля равно `1`. Если сообщение было отозвано, значение этого поля равно `2`. |
| MsgPriority | Integer | Приоритет сообщения, используемый для дедупликации сообщений. Значение вводится при отправке сообщения клиентом. Если значение не введено, сервер автоматически генерирует его. `1`: высокий приоритет; `2`: нормальный приоритет; `3`: низкий приоритет; `4`: самый низкий приоритет |
| MsgRandom | Integer | Случайное значение сообщения, используемое для дедупликации сообщений. Значение вводится при отправке сообщения клиентом. Если значение не введено, сервер автоматически генерирует его. |
| MsgSeq | Integer | Уникальный seq сообщения. Чем меньше значение, тем раньше было отправлено сообщение. |
| MsgTimeStamp | Integer | Временная метка отправки сообщения, которая следует системному времени сервера |
| MsgBody | Array | Тело сообщения. Для получения дополнительной информации см. [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). Если IsSystemMsg установлено на 1, это поле является объектом структуры MsgBody. Если IsSystemMsg установлено на 0, это поле является массивом, содержащим несколько структур MsgElement. |
| IsSystemMsg | Integer | Является ли сообщение системным сообщением. `1`: Да |

## Коды ошибок

HTTP код состояния, возвращаемый этим API, всегда равен 200, если не происходит сетевая ошибка (например, ошибка 502). Конкретный код ошибки и сведения можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10003 | Недействительное слово команды. |
| 10004 | Недействительный параметр. Проверьте описание ошибки и устраните проблему. |
| 10007 | Нет прав доступа. Оператор должен иметь разрешения на выполнение соответствующих операций. |
| 10010 | Группа не существует или была удалена. |
| 10015 | Недействительный ID группы. Используйте правильный ID группы. |

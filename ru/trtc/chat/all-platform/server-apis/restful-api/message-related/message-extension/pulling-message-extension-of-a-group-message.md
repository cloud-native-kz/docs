# Получение расширения сообщения групповой беседы

## Обзор функции

Администраторы приложений и члены группы могут получать расширение сообщения из личной беседы. Расширение сообщения — это набор пользовательских пар ключ-значение.

> **Примечание**Для использования этой функции необходимо [приобрести выпуск Pro, выпуск Pro Plus или Enterprise](https://www.tencentcloud.com/document/product/1047/34577) и включить функцию **Set message extension** (Установить расширение сообщения) на странице **Login and Message** (Вход и сообщения) в консоли Chat. Для каждого группового сообщения можно установить до 300 пар ключ-значение. Для групповых сообщений, для которых установлены пары ключ-значение, при отправке необходимо указать поддержку расширения сообщения. Подробнее см. [Отправка обычных сообщений в группе](https://intl.cloud.tencent.com/document/product/1047/34959).

## Описание вызова API

### Применимые типы групп

| ID типа группы | Поддержка RESTful API |
| --- | --- |
| Private | Да. Аналогично рабочим группам (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. Аналогично группе встреч (Meeting) в новой версии. |
| AVChatRoom | Нет |
| Community | Нет |

Это встроенные типы групп в Chat. Дополнительные сведения см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

### Пример URL запроса

```
https://xxxxxx/v4/openim_msg_ext_http_svc/group_get_key_values?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для получения информации о других параметрах см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, в котором находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/openim_msg_ext_http_svc/group_get_key_values | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Дополнительные сведения см. в разделе **App Admin** (Администратор приложения) в [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробнее см. [Создание UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса. Значение фиксировано на `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

**Основной формат**

Получить расширения сообщения группового сообщения, у которого `MsgSeq` равен 158:

```
{    "GroupId": "@TGS#1YMVAB3IZ",    "MsgSeq": 158}
```

Получить расширения сообщения группового сообщения, у которого `MsgSeq` равен 158, а `Seq` больше или равен 2:

```
{    "GroupId": "@TGS#1YMVAB3IZ",    "MsgSeq": 158,    "StartSeq": 2}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы |
| MsgSeq | Integer | Да | Seq группового сообщения для настройки |
| StartSeq | Integer | Нет | Расширение сообщения будет получено для сообщений, `Seq` которых больше или равен `StartSeq`. |

### Пример ответа

```
{    "ErrorCode": 0,    "ErrorInfo": "",    "ActionStatus": "OK",    "CompleteFlag": 1,    "LatestSeq": 2,    "ClearSeq": 0,    "ExtensionList": [        {            "Key": "key1",            "Value": "1",            "Seq": 1        },        {            "Key": "key2",            "Value": "2",            "Seq": 1        },        {            "Key": "key3",            "Value": "3",            "Seq": 2        }    ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: ошибка |
| ErrorInfo | String | Информация об ошибке |
| ExtensionList | Array | Список возвращаемых пар ключ-значение, отсортированных по `Seq`. За один запрос можно вернуть до 40 пар ключ-значение. Если пар ключ-значение больше 40, их необходимо получить партиями на основе `StartSeq`. |
| CompleteFlag | Integer | `1`: все пары ключ-значение возвращены. `0`: не все пары ключ-значение возвращены. |
| LatestSeq | Integer | Последний `Seq` на бэкэнде, который является максимальным значением `Seq` среди всех пар ключ-значение. |
| ClearSeq | Integer | `Seq` для операции очистки. Все пары ключ-значение, `Seq` которых меньше или равен `ClearSeq`, должны быть удалены локально на клиенте. |

## Коды ошибок

HTTP-код состояния, возвращаемый этим API, всегда равен 200, если не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для публичных кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10004 | Неверный параметр. Проверьте описание ошибки и устраните проблему. |
| 10008 | Недействительный запрос: запрос не является запросом плана Pro edition, Pro Plus edition или Enterprise edition. |
| 23004 | Групповое сообщение не существует. |

# Изменение исторических личных сообщений

## Описание вызова API

- Этот API используется администратором для изменения исторических личных сообщений.
- Вы можете изменять поля `MsgBody` и `CloudCustomData` отдельно или одновременно для сообщения путем перезаписи значений полей в историческом сообщении значениями, указанными в запросах.
- Вы можете получить `MsgKey` личного сообщения для изменения следующими способами:
  - Включить [обратный вызов перед отправкой личного сообщения](https://intl.cloud.tencent.com/document/product/1047/34364) или [обратный вызов после отправки личного сообщения](https://intl.cloud.tencent.com/document/product/1047/34365) для записи `MsgKey` каждого личного сообщения.
  - Использовать [API для запроса личных сообщений](https://intl.cloud.tencent.com/document/product/1047/35478) для запроса `MsgKey` личного сообщения для изменения.
  - Для личных сообщений, отправленных через RESTful API [отправки личного сообщения одному пользователю](https://intl.cloud.tencent.com/document/product/1047/34919) и [отправки личного сообщения нескольким пользователям](https://intl.cloud.tencent.com/document/product/1047/34920), `MsgKey` сообщения содержится в пакетах ответов.

> **Примечание** Сообщения, измененные этим API, не могут быть восстановлены.

### Пример URL запроса

```
https://xxxxxx/v4/openim/modify_c2c_msg?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/openim/modify_c2c_msg | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Для получения дополнительной информации см. раздел **Администратор приложения** в [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Для подробностей см. [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса, который всегда должен быть `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

#### Изменение только информации `MsgBody` объекта сообщения

```
{    "From_Account": "dramon1",    "To_Account": "dramon2",    "MsgKey": "1_2_3",    "MsgBody": [        {            "MsgType": "TIMTextElem",            "MsgContent": {                "Text": "hello"            }        }    ]}
```

#### Изменение только информации `CloudCustomData` объекта сообщения

```
{    "From_Account": "dramon1",    "To_Account": "dramon2",    "MsgKey": "1_2_3",    "CloudCustomData": "your cloud custom data"}
```

#### Изменение информации `MsgBody` и `CloudCustomData` объекта сообщения

```
{    "From_Account": "dramon1",    "To_Account": "dramon2",    "MsgKey": "1_2_3",    "MsgBody": [        {            "MsgType": "TIMTextElem",            "MsgContent": {                "Text": "hello"            }        }    ],    "CloudCustomData": "your cloud custom data"}
```

### Поля запроса

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| From_Account | String | Да | UserID отправителя сообщения. |
| To_Account | String | Да | `UserID` получателя |
| MsgKey | String | Да | Уникальный идентификатор сообщения, которое необходимо изменить. Информацию о том, как получить `MsgKey` сообщения, см. в описании API. |
| MsgBody | Array | Нет | Текст сообщения. Подробные сведения о форматах см. в разделе [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). (Примечание: сообщение может содержать несколько элементов сообщения, в этом случае `MsgBody` является массивом.) |
| CloudCustomData | String | Нет | Пользовательские данные сообщения. Они сохраняются в облаке и отправляются получателю. Такие данные могут быть получены после удаления и переустановки приложения. |

#### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorCode": 0,    "ErrorInfo": "succeed"}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: Успешно. `FAIL`: Ошибка |
| ErrorCode | Integer | Код ошибки. `0`: Успешно. Другие значения: Ошибка |
| ErrorInfo | String | Информация об ошибке |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда равен 200, если не происходит ошибка сети (например, ошибка 502). Конкретный код ошибки и подробности можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, характерные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 20001 | Неверный запрос. |
| 20002 | `UserSig` или `A2` истекли. |
| 20003 | `UserID` отправителя или получателя неверен или не существует. Убедитесь, что `UserID` был импортирован в Chat. |
| 20004 | Ошибка сети. Повторите попытку. |
| 20005 | Внутренняя ошибка сервера. Повторите попытку. |
| 90001 | Не удалось проанализировать JSON запрос. Убедитесь, что формат верен. |
| 90002 | `MsgBody` в JSON запросе не соответствует требованиям формата сообщения или `MsgBody` не является массивом. Для получения дополнительной информации см. раздел **Элемент сообщения TIMMsgElement** в [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527#.E6.B6.88.E6.81.AF.E5.85.83.E7.B4.A0-timmsgelement). |
| 90003 | JSON запрос не содержит поле `To_Account` или поле `To_Account` не является строкой. |
| 90007 | Поле `MsgBody` в JSON запросе не является массивом. Измените его на массив. |
| 90009 | Запрос требует прав администратора приложения. |
| 90010 | JSON запрос не соответствует требованиям формата сообщения. Для получения дополнительной информации см. раздел **Элемент сообщения TIMMsgElement** в [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527#.E6.B6.88.E6.81.AF.E5.85.83.E7.B4.A0-timmsgelement). |
| 90012 | Учетная запись, указанная в `To_Account`, не существует или не зарегистрирована. Убедитесь, что учетная запись была импортирована в Chat и является правильной. |
| 91000 | Внутренняя ошибка сервиса. Повторите попытку. |
| 90992 | Внутренняя ошибка сервиса. Повторите попытку. Если этот код ошибки возвращается для всех запросов и включен webhook, убедитесь, что сервер приложения правильно возвращает результаты на backend Chat. |

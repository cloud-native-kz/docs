# Отправка сообщений всем пользователям

Отправка сообщений всем пользователям — это превосходный инструмент для операций с пользователями приложения. Он не только поддерживает отправку определённого контента всем пользователям, но также позволяет отправлять персонализированный контент определённым группам пользователей на основе тегов и атрибутов, таких как события для участников и региональные уведомления. Это помогает эффективно привлекать, преобразовывать и активировать новых пользователей.

## Обзор функции

- Этот API можно использовать для отправки сообщений всем пользователям.
- Этот API можно использовать для отправки сообщений по атрибутам пользователя.
- Этот API можно использовать для отправки сообщений по тегам пользователя.
- Когда администратор отправляет сообщения, отправитель сообщения, отображаемый для получателей, является администратором.
- Когда администратор указывает учетную запись для отправки сообщений другим учетным записям, отправитель, отображаемый для получателей, не является администратором, а является учетной записью, указанной администратором.
- Этот API поддерживает автономное хранилище сообщений, но не поддерживает синхронизацию сообщений.
- Отправка сообщений всем пользователям занимает время. Требуемое время зависит от количества учетных записей. Обычно это занимает менее одной минуты.
- Этот API позволяет отправлять сообщения только онлайн-пользователям, установив параметр `MsgLifeTime` в значение `0`.

> **Примечание:** Функция "отправка сообщений всем пользователям" доступна только в редакции IM Pro, редакции Pro Plus или Enterprise Edition. Для её использования [приобретите редакцию Pro, Pro Plus или Enterprise Edition](https://www.tencentcloud.com/document/product/1047/34577#.E5.8D.87.E7.BA.A7.E5.BA.94.E7.94.A8). Дополнительную информацию см. в разделе [Ценообразование](https://intl.cloud.tencent.com/document/product/1047/34350).

## Описание вызова API

Функция отправки сообщений всем пользователям доступна только пользователям с учетными записями редакции Pro, Pro Plus или Enterprise Edition. См. раздел [Заявка на изменение конфигурации](https://intl.cloud.tencent.com/document/product/1047/44322) для подачи заявки на эту функцию. Функция будет включена **через 48 часов** после одобрения вашей заявки.

### Примеры URL запроса

```
https://xxxxxx/v4/all_member_push/im_push?usersig=xxx&identifier=admin&sdkappid=88888888&random=99999999&contenttype=json
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| xxxxxx | Доменное имя, соответствующее стране/региону, где расположен ваш SDKAppID. Материковая часть Китая: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Силиконовая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/all_member_push/im_push | API запроса |
| usersig | Подпись, сгенерированная в учетной записи администратора приложения. Дополнительную информацию см. в разделе [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| identifier | Учетная запись администратора приложения. |
| sdkappid | `SDKAppID`, присвоенный консолью IM при создании приложения |
| random | Случайное 32-битное целое число без знака |
| contenttype | Значение всегда равно `json`. |

### Частота вызовов

Этот API включает отправку всем, по атрибутам и по тегам. По умолчанию его можно вызывать 100 раз в день. Интервал между двумя отправками должен быть больше одной секунды.

### Примеры запроса

- **Отправка сообщений всем пользователям**
Администратор отправляет сообщения всем пользователям и сохраняет сообщения в автономном режиме в течение 120 секунд.

```
{  "From_Account": "admin",  "MsgRandom": 56512,  "MsgLifeTime": 120,  "MsgBody":[      {          "MsgType": "TIMTextElem",          "MsgContent":{              "Text": "hi, beauty"          }      }  ]}
```

Администратор указывает учетную запись для отправки всем пользователям и сохраняет сообщения в автономном режиме в течение 120 секунд. В примере учетная запись отправителя — `xiaoming`:

```
{  "From_Account": "xiaoming",  "MsgRandom": 3674128,  "MsgLifeTime": 120,  "MsgBody":[      {          "MsgType": "TIMTextElem",          "MsgContent":{              "Text": "hi, beauty"          }      }  ]}
```

Администратор указывает учетную запись для отправки всем пользователям, устанавливает информацию об автономной отправке и сохраняет сообщения в автономном режиме в течение 120 секунд.

```
{  "From_Account": "xiaoming",  "MsgRandom": 3674128,  "MsgLifeTime": 120,  "MsgBody":[      {          "MsgType": "TIMTextElem",          "MsgContent":{              "Text": "hi, beauty"          }      }  ],  "OfflinePushInfo": {      "PushFlag": 0,      "Desc": "Content to push offline",      "Ext": "Passthrough content",      "AndroidInfo": {          "Sound": "android.mp3"      },      "ApnsInfo": {          "Sound": "apns.mp3",          "BadgeMode": 1, // If this field is left as default or is set to `0`, the message is counted. If this field is set to `1`, the message is not counted, that is, the badge counter in the upper-right corner does not increase.          "Title":"apns title", // APNs title          "SubTitle":"apns subtitle", // APNs subtitle          "Image":"www.image.com" // Image URL      }  }}
```

Администратор отправляет сообщения всем пользователям и сохраняет сообщения в автономном режиме в течение 120 секунд.

```
{  "From_Account": "admin",  "MsgRandom": 21302570,  "MsgLifeTime": 120,  "MsgBody":[      {          "MsgType": "TIMTextElem",          "MsgContent":{              "Text": "hi, beauty"          }      }  ]}
```

- **Отправка сообщений по тегам пользователя**
Администратор отправляет сообщения пользователям с тегами "Stock A" и "Stock B" и сохраняет сообщения в автономном режиме в течение 120 секунд.

```
{  "From_Account": "admin",  "MsgRandom": 214,  "MsgLifeTime": 120,  "Condition":{      "TagsAnd": ["Stock A","Stock B"]  },  "MsgBody":[      {          "MsgType": "TIMTextElem",          "MsgContent":{              "Text": "hi, beauty"          }      }  ]}
```

Администратор отправляет сообщения пользователям с тегами "Stock A" и "Stock B" и сохраняет сообщения в автономном режиме в течение 120 секунд.

```
{  "From_Account": "admin",  "MsgRandom": 124032,  "MsgLifeTime": 120,  "Condition":{      "TagsOr": ["Stock A","Stock B"]  },  "MsgBody":[      {          "MsgType": "TIMTextElem",          "MsgContent":{              "Text": "hi, beauty"          }      }  ]}
```

- **Отправка сообщений по атрибутам пользователя**
Администратор отправляет сообщения пользователям Platinum Premier из Шэньчжэня и сохраняет сообщения в автономном режиме в течение 120 секунд.

```
{  "From_Account": "admin",  "MsgRandom": 389475,  "MsgLifeTime": 120,  "Condition":{      "AttrsAnd": {          "Membership Level": "Platinum Premier members",          "city": "Shenzhen"      }  },  "MsgBody":[      {          "MsgType": "TIMTextElem",          "MsgContent":{              "Text": "hi, beauty"          }      }  ]}
```

Администратор отправляет сообщения пользователям Platinum Premier из Шэньчжэня и сохраняет сообщения в автономном режиме в течение 120 секунд.

```
{  "From_Account": "admin",  "MsgRandom": 9312457,  "MsgLifeTime": 120,  "Condition":{      "AttrsAnd": {          "Membership Level": "Platinum Premier users",          "city": "Shenzhen"      }  },  "MsgBody":[      {          "MsgType": "TIMTextElem",          "MsgContent":{              "Text": "hi, beauty"          }      }  ]}
```

### Поля запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| Condition | Object | Необязательный | Допустимые значения: AttrsOr, AttrsAnd, TagsOr, TagsAnd. `AttrsOr` и `AttrsAnd` могут сосуществовать, а `TagsOr` и `TagsAnd` могут сосуществовать. Однако условия на основе тегов и атрибутов не могут сосуществовать. Если условие не указано, сообщения отправляются всем пользователям. |
| MsgRandom | Integer | Обязательный | Случайное число (32-битное целое число без знака) сообщения. Оно используется сервером для дедупликации сообщений в течение секунды. Убедитесь, что введено случайное число. |
| MsgBody | Array | Обязательный | Текст сообщения. Дополнительную информацию о формате сообщения см. в разделе [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). Обратите внимание, что `MsgBody` — это массив, который может содержать несколько элементов сообщения. |
| MsgType | String | Обязательный | Тип объекта сообщения TIM. Допустимые значения: `TIMTextElem` (текстовое сообщение), `TIMLocationElem` (сообщение о местоположении), `TIMFaceElem` (сообщение с эмодзи), `TIMCustomElem` (пользовательское сообщение), `TIMSoundElem` (голосовое сообщение), `TIMImageElem` (сообщение с изображением), `TIMFileElem` (сообщение с файлом), `TIMVideoFileElem` (видеосообщение) |
| MsgContent | Object | Обязательный | Различные типы объектов сообщений (`MsgType`) имеют разные форматы (`MsgContent`). Дополнительную информацию см. в разделе **Элемент сообщения TIMMsgElement** в [Форматах сообщений](https://intl.cloud.tencent.com/document/product/1047/33527#.E6.B6.88.E6.81.AF.E5.85.83.E7.B4.A0-timmsgelement). |
| MsgLifeTime | Integer | Необязательный | Время хранения сообщения в автономном режиме, в секундах. Максимальная длительность составляет 604 800 секунд (7 дней). Значение по умолчанию — 0, что означает, что сообщения не сохраняются в автономном режиме и отправляются только онлайн-пользователям. |
| From_Account | String | Необязательный | Учетная запись отправителя сообщения |
| AttrsOr | Object | Необязательный | Набор условий по атрибутам, связанных логическим OR. Обратите внимание, что условия по атрибутам и условия по тегам не могут использоваться одновременно. |
| AttrsAnd | Object | Необязательный | Набор условий по атрибутам, связанных логическим AND. Обратите внимание, что условия по атрибутам и условия по тегам не могут использоваться одновременно. |
| TagsOr | Array | Необязательный | Объединение условий по тегам. Тег — это строка из максимум 50 байт. Обратите внимание, что условия по атрибутам и условия по тегам не могут использоваться одновременно. Условие `TagsOr` может содержать до десяти тегов. |
| TagsAnd | Array | Необязательный | Пересечение условий по тегам. Тег — это строка из максимум 50 байт. Обратите внимание, что условия по атрибутам и условия по тегам не могут использоваться одновременно. Условие `TagsAnd` может содержать до десяти тегов. |
| OfflinePushInfo | Object | Необязательный | Информация для отправки в автономном режиме. Дополнительную информацию см. в разделе [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0,    "TaskId": "1400123456_144115212910570789_4155518400_15723514"}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки |
| ErrorInfo | String | Сообщение об ошибке |
| TaskId | String | ID задачи отправки |

## Коды ошибок

Если не возникает сетевая ошибка (например, ошибка 502), код возврата HTTP для этого API всегда равен 200. `ErrorCode` **и** `ErrorInfo` **в ответе представляют фактический код ошибки и сообщение об ошибке.** Для общих кодов ошибок (60000–79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 90001 | Ошибка при разборе формата JSON. Проверьте, соответствует ли запрос JSON спецификациям JSON. |
| 90002 | Поле `MsgBody` в запросе JSON не соответствует требованиям формата сообщения или это не массив. Дополнительную информацию см. в разделе **Элемент сообщения TIMMsgElement** в [Форматах сообщений](https://intl.cloud.tencent.com/document/product/1047/33527#.E6.B6.88.E6.81.AF.E5.85.83.E7.B4.A0-timmsgelement). |
| 90005 | Поле `MsgRandom` отсутствует в запросе JSON или это не целое число. |
| 90007 | Поле `MsgBody` в запросе JSON — это не массив. Измените его на массив. |
| 90009 | Запрос требует разрешений администратора приложения. |
| 90010 | Запрос JSON не соответствует требованиям формата сообщения. Дополнительную информацию см. в разделе **Элемент сообщения TIMMsgElement** в [Форматах сообщений](https://intl.cloud.tencent.com/document/product/1047/33527#.E6.B6.88.E6.81.AF.E5.85.83.E7.B4.A0-timmsgelement). |
| 90020 | Длина тега превышает лимит (максимально допустимая длина составляет 50 байт). |
| 90022 | `TagsOr` и `TagsAnd` в условиях отправки содержат повторяющиеся теги. |
| 90024 | Отправка слишком частая. Интервал между двумя отправками должен быть больше 1 секунды. |
| 90026 | Неправильный период хранения сообщения в автономном режиме. Значение не может превышать 7 дней. |
| 90032 | Количество тегов в условиях отправки превышает 10 или количество тегов в запросе на добавление тега превышает 10. |
| 90033 | Неправильный атрибут. |
| 90039 | Отправка сообщений по атрибутам и отправка сообщений по тегам являются взаимоисключающими. |
| 90040 | Тег в условиях отправки имеет нулевое значение. |
| 90045 | Функция отправки сообщений всем пользователям не включена. |
| 90047 | Количество отправок превышает дневную квоту (квота по умолчанию: 100). |
| 91000 | Внутренняя ошибка сервиса. Повторите попытку. |

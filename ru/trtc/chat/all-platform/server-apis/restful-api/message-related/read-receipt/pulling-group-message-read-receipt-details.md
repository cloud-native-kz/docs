# Получение сведений о прочтении групповых сообщений

## Обзор функции

Администратор приложения может использовать этот API для получения списка участников, которые прочитали или не прочитали групповое сообщение.

> **Примечание:** Эта функция доступна только в редакциях Pro, Pro Plus и Enterprise. Чтобы использовать её, необходимо [приобрести редакцию Pro, Pro Plus или Enterprise](https://www.tencentcloud.com/document/product/1047/34577), перейти в [**консоль**](https://console.tencentcloud.com/im/qun-setting), выбрать **Feature Configuration** > **Group configuration** > **Group Message Read Receipts** и включить функцию.

## Описание вызова API

| --- | --- |

| Public | Да |
| ChatRoom | Да. То же самое, что и группы встреч (Meeting) в новой версии. |
| AVChatRoom | Нет |

Выше приведены встроенные группы Chat. Дополнительные сведения см. в [Системе групп](https://intl.cloud.tencent.com/document/product/1047/33529).

```
https://xxxxxx/v4/group_open_http_svc/get_group_msg_receipt_detail?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

Получение списка участников, которые прочитали или не прочитали групповое сообщение, в пакетном режиме:

- **Базовый формат**
Вы можете использовать поля `Cursor` и `Count` для управления режимом постраничного получения:
- `Count`: Указывает максимальное количество участников в массиве `ReadList` или `UnreadList` в ответе. Максимальное значение: 200
- `Cursor`: Указывает позицию участника, с которой должна начинаться последующая выборка информации. Для первого запроса параметр клиента `Cursor` должен передать "". Для последнего запроса сервер возвращает `1` для параметра `IsFinish`, указывая, что выборка информации завершена.

```
{ "GroupId":"@TGS#2TTV7VSII", "MsgSeq": 1, "Filter": 1,        // `0`: получить список участников, которые прочитали сообщение; `1`: получить список участников, которые не прочитали сообщение "Cursor":"",      // Позиция последнего полученного участника. Введите "" для первого запроса "Count":5           // Максимальное количество участников для получения}
```

### Поля запроса

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы, для которой должны быть получены подтверждения прочтения сообщений |
| MsgSeq | Integer | Да | Seq сообщения для получения |
| Filter | Integer | Да | Получить список участников, которые прочитали или не прочитали групповое сообщение. `0`: получить список участников, которые прочитали групповое сообщение; `1`: получить список участников, которые не прочитали групповое сообщение |
| Cursor | String | Да | Позиция последнего полученного участника. Введите "" для первого запроса. |
| Count | Integer | Да | Максимальное количество участников, которое можно получить за один раз. Максимальное значение — 200. |

### Пример ответа

- **Получение списка участников, которые прочитали групповое сообщение**

```
{    "ActionStatus": "OK",    "Cursor": "0",    "ErrorCode": 0,    "ErrorInfo": "",    "IsFinish": 1,     // Список полностью получен. Дополнительная выборка не требуется.     "MsgSeq": 1,      "ReadList": [    // Список участников, которые прочитали сообщение        {            "Read_Account": "test1"        }    ]}
```

- **Получение списка участников, которые не прочитали сообщение**

```
{  "ActionStatus": "OK",  "Cursor": "144115213529088617", // Значение `Cursor` для передачи в следующем запросе  "ErrorCode": 0,  "ErrorInfo": "",  "IsFinish": 0,   // Список не полностью получен, требуется дополнительная выборка.  "MsgSeq": 1,  "UnreadList": [  // Список участников, которые не прочитали сообщение      {          "Unread_Account": "test"      },      {          "Unread_Account": "test6"      },      {          "Unread_Account": "test3"      },      {          "Unread_Account": "test5"      },      {          "Unread_Account": "test4"      }  ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorInfo | String | Информация об ошибке |
| ErrorCode | Integer | Код ошибки. `0`: Успешно; другие значения: Ошибка |
| IsFinished | Integer | Полностью ли получен список. `0`: Нет; `1`: Да |
| MsgSeq | Integer | Seq сообщения для получения |
| ReadList | Array | Список сообщений, которые были прочитаны |
| Read_Account | String | Участники, которые прочитали сообщение |
| UnreadList | Array | Список сообщений, которые не были прочитаны |
| Unread_Account | String | Участники, которые не прочитали сообщение |
| Cursor | String | Значение `Cursor` для передачи в следующем запросе |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда равен 200, если не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и подробности можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, характерные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10003 | Неверное слово команды. |
| 10004 | Неверный параметр. Проверьте описание ошибки и устраните проблему. |
| 10007 | Нет разрешений на операцию. Оператор должен иметь разрешения на выполнение соответствующих операций. |
| 10010 | Группа не существует или была удалена. |
| 10015 | Неверный ID группы. Используйте правильный ID группы. |
| 10062 | Подтверждение прочтения не существует. |

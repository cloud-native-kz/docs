# Импорт одноранговых сообщений

## Обзор функций

- Этот API импортирует историю чата один-на-один в Tencent Cloud Chat **в хронологическом порядке**.
- Он импортирует ваши сообщения с других платформ мгновенного обмена сообщениями в Tencent Cloud Chat **в хронологическом порядке**.
- Он обновляет беседу.
- Он не вызывает обратный вызов.
- Он устраняет дубликаты импортированных сообщений в одноранговой беседе на основе полей `MsgSeq`, `MsgRandom` и `MsgTimeStamp`. **Когда значения всех этих трех полей совпадают с их соответствиями, два сообщения считаются дубликатами, независимо от их содержания.** Кроме того, если поля `MsgSeq`, `MsgRandom` и `MsgTimeStamp` двух сообщений совпадают друг с другом, но `from_account` и `to_account` противоположны, два сообщения также считаются дубликатами.
- Импортированные сообщения не будут перезаписаны теми же сообщениями из последующих импортов.
- Для одноранговых сообщений поле `MsgSeq` указывается пользователем при отправке сообщения. Его значение может повторяться. Оно не генерируется серверной частью и не является глобально уникальным. **Для групповых сообщений значение поля `MsgSeq` генерируется серверной частью, и каждая группа сохраняет свой `MsgSeq`, который строго увеличивается с 1. Для одноранговых сообщений исторические сообщения одной беседы сортируются сначала по временной метке, а затем исторические сообщения одной секунды сортируются по `MsgSeq`.**

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/openim/importmsg?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где расположен ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/openim/importmsg | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Для получения дополнительной информации см. раздел **Администратор приложения** в [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробнее см. [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса, который всегда должен быть `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Импорт сообщений в реальном времени**

```
{  "SyncFromOldSystem": 5, // Импортирует сообщения в реальном времени, помечает их как непрочитанные и отправляет клиенту.  "From_Account": "lumotuwe1", // Учетная запись отправителя  "To_Account": "lumotuwe2", // Учетная запись получателя  "MsgSeq": 827092, // Номер последовательности сообщения  "MsgRandom": 1287657, // Случайное число, присвоенное сообщению  "MsgTimeStamp": 1556178721, // UNIX временная метка в секундах  "MsgBody": [ // Тело сообщения. Это текстовое сообщение      {          "MsgType": "TIMTextElem", // Элемент текстового сообщения          "MsgContent": {              "Text": "hi, beauty"          }      }  ],  "CloudCustomData": "your cloud custom data"}
```

- **Импорт исторических сообщений**

```
 {  "SyncFromOldSystem": 2, // Импортирует исторические сообщения, помечает их как прочитанные и не отправляет клиенту.  "From_Account": "lumotuwe1", // Учетная запись отправителя  "To_Account": "lumotuwe2", // Учетная запись получателя  "MsgSeq": 827092, // Номер последовательности сообщения  "MsgRandom": 1287657, // Случайное число, присвоенное сообщению  "MsgTimeStamp": 1556178721, // UNIX временная метка в секундах  "MsgBody": [ // Тело сообщения. Это текстовое сообщение      {          "MsgType": "TIMTextElem", // Элемент текстового сообщения          "MsgContent": {              "Text": "hi, beauty" // Содержание сообщения          }      }  ],  "CloudCustomData": "your cloud custom data"}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| SyncFromOldSystem | Integer | Да | Допустимые значения: `2` и `5`. `2`: импортирует исторические сообщения, помечает их как прочитанные и не отправляет клиенту. `5`: импортирует сообщения в реальном времени, помечает их как непрочитанные и отправляет клиенту. |
| From_Account | String | Да | `UserID` отправителя, используется для указания отправителя сообщения |
| To_Account | String | Да | `UserID` получателя |
| MsgSeq | Integer | Нет | Номер последовательности (32-битное целое число без знака) сообщения. Серверная часть использует это поле для удаления дубликатов сообщений и сортировки сообщений в одну секунду. Подробнее см. **Описание функций**. Если это поле оставить пустым, серверная часть введет случайное число. |
| MsgRandom | Integer | Да | Случайное число (32-битное целое число без знака) сообщения. Используется серверной частью для удаления дубликатов сообщений в пределах одной секунды. Убедитесь, что введено случайное число. |
| MsgTimeStamp | Integer | Да | UNIX временная метка в секундах. Она отмечает время отправки сообщения и используется для удаления дубликатов сообщений. Подробнее см. **Описание функций**. |
| MsgBody | Array | Да | Тело сообщения. Подробнее о форматах см. [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). (Примечание: сообщение может содержать несколько элементов сообщения, в этом случае `MsgBody` является массивом.) |
| MsgType | String | Да | Тип объекта TIM сообщения. Допустимые значения: `TIMTextElem` (текстовое сообщение) `TIMLocationElem` (сообщение о местоположении) `TIMFaceElem` (сообщение эмодзи) `TIMCustomElem` (пользовательское сообщение) `TIMSoundElem` (голосовое сообщение) `TIMImageElem` (сообщение с изображением) `TIMFileElem` (сообщение с файлом) `TIMVideoFileElem` (видео сообщение) |
| MsgContent | Object | Да | Различные типы объектов сообщений (`MsgType`) имеют различные форматы (`MsgContent`). Подробнее см. [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| CloudCustomData | String | Нет | Пользовательские данные сообщения. Они сохраняются в облаке и будут отправлены другой стороне. Такие данные можно извлечь после удаления и переустановки приложения. |

### Примеры ответов

```
{   "ActionStatus" : "OK",   "ErrorInfo" : "",   "ErrorCode" : 0}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: ошибка |
| ErrorInfo | String | Информация об ошибке |

## Коды ошибок

HTTP код состояния для этого API всегда 200, если не происходит сетевая ошибка (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для публичных кодов ошибок (60000-79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).

В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 90001 | Ошибка парсинга JSON запроса. Убедитесь, что формат верен. |
| 90002 | `MsgBody` в JSON запросе не соответствует требованиям формата сообщения или `MsgBody` не является массивом. Для получения дополнительной информации см. раздел **Элемент сообщения TIMMsgElement** в [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527#.E6.B6.88.E6.81.AF.E5.85.83.E7.B4.A0-timmsgelement). |
| 90003 | Поле `To_Account` отсутствует в JSON запросе или это не строка. |
| 90005 | Поле `MsgRandom` отсутствует в JSON запросе или это не целое число. |
| 90006 | Поле `MsgTimeStamp` отсутствует в JSON запросе или это не целое число. |
| 90007 | Поле `MsgBody` в JSON запросе не является массивом. Измените его на массив. |
| 90008 | Поле `From_Account` отсутствует в JSON запросе или это не целое число. |
| 90009 | Запрос требует разрешений администратора приложения. |
| 90010 | JSON запрос не соответствует требованиям формата сообщения. Для получения дополнительной информации см. раздел **Элемент сообщения TIMMsgElement** в [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527#.E6.B6.88.E6.81.AF.E5.85.83.E7.B4.A0-timmsgelement). |
| 90011 | Число получателей превысило 500. Попытайтесь уменьшить количество учетных записей в To_Account. |
| 90012 | Учетная запись, указанная в `To_Account`, не существует или не зарегистрирована. Убедитесь, что учетная запись импортирована в Chat и корректна. |
| 90026 | Время автономного сохранения сообщения некорректно. Сообщения не могут быть сохранены автономно более чем на 7 дней. |
| 90030 | Поле `SyncFromOldSystem` отсутствует в JSON запросе или это не целое число. |
| 90048 | Запрашиваемая учетная запись не существует. |
| 90992 | Внутренняя ошибка сервиса. Попробуйте еще раз. Если этот код ошибки возвращается для всех запросов и обратный вызов третьей стороны включен, убедитесь, что сервер приложения нормально возвращает результаты обратного вызова серверной части Chat. |
| 91000 | Внутренняя ошибка сервиса. Попробуйте еще раз. |
| 93000 | Пакет JSON превышает максимальный размер 12 КБ. |

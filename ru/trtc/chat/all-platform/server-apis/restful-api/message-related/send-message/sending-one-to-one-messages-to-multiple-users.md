# Отправка личных сообщений нескольким пользователям

## Обзор функции

- Этот API используется для отправки личных сообщений нескольким пользователям (до 500) одновременно.
- По сравнению с API отправки личного сообщения одному пользователю, этот API лучше подходит для срочных сообщений, таких как маркетинговые сообщения и системные уведомления.
- Если вы не хотите считать сообщения как непрочитанные или хранить их, вы можете установить `MsgLifeTime` в значение `1`. Это ускорит доставку сообщений.
- Когда администратор указывает аккаунт для отправки сообщения нескольким целевым аккаунтам, отправитель, отображаемый получателям, — это не администратор, а аккаунт, указанный администратором.
- Этот API не вызывает запросы обратного вызова.
- Этот API не проверяет, являются ли отправитель и получатели друзьями или в списке блокировки друг друга, а также не проверяет, замучены ли отправители.
- Этот API не проверяет, отключил ли получатель уведомления о сообщениях от отправителя по умолчанию. При необходимости введите `WithMuteNotifications` в параметр `SendMsgControl`.
- Для личных сообщений поле `MsgSeq` указывается пользователем при отправке сообщения. Его значение может повторяться. Оно не генерируется сервером и не является глобально уникальным. Для групповых чатов значение поля `MsgSeq` генерируется сервером, и каждая группа поддерживает свой `MsgSeq`, который строго возрастает с 1. Для личных сообщений исторические сообщения одного диалога сначала сортируются по временной метке, а затем исторические сообщения одной секунды сортируются по `MsgSeq`.

> **Примечание:** При вызове этого API для массовой отправки сообщения вы должны указать, нужно ли синхронизировать сообщение с отправителем — администраторским аккаунтом или аккаунтом, указанным администратором. Синхронизация может быть осуществлена через онлайн-терминалы и серверы роуминга. Этот API предоставляет параметр `SyncOtherMachine` для определения того, нужно ли синхронизировать сообщение. Для получения дополнительной информации см. **Пример запроса** ниже.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/openim/batchsendmsg?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/openim/batchsendmsg | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Аккаунт администратора приложения. Для получения дополнительной информации см. раздел **Администратор приложения** в [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная аккаунтом администратора приложения. Для получения деталей см. [Генерация UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса, всегда должно быть `json`. |

### Максимальная частота вызовов

12 000 сообщений/минуту. Если отправить 500 пользователям одновременно, это будет засчитано как 500 сообщений.

### Пример запроса

Здесь мы используем отправку текстового сообщения в качестве примера. Для отправки сообщений других типов установите `MsgBody` на соответствующий тип сообщения. Для получения дополнительной информации см. [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527).

#### Администратор отправляет сообщение нескольким целевым аккаунтам.

> **Примечание:** Если вы не хотите синхронизировать сообщение с `From_Account`, установите `SyncOtherMachine` в значение `2`.
> Для синхронизации сообщения с `From_Account` установите `SyncOtherMachine` в значение `1`.

```
{    "SyncOtherMachine": 2, // Do not synchronize the message to the sender.    "To_Account": [ // A list of target accounts        "bonnie",        "rong"    ],    "MsgSeq": 28360, // Sequence number of the message.    "MsgRandom": 19901224, // Random number of the message    "MsgBody": [ // Message body        {            "MsgType": "TIMTextElem", // Message type. `TIMTextElem` indicates text messages.            "MsgContent": {                "Text": "hi, beauty" // Message text            }        }    ],    "CloudCustomData": "your cloud custom data"}
```

#### Администратор указывает аккаунт для отправки сообщения нескольким целевым аккаунтам и устанавливает информацию для автономной отправки уведомлений.

`From_Account` — это отправитель, указанный администратором. Отправитель, отображаемый получателям, — это не администратор, а аккаунт, указанный администратором. В следующем JSON запросе dave отправляет сообщение bonnie и rong. Когда bonnie и rong получают сообщение, отправитель, отображаемый им, — это dave.

> **Примечание:** Если вы не хотите синхронизировать сообщение с `From_Account`, установите `SyncOtherMachine` в значение `2`.
> Для синхронизации сообщения с `From_Account` установите `SyncOtherMachine` в значение `1`.

```
{    "SyncOtherMachine": 1, // Synchronize the message to the sender.    "From_Account": "dave",    "To_Account": [        "bonnie",        "rong"    ],    "MsgSeq": 28360, // Sequence number of the message.    "MsgRandom": 19901224, // Random number of the message    "MsgBody": [        {            "MsgType": "TIMTextElem",            "MsgContent": {                "Text": "hi, beauty"            }        }    ],    "CloudCustomData": "your cloud custom data",    "OfflinePushInfo": {        "PushFlag": 0,        "Desc": "Content to push offline",        "Ext": "Passthrough content",        "AndroidInfo": {            "Sound": "android.mp3"        },        "ApnsInfo": {            "Sound": "apns.mp3",            "BadgeMode": 1, // If this field is left as default or is set to `0`, the message is counted. If this field is set to `1`, the message is not counted, that is, the badge counter in the upper-right corner does not increase.            "Title":"apns title", // APNs title            "SubTitle":"apns subtitle", // APNs subtitle            "Image":"www.image.com" // Image URL        }    }}
```

### Поля запроса

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| SyncOtherMachine | Integer | Нет | `1`: синхронизировать сообщение с онлайн-терминалом и сервером роуминга `From_Account`. `2`: не синхронизировать сообщение с `From_Account`. Если это поле не указано, сообщение синхронизируется с сервером роуминга `From_Account`. |
| From_Account | String | Нет | Аккаунт отправителя, указанный администратором. Для установки информации `From_Account` значение этого поля не может быть пустым. |
| To_Account | Array | Да | `UserID` получателя сообщения |
| MsgSeq | Integer | Нет | Последовательный номер (32-битное целое число без знака) сообщения. Сервер будет использовать это поле для удаления дублирующихся сообщений и сортировки сообщений в одну секунду. Для получения деталей см. **Описание функции**. Если это поле оставлено пустым, сервер введет случайное число. |
| MsgRandom | Integer | Да | Случайное число (32-битное целое число без знака) сообщения. Оно используется сервером для удаления дублирующихся сообщений в течение одной секунды. Убедитесь, что введено случайное число. |
| MsgBody | Array | Да | TIM сообщение. Для получения дополнительной информации см. [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| OnlineOnlyFlag | Integer | Нет | Если это поле установлено в `0`, сообщение будет сохранено в исторических записях чата. `1` означает, что сообщения не будут сохранены в исторических записях чата, то есть если получатель онлайн, он получит это сообщение; если в автономном режиме, нет. Подходит для реализации некоторых функций отображения состояния в реальном времени. |
| MsgType | String | Да | Тип объекта TIM сообщения. Допустимые значения: `TIMTextElem` (текстовое сообщение) `TIMLocationElem` (сообщение о местоположении) `TIMFaceElem` (сообщение с эмодзи) `TIMCustomElem` (пользовательское сообщение) `TIMSoundElem` (голосовое сообщение) `TIMImageElem` (сообщение с изображением) `TIMFileElem` (файловое сообщение) `TIMVideoFileElem` (видеосообщение) |
| MsgContent | Object | Да | Объект TIM сообщения. Для получения дополнительной информации см. [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| CloudCustomData | String | Нет | Данные пользовательского сообщения. Они сохраняются в облаке и отправляются получателю. Такие данные можно получить после удаления и переустановки приложения. |
| SendMsgControl | Array | Нет | Параметры управления отправкой сообщений — это массив строк, который действителен только для этого запроса. "NoUnread" означает, что это сообщение не считается непрочитанным сообщением. "NoLastMsg" указывает на то, что это сообщение не обновляет список диалогов. "WithMuteNotifications" означает, что параметры «Не беспокоить» получателя для отправителя действуют для этого сообщения (по умолчанию это не вступает в силу). "NoMsgCheck" означает, что при включенной облачной проверке это сообщение не будет отправлено на проверку. Пример: "SendMsgControl": ["NoUnread","NoLastMsg","WithMuteNotifications","NoMsgCheck"] |
| OfflinePushInfo | Object | Нет | Информация для отправки в автономном режиме. Для получения дополнительной информации см. [Описание OfflinePushInfo](https://www.tencentcloud.com/document/product/1047/77715). |
| IsNeedReadReceipt | Integer | Нет | Требует ли сообщение подтверждение прочтения. `0`: Нет. `1`: Да. Значение по умолчанию — `0`. |

### Пример ответа

- Ответ при отправке сообщения всем целевым аккаунтам

```
 {  "ErrorInfo": "",  "ActionStatus": "OK",  "ErrorCode": 0,  "MsgKey": "128493_903762_1572870301",  "MsgId": "144015217074523023-1734398005-1067690400"}
```

- Ответ при отправке сообщения не всем целевым аккаунтам

```
{  "ActionStatus": "SomeError",  "ErrorCode": 0,  "ErrorInfo": "",  "MsgKey": "4852_28135_1579678877",  "MsgId": "144015217074523023-1734398005-1067690400",  "ErrorList": [ // List of the accounts to which the message was not sent      {          "To_Account": "rong", // Account to which the message was not sent          "ErrorCode":  70107 // Error code. `70107` indicates that the account does not exist.      }  ]}
```

- Ответ при отправке сообщения ни одному целевому аккаунту

```
{  "ActionStatus": "FAIL",  "ErrorInfo": "invalid To_Account",  "ErrorCode": 90012}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки, возвращаемый для запроса. Если сообщение отправлено на любой аккаунт, значение равно `0`. Если сообщение не отправлено ни на один целевой аккаунт, значение не равно `0`. |
| ErrorInfo | String | Информация об ошибке |
| ErrorList | Array | Список целевых аккаунтов, на которые сообщение не было отправлено или которые не существуют. Если сообщение отправлено на все целевые аккаунты, значение этого поля пусто. |
| ErrorList.To_Account | String | Целевой аккаунт, на который сообщение не было отправлено. |
| ErrorList.ErrorCode | Integer | Код ошибки, указывающий, что сообщение не было отправлено. Если код ошибки — 70107, аккаунт не существует. |
| MsgKey | String | Уникальный идентификатор сообщения. Это поле требуется для отозвания сообщения. Значение — это строка не более 50 символов. |
| MsgId | String | Уникальный идентификатор сообщения на клиенте. |

## Коды ошибок

Код состояния HTTP, возвращаемый для этого API, всегда 200, если не возникает ошибка сети (например, ошибка 502). Конкретный код ошибки и деталь можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, относящиеся к этому API:

| Код ошибки | Описание |
| --- | --- |
| 70107 | Запрашиваемый аккаунт не существует. |
| 70169 | Время ожидания сервера истекло. Повторите попытку позже. |
| 90001 | Не удалось разобрать JSON запрос. Убедитесь, что формат правильный. |
| 90002 | `MsgBody` в JSON запросе не соответствует требованиям формата сообщения или `MsgBody` не является массивом. Для получения дополнительной информации см. раздел **Элемент сообщения TIMMsgElement** в [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| 90004 | Неправильное поле `MsgSeq` в JSON запросе. |
| 90007 | Поле `MsgBody` в JSON запросе не является массивом. Измените его на массив. |
| 90008 | JSON запрос не содержит поле `From_Account` или аккаунт, указанный в `From_Account`, не существует. |
| 90009 | Запрос требует разрешения администратора приложения. |
| 90010 | JSON запрос не соответствует требованиям формата сообщения. Для получения дополнительной информации см. раздел **Элемент сообщения TIMMsgElement** в [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| 90011 | Количество целевых аккаунтов превышает 500. Удалите некоторые `To_Account`. |
| 90012 | Аккаунт, указанный в `To_Account`, не существует или не зарегистрирован. Убедитесь, что аккаунт был импортирован в Chat и правильный. |
| 90026 | Время хранения сообщений в автономном режиме неправильно. Сообщения не могут храниться в автономном режиме более 7 дней. |
| 90048 | Запрашиваемый аккаунт не существует. |
| 90992 | Ошибка внутреннего сервиса. Повторите попытку. Если этот код ошибки возвращается для всех запросов и webhook включен, убедитесь, что сервер приложения нормально возвращает результаты в сервер Chat. |
| 91000 | Ошибка внутреннего сервиса. Повторите попытку. |
| 93000 | JSON пакет превышает максимальный размер 12 КБ. |

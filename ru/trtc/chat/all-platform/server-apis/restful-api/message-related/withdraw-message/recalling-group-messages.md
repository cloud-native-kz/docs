# Отзыв групповых сообщений

## Обзор функции

- Этот API позволяет администраторам отзывать групповые сообщения.
- Этот API позволяет отзывать все групповые сообщения, действительные для роуминга, включая групповые сообщения, отправленные с клиента, и те, которые отправлены через RESTful API.
- Для отзыва групповых сообщений, отправленных с клиента, сначала включите [After a Group Message Is Sent](https://intl.cloud.tencent.com/document/product/1047/34375), который записывает `GroupId` и `MsgSeq` каждого группового сообщения, а затем введите `GroupId` и `MsgSeqList`, полученные из webhook, в этот API для отзыва сообщения. Кроме того, вы можете получить информацию о групповых сообщениях, которые необходимо отозвать, используя [Pulling Historical Messages](https://intl.cloud.tencent.com/document/product/1047/34971), а затем отозвать сообщение с помощью этого API.
- Для отзыва групповых сообщений, отправленных через RESTful API [Sending Ordinary Messages in a Group](https://intl.cloud.tencent.com/document/product/1047/34959), получите `MsgSeq`, содержащийся в пакетах ответов этих API, а затем отзовите сообщения.
- После отзыва сообщения этим API оно удаляется из локальных кэшей клиентов отправителя и получателей.
- Этот API может отзывать групповые сообщения, отправленные в любой период времени, но групповые сообщения должны быть действительны для роуминга.

## Описание вызова API

### Применимые типы групп

| ID типа группы | Отзыв группового сообщения |
| --- | --- |
| Private | Да. Аналогично рабочим группам (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. Аналогично группе собраний (Meeting) в новой версии. |
| AVChatRoom | Да. Поддерживается с Native SDK 7.4.4634 и Web SDK 3.1.2. |
| Community | Да |

Это предустановленные типы групп в Chat. Для получения дополнительной информации см. [Group System](https://intl.cloud.tencent.com/document/product/1047/33529).

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/group_msg_recall?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [RESTful API Overview](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, в котором находится ваш SDKAppID.Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Сеул: `adminapikr.im.qcloud.com`Токио: `adminapijpn.im.qcloud.com`Франкфурт: `adminapiger.im.qcloud.com`Кремниевая долина: `adminapiusa.im.qcloud.com`Джакарта: `adminapiidn.im.qcloud.com` |
| v4/group_open_http_svc/group_msg_recall | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Для получения дополнительной информации см. раздел **App Admin** в [Login Authentication](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, сгенерированная учетной записью администратора приложения. Для получения подробной информации см. [Generating UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса, который всегда должен быть `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

Операция отзыва сообщения необратима. Пожалуйста, вызывайте этот API с осторожностью.

```
{    "GroupId": "@TGS#2J4SZEAEL",    "MsgSeqList":[        {            "MsgSeq":100        },        {            "MsgSeq":101        }    ]}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы, чьи исторические сообщения будут изменены |
| MsgSeqList | Array | Да | Список последовательности сообщений, которые должны быть отозваны. Запрос может содержать до 10 последовательностей сообщений. |
| MsgSeq | Integer | Да | Последовательность сообщения, которое должно быть отозвано. |
| TopicId | String | Нет | ID темы, который указывает отзыв сообщения в соответствующей теме и применяется только к сообществам с поддержкой тем. |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0,    "RecallRetList":[        {            "MsgSeq":100,            "RetCode":10030        },        {            "MsgSeq":101,            "RetCode":0        }    ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: не удалось |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: не удалось |
| ErrorInfo | String | Информация об ошибке |
| RecallRetList | Array | Детали результатов запроса на отзыв сообщения. |
| MsgSeq | Integer | Последовательность отозванного сообщения. |
| RetCode | Integer | Результат отзыва отдельного сообщения. `0` — успех, остальное — ошибки. Подробнее см. коды ошибок ниже. |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда 200, если не возникает ошибка сети (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для публичных кодов ошибок (60000–79999) см. [Error Codes](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Попробуйте еще раз. |
| 10003 | Недопустимое слово команды. |
| 10004 | Недопустимый параметр. Проверьте описание ошибки и устраните проблему. |
| 10007 | Нет прав на операцию. Оператор должен иметь разрешения на выполнение соответствующих операций. |
| 10010 | Группа не существует или была удалена. |
| 10015 | Недопустимый ID группы. Используйте правильный ID группы. |
| 10030 | Запрашиваемое сообщение не существует. |
| 10031 | Запрашиваемое сообщение находится за пределами допустимого временного диапазона. |
| 10032 | Запрашиваемое сообщение не может быть отозвано. |
| 10033 | Этот тип группы не поддерживает отзыв сообщений. |
| 10041 | Приложение (SDKAppID) настроено на отсутствие поддержки отзыва групповых сообщений. |

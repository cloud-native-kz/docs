# Загрузка недавних сообщений

## Обзор функции

Администратор приложения может использовать этот API для получения адресов скачивания всех записей сообщений один-на-один или групповых сообщений в приложении, которые произошли в указанный момент времени за последние семь дней.

> **Примечание** Вы можете скачивать изображения, аудио, файлы и короткие видео из записей сообщений. Эта функция применима только к Chat SDK версии 4.X и более поздних. Загрузку можно выполнять на основе полей URL в записях чата. Если вы используете Chat SDK версии 2.X или 3.X, вы не можете получить указанную выше информацию этим методом. Если вам нужна эта функция, обновите ваш Chat SDK на версию 4.X или более позднюю. Записи сообщений хранятся в виде журналов и сжимаются с помощью GZip. После получения адресов скачивания через API вы можете загрузить и обработать записи сообщений самостоятельно. Файлы записей сообщений генерируются каждый час по пекинскому времени. Например, данные, созданные в полночь (00:00-00:59), будут обработаны с 01:00. Обычно данные обрабатываются в течение одного часа. Однако если количество сообщений велико, обработка займет больше времени. Файлы записей сообщений действительны только семь дней и будут удалены через семь дней независимо от статуса загрузки. Удаленные записи нельзя экспортировать повторно. Адрес скачивания, полученный через этот API, имеет срок действия. Пожалуйста, загрузите записи сообщений до истечения срока действия адреса скачивания. Если адрес скачивания становится недействительным, получите адрес скачивания снова через этот API. Этот API используется только для загрузки исторических записей чата за последние семь дней для резервного копирования, статистики или других целей. Мы не рекомендуем использовать его для реальных онлайн-бизнесов.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/open_msg_svc/get_history?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где расположен ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` |
| v4/open_msg_svc/get_history | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Для получения дополнительной информации см. раздел **Администратор приложения** в [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробную информацию см. в [Создание UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное беззнаковое целое число в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса. Значение фиксировано на `json`. |

### Максимальная частота вызовов

10 раз/сек

### Пример запроса

```
{    "ChatType": "C2C",    "MsgTime": "2015120121"}
```

### Поля запроса

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| ChatType | String | Обязательное | Тип сообщения. `C2C` указывает сообщение один-на-один, тогда как `Group` указывает групповое сообщение. |
| MsgTime | String | Обязательное | Период для загрузки записей сообщений. MsgTime использует пекинское время и не поддерживает другие часовые пояса. Например, 2015120121 указывает на то, что будут загружены сообщения за период с 21:00 до 21:59 1 декабря 2015 г. Это поле должно быть задано по часам. Каждый запрос может использоваться только для получения всех сообщений один-на-один или групповых сообщений, которые происходят в указанный час в конкретный день. |

### Пример ответа

```
{    "File": [        {            "URL": "https://download.tim.qq.com/msg_history/2/9b8f8f063b73f61698ce11e58207e89ade40.gz",            "ExpireTime": "2015-12-02 16:45:23",            "FileSize": 65207,            "FileMD5": "cceece008bb7f469a47cf8c4b7acb84e",            "GzipSize": 1815,            "GzipMD5": "c3a0269dde393fd7a8bb18bfdeaeee2e"        }    ],    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode":0}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса. OK: успешно. FAIL: ошибка. |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: ошибка |
| ErrorInfo | String | Информация об ошибке |
| File | Array | Информация о загрузке файла записи сообщения |
| URL | String | Адрес скачивания файла записи сообщения |
| ExpireTime | String | Время истечения срока действия адреса скачивания. Всегда загружайте файл до истечения срока действия адреса. Если адрес истек, получите новый адрес через API. |
| FileSize | Integer | Размер файла (в байтах) до сжатия GZip |
| FileMD5 | String | MD5 файла до сжатия GZip |
| GzipSize | Integer | Размер файла (в байтах) после сжатия GZip |
| GzipMD5 | String | MD5 файла после сжатия GZip |

## Коды ошибок

Возвращаемый этим API код состояния HTTP всегда равен 200, если не возникает ошибка сети (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000-79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 1001 | Неверный запрос. Проверьте, правильный ли URL запроса. |
| 1002 | Неверный параметр. Проверьте, является ли учетная запись администратором, указаны ли обязательные поля и соответствуют ли значения требованиям протокола. |
| 1003 | Системная ошибка. |
| 1004 | Файл еще не создан, или в запрашиваемый период не было доставлено сообщений. |
| 1005 | Срок действия файла истек. |

## Формат файлов записей сообщений

```
//For one-to-one messages{"SdkAppId":1104620500,"ChatType":"C2C","MsgTime":"2015120121","MsgList":[{"From_Account":"peakerdong","To_Account":"qiyueliuhuo2018","MsgTimestamp":1448974806,"MsgSeq":3452069198,"MsgRandom":45838,"MsgBody":[{"MsgType":"TIMTextElem","MsgContent":{"Text":"Quartering"}}]},{"From_Account":"group_root","To_Account":"group_test4","MsgTimestamp":1448974808,"MsgSeq":462709847,"MsgRandom":19196437,"MsgBody":[{"MsgType":"TIMTextElem","MsgContent":{"Text":"hi, beauty"}}]}]}//For group messages{"SdkAppId":1104620500,"ChatType":"Group","MsgTime":"2015120121","MsgList":[{"From_Account":"Test_1","GroupId":"@TGS#1FDFVPAE2","MsgTimestamp":1448975384,"MsgSeq":1,"MsgBody":[{"MsgType":"TIMTextElem","MsgContent":{"Text":"Private activate"}}]},{"From_Account":"Test_1","GroupId":"@TGS#1FDFVPAE2","MsgTimestamp":1448975384,"MsgSeq":1,"MsgBody":[{"MsgType":"TIMTextElem","MsgContent":{"Text":"Private activate"}}]}]}
```

Первая строка файла записывает основную информацию о файле. Каждая следующая строка записывает сообщение до последней строки, которая заканчивается на "]}". Для формата каждого сообщения см. определения в [TIMMsgElement Objects](https://intl.cloud.tencent.com/document/product/1047/33527).

- Если файл небольшой, вы можете использовать базу данных JSON для анализа всего файла. `MsgList` указывает массив сообщений за указанный период. Например:

```
# Python sample codeimport gzip, jsonwith gzip.open('1104620500_Group_2015120121.gz', 'rb') as fp:  info = json.load(fp)for msg in info['MsgList']:  pass #do sth with msg
```

- Если файл большой, мы рекомендуем анализировать его строка за строкой. Например:

```
# Python sample codeimport gzip, jsonwith gzip.open('1104620500_Group_2015120121.gz', 'rb') as fp:  cnt = -1  for line in fp:      line = line.strip().rstrip(b',')      if line == b']}': break      if cnt < 0:          info = json.loads(line + b']}')      else:          msg = json.loads(line)          #do sth with msg      cnt += 1
```

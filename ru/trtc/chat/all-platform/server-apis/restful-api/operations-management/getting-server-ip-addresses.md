# Получение IP-адресов серверов

## Обзор функции

Иногда может потребоваться получить список IP-адресов серверов в целях безопасности. Администратор приложения может использовать этот API для получения списка IP-адресов или диапазонов IP, используемых SDK и вебхуком.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/ConfigSvc/GetIPList?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json&nettype=0
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Имя домена, соответствующее стране/региону, где находится ваш SDKAppID.Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Сеул: `adminapikr.im.qcloud.com`Токио: `adminapijpn.im.qcloud.com`Франкфурт: `adminapiger.im.qcloud.com`Кремниевая долина: `adminapiusa.im.qcloud.com`Джакарта: `adminapiidn.im.qcloud.com` |
| v4/ConfigSvc/GetIPList | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Аккаунт администратора приложения. Для получения дополнительной информации см. раздел **Администратор приложения** в документе [Аутентификация при входе](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная аккаунтом администратора приложения. Для получения подробной информации см. [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса. Значение зафиксировано как `json`. |
| nettype | Запрос списка IP-адресов серверов указанного типа: nettype=0: Список всех IP-адресов сервераnettype=1: Список IP-адресов серверов, к которым обращается собственный SDK в Китае (включая Гонконг, Китай)nettype=2: Список IP-адресов серверов, к которым обращается собственный SDK вне Китаяnettype=3: Список IP-адресов серверов, к которым обращается веб-SDK в Китае (включая Гонконг, Китай)nettype=4: Список IP-адресов серверов, к которым обращается веб-SDK вне Китаяnettype=5: Список адресов выхода, используемых Chat для вебхуков в Китае (включая Гонконг, Китай)nettype=6: Список адресов выхода, используемых Chat для вебхуков вне Китая |

> **Примечание**Для бизнеса вебхуков сервер бизнеса должен добавить адреса выхода сервиса Chat в список разрешений. Вы можете установить `nettype` на 5 (или 6) в соответствии с регионом адреса бизнес-сервера, чтобы получить список адресов выхода сервиса Chat.

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

```
{}
```

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorCode": 0,    "IPList": [ "127.0.0.2",   "183.192.202.0/25" ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| IPList | Array | Список IP-адресов серверов |
| ErrorCode | Integer | Код ошибки. `0`: Успешно; другие значения: Ошибка |
| ErrorInfo | String | Информация об ошибке |

## Коды ошибок

HTTP-код состояния, возвращаемый этим API, всегда равен 200, если не происходит ошибка сети (например, ошибка 502). Конкретный код ошибки и подробности можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для публичных кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).

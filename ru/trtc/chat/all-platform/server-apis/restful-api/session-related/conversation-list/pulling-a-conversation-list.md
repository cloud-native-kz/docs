# Получение списка разговоров

## Обзор функции

Этот API используется для получения списка разговоров по страницам.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/recentcontact/get_list?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где расположен ваш SDKAppID.Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Сеул: `adminapikr.im.qcloud.com`Франкфурт: `adminapiger.im.qcloud.com`Токио: `adminapijpn.im.qcloud.com`Кремниевая долина: `adminapiusa.im.qcloud.com`Джакарта: `adminapiidn.im.qcloud.com` |
| v4/recentcontact/get_list | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Для получения дополнительной информации см. раздел **App Admin** в разделе [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Дополнительные сведения см. в разделе [Генерация UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса, который всегда должен быть `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

```
{  "From_Account": "id1",  "TimeStamp": 0,  "StartIndex": 0,  "TopTimeStamp": 0,  "TopStartIndex": 0,  "AssistFlags": 7}
```

### Поля запроса

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| From_Account | String | Да | `UserID` учетной записи, для которой нужно получить список разговоров |
| TimeStamp | Integer | Да | Время начала обычных разговоров. Введите `0` для первой страницы. |
| StartIndex | Integer | Да | Начальная точка обычных разговоров. Введите `0` для первой страницы. |
| TopTimeStamp | Integer | Да | Время начала закрепленных разговоров. Введите `0` для первой страницы. |
| TopStartIndex | Integer | Да | Начальная точка закрепленных разговоров. Введите `0` для первой страницы. |
| AssistFlags | Integer | Да | Флаги разговоров:Бит 0: поддержка ли закрепленных разговоровБит 1: возвращать ли пустой разговорБит 2: поддержка ли разбиения на страницы закрепленных разговоров |

### Пример ответа

```
{  "SessionItem": [    {      "Type": 1,      "To_Account": "id2",      "MsgTime": 1630997627,      "TopFlag": 1    },    {      "Type": 2,      "GroupId": "id3",      "MsgTime": 1630997628,      "TopFlag": 1    },    {      "Type": 1,      "To_Account": "id4",      "MsgTime": 1630997630,      "TopFlag": 0    },    {      "Type": 2,      "GroupId": "id5",      "MsgTime": 1630997650,      "TopFlag": 0    }  ],  "CompleteFlag": 1,  "TimeStamp": 1631012800,  "StartIndex": 0,  "TopTimeStamp": 1631012800,  "TopStartIndex": 0,  "ActionStatus": "OK",  "ErrorCode": 0,  "ErrorInfo": "",  "ErrorDisplay": ""}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| SessionItem | Array | Массив объектов разговоров |
| Type | Integer | Тип разговора. `1`: разговор один на один; `2`: групповой разговор |
| To_Account | String | `UserID` другого участника разговора, который будет возвращен только для разговора один на один |
| GroupId | String | ID группы, который будет возвращен только для группового разговора |
| MsgTime | Integer | Продолжительность разговора |
| TopFlag | Integer | Флаг закрепления разговора. `0`: обычный разговор; `1`: закрепленный разговор |
| CompleteFlag | Integer | Флаг завершения. `1`: все разговоры возвращены; `0`: получение еще не завершено. |
| TimeStamp | Integer | Время начала следующей извлеченной страницы для обычного разговора, которое отправляется в серверную часть Chat через поле `TimeStamp` запроса при получении по страницам |
| StartIndex | Integer | Начальная точка следующей извлеченной страницы для обычного разговора, которая отправляется в серверную часть Chat через поле `StartIndex` запроса при получении по страницам |
| TopTimeStamp | Integer | Время начала следующей извлеченной страницы для закрепленного разговора, которое отправляется в серверную часть Chat через поле `TopTimeStamp` запроса при получении по страницам |
| TopStartIndex | Integer | Начальная точка следующей извлеченной страницы для закрепленного разговора, которая отправляется в серверную часть Chat через поле `TopStartIndex` запроса при получении по страницам |
| ActionStatus | String | Результат запроса. `OK`: успешно. `FAIL`: ошибка. |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: ошибка. Подробнее о ненулевых результатах см. в разделе [Коды ошибок](#ErrorCode). |
| ErrorInfo | String | Подробное сообщение об ошибке |
| ErrorDisplay | String | Подробная информация, отображаемая на клиенте |

## Коды ошибок

| Код ошибки | Описание |
| --- | --- |
| 50001 | Запрошенный UserID не был импортирован в серверную часть Tencent Cloud Chat. Пожалуйста, импортируйте. |
| 50002 | Неправильный параметр запроса. Проверьте ваш запрос в соответствии с описанием ошибки. |
| 50003 | Запрос требует разрешений администратора приложения. |
| 50004 | Внутренняя ошибка сервера. Пожалуйста, попробуйте снова. |
| 50005 | Тайм-аут сети. Попробуйте позже. |

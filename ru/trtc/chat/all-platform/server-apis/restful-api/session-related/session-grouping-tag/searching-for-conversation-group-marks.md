# Поиск отметок групп разговоров

## Обзор функции

Этот API используется для запроса данных отметок групп разговоров для указанного разговора.

> **Примечание:** группы разговоров, сеанс может присоединиться к нескольким группам, максимум сеансов на группу составляет 1000. При превышении возвращается код ошибки: [51008](https://trtc.io/document/34348?product=chat&menulabel=uikit&platform=react). Теги сеансов и пользовательские поля имеют объединенный верхний предел в 1000 сеансов. При превышении возвращается код ошибки: [51008](https://trtc.io/document/34348?product=chat&menulabel=uikit&platform=react).

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/recentcontact/search_contact_group?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Другие параметры см. в [обзоре RESTful API](https://trtc.io/document/34620?product=chat&menulabel=serverapi).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/recentcontact/search_contact_group | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Дополнительные сведения см. в разделе **Администратор приложения** в [аутентификации входа](https://trtc.io/document/33517?platform=javascript&product=chat&menulabel=coresdk). |
| usersig | Подпись, созданная учетной записью администратора приложения. Дополнительные сведения см. в [генерировании UserSig](https://trtc.io/document/34385?product=chat&menulabel=serverapi). |
| random | 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса, который всегда должен быть `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

```
{    "From_Account":"yahaha",    "ContactItem":[        {            "Type":1,            "To_Account":"xixi"        }    ]}
```

### Поля запроса

| Поле | Тип | Требуется | Описание |
| --- | --- | --- | --- |
| From_Account | String | Да | Заполните UserID, запрос для поиска группы разговоров/отметки для этого пользователя. |
| ContactItem | Array | Да | Список объектов разговоров для поиска. За один раз можно выполнить поиск до 300 разговоров. |
| Type | Integer | Да | Тип разговора: 1 — один-на-один разговор. 2 — групповой разговор |
| ToGroupId | String | Нет | Присваивается только для группового разговора, заполните ID группы стороны разговора. |
| To_Account | String | Нет | Присваивается только для один-на-один разговора, заполните UserID стороны разговора. |

### Пример ответа

```
{    "ContactResultItem": [        {            "Type": 1,            "To_Account": "xixi",            "StandardMark": "11111110",            "CustomMark": "xxyyyzzzfffffffxxx111111",            "ContactGroupId": [                1,                2            ],            "Timestamp": 1673500546        }    ],    "GroupItem": [        {            "GroupName": "test1x",            "GroupId": 1        },        {            "GroupName": "test10",            "GroupId": 2        }    ],    "ActionStatus": "OK",    "ErrorCode": 0,    "ErrorInfo": "",    "ErrorDisplay": ""}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK` — успешно; `FAIL` — ошибка |
| ErrorCode | Integer | Код ошибки. `0` — успешно; другие значения — ошибка |
| ErrorInfo | String | Информация об ошибке |
| ContactItem | Integer | Объект разговора |
| Type | String | Тип разговора: 1 — один-на-один разговор. 2 — групповой разговор |
| ToGroupId | String | Присваивается только для группового разговора, заполните ID группы стороны разговора |
| To_Account | String | Присваивается только для один-на-один разговора, заполните UserID стороны разговора |
| Timestamp | Integer | Последний временной мestamp обновления группы разговоров |
| StandardMark | String | Бит возможности стандартной отметки, например `011111110` |
| CustomMark | String | Пользовательская отметка |
| ContactGroupId | Array | Группа, к которой принадлежит разговор |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда равен 200, если не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и подробности можно найти в полях ответа `ResultCode`, `ResultInfo`, `ErrorCode` и `ErrorInfo`.

Для общих кодов ошибок (60000–79999) см. [коды ошибок](https://trtc.io/document/34348?platform=javascript&product=chat&menulabel=coresdk).

В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 50001 | Запрашиваемый UserID не был импортирован в бэкэнд Tencent Cloud Chat. Пожалуйста, выполните импорт. |
| 50002 | Неверный параметр запроса. Проверьте ваш запрос в соответствии с описанием ошибки. |
| 50003 | Запрос требует прав администратора приложения. |
| 50004 | Внутренняя ошибка сервера. Пожалуйста, попробуйте еще раз. |
| 50005 | Истечение времени ожидания сети. Повторите попытку позже. |
| 51006 | При изменении отметки разговора количество разговоров пусто или превышает верхний предел в 100. |
| 51007 | Ошибка замены GroupID на GroupCode из-за внутренней ошибки или расформирования группы. |
| 51008 | Общее количество разговоров в группе разговоров превышает 1000. |
| 51009 | Группа разговоров не существует при попытке удаления. |
| 51010 | Количество групп разговоров превышает верхний предел в 20. |
| 51011 | Имя группы разговоров содержит более 32 байтов. |
| 51012 | Превышено максимальное количество разговоров, закрепленных в верхней части. |
| 51013 | Если запрошенное приложение не является изданием Pro, Pro Plus или Enterprise, при изменении стандартной отметки разговора или группы разговоров будет выведена ошибка. |

# Блокировка пользователей

## Описание API

Этот API используется для добавления одного или нескольких пользователей в список блокировки.

> **Примечание**: Если пользователи A и B являются друзьями, то добавление одного из них в список блокировки другого удаляет их из списков друзей друг друга.
> Если пользователь A заблокирует пользователя B или наоборот, то ни один из них не сможет отправить запрос на добавление в друзья другому пользователю. Если пользователь B находится в списке блокировки пользователя A, а пользователь A находится в списке блокировки пользователя B, то пользователи A и B не смогут начать беседу друг с другом.
> -> Если пользователь B находится в списке блокировки пользователя A, но пользователь A не находится в списке блокировки пользователя B, то пользователь A может отправлять сообщения пользователю B, но не наоборот.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/sns/black_list_add?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

Приведенный ниже список содержит только часто используемые параметры при вызове этого API и их описания. Дополнительные параметры см. в разделе [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| xxxxxx | Страна/регион, в котором находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/sns/black_list_add | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения. |
| identifier | Учетная запись администратора приложения. Дополнительные сведения см. в разделе [Администратор приложения](https://intl.cloud.tencent.com/document/product/1047/33517#app-.E7.AE.A1.E7.90.86.E5.91.98). |
| usersig | Подпись, созданная в учетной записи администратора приложения. Подробные сведения о создании подписи см. в разделе [Создание UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса. Значение всегда `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

```
{    "From_Account":"id",    "To_Account":["id1","id2","id3"]}
```

### Поля запроса

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| From_Account | String | Да | UserID, инициирующий запрос на блокировку. |
| To_Account | Array | Да | Список UserID, которые будут добавлены в список блокировки. Этот массив должен содержать не более 1000 UserID. |

### Пример ответа

```
{    "ResultItem":    [        {            "To_Account":"id1",            "ResultCode":0,            "ResultInfo":""        },        {            "To_Account":"id2",            "ResultCode":30001,            "ResultInfo":"Err_SNS_BlackListAdd_Already_Exist"        },        {            "To_Account":"id3",            "ResultCode":30002,            "ResultInfo":"Err_SNS_BlackListAdd_SdkAppId_Illegal"        }    ],    "Fail_Account":["id2","id3"],    "ActionStatus":"OK",    "ErrorCode":0,    "ErrorInfo":"",    "ErrorDisplay":""}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ResultItem | Array | Результат пакетной блокировки — это массив UserID и соответствующих результатов. |
| To_Account | String | UserID, который был запрошен для добавления в список блокировки. |
| ResultCode | Integer | Результат. `0` означает успех, другие значения означают ошибку. Подробные сведения о ненулевых результатах см. в разделе [Коды ошибок](#ErrorCode). |
| ResultInfo | String | Описание ошибки. Это поле пусто, когда запрос выполнен успешно. |
| Fail_Account | Array | Список пользователей, которые не смогли быть добавлены в список блокировки. Это поле возвращается только в том случае, если не удалось добавить хотя бы одного пользователя. |
| ActionStatus | String | Результат запроса. `OK` означает, что запрос успешно обработан. `FAIL` означает, что запрос не удался. |
| ErrorCode | Integer | Код ошибки. `0` означает успех, другие значения означают ошибку. Подробные сведения о ненулевых результатах см. в разделе [Коды ошибок](#ErrorCode). |
| ErrorInfo | String | Подробная информация об ошибке. |
| ErrorDisplay | String | Подробная информация, отображаемая на клиенте. |

## Коды ошибок

Если не происходит сетевая ошибка (такая как ошибка 502), то код состояния HTTP, возвращаемый этим API, всегда равен 200. Конкретный код ошибки и детали можно найти в полях ответа, таких как `ResultCode`, `ResultInfo`, `ErrorCode` и `ErrorInfo`.
Для общих кодов ошибок (60000–79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
Приведенный ниже список содержит только коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 30001 | Неправильный параметр запроса. Проверьте запрос согласно описанию ошибки. |
| 30002 | SDKAppID не совпадает. |
| 30003 | Запрошенная учетная запись пользователя не существует. |
| 30004 | Запрос требует разрешений администратора приложения. |
| 30006 | Внутренняя ошибка сервера. Пожалуйста, попробуйте снова. |
| 30007 | Истекло время ожидания запроса. Пожалуйста, попробуйте позже. |
| 30008 | Произошел конфликт записи из-за одновременных операций записи. Рекомендуется использовать пакетную обработку. |
| 30013 | Достигнут максимальный лимит заблокированных пользователей. |

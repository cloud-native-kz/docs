# Запрос статуса онлайн-активности учетной записи

## Описание функции

Этот API используется для запроса текущего статуса входа пользователя.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/openim/query_online_status?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

Ниже приведен список параметров, обычно используемых при вызове этого API, и их описания. Дополнительные параметры см. в [обзоре RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| xxxxxx | Страна/регион, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/openim/query_online_status | API запроса, которое используется. |
| sdkappid | SDKAppID, назначенный через консоль Chat при создании приложения. |
| identifier | Значение должно быть учетной записью администратора приложения. Для получения дополнительной информации см. [Администратор приложения](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Дополнительные сведения об операции см. в [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Значение должно быть случайным 32-битным целым числом без знака. Диапазон значений: от 0 до 4294967295. |
| contenttype | Формат запроса. Значение всегда `json`. |

### Максимальная частота вызовов

Этот API может быть вызван до 200 раз в секунду.

### Примеры пакетов запросов

#### Когда подробная информация о платформе входа не требуется

```
{    "To_Account":["id1","id2","id3","id4"],}
```

#### Когда требуется подробная информация о платформе входа

```
{    "IsNeedDetail": 1,    "To_Account": ["id1", "id2", "id4"]}
```

### Поля пакета запроса

| Поле | Тип | Свойство | Описание |
| --- | --- | --- | --- |
| To_Account | Array | Обязательное | Один или несколько UserID, статусы входа которых требуется запросить. Этот API можно использовать для запроса статусов входа до 500 UserID за один раз. |
| IsNeedDetail | Integer | Необязательное | Указывает, требуется ли подробная информация о платформе входа в ответе. 0: не требуется. 1: требуется. |

### Примеры тела пакета ответа

#### Когда подробная информация о платформе входа не требуется

```
{    "ActionStatus":"OK",    "ErrorInfo":"",    "ErrorCode": 0    "QueryResult": [        {            "To_Account": "id1",            "State": "Offline"        },        {            "To_Account": "id2",            "State": "Online"        },        {            "To_Account": "id3",            "State": "PushOnline"        }    ],    "ErrorList": [        {            "To_Account": "id4",            "ErrorCode": 70107        }    ]    }
```

#### Когда требуется подробная информация о платформе входа

```
{    "ActionStatus":"OK",    "ErrorInfo":"",    "ErrorCode": 0    "QueryResult": [        {            "To_Account": "id1",            "State": "Online",            "Detail": [                {                    "Platform": "IPhone",                    "Status": "PushOnline",                    "IsBackground": 1,                    "Instid": 74224656,                    "CustomIdentifier": "device-1"                },                {                    "Platform": "Web",                    "Status": "Online",                    "IsBackground": 1,                    "Instid": 74224656,                    "CustomIdentifier": "device-2"                }            ]        },        {            "To_Account": "id2",            "State": "Offline",        }            ],    "ErrorList": [        {            "To_Account": "id4",            "ErrorCode": 70107        }    ]      }
```

#### Ошибка запроса

```
{    "ActionStatus": "FAIL",    "ErrorInfo": "Fail to Parse json data of body, Please check it",    "ErrorCode": 90001}
```

### Поля пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса. OK: успешно. FAIL: ошибка. |
| ErrorInfo | String | Подробная информация об ошибке. |
| ErrorCode | Integer | Код ошибки, возвращаемый для запроса. Если запрос статуса для любой учетной записи успешен, значение равно 0. Если запрос для всех учетных записей не удался, возвращаемое значение отлично от 0. |
| QueryResult | Array | Возвращаемая структурированная информация о статусе входа пользователя. |
| QueryResult.To_Account | String | UserID пользователя, чей статус возвращается. |
| QueryResult.Status | String | Возвращаемый статус входа. Допустимые значения: Online: после входа пользователя в клиент клиент остается в постоянном соединении с серверной частью Chat. PushOnline: клиент переходит в состояние PushOnline, когда процесс iOS или Android отключается из-за ошибки сети или завершается операционной системой. В этом состоянии клиент все еще может получать автономные сообщения. Однако если процесс клиента не завершена операционной системой после переключения клиента в фоновый режим, клиент находится в состоянии Online. Offline: пользователь правильно вышел из клиента или не входил в клиент в течение как минимум 7 дней с момента последнего входа. Если пользователь входит в клиент на нескольких устройствах, значение Online предполагается, при условии, что клиент находится в состоянии Online на любом устройстве. |
| QueryResult.Detail | Object | Подробная информация о платформе входа. |
| QueryResult.Detail.Platform | String | Тип платформы входа. Допустимые значения: "iPhone", "Android", "Web", "PC", "iPad" и "Mac". |
| QueryResult.Detail.Instid | Integer | Уникальный идентификатор этого устройства. |
| QueryResult.Detail.CustomIdentifier | String | Пользовательский идентификатор этого устройства. |
| ErrorList | Array | Список учетных записей, для которых не удалось запросить статусы. Целевые учетные записи в этом списке не найдены или не удалось запросить их статусы. Если запрос статуса для всех учетных записей выполнен успешно, значение поля ErrorList пусто. |
| ErrorList.To_Account | String | Целевая учетная запись, для которой не удалось запросить статус. |
| ErrorList.ErrorCode | Integer | Код ошибки, указывающий на ошибку запроса статуса. Если код ошибки для целевой учетной записи равен 70107, учетная запись не найдена. |

> **Внимание** Серверная часть Chat сохраняет состояние PushOnline только на протяжении 7 дней. Если пользователь не входил в клиент в течение 7 дней с момента предыдущего входа, пользователь переходит в состояние Offline.

## Коды ошибок

Код возврата HTTP для этого API равен 200, если не происходит ошибка сети, такая как ошибка 502. Фактический код ошибки и информация об ошибке указаны в полях ErrorCode и ErrorInfo соответственно в теле пакета ответа.
Для общих кодов ошибок от 60000 до 79999 см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
Частные коды ошибок для этого API приведены ниже:

| Код ошибки | Описание |
| --- | --- |
| 70107 | Запрашиваемый UserID не существует. |
| 70169 | Истекло время ожидания сервера. Повторите попытку позже. |
| 90001 | Ошибка анализа формата JSON. Проверьте, соответствует ли пакет запроса спецификации JSON или является ли To_Account пустым массивом. |
| 90003 | Значение поля To_Account в пакете запроса формата JSON не соответствует требованиям формата сообщения. Проверьте, является ли тип поля To_Account String. |
| 90009 | Запрос требует прав администратора приложения. |
| 90011 | Количество целевых учетных записей, которым должно быть отправлено сообщение, превышает 500. Уменьшите количество целевых учетных записей в To_Account. |
| 90992 | Истекло время ожидания серверной части. Повторите попытку. |
| 90994 | Произошла ошибка внутреннего сервиса. Повторите попытку. |
| 90995 | Произошла ошибка внутреннего сервиса. Повторите попытку. |
| 91000 | Произошла ошибка внутреннего сервиса. Повторите попытку. |

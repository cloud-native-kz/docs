# Обратный вызов после отправки личного сообщения чатботу

## Описание функции

Бэкенд приложения может выполнять операции в реальном времени над личными сообщениями, используя этот обратный вызов, включая:

- Выполнение логирования личных сообщений в реальном времени (например, запись логов или синхронизацию сообщений с другими системами).
- Выполнение статистики данных по личным сообщениям (например, статистика по количеству участников или сообщений).

## Обязательные знания

- Для включения обратного вызова необходимо настроить URL обратного вызова и включить переключатель, соответствующий этому обратному вызову. Методику настройки см. в разделе [Конфигурация Webhook](https://www.tencentcloud.com/document/product/1047/34520#).
- Направление обратного вызова таково: бэкенд Chat отправляет HTTPS POST запросы на бэкенд приложения.
- После получения запроса обратного вызова бэкенд приложения должен проверить, совпадает ли значение параметра SDKAppID в URL запроса с его собственным SDKAppID.
- Если одновременно включены оба обратных вызова — перед отправкой личного сообщения и после отправки личного сообщения — и обратный вызов перед отправкой личного сообщения возвращает код ошибки, указывающий на запрет говорить, обратный вызов после отправки личного сообщения не будет срабатывать.
- Если одновременно включены оба обратных вызова — перед отправкой личного сообщения и после отправки личного сообщения — и текст сообщения изменен для обратного вызова перед отправкой личного сообщения, измененное сообщение будет использовано для срабатывания обратного вызова после отправки личного сообщения.
- По другим вопросам безопасности см. раздел "Рассмотрение безопасности" в [Обзор Webhook](https://www.tencentcloud.com/document/product/1047/34354#security-considerations).

## Сценарии срабатывания обратного вызова

- Пользователь приложения отправляет личное сообщение через клиент, и получателем является чатбот.
- Администратор приложения отправляет личное сообщение через REST API (sendmsg), и получателем является чатбот.

## Время срабатывания обратного вызова

Обратный вызов срабатывает после того, как бэкенд Chat получает личное сообщение, отправленное пользователем, и отправляет сообщение целевому пользователю. Целевой пользователь определяется как чатбот.

## Описание API

### Пример URL запроса

В следующем примере URL обратного вызова, настроенный для приложения, — это `https://www.example.com`.

**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL обратного вызова. |
| SdkAppid | SDKAppID, назначенный приложению при его создании в консоли Chat. |
| CallbackCommand | Значение зафиксировано как Bot.OnC2CMessage. |
| contenttype | Значение зафиксировано как json. |
| ClientIP | IP-адрес клиента. Например, 127.0.0.1. |
| OptPlatform | Платформа клиента. Допустимые значения см. в описании параметра OptPlatform в разделе "Протокол Webhook" [Обзор Webhook](https://www.tencentcloud.com/document/product/1047/34354#webhook-protocol). |

### Пример пакета запроса

```
{    "CallbackCommand": "Bot.OnC2CMessage", // Команда обратного вызова.    "From_Account": "jared", // Отправитель.    "To_Account": "Jonh", // Получатель.    "MsgSeq": 48374, // Порядковый номер сообщения.    "MsgRandom": 2837546, // Случайный номер сообщения.    "MsgTime": 1557481126, // Временная метка отправки сообщения, в секундах.     "MsgKey": "48374_2837546_1557481126", // Уникальный идентификатор сообщения, который можно использовать для отзыва личного сообщения путем вызова REST API.    "OnlineOnlyFlag":1,//Флаг, указывающий, следует ли отправлять сообщение только онлайн-пользователям. 1: отправлять только онлайн-пользователям; 0: отправлять всем пользователям.    "SendMsgResult": 0, // Результат отправки сообщения.    "ErrorInfo": "send msg succeed", // Сообщение об ошибке, возвращаемое при ошибке отправки сообщения. Если сообщение успешно отправлено, возвращается "send msg succeed".    "MsgBody": [ // Текст сообщения.        {            "MsgType": "TIMTextElem", // Текст.            "MsgContent": {                "Text": "red packet"            }        }    ],    "CloudCustomData": "your cloud custom data",    "EventTime": 1670574414123 // Временная метка срабатывания события, в миллисекундах.}
```

### Поля пакета запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда обратного вызова. |
| From_Account | String | Идентификатор пользователя отправителя сообщения. |
| To_Account | String | Идентификатор пользователя получателя сообщения. |
| MsgSeq | Integer | Порядковый номер сообщения (32-битное беззнаковое целое число), используется для идентификации сообщения. |
| MsgRandom | Integer | Случайный номер сообщения (32-битное беззнаковое целое число), используется для идентификации сообщения. |
| MsgTime | Integer | Временная метка отправки сообщения, в секундах. MsgTime в приоритетном порядке используется для сортировки личных сообщений. Для сообщений, отправленных в одну и ту же секунду, используется MsgSeq для сортировки. Сообщение отправляется более недавно, если его значение MsgSeq больше. |
| MsgKey | String | Уникальный идентификатор сообщения, который можно использовать для [отзыва личного сообщения путем вызова REST API](https://www.tencentcloud.com/document/product/1047/35015#). |
| OnlineOnlyFlag | Integer | Флаг, указывающий, следует ли отправлять сообщение только онлайн-пользователям. 1: отправлять только онлайн-пользователям; 0: отправлять всем пользователям. |
| SendMsgResult | Integer | Результат отправки сообщения. 0: отправка успешна; другие значения: отправка не удалась. Подробнее см. в разделе [Коды ошибок](https://www.tencentcloud.com/document/product/1047/34348#message-error-codes2). |
| ErrorInfo | String | Сообщение об ошибке, возвращаемое при ошибке отправки сообщения. Если сообщение успешно отправлено, возвращается "send msg succeed". |
| MsgBody | Array | Текст сообщения. Подробнее см. в разделе [Форматы сообщений](https://www.tencentcloud.com/document/product/1047/33527#). |
| CloudCustomData | String | Пользовательские данные сообщения. (Сохраняются в облаке и отправляются получателю. Могут быть получены даже после удаления и переустановки Chat.) |
| EventTime | Integer | Временная метка срабатывания события, в миллисекундах. |

### Пример пакета ответа

Пакет ответа возвращается после синхронизации данных бэкендом приложения.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // 0: обратный вызов успешен; 1: ошибка обратного вызова.}
```

### Поля пакета ответа

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательно | Результат обработки запроса: OK: обработка успешна. FAIL: обработка не удалась. |
| ErrorCode | Integer | Обязательно | Код ошибки: 0: обратный вызов успешен. 1: ошибка обратного вызова. |
| ErrorInfo | String | Обязательно | Сообщение об ошибке. |

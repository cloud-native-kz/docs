# После выхода пользователя из группы

## Обзор функции

Этот обратный вызов позволяет просматривать статус выхода пользователей из группы в бэкэнде приложения. Вы можете записывать информацию о выходе пользователей из группы в реальном времени, например, путем создания журнала или синхронизации информации с другой системой.

## Примечания

- Чтобы включить этот обратный вызов, необходимо настроить URL обратного вызова и переключить соответствующий протокол. Дополнительные сведения о методе настройки см. в разделе [Конфигурация обратного вызова](https://intl.cloud.tencent.com/document/product/1047/34520).
- Во время этого обратного вызова бэкэнд Chat инициирует HTTP POST запрос к бэкэнду приложения.
- После получения запроса обратного вызова бэкэнд приложения должен проверить, является ли `SDKAppID`, содержащийся в URL запроса, значением `SDKAppID` приложения.
- Дополнительные соображения безопасности см. в разделе **Security Considerations** в документе [Обзор сторонних обратных вызовов](https://intl.cloud.tencent.com/document/product/1047/34354).

## Сценарии инициирования обратного вызова

- Пользователь приложения выходит из группы на клиенте.
- Пользователь приложения удаляет участника группы из группы на клиенте.
- Администратор приложения удаляет участника группы через RESTful API.

## Время инициирования обратного вызова

Этот обратный вызов будет инициирован после того, как пользователь выходит из группы или удаляется из группы владельцем/администратором группы.

## Описание вызова API

### Пример URL запроса

В следующем примере настроенный в приложении URL обратного вызова — это `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL обратного вызова |
| SdkAppid | `SDKAppID`, назначенный консолью Chat при создании приложения |
| CallbackCommand | Фиксированное значение: `Group.CallbackAfterMemberExit`. |
| contenttype | Фиксированное значение: `JSON`. |
| ClientIP | IP-адрес клиента, например 127.0.0.1 |
| OptPlatform | Платформа клиента. Допустимые значения см. в описании `OptPlatform` в разделе **Callback Protocols** документа [Обзор сторонних обратных вызовов](https://intl.cloud.tencent.com/document/product/1047/34354). |

### Пример запроса

```
{    "CallbackCommand": "Group.CallbackAfterMemberExit", // Команда обратного вызова    "GroupId": "@TGS#2J4SZEAEL", // ID группы    "Type": "Public", // Тип группы    "ExitType": "Kicked", // Режим выхода из группы. Допустимые значения: `Kicked` (участник удален из группы); `Quit` (участник выходит из группы).    "Operator_Account": "leckie", // Оператор    "ExitMemberList": [ // Список участников, вышедших из группы        {            "Member_Account": "jared"        },        {            "Member_Account": "tommy"        }    ],    "EventTime":"1670574414123"// Временная метка инициирования события в миллисекундах		}
```

### Поля запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда обратного вызова |
| GroupId | String | ID группы, в которой генерируются сообщения группы |
| Type | String | Тип группы, в которой генерируются сообщения группы, например `Public`. Подробности см. в разделе **Group Types** документа [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529). |
| ExitType | String | Режим выхода из группы. Допустимые значения: `Kicked` (участник удален из группы); `Quit` (участник выходит из группы). |
| Operator_Account | String | `UserID` пользователя, выходящего из группы |
| ExitMemberList | Array | Список участников, выходящих из группы |
| EventTime | Integer | Временная метка инициирования события в миллисекундах |

### Пример ответа

Ответ возвращается после синхронизации данных бэкэндом приложения.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // Игнорировать результат ответа}
```

### Поля ответа

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Да | Результат запроса. `OK`: Успешно; `FAIL`: Неудачно |
| ErrorCode | Integer | Да | Код ошибки. Значение `0` указывает на игнорирование результата ответа. |
| ErrorInfo | String | Да | Информация об ошибке |

# До отправки группового сообщения

## Обзор функции

Это событие вебхука используется серверной частью приложения для проверки групповых сообщений пользователей в реальном времени, включая:

- Запись групповых сообщений в реальном времени, например ведение журнала или синхронизация сообщений с другими системами.
- Блокировка запросов пользователей на отправку сообщений в группу.

> **Примечание:** Период ожидания вебхука перед отправкой сообщения по умолчанию составляет две секунды, и мы рекомендуем вам не изменять его. Этот вебхук может превысить время ожидания при использовании для модерации текста. Если вам необходимо модерировать текст, используйте вебхук контента.

## Ограничения

- Чтобы включить этот вебхук, необходимо настроить URL вебхука и включить соответствующий переключатель для этого вебхука. Дополнительные сведения о методе конфигурации см. в разделе [Конфигурация вебхука](https://intl.cloud.tencent.com/document/product/1047/34520).
- Во время этого события вебхука серверная часть Chat инициирует HTTP POST-запрос к серверной части приложения.
- После получения запроса вебхука серверная часть приложения должна проверить, является ли `SDKAppID`, содержащийся в URL запроса, `SDKAppID` приложения.
- Дополнительные соображения безопасности см. в разделе **Соображения безопасности** в [Обзор вебхук-обратных вызовов](https://intl.cloud.tencent.com/document/product/1047/34354).

## Сценарии срабатывания вебхука

- Пользователь приложения отправляет групповое сообщение с клиента.
- Администратор приложения отправляет групповое сообщение через API RESTful.

## Время срабатывания вебхука

Вебхук срабатывает перед тем, как серверная часть Chat отправляет групповое сообщение членам группы.

## Описание вызова API

### Пример URL запроса

В следующем примере URL вебхука, настроенный в приложении, это `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL вебхука |
| SdkAppid | `SDKAppID`, назначенный консолью Chat при создании приложения |
| CallbackCommand | Фиксированное значение: `Group.CallbackBeforeSendMsg`. |
| contenttype | Фиксированное значение: `json`. |
| ClientIP | IP-адрес клиента, например 127.0.0.1 |
| OptPlatform | Платформа клиента. Допустимые значения см. в описании `OptPlatform` в разделе **Протоколы вебхука** в [Обзор вебхук-обратных вызовов](https://intl.cloud.tencent.com/document/product/1047/34354). |

### Примеры запросов

```
{    "CallbackCommand": "Group.CallbackBeforeSendMsg", // Команда вебхука    "GroupId": "@TGS#2J4SZEAEL", // ID группы    "Type": "Community", // Тип группы    "From_Account": "jared", // Отправитель    "Operator_Account":"admin", // Инициатор запроса    "Random": 123456, // Случайное число    "MsgId": "144115233406643804-1727580296-4026038328", // Уникальный идентификатор сообщения на клиенте    "OnlineOnlyFlag": 1, // Значение равно `1` если это онлайн-сообщение и `0` (по умолчанию) если нет. Для аудиовизуальных групп значение равно `0`.    "MsgBody": [ // Тело сообщения. Дополнительные сведения см. в объекте сообщения `TIMMessage`.        {            "MsgType": "TIMTextElem", // Текст            "MsgContent":{                "Text": "red packet"            }        }    ],    "CloudCustomData": "your cloud custom data",    "TopicId":"@TGS#_@TGS#cQVLVHIM62CJ@TOPIC#_TestTopic",// ID темы, применяется только для сообществ с включенной поддержкой тем    "EventTime":"1670574414123"// Временная метка срабатывания события в миллисекундах		}
```

### Поля запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда вебхука. |
| GroupId | String | ID группы, которая генерирует групповые сообщения. |
| Type | String | Тип группы, которая генерирует групповые сообщения, например `Public`. Подробные сведения см. в разделе **Типы групп** в [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529). |
| From_Account | String | `UserID` отправителя сообщения. |
| Operator_Account | String | `UserID` инициатора запроса, на основе которого система может определить, инициирован ли запрос администратором. |
| Random | Integer | 32-битовое случайное число в запросе. |
| MsgId | String | Уникальный идентификатор сообщения на клиенте. |
| OnlineOnlyFlag | Integer | Значение равно `1` если это онлайн-сообщение и `0` (по умолчанию) если нет. Для аудиовизуальных групп значение равно `0`. |
| MsgBody | Array | Тело сообщения. Дополнительные сведения см. в разделе [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| CloudCustomData | String | Пользовательские данные сообщения. Они сохраняются в облаке и будут отправлены получателю. Такие данные можно получить после удаления и переустановки приложения. |
| TopicId | String | ID темы, который указывает на отправку сообщения в теме и применяется только для сообществ с включенной поддержкой тем. |
| EventTime | Integer | Временная метка срабатывания события в миллисекундах. |

### Примеры ответов

#### Разрешение отправки группового сообщения

Пользователю разрешено отправлять групповые сообщения, и содержание сообщения не изменяется.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // `0` указывает, что пользователю разрешено отправлять сообщения.}
```

#### Запрещение отправки группового сообщения

Пользователю не разрешено отправлять групповые сообщения. В этом случае сообщение не отправляется, и вызывающей стороне возвращается код ошибки `10016`.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 1 // `1` указывает, что пользователю не разрешено отправлять сообщения.}
```

#### Молчаливое отбрасывание сообщений

Пользователю не разрешено отправлять групповые сообщения. В этом случае сообщение не отправляется, но вызывающей стороне будет возвращено сообщение об успехе.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 2 // Значение `2` указывает, что сообщение молча отбрасывается.}
```

#### Изменение содержания сообщения

В следующем примере ответа групповое сообщение, отправленное пользователем, изменяется (добавляется пользовательское сообщение), и серверная часть Chat отправляет измененное сообщение. С помощью этой функции серверная часть приложения может добавить специальное содержание, такое как уровень и звание пользователя, к сообщению, отправленному пользователем.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0, // Это поле должно быть установлено в `0`, чтобы измененное сообщение было доставлено нормально.    "MsgBody": [ // Сообщение, измененное серверной частью приложения. Если серверная часть приложения не изменяет сообщение, сообщение, отправленное пользователем, доставляется.        {            "MsgType": "TIMTextElem", // Текст            "MsgContent":{                "Text": "red packet"            }        },        {            "MsgType": "TIMCustomElem", // Пользовательское сообщение            "MsgContent":{                "Desc": "CustomElement.MemberLevel", // Описание                "Data": "LV1" // Данные            }        }    ],    "CloudCustomData": "your cloud custom data"}
```

### Поля ответа

| Поле | Тип | Требуется | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Да | Результат запроса. `OK`: успешно; `FAIL`: не удалось. |
| ErrorCode | Integer | Да | Возвращаемый код ошибки. `0`: разрешить отправку группового сообщения; `1`: запретить отправку группового сообщения; `2`: молча отбросить сообщение. Если бизнес-сторона хочет запретить пользователю отправлять групповые сообщения и отправить `ErrorCode` и `ErrorInfo` клиенту, убедитесь, что значение `ErrorCode` находится в диапазоне [10100, 10200]. |
| ErrorInfo | String | Да | Информация об ошибке. |
| MsgBody | Array | Нет | Тело сообщения, измененное серверной частью приложения. Серверная часть Chat отправляет измененное сообщение в группу. Дополнительные сведения о формате см. в разделе [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| CloudCustomData | String | Нет | Пользовательские данные сообщения. Они сохраняются в облаке и будут отправлены другой стороне. Такие данные можно получить после удаления и переустановки приложения. |

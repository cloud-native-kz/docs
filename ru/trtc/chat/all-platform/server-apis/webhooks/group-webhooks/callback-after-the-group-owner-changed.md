# Обратный вызов после изменения владельца группы

## Обзор функции

Backend приложения может использовать этот обратный вызов для просмотра информации об изменении владельца группы в реальном времени. Backend приложения может выполнять операции, такие как синхронизация данных, на основе этого обратного вызова.

## Примечания

- Для включения обратного вызова необходимо настроить URL обратного вызова и включить соответствующий переключатель протокола. Подробные методы конфигурации см. в документе [Конфигурация обратных вызовов третьих сторон](https://www.tencentcloud.com/document/product/1047/34520).
- При этом обратном вызове backend Chat инициирует HTTP POST запрос на backend приложения.
- После получения запроса обратного вызова backend приложения должен проверить, соответствует ли SDKAppID, содержащийся в URL запроса, его собственному SDKAppID.
- По другим вопросам безопасности обратитесь к документу [Обзор Webhook - Соображения безопасности](https://www.tencentcloud.com/document/product/1047/34354#.E5.AE.89.E5.85.A8.E8.80.83.E8.99.91).

## Сценарии, которые могут запустить этот обратный вызов

- Пользователь приложения успешно передает владельца группы через клиент.
- Администратор приложения успешно передает владельца группы через REST API.

## Время срабатывания обратного вызова

После успешной передачи владельца группы.

## Описание API

### Пример URL запроса

В последующем примере URL обратного вызова, настроенный в приложении, — это `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST |
| www.example.com | URL обратного вызова |
| SdkAppid | SDKAppID, выделенный консолью Instant Messaging при создании приложения |
| CallbackCommand | Фиксированное значение: Group.CallbackAfterChangeGroupOwner |
| contenttype | Фиксированное значение: JSON |
| ClientIP | IP адрес клиента, например: 127.0.0.1 |
| OptPlatform | Платформа клиента, см. [Обзор Webhook - Протокол обратного вызова](https://www.tencentcloud.com/document/product/1047/34354#.E5.9B.9E.E8.B0.83.E5.8D.8F.E8.AE.AE) для понимания значения параметра OptPlatform |

### Пример пакета запроса

```
{    "CallbackCommand": "Group.CallbackAfterChangeGroupOwner",    // Обратный вызов после изменения владельца группы    "GroupId": "@TGS#2TTV7VSII", // ID группы    "Type": "Public", // Тип группы       "Operator_Account": "admin", // UserID пользователя, выполняющего операцию       "OldOwner_Account": "user1", // Исходный владелец группы       "NewOwner_Account": "user2", // Новый владелец группы       "EventTime":"1670574414123"// Временная метка срабатывания события в миллисекундах}
```

### Описание полей пакета запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда обратного вызова |
| GroupId | String | ID действующей группы |
| Type | String | Тип группы [Введение в типы групп](https://www.tencentcloud.com/document/product/1047/33529#GroupType), например Public |
| Operator_Account | String | UserID пользователя, выполняющего операцию |
| OldOwner_Account | String | UserID исходного владельца группы |
| NewOwner_Account | String | UserID нового владельца группы |
| EventTime | Integer | Временная метка срабатывания события в миллисекундах |

### Пример пакета ответа

После синхронизации данных backend приложения отправляет пакет ответа обратного вызова.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // Игнорировать результат обратного вызова}
```

### Описание полей пакета ответа

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательно | Результат обработки запроса: OK: Указывает на успешную обработку; FAIL: Указывает на ошибку |
| ErrorCode | Integer | Обязательно | Код ошибки, введение 0 здесь означает игнорирование результата ответа |
| ErrorInfo | String | Обязательно | Сообщение об ошибке |

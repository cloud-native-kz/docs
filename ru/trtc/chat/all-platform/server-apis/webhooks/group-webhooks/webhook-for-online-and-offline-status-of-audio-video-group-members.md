# Вебхук для отслеживания статуса онлайн и офлайн членов аудио-видео группы

## Обзор функции

Этот вебхук позволяет просматривать статус онлайн и офлайн (например, логи записей) членов аудио-видео группы в реальном времени или синхронизировать информацию с другими системами на бэкенде приложения.

## Примечания

- Для использования этого вебхука необходимо приобрести [Pro edition, Pro Plus edition или Enterprise edition](https://www.tencentcloud.com/document/product/1047/34577) и включить функцию **List of online audio-video group members** на странице **Group feature configuration** в консоли. После включения функции в консоли будет сохраняться список последних 1000 онлайн членов аудио-видео группы, который можно загружать на клиентах. Если функция отключена в консоли, список невозможно загружать на клиентах, и можно загружать только список последних 30 членов группы.
- Для включения этого вебхука необходимо настроить URL вебхука и переключить соответствующий протокол. Дополнительную информацию о методе настройки см. в разделе [Webhook Configuration](https://intl.cloud.tencent.com/document/product/1047/34520).
- Во время этого события вебхука бэкенд Chat инициирует HTTP POST запрос на бэкенд приложения.
- После получения запроса вебхука бэкенд приложения должен проверить, является ли `SDKAppID`, содержащийся в URL запроса, `SDKAppID` приложения.
- Дополнительную информацию о соображениях безопасности см. в разделе **Security Considerations** в [Webhook Overview.](https://www.tencentcloud.com/document/product/1047/34520#b25a53ea-c02c-447f-9acb-7ae9b4c21b35)

## Сценарии срабатывания вебхука

- Пользователь приложения аномально выходит после входа в аудио-видео группу. Система выполняет внутреннюю проверку каждые 20 секунд. Если она обнаруживает, что пользователь не отправлял сердцебиение более 30 секунд, это запускает обратный вызов офлайна.
- Пользователь приложения аномально выходит после входа в аудио-видео группу. После срабатывания обратного вызова офлайна из-за отсутствия сердцебиения, последующий обратный вызов онлайна инициируется при получении сердцебиения.
- Пользователь приложения впервые присоединяется к аудио-видео группе. Когда пользователь присоединяется к одной и той же аудио-видео группе с нескольких терминалов приложения, включая веб-терминал, обратный вызов онлайна срабатывает только один раз.
- Пользователь приложения последний раз активно выходит из аудио-видео группы. Когда пользователь имеет несколько терминалов приложения в аудио-видео группе, включая веб-терминал, обратный вызов офлайна срабатывает при выходе последнего терминала.

## Время срабатывания вебхука

После входа пользователя в аудио-видео группу событие `Offline` срабатывает из-за истечения сердцебиения, вызванного разрывом сетевого соединения, и событие `Online` срабатывает, когда пользователь переходит в онлайн через 20 секунд после истечения. Если пользователь входит с несколькими устройствами и одновременно присоединяется к одной и той же аудио-видео группе, событие `Offline` будет срабатывать, когда все устройства одновременно переходят в офлайн. Событие `Online` будет срабатывать, когда любое устройство переходит обратно в онлайн (только одно событие `Online` будет срабатывать даже если несколько устройств переходят обратно в онлайн).

## Описание вызова API

### Пример URL запроса

В следующем примере URL вебхука, настроенный в приложении, это `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL обратного вызова |
| SdkAppid | `SDKAppID`, назначенный консолью Chat при создании приложения |
| CallbackCommand | Фиксированное значение: `Group.CallbackOnMemberStateChange`. |
| contenttype | Фиксированное значение: `JSON`. |

### Пример запроса

```
{    "CallbackCommand": "Group.CallbackOnMemberStateChange", // Команда вебхука    "GroupId": "@TGS#2J4SZEAEL", // ID группы    "EventType": "Offline", // Тип события: Offline (отключено); Online (переподключено)    //Причина события: HeartbeatInterrupt (прерывание сердцебиения), HeartbeatRecover (восстановление сердцебиения), Join (первый вход в группу), Quit (последний выход из группы)    "MemberList": [ // Список членов, покинувших группу        {            "Member_Account": "jared"        },        {            "Member_Account": "tommy"        }    ],    "EventTime":"1670574414123"// Временная метка срабатывания события в миллисекундах	}
```

### Поля запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда вебхука |
| GroupId | String | ID группы, генерирующей групповые сообщения |
| EventType | String | Тип события: Offline (отключено); Online (переподключено) |
| EventCause | String | Причина события: HeartbeatInterrupt (прерывание сердцебиения), HeartbeatRecover (восстановление сердцебиения), Join (первый вход в группу), Quit (последний выход из группы) |
| MemberList | Array | Список членов, срабатывающих событие |

### Пример ответа

Ответ возвращается после синхронизации данных бэкендом приложения.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // Значение `0` указывает, что результат ответа игнорируется.}
```

### Поля ответа

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Да | Результат запроса. `OK`: Успешно; `FAIL`: Неудачно |
| ErrorCode | Integer | Да | Код ошибки. Значение `0` указывает, что результат ответа игнорируется. |
| ErrorInfo | String | Да | Информация об ошибке |

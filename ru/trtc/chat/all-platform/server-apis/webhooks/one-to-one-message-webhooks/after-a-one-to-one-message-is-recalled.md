# После отзыва личного сообщения

## Описание функции

Этот API используется серверной частью приложения для просмотра отзывов личных сообщений в реальном времени.

## Примечания

- Для включения этого обратного вызова необходимо настроить URL обратного вызова и включить соответствующий переключатель для этого обратного вызова. Дополнительную информацию о методе конфигурации см. в разделе [Конфигурация обратного вызова](https://intl.cloud.tencent.com/document/product/1047/34520).
- Во время этого обратного вызова серверная часть Chat инициирует HTTPS POST-запрос к серверной части приложения.
- После получения запроса обратного вызова серверная часть приложения должна проверить, что `SDKAppID`, содержащийся в URL запроса, соответствует `SDKAppID` приложения.
- Для получения дополнительной информации о соображениях безопасности см. раздел **Соображения безопасности** в [Обзор сторонних обратных вызовов](https://intl.cloud.tencent.com/document/product/1047/34354).

## Сценарии срабатывания обратного вызова

- Пользователь приложения отзывает личное сообщение на клиенте.
- Администратор приложения отзывает личное сообщение путем вызова API [admin_msgwithdraw](https://intl.cloud.tencent.com/document/product/1047/35015).

## Время срабатывания обратного вызова

После успешного отзыва личного сообщения

## Описание вызова API

### Пример URL запроса

В следующем примере URL обратного вызова, настроенный в приложении, имеет значение `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL обратного вызова |
| SdkAppid | `SDKAppID`, назначенный консолью Chat при создании приложения |
| CallbackCommand | Всегда `C2C.CallbackAfterMsgWithDraw` |
| contenttype | Всегда `json` |
| ClientIP | IP-адрес клиента, например 127.0.0.1 |
| OptPlatform | Платформа клиента. Допустимые значения см. в описании `OptPlatform` в разделе **Протоколы обратного вызова** в [Обзор сторонних обратных вызовов](https://intl.cloud.tencent.com/document/product/1047/34354). |

### Пример запроса

```
{  "CallbackCommand": "C2C.CallbackAfterMsgWithDraw", // Команда обратного вызова  "From_Account": "jared", // Отправитель  "To_Account": "Jonh", // Получатель  "MsgKey": "48374_2837546_1557481126", // Уникальный идентификатор сообщения  "UnreadMsgNum": 7 // Общее количество непрочитанных личных сообщений `To_Account`}
```

### Поля запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда обратного вызова |
| From_Account | String | `UserID` отправителя сообщения |
| To_Account | String | `UserID` получателя сообщения |
| MsgKey | String | Уникальный идентификатор сообщения |
| UnreadMsgNum | Integer | Общее количество непрочитанных личных сообщений `To_Account` (включая все личные беседы) |

### Пример ответа

```
{  "ActionStatus": "OK",  "ErrorInfo": "",  "ErrorCode": 0 // `0`: обратный вызов выполнен успешно; `1`: произошла ошибка при обратном вызове.}
```

### Поля ответа

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Да | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Да | Код ошибки. `0`: обратный вызов выполнен успешно; `1`: произошла ошибка при обратном вызове. |
| ErrorInfo | String | Да | Информация об ошибке |

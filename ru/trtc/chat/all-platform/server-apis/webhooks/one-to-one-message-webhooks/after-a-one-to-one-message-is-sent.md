# После отправки личного сообщения

## Описание функции

Этот API используется бэкендом приложения для обработки личных сообщений пользователей в режиме реального времени, включая:

- Запись личных сообщений в режиме реального времени, например, путем регистрации журнала или синхронизации сообщений с другими системами.
- Сбор статистики по личным сообщениям, например, по количеству пользователей или количеству сообщений.

## Примечания

- Чтобы включить этот callback, необходимо настроить URL callback и включить соответствующий переключатель для этого callback. Дополнительные сведения о методе конфигурации см. в [Конфигурация Callback](https://intl.cloud.tencent.com/document/product/1047/34520).
- Во время этого callback бэкенд Chat инициирует HTTPS POST запрос на бэкенд приложения.
- После получения запроса callback бэкенд приложения должен проверить, что `SDKAppID`, содержащийся в URL запроса, соответствует `SDKAppID` приложения.
- Если включены оба callback (до отправки личного сообщения и после отправки личного сообщения) и callback перед отправкой личного сообщения возвращает запрет на отправку сообщения, то callback после отправки личного сообщения не может быть срабатывает.
- Если включены оба callback (до отправки личного сообщения и после отправки личного сообщения) и тело сообщения изменяется во время callback перед отправкой личного сообщения, то callback после отправки личного сообщения будет использовать модифицированное сообщение для callback.
- Дополнительные сведения о безопасности см. в разделе **Security Considerations** (Рекомендации по безопасности) в [Обзор сторонних Callback](https://intl.cloud.tencent.com/document/product/1047/34354).

## Сценарии срабатывания Callback

- Пользователь приложения отправляет личное сообщение с клиента.
- Администратор приложения отправляет личное сообщение, вызывая RESTful API `sendmsg`.

## Время срабатывания Callback

Callback срабатывает после того, как бэкенд Chat получает личное сообщение, отправленное пользователем, и отправляет сообщение целевому пользователю.

> **Примечание:** Если бэкенд Chat не может отправить сообщение, например, из-за фильтрации запрещенных слов, callback все равно срабатывает.

## Описание вызова API

### Пример URL запроса

В следующем примере URL callback, настроенный в приложении, равен `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL Callback |
| SdkAppid | `SDKAppID`, назначенный консолью Chat при создании приложения |
| CallbackCommand | Значение всегда `C2C.CallbackAfterSendMsg`. |
| contenttype | Значение всегда `JSON`. |
| ClientIP | IP-адрес клиента, например `127.0.0.1` |
| OptPlatform | Платформа клиента. Дополнительные сведения о допустимых значениях см. в описании `OptPlatform` в разделе **Callback Protocols** (Протоколы Callback) в [Обзор сторонних Callback](https://intl.cloud.tencent.com/document/product/1047/34354). |

### Примеры запросов

```
{    "CallbackCommand": "C2C.CallbackAfterSendMsg", // Команда callback    "From_Account": "jared", // Отправитель    "To_Account": "Jonh", // Получатель    "MsgSeq": 48374, // Порядковый номер сообщения    "MsgRandom": 2837546, // Случайное число сообщения    "MsgTime": 1557481126, // Временная метка в секундах, указывающая время отправки сообщения     "MsgKey": "48374_2837546_1557481126", // Уникальный идентификатор сообщения. Может использоваться для отзыва сообщения через вызов RESTful API.    "OnlineOnlyFlag":1, // Значение `1`, если это онлайн-сообщение, и `0`, если нет    "SendMsgResult": 0, // Результат отправки сообщения    "ErrorInfo": "send msg succeed", // Информация об ошибке, связанная с ошибкой отправки сообщения. Если сообщение отправлено успешно, значение этого поля равно `send msg succeed`.    "UnreadMsgNum": 7, // Общее количество непрочитанных личных сообщений `To_Account`    "MsgBody": [ // Тело сообщения        {            "MsgType": "TIMTextElem", // Текст            "MsgContent": {                "Text": "красный пакет"            }        }    ],    "CloudCustomData": "ваши облачные пользовательские данные"}
```

### Поля запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда callback |
| From_Account | String | `UserID` отправителя сообщения |
| To_Account | String | `UserID` получателя сообщения |
| MsgSeq | Integer | Порядковый номер сообщения. Используется для идентификации сообщения, значение представляет собой случайное 32-битное целое число без знака. |
| MsgRandom | Integer | Случайное число сообщения. Используется для идентификации сообщения, значение представляет собой случайное 32-битное целое число без знака. |
| MsgTime | Integer | Временная метка в секундах, указывающая время отправки сообщения. Личные сообщения в приоритетном порядке сортируются по `MsgTime`. Сообщения, отправленные в одну секунду, сортируются по `MsgSeq`. Сообщения с большими значениями `MsgSeq` идут после сообщений с меньшими значениями `MsgSeq`. |
| MsgKey | String | Уникальный идентификатор сообщения. Может использоваться для [отзыва сообщения](https://intl.cloud.tencent.com/document/product/1047/35015) через вызов RESTful API. |
| MsgId | String | Уникальный идентификатор сообщения на клиенте. |
| OnlineOnlyFlag | Integer | Значение `1`, если это означает только для онлайн пользователей, и `0`, если нет. |
| SendMsgResult | Integer | Результат отправки сообщения. `0`: успешно; другие значения: ошибка. Дополнительные сведения см. в [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348). |
| ErrorInfo | String | Информация об ошибке, связанная с ошибкой отправки сообщения. Если сообщение отправлено успешно, значение этого поля равно `send msg succeed`. |
| UnreadMsgNum | Integer | Общее количество непрочитанных личных сообщений `To_Account` (включая все личные беседы). Если сообщение не удается отправить, например, из-за фильтрации запрещенных слов, значение этого поля равно `-1`. |
| MsgBody | Array | Тело сообщения. Дополнительные сведения см. в [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| CloudCustomData | String | Пользовательские данные сообщения. Они сохраняются в облаке и будут отправлены на другую сторону. Такие данные можно получить после удаления и переустановки приложения. |

### Пример ответа

```
{    "ActionStatus":"OK",    "ErrorInfo": "",    "ErrorCode": 0 // `0`: callback успешен; `1`: во время callback произошла ошибка.}
```

### Поля ответа

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Да | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Да | Код ошибки. `0`: callback успешен; `1`: во время callback произошла ошибка. |
| ErrorInfo | String | Да | Информация об ошибке |

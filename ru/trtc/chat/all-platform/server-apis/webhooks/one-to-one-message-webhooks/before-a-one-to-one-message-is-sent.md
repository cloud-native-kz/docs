# Перед отправкой личного сообщения

## Обзор

Этот webhook используется бэкендом приложения для управления личными сообщениями пользователей в реальном времени, включая:

- Запись личных сообщений, отправляемых в реальном времени, например через логирование или синхронизацию сообщений с другими системами.
- Блокирование запросов пользователей на отправку несоответствующих сообщений любого типа, таких как текстовые, графические и пользовательские сообщения.

> **Примечание:** период ожидания webhook перед отправкой сообщения по умолчанию составляет две секунды, и мы рекомендуем вам не изменять это значение. Этот webhook может истечь при использовании для модерации текста.

## Ограничения

- Для включения этого webhook необходимо настроить URL webhook и включить соответствующий переключатель для этого webhook. Для получения дополнительной информации о методе конфигурации см. [Конфигурация Webhook](https://intl.cloud.tencent.com/document/product/1047/34520).
- Во время этого события webhook бэкенд Chat инициирует HTTPS POST запрос к бэкенду приложения.
- После получения запроса webhook бэкенд приложения должен проверить, совпадает ли SDKAppID, содержащийся в URL запроса, с SDKAppID приложения.
- Если webhook перед отправкой личных сообщений и webhook после отправки личных сообщений оба включены и для webhook перед отправкой личных сообщений возвращается запрет на отправку сообщения, webhook после отправки личных сообщений не может быть запущен.
- Если webhook перед отправкой личных сообщений и webhook после отправки личных сообщений оба включены и тело сообщения изменяется во время webhook перед отправкой личных сообщений, webhook после отправки личных сообщений будет использовать измененное сообщение для webhook.
- Для получения дополнительной информации о соображениях безопасности см. раздел **Security Considerations** в [Обзоре Webhook](https://intl.cloud.tencent.com/document/product/1047/34354).

## Сценарии запуска Webhook

- Пользователь приложения отправляет личное сообщение на клиенте.
- Администратор приложения отправляет личное сообщение путем вызова RESTful API `sendmsg`.

## Время запуска Webhook

Бэкенд Chat получил личное сообщение, отправленное пользователем, но еще не доставил сообщение целевому пользователю.

## Описание вызова API

### Пример URL запроса

В следующем примере webhook URL, настроенный в приложении, — это `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| [www.example.com](http://www.example.com) | URL обратного вызова |
| SdkAppid | `SDKAppID`, назначенный консолью Chat при создании приложения |
| CallbackCommand | Значение всегда `C2C.CallbackBeforeSendMsg`. |
| contenttype | Всегда `json` |
| ClientIP | IP адрес клиента, например 127.0.0.1 |
| OptPlatform | Платформа клиента. Для допустимых значений см. описание `OptPlatform` в разделе **Webhook Protocols** в [Обзоре Webhook](https://intl.cloud.tencent.com/document/product/1047/34354). |

### Пример запросов

```
{    "CallbackCommand": "C2C.CallbackBeforeSendMsg", // Команда webhook    "From_Account": "jared", // Отправитель    "To_Account": "John", // Получатель    "MsgSeq": 48374, // Порядковый номер сообщения    "MsgRandom": 2837546, // Случайное число сообщения    "MsgTime": 1557481126, // Временная метка в секундах, указывающая, когда отправляется сообщение     "MsgKey": "48374_2837546_1557481126", // Уникальный идентификатор сообщения. Его можно использовать для отзыва сообщения через вызов RESTful API.    "OnlineOnlyFlag":1, // Значение `1`, если это онлайн сообщение, и `0`, если это не так    "MsgBody": [ // Тело сообщения. Для получения дополнительной информации см. объект сообщения `TIMMessage`.        {            "MsgType": "TIMTextElem", // Текст            "MsgContent":{                "Text": "красный пакет"            }        }    ],    "CloudCustomData": "ваши облачные пользовательские данные"}
```

### Поля запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда webhook |
| From_Account | String | `UserID` отправителя сообщения |
| To_Account | String | `UserID` получателя сообщения |
| MsgSeq | Integer | Порядковый номер сообщения. Используется для идентификации сообщения, значение — случайное 32-битное целое число без знака. |
| MsgRandom | Integer | Случайное число сообщения. Используется для идентификации сообщения, значение — случайное 32-битное целое число без знака. |
| MsgTime | Integer | Временная метка в секундах, указывающая, когда отправляется сообщение. Личные сообщения сортируются предпочтительно по `MsgTime`. Сообщения, отправленные в одну секунду, сортируются по `MsgSeq`. Сообщения с большими значениями `MsgSeq` идут после сообщений с меньшими значениями. |
| MsgKey | String | Уникальный идентификатор сообщения. Его можно использовать для [отзыва сообщения](https://intl.cloud.tencent.com/document/product/1047/35015) через вызов RESTful API. |
| OnlineOnlyFlag | Integer | Значение `1`, если это онлайн сообщение, и `0`, если это не так. |
| MsgBody | Array | Тело сообщения. Для получения дополнительной информации см. [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| CloudCustomData | String | Пользовательские данные сообщения. Они сохраняются в облаке и будут отправлены получателю. Такие данные можно извлечь после удаления и переустановки приложения. |

### Пример ответа, когда разрешена отправка сообщений

Пользователю разрешено отправлять групповые сообщения, и содержимое сообщения не изменяется.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // `0` означает, что пользователю разрешено отправлять сообщения.}
```

### Пример ответа, когда отправка сообщений запрещена

Пользователю не разрешено отправлять групповые сообщения. В этом случае сообщение не отправляется, и пользователю (отправителю сообщения) возвращается код ошибки `20006`.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 1 // `1` означает, что пользователю не разрешено отправлять сообщения.}
```

### Пример ответа, когда сообщения отбрасываются молча

Пользователю не разрешено отправлять групповые сообщения. В этом случае сообщение не отправляется, но отправителю будет возвращено сообщение об успехе.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 2 // Значение `2` означает, что сообщение отбрасывается молча.}
```

### Пример ответа, когда содержимое сообщения изменено

В следующем примере сообщение, отправленное пользователем, изменяется (добавляется пользовательское сообщение или пользовательские данные сообщения), и бэкенд Chat отправит измененное сообщение. С помощью этой функции бэкенд приложения может добавить специальное содержимое, такое как уровень пользователя и название, к сообщению, отправленному пользователем.
**Пример:**

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0, // Это поле должно быть установлено в `0`, чтобы измененное сообщение могло быть отправлено нормально.    "MsgBody": [ // Сообщение, измененное бэкендом приложения. Если бэкенд приложения не изменяет сообщение, отправляется сообщение, отправленное пользователем.        {            "MsgType": "TIMTextElem", // Текст            "MsgContent":{                "Text": "красный пакет"            }        },        {            "MsgType": "TIMCustomElem", // Пользовательское сообщение            "MsgContent":{                "Desc": " CustomElement.MemberLevel ", // Описание                "Data": " LV1" // Данные            }        }    ],    "CloudCustomData": "ваши новые облачные пользовательские данные" // Пользовательские данные сообщения}
```

### Поля ответа

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Да | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Да | Возвращаемый код ошибки. `0`: разрешить отправку сообщения; `1`: запретить отправку сообщения; `2`: молча отбросить сообщение. Если деловая сторона хочет запретить пользователю отправлять сообщения и отправить `ErrorCode` и `ErrorInfo` клиенту, убедитесь, что значение `ErrorCode` находится в диапазоне [120001, 130000]. |
| ErrorInfo | String | Да | Информация об ошибке |
| MsgBody | Array | Нет | Тело сообщения, измененное бэкендом приложения. Бэкенд Chat отправляет измененное сообщение получателю. Для получения дополнительной информации о формате см. [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| CloudCustomData | String | Нет | Пользовательские данные сообщения, измененные бэкендом приложения. Они сохраняются в облаке и будут отправлены второй стороне. Такие данные можно извлечь после удаления и переустановки приложения. Бэкенд Chat отправляет измененное сообщение получателю. |

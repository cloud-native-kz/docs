# Webhook-уведомления об изменении статуса

## Обзор

Серверная часть приложения может использовать этот callback для просмотра в реальном времени поведения пользователей при входе или выходе, включая:

- Вход пользователя (устанавливается TCP-соединение)
- Выход пользователя или отключение сети (TCP-соединение разрывается)
- Timeout heartbeat приложения (приложение аномально закрыто или произошел сбой)
- Установка пользовательского статуса

## Примечания

- Чтобы включить этот callback, необходимо настроить URL callback и включить соответствующий переключатель для этого callback. Дополнительные сведения о методе конфигурации см. в разделе [Callback Configuration](https://intl.cloud.tencent.com/document/product/1047/34520).
- Во время этого callback серверная часть Chat инициирует HTTP POST запрос к серверной части приложения.
- После получения запроса callback серверная часть приложения должна проверить, является ли SDKAppID, содержащийся в URL запроса, SDKAppID приложения.
- Для получения дополнительной информации об аспектах безопасности см. раздел **Security Considerations** в документе [Third-Party Callback Overview](https://intl.cloud.tencent.com/document/product/1047/34354).
- Для встроенного SDK улучшенной версии 5.6.1200 или более поздней и веб-SDK 2.14.0 или более поздней принудительный выход из-за входа с нескольких устройств или входа с нескольких экземпляров вызовет только callback Login (Register). Для других версий SDK принудительный выход из-за входа с нескольких клиентов или входа с нескольких экземпляров вызовет как callback Login (Register), так и callback Logout (Unregister).

## Сценарии запуска Callback

- Пользователь инициирует запрос на вход через клиента.
- Пользователь инициирует запрос на выход через клиента.
- Клиент пользователя отключается, а затем снова подключается.
- Пользователь проактивно закрывает процесс клиента, приложение закрывается операционной системой мобильного телефона после того, как пользователь переводит приложение в фоновый режим, или процесс завершается аномально из-за сбоя приложения. При обнаружении отключения клиента от сети экземпляр CVM запускает callback отключения сети.
- Timeout heartbeat клиента, например, из-за отключения сети или полной недоступности сети. При обнаружении timeout heartbeat клиента экземпляр CVM запускает callback отключения сети. Интервал timeout heartbeat составляет 400 секунд.
- Установка пользовательского статуса пользователем

## Callback в реальном времени

### Android, iOS и PC

В большинстве случаев экземпляр CVM Chat может обнаружить изменение статуса пользователя и запустить callback в реальном времени. Например:

- Когда пользователь проактивно входит, экземпляр CVM Chat запускает callback Login (Register).
- Когда пользователь проактивно выходит, экземпляр CVM Chat запускает callback Logout (Unregister).
- Когда пользователь проактивно закрывает процесс клиента или переводит его в фоновый режим, либо процесс клиента закрывается операционной системой мобильного телефона, экземпляр CVM Chat запускает callback Disconnect (LinkClose).

Есть особый случай, когда серверу Chat требуется подождать до 400 секунд для обнаружения изменения статуса пользователя:

Когда сеть полностью недоступна и клиент не может даже отправить TCP пакеты FIN или RST через TCP, если сервер Chat обнаружит отключение TCP в течение 400 секунд, он запустит callback Disconnect (LinkClose). Если сервер Chat не обнаружит отключение TCP в течение 400 секунд, он проверит timeout heartbeat клиента и запустит callback Disconnect (TimeOut). Это обычно происходит, когда пользователь отключает клиента от сети (например, включив режим полета на мобильном телефоне) или пользователь входит в туннель без сигнала сети.

Кроме того, если клиент переподключится или снова войдет до получения callback Disconnect, сервер Chat обновит статус клиента на новое соединение и больше не запустит callback Disconnect для старого соединения.

### Web

Когда пользователь проактивно входит на веб-клиент, экземпляр CVM Chat запускает callback Login (Register) в реальном времени.

Своевременность callback изменения статуса в различных сценариях выхода/отключения выглядит следующим образом:

- Прямое закрытие страницы запускает callback Disconnect (LinkClose) в реальном времени.
- Отключение сети без закрытия текущей страницы требует около 60 секунд для запуска callback Disconnect (LinkClose).
- Проактивный вызов API `destroy` запускает callback Logout (Unregister) в реальном времени.

### Mini Program

Когда пользователь входит в Mini Program, экземпляр CVM Chat запускает callback Login (Register) в реальном времени.

Своевременность изменения статуса в различных сценариях выхода/отключения выглядит следующим образом:

- Когда пользователь нажимает в верхнем правом углу для выхода, callback Disconnect (LinkClose) запускается через пять секунд.
- Отключение сети (например, включение режима полета на телефоне) требует около 60 секунд для запуска callback Disconnect (LinkClose).
- Переключение WeChat в фоновый режим требует около 30 секунд для запуска callback Disconnect (LinkClose).
- Завершение процесса WeChat запускает callback Disconnect (LinkClose) в реальном времени.
- Проактивный вызов API `destroy` запускает callback Logout (Unregister) в реальном времени.

## Описание API

### Пример URL запроса

В следующем примере URL callback, настроенный в приложении, — это `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL callback |
| SdkAppid | SDKAppID, назначенный консолью Chat при создании приложения |
| CallbackCommand | Фиксированное значение: State.StateChange |
| contenttype | Фиксированное значение: JSON. |
| ClientIP | IP-адрес клиента, например 127.0.0.1 |
| OptPlatform | Платформа клиента. Допустимые значения см. в описании OptPlatform в разделе **Callback Protocols** документа [Third-Party Callback Overview](https://intl.cloud.tencent.com/document/product/1047/34354). |

### Пример запроса

```
{    "CallbackCommand": "State.StateChange",    "EventTime": 1629883332497,    "Info": {        "Action": "Login",        "To_Account": "testuser316",        "Reason": "Register"    },    "KickedDevice": [        {            "Platform": "Windows"        },        {            "Platform": "Android"        }    ]}
```

### Поля запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда callback |
| Info | Object | Информация о входе или выходе пользователя |
| Action | String | Поведение пользователя при входе или выходе. Допустимые значения: Login (установлено TCP-соединение); Logout (TCP-соединение разорвано); Disconnect (сеть отключена) |
| To_Account | String | UserID пользователя |
| Reason | String | Причина запуска входа или выхода пользователя:Причина входа: Register, указывает, что TCP-соединение установлено с приложением или что сеть была отключена, а затем подключена снова.Причина выхода: Unregister, указывает, что пользователь приложения отменяет регистрацию аккаунта и разрывает TCP-соединение.Причина отключения: LinkClose, указывает, что Chat обнаружил, что TCP-соединение с приложением разорвано, например, когда приложение закрыто или клиент отправляет TCP пакет FIN или RST. TimeOut: Chat обнаружил, что heartbeat пакет приложения timeout и определил, что TCP-соединение разорвано. Например, когда сеть клиента аномально отключена, клиент не отправляет TCP пакет FIN или RST и не может отправлять heartbeat пакеты. Интервал timeout heartbeat составляет 400 секунд.Причины CustomStatusChange: SetCustomStatus, что пользователь установил пользовательский статусДля callback причин в конкретных сценариях см. [Callback Trigger Scenarios](https://intl.cloud.tencent.com/document/product/1047/34357). |
| CustomStatus | String | Пользовательский статус после изменения пользователем |
| KickedDevice | Array | Информация о других устройствах, которые отключены. Это поле доступно только когда текущее изменение статуса — это Login (Register) и другие устройства отключены. |
| KickedDevice.Platform | String | Тип платформы отключенного устройства. Допустимые значения: iOS, Android, Web, Windows, iPad, Mac, Linux |
| EventTime | Integer | Временная метка при запуске текущего callback в миллисекундах. |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorCode": 0,    "ErrorInfo": ""}
```

### Поля ответа

| Поле | Тип | Требуется | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Да | Результат запроса. OK: успешно; FAIL: ошибка |
| ErrorCode | Integer | Да | Код ошибки. 0: обработка серверной частью приложения прошла успешно; 1: обработка серверной частью приложения не удалась. |
| ErrorInfo | String | Да | Информация об ошибке |

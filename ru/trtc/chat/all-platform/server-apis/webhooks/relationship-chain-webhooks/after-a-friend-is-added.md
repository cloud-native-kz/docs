# После добавления друга

## Описание функции

Бэкэнд приложения использует этот обратный вызов для просмотра информации о друзьях, добавленных пользователями в режиме реального времени.

## Примечания

- Чтобы включить этот обратный вызов, необходимо настроить URL обратного вызова и включить соответствующий протокол. Подробнее о методе настройки см. в разделе [Конфигурация обратных вызовов третьих лиц](https://intl.cloud.tencent.com/document/product/1047/34520).
- Направление обратного вызова: бэкэнд Chat инициирует HTTP POST запрос на бэкэнд приложения.
- После получения запроса обратного вызова бэкэнд приложения должен проверить, совпадает ли SDKAppID, содержащийся в URL запроса, с собственным SDKAppID.
- По другим вопросам безопасности см. [Обзор обратных вызовов третьих лиц: Рассмотрение вопросов безопасности](https://intl.cloud.tencent.com/document/product/1047/34354#.E5.AE.89.E5.85.A8.E8.80.83.E8.99.91).

## Сценарии срабатывания обратного вызова

- Бэкэнд приложения использует RESTful API для инициирования двусторонней заявки на добавление в друзья, а режим проверки заявки друга равного узла — "AllowAny".
- Бэкэнд приложения использует клиента для инициирования двусторонней заявки на добавление в друзья, а режим проверки друга равного узла — "AllowAny".
- Бэкэнд приложения использует RESTful API для инициирования односторонней заявки на добавление в друзья.
- Бэкэнд приложения использует клиента для инициирования односторонней заявки на добавление в друзья.
- После получения заявки на добавление в друзья пользователь приложения разрешает добавить равного узла в качестве друга.
- Бэкэнд приложения использует RESTful API для принудительного добавления друга.

## Время срабатывания обратного вызова

Этот обратный вызов срабатывает при успешном добавлении друга.

> **Внимание:** Этот обратный вызов не будет срабатывать, если API для [импорта друзей](https://intl.cloud.tencent.com/document/product/1047/34903) используется для добавления друга.

## Описание API

### Пример URL запроса

В следующем примере URL обратного вызова, настроенный в приложении, — это `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL обратного вызова. |
| SdkAppid | SDKAppID, назначенный консолью Chat при создании приложения. |
| CallbackCommand | Значение зафиксировано как Sns.CallbackFriendAdd. |
| contenttype | Значение зафиксировано как JSON. |
| ClientIP | IP-адрес клиента, формат аналогичен: 127.0.0.1. |
| OptPlatform | Платформа клиента. Подробнее о возможных значениях см. параметр OptPlatform в разделе [Обзор обратных вызовов третьих лиц: Протоколы обратных вызовов](https://intl.cloud.tencent.com/document/product/1047/34354#.E5.9B.9E.E8.B0.83.E5.8D.8F.E8.AE.AE). |

### Пример пакета запроса

```
{    "CallbackCommand": "Sns.CallbackFriendAdd",    "PairList": [        {            "From_Account": "id",            "To_Account": "id1",            "Initiator_Account": "id"        },        {            "From_Account": "id",            "To_Account": "id2",            "Initiator_Account": "id"        },        {            "From_Account": "id",            "To_Account": "id3",            "Initiator_Account": "id"        }    ],    "ClientCmd":"friend_add",    "Admin_Account":"",    "ForceFlag":1}
```

### Поля пакета запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда обратного вызова. |
| PairList | Array | Пара друзей, которые были успешно добавлены. |
| From_Account | String | From_Account добавляет To_Account в список друзей. |
| To_Account | String | To_Account добавляется в список друзей From_Account. |
| Initiator_Account | String | UserID пользователя, который инициирует заявку на добавление в друзья. |
| ClientCmd | String | Ключевое слово команды, которое срабатывает обратный вызов: для заявки на добавление в друзья допустимые значения — friend_add и FriendAdd. Для ответа на добавление в друзья допустимые значения — friend_response и FriendResponse. |
| Admin_Account | String | Если текущий запрос является заявкой на добавление в друзья, инициированной бэкэндом, это поле устанавливается на учетную запись администратора. В противном случае это поле пусто. |
| ForceFlag | Integer | Флаг для принудительного добавления друга администратором. 1: друг добавлен принудительно. 0: друг добавлен обычным способом. |

### Пример пакета ответа

```
{    "ActionStatus": "OK",    "ErrorCode": 0,    "ErrorInfo": ""}
```

### Поля пакета ответа

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательное | Результат обработки запроса. OK: успешно. FAIL: не удалось. |
| ErrorCode | Integer | Обязательное | Код ошибки. 0 указывает на успешную обработку бэкэндом приложения, 1 указывает на неудачную обработку бэкэндом приложения. |
| ErrorInfo | String | Обязательное | Информация об ошибке. |

# После добавления пользователя в черный список

## Описание функции

Бэкэнд приложения использует этот обратный вызов для получения информации о пользователях, добавленных в черный список в реальном времени.

## Меры предосторожности

- Для включения этого обратного вызова необходимо настроить URL обратного вызова и включить соответствующий протокол. Дополнительные сведения о методе конфигурации см. в разделе [Конфигурация обратных вызовов третьих сторон](https://intl.cloud.tencent.com/document/product/1047/34520).
- Направление обратного вызова: бэкэнд Chat инициирует HTTP POST запрос к бэкэнду приложения.
- После получения запроса обратного вызова бэкэнд приложения должен проверить, совпадает ли SDKAppID, содержащийся в URL запроса, с его собственным SDKAppID.
- Для получения информации о других проблемах безопасности см. раздел [Обзор обратных вызовов третьих сторон: рассмотрение вопросов безопасности](https://intl.cloud.tencent.com/document/product/1047/34354#.E5.AE.89.E5.85.A8.E8.80.83.E8.99.91).

## Сценарии срабатывания обратного вызова

- Пользователь приложения инициирует запрос на добавление пользователя в черный список через клиент.
- Бэкэнд приложения инициирует запрос на добавление пользователя в черный список через RESTful API.

## Время срабатывания обратного вызова

Обратный вызов срабатывает, когда бэкэнд Chat получает запрос на добавление пользователя в черный список и успешно добавляет пользователя в черный список.

## Описание API

### Пример URL запроса

В следующем примере URL обратного вызова, настроенный в приложении, имеет значение `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL обратного вызова. |
| SdkAppid | SDKAppID, назначенный консолью Chat при создании приложения. |
| CallbackCommand | Значение зафиксировано на Sns.CallbackBlackListAdd. |
| contenttype | Значение зафиксировано на JSON. |
| ClientIP | IP адрес клиента, формат которого аналогичен: 127.0.0.1. |
| OptPlatform | Платформа клиента. Дополнительные сведения о возможных значениях см. в параметре OptPlatform в разделе [Обзор обратных вызовов третьих сторон: протоколы обратных вызовов](https://intl.cloud.tencent.com/document/product/1047/34354#.E5.9B.9E.E8.B0.83.E5.8D.8F.E8.AE.AE). |

### Пример пакета запроса

```
{   "CallbackCommand": "Sns.CallbackBlackListAdd",   "PairList": [       {           "From_Account": "id",           "To_Account": "id1"       },       {           "From_Account": "id",           "To_Account": "id2"       },       {           "From_Account": "id",           "To_Account": "id3"       }   ]}
```

### Поля пакета запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда обратного вызова. |
| PairList | Array | Пара контактов черного списка, успешно добавленная. |
| From_Account | String | From_Account добавляет To_Account в черный список. |
| To_Account | String | To_Account добавляется в черный список From_Account. |

### Пример пакета ответа

```
{   "ActionStatus": "OK",   "ErrorCode": 0,   "ErrorInfo": ""}
```

### Поля пакета ответа

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательное | Результат обработки запроса. OK: успешно. FAIL: ошибка. |
| ErrorCode | Integer | Обязательное | Код ошибки. 0 указывает, что обработка бэкэнда приложения прошла успешно, 1 указывает, что обработка бэкэнда приложения не удалась. |
| ErrorInfo | String | Обязательное | Информация об ошибке. |

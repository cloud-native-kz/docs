# После удаления пользователя из чёрного списка

## Описание функции

Посредством этого обратного вызова бэкэнд приложения может в реальном времени отслеживать удаление пользователя из чёрного списка.

## Меры предосторожности

- Чтобы включить этот обратный вызов, необходимо настроить URL обратного вызова и включить соответствующий протокол. Подробнее о методе конфигурации см. в разделе [Конфигурация обратного вызова третьей стороны](https://intl.cloud.tencent.com/document/product/1047/34520).
- Направление обратного вызова: бэкэнд Chat инициирует HTTP POST запрос на бэкэнд приложения.
- После получения запроса обратного вызова бэкэнд приложения должен проверить, совпадает ли SDKAppID, содержащийся в URL запроса, с его собственным SDKAppID.
- По другим вопросам безопасности см. раздел [Обзор обратного вызова третьей стороны: Рассмотрение безопасности](https://intl.cloud.tencent.com/document/product/1047/34354#.E5.AE.89.E5.85.A8.E8.80.83.E8.99.91).

## Сценарии срабатывания обратного вызова

- Пользователь приложения использует клиент для инициирования запроса на удаление пользователей из чёрного списка.
- Бэкэнд приложения инициирует запрос на удаление пользователей из чёрного списка через RESTful API.

## Время срабатывания обратного вызова

Обратный вызов срабатывает после того, как бэкэнд Chat получит запрос на удаление пользователей из чёрного списка и успешно удалит пользователей.

## Описание API

### Пример URL запроса

В следующем примере настроенный в приложении URL обратного вызова — это `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL обратного вызова. |
| SdkAppid | SDKAppID, назначенный консолью Chat при создании приложения. |
| CallbackCommand | Значение фиксировано как Sns.CallbackBlackListDelete. |
| contenttype | Значение фиксировано как JSON. |
| ClientIP | IP-адрес клиента, формат аналогичен: 127.0.0.1. |
| OptPlatform | Платформа клиента. Подробнее о возможных значениях см. параметр OptPlatform в разделе [Обзор обратного вызова третьей стороны: Протоколы обратного вызова](https://intl.cloud.tencent.com/document/product/1047/34354#.E5.9B.9E.E8.B0.83.E5.8D.8F.E8.AE.AE). |

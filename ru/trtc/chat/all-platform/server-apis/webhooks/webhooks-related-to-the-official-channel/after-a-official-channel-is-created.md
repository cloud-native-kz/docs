# После создания официального канала

## Обзор функции

Это событие вебхука позволяет бэкенду приложения просматривать информацию, связанную с созданием официальных каналов пользователями в реальном времени, включая уведомление бэкенда приложения об успешном создании официального канала, что позволяет выполнять синхронизацию данных и другие операции.

## Примечания

- Чтобы включить вебхук, необходимо настроить URL вебхука и включить соответствующий переключатель для этого вебхука. Дополнительные сведения о методе конфигурации см. в разделе [Конфигурация вебхука](https://www.tencentcloud.com/document/product/1047/34520).
- Во время этого вебхука сервер Chat инициирует HTTP POST запрос на бэкенд приложения.
- После получения запроса вебхука сервер приложения должен проверить, соответствует ли SDKAppID, содержащийся в URL запроса, SDKAppID приложения.
- По дополнительным вопросам безопасности обратитесь к документу [Обзор вебхука: Соображения безопасности](https://www.tencentcloud.com/document/product/1047/34354).

## Сценарии срабатывания вебхука

Администратор приложения создает официальный аккаунт через REST API.

## Время срабатывания вебхука

После успешного создания официального аккаунта.

## Описание вызова API

### Пример URL запроса

В следующем примере URL вебхука, настроенный в приложении, это `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL обратного вызова. |
| SdkAppid | SDKAppID, назначенный консолью Chat при создании приложения. |
| CallbackCommand | Зафиксировано: OfficialAccount.CallbackAfterCreateOfficialAccount. |
| contenttype | Фиксированное значение: JSON. |
| ClientIP | IP-адрес клиента, например: 127.0.0.1. |
| OptPlatform | Платформа клиента, значения см. в разделе [Обзор вебхука: Протокол обратного вызова](https://www.tencentcloud.com/document/product/1047/34354) для понимания значения параметра OptPlatform. |

### Пример пакетов запроса

```
 {    "CallbackCommand": "OfficialAccount.CallbackAfterCreateOfficialAccount", // Команда обратного вызова    "Official_Account" : "@TOA#_test_OA_for_penn",    "Operator_Account": "107867", // Оператор    "Owner_Account": "107867", // Создатель    "Name": "TestOfficialAccount", // Название официального канала    "EventTime": 1670574414123// Уровень миллисекунд, временная метка срабатывания события		}
```

### Поля пакета запроса

| Объект | Характеристики | Функция |
| --- | --- | --- |
| CallbackCommand | String | Команда вебхука. |
| Operator_Account | String | UserID оператора, инициирующего запрос на создание. |
| Official_Account | String | Идентификатор пользователя созданного официального аккаунта. |
| Owner_Account | String | Создатель официального канала, который также является владельцем официального канала. |
| Name | String | Название создаваемого официального канала. |
| EventTime | Integer | Временная метка срабатывания события в миллисекундах. |

### Пример ответа

После синхронизации данных бэкенд приложения отправляет пакет ответа вебхука.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 }
```

### Поля ответа

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательно | Результат обработки запроса: OK: Указывает на успешную обработку, FAIL: Указывает на сбой |
| ErrorCode | Integer | Обязательно | Код ошибки, значение 0 означает игнорировать ответ. |
| ErrorInfo | String | Обязательно | Сообщение об ошибке. |

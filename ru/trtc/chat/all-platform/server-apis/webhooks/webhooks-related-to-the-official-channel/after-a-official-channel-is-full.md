# После того как официальный канал заполнен

## Обзор функции

Этот вебхук позволяет серверной части приложения отслеживать в реальном времени, достигнут ли лимит количества подписчиков официального канала, обеспечивая возможность подписки пользователям.

## Примечания

- Чтобы включить вебхук, необходимо настроить URL вебхука и включить соответствующий протокол. Дополнительную информацию о методе конфигурации см. в разделе [Конфигурация вебхука](https://www.tencentcloud.com/document/product/1047/34520).
- Во время этого вебхука серверная часть Chat инициирует HTTP POST запрос к серверной части приложения.
- После получения запроса вебхука серверная часть приложения должна проверить, является ли `SDKAppID`, содержащийся в URL запроса, `SDKAppID` приложения.
- Для получения дополнительной информации о мерах безопасности см. документ [Обзор вебхука: Соображения безопасности](https://www.tencentcloud.com/document/product/1047/34354).

## Сценарии срабатывания вебхука

- Пользователи приложения подписываются на официальный канал через клиент.
- Администратор приложения добавляет подписчиков через REST API.

## Время срабатывания вебхука

После успешной подписки пользователя на официальный канал.

## Описание вызова API

### Пример URL запроса

В приведенном ниже примере URL вебхука, настроенный в приложении, — `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL вебхука. |
| SdkAppid | SDKAppID, назначенный консолью Chat при создании приложения. |
| CallbackCommand |固定值为:OfficialAccount.CallbackAfterOfficialAccountFull. |
| contenttype | Фиксированное значение: JSON. |
| ClientIP | IP адрес клиента, например: 127.0.0.1. |
| OptPlatform | Платформа клиента. Значения см. в [Обзоре вебхука: Протокол вебхука](https://www.tencentcloud.com/document/product/1047/34354) для получения значений параметра OptPlatform. |

### Пример запроса

```
{    "CallbackCommand": "OfficialAccount.CallbackAfterOfficialAccountFull", // Команда вебхука    "Official_Account" : "@TOA#_test_OA_for_penn",    "EventTime": 1670574414123// Уровень миллисекунд, временная метка срабатывания события		}
```

### Поля запроса

| Объект | Особенности | Функция |
| --- | --- | --- |
| CallbackCommand | String | Команда вебхука. |
| Official_Account | String | Идентификатор пользователя официального канала с максимальным количеством подписчиков. |
| EventTime | Integer | Временная метка срабатывания события в миллисекундах. |

### Пример ответа

После того как серверная часть приложения записывает, что поддерживаемое количество подписчиков официального канала достигло своего лимита, она отправляет пакет ответа вебхука.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // Игнорировать результаты ответа}
```

### Поля ответа

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательное | Результат обработанного запроса:OK: Указывает на успешную обработкуFAIL: Указывает на сбой |
| ErrorCode | Integer | Обязательное | Код ошибки, 0 означает, что результаты ответа можно игнорировать. |
| ErrorInfo | String | Обязательное | Сообщение об ошибке. |

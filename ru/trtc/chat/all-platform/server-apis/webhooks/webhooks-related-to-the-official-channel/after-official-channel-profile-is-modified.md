# После изменения профиля официального канала

## Обзор функции

Через этот вебхук приложение может просматривать изменения информации официального канала (например, имя официального канала, описание официального канала, аватар официального канала и т. д.) в реальном времени, включая записи об изменениях информации официального канала в реальном времени (например, логирование или синхронизацию с другими системами).

## Примечания

- Чтобы включить вебхук, необходимо настроить URL вебхука и включить соответствующий протокол. Дополнительные сведения о методе конфигурации см. в разделе [Конфигурация вебхука](https://www.tencentcloud.com/document/product/1047/34520).
- Во время работы этого вебхука серверная часть Chat инициирует HTTP POST запрос к серверной части приложения.
- После получения запроса вебхука серверная часть приложения должна проверить, совпадает ли SDKAppID, содержащийся в URL запроса, с `SDKAppID` приложения.
- По другим вопросам безопасности обратитесь к разделу [Введение в вебхуки: Рассмотрение безопасности](https://www.tencentcloud.com/document/product/1047/34354).

## Сценарии срабатывания вебхука

- Администраторы приложения изменяют информацию официального канала через REST API

## Время срабатывания вебхука

После успешного изменения информации официального канала

## Описание вызова API

### Пример URL запроса

В последующем примере URL вебхука, настроенный в приложении, — `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST |
| www.example.com | URL вебхука |
| SdkAppid | SDKAppID, выданный консолью Instant Messaging при создании приложения |
| CallbackCommand | Установить значение: OfficialAccount.CallbackAfterOfficialAccountInfoChanged |
| contenttype | Фиксированное значение: JSON |
| ClientIP | IP-адрес клиента, например 127.0.0.1 |
| OptPlatform | Платформа клиента, значения см. в разделе [введение в вебхуки: Протокол вебхука](https://www.tencentcloud.com/document/product/1047/34354) относительно значений параметров OptPlatform |

### Пример запроса

```
{    "CallbackCommand": "OfficialAccount.CallbackAfterOfficialAccountInfoChanged", // Команда обратного вызова    "Official_Account" : "@TOA#_test_OA_for_penn",    "Operator_Account": "leckie", // Инициатор    "Introduction": "NewNotification", // Обновленное описание официального аккаунта    "Name": "NewName",		// Обновленное имя официального аккаунта    "FaceUrl": "http://this.is.newface.url"	// Обновленная фотография профиля официального аккаунта  	"Organization": "NewOrganization"		// Обновленная организация официального аккаунта    "EventTime": 1670574414123// Временная метка срабатывания события в миллисекундах		}
```

### Поля запроса

| Объект | Особенности | Функция |
| --- | --- | --- |
| CallbackCommand | String | Команда вебхука |
| Operator_Account | String | UserID инициатора |
| Official_Account | String | ID созданного официального канала |
| Introduction | String | Обновленное описание официального аккаунта |
| Name | String | Обновленное имя официального канала |
| FaceUrl | String | Обновленный аватар официального канала |
| Organization | String | Обновленная организация официального канала |
| EventTime | Integer | Временная метка срабатывания события в миллисекундах |

### Пример ответа

После синхронизации данных серверная часть приложения отправляет пакет ответа вебхука.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 }
```

### Поля ответа

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательное | Результаты обработки запросаOK: Указывает на успешную обработкуFAIL: Указывает на ошибку |
| ErrorCode | Integer | Обязательное | Код ошибки, введение 0 здесь означает игнорирование результата ответа |
| ErrorInfo | String | Обязательное | Сообщение об ошибке |

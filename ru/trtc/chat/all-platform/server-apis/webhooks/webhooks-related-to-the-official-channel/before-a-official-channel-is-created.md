# Перед созданием официального канала

## Обзор функции

Серверная часть приложения может использовать этот webhook для просмотра запросов на создание официальных каналов в реальном времени, включая возможность отклонения этих запросов.

## Примечания

- Для включения webhook необходимо настроить URL webhook и включить соответствующий переключатель для этого протокола webhook. Дополнительные сведения о методе конфигурации см. в разделе [Конфигурация Webhook](https://www.tencentcloud.com/document/product/1047/34520).
- Во время этого webhook серверная часть Chat инициирует HTTP POST запрос к серверной части приложения.
- После получения запроса webhook серверная часть приложения должна проверить, что SDKAppID, содержащийся в URL запроса, является SDKAppID приложения.
- По вопросам безопасности обратитесь к документации [Обзор Webhook: Соображения безопасности](https://www.tencentcloud.com/document/product/1047/34354).

## Сценарии срабатывания Webhook

Администратор приложения создает официальный аккаунт через REST API.

## Время срабатывания Webhook

Серверная часть Chat готовится к созданию официального аккаунта.

## Описание вызова API

### Пример URL запроса

В последующем примере URL webhook, настроенный в приложении: `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL обратного вызова. |
| SdkAppid | SDKAppID, назначенный консолью Chat при создании приложения. |
| CallbackCommand | Фиксированное значение: OfficialAccount.CallbackBeforeCreateOfficialAccount. |
| contenttype | Фиксированное значение: JSON. |
| ClientIP | IP адрес клиента, например: 127.0.0.1. |
| OptPlatform | Платформа клиента. Значения и описание параметра OptPlatform см. в разделе [Обзор Webhook: Протокол обратного вызова](https://www.tencentcloud.com/document/product/1047/34354). |

### Пример запроса

```
{    "CallbackCommand": "OfficialAccount.CallbackBeforeCreateOfficialAccount", // Команда обратного вызова    "Operator_Account": "107867", // Оператор    "Owner_Account": "107867", // Создатель    "Name": "TestOfficialAccount", // Имя официального канала    "EventTime": 1670574414123// Временная метка срабатывания события в миллисекундах}
```

### Поля запроса

| Объект | Тип | Свойство |
| --- | --- | --- |
| CallbackCommand | String | Команда webhook. |
| Operator_Account | String | UserID оператора, инициирующего запрос на создание. |
| Owner_Account | String | Создатель официального канала, также являющийся владельцем официального канала. |
| Name | String | Имя создаваемого официального канала. |
| EventTime | Integer | Временная метка срабатывания события в миллисекундах. |

### Пример ответа

#### Разрешить создание

Разрешить пользователям создавать официальные каналы.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // Разрешить создание}
```

#### Запретить создание

Пользователям не разрешено создавать официальные каналы. Официальный канал не будет создан, и коду ошибки 20006 будет возвращен вызывающей стороне.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 1 // Отклонить создание}
```

### Поля ответа

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательно | Результат обработки запроса:OK: указывает на успешную обработкуFAIL: указывает на ошибку |
| ErrorCode | Integer | Обязательно | Код ошибки: 0 разрешает создание; 1 запрещает создание. Если служба хочет использовать собственный код ошибки для отклонения создания группы пользователей, передайте код ошибки ErrorCode и ErrorInfo клиенту. Установите ErrorCode в диапазоне [120001, 130000]. |
| ErrorInfo | String | Обязательно | Сообщение об ошибке. |

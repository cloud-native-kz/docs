# Перед отправкой сообщения по официальному каналу

## Обзор функции

Этот webhook позволяет бэкенду приложения просматривать трансляционные сообщения официального канала в реальном времени, включая:

- Запись сообщений официального канала в реальном времени, например, ведение журнала или синхронизацию сообщений с другими системами.
- Блокировка запросов пользователей на отправку сообщений в официальный канал.

> **Внимание** Webhook предварительной проверки сообщений имеет стандартное время ожидания 2 секунды и не поддерживает изменения. Использование этого для проверки содержимого добавляет две передачи по открытой сети, что как минимум удваивает общую задержку по сравнению с обычными сценариями.

## Примечания

- Для включения webhook необходимо настроить URL webhook и активировать переключатель, соответствующий этому протоколу webhook. Подробные инструкции по настройке см. в документе [Конфигурация webhook](https://www.tencentcloud.com/document/product/1047/34520).
- Во время этого webhook бэкенд IM инициирует HTTP POST запрос к бэкенду приложения.
- После получения запроса webhook бэкенд приложения должен проверить соответствие SDKAppID, содержащегося в URL запроса, собственному SDKAppID.
- Для получения дополнительной информации о безопасности обратитесь к документу [Обзор webhook: рассмотрение вопросов безопасности](https://www.tencentcloud.com/document/product/1047/34354).

## Сценарии срабатывания webhook

- Пользователь приложения отправляет сообщение официального аккаунта через клиент.
- Администратор приложения отправляет сообщение официального аккаунта через RESTful API.

## Время срабатывания webhook

Webhook срабатывает перед тем, как бэкенд Chat отправит сообщение официального аккаунта подписчикам.

## Описание вызова API

### Пример URL запроса

В последующем примере URL webhook, настроенный в приложении, это `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса HTTPS, метод запроса POST. |
| www.example.com | URL webhook. |
| SdkAppid | SDKAppID, назначенный консолью Chat при создании приложения. |
| CallbackCommand | Зафиксировано как OfficialAccount.CallbackBeforeSendMsg. |
| contenttype | Фиксированное значение: JSON. |
| ClientIP | IP адрес клиента, например: 127.0.0.1. |
| OptPlatform | Платформа клиента, значения см. в [Обзор webhook: протокол webhook](https://www.tencentcloud.com/document/product/1047/34354) для значения параметра OptPlatform. |

### Пример запроса

```
{    "CallbackCommand": "OfficialAccount.CallbackBeforeSendMsg", // Команда webhook    "Official_Account": "@TOA#_2J4SZEAEL", // ID пользователя официального аккаунта    "OnlineOnlyFlag": 1, // Значение `1` если это сообщение онлайн, `0` если нет    "MsgBody": [ // Тело сообщения, см. объект TIMMessage        {            "MsgType": "TIMTextElem", // Текст            "MsgContent": {                "Text": "red packet"            }        }    ],    "CloudCustomData": "your cloud custom data",    "EventTime": 1670574414123 // Временная метка события срабатывания в миллисекундах		}
```

### Поля запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда webhook. |
| Official_Account | String | ID пользователя официального аккаунта. |
| OnlineOnlyFlag | Integer | Сообщение онлайн, `1` если да, иначе `0`. |
| MsgBody | Array | Тело сообщения, см. [Форматы сообщений](https://www.tencentcloud.com/document/product/1047/33527). |
| CloudCustomData | String | Пользовательские данные сообщения (хранятся в облаке, будут отправлены получателю и могут быть извлечены даже после удаления и переустановки приложения). |
| EventTime | Integer | Временная метка события срабатывания в миллисекундах. |

### Пример ответа

#### Разрешить отправку

Разрешить пользователям отправлять сообщения без изменения содержимого отправляемого сообщения.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // 0 означает разрешение на отправку}
```

#### Запретить отправку

Пользователю не разрешается отправлять сообщения. Следовательно, сообщение не будет отправлено, и код ошибки `10016` возвращается запрашивающему.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 1 // 1 означает запрещение на отправку}
```

#### Тихое отклонение

Пользователю не разрешается отправлять сообщения. В этом случае сообщение не будет отправлено, но успешный ответ будет возвращен вызывающему, заставляя его думать, что сообщение было отправлено.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 2 // 2 означает тихое отклонение}
```

#### Изменить содержимое сообщения

В следующем примере ответа: сообщение, отправленное на официальный канал, изменяется (было добавлено сообщение с пользовательским определением). Бэкенд Chat отправит модифицированное сообщение. На основе этой функции бэкенд приложения может добавлять специальное содержимое, такое как уровень и звание пользователя, к сообщению, отправленному пользователем.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0, // Должен быть 0, только тогда модифицированное сообщение может быть правильно отправлено    "MsgBody": [ // Сообщение после изменений приложением; если отсутствует, используется сообщение, отправленное пользователем        {            "MsgType": "TIMTextElem", // Текст            "MsgContent": {                "Text": "red packet"            }        },        {            "MsgType": "TIMCustomElem", // Пользовательское сообщение            "MsgContent": {                "Desc": "CustomElement.MemberLevel", // Описание                "Data": "LV1" // Данные            }        }    ],    "CloudCustomData": "your cloud custom data"}
```

### Поля ответа

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательный | Результат обработанного запроса: OK означает успешную обработку, FAILURE означает неудачное выполнение |
| ErrorCode | Integer | Обязательный | Идентификатор ошибки: 0: разрешить отправку, 1: отказать в отправке, 2: тихое отклонение. Если сервис хочет отклонить отправку с указанием кода ошибки ErrorCode и ErrorInfo для клиента, установите ErrorCode в диапазоне [120001, 130000]. |
| ErrorInfo | String | Обязательный | Сообщение об ошибке. |
| MsgBody | Array | Опциональный | После изменений приложением модифицированное тело сообщения будет отправлено сообщениям официального аккаунта бэкенд облачных коммуникаций. Для конкретного формата см. [Описание формата сообщения](https://www.tencentcloud.com/document/product/1047/33527). |
| CloudCustomData | String | Опциональный | Пользовательские данные сообщения (хранятся в облаке, будут отправлены получателю и могут быть извлечены даже после удаления и переустановки приложения). |

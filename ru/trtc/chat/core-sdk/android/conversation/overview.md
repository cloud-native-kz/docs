# Обзор

## Обзор

Когда пользователь создает личный или групповой чат и отправляет или получает в нем сообщение, создается беседа.
В Chat SDK класс управления беседами — это `V2TIMConversationManager(Java)` или `V2TIMManager(Conversation)(Objective-C)`. Вы можете использовать API класса управления беседами для отображения/обновления списка бесед, обновления счетчика непрочитанных сообщений, закрепления беседы в верхней части, создания черновика беседы и отключения уведомлений о сообщениях.

## Класс Conversation

Класс беседы — это `V2TIMConversation` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMConversation.html) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMConversation.html) /  [Objective-C](https://im.sdk.qcloud.com/doc/en/interfaceV2TIMConversation.html) / [C++](https://im.sdk.qcloud.com/doc/en/structV2TIMConversation.html)), который определяет следующее содержимое:

| Атрибут | Определение | Описание |
| --- | --- | --- |
| type | Тип беседы | См. определение `V2TIMConversationType`, которое может быть `C2C` или `Group`. |
| conversationID | Уникальный идентификатор беседы | Имеет формат `c2c_userID` для личного чата или `group_groupID` для группового чата. |
| userID | ID пользователя получателя сообщения | `userID` хранит ID пользователя получателя сообщения, если тип беседы — личный чат; в противном случае пусто. |
| groupID | ID группы | `groupID` хранит ID группы, если тип беседы — групповой чат; в противном случае пусто. |
| groupType | Тип группы | `groupType` — это тип группы, если тип беседы — групповой чат; в противном случае пусто. |
| showName | Отображаемое имя беседы | Имя групповой беседы отображается в следующем порядке приоритета: имя группы > ID группы; имя личной беседы отображается в следующем порядке приоритета: замечание о друге получателя сообщения > никнейм получателя сообщения > `userID` получателя сообщения. |
| faceUrl | Отображаемая фотография профиля беседы | Это фото профиля группы для группового чата или фото профиля получателя сообщения для личного чата. |
| unreadCount | Количество непрочитанных сообщений в беседе | Подробные инструкции см. в разделе [Счетчик непрочитанных сообщений в беседе](https://intl.cloud.tencent.com/document/product/1047/48320). Недействительно и по умолчанию `0` для аудио-видео группы (AVChatRoom). |
| recvOpt | Опция получения сообщений | См. определение `V2TIMReceiveMessageOpt` и инструкции по использованию [Не уведомлять](https://intl.cloud.tencent.com/document/product/1047/48032). |
| lastMessage | Последнее сообщение в беседе | Подробные инструкции см. в разделе [Список бесед](https://intl.cloud.tencent.com/document/product/1047/48326). |
| groupAtInfolist | Список информации @ в групповой беседе | Обычно используется для отображения уведомлений о "кто-то@меня" и "@all". |
| draftText | Информация черновика | Информацию черновика можно установить, вызвав API `setConversationDraft`. Подробные инструкции см. в разделе [Черновик беседы](https://intl.cloud.tencent.com/document/product/1047/48311). |
| draftTimestamp | Время редактирования черновика | Генерируется автоматически при установке черновика. |
| isPinned | Закрепить ли беседу в верхней части | Подробные инструкции см. в разделе [Закрепление беседы в верхней части](https://intl.cloud.tencent.com/document/product/1047/48317). |
| orderKey | Поле для сортировки бесед | Подробные инструкции см. в разделе [Список бесед](https://intl.cloud.tencent.com/document/product/1047/48326). |
| markList | Список отметок беседы | Инструкции по использованию см. в разделе [Отметка беседы](https://intl.cloud.tencent.com/document/product/1047/48853). Этот атрибут поддерживается с версии 6.5. |
| customData | Пользовательские данные беседы | Для установки пользовательских данных вызовите API `setConversationCustomData`. Пользовательские данные будут сохранены в облаке. Поддерживается до 256 байт. Этот атрибут поддерживается с версии 6.5. |
| conversationGroupList | Список групп, к которым принадлежит беседа | Инструкции по использованию см. в разделе [Группа бесед](https://intl.cloud.tencent.com/document/product/1047/48854). Этот атрибут поддерживается с версии 6.5. |
| c2cReadTimestamp | Временная метка последнего прочитанного сообщения | Действительно только для личного чата. Поддерживается с версии 7.1. |
| groupReadSequence | Последовательность последнего прочитанного сообщения | Действительно только для группового чата. Поддерживается с версии 7.1. |

## Политика хранения бесед

Нет ограничений на количество локально сохраняемых бесед.

В облаке можно сохранить до 100 бесед. Чтобы увеличить это ограничение, обновитесь до версии Pro, Pro Plus или Enterprise. Затем вы можете установить лимит на 500 в консоли.

Если информация беседы длительное время не обновлялась, беседа может храниться в облаке до 7 дней.

Локально сохраняемые беседы могут не всегда совпадать с теми, которые сохраняются в облаке. Если вы не вызовете API `deleteConversation` для удаления локальных бесед, эти беседы всегда будут существовать. Однако в облаке можно сохранить максимум 100 бесед. Кроме того, если информация беседы длительное время не обновлялась, эта беседа может храниться в облаке максимум 7 дней. Поэтому локальные беседы, отображаемые на разных мобильных телефонах, могут отличаться друг от друга.

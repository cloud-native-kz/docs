# Количество непрочитанных сообщений

## Обзор

Количество непрочитанных сообщений — это число сообщений, которые пользователь еще не прочитал в приложении чата. Это число обычно обозначается красной точкой или цифрами в списке диалогов, напоминая пользователю, что есть новые непрочитанные сообщения для просмотра. Количество непрочитанных сообщений побуждает пользователей оперативно просматривать и отвечать на новые сообщения. Это может повысить вовлеченность пользователей и эффективность общения, предотвращая игнорирование важной информации.

## Описание API

### Получение общего количества непрочитанных сообщений

> **Примечание:** Эта функция поддерживается только SDK Enhanced Edition версии 5.3.425 и выше. Эта функция применяется только к рабочим группам (Work), публичным группам (Public) и сообществам (Community), но не к аудиовидео группам (AVChatRoom) или группам встреч (Meeting). Дополнительную информацию о типах групп см. в разделе [Обзор групп](https://intl.cloud.tencent.com/document/product/1047/48169).

Ниже приведены подробные шаги.

### Получение количества непрочитанных сообщений по фильтру

Вы можете вызвать `getUnreadMessageCountByFilter`, передав различные `V2TIMConversationListFilter`, чтобы получить количество непрочитанных сообщений для диалогов под определенным фильтром.

`V2TIMConversationListFilter` объясняется следующим образом:

| Атрибут | Значение | Описание |
| --- | --- | --- |
| type | Тип чата (введите 0, чтобы не фильтровать этот элемент) | Личный или групповой чат |
| conversationGroup | Название группы сеанса (если оставить пустым, этот элемент не будет фильтроваться) | Не название группы, а название группировки сеанса, см. документ [Группировка сеансов](https://intl.cloud.tencent.com/document/product/1047/48854) |
| markType | Тип метки сеанса (введите 0, чтобы не фильтровать этот элемент) | См. документ [Метка сеанса](https://intl.cloud.tencent.com/document/product/1047/48853) |
| hasUnreadCount | Количество сеансов с непрочитанными сообщениями | true: возвращает сеансы с непрочитанными сообщениями; false: возвращает все сеансы |
| hasGroupAtInfo | Сеанс включает сообщения группы @ | true: возвращает сеансы с сообщениями группы @; false: возвращает все сеансы |

Если вам нужно отслеживать изменения в общем количестве непрочитанных сообщений диалога, вы можете вызвать `subscribeUnreadMessageCountByFilter` для регистрации уведомлений. Когда общее количество непрочитанных сообщений в отфильтрованном диалоге изменится, SDK проактивно уведомит вас о последнем общем количестве непрочитанных через обратный вызов `onUnreadMessageCountChangedByFilter`.

> **Примечание:** Получение общего количества непрочитанных сообщений определенных сеансов на основе критериев фильтра поддерживается только SDK Enhanced Edition версии 7.0.3754 и выше. Применимо только к рабочим группам друзей (Work), странным социальным группам (Public) и сообществам (Community), но неприменимо к группам трансляции (AVChatRoom) и временным группам встреч (Meeting). Дополнительные сведения о типах групп см. в разделе [Введение в группы](https://intl.cloud.tencent.com/document/product/1047/48169).

Ниже приведены некоторые типичные случаи извлечения с фильтрами.

### Уведомление об изменении количества непрочитанных

#### Подписка на уведомления

Вызовите `addConversationListener` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMConversationManager.html#a806534684e5d4d01b94126cd1397fee4)/ [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMManager+Conversation.html#v2timmanager.addconversationlistener(listener:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/categoryV2TIMManager_07Conversation_08.html#a39b4f352f1740171fb56143149201cd9)/[C++](https://im.sdk.qcloud.com/doc/en/classV2TIMConversationManager.html#adb2c20ca824cac69d0703169f3a025a1)), чтобы добавить слушателя диалога. class SendCallback final : public V2TIMSendCallback {
public:
    using SuccessCallback = std::function<void(const V2TIMMessage&)>;
    using ErrorCallback = std::function<void(int, const V2TIMString&)>;
    using ProgressCallback = std::function<void(uint32_t)>;
    SendCallback() = default;
    ~SendCallback() override = default;
    void SetCallback(SuccessCallback success_callback, ErrorCallback error_callback,
                     ProgressCallback progress_callback) {
        success_callback_ = std::move(success_callback);
        error_callback_ = std::move(error_callback);
        progress_callback_ = std::move(progress_callback);
    }
    void OnSuccess(const V2TIMMessage& message) override {
        if (success_callback_) {
            success_callback_(message);
        }
    }
    void OnError(int error_code, const V2TIMString& error_message) override {
        if (error_callback_) {
            error_callback_(error_code, error_message);
        }
    }
    void OnProgress(uint32_t progress) override {
        if (progress_callback_) {
            progress_callback_(progress);
        }
    }
private:
    SuccessCallback success_callback_;
    ErrorCallback error_callback_;
    ProgressCallback progress_callback_;
};
// Создать объект сообщения
V2TIMMessage message = V2TIMManager::GetInstance()->GetMessageManager()->CreateTextMessage(u8"content");
// Установить, чтобы не обновлять `lastMessage` диалога
message.isExcludedFromUnreadCount = true;
auto callback = new SendCallback{};
callback->SetCallback([=](const V2TIMMessage& message) { delete callback; },
                      [=](int error_code, const V2TIMString& error_message) { delete callback; },
                      [=](uint32_t progress) {});
V2TIMManager::GetInstance()->GetMessageManager()->SendMessage(
    message, u8"userID", {}, V2TIMMessagePriority::V2TIM_PRIORITY_NORMAL, false, {}, callback);

# Профиль

## Обзор

Профиль группы содержит информацию о группе, такую как ID группы, тип группы, пользовательское поле группы и количество присоединённых участников. Он сохраняется в объекте `V2TIMGroupInfo`, который создаётся и возвращается SDK IM и не может быть настроен.

## Получение профиля группы

Вызовите `getGroupsInfo` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMGroupManager.html#ada614335043d548c11f121500e279154) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMManager+Group.html#v2timmanager.getgroupsinfo(groupidlist:succ:fail:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/categoryV2TIMManager_07Group_08.html#a9bca7e5318cfed44335566a783a6b568) / [C++](https://im.sdk.qcloud.com/doc/en/classV2TIMGroupManager.html#a8c98b92b45c3a2c4e57901e6c4cd3435)) для получения профиля группы. Этот API поддерживает передачу нескольких значений `groupID` одновременно для пакетного получения профилей групп.

## Изменение профилей групп

Вызовите `setGroupInfo` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMGroupManager.html#ad4ceef92975fa00c4a5dddc8f7e1edcf) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMManager+Group.html#v2timmanager.setgroupinfo(info:succ:fail:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/categoryV2TIMManager_07Group_08.html#aa9519a479493e56d7920e40aba796144) / [C++](https://im.sdk.qcloud.com/doc/en/classV2TIMGroupManager.html#a10785c46e166879250c2c2ba2001b354)) для изменения профиля группы.

Если вы вызвали `addGroupListener` для добавления прослушивателя событий группы, после изменения профиля группы все участники группы получат обратный вызов `onGroupInfoChanged` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMGroupListener.html#ac3a88ea430c6dc35845472ed98ad049b) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMGroupListener.html#v2timgrouplistener.ongroupinfochanged(groupid:changeinfolist:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/protocolV2TIMGroupListener-p.html#abbe2208a234d77364bff697eea503d27) / [C++](https://im.sdk.qcloud.com/doc/en/classV2TIMGroupListener.html#a37aa57b23295e6b295413fafba2eda07)).

Роли участников, которые могут изменять профиль группы, различаются в зависимости от типа группы:

| Тип группы | Роли участников, имеющие право изменять профиль группы |
| --- | --- |
| Рабочая группа (Work) | Все участники группы |
| Публичная группа (Public) | Владелец группы и администраторы |
| Встречная группа (Meeting) | Владелец группы и администраторы |
| Сообщество (Community) | Владелец группы и администраторы |
| Аудиовизуальная группа (AVChatRoom) | Владелец группы |

> **Внимание** Не все поля профиля группы `V2TIMGroupInfo` можно изменять, только те, которые доступны для записи.

## Установка пользовательского поля группы

В отличие от установки других полей в `V2TIMGroupInfo`, установка пользовательского поля группы состоит из двух шагов:

1. В [консоли](https://console.trtc.io/) настройте ключ для поля пользовательского определения группы. Ключ имеет тип строки и его длина не должна превышать 16 байт. Если ключ не был предварительно настроен, попытка прямой установки пары ключ-значение будет неудачной. Путь конфигурации: Приложения > Ваше приложение > Chat > Конфигурация > Конфигурация группы > Пользовательское поле группы.
2. Вызовите API `setGroupInfo` для установки поля объёмом до 512 байт.

> **Внимание** Вы можете установить до десяти пользовательских полей, которые нельзя удалить, и чьё имя и тип нельзя изменить. Пользовательское поле в основном используется для обеспечения совместимости с v1 и v2. Если вы используете API на v2, мы рекомендуем использовать API `initGroupAttributes` для установки атрибутов группы. Это обеспечивает большую гибкость (не требуется конфигурация в консоли) и хранилище (до 16 КБ). Дополнительную информацию см. в разделе [Пользовательский атрибут группы](https://intl.cloud.tencent.com/document/product/1047/48175).

## Установка опции получения сообщений группы

Любой участник группы может вызвать API `setGroupReceiveMessageOpt` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMMessageManager.html#a2735427ac22485626aea278a9d465b3e) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMManager+Message.html#v2timmanager.setgroupreceivemessageopt(groupid:opt:succ:fail:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/categoryV2TIMManager_07Message_08.html#a379eeef926e41ec5d48287e7fb55b80a) / [C++](https://im.sdk.qcloud.com/doc/en/classV2TIMMessageManager.html#a866d06c28faf058f253f29be6f5b3fe2)) для изменения опции получения сообщений группы. Этот параметр действует только для участника группы и не влияет на параметры других участников группы.

`V2TIMReceiveMessageOpt` имеет следующие опции:

| Опция получения сообщений | Поддерживаемый тип | Описание |
| --- | --- | --- |
| V2TIM_RECEIVE_MESSAGE | Персональный чат, групповой чат, все сообщения | Получать сообщения в нормальном режиме в сети и получать уведомления о доставке в автономном режиме |
| V2TIM_NOT_RECEIVE_MESSAGE | Персональный чат, групповой чат | Не получать сообщения ни в сети, ни в автономном режиме |
| V2TIM_RECEIVE_NOT_NOTIFY_MESSAGE | Персональный чат, групповой чат, все сообщения | Получать сообщения в нормальном режиме в сети, но не получать уведомления о доставке в автономном режиме |
| V2TIM_RECEIVE_NOT_NOTIFY_MESSAGE_EXCEPT_AT | Групповой чат | Получать сообщения в сети, получать уведомления только для сообщений `at` в автономном режиме |
| V2TIM_NOT_RECEIVE_MESSAGE_EXCEPT_AT | Тема | Получать только сообщения `at` в сети и в автономном режиме |

Установка различных опций `V2TIMReceiveMessageOpt` может реализовать различные эффекты режима «Не беспокоить»:

| Эффект режима «Не беспокоить» | Опция получения сообщений | Описание |
| --- | --- | --- |
| Не получать никаких сообщений | V2TIM_NOT_RECEIVE_MESSAGE | Сообщения не могут быть получены, и список разговоров не будет обновлён. |
| Получать сообщения без уведомлений | V2TIM_RECEIVE_NOT_NOTIFY_MESSAGE | В этом случае рекомендуется отображать красную точку в списке разговоров (без отображения счётчика непрочитанных): при получении нового сообщения и необходимости обновления списка разговоров получите счётчик непрочитанных сообщений через unreadCount в V2TIMConversation разговора. Если счётчик непрочитанных больше нуля, отображайте красную точку вместо количества непрочитанных. |
| Получать сообщения, но только с уведомлением для сообщений `at` | V2TIM_RECEIVE_NOT_NOTIFY_MESSAGE_EXCEPT_AT | Получать сообщения в группе в нормальном режиме будет напоминать только об сообщениях `at`. |
| Получать только сообщения `at` | V2TIM_NOT_RECEIVE_MESSAGE_EXCEPT_AT | Получать только сообщения `at` в теме в сети и в автономном режиме |

> **Примечание:** В приведённых выше реализациях, если требуется функция `unreadCount` в `V2TIMConversation`, она применяется только к рабочим группам (Work), публичным группам (Public) и сообществам (Community), но не к аудиовизуальным группам (AVChatRoom) или встречным группам (Meeting). Дополнительную информацию о типах групп см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

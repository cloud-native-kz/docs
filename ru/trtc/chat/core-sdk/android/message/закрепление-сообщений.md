# Закрепление сообщений

## Описание

Функция закрепления сообщений позволяет зафиксировать определённые сообщения в верхней части списка в приложении Chat, сохраняя их позицию в верхней части даже при получении новых сообщений. Эта функция помогает пользователям легко находить и получать доступ к сообщениям, которые они считают важными или часто просматриваемыми.

> **Примечание:** Функция закрепления сообщений поддерживается в расширенной версии 7.9 и позже. Эта функция доступна исключительно для клиентов премиум-издания, пожалуйста [приобретите премиум-издание](https://trtc.io/buy/chat) сначала. В настоящее время функция закрепления сообщений поддерживается только для обычных групп.

### Закрепление сообщения

Вызов API `PinGroupMessage` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMMessageManager.html#aa345a7876bf0df29491429329a10f469)/ [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMManager+Message.html#v2timmanager.pingroupmessage(groupid:message:ispinned:succ:fail:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/categoryV2TIMManager_07Message_08.html#a4bd9a32e4525e75778708bfde5b95c62)/[C++](https://im.sdk.qcloud.com/doc/en/classV2TIMMessageManager.html#ace35ac347c02c608f5f61ce2a76599ad)) позволяет установить сообщение для закрепления в верхней части.

Подробное объяснение входных параметров API закрепления сообщений:

| Атрибут | Значение | Описание |
| --- | --- | --- |
| groupID | ID группы | Указывает ID группового чата, требующего закрепления/открепления сообщений. |
| message | Объект сообщения | Сообщение должно быть успешно отправлено. |
| isPinned | Действие закрепления/открепления | Указывает, следует ли закреплять или откреплять групповые сообщения. |

> **Примечание:** Групповой чат поддерживает до 10 закреплённых сообщений. Если количество закреплённых сообщений превышает лимит, вызов API приведёт к ошибке ERR_SVR_GROUP_PINNED_MESSAGE_COUNT_LIMIT.

### Получение закреплённых сообщений

Вызов API `GetPinnedGroupMessageList` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMMessageManager.html#a35c28770388fa0b1c51946b314287786) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMManager+Message.html#v2timmanager.getpinnedgroupmessagelist(groupid:succ:fail:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/categoryV2TIMManager_07Message_08.html#afee75362418c0dbd3f2337fea5865179) / [C++](https://im.sdk.qcloud.com/doc/en/classV2TIMMessageManager.html#a174f9f9595661f8229975f537fd741eb)) позволяет получить список закреплённых сообщений.

Если вам нужно получить информацию о пользователе, который закрепил сообщение, вы можете сделать это через поле pinnerInfo объекта V2TIMMessage.

Подробное объяснение входных параметров API закрепления сообщений:

| Атрибут | Значение | Описание |
| --- | --- | --- |
| groupID | ID группы | Указывает ID группового чата, для которого необходимо получить список закреплённых сообщений. |

> **Примечание:** Этот API используется для получения списка закреплённых сообщений. Если срок хранения закреплённых сообщений истёк (7 дней для пробной версии и 30 дней для премиум-издания), результат не будет возвращён.

### Уведомление об изменении списка закреплённых сообщений

Если вы добавили слушателя событий расширенных сообщений, вызвав `addAdvancedMsgListener`, вы получите обратный вызов `onGroupMessagePinned` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMAdvancedMsgListener.html#a69954fdd1093ee4b7c3a08c277956198)/ [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMAdvancedMsgListener.html#v2timadvancedmsglistener.ongroupmessagepinned(groupid:message:ispinned:opuser:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/protocolV2TIMAdvancedMsgListener-p.html#a0946f7c3e62d1b5c7f2157aab11cb9e8) /[C++](https://im.sdk.qcloud.com/doc/en/classV2TIMAdvancedMsgListener.html#a1b6226155e451274ad8b55b2a5157aa6)) при обновлении списка закреплённых сообщений.

> **Примечание:** Если тип изменения — открепление, параметр сообщения будет содержать только отправителя, порядковый номер, случайное число и временную метку как действительные поля, без полного тела сообщения.

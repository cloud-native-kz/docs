# Обзор

## Класс Message

В Chat SDK класс сообщения — это `V2TIMMessage` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMMessage.html) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMMessage.html) / [Objective-C](https://im.sdk.qcloud.com/doc/en/interfaceV2TIMMessage.html) / [C++](https://im.sdk.qcloud.com/doc/en/structV2TIMMessage.html)), который часто используется для отправки и получения сообщений.

Класс `V2TIMMessage` определяет следующее содержимое:

| Атрибут | Определение | Описание |
| --- | --- | --- |
| msgID | ID сообщения | Пусто при создании сообщения и возвращается синхронно при вызове `sendMessage`. |
| timestamp | Временная метка сообщения | Время отправки сообщения на сервер, может использоваться для сортировки сообщений |
| sender | `userID` отправителя сообщения | Может быть установлено пользователями и должно совпадать с `userID`, переданным при входе. |
| nickName | Прозвище отправителя сообщения | Может быть установлено пользователями и изменено путём вызова `setSelfInfo`. Дополнительные сведения см. в разделе [Профиль пользователя](https://intl.cloud.tencent.com/document/product/1047/48162). |
| friendRemark | Примечание к другу отправителя сообщения | Используется получателем. Например, если Alice установила примечание Bob как `bob01`, когда Bob отправляет сообщение Alice, Alice увидит, что `friendRemark` в сообщении это `bob01`. Может быть установлено путём вызова `setFriendInfo`. Дополнительные сведения см. в разделе [Профиль члена группы](https://www.tencentcloud.com/document/product/1047/48178). |
| nameCard | Карточка имени группы отправителя | Применяется только к групповым сообщениям. Например, если Alice изменяет свою карточку имени в группе на `doctorA` и отправляет сообщение в группу, другие члены группы получат сообщение, где поле `nameCard` это `doctorA`. Получатель может отдать приоритет этому полю как имя пользователя для отображения. `nameCard` может быть установлено путём вызова `setGroupMemberInfo`. |
| faceURL | Фотография профиля отправителя сообщения | Может быть установлена пользователями и может использоваться для загрузки фото. |
| groupID | ID группы | `groupID` это ID группы в групповом сообщении или `nil` в личном сообщении. |
| userID | ID пользователя | `userID` это ID получателя в личном сообщении или `nil` в групповом сообщении. |
| seq | Серийный номер сообщения | Значения `seq` для личных сообщений генерируются локально и могут быть не в порядке возрастания и не уникальны; значения `seq` для групповых сообщений генерируются сервером и находятся в порядке возрастания и уникальны в группе. |
| random | Случайный код сообщения | Генерируется в SDK. |
| status | Статус отправки сообщения | В настоящее время поддерживаются следующие статусы: отправка, успешно отправлено, ошибка отправки, удалено, импортировано локально и отозвано. |
| isSelf | Было ли сообщение отправлено самим собой | Может использоваться для фильтрации сообщений. |
| needReadReceipt | Требуется ли сообщению подтверждение прочтения | Устанавливается отправителем и действительно для Pro edition, Pro Plus edition, Enterprise edition на v6.1 или позже. Дополнительные сведения см. в разделе [Подтверждение прочтения](https://intl.cloud.tencent.com/document/product/1047/48022). |
| priority | Приоритет сообщения | Применяется только к групповым сообщениям и может быть установлено путём вызова API `sendMessage`. |
| groupAtUserList | Список пользователей, упомянутых в групповом сообщении | Применяется только к групповым сообщениям и хранит `userID`. Дополнительные сведения см. в разделе [Групповое @ сообщение](https://intl.cloud.tencent.com/document/product/1047/48027). |
| elemType | Тип сообщения | В настоящее время поддерживаются следующие типы сообщений: текст, пользовательский, изображение, аудио, видео, файл, географическое местоположение, эмодзи, подсказка группы и объединённые сообщения. Дополнительные сведения см. в разделе [Типы сообщений](https://www.tencentcloud.com/document/product/1047/47989?lang=en&pg=#message-types). |
| textElem | Элемент хранилища текстового сообщения | Требуется создать и заполнить содержимое при отправке текстового сообщения. При получении сообщения и идентификации `elemType` как `TEXT`, содержимое можно проанализировать из `textElem`. Дополнительные сведения см. в разделах [Отправка сообщения](https://intl.cloud.tencent.com/document/product/1047/47994) / [Получение сообщения](https://intl.cloud.tencent.com/document/product/1047/47995). |
| customElem | Элемент хранилища пользовательского сообщения | Требуется создать и заполнить содержимое при отправке пользовательского сообщения. При получении сообщения и идентификации `elemType` как `CUSTOM`, содержимое можно проанализировать из `customElem`. |
| imageElem | Элемент хранилища сообщения с изображением | Требуется создать и заполнить содержимое при отправке сообщения с изображением. При получении сообщения и идентификации `elemType` как `IMAGE`, содержимое можно проанализировать из `imageElem`. |
| soundElem | Элемент хранилища аудиосообщения | Требуется создать и заполнить содержимое при отправке аудиосообщения. При получении сообщения и идентификации `elemType` как `SOUND`, содержимое можно проанализировать из `soundElem`. |
| videoElem | Элемент хранилища видеосообщения | Требуется создать и заполнить содержимое при отправке видеосообщения. При получении сообщения и идентификации `elemType` как `VIDEO`, содержимое можно проанализировать из `videoElem`. |
| fileElem | Элемент хранилища сообщения с файлом | Требуется создать и заполнить содержимое при отправке сообщения с файлом. При получении сообщения и идентификации `elemType` как `FILE`, содержимое можно проанализировать из `fileElem`. |
| locationElem | Элемент хранилища сообщения с географическим местоположением | Требуется создать и заполнить содержимое при отправке сообщения с географическим местоположением. При получении сообщения и идентификации `elemType` как `LOCATION`, содержимое можно проанализировать из `locationElem`. |
| faceElem | Элемент хранилища сообщения с эмодзи | Требуется создать и заполнить содержимое при отправке сообщения с эмодзи. При получении сообщения и идентификации `elemType` как `FACE`, содержимое можно проанализировать из `faceElem`. |
| mergerElem | Элемент хранилища объединённого сообщения | Требуется создать и заполнить содержимое при отправке объединённого сообщения. При получении сообщения и идентификации `elemType` как `MERGER`, содержимое можно проанализировать из `mergerElem`. Дополнительные сведения см. в разделе [Переадресация сообщения](https://intl.cloud.tencent.com/document/product/1047/48001). |
| groupTipsElem | Элемент хранилища сообщения с подсказкой группы | В настоящее время сообщение с подсказкой группы может быть отправлено только сервером. При получении сообщения SDK и идентификации `elemType` как `GROUP_TIPS`, содержимое можно проанализировать из `groupTipsElem`. |
| cloudCustomData | Пользовательские данные сообщения | Устанавливаются отправителем и могут быть настроены. Сохраняются в облаке, отправляются получателю и могут по-прежнему быть получены после удаления и переустановки приложения. |
| isExcludedFromUnreadCount | Исключено ли сообщение из счёта непрочитанных сообщений в беседе | Устанавливается отправителем. По умолчанию сообщения включаются в счёт непрочитанных сообщений. Это поле поддерживается v5.3.425 или позже. |
| isExcludedFromLastMessage | Исключено ли сообщение из `lastMsg` беседы | Устанавливается отправителем. По умолчанию сообщения включаются в `lastMsg` беседы. Это поле поддерживается v5.4.666 или позже. Дополнительные сведения см. в разделе [Список бесед](https://intl.cloud.tencent.com/document/product/1047/48326). |
| isExcludedFromContentModeration | Исключено ли сообщение из модерирования содержимого (облачной модерации) | Устанавливается отправителем. По умолчанию сообщения включаются в модерирование содержимого. Это поле поддерживается v7.1 или позже. Параметры будут эффективны только при включении функции облачной модерации. |
| hasRiskContent | Определено ли сообщение как рискованное | Поддерживаются только сообщения, содержащие звук и видео. Вступит в силу после включения функции облачной модерации. Если вы отправляете рискованное сообщение со звуком или видео, вы получите обратный вызов `onRecvMessageModified` с этим полем, установленным в true после облачной модерации. Это поле поддерживается v7.4 или позже. |
| disableCloudMessagePreHook | Отключить ли облачный хук сообщения перед отправкой | Устанавливается отправителем. По умолчанию облачный хук сообщения перед отправкой включён. Это поле поддерживается v8.1 или позже. |
| disableCloudMessagePostHook | Отключить ли облачный хук сообщения после отправки | Устанавливается отправителем. По умолчанию облачный хук сообщения после отправки включён. Это поле поддерживается v8.1 или позже. |
| isRead | Прочитано ли сообщение на локальном конце | Сообщение считается прочитанным по умолчанию, если оно отправлено самим собой. |
| isPeerRead | Прочитано ли сообщение получателем | Применяется только к личным сообщениям. |
| localCustomData | Пользовательские данные сообщения | Устанавливаются отправителем. Сохраняются локально, не отправляются получателю и становятся недействительными после удаления и переустановки приложения. |
| localCustomInt | Пользовательские данные сообщения | Устанавливаются отправителем. Сохраняются локально, не отправляются получателю и становятся недействительными после удаления и переустановки приложения. Может использоваться для отметки того, было ли воспроизведено аудио или видеосообщение. |

## Типы сообщений

Сообщения можно классифицировать по пункту назначения сообщения на два типа: личные сообщения (также называемые C2C сообщениями) и групповые сообщения.

| Тип сообщения | Ключевое слово API | Описание |
| --- | --- | --- |
| Личное сообщение | C2CMessage | Также называется C2C сообщением. При отправке личного сообщения вы должны указать `UserID` получателя, и только получатель может получить это сообщение. |
| Групповое сообщение | GroupMessage | При отправке группового сообщения вы должны указать `groupID` целевой группы, и все пользователи в группе могут получить это сообщение. |

Сообщения чата также можно классифицировать по содержимому на текстовые сообщения, пользовательские (сигнальные) сообщения, сообщения с изображениями, видеосообщения, аудиосообщения, сообщения с файлами, сообщения с местоположением и объединённые сообщения. Существует ограничение на размер файлов сообщений. Дополнительные сведения см. в разделе [Ограничения использования](https://www.tencentcloud.com/document/product/1047/34381).

| Тип сообщения | Ключевое слово API | Описание |
| --- | --- | --- |
| Текстовое сообщение | TextElem | Обычное текстовое сообщение |
| Пользовательское сообщение | CustomElem | Это раздел двоичного буфера и часто используется для передачи пользовательского сигнала в вашем приложении. |
| Сообщение с изображением | ImageElem | Когда SDK отправляет исходное изображение, он автоматически генерирует два изображения разных размеров. Эти три изображения называются исходным изображением, большим изображением и миниатюрой. SDK поддерживает максимальный размер изображения 28 МБ для любого одного отправляемого файла. |
| Видеосообщение | VideoElem | Видеосообщение содержит видеофайл и миниатюру. |
| Аудиосообщение | SoundElem | Поддерживает отображение красной точки перед воспроизведением аудиосообщения. |
| Сообщение с файлом | FileElem | Сообщение с файлом не может превышать 100 МБ. |
| Сообщение с местоположением | LocationElem | Сообщение с местоположением содержит три поля: описание местоположения, долготу и широту. |
| Объединённое сообщение | MergerElem | Подходит для объединения и переадресации истории чата. Может быть объединено до 300 сообщений. |

## Стратегия хранения сообщений

Сообщения чата можно классифицировать по стратегии хранения сообщений на два типа: онлайн-сообщения и оффлайн-сообщения.

Онлайн-сообщения могут получить только пользователи, находящиеся в сети, и не будут отправляться, когда они оффлайн. Оффлайн-сообщения могут получить пользователи как в сети, так и оффлайн.

Онлайн-сообщения доставляются в реальном времени и не хранятся на сервере или в SDK. Поэтому их невозможно получить из истории сообщений после изменения устройства или удаления и переустановки приложения.

> **Примечание** Все сообщения в аудио-видео группе являются онлайн-сообщениями. Сообщения, отправленные всем пользователям, являются онлайн-сообщениями.

Оффлайн-сообщения хранятся в SDK и на сервере. По умолчанию они хранятся на сервере перемещения в течение семи дней. Если вы хотите более длительный период хранения, вам необходимо приобрести платную услугу. Дополнительные сведения о содержании услуги и выставлении счётов см. в разделе [Цены](https://trtc.io/pricing/chat).

Эти сообщения можно получить из истории сообщений после изменения устройства или удаления и переустановки приложения.

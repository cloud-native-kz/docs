# Отправка онлайн-сообщения

## Обзор

В некоторых случаях может потребоваться, чтобы сообщение было получено получателем только при наличии подключения; то есть получатель не будет видеть сообщение в автономном режиме.
Вам нужно только установить `onlineUserOnly` в `true/YES` при вызове `sendMessage`. Сообщение, отправленное таким образом, отличается от обычного тем, что:

1. Оно не может быть сохранено в автономном режиме; то есть оно не может быть получено, если получатель находится в автономном режиме.
2. Оно не может синхронизироваться между устройствами; то есть если оно получено на одном устройстве, оно не может быть получено на другом, независимо от того, было ли оно прочитано или нет.
3. Оно не может быть сохранено локально; то есть его нельзя получить из локальных или облачных исторических сообщений.

## Пример

### Реализация индикатора набора текста

В личных чатах вы можете использовать `sendMessage` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMMessageManager.html#a28e01403acd422e53e999f21ec064795) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMManager+Message.html#v2timmanager.sendmessage(message:receiver:groupid:priority:onlineuseronly:offlinepushinfo:progress:succ:fail:)) /  [Objective-C](https://im.sdk.qcloud.com/doc/en/categoryV2TIMManager_07Message_08.html#a681947465d6ab718da40f7f983740a21) / [C++](https://im.sdk.qcloud.com/doc/en/classV2TIMMessageManager.html#a42db237e7ae52cd2aa7edebf4f435c61)) для отправки указанного пользовательского онлайн-сообщения. После получения указанного пользовательского онлайн-сообщения и определения того, что отправитель печатает, получатель может отобразить на пользовательском интерфейсе "Печатает...".

Фактически, реализация индикатора набора текста с помощью SDK для отправки онлайн-сообщений — это лишь небольшая часть работы. Требуется гораздо больше работ по разработке, например:

- Мониторинг пользовательского интерфейса и обновление статуса: прослушиватель событий добавляется в поле ввода для отправки сообщений, чтобы обнаружить начало ввода текста пользователем. Когда начинается ввод, клиент обновляет статус пользователя, например "печатает".
- Отправка статуса на сервер: в соответствующее время статус ввода пользователя отправляется на сервер.
- Обновление пользовательского интерфейса: на основе полученного статуса ввода на пользовательском интерфейсе получателя отображается индикатор, например "Daniel печатает". Это может быть текстовое уведомление, значок или анимация.
- Рассмотрение дросселирования: чтобы уменьшить ненужные обновления статуса и снизить нагрузку на сеть, можно установить временной интервал. В пределах этого интервала отправляется только одно обновление статуса ввода.

# Получение сообщения

## Описание функции

- API `addSimpleMsgListener` используется для прослушивания и получения текстовых и пользовательских сообщений с обратным вызовом, определенным в протоколе `V2TIMSimpleMsgListener`.
- API `addAdvancedMsgListener` используется для прослушивания и получения всех типов сообщений (текстовых, пользовательских и мультимедийных сообщений) с обратным вызовом, определенным в протоколе `V2TIMAdvancedMsgListener`.

## Установка слушателя сообщений

SDK предоставляет простой слушатель сообщений `V2TIMSimpleMsgListener` и продвинутый слушатель сообщений `V2TIMAdvancedMsgListener`.
Они отличаются тем, что:

1. Простой слушатель сообщений **может получать только** текстовые и пользовательские сообщения. Вы можете использовать его, если ваш бизнес требует только эти два типа сообщений.
2. Продвинутый слушатель сообщений может получать **все** типы сообщений. Вы можете использовать его, если ваш бизнес также требует мультимедийные, объединенные и другие сообщения.

> **Примечание:** `addSimpleMsgListener` и `addAdvancedMsgListener` являются взаимоисключающими. **Не используйте их одновременно**; в противном случае возникнут непредсказуемые логические ошибки. Слушатель сообщений **должен** быть добавлен в первую очередь для получения следующих типов сообщений.

## Получение сообщения с мультимедийным контентом

Получатель может получить сообщение с мультимедийным контентом **только используя** расширенный прослушиватель сообщений следующим образом:

1. Вызовите API `addAdvancedMsgListener` для установки расширенного прослушивателя сообщений.
2. Прослушивайте обратный вызов `onRecvNewMessage` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMAdvancedMsgListener.html#a6771cfa1a897e24b05c17788aba15ff6) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMAdvancedMsgListener.html#v2timadvancedmsglistener.onrecvnewmessage(msg:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/protocolV2TIMAdvancedMsgListener-p.html#ae36dde0fab51c29f6d1620ab4b13d7a3) / [C++](https://im.sdk.qcloud.com/doc/en/classV2TIMAdvancedMsgListener.html#acbd1dba3526377d88f14c2183901511c)) для получения сообщения `V2TIMMessage`.
3. Разберите атрибут `elemType` в сообщении `V2TIMMessage`, затем снова разберите сообщение в соответствии с типом, чтобы получить конкретный контент элементов в сообщении.

### Сообщение с изображением

Изображение в сообщении с изображением может быть в трёх форматах: оригинальное, крупное и эскиз. Последние два автоматически создаются SDK при отправке сообщения и могут быть проигнорированы.

- Крупное изображение: Крупное изображение — это изображение, полученное после пропорционального сжатия исходного изображения. После сжатия высота или ширина (в зависимости от того, что меньше) равна 720 пикселям.
- Эскиз: Эскиз — это изображение, полученное после пропорционального сжатия исходного изображения. После сжатия высота или ширина (в зависимости от того, что меньше) равна 198 пикселям.

После получения сообщения с изображением рекомендуется вызвать функцию SDK `downloadImage` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMImageElem_1_1V2TIMImage.html#a9114c16b095bbf608027c71eb40e9025) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMImage.html#v2timimage.downloadimage(path:progress:succ:fail:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/interfaceV2TIMImage.html#a2ef83a4fb5c2a047acaf881ee90df58e) / [C++](https://im.sdk.qcloud.com/doc/en/structV2TIMImage.html#a68989f22baa77f5c4700be434d5d7fa9)) для загрузки изображения, а затем визуализировать его на уровне пользовательского интерфейса.

## Получение трансляционного сообщения

Вы можете использовать только [Трансляционное сообщение аудиовизуальной группы](https://www.tencentcloud.com/document/product/1047/49440) для отправки трансляционных сообщений всем AVChatRooms с одним и тем же AppID, и все участники группы в сети в AVChatRooms могут получить это сообщение. SDK не может отправлять трансляционные сообщения, он может только получать сообщения из REST API. Получение сообщений по-прежнему происходит в уведомлении `onRecvNewMessage`, упомянутом ранее. После получения сообщения вы можете определить, является ли сообщение трансляционным, на основе свойства `isBroadcastMessage`.

> **Примечание**
> Для использования этой функции необходимо [приобрести издание Pro, Pro Plus или Enterprise](https://trtc.io/buy/chat). Эта функция доступна только в SDK расширенном издании v6.5.2803 или более поздней версии. Трансляционное сообщение поддерживается только в AVChatRoom, другие типы групп временно не поддерживаются.

## Получение сообщения, содержащего несколько элементов

1. Проанализируйте первый объект элемента через объект Message.
2. Получите следующий объект элемента через метод `nextElem` первого объекта элемента. Если следующий объект существует, возвращается экземпляр объекта элемента; в противном случае возвращается `nil` или `null`.

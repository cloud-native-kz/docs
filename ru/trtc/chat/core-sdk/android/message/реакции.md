# Реакции

## Описание функции

Функция ответа на сообщение относится к взаимодействию с конкретным сообщением, типичный сценарий — эмодзи ответы. Эмодзи ответы — это интерактивные ответы с использованием пиктограмм, и мы можем видеть количество ответивших пользователей и список ответивших для каждого эмодзи.

> **Примечание:** Эта функция доступна только для клиентов Pro edition, Pro Plus edition или Enterprise edition. Вы можете использовать её после приобретения [Pro edition, Pro Plus edition или Enterprise edition](https://trtc.io/buy/chat). Эта функция поддерживается только Enhanced SDK версии 7.4 и выше

## Описание API

Вы можете реализовать возможности эмодзи ответов на основе API SDK:

- Вызовите интерфейс `addMessageReaction` для добавления эмодзи к сообщению. После успешного добавления текущий пользователь будет сохранен под эмодзи.
- Вызовите интерфейс `removeMessageReaction` для удаления добавленного эмодзи. После успешного удаления текущий пользователь больше не будет сохранен под эмодзи.
- Вызовите интерфейс `getMessageReactions` для массового получения списка эмодзи для нескольких сообщений. Каждый эмодзи содержит общее количество текущих пользователей и первые N (по умолчанию 10) профилей пользователей.
- Вызовите интерфейс `getAllUserListOfMessageReaction` для постраничного получения полного списка профилей пользователей, использующих эмодзи сообщения.
- Прослушивайте callback `onRecvMessageReactionsChanged` для отслеживания изменений профиля пользователя эмодзи. Callback будет содержать последний профиль пользователя эмодзи (включая общее количество пользователей и первые N профилей пользователей).

Для получения подробных инструкций по использованию обратитесь к следующему тексту.

### Добавить реакцию на сообщение

Вызов интерфейса `addMessageReaction` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMMessageManager.html#af47c3e1fb1ac73f7e2ba7bf739c9452a) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMManager+Message.html#v2timmanager.addmessagereaction(message:reactionid:succ:fail:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/categoryV2TIMManager_07Message_08.html#a425d818c392bef6c6daeddef8c9c0cc1) / [C++](https://im.sdk.qcloud.com/doc/en/classV2TIMMessageManager.html#a3c961027e9f295e472219116a3c8f90e)) позволяет добавить реакцию на сообщение.

Объяснение входных параметров интерфейса добавления реакции на сообщение выглядит следующим образом:

| Входной параметр | Определение | Описание |
| --- | --- | --- |
| message | Объект сообщения | Сообщение должно находиться в статусе успешно отправленного. |
| reactionID | ID реакции на сообщение | В сценарии эмодзи ответа reactionID — это ID эмодзи. |

> **Примечание:** Одно сообщение поддерживает до 10 реакций, и одна реакция поддерживает до 100 пользователей. Если общее количество реакций в одном сообщении превышает максимальное ограничение, интерфейс сообщит об ошибке ERR_SVR_MSG_REACTION_COUNT_LIMIT. Если общее количество пользователей в одной реакции превышает максимальное ограничение, интерфейс сообщит об ошибке ERR_SVR_MSG_REACTION_USER_COUNT_LIMIT. Если реакция уже содержит текущего пользователя, интерфейс сообщит об ошибке ERR_SVR_MSG_REACTION_ALREADY_CONTAIN_USER.

### Удалить реакцию на сообщение

Вызов интерфейса `removeMessageReaction` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMMessageManager.html#a8af85ef9a47c6d498c601dad1370de32) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMManager+Message.html#v2timmanager.removemessagereaction(message:reactionid:succ:fail:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/categoryV2TIMManager_07Message_08.html#a88d3bf59edf75dd06ef9067b08b7db00)/ [C++](https://im.sdk.qcloud.com/doc/en/classV2TIMMessageManager.html#a4fa45bdf386beefc5544935eb6122649)) позволяет удалить реакцию на сообщение.

### Получить реакции на сообщение

Вызов интерфейса `getMessageReactions` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMMessageManager.html#aa6d5f0421950c7354dd4f0de48814881) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMManager+Message.html#v2timmanager.getmessagereactions(messagelist:maxusercountperreaction:succ:fail:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/categoryV2TIMManager_07Message_08.html#ab3d6d68379e25750ba1b0154d956e62c)/ [C++](https://im.sdk.qcloud.com/doc/en/classV2TIMMessageManager.html#a4535e7faaba45905ec4b5d0a658935cb)) позволяет получить реакции нескольких сообщений.

Объяснение входных параметров интерфейса получения реакций на сообщение выглядит следующим образом:

| Входной параметр | Определение | Описание |
| --- | --- | --- |
| messageList | Список объектов сообщений | Поддерживает до 20 сообщений одновременно. |
| maxUserCountPerReaction | Максимальное количество профилей пользователей, возвращаемых для каждой реакции | Диапазон значений: [0,10]. Каждая реакция поддерживает возврат не более 10 профилей пользователей. Если вам нужно больше профилей пользователей, вы можете вызвать интерфейс `getAllUserListOfMessageReaction` для постраничного получения. |

Объяснение объекта результата получения информации о реакциях на сообщение `V2TIMMessageReactionResult` выглядит следующим образом:

| Входной параметр | Определение | Описание |
| --- | --- | --- |
| resultCode | Код результата | 0 — указывает успех. Ненулевое значение — указывает ошибку. |
| resultInfo | Информация о результате | Информация о результате. |
| msgID | Уникальный ID сообщения | Уникальный ID сообщения. |
| reactionList | Список реакций на сообщение | Список объектов реакций на сообщение `V2TIMMessageReaction`. |

Подробное объяснение объекта `V2TIMMessageReaction` выглядит следующим образом:

| Входной параметр | Определение | Описание |
| --- | --- | --- |
| reactionID | ID реакции на сообщение | В сценарии эмодзи ответа reactionID — это ID эмодзи. |
| totalUserCount | Общее количество пользователей | Общее количество пользователей, добавивших реакцию с тем же ID к сообщению. |
| partialUserList | Частичный список пользователей | Частичный список пользователей, добавивших реакцию с тем же ID к сообщению. Количество пользователей зависит от значения maxUserCountPerReaction, установленного при вызове интерфейса `getMessageReactions`. |
| reactedByMyself | Использовал ли я эту реакцию | Это поле возвращает true, если я использовал реакцию. |

### Уведомление об изменении реакции на сообщение

Если вы предварительно добавили прослушиватель расширенных событий сообщения, вызвав `addAdvancedMsgListener`, вы получите callback `onRecvMessageReactionsChanged` ([Java](https://im.sdk.qcloud.com/doc/en/classcom_1_1tencent_1_1imsdk_1_1v2_1_1V2TIMAdvancedMsgListener.html#ac586fecabcd833cb2d10a4b11600cf91) / [Swift](https://im.sdk.qcloud.com/doc/en/swift_V2TIMAdvancedMsgListener.html#v2timadvancedmsglistener.onrecvmessagereactionschanged(changelist:)) / [Objective-C](https://im.sdk.qcloud.com/doc/en/protocolV2TIMAdvancedMsgListener-p.html#a28e76f71176d3413db0f4e525e2f3804) / [C++](https://im.sdk.qcloud.com/doc/en/classV2TIMAdvancedMsgListener.html#abd17cfaf5262b2cda8fa6b74066e94cd)) при обновлении информации о реакции на сообщение. Следует отметить, что этот callback является добавочным callback для реакции сообщения и будет нести только измененную информацию о реакции. Когда значение поля totalUserCount в измененной информации о реакции равно 0, это означает, что ни один пользователь не использует эту реакцию, и вы можете удалить отображение этой реакции в пользовательском интерфейсе.

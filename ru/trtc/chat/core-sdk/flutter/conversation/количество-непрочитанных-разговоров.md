# Количество непрочитанных сообщений в беседе

## Описание функции

Список бесед пользователя обычно содержит несколько бесед. Если в одной из бесед есть новое сообщение, в ячейке списка должен отображаться значок, указывающий на количество непрочитанных сообщений.
После того как пользователь входит в беседу и возвращается в список бесед, количество непрочитанных сообщений очищается, и значок исчезает.
В некоторых приложениях рассчитывается и отображается общее количество непрочитанных сообщений во всех бесед в нижней вкладке списка бесед.

В этом документе описано, как реализовать функцию уведомления о количестве непрочитанных сообщений в беседе.

## Получение общего количества непрочитанных сообщений во всех бесед

В общих случаях для получения общего количества непрочитанных сообщений во всех бесед можно пройти по списку бесед, получить информацию `V2TimConversation` каждой беседы и сложить значения `unreadCount` всех объектов `V2TimConversation`, чтобы получить окончательный результат и отобразить его в интерфейсе.

Chat SDK предоставляет API `getTotalUnreadMessageCount` для запроса общего количества непрочитанных сообщений во всех бесед.
Когда общее количество непрочитанных сообщений изменится, SDK уведомит вас о последнем количестве непрочитанных сообщений через обратный вызов `onTotalUnreadMessageCountChanged`.

> **Примечание:** Эта функция поддерживается только в SDK для Flutter версии 3.0.0 или позже.

Ниже приведены подробные шаги.

### Получение общего количества непрочитанных сообщений

Вызовите `getTotalUnreadMessageCount` ([Подробнее](https://pub.dev/documentation/tencent_im_sdk_plugin_platform_interface/latest/im_flutter_plugin_platform_interface/ImFlutterPlatform/getTotalUnreadMessageCount.html)), чтобы получить общее количество непрочитанных сообщений во всех бесед и обновить его в интерфейсе.

Пример кода:

```
// Получить общее количество непрочитанных сообщений
V2TimValueCallback<int> unread = await conversationManager.getTotalUnreadMessageCount();
```

### Уведомление об изменении общего количества непрочитанных сообщений

Вызовите `addConversationListener` ([Подробнее](https://pub.dev/documentation/tencent_im_sdk_plugin_platform_interface/latest/im_flutter_plugin_platform_interface/ImFlutterPlatform/addConversationListener.html)), чтобы добавить прослушиватель бесед для получения уведомлений об изменении общего количества непрочитанных сообщений.

Вы можете получить измененное общее количество непрочитанных сообщений в `onTotalUnreadMessageCountChanged` ([Подробнее](https://pub.dev/documentation/tencent_im_sdk_plugin_platform_interface/latest/enum_callbacks/OnTotalUnreadMessageCountChanged.html)) объекта `V2TIMConversationListener`.

Пример кода:

```
conversationManager.addConversationListener(listener: V2TimConversationListener(onTotalUnreadMessageCountChanged: (totalUnreadCount) {    // Последнее количество непрочитанных сообщений  },));
```

## Очистка количества непрочитанных сообщений в беседе

После того как пользователь входит в беседу и возвращается в список бесед, количество непрочитанных сообщений должно быть очищено, после чего значок в списке бесед должен исчезнуть.

Вы можете вызвать cleanConversationUnreadMessageCount ([Подробнее](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_conversation_manager/V2TIMConversationManager/cleanConversationUnreadMessageCount.html)), чтобы очистить количество непрочитанных сообщений в указанной беседе. Передав параметры в определенном формате, вы можете отдельно очистить количество непрочитанных сообщений в указанной личной беседе, всех личных бесед, указанной групповой беседе, всех групповых бесед и всех бесед. Конкретные шаги операции следующие.

> **Примечание:** Эта функция поддерживается только в SDK Enhanced Edition версии 8.0 или позже.

Описание параметров:

| Параметр | Значение | Действителен в личной беседе | Действителен в групповой беседе | Примечания |
| --- | --- | --- | --- | --- |
| conversationID | Уникальный ID беседы | ДА | ДА | Метод формирования личной беседы (C2C): "c2c_userID"; метод формирования групповой беседы: "group_groupID". |
| cleanTimestamp | Временная отметка очистки | ДА | НЕТ | Укажите временную отметку, до которой будет очищено количество непрочитанных сообщений. Если передать 0, все непрочитанные сообщения в соответствующей беседе будут очищены, и количество непрочитанных сообщений в беседе будет установлено в 0. |
|  cleanSequence | Последовательность очистки | НЕТ | ДА | Укажите последовательность, до которой будет очищено количество непрочитанных сообщений. Если передать 0, все непрочитанные сообщения в соответствующей беседе будут очищены, и количество непрочитанных сообщений в беседе будет установлено в 0. |

> **Примечание:** Когда вы хотите очистить количество непрочитанных сообщений всех личных бесед, передайте "c2c" в conversationID, то есть не указывайте конкретный userID. Когда вы хотите очистить количество непрочитанных сообщений всех групповых бесед, передайте "group" в conversationID, то есть не указывайте конкретный groupID. Когда вы хотите очистить количество непрочитанных сообщений всех бесед, передайте "" в conversationID. После успешного вызова этого API SDK уведомит вас о последнем количестве непрочитанных сообщений соответствующей беседы через обратный вызов onConversationChanged.

### Очистка количества непрочитанных сообщений в указанной личной беседе

Вы можете очистить количество непрочитанных сообщений в указанной личной беседе, передав conversationID с префиксом "c2c_". Вы также можете очистить количество непрочитанных сообщений до указанной временной отметки, передав cleanTimestamp. Если переданный cleanTimestamp равен 0, количество непрочитанных сообщений в указанной личной беседе будет очищено до 0.

Пример кода:

```
cleanConversationUnreadMessageCount
```

После успешного вызова `cleanConversationUnreadMessageCount`:

1. Если вызывающее приложение вызвало `addConversationListener` для добавления прослушивателя бесед, оно получит обратный вызов `onConversationChanged` и обновит интерфейс.
2. Отправитель получит обратный вызов `onRecvC2CReadReceipt`, содержащий временную отметку, когда было очищено количество непрочитанных сообщений в беседе.

Пример кода:

```
// Получатель
conversationManager.addConversationListener(listener: V2TimConversationListener(,onConversationChanged: (conversationList) {    // Последняя беседа после изменения  },));
// Отправитель
TencentImSDKPlugin.v2TIMManager.getMessageManager().addAdvancedMsgListener(listener: V2TimAdvancedMsgListener(onRecvC2CReadReceipt: (receiptList) {    // Сообщение прочитано получателем.  },));
```

### Очистка количества непрочитанных сообщений всех личных бесед

Вы можете передать "c2c" в качестве conversationID, указывая, что количество непрочитанных сообщений всех личных бесед будет очищено до 0. Обратите внимание, что cleanTimestamp больше не будет действителен в этом случае.

Пример кода:

```
cleanConversationUnreadMessageCount
```

После успешного вызова `cleanConversationUnreadMessageCount`, если вызывающее приложение вызвало `addConversationListener` для добавления прослушивателя бесед, оно получит обратный вызов `onConversationChanged` и обновит интерфейс.

Пример кода:

```

```

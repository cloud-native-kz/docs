# Поиск сообщений

## Описание функции

Локальный поиск сообщений — это функция, необходимая для улучшения взаимодействия пользователей с приложением. Она позволяет пользователям быстро и удобно найти нужную информацию в большом объеме сложных данных. Её также можно использовать в качестве инструмента управления для удобной и эффективной навигации по обширному контенту.

> **Примечание:** Поиск доступен только для локальных сообщений, например полученных сообщений или исторических сообщений, полученных после вызова API. Эта функция поддерживается только SDK для Flutter версии 3.8.0 и выше. Функция локального поиска сообщений доступна только в выпуске Chat Ultimate. Для её использования приобретите [выпуск Ultimate](https://www.tencentcloud.com/document/product/1047/34577#.E5.8D.87.E7.BA.A7.E5.BA.94.E7.94.A8). Дополнительные сведения см. в разделе [Тарификация](https://intl.cloud.tencent.com/document/product/1047/34350).

## Класс поиска сообщений

### Класс параметров поиска сообщений

Класс параметров поиска сообщений — это `V2TimMessageSearchParam` ([Подробности](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/models_v2_tim_message_search_param/V2TimMessageSearchParam-class.html)). При поиске сообщений SDK будет выполнять различную логику поиска в зависимости от параметров объекта.

Параметры `V2TIMMessageSearchParam` описаны ниже:

| Параметр | Определение | Описание |
| --- | --- | --- |
| keywordList | Список ключевых слов | Может содержать до пяти ключевых слов. Если отправитель сообщения и тип сообщения не указаны, список ключевых слов должен быть установлен; в других случаях может быть пуст. |
| keywordListMatchType | Тип совпадения списка ключевых слов | Может быть установлено значение `OR` или `AND`. Допустимые значения: V2TIM_KEYWORD_LIST_MATCH_TYPE_OR (по умолчанию), V2TIM_KEYWORD_LIST_MATCH_TYPE_AND. |
| senderUserIDList | `userID` отправителя сообщений | Может содержать до пяти идентификаторов пользователей. |
| messageTypeList | Набор типов сообщений для поиска | Если это поле оставить пустым, будут искаться сообщения всех поддерживаемых типов (исключая `V2TIMFaceElem` и `V2TIMGroupTipsElem`). Дополнительные сведения о значениях других типов см. в разделе `V2TIMElemType` ([Подробности](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/enum_message_elem_type/MessageElemType-class.html)). |
| conversationID | Искать во всех диалогах или в указанном диалоге | Если оставить пусто, будут выполнены поиск во всех диалогах; в противном случае будет выполнен поиск в указанном диалоге. |
| searchTimePosition | Время начала поиска | По умолчанию имеет значение `0`, что указывает на поиск с текущего момента времени. Это UTC-временная метка в секундах. |
| searchTimePeriod | Период времени в прошлом, начиная с времени начала | По умолчанию имеет значение `0`, что означает отсутствие ограничения временного диапазона. Измеряется в секундах. `24x60x60` представляет прошлый день. |
| pageIndex | Номер страницы | Используется для отображения результатов поиска по страницам. `0` указывает на первую страницу. |
| pageSize | Количество элементов результата на странице | Используется для отображения результатов поиска по страницам. Установите значение `0`, если вы не хотите постраничное отображение. Если одновременно получить слишком много элементов результата, может возникнуть проблема производительности. |

### Класс результатов поиска сообщений

Класс результатов поиска сообщений — это `V2TIMMessageSearchResult` ([Подробности](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/models_v2_tim_message_search_result/V2TimMessageSearchResult-class.html)). Параметры описаны ниже:

| Параметр | Определение | Описание |
| --- | --- | --- |
| totalCount | Общее количество элементов результатов поиска | Если выполняется поиск в указанном диалоге, будет возвращено **общее количество сообщений**, соответствующих критериям поиска. Если выполняется поиск во всех диалогах, будет возвращено **общее количество диалогов**, которым принадлежат сообщения, соответствующие критериям поиска. |
| messageSearchResultItems | Тип совпадения списка ключевых слов | Если выполняется поиск в указанном диалоге, в возвращаемый список результатов будут включены только элементы результатов в этом диалоге. Если выполняется поиск во всех диалогах, сообщения, соответствующие критериям поиска, будут сгруппированы по идентификатору диалога и возвращены по страницам. |

Здесь `messageSearchResultItems` — это список, содержащий объекты `V2TIMMessageSearchResultItem` ([Подробности](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/models_v2_tim_message_search_result_item/V2TimMessageSearchResultItem-class.html)). Параметры описаны ниже:

| Параметр | Определение | Описание |
| --- | --- | --- |
| conversationID | Идентификатор диалога | – |
| messageCount | Количество сообщений | Количество сообщений, соответствующих критериям поиска и найденных в текущем диалоге. |
| messageList | Список сообщений, соответствующих критериям поиска | Если выполняется поиск в указанном диалоге, `messageList` — это список сообщений в диалоге, соответствующих критериям поиска. Если выполняется поиск во всех диалогах, `messageList` может отображать один из следующих результатов: Если количество совпадающих сообщений в диалоге больше 1, `messageList` пусто, и на пользовательском интерфейсе можно отобразить "{messageCount} соответствующих записей". Если количество совпадающих сообщений в диалоге равно 1, `messageList` — это совпадающее сообщение, которое можно отобразить на пользовательском интерфейсе и выделить совпадающее ключевое слово. |

## Поиск сообщений во всех диалогах

Когда пользователь вводит ключевое слово в поле поиска для поиска сообщений, можно вызвать `searchLocalMessages` ([Подробности](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_message_manager/V2TIMMessageManager/searchLocalMessages.html)) для поиска локальных сообщений SDK Chat.

Если вы хотите выполнить поиск во всех диалогах, вам нужно только оставить `conversationID` в `V2TIMMessageSearchParam` пустым (`null`/`nil`) или не указывать его.

Пример кода:

```
// Search for local messages by keywordV2TimValueCallback<V2TimMessageSearchResult> searchMessage = await messageManager.searchLocalMessages(searchParam: V2TimMessageSearchParam(keywordList: ['Keyword 1'],pageIndex: 0,pageSize: 10,type: 1));
```

## Поиск сообщений в указанном диалоге

Когда пользователь вводит ключевое слово в поле поиска для поиска сообщений, можно вызвать `searchLocalMessages` для поиска локальных сообщений SDK Chat.

Пример кода:

```
// Search for local messages by conversation ID and keywordV2TimValueCallback<V2TimMessageSearchResult> searchMessage = await messageManager.searchLocalMessages(searchParam: V2TimMessageSearchParam(keywordList: ['Keyword 1'],pageIndex: 0,pageSize: 10,type: 1,conversationID:'conversationID'));
```

### Отображение последних активных диалогов

Как показано на рисунке 1, внизу отображается список последних трёх диалогов, которым принадлежат найденные сообщения. Метод реализации следующий:

1. Установите параметр поиска `V2TIMMessageSearchParam`.
  - Установите `conversationID` на `null`, что означает поиск во всех диалогах.
  - Установите `pageIndex` на `0`, что означает первую страницу диалогов, которым принадлежат найденные сообщения.
  - Установите `pageSize` на `3`, что означает количество последних диалогов, которые будут возвращены. Обычно на пользовательском интерфейсе будут отображены последние три диалога.
2. Обработайте результат обратного вызова поиска `V2TIMMessageSearchResult`.
  - `totalCount` указывает количество всех диалогов, которым принадлежат совпадающие сообщения.
  - Список `messageSearchResultItems` содержит информацию о последних трёх диалогах (то есть параметр `pageSize`). Здесь `messageCount` элемента `V2TIMMessageSearchResultItem` указывает общее количество сообщений, найденных в текущем диалоге.
  - Если количество найденных сообщений больше 1, `messageList` пусто, и на пользовательском интерфейсе можно отобразить «4 соответствующие записи». Здесь `4` — это `messageCount`.
  - Если количество найденных сообщений равно 1, `messageList` — это совпадающее сообщение, и его можно отобразить на пользовательском интерфейсе и выделить ключевое слово поиска, например «test» на рисунке выше.

Пример кода:

```
// Search for messages of a specified type through `messageTypeList`V2TimValueCallback<V2TimMessageSearchResult> searchMessage = await messageManager.searchLocalMessages(searchParam: V2TimMessageSearchParam(keywordList: ['Keyword 1'],pageIndex: 0,pageSize: 10,type: 1,conversationID: "",messageTypeList: [MessageElemType.V2TIM_ELEM_TYPE_TEXT]));
```

## Поиск пользовательского сообщения

В целом, если вы вызовете API `createCustomMessage(data)` ([Подробности](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_message_manager/V2TIMMessageManager/createCustomMessage.html)) для создания пользовательского сообщения, это сообщение не будет найдено, так как SDK сохранит его как поток бинарных данных.

Чтобы позволить пользователю найти пользовательское сообщение, необходимо вызвать API `createCustomMessage(data, description, extension)` [dart](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_message_manager/V2TIMMessageManager/createCustomMessage.html) для создания и отправки пользовательского сообщения и поместить текст для поиска в параметр `description`.

Если вы настроили функцию автономной передачи, после установки параметра `description` пользовательское сообщение будет передано в автономном режиме, и содержимое параметра будет отображено на панели уведомлений.
Если вам не нужна автономная передача, можно использовать `disablePush` в параметре `V2TIMOfflinePushInfo` API `sendMessage` ([Подробности](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_message_manager/V2TIMMessageManager/sendMessage.html)).

Если вы не хотите, чтобы содержимое, отправленное на панель уведомлений, отображалось в качестве текста для поиска, можно использовать `desc` в параметре `V2TIMOfflinePushInfo` для установки содержимого передачи.

## Поиск сообщения с мультимедийным содержимым

Сообщения с мультимедийным содержимым включают сообщения файлов, изображений, аудио и видео.

Для сообщения файла имя файла обычно отображается на пользовательском интерфейсе. Если при вызове `createFileMessage` для создания сообщения файла передать параметр `fileName`, то `fileName` будет использоваться в качестве содержимого для поиска и совпадения с ключевым словом поиска. Если `fileName` не установлено, SDK автоматически извлечёт имя файла из `filePath` в качестве содержимого для поиска.
Оба параметра `fileName` и `filePath` будут сохранены в локальной базе данных и на сервере, что позволяет выполнять поиск после получения на другом устройстве.

Для сообщений изображений, аудио и видео нет такого имени, как `fileName`, и обычно на пользовательском интерфейсе отображается эскиз или продолжительность; в этом случае `keywordList` неактуально.
Чтобы позволить пользователю найти такие сообщения, можно указать `messageTypeList` как `V2TIM_ELEM_TYPE_IMAGE`/`V2TIM_ELEM_TYPE_SOUND`/`V2TIM_ELEM_TYPE_VIDEO` для категоризированного поиска.

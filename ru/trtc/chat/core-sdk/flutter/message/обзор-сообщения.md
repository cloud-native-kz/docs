# Обзор сообщений

## Класс сообщения

В SDK Tencent Cloud Chat класс сообщения — это `V2TimMessage` ([Подробнее](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/models_v2_tim_message/V2TimMessage-class.html)), который часто используется при отправке и получении сообщений.

Класс `V2TimMessage` определяет следующее содержание:

| Атрибут | Описание | Примечания |
| --- | --- | --- |
| msgID | ID сообщения | Пусто при создании сообщения, возвращается синхронно при вызове `sendMessage`. |
| timestamp | Временная метка сообщения | Время отправки сообщения на сервер, можно использовать для сортировки сообщений |
| sender | `userID` отправителя сообщения | Может быть установлен пользователями и должен совпадать с `userID`, переданным при входе. |
| nickName | Никнейм отправителя сообщения | Может быть установлен пользователями и изменён путём вызова `setSelfInfo`. Дополнительную информацию см. в разделе [Профиль пользователя](https://intl.cloud.tencent.com/document/product/1047/48160). |
| friendRemark | Примечания друга об отправителе сообщения | Используется получателем. Например, если Алиса установила примечание к Бобу как `bob01`, когда Боб отправляет сообщение Алисе, Алиса увидит, что `friendRemark` в сообщении — это `bob01`. Может быть установлено путём вызова `setFriendInfo`. |
| nameCard | Карточка имени отправителя в группе | Применяется только к групповым сообщениям. Например, если Алиса изменит свою карточку имени в группе на `doctorA` и отправит сообщение в группу, другие члены группы получат сообщение, где поле `nameCard` — это `doctorA`. Получатель может приоритизировать это поле в качестве имени пользователя для отображения. Это поле можно установить путём вызова `setGroupMemberInfo`. |
| faceURL | Фотография профиля отправителя сообщения | Может быть установлена пользователями и использована для загрузки фотографии. |
| groupID | ID группы | `groupID` — это ID группы в групповом сообщении или `nil` в личном сообщении. |
| userID | ID пользователя | `userID` — это ID получателя в личном сообщении или `nil` в групповом сообщении. |
| seq | Порядковый номер сообщения | Значения `seq` личных сообщений генерируются локально и могут быть не в порядке возрастания и не уникальны; значения `seq` групповых сообщений генерируются сервером и находятся в порядке возрастания и уникальны в пределах группы. |
| random | Код случайного сообщения | Генерируется в SDK. |
| status | Статус отправки сообщения | В настоящее время поддерживаются следующие статусы: отправка, успешно отправлено, ошибка отправки, удалено, импортировано локально и отозвано. |
| isSelf | Отправлено ли сообщение самим собой | Может использоваться для фильтрации сообщений. |
| needReadReceipt | Требуется ли для сообщения подтверждение прочтения | Устанавливается отправителем и действительно для издания Ultimate на версии 6.1 или позже. Дополнительную информацию см. в разделе [Подтверждение прочтения](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_message_manager/V2TIMMessageManager/sendMessageReadReceipts.html). |
| priority | Приоритет сообщения | Применяется только к групповым сообщениям и может быть установлен путём вызова API `sendMessage`. |
| groupAtUserList | Список пользователей, отмеченных в групповом сообщении | Применяется только к групповым сообщениям и хранит `userID`. Дополнительную информацию см. в разделе [метод createAtSignedGroupMessage](https://comm.qq.com/im/doc/flutter/en/SDKAPI/Api/V2TIMMessageManager/createTextAtMessage.html). |
| elemType | Тип сообщения | В настоящее время поддерживаются следующие типы сообщений: текст, пользовательский, изображение, аудио, видео, файл, географическое местоположение, эмодзи, совет группы и объединённые сообщения. Дополнительную информацию см. в разделе Тип сообщения. |
| textElem | Элемент хранилища текстового сообщения | Должен быть создан и введён при отправке текстового сообщения. При получении сообщения и определении `elemType` как `TEXT` содержимое можно распарсить из `textElem`. Дополнительную информацию см. в разделе [метод sendMessage](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_message_manager/V2TIMMessageManager/sendMessage.html) / [свойство onRecvNewMessage](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/enum_callbacks/OnRecvNewMessageCallback.html). |
| customElem | Элемент хранилища пользовательского сообщения | Должен быть создан и введён при отправке пользовательского сообщения. При получении сообщения и определении `elemType` как `CUSTOM` содержимое можно распарсить из `customElem`. |
| imageElem | Элемент хранилища сообщения с изображением | Должен быть создан и введён при отправке сообщения с изображением. При получении сообщения и определении `elemType` как `IMAGE` содержимое можно распарсить из `imageElem`. |
| soundElem | Элемент хранилища аудиосообщения | Должен быть создан и введён при отправке аудиосообщения. При получении сообщения и определении `elemType` как `SOUND` содержимое можно распарсить из `soundElem`. |
| videoElem | Элемент хранилища видеосообщения | Должен быть создан и введён при отправке видеосообщения. При получении сообщения и определении `elemType` как `VIDEO` содержимое можно распарсить из `videoElem`. |
| fileElem | Элемент хранилища сообщения с файлом | Должен быть создан и введён при отправке сообщения с файлом. При получении сообщения и определении `elemType` как `FILE` содержимое можно распарсить из `fileElem`. |
| locationElem | Элемент хранилища сообщения о географическом местоположении | Должен быть создан и введён при отправке сообщения о географическом местоположении. При получении сообщения и определении `elemType` как `LOCATION` содержимое можно распарсить из `locationElem`. |
| faceElem | Элемент хранилища сообщения с эмодзи | Должен быть создан и введён при создании сообщения с эмодзи. При получении сообщения и определении `elemType` как `FACE` содержимое можно распарсить из `faceElem`. |
| mergerElem | Элемент хранилища объединённого сообщения | Должен быть создан и введён при отправке объединённого сообщения. При получении сообщения и определении `elemType` как `MERGER` содержимое можно распарсить из `mergerElem`. Дополнительную информацию см. в разделе [метод createForwardMessage](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/models_v2_tim_merger_elem/V2TimMergerElem-class.html). |
| groupTipsElem | Элемент хранилища сообщения с советом группы | В настоящее время сообщение с советом группы может быть отправлено только сервером. При получении сообщения SDK и определении `elemType` как `GROUP_TIPS` содержимое можно распарсить из `groupTipsElem`. Дополнительную информацию см. в разделе [свойство onRecvNewMessage](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/enum_callbacks/OnRecvNewMessageCallback.html). |
| cloudCustomData | Пользовательские данные сообщения | Устанавливаются отправителем и могут быть настроены. Сохраняются в облаке, отправляются получателю и могут быть получены после удаления и переустановки приложения. |
| isExcludedFromUnreadCount | Исключено ли сообщение из счётчика непрочитанных сообщений разговора | Устанавливается отправителем. По умолчанию сообщения включены в счётчик непрочитанных сообщений. Это поле поддерживается версией 5.3.425 или позже. |
| isExcludedFromLastMessage | Исключено ли сообщение из `lastMsg` разговора | Устанавливается отправителем. По умолчанию сообщения включены в `lastMsg` разговора. Это поле поддерживается версией 5.4.666 или позже. |
| offlinePushInfo | Пользовательское сообщение автономной отправки | Дополнительную информацию см. в разделе Автономная отправка. |
| isRead | Прочитано ли сообщение на локальном конце | Сообщение по умолчанию считается прочитанным, если оно отправлено самим собой. |
| isPeerRead | Прочитано ли сообщение получателем | Применяется только к личным сообщениям. |
| localCustomData | Пользовательские данные сообщения | Устанавливаются отправителем. Сохраняются локально, не отправляются получателю и становятся недействительными после удаления и переустановки приложения. |
| localCustomInt | Пользовательские данные сообщения | Устанавливаются отправителем. Сохраняются локально, не отправляются получателю и становятся недействительными после удаления и переустановки приложения. Могут использоваться для отметки того, было ли воспроизведено аудио- или видеосообщение. |

## Тип сообщения

Сообщения чата можно классифицировать по назначению на два типа: личные сообщения (также называемые сообщениями C2C) и групповые сообщения.

| Тип сообщения | Ключевое слово API | Описание |
| --- | --- | --- |
| Личное сообщение | C2CMessage | Также называется сообщением C2C. При отправке личного сообщения необходимо указать `UserID` получателя, и только получатель может получить это сообщение. |
| Групповое сообщение | GroupMessage | При отправке группового сообщения необходимо указать `groupID` целевой группы, и все пользователи в группе могут получить это сообщение. |

Сообщения чата также можно классифицировать по содержанию на текстовые сообщения, пользовательские (сигнальные) сообщения, сообщения с изображениями, видеосообщения, аудиосообщения, сообщения с файлами, сообщения о местоположении и объединённые сообщения.

| Тип сообщения | Ключевое слово API | Описание |
| --- | --- | --- |
| Текстовое сообщение | TextElem | Обычное текстовое сообщение |
| Пользовательское сообщение | CustomElem | Это раздел двоичного буфера, часто используемый для передачи пользовательской сигнализации в приложении. |
| Сообщение с изображением | ImageElem | При отправке исходного изображения SDK автоматически генерирует два изображения разных размеров. Три изображения называются исходным изображением, большим изображением и миниатюрой. |
| Видеосообщение | VideoElem | Видеосообщение содержит видеофайл и миниатюру. |
| Аудиосообщение | SoundElem | Поддерживает отображение красной точки перед воспроизведением аудиосообщения. |
| Сообщение с файлом | FileElem | Сообщение с файлом не может превышать 100 МБ. |
| Сообщение о местоположении | LocationElem | Сообщение о местоположении содержит три поля: описание местоположения, долгота и широта. |
| Объединённое сообщение | MergerElem | Можно объединить до 300 сообщений. |

## Политика хранения сообщений

Сообщения чата можно классифицировать по политике хранения сообщений на два типа: сообщения в сети и сообщения вне сети.

Сообщения в сети могут быть получены только онлайн-пользователями и не будут отправлены в автономном режиме. Сообщения вне сети могут быть получены пользователями независимо от того, находятся ли они в сети или нет.

Сообщения в сети доставляются в реальном времени и не сохраняются на сервере или в SDK. Поэтому они не могут быть получены из исторических сообщений после изменения устройства или удаления и переустановки приложения.

> **Примечание:** Все сообщения в аудио-видео группе являются сообщениями в сети. Сообщения, отправленные всем пользователям, являются сообщениями в сети.

Сообщения вне сети хранятся в SDK и на сервере. По умолчанию они хранятся на сервере маршрутизации в течение семи дней. Если требуется более длительный период хранения, необходимо приобрести услугу добавленной стоимости. Дополнительную информацию о содержании услуги и выставлении счетов см. в разделе [Ценообразование](https://intl.cloud.tencent.com/document/product/1047/34350).

Эти сообщения можно получить из исторических сообщений после изменения устройства или удаления и переустановки приложения.

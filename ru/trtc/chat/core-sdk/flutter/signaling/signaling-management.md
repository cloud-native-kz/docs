# Управление сигнализацией

## Обзор

API сигнализации — это набор API управления процессом приглашения на основе сообщений Chat. Они могут использоваться для реализации различных функций в реальном времени, включая:

- Аудио-видео чат-румы: микрофон включен или выключен
- Чат: аудио/видео звонки
- Образование: управление процессом приглашения преподавателем студентов для "поднятия руки"

## Функции

API сигнализации поддерживают следующие функции:

### Приглашение в личный чат

Вы можете использовать API сигнализации [invite](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_signaling_manager/V2TIMSignalingManager/invite.html) для осуществления точка-в-точку звонка. При получении уведомления о приглашении [onReceiveNewInvitation](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/enum_V2TimSignalingListener/V2TimSignalingListener/onReceiveNewInvitation.html) приглашенный может выбрать принять приглашение, отклонить его или дождаться истечения времени приглашения.

### Приглашение в групповой чат

Участники группы могут инициировать групповое приглашение на звонок через [inviteInGroup](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_signaling_manager/V2TIMSignalingManager/inviteInGroup.html) в группе. При получении приглашения [onReceiveNewInvitation](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/enum_V2TimSignalingListener/V2TimSignalingListener/onReceiveNewInvitation.html) приглашенный может выбрать принять, отклонить приглашение или дождаться истечения времени приглашения.

### Отмена приглашения

До истечения времени приглашения и обработки приглашения приглашенным, приглашающий может отменить приглашение через [cancel](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_signaling_manager/V2TIMSignalingManager/cancel.html). После того как приглашающий отменяет приглашение, приглашенный получает уведомление об отмене [onInvitationCancelled](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/enum_V2TimSignalingListener/V2TimSignalingListener/onInvitationCancelled.html), и процесс приглашения завершается.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2532342b18bd11efb8185254005ac0ca.png)

### Принятие приглашения

При получении приглашения `onReceiveNewInvitation` приглашенный может принять приглашение через [accept](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_signaling_manager/V2TIMSignalingManager/accept.html) до истечения времени приглашения и отмены приглашения приглашающим. Если приглашенный принимает приглашение, приглашающий получит уведомление о принятии приглашения [onInviteeAccepted](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/enum_V2TimSignalingListener/V2TimSignalingListener/onInviteeAccepted.html). После завершения обработки (включая принятие, отклонение и истечение времени) на стороне приглашенного процесс приглашения завершается.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/25301ebf18bd11efb8185254005ac0ca.png)

### Отклонение приглашения

При получении уведомления о приглашении `onReceiveNewInvitation` приглашенный может отклонить приглашение через [reject](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_signaling_manager/V2TIMSignalingManager/reject.html) до истечения времени приглашения и отмены приглашения приглашающим. Если приглашенный отклоняет приглашение, приглашающий получит уведомление об отклонении приглашения [onInviteeRejected](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/enum_V2TimSignalingListener/V2TimSignalingListener/onInviteeRejected.html). После завершения обработки (включая принятие, отклонение и истечение времени) на стороне приглашенного процесс приглашения завершается.

### Истечение времени приглашения

Если длительность времени ожидания API приглашения больше 0 и приглашенный не отвечает в течение этого времени, приглашение истекает, и приглашающий и приглашенный получат уведомление об истечении времени [onInvitationTimeout](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/enum_V2TimSignalingListener/V2TimSignalingListener/onInvitationTimeout.html). После завершения обработки (включая принятие, отклонение и истечение времени) на стороне приглашенного процесс приглашения завершается. Если длительность времени ожидания API приглашения равна 0, уведомлений об истечении времени не будет.

## Варианты использования

### Аудио/видео звонок

В проекте с открытым исходным кодом [TRTCFlutterScenesDemo](https://github.com/tencentyun/TRTCFlutterScenesDemo) мы разработали решение для аудио/видео звонков, подходящее для личных и многопользовательских чатов на основе [TRTC](https://intl.cloud.tencent.com/document/product/647/41691). Вы можете напрямую модифицировать демонстрацию для адаптации. Ниже на примере процесса видеозвонка один-на-один показано, как API сигнализации работают с SDK TRTC.

**Процесс видеозвонка один-на-один:**

1. Приглашающий входит в TRTC-комнату на основе ID комнаты, созданного на уровне сервиса, и вызывает API приглашения сигнализации `invite` для инициирования запроса аудио/видео звонка, включая ID комнаты в пользовательское поле API приглашения.
2. Приглашенный получает уведомление о приглашении сигнализации `onReceiveNewInvitation` и получает ID комнаты на основе пользовательских данных. На телефоне приглашенного начинает звонить.
3. Приглашенный обрабатывает уведомление о приглашении:
  - Принятие приглашения: приглашенный вызывает API сигнализации `accept` и входит в TRTC-комнату на основе ID комнаты, затем вызывает функцию `openCamera()` для включения локальной камеры. Приглашающий и приглашенный получают обратный вызов `onRemoteUserEnterRoom` из SDK TRTC, и системы обеих сторон записывают время начала звонка.
  - Отклонение приглашения: приглашенный вызывает API сигнализации `reject` для завершения звонка.
  - Если приглашенный разговаривает с кем-то еще, приглашенный вызывает API сигнализации `reject` для отклонения приглашения и указывает причину отклонения (занято) в пользовательских данных.
4. После того как приглашенный ответил на звонок и между приглашающим и приглашенным установлен аудио/видео канал, оба получат уведомление о событии `onUserVideoAvailable` из SDK TRTC, которое указывает, что они получили видеоизображение друг друга. В этом случае они могут вызвать API `startRemoteView` SDK TRTC для отображения удаленного видеоизображения, а удаленный звук будет воспроизводиться по умолчанию.
5. После завершения звонка (независимо от того, кто положит трубку — приглашающий или приглашенный) сторона, положившая трубку, выходит из TRTC-комнаты. Другая сторона получает обратный вызов `onRemoteUserLeaveRoom` из SDK TRTC, и ее система рассчитывает общую длительность звонка и инициирует приглашение снова, включая событие завершения звонка и длительность звонка в пользовательские данные для удобства отображения UI.

### Образование: преподаватель приглашает студентов "поднять руку"

В этом сценарии преподаватель просит студентов "поднять руку" и затем выбирает одного из студентов для выступления. Процесс выглядит следующим образом:

1. Преподаватель вызывает API `inviteInGroup` для приглашения студентов "поднять руку", указав операцию "поднятия руки" в пользовательском поле `data`. Студенты получают обратный вызов `onReceiveNewInvitation`.
2. На основе полей `inviteeList` и `data` в `onReceiveNewInvitation` студент определяет, что он/она является одним из приглашенных и операция — это поднятие руки. Затем студент вызывает API `accept` для поднятия руки.
3. Если студент поднимает руку, остальные могут получить обратный вызов `onInviteeAccepted`. Система определяет, что поле `data` — это поднятие руки, и отображает список студентов, поднявших руку.
4. Преподаватель вызывает API `inviteInGroup` для приглашения одного из студентов, поднявших руку, для выступления. В этом случае система указывает операцию "выступления" в пользовательском поле `data`. Студенты получают обратный вызов `onReceiveNewInvitation`.
5. На основе полей `inviteeList` и `data` в обратном вызове `onReceiveNewInvitation` студент определяет, что он/она является одним из приглашенных и операция — это выступление. Затем студент вызывает API `accept` для выступления.
6. Если студент выступает, остальные могут получить обратный вызов `onInviteeAccepted`, и их системы определяют, что поле данных — это выступление, и отображают список студентов, которые выступают.

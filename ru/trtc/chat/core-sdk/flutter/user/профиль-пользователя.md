# Профиль пользователя

## Обзор

Пользователи могут устанавливать и получать свои прозвища, фотографии профиля и статусы, а также информацию профиля пользователей, не являющихся друзьями. Методы находятся в основном классе `TencentImSDKPlugin.v2TIMManager.getFriendshipManager()`.

## Слушатель событий цепочки отношений

Вызовите API `addFriendListener` ([подробнее](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_friendship_manager/V2TIMFriendshipManager/addFriendListener.html)), чтобы добавить слушатель событий контактов.

Чтобы перестать получать события контактов, вызовите API `removeFriendListener` ([подробнее](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_friendship_manager/V2TIMFriendshipManager/removeFriendListener.html)) для удаления слушателя событий контактов.

> **Примечание:** Необходимо заранее установить слушатель событий контактов, чтобы получать уведомления о событиях.

Пример кода:

```
    // Set the contacts listener    V2TimFriendshipListener listener = V2TimFriendshipListener(      onBlackListAdd: (List<V2TimFriendInfo> infoList) async {        // Callback for adding a user to the blocklist        // infoList: List of information of the user added      },      onBlackListDeleted: (List<String> userList) async {        // Callback for removing users from the blocklist        // userList: List of IDs of the users deleted      },      onFriendApplicationListAdded:          (List<V2TimFriendApplication> applicationList) async {        // Callback for the increase of friend requests        // applicationList: List of the information of new friend requests      },      onFriendApplicationListDeleted: (List<String> userIDList) async {        // Callback for the decrease of friend requests        // userIDList: List of user IDs corresponding to the friend requests decreased      },      onFriendApplicationListRead: () async {        // Callback for read friend request      },      onFriendInfoChanged: (List<V2TimFriendInfo> infoList) async {        // Callback for friend information changes        // infoList: list of friends whose information changes      },      onFriendListAdded: (List<V2TimFriendInfo> users) async {        // Callback for the increase of users in the friend list        // users: list of users added      },      onFriendListDeleted: (List<String> userList) async {        // Callback for the decrease of users in the friend list        // userList: list of users deleted      },    );    TencentImSDKPlugin.v2TIMManager        .getFriendshipManager()        .addFriendListener(listener: listener); // Add a contacts listener // Remove the contacts listener friendshipManager.removeFriendListener(listener: friendshipListener);
```

## Управление профилем пользователя

### Запрос и изменение собственного профиля

Вызовите API `getUsersInfo` ([подробнее](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_manager/V2TIMManager/getUsersInfo.html)) с параметром `userIDList`, установленным на `UserID` пользователя, чтобы запросить профиль пользователя.

Вызовите API `setSelfInfo` ([подробнее](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_manager/V2TIMManager/setSelfInfo.html)) для изменения профиля пользователя.

После успешного изменения профиля вы получите обратный вызов `onSelfInfoUpdated` ([подробнее](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/enum_V2TimSDKListener/V2TimSDKListener/onSelfInfoUpdated.html)).

Пример кода:

```
// Obtain a user's personal profileV2TimValueCallback<String> self = await TencentImSDKPlugin.v2TIMManager.getLoginUser();TencentImSDKPlugin.v2TIMManager.getUsersInfo(userIDList: [self.data]);// Set the user's profile TencentImSDKPlugin.v2TIMManager.setSelfInfo(userFullInfo: V2TimUserFullInfo(nickName: "",role: 0,faceUrl: ""));
```

### Запрос профиля пользователя, не являющегося другом

Вызовите API `getUsersInfo` ([подробнее](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_manager/V2TIMManager/getUsersInfo.html)) с параметром `userIDList`, установленным на `UserID` пользователя, не являющегося другом, чтобы запросить профиль этого пользователя.

> **Примечание:** Профиль пользователя, не являющегося другом, не может быть изменен.

### Запрос и изменение профиля друга

Вызовите API `getFriendsInfo` ([подробнее](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_friendship_manager/V2TIMFriendshipManager/getFriendsInfo.html)) для запроса профиля указанного друга. Взаимосвязь между пользователем и другом можно получить через поле `relation` объекта `V2TIMFriendInfoResult` в обратном вызове:

| relation | Взаимосвязь |
| --- | --- |
| `V2TIM_FRIEND_RELATION_TYPE_NONE` | Не друг |
| `V2TIM_FRIEND_RELATION_TYPE_BOTH_WAY` | Двусторонний друг |
| `V2TIM_FRIEND_RELATION_TYPE_IN_MY_FRIEND_LIST` | Пользователь находится в ваших контактах. |
| `V2TIM_FRIEND_RELATION_TYPE_IN_OTHER_FRIEND_LIST` | Вы находитесь в контактах пользователя. |

```
// Get friend informationV2TimValueCallback<List<V2TimFriendInfoResult>> friendsInfo = await friendshipManager.getFriendsInfo(userIDList: []);
```

Вызовите API `setFriendInfo` ([подробнее](https://pub.dev/documentation/tencent_cloud_chat_sdk/latest/manager_v2_tim_friendship_manager/V2TIMFriendshipManager/setFriendInfo.html)) для изменения информации о друге, такой как примечания.

```
// Set friend informationTencentImSDKPlugin.v2TIMManager.setSelfInfo(userFullInfo: V2TimUserFullInfo(nickName: "",role: 0,faceUrl: ""));
```

## Часто задаваемые вопросы

### Почему расширенное издание SDK не может получить последние профили пользователей?

В расширенном SDK есть два типа обновлений профиля пользователя:

- Профиль друга: когда профиль друга обновляется, бэкенд отправляет системное уведомление в SDK, поэтому профиль друга будет обновлен в реальном времени.
- Профиль незнакомца: когда профиль незнакомца обновляется, бэкенд не отправляет никаких системных уведомлений, так как незнакомец не является вашим другом, поэтому профиль незнакомца не будет обновлен в реальном времени. Чтобы избежать отправки сетевого запроса на бэкенд каждый раз при получении профиля пользователя, SDK добавляет логику кэширования с интервалом в 10 минут между запросами профиля одного пользователя с бэкенда.

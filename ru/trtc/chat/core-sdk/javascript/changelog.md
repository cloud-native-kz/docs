# Журнал изменений

### 3.5.0 @2024.12.18

**Новые возможности**

- [searchCloudUsers](https://www.tencentcloud.com/document/product/1047/67679#b57f11f2-d80b-4e40-b9e3-8f9463195624), позволяет выполнять поиск облачных пользователей.
- [searchCloudGroups](https://www.tencentcloud.com/document/product/1047/67681#b57f11f2-d80b-4e40-b9e3-8f9463195624), облегчает поиск облачных списков групп.
- [searchCloudGroupMembers](https://www.tencentcloud.com/document/product/1047/67683#b57f11f2-d80b-4e40-b9e3-8f9463195624), позволяет выполнять поиск облачных списков членов группы.

### 3.4.9 @2024.12.6

**Новые возможности**

- Поддержка IPv6; платформы мини-программ должны добавить доверенный домен сокета `wss://wssv6.im.qcloud.com`.
- Автономная доставка теперь поддерживает категоризацию сообщений OPPO (Category).
- Совместимость с версией ReactNative 0.76.x.

**Исправления**

- Неточности в данных `groupAtInfo` разговора.

### 3.4.8 @2024.11.21

**Новые возможности**

- Поддержка защиты сообщений от подделок для обеспечения безопасности сообщений.
- Поддержка повторного получения чёрного списка.

**Исправления**

- Проблемы, при которых несколько изменений одного сообщения позволяли вступить в силу только первому изменению.
- Ошибки, связанные с редактированием сообщений в личных чатах с использованием ESM SDK из-за ошибки модуля.
- Неправильное назначение `group.selfInfo.role` владельцу группы после изменения информации группы.

### 3.4.7 @2024.10.31

**Исправления**

- Данные группы не очищаются при выходе.
- Невозможность остановить опрос при получении уведомления о выходе из группы прямых трансляций.

### 3.4.6 @2024.10.16

**Добавлено**

- Поддержка выделенных доменов, повышающая стабильность сервиса. Если ваш бизнес работает на платформе мини-программ, перед обновлением необходимо настроить доверенный домен сокета.
- Поддержка изменения комбинированных сообщений.
- После обновления замечания друга выполняется событие `CONVERSATION_LIST_UPDATED`, улучшающее пользовательский опыт.

### 3.4.5 @2024.9.24

**Добавлено**

- Оптимизация пользовательского опыта после добавления в группу, сокращение сетевых запросов.

**Исправление**

- Исправлена проблема, при которой RESTAPI, изменяющий пользовательские данные сеанса `Conversation.customData`, не обновлялся.

### 3.4.4 @2024.9.18

**Добавлено**

- `getServerTime`, поддержка получения времени сервера.
- Рабочая группа позволяет устанавливать роли членов группы.
- Совершенное обнаружение окружения для React Native.

**Исправление**

- Проблема интеграции uni-app TencentCloud-TIMPush, при которой автономные push-уведомления не получались в некоторых сценариях.
- Проблема с отказом имени домена медиа-сообщения при одновременном включении внутреннего прокси сети и приватного чтения сценариев мультимедиа.

### 3.4.3 @2024.9.2

**Добавлено**

- Поддержка React Native.
- Поддержка независимых доменных имён COS для обеспечения стабильности услуг мультимедиа.
- Оптимизация логики подсчёта непрочитанных сообщений в разговоре C2C для обеспечения точности.
- Новый разговор, созданный `getConversationProfile`, дополнительно запрашивает `messageRemindType` и информацию о группе разговора.

### 3.4.2 @2024.7.31

**Добавлено**

- Поддержка приватного чтения медиа-сообщений.
- Когда информация о группе изменяется, сообщение `payload` добавляет `operatorInfo`, представляющий информацию оператора, сделавшего изменение.
- Добавлен тип `TencentCloudChat.TYPES.NOT_RECEIVE_MSG_EXCEPT_AT`, поддерживающий получение только сообщений @.

### 3.4.1 @2024.7.3

**Добавлено**

- `getJoinedCommunityList` получает список поддерживающих темы сообществ по умолчанию из поля Definition сообщества.
- Управление памятью сообщений темы.
- `pinConversation` поддерживает закрепление разговоров, которых нет в списке разговоров.
- Оптимизация производительности SDK.
- Вновь созданные разговоры, если lastTime равно 0, будут отображаться после закреплённых разговоров и перед незакреплёнными разговорами в первый раз.
- TIMPush поддерживает изображения в панели уведомлений.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/d722695f906d11ef8a1b525400bdab9d.png)

**Исправление**

- Аномалии в подсчёте непрочитанных разговоров группы после переподключения.
- Отредактированные сообщения, полученные из роумера, имеют `Message.isModified` неустановленным на true.
- Проблемы с своевременным обновлением верхнего слоя в сценариях добавления или удаления членов `Group.memberCount`.
- В некоторых сценариях ошибки в расчёте `isCompleted` ответа интерфейса `getMessageListHopping`.

### 3.4.0 @2024.6.13

**Добавлено**

- Временная метка уведомления о прочтении для сообщений C2C (`Message.readReceiptInfo.timestamp`).
- Поддержка сжатия на уровне приложения JS для уменьшения объёма передачи и повышения производительности и опыта клиента.

**Исправление**

- Проблема с аномальными payload и типом lastMessage в некоторых сценариях.

### 3.3.3 @2024.4.26

**Добавлено**

- При создании нового разговора заполняется информация, такая как группа и теги разговора.
- После удаления членов группы через RestAPI и последующего приглашения удалённых членов обратно в группу SDK активно синхронизирует сообщения группы в период, когда удалённые члены не находились в группе.

**Исправление**

- Повторные вызовы API `sendMessageReadReceipt` отправляют уведомления о прочтении сообщений C2C, вызывая множественное направление события `MESSAGE_READ_RECEIPT_RECEIVED`.

### 3.3.2 @2024.4.12

**Добавлено**

- Параметр API `sendMessage` `offlinePushInfo` поддерживает пользовательское определение звуков автономной доставки. Получатель должен интегрировать `TIMPush`.
- Для типов напоминания о сообщениях в сеансе добавлен новый тип `RECEIVE_WITH_OFFLINE_PUSH_EXCEPT_AT`. Сообщения @ группы поддерживают автономную доставку.
- После выхода из группы, исключения или распуска группы локальный счётчик непрочитанных сообщений соответствующего сеанса группы сбрасывается на ноль.
- Оптимизация опыта интеграции встроенного WebView в сценарии Chat SDK.

**Исправление**

- В некоторых сценариях `setMessageRead` не может очистить непрочитанные номера для групповых чатов.

### 3.3.1 @2024.3.20

**Добавлено**

- Оптимизирована работа переподключения при слабой сети для обеспечения надёжности и своевременной доставки сообщений.

**Исправление**

- В сценариях многоэкземплярного входа Web в систему после восстановления определённого экземпляра со слабой сети проблема не обновления статуса пользовательского определения (`customStatus`).
- Закреплённые разговоры с непрочитанными сообщениями имеют проблему, при которой `isPinned` имеет значение `false`.

### 3.3.0 @2024.3.6

**Добавлено**

- Интерфейсы создания сообщений `createTextMessage` и другие теперь включают `customModerationConfigurationID`, поддерживающий политики пользовательской проверки сообщений.
- Интерфейсы `setAllReceiveMessageOpt` и `getAllReceiveMessageOpt` теперь поддерживают глобальный режим недостижимости.
- Интерфейс `getConversationList` добавляет параметры `hasUnreadCount` и `hasGroupAtInfo`, поддерживающие фильтрацию разговоров по наличию непрочитанных сообщений и информации @.

**Исправление**

- Проблема неточного подсчёта членов (memberCount) в некоторых сценариях после добавления или удаления членов группы.

### 3.2.6 @2024.1.31

**Добавлено**

- Интерфейсы `followUser`/`getMyFollowersList` и другие теперь поддерживают важные функции для социальных сценариев с незнакомцами, такие как подписка и подписчики (требуется версия флагманского продукта).
- Интерфейс `create` добавляет параметр `eventThrottling`, позволяющий интеграции использовать этот параметр для пользовательских политик дросселирования событий.

**Изменено**

**Исправление**

- Ошибка создания группы разговоров.
- Проблемы, связанные с сообществом и темами.

### 3.2.0 @2023.9.13

**Добавлено**

- addMessageReaction добавляет реакции на сообщения (требуется версия флагманского продукта). Этот интерфейс поддерживает сценарии, такие как "реакции эмодзи", например:
  - ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/d71acb53906d11ef8a1b525400bdab9d.png)
  - ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/d71cd78c906d11efa3675254002693fd.png)
- `removeMessageReaction` удаляет реакции на сообщения.
- `getMessageReactions` массово получает несколько реакций на сообщения.
- `getAllUserListOfMessageReaction` постранично получает список пользователей заданных реакций на сообщения.
- `getGroupMemberList` поддерживает получение списка членов группы, включая состояние в сети (требуется версия флагманского продукта).
- `getGroupOnlineMemberCount` поддерживает запрос счётчика участников в сети для всех типов групп.

**Исправление**

- Получение пакета облачного ответа роумера группы не может продолжаться повторно для направленных сообщений только.
- Данные списка запросов друзей неожиданно были очищены.
- В некоторых сценариях ранее полученные сообщения роумера группы неожиданно были очищены.

### 3.1.3 @2023.8.21

**Добавлено**

- `convertVoiceToText` поддерживает преобразование голоса в текст.
- `getGroupMemberList` поддерживает получение списка членов группы с заданными ролями.

### 3.1.2 @2023.8.15

**Добавлено**

- `getGroupApplicationList` добавляет информацию о замечаниях группы при получении списка ожидания группы.
- `revokeMessage` поддерживает отзыв сообщений прямой группы.
- `searchCloudMessages` поддерживает ввод только типа сообщения и отправителя.
- Медиа-сообщения автоматически обновляются при пересылке в слиянии.
- Поддержка асинхронных уведомлений о проверке голосовых и видеосообщений (прослушивание события `MESSAGE_MODIFIED`).
- `Message` добавлено новое свойство `hasRiskContent`, указывающее, отмечены ли голосовые или видеосообщения как имеющие риски безопасности после `облачной модерации`.
- `Message` добавлено новое свойство `revokerInfo`, которое включает информацию об отозвавшем сообщение: userID, прозвище (nick) и аватар.
- `Message` добавлено новое свойство `revokeReason`, которое указывает причину отзыва сообщения (причину можно установить при отзыве сообщения через REST API).

**Изменено**

- Оптимизированные подсказки и руководство, когда коммерческие возможности не поддерживаются.
- В приватных и внутренних сценариях прокси SDK не загружает информацию журнала через HTTP, чтобы избежать проблем соответствия.

### 2.25.0 @2022.12.8

**Новые возможности**

- Добавлен API [clearHistoryMessage](https://web.sdk.qcloud.com/im/doc/en/SDK.html#clearHistoryMessage) для очистки сообщений из локального хранилища и облака.
- Поддержка расширения сообщений (функция Premium издания).
- Поддержка атрибутов обычной группы и сообщества.

- Исправлена ошибка выполнения (не удалось прочитать свойство 'localStorage' из 'Window': доступ запрещён для этого документа), вызванная заблокированными файлами cookie.

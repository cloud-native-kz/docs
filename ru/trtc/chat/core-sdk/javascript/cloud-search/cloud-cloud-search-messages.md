# Поиск облачных сообщений

## Описание функции

Поиск облачных сообщений помогает пользователям быстро находить и индексировать связанные сообщения по ключевым словам. Через этот API пользователи могут эффективно находить конкретное содержимое сообщений, повышая скорость и точность извлечения информации.

> **Примечание:** Функция поиска облачных сообщений доступна только для клиентов **Pro Plus и Enterprise** и может быть использована после [приобретения Pro Plus и Enterprise](https://console.trtc.io/subscription/buy/chat?packType=pro&language=en); версия Free Trial поддерживает [определенный лимит бесплатного пробного использования](https://www.tencentcloud.com/document/product/1047/67651#d1113f0d-47e8-4211-82c0-00d2efb72586), действительный в течение одного месяца. Функция поиска облачных сообщений поддерживается с версии **v3.1.0**. Этот API имеет локальное ограничение в 2 вызова в секунду. Сообщества, темы и группы прямых трансляций не поддерживают поиск облачных сообщений. Поиск сообщений во **всех диалогах**: Если количество найденных сообщений messageCount > 1, API возвращает messageList как []. Вы можете отобразить [${messageCount} релевантные записи] в интерфейсе. Если вы хотите выделить найденные сообщения, обратитесь к разделу [поиск в указанном диалоге](https://www.tencentcloud.com/document/product/1047/67677#0a53dccd-3aa7-4cec-9ee1-9baa245d32ff) и выделите messageList, возвращаемый API. Если количество найденных сообщений в диалоге = 1, messageList содержит найденное сообщение.

## Поиск облачных сообщений

##### API

```
chat.searchCloudMessages(options);
```

##### Параметры

Параметр options имеет тип Object и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| keywordList | Array \| undefined | Список ключевых слов, поддерживает до 5 ключевых слов. **Примечание:** Когда идентификатор отправителя сообщения `senderUserIDList` и тип сообщения `messageTypeList` не указаны, список ключевых слов должен быть непустым; в противном случае список ключевых слов может быть пустым. |
| keywordListMatchType | String \| undefined | Тип совпадения списка ключевых слов или: поиск с отношением "или" (по умолчанию) и: поиск с отношением "и" |
| senderUserIDList | Array \| undefined | Сообщения, отправленные указанным userID, поддерживает до 5. |
| messageTypeList | Array \| undefined | Набор указанных типов сообщений для поиска, по умолчанию все типы. Если не передано, это означает поиск всех поддерживаемых типов сообщений (не поддерживаются `TencentCloudChat.TYPES.MSG_FACE`, `TencentCloudChat.TYPES.MSG_GRP_TIP` и `TencentCloudChat.TYPES.MSG_GRP_SYS_NOTICE`). |
| conversationID | String \| undefined | Поиск во "всех диалогах" или "указанных диалогах". Если не передано, это означает все диалоги. По умолчанию: все диалоги. Метод формирования ID диалога: `C2C${userID}` (личный чат) `GROUP${groupID}` (групповой чат) Сообщества, темы и группы прямых трансляций не поддерживают поиск облачных сообщений. |
| timePosition | Number \| undefined | Начальная точка времени для поиска. По умолчанию 0, что означает поиск с текущего момента, в секундах. |
| timePeriod | Number \| undefined | Период времени от начальной точки, в секундах. По умолчанию 0, что означает отсутствие ограничения по времени. Передача 24 * 60 * 60 означает последний день. |
| cursor | String \| undefined | Начальная позиция для каждого облачного поиска. Не передавайте `cursor` для первого поиска. Для последующих поисков передавайте значение `cursor`, возвращаемое предыдущим вызовом API `searchCloudMessages`. **Примечание:** Курсор действителен 2 минуты во время полнотекстового поиска. |

##### **Возвращаемое значение**

`Promise`

| Имя | Тип | Описание |
| --- | --- | --- |
| totalCount | Number | Если выполняется поиск в указанном диалоге, возвращается общее количество сообщений, соответствующих критериям поиска. Если выполняется поиск во всех сессиях, возвращается общее количество сессий, содержащих сообщения, соответствующие критериям поиска. |
| searchResultList | Array | Сообщения, соответствующие критериям поиска, сгруппированные по ID диалога. |
| cursor | String | Курсор, необходимый для вызова API поиска для продолжения извлечения. |

Здесь `searchResultList` — это список, содержащий объекты результатов поиска. Параметры описаны ниже:

| Имя | Тип | Описание |
| --- | --- | --- |
| conversationID | String | ID диалога. |
| messageCount | Number | Общее количество сообщений, соответствующих критериям, найденных в текущем диалоге. |
| messageList | Array | Список всех сообщений, соответствующих критериям поиска в текущем диалоге. **Примечание:** При поиске в указанном диалоге `messageList` — это список всех сообщений в этом диалоге, соответствующих критериям поиска. При поиске во всех диалогах количество сообщений в `messageList` может иметь следующие две возможности: Если количество найденных сообщений `messageCount` > 1, `messageList`, возвращаемый API, будет []. Вы можете отобразить [${messageCount} релевантные записи] в интерфейсе. Если вы хотите выделить найденные сообщения, обратитесь к разделу [поиск в указанном диалоге](https://www.tencentcloud.com/document/product/1047/67677#0a53dccd-3aa7-4cec-9ee1-9baa245d32ff) и выделите `messageList`, возвращаемый API. Если количество найденных сообщений в диалоге = 1, то `messageList` — это найденное сообщение. |

## Поиск сообщений во всех сессиях

Не указывайте `conversationID`, укажите поиск по ключевому слову.

##### Пример

```
// Полнотекстовый поиск, укажите ключевое слово// - Поиск сообщений, содержащих 'Hello' или 'Where are you'let promise = chat.searchCloudMessages({   keywordList: ['Hello', 'Where are you']});promise.then(function(imResponse) {   // Успешный поиск сообщения   const { totalCount, cursor, searchResultList } = imResponse.data;   console.log(totalCount); // Общее количество сессий, содержащих сообщения, соответствующие критериям поиска   console.log(cursor); // Начальная позиция для следующего облачного поиска, если отсутствует, указывает, что результаты поиска полностью получены   console.log(searchResultList); // Сообщения, соответствующие критериям поиска, сгруппированные по ID диалога и возвращаемые постранично.   for (let i = 0; i < searchResultList.length; i++) {      const searchResultItem = searchResultList[i];      const { conversationID, messageCount, messageList } = searchResultItem;      console.log(conversationID); // ID диалога      console.log(messageCount); // Общее количество сообщений, соответствующих критериям, в текущем диалоге      // При поиске во всех диалогах количество сообщений в messageList может иметь следующие две возможности:      // - Если количество найденных сообщений в диалоге больше 1, messageList пуст, и вы можете отобразить "messageCount релевантные записи" в интерфейсе.      // - Если количество найденных сообщений в диалоге равно 1, messageList — это найденное сообщение, и вы можете отобразить его в интерфейсе и выделить найденное ключевое слово.      console.log(messageList);    }}).catch(function(imError) {   console.error(imError); // ошибка при поиске сообщений});
```

## Поиск сообщений в указанном диалоге

Укажите `conversationID` и выполните поиск по ключевому слову.

##### Пример

```
// Укажите диалог и выполните поиск по ключевому слову// - Поиск сообщений в диалоге 'GROUPPublic001', содержащих 'Hello' или 'Where are you'.let promise = chat.searchCloudMessages({   keywordList: ['Hello', 'Where are you'],   conversationID: 'GROUPPublic001'});promise.then(function(imResponse) {   // Успешный поиск сообщения   const { totalCount, cursor, searchResultList } = imResponse.data;   console.log(totalCount); // Общее количество сообщений, соответствующих критериям поиска   console.log(cursor); // Начальная позиция для следующего облачного поиска, если отсутствует, указывает, что результаты поиска полностью получены   console.log(searchResultList); // Результаты поиска сообщений текущего диалога   if(searchResultList.length){       const { conversationID, messageCount, messageList } = searchResultList[0];       console.log(conversationID); // ID диалога       console.log(messageCount); // Общее количество сообщений, соответствующих критериям, в текущем диалоге       console.log(messageList); // Список всех сообщений в этом диалоге, соответствующих критериям поиска   }}).catch(function(imError){   console.error(imError); // ошибка при поиске сообщений});
```

## Поиск пользовательского сообщения

1. При использовании API [`createCustomMessage`](https://www.tencentcloud.com/document/product/1047/47993#creating-a-custom-message) для создания пользовательского сообщения необходимо поместить текст для поиска в параметр `description`. Он поддерживает сопоставление ключевых слов с `description`.
2. Укажите `messageTypeList` как `TencentCloudChat.TYPES.MSG_CUSTOM` для классифицированного поиска, который будет искать все пользовательские сообщения.

##### Пример

```
description
```

## Поиск мультимедийного сообщения

Мультимедийные сообщения включают сообщения о файлах, изображениях, аудио и видео.

1. Для сообщения о файле имя файла обычно отображается в интерфейсе. Если при создании сообщения о файле вы передаете параметр [`createFileMessage`](https://www.tencentcloud.com/document/product/1047/47993#creating-a-file-message) `fileName`, `fileName` будет использоваться как содержимое для поиска и сопоставления ключевого слова поиска.
2. Для сообщений об изображениях, голосе и видео нет имени, как `fileName`. Интерфейс обычно отображает миниатюру или длительность, поэтому указание `keywordList` для поиска недействительно.
3. Вы можете указать `messageTypeList` как `TencentCloudChat.TYPES.MSG_IMAGE`/`TencentCloudChat.TYPES.MSG_AUDIO`/`TencentCloudChat.TYPES.MSG_VIDEO` для классифицированного поиска, который будет искать все сообщения указанных типов.

##### Пример

```
// Полнотекстовый поиск, укажите только отправителя сообщения и тип сообщения для поиска (при указании отправителя и типа сообщения список ключевых слов может быть пустым)// - Поиск сообщений, отправленных 'user1' или 'user2', и тип сообщения — 'сообщение изображения', 'голосовое сообщение' или 'сообщение файла'let promise = chat.searchCloudMessages({   senderUserIDList: ['user1', 'user2'],   messageTypeList: [      TencentCloudChat.TYPES.MSG_IMAGE,      TencentCloudChat.TYPES.MSG_AUDIO,      TencentCloudChat.TYPES.MSG_FILE   ],});promise.then(function(imResponse) { // Успешный поиск сообщения   const { totalCount, cursor, searchResultList } = imResponse.data;   console.log(totalCount); // Общее количество сессий, содержащих сообщения, соответствующие критериям поиска   console.log(cursor); // Начальная позиция для следующего облачного поиска, если отсутствует, указывает, что результаты поиска полностью получены   console.log(searchResultList); // Сообщения, соответствующие критериям поиска, сгруппированные по ID диалога и возвращаемые постранично.   for (let i = 0; i < searchResultList.length; i++) {      const searchResultItem = searchResultList[i];      const { conversationID, messageCount, messageList } = searchResultItem;      console.log(conversationID); // ID диалога      console.log(messageCount); // Общее количество сообщений, соответствующих критериям, в текущем диалоге      // При поиске во всех диалогах количество сообщений в messageList может иметь следующие две возможности:      // - Если количество найденных сообщений в диалоге больше 1, messageList пуст, и вы можете отобразить "messageCount релевантные записи" в интерфейсе.      // - Если количество найденных сообщений в диалоге равно 1, messageList — это найденное сообщение, и вы можете отобразить его в интерфейсе и выделить найденное ключевое слово.      console.log(messageList);    }}).catch(function(imError) {   console.error(imError); // ошибка при поиске сообщений});
```

## Поиск сообщений геолокации

1. Поддерживает сопоставление `latitude`, `longitude`, `description` с ключевыми словами.
2. Укажите `messageTypeList` как `TencentCloudChat.TYPES.MSG_LOCATION` для классифицированного поиска, который будет искать все сообщения геолокации.

##### Пример

```
latitude, longitude, description
```

## Поиск объединенных сообщений

1. При использовании API [`createMergerMessage`](https://www.tencentcloud.com/document/product/1047/47993#creating-a-merged-message) для создания объединенного сообщения необходимо поместить текст для поиска в параметр `title` или `abstractList`. Он поддерживает сопоставление `title` и `abstractList` с ключевыми словами.
2. Укажите `messageTypeList` как `TencentCloudChat.TYPES.MSG_MERGER` для классифицированного поиска, который будет искать все объединенные сообщения.

##### Пример

```
title or abstractList
```

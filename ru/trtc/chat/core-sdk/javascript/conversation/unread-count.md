# Счётчик непрочитанных сообщений

## Обзор функции

Список диалогов пользователя обычно содержит несколько диалогов. Если в одном из диалогов появляется новое сообщение, необходимо отобразить значок счётчика в ячейке списка, чтобы указать количество непрочитанных сообщений.

После того как пользователь входит в диалог и возвращается в список диалогов, счётчик непрочитанных сообщений очищается, и значок исчезает. В некоторых приложениях рассчитывается и отображается общее количество непрочитанных сообщений всех диалогов на вкладке внизу списка диалогов.

## Получение общего количества непрочитанных сообщений во всех диалогах

> **Примечание:** Применяется только к групповым диалогам Work/Public/Community/Meeting. AVChatRoom в настоящий момент не поддерживается. Общее количество непрочитанных сообщений будет уменьшаться на количество непрочитанных сообщений в диалогах, установленных в режим «Не беспокоить». Общее количество непрочитанных сообщений не включает количество непрочитанных сообщений в системных диалогах. Поскольку этот интерфейс зависит от синхронизации списка диалогов из облака SDK, общее количество непрочитанных сообщений, полученное сразу после успешного входа, может быть равно 0. Сторона интеграции может обновить локальное общее количество непрочитанных сообщений в сочетании с событием `TencentCloud.EVENT.TOTAL_UNREAD_MESSAGE_COUNT_UPDATED`.

##### **API**

```
chat.getTotalUnreadMessageCount();
```

##### **Возвращаемое значение**

`Number`

##### **Примеры**

```
// Общее количество непрочитанных сообщений в личных и групповых диалогахlet totalUnreadCount = chat.getTotalUnreadMessageCount();
```

## Уведомление об изменении общего количества непрочитанных сообщений во всех диалогах

##### **Примеры**

```
let onTotalUnreadMessageCountUpdated = function(event) {  console.log(event.data);};chat.on(TencentCloudChat.EVENT.TOTAL_UNREAD_MESSAGE_COUNT_UPDATED, onTotalUnreadMessageCountUpdated);
```

## Очистка счётчика непрочитанных сообщений диалога

##### **API**

```
chat.setMessageRead(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Он содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| conversationID | String | ID диалога. Допустимые значения: `C2C${userID}` (для личного чата)`GROUP{groupID}`(для группового чата)`@TIM#SYSTEM` (для диалога системных уведомлений)`GROUP${topicID}` (для темы) |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Установить все непрочитанные сообщения в диалоге как прочитанныеlet promise = chat.setMessageRead({conversationID: 'C2Cexample'});promise.then(function(imResponse) {  // Непрочитанные сообщения успешно установлены как прочитанные.  // Значение атрибута `unreadCount` диалога с указанным ID устанавливается на `0`.}).catch(function(imError) {  // Не удалось установить непрочитанные сообщения как прочитанные  console.warn('setMessageRead error:', imError);});
```

## Очистка счётчика непрочитанных сообщений всех диалогов

##### API

```
chat.setAllMessageRead(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Он содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| scope | String \| undefined | Установить область действия обработки сообщений. Допустимые значения: `TencentCloudChat.TYPES.READ_ALL_C2C_MSG`: установить непрочитанные сообщения всех личных диалогов как прочитанные`TencentCloudChat.TYPES.READ_ALL_GROUP_MSG`: установить непрочитанные сообщения всех групповых диалогов как прочитанные`TencentCloudChat.TYPES.READ_ALL_MSG `(значение по умолчанию): установить непрочитанные сообщения всех личных и групповых диалогов как прочитанные |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Установить все непрочитанные сообщения во всех диалогах как прочитанные// Эквивалентно `chat.setAllMessageRead({scope: TencentCloudChat.TYPES.READ_ALL_MSG})`let promise = chat.setAllMessageRead();promise.then(function(imResponse) {  // Непрочитанные сообщения успешно установлены как прочитанные.  // Значения атрибута `unreadCount` всех диалогов устанавливаются на `0`.}).catch(function(imError) {  // Не удалось установить непрочитанные сообщения как прочитанные  console.warn('setAllMessageRead error:', imError);});
```

```
// Установить все непрочитанные сообщения в личных диалогах как прочитанныеlet promise = chat.setAllMessageRead({scope: TencentCloudChat.TYPES.READ_ALL_C2C_MSG});promise.then(function(imResponse) {  // Непрочитанные сообщения успешно установлены как прочитанные.  // Значения атрибута `unreadCount` всех личных диалогов устанавливаются на `0`.}).catch(function(imError) {  // Не удалось установить непрочитанные сообщения как прочитанные  console.warn('setAllMessageRead error:', imError);});
```

```
// Установить все непрочитанные сообщения в групповых диалогах как прочитанныеlet promise = chat.setAllMessageRead({scope: TencentCloudChat.TYPES.READ_ALL_GROUP_MSG});promise.then(function(imResponse) {  // Непрочитанные сообщения успешно установлены как прочитанные.  // Значения атрибута `unreadCount` всех групповых диалогов устанавливаются на `0`.}).catch(function(imError) {  // Не удалось установить непрочитанные сообщения как прочитанные  console.warn('setAllMessageRead error:', imError);});
```

## Отправка сообщения, исключённого из счётчика непрочитанных сообщений диалога

В обычных случаях отправленные личные сообщения и групповые сообщения включаются в счётчик непрочитанных сообщений. Атрибут `unreadCount` объекта `Conversation` указывает на количество непрочитанных сообщений в диалоге.
Если вы хотите отправлять сообщения, которые не будут включаться в счётчик непрочитанных сообщений, такие как подсказки или контрольные сообщения, обратитесь к следующему примеру кода.

##### **Примеры**

```
chat.sendMessage(message, {  messageControlInfo: {    // `unreadCount` диалога не обновляется (сообщение сохраняется на сервере роуминга).    excludedFromUnreadCount: true,    // `lastMessage` диалога не обновляется (сообщение сохраняется на сервере роуминга).     excludedFromLastMessage: true  }});
```

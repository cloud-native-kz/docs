# Обзор

## Обзор Conversation

В Chat SDK существует три типа `Conversation`:

- One-to-one (C2C): Один-на-один разговор только между двумя пользователями. Его метод формирования: `C2C${userID}`.
- GROUP: Групповой разговор между участниками группы. Его метод формирования: `GROUP${groupID}`.
- `@TIM#SYSTEM` (Сеанс системных уведомлений)

Вы можете использовать API класса conversation для отображения/обновления списка разговоров, обновления количества непрочитанных сообщений, закрепления разговора в верхней части и отключения уведомлений о сообщениях.

## Политика сохранения разговоров

По умолчанию клиент может получить 100 недавних контактных разговоров из облака, а после обновления на [Pro edition, Pro Plus edition или Enterprise edition](https://trtc.io/buy/chat) может быть настроен для получения до 500 недавних контактных разговоров. Продолжительность разговора соответствует времени сохранения последнего сообщения в разговоре. Сообщения сохраняются в течение 7 дней по умолчанию, что означает, что разговор также сохраняется в течение 7 дней по умолчанию. Обратитесь к [Ценообразованию](https://trtc.io/pricing/chat).

## Класс Conversation

| Атрибут | Тип | Описание |
| --- | --- | --- |
| conversationID | String | Его метод формирования:`C2C${userID}` - Один-на-один разговор только между двумя пользователями`GROUP${groupID}` - Групповой разговор между участниками группы`@TIM#SYSTEM` - Сеанс системных уведомлений |
| type | String | `TencentCloudChat.TYPES.CONV_C2C` - Один-на-один разговор только между двумя пользователями`TencentCloudChat.TYPES.CONV_GROUP` - Групповой разговор между участниками группы`TencentCloudChat.TYPES.CONV_SYSTEM` - Сеанс системных уведомлений |
| unreadCount | Number | Количество непрочитанных сообщений. Групповые разговоры типа `TencentCloudChat.TYPES.GRP_MEETING` / `TencentCloudChat.TYPES.GRP_AVCHATROOM` не записывают количество непрочитанных сообщений, и значение этого поля равно 0. |
| lastMessage | Object | Последнее сообщение в разговоре. Его атрибуты описаны ниже.`nick` - String - Никнейм отправителя сообщения`nameCard` - String - Карточка имени в группе отправителя сообщения`lastTime` - Number - Временная метка сообщения, в секундах`lastSequence` - Number - Последовательность сообщения`fromAccount` - String - userID отправителя сообщения`isRevoked` - Boolean - Было ли сообщение отозвано, `true` означает, что оно было отозвано, значение по умолчанию `false``revoker` - String \| null - userID инициатора отзыва сообщения`isPeerRead` - Boolean - Было ли прочитано сообщение в один-на-один разговоре другой стороной; после того, как другая сторона вызовет `setMessageRead` для отчета о том, что она прочитала, `isPeerRead` становится `true`. По умолчанию `false``messageForShow` - String - Содержание сообщения для отображения. Возможные значения: содержание текстового сообщения, "[Изображение]", "[Голос]", "[Геолокация]", "[Эмодзи]", "[Файл]", "[Пользовательское сообщение]". Если это поле не соответствует вашим требованиям, вы можете использовать payload для пользовательского отображения.`type` - String`TencentCloudChat.TYPES.MSG_TEXT` \| Текстовое сообщение`TencentCloudChat.TYPES.MSG_IMAGE` \| Сообщение с изображением`TencentCloudChat.TYPES.MSG_AUDIO` \| Голосовое сообщение`TencentCloudChat.TYPES.MSG_VIDEO` \| Видеосообщение`TencentCloudChat.TYPES.MSG_FILE` \| Сообщение с файлом`TencentCloudChat.TYPES.MSG_LOCATION` \| Сообщение о геолокации`TencentCloudChat.TYPES.MSG_CUSTOM` \| Пользовательское сообщение`TencentCloudChat.TYPES.MSG_GRP_TIP` \| Сообщение с подсказкой группы`TencentCloudChat.TYPES.MSG_GRP_SYS_NOTICE` \| Сообщение системного уведомления группы.`payload` - Object - Содержание сообщения |
| groupProfile | Group | Профиль группы группового разговора. |
| userProfile | Profile | Профиль пользователя один-на-один разговора. |
| groupAtInfoList | Array.<GroupAtInfo> | Список информации об @упоминаниях в групповом разговоре. Разработчик может отображать эффекты такие как [Кто-то упомянул меня] или [@all] в списке разговоров на основе этой информации. |
| remark | String | Примечание друга. Имеет значение только когда есть C2C разговор, другая сторона является моим другом, и я установил примечание для этого друга. |
| isPinned | Boolean | Закреплен ли разговор. |
| messageRemindType | String | Типы напоминаний о сообщениях, конкретно следующие:`TencentCloudChat.TYPES.MSG_REMIND_ACPT_AND_NOTE` - Сообщения будут получены, когда пользователь в сети, и будут получены push-уведомления об автономном режиме, когда пользователь в автономном режиме.`TencentCloudChat.TYPES.MSG_REMIND_ACPT_NOT_NOTE` - SDK получает сообщение и уведомляет вас (отправляя событие получения сообщения), вы не отображаете уведомление. Этот параметр обычно используется для реализации отключения уведомлений о сообщениях.`TencentCloudChat.TYPES.MSG_REMIND_DISCARD` - SDK отклоняет сообщение.`TencentCloudChat.TYPES.NOT_RECEIVE_OFFLINE_PUSH_EXCEPT_AT` - Сообщения будут получены, когда пользователь в сети, и только сообщения группы @ будут получены, когда пользователь в автономном режиме. |
| markList | Array | Список тегов разговора, конкретно следующие:`TencentCloudChat.TYPES.CONV_MARK_TYPE_STAR` - Добавить разговор в избранное`TencentCloudChat.TYPES.CONV_MARK_TYPE_UNREAD` - Отметить разговор как непрочитанный`TencentCloudChat.TYPES.CONV_MARK_TYPE_FOLD` - Свернуть разговор`TencentCloudChat.TYPES.CONV_MARK_TYPE_HIDE` - Скрыть разговор |
| customData | String | Пользовательские данные для разговора. |
| conversationGroupList | Array | Список групп разговоров, к которым принадлежит разговор. |
| draftText | String | Черновик разговора. Сохраняется только локально, а не в облаке. |

### GroupAtInfo

| Имя | Тип | Описание |
| --- | --- | --- |
| groupID | String | ID группы. |
| messageSequence | Number | Последовательность сообщений с информацией об @упоминании. |
| atTypeArray | Array.<Number> | Массив типов `at`, значения перечисления `at` в групповых разговорах следующие:`TencentCloudChat.TYPES.CONV_AT_ME` - Кто-то упомянул меня.`TencentCloudChat.TYPES.CONV_AT_ALL` - Упомянуть всех в группе.`TencentCloudChat.TYPES.CONV_AT_ALL_AT_ME` - Упомянуть всех в группе и также упомянуть меня отдельно. |

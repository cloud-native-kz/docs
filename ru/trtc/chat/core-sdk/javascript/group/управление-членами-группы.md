# Управление участниками группы

## Обзор функции

Управление участниками группы включает получение списка участников, отключение звука участникам группы, удаление участников группы, предоставление прав доступа и передачу права собственности на группу.

## Получение списка участников группы

> **Примечание:** Этот API возвращает информацию об участниках группы с полем `isOnline`, указывающим, находится ли участник в сети. Этот API поддерживает получение временной метки окончания отключения звука (`muteUntil`) участников группы. На основе этого значения вы можете определить, отключен ли звук у участника, и оставшееся время отключения звука. Этот API получает участников группы постранично и не может использоваться напрямую для получения общего количества участников группы. Чтобы получить общее количество участников группы (`memberCount`), используйте `getGroupProfile`. Когда API возвращает смещение 0, это означает, что все участники группы получены. Выпуск Pro, выпуск Pro Plus или выпуск Enterprise поддерживают получение списка участников группы с указанными тегами в AVChatRoom на основе параметра `filter`. Пожалуйста, обратитесь к `mark` и `GroupMemberList` для маркировки участников группы. Следующие специальные ограничения добавлены для AVChatRoom: Выпуск Pro, выпуск Pro Plus или выпуск Enterprise поддерживают получение до 1000 недавно присоединившихся участников группы, при этом новые участники ранжируются первыми. Для использования этой функции необходимо приобрести пакет выпуска Pro, выпуска Pro Plus или выпуска Enterprise и включить функцию в [Chat Console](https://console.trtc.io/chat/qun-setting). В выпуске Pro, выпуске Pro Plus или выпуске Enterprise SDK будет игнорировать параметр count, и один запрос вернёт до 500 участников группы по умолчанию. В выпуске Pro, выпуске Pro Plus или выпуске Enterprise этот API можно вызывать не более одного раза в три секунды. Для периодического запроса списка участников группы рекомендуется вызывать API один раз каждые десять секунд. Информация профиля участника группы поддерживает только поля `userID`, `nick`, `avatar`, `joinTime` и `role`. Для установки информации о нике и аватаре вызовите: `updateMyProfile`.

Чтобы включить функцию списков участников группы в режиме реального времени в AVChatRoom для выпуска Pro, выпуска Pro Plus или выпуска Enterprise, войдите в [Chat Console](https://console.trtc.io/chat/qun-setting) и измените соответствующую конфигурацию. Страница конфигурации показана ниже:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/93415a0f137e11efa2935254005ac0ca.png)

##### **API**

```
chat.getGroupMemberList(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| groupID | String | ID группы |
| role | String | Роли участников группы. По умолчанию SDK получает всех участников группы, но вы можете установить это поле для получения списка участников группы с указанными ролями. Поддерживаемые значения: `TencentCloudChat.GRP_MBR_ROLE_OWNER` - Владелец группы`TencentCloudChat.GRP_MBR_ROLE_ADMIN` - Администратор группы`TencentCloudChat.GRP_MBR_ROLE_MEMBER` - Обычный участник группы |
| count | Number | Количество записей для получения. Значение по умолчанию: 15, максимальное значение: 100, чтобы избежать ошибок ответа из-за больших пакетов. Если передано более 100, будут получены только первые 100 (выпуск Pro, выпуск Pro Plus или выпуск Enterprise для AVChatRoom игнорируют параметр count при использовании этого API). |
| offset | Number \| String | Смещение, по умолчанию начинается получение с 0. При использовании в Community это поле имеет тип `String`. |
| filter | Number | Пользовательские теги для участников группы, поддерживаются только в AVChatRoom. |

##### **Возвращаемое значение**

`Promise`

##### **Пример**

```
// Получить всех администраторов группыlet promise = chat.getGroupMemberList({  groupID: 'group1',  role: TencentCloudChat.TYPES.GRP_MBR_ROLE_ADMIN,  count: 15,  offset: 0});promise.then(function(imResponse) {  console.log(imResponse.data.memberList);}).catch(function(imError) {  console.warn('getGroupMemberList error:', imError);});
```

```
// Получить временную метку окончания отключения звука участников группы.let promise = tim.getGroupMemberList({   groupID: 'group1',   count: 30,   offset:0, }); // Получить 30 участников группы начиная с 0promise.then(function(imResponse) {  console.log(imResponse.data.memberList); // Список участников группы  for (let groupMember of imResponse.data.memberList) {    if (groupMember.muteUntil * 1000  > Date.now()) {      console.log(`${groupMember.userID} muted`);    } else {      console.log(`${groupMember.userID} not muted`);    }  }}).catch(function(imError) {    console.warn('getGroupMemberProfile error:', imError);});
```

```
// Выпуск Pro, выпуск Pro Plus или выпуск Enterprise поддерживают получение списка участников в режиме реального времени в AVChatRoomlet promise = chat.getGroupMemberList({   groupID: 'group1',   filter: 1000,   offset:0, // По умолчанию начинается получение с 0 });promise.then(function(imResponse) {  console.log(imResponse.data.memberList); // Список участников группы}).catch(function(imError) {  console.warn('getGroupMemberList error:', imError);});
```

```
// Отфильтровать список участников в режиме реального времени в AVChatRoomlet promise = chat.getGroupMemberList({ groupID: 'group1', filter: 1000, offset: 0 });
promise.then(function(imResponse) {
  console.log(imResponse.data.memberList);
}).catch(function(imError) {
  console.warn('getGroupMemberList error:', imError);
});
```

## Отключение звука

### Отключить звук конкретным участникам группы

> **Примечание:** Группы типа `TencentCloudChat.TYPES.GRP_WORK` не поддерживают эту операцию. Группы типа `TencentCloudChat.TYPES.GRP_AVCHATROOM` не поддерживают эту операцию. Чтобы заблокировать или удалить участников группы в AVChatRoom, необходимо приобрести выпуск Pro, выпуск Pro Plus или выпуск Enterprise и использовать интерфейс `deleteGroupMember`. Если вам нужно отключить звук всем участникам, обратитесь к `updateGroupProfile`. Поддерживает установку длительности отключения звука для участников сообщества в разделах путём передачи topicID как groupID. Только владелец группы и администраторы имеют это разрешение: владелец группы может отключить/включить звук администраторам и обычным участникам группы. Администраторы могут отключить/включить звук обычным участникам группы.

##### **API**

```
chat.setGroupMemberMuteTime(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| groupID | String | ID группы или ID раздела |
| userID | String | ID пользователя |
| muteTime | Number | Длительность отключения звука в секундах. Например, установка значения 1000 означает отключение звука пользователю на 1000 секунд с текущего момента; установка значения 0 означает включение звука. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.setGroupMemberMuteTime({  groupID: 'group1',  userID: 'user1',  muteTime: 600 // Отключить звук на 10 минут; установите 0 для включения звука});promise.then(function(imResponse) {  console.log(imResponse.data.group); // Новый профиль группы  console.log(imResponse.data.member); // Новый профиль участника}).catch(function(imError) {  console.warn('setGroupMemberMuteTime error:', imError); // Информация об ошибке});
```

```
// Установить период отключения звука участнику группы в разделеlet promise = chat.setGroupMemberMuteTime({  groupID: 'topicID',  userID: 'user1',  muteTime: 600 // Отключить звук на 10 минут; установите 0 для включения звука});promise.then(function(imResponse) {  console.log(imResponse.data.group); // Новый профиль группы  console.log(imResponse.data.member); // Новый профиль участника}).catch(function(imError) {  console.warn('setGroupMemberMuteTime error:', imError); // Информация об ошибке});
```

### Отключить звук всей группе

```
// Глобальное отключение звукаlet promise = chat.updateGroupProfile({  groupID: 'group1',  muteAllMembers: true, // true: отключить звук всем; false: включить звук всем});promise.then(function(imResponse) {  console.log(imResponse.data.group) // Подробный профиль группы после изменения}).catch(function(imError) {  console.warn('updateGroupProfile error:', imError); // Информация об ошибке});
```

## Добавление участников группы

> **Примечание:** `TencentCloudChat.TYPES.GRP_WORK` (Рабочая группа): По умолчанию любой участник группы может напрямую приглашать других в группу без необходимости согласия приглашаемого, и они добавляются в группу напрямую. Вы можете установить `inviteOption` в `TencentCloudChat.TYPES.INVITE_OPTIONS_DISABLE_INVITE`, чтобы запретить любому участнику группы приглашать других в группу. `TencentCloudChat.TYPES.GRP_PUBLIC` (Публичная) / `TencentCloudChat.TYPES.GRP_MEETING` (Встреча): По умолчанию участники группы не могут приглашать других в группу. Вы можете управлять параметрами приглашения, установив `inviteOption`. `TencentCloudChat.TYPES.GRP_COMMUNITY` (Сообщество): По умолчанию участники группы могут приглашать других в группу. `TencentCloudChat.TYPES.GRP_AVCHATROOM` (AVChatRoom): Никому не разрешается приглашать других в группу (включая администраторов приложения). ID пользователя добавляемого участника должен быть действительным учётным записям пользователя; в противном случае API вернёт 10019.

##### **API**

```
chat.addGroupMember(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| groupID | String | ID группы |
| userIDList | Array.<String> | Массив ID участников группы для добавления, максимум 20 участников за один раз. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.addGroupMember({groupID: 'group1', userIDList: ['user1','user2','user3']});promise.then(function(imResponse) {  console.log(imResponse.data.successUserIDList);  console.log(imResponse.data.failureUserIDList);  console.log(imResponse.data.existedUserIDList);  // Пользователь userX может присоединиться к максимум N группам.   // Если userX уже присоединился к N группам, попытка добавить userX в качестве участника группы снова приведёт к тому, что userX не сможет нормально присоединиться к группе.  // SDK помещает информацию userX в overLimitUserIDList для обработки стороной доступа.  console.log(imResponse.data.overLimitUserIDList);  console.log(imResponse.data.group);}).catch(function(imError) {  console.warn('addGroupMember error:', imError);});
```

## Удаление

> **Примечание:** Выпуск Pro, выпуск Pro Plus или выпуск Enterprise поддерживают удаление участников AVChatRoom. Вы можете достичь эффекта [Блокировка участников группы] в AVChatRoom, вызвав этот интерфейс. Поле длительности удаления поддерживается только AVChatRoom. После удаления участника группы из AVChatRoom в течение **периода удаления**, если пользователь хочет повторно присоединиться к группе, администратор приложения должен вызвать [REST API](https://trtc.io/document/50297?product=chat&menulabel=serverapis) для его разблокирования. **После периода удаления** пользователь может самостоятельно снова присоединиться к группе живого чата.

##### **API**

```
chat.deleteGroupMember(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| groupID | String | ID группы или ID раздела |
| userIDList | Array | Список ID участников группы для удаления |
| reason | String | Причина удаления |
| duration | Number | Длительность удаления должна быть больше 0 (поддерживается только AVChatRoom) |

##### **Возвращаемое значение**

`Promise`

##### **Пример**

```
// Удаление участников группы из групп, не являющихся живыми группамиlet promise = chat.deleteGroupMember({   groupID: 'group1',   userIDList:['user1'],   reason: 'You violated the rules, I am kicking you out!', });promise.then(function(imResponse) {  console.log(imResponse.data.group); // Профиль группы после удаления участника  console.log(imResponse.data.userIDList); // Список userID удалённого участника группы}).catch(function(imError) {  console.warn('deleteGroupMember error:', imError); // Информация об ошибке});
```

```
// Удаление участников группы из AVChatRoomlet promise = chat.deleteGroupMember({   groupID: 'group1',   userIDList:['user1'],   reason: "You violated the rules, I'm kicking you out!",   duration: 60, });promise.then(function(imResponse) {  console.log(imResponse.data.group); // Профиль группы после удаления участника  console.log(imResponse.data.userIDList); // Список userID удалённого участника группы}).catch(function(imError) {  console.warn('deleteGroupMember error:', imError); // Информация об ошибке});
```

## Изменение роли участника группы

> **Примечание:** Группы типа `TencentCloudChat.TYPES.GRP_WORK` не поддерживают эту операцию. Группы типа `TencentCloudChat.TYPES.GRP_AVCHATROOM` не поддерживают эту операцию.

##### **API**

```
chat.setGroupMemberRole(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| groupID | String | ID группы или ID раздела |
| userID | String | ID пользователя |
| role | String | Допустимые значения: `TencentCloudChat.TYPES.GRP_MBR_ROLE_ADMIN` (Администратор группы), `TencentCloudChat.TYPES.GRP_MBR_ROLE_MEMBER` (Обычный участник группы), `TencentCloudChat.TYPES.GRP_MBR_ROLE_CUSTOM` (Пользовательские роли участников группы, поддерживаются только в Community) |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.setGroupMemberRole({  groupID: 'group1',  userID: 'user1',  role: TencentCloudChat.TYPES.GRP_MBR_ROLE_ADMIN // Установить user1 администратором в группе group1});promise.then(function(imResponse) {  console.log(imResponse.data.group); // Новый профиль группы  console.log(imResponse.data.member); // Новый профиль участника}).catch(function(imError) {  console.warn('setGroupMemberRole error:', imError); // Информация об ошибке});
```

## Получение количества участников группы в режиме реального времени

> **Примечание:** Рекомендуется управлять частотой вызова этого интерфейса для запроса количества людей в AVChatRoom между 5-10 секундами один раз; для запроса количества участников в режиме реального времени других типов групп ограничение частоты составляет один раз в 60 секунд.

##### **API**

```
chat.getGroupOnlineMemberCount(groupID);
```

##### **Параметры**

| Имя | Тип | Описание |
| --- | --- | --- |
| groupID | String | ID группы |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Запрос количества участников в режиме реального времени в AVChatRoomlet promise = chat.getGroupOnlineMemberCount('group1');promise.then(function(imResponse) {  console.log(imResponse.data.memberCount);}).catch(function(imError) {  console.warn('getGroupOnlineMemberCount error:', imError); // Информация об ошибке});
```

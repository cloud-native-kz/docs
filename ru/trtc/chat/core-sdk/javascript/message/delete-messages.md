# Удаление сообщений

## Обзор функции

После успешного удаления сообщения его `isDeleted` становится `true`. В личных разговорах удаленные сообщения не могут быть загружены при следующем входе, но получатель не будет затронут. В групповых разговорах удаленные сообщения не могут быть загружены при следующем входе, но другие члены группы не будут затронуты.

> **Примечание:** За один раз можно удалить до 30 сообщений. Если выбрано более 30 сообщений, будут удалены первые 30 сообщений. Удаляемые сообщения должны быть из одного разговора, то есть разговора первого сообщения в списке сообщений. Этот API можно вызывать только один раз в секунду. Удаленные сообщения не синхронизируются. Нельзя удалять сообщения из аудио-видео групп (AVChatRoom), и если вы вызовете этот API, будет возвращен код ошибки 10035. Этот API не поддерживает удаление системных уведомлений группы.

## Отображение интерфейса

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/5b31858bcbe011efa2ff52540044a08e.png)

##### **API**

```
chat.deleteMessage(messageList);
```

##### **Параметры**

| Имя | Тип | Описание |
| --- | --- | --- |
| messageList | Array | Список сообщений (до 30) из одного разговора |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.deleteMessage([message1, message2, message3, ...]);promise.then(function(imResponse) {  // Messages deleted successfully}).catch(function(imError) {  // Failed to delete the messages  console.warn('deleteMessage error:', imError);});
```

# Исторические сообщения

## Обзор функции

SDK поддерживает получение сообщений по страницам, последовательности или диапазону времени. Исторические сообщения, хранящиеся в облаке, подчиняются следующим временным ограничениям:

- Издание Developer: Бесплатный период хранения составляет 7 дней и не может быть продлен.
- Издание Standard: Бесплатный период хранения составляет 7 дней и не может быть продлен.
- Издание Premium: Бесплатный период хранения составляет 30 дней и не может быть продлен.

> **Примечание** Продление периода хранения исторических сообщений — это услуга с добавленной стоимостью. Вы можете войти в [Chat Console](https://console.trtc.io/) для изменения соответствующей конфигурации. Дополнительную информацию о выставлении счетов см. в разделе [Цены](https://trtc.io/pricing/chat). Сообщения с форматированным контентом (такие как изображения, файлы и аудио) имеют те же периоды хранения, что и исторические сообщения.

## Получение списка сообщений

### getMessageList

Этот API используется для получения списка сообщений в указанной беседе по страницам. Его необходимо вызвать при первом отображении списка сообщений после входа пользователя в беседу или при прокрутке пользователем списка вниз для просмотра дополнительных сообщений.

> **Примечание** Этот API может получать историю сообщений для всех типов групп **за исключением AVChatRoom**. По умолчанию **AVChatRoom** не хранит историю сообщений. Новые участники могут видеть только сообщения, отправленные после их присоединения к комнате. Для повышения вовлеченности вы можете включить историю сообщений для новых участников: Перейдите в: **Chat Console > Configuration > Group Configuration > Message History for New Members** Конфигурация: Установите количество последних сообщений, видимых новым участникам Доступность: только **Pro edition, Pro Plus edition** или **Enterprise edition** Ограничения: Новые участники могут получить до **50** сообщений из **последних 24 часов** через событие `MESSAGE_RECEIVED` Если вы хотите выполнить поиск локальных сообщений, используйте `findMessage`. Если вы хотите выполнить поиск всех облачных сообщений, используйте `searchCloudMessages` (это услуга с добавленной стоимостью, требующая плагина Cloud Search. [Купить здесь](https://console.trtc.io/chat/plugin/TUICloudSearch)).

##### **API**

```
chat.getMessageList(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Он содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| conversationID | String | Идентификатор беседы, который состоит из: `C2C${userID}` (для одиночных чатов) `GROUP${groupID}` (для групповых чатов) `GROUP${topicID}` (тема) `@TIM#SYSTEM` (беседа системных уведомлений) |
| nextReqMessageID | String \| undefined | Идентификатор сообщения для последующего получения по страницам. Оставьте это поле пустым для первого получения и укажите значение, возвращаемое API `getMessageList` для последующего получения. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Получить список сообщений в первый раз при открытии беседыlet promise = chat.getMessageList({conversationID: 'C2Ctest'});promise.then(function(imResponse) {  const messageList = imResponse.data.messageList; // Список сообщений  // Этот параметр должен быть передан при следующем получении по страницам.  const nextReqMessageID = imResponse.data.nextReqMessageID;  // Указывает, получены ли все сообщения.  const isCompleted = imResponse.data.isCompleted;});
```

```
// Прокрутить список сообщений вниз для просмотра дополнительных сообщенийlet promise = chat.getMessageList({conversationID: 'C2Ctest', nextReqMessageID});promise.then(function(imResponse) {  const messageList = imResponse.data.messageList; // Список сообщений  // Этот параметр должен быть передан при следующем получении по страницам.  const nextReqMessageID = imResponse.data.nextReqMessageID;  // Указывает, получены ли все сообщения.  const isCompleted = imResponse.data.isCompleted; });
```

### getMessageListHopping

Этот API используется для получения списка сообщений по указанной последовательности или диапазону времени.

##### **API**

```
chat.getMessageListHopping(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Он содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| conversationID | String | Идентификатор беседы, который состоит из: `C2C${userID}` (для одиночных чатов) `GROUP${groupID}` (для групповых чатов) `GROUP${topicID}` (тема) |
| sequence | Number \| undefined | `sequence` сообщения, начиная с которого получить ротирующие групповые сообщения |
| time | Number \| undefined | Серверное время сообщения, начиная с которого получить ротирующие одиночные сообщения |
| direction | Number | Направление получения сообщений, по умолчанию `0`. 0: получение в обратном хронологическом порядке для получения более старых сообщений; 1: получение в хронологическом порядке для получения более новых сообщений |
| count | Number \| undefined | Количество получаемых сообщений. По умолчанию составляет 15, что также является максимальным значением, указывающим, что одновременно можно получить и вернуть до 15 сообщений. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Получить ротирующие групповые сообщения по последовательности.// `direction`: 0: получение в обратном хронологическом порядке для получения более старых сообщений// 1: получение в хронологическом порядке для получения более новых сообщенийlet promise = chat.getMessageListHopping({  conversationID: 'GROUPtest',  sequence: 100,  count: 15,  direction: 0});promise.then(function(imResponse) {  const messageList = imResponse.data.messageList; // Список сообщений});
```

****

```
// Получить ротирующие одиночные сообщения по диапазону времени.// `direction`: 0: получение в обратном хронологическом порядке для получения более старых сообщений// 1: получение в хронологическом порядке для получения более новых сообщенийlet promise = chat.getMessageListHopping({  conversationID: 'C2Ctest',  time: xxx,  count: 15,  direction: 0});promise.then(function(imResponse) {  const messageList = imResponse.data.messageList; // Список сообщений});
```

# Обзор

## Класс Message

В Chat SDK `Message` представляет объект сообщения, который используется для описания атрибутов сообщения, таких как тип, содержание и ID беседы.

| Атрибут | Тип | Значение по умолчанию | Описание |
| --- | --- | --- | --- |
| ID | String | - | ID сообщения. Правило для объединения ID сообщения: `${senderTinyID}-${clientTime}-${random}`, что совпадает с правилом объединения ID сообщения нативного чата. |
| type | String | - | Тип сообщения. Допустимые значения:`TencentCloudChat.TYPES.MSG_TEXT` - текстовое сообщение`TencentCloudChat.TYPES.MSG_IMAGE` - сообщение с изображением`TencentCloudChat.TYPES.MSG_AUDIO` - аудиосообщение`TencentCloudChat.TYPES.MSG_VIDEO` - видеосообщение`TencentCloudChat.TYPES.MSG_FILE` - сообщение с файлом`TencentCloudChat.TYPES.MSG_CUSTOM` - пользовательское сообщение`TencentCloudChat.TYPES.MSG_MERGER` - объединённое сообщение`TencentCloudChat.TYPES.MSG_LOCATION` - сообщение о местоположении`TencentCloudChat.TYPES.MSG_GRP_TIP` - подсказка группы`TencentCloudChat.TYPES.MSG_GRP_SYS_NOTICE` - системное уведомление группы |
| payload | Object | - | Содержание сообщения. Допустимые значения:`TextPayload``ImagePayload``AudioPayload``VideoPayload``FilePayload``CustomPayload``MergerPayload``LocationPayload``FacePayload``GroupTipPayload``GroupSystemNoticePayload` |
| conversationID | String | - | ID беседы сообщения. |
| conversationType | String | - | Тип беседы сообщения. Допустимые значения:`TencentCloudChat.TYPES.CONV_C2C` --- беседа C2C (один на один)`TencentCloudChat.TYPES.CONV_GROUP` --- групповая беседа`TencentCloudChat.TYPES.CONV_SYSTEM` --- системная беседа |
| to | String | - | `userID` получателя. |
| from | String | - | `userID` отправителя. По умолчанию при отправке сообщения устанавливается ID текущего вошедшего в систему пользователя. |
| flow | String | - | Поток сообщения.`in` --- полученное сообщение`out` --- отправленное сообщение |
| time | Number | - | Временная метка сообщения в секундах. |
| status | String | - | Статус сообщения.`unSend` --- не отправлено`success` --- успешно отправлено`fail` --- ошибка отправки |
| isRevoked | Boolean | false | Является ли сообщение отозванным сообщением. `true` означает да. |
| priority | String | `TencentCloudChat.TYPES.MSG_PRIORITY_NORMAL` | Приоритет сообщения. Если сообщения в группе превышают частотный лимит, бэкенд будет доставлять сообщения с высоким приоритетом в первую очередь. Поддерживаемые значения перечисления:`TencentCloudChat.TYPES.MSG_PRIORITY_HIGH``TencentCloudChat.TYPES.MSG_PRIORITY_NORMAL (по умолчанию)``TencentCloudChat.TYPES.MSG_PRIORITY_LOW``TencentCloudChat.TYPES.MSG_PRIORITY_LOWEST` |
| nick | String | - | Прозвище отправителя сообщения. Этот атрибут необходимо установить путём заранее вызова API `updateMyProfile`. |
| avatar | String | - | URL фото профиля отправителя сообщения. Этот атрибут необходимо установить путём заранее вызова API `updateMyProfile`. |
| nameCard | String | - | Карточка имени группы отправителя, отправляющего не аудио-видео сообщение группы. Этот атрибут также называется прозвищем отправителя сообщения в группе и должен быть установлен заранее путём вызова API `setGroupMemberNameCard`. |
| atUserList | Array | - | Это поле хранит значения `userID` членов группы, упомянутых в групповом сообщении. |
| cloudCustomData | String | '' | Пользовательские данные сообщения, которые сохраняются в облаке, будут отправлены получателю и могут быть загружены после удаления и переустановки приложения. |
| isDeleted | Boolean | false | Является ли сообщение удалённым сообщением. `true` означает да. |
| isModified | Boolean | false | Является ли сообщение изменённым сообщением. `true` означает да. |
| needReadReceipt | Boolean | false | Требуется ли квитанция о прочтении. `true` означает да. Для использования необходимо приобрести пакет Pro edition, Pro Plus edition или Enterprise edition. |
| readReceiptInfo | Object | - | Информация о квитанции о прочтении сообщения`readCount` - количество прочитанных сообщений, которое можно запросить путём вызова API `getMessageReadReceiptList`; для запроса членов группы, которые прочитали сообщение, вызовите `getGroupMessageReadMemberList`.`unreadCount` - количество непрочитанных сообщений, которое можно запросить путём вызова API `getMessageReadReceiptList`.`isPeerRead` - была ли отправлена квитанция о прочтении для сообщения C2C со стороны собеседника, это свойство будет обновлено, когда отправитель сообщения получит уведомление о квитанции о прочтении или загрузит роуминг после успешного вызова `sendMessageReadReceipt` получателем сообщения.`timestamp` - время отправки квитанции о прочтении со стороны собеседника для сообщения C2C в секундах. Этот атрибут будет обновлен, когда отправитель сообщения получит уведомление о квитанции о прочтении или загрузит роуминг после успешного вызова `sendMessageReadReceipt` получателем сообщения. |
| isBroadcastMessage | Boolean | false | Является ли сообщение трансляционным сообщением. `true` означает да. |
| isSupportExtension | Boolean | false | поддерживает ли сообщение расширения. `true` означает да. Для использования необходимо приобрести пакет Pro edition, Pro Plus edition или Enterprise edition. |
| revokerInfo | Object | - | Информация об отзывающем сообщение. |
| revokeReason | String | - | Причина отзыва сообщения. |
| hasRiskContent | Boolean | - | Отмечены ли сообщения с изображением, голосом и видео как имеющие угрозы безопасности, по умолчанию false. Если содержимое отправляемого вами голосового или видеосообщения не соответствует требованиям, SDK вызовет событие `MESSAGE_MODIFIED` после асинхронной облачной проверки. |

### TextPayload

| Имя | Тип | Описание |
| --- | --- | --- |
| text | String | Содержание текстового сообщения |

### ImagePayload

| Имя | Тип | Описание |
| --- | --- | --- |
| uuid | String | Уникальный идентификатор изображения |
| imageFormat | Format | Тип формата изображения. JPG/JPEG = 1, GIF = 2, PNG = 3, BMP = 4, other = 255 |
| imageInfoArray | Array.<Object> | Информация об изображении. Каждый элемент содержит следующие атрибуты:width - Number - ширина изображенияheight - Number - высота изображенияurl - String - адрес изображения, который можно использовать для отображенияsize - Number - размер изображения в байтахtype - Number - 0, исходное изображение; 1, сжатое изображение 198p; 2, сжатое изображение 720p |

### AudioPayload

| Имя | Тип | Описание |
| --- | --- | --- |
| uuid | String | Уникальный идентификатор |

| size | Number | Размер файла в байтах |
| second | Number | Продолжительность аудио в секундах |

### VideoPayload

| --- | --- | --- |

| videoSize | Number | Размер видеофайла в байтах |
| videoUrl | String | Адрес видеофайла, который можно использовать для воспроизведения |

| snapshotUrl | String | адрес изображения снимка экрана, который можно использовать для отображения (требуется [tim-upload-plugin](https://www.npmjs.com/package/tim-upload-plugin)) |

### MergerPayload

| Имя | Тип | Описание |
| --- | --- | --- |
| downloadKey | String | Ключ для загрузки объединённого сообщения. Если объединённое сообщение имеет большой размер, SDK будет хранить его в облаке, используя этот ключ в качестве уникального идентификатора сообщения. |
| messageList | Array.<SimplifiedMessage> | Список объединённых сообщений. |
| title | String | Заголовок объединённого сообщения, например "История чатов центра талантов в Большом заливе". |
| abstractList | String | Список дайджестов. Вы можете установить информацию дайджеста в разных форматах для разных типов сообщений, например: для текстового сообщения дайджест может быть в формате "отправитель:текст". Для сообщения с изображением дайджест может быть в формате "отправитель:[изображение]". Для сообщения с файлом дайджест может быть в формате "отправитель:[файл]". |
| compatibleText | String | Совместимый текст. SDK ранних версий не поддерживает объединённые сообщения и по умолчанию будет отправлять текстовое сообщение с содержимым `compatibleText`. |
| layersOverLimit | Boolean | Превышает ли количество уровней вложенности объединений лимит. true: лимит превышен и сообщение усечено. Значение по умолчанию: false. |

### GroupTipPayload

| Имя | Тип | Описание |
| --- | --- | --- |
| groupJoinType | Number | Тип присоединения к группе. Поддерживаемые значения следующие:0 - значение по умолчанию, указывающее, что операция не является операцией присоединения к группе.1 - пользователь запрашивает присоединение к группе.2 - пользователь приглашён в группу. |
| operatorID | String | ID пользователя, выполняющего операцию. |
| operatorInfo | Object | Информация пользователя, выполняющего эту операцию (поддерживается с версии v3.4.2 и выше). |
| operationType | Number | Тип операции. Поддерживаемые значения следующие:1 - член группы присоединился к группе.2 - член группы покинул группу. 3 - член группы удален из группы. 4 - члену группы предоставлена роль администратора группы. 5 - роль администратора группы отозвана у члена группы. 6 - профиль группы изменён. 7 - профиль члена группы изменён. Например, член группы заглушён.10 - член аудио-видео группы заблокирован.11 - блокировка члена аудио-видео группы снята. |
| userIDList | Array.<String> | Список соответствующих userID. |
| newGroupProfile | Object | Если профиль группы изменён, это поле хранит новый профиль группы. |
| memberList | Array.<Object> | Если член группы заглушён, это поле хранит соответствующую информацию. Каждый элемент содержит следующие атрибуты:`userID` - String - userID члена группы`muteTime` - Number - длительность заглушения в секундах`nick` - String - прозвище члена группы`avatar` - String - аватар членов группы`duration` - Number - длительность блокировки в секундах`reason` - String - причина блокировки |

### GroupSystemNoticePayload

| Имя | Тип | Описание |
| --- | --- | --- |
| operatorID | String | ID пользователя, выполняющего операцию. |
| operationType | Number | Тип операции. Поддерживаемые значения следующие:1 - пользователь запрашивает присоединение к группе. 2 - запрос на присоединение к группе одобрен. 3 - запрос на присоединение к группе отклонен. 4 - пользователь удален из группы. 5 - группа удалена. 6 - группа создана. 7 - пользователь приглашён в группу. 8 - пользователь покинул группу. 9 - администратор изменён. 10 - администратор отменён. 11 - группа вновь восстановлена. 12 - пользователь получает приглашение на присоединение к группе и должен принять или отклонить приглашение.13 - пользователь приглашает других присоединиться к группе и приглашение принято. 14 - пользователь приглашает других присоединиться к группе и приглашение отклонено.15 - отчёт о прочтении синхронизирован на несколько терминалов.20 - тип напоминания о сообщении группы синхронизирован на несколько терминалов.21 - заблокировать члена аудио-видео группы.22 - разблокировать члена аудио-видео группы.23 - член группы приглашает других присоединиться к группе. 255 - пользовательское уведомление запущено. |
| groupProfile | Object | Профиль соответствующей группы. |
| userDefinedField | String | Пользовательское поле. |
| handleMessage | Object | Замечания по обработке. Например, если user1 вводит замечания на приложение для присоединения к group1, которая требует одобрения, администратор group1 увидит это поле в системном сообщении группы. |
| messageRemindType | String | `TencentCloudChat.TYPES.MSG_REMIND_ACPT_AND_NOTE` - SDK получает сообщение и генерирует событие MESSAGE_RECEIVED для уведомления стороны доступа, которая затем отправляет уведомление.`TencentCloudChat.TYPES.MSG_REMIND_ACPT_NOT_NOTE` - SDK получает сообщение и генерирует событие MESSAGE_RECEIVED для уведомления стороны доступа, которая затем не отправляет уведомление.`TencentCloudChat.TYPES.MSG_REMIND_DISCARD` - SDK отклоняет сообщение.`TencentCloudChat.TYPES.NOT_RECEIVE_OFFLINE_PUSH_EXCEPT_AT` получать сообщения в режиме онлайн и получать push-уведомления только для групповых сообщений @mentions в режиме офлайн |

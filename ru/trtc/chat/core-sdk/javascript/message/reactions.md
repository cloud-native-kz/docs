# Реакции

## Обзор функции

Функция ответа на сообщение — это интерактивные ответы на конкретное сообщение, типичным примером является ответ с помощью эмодзи. Ответы с эмодзи создаются с использованием эмодзи, что позволяет нам видеть количество отвечающих и список отвечающих для каждого эмодзи.

Текущие распространённые методы отображения ответов с эмодзи включают следующие два стиля:

| Стиль один | ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/38f82b4c58f811ee974d5254005f490f.png) |
| --- | --- |
| Стиль два | ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/3dee908d58f811ee974d5254005f490f.png) |

Пользователи могут осуществлять постраничный просмотр и получать профили всех пользователей, которые использовали определённый эмодзи.

Вы можете реализовать возможность ответа с эмодзи на основе API SDK:

- Вызовите интерфейс `addMessageReaction` для добавления эмодзи к сообщению. После успешного добавления эмодзи будет хранить информацию пользователя, выполнившего операцию.
- Вызовите интерфейс `removeMessageReaction` для удаления уже добавленного эмодзи. После успешного удаления эмодзи больше не будет хранить информацию пользователя, выполнившего операцию.
- Вызовите интерфейс `getMessageReactions` для пакетного получения списка эмодзи нескольких сообщений, где каждый эмодзи включает общее количество пользователей, использующих его в настоящее время, и профили первых N (по умолчанию 10) пользователей.
- Вызовите интерфейс `getAllUserListOfMessageReaction` для постраничного получения полного списка профилей пользователей, использующих эмодзи в сообщении.
- Слушайте событие `TencentCloudChat.EVENT.MESSAGE_REACTIONS_UPDATE` для обнаружения изменений в информации пользователей, использующих эмодзи. Это событие будет содержать последнюю информацию пользователей эмодзи (включая общее количество пользователей и профили топ N пользователей).

> **Примечание:** Поддерживается начиная с v3.2.0, эта функция доступна только для продвинутых клиентов. После [приобретения издания Pro, Pro Plus или Enterprise](https://trtc.io/pricing/chat) она может быть использована.

### Добавление ответа на сообщение

Вызывая интерфейс `addMessageReaction`, вы можете добавить ответ на сообщение.

##### **API**

```
chat.addMessageReaction(message, reactionID);
```

**Параметры**

| Атрибут | Значение | Описание |
| --- | --- | --- |
| message | Объект сообщения | Сообщение отправлено успешно. |
| reactionID | ID ответа на сообщение | В сценариях ответов с эмодзи reactionID — это ID эмодзи. |

> **Примечание:** Одно сообщение поддерживает до 10 реакций, а одна реакция поддерживает до 100 пользователей. Если количество реакций для одного сообщения превышает максимальный лимит (10), вызов API вернёт код ошибки 23005. Если количество пользователей для одной реакции превышает максимальный лимит (100), вызов API вернёт код ошибки 23006. Если реакция уже включает текущего пользователя, вызов API вернёт код ошибки 23007.

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.addMessageReaction(message, 'smile');promise.then(function(imResponse) {  // Ответ на сообщение успешно добавлен}).catch(function(imError) {  // Ошибка при добавлении ответа на сообщение  console.warn('addMessageReaction error:', imError);});
```

### Удаление ответа на сообщение

Вызовите API `removeMessageReaction` для удаления ответа на сообщение.

##### **API**

```
chat.removeMessageReaction(message, reactionID);
```

**Параметры**

| Атрибут | Значение | Описание |
| --- | --- | --- |
| message | Объект сообщения | Сообщение отправлено успешно. |
| reactionID | ID ответа на сообщение | В сценариях ответов с эмодзи reactionID — это ID эмодзи. |

> **Примечание:** Если реакция не существует, вызов API вернёт код ошибки 23008. Если реакция не включает текущего пользователя, вызов API вернёт код ошибки 23009.

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.removeMessageReaction(message, 'smile');promise.then(function(imResponse) {  // Ответ на сообщение успешно удалён}).catch(function(imError) {  // Ошибка при удалении ответа на сообщение  console.warn('removeMessageReaction error:', imError);});
```

### Пакетное получение информации о нескольких ответах на сообщения

Вызовите интерфейс `getMessageReactions` для пакетного получения информации о нескольких ответах на сообщения.

##### **API**

```
chat.getMessageReactions(options);
```

**Параметры**

| Атрибут | Значение | Описание |
| --- | --- | --- |
| messageList | Список сообщений | Сообщения должны принадлежать одной сессии и должны находиться в состоянии успешно отправленного сообщения. |
| maxUserCountPerReaction | Максимальное количество профилей пользователей, возвращаемых для каждой реакции | Диапазон значений [0,10], каждая реакция возвращает профили первых 10 пользователей только. Для получения дополнительных профилей пользователей вы можете вызвать интерфейс `getAllUserListOfMessageReaction` при необходимости для постраничного получения. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.getMessageReactions({  messageList: [message1, message2],  maxUserCountPerReaction: 10,});promise.then(function(imResponse) {  // Пакетное получение успешно   const resultList = imResponse.data;   resultList.forEach((item) => {     const { messageID, reactionList } = item;     // Информация, возвращаемая reactionList, выглядит следующим образом:     // reactionID - ID ответа на сообщение     // totalUserCount - Общее количество пользователей, ответивших на сообщение с одинаковым reactionID     // partialUserList - Частичный список пользователей с одинаковым reactionID, содержащий информацию userID, nick, avatar пользователя   });}).catch(function(imError) {  // Ошибка пакетного получения  console.warn('getMessageReactions error:', imError);});
```

### Постраничное получение полного списка пользователей, ответивших на сообщение

Вызовите интерфейс `getAllUserListOfMessageReaction` для постраничного получения полного списка пользователей, ответивших на сообщение.

##### **API**

```
chat.getAllUserListOfMessageReaction(options);
```

**Параметры**

| Атрибут | Значение | Описание |
| --- | --- | --- |
| message | Объект сообщения | Сообщение отправлено успешно. |
| reactionID | ID ответа на сообщение | В сценариях ответов с эмодзи reactionID — это ID эмодзи. |
| nextSeq | Курсор для постраничного получения | При первой передаче передайте 0, затем при постраничном просмотре передайте nextSeq, возвращённый функцией успеха. |
| count | Максимальное количество элементов, которые можно получить на одной странице | Поддерживает до 100 |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.getAllUserListOfMessageReaction({  message: message,  reactionID: 'smile',  nextSeq: 0,  count: 100,});promise.then(function(imResponse) {  // Успешно получено  const { userList, nextSeq, isCompleted } = imResponse.data;  // userList - Список информации пользователя, включающий информацию userID, nick, avatar пользователя  // nextSeq - Этот параметр должен быть передан при следующем получении при постраничном просмотре  // isCompleted - Указывает, была ли получена вся информация пользователя. Когда isCompleted равен true, nextSeq равен 0}).catch(function(imError) {  // Ошибка при получении  console.warn('getAllUserListOfMessageReaction error:', imError);});
```

### Обновление информации об ответе на сообщение

При изменении информации об ответе на сообщение SDK отправляет событие `TencentCloudChat.EVENT.MESSAGE_REACTIONS_UPDATED`. Вы можете получить обновленную информацию об ответе на сообщение в зарегистрированном обратном вызове.

> **Примечание:** Когда значение поля totalUserCount в изменённой информации реакции равно 0, это указывает, что никакие пользователи больше не используют эту реакцию. Вы можете удалить отображение этой реакции из интерфейса.

##### **Примеры**

```
let onMessageReactionsUpdated = function(event) {  const { messageID, reactionList } = event.data;  // messageID - ID сообщения  // reactionList - Список обновлений ответов на сообщение  reactionList.forEach((item) => {   const { reactionID, totalUserCount, partialUserList } = item;   // reactionID - ID ответа на сообщение   // totalUserCount - Общее количество пользователей, ответивших на сообщение с одинаковым reactionID   // partialUserList - Частичный список пользователей с одинаковым reactionID  });};chat.on(TencentCloudChat.EVENT.MESSAGE_REACTIONS_UPDATED, onMessageReactionsUpdated);
```

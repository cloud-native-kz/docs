# Отправка сообщения только для онлайн-пользователей

## Обзор функции

В некоторых случаях может потребоваться, чтобы сообщение было получено получателем только при его нахождении онлайн; то есть получатель не заметит сообщение при отсутствии в сети. Вам просто нужно установить `onlineUserOnly` в `true` при вызове `sendMessage`. Сообщение, отправленное таким образом, отличается от обычного тем, что:

1. Оно не может быть сохранено в оффлайн; то есть оно не может быть получено, если получатель находится в оффлайн.
2. Оно не может синхронизироваться между устройствами; то есть если оно получено на одном устройстве, оно не может быть получено на другом, прочитано оно или нет.
3. Оно не может быть сохранено локально; то есть оно не может быть извлечено из локальных или облачных исторических сообщений.

## Индикаторы печати

В личных чатах вы можете вызвать API `sendMessage` для отправки уведомления "Печать...". После получения такого сообщения получатель может отобразить "Печать..." в интерфейсе.

##### **Примеры**

```
chat.sendMessage(message, {  // Если получатель находится в оффлайн, сообщение не будет  // сохранено на сервере синхронизации и не будет доставлено в оффлайн.  onlineUserOnly: true});
```

Фактически, для реализации полного уведомления "Печать..." отправка сообщений только для онлайн-пользователей с помощью Chat SDK — это лишь небольшая часть. Требуется дополнительная работа разработки, например:

- Мониторинг пользовательского интерфейса и обновление статуса: В поле ввода сообщений добавляется прослушиватель событий для определения момента, когда пользователь начинает печать. При начале печати клиент обновляет статус пользователя, например, "печать".
- Отправка статуса на сервер: В нужный момент статус ввода пользователя отправляется на сервер.
- Обновление пользовательского интерфейса: На основе полученного статуса ввода на интерфейсе получателя отображается подсказка, например "Печать". Это может быть текстовая подсказка, значок или анимация.
- Учет дросселирования: Для уменьшения ненужных обновлений статуса и нагрузки на связь можно установить временной интервал. Только одно обновление статуса ввода отправляется в течение этого интервала.

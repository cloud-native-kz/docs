# Статус пользователя

## Обзор функции

JavaScript SDK предоставляет функцию управления статусом пользователя, где каждый пользователь имеет два разных типа статуса:

- Обычный статус. Встроенный в SDK статус, который не может быть непосредственно изменен клиентами.
- Пользовательский статус. Определяемый клиентом статус, который можно изменять. Используя пользовательский статус, вы можете установить информацию, например «Слушаю музыку», «Во время звонка» и т. д.

> **Примечание:** Статус пользователя зависит от текущего пользователя и не связан с устройством. Если несколько устройств одновременно авторизованы в одном аккаунте, запрос или установка статуса по устройству не поддерживаются.

Существует три типа обычного статуса пользователя:

- В сети (`TencentCloudChat.TYPES.USER_STATUS_ONLINE`): текущий пользователь авторизован и в сети, может нормально отправлять и получать сообщения.
- Не в сети (`TencentCloudChat.TYPES.USER_STATUS_OFFLINE`): вход/выход из веб-приложения не вызывает статус «не в сети». В приложениях, интегрирующих Native ChatSDK, будет вызван статус «не в сети».
- Не авторизован (`TencentCloudChat.TYPES.USER_STATUS_UNLOGINED`): пользователь никогда не авторизовался после регистрации аккаунта или пользователь активно вызвал `logout` для выхода.

## Установка пользовательского статуса

Вы можете вызвать интерфейс `setSelfStatus` для установки поля `customStatus` для вашего пользовательского статуса. После успешной установки вы получите уведомление об изменении статуса через событие `TencentCloudChat.EVENT.USER_STATUS_UPDATED`.

> **Примечание:** Вызов `setSelfStatus` не требует обновления до версии Pro, Pro Plus или Enterprise, а также не требует включения переключателя консоли. Этот интерфейс не выполняет контроль частоты запросов. Вы можете активно очистить ваш статус, установив поле `customStatus` на пустую строку при вызове интерфейса `setSelfStatus`. После выхода из системы на некоторое время (веб), серверная часть чата автоматически очистит пользовательский статус. Это также вызовет уведомление об изменении статуса.

##### **API**

```
chat.setSelfStatus(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Название | Тип | Описание |
| --- | --- | --- |
| customStatus | String | Пользовательский статус пользователя |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Установка customStatus на пустую строку '', это очистит ваш пользовательский статусlet promise = chat.setSelfStatus({customStatus: 'xxx'});promise.then(function(imResponse) {  console.log(imResponse.data);  const { userID, statusType, customStatus } = imResponse.data;  // userID - ID пользователя  // statusType - Статус пользователя. Перечисленные значения и описания следующие:  // TencentCloudChat.TYPES.USER_STATUS_UNKNOWN - Неизвестно  // TencentCloudChat.TYPES.USER_STATUS_ONLINE - В сети  // TencentCloudChat.TYPES.USER_STATUS_OFFLINE - Не в сети  // TencentCloudChat.TYPES.USER_STATUS_UNLOGINED - Не авторизован  // customStatus - Пользовательский статус пользователя}).catch(function(imError) {  console.warn('setSelfStatus error:', imError); // Ошибка установки пользовательского статуса});
```

## Запрос статуса пользователя

API вернет как обычный статус, так и пользовательский статус запрашиваемых пользователей.

> **Примечание:** Запрос собственного статуса не требует обновления до версии Pro, Pro Plus или Enterprise и не требует включения переключателя консоли. Запрос собственного статуса не подлежит ограничению частоты API. Запрос статуса других пользователей требует обновления до версии Pro, Pro Plus или Enterprise с ограничением по умолчанию 20 запросов за 5 секунд, и один запрос может включать не более 500 пользователей. Запрос статуса других пользователей требует предварительного включения опции «Установить запрос статуса пользователя и уведомление об изменении статуса» в [консоли чата](https://console.trtc.io/chat/login-message). Если переключатель отключен, вызов `getUserStatus` приведет к ошибке.

##### **API**

```
chat.getUserStatus(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Название | Тип | Описание |
| --- | --- | --- |
| userIDList | Array | Список пользовательских ID для запроса. При запросе собственного статуса нужно передать только собственный userID. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Проверка собственного статуса пользователя// userIDList содержит только собственный userID для запроса собственного статусаlet promise = chat.getUserStatus({userIDList: [`${myUserID}`]});promise.then(function(imResponse) {   const { successUserList } = imResponse.data;   successUserList.forEach((item) => {     const { userID, statusType, customStatus } = item;     // userID - ID пользователя     // statusType - Статус пользователя, перечисленные значения и описания следующие:     // TencentCloudChat.TYPES.USER_STATUS_UNKNOWN - Неизвестно     // TencentCloudChat.TYPES.USER_STATUS_ONLINE - В сети     // TencentCloudChat.TYPES.USER_STATUS_OFFLINE - Не в сети     // TencentCloudChat.TYPES.USER_STATUS_UNLOGINED - Не авторизован     // customStatus - Пользовательский статус пользователя   });}).catch(function(imError) {  console.warn('getUserStatus error:', imError); // Ошибка получения статуса пользователя});
```

### Проверка статуса других пользователей

Установите `userIDList` в список userID других пользователей для проверки их статуса.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/15fb0819138011ef83b95254002977b6.png)

##### **Примеры**

```
// Запрос статуса других пользователейlet promise = chat.getUserStatus({userIDList: ['user0', 'user1']});promise.then(function(imResponse) {   const { successUserList, failureUserList } = imResponse.data;   // Список пользователей с успешными запросами   successUserList.forEach((item) => {     const { userID, statusType, customStatus } = item;     // userID - ID пользователя     // statusType - Статус пользователя, перечисленные значения и описания следующие:     // TencentCloudChat.TYPES.USER_STATUS_UNKNOWN - Неизвестно     // TencentCloudChat.TYPES.USER_STATUS_ONLINE - В сети     // TencentCloudChat.TYPES.USER_STATUS_OFFLINE - Не в сети     // TencentCloudChat.TYPES.USER_STATUS_UNLOGINED - Не авторизован     // customStatus - Пользовательский статус пользователя   });   // Список пользователей с неудачными запросами   failureUserList.forEach((item) => {     const { userID, code, message } = item;     // userID - ID пользователя с неудачным запросом     // code - Код ошибки неудачного запроса     // message - Сообщение об ошибке неудачного запроса   });}).catch(function(imError) {  console.warn('getUserStatus error:', imError); // Ошибка получения статуса пользователя});
```

### Подписка на статус пользователя

После успешной подписки при изменении статуса подписанного пользователя (включая обычный статус и пользовательский статус) вы получите уведомление об изменении статуса пользователя через событие `TencentCloudChat.EVENT.USER_STATUS_UPDATED`.

> **Примечание:** Этот API требует обновления до версии Pro, Pro Plus или Enterprise. API ограничен 20 запросами за 5-секундный интервал, и максимальное количество пользователей за одну подписку не может превышать 100. Существует ограничение на количество подписок в списке. После превышения ограничения самые ранние подписанные пользователи будут автоматически удалены.

##### **API**

```
chat.subscribeUserStatus(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Название | Тип | Описание |
| --- | --- | --- |
| userIDList | Array | Список пользовательских ID, максимум 100 за один запрос. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.subscribeUserStatus({userIDList: ['user0', 'user1']});promise.then(function(imResponse) {  const { failureUserList } = imResponse.data;   // Список пользователей с неудачными подписками   failureUserList.forEach((item) => {     const { userID, code, message } = item;     // userID - ID пользователя с неудачным запросом     // code - Код ошибки неудачного запроса     // message - Сообщение об ошибке неудачного запроса   });}).catch(function(imError) {  // Информация об ошибке при неудачной подписке на статус пользователя  console.warn('subscribeUserStatus error:', imError);});
```

## Отписка от статуса пользователя

После успешной отписки, когда статус пользователя изменяется (включая обычный статус и пользовательский статус), SDK не будет отправлять событие `TencentCloudChat.EVENT.USER_STATUS_UPDATED`.

> **Примечание:** API ограничен 20 запросами за 5-секундный интервал, и максимальное количество пользователей за одну отписку не может превышать 100.

##### **API**

```
chat.unsubscribeUserStatus(options);
```

##### **Параметры**

| Название | Тип | Описание |
| --- | --- | --- |
| userIDList | Array \| undefined | Список ID пользователей, максимум 100 за один запрос. Когда userIDList является пустым массивом или `undefined`, это отменяет все текущие подписки. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Отписка от текущих отдельных пользователейlet promise = chat.unsubscribeUserStatus({userIDList: ['user0', 'user1']});promise.then(function(imResponse) {  const { failureUserList } = imResponse.data;   // Список пользователей с неудачными отписками   failureUserList.forEach((item) => {     const { userID, code, message } = item;     // userID - ID пользователя с неудачным запросом     // code - Код ошибки неудачного запроса     // message - Сообщение об ошибке неудачного запроса   });}).catch(function(imError) {  console.warn('unsubscribeUserStatus error:', imError); // Информация об ошибке отписки});
```

```
// Отписка от всех текущих подписокlet promise = chat.unsubscribeUserStatus();promise.then(function(imResponse) {  const { failureUserList } = imResponse.data;   // Список пользователей с неудачными отписками   failureUserList.forEach((item) => {     const { userID, code, message } = item;     // userID - ID пользователя с неудачным запросом     // code - Код ошибки неудачного запроса     // message - Сообщение об ошибке неудачного запроса   });}).catch(function(imError) {  console.warn('unsubscribeUserStatus error:', imError); // Информация об ошибке отписки});
```

### Уведомление об изменении собственного статуса

Если вы предварительно зарегистрировали слушатель события `TencentCloudChat.EVENT.USER_STATUS_UPDATED`, при изменении вашего собственного статуса SDK отправит событие `TencentCloudChat.EVENT.USER_STATUS_UPDATED`, и вы сможете получить ваш последний статус там.

### Уведомление об изменении статуса друга

1. Если вы включили автоматическое уведомление о статусе друга в [консоли чата](https://console.trtc.io/), при изменении статуса друга SDK отправит событие `TencentCloudChat.EVENT.USER_STATUS_UPDATED`, и вы сможете получить последний статус вашего друга там.
2. Если вы не включили автоматические уведомления о статусе друга и все еще хотите узнавать об изменениях статуса друга, вам нужно вызвать `subscribeUserStatus` для активной подписки на статус друга. При изменении статуса друга SDK отправит обратный вызов `TencentCloudChat.EVENT.USER_STATUS_UPDATED`.
3. Если вы ни включили автоматическое уведомление о статусе друга, ни вызвали `subscribeUserStatus` для активной подписки на статус друга, то при изменении статуса друга вы не сможете его заметить.

### Изменение статуса обычных пользователей (не друзей)

Если вы хотите узнавать об изменениях статуса обычных пользователей (не друзей), вы можете только вызвать `subscribeUserStatus` для активной подписки. При изменении статуса этого пользователя будет вызван обратный вызов `TencentCloudChat.EVENT.USER_STATUS_UPDATED`, и вы сможете получить его последний статус там.

##### **Примеры**

```
/** * Сценарии получения уведомлений: * 1. Когда статус подписанного пользователя изменяется (включая статус в сети и определяемый пользователем статус), это событие будет вызвано * 2. Если переключатель уведомления о статусе друга включен в консоли, даже если вы не активно подписались, это событие будет вызвано при изменении статуса друга * 3. Когда один и тот же аккаунт авторизован на нескольких устройствах, если одно устройство изменит пользовательский статус, все устройства получат это событие */let onUserStatusUpdated = function(event) {   console.log(event.data);   const userStatusList = event.data;   userStatusList.forEach((item) => {     const { userID, statusType, customStatus } = item;     // userID - ID пользователя     // statusType - Статус пользователя, перечисленные значения и описания следующие:     // TencentCloudChat.TYPES.USER_STATUS_UNKNOWN - Неизвестно     // TencentCloudChat.TYPES.USER_STATUS_ONLINE - В сети     // TencentCloudChat.TYPES.USER_STATUS_OFFLINE - Не в сети     // TencentCloudChat.TYPES.USER_STATUS_UNLOGINED - Не авторизован     // customStatus - Пользовательский статус пользователя   })};chat.on(TencentCloudChat.EVENT.USER_STATUS_UPDATED, onUserStatusUpdated);
```

### Уведомление об изменении статуса на нескольких платформах, синхронизация между несколькими экземплярами

Если вы включили вход с нескольких клиентов или вход с нескольких экземпляров на одной платформе (подробнее см. [Вход с нескольких устройств или несколько экземпляров входа на одной платформе](https://trtc.io/document/33515?platform=javascript&product=chat&menulabel=sdk)), один аккаунт может быть авторизован на разных устройствах. Когда пользовательский статус пользователя, авторизованного на одном устройстве, изменяется, фоновый сервис чата отправит уведомление об изменении статуса на другие авторизованные устройства.

## Часто задаваемые вопросы

### При вызове API подписки/отписки интерфейс выдает код ошибки «72001»

Код ошибки 72001 указывает на то, что соответствующая функция не включена в консоли. Пожалуйста, авторизуйтесь в [консоли чата](https://console.trtc.io/) и включите соответствующий переключатель функции.

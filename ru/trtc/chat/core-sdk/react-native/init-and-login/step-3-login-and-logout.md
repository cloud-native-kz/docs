# Шаг 3: Вход и выход

## Описание функции

Пользователи могут отправлять и получать сообщения только после входа в Chat SDK. Для входа на сервер Tencent требуется предоставить информацию, включая `UserID` и `UserSig`. Дополнительные сведения об этих параметрах см. в разделе [Аутентификация при входе](https://www.tencentcloud.com/document/product/1047/33517).

## Вход

> **Примечание:** После успешного входа необходимо дождаться, пока SDK перейдет в состояние готовности (путем прослушивания события `TencentCloudChat.EVENT.SDK_READY`), прежде чем вызывать интерфейсы, требующие аутентификации, такие как `sendMessage`. По умолчанию вход с нескольких экземпляров не поддерживается, то есть если эта учетная запись уже вошла на другую страницу, продолжение входа на текущей странице может отключить другие страницы. Когда пользователь отключен, инициируется событие `TencentCloudChat.EVENT.KICKED_OUT`, и пользователь может обработать его после прослушивания события. Для поддержки входа с нескольких экземпляров (позволяет одновременно входить в одну учетную запись на нескольких веб-страницах) войдите в [консоль Chat](https://console.trtc.io/), найдите целевой SDKAppID и нажмите **Application > Configuration > Login and Message > Login Settings > Max Login Instances per User per Platform**, чтобы настроить количество экземпляров. Конфигурация вступит в силу через 5 минут. Для платформ мини-программ, мини-игр, uni-app и **React Native** (даже если упакованы в собственное приложение) входы считаются веб-экземплярами. Если этот интерфейс возвращает код ошибки 60020, это означает, что вы не приобрели пакет, пакет истек или приобретенный пакет все еще находится в конфигурации и еще не вступил в силу. Пожалуйста, войдите на страницу покупки Chat, чтобы переобрести пакет. После покупки он вступит в силу в течение 5 минут. Если этот интерфейс возвращает код ошибки 2801 на платформе мини-программы, проверьте конфигурацию доверенного домена мини-программы.

##### API

```
chat.login(options);
```

##### Параметры

Параметр `options` имеет тип `Object`. Он содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| UserID | String | ID пользователя. |
| UserSig | String | Пароль, с помощью которого пользователь входит в консоль Chat. По сути, это зашифрованный текст, созданный путем шифрования информации, такой как UserID. Подробный метод создания см. в разделе [Создание UserSig](https://www.tencentcloud.com/document/product/1047/34385). |

##### Возвращаемое значение

`Promise`

##### Примеры

```
let promise = chat.login({userID: 'your userID', userSig: 'your userSig'});promise.then(function(imResponse) {  console.log(imResponse.data);  // Logged in successfully  if (imResponse.data.repeatLogin === true) {  // This indicates that the account is already logged in.    console.log(imResponse.data.errorInfo);  }}).catch(function(imError) {  console.warn('login error:', imError);});
```

## Готовность SDK

После подготовки SDK разработчики могут вызывать API SDK для отправки сообщений и использования различных функций SDK.

Разработчики также могут прослушивать:

- `SDK_READY`, которое инициируется, когда SDK входит в состояние готовности.
- `SDK_NOT_READY`, которое инициируется, когда SDK входит в состояние неготовности.

##### API

```
chat.isReady();
```

##### Параметры

Нет

##### Возвращаемое значение

`Boolean`

##### Примеры

```
let isReady = chat.isReady();
```

## Выход

Этот API используется для выхода из Chat SDK. Он обычно вызывается при переключении между учетными записями. Этот API очищает статус входа текущей учетной записи и все данные в памяти.

> **Примечание:** При вызове этого API экземпляр публикует событие `SDK_NOT_READY`. В этом случае экземпляр автоматически выходит и не может получать или отправлять сообщения. Предположим, что значение элемента конфигурации Web в [консоли Chat](https://console.trtc.io/) больше 1, и одна и та же учетная запись использовалась для входа на экземпляры a1 и a2. После выполнения a1.logout() a1 автоматически выйдет и не сможет получать или отправлять сообщения, в то время как a2 не будет затронут. Предположим, что для Online Web Instances установлено значение 2, и ваша учетная запись использовалась для входа на экземпляры a1 и a2. Когда вы используете эту учетную запись для входа на экземпляр a3, a1 или a2 будет принудительно выведена. В большинстве случаев экземпляр, который первым вошел в состояние входа, принудительно выводится. Это называется принудительным выходом из-за входа с несколько учетных записей. Если a1 принудительно выведена, процесс выхода выполняется в пределах a1 и инициируется событие `KICKED_OUT`. Вы можете прослушивать это событие и перенаправить на страницу входа при инициировании события. В этом случае a1 принудительно выведена, тогда как a2 и a3 могут продолжать работать нормально.

##### API

```
chat.logout();
```

##### Параметры

Нет

##### Возвращаемое значение

`Promise`

##### Примеры

```
let promise = chat.logout();promise.then(function(imResponse) {  console.log(imResponse.data);// Logged out successfully}).catch(function(imError) {  console.warn('logout error:', imError);});
```

## Завершение

Вы можете завершить экземпляры SDK, включая выход, отключение постоянного соединения и освобождение всех ресурсов.

##### API

```
chat.destroy();
```

##### Параметры

Нет

##### Возвращаемое значение

`Promise`

##### Примеры

```
let promise = chat.destroy();
```

## Параметры входа

По умолчанию вход с нескольких учетных записей не поддерживается. Если вы используете учетную запись, которая уже вошла на другую страницу, для входа на текущую страницу, учетная запись может быть принудительно выведена на другой странице, что инициирует событие `KICKED_OUT`. Вы можете действовать соответствующим образом после обнаружения события через прослушивание. Ниже приведен пример:

```
let onKickedOut = function (event) {  console.log(event.data.type);  // TencentCloudChat.TYPES.KICKED_OUT_MULT_ACCOUNT (The user is forcibly logged out because the same account logs in from multiple webpages on the web client.)  // TencentCloudChat.TYPES.KICKED_OUT_MULT_DEVICE (The user is forcibly logged out because the same account logs in from multiple terminals.)  // TencentCloudChat.TYPES.KICKED_OUT_USERSIG_EXPIRED (The signature expired.)  // TencentCloudChat.TYPES.KICKED_OUT_REST_API (The user is forcibly logged out by the RESTful API.)};chat.on(TencentCloudChat.EVENT.KICKED_OUT, onKickedOut);
```

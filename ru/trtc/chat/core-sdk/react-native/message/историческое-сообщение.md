# Историческое сообщение

## Обзор функции

SDK поддерживает получение сообщений по страницам, последовательности или диапазону времени. Исторические сообщения, хранящиеся в облаке, подлежат следующим ограничениям по времени:

- Версия Developer: Бесплатный период хранения составляет 7 дней и не может быть продлен.
- Версия Standard: Бесплатный период хранения составляет 7 дней и не может быть продлен.
- Версия Pro: Бесплатный период хранения составляет 30 дней и не может быть продлен.
- Версия Pro Plus: Бесплатный период хранения составляет 60 дней и не может быть продлен.
- Версия Enterprise: Бесплатный период хранения составляет 90 дней и не может быть продлен.

> **Примечание** Продление периода хранения исторических сообщений — это дополнительная услуга. Вы можете войти в [Chat Console](https://console.trtc.io/) для изменения соответствующей конфигурации. Сведения о выставлении счетов см. в разделе [Pricing](https://trtc.io/pricing/chat). Мультимедийные сообщения (такие как изображения, файлы и аудио) имеют те же периоды хранения, что и исторические сообщения.

## Получение списка сообщений

### getMessageList

Этот API используется для получения списка сообщений в указанном диалоге по страницам. Его необходимо вызвать при первой отрисовке списка сообщений после входа пользователя в диалог или когда пользователь прокручивает список вниз для просмотра дополнительных сообщений.

> **Примечание** Этот интерфейс может использоваться для "получения исторических сообщений" (исключая AVChatRoom). AVChatRoom не хранит исторические сообщения по умолчанию. Когда новые пользователи входят в AVChatRoom, они могут видеть только сообщения, отправленные после их присоединения. Для повышения вовлеченности и участия пользователей в AVChatRoom вы можете обновить количество недавних сообщений, которые новые члены могут просматривать в Chat Console > Application Configuration > Feature Configuration > Group Configuration -> Live Group New Member View Pre-join Message Quantity Configuration. Эта функция доступна только в версиях Pro edition, Pro Plus edition или Enterprise edition. Новые члены группы могут просматривать последние сообщения, созданные в течение 24 часов до их присоединения, максимум 20 сообщений. Если вы хотите найти локальные сообщения, используйте `findMessage`. Если вы хотите найти все облачные сообщения, используйте `searchCloudMessages` (эта функция — дополнительная услуга, и вам необходимо приобрести плагин облачного поиска. Пожалуйста, нажмите для [покупки](https://console.trtc.io/chat/plugin/TUICloudSearch)).

##### **API**

```
chat.getMessageList(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Он содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| conversationID | String | ID диалога, который состоит из: `C2C${userID}` (для личных чатов) `GROUP${groupID}` (для групповых чатов) `GROUP${topicID}` (тема) `@TIM#SYSTEM` (диалог системного уведомления) |
| nextReqMessageID | String \| undefined | ID сообщения, используемый для последующего получения по страницам. Оставьте это поле пустым для первого получения и введите значение, возвращаемое API `getMessageList` для последующего получения. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Получение списка сообщений в первый раз при открытии диалогаlet promise = chat.getMessageList({conversationID: 'C2Ctest'});promise.then(function(imResponse) {  const messageList = imResponse.data.messageList; // Список сообщений  // Этот параметр должен быть передан для следующего получения по страницам.  const nextReqMessageID = imResponse.data.nextReqMessageID;  // Это указывает, были ли получены все сообщения.  const isCompleted = imResponse.data.isCompleted;});
```

```
// Прокрутка списка сообщений вниз для просмотра дополнительных сообщенийlet promise = chat.getMessageList({conversationID: 'C2Ctest', nextReqMessageID});promise.then(function(imResponse) {  const messageList = imResponse.data.messageList; // Список сообщений  // Этот параметр должен быть передан для следующего получения по страницам.  const nextReqMessageID = imResponse.data.nextReqMessageID;  // Это указывает, были ли получены все сообщения.  const isCompleted = imResponse.data.isCompleted; });
```

### getMessageListHopping

Этот API используется для получения списка сообщений по указанной последовательности или диапазону времени.

##### **API**

```
chat.getMessageListHopping(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Он содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| conversationID | String | ID диалога, который состоит из: `C2C${userID}` (для личных чатов) `GROUP${groupID}` (для групповых чатов) `GROUP${topicID}` (тема) |
| sequence | Number \| undefined | `sequence` сообщения, начиная с которого получать роуминговые групповые сообщения |
| time | Number \| undefined | Серверное время сообщения, начиная с которого получать роуминговые личные сообщения |
| direction | Number | Направление получения сообщений, по умолчанию `0`. 0: получить в обратном хронологическом порядке для получения старых сообщений; 1: получить в хронологическом порядке для получения недавних сообщений |
| count | Number \| undefined | Количество получаемых сообщений. По умолчанию равно 15, что также является максимальным значением, указывая, что одновременно можно получить и вернуть до 15 сообщений. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Получение роуминговых групповых сообщений по последовательности.// `direction`: 0: получить в обратном хронологическом порядке для получения старых сообщений// 1: получить в хронологическом порядке для получения недавних сообщенийlet promise = chat.getMessageListHopping({  conversationID: 'GROUPtest',  sequence: 100,  count: 15,  direction: 0});promise.then(function(imResponse) {  const messageList = imResponse.data.messageList; // Список сообщений});
```

****

```
// Получение роуминговых личных сообщений по диапазону времени.// `direction`: 0: получить в обратном хронологическом порядке для получения старых сообщений// 1: получить в хронологическом порядке для получения недавних сообщенийlet promise = chat.getMessageListHopping({  conversationID: 'C2Ctest',  time: xxx,  count: 15,  direction: 0});promise.then(function(imResponse) {  const messageList = imResponse.data.messageList; // Список сообщений});
```

# Обзор сообщений

## Класс Message

В Chat SDK `Message` представляет объект сообщения, используемый для описания атрибутов сообщения, таких как тип, содержание и ID беседы.

| Атрибут | Тип | Значение по умолчанию | Описание |
| --- | --- | --- | --- |
| ID | String | - | ID сообщения. Правило конкатенации ID сообщения: `${senderTinyID}-${clientTime}-${random}`, что совпадает с правилом конкатенации ID сообщения в нативном Chat. |
| type | String | - | Тип сообщения. Допустимые значения:`TencentCloudChat.TYPES.MSG_TEXT` - текстовое сообщение`TencentCloudChat.TYPES.MSG_IMAGE` - сообщение с изображением`TencentCloudChat.TYPES.MSG_AUDIO` - аудиосообщение`TencentCloudChat.TYPES.MSG_VIDEO` - видеосообщение`TencentCloudChat.TYPES.MSG_FILE` - файловое сообщение`TencentCloudChat.TYPES.MSG_CUSTOM` - пользовательское сообщение`TencentCloudChat.TYPES.MSG_MERGER` - объединённое сообщение`TencentCloudChat.TYPES.MSG_LOCATION` - сообщение о местоположении`TencentCloudChat.TYPES.MSG_GRP_TIP` - подсказка группы`TencentCloudChat.TYPES.MSG_GRP_SYS_NOTICE` - системное уведомление группы |
| payload | Object | - | Содержание сообщения. Допустимые значения:`TextPayload``ImagePayload``AudioPayload``VideoPayload``FilePayload``CustomPayload``MergerPayload``LocationPayload``FacePayload``GroupTipPayload``GroupSystemNoticePayload` |
| conversationID | String | - | ID беседы сообщения. |
| conversationType | String | - | Тип беседы сообщения. Допустимые значения:`TencentCloudChat.TYPES.CONV_C2C` --- беседа C2C (один-на-один)`TencentCloudChat.TYPES.CONV_GROUP` --- групповая беседа`TencentCloudChat.TYPES.CONV_SYSTEM` --- системная беседа |
| to | String | - | `userID` получателя. |
| from | String | - | `userID` отправителя. При отправке сообщения по умолчанию устанавливается ID текущего авторизованного пользователя. |
| flow | String | - | Поток сообщения.`in` --- полученное сообщение`out` --- отправленное сообщение |
| time | Number | - | Временная метка сообщения в секундах. |
| status | String | - | Статус сообщения.`unSend` --- не отправлено`success` --- успешно отправлено`fail` --- ошибка при отправке |
| isRevoked | Boolean | false | Является ли сообщение отозванным. `true` указывает на да. |
| priority | String | `TencentCloudChat.TYPES.MSG_PRIORITY_NORMAL` | Приоритет сообщения. Если сообщения в группе превышают ограничение по частоте, сервер будет сначала доставлять сообщения с высоким приоритетом. Поддерживаемые значения:`TencentCloudChat.TYPES.MSG_PRIORITY_HIGH``TencentCloudChat.TYPES.MSG_PRIORITY_NORMAL (по умолчанию)``TencentCloudChat.TYPES.MSG_PRIORITY_LOW``TencentCloudChat.TYPES.MSG_PRIORITY_LOWEST` |
| nick | String | - | Имя отправителя сообщения. Этот атрибут должен быть задан предварительно путём вызова API `updateMyProfile`. |
| avatar | String | - | URL фото профиля отправителя сообщения. Этот атрибут должен быть задан предварительно путём вызова API `updateMyProfile`. |
| nameCard | String | - | Групповая визитная карточка отправителя, отправляющего не-аудио-видео групповое сообщение. Этот атрибут также называется прозвищем отправителя сообщения в группе и должен быть задан предварительно путём вызова API `setGroupMemberNameCard`. |
| atUserList | Array | - | Это поле хранит значения `userID` членов группы, упомянутых в групповом сообщении. |
| cloudCustomData | String | '' | Пользовательские данные сообщения, которые сохраняются в облаке, отправляются получателю и могут быть получены после удаления и переустановки приложения. |
| isDeleted | Boolean | false | Является ли сообщение удалённым. `true` указывает на да. |
| isModified | Boolean | false | Является ли сообщение изменённым. `true` указывает на да. |
| needReadReceipt | Boolean | false | Требуется ли подтверждение прочтения. `true` указывает на да. Для использования требуется приобретение пакета Premium edition. |
| readReceiptInfo | Object | - | Информация о подтверждении прочтения сообщения`readCount` - количество прочитанных сообщений, может быть запрошено путём вызова API `getMessageReadReceiptList`; для запроса членов группы, которые прочитали сообщение, вызовите `getGroupMessageReadMemberList`.`unreadCount` - количество непрочитанных сообщений, может быть запрошено путём вызова API `getMessageReadReceiptList`.`isPeerRead` - отправил ли собеседник сообщения C2C подтверждение прочтения, это свойство будет обновлено при получении уведомления о подтверждении прочтения или загрузки роумингового хранилища после успешного вызова `sendMessageReadReceipt` получателем.`timestamp` - время отправки подтверждения прочтения собеседником сообщения C2C в секундах. Этот атрибут будет обновлён при получении уведомления о подтверждении прочтения или загрузки роумингового хранилища после успешного вызова `sendMessageReadReceipt` получателем. |
| isBroadcastMessage | Boolean | false | Является ли сообщение трансляционным. `true` указывает на да. |
| isSupportExtension | Boolean | false | поддерживает ли сообщение расширения. `true` указывает на да. Для использования требуется приобретение пакета Premium edition. |
| revokerInfo | Object | - | Информация об отзывающем сообщение. |
| revokeReason | String | - | Причина отзыва сообщения. |
| hasRiskContent | Boolean | - | Отмечены ли изображения, голосовые и видеосообщения как имеющие риск безопасности, по умолчанию false. Если содержимое отправленного вами голосового или видеосообщения не соответствует нормам, SDK вызовет событие `MESSAGE_MODIFIED` после асинхронной облачной проверки. |

### TextPayload

| Имя | Тип | Описание |
| --- | --- | --- |
| text | String | Содержание текстового сообщения |

### ImagePayload

| Имя | Тип | Описание |
| --- | --- | --- |
| uuid | String | Уникальный идентификатор изображения |
| imageFormat | Format | Тип формата изображения. JPG/JPEG = 1, GIF = 2, PNG = 3, BMP = 4, другое = 255 |
| imageInfoArray | Array.<Object> | Информация об изображении. Каждый элемент содержит описываемые ниже атрибуты:width - Numberheight - Numberurl - String - адрес изображения, который может быть использован для отрисовкиsize - Number - размер изображения в байтахtype - Number - 0, исходное изображение; 1, сжатое изображение 198p; 2, сжатое изображение 720p |

### AudioPayload

| Имя | Тип | Описание |
| --- | --- | --- |
| uuid | String | Уникальный идентификатор |
| url | String | Адрес аудио, который может быть использован для воспроизведения |
| size | Number | Размер файла в байтах |
| second | Number | Продолжительность аудио в секундах |

### VideoPayload

| Имя | Тип | Описание |
| --- | --- | --- |
| videoFormat | String | Формат видеофайла |
| videoSecond | Number | Продолжительность видеофайла в секундах, целочисленного типа |
| videoSize | Number | Размер видеофайла в байтах |
| videoUrl | String | Адрес видеофайла, который может быть использован для воспроизведения |
| videoUUID | String | Уникальный идентификатор видео |
| snapshotWidth | Number | ширина изображения миниатюры |
| snapshotHeight | Number | высота изображения миниатюры |
| snapshotUrl | String | адрес изображения миниатюры, который может быть использован для отрисовки (требуется [tim-upload-plugin](https://www.npmjs.com/package/tim-upload-plugin)) |

### FilePayload

| Имя | Тип | Описание |
| --- | --- | --- |

| fileName | String | Имя файла |
| fileUrl | String | Адрес файла |

### CustomPayload

| Имя | Тип | Описание |
| --- | --- | --- |
| data | String | Поле "data" пользовательского сообщения |
| description | String | Поле "description" пользовательского сообщения |
| extension | String | Поле "extension" пользовательского сообщения |

### MergerPayload

| Имя | Тип | Описание |
| --- | --- | --- |
| downloadKey | String | Ключ для загрузки объединённого сообщения. Если объединённое сообщение большого размера, SDK сохранит его в облаке, используя этот ключ в качестве уникального идентификатора сообщения. |
| messageList | Array.<SimplifiedMessage> | Список объединённых сообщений. |
| title | String | Название объединённого сообщения, например, "История чатов Центра талантов в Большом заливе". |
| abstractList | String | Список кратких содержаний. Вы можете установить информацию кратких содержаний в разных форматах для разных типов сообщений, например: для текстового сообщения краткое содержание может быть в формате "отправитель:текст". Для сообщения с изображением краткое содержание может быть в формате "отправитель:[изображение]". Для файлового сообщения краткое содержание может быть в формате "отправитель:[файл]". |
| compatibleText | String | Совместимый текст. SDK ранних версий не поддерживают объединённые сообщения и по умолчанию отправят текстовое сообщение с содержанием `compatibleText`. |
| layersOverLimit | Boolean | Превышен ли лимит уровней вложенности объединения. true: лимит превышен, сообщение усечено. Значение по умолчанию: false. |

### GroupTipPayload

| Имя | Тип | Описание |
| --- | --- | --- |
| groupJoinType | Number | Тип присоединения к группе. Поддерживаемые значения:0 - значение по умолчанию, указывающее, что операция не является операцией присоединения к группе.1 - пользователь запрашивает присоединение к группе.2 - пользователь приглашается в группу. |
| operatorID | String | ID пользователя, выполняющего операцию. |
| operatorInfo | Object | Информация пользователя, выполняющего эту операцию (поддерживается с версии v3.4.2 и выше). |
| operationType | Number | Тип операции. Поддерживаемые значения:1 - член присоединяется к группе.2 - член покидает группу.3 - член группы удаляется из группы.4 - члену группы предоставляется роль администратора группы.5 - роль администратора группы отозвана у члена группы.6 - профиль группы изменён.7 - профиль члена группы изменён. Например, член группы отключен звук.10 - член аудио-видео группы заблокирован.11 - член аудио-видео группы разблокирован. |
| userIDList | Array.<String> | Список связанных userID. |
| newGroupProfile | Object | Если профиль группы изменён, это поле хранит новый профиль группы. |
| memberList | Array.<Object> | Если члену группы отключен звук, это поле хранит связанную информацию. Каждый элемент содержит описываемые ниже атрибуты:`userID` - String - userID члена группы`muteTime` - Number - продолжительность отключения звука в секундах`nick` - String - прозвище члена группы`avatar` - String - аватар членов группы`duration` - Number - продолжительность блокировки, единица: секунды`reason` - String - причина блокировки |

### GroupSystemNoticePayload

| Имя | Тип | Описание |
| --- | --- | --- |
| operatorID | String | ID пользователя, выполняющего операцию. |
| operationType | Number | Тип операции. Поддерживаемые значения:1 - пользователь запрашивает присоединение к группе.2 - запрос на присоединение к группе одобрен.3 - запрос на присоединение к группе отклонён.4 - пользователь удалён из группы.5 - группа удалена.6 - группа создана.7 - пользователь приглашен в группу.8 - пользователь покидает группу.9 - администратор изменён.10 - администратор отменён.11 - группа восстановлена.12 - пользователь получает приглашение присоединиться к группе и должен принять или отклонить приглашение.13 - пользователь приглашает других присоединиться к группе и приглашение принято.14 - пользователь приглашает других присоединиться к группе и приглашение отклонено.15 - отчёт о прочтении синхронизирован между несколькими клиентами.20 - тип напоминания группового сообщения синхронизирован между несколькими клиентами.21 - заблокировать члена аудио-видео группы.22 - разблокировать члена аудио-видео группы.23 - член группы приглашает других присоединиться к группе.255 - запущено пользовательское уведомление. |
| groupProfile | Object | Профиль соответствующей группы. |
| userDefinedField | String | Пользовательское поле. |
| handleMessage | Object | Замечания по обработке. Например, если пользователь1 вводит замечание на заявку на присоединение к группе1, требующую одобрения, администратор группы1 увидит это поле в системном сообщении группы. |
| messageRemindType | String | `TencentCloudChat.TYPES.MSG_REMIND_ACPT_AND_NOTE` - SDK получает сообщение и выбрасывает событие MESSAGE_RECEIVED для уведомления стороны доступа, которая затем отправляет уведомление.`TencentCloudChat.TYPES.MSG_REMIND_ACPT_NOT_NOTE` - SDK получает сообщение и выбрасывает событие MESSAGE_RECEIVED для уведомления стороны доступа, которая затем не отправляет уведомление.`TencentCloudChat.TYPES.MSG_REMIND_DISCARD` - SDK отклоняет сообщение.`TencentCloudChat.TYPES.NOT_RECEIVE_OFFLINE_PUSH_EXCEPT_AT` получать сообщения в сети и получать уведомления о push только для сообщений группы @ в офлайне |

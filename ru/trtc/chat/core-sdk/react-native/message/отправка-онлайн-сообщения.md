# Отправка онлайн-сообщения

## Обзор функции

В некоторых случаях может потребоваться, чтобы сообщение было получено получателем только при его онлайн-статусе; то есть получатель не заметит сообщение при оффлайне. Вам только нужно установить `onlineUserOnly` на `true` при вызове `sendMessage`. Сообщение, отправленное таким образом, отличается от обычного следующим образом:

1. Оно не может быть сохранено в оффлайне; то есть не может быть получено, если получатель оффлайн.
2. Оно не может быть синхронизировано между устройствами; то есть если оно было получено на одном устройстве, оно не может быть получено на другом, независимо от того, прочитано оно или нет.
3. Оно не может быть сохранено локально; то есть не может быть извлечено из локальных или облачных исторических сообщений.

## Индикаторы печати

В индивидуальных чатах вы можете вызвать API `sendMessage` для отправки сообщения с подсказкой "Печать...". После получения такого сообщения получатель может отобразить "Печать..." в интерфейсе.

##### **Примеры**

```
chat.sendMessage(message, {  // If the receiver is offline, the message will be neither stored on the roaming server  // nor pushed offline.  onlineUserOnly: true});
```

На самом деле, чтобы реализовать полную подсказку "Печать...", отправка онлайн-сообщений с помощью Chat SDK — это только небольшая часть. Требуется дополнительная работа по разработке, например:

- Мониторинг интерфейса пользователя и обновление статуса: К полю ввода сообщений добавляется прослушиватель событий для обнаружения момента, когда пользователь начинает печатать. Когда начинается печать, клиент обновляет статус пользователя, например "печатает".
- Отправка статуса на сервер: В подходящее время статус ввода пользователя отправляется на сервер.
- Обновление интерфейса: На основе полученного статуса ввода на интерфейсе получателя отображается подсказка, такая как "Печать". Это может быть текстовая подсказка, значок или анимация.
- Рассмотрение регулирования частоты: Чтобы снизить количество ненужных обновлений статуса и коммуникационных затрат, можно установить временной интервал. Только одно обновление статуса ввода отправляется в течение этого интервала.

# Отправка сообщения

### Создание текстового сообщения

Этот API используется для создания текстового сообщения. Он возвращает экземпляр сообщения, который можно отправить, вызвав API [sendMessage](https://web.sdk.qcloud.com/im/doc/v3/zh-cn/SDK.html#sendMessage), когда вам нужно отправить экземпляр текстового сообщения.

##### **API**

```
chat.createTextMessage(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Содержащиеся в нём атрибуты описаны ниже:

| Параметры | Тип | Описание |
| --- | --- | --- |
| to | String | UserID или groupID получателя сообщения |
| conversationType | String | Тип беседы со значениями `TencentCloudChat.TYPES.CONV_C2C` (беседа один-на-один) или `TencentCloudChat.TYPES.CONV_GROUP` (групповая беседа) |
| priority | String | Приоритет сообщения. Если превышен лимит частоты сообщений в группе, серверная часть будет приоритизировать доставку сообщений с высоким приоритетом. Поддерживаемые значения перечисления:`TencentCloudChat.TYPES.MSG_PRIORITY_HIGH``TencentCloudChat.TYPES.MSG_PRIORITY_NORMAL (по умолчанию)``TencentCloudChat.TYPES.MSG_PRIORITY_LOW``TencentCloudChat.TYPES.MSG_PRIORITY_LOWEST` |
| payload | Object | Контейнер содержимого сообщения |
| cloudCustomData | String | Пользовательские данные сообщения, которые сохраняются в облаке, будут отправлены адресату и могут быть загружены после удаления и переустановки приложения |
| receiverList | Array \| undefined | Список членов группы для целевого приёма сообщения (не поддерживается для сообществ и групп прямых трансляций) |
| isSupportExtension | Boolean | Поддерживается ли расширение сообщения, true — поддерживается, false — не поддерживается (требуется приобрести [Pro edition ãPro Plus edition или Enterprise edition](https://console.trtc.io/subscription/buy/chat?packType=pro)) |

Описание `payload` приведено ниже:

| Параметры | Тип | Описание |
| --- | --- | --- |
| text | String | Текстовое содержимое сообщения |

##### **Возвращаемое значение**

[Message](https://web.sdk.qcloud.com/im/doc/v3/zh-cn/Message.html)

##### **Пример**

```
// Отправка текстовых сообщений, одинаково для Web и Mini Program// 1. Создайте экземпляр сообщения, экземпляр, возвращаемый интерфейсом, может быть отображён на экранеlet message = chat.createTextMessage({  to: 'user1',  conversationType: TencentCloudChat.TYPES.CONV_C2C,  // Приоритет сообщения для группового чата. Если превышен лимит частоты сообщений в группе, серверная часть будет приоритизировать сообщения с высоким приоритетом  // priority: TencentCloudChat.TYPES.MSG_PRIORITY_NORMAL,  payload: {    text: 'Hello world!'  },  // Если вам нужны уведомления о прочтении для ваших сообщений, вам нужно приобрести флагманский пакет и установить needReadReceipt на true при создании сообщения  needReadReceipt: true  // Пользовательские данные сообщения, сохранённые в облаке, будут отправлены адресату и могут быть загружены после удаления и переустановки приложения  // cloudCustomData: 'your cloud custom data'});// 2. Отправить сообщениеlet promise = chat.sendMessage(message);promise.then(function(imResponse) {  // Отправка успешна.  console.log(imResponse);}).catch(function(imError) {  // Ошибка отправки.  console.warn('sendMessage error:', imError);});
```

```
// Отправка целевого группового сообщения// Примечание: Целевые групповые сообщения исключаются из подсчёта непрочитанных сообщений беседы. `receiverList` поддерживает до 50 получателей.let message = chat.createTextMessage({  to: 'group1',  conversationType: TencentCloudChat.TYPES.CONV_GROUP,  payload: {    text: 'Hello world!'  },  // Для отправки целевого группового сообщения необходимо приобрести пакет Enterprise Edition и указать получателей сообщения через receiverList при создании сообщения  receiverList: ['user0', 'user1']});// Отправить сообщение.let promise = chat.sendMessage(message);promise.then(function(imResponse) {  // Отправка успешна.  console.log(imResponse);}).catch(function(imError) {  // Ошибка отправки.  console.warn('sendMessage error:', imError);});

### Создание сообщения с упоминанием (@mention)

Этот API используется для создания текстового сообщения с упоминанием кого-либо символом @. Он возвращает экземпляр сообщения, который можно отправить, вызвав API [sendMessage](https://web.sdk.qcloud.com/im/doc/v3/zh-cn/SDK.html#sendMessage), когда вам нужно отправить экземпляр текстового сообщения.

> **Примечания:** Этот API предназначен только для групповых чатов, и @ALL не поддерживается в сообществах и темах под сообществами. Создание групповых сообщений @ не поддерживает указание получателей сообщения.

##### **API**

```
chat.createTextAtMessage(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Содержащиеся в нём атрибуты описаны ниже:

| Параметры | Тип | Описание |
| --- | --- | --- |
| to | String | UserID или groupID получателя сообщения |
| conversationType | String | Тип беседы со значениями `TencentCloudChat.TYPES.CONV_C2C` (беседа один-на-один) или `TencentCloudChat.TYPES.CONV_GROUP` (групповая беседа) |
| priority | String | Приоритет сообщения. Если превышен лимит частоты сообщений в группе, серверная часть будет приоритизировать доставку сообщений с высоким приоритетом. Поддерживаемые значения перечисления:`TencentCloudChat.TYPES.MSG_PRIORITY_HIGH``TencentCloudChat.TYPES.MSG_PRIORITY_NORMAL (по умолчанию)``TencentCloudChat.TYPES.MSG_PRIORITY_LOW``TencentCloudChat.TYPES.MSG_PRIORITY_LOWEST` |
| payload | Object | Контейнер содержимого сообщения |
| cloudCustomData | String | Пользовательские данные сообщения, которые сохраняются в облаке, будут отправлены адресату и могут быть загружены после удаления и переустановки приложения |

Описание `payload` приведено ниже:

| Параметры | Тип | Описание |
| --- | --- | --- |
| text | String | Текстовое содержимое сообщения |
| atUserList | Array | Список пользователей для упоминания (@). Если вам нужно упомянуть всех, передайте [TencentCloudChat.TYPES.MSG_AT_ALL](https://web.sdk.qcloud.com/im/doc/v3/zh-cn/module-TYPES.html#.MSG_AT_ALL). Например, если вы хотите упомянуть в этом текстовом сообщении как denny, так и lucy, а также упомянуть всех, передайте ['denny', 'lucy', TencentCloudChat.TYPES.MSG_AT_ALL] в atUserList. |

##### **Возвращаемое значение**

[Message](https://web.sdk.qcloud.com/im/doc/v3/zh-cn//Message.html)

##### **Пример**

```
// Отправка текстовых сообщений, одинаково для Web и Mini Program// 1. Создайте экземпляр сообщения, экземпляр, возвращаемый интерфейсом, может быть отображён на экранеlet message = chat.createTextAtMessage({  to: 'group1',  conversationType: TencentCloudChat.TYPES.CONV_GROUP,  // Приоритет сообщения для группового чата  // priority: TencentCloudChat.TYPES.MSG_PRIORITY_NORMAL,  payload: {    text: '@denny @lucy @everyone Ужин сегодня, пожалуйста, ответьте 1, если получили',    // 'denny' и 'lucy' — это userID, не никнеймы    atUserList: ['denny', 'lucy', TencentCloudChat.TYPES.MSG_AT_ALL]  },  // Пользовательские данные сообщения, сохранённые в облаке, будут отправлены адресату и могут быть загружены после удаления и переустановки приложения  // cloudCustomData: 'your cloud custom data'});// 2. Отправить сообщениеlet promise = chat.sendMessage(message);promise.then(function(imResponse) {  // Отправка успешна.  console.log(imResponse);}).catch(function(imError) {  // Ошибка отправки.  console.warn('sendMessage error:', imError);});
```

### Создание сообщения с изображением

API для создания сообщения с изображением возвращает экземпляр сообщения, который можно отправить с помощью API [send message](https://web.sdk.qcloud.com/im/doc/v3/zh-cn/SDK.html#sendMessage) при необходимости отправки сообщения с изображением. Для отправки сообщений с изображениями в React Native необходимо интегрировать [react-native-image-picker](https://www.npmjs.com/package/react-native-image-picker).

```
npm install react-native-image-picker --save
```

##### **API**

```
chat.createImageMessage(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Содержащиеся в нём атрибуты описаны в таблице ниже:

| Параметры | Тип | Описание |
| --- | --- | --- |
| to | String | Получатель сообщения |
| conversationType | String | Тип беседы со значениями `TencentCloudChat.TYPES.CONV_C2C` или `TencentCloudChat.TYPES.CONV_GROUP` |
| priority | String | Приоритет сообщения. Если превышен лимит частоты сообщений в группе, серверная часть будет приоритизировать доставку сообщений с высоким приоритетом. Поддерживаемые значения перечисления:`TencentCloudChat.TYPES.MSG_PRIORITY_HIGH``TencentCloudChat.TYPES.MSG_PRIORITY_NORMAL (по умолчанию)``TencentCloudChat.TYPES.MSG_PRIORITY_LOW``TencentCloudChat.TYPES.MSG_PRIORITY_LOWEST` |
| payload | Object | Контейнер содержимого сообщения |
| onProgress | function | Функция обратного вызова для получения прогресса загрузки |
| cloudCustomData | String | Пользовательские данные сообщения, которые сохраняются в облаке, будут отправлены адресату и могут быть загружены после удаления и переустановки приложения |

Описание `payload` приведено ниже:

| Параметры | Тип | Описание |
| --- | --- | --- |
| file | HTMLInputElement \| Object \| File | Узел DOM для выбора изображений (Web) или объект File (Web)Параметр обратного вызова успешного выполнения интерфейса wx.chooseImage. SDK будет читать данные и загружать изображение. |

### Создание аудиосообщения

Интерфейс для создания экземпляра аудиосообщения. Этот интерфейс возвращает экземпляр сообщения, который можно использовать для вызова интерфейса [Отправка сообщения](https://web.sdk.qcloud.com/im/doc/v3/zh-cn/SDK.html#sendMessage) при необходимости отправки аудиосообщения. Для отправки аудиосообщений в React Native необходимо интегрировать [react-native-audio-recorder-player](https://www.npmjs.com/package/react-native-audio-recorder-player).

```
npm install react-native-audio-recorder-player --save
```

##### **API**

```
chat.createAudioMessage(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Содержащиеся в нём атрибуты описаны ниже:

| Параметры | Тип | Описание |
| --- | --- | --- |
| to | String | Получатель сообщения |
| conversationType | String | Тип беседы со значениями `TencentCloudChat.TYPES.CONV_GROUP` или `TencentCloudChat.TYPES.CONV_GROUP` |
| priority | String | Приоритет сообщения. Если превышен лимит частоты сообщений в группе, серверная часть будет приоритизировать доставку сообщений с высоким приоритетом. Поддерживаемые значения перечисления:`TencentCloudChat.TYPES.MSG_PRIORITY_HIGH``TencentCloudChat.TYPES.MSG_PRIORITY_NORMAL (по умолчанию)``TencentCloudChat.TYPES.MSG_PRIORITY_LOW``TencentCloudChat.TYPES.MSG_PRIORITY_LOWEST` |

| --- | --- | --- |
| file | Object | Аудиофайл, полученный после записи |

##### **Возвращаемое значение**

[Message](https://web.sdk.qcloud.com/im/doc/v3/zh-cn/Message.html)

##### **Пример**

```
import AudioRecorderPlayer, {AVEncodingOption} from 'react-native-audio-recorder-player'; // Записывайте длительность записи, единица: мсlet duration = 0;// Записывайте путь файла записиlet uri = '';// 1. ### Создание сообщения с эмодзи

Этот API используется для создания сообщения с эмодзи. Он возвращает экземпляр сообщения, который можно отправить, вызвав API [sendMessage](https://web.sdk.qcloud.com/im/doc/v3/zh-cn/SDK.html#sendMessage) при необходимости отправки сообщения с эмодзи.

##### **API**

```
chat.createFaceMessage(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Содержащиеся в нём атрибуты описаны ниже:

| Параметры | Тип | Описание |

| to | String | UserID или groupID получателя сообщения |
| conversationType | String | Тип беседы со значениями `TencentCloudChat.TYPES.CONV_C2C` или `TencentCloudChat.TYPES.CONV_GROUP` |
| priority | String | Приоритет сообщения. Если превышен лимит частоты сообщений в группе, серверная часть будет приоритизировать доставку сообщений с высоким приоритетом. Поддерживаемые значения перечисления:`TencentCloudChat.TYPES.MSG_PRIORITY_HIGH``TencentCloudChat.TYPES.MSG_PRIORITY_NORMAL (по умолчанию)``TencentCloudChat.TYPES.MSG_PRIORITY_LOW``TencentCloudChat.TYPES.MSG_PRIORITY_LOWEST` |
| payload | Object | Контейнер содержимого сообщения |
| cloudCustomData | String | Пользовательские данные сообщения, которые сохраняются в облаке, будут отправлены адресату и могут быть загружены после удаления и переустановки приложения |

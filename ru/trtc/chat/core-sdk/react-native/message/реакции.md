# Реакции

## Обзор функции

Функция "Ответ на сообщение" относится к интерактивным ответам на конкретное сообщение, типичным примером которых является "Ответ с эмодзи". Ответы с эмодзи создаются с использованием эмодзи, что позволяет нам видеть количество ответивших и список ответивших для каждого эмодзи.

Распространённые методы отображения "Ответа с эмодзи" в настоящее время включают следующие два стиля:

| Стиль один | ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/38f82b4c58f811ee974d5254005f490f.png) |
| --- | --- |
| Стиль два | ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/3dee908d58f811ee974d5254005f490f.png) |

Пользователи могут переходить по страницам и получать профили всех пользователей, использовавших определённый эмодзи.

Вы можете реализовать возможность "Ответ с эмодзи" на основе API SDK:

- Вызовите интерфейс `addMessageReaction` для добавления эмодзи к сообщению. После успешного добавления эмодзи будет хранить информацию пользователя, который выполнил операцию.
- Вызовите интерфейс `removeMessageReaction` для удаления уже добавленного эмодзи. После успешного удаления эмодзи больше не будет хранить информацию пользователя, который выполнил операцию.
- Вызовите интерфейс `getMessageReactions` для массового получения списка эмодзи нескольких сообщений, где каждый эмодзи включает общее количество пользователей, использующих его в настоящее время, и профили первых N (по умолчанию 10) пользователей.
- Вызовите интерфейс `getAllUserListOfMessageReaction` для постраничного получения полного списка профилей пользователей, использующих эмодзи сообщения.
- Слушайте событие `TencentCloudChat.EVENT.MESSAGE_REACTIONS_UPDATE` для обнаружения изменений в информации пользователей, использующих эмодзи. Это событие будет содержать последнюю информацию пользователей эмодзи (включая общее количество пользователей и профили топ N пользователей).

> **Примечание:** Поддерживается с версии v3.2.0, эта функция доступна только для расширенных клиентов. После [приобретения Pro edition, Pro Plus edition или Enterprise edition](https://trtc.io/pricing/chat) её можно использовать.

### Добавление ответа на сообщение

Вызывая интерфейс `addMessageReaction`, вы можете добавить ответ на сообщение.

##### **API**

```
chat.addMessageReaction(message, reactionID);
```

**Параметры**

| Атрибут | Значение | Описание |
| --- | --- | --- |
| message | Объект сообщения | Сообщение успешно отправлено. |
| reactionID | ID ответа на сообщение | В сценариях ответа с эмодзи reactionID — это ID эмодзи. |

> **Примечание:** Одно сообщение поддерживает до 10 реакций, а одна реакция поддерживает до 100 пользователей. Если количество реакций для одного сообщения превышает максимальный лимит (10), вызов API вернёт код ошибки 23005. Если количество пользователей для одной реакции превышает максимальный лимит (100), вызов API вернёт код ошибки 23006. Если реакция уже содержит текущего пользователя, вызов API вернёт код ошибки 23007.

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.addMessageReaction(message, 'smile');promise.then(function(imResponse) {  // Ответ на сообщение успешно добавлен}).catch(function(imError) {  // Не удалось добавить ответ на сообщение  console.warn('addMessageReaction error:', imError);});
```

### Удаление ответа на сообщение

Вызовите API `removeMessageReaction` для удаления ответа на сообщение.

##### **API**

```
chat.removeMessageReaction(message, reactionID);
```

**Параметры**

| Атрибут | Значение | Описание |
| --- | --- | --- |
| message | Объект сообщения | Сообщение успешно отправлено. |
| reactionID | ID ответа на сообщение | В сценариях ответа с эмодзи reactionID — это ID эмодзи. |

> **Примечание:** Если реакция не существует, вызов API вернёт код ошибки 23008. Если реакция не содержит текущего пользователя, вызов API вернёт код ошибки 23009.

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.removeMessageReaction(message, 'smile');promise.then(function(imResponse) {  // Ответ на сообщение успешно удалён}).catch(function(imError) {  // Не удалось удалить ответ на сообщение  console.warn('removeMessageReaction error:', imError);});
```

### Массовое получение нескольких ответов на сообщения

Вызовите интерфейс `getMessageReactions` для массового получения информации о нескольких ответах на сообщения.

##### **API**

```
chat.getMessageReactions(options);
```

**Параметры**

| Атрибут | Значение | Описание |
| --- | --- | --- |
| messageList | Список сообщений | Сообщения должны принадлежать одной сессии и должны быть в успешно отправленном состоянии. |
| maxUserCountPerReaction | Максимальное количество профилей пользователей, возвращаемых для каждой реакции | Диапазон значений [0,10], каждая реакция возвращает профили максимум первых 10 пользователей. Для получения дополнительных профилей пользователей вы можете при необходимости вызвать интерфейс `getAllUserListOfMessageReaction` для постраничного получения. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.getMessageReactions({  messageList: [message1, message2],  maxUserCountPerReaction: 10,});promise.then(function(imResponse) {  // Массовое получение успешно   const resultList = imResponse.data;   resultList.forEach((item) => {     const { messageID, reactionList } = item;     // Информация, возвращённая reactionList, выглядит следующим образом:     // reactionID — ID ответа на сообщение     // totalUserCount — общее количество пользователей, ответивших на сообщение с тем же reactionID     // partialUserList — частичный список пользователей с тем же reactionID, содержащий информацию userID, nick, avatar пользователя   });}).catch(function(imError) {  // Массовое получение не удалось  console.warn('getMessageReactions error:', imError);});
```

### Постраничное получение полного списка пользователей, ответивших на сообщение

Вызовите интерфейс `getAllUserListOfMessageReaction` для постраничного получения полного списка пользователей, ответивших на сообщение.

##### **API**

```
chat.getAllUserListOfMessageReaction(options);
```

**Параметры**

| Атрибут | Значение | Описание |
| --- | --- | --- |
| message | Объект сообщения | Сообщение успешно отправлено. |
| reactionID | ID ответа на сообщение | В сценариях ответа с эмодзи reactionID — это ID эмодзи. |
| nextSeq | Курсор для постраничного получения | При первой передаче передайте 0, затем для постраничного получения передайте nextSeq, возвращённый функцией succ. |
| count | Максимальное количество элементов, которые можно получить на одной странице | Поддерживает до 100 |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.getAllUserListOfMessageReaction({  message: message,  reactionID: 'smile',  nextSeq: 0,  count: 100,});promise.then(function(imResponse) {  // Получено успешно  const { userList, nextSeq, isCompleted } = imResponse.data;  // userList — список информации пользователей, включающий информацию userID, nick, avatar пользователя  // nextSeq — этот параметр должен быть передан при следующем постраничном получении  // isCompleted — указывает, получена ли вся информация пользователей. Когда isCompleted имеет значение true, nextSeq равен 0}).catch(function(imError) {  // Не удалось получить  console.warn('getAllUserListOfMessageReaction error:', imError);});
```

### Обновление информации об ответе на сообщение

Когда информация об ответе на сообщение изменяется, SDK отправит событие `TencentCloudChat.EVENT.MESSAGE_REACTIONS_UPDATED`. Вы можете получить обновленную информацию об ответе на сообщение в зарегистрированном обратном вызове.

> **Примечание:** Когда значение поля totalUserCount в изменённой информации реакции равно 0, это указывает на то, что никто больше не использует эту реакцию. Вы можете удалить отображение этой реакции из UI.

##### **Примеры**

```
let onMessageReactionsUpdated = function(event) {  const { messageID, reactionList } = event.data;  // messageID — ID сообщения  // reactionList — список обновлений ответов на сообщение  reactionList.forEach((item) => {   const { reactionID, totalUserCount, partialUserList } = item;   // reactionID — ID ответа на сообщение   // totalUserCount — общее количество пользователей, ответивших на сообщение с тем же reactionID   // partialUserList — частичный список пользователей с тем же reactionID  });};chat.on(TencentCloudChat.EVENT.MESSAGE_REACTIONS_UPDATED, onMessageReactionsUpdated);
```

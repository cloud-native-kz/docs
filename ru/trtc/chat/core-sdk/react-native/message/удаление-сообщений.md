# Удаление сообщений

## Обзор функции

После успешного удаления сообщения его свойство `isDeleted` устанавливается в значение `true`. В персональных беседах удаленные сообщения не могут быть получены при следующем входе, но получатель на это не повлияет. В групповых беседах удаленные сообщения не могут быть получены при следующем входе, но другие члены группы на это не повлияют.

> **Примечание:** За один раз можно удалить до 30 сообщений. Если выбрано более 30 сообщений, будут удалены первые 30 сообщений. Удаляемые сообщения должны быть из одной беседы, то есть из беседы первого сообщения в списке сообщений. Этот API можно вызывать только один раз в секунду. Удаленные сообщения не синхронизируются. Сообщения нельзя удалять из аудио-видео групп (AVChatRoom), и если вы вызовете этот API, будет возвращен код ошибки 10035. Этот API не поддерживает удаление системных уведомлений группы.

## Отображение в интерфейсе

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/7058314ecbe011ef81865254005ef0f7.png)

##### **API**

```
chat.deleteMessage(messageList);
```

##### **Параметры**

| Имя | Тип | Описание |
| --- | --- | --- |
| messageList | Array | Список сообщений (до 30) из одной беседы |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.deleteMessage([message1, message2, message3, ...]);promise.then(function(imResponse) {  // Messages deleted successfully}).catch(function(imError) {  // Failed to delete the messages  console.warn('deleteMessage error:', imError);});
```

# Упоминания

## Обзор функции

Отправитель прослушивает символы в поле ввода. Когда пользователь вводит @, откроется пользовательский интерфейс выбора членов группы. После выбора целевых членов группы сообщение будет отображаться в поле ввода в формате `"@A @B @C......"`, которое можно дополнительно редактировать перед отправкой.

В списке групповых чатов пользовательского интерфейса переписки получателя будет отображаться идентификатор `"someone@me"` или `"@all"`, чтобы напомнить пользователю, что он был упомянут кем-то в групповом чате.

> **Примечание** В настоящее время поддерживаются только текстовые @-сообщения.

## Отправка сообщения с упоминанием группы @

1. Отправитель прослушивает текстовое поле ввода в пользовательском интерфейсе чата и запускает пользовательский интерфейс выбора членов группы. После выбора членов группы вызывается информация об ID и никнеймах членов.
2. Отправитель вызывает API `createTextAtMessage` для создания сообщения группового @.
3. Отправитель вызывает `sendMessage` для отправки созданного объекта сообщения @.

##### API

```
chat.createTextAtMessage(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Он содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| to | String | `groupID` получателя сообщения |
| conversationType | String | Тип переписки, который должен быть `TencentCloudChat.TYPES.CONV_GROUP` |
| payload | Object | Контейнер содержимого сообщения |
| cloudCustomData | String | Пользовательские данные сообщения, которые сохраняются в облаке, будут отправлены получателю и могут быть восстановлены после удаления и переустановки приложения. |
| priority | String | Приоритет сообщения. Если сообщения в группе превышают лимит частоты, сервер будет доставлять сообщения с высоким приоритетом в первую очередь. Поддерживаемые перечисляемые значения: `TencentCloudChat.TYPES.MSG_PRIORITY_HIGH``TencentCloudChat.TYPES.MSG_PRIORITY_NORMAL (по умолчанию)``TencentCloudChat.TYPES.MSG_PRIORITY_LOW``TencentCloudChat.TYPES.MSG_PRIORITY_LOWEST` |

`payload` описывается следующим образом:

| Имя | Тип | Описание |
| --- | --- | --- |
| text | String | Текстовое содержимое сообщения |
| atUserList | Array.<String> | Список пользователей, которые должны быть упомянуты @. Если вам нужно упомянуть всех, передайте `TencentCloudChat.TYPES.MSG_AT_ALL`. Например, если это текстовое сообщение хочет упомянуть denny и lucy, а также упомянуть всех, `atUserList` будет передан как `['denny', 'lucy', TencentCloudChat.TYPES.MSG_AT_ALL]`. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let message = chat.createTextAtMessage({  to: 'group1',  conversationType: TencentCloudChat.TYPES.CONV_GROUP,  payload: {    text: '@denny @lucy Dinner gathering tonight, reply with 1 if you are coming.',    atUserList: ['denny', 'lucy', TencentCloudChat.TYPES.MSG_AT_ALL] // 'denny' 'lucy' are both userID  },  // cloudCustomData: 'your cloud custom data'});let promise = chat.sendMessage(message);promise.then(function(imResponse) {  console.log(imResponse);}).catch(function(imError) {  console.warn('sendMessage error:', imError);});
```

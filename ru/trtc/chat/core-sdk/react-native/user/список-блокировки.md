# Список блокировки

## Обзор функции

Для блокировки сообщений пользователя добавьте пользователя в список блокировки.

> **Примечание:** Максимальное количество пользователей в списке блокировки составляет 1000, и оно не может быть изменено. Если вам нужно добавить друга в список блокировки, сохраняя при этом дружбу, перейдите в [консоль чата](https://console.trtc.io/chat/friends-diy-vars), найдите соответствующий SDKAppID, перейдите в Chat > Configuration > Friends and Relationship > Block configuration и выберите `Remain Friends`.

## Блокировка пользователя

Добавляя пользователя в список блокировки, вы можете заблокировать все сообщения, отправляемые этим пользователем. Таким образом, этот API можно использовать для блокировки сообщений указанного пользователя.

- Если пользователи A и B являются друзьями, двусторонняя связь дружбы прерывается, когда один из них блокирует другого.
- Если пользователь A или B блокирует другого, ни пользователь A, ни пользователь B не могут начать беседу с другим пользователем.
- Если пользователь A или B блокирует другого, ни пользователь A, ни пользователь B не могут отправить запрос на добавление в друзья другому пользователю.

##### **API**

```
chat.addToBlacklist(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Он содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| userIDList | Array | Список значений `userID` пользователей, которые должны быть добавлены в список блокировки. Количество значений `userID` не может превышать 1000 в одном запросе. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.addToBlacklist({userIDList: ['user1', 'user2']});promise.then(function(imResponse) {  console.log(imResponse.data);  console.warn('addToBlacklist error:', imError); // Ошибка при добавлении пользователя в список блокировки});
```

## Разблокировка пользователя

После удаления пользователя из списка блокировки сообщения от этого пользователя можно будет получать.

##### **API**

```
chat.removeFromBlacklist(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object`. Он содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| userIDList | Array | Список значений `userID` пользователей, которые должны быть удалены из списка блокировки. Количество значений `userID` не может превышать 1000 в одном запросе. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.removeFromBlacklist({userIDList: ['user1', 'user2']});  console.log(imResponse.data);  console.warn('removeFromBlacklist error:', imError); // Ошибка при удалении пользователя из списка блокировки});
```

## Получение списка блокировки

Можно получить список блокировки, кэшированный в SDK.

##### **API**

```
chat.getBlacklist();
```

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.getBlacklist();promise.then(function(imResponse) {  console.log(imResponse.data);}).catch(function(imError) {  console.warn('getBlacklist error:', imError); // Ошибка при получении списка блокировки});
```

## Уведомление об изменении списка блокировки

Когда список блокировки обновляется, SDK отправляет событие `TencentCloudChat.EVENT.BLACKLIST_UPDATED`.

##### Примеры

```
let onBlacklistUpdated = function(event) {  console.log(event.data);};chat.on(TencentCloudChat.EVENT.BLACKLIST_UPDATED, onBlacklistUpdated);
```

# Статус пользователя

## Обзор функции

React Native SDK предоставляет функцию управления статусом пользователя, где каждый пользователь имеет два различных типа статуса:

- Обычный статус. Встроенный в SDK статус, который не может быть напрямую изменен клиентами.
- Пользовательский статус. Определяемый клиентом статус, который может быть изменен. Используя пользовательский статус, вы можете устанавливать информацию, такую как «Слушаю музыку», «Во время звонка» и т. д.

> **Примечание:** Статус пользователя относится к текущему пользователю и не зависит от устройства. Если несколько устройств одновременно авторизованы на одной учетной записи, запрос или установка статуса по устройству не поддерживается.

Существуют три типа обычного статуса пользователя:

- В сети (`TencentCloudChat.TYPES.USER_STATUS_ONLINE`): текущий пользователь вошел в систему и находится в сети, может нормально отправлять и получать сообщения.
- Не в сети (`TencentCloudChat.TYPES.USER_STATUS_OFFLINE`): вход/выход из веб-приложения не вызовет статус отключения. В приложениях, интегрирующих Native ChatSDK, статус отключения будет активирован.
- Не авторизован (`TencentCloudChat.TYPES.USER_STATUS_UNLOGINED`): пользователь никогда не входил в систему после регистрации учетной записи или пользователь активно вызвал `logout` для выхода.

## Отображение пользовательского интерфейса

## ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/0de73078c8be11ef86f2525400bf7822.png)Установка вашего пользовательского статуса

Вы можете вызвать интерфейс `setSelfStatus` для установки поля `customStatus` для вашего пользовательского статуса. После успешной установки вы получите уведомление об изменении вашего статуса через событие `TencentCloudChat.EVENT.USER_STATUS_UPDATED`. Для подробностей см. [раздел «Уведомление об изменении статуса»](https://www.tencentcloud.com/document/product/1047/66156#7118302a-bb4e-4104-9d2b-75b9962a4b77) ниже.

> **Примечание:** вызов `setSelfStatus` не требует обновления на Pro edition, Pro Plus edition или Enterprise edition, а также не требует включения переключателя консоли. Этот интерфейс не выполняет контроль частотного ограничения. Вы можете активно очистить свой статус, установив поле `customStatus` на пустую строку при вызове интерфейса `setSelfStatus`. После выхода из системы на некоторое время (Web) фоновый сервис Chat автоматически очистит пользовательский статус. Это также вызовет уведомление об изменении статуса.

##### **API**

```
chat.setSelfStatus(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| customStatus | String | Пользовательский статус пользователя |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Set customStatus to an empty string '', this will clear your custom statuslet promise = chat.setSelfStatus({customStatus: 'xxx'});promise.then(function(imResponse) {  console.log(imResponse.data);  const { userID, statusType, customStatus } = imResponse.data;  // userID - User ID  // statusType - User status. The enumerated values and descriptions are as follows:  // TencentCloudChat.TYPES.USER_STATUS_UNKNOWN - Unknown  // TencentCloudChat.TYPES.USER_STATUS_ONLINE - Online  // TencentCloudChat.TYPES.USER_STATUS_OFFLINE - Offline  // TencentCloudChat.TYPES.USER_STATUS_UNLOGINED - Not Logged In  // customStatus - User Custom Status}).catch(function(imError) {  console.warn('setSelfStatus error:', imError); // Failed to set user's custom status});
```

## Запрос статуса пользователя

API возвращает как обычный статус, так и пользовательский статус запрашиваемых пользователей.

> **Примечание:** запрос собственного статуса не требует обновления на Pro edition, Pro Plus edition или Enterprise edition или включения переключателя консоли. Запрос собственного статуса не подлежит ограничению частоты вызова API. Запрос статуса других пользователей требует обновления на Pro edition, Pro Plus edition или Enterprise edition с ограничением вызова интерфейса по умолчанию 20 запросов за 5 секунд, и один запрос может включать не более 500 пользователей. Запрос статуса других пользователей требует предварительного включения функции «Установить запрос статуса пользователя и уведомление об изменении статуса» в [консоли Chat](https://console.trtc.io/chat/login-message). Если переключатель отключен, вызов `getUserStatus` приведет к ошибке.

##### **API**

```
chat.getUserStatus(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| userIDList | Array | Список userID для запроса. При запросе себя нужно передать только ваш собственный userID. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Check your own user status// userIDList includes only your own userID to query your own statuslet promise = chat.getUserStatus({userIDList: [`${myUserID}`]});promise.then(function(imResponse) {   const { successUserList } = imResponse.data;   successUserList.forEach((item) => {     const { userID, statusType, customStatus } = item;     // userID - User ID     // statusType - User status, the enumerated values and descriptions are as follows:     // TencentCloudChat.TYPES.USER_STATUS_UNKNOWN - Unknown     // TencentCloudChat.TYPES.USER_STATUS_ONLINE - Online     // TencentCloudChat.TYPES.USER_STATUS_OFFLINE - Offline     // TencentCloudChat.TYPES.USER_STATUS_UNLOGINED - Not Logged In     // customStatus - User Definition Status   });}).catch(function(imError) {  console.warn('getUserStatus error:', imError); // Failed to obtain user status});
```

### Проверка статуса других пользователей

Установите `userIDList` на список userID других пользователей для проверки их статуса.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/15fb0819138011ef83b95254002977b6.png)

##### **Примеры**

```
// Query the status of otherslet promise = chat.getUserStatus({userIDList: ['user0', 'user1']});promise.then(function(imResponse) {   const { successUserList, failureUserList } = imResponse.data;   // List of users with successful queries   successUserList.forEach((item) => {     const { userID, statusType, customStatus } = item;     // userID - User ID     // statusType - User status, the enumerated values and descriptions are as follows:     // TencentCloudChat.TYPES.USER_STATUS_UNKNOWN - Unknown     // TencentCloudChat.TYPES.USER_STATUS_ONLINE - Online     // TencentCloudChat.TYPES.USER_STATUS_OFFLINE - Offline     // TencentCloudChat.TYPES.USER_STATUS_UNLOGINED - Not Logged In     // customStatus - User Definition Status   });   // List of users with failed queries   failureUserList.forEach((item) => {     const { userID, code, message } = item;     // userID - User ID of the failed query     // code - Error code of the failed query     // message - Error message of the failed query   });}).catch(function(imError) {  console.warn('getUserStatus error:', imError); // Failed to obtain user status});
```

### Подписка на статус пользователя

После успешной подписки при изменении статуса подписанного пользователя (включая обычный статус и пользовательский статус) вы получите уведомление об изменении статуса пользователя через событие `TencentCloudChat.EVENT.USER_STATUS_UPDATED`.

> **Примечание:** этот API требует обновления на Pro edition, Pro Plus edition или Enterprise edition. API имеет ограничение частоты 20 запросов за 5-секундный интервал, и максимальное количество пользователей на одну подписку не может превышать 100. Существует ограничение на количество подписок в списке. После превышения лимита пользователи с самой ранней подпиской будут автоматически исключены.

##### **API**

```
chat.subscribeUserStatus(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| userIDList | Array | Список userID пользователей, до 100 на запрос. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.subscribeUserStatus({userIDList: ['user0', 'user1']});promise.then(function(imResponse) {  const { failureUserList } = imResponse.data;   // List of users with failed subscriptions   failureUserList.forEach((item) => {     const { userID, code, message } = item;     // userID - User ID of the failed query     // code - Error code of the failed query     // message - Error message of the failed query   });}).catch(function(imError) {  // Error information regarding the failure to subscribe to user status  console.warn('subscribeUserStatus error:', imError);});
```

## Отписка от статуса пользователя

После успешной отписки при изменении статуса пользователя (включая обычный статус и пользовательский статус) SDK не будет отправлять событие `TencentCloudChat.EVENT.USER_STATUS_UPDATED`.

> **Примечание:** API имеет ограничение частоты 20 запросов за 5-секундный интервал, и максимальное количество пользователей на одну отписку не может превышать 100.

##### **API**

```
chat.unsubscribeUserStatus(options);
```

##### **Параметры**

| Имя | Тип | Описание |
| --- | --- | --- |
| userIDList | Array \| undefined | Список ID пользователей, максимум 100 на один запрос. Когда userIDList представляет собой пустой массив или `undefined`, отменяет все текущие подписки. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Unsubscribe current partial userslet promise = chat.unsubscribeUserStatus({userIDList: ['user0', 'user1']});promise.then(function(imResponse) {  const { failureUserList } = imResponse.data;   // List of users with failed unsubscriptions   failureUserList.forEach((item) => {     const { userID, code, message } = item;     // userID - User ID of the failed query     // code - Error code of the failed query     // message - Error message of the failed query   });}).catch(function(imError) {  console.warn('unsubscribeUserStatus error:', imError); // Information related to the unsubscription failure});
```

```
// Unsubscribe from all current subscriptionslet promise = chat.unsubscribeUserStatus();promise.then(function(imResponse) {  const { failureUserList } = imResponse.data;   // List of users with failed unsubscriptions   failureUserList.forEach((item) => {     const { userID, code, message } = item;     // userID - User ID of the failed query     // code - Error code of the failed query     // message - Error message of the failed query   });}).catch(function(imError) {  console.warn('unsubscribeUserStatus error:', imError); // Information related to the unsubscription failure});
```

### Уведомление об изменении статуса

В зависимости от типа пользователей, статус которых вы хотите отслеживать, уведомление об изменении статуса можно разделить на три типа:

1. Отслеживание изменения собственного статуса.

2. Отслеживание изменения статуса друзей.

3. Отслеживание изменения статуса пользователей (не друзей).

Уведомления об изменении статуса всех трех вышеупомянутых методов возвращаются через обратный вызов onUserStatusChanged (Java / Swift / Objective-C / C++). Однако различные типы пользователей вызывают это уведомление по-разному.

### Уведомление об изменении собственного статуса

Если вы предварительно зарегистрировали слушатель события `TencentCloudChat.EVENT.USER_STATUS_UPDATED`, при изменении вашего собственного статуса SDK отправит событие `TencentCloudChat.EVENT.USER_STATUS_UPDATED`, и вы сможете получить там ваш последний статус.

### Уведомление об изменении статуса друга

1. Если вы включили автоматическое уведомление о статусе друга в [консоли Chat](https://console.trtc.io/), при изменении статуса друга SDK отправит событие `TencentCloudChat.EVENT.USER_STATUS_UPDATED`, и вы сможете получить там последний статус вашего друга.
2. Если вы не включили автоматическое уведомление о статусе друга и все еще хотите получать уведомления об изменении статуса друга, вам необходимо вызвать `subscribeUserStatus` для активной подписки на статус друга. При изменении статуса друга SDK отправит обратный вызов `TencentCloudChat.EVENT.USER_STATUS_UPDATED`.
3. Если вы ни не включили автоматическое уведомление о статусе друга, ни не вызывали `subscribeUserStatus` для активной подписки на статус друга, то при изменении статуса друга вы не сможете его заметить.

### Изменение статуса обычных пользователей (не друзей)

Если вы хотите получать уведомления об изменении статуса обычного пользователя (не друга), вы можете только вызвать `subscribeUserStatus` для активной подписки. При изменении статуса этого пользователя будет запущен обратный вызов `TencentCloudChat.EVENT.USER_STATUS_UPDATED`, и вы сможете получить его последний статус там.

##### **Примеры**

```
/** * Scenarios for receiving notifications: * 1. When a subscribed user's status changes (including online status and user-defined status), this event will be triggered * 2. If the Friend Status Notification Switch is enabled in the console, even if you haven't actively subscribed, this event will be triggered when a friend's status changes * 3. When the same account logs in to multiple devices, if one device modifies the user-defined status, all devices will receive this event */let onUserStatusUpdated = function(event) {   console.log(event.data);   const userStatusList = event.data;   userStatusList.forEach((item) => {     const { userID, statusType, customStatus } = item;     // userID - User ID     // statusType - User status, the enumerated values and descriptions are as follows:     // TencentCloudChat.TYPES.USER_STATUS_UNKNOWN - Unknown     // TencentCloudChat.TYPES.USER_STATUS_ONLINE - Online     // TencentCloudChat.TYPES.USER_STATUS_OFFLINE - Offline     // TencentCloudChat.TYPES.USER_STATUS_UNLOGINED - Not Logged In     // customStatus - User Definition Status   })};chat.on(TencentCloudChat.EVENT.USER_STATUS_UPDATED, onUserStatusUpdated);
```

### Уведомление об изменении статуса на нескольких платформах, синхронизация между несколькими экземплярами

Если вы включили вход на несколько клиентов или несколько экземпляров на одной платформе (подробнее см. [Вход на несколько устройств или несколько экземпляров на одной платформе](https://trtc.io/document/33515?platform=javascript&product=chat&menulabel=sdk)), то одна учетная запись может быть авторизована на разных устройствах. Когда определяемый пользователем статус пользователя, авторизованного на одном устройстве, изменяется, фоновый сервис Chat отправит уведомление об изменении статуса и на другие авторизованные устройства.

## Часто задаваемые вопросы

### При вызове API подписки/отписки интерфейс выдает код ошибки «72001»

Код ошибки 72001 указывает на то, что соответствующая функция не была включена в консоли. Пожалуйста, войдите в [консоль Chat](https://console.trtc.io/) и включите соответствующий переключатель функции.

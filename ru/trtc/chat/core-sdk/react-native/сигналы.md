# Сигнализация

## Прослушивание событий сигнализации.

> **Примечание:** Пожалуйста, вызовите этот интерфейс для прослушивания событий перед вызовом интерфейса `login`, чтобы избежать пропуска событий, отправляемых SDK.

##### **API**

```
chat.addSignalingListener(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| eventName | String | Имя события. Все имена событий хранятся в переменной `TencentCloudChat.TSignaling`. Если вам нужно их просмотреть, вы можете использовать `console.log(TencentCloudChat.TSignaling)` для отображения всех событий. |
| handler | Function | Метод для обработки событий; когда событие срабатывает, этот обработчик будет вызван для обработки. |
| context | any | Ожидаемый контекст, в котором выполняется обработчик. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let onNewInvitationReceived = function(event) {};chat.addSignalingListener(TencentCloudChat.TSignaling.NEW_INVITATION_RECEIVED, onNewInvitationReceived);
```

## Пригласить пользователя

##### **API**

```
chat.invite(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| userID | String | ID пользователя |
| data | String | пользовательские данные |
| timeout | Number | Период тайм-аута в секундах. Если установлено значение 0, SDK не будет выполнять обнаружение тайм-аута и не будет запускать обратный вызов `onInvitationTimeout`. |
| onlineUserOnly | Boolean | Флаг, указывающий, отправляется ли сообщение только онлайн-пользователям; значение по умолчанию — `false`. Если установлено значение `true`, сообщение не будет перемещаться в историю, не будет считаться непрочитанным, и не будет отправлено в автономном режиме получателю. Кроме того, операция приглашения не будет генерировать исторические сообщения (последующие операции `cancel`, `accept`, `reject`, timeout для этого приглашения также не будут генерировать исторические сообщения). |
| offlinePushInfo | Object | Конфигурация автономной отправки. |

`offlinePushInfo` описывается следующим образом:

| Имя | Тип | Описание |
| --- | --- | --- |
| disablePush | Boolean | `true`: автономная отправка отключена; `false` (по умолчанию): автономная отправка включена. |
| disableVoipPush | Boolean | `true` для отключения push VoIP (по умолчанию); `false` для включения push VoIP (включение push VoIP требует одновременного включения автономной отправки) |
| title | String | Заголовок автономной отправки. Этот параметр используется как для iOS, так и для Android. |
| description | String | Содержимое автономной отправки. Этот параметр переопределит текст отображения автономной отправки экземпляра сообщения. Если отправленное сообщение является пользовательским сообщением, этот параметр переопределит `message.payload.description`. Если оба параметра `description` и `message.payload.description` остаются пустыми, получатель не сможет получить уведомление об автономной отправке пользовательского сообщения. |
| extension | String | Содержимое, передаваемое через автономную отправку. |
| androidInfo | Object | Конфигурация Android push. |
| apnsInfo | Object | Конфигурация iOS push. |

`androidInfo` описывается следующим образом:

| Имя | Тип | Описание |
| --- | --- | --- |
| sound | String | Параметры звука автономной отправки Android. Поддерживает только Huawei, Xiaomi и Google. Для телефонов Xiaomi с Android 8.0 и выше необходимо установить androidInfo.XiaoMiChannelID. Для телефонов Google с FCM push на системах Android 8.0 и выше для установки звуковых подсказок необходимо установить androidInfo.FCMChannelID. Пользовательские рингтоны требуют установки пути к звуковому файлу. Например: androidInfo.sound = "shake.xxx" соответствует звуковому файлу "nativeResources/android/res/raw/shake.xxx" в uniapp. Соответствует звуковому файлу "/res/raw/shake.xxx" в нативном приложении. |
| XiaoMiChannelID | String | Поле адаптации категории уведомлений (Channel) для телефонов Xiaomi с MIUI 10 и выше. |
| OPPOChannelID | String | Поле адаптации NotificationChannel для телефонов OPPO с Android 8.0 и выше. |
| FCMChannelID | String | Поле канала уведомлений для телефонов Google с Android 8.0 и выше. |
| VIVOClassification | Number | Классификация push-сообщений телефонов VIVO, 0: оперативное сообщение, 1: системное сообщение, по умолчанию 1. |
| VIVOCategory | String | Используется для идентификации типа сообщения на телефонах VIVO. |
| HuaWeiCategory | String | Используется для идентификации типа сообщения на телефонах Huawei. |
| HuaWeiImage | String | URL изображения уведомления на панели push Huawei. Используемый протокол должен быть HTTPS, пример значения: https://example.com/image.png. Файл изображения должен быть меньше 512KB с рекомендуемым размером 40dp x 40dp и скругленными углами 8dp. Изображения, превышающие рекомендуемые спецификации, могут испытать сжатие или неполное отображение. Рекомендуемые форматы изображений — JPG/JPEG/PNG. |
| HonorImage | String | URL изображения уведомления на панели push Honor. Используемый протокол должен быть HTTPS, пример значения: https://example.com/image.png. Размер файла значка должен быть меньше 100KB с рекомендуемым размером спецификации 160px x 160px и скругленными углами 32px. Значки, превышающие рекомендуемые спецификации, могут испытать сжатие или неполное отображение. |
| GoogleImage | String | URL изображения уведомления на панели push Google. Используемый протокол должен быть HTTPS, пример значения: https://example.com/image.png. Размер файла значка должен быть меньше 1 MB, значки, превышающие размер спецификации, могут испытать сжатие или неполное отображение. |

`apnsInfo` описывается следующим образом:

| Имя | Тип | Описание |
| --- | --- | --- |
| sound | String | Параметры звука автономной отправки iOS. Когда `apnsInfo.sound = TencentCloudChat.TYPES.IOS_OFFLINE_PUSH_NO_SOUND`, это означает, что при получении звук не будет воспроизведен. Когда `apnsInfo.sound = TencentCloudChat.TYPES.IOS_OFFLINE_PUSH_DEFAULT_SOUND`, это означает, что при получении будет воспроизведен системный звук. Пользовательские рингтоны требуют установки пути к звуковому файлу. Например: apnsInfo.sound = "shake.xxx" соответствует файлу "nativeResources/ios/Resources/shake.xxx" в uniapp, uniapp должен быть официальным пакетом. Соответствует звуковому файлу "shake.xxx" в нативном проекте Xcode. |
| ignoreIOSBadge | Boolean | Игнорировать счет значков при автономной отправке (действует только для iOS), значение по умолчанию — `false`. Если установлено значение `true`, на стороне iOS получателя это сообщение не увеличит количество непрочитанных на значке приложения. |
| disableVoipPush | Boolean | `true` для отключения push VoIP (по умолчанию); `false` для включения push VoIP (включение push VoIP требует одновременного включения автономной отправки) |
| image | String | Определяет адрес изображения уведомления на панели APNs. Когда клиент получает это поле, он может отобразить изображение во всплывающем окне, загрузив ресурс изображения. Используемый протокол должен быть HTTPS, пример значения: https://example.com/image.png Поддерживает JPEG, GIF, PNG с размером не более 10 MB. Атрибут mutable-content должен быть включен в консоли IM для поддержки функций iOS 10 Service Extension. Ключевое значение ресурса — "image". |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// пригласить пользователяlet promise = chat.invite({  userID: 'user1',  data: ''});promise.then(function(imResponse) { console.warn('invite success:', imResponse)}).catch(function(imError) {  console.warn('invite error:', imError);});
```

```
// Пригласить пользователя с установкой параметра для невключения обновления unreadCount и lastMessage беседы.let promise = chat.invite({  userID: 'user1',  data: JSON.stringify({ excludeFromHistoryMessage: true }),});promise.then(function(imResponse) { console.warn('invite success:', imResponse)}).catch(function(imError) {  console.warn('invite error:', imError);});
```

```
// Пригласить пользователя и установить тайм-аут 30 секунд. Если в течение 30 секунд не будет никаких операций// отправитель приглашения получит событие TencentCloudChat.TSignaling.INVITATION_TIMEOUT.let promise = chat.invite({  userID: 'user1',  data: '',  timeout: 30});promise.then(function(imResponse) { console.warn('invite success:', imResponse)}).catch(function(imError) {  console.warn('invite error:', imError);});
```

```
// Пригласить пользователя, установить тайм-аут 30 секунд и связанная сигнализация не будет сохранена в роуминге// и не будет отправлена в автономном режиме.let promise = chat.invite({  userID: 'user1',  data: '',  timeout: 30,  onlineUserOnly: true});promise.then(function(imResponse) {  console.warn('invite success:', imResponse)}).catch(function(imError) {  console.warn('invite error:', imError);});
```

## Пригласить определенных пользователей в группу.

##### **API**

```
chat.inviteInGroup(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| groupID | String | ID группы |
| inviteeList | Array.<String> | Список приглашенных |
| data | String | пользовательские данные |
| timeout | Number | Период тайм-аута в секундах. Если установлено значение 0, SDK не будет выполнять обнаружение тайм-аута и не будет запускать обратный вызов `onInvitationTimeout`. |
| onlineUserOnly | Boolean | Флаг, указывающий, отправляется ли сообщение только онлайн-пользователям; значение по умолчанию — `false`. Если установлено значение `true`, сообщение не будет перемещаться в историю, не будет считаться непрочитанным, и не будет отправлено в автономном режиме получателю. Кроме того, операция приглашения не будет генерировать исторические сообщения (последующие операции `cancel`, `accept`, `reject`, timeout для этого приглашения также не будут генерировать исторические сообщения). |
| offlinePushInfo | Object | Конфигурация автономной отправки. |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
// Пригласить определенных пользователей в группу.let promise = chat.inviteInGroup({  groupID: 'AV1',  inviteeList: ['user1', 'user2'],  data: '',});promise.then(function(imResponse) { console.warn('inviteInGroup success:', imResponse)}).catch(function(imError) {  console.warn('inviteInGroup error:', imError);});
```

## Получить информацию о сигнализации.

##### **API**

```
chat.getSignalingInfo(message);
```

##### **Параметры**

| Имя | Тип | Описание |
| --- | --- | --- |
| message | Message | экземпляр сообщения |

##### **Возвращаемое значение**

`Message | Null`

##### **Примеры**

```
// Когда сигнализация равна null, это указывает на то, что сообщение является обычным сообщением, а не сигнальным сообщением.let signaling = chat.getSignalingInfo(message);console.log('signaling info', signaling);
```

## Изменить сигнализацию приглашения.

> **Примечание:** Поддерживает изменение `data` сигнализации приглашения. Когда `onlineUserOnly = true` сигнализации приглашения, изменения не поддерживаются.

##### **API**

```
chat.modifyInvitation(options);
```

##### **Параметры**

Параметр `options` имеет тип `Object` и содержит следующие значения атрибутов:

| Имя | Тип | Описание |
| --- | --- | --- |
| inviteID | String | ID приглашения |
| data | String | пользовательские данные |

##### **Возвращаемое значение**

`Promise`

##### **Примеры**

```
let promise = chat.modifyInvitation({  inviteID:'38897dbf-ecd4-4b59-a132-bc31529a2b18',  data: 'xxxx'});promise.then(function(imResponse) { console.log('modifyInvitation OK', imResponse);}).catch(function(error) {  console.warn('modifyInvitation failed:', error);});
```

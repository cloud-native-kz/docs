# Обзор беседы

## Беседа

Когда пользователь создает индивидуальный чат или групповой чат, создается беседа.
В Tencent Cloud IM SDK класс беседы — это `ConvInfo`. Вы можете использовать API класса управления беседами для отображения/обновления списка бесед, обновления количества непрочитанных сообщений, закрепления беседы в верхней части, создания черновика беседы и отключения уведомлений о сообщениях.

## Класс беседы

Класс беседы — это `ConvInfo` ([Подробности](https://comm.qq.com/im/doc/unity/en/types/ConvAttributes/ConvInfo.html)), которая определяет следующее содержание:

| Атрибут | Определение | Описание |
| --- | --- | --- |
| conv_type | Тип беседы | См. определение `TIMConvType`, которое может быть `C2C` или `Group`. |
| conv_id | Уникальный идентификатор беседы | Формат: `c2c_userID` для индивидуального чата или `group_groupID` для группового чата. |
| conv_active_time | Время активации беседы |  |
| conv_is_has_lastmsg | Наличие последнего сообщения в беседе |  |
| conv_show_name | Отображаемое имя беседы | Имя групповой беседы отображается в следующем порядке приоритета: имя группы > ID группы; Имя индивидуальной беседы отображается в следующем порядке приоритета: замечание о друге получателя сообщения > никнейм получателя сообщения > `userID` получателя сообщения. |
| conv_unread_num | Количество непрочитанных сообщений в беседе | Недействительно и по умолчанию равно `0` для аудио-видео группы (AVChatRoom). |
| conv_recv_opt | Параметр получения сообщения | См. определение `TIMReceiveMessageOpt`. |
| conv_last_msg | Последнее сообщение в беседе | Подробные инструкции см. в разделе [ConvGetConvInfo(List, ValueCallback)](https://comm.qq.com/im/doc/unity/en/api/ConvApi/ConvGetConvInfo.html). |
| conv_group_at_info_array | Список информации об @ в групповой беседе | Обычно используется для отображения уведомлений «кто-то упомянул меня» и «@все». |
| conv_draft | Информация о черновике | Информация о черновике может быть установлена путем вызова API `ConvSetDraft`. Дополнительную информацию см. в разделе [ConvSetDraft(String, TIMConvType, DraftParam)](https://comm.qq.com/im/doc/unity/en/api/ConvApi/ConvSetDraft.html). |
| conv_is_has_draft | Наличие черновика беседы |  |
| conv_is_pinned | Закрепление беседы в верхней части | Подробные инструкции см. в разделе [ConvPinConversation](https://comm.qq.com/im/doc/unity/en/api/ConvApi/ConvPinConversation.html). |

## Политика хранения бесед

Ограничений на количество локально хранящихся бесед нет.

В облаке можно хранить до 100 бесед. Чтобы увеличить это ограничение, произведите обновление до редакции Pro, Pro Plus или Enterprise. Затем вы можете установить ограничение на 500 в консоли.

В облаке можно хранить до 100 бесед. Чтобы увеличить это ограничение, произведите обновление до редакции Pro, Pro Plus или Enterprise. Затем вы можете установить ограничение на 500 в консоли:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/f9a9ba91c4ca11eebedb525400461a83.png)

Если информация о беседе не обновлялась долгое время, беседу можно хранить в облаке в течение семи дней максимум. Чтобы продлить период хранения беседы, [свяжитесь с нами](https://console.tencentcloud.com/workorder/category) для подачи заявки.

Локально хранящиеся беседы не всегда совпадают с теми, что хранятся в облаке. Если пользователь не вызывает API `ConvDelete` для удаления локальных бесед, беседы будут всегда существовать. Однако в облаке можно хранить до 100 бесед, и если информация о беседе не обновлялась долгое время, беседу можно хранить в облаке в течение семи дней максимум. Поэтому локальные беседы, отображаемые на разных терминалах, могут отличаться.

# Список разговоров

## Обзор

После того, как пользователь входит в приложение, можно отобразить список последних разговоров, чтобы облегчить поиск целевого разговора.

Возможности списка разговоров включают получение списка разговоров и обработку обновлений списка разговоров.
В этом разделе описано, как реализовать такие функции.

## Получение списка разговоров

Вызовите API `ConvGetConvList` ([подробности](https://comm.qq.com/im/doc/unity/zh/api/ConvApi/ConvGetConvList.html)), чтобы получить список разговоров. Этот API получает локально кэшированные разговоры. Если какой-либо разговор на сервере обновится, SDK автоматически синхронизирует обновление и уведомит вас через обратный вызов `TIMConvEventCallback`.

Разговоры пользователя возвращаются в списке, который содержит объекты `ConvInfo`.

Пример кода:

```
TIMResult res = TencentIMSDK.ConvGetConvList((int code, string desc, List<ConvInfo> info_list, string user_data)=>{ // Process the async logic});
```

## Обновление списка разговоров

После успешного входа Chat SDK, или когда пользователь выходит в сеть, или после восстановления соединения, прерванного на время, Chat SDK автоматически обновит список разговоров.
Вы можете получить обновленный список разговоров, выполнив следующие действия:

1. Добавьте прослушиватель разговоров.
2. Получите и обработайте изменения разговоров.
3. Удалите прослушиватель разговоров. Этот шаг является необязательным и может быть выполнен по мере необходимости.

### Добавление прослушивателя разговоров

Вызовите API `SetConvEventCallback` ([подробности](https://comm.qq.com/im/doc/unity/zh/api/SDKRegisteringCallback/SetConvEventCallback.html)), чтобы добавить прослушиватель разговоров для получения событий изменения разговоров.

Пример кода:

```
TencentIMSDK.SetConvEventCallback((TIMConvEvent conv_event, List<ConvInfo> conv_list, string user_data)=>{ // Process the callback logic});
```

Вы можете прослушивать событие в `TIMConvEvent` ([подробности](https://comm.qq.com/im/doc/unity/zh/enums/TIMConvEvent.html)), чтобы получить уведомление об изменении списка разговоров.

В настоящее время Chat SDK поддерживает следующие события изменения разговоров:

| Событие | Описание | Рекомендация |
| --- | --- | --- |
| kTIMConvEvent_Add | Добавлен новый разговор. | Пересортируйте разговоры, когда пользователь получает личное сообщение от нового коллеги или приглашается в новую группу. |
| kTIMConvEvent_Del | Разговор был удален. | Вызовите это событие, когда пользователь удаляет разговор. |
| kTIMConvEvent_Update | Разговор был обновлен. | Вызовите это событие, когда изменяется количество непрочитанных сообщений в разговоре или получается новое сообщение. |
| kTIMConvEvent_Start | Разговор был начат. | - |
| kTIMConvEvent_Finish | Разговор был завершен. | - |

### Удаление прослушивателя разговоров

Вызовите API `RemoveConvEventCallback` ([подробности](https://comm.qq.com/im/doc/unity/zh/api/SDKRegisteringCallback/RemoveConvEventCallback.html)), чтобы удалить указанный прослушиватель разговоров и перестать получать события изменения разговоров. Если передать `null`, будут удалены все прослушиватели разговоров.
Этот шаг является необязательным и может быть выполнен по мере необходимости.

Пример кода:

```
TencentIMSDK.RemoveConvEventCallback();
```

### Отправка сообщения без обновления conv_last_msg

В интерфейсе списка разговоров обычно необходимо отображать предпросмотр и время отправки последнего сообщения в каждом разговоре. В этом случае вы можете использовать `conv_last_msg` из `ConvInfo` в качестве источника данных для реализации. Однако в некоторых случаях, если вы не хотите, чтобы некоторые сообщения (например, системные подсказки) отображались как последнее сообщение в разговоре, вы можете установить `message_excluded_from_last_message` в значение `false`/`no` при вызове `MsgSendMessage`.

Инструкции по отправке сообщения см. [здесь](https://comm.qq.com/im/doc/unity/zh/api/MessageApi/MsgSendMessage.html).

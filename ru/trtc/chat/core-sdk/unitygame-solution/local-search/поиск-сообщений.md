# Поиск сообщений

## Описание функции

Поиск возможен только в локальных сообщениях, например в полученных сообщениях или исторических сообщениях, полученных после вызова API.

> **Примечание:** Эта функция поддерживается только в редакции Ultimate.

## Класс поиска сообщений

### Класс параметров поиска сообщений

Класс параметров поиска сообщений — это `MessageSearchParam` ([Подробности](https://comm.qq.com/im/doc/unity/en/types/MessageAttributes/MessageSearchParam.html)). При поиске сообщений SDK будет выполнять различную логику поиска на основе параметров объекта.

Параметры `MessageSearchParam` описаны ниже:

| Параметр | Определение | Описание |
| --- | --- | --- |
| msg_search_param_keyword_array | Список ключевых слов | Может содержать до пяти ключевых слов. Когда отправитель сообщения и тип сообщения не указаны, список ключевых слов должен быть установлен; в остальных случаях его можно оставить пустым. |
| msg_search_param_keyword_list_match_type | Тип сопоставления списка ключевых слов | Можно установить значение `OR` или `AND`. Допустимые значения: TIMKeywordListMatchType_Or (по умолчанию), TIMKeywordListMatchType_And. |
| msg_search_param_send_indentifier_array | `userID`, который отправил сообщения | Может содержать до пяти ID пользователей. |
| msg_search_param_message_type_array | Набор типов сообщений для поиска | Если оставить пусто, будут выполнен поиск сообщений всех поддерживаемых типов (исключая `FaceElem` и `GroupTipsElem`). Дополнительные сведения о значениях других типов см. в `TIMElemType` ([Подробности](https://comm.qq.com/im/doc/unity/en/enums/TIMElemType.html)). |
| msg_search_param_conv_id | Поиск во всех диалогах или в указанном диалоге | Если не оставлено пусто, будет выполнен поиск в указанном диалоге. |
| msg_search_param_search_time_position | Время начала поиска | По умолчанию `0`, что указывает на поиск с текущей точки времени. Это временная метка UTC в секундах. |
| msg_search_param_search_time_period | Прошедший период времени, начиная со времени начала | По умолчанию `0`, что указывает на отсутствие ограничения по времени. Указывается в секундах. `24x60x60` представляет прошедший день. |
| msg_search_param_page_index | Номер страницы | Используется для отображения результатов поиска по страницам. `0` означает первую страницу. |
| msg_search_param_page_size | Количество элементов результата на странице | Используется для отображения результатов поиска по страницам. Установите значение `0`, если вы не хотите постраничное отображение. Если одновременно извлечь слишком много элементов результата, может возникнуть проблема с производительностью. |

### Класс результатов поиска сообщений

Класс результатов поиска сообщений — это `MessageSearchResult` ([Подробности](https://comm.qq.com/im/doc/unity/en/types/MessageAttributes/MessageSearchResult.html)). Параметры описаны ниже:

| Параметр | Определение | Описание |
| --- | --- | --- |
| msg_search_result_total_count | Общее количество элементов результатов поиска | Если выполняется поиск в указанном диалоге, будет возвращено **общее количество сообщений**, соответствующих критериям поиска. Если выполняется поиск во всех диалогах, будет возвращено **общее количество диалогов**, которым принадлежат сообщения, соответствующие критериям поиска. |
| msg_search_result_item_array | Тип сопоставления списка ключевых слов | Если выполняется поиск в указанном диалоге, в возвращаемый список результатов будут включены только элементы результатов в этом диалоге. Если выполняется поиск во всех диалогах, сообщения, соответствующие критериям поиска, будут сгруппированы по ID диалога и возвращены по страницам. |

Здесь `msg_search_result_item_array` — это список, содержащий объекты `MessageSearchResultItem` ([Подробности](https://comm.qq.com/im/doc/unity/en/types/MessageAttributes/MessageSearchResultItem.html)). Параметры описаны ниже:

| Параметр | Определение | Описание |
| --- | --- | --- |
| msg_search_result_item_conv_id | ID диалога | — |
| msg_search_result_item_total_message_count | Количество сообщений | Количество сообщений, соответствующих критериям поиска и найденных в текущем диалоге. |
| messageList | Список сообщений, соответствующих критериям поиска | Если выполняется поиск в указанном диалоге, `messageList` — это список сообщений в диалоге, соответствующих критериям поиска. Если выполняется поиск во всех диалогах, `messageList` может отображать один из следующих результатов: Если количество совпавших сообщений в диалоге больше 1, `messageList` пусто, и вы можете отобразить на пользовательском интерфейсе "{messageCount} соответствующих записей". Если количество совпавших сообщений в диалоге равно 1, `messageList` — это совпавшее сообщение, и вы можете отобразить его на пользовательском интерфейсе и выделить совпавшее ключевое слово. |
| msg_search_result_item_conv_type | Тип диалога | `TIMConvType` ([Подробности](https://comm.qq.com/im/doc/unity/en/enums/TIMConvType.html)) |

## Поиск сообщений во всех диалогах

Когда пользователь вводит ключевое слово в поле поиска для поиска сообщений, вы можете вызвать `MsgSearchLocalMessages` ([Подробности](https://comm.qq.com/im/doc/unity/en/api/MessageApi/MsgSearchLocalMessages.html)) для поиска локальных сообщений SDK IM.

Если вы хотите выполнить поиск во всех диалогах, вам нужно только оставить `msg_search_param_conv_id` в `MessageSearchParam` пусто (`null`) или не указывать его.

Пример кода:

```
// Search for local messages by keywordMessageSearchParam param = new MessageSearchParam{  msg_search_param_keyword_array = new List<string>  {    "Keyword 1"  },  msg_search_param_page_index = 0,  msg_search_param_page_size = 10,  msg_search_param_conv_type = TIMConvType.kTIMConv_C2C};TIMResult res = TencentIMSDK.MsgSearchLocalMessages(param, (int code, string desc, MessageSearchResult result, string user_data)=>{ // Process the async logic});
```

## Поиск сообщений в указанном диалоге

Когда пользователь вводит ключевое слово в поле поиска для поиска сообщений, вы можете вызвать `MsgSearchLocalMessages` ([Подробности](https://comm.qq.com/im/doc/unity/en/api/MessageApi/MsgSearchLocalMessages.html)) для поиска локальных сообщений SDK IM.

Пример кода:

```
// Search for local messages by conversation ID and keywordMessageSearchParam param = new MessageSearchParam{  msg_search_param_keyword_array = new List<string>  {    "Keyword 1"  },  msg_search_param_page_index = 0,  msg_search_param_page_size = 10,  msg_search_param_conv_id = "group_id"};TIMResult res = TencentIMSDK.MsgSearchLocalMessages(param, (int code, string desc, MessageSearchResult result, string user_data)=>{ // Process the async logic});
```

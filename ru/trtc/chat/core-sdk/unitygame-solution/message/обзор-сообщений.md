# Обзор сообщений

## Класс сообщения

В SDK Tencent Cloud Chat класс сообщения — это `Message` ([подробности](https://comm.qq.com/im/doc/unity/zh/types/MessageAttributes/Message.html)), который часто используется для отправки и получения сообщений.

## Значения полей сообщения

| Поле | Описание |
| --- | --- |
| message_elem_array | Список элементов в сообщении |
| message_conv_id | ID беседы сообщения |
| message_conv_type | Тип беседы сообщения |
| message_sender | Отправитель сообщения |
| message_priority | Приоритет сообщения |
| message_client_time | Время клиента |
| message_server_time | Время сервера |
| message_is_from_self | Является ли отправитель сообщения текущим пользователем |
| message_platform | Платформа, с которой отправляется сообщение |
| message_is_read | Прочитано ли сообщение |
| message_is_online_msg | Является ли сообщение сетевым сообщением. `false` (по умолчанию): обычное сообщение; `true`: сетевое сообщение |
| message_is_peer_read | Прочитано ли сообщение получателем |
| message_need_read_receipt | Требует ли сообщение подтверждение прочтения (для использования этой функции необходимо приобрести издание Pro, Pro Plus или Enterprise Edition в версии 6.1 или позже). Для использования этой функции с групповыми сообщениями перейдите в консоль Chat и сначала установите целевой тип группы. |
| message_status | Текущий статус сообщения |
| message_target_group_member_array | Указывает целевых получателей группового сообщения (целевое сообщение). Это поле не поддерживается для групповых @ сообщений, сообщений сообщества или аудио-видео групповых сообщений. После установки этого поля сообщение не будет включено в количество непрочитанных сообщений беседы. |
| message_unique_id | Уникальный ID сообщения. Рекомендуется использовать `kTIMMsgMsgId`. |
| message_msg_id | Уникальный ID сообщения. |
| message_rand | Случайный код сообщения |
| message_has_sent_receipt | Отправлено ли подтверждение прочтения (действительно только для групповых сообщений) |
| message_group_receipt_read_count | Это внутреннее поле и его использование не рекомендуется. Рекомендуется вызвать `TIMMsgGetMessageReadReceipts` для получения подтверждения прочтения группового сообщения. |
| message_group_receipt_unread_count | Это внутреннее поле и его использование не рекомендуется. Рекомендуется вызвать `TIMMsgGetMessageReadReceipts` для получения подтверждения прочтения группового сообщения. |
| message_seq | Порядковый номер сообщения |
| message_custom_int | Пользовательское целое число, которое сохраняется локально, не отправляется получателю и становится недействительным после удаления и переустановки приложения |
| message_custom_str | Пользовательское поле данных, которое сохраняется локально, не отправляется получателю и становится недействительным после удаления и переустановки приложения |
| message_cloud_custom_str | Пользовательские данные сообщения, которые сохраняются в облаке, отправляются получателю и могут быть получены после удаления и переустановки приложения |
| message_is_excluded_from_unread_count | Исключено ли сообщение из количества непрочитанных сообщений беседы. `NO` (по умолчанию): включено в количество непрочитанных сообщений беседы; `YES`: исключено из количества непрочитанных сообщений беседы |
| message_is_forward_message | Является ли сообщение переадресованным сообщением |
| message_group_at_user_array | Список UserID пользователей, которые должны быть упомянуты (@). Для упоминания всех пользователей передайте `kImSDK_MesssageAtALL`. |
| message_sender_profile | Профиль пользователя отправителя сообщения |
| message_sender_group_member_info | Информация об отправителе сообщения в группе. Этот параметр действителен только в групповой беседе. В настоящее время можно получить только поля `kTIMGroupMemberInfoIdentifier` и `kTIMGroupMemberInfoNameCard`. Рекомендуется получать другие поля через API `TIMGroupGetMemberInfoList`. |
| message_offlie_push_config | Параметры автономной доставки сообщения |
| message_excluded_from_last_message | Используется ли сообщение как последнее сообщение (`lastMessage`) в беседе. `true`: нет; `false`: да |

## Типы сообщений

Сообщения Chat классифицируются по назначению сообщения на два типа: личные сообщения (также называемые сообщениями C2C) и групповые сообщения.

| Тип сообщения | Ключевое слово API | Описание |
| --- | --- | --- |
| Личное сообщение | C2CMessage | Также называется сообщением C2C. При отправке личного сообщения необходимо указать UserID получателя сообщения, и только получатель может получить это сообщение. |
| Групповое сообщение | GroupMessage | При отправке группового сообщения необходимо указать groupID целевой группы, и все пользователи в этой группе могут получить это сообщение. |

Сообщения Tencent Cloud Chat также можно классифицировать по содержанию на текстовые сообщения, сообщения с изображениями, видеосообщения, голосовые сообщения, сообщения с файлами, сообщения о местоположении, объединённые сообщения и подсказки группы.

| Тип сообщения | Ключевое слово API | Описание |
| --- | --- | --- |
| Текстовое сообщение | kTIMElem_Text | Обычное текстовое сообщение |
| Пользовательское сообщение | kTIMElem_Custom | Это раздел двоичного буфера, часто используется для передачи пользовательских сигналов в приложении. |
| Сообщение с изображением | kTIMElem_Image | Когда SDK Chat отправляет исходное изображение, он автоматически генерирует два изображения разных размеров. Эти три изображения называются исходным изображением, большим изображением и миниатюрой. |
| Видеосообщение | kTIMElem_Video | Видеосообщение содержит видеофайл и миниатюру. |
| Голосовое сообщение | kTIMElem_Sound | Поддерживает отображение красной точки перед воспроизведением голосового сообщения. |
| Сообщение с файлом | kTIMElem_File | Сообщение с файлом не может превышать 100 МБ. |
| Сообщение о местоположении | kTIMElem_Location | Сообщение о местоположении содержит три поля: описание местоположения, долготу и широту. |
| Объединённое сообщение | kTIMElem_Merge | Можно объединить до 300 сообщений. |
| Подсказка группы | kTIMElem_GroupTips | Подсказка группы часто используется для передачи системного уведомления в группе, например уведомления о присоединении или отходе участника от группы, изменении описания группы или изменении профиля участника группы. |

## Политика хранения сообщений

Сообщения Tencent Cloud Chat можно классифицировать по политике хранения сообщений на два типа: сетевые сообщения и обычные сообщения.
Сетевые сообщения могут получить только онлайн пользователи и не будут отправлены при автономном режиме. Обычные сообщения могут получить пользователи независимо от их статуса подключения.

Сетевые сообщения доставляются в реальном времени и не сохраняются на сервере или в SDK. Поэтому они не могут быть получены из истории сообщений после смены устройства или удаления и переустановки приложения.

> **Примечание:** Все сообщения в аудио-видео группе являются сетевыми сообщениями. Сообщения, отправленные всем пользователям, являются сетевыми сообщениями.

Обычные сообщения хранятся в SDK и на сервере. По умолчанию они хранятся на роуминг-сервере в течение семи дней. Если вам требуется более длительный период хранения, необходимо приобрести дополнительную услугу. Дополнительная информация о содержании услуги и выставлении счетов см. в разделе [Ценообразование](https://intl.cloud.tencent.com/document/product/1047/34350).

Эти сообщения, например все типы обычных сообщений, могут быть получены из истории сообщений после смены устройства или удаления и переустановки приложения.

#   Отключение уведомлений

## Описание функции

Вы можете установить опцию получения сообщений для **личного или группового чата**, чтобы реализовать функцию отключения уведомлений.

SDK Chat поддерживает следующие три опции получения сообщений, определённые в `TIMReceiveMessageOpt`:

| Опция получения сообщений | Описание функции |
| --- | --- |
| TIMReceiveMessageOpt.kTIMRecvMsgOpt_Receive | Сообщения будут получены, когда пользователь онлайн, и будут получены автономные push-уведомления, когда пользователь оффлайн. |
| TIMReceiveMessageOpt.kTIMRecvMsgOpt_Not_Receive | Сообщения не будут получены независимо от того, онлайн или оффлайн пользователь. |
| TIMReceiveMessageOpt.kTIMRecvMsgOpt_Not_Notify | Сообщения будут получены, когда пользователь онлайн, и автономные push-уведомления не будут получены, когда пользователь оффлайн. |

Различные опции `TIMReceiveMessageOpt` могут использоваться для реализации отключения групповых уведомлений:
**Сообщения не будут получены.**
После установки опции получения сообщений на `kTIMRecvMsgOpt_Not_Receive` личные и групповые сообщения не будут получены, и список диалогов не будет обновлён.

**Сообщения будут получены, но не будут уведомлены пользователю, и на пользовательском интерфейсе списка диалогов будет отображён значок без счётчика непрочитанных сообщений.**

1. Опция получения сообщений установлена на `kTIMRecvMsgOpt_Not_Notify`.
2. Когда получатель получает личное или групповое сообщение и ему необходимо обновить список диалогов, он может получить счётчик непрочитанных через `total_unread_count` ([Подробнее](https://comm.qq.com/im/doc/unity/en/callback/ConvTotalUnreadMessageCountChangedCallback.html)) в обратном вызове `SetConvTotalUnreadMessageCountChangedCallback` для изменения общего счётчика непрочитанных сообщений диалога.
3. Получатель отображает значок вместо счётчика непрочитанных, идентифицируя опцию получения сообщений как `kTIMRecvMsgOpt_Not_Notify` на основе `conv_recv_opt` ([Подробнее](https://comm.qq.com/im/doc/unity/en/types/ConvAttributes/ConvInfo.html)) в `ConvInfo`.

> **Примечание:** Поскольку этот метод требует функции счётчика непрочитанных, он применяется только к рабочим группам (Work) и публичным группам (Public). Для получения дополнительной информации о типах групп см. раздел "Обзор групп".

## Установка опции получения сообщений для личного чата

Вызовите API `MsgSetC2CReceiveMessageOpt` ([Подробнее](https://comm.qq.com/im/doc/unity/en/api/MessageApi/MsgSetC2CReceiveMessageOpt.html)), чтобы установить опцию получения сообщений для личного чата.
Вы можете использовать параметр `userIDList` для одновременной указания до 30 пользователей.

> **Внимание:** Этот API можно вызывать не более 5 раз в секунду.

Пример кода:

```
// Set not to receive messages no matter whether the user is online or offlineTIMResult res = TencentIMSDK.MsgSetC2CReceiveMessageOpt(user_id_list, TIMReceiveMessageOpt.kTIMRecvMsgOpt_Not_Receive, (int code, string desc, string user_data)=>{ // Process the async logic});
```

## Получение опции получения сообщений для личного чата

Вызовите API `MsgGetC2CReceiveMessageOpt` ([Подробнее](https://comm.qq.com/im/doc/unity/en/api/MessageApi/MsgGetC2CReceiveMessageOpt.html)), чтобы получить опцию получения сообщений для личного чата.

Пример кода:

```
TIMResult res = TencentIMSDK.MsgGetC2CReceiveMessageOpt(user_id_list, (int code, string desc, List<GetC2CRecvMsgOptResult> msg_opts, string user_data)=>{ // Process the async logic});
```

## Установка опции получения сообщений для группового чата

Вызовите API `MsgSetGroupReceiveMessageOpt` ([Подробнее](https://comm.qq.com/im/doc/unity/en/api/MessageApi/MsgSetGroupReceiveMessageOpt.html)), чтобы установить опцию получения сообщений для группового чата.

Пример кода:

```
TIMResult res = TencentIMSDK.MsgSetGroupReceiveMessageOpt(group_id, TIMReceiveMessageOpt.kTIMRecvMsgOpt_Not_Receive, (int code, string desc, string user_data)=>{ // Process the async logic});
```

#  Статус пользователя

## Обзор

Нативный SDK версии 6.3 и выше предоставляет функцию управления статусом пользователя. Каждый пользователь имеет два статуса:

- Общий статус. Он предустановлен в SDK и не может быть изменен.
- Пользовательский статус. Может быть настроен и изменен пользователями. Например, может быть установлен на "Слушаю музыку" или "На звонке".

> **Примечание:** Статус пользователя относится к текущему пользователю, но не к устройству. Если учетная запись одновременно активна на нескольких устройствах, статус не может быть запрошен или установлен.

Общий статус пользователя доступен в следующих трех типах:

- В сети (ONLINE): Текущий пользователь вошел в систему и может получать и отправлять сообщения.
- Не в сети (OFFLINE): Пользователь не вызвал `logout` ([подробнее](https://comm.qq.com/im/doc/unity/zh/api/loginOrlogout/Logout.html)) для выхода, и постоянное соединение разорвано. Как правило, пользователь может получать сообщения оффлайн-уведомлений.
- Не авторизован (UNLOGINED): Пользователь не вошел в систему после регистрации или вызвал `logout` для выхода.

Учитывайте следующее в отношении статуса оффлайн:

1. Учетная запись будет иметь статус оффлайн, если приложение завершено или сетевое соединение разорвано ненормально (например, из-за переключения между 4G/Wi-Fi или слабого сигнала в лифте), когда приложение авторизуется.
2. Учетная запись будет иметь статус оффлайн, если процесс приложения завершен после входа пользователя в приложение и нажатия кнопки Home для перехода в фоновый режим. Учетная запись будет иметь статус онлайн, если процесс приложения остается активным в фоне.
3. Переключение между статусами онлайн и оффлайн зависит от TCP постоянного соединения между Chat SDK и бэкэндом. Если клиент находится в режиме полета, сеть полностью отключена или не поддерживается некоторыми производителями устройств, пакеты TCP FIN или RST могут не быть отправлены, и статус оффлайн не может быть переключен немедленно. Поскольку бэкэнд не может получать пакеты heartbeat, он установит статус текущего пользователя на оффлайн через 400 секунд.

> **Внимание:** **Некоторые из следующих функций** поддерживаются только издением Ultimate. Убедитесь, что вы приобрели его перед использованием. Вам необходимо включить статус пользователя в [консоли Chat](https://console.tencentcloud.com/im) для **некоторых из следующих функций**. Убедитесь, что вы включили это перед использованием.

## Установка пользовательского статуса

Ниже описано, как очистить пользовательский статус:

1. При вызове API `SetSelfStatus` вы можете оставить поле `current_user_status` пустым для очистки статуса.
2. Когда SDK замечает, что текущая учетная запись находится в статусе оффлайн, он автоматически очистит пользовательский статус и запустит уведомление об изменении статуса.

> **Примечание:** Для вызова `SetSelfStatus` вам не требуется обновление до издания Ultimate или включение функции в консоли. Этот API можно вызывать неограниченное количество раз.

Пример кода:

```
    // Set user status    UserStatus status = new UserStatus    {      user_status_identifier = "userID",      user_status_status_type = TIMUserStatusType.kTIMUserStatusType_Online    };    TIMResult res = TencentIMSDK.SetSelfStatus(status, (int code, string desc, string result, string user_data)=>{      // Async result of the user status setting    });
```

## Запрос статуса пользователя

### Запрос собственного статуса пользователя

> **Примечание:** Для того чтобы позволить пользователям запрашивать собственный статус, вам не требуется обновление до издания Ultimate или включение функции в консоли. Этот API можно вызывать неограниченное количество раз.

Пример кода:

```
  // Query user status    TIMResult res = TencentIMSDK.GetUserStatus(new List<string> {"userID1"}, (int code, string desc, List<UserStatus> results, string user_data)=>{      // Async result of the user status query    });
```

### Запрос статуса другого пользователя

> **Примечание:** Для использования этой функции вам необходимо обновиться до издания Ultimate. Подробнее см. [Тарификация](https://intl.cloud.tencent.com/document/product/1047/34350). Для использования этой функции вам необходимо предварительно включить **Установить запрос статуса пользователя и уведомление об изменении статуса** в [консоли Chat](https://console.tencentcloud.com/im); в противном случае при вызове `GetUserStatus` будет сообщена об ошибке.
> ![](https://qcloudimg.tencent-cloud.cn/raw/1adfc878b14e6a6a62de3a7db159b0a6.png)По умолчанию этот API можно вызывать 20 раз каждые пять секунд, а статусы до 500 пользователей можно запрашивать одновременно.

## Подписка на статус пользователя

Функции API:

1. Этот API не поддерживает подписку на статус собственного пользователя, который можно получить в обратном вызове `UserStatusChangedCallback`. Подробнее см. [Уведомление об изменении статуса](#notify).
2. Этот API поддерживает подписку на статус друга, которая будет занимать квоту из 200, упомянутую выше.
  - Для получения изменений статусов всех друзей пользователя вам не нужно вызывать этот API, и вы можете включить автоматические уведомления о статусах друзей в [консоли Chat](https://console.tencentcloud.com/im), после чего уведомление об изменении статуса можно получить в обратном вызове `UserStatusChangedCallback`.
На следующем рисунке показан переключатель для автоматического уведомления о статусах друзей.
![](https://qcloudimg.tencent-cloud.cn/raw/17b70799301407e46b14ca144646eb98.png)
  - Для получения изменений статусов некоторых друзей пользователя вы можете только вызвать `SubscribeUserStatus` для подписки, после чего уведомление об изменении статуса можно получить в обратном вызове `UserStatusChangedCallback`.

> **Примечание:** Для использования этой функции вам необходимо обновиться до издания Ultimate. Подробнее см. [Тарификация](https://intl.cloud.tencent.com/document/product/1047/34350). Для использования этой функции вам необходимо предварительно включить **Установить запрос статуса пользователя и уведомление об изменении статуса** в [консоли Chat](https://console.tencentcloud.com/im); в противном случае при вызове `SubscribeUserStatus` будет сообщена об ошибке.
> ![](https://qcloudimg.tencent-cloud.cn/raw/1adfc878b14e6a6a62de3a7db159b0a6.png)По умолчанию этот API можно вызывать 20 раз каждые пять секунд, а подписаться на статусы до 100 пользователей можно одновременно.

Пример кода:

```
    // Subscribe to the user status    TIMResult res = TencentIMSDK.SubscribeUserStatus(new List<string> {"userID1"}, (int code, string desc, string result, string user_data)=>{      // Async result of the user status subscription    });
```

## Отписка от статуса пользователя

Ограничения использования API для отписки от статуса пользователя согласуются с ограничениями [API для подписки на статус пользователя](#subscribe).

Пример кода:

```
    // Unsubscribe from the user status    TIMResult res = TencentIMSDK.UnsubscribeUserStatus(new List<string> {"userID1"}, (int code, string desc, string result, string user_data)=>{      // Async result of the user status unsubscription    });
```

## Уведомление об изменении статуса

1. Изменение статуса собственного пользователя.
2. Изменение статуса друга.
3. Изменение статуса пользователя, не являющегося другом.

Уведомления об всех этих изменениях статуса могут быть получены обратным вызовом через обратный вызов `UserStatusChangedCallback` ([подробнее](https://comm.qq.com/im/doc/unity/zh/callback/UserStatusChangedCallback.html)), который запускается по-разному для разных типов пользователей.

### Уведомление об изменении статуса собственного пользователя

Если вы вызвали `SetUserStatusChangedCallback` для добавления слушателя SDK, после успешной установки статуса собственного пользователя, когда статус изменяется, будет запущен обратный вызов `UserStatusChangedCallback`, где пользователь может получить последний собственный статус.

### Уведомление об изменении статуса друга

1. Если вы включили автоматические уведомления о статусах друзей в [консоли Chat](https://console.tencentcloud.com/im), когда статус друга пользователя изменяется, обратный вызов `UserStatusChangedCallback` будет автоматически запущен.
2. Если вы не включаете автоматические уведомления о статусах друзей, но все еще хотите получать изменения статусов друзей, вам необходимо вызвать `SubscribeUserStatus` для подписки на статусы друзей. Когда статус друга изменяется, обратный вызов `UserStatusChangedCallback` будет автоматически запущен.
Для ограничений использования `SubscribeUserStatus` см. [Подписка на статус пользователя](#subscribe).
3. Если вы ни не включаете автоматические уведомления о статусах друзей, ни не вызываете `SubscribeUserStatus` для подписки на статусы друзей, изменения в статусах друзей не могут быть получены.

### Изменение статуса пользователя, не являющегося другом

Для получения изменения статуса пользователя, не являющегося другом, вы можете только вызвать `SubscribeUserStatus` для подписки на статус. Когда статус пользователя изменяется, будет запущен обратный вызов `UserStatusChangedCallback`.
Для ограничений использования `SubscribeUserStatus` см. [Подписка на статус пользователя](#subscribe).

Пример кода:

```
    // Set the user status change notification callback    TencentIMSDK.SetUserStatusChangedCallback((List<UserStatus> json_user_status_array, string user_data)=>{      // Process the callback logic    });
```

### Синхронизация уведомлений об изменении статуса на нескольких устройствах

Если вы включили вход с нескольких устройств ([подробнее](https://intl.cloud.tencent.com/document/product/1047/34419)), учетная запись может быть активна на разных устройствах. Когда статус пользователя, вошедшего на одно из устройств, изменяется, бэкэнд отправит уведомление `UserStatusChangedCallback` ([подробнее](https://comm.qq.com/im/doc/unity/zh/callback/UserStatusChangedCallback.html)) на другие устройства с активными сеансами.

## Ограничения API

### Ограничения плана

- Вам не требуется обновление до издания Ultimate для использования API `GetUserStatus` для запроса собственного статуса пользователя.
- Вам необходимо обновиться до издания Ultimate для использования API `GetUserStatus` для запроса статуса другого пользователя.
- Вам необходимо обновиться до издания Ultimate для использования API `SubscribeUserStatus`/`UnsubscribeUserStatus`.

### Ограничение частоты вызовов API

- API `SetSelfStatus` можно вызывать неограниченное количество раз.
- API `GetUserStatus`, используемый для запроса собственного статуса пользователя, можно вызывать неограниченное количество раз.
- По умолчанию API `GetUserStatus` можно вызывать 20 раз каждые пять секунд при использовании для запроса статуса другого пользователя, а статусы до 500 пользователей можно запрашивать одновременно.
- По умолчанию API `SubscribeUserStatus` можно вызывать 20 раз каждые пять секунд, а подписаться на статусы до 100 пользователей можно одновременно.
- По умолчанию API `UnsubscribeUserStatus` можно вызывать 20 раз каждые пять секунд, и можно отписать до 100 пользователей одновременно.

## Часто задаваемые вопросы

### Что делать, если при вызове API подписки/отписки сообщается код ошибки 72001?

Код ошибки 72001 указывает, что функция не включена в консоли. Вам необходимо войти в [консоль Chat](https://console.tencentcloud.com/im) и включить функцию.

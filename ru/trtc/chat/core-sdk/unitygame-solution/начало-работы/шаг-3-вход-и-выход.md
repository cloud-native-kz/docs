# Шаг 3: Вход и выход

## Описание функции

После инициализации Chat SDK необходимо вызвать API входа SDK для аутентификации вашей учётной записи, чтобы получить разрешения на использование функций сообщений, диалогов и других возможностей.

> **Осторожно:** После вызова API входа можно сразу же вызывать только API получения диалогов, остальные API можно вызывать только после успешного входа в SDK. Поэтому **убедитесь, что вы успешно вошли в систему** перед использованием других функций; в противном случае эти функции могут работать неправильно или быть недоступными.

## Вход

При первом входе не нужно регистрироваться, так как вы будете автоматически зарегистрированы при успешном входе.
Вы можете вызвать API `Login` ([Подробнее](https://comm.qq.com/im/doc/unity/en/api/loginOrlogout/Login.html)) для входа.

Ключевые параметры API `Login` следующие:

| Параметр | Описание | Примечания |
| --- | --- | --- |
| user_id | Уникальный идентификатор пользователя | Может содержать до 32 байт букв (a-z и A-Z), цифр (0-9), подчёркиваний (_) и дефисов (-). |
| user_sig | Билет входа | Вычисляется вашим бизнес-сервером для обеспечения безопасности. Для получения дополнительной информации см. [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| callback | Асинхронный обратный вызов | Может быть [NullValueCallback](https://comm.qq.com/im/doc/unity/en/callback/NullValueCallback.html) или [ValueCallback`<String>`](https://comm.qq.com/im/doc/unity/en/callback/ValueCallback.html). |

Вы можете вызвать API `Login` в следующих сценариях:

- Вы используете функции Chat SDK в первый раз после запуска приложения.
- Ваш билет истёк при входе: обратный вызов API `Login` возвращает код ошибки `ERR_USER_SIG_EXPIRED (6206)` или `ERR_SVR_ACCOUNT_USERSIG_EXPIRED (70001)`. В этом случае вам необходимо сгенерировать новый `user_sig` для повторного входа.
- Ваш билет истёк, когда пользователь находится в сети: Вы можете получить обратный вызов `UserSigExpiredCallback` ([Подробнее](https://comm.qq.com/im/doc/unity/en/callback/UserSigExpiredCallback.html)) в то время, когда пользователи находятся в сети. В этом случае вам необходимо сгенерировать новый `user_sig` для повторного входа.
- Пользователь был выведен из системы: Когда пользователь выводится из системы, Chat SDK уведомит вас через обратный вызов `KickedOfflineCallback` ([Подробнее](https://comm.qq.com/im/doc/unity/en/callback/KickedOfflineCallback.html)). В этом случае вы можете уведомить пользователя и вызвать `Login` для повторного входа.

Вам не нужно вызывать API `Login` в следующих сценариях:

- После того как сетевое соединение пользователя разорвалось, а затем восстановилось, вам не нужно вызывать функцию `Login`, так как Chat SDK автоматически перейдёт в режим онлайн.
- Процесс входа выполняется.

> **Осторожно:** 1. После вызова API Chat SDK и успешного входа начнётся подсчёт DAU; поэтому вызывайте API входа по мере необходимости, чтобы избежать высокого значения DAU. 2. Вы не можете одновременно войти в несколько учётных записей Chat SDK одного приложения; в противном случае в сети будет находиться только последняя учётная запись.

Пример кода:

```
public static void Login() {  if (userid == "" || user_sig == "")  {      return;  }  TIMResult res = TencentIMSDK.Login(userid, user_sig, (int code, string desc, string json_param, string user_data)=>{    // Process the login callback logic  });
```

### Получение UserID

После успешного входа вызовите `GetLoginUserID` ([Подробнее](https://comm.qq.com/im/doc/unity/en/api/loginOrlogout/GetLoginUserID.html)) для получения `UserID`.
Если вход не удастся, `UserID` будет пустым.

Пример кода:

```
public static void GetLoginUserID()  {    StringBuilder userId = new StringBuilder(128);    TIMResult res = TencentIMSDK.GetLoginUserID(userId);    Debug.Log(userId.ToString());  }
```

### Получение статуса входа

Вызовите `GetLoginStatus` ([Подробнее](https://comm.qq.com/im/doc/unity/en/api/loginOrlogout/GetLoginStatus.html)) для получения статуса входа. Если пользователь вошёл или входит в систему, не вызывайте API входа часто. IM SDK поддерживает следующие статусы входа:

| Статус входа | Описание |
| --- | --- |
| kTIMLoginStatus_Logined | Вошли в систему |
| kTIMLoginStatus_Logining | Вход в систему |
| kTIMLoginStatus_Logouting | Выход из системы |
| kTIMLoginStatus_UnLogined | Не вошли в систему |

Пример кода:

```
 public static void GetLoginStatus()  {    TIMLoginStatus res = TencentIMSDK.GetLoginStatus();  }
```

## Многоклиентский вход и взаимное вытеснение

Вы можете настроить политики многоклиентского входа для Chat SDK в консоли Tencent Cloud.

Существует несколько политик многоклиентского входа, такие как **пользователь может одновременно находиться в сети на мобильной, настольной платформе и веб-платформе** или **пользователь может одновременно находиться в сети на всех платформах**.
Дополнительную информацию о конфигурации см. в разделе [Конфигурация функций](https://intl.cloud.tencent.com/document/product/1047/34419).

Вы можете настроить **максимальное количество экземпляров входа на пользователя на платформу** для Chat SDK в консоли Tencent Cloud, то есть максимальное количество экземпляров на одной платформе, которые могут одновременно находиться в сети.

Эта функция доступна только для издания Pro, Pro Plus или Enterprise. Пользователь может одновременно находиться в сети на веб-платформе до десяти клиентов или на платформах Android, iPhone, iPad, Windows или macOS — до трёх клиентов (максимальное количество клиентов, которые могут одновременно находиться в сети на Flutter, зависит от результата фактической компиляции).

Дополнительную информацию о конфигурации см. в разделе [Конфигурация функций](https://intl.cloud.tencent.com/document/product/1047/34419).

При вызове API `Login` ([Подробнее](https://comm.qq.com/im/doc/unity/en/api/loginOrlogout/Login.html)) для входа, если превышено ограничение, установленное политикой многоклиентского входа для вашей учётной записи, новый вошедший экземпляр вытеснит более ранний.
Вытесненный экземпляр получит `KickedOfflineCallback` ([Подробнее](https://comm.qq.com/im/doc/unity/en/callback/KickedOfflineCallback.html)).

## Выход

Обычно, если жизненный цикл приложения совпадает с жизненным циклом Chat SDK, нет необходимости выходить перед выходом из приложения.

В специальных случаях, например, если вы используете Chat SDK только после входа на определённый интерфейс и больше не используете его после выхода из интерфейса, вы можете вызвать API `Logout` ([Подробнее](https://comm.qq.com/im/doc/unity/en/api/loginOrlogout/Logout.html)) для выхода из SDK, после чего вы больше не будете получать новые сообщения. Обратите внимание, что в этом случае вам также необходимо вызвать `Uninit` ([Подробнее](https://comm.qq.com/im/doc/unity/en/api/IMSDKInit/Uninit.html)) после выхода для отключения инициализации SDK.

Пример кода:

```
public static void Logout()  {    TIMResult res = TencentIMSDK.Logout((int code, string desc, string json_param, string user_data)=>{    // Process the logout callback logic    });  }
```

## Переключение учётной записи

Вызовите `Login` ([Подробнее](https://comm.qq.com/im/doc/unity/en/api/loginOrlogout/Login.html)) для переключения между учётными записями в приложении.

Например, чтобы переключить вошедшего пользователя с `Alice` на `Bob`, просто вызовите `Login bob`. Вам не нужно явно вызывать `logout alice`, так как эта операция будет обработана автоматически внутри Chat SDK.

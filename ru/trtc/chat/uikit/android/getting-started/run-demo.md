# Запуск демоверсии

Это руководство содержит пошаговые инструкции, которые помогут вам быстро начать работу с демоверсией Chat и изучить функции, включая текстовые, голосовые и видеосообщения. После настройки интерфейс приложения Demo выглядит следующим образом:

## Приложение Demo

Вы можете перейти на страницу [Бесплатная демоверсия](https://trtc.io/document/35076#42b9c204-f927-4326-9af6-370e511773f3), чтобы скачать приложение Chat Demo и ознакомиться со всеми функциями мгновенного обмена сообщениями.

## Предварительные требования

### Включение сервиса

1. Войдите в [Консоль](https://console.trtc.io/). Если у вас уже есть приложение, сохраните `SDKAppID` и `SDKSecretKey` и переходите к [Подготовке окружения](#1c7f5d4b-cacf-4b54-9cbb-0fbf990847f5).
2. Нажмите `Create Application`, введите имя приложения, выберите **Chat** в качестве продукта, выберите регион и нажмите `Create` для добавления приложения.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2b6c10ebb55a11f0b4c35254001c06ec.png)

3. После создания вы сможете просмотреть `SDKAppID` и `SDKSecretKey` приложения на странице обзора консоли. Эта информация потребуется вам для запуска демоверсии.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2b60badeb55a11f09a6b525400454e06.png)

> **Запрещено:** Защищайте ваш `SDKSecretKey`, чтобы предотвратить несанкционированный доступ к вашему приложению.

### Подготовка окружения

Перед началом убедитесь, что вы соответствуете следующим требованиям:

- Android Studio версии 2022.3.1 или позже
- Физическое устройство или эмулятор с Android 5.0 или выше
- Действительное приложение Chat. Обратитесь к разделу «Включение сервиса» выше, чтобы получить или создать приложение.

> **Примечание о совместимости версий:** Для обеспечения стабильной среды сборки строго следуйте официальным требованиям совместимости: Для совместимости между Gradle, Android Gradle Plugin, JDK и Android Studio см. официальную документацию Android: [Примечания о версиях](https://developer.android.com/build/releases/gradle-plugin#updating-gradle) Для сопоставления версий между Kotlin, Android Gradle Plugin и Gradle см. официальную документацию Kotlin: [Совместимость Kotlin-Gradle Plugin](https://kotlinlang.org/docs/gradle-configure-project.html#apply-the-plugin) Рекомендуется выбрать комбинацию версий, соответствующую требованиям вашего проекта, на основе приведенных выше рекомендаций.

## Инструкции

### Получение демоверсии

> **Примечание:** Эта демоверсия использует набор эмодзи TRTC с ограничениями по лицензированию. **Опции для коммерческого использования** **Вариант A: Сохранить наши эмодзи (рекомендуется)** Обновитесь до [плана Chat Pro Plus или Enterprise Plan](https://console.trtc.io/subscription/buy/chat?packType=pro) и используйте наш набор эмодзи без дополнительных затрат. **Вариант B: Использовать свой** Замените эмодзи по умолчанию на свои собственные дизайны или используйте наборы эмодзи с надлежащим коммерческим лицензированием. ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/838c3fd6b88211f0a808525400bf7822.png)

1. Скачайте [проект Android Chat UIKit](https://github.com/TencentCloud/chat-uikit-android).
2. Откройте директорию проекта в терминале и найдите файл `GenerateTestUserSig.java` в пути `Demo/app/src/main/java/com/tencent/qcloud/tim/demo/signature/GenerateTestUserSig.java`.
3. Настройте следующие параметры в файле `GenerateTestUserSig.java`:
  - SDKAPPID: установите значение вашего реального SDKAppID, полученного выше.
  - SECRETKEY: установите значение вашего реального SecretKey.

> **Запрещено:** В этой демоверсии аутентификация выполняется путем настройки `SECRETKEY` в коде клиента. Однако `SECRETKEY` может быть легко извлечено посредством обратного инжиниринга. В случае утечки злоумышленники смогут неправомерно использовать ваши ресурсы Tencent Cloud. Этот подход подходит только для локальных пробных использований демоверсии и отладки функций. Для производственных сред мы настоятельно рекомендуем генерировать UserSig на вашем сервере. Когда вашему приложению потребуется UserSig, оно должно запросить динамический UserSig у вашего бизнес-сервера для аутентификации. Для получения подробной информации см. [Генерирование UserSig на сервере](https://intl.cloud.tencent.com/document/product/1047/34385).

### Сборка и запуск демоверсии

Откройте Android Studio и запустите директорию Demo.

> **Примечание:** При первом импорте проекта Demo зависимости будут загружены автоматически. Дождитесь завершения загрузки и синхронизации перед запуском проекта.

Для запуска на устройстве Android:

1. Подключите устройство Android к компьютеру, включите режим разработчика и отладку по USB на устройстве, при необходимости установите USB режим на передачу файлов.
2. Установите Gradle JDK:
  - В Android Studio выберите **Settings** > **Build, Execution, Deployment** > **Build Tools** > **Gradle**.
  - В раскрывающемся меню **Gradle JDK** выберите **JDK 17** (установите JDK 17, если его нет). Затем выполните **File > Sync Project with Gradle Files** для проверки конфигурации.
3. На панели инструментов Android Studio выберите ваше тестовое устройство Android из раскрывающегося меню **Running devices**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2aea5b88b55a11f0b4c35254001c06ec.png)

4. Нажмите кнопку **Run** для сборки проекта. После успешной сборки приложение Tencent Cloud IM будет автоматически установлено на вашем устройстве.
5. Откройте приложение и введите любой userID для создания и входа в учетную запись пользователя.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2b124b6ab55a11f09a6b525400454e06.png)

## Изучение основных функций

Чтобы изучить основные функции, выполните следующие шаги после успешного запуска демоверсии.

### Создание учетной записи пользователя

Для создания учетной записи пользователя выберите один из следующих способов:

Регистрация на клиенте

Регистрация в консоли

Введите userID в Demo и войдите.

1. Щелкните, чтобы открыть созданное приложение. На левой боковой панели щелкните запись продукта Chat для входа.
2. На подстранице продукта Chat щелкните `Users`, чтобы открыть страницу управления пользователями.
3. Щелкните `Create account`, чтобы открыть диалоговое окно создания учетной записи. Для обычных пользователей выберите тип `General`. Хотя `Nickname` не является обязательным, рекомендуется его установить. Если отображение `userID` не подходит для вашего пользовательского интерфейса, используйте `Nickname` для различия пользователей.

Иллюстрация:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/30ca9a99b55b11f0a808525400bf7822.png)

> **Примечание:** Для обмена сообщениями требуется минимум два пользователя. На этом этапе создайте минимум два аккаунта и запишите их userID для следующих шагов.

### Добавление контактов

Чтобы добавить контакты:

1. На странице контактов нажмите кнопку `+` в верхнем правом углу и выберите `Add to Contacts` из подменю.
2. Введите действительный userID для поиска пользователя. Если вы создали учетные записи в консоли, получите действительные userID со страницы `Account Management` консоли. Путь: **Applications** > **Your App** > **Chat** > **Users** > **Account Management**.
3. Добавьте пользователя как контакт.

| Нажмите `Add to Contacts` | Поиск пользователя | Отправка запроса контакта |
| --- | --- | --- |
| ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2b575b84b55a11f085a55254007c27c5.png) | ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2b16c558b55a11f0b4c35254001c06ec.png) | ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2b52ceadb55a11f085a55254007c27c5.png) |

После успешного добавления пользователя в списке контактов отображается только что добавленный пользователь:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2b4be4d9b55a11f0a5e052540099c741.png)

### Отправка сообщений

Выберите пользователя, нажмите Message, чтобы войти в интерфейс чата:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2b6e8855b55a11f0a808525400bf7822.png)

Теперь вы можете обмениваться текстом, голосом, изображениями и проводить аудио/видеозвонки с этим пользователем:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2b7c6974b55a11f0a5e052540099c741.png)

# Отправка вашего первого сообщения

Этот документ проведет вас через интеграцию TUIKit или TUIChat и успешную отправку вашего первого сообщения.

Если же вы предпочитаете более быстрый и автоматизированный подход, обратитесь к разделу [Начало работы с интеграцией ИИ](https://www.tencentcloud.com/document/product/1047/72277) для упрощения процесса.

## Требования к среде разработки

- Android Studio-Giraffe
- Gradle-7.2
- Android Gradle Plugin Version-7.0.0
- kotlin-gradle-plugin-1.5.31

## Создание приложения

Перед интеграцией TUIKit необходимо создать приложение Chat в консоли. Шаги выполняются следующим образом:

1. Зарегистрируйте аккаунт [TRTC](https://trtc.io/register?s_url=https://console.trtc.io) и добавьте способ оплаты.
2. Войдите в [консоль Chat](https://console.trtc.io) и нажмите **Create Application**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e400d41bcc2811f093295254001c06ec.png)

3. Введите имя приложения в появившемся окне создания, выберите **Chat**, выберите подходящий **Deployment Region**, затем нажмите **Create**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e3e7e92dcc2811f0ae4f52540099c741.png)

4. После создания вы можете просмотреть статус приложения, версию сервиса, SDKAppID, SDKSecretKey, время создания и время истечения на текущей странице [**Chat Product Details Page**](https://console.trtc.io/chat) или в разделе [**My Applications**](https://console.trtc.io/app).

> **Примечание:** Версия сервиса нового приложения по умолчанию установлена на Trial Edition, статус включен по умолчанию. Один аккаунт Tencent Cloud может создать до 300 приложений Chat. Если у вас уже есть 300 приложений, вы можете сначала [деактивировать и удалить](https://www.tencentcloud.com/document/product/1047/34540#357049f6-9802-456e-8f51-755d906f014a) неиспользуемые приложения перед созданием новых. **После удаления приложения все данные и сервисы, соответствующие SDKAppID, не могут быть восстановлены. Действуйте с осторожностью.**

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e4051d9dcc2811f096d1525400e889b2.png)

## Создание учетных записей

Создание приложения только обеспечивает нормальную инициализацию SDK. Для успешной отправки сообщений вам также необходимо создать учетные записи пользователей в приложении. Существует множество способов создания учетных записей, например, непосредственно в консоли или через регистрацию клиента API. Вы можете выбрать любой подходящий метод.

> **Примечание:** Отправка сообщений требует как минимум двух пользователей для взаимодействия друг с другом, поэтому вам необходимо создать по крайней мере 2 учетные записи на этом этапе. Пожалуйста, запишите userID этих 2 учетных записей, так как они будут использоваться на следующих этапах.

**Вариант A: Создание в консоли**

> **Примечание:** Дополнительные операции, связанные с учетными записями, см. в разделе [Account Management](https://www.tencentcloud.com/document/product/1047/50301).

1. Войдите в [консоль Chat](https://console.trtc.io), выберите **Chat >**[**Users**](https://console.trtc.io/chat/account-management) в левой боковой панели и выберите целевое приложение в **верхней части**.
2. На странице управления учетными записями нажмите Create Account.
3. В появившемся диалоговом окне Create Account настройте следующие параметры:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e3e44795cc2811f0ae4f52540099c741.png)

  - Account type: с учетом регистра для обычной учетной записи и учетной записи администратора. "App admin" — это роль с наивысшим уровнем привилегий для приложения, позволяющая вызывать RESTful API для выполнения таких операций, как создание/расформирование групп и отправка сообщений всем пользователям. Каждое приложение поддерживает конфигурацию до 10 администраторов.
  - Username: Введите имя пользователя (UserID), обязательно.
  - User nickname: Введите прозвище пользователя, опционально.
  - Avatar: Введите URL аватара пользователя, опционально.
4. Нажмите **Confirm** для сохранения конфигурации.
5. После создания учетной записи вы можете просмотреть имя пользователя, прозвище, тип учетной записи, аватар и время создания в списке учетных записей.

**Вариант B: Создание учетных записей при входе**

Для регистрации через клиент просто введите совершенно новый UserID в разделе [Log In to TUIKit](#288b702c-cc18-4c33-976d-af38aabcac80) ниже. В этом случае TUIKit автоматически зарегистрирует UserID для вас.

## Генерация UserSig

> **Примечание:** Дополнительные операции, связанные с UserSig, см. в разделе [UserSig Generation & Validation](https://www.tencentcloud.com/document/product/1047/34580#usersig-.E7.94.9F.E6.88.90.26.E6.A0.A1.E9.AA.8C).

1. Войдите в [консоль Chat > Development Tools > UserSig Tools](https://console.trtc.io/usersig).
2. В разделе инструмента генерации UserSig **выберите приложение** и **вручную введите UserID**.
3. Нажмите **Generate Signature (UserSig)** для генерации подписи. Период действия подписи по умолчанию составляет 180 дней.
4. Нажмите **Copy Signature (UserSig)** для вставки и сохранения подписи.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e3ed9a88cc2811f091ab5254007c27c5.png)

## Интеграция TUIKit

Функция отправки сообщений в чате реализована компонентом `TUIChat`. Вам необходимо интегрировать по крайней мере `TUIChat` для надлежащей отправки и получения сообщений. Другие компоненты, такие как `TUIConversation`, `TUIContact` и `TUIGroup`, могут быть интегрированы по мере необходимости.

- Если вам нужны несколько компонентов пользовательского интерфейса, вы можете интегрировать TUIKit. Дополнительную информацию см. в разделе [Integrate TUIKit](https://www.tencentcloud.com/document/product/1047/50057?lang=en&pg=).
- Если вам требуется только интеграция TUIChat, см. раздел [Integrate TUIChat Only](https://www.tencentcloud.com/document/product/1047/60168?lang=en&pg=).

## Вход в TUIKit

Для использования функций компонента TUIKit требуется вход. TUILogin предоставляет API входа следующим образом:

```
// API location: TUICore/TUILogin.java// Called when login is clicked on the user UITUILogin.login(context, sdkAppID, userID, userSig, new TUICallback() {    @Override    public void onSuccess() {    }        @Override    public void onError(final int code, final String desc) {    }});
```

Этот API требует три параметра:

- SDKAppID, SDKAppID только что созданного приложения, был получен на предыдущем этапе [Create Application](#91aad40c-f055-48a6-b64b-c507d16cad7f).
- UserID, UserID пользователя1, был получен на предыдущем этапе [Create Account](#bafe82cd-d58d-4d09-956d-c318e2e49bc5). Обратите внимание, что это не прозвище пользователя.
- UserSig, UserSig пользователя1, был получен в контексте раздела [Generating UserSig](#1678b9aa-cc39-49ab-8a9d-d5e8857df988).

## Переход в интерфейс чата

Для отправки сообщений следующий шаг:

1. Используя одну из ранее зарегистрированных учетных записей (далее называемых user1), войдите в TUIKit, после чего user1 находится в сети.
2. User1 отправляет сообщение другой учетной записи (далее называемой user2). User2 не обязательно входить и не нужно иметь никаких отношений дружбы с user1.

> **Примечание:** Следующие шаги объясняют, как отправить сообщение user2 после входа как user1. Если вы хотите, чтобы user1 и user2 взаимодействовали через чат, вам нужно использовать те же шаги для входа как user2 и входа в интерфейс чата с user1.

Вы можете перейти в интерфейс чата в обратном вызове успешного входа user1 и отправить сообщение user2.

Пример кода показан ниже, где chatID должен быть идентификатором user2.

```
Intent intent = new Intent(context, TUIC2CChatMinimalistActivity.class);
intent.putExtra(TUIConstants.TUIChat.CHAT_ID, "chatID");
intent.putExtra(TUIConstants.TUIChat.CHAT_TYPE, V2TIMConversation.V2TIM_C2C);
intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK);
context.startActivity(intent);
```

# Локализация

## Описание функции

Android TUIKit по умолчанию поставляется с языковыми пакетами **английский**, **арабский** и **упрощённый китайский**, которые служат языком отображения интерфейса. В соответствии с указаниями в этом документе вы можете использовать языковой пакет по умолчанию или настроить выражения переводов и добавить другие языковые пакеты.

> **Примечание:** Начиная с версии 7.5.4852, TUIKit добавил поддержку языков RTL (языки с направлением текста справа налево, такие как арабский, иврит и т. д.). Когда язык в приложении является языком RTL, TUIKit автоматически переключится на стиль RTL; при этом встроенные языки расширены и включают арабский.

## Использование встроенного языка

Если ваше приложение требует только **английский**, **арабский** и **упрощённый китайский** языки, обратитесь к этому разделу.

### Следование языку системы

Вы можете использовать TUIKit напрямую без каких-либо дополнительных шагов. Язык внутри компонента будет следовать языку системы.

### Указание отображаемого языка

Если вам необходимо указать язык для интерфейса TUIKit, вам нужно вызвать следующий код во время инициализации Application для установки. Например, для установки упрощённого китайского:

```
public class MyApplication extends Application {    @Override    protected void onCreate() {        super.onCreate();        TUIThemeManager.getInstance().changeLanguage(this, TUIThemeManager.LANGUAGE_ZH_CN);    }    /**    * Доступные параметры языка перечислены следующим образом:    * TUIThemeManager.LANGUAGE_EN        ---- Английский    * TUIThemeManager.LANGUAGE_ZH_CN     ---- Упрощённый китайский    * TUIThemeManager.LANGUAGE_AR        ---- Арабский    */}
```

> **Примечание:** Вызов только метода changeLanguage не приведёт к автоматическому обновлению пользовательского интерфейса. После изменения языка вам потребуется получить переведённые строки и установить их снова в соответствующие компоненты пользовательского интерфейса, чтобы изменения вступили в силу.

### Динамическое изменение языка

Вы можете обратиться к коду в файле [LanguageSelectActivity.java](https://github.com/TencentCloud/chat-uikit-android/blob/main/Demo/app/src/main/java/com/tencent/qcloud/tim/demo/login/LanguageSelectActivity.java) TUIKitDemo. Вы также можете использовать следующий метод для переключения языка, например на упрощённый китайский:

```
TUIThemeManager.getInstance().changeLanguage(context, TUIThemeManager.LANGUAGE_ZH_CN);System.exit(0);Intent intent = context.getPackageManager().getLaunchIntentForPackage(context.getPackageName());context.startActivity(intent);
```

### Решение для сбоя переключения языка после использования WebView

Сбой переключения языков после использования WebView — это ошибка в Android 7 и более поздних версиях. Это связано с процессом инициализации WebView, который изменяет язык приложения в соответствии с языком системы телефона. Чтобы решить эту проблему, следующий код должен быть вызван в методе **setThemeInternal** в [TUIThemeManager.java](https://github.com/TencentCloud/chat-uikit-android/blob/main/TUIKit/TUICore/tuicore/src/main/java/com/tencent/qcloud/tuicore/TUIThemeManager.java):

```
setWebViewLanguage(appContext);
```

После добавления:

```
private void setThemeInternal(Context context) {    if (context == null) {        return;    }    Context appContext = context.getApplicationContext();    if (!isInit) {        isInit = true;        if (appContext instanceof Application) {            ((Application) appContext).registerActivityLifecycleCallbacks(new ThemeAndLanguageCallback());        }        /**        * добавить код здесь  начало        */        setWebViewLanguage(appContext);        /**        * добавить код здесь  конец        */        Locale defaultLocale = getLocale(appContext);        SPUtils spUtils = SPUtils.getInstance(SP_THEME_AND_LANGUAGE_NAME);        currentLanguage = spUtils.getString(SP_KEY_LANGUAGE, defaultLocale.getLanguage());        currentThemeID = spUtils.getInt(SP_KEY_THEME, THEME_LIGHT);        // Язык требует инициализации только один раз        applyLanguage(appContext);    }    // Тему необходимо обновлять несколько раз    applyTheme(appContext);}
```

## Использование дополнительных языков/Настройка выражений переводов

Если ваше приложение требует добавления дополнительных языков или изменения определённых записей перевода, обратитесь к этому разделу. В этой главе описывается процесс добавления новых языковых пакетов и настройки переводов с использованием примера добавления корейского языкового пакета к компоненту **TUIGroup**.

### Добавление новых файлов языковых ресурсов

В каталоге компонента TUIGroup в Android Studio добавьте новый Android Resource File через контекстное меню.

Создайте каталоги ресурсов по измерению Locale, введя имя файла как 'strings'.

Выбор языка установлен на корейский ("ko: Korean"), регион назначен как "KR: South Korea", и после подтверждения успешно создан файл корейских ресурсов values-ko-rKR/strings.xml.

### Персонализированный пользовательский перевод

На предыдущем шаге успешно сгенерирован файл корейских ресурсов values-ko-rKR/strings.xml. Теперь скопируйте содержимое из файла values/strings.xml в файл values-ko-rKR/strings.xml, заменив соответствующий английский текст корейским, как показано на иллюстрации.

Атрибут 'name' в различных файлах языковых ресурсов остаётся согласованным, а конкретное содержимое может быть настроено путём перевода.

### Следование языку системы

Просто используйте TUIKit, и при установке языка вашего мобильного устройства по умолчанию на корейский и запуске приложения язык приложения будет автоматически отображаться на корейском языке.

### Указанный язык отображения

Если вам требуется установить язык интерфейса TUIKit на корейский, рекомендуется сначала добавить корейский в менеджер языков во время инициализации Application, а затем установить язык интерфейса TUIKit на корейский.

```
public class MyApplication extends Application {    @Override    protected void onCreate() {        super.onCreate();        // Добавить корейский        TUIThemeManager.addLanguage("ko-rKR", Locale.KOREA);        // Изменить язык приложения на корейский.        TUIThemeManager.getInstance().changeLanguage(this, "ko-rKR");    }}
```

# Начало работы

Этот документ будет содержать рекомендации по интеграции TUIKit и успешной отправке вашего первого сообщения.

Или, если вы предпочитаете более быстрый и автоматизированный подход, вы можете обратиться к разделу [Get started with AI integration](https://www.tencentcloud.com/document/product/1047/72277) для упрощения процесса.

## Предварительные требования

- Flutter >= 3.29.0, Dart >= 3.7.0, Kotlin >= 1.9.
- Android Studio Ladybug | 2024.2.1 или позже, Android Gradle plugin 7.3.1 или позже, JDK 17.
- Xcode 12.0 или позже.

## Создание приложения

Перед интеграцией TUIKit вам необходимо создать новое приложение Chat в консоли. Следуйте этим шагам:

1. [Зарегистрируйтесь в аккаунте Tencent RTC](https://trtc.io/register?s_url=https://console.trtc.io) и добавьте способ оплаты.
2. Войдите в [Chat Console](https://console.trtc.io), затем нажмите **Create Application**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2830c87a023811f1b9ec525400a31896.png)

3. В появившемся окне введите название приложения, выберите **Chat**, выберите соответствующий **Deployment Region** и нажмите **Create**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2841ae10023811f198e252540097cba1.png)

4. После создания приложения вы сможете получить ваши SDKAppID и SDKSecretKey.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/edeadd1c072211f1a751525400380f7d.png)

> **Примечание:** Вновь созданные приложения по умолчанию используют версию Trial Edition и включены по умолчанию. Каждый аккаунт может создать до 300 приложений Chat. Если вы достигли этого лимита, вы можете [отключить и удалить](https://www.tencentcloud.com/zh/document/product/1047/34540#357049f6-9802-456e-8f51-755d906f014a) неиспользуемые приложения перед созданием новых. **После удаления все данные и сервисы, связанные с SDKAppID, будут безвозвратно потеряны и не смогут быть восстановлены. Действуйте с осторожностью**.

## Создание аккаунта

Создание приложения позволяет вам инициализировать SDK, но для отправки сообщений вам необходимо создать учетные записи пользователей в приложении. Вы можете создавать аккаунты прямо в консоли или регистрировать их через API на стороне клиента. Выберите метод, который лучше всего подходит для вашего рабочего процесса.

> **Примечание:** Для отправки сообщений вам нужны как минимум два пользователя, поэтому создайте как минимум два аккаунта сейчас. Запишите userID обоих аккаунтов, так как они понадобятся вам на последующих этапах.

**Вариант A: Создание в консоли**

> **Примечание:** Для получения дополнительных опций управления аккаунтами см. [Account Management](https://www.tencentcloud.com/zh/document/product/1047/50301).

1. Войдите в [Chat Console](https://console.trtc.io), выберите **Chat >** [**Users**](https://console.trtc.io/chat/account-management) в левой панели навигации и **выберите ваше целевое приложение в верхней части**.
2. На странице управления аккаунтами нажмите **Create Account**.
3. В диалоговом окне настройте следующие параметры:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2838798f023811f18e6252540073fd3b.png)

  - Account Type: Выберите между обычными аккаунтами и аккаунтами администратора. "App Administrator" обладает наивысшими привилегиями управления и может использовать REST API для создания/распуска групп, отправки сообщений All Users Push и многого другого. Каждое приложение поддерживает до 10 администраторов.
  - Username: Введите имя пользователя (UserID). Это поле обязательно.
  - User Nickname: При необходимости введите никнейм.
  - Avatar: При необходимости введите URL аватара.
4. Нажмите **OK** для сохранения аккаунта.
5. После создания вы сможете просмотреть имя пользователя, никнейм, тип аккаунта, аватар и время создания в списке аккаунтов.

**Вариант B: Создание аккаунтов при входе**

Для регистрации аккаунта на стороне клиента просто передайте новый UserID при входе в TUIKit, как описано ниже. TUIKit автоматически зарегистрирует UserID за вас — дополнительные шаги не требуются.

## Генерация UserSig

> **Примечание:** Для получения дополнительной информации о UserSig см. [UserSig Generation & Verification](https://trtc.io/document/34385).

1. Войдите в [Chat Console > Development Tools > UserSig Tools](https://console.trtc.io/usersig).
2. В инструменте UserSig **выберите ваше приложение** и **введите UserID**.
3. Нажмите **Generate** для создания подписи. Период действия по умолчанию составляет 180 дней.
4. Нажмите **Copy Signature (UserSig)** для копирования и сохранения подписи.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/283577ef023811f18ab45254001d6acc.png)

## Интеграция TUIKit

Отправка сообщений в чате осуществляется через MessageList. Для получения деталей интеграции обратитесь к документации [Full Feature Integration](https://www.tencentcloud.com/document/product/1047/77489).

## Вход в TUIKit

Все функции TUIKit требуют входа пользователя, который предоставляется интерфейсом входа `LoginStore`:

```
final result = await LoginStore.shared.login(  sdkAppID: SDKAPPID,  userID: userID,  userSig: userSig,);if (result.errorCode == 0) {
```

Этот интерфейс требует трех параметров:

- SDKAppID: SDKAppID для вашего приложения, полученный в разделе Create Application.
- UserID: UserID для User1, полученный в разделе Create Account. Примечание: это не никнейм пользователя.
- UserSig: UserSig для User1, полученный в разделе Generate UserSig.

## Переход к интерфейсу чата

Для отправки сообщений выполните следующие шаги:

1. Войдите в TUIKit, используя один из зарегистрированных аккаунтов (именуемый user1). После входа user1 будет в сети.
2. user1 отправляет сообщение другому аккаунту (именуемый user2). user2 не обязан быть в сети и не обязан быть другом user1.

> **Примечание:** Этот раздел описывает, как user1 может отправить сообщение user2 после входа в качестве user1. Если вы хотите, чтобы user1 и user2 общались интерактивно, войдите в качестве user2 согласно тем же шагам и откройте интерфейс чата с user1.

После успешного входа user1 вы можете перейти или встроить интерфейс чата и отправить сообщение user2.

Используйте следующий пример кода. Установите userID на userID целевого пользователя чата (user2):

```
final conversation = ConversationInfo(  conversationID: 'c2c_${userID}',  type: ConversationType.c2c,);Navigator.push(  context,  MaterialPageRoute(    builder: (context) => ChatPage(      conversation: conversation,    ),  ),);
```

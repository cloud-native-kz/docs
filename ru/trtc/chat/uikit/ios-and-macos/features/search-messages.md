# Поиск сообщений

Локальный поиск реализован в компоненте TUISearch интерфейса TUIKit. Он позволяет пользователям быстро найти нужную информацию в больших объемах сложных данных, таких как история чата, контакты и групповые чаты. Его также можно использовать в качестве инструмента управления для удобной и эффективной навигации по обширному содержимому.

> **Примечание:** Функция локального поиска доступна только в версии Chat Pro, Pro Plus или Enterprise. Для использования [приобретите версию Pro, Pro Plus или Enterprise](https://www.tencentcloud.com/document/product/1047/34577). Дополнительные сведения см. в разделе [Тарифы](https://www.tencentcloud.com/document/product/1047/34350).

## Демонстрация функций

API поиска пользовательского интерфейса состоит из трех частей: первая часть предназначена для поиска друзей, вторая часть предназначена для поиска групп и участников групп, а третья часть предназначена для поиска сообщений, где сообщения классифицируются по беседам.
[Скачайте и установите приложение](https://www.tencentcloud.com/document/product/1047/34279), чтобы попробовать его прямо сейчас.

## Руководство по интеграции

Ниже описано, как интегрировать компонент TUISearch.

### Приобретение пакета

[Приобретите версию Pro, Pro Plus или Enterprise](https://www.tencentcloud.com/document/product/1047/34577).

### Интеграция TUISearch

Добавьте следующее содержимое в ваш Podfile:

```
// Integrate TUISearchpod 'TUISearch'
```

Запустите `pod install`.

### Вход в TUIKit

Необходимо вызвать `TUILogin` из `TUICore` для входа в TUIKit. Инициализация по умолчанию реализуется внутри API входа, и дополнительный вызов API инициализации не требуется.

Swift

Objective-C

```
TUILogin.login(SDKAppID, userID: userID, userSig: userSig) {    // Login succeeded    print("login succeeded")} fail: { code, desc in    // Login failed    print("login failed, \\(code): \\(desc)")}
```

```
[TUILogin login:SDKAPPID userID:userID userSig:userSig succ:^{   // Login succeeded} fail:^(int code, NSString *msg) {   // Login failed}];
```

### Запуск пользовательского интерфейса поиска

Если вы интегрировали компоненты TUIConversation и TUISearch, дополнительная обработка на этом этапе не требуется, и searchBar отображается над списком бесед по умолчанию.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b033b04c135911efbf645254007bbd8c.png)

Если вы интегрировали только TUISearch, вы можете напрямую инициализировать TUISearchBar и добавить его в свое представление.
Логика поиска пользовательского интерфейса и пользовательский интерфейс инкапсулированы внутри TUISearchBar. После добавления TUISearchBar вы можете нажать на него, чтобы запустить поиск.

Пример кода:

Swift

Objective-C

```
// initlet searchBar = TUISearchBar()// self.containerView is your own viewself.containerView.addSubview(searchBar)
```

```
// initTUISearchBar *searchBar = [[TUISearchBar alloc] init];// self.containerView is your own view[self.containerView addSubview:searchBar];
```

## Часто задаваемые вопросы

#### Как выполнить поиск по пользовательским сообщениям?

Необходимо использовать API `createCustomMessage:desc:extension` для создания и отправки пользовательского сообщения и указать текст для поиска в параметре `desc`.

- Если вы используете API `createCustomMessage` для создания пользовательского сообщения, локально сохраняется поток двоичных данных, и пользовательское сообщение невозможно найти.
- Если вы настроили функцию автономной отправки и указали параметр `desc`, пользовательское сообщение также будет отправлено автономно, и содержимое, указанное в параметре `desc`, будет отображено в строке уведомлений.
- Если вам не нужна функция автономной отправки, используйте параметр `disablePush` в `V2TIMOfflinePushInfo` API `sendMessage` для ее отключения.
- Если вы не хотите, чтобы искомый текст отображался как отправляемое содержимое, используйте параметр `desc` в `V2TIMOfflinePushInfo` для указания отправляемого содержимого.

#### Как выполнить поиск по сообщениям с мультимедиа?

Сообщения с мультимедиа включают сообщения с файлами, изображениями, аудио и видео.

- Для сообщения с файлом имя файла обычно отображается в пользовательском интерфейсе. Поэтому при создании сообщения с файлом можно установить параметр `fileName` в качестве искомого содержимого. Если параметр `fileName` не установлен, система получает имя файла из `filePath` и сохраняет его как на локальном устройстве, так и на сервере.
- Для сообщения с изображением, аудио или видео в пользовательском интерфейсе обычно отображается миниатюра или продолжительность. В этом случае вы можете указать тип сообщения для поиска, но не можете указать ключевые слова для поиска.

Это руководство содержит пошаговые инструкции, которые помогут вам быстро начать работу с демо-приложением Chat и испытать такие функции, как отправка текстовых сообщений, голосовых и видеовызовов. После настройки интерфейс приложения будет выглядеть следующим образом:

## Демо-приложение

Вы можете перейти на страницу [Бесплатное демо](https://trtc.io/document/35076#42b9c204-f927-4326-9af6-370e511773f3), чтобы загрузить демо-приложение Chat и изучить полный спектр функций обмена мгновенными сообщениями.

## Предварительные требования

### Включение сервиса

1. Войдите в [Консоль](https://console.trtc.io/). Если у вас уже есть приложение, сохраните `SDKAppID` и `SDKSecretKey` и перейдите к разделу [Подготовка среды](#env).
2. Нажмите `Create Application`, введите название вашего приложения, выберите продукт **Chat**, выберите регион и нажмите `Create`, чтобы добавить приложение.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/8f0d0930ab0511f0b08552540044a08e.png)

3. После создания вы сможете увидеть `SDKAppID` и `SDKSecretKey` вашего приложения на странице обзора консоли. Эта информация потребуется вам для запуска демо.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/8ed5f433ab0511f09710525400e889b2.png)

> **Запрещено:**Защитите ваш `SDKSecretKey`, чтобы предотвратить несанкционированный доступ к приложению.

### Подготовка среды

Перед началом убедитесь, что выполнены следующие требования:

- Xcode 10 или более поздняя версия
- iOS 9.0 или более поздняя версия (физическое устройство или симулятор)
- CocoaPods 1.7.5 или более поздняя версия. Если не установлена, обратитесь к [Руководству CocoaPods — Начало работы](https://guides.cocoapods.org/using/getting-started.html#getting-started).

### Построение и запуск демо

> **Примечание:**Демо по умолчанию интегрирует функции аудио и видеовызовов. SDK аудио/видео в настоящее время не поддерживает симуляторы. Используйте физическое устройство для отладки или запуска демо.

- Для проекта Swift перейдите в папку `Swift/TUIKitDemo` и откройте `TUIKitDemo.xcworkspace` для построения и запуска.
- Для проекта Objective-C перейдите в папку `chat-uikit-ios-main/Demo` и откройте `TUIKitDemo.xcworkspace` для построения и запуска.

Например, на устройстве iOS:

1. Подключите ваше устройство iOS к компьютеру. На устройстве перейдите в **Параметры** > **Конфиденциальность и безопасность** > **Режим разработчика** и включите режим разработчика.
2. В Xcode выберите ваше тестовое устройство iOS из раскрывающегося меню **iOS Device** в верхней части.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/8e777c29ab0511f0b5345254005ef0f7.png)

3. В разделе **TARGETS** > **Signing & Capabilities** проекта установите флажок **Automatically manage signing**, затем настройте вашу учетную запись разработчика Apple и Bundle Identifier для каждого Target. Если вы еще не вошли в свой Apple ID в Xcode, перейдите в **Xcode** > **Preferences** > **Accounts**, чтобы добавить учетную запись разработчика или Apple ID.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/8e752bebab0511f0a0a052540099c741.png)

4. Нажмите кнопку запуска для построения. После успешного построения TUIKitDemo автоматически устанавливается на ваше устройство.
5. Откройте приложение и введите любой userid для создания и входа в учетную запись пользователя.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/8e9881c0ab0511f0a68e5254001c06ec.png)

Сначала создайте учетную запись пользователя. Вы можете зарегистрироваться прямо в клиенте демо или создать учетную запись в консоли. Выберите один из следующих методов:

## Часто задаваемые вопросы

#### 1. Как исправить ошибку "pod does not exist" или "pod version less than 1.7.5" при установке pod?

Выполните следующие команды для установки последней версии pod.

```
# 1. Установка pod sudo gem install cocoapods -n /usr/local/bin# Если у вас установлено несколько версий Xcode, используйте следующую команду для выбора версии Xcode (обычно выберите последнюю).sudo xcode-select -switch /Applications/Xcode.app/Contents/Developer# 2. Обновление локального репозитория podpod setup
```

#### 2. Несовместимость версии TUICore

При использовании CocoaPods для получения демо, если версия TUICore, заблокированная в файле `Podfile.lock`, несовместима с зависимостью плагина, может возникнуть следующая ошибка: `CocoaPods could not find compatible versions for pod "TUICore"`

Выполните следующие шаги для решения:

1. В корневом каталоге проекта демо выполните следующую команду для удаления файла `Podfile.lock`:

```
rm Podfile.lock
```

2. Выполните следующую команду для обновления локального репозитория кода:

```
pod repo update
```

3. Выполните следующую команду для переустановки:

```
pod update
```

После выполнения этих шагов будет создан новый файл `Podfile.lock` и зависимости будут установлены успешно.

#### 3. Недоверенный разработчик

Если вы выполняете отладку на физическом устройстве и видите подсказку "Untrusted Developer" после успешного построения, нажмите Cancel, чтобы закрыть подсказку. Затем на вашем устройстве iOS перейдите в **Параметры** > **Основные** > **VPN и управление устройством**, выберите приложение разработчика и доверьте разработчику. После этого откройте приложение демо, чтобы продолжить.

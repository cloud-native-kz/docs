# Конфигурация производителя

Данная статья поможет вам создать и использовать **Push-сертификаты** и **Файлы профиля**, при условии, что у вас есть **Учетная запись разработчика** и **Сертификаты разработки/распространения**.

Если у вас есть вопросы о "Учетной записи разработчика" или "Сертификатах разработки/распространения", вы можете [обратиться к соответствующим концепциям в приложениях](https://www.tencentcloud.com/document/product/1047/67573#d5744498-1f37-4255-b3e8-7e55dda2fdc4).

## Конфигурация APNs Push-сертификата

Перед интеграцией компонента TIMPush необходимо получить push-сертификат APNs у Apple, а затем загрузить сертификат в консоль CHAT. После этого вы можете следовать инструкциям быстрого доступа для интеграции.

Apple в настоящее время предлагает два основных типа сертификатов для конфигурации производителя: сертификаты p12 и сертификаты p8. Каждый имеет свои преимущества и недостатки, поэтому вы можете выбрать один в соответствии с вашими потребностями.

|  | **Тип сертификата** | **Срок действия и управление** | **Безопасность** | **Dynamic Island** |
| --- | --- | --- | --- | --- |
| **Сертификат P12** | Сертификат P12 — это двоичный файл, содержащий открытый ключ и приватный ключ, используется для проверки подлинности на основе сертификата. Он объединяет сертификат открытого ключа и приватный ключ в один файл с расширением .p12 или .pfx. | Сертификат P12 обычно имеет период действия один год и должен быть пересоздан и переразвернут по истечении срока действия. Каждому приложению требуется отдельный сертификат P12 для обработки push-уведомлений. | Сертификаты: P12 Certificate использует проверку подлинности на основе сертификата и требует хранения приватного ключа на сервере. Это может увеличить риски безопасности, так как приватный ключ может быть получен неавторизованными пользователями. | Не поддерживается. |
| **Сертификат P8** | Сертификат P8 — это ключ аутентификации, используемый для аутентификации на основе токена. Это текстовый файл, содержащий приватный ключ, с расширением .p8. | Сертификат P8 не имеет срока действия, поэтому вам не нужно беспокоиться об истечении срока действия сертификата. Кроме того, использование сертификата P8 упрощает управление сертификатом, так как вы можете использовать один сертификат P8 для предоставления услуги push-уведомлений для нескольких приложений. | Сертификат P8 использует аутентификацию на основе токена, означающую, что сервер периодически генерирует JSON Web Token (JWT) для установления соединения с APNs. Этот метод более безопасен, так как не требует хранения приватного ключа на сервере. | Поддержка push Dynamic Island |

### Метод 1: использование сертификата p12 (традиционный сертификат APNs push)

### Шаг 1: получение сертификата APNs

1. Войдите на веб-сайт [Apple Developer Center](https://developer.apple.com/account/), нажмите **Certificates, Identifiers & Profiles** или **Certificates, IDs & Profiles** на боковой панели, чтобы перейти на страницу **Certificates, IDS & Profiles**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/340e3623b30b11ef8b1b525400f69702.jpeg)

2. Нажмите на **+** рядом с Identifiers на правой стороне.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/33da1734b30b11ef96e55254002693fd.png)

3. Вы можете выполнить следующие шаги, чтобы создать новый AppID или добавить сервис `Push Notification` в существующий AppID.

> **Примечание:** `Bundle ID` вашего приложения не может использовать подстановочный знак `*`, в противном случае услуга удаленной доставки push будет недоступна.

4. Выберите **App IDs**, нажмите **Continue**, чтобы перейти к следующему шагу.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/33ef60e2b30b11ef96e55254002693fd.png)

5. Выберите **App**, нажмите **Continue**, чтобы перейти к следующему шагу.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/33f569b6b30b11ef96e55254002693fd.jpeg)

6. Настройте `Bundle ID` и другую информацию, нажмите **Continue**, чтобы перейти к следующему шагу.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/33d57336b30b11ef8c01525400fdb830.png)

7. Выберите **Push Notifications**, чтобы активировать услугу удаленной доставки push.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/33ef55dbb30b11efa2e952540075b605.png)

#### Создание сертификата

1. Выберите ваш AppID, выберите **Configure**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/354711d7b30b11ef8b1b525400f69702.png)

2. Вы можете увидеть в окне **Apple Push Notification service SSL Certificates** два `SSL Certificate`: один для среды разработки (Development) и один для производственной среды (Production) push-сертификат удаленной доставки, как показано ниже:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/34268eafb30b11ef9dc0525400329841.jpeg)

3. Сначала выберите Development **Create Certificate**; система подскажет вам, что требуется запрос подписи сертификата (CSR).

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/35b34bd2b30b11ef8b1b525400f69702.png)

4. Откройте **Keychain Access** на вашем Mac, в меню выберите **Keychain Access** > **Certificate Assistant** > **Request a Certificate From a Certificate Authority** (`Keychain Access - Certificate Assistant - Request a Certificate From a Certificate Authority`).

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/fdeb3532c1cd11efbcd1525400bf7822.png)

5. Введите адрес электронной почты пользователя (ваша электронная почта), общее имя (ваше имя или название компании), выберите **Save to disk**, нажмите **continue**, и система создаст файл `*.certSigningRequest`.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/0499312dc1ce11ef8f945254007c27c5.png)

6. Вернитесь на страницу веб-сайта Apple Developer, на которой вы находились на шаге 3 приведенных выше инструкций, нажмите **Choose File**, чтобы загрузить файл `*.certSigningRequest`.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/34aa86b8b30b11ef9d2952540055f650.png)

7. Нажмите **Continue**, чтобы создать push-сертификат.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/34c31fe8b30b11ef970f525400d5f8ef.png)

8. Нажмите **Download**, чтобы загрузить `Development SSL Certificate` на локальный компьютер.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/34579aa4b30b11efbfb3525400bdab9d.jpeg)

9. Повторите шаги 1–8 выше, чтобы загрузить `Production SSL Certificate` на локальный компьютер.

> **Примечание:** На самом деле этот сертификат представляет собой объединенный сертификат Sandbox и Production, который применяется как к среде разработки, так и к производственной среде.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/3551083cb30b11ef9d2952540055f650.png)

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/34f964f4b30b11ef96e55254002693fd.png)

10. Дважды щелкните загруженный `SSL Certificate` для сред разработки и производства. Система импортирует его в цепочку ключей.
11. Откройте приложение Keychain Access, перейдите в **log in to** > **My Certificates**, щелкните правой кнопкой мыши, чтобы экспортировать недавно созданные файлы `p12` для среды разработки (`Apple Development IOS Push Service`) и производственной среды (`Apple Push Services`).

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/10177c7cc1ce11ef98e4525400329841.png)

> **Примечание:** При сохранении файла `.p12` обязательно установите пароль. Шаг 2: Загрузите сертификат в консоль.

### Шаг 2: загрузите сертификат p12 в консоль Chat

1. Войдите в [консоль Chat](https://console.trtc.io/chat/push-plugin-push-identifier).
2. Нажмите на карточку целевого приложения, чтобы перейти на страницу основной конфигурации приложения.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/14e26a91c1ce11ef95c1525400d5f8ef.png)

3. Нажмите **iOS native offline push settings** справа, нажмите **Add Certificate**.
4. Выберите тип сертификата, загрузите сертификат iOS (p12), установите пароль сертификата и нажмите **Confirm**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1a079c4bc1ce11efbcd1525400bf7822.png)

> **Примечание:** При загрузке push-сертификата убедитесь, что Bundle ID совпадает с Bundle ID основного приложения. Рекомендуется назвать загруженный сертификат на английском языке (специальные символы, такие как скобки, не допускаются). Вы должны установить пароль для загруженного сертификата. Без пароля push-уведомления не могут быть получены. Для приложения, опубликованного в App Store, среда сертификата должна быть производственной средой. В противном случае push-уведомления не могут быть получены. Загруженный сертификат .p12 должен быть вашим подлинным и действующим сертификатом.

5. После создания информации push-сертификата запишите **ID** сертификата.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1fd5d862c1ce11ef87c05254005ef0f7.png)

### Метод 2: использование сертификата p8 (поддержка Dynamic Island Push)

Сертификат p8: сертификат p8 не имеет срока действия, поэтому вам не нужно беспокоиться об истечении срока действия сертификата. Кроме того, использование сертификата p8 может упростить управление сертификатом, так как вы можете использовать один сертификат p8 для предоставления услуг push-уведомлений для нескольких приложений. Кроме того, сертификат p8 поддерживает Dynamic Island push.

### Шаг 1: получение сертификата APNs

Чтобы создать файл сертификата p8, сначала необходимо войти в [Apple Developer Center](https://developer.apple.com/).

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/347f9580b30b11efbfb3525400bdab9d.png)

1. Перейдите в Certificates, Identifiers & Profiles: нажмите на **Account** в верхнем правом углу страницы, а затем выберите **Certificates, Identifiers & Profiles** из раскрывающегося меню.
2. Создайте новый App ID: нажмите на **Identifiers** в левом меню, затем нажмите на **+** справа, чтобы создать новый App ID. Заполните соответствующую информацию и нажмите **Continue**.
3. Создайте новый ключ: в левом меню нажмите на **Keys**, затем нажмите **+** справа, чтобы создать новый ключ. Введите имя ключа, а затем установите флажок для **Apple Push Notifications service (APNs)**, нажмите **Continue**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/340cd9acb30b11ef8c01525400fdb830.png)

Подтвердите и создайте ключ: на странице подтверждения проверьте информацию о ключе, затем нажмите **Register**. Затем вы увидите страницу с предложением загрузить ключ. Нажмите **Download**, чтобы сохранить созданный файл .p8 на компьютер.

> **Примечание:** Сертификат p8 можно загрузить только один раз, поэтому сохраните его в безопасном месте. Сохраняйте загруженный файл p8 в безопасном месте, так как вы не сможете загрузить его снова. Вы можете использовать этот сертификат P8 для конфигурации приложения iOS для получения push-уведомлений.

### Шаг 2: загрузите сертификат p8 в консоль Chat

1. Войдите в [консоль Chat](https://console.trtc.io/chat/push-plugin-push-identifier).
2. Нажмите на карточку целевого приложения, чтобы перейти на страницу основной конфигурации приложения.
3. Нажмите **iOS native offline push settings** справа, нажмите **Add Certificate**.
4. Выберите сертификат .p8
5. Выберите производственную среду и среду разработки для загрузки файла (вам нужно подготовить одинаковые материалы, только бэкэнд назначит вам среду APNs push)

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/43037b7ac1ce11efb6165254001c06ec.png)

> **Примечание:** **KeyID**: это уникальный идентификатор вашего APNs Auth Key. Когда вы создаете новый APNs Auth Key в Apple Developer Center, система генерирует для вас идентификатор ключа. Вы можете найти его в разделе "Certificates, Identifiers & Profiles" в подразделе "Keys". **TeamID**: это уникальный идентификатор вашей учетной записи разработчика. Вы можете найти его на странице сведений об учетной записи в Apple Developer Center. Нажмите "Membership" в верхнем правом углу, и вы найдете свой Team ID в разделе "Membership Details". **BundleID**: это уникальный идентификатор вашего приложения, также известный как Application ID. Вы можете найти его в разделе "Certificates, Identifiers & Profiles" Apple Developer Center. Выберите "Identifiers", затем найдите соответствующий Bundle ID в списке приложений.

## Создание файла профиля

### Шаг 1: создание App ID

1. **Вход в Apple Developer Center:**
- Посетите [Apple Developer Center](https://developer.apple.com/account/) и войдите в свою учетную запись разработчика.
2. **Создание App ID:**
- В разделе "Certificates, Identifiers & Profiles" выберите "Identifiers".
- Нажмите кнопку "+" для создания нового App ID.
- Выберите "App IDs" и нажмите "Continue".
- Введите имя App ID и Bundle ID (например, **com.yourcompany.yourapp**).
- Обязательно выберите соответствующие функции (например, Push Notifications, App Groups и т. д.), затем нажмите "Continue" и подтвердите.

### Шаг 2: создание App ID для расширения сервиса (дополнительно)

- Повторите приведенные выше шаги, чтобы создать новый App ID для расширения сервиса.
- Bundle ID обычно является Bundle ID основного приложения, за которым следует идентификатор расширения, например, com.yourcompany.yourapp.extension.

### Шаг 3: создание файла профиля

1. **Создание файла профиля:**
- В разделе "Certificates, Identifiers & Profiles" выберите "Profiles".
- Нажмите кнопку "+" для создания нового файла профиля.
- Выберите "iOS App Development" или "App Store" (в зависимости от ваших потребностей), затем нажмите "Continue".
- Выберите только что созданный App ID (App ID основного приложения), затем нажмите "Continue".
- Выберите сертификат разработки и устройство (если это файл профиля разработки), затем нажмите "Continue".
- Введите имя файла профиля, затем нажмите "Generate".
2. **Создание файла профиля для расширения сервиса (дополнительно):**
- Повторите приведенные выше шаги, чтобы создать новый файл профиля для расширения сервиса, убедитесь, что вы выбрали соответствующий App ID.

### Шаг 4: загрузка и установка файла профиля

1. **Загрузка файла профиля:**
- После создания файла профиля нажмите кнопку "Download", чтобы загрузить файл профиля.
2. **Установка файла профиля:**
- Дважды щелкните загруженный файл профиля. Он будет автоматически установлен в Xcode.

## Приложение: учетная запись разработчика, тип сертификата и файл профиля

**Учетная запись разработчика Apple:**

Создание сертификатов и профилей iOS требует платную учетную запись Apple. Существует три типа учетных записей разработчика Apple: индивидуальный разработчик, разработчик компании и корпоративный разработчик, каждый со своим назначением.

- Учетные записи индивидуального разработчика и разработчика компании стоят 99 долларов США в год, тогда как учетные записи корпоративного разработчика стоят 299 долларов США в год.
- Корпоративные разработчики обычно используются крупными предприятиями при разработке внутренних приложений и не могут быть опубликованы в App Store.
- Убедитесь, что у вас уже есть личная или корпоративная учетная запись разработчика.

**Введение сертификатов:**

Сертификаты бывают различных типов и служат разным целям. Вот классификация сертификатов iOS:

| **Тип сертификата** | **Различия** |
| --- | --- |
| **Сертификат разработчика** (Developer Certificate) | Используется для подписи и отладки приложений на этапе разработки. Сертификат разработчика выдается Apple Developer Center и требует использования Xcode или других инструментов разработки для применения и управления. |
| **Сертификат распространения** (Distribution Certificate) | Используется для распространения приложений другим пользователям или загрузки их в App Store на рецензирование. Сертификат распространения выдается Apple Developer Center и требует использования Xcode или других инструментов разработки для применения и управления. |
| **Push-сертификат** (Push Certificate) | Используется для реализации функции APNs удаленной доставки push, позволяющей приложению получать push-уведомления с сервера. Push-сертификат выдается Apple Developer Center и требует использования Xcode или других инструментов разработки для применения и управления. |
| **Корпоративный сертификат** (Enterprise Certificate) | Используется для распространения приложений внутренним сотрудникам или клиентам. Корпоративный сертификат выдается Apple Developer Center и требует использования Xcode или других инструментов разработки для применения и управления. |

**Введение файла профиля:**

Provisioning Profiles также делятся на два типа: Development и Distribution. Файл конфигурации включает сертификаты, App ID и устройства.

Расширение файла — .mobileprovision. Он играет роль конфигурации и валидации в системе учетной записи разработчика и необходим для отладки на реальном устройстве и упаковки для выпуска.

| **Файл профиля (Provisioning Profile)** | **Функция** |
| --- | --- |
| Development | Provisioning Profile соответствует одному App ID (Bundle ID). Provisioning Profile определяет, какую пару ключей (открытый ключ/приватный ключ) Xcode использует для подписи приложения, и он будет встроен в пакет .ipa при упаковке приложения. Provisioning Profile объединяет всю эту информацию, облегчая использование при отладке и упаковке приложения для выпуска. Таким образом, достаточно просто выбрать разные файлы Provisioning Profile в разных ситуациях. Provisioning Profiles также делятся на типы Development и Distribution со сроком действия, аналогичным сертификатам. Версия Development Provisioning Profile используется для разработки и отладки, тогда как версия Distribution в основном используется для отправки в App Store на рецензирование и не указывает устройства для тестирования разработки. |

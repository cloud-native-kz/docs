# Локализация

## Описание функции

iOS версия TUIKit поставляется со встроенными языковыми пакетами **Упрощённого китайского**, **Английского** и **Арабского** языков, которые служат языками отображения пользовательского интерфейса. Следуя этому документу, у вас есть возможность использовать языковые пакеты по умолчанию или настроить переводы и добавить дополнительные языковые пакеты.

> **Примечание:** Начиная с версии 7.5.4852 TUIKit добавил поддержку языков с RTL-письменностью (языков с написанием справа налево, таких как арабский и иврит), одновременно включив арабский язык в встроенные языки.

## Использование встроенного языка

Если ваше приложение требует только **Упрощённого китайского**, **Английского** и **Арабского** языков, обратитесь к этому разделу.

### Следить за системным языком

Вы можете использовать TUIKit напрямую без дополнительных шагов. Язык внутри компонента будет соответствовать системному языку.

### Указание отображаемого языка

Если требуется указать язык интерфейса для TUIKit, введите нужный язык в [TUIGlobalization setCustomLanguage:@""]. После указания языка внутренние компоненты больше не будут следовать системному языку.
Выбор языка, значение:

```
@"zh-Hans" ,//Упрощённый китайский @"en", // Английский@"ar", // Арабский
```

> **Примечание:** Реестр кодов языков можно найти в [приложении](#code).

## Использование дополнительных языков/Настройка выражений перевода

Если ваше приложение требует добавления дополнительных языков или изменения определённых записей перевода, обратитесь к этому разделу. В этой главе показан процесс добавления новых языковых пакетов и настройки переводов на примере добавления корейского языкового пакета к компоненту **TUIGroup**.

### Добавление новых файлов языковых ресурсов

Все встроенные языковые пакеты хранятся в компоненте TUICore вашего проекта в Pods в виде шаблонов String файлов, расположенных по пути TUIKitLocalizable/Localizable/.

Создайте новый каталог и назовите его {language code}.lproj. В этом каталоге добавьте новый файл с именем Localizable.strings. Здесь ${language code} необходимо заменить кодом языка ISO 639-1. (Вы можете напрямую скопировать знакомый вам языковой файл, такой как zh-Hans.lproj, и просто изменить имя каталога.)
Если требуется поддержка совместимости для нескольких новых языков, просто дублируйте необходимые файлы, обеспечивая каждому его правильное назначение кодировки языка.

Например, при добавлении корейского языка создайте новый файл языкового ресурса: ko.lproj/Localizable.strings.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e01d2855167a11efb8ef5254002fd0a8.png)

### Персонализированный пользовательский перевод

На предыдущем этапе был успешно создан корейский файл ресурсов ko.lproj/Localizable.strings. Ключи в файлах языковых ресурсов разных языков остаются идентичными, что позволяет настраивать переводы конкретного контента.

### Следить за системным языком

После добавления пакетов ресурсов lproj для упрощённого китайского, традиционного китайского, английского, корейского, русского и украинского языков можно напрямую использовать TUIKit без необходимости дополнительных шагов.

Если есть другие языки, их необходимо добавить в + (NSString *)tk_localizableLanguageKey в TUIGlobalization.m. Наименование вновь добавленных языковых записей должно соответствовать [стандарту](#code), поскольку компонент будет адаптироваться к системному языку внутри.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e507bcbe167a11efb5545254007bbd8c.png)

> **Примечание:** Реестр кодов языков можно найти в [кодах языков ISO 639-1](#code).

### Указание отображаемого языка

Если требуется указать язык интерфейса TUIKit, введите нужный язык в [TUIGlobalization setCustomLanguage:@""]. После указания языка внутренние компоненты больше не будут следовать системному языку.
Выбор языка, представленный [кодами языков ISO 639-1](#code).
Используя корейский язык в качестве примера, [TUIGlobalization setCustomLanguage:@"ko"];

# Состояние входа

## Введение

`LoginState` — это центр управления состоянием, связанный с входом, специализирующийся на управлении состояниями входа и операциями. Он обеспечивает автоматический вход, ручной вход, выход и другие функции, а также поддерживает мониторинг статуса входа в реальном времени и обработку ошибок.

Hook имеет следующие особенности:

- **Управление автоматическим входом** — поддерживает автоматическую инициализацию и вход после конфигурации
- **Управление ручным входом** — предоставляет методы ручного входа и выхода
- **Синхронизация статуса в реальном времени** — обновляет статус входа и информацию об ошибках в реальном времени
- **Предотвращение повторного входа** — предотвращает повторный вход и интеллектуальную обработку при изменении конфигурации

## Параметр UseLoginState

| Значение | Тип | Описание |
| --- | --- | --- |
| config | LoginConfig | конфигурация входа |

- `config` — параметр входа для инициализации состояния входа. Прямое использование в компоненте включает автоматический вход.
- Определение `LoginConfig` выглядит следующим образом:

```
interface LoginConfig {  userID: string;   // ID пользователя  SDKAppID: number; // ID приложения  userSig: string;  // подпись пользователя  onSuccess?: () => void; // обратный вызов при успешном входе  onError?: (error: Error) => void; // обратный вызов при ошибке входа}
```

## Возвращаемое значение UseLoginState

| Значение | Тип | Описание |
| --- | --- | --- |
| status | LoginStatus | Текущий статус входа |
| error | Error \| null | Информация об ошибке при входе |
| currentConfig | LoginConfig \| null | текущая используемая конфигурация входа |
| login | (config: LoginConfig) => Promise<void> | Метод ручного входа |
| logout | () => Promise<void> | Метод выхода |

## Описание полей

### status

- **Тип**: `LoginStatus`
- **Описание**: текущий статус входа, используется для отслеживания каждого этапа процесса входа.
- **Определение enum**:

```
enum LoginStatus {  IDLE,     // Неактивно, не начато  LOADING,  // Вход в процессе  SUCCESS,  // Вход успешен  ERROR,    // Ошибка входа}
```

### error

- **Тип**: `Error | null`
- **Описание**: ошибка, произошедшая во время входа. Это `null`, когда вход успешен или ошибка не произошла.

### currentConfig

- **Тип**: `LoginConfig | null`
- **Описание**: текущая используемая конфигурация входа. Используется для сравнения изменений конфигурации и предотвращения повторного входа.

### login

- **Тип**: `(config: LoginConfig) => Promise<void>`
- **Описание**: метод ручного входа. Можно передать полную конфигурацию входа для входа.
- **Параметр**: `config` — объект конфигурации входа
- **Возвращаемое значение**: `Promise<void>` — Promise завершения входа

### logout

- **Тип**: `() => Promise<void>`
- **Описание**: метод выхода. Очищает состояние входа и вызывает API выхода SDK.
- **Возвращаемое значение**: `Promise<void>` — Promise завершения выхода

## Примеры использования

### Способ один: автоматический вход (рекомендуется)

Самый простой способ использования: введите конфигурацию входа для автоматической инициализации и входа.

```
import React from 'react';import { useLoginState, LoginStatus } from './states/LoginState/LoginState';function App() {  const { status, error, logout } = useLoginState({    userID: 'your-userID-here',    SDKAppID: 0, // your-SDKAppID-here, this is number type!    userSig: 'your-userSig-here',    onSuccess: () => console.log('ð Вход успешен.'),    onError: (error) => console.error('â Ошибка входа:', error),  });  // Отображать разное содержимое в зависимости от статуса входа  if (status === LoginStatus.LOADING) {    return <div className="loading">Вход в процессе...</div>;  }  if (status === LoginStatus.ERROR) {    return (      <div className="error">        <p>Ошибка входа: {error?.message}</p>        <button onClick={() => window.location.reload()}>Повторить</button>      </div>    );  }  if (status !== LoginStatus.SUCCESS) {    return <div className="waiting">Ожидание входа...</div>;  }  return (    <div className="app">      <header className="app-header">        <h1>приложение чата</h1>        <button onClick={logout} className="logout-btn">          Выход        </button>      </header>      <UIKitProvider>              </UIKitProvider>    </div>  );}export default App;
```

### Способ 2: управление ручным входом

Этот подход обеспечивает больше контроля и позволяет вручную запускать вход при необходимости.

```
import React, { useState } from 'react';import { useLoginState, LoginStatus } from './states/LoginState/LoginState';function ManualLoginApp() {  const { login, logout, status, error, client } = useLoginState(); // config не передан  const [loginForm, setLoginForm] = useState({    userID: '',    SDKAppID: 0,    userSig: '',  });  const handleLogin = async () => {    try {      await login({        ...loginForm,        onSuccess: () => {          console.log('Вход успешен.');          // Здесь вы можете выполнить операции после входа        },        onError: (error) => {          console.error('Ошибка входа:', error);          // Здесь вы можете исправить ошибки входа        },      });    } catch (error) {      console.error('Исключение входа:', error);    }  };  const handleLogout = async () => {    try {      await logout();      console.log('Выход успешен.');    } catch (error) {      console.error('Ошибка выхода:', error);    }  };  const isLoggedIn = status === LoginStatus.SUCCESS;  return (    <div className="manual-login-app">      {!isLoggedIn ? (        <div className="login-form">          <h2>Вход</h2>          <div className="form-group">            <label>ID пользователя:</label>            <input              type="text"              value={loginForm.userID}              onChange={(e) => setLoginForm(prev => ({ ...prev, userID: e.target.value }))}              placeholder="Введите ID пользователя"            />          </div>          <div className="form-group">            <label>SDK App ID:</label>            <input              type="number"              value={loginForm.SDKAppID}              onChange={(e) => setLoginForm(prev => ({ ...prev, SDKAppID: Number(e.target.value) }))}              placeholder="Введите SDK App ID"            />          </div>          <div className="form-group">            <label>Подпись пользователя:</label>            <textarea              value={loginForm.userSig}              onChange={(e) => setLoginForm(prev => ({ ...prev, userSig: e.target.value }))}              placeholder="Введите подпись пользователя"            />          </div>          <button             onClick={handleLogin}            disabled={status === LoginStatus.LOADING}            className="login-btn"          >            {status === LoginStatus.LOADING ? 'Вход в процессе...' : 'Вход'}          </button>          {error && (            <div className="error-message">              Ошибка: {error.message}            </div>          )}        </div>      ) : (        <div className="app-content">          <div className="app-header">            <h1>приложение чата</h1>            <button onClick={handleLogout} className="logout-btn">              Выход            </button>          </div>          <ChatApp client={client} />        </div>      )}    </div>  );}export default ManualLoginApp;
```

## Меры предосторожности

1. **Проверка параметров**: убедитесь, что переданные `userID`, `SDKAppID` и `userSig` действительны.
2. **Обработка ошибок**: всегда обрабатывайте возможные ошибки входа
3. **Мониторинг статуса**: разумно используйте статус входа для управления отображением интерфейса
4. **Очистка ресурсов**: вызывайте метод `logout` при удалении компонента.
5. **Безопасность**: не жестко кодируйте конфиденциальную информацию на клиенте. `userSig` должна быть получена с сервера.

# AI интеграция (MCP)

Значительно ускорьте ваш рабочий процесс, превратив ваш AI IDE в эксперта по TRTC.

Забудьте о шаблонном коде и позвольте Model Context Protocol (MCP) выполнять тяжелую работу — scaffolding, настройку SDK и интеграцию пользовательского интерфейса.

## Смотрите в действии

Представьте себе, что вы печатаете одну подсказку, и весь ваш Chat UI оживает.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/8d87986a07dd11f1a751525400380f7d.gif)

## Дорожная карта реализации

Чтобы успешно интегрировать Chat с использованием AI, просто следуйте этим 3 шагам:

1. **Настройте MCP сервер**
2. **Запустите интеграцию**
3. **Запустите ваш проект**

### Предварительные требования

- **Node.js:** версия v18.x или v20.x (рекомендуется LTS). Запустите `node -v` для проверки.
- **Аккаунт TRTC:** см. [Активация сервиса](https://trtc.io/document/74511) для создания приложения Chat и получения `SDKAppID` и `secretKey`.

### 1. Настройте MCP сервер

Включите инструменты MCP в ваш предпочитаемый редактор. Рекомендуется настроить это в **корневом каталоге** вашего проекта.

Cursor

Trae

CodeBuddy

1. [Загрузите Cursor](https://cursor.com/) (если уже загружен, пропустите этот шаг).
2. Перейдите в корень проекта.
3. Создайте или обновите `.cursor/mcp.json` следующим образом, заполнив `SDKAPPID` и `SECRETKEY`:

```
{  "mcpServers": {    "tencent-rtc": {      "command": "npx",      "args": ["-y", "@tencent-rtc/mcp"],      "env": {        "SDKAPPID": "YOUR_SDKAPPID",        "SECRETKEY": "YOUR_SECRET_KEY"      }    }  }}
```

4. Сохраните изменения. Когда Cursor предложит "New MCP server detected", нажмите **Enable**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/612f8735033211f1880952540073fd3b.png)

5. Перейдите в **Settings - Tools & MCP** в Cursor, чтобы проверить, успешно ли включен инструмент MCP.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/611f119f033211f18b07525400ecee81.png)

1. [Загрузите Trae](https://www.trae.ai/) (если уже загружен, пропустите этот шаг).
2. Нажмите **Settings** > **MCP.**

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/616fd85a033211f1a92352540097cba1.png)

3. Нажмите **Add** > выберите Configure Manually, затем нажмите **Raw Config (JSON)** для обновления раздела `mcpServers`.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/6120c64f033211f1a92352540097cba1.png)

```
{  "mcpServers": {    "tencent-rtc": {      "command": "npx",      "args": ["-y", "@tencent-rtc/mcp"],      "env": {        "SDKAPPID": "YOUR_SDKAPPID",        "SECRETKEY": "YOUR_SECRET_KEY"      }    }  }}
```

4. Сохраните изменения. Выберите стандартный **Builder with MCP** редактора или добавьте MCP в ваш пользовательский Builder.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/611f890d033211f1ac395254001d6acc.png)

1. [Загрузите CodeBuddy](https://www.codebuddy.com/) (если уже загружен, пропустите этот шаг).
2. Перейдите в **Settings** > **Add MCP** для открытия `settings.json` и настройки MCP.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/6128c003033211f19589525400370dda.png)

3. Обновите раздел `mcpServers` и сохраните изменения.

```
{  "mcpServers": {    "tencent-rtc": {      "command": "npx",      "args": ["-y", "@tencent-rtc/mcp"],      "env": {        "SDKAPPID": "YOUR_SDKAPPID",        "SECRETKEY": "YOUR_SECRET_KEY"      }    }  }}
```

4. Проверьте, успешно ли загружен инструмент MCP в tools.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/6133bee8033211f18b07525400ecee81.png)

### 2: Запустите интеграцию

Теперь, когда ваша среда настроена и защищена, используйте естественный язык для построения вашего приложения.

Scaffolding нового проекта

Расширение существующего приложения

Идеально подходит для начала с нуля.

**Примеры подсказок**:

- "Используйте компонент TUIKit для создания Vue3 приложения чата."
- "Используйте компонент TUIKit для создания iOS приложения чата."

Направьте интеграцию на конкретные файлы.

**Пример подсказки**:

- "Интегрируйте компонент Vue3 Chat window в src/views/ChatPage.tsx"

### 3: Запустите и проверьте

#### Создайте тестовые аккаунты

1. Для проверки совместимости отправки и получения сообщений необходимо создать два аккаунта. Позвольте AI сгенерировать userSig для двух тестовых аккаунтов.

**Пример подсказки:**

```
Используйте инструмент MCP для генерации userSig для test001 и test002.
```

2. После того как AI выведет userSig, позвольте AI автоматически заполнить всю информацию для входа в код.

**Пример подсказки:**

```
Заполните userSig test001 и test002 в коде.
```

#### Запустите ваш проект

После завершения генерации кода AI запустите сервер разработки для тестирования приложения.

Вы можете просто попросить AI "запустить проект".

## Устранение неполадок и часто задаваемые вопросы

| **Проблема** | **Причина** | **Решение** |
| --- | --- | --- |
| MCP сервер оффлайн | Отсутствует путь Node/NPM. | Запустите `npx -y @tencent-rtc/mcp` вручную в терминале для проверки установки и доступности сервера. |
| Ошибки авторизации | Неверный UserSig. | Подсказка AI: "Переgenerate the UserSig using the MCP tool and update the config." |
| Отказано в разрешении при scaffolding | Недостаточные разрешения папки. | Проверьте разрешения папки. В macOS/Linux попробуйте запустить IDE или команду с повышенными привилегиями (или sudo для команд терминала). |

### Как вручную установить MCP сервер, если IDE не может?

Вы всегда можете обойти автоматическую установку IDE, запустив следующую команду непосредственно в корне вашего проекта:

```
npx -y @tencent-rtc/mcp
```

### Профессиональные советы для лучших результатов AI

Как убедиться, что AI каждый раз выбирает правильный инструмент?

Модели AI работают лучше с конкретными ключевыми словами.

- **Хорошая подсказка**: "Создайте приложение React чата с использованием TUIKit. Включите компоненты ConversationList и Chat Window."
- **Ключевой вывод**: Всегда упоминайте "TUIKit", "React" (или Vue) и конкретные имена компонентов в вашей подсказке.

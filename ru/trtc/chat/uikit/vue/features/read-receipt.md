# Уведомление о прочтении сообщений

## Описание

После включения функции "Уведомление о прочтении сообщений" компонент TUIChat будет отслеживать прокрутку сообщений. Когда непрочитанные сообщения появляются в видимой зоне окна чата получателя, система автоматически отправляет уведомления о прочтении отправителю, тщательно отслеживая статус прочтения каждого сообщения.

> **Примечание:** Начиная с версии v2.0.0, TUIKit поддерживает уведомления о прочтении сообщений как для групповых чатов, так и для личных переписок. **Эта функция доступна исключительно в Pro edition, Pro Plus edition и Enterprise edition. Пожалуйста, [**приобретите Pro edition, Pro Plus edition или Enterprise edition**](https://console.trtc.io/subscription/buy/chat?packType=pro)**для использования.**

## Как включить уведомление о прочтении сообщений

### Указание типа группы, поддерживающего уведомления о прочтении

Для уведомлений о прочтении групповых сообщений сначала перейдите в [Chat Console > Chat > Configuration > Group Configuration > Read receipts for group messages](https://console.trtc.io/chat/qun-setting) и установите тип группы, который поддерживает уведомления о прочтении.

> **Примечание:** Группы Live Chat (AVChatRoom) или Communities (Community) **не поддерживают** функцию уведомления о прочтении.

### Управление на стороне пользователя для включения/отключения уведомления о прочтении

> **Примечание:** После настройки уведомления о прочтении для групповых чатов в консоли функция уведомления включена по умолчанию. Если это не требуется специально, нет необходимости в переключении на стороне пользователя.

На стороне пользователя также поддерживается ручное включение/отключение функции уведомления о прочтении (включена по умолчанию). Однако, если функция отключена, другие не смогут видеть, прочитали ли они сообщение (т. е. уведомления о прочтении не будут отправляться), и вы не сможете видеть, прочитали ли другие ваши сообщения (статус прочтения отправленных сообщений не будет отображаться).

После успешной авторизации используйте API `TUIUserService.switchMessageReadStatus(isDisplay: boolean)` для управления этим переключателем.

```
import { TUIUserService } from "@tencentcloud/chat-uikit-engine";TUIUserService.switchMessageReadStatus(true);  // ВключитьTUIUserService.switchMessageReadStatus(false); // Отключить
```

## Дополнительные материалы

> **Примечание:** Содержание ниже предназначено только в качестве справочного материала. Функция уведомления о прочтении уже включена в флагманский TUIKit. Пользователям не требуется инициировать применение этой функции.

### Правила уведомления о прочтении для групповых чатов и личной переписки

После активации функции уведомления о прочтении поле `needReadReceipt` объекта `Message` предустановлено как `true`. Когда сообщение находится в видимой позиции списка сообщений другого пользователя, будет отправлено уведомление о прочтении. Однако следует учитывать, что правила для личных сообщений и групповых сообщений различаются до и после активации функции уведомления о прочтении.

#### Правила уведомления о прочтении в групповых сообщениях

1. До активации функции уведомления о прочтении в групповых сообщениях:

Статус прочтения не отображается.

2. После включения уведомления о прочтении в групповом чате:

Получайте информацию о количестве прочитавших и непрочитавших на основе поля `Message`'s `readReceiptInfo`.

  - Для количества прочитавших 0: отобразить "Не прочитано".
  - Для количества непрочитавших 0: отобразить "Все прочитали".
  - В противном случае отобразить "x человек прочитали", где x — количество прочитавших.

#### Правила уведомления о прочтении в личных переписках

1. До включения уведомления о прочтении в личных переписках:

Отображается статус прочтения, но это полное прочтение. Когда пользователь переходит в беседу, независимо от того, видит ли он сообщение, все непрочитанные сообщения будут отмечены как прочитанные. Определите, прочитано ли сообщение или нет, на основе поля `Message`'s `isPeerRead`.

2. После включения уведомления о прочтении в личных переписках:

На основе поля `Message` `readReceiptInfo.isReceiptPeerRead` (логический тип) можно определить статус прочтения — "прочитано" или "не прочитано".

## Часто задаваемые вопросы

### 1. Ошибка: использование этого API не поддерживается вашим пакетом. Пожалуйста, обновитесь до премиум-версии.

Функция "Уведомление о прочтении групповых сообщений" поддерживается только флагманским пакетом. Сообщение об ошибке означает, что ваш текущий пакет не поддерживает эту функцию.

### 2. Как отключить функцию прочтения?

Пожалуйста, обратитесь к разделу [Управление на стороне пользователя для отключения уведомления о прочтении](https://www.tencentcloud.com/document/product/1047/58661#f81aa831-f9a8-40a1-a581-f828f4393361) этого документа и используйте `TUIUserService.switchMessageReadStatus(isDisplay: boolean)` для отключения функции прочтения.

```
import { TUIUserService } from "@tencentcloud/chat-uikit-engine";TUIUserService.switchMessageReadStatus(false);
```

# Отзыв отправленного уведомления

Если содержимое отправленного уведомления для всех пользователей/тегов некорректно, это может негативно повлиять на продукт при просмотре или клике конечным пользователем. Требуется своевременная обработка. Вы можете выбрать отзыв отправленного уведомления.

## Обзор функции

- Прерывание: выпуск задач отправки занимает некоторое время, и доставка учетным записям, которым уведомление еще не было выпущено, будет прервана.
- Перезапись: если выпущенные учетные записи получили офлайн-уведомление, поддерживается перезапись уведомления.
- Этот интерфейс поддерживает прерывание/перезапись задач отправки для всех пользователей/тегов. В дальнейшем прерывание/перезапись именуется отзывом по умолчанию.
- Период действительности отзыва составляет 24 часа, начиная с момента инициирования задачи. Задачи отправки, которым более 24 часов, не могут быть отозваны.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/timpush/revoke?usersig=xxx&identifier=admin&sdkappid=88888888&random=99999999&contenttype=json
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса: HTTPSМетод запроса: POST |
| xxxxxx | Выделенный домен, соответствующий стране/региону вашего SDKAppID.Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Сеул: `adminapikr.im.qcloud.com`Франкфурт: `adminapiger.im.qcloud.com`Кремниевая долина: `adminapiusa.im.qcloud.com`Джакарта: `adminapiidn.im.qcloud.com` |
| v4/timpush/revoke | API запроса |
| usersig | Подпись, сгенерированная учетной записью администратора приложения. Дополнительные сведения см. в разделе [Создание UserSig](https://www.tencentcloud.com/document/product/1047/34385) |
| identifier | Должна быть учетная запись администратора приложения. Дополнительные сведения см. в разделе [Администраторы приложений](https://www.tencentcloud.com/document/product/1047/33517) |
| sdkappid | SdkAppid, присвоенный консолью Chat при создании приложения |
| random | Случайное 32-битное беззнаковое целое число в диапазоне от 0 до 4294967295 |
| contenttype | Фиксированное значение: `json` |

### Частота вызова

Вызов этого интерфейса ограничен 1 разом в секунду.

### Пример пакета запроса

```
{    "TaskId": "660cc447_537ed82a_200000cd7ee17f5_84035729_bc614e",  // Push task taskId sent within 24 hours     "OfflinePushInfo": {        "Title": "Recall Title",        "Desc": "The other party has recalled a message",        "Ext": "{\\"entity\\":{\\"key1\\":\\"value1\\",\\"key2\\":\\"value2\\"}}" // Passthrough field, using JSON format string for push    }}
```

> **Примечание:** Поддерживаемые поставщики для перезаписи офлайн-уведомления: APNS/Google FCM/Huawei/Honor. Перезапись офлайн-уведомления не поддерживается для других поставщиков. (Режим уведомлений Google FCM поддерживает перезапись, режим данных пока не поддерживает перезапись.) При отзыве, если получатель находится на переднем плане и прочитал уведомление по умолчанию (сообщение панели уведомлений), это офлайн-уведомление не будет перезаписано.

### Описание полей пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| TaskId | String | Обязательно | ID задачи отправки для всех пользователей/тегов. |
| OfflinePushInfo | Object | Обязательно | Конфигурация уведомления офлайн-отправки, см. [Описание OfflinePushInfo](https://www.tencentcloud.com/document/product/1047/77715). |

### Пример пакета ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработанного запроса:OK: указывает успешную обработкуFAIL: указывает на сбой |
| ErrorCode | Integer | Идентификатор ошибки:0 указывает на успехНе ноль указывает на сбой |
| ErrorInfo | String | Сообщение об ошибке |

## Коды ошибок

Если не происходит ошибка сети (например, ошибка 502), код возврата HTTP для этого интерфейса всегда равен 200. Фактический код ошибки и информация об ошибке представлены через ErrorCode и ErrorInfo в полезной нагрузке ответа. Для общих кодов ошибок (60000–79999) см. документ [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).

| Код ошибки | Описание |
| --- | --- |
| 90001 | Ошибка при парсинге формата JSON. Убедитесь, что пакет запроса соответствует спецификации JSON. |
| 90009 | Запрос требует прав администратора приложения. |
| 90049 | Недействительный TaskId для отзыва, нет записей отправки. Используйте интерфейс [Отправка для всех пользователей/тегов](https://www.tencentcloud.com/document/product/1047/60561) или [Одиночная отправка](https://www.tencentcloud.com/document/product/1047/67553) и только возвращаемый TaskId можно использовать для отзыва. |
| 90050 | Дублирование отзыва, задача отправки, которая уже была отозвана, не может быть отозвана снова. |
| 90051 | Отзыв слишком частый, лимит частоты отзыва составляет 1 раз в секунду. |
| 90052 | Превышен период действительности отзыва, отзыв должен быть выполнен в течение 24 часов. Задачи отправки, которым более 24 часов, не могут быть отозваны. |
| 90053 | Отзыв недействителен. Задача отправки не указала отсутствие роамингового/непрочитанного (OnlineOnlyFlag=0), но OfflinePushInfo не была включена при отзыве. |
| 91000 | Ошибка внутреннего сервиса, пожалуйста, повторите попытку. |

# Импорт профилей тем

## Обзор функции

Этот API позволяет администратору приложения импортировать данные тем без запуска обратных вызовов или доставки уведомлений. Когда вашему приложению необходимо миграция на Chat из другой системы мгновенного обмена сообщениями, вы можете использовать этот API для импорта существующих данных тем.

## Описание вызова API

### Применимые типы групп

| Тип группы | Поддержка этого API RESTful |
| --- | --- |
| Private | Нет |
| Public | Нет |
| ChatRoom | Нет |
| AVChatRoom | Нет |
| Community | Этот API применяется только к сообществам с поддержкой тем. |

Это предустановленные типы групп в Chat. Дополнительную информацию см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

> **Примечание:** Для использования функции тем необходимо перейти на [консоль](https://console.trtc.io/chat/qun-setting), выбрать **Конфигурация функций** > **Конфигурация группы** > **Конфигурация функций группы** > **Сообщество**, включить функцию сообщества, а затем включить функцию тем.

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/import_topic?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны только измененные параметры при вызове этого API. Другие параметры см. в разделе [Обзор API RESTful](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/group_open_http_svc/import_topic | Запрос API |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Дополнительную информацию см. в разделе **Администратор приложения** в разделе [Аутентификация при входе](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Дополнительную информацию см. в разделе [Генерация UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса. Значение фиксировано на `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Базовый формат**
Укажите группу, к которой принадлежит импортируемая тема. Вы можете использовать `CreateTime` для указания времени создания темы.

```
{"GroupId": "@TGS#_@TGS#cBZXAIIM62CV",     // (Обязательно) ID группы, к которой принадлежит импортируемая тема"TopicName": "test_topic3",           // (Дополнительно) Название темы"CreateTime": 1448357837              // (Дополнительно) Время создания темы}
```

- **Указание других дополнительных полей**
Укажите дополнительные поля, такие как `Introduction` и `Notification`.

```
{  "Type": "Community", // (Дополнительно) Тип группы, к которой принадлежит тема, должен быть `Community`  "GroupId": "@TGS#_@TGS#cBZXAIIM62CV",     // (Обязательно) ID группы, к которой принадлежит импортируемая тема  "TopicName": "test_topic3",           // (Дополнительно) Название темы  "Introduction": "This is topic Introduction", // (Дополнительно) Введение в тему  "Notification": "This is topic Notification", // (Дополнительно) Уведомление по теме  "FaceUrl": "http://this.is.face.url",  "CreateTime": 1448357837 // (Дополнительно) Время создания темы. Если это поле не указано, время создания по умолчанию - время запроса.}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| Type | String | Нет | Тип группы, к которой принадлежит тема. В настоящее время поддерживается только группа Community. |
| GroupId | String | Да | ID группы, к которой принадлежит импортируемая тема. |
| TopicId | String | Нет | Чтобы упростить ID тем и сделать их легко запоминающимися, Tencent Cloud позволяет приложениям настраивать ID тем при создании темы через API RESTful. Дополнительную информацию см. [здесь](https://intl.cloud.tencent.com/document/product/1047/33529#customize). |
| TopicName | String | Да | Название темы, максимальная длина которого составляет 30 байт. Это поле кодируется в UTF-8, и один китайский символ занимает три байта. |
| From_Account | uint64 | Нет | Учетная запись пользователя, который хочет создать тему. |
| CustomString | String | Нет | Пользовательская строка, которая может содержать до 3000 байт, закодированная в UTF-8. |
| FaceUrl | String | Нет | URL фотографии профиля темы, максимальная длина которого составляет 100 байт. |
| Notification | String | Нет | Объявление по теме, максимальная длина которого составляет 300 байт. Это поле кодируется в UTF-8, и один китайский символ занимает три байта. |
| Introduction | String | Нет | Введение в тему, максимальная длина которого составляет 240 байт. Это поле кодируется в UTF-8, и один китайский символ занимает три байта. |
| CreateTime | Integer | Нет | Время создания темы |

### Пример ответа

- **Базовый формат**

```
{  "ActionStatus": "OK",  "ErrorInfo": "",  "ErrorCode": 0,  "TopicId": "@TGS#_@TGS#cBZXAIIM62CV@TOPIC#_@TOPIC#cTCCCIIM62CW"}
```

- **Указание других дополнительных полей**

```
{  "ActionStatus": "OK",  "ErrorInfo": "",  "ErrorCode": 0,  "TopicId": "@TGS#_@TGS#cBZXAIIM62CV@TOPIC#_@TOPIC#c5CCCIIM62CW"}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки. `0`: Успешно; другие значения: Ошибка |
| ErrorInfo | String | Информация об ошибке |
| TopicId | String | ID темы после успешного создания, который назначается сервером Chat или указывается пользователями. |

## Коды ошибок

Возвращаемый код состояния HTTP для этого API всегда 200, если не возникает ошибка сети (например, ошибка 502). Конкретный код ошибки и информацию можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для публичных кодов ошибок (от 60000 до 79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Попробуйте снова. |
| 10003 | Недопустимое слово команды. |
| 10004 | Недопустимый параметр. Проверьте описание ошибки и устраните проблему. |
| 10007 | Недостаточно прав на операцию. Проверьте параметры запроса в соответствии с сообщением об ошибке. |
| 10010 | Текущая группа не существует или была удалена. |
| 10015 | Запрошенный ID группы недопустим. Проверьте параметр запроса в соответствии с сообщением об ошибке. |
| 10021 | ID темы уже используется. Укажите другой ID темы. |
| 10037 | Количество созданных предоплачиваемых тем превышает лимит. Удалите некоторые темы или обновите свой сервис. Дополнительную информацию см. в разделе [Ценообразование](https://intl.cloud.tencent.com/document/product/1047/34350). |
| 11000 | Текущая группа не поддерживает функцию темы сообщества. Для использования этой функции необходимо приобрести [Pro edition, Pro Plus edition или Enterprise edition](https://www.tencentcloud.com/document/product/1047/34577) и [включить её на консоли](https://intl.cloud.tencent.com/document/product/1047/34419). |
| 80001 | Не пройдена проверка безопасности. Проверьте параметры запроса в соответствии с сообщением об ошибке. |
| 80005 | Не пройдена проверка безопасности: истекло время ожидания проверки безопасности. |

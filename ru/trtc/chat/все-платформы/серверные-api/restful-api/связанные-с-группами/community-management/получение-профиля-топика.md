# Получение профиля топика

## Обзор функции

Этот API используется для получения профиля топика.

## Описание вызова API

### Применимые типы групп

| ID типа группы | Поддерживается ли этот RESTful API |
| --- | --- |
| Private | Нет |
| Public | Нет |
| ChatRoom | Нет |
| AVChatRoom | Нет |
| Community | Этот API применяется только к сообществам с поддержкой топиков. |

Это предустановленные типы групп в Chat. Для получения дополнительной информации см. [Group System](https://intl.cloud.tencent.com/document/product/1047/33529).

> **Примечание**Для использования функции топиков необходимо перейти в [консоль](https://console.trtc.io/chat/qun-setting), выбрать **Feature Configuration** > **Group configuration** > **Group feature configuration** > **Community**, включить функцию сообщества, а затем включить функцию топиков.

### Пример URL запроса

```
https://xxxxxx/v4/million_group_open_http_svc/get_topic?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [RESTful API Overview](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, в котором находится ваш SDKAppID.Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Сеул: `adminapikr.im.qcloud.com`Токио: `adminapijpn.im.qcloud.com`Франкфурт: `adminapiger.im.qcloud.com`Кремниевая долина: `adminapiusa.im.qcloud.com`Джакарта: `adminapiidn.im.qcloud.com` |
| v4/million_group_open_http_svc/get_topic | API запроса |
| sdkappid | `SDKAppID`, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Для получения дополнительной информации см. раздел **App Admin** в [Login Authentication](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробнее см. [Generating UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса. Фиксированное значение: `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Базовый формат**

Используется для получения деталей топика. Запрос содержит ID группы и список всех топиков для получения.

```
{    "GroupId":"@TGS#_@TGS#cQVLVHIM62CJ",	// Необходимо получить ID группы топика, обязательный параметр.    "From_Account":"1400187352"	// Получить информацию указанного пользователя в топике, обязательный параметр.}
```

- **Указание формы списка топиков для получения**

```
{  "GroupId":"@TGS#_@TGS#cQVLVHIM62CJ",	// Необходимо получить ID группы топика, обязательный параметр.  "From_Account":"1400187352",	// Получить информацию указанного пользователя в топике, обязательный параметр.  "TopicIdList":[		// Указать список топиков для получения, опциональный параметр.      "@TGS#_@TGS#cQVLVHIM62CJ@TOPIC#_@TOPIC#cRTE3HIM62C5"  ]}
```

- **Использование фильтра для получения указанной формы информации**
Вы можете указать поле топика для получения в `ResponseFilter`. Если установлен фильтр, поля, не указанные в фильтре, не будут получены.

```
{  "GroupId":"@TGS#_@TGS#cQVLVHIM62CJ",	// Необходимо получить ID группы топика, обязательный параметр.  "From_Account":"1400187352",	// Получить информацию указанного пользователя в топике, обязательный параметр.  "TopicIdList":[	// Указать список топиков для получения, опциональный параметр.      "@TGS#_@TGS#cQVLVHIM62CJ@TOPIC#_@TOPIC#cRTE3HIM62C5"  ],  "ResponseFilter":[	// Фильтр ответа. Указать информацию топика для получения, опциональный параметр.      "TopicName",// Название топика        "FaceUrl",// URL фото профиля топика      "Notification",// Уведомление топика      "Introduction"// Описание топика  ]}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupId | String | Обязательно | Необходимо получить ID группы топика. |
| From_Account | String | Обязательно | Получить информацию указанного пользователя в топике. |
| TopicIdList | Array | Опционально | Необходимо получить список топиков. Если оставить пусто, будет получена вся информация о топиках группы. |
| ResponseFilter | Array | Опционально | Фильтр ответа. Указать информацию топика для получения. |

### Пример ответа

- **Базовая форма; указание формы списка топиков для получения**

```
{  "ActionStatus": "OK",  "ErrorInfo": "ok",// Параметр `ErrorInfo` здесь не имеет значения. Необходимо проверить параметр `ErrorInfo` каждого топика.  "ErrorCode": 0, // Параметр `ErrorCode` здесь не имеет значения. Необходимо проверить параметр `ErrorCode` каждого топика.  "TopicAndSelfInfo": [	// Возвращаемый результат - это массив информации о топиках. Здесь для простой демонстрации указан только один топик.      {          "ErrorCode": 0,          "ErrorInfo": "ok",// Результат возврата для этого топика          "SelfInfo": {	 // Информация указанного пользователя в топике              "ShutUpTime": 0,// Период молчания пользователя              "MsgSeq": 0,// Порядковый номер прочитанного сообщения участника              "MsgFlag": "AcceptAndNotify"	          },          "TopicInfo": {	// Возвращенная информация о топике              "TopicName": "TestTopicName",	// Название топика              "GroupID": "@TGS#_@TGS#cQVLVHIM62CJ",	// ID группы топика              "FaceUrl": "http://this.is.new.face.url",// URL фото профиля топика              "TopicId": "@TGS#_@TGS#cQVLVHIM62CJ@TOPIC#_@TOPIC#cRTE3HIM62C5",	// ID топика              "NextMsgSeq": 1,	              "Notification": "Notification",// Уведомление топика              "Introduction": "Introduction",// Описание топика              "CreateTime": 1658839448,// Время создания топика в UTC              "LastMsgTime": 0,// Время UTC отправки последнего сообщения в топике              "MuteAllMember": "Off",  // Флаг, указывающий на отключение звука для всех в топике. `On`: Да; `Off`: Нет              "CustomString": "This is a customs string."	// Пользовательская строка в топике          }      }  ]}
```

- **Использование фильтра для получения указанной формы информации о топике**

```
{    "ActionStatus": "OK",    "ErrorInfo": "ok",// Параметр `ErrorInfo` здесь не имеет значения. Необходимо проверить параметр `ErrorInfo` каждого топика.    "ErrorCode": 0, // Параметр `ErrorCode` здесь не имеет значения. Необходимо проверить параметр `ErrorCode` каждого топика.    "TopicAndSelfInfo": [	// Возвращаемый результат - это массив информации о топиках. Здесь для простой демонстрации указан только один топик.        {            "ErrorCode": 0,            "ErrorInfo": "ok",// Результат возврата для этого топика            "SelfInfo": {	 // Информация указанного пользователя в топике                "ShutUpTime": 0,// Период молчания пользователя                "MsgSeq": 0,	                "MsgFlag": "AcceptAndNotify"            },            "TopicInfo": {	// Возвращенная информация о топике                "TopicName": "TestTopicName",	// Название топика                "GroupID": "@TGS#_@TGS#cQVLVHIM62CJ",	// ID группы топика                "FaceUrl": "http://this.is.new.face.url",// URL фото профиля топика                "TopicId": "@TGS#_@TGS#cQVLVHIM62CJ@TOPIC#_@TOPIC#cRTE3HIM62C5",	// ID топика                "Notification": "Notification",// Уведомление топика                "Introduction": "Introduction"// Описание топика            }        }    ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: ошибка |
| ErrorInfo | String | Информация об ошибке |
| TopicAndSelfInfo | Array | Возвращаемый результат - это массив информации о топиках, содержащий базовую информацию о топике и информацию указанного пользователя в топике. |

## Коды ошибок

HTTP код состояния, возвращаемый этим API, всегда 200, если не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. [Error Codes](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | **Описание** |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10003 | Неверная команда. |
| 10004 | Неверный параметр. Проверьте описание ошибки и устраните проблему. |
| 10006 | Операция превышает лимит частоты. Пожалуйста, уменьшите частоту вызовов. Эта ошибка обычно вызвана слишком большим увеличением групп в один день или слишком частыми вызовами для получения всех групп приложения. |
| 10007 | Недостаточны права на операцию. Проверьте параметры запроса на основе сообщения об ошибке. |
| 10008 | Запрос недействителен, вероятно, потому что не удалось проверить информацию подписи в запросе. Пожалуйста, повторите попытку или [отправьте тикет](https://console.tencentcloud.com/workorder/category?level1_id=29&level2_id=40&source=0&data_title=%E4%BA%91%E9%80%9A%E4%BF%A1%20%20IM&step=1). |
| 10010 | Неверный запрос. Группа или топик был удален. |
| 11000 | Текущая группа не поддерживает функцию топиков сообщества. Для использования этой функции необходимо приобрести [Pro edition, Pro Plus edition или Enterprise edition](https://www.tencentcloud.com/document/product/1047/34577) и [включить его в консоли](https://intl.cloud.tencent.com/document/product/1047/34419). |
| 110003 | Запрос не выполнен. Топик был удален. |

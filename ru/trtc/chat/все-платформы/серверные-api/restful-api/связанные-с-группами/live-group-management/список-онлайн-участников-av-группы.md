# Получение списка участников онлайн в аудио-видео группе

## Обзор функции

Этот API используется для получения списка участников онлайн в аудио-видео группе на основе ID группы. Для использования этой функции необходимо приобрести издание Pro, Pro Plus или Enterprise, перейти в консоль Chat, выбрать **Конфигурация функций группы** и включить **Список участников онлайн аудио-видео группы**. В других редакциях плана после вызова этого API будет возвращен список до 30 последних участников группы.

## Описание вызова API

### Применимые типы групп

| ID типа группы | Поддержка RESTful API |
| --- | --- |
| Private | Нет. Аналогично рабочим группам (Work) в новой версии. |
| Public | Нет |
| ChatRoom | Нет. Аналогично группам совещаний (Meeting) в новой версии. |
| AVChatRoom | Да |
| Community | Да |

Это предустановленные типы групп в Chat. Дополнительную информацию см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

> **Примечание:** Для использования этой функции необходимо [приобрести издание Pro, Pro Plus или Enterprise](https://www.tencentcloud.com/document/product/1047/34577) и включить функцию **Список участников онлайн аудио-видео группы** на странице **Конфигурация функций группы** в консоли. В других редакциях плана после вызова этого API будет возвращен список до 30 последних участников группы. Список участников группы обновляется один раз каждые 10 секунд. Если в аудио-видео группе более 1000 участников, после вызова этого API будут возвращены только 1000 последних участников онлайн. Когда размер группы больше или равен 300, или когда есть веб-пользователи, количество онлайн-пользователей обновляется через 10–20 секунд после присоединения или выхода пользователя из группы. Когда размер группы менее 300 и нет веб-пользователей, количество онлайн-пользователей обновляется немедленно после присоединения или выхода пользователя из группы.

### Пример URL запроса

```
https://xxxxxx/v4/group_open_avchatroom_http_svc/get_members?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

Этот API используется для получения количества участников онлайн в аудио-видео группе (AVChatRoom), которое собирается один раз каждые 10 секунд.

```
{    "GroupId":"@TGS#a6I4ZUUGO",    "Timestamp": 0}
```

Будут получены только участники группы, отмеченные как `1000`.

```
{    "GroupId":"@TGS#a6I4ZUUGO",    "Mark": 1000,    "Timestamp": 0}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID целевой группы |
| Timestamp | Integer | Да | Для первого запроса значение равно `0`. Если аудио-видео группа имеет 500 или менее участников, бэкенд возвращает всех участников и `NextTimestamp` со значением `0`. Если аудио-видео группа имеет более 500 участников, бэкенд возвращает `NextTimestamp` с ненулевым значением, которое используется как значение `Timestamp` для следующего запроса на получение оставшихся участников. |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorCode": 0,    "ErrorInfo": "",    "MemberList": [        {            "JoinTime": 1657773110,            "Member_Account": "teacher2",            "NickName": "Teacher t02",            "Avatar": "http://www.example.com/teacher2.jpg"        }    ],    "NextTimestamp": 0}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки. `0`: Успешно; другие значения: ошибка |
| ErrorInfo | String | Информация об ошибке |
| MemberList | Object | Онлайн-аккаунты группы. `Member_Account` — это `UserID` пользователя, а `JoinTime` — время присоединения пользователя к группе. |
| NextTimestamp | Integer | Флаг постраничного получения. Если значение не равно `0`, можно получить больше участников. Вам нужно установить значение в параметр запроса `Timestamp` для получения дополнительных участников. Если это `0`, все участники уже возвращены. |

## Коды ошибок

HTTP код состояния, возвращаемый этим API, всегда равен 200, если не возникает ошибка сети (например, ошибка 502). Конкретный код ошибки и подробности можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (от 60000 до 79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10001 | Пользователь не авторизован. Проверьте правильность учетной записи `admin` и `usersig`. |
| 10002 | Произошла системная ошибка. Повторите попытку или обратитесь в техническую поддержку. |
| 10003 | Команда запроса недействительна. Повторите попытку или обратитесь в техническую поддержку. |
| 10004 | Параметр недействителен. Проверьте поле `ErrorInfo` в пакете ответа, чтобы убедиться, что указаны обязательные поля или что поля установлены в соответствии с требованиями протокола. |
| 10007 | У вас нет необходимых разрешений. Вы можете получить только количество онлайн-пользователей для аудио-видео групп. |
| 10010 | Группа не существует или была удалена. |
| 10015 | ID группы недействителен. Проверьте ID группы. |

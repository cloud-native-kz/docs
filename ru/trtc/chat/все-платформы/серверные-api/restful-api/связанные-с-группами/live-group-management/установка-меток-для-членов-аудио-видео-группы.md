# Установка меток для членов аудио-видео группы

## Обзор функции

Администраторы приложений и владельцы групп могут устанавливать различные метки/ярлыки для членов аудио-видео группы, чтобы различать разные типы членов группы. Для использования этой функции необходимо приобрести издание Pro, издание Pro Plus или Enterprise, включить **Список членов онлайн аудио-видео группы** на странице **Конфигурация группы** в консоли Chat. Для других редакций плана вызов этого API будет неудачным.

## Описание вызова API

### Применимые типы групп

| ID типа группы | Поддержка RESTful API |
| --- | --- |
| Private | Нет. Аналогично рабочим группам (Work) в новой версии. |
| Public | Нет |
| ChatRoom | Нет. Аналогично группам встреч (Meeting) в новой версии. |
| AVChatRoom | Да |
| Community | Нет |

Это предустановленные типы групп в Chat. Дополнительные сведения см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

> **Примечание** Для использования этой функции необходимо [приобрести издание Pro, издание Pro Plus или Enterprise](https://www.tencentcloud.com/document/product/1047/34577) и включить функцию **Список членов онлайн аудио-видео группы** на странице **Конфигурация группы** в консоли. Для других редакций плана вызов этого API будет неудачным. Для аудио-видео группы можно установить до 10 различных меток. Каждая метка может быть установлена для до 1000 членов. Это ограничение независимо от ограничения в 1000 членов в списке онлайн членов аудио-видео группы. Метки можно устанавливать только для онлайн членов группы, за исключением специальной метки 500, которая описана в разделе "Специальные метки". Когда член покидает группу или переходит в режим офлайн, информация о его метке будет очищена. Когда член присоединяется к группе или снова переходит в режим онлайн, вы можете повторно установить метку для члена через вебхук.

### Пример URL запроса

```
https://xxxxxx/v4/group_open_avchatroom_http_svc/modify_user_info?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны только измененные параметры при вызове этого API. Для других параметров см. раздел [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваше SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/group_open_http_svc/modify_group_member_info | API запроса. |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Дополнительные сведения см. в разделе **Администратор приложения** в статье [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробнее см. в разделе [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное беззнаковое целое число в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса. Значение фиксировано на `json`. |

### Максимальная частота вызова

200 вызовов в секунду

### Пример запроса

Установить метку 1000 для члена "teacher10" и метку 1001 для члена "student9" аудио-видео группы:

```
{    "GroupId": "@TGS#a6I4ZUUGO",    "CommandType": 1,    "MemberList": [        {            "Member_Account": "teacher10",            "Marks": [1000]        },        {            "Member_Account": "student9",            "Marks": [1001]        },    ]}
```

Отменить метку 1000 для члена "teacher10" и метку 1001 для члена "student9" аудио-видео группы:

```
{    "GroupId": "@TGS#a6I4ZUUGO",    "CommandType": 2,    "MemberList": [        {            "Member_Account": "teacher10",            "Marks": [1000]        },        {            "Member_Account": "student9",            "Marks": [1001]        },    ]}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы, исторические сообщения которой будут изменены |
| CommandType | Integer | Да | `1`: установка метки; `2`: удаление метки |
| MemberList | Array | Да | Список учетных записей, для которых должны быть установлены метки. За один раз поддерживается до 500 учетных записей, и для каждой группы можно установить до 10 различных меток. Метка — это число, большее или равное 1000. |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorCode": 0,    "ErrorInfo": "",    "CommandType": 1,    "MemberList": [        {            "Member_Account": "teacher10",            "Marks": [1000]        },        {            "Member_Account": "student9",            "Marks": [1001]        }    ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: ошибка |
| ErrorInfo | String | Информация об ошибке |
| MemberList | Object | Список членов, для которых метки успешно установлены. В случае установки меток для набора учетных записей, если определенные учетные записи находятся в режиме офлайн, сервер не возвращает офлайн учетные записи; если все учетные записи находятся в режиме офлайн или не существуют, сервер возвращает результат ошибки. |

### Специальные метки

Администраторы приложений и владельцы групп могут устанавливать метки в диапазоне [1000, +) для членов группы. Кроме того, администраторы приложений могут устанавливать следующие специальные метки, предустановленные в Chat:

| Метка | Описание |
| --- | --- |
| 500 | Когда для члена установлена метка 500, член считается постоянно онлайн (события отключения игнорируются) в течение максимум трех дней. |
| 600 | Когда для члена установлена метка 600, член скрывается при получении списка онлайн членов, если только список явно не указан как полученный по метке члена. |

## Коды ошибок

Возвращаемый этим API код состояния HTTP всегда равен 200, если только не происходит сетевая ошибка (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10001 | Пользователь не вошел в систему. Проверьте, правильна ли учетная запись `admin` и `usersig`. |
| 10002 | Произошла системная ошибка. Повторите попытку или свяжитесь с технической поддержкой. |
| 10003 | Команда запроса недействительна. Повторите попытку или свяжитесь с технической поддержкой. |
| 10004 | Параметр недействителен. Проверьте поле `ErrorInfo` в пакете ответа, чтобы убедиться, что все необходимые поля указаны или поля установлены в соответствии с требованиями протокола. |
| 10007 | Недостаточно разрешений. Только администратор приложения или владелец группы могут вызвать этот API. |
| 10010 | Группа не существует или была удалена. |
| 10015 | ID группы недействителен. Проверьте ID группы. |

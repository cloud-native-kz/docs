# Получение профилей групп

## Обзор функции

Этот API используется администратором приложения для запроса подробной информации о группе на основе одного или нескольких идентификаторов групп.

## Описание вызова API

### Применимые типы групп

| ID типа группы | Поддержка RESTful API |
| --- | --- |
| Private | Да. То же самое, что рабочие группы (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. То же самое, что группы собраний (Meeting) в новой версии. |
| AVChatRoom | Поддерживает только получение деталей группы (количество участников не поддерживается) и не поддерживает получение списка участников. |
| Community | Да |

Это предустановленные типы групп в Chat. Дополнительную информацию см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

> **Примечание:** Для получения количества пользователей в сети в группах прямой трансляции см. подробности в API [количество пользователей в сети в группах прямой трансляции](https://www.tencentcloud.com/document/product/1047/38521). Для получения списка участников в сети AVChatRoom см. подробности в API [список участников в сети в группе потоковой трансляции](https://www.tencentcloud.com/document/product/1047/60524).

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/get_group_info?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

В следующей таблице описаны измененные параметры при вызове этого API. Другие параметры см. в разделе [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |

| v4/group_open_http_svc/get_group_info | Вызываемый API запроса. |

| identifier | Аккаунт администратора приложения. Дополнительную информацию см. в разделе **Администратор приложения** в [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная аккаунтом администратора приложения. Подробности см. в разделе [Создание UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса, который всегда должен быть `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Базовый формат**
Следующий пример используется для получения подробной информации о группе, включая всю базовую информацию о группе, информацию о членах группы и пользовательскую информацию. Запрос содержит только список идентификаторов групп. Каждый запрос поддерживает запрос информации максимум по 50 группам.

```
{  "GroupIdList": [ // Список идентификаторов групп, указанных для запроса. Этот параметр является обязательным.      "@TGS#1NVTZEAE4",      "@TGS#1CXTZEAET"  ]}
```

- **Получение указанной информации**
- Для указания полей базовой информации, подлежащих извлечению, установите `GroupBaseInfoFilter`.
- Для указания полей информации о членах, подлежащих извлечению, установите `MemberInfoFilter`.
- Для указания специфичных для группы пользовательских полей, подлежащих извлечению, установите AppDefinedDataFilter_Group. Это поле недоступно по умолчанию и должно быть включено дополнительно. Дополнительную информацию см. в разделе Поля пакета запроса.
- Для указания специфичных для членов группы пользовательских полей, подлежащих извлечению, установите AppDefinedDataFilter_GroupMember. Это поле недоступно по умолчанию и должно быть включено дополнительно. Дополнительную информацию см. в разделе Поля пакета запроса.

Если установлены фильтры, извлекаются только поля, указанные фильтрами.

```
{    "GroupIdList": [ // Список идентификаторов групп, указанных для запроса. Этот параметр является обязательным.        "@TGS#1NVTZEAE4",        "@TGS#1CXTZEAET"    ],    "ResponseFilter": { // Фильтры, которые вы указываете для ответа.        "GroupBaseInfoFilter": [ // Добавьте этот массив, если вы хотите запросить базовую информацию.            "Type",            "Name",            "Introduction",            "Notification"        ],        "MemberInfoFilter": [ // Добавьте этот массив, если вы хотите запросить информацию о членах.            "Account", // ID участника            "Role"        ],        "AppDefinedDataFilter_Group": [ // Фильтр для специфичных для группы пользовательских полей. Этот фильтр отключен по умолчанию.            "GroupTestData1",            "GroupTestData2"        ],        "AppDefinedDataFilter_GroupMember": [ // Фильтр для специфичных для членов группы пользовательских полей. Этот фильтр отключен по умолчанию.            "MemberDefined2",            "MemberDefined1"        ]    }}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupIdList | Array | Обязательно | Список идентификаторов групп, которые вы указываете для запроса. |
| ResponseFilter | Object | Опционально | Содержит три фильтра: `GroupBaseInfoFilter`, `MemberInfoFilter` и `AppDefinedDataFilter_Group`. |
| GroupBaseInfoFilter | Array | Опционально | Фильтр полей базовой информации, который указывает полями базовой информации для запроса. Дополнительную информацию см. в разделе [Базовые данные группы](https://intl.cloud.tencent.com/document/product/1047/33529#GroupBaseInfoFilter). |
| MemberInfoFilter | Array | Опционально | Фильтр полей информации о членах, который указывает поля информации о членах для запроса. Дополнительную информацию см. в разделе [Информация о членах группы](https://intl.cloud.tencent.com/document/product/1047/33529#SelfInfoFilter). |
| AppDefinedDataFilter_Group | Array | Опционально | Фильтр специфичных для группы пользовательских полей, который указывает специфичные для группы пользовательские поля для запроса. Дополнительную информацию см. в разделе [Пользовательские поля](https://intl.cloud.tencent.com/document/product/1047/33529#.E8.87.AA.E5.AE.9A.E4.B9.89.E5.AD.97.E6.AE.B5). |
| AppDefinedDataFilter_GroupMember | Array | Опционально | Фильтр специфичных для членов группы пользовательских полей, который указывает специфичные для членов группы пользовательские поля для запроса. Дополнительную информацию см. в разделе [Пользовательские поля](https://intl.cloud.tencent.com/document/product/1047/33529#.E8.87.AA.E5.AE.9A.E4.B9.89.E5.AD.97.E6.AE.B5). |

## Коды ошибок

HTTP-код состояния, возвращаемый этим API, всегда равен 200, если не происходит ошибка сети (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Произошла системная ошибка. Повторите попытку или обратитесь в техническую поддержку. |
| 10003 | Команда запроса является недействительной. Повторите попытку или обратитесь в техническую поддержку. |
| 10004 | Параметр является недействительным. Проверьте поле `ErrorInfo` в пакете ответа, чтобы убедиться, что все обязательные поля указаны или что поля установлены в соответствии с требованиями протокола. |
| 10007 | Недостаточно прав на операцию. Проверьте, является ли оператор администратором приложения. |
| 10010 | Группа не существует или была удалена. |
| 10015 | ID группы является недействительным. Пожалуйста, проверьте ID группы. |
| 10018 | Длина пакета ответа превышает максимальную длину пакета 1 МБ, так как было запрошено слишком много содержимого. Попробуйте уменьшить количество запрашиваемых данных за один раз. |

# Получение всех групп в приложении

## Обзор функции

Данный API используется администратором приложения для получения идентификаторов всех групп в приложении.

## Описание вызова API

### Применимые типы групп

Tencent Cloud Chat предоставляет различные типы встроенных групп. Дополнительные сведения см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/get_appid_group_list?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для получения других параметров см. раздел [Обзор REST API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, в котором находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/group_open_http_svc/get_appid_group_list | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Дополнительные сведения см. в разделе **Администратор приложения** в документе [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробные сведения см. в разделе [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |

| contenttype | Формат запроса, который должен быть `json`. |

### Максимальная частота вызовов

1 раз в секунду

### Пример запроса

- **Базовый формат**
Если общее количество групп в приложении превышает 10 000, возвращается максимум 10 000 идентификаторов групп. Для получения всех идентификаторов групп необходимо получить их постранично.

```
{}
```

- **Постраничное получение**
Вы можете использовать поля `Limit` и `Next` для управления постраничным получением:
- Поле `Limit` определяет максимальное количество групп в `GroupIdList` в пакете ответа, которое не может превышать 10 000.
- Поле `Next` используется для управления разбиением на страницы. Для начального запроса разбиения на страницы `Next` установлено значение 0. Для последующих запросов установлено ранее возвращенное значение `Next`. Если возвращенное значение `Next` равно 0, все группы были получены.
Например, если запрашивается постраничное получение и 20 идентификаторов групп отображаются на странице, параметры запроса для первой страницы: `{âLimitâ : 20, âNextâ : 0}`, а для второй страницы: `{âLimitâ : 20, âNextâ : Ранее возвращенное значение Next}` и так далее.

Значение `Limit` или `Next` не влияет на `TotalCount` в пакете ответа.

```
{   "Limit": 1000,   "Next": 0}
```

- **Указание типа группы**
Вы можете указать тип групп для получения, такие как Public, Private, ChatRoom, AVChatRoom или BChatRoom.

```
{  "GroupType" : "Public" // Тип групп для получения. Если этот параметр не указан, будут получены группы всех типов.}
```

- **ВСЕ В ОДНОМ**

```
{  "Limit": 1000,  "Next": 0,  "GroupType" : "Public" // Тип групп для получения. Если этот параметр не указан, будут получены группы всех типов.}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| Limit | Integer | Нет | Максимальное количество идентификаторов групп для получения, которое не может превышать 10 000. Если значение не указано, по умолчанию используется максимальное значение 10000. |
| Next | Integer | Нет | Флаг постраничного получения, когда количество групп слишком велико. Изначально установлено значение 0, впоследствии установлено значение `Next`, возвращенное в предыдущем ответе. Если возвращенное значение `Next` равно 0, все группы были получены. |
| GroupType | String | Нет | Для получения групп определенного типа вы можете использовать `GroupType` для фильтрации. В этом случае возвращаемое значение `TotalCount` указывает общее количество групп указанного типа в приложении. Если это поле не указано, получаются группы всех типов. Возможные типы групп: Public, Private, ChatRoom, AVChatRoom, BChatRoom и Community. |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0,    "TotalCount": 2,    "GroupIdList": [        {            "GroupId": "@TGS#2J4SZEAEL"        },        {            "GroupId": "@TGS#2C5SZEAEF"        }    ],    "Next": 4454685361}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки. `0`: Успешно; другие значения: Ошибка |
| ErrorInfo | String | Информация об ошибке |
| TotalCount | Integer | Общее количество существующих групп в приложении. Для получения групп определенного типа вы можете использовать `GroupType` для фильтрации. В этом случае возвращаемое значение `TotalCount` указывает общее количество групп указанного типа в приложении. Если это поле не указано, получаются группы всех типов. Например, предположим, что приложение содержит всего 50 000 групп, включая 20 000 открытых групп. Если `GroupType` в пакете запроса установлено значение `Public`, `TotalCount` в пакете ответа равно 20 000 независимо от значений `Limit` и `Offset`. Кроме того, все группы в `GroupIdList` являются открытыми группами. |
| GroupIdList | Array | Список полученных идентификаторов групп. |
| Next | Integer | Флаг постраничного получения. |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда равен 200, если не возникает ошибка сети (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (от 60000 до 79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10004 | Параметр указан неправильно. Чтобы исправить это, проверьте параметры запроса, такие как `GroupType`, на основе описания ошибки. |
| 10007 | Оператор не имеет необходимых разрешений для выполнения этой операции. Только администратор приложения может вызывать этот API. |
| 10018 | Пакет ответа превышает ограничение по длине 1 МБ из-за чрезмерного содержимого запроса. Попытайтесь уменьшить объем данных в отдельных запросах. |

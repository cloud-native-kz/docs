# Получение списка групп, в которых состоит пользователь

## Обзор функции

Этот API используется администраторами приложения для получения списка групп, в которых состоит определённый пользователь. По умолчанию информация о рабочих группах (Work) и аудиовидео-группах (AVChatRoom), в которых пользователь состоит, но которые не активированы, не выводится.

## Описание вызова API

### Применимые типы групп

| Идентификатор типа группы | Поддержка RESTful API |
| --- | --- |
| Private | Да. В новой версии совпадает с рабочими группами (Work). По умолчанию информация о рабочих группах, в которых пользователь состоит, но которые не активированы, не возвращается. |
| Public | Да |
| ChatRoom | Да. В новой версии совпадает с группами встреч (Meeting). |
| AVChatRoom | Да. По умолчанию информация об аудиовидео-группах (AVChatRoom), в которых пользователь состоит, не возвращается. При запросе информации об аудиовидео-группах полученная информация может быть неполной. Аудиовидео-группы не хранят полные данные всех членов группы. |
| Community | Да |

Это предустановленные типы групп в Chat. Дополнительную информацию см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Базовый формат**
Базовый запрос используется для получения информации о группах, в которых состоит пользователь. Информация о группе содержит только идентификаторы пользователей в группе.

```
{  "Member_Account": "leckie"}
```

- **Получение по страницам**
Вы можете использовать поля `Limit` и `Offset` для управления режимом постраничного вывода:
- Поле `Limit` указывает максимальное количество групп в `GroupIdList` в пакете ответа, которое не может превышать 5000.
- Поле `Offset` указывает порядковый номер группы в полном списке групп, начиная с которого читается информация. По умолчанию Offset равен 0. При запросе постраничного вывода (номер страницы начинается с 1) значение `Offset` для каждой страницы должно быть: (Номер страницы - 1) × Количество групп, отображаемых на каждой странице.
Например, для отображения 20 членов группы на каждой странице параметры запроса для первой страницы должны быть `{"Limit": 20, "Offset": 0}`, параметры запроса для второй страницы должны быть `{"Limit": 20, "Offset": 20}` и так далее.

Значение `Limit` или `Offset` не влияет на `TotalCount` в пакете ответа.

```
{    "Member_Account": "leckie",    "Limit": 10, // Количество групп для получения. Если это поле не указано, будут получены все группы.    "Offset": 0 // Порядковый номер группы, начиная с которой получается информация.}
```

- **Указание типа группы**
Вы можете указать тип групп для получения, например Public (социальная сетевая группа для незнакомцев), Private (совпадает с рабочими группами в новой версии) и ChatRoom (совпадает с группами встреч в новой версии). Если указана AVChatRoom (аудиовидео-группа), вы можете получить неполный список членов.

```
{  "Member_Account": "leckie",  "GroupType" : "Public" // Тип групп для получения. Если этот параметр не указан, будут получены группы всех типов.}
```

- **Получение указанной информации**
Для указания полей информации о группе для получения установите `GroupBaseInfoFilter`.
Для указания информации о членах группы, которая должна быть получена, установите `SelfInfoFilter`.

```
{  "Member_Account": "leckie",  "WithHugeGroups":1, // Поддерживает получение информации об аудиовидео-группах (AVChatRoom).  "WithNoActiveGroups":1,// Поддерживает получение информации о приватных группах (совпадает с рабочими группами в новой версии), в которых пользователь состоит, но которые не активированы.  "Limit": 10, // Количество групп для получения. Если это поле не указано, будут получены все группы.  "Offset": 0, // Порядковый номер группы, начиная с которой получается информация.  "ResponseFilter": {      "GroupBaseInfoFilter": [ // Поля базовой информации для получения          "Type",          "Name",          "Introduction",          "Notification"      ],      "SelfInfoFilter": [ // Личная информация члена в группе          "Role", // Роль в группе          "JoinTime" // Время, когда член присоединился к группе      ]  }}
```

- **Получение групп сообщества с поддержкой тематик**

```
{  "Member_Account": "107867",// Аккаунт пользователя для запроса (обязательно)  "SupportTopic": 1// Поддерживает ли указанный тип группы тематики. Это поле поддерживается только группами сообщества.}
```

- **ВСЕ В ОДНОМ**

```
{"Member_Account": "leckie","WithHugeGroups":1, "WithNoActiveGroups":1,"ResponseFilter": {      "GroupBaseInfoFilter": [          "Type",          "Name",          "Introduction",          "Notification",          "FaceUrl",          "CreateTime",          "Owner_Account",          "LastInfoTime",          "LastMsgTime",          "NextMsgSeq",          "MemberNum",          "MaxMemberNum",          "ApplyJoinOption",          "MuteAllMember"      ],      "SelfInfoFilter": [          "Role",           "JoinTime",          "MsgFlag",           "MsgSeq"       ]  }}
```

### Поля запроса

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| Member_Account | String | Да | Аккаунт пользователя для запроса |
| WithHugeGroups | Integer | Нет | Получать ли информацию об аудиовидео-группах (AVChatRoom), в которых состоит пользователь. `0`: Нет (значение по умолчанию); `1`: Да |
| WithNoActiveGroups | Integer | Нет | Получать ли информацию о приватных группах (совпадает с рабочими группами в новой версии), в которых пользователь состоит, но которые не активированы. `0`: Нет (значение по умолчанию); `1`: Да |
| Limit | Integer | Нет | Количество групп, получаемых каждый раз. Если значение не указано, будут получены все группы. |
| Offset | Integer | Нет | Порядковый номер группы, начиная с которой получается информация. |
| GroupType | String | Нет | Тип групп для получения, например Public (социальная сетевая группа для незнакомцев), Private (совпадает с рабочими группами в новой версии), ChatRoom (совпадает с группами встреч в новой версии), AVChatRoom (аудиовидео-группы) и Community. Если значение не указано, получаются группы всех типов. |
| ResponseFilter | Object | Нет | Содержит два фильтра: `GroupBaseInfoFilter` и `SelfInfoFilter`. `GroupBaseInfoFilter` указывает поля базовой информации для получения. Дополнительную информацию см. в [Базовая информация группы](https://intl.cloud.tencent.com/document/product/1047/33529#GroupBaseInfoFilter). `SelfInfoFilter` указывает личную информацию членов для получения из групп. Дополнительную информацию см. в [Профиль члена группы](https://intl.cloud.tencent.com/document/product/1047/33529#SelfInfoFilter). |
| SupportTopic | Integer | Нет | Поддерживает ли указанный тип группы тематики. `1`: Да; `0`: Нет. Если это поле указано, `GroupType` должен быть `Community`. |

## Коды ошибок

HTTP код состояния, возвращаемый этим API, всегда равен 200, если только не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для публичных кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, характерные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Произошла системная ошибка. Повторите попытку или обратитесь в техническую поддержку. |
| 10003 | Команда запроса неправильна. Повторите попытку или обратитесь в техническую поддержку. |
| 10004 | Параметр неправильный. Проверьте поле `ErrorInfo` в пакете ответа, чтобы убедиться, что были указаны обязательные поля или что поля установлены в соответствии с требованиями протокола. |
| 10018 | Длина пакета ответа превышает максимальную длину пакета 1 МБ из-за того, что был запрошен слишком большой объём содержимого. Попробуйте сократить объём данных, запрашиваемых за один раз. |

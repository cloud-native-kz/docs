# Изменение владельца группы

## Обзор функции

- Администратор приложения может вызвать этот API для передачи прав владельца группы другому пользователю.
- Для группы без владельца администратор приложения может вызвать этот API, чтобы назначить пользователя владельцем группы.
- Новый владелец группы должен быть членом группы.

## Описание вызова API

### Применимые типы групп

| ID типа группы | Поддержка RESTful API |
| --- | --- |
| Private | Да. Аналогично рабочим группам (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. Аналогично группам для встреч (Meeting) в новой версии. |
| AVChatRoom | Нет (см. примечание ниже) |
| Community | Да |

Это предустановленные типы групп в Chat. Дополнительную информацию см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

> **Примечание:** Аудиовидеогруппы (AVChatRoom) не поддерживают передачу прав владельца группы. При вызове этого API для такой группы будет возвращен код ошибки 10007.

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/change_group_owner?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Силиконовая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/group_open_http_svc/change_group_owner | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Аккаунт администратора приложения. Дополнительную информацию см. в разделе **Администратор приложения** в документе [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная аккаунтом администратора приложения. Подробности см. в разделе [Создание UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса. Значение зафиксировано как `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

Передача прав владельца группы другому члену группы. Новый владелец группы должен быть членом группы.

```
{    "GroupId": "@TGS#1NVTZEAE4",  // (Обязательно) ID группы, права владельца которой передаются    "NewOwner_Account": "peter" // (Обязательно) ID нового владельца группы}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы, права владельца которой передаются |
| NewOwner_Account | String | Да | ID нового владельца группы |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode":0}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: не удалось |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: не удалось |
| ErrorInfo | String | Информация об ошибке |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда равен 200, если только не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, относящиеся к этому API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Произошла системная ошибка. Попробуйте снова или обратитесь в техническую поддержку. |
| 10004 | Параметр недействителен. Проверьте поле `ErrorInfo` в пакете ответа, чтобы убедиться, что указаны обязательные поля или что поля установлены в соответствии с требованиями протокола. |
| 10007 | Недостаточно прав на выполнение операции. Проверьте, является ли оператор администратором приложения. |
| 10010 | Группа не существует или была удалена. |
| 10015 | Недействительный ID группы. Проверьте, правильно ли введен ID группы. |

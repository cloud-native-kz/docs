# Создание или обновление данных метки беседы

## Обзор функции

Данные метки группирования бесед независимы от недавних контактов. REST API поддерживает добавление, удаление, изменение и запрос полей для групп бесед, стандартных меток бесед и пользовательских меток бесед. Этот API поддерживает создание или обновление как стандартных меток бесед, так и пользовательских меток бесед. Данные пользовательской метки беседы не ограничены по объему.

> **Примечание:** Каждый пользователь может создать до 20 групп бесед. Сеанс может присоединиться к нескольким группам. Максимальный объем сеансов установлен на уровне 1000, превышение которого вернет код ошибки: [51008](https://www.tencentcloud.com/document/product/1047/34348). Теги сеансов и пользовательские поля сеансов имеют объединённый верхний лимит в 1000 сеансов. Превышение этого лимита вернет код ошибки: [51008](https://www.tencentcloud.com/document/product/1047/34348). Функции группирования и маркировки бесед требуют включения **Ultimate Edition или Enterprise Edition**. Количество облачных сеансов, которые можно получить, ограничено параметром ["Количество бесед для получения"](https://www.tencentcloud.com/document/product/1047/67650) в базовом пакете Chat. Если количество сгруппированных или помеченных сеансов превышает соответствующий лимит, полное получение сеансов может не пройти. Вы можете обновить версию пакета Chat, чтобы увеличить лимит облачных сеансов.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/recentcontact/mark_contact?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны изменённые параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://trtc.io/document/34620?product=chat&menulabel=serverapi).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/recentcontact/mark_contact | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учётная запись администратора приложения. Для получения дополнительной информации см. раздел **App Admin** в [Аутентификация входа](https://trtc.io/document/33517?platform=javascript&product=chat&menulabel=coresdk). |
| usersig | Подпись, созданная учётной записью администратора приложения. Подробности см. в [Генерация UserSig](https://trtc.io/document/34385?product=chat&menulabel=serverapi). |
| random | Случайное 32-битное беззнаковое целое число от 0 до 4294967295. |
| contenttype | Формат запроса, всегда должен быть `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

```
{    "From_Account":"user0",    "MarkItem":[        {            "OptType":3,            "ContactItem":{                "Type":1,                                   "To_Account":"user1"            },            "SetMark":[1,2,3],            "CustomMark":"abcd"        }    ]}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| From_Account | String | Да | Укажите UserID, запрос для создания меток бесед для этого пользователя. |
| MarkItem | Array | Да | Массив меток бесед. Массив может содержать до 100 меток бесед. |
| OptType | Integer | Да | Тип операции. `1`: Установка стандартных меток бесед; `2`: Установка пользовательских меток бесед; `3`: Установка как стандартных, так и пользовательских меток бесед |
| ContactItem | Object | Да | Объект беседы для операции |
| Type | Integer | Да | Тип беседы: 1: Персональная беседа. 2: Групповая беседа |
| ToGroupId | String | Нет | Указывайте значение только для групповой беседы, укажите ID группы стороны беседы. |
| To_Account | String | Нет | Указывайте значение только для персональной беседы, укажите UserID стороны беседы. |
| ClearMark | Array | Нет | Номер последовательности бита, для которого должна быть отменена стандартная метка. Значение `n` указывает на отмену значения метки для бита n. |
| SetMark | Array | Нет | Номер последовательности бита, для которого должна быть установлена стандартная метка. Значение `n` указывает на установку значения метки для бита n. |
| CustomMark | String | Нет | Установить пользовательскую метку беседы. Поддерживается до 256 байт. |

```
Стандартная метка беседы имеет в общей сложности 64 бита: первые 32 бита используются внутри Chat, а последние 32 бита являются расширенными битами, которые могут быть настроены клиентами. Описание внутренних битов:Бит 0: Отметить беседу как "избранное"Бит 1: Отметить беседу (важная беседа) как непрочитаннуюБит 2: Отметить беседу как свёрнутуюБит 3: Отметить беседу как скрытую
```

### Пример ответа

```
{    "ResultItem": [        {            "OptType":3,            "ContactItem": {                "Type": 1,                "To_Account": "user1"            },            "ResultCode": 0,            "ResultInfo": ""        }    ],    "ActionStatus": "OK",    "ErrorCode": 0,    "ErrorInfo": "",    "ErrorDisplay": ""}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: Успешно; `FAIL`: Ошибка |
| ErrorCode | Integer | Код ошибки. `0`: Успешно; другие значения: Ошибка |
| ErrorInfo | String | Информация об ошибке |
| ResultItem | Array | Результат операции метки беседы |
| OptType | Integer | Да |
| ContactItem | Integer | Объект беседы |
| Type | String | Тип беседы: 1: Персональная беседа. 2: Групповая беседа |
| ToGroupId | String | Указывайте значение только для групповой беседы, укажите ID группы стороны беседы |
| To_Account | String | Указывайте значение только для персональной беседы, укажите UserID стороны беседы |
| ResultCode | Integer | Код ошибки в результате операции беседы. `0`: Успешно |
| ResultInfo | String | Описание ошибки в результате операции беседы |

## Коды ошибок

HTTP код состояния, возвращаемый этим API, всегда равен 200, если не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и подробности можно найти в полях ответа `ResultCode`, `ResultInfo`, `ErrorCode` и `ErrorInfo`.

Для общих кодов ошибок (60000–79999) см. [Коды ошибок](https://trtc.io/document/34348?platform=javascript&product=chat&menulabel=coresdk).

В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 50001 | Запрашиваемый UserID не был импортирован в серверную часть Tencent Cloud Chat. Пожалуйста, выполните импорт. |
| 50002 | Неверный параметр запроса. Проверьте свой запрос в соответствии с описанием ошибки. |
| 50003 | Запрос требует прав администратора приложения. |
| 50004 | Внутренняя ошибка сервера. Пожалуйста, попробуйте ещё раз. |
| 50005 | Истекло время ожидания сети. Попробуйте позже. |
| 51006 | При изменении метки беседы количество бесед пусто или превышает верхний лимит в 100. |
| 51007 | Ошибка при замене GroupID на GroupCode из-за внутренней ошибки или распада группы. |
| 51008 | Общее количество бесед в группе бесед превышает 1000. |
| 51009 | Группа бесед не существует при попытке удаления. |
| 51010 | Количество групп бесед превышает верхний лимит в 20. |
| 51011 | Имя группы бесед содержит более 32 байт. |
| 51012 | Превышено максимальное количество бесед, закреплённых в начале. |
| 51013 | Если запрашиваемое приложение не является Pro edition, Pro Plus edition или Enterprise edition, при изменении стандартной метки беседы или группы бесед будет выдана ошибка. |

# Удаление сообщений, отправленных указанным пользователем

## Обзор функции

Этот API можно использовать для отзыва сообщений, отправленных указанным пользователем среди последних 1000 сообщений.

## Описание вызова API

### Применимые типы групп

| ID типа группы | Поддержка RESTful API |
| --- | --- |
| Private | Да. Аналогично рабочим группам (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. Аналогично группам встреч (Meeting) в новой версии. |
| AVChatRoom | Нет |
| Community | Да |

Это встроенные типы групп в Chat. Дополнительные сведения см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

> **Примечание:** Этот API отзывает сообщения, находящиеся в историческом хранилище сообщений, но аудиовизуальные группы (AVChatRoom) не поддерживают хранение исторических сообщений. Поэтому этот API не работает при использовании для отзыва сообщений пользователя в аудиовизуальной группе (но ошибка не будет возвращена). Этот API в настоящее время поддерживает только автоматический отзыв. Отозванное сообщение будет отмечено как отозванное на сервере, и клиент не будет уведомлен об отзыве. Клиент может узнать, что сообщение отозвано, только когда он получает бродячие сообщения.

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/delete_group_msg_by_sender?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны модифицированные параметры при вызове этого API. Для других параметров см. раздел [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Имя домена, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/group_open_http_svc/delete_group_msg_by_sender | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Дополнительные сведения см. в разделе **Администратор приложения** в документе [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробнее см. раздел [Создание UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-разрядное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса. Значение зафиксировано как `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

Отозвать сообщения, отправленные указанным пользователем среди последних 1000 сообщений группы:

```
{    "GroupId": "@TGS#2C5SZEAEF",    // Обязательно    "Sender_Account": "leckie"      // Обязательно}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы, в которой нужно отозвать сообщения |
| Sender_Account | String | Да | ID отправителя сообщений, которые нужно отозвать |
| TopicId | String | Нет | ID темы, который указывает отзыв сообщения в соответствующей теме и применяется только к сообществам с поддержкой тем. |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode":0}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorInfo | String | Информация об ошибке |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: ошибка |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда равен 200, если не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, относящиеся к этому API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Попробуйте еще раз. |
| 10003 | Неверная команда. |
| 10004 | Неверный параметр. Проверьте описание ошибки и устраните проблему. |
| 10007 | Отсутствуют разрешения на операцию. Например, обычный участник в открытой группе пытается удалить других пользователей из группы, но это может делать только администратор приложения. |
| 10010 | Группа не существует или была удалена. |
| 10015 | Неверный ID группы. Используйте правильный ID группы. |

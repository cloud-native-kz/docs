# Получение атрибутов пользователя

## Обзор функции

Этот API используется для получения атрибутов пользователя. Для вызова этого API необходимо использовать учетную запись администратора. Вы можете получить атрибуты максимум 100 пользователей за раз. Необходимо [установить имена атрибутов приложения](https://intl.cloud.tencent.com/document/product/1047/37167) перед вызовом API.

## Описание вызова API

Отправка всем пользователям доступна только для редакций Pro, Pro Plus и Enterprise. Чтобы использовать эту функцию, необходимо [приобрести редакции Pro, Pro Plus или Enterprise](https://www.tencentcloud.com/document/product/1047/34577), перейти в [консоль](https://console.trtc.io/chat), выбрать **Feature Configuration** > **Login and Message** > **Push to all users** и включить функцию.

### Пример URL запроса

```
https://xxxxxx/v4/all_member_push/im_get_attr?usersig=xxx&identifier=admin&sdkappid=88888888&random=99999999&contenttype=json
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/all_member_push/im_get_attr | API запроса |
| usersig | Подпись, созданная в учетной записи администратора приложения. Дополнительную информацию см. в разделе [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| identifier | Учетная запись администратора приложения. |
| sdkappid | `SDKAppID`, назначенный консолью Chat при создании приложения |
| random | Случайное 32-битное беззнаковое целое число |
| contenttype | Значение всегда `json`. |

### Максимальная частота вызовов

100 раз/сек

### Пример запроса

```
{    "To_Account": [        "Zhang Xiaohong",        "Chen Xiaoming",        "abc"    ]}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| To_Account | Array | Да | Список целевых учетных записей пользователей |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0,    "UserAttrs": [        {            "To_Account": "Zhang Xiaohong",            "Attrs": {                "sex" : "Female",                "city": "New York"            }        },        {            "To_Account": "abc",            "Attrs": {}        },        {            "To_Account": "Chen Xiaoming",            "Attrs": {                "sex": "M",                "city": "Shenzhen"            }        }    ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно. `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки |
| ErrorInfo | String | Информация об ошибке |
| UserAttrs | Array | Список тегов пользователя |
| To_Account | String | Учетная запись пользователя |
| Attrs | Object | Содержимое атрибута |

## Коды ошибок

За исключением сетевых ошибок (например, ошибка 502), HTTP-код возврата для этого API всегда равен 200. `ErrorCode` **и** `ErrorInfo` **в ответе представляют фактический код ошибки и информацию об ошибке.** Для общих кодов ошибок (60000–79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 90001 | Ошибка разбора формата JSON. Убедитесь, что JSON-запрос соответствует спецификациям JSON. |
| 90018 | Количество запрашиваемых учетных записей превышает лимит. |
| 91000 | Ошибка внутреннего сервиса. Попробуйте еще раз. |

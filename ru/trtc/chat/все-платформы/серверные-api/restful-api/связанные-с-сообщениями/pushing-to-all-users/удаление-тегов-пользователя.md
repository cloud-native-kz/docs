# Удаление тегов пользователя

## Обзор функции

Этот API используется администратором для удаления тегов пользователей. Каждый запрос может удалять теги максимум для 100 пользователей.

## Описание вызова API

Отправка всем пользователям доступна только для издания Pro, Pro Plus и Enterprise. Чтобы использовать эту функцию, необходимо [приобрести издание Pro, Pro Plus или Enterprise](https://www.tencentcloud.com/document/product/1047/34577), перейти в [консоль](https://console.trtc.io/chat), выбрать **Feature Configuration** > **Login and Message** > **Push to all users** и включить функцию.

### Пример URL запроса

```
https://xxxxxx/v4/all_member_push/im_remove_tag?usersig=xxx&identifier=admin&sdkappid=88888888&random=99999999&contenttype=json
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| xxxxxx | Доменное имя, соответствующее стране/региону расположения вашего SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/all_member_push/im_remove_tag | API запроса |
| usersig | Подпись, созданная в учетной записи администратора приложения. Для получения дополнительной информации см. [Создание UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| identifier | Учетная запись администратора приложения. |
| sdkappid | `SDKAppID`, назначенный консолью Chat при создании приложения |
| random | Случайное 32-битное целое число без знака |
| contenttype | Значение всегда `json`. |

### Максимальная частота вызовов

100 раз/сек

### Пример запроса

```
{    "UserTags": [        {            "To_Account": "xiaojun012",            "Tags": ["a", "b"]        },        {            "To_Account": "xiaojun013",            "Tags": ["a", "b"]        }    ]}
```

### Поля запроса

| Поле | Тип | Требуется | Описание |
| --- | --- | --- | --- |
| To_Account | String | Да | Учетная запись целевого пользователя |
| Tags | Array | Да | Набор тегов |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode":0}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: Успешно; `FAIL`: Ошибка |
| ErrorCode | Integer | Код ошибки |
| ErrorInfo | String | Информация об ошибке |

## Коды ошибок

За исключением сетевых ошибок (например, ошибка 502), код возврата HTTP для этого API всегда составляет 200. `ErrorCode` **и** `ErrorInfo` **в ответе представляют фактический код ошибки и информацию об ошибке.** Для общих кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 90001 | Ошибка анализа формата JSON. Проверьте, соответствует ли JSON-запрос спецификациям JSON. |
| 90009 | Запрос требует разрешения администратора приложения. |
| 90018 | Количество запрашиваемых учетных записей превышает лимит. |
| 91000 | Внутренняя ошибка сервиса. Попробуйте снова. |

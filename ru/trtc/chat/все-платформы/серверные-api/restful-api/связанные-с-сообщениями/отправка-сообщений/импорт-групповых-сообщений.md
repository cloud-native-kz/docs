# Импорт групповых сообщений

## Обзор функции

- Этот API используется для импорта групповых сообщений без срабатывания обратных вызовов и отправки уведомлений.
- При миграции приложения из другой системы мгновенного обмена сообщениями в Tencent Cloud Chat вы можете использовать этот API для импорта данных групповых сообщений.

## Описание вызова API

### Применимые типы групп

| Тип группы | Поддержка этого RESTful API |
| --- | --- |
| Private | Да. То же самое, что рабочие группы (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. То же самое, что группы встреч (Meeting) в новой версии. |
| AVChatRoom | Нет |
| Community | Да |

Выше перечислены встроенные типы групп сервиса Chat. Дополнительные сведения см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

> **Примечание** Аудиовизуальные группы (AVChatRoom) не поддерживают импорт групповых сообщений. При попытке импорта групповых сообщений для аудиовизуальной группы будет возвращена ошибка 10007. Поэтому члены аудиовизуальной группы не могут просматривать сообщения, отправленные до их присоединения к группе.

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/import_group_msg?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/group_open_http_svc/import_group_msg | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Аккаунт администратора приложения. Дополнительные сведения см. в разделе **Администратор приложения** в разделе [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная аккаунтом администратора приложения. Дополнительные сведения см. в разделе [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное беззнаковое целое число в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса. Значение зафиксировано как `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

В одном запросе можно импортировать до семи групповых сообщений.
После импорта сообщений через этот API количество непрочитанных сообщений для всех членов будет равно `0`. Чтобы сохранить количество непрочитанных сообщений, необходимо импортировать членов группы или установить количество непрочитанных сообщений для членов после импорта всех сообщений.
Сообщения должны быть импортированы в порядке возрастания по временной метке, и временные метки импортируемых сообщений должны быть раньше текущего времени и позже времени создания группы и времени создания последнего сообщения в группе. В противном случае импорт не будет выполнен.

```
{    "GroupId": "@TGS#2C5SZEAEF",    "RecentContactFlag":1,// Означает запуск обновления беседы (это поле не поддерживается группами AVChatRoom.)    "MsgList": [        {            "From_Account": "leckie", // Отправитель сообщения            "SendTime":1620808101,            "Random": 8912345, // Случайное число сообщения (необязательно)            "MsgBody": [ // Тело сообщения, состоящее из массива элементов. Дополнительные сведения см. в объекте сообщения `TIMMessage`.                {                    "MsgType": "TIMTextElem", // Текст                    "MsgContent": {                        "Text": "красный пакет"                    }                },                {                    "MsgType": "TIMFaceElem", // Смайлик                    "MsgContent": {                        "Index": 6,                        "Data": "abc\\u0000\\u0001"                    }                }            ]        },        {            "From_Account": "peter", // Отправитель сообщения            "SendTime":1620892821,            "MsgBody": [ // Тело сообщения, состоящее из массива элементов. Дополнительные сведения см. в объекте сообщения `TIMMessage`.                {                    "MsgType": "TIMTextElem", // Текст                    "MsgContent": {                        "Text": "красный пакет"                    }                }            ]        }    ],    "CloudCustomData": "your cloud custom data"}
```

**Указание получателей сообщения**

Если добавить поле To_Account в тело сообщения для указания получателей сообщения, сообщение будет отправлено только этим назначенным получателям, и сообщение не будет учитываться как непрочитанное. (Поддерживаемые типы групп: Private, Public и ChatRoom).

> **Примечание** Эту функцию поддерживают только выпуск Flagship Edition и выпуск Enterprise Edition.

```
{    "GroupId": "@TGS#2C5SZEAEF",    "MsgList": [        {            "From_Account": "peter", // Указывает отправителя сообщения            "SendTime":1620892821,            "MsgBody": [ // Тело сообщения, состоящее из массива элементов. Дополнительные сведения см. в объекте TIMMessage.                {                    "MsgType": "TIMTextElem", // Текст                    "MsgContent": {                        "Text": "красный пакет"                    }                }            ],            "To_Account":["brennanli2", "brennanli3"] // Указывает получателей сообщения (до 50 членов). При использовании этого поля сообщение не будет учитываться как непрочитанное.        }    ]}
```

**Управление импортом сообщений**

Если добавить SendMsgControl в запрос, можно управлять поведением этого импорта сообщений. Если установлено NoMsgCheck, это означает, что после включения облачного аудита все сообщения, импортированные в этой операции, не будут отправлены на проверку.

```
{    "GroupId": "@TGS#2C5SZEAEF",    "SendMsgControl": ["NoMsgCheck"],    "MsgList": [        {            "From_Account": "peter", // Указывает отправителя сообщения            "SendTime": 1620892821,            "MsgBody": [ // Тело сообщения, состоящее из массива элементов. Дополнительные сведения см. в объекте TIMMessage.                {                    "MsgType": "TIMTextElem", // Текст                    "MsgContent": {                        "Text": "красный пакет"                    }                }            ],            "To_Account": ["brennanli2", "brennanli3"] // Указывает получателей сообщения (до 50 членов). При использовании этого поля сообщение не будет учитываться как непрочитанное.        }    ]}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы, для которой импортировать сообщения |
| RecentContactFlag | Integer | Нет | Следует ли запускать обновление беседы. Значение `1` означает запуск обновления беседы. Обновление беседы не запускается по умолчанию. Это поле не поддерживается аудиовизуальными группами (AVChatRoom) и группами сообществ. |
| MsgList | Object | Да | Список импортируемых сообщений |
| From_Account | String | Да | Отправитель сообщения |
| SendTime | Integer | Да | Время отправки сообщения |
| Random | Integer | Нет | 32-битное беззнаковое целое число. Если случайные числа двух сообщений в течение пяти минут совпадают, более позднее сообщение будет отклонено как повторяющееся сообщение. |
| MsgBody | Array | Да | TIM сообщение. Дополнительные сведения см. в определении `TIMMsgElement` в разделе [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| MsgType | String | Да | Тип объекта TIM сообщения. Допустимые значения: `TIMTextElem` (текстовое сообщение), `TIMFaceElem` (сообщение смайлика), `TIMLocationElem` (сообщение о местоположении), `TIMCustomElem` (пользовательское сообщение) |
| MsgContent | Object | Да | Объект TIM сообщения. Дополнительные сведения см. в определении `TIMMsgElement` в разделе [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| TopicId | String | Нет | ID темы, в которую импортируются сообщения. Это поле применяется только к группам сообществ с включенными темами. |
| CloudCustomData | String | Нет | Пользовательские данные сообщения. Они сохраняются в облаке и отправляются получателю. Такие данные могут быть загружены после удаления и переустановки приложения. |
| To_Account | Array | Нет | Укажите получателей сообщения (до 50 членов). При использовании этого поля сообщение не будет учитываться как непрочитанное. Эта функция поддерживается только в выпуске Flagship Edition и Enterprise Edition и совместима с рабочими группами (Work), открытыми социальными группами (Public) и временными группами встреч (Meeting). |
| SendMsgControl | String | Нет | Разрешение на импорт сообщений, действительно только для текущего сообщения. `NoMsgCheck` указывает, что после включения облачного аудита сообщения не будут отправлены на проверку. |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0,    "ImportMsgResult": [        {            "MsgSeq": 1,            "MsgTime": 1620808101,            "Result": 0        },        {            "MsgSeq": 2,            "MsgTime": 1620892821,            "Result": 0        },    ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorInfo | String | Информация об ошибке |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: ошибка |
| ImportMsgResult | Array | Результат импорта сообщения |
| Result | Integer | Результат каждого сообщения. `0`: сообщение успешно импортировано. `10004`: время отправки сообщения недействительно. `80002`: содержимое сообщения превышает лимит 12 000 байт. Пожалуйста, отрегулируйте размер сообщения. |
| MsgTime | Integer | Временная метка сообщения |
| MsgSeq | Integer | Порядковый номер сообщения, уникальный идентификатор сообщения |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда составляет 200, если не возникает ошибка сети (например, ошибка 502). Конкретный код ошибки и подробности можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для публичных кодов ошибок (60000-79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, относящиеся к этому API:

| Код ошибки | Описание |
| --- | --- |
| 10004 | Неправильный параметр. Проверьте описание ошибки и устраните проблему. |
| 10007 | Отсутствуют права доступа. Эта ошибка возникает, например, когда участник открытой группы пытается удалить других пользователей из группы (это может делать только администратор приложения). |
| 10010 | Группа не существует или была удалена. |
| 10015 | ID группы недействителен. Используйте правильный ID группы. |
| 10020 | Содержимое сообщения слишком длинное. В настоящее время максимальная поддерживаемая длина сообщения составляет 12 КБ. Пожалуйста, отрегулируйте длину сообщения. |

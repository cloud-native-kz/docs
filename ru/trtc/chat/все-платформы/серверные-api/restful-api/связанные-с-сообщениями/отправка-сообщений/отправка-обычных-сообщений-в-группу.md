# Отправка обычных сообщений в группу

## Обзор функции

Этот API используется администратором приложения для отправки обычных сообщений в группу.

## Описание вызова API

### Поддерживаемые типы групп

| ID типа группы | Поддержка RESTful API |
| --- | --- |
| Private | Да. Аналогично рабочим группам (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. Аналогично группам встреч (Meeting) в новой версии. |
| AVChatRoom | Да |
| Community | Да |

Это предустановленные типы групп в Chat. Дополнительную информацию см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/send_group_msg?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Другие параметры см. в разделе [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Силиконовая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/group_open_http_svc/send_group_msg | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Дополнительную информацию см. в разделе **Администратор приложения** в [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробнее см. в разделе [Создание UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса, который всегда должен быть `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

> **Примечание:** Когда пользователь отправляет сообщение, аватар и никнейм отправителя автоматически встраиваются в сообщение. Это позволяет получателю использовать эту информацию напрямую без необходимости запрашивать данные учетной записи у сервера после получения сообщения, что позволяет избежать ограничения частоты API и улучшить пользовательский опыт. Если требуется дополнительная информация о пользователе, рекомендуется встроить ее в полезную нагрузку сообщения и отправить вместе с сообщением.

### Пример запроса

- **Основной формат**
Администратор приложения отправляет обычные групповые сообщения, отправитель — администратор приложения.

```
{  "GroupId": "@TGS#2C5SZEAEF",  "Random": 8912345, // Случайное число. Если случайные номера двух сообщений одинаковы в течение пяти минут, они считаются одним сообщением.  "MsgBody": [ // Тело сообщения, состоящее из массива элементов. Подробности см. в описании полей.      {          "MsgType": "TIMTextElem", // Текст          "MsgContent": {              "Text": "red packet"          }      },      {          "MsgType": "TIMFaceElem", // Эмодзи          "MsgContent": {              "Index": 6,              "Data": "abc\\u0000\\u0001"          }      }  ],  "CloudCustomData": "your cloud custom data",  "SupportMessageExtension": 0,  "OfflinePushInfo": {      "PushFlag": 0, // Обычная отправка      "Desc": "Content to push offline",      "Ext": "Passthrough content",      "AndroidInfo": {          "Sound": "android.mp3"      },      "ApnsInfo": {          "Sound": "apns.mp3",          "BadgeMode": 1, // Если это поле оставить по умолчанию или установить значение `0`, сообщение подсчитывается. Если это поле установлено как `1`, сообщение не подсчитывается, т.е. число в верхнем правом углу значка не увеличивается.          "Title":"apns title", // Заголовок APNs          "SubTitle":"apns subtitle", // Подзаголовок APNs          "Image":"www.image.com" // URL изображения      }  }}
```

- **Указание отправителя сообщения**
Администратор приложения может указать члена группы в качестве отправителя сообщения в `From_Account`.
После получения сообщения другие члены увидят, что сообщение отправлено членом группы, указанным администратором приложения.

```
{  "GroupId": "@TGS#2C5SZEAEF",  "From_Account": "leckie", // Отправитель сообщения (опционально)  "Random": 8912345, // Случайное число. Если случайные номера двух сообщений одинаковы в течение пяти минут, они считаются одним сообщением.  "MsgBody": [ // Тело сообщения, состоящее из массива элементов. Подробности см. в описании полей.      {          "MsgType": "TIMTextElem", // Текст          "MsgContent": {              "Text": "red packet"          }      },      {          "MsgType": "TIMFaceElem", // Эмодзи          "MsgContent": {              "Index": 6,              "Data": "abc\\u0000\\u0001"          }      }  ]}
```

- **Указание получателя сообщения**
Если добавить `To_Account` в тело сообщения и указать получателя, сообщение будет отправлено только указанному получателю, и сообщение исключается из счетчика непрочитанных. (Поддерживаемые типы групп: Private, Public, ChatRoom)

> **Примечание:** Эта функция поддерживается только в издании Pro, Pro Plus и Enterprise.

```
{  "GroupId":"@TGS#12DEVUDHQ",  "Random":2784275388,  "MsgBody":[      {          "MsgType":"TIMCustomElem",          "MsgContent":{              "Data":"1cddddddddq1"          }      }  ],  "To_Account":["brennanli2", "brennanli3"] // Укажите получателя сообщения (можно указать до 50 получателей). Если это поле используется, сообщение исключается из счетчика непрочитанных.}
```

- **Указание, что сообщения не вызывают обновление диалога**
Если `SendMsgControl` установлен на `NoLastMsg`, это означает, что не следует вызывать обновление диалога; если установлен на `NoUnread`, это означает, что сообщение не включается в счетчик непрочитанных.

```
{   "GroupId": "@TGS#2C5SZEAEF",   "Random": 8912345, // Случайное число. Если случайные номера двух сообщений одинаковы в течение пяти минут, они считаются одним сообщением.   "SendMsgControl":["NoLastMsg"],// Не вызывать обновление диалога.   "MsgBody": [ // Тело сообщения, состоящее из массива элементов. Подробности см. в описании полей.       {           "MsgType": "TIMTextElem", // Текст           "MsgContent": {               "Text": "red packet"           }       },       {           "MsgType": "TIMFaceElem", // Эмодзи           "MsgContent": {               "Index": 6,               "Data": "abc\\u0000\\u0001"           }       }   ] }
```

- **Указание приоритета сообщения**
Вы можете указать приоритет сообщения. Приоритет по умолчанию — `Normal`.
Существует три варианта приоритета в порядке убывания: `High`, `Normal` и `Low`. Они чувствительны к регистру.

```
{  "GroupId": "@TGS#2C5SZEAEF",  "Random": 8912345, // Случайное число. Если случайные номера двух сообщений одинаковы в течение пяти минут, они считаются одним сообщением.  "MsgPriority": "High", // Приоритет сообщения  "MsgBody": [ // Тело сообщения, состоящее из массива элементов. Подробности см. в описании полей.      {          "MsgType": "TIMTextElem", // Текст          "MsgContent": {              "Text": "red packet"          }      },      {          "MsgType": "TIMFaceElem", // Эмодзи          "MsgContent": {              "Index": 6,              "Data": "abc\\u0000\\u0001"          }      }  ]}
```

- **Запрет обратного вызова для сообщения**
Когда переключатель обратного вызова включен, пользователи могут указать `ForbidCallbackControl` для управления тем, инициировать ли обратный вызов для одного сообщения. По умолчанию инициируется обратный вызов.

```
{  "GroupId": "@TGS#2C5SZEAEF",  "Random": 8912345, // Случайное число. Если случайные номера двух сообщений одинаковы в течение пяти минут, они считаются одним сообщением.  "ForbidCallbackControl":[          "ForbidBeforeSendMsgCallback",          "ForbidAfterSendMsgCallback"], // Опция управления запретом обратного вызова  "MsgBody": [ // Тело сообщения, состоящее из массива элементов. Подробности см. в описании полей.      {          "MsgType": "TIMTextElem", // Текст          "MsgContent": {              "Text": "red packet"          }      },      {          "MsgType": "TIMFaceElem", // Эмодзи          "MsgContent": {              "Index": 6,              "Data": "abc\\u0000\\u0001"          }      }  ]}
```

- **Отправка групповых сообщений с упоминанием (@)**
Упомянутые пользователи (@) в поле `GroupAtInfo` имеют взаимно однозначное и последовательное соответствие упомянутым пользователям @ в теле сообщения.

> **Внимание:** Аудиовидео-группы (AVChatRoom) не поддерживают функцию @all.

```
    {        "GroupId": "@TGS#2C5SZEAEF",        "Random": 8912345, // Случайное число. Если случайные номера двух сообщений одинаковы в течение пяти минут, они считаются одним сообщением.        "MsgBody": [ // Тело сообщения, состоящее из массива элементов. Подробности см. в описании полей.            {                "MsgType": "TIMTextElem", // Текст                "MsgContent": {                    "Text": "red @all @tommy @brennanli packet"                }            }        ],        // Это соответствует @all @tommy @brennanli в информации текста.        "GroupAtInfo":[        {            "GroupAtAllFlag":1 // `1`: @ все ; `0`: @ определенный член группы        },        {            "GroupAtAllFlag":0,            "GroupAt_Account":"tommy" // @ конкретный член группы        },        {            "GroupAtAllFlag":0,            "GroupAt_Account":"brennanli"        }     ]        }
```

- **Указание сообщений для доставки в режиме онлайн без хранения в оффлайне или в истории**
Если **OnlineOnlyFlag** в теле сообщения установлен на значение больше `0`, сообщение предназначено только для онлайн-доставки, без сохранения в оффлайне или в истории (недоступно для AVChatRoom или BChatRoom).

```
  {      "GroupId": "@TGS#2C5SZEAEF",      "Random": 8912345, // Случайное число. Если случайные номера двух сообщений одинаковы в течение пяти минут, они считаются одним сообщением.      "OnlineOnlyFlag": 1, // Сообщение предназначено только для онлайн-доставки (только онлайн-члены группы его получат), без сохранения в оффлайне или в истории.      "MsgBody": [ // Тело сообщения, состоящее из массива элементов. Подробности см. в описании полей.          {              "MsgType": "TIMTextElem", // Текст              "MsgContent": {                  "Text": "red packet"              }          },          {              "MsgType": "TIMFaceElem", // Эмодзи              "MsgContent": {                  "Index": 6,                  "Data": "abc\\u0000\\u0001"              }          }      ]  }
```

### Поля запроса

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы, в которую будет отправлено сообщение |
| Random | Integer | Да | 32-битное целое число без знака. Если содержание и случайные номера двух сообщений в течение пяти минут одинаковы, более позднее сообщение будет отклонено как дублирующееся. |
| MsgPriority | String | Нет | Приоритет сообщения |
| MsgBody | Array | Да | Тело сообщения. Дополнительную информацию см. в разделе [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| From_Account | String | Нет | Учетная запись источника сообщения. Если это поле не указано, отправитель сообщения — это учетная запись администратора приложения, используемая для вызова API. Кроме того, приложения могут указать отправителя сообщения в этом поле для реализации некоторых специальных функций. Обратите внимание, что если это поле указано, вы должны убедиться, что учетная запись в этом поле существует. |
| OfflinePushInfo | Object | Нет | Информация об оффлайн-доставке. Дополнительную информацию см. в разделе [Описание OfflinePushInfo](https://www.tencentcloud.com/document/product/1047/77715). |
| ForbidCallbackControl | Array | Нет | Опция запрета обратного вызова сообщения, действительна только для одного сообщения. `ForbidBeforeSendMsgCallback`: обратный вызов перед отправкой сообщения запрещен; `ForbidAfterSendMsgCallback`: обратный вызов после отправки сообщения запрещен. |
| OnlineOnlyFlag | Integer | Нет | `1`: отправлять только онлайн-членам; `0` (значение по умолчанию): отправлять всем членам. Это поле не действительно для аудиовидео-групп (AVChatRoom). |
| SendMsgControl | Array | Нет | Разрешение на отправку сообщения, действительно только для текущего сообщения. `NoLastMsg`: не вызывать обновление диалога; `NoUnread`: не включать сообщение в счетчик непрочитанных. (Если для сообщения установлен `OnlineOnlyFlag` в значение `1`, это поле не может быть использовано.) |
| CloudCustomData | String | Нет | Пользовательские данные сообщения. Они сохраняются в облаке и будут отправлены получателю. Такие данные могут быть извлечены после удаления и переустановки приложения. |
| SupportMessageExtension | Integer | Нет | Поддерживает ли сообщение расширение сообщения. `0`: Нет. `1`: Да |
| To_Account | Array | Нет | Укажите до 50 получателей сообщения. Если это поле используется, сообщение будет исключено из счетчика непрочитанных. Доступно только в издании Pro, Pro Plus и Enterprise для рабочих групп (Work), публичных групп (Public) и групп встреч (Meeting). |
| TopicId | String | Нет | ID темы, который указывает отправку обычных сообщений в тему и применяется только к сообществам с включенными темами. |
| NeedReadReceipt | Integer | Нет | Требуется ли для этого сообщения квитанция о прочтении. 0: Нет (по умолчанию), 1: Да. |
| GroupAtInfo | Array | Нет | Уведомление @ этого сообщения. GroupAtAllFlag: 1 обозначает @all; 0 обозначает конкретных членов. GroupAtUserList: Указывает UserID упомянутых членов. |

### Примеры ответов

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0,    "MsgTime": 1497249503,    "MsgSeq": 1,    "MsgDropReason" : "MsgFreqCtrl"}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: Успешно; `FAIL`: Ошибка |
| ErrorCode | Integer | Код ошибки. `0`: Успешно; другие значения: Ошибка |
| ErrorInfo | String | Информация об ошибке |
| MsgTime | Integer | Временная метка отправки сообщения, соответствующая серверному времени |
| MsgSeq | Integer | Порядковый номер сообщения, уникальный идентификатор сообщения |
| MsgDropReason | String | Причина отклонения сообщения. По умолчанию пусто. В настоящее время отклоняются только сообщения, превышающие лимит частоты. |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда равен 200, если не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000-79999) см. раздел [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10004 | Неверный параметр. Проверьте описание ошибки и устраните проблему. |
| 10007 | Нет прав на операцию. Эта ошибка возникает, например, когда член публичной группы пытается удалить других пользователей из группы (эту операцию может выполнить только администратор приложения). |
| 10010 | Группа не существует или была удалена. |
| 10015 | Неверный ID группы. Используйте корректный ID группы. |
| 10016 | Администратор приложения отклонил эту операцию через сторонний обратный вызов. |
| 10017 | Сообщение не может быть отправлено из-за отключения звука. Проверьте, отключен ли отправитель. |
| 10023 | Достигнут лимит частоты отправки сообщений. Повторите попытку позже. |
| 80002 | Содержимое сообщения слишком большое. В настоящее время максимальная поддерживаемая длина сообщения составляет 12 КБ. Пожалуйста, отрегулируйте длину сообщения. |

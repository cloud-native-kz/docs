# Отправка персональных сообщений одному пользователю

## Обзор функции

- Когда администратор отправляет сообщение учётной записи, получатель видит отправителя как администратора.
- Когда администратор указывает учётную запись для отправки сообщения другой учётной записи, получатель видит отправителя не как администратора, а как учётную запись, указанную администратором.
- Данный API не проверяет, являются ли отправитель и получатели друзьями, заблокированы ли друг другом или отключена ли отправка сообщений.
- Данный API не проверяет, отключены ли уведомления о сообщениях от отправителя по умолчанию. При необходимости укажите `WithMuteNotifications` в параметре `SendMsgControl`.
- Для персональных сообщений поле `MsgSeq` указывается пользователем при отправке сообщения. Его значение может повторяться. Оно не генерируется бэкэндом и не является глобально уникальным. Для групповых чатов значение поля `MsgSeq` генерируется бэкэндом, и каждая группа имеет свой `MsgSeq`, который строго возрастает от 1. Для персональных сообщений исторические сообщения одной беседы сортируются сначала по временной метке, а затем исторические сообщения одной секунды сортируются по `MsgSeq`.
- Сообщения C2C используют ключ дедупликации, состоящий из ID отправителя, `MsgSeq`, `MsgRandom` и контрольной суммы CRC32 содержимого сообщения. Этот ключ остаётся действительным в течение 120 секунд. В течение этого периода любые сообщения с одинаковыми значениями этих четырёх компонентов считаются дубликатами и автоматически фильтруются как бэкэндом, так и SDK.

> **Внимание** При вызове этого API для отправки персонального сообщения вы должны указать, следует ли синхронизировать сообщение с отправителем, то есть учётной записью администратора или учётной записью, указанной администратором. Синхронизация может быть реализована через онлайн-терминалы и серверы перемещаемости. Этот API предоставляет параметр `SyncOtherMachine` для определения того, синхронизировать ли сообщение. Для получения дополнительной информации см. **Примеры запросов** ниже.

### URL примера запроса

```
https://xxxxxx/v4/openim/sendmsg?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны изменённые параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/openim/sendmsg | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учётная запись администратора приложения. Дополнительную информацию см. в разделе **Администратор приложения** в [Аутентификация при входе](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учётной записью администратора приложения. Дополнительные сведения см. в разделе [Создание UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное беззнаковое целое число в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса, который всегда должен быть `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

Здесь мы используем отправку текстового сообщения в качестве примера. Для отправки сообщений других типов установите `MsgBody` на соответствующий тип сообщения. Дополнительную информацию см. в разделе [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527).

#### Администратор отправляет сообщение другой учётной записи.

> **Внимание** Если вы не хотите синхронизировать сообщение с `From_Account`, установите `SyncOtherMachine` в `2`. Чтобы синхронизировать сообщение с `From_Account`, установите `SyncOtherMachine` в `1`.

```
{    "SyncOtherMachine": 2, // Do not synchronize the message to the sender.    "To_Account": "lumotuwe2",    "MsgSeq": 93847636,    "MsgRandom": 1287657,    "MsgBody": [        {            "MsgType": "TIMTextElem",            "MsgContent": {                "Text": "hi, beauty"            }        }    ],    "CloudCustomData": "your cloud custom data",    "SupportMessageExtension": 0}
```

#### Администратор отправляет сообщение другой учётной записи и запрещает обратные вызовы для сообщения.

> **Внимание** Если вы не хотите синхронизировать сообщение с `From_Account`, установите `SyncOtherMachine` в `2`. Чтобы синхронизировать сообщение с `From_Account`, установите `SyncOtherMachine` в `1`.

```
{    "SyncOtherMachine": 2, // Do not synchronize the message to the sender.    "To_Account": "lumotuwe2",    "MsgSeq": 93847636,    "MsgRandom": 1287657,    "ForbidCallbackControl":[        "ForbidBeforeSendMsgCallback",        "ForbidAfterSendMsgCallback"], // Callback forbidding control option    "MsgBody": [        {            "MsgType": "TIMTextElem",            "MsgContent": {                "Text": "hi, beauty"            }        }    ],    "CloudCustomData": "your cloud custom data"}
```

#### Администратор указывает учётную запись для отправки сообщения другой учётной записи и устанавливает информацию об автономной отправке без синхронизации сообщения с `From_Account`.

> **Внимание** Если вы не хотите синхронизировать сообщение с `From_Account`, установите `SyncOtherMachine` в `2`.

```
{    "SyncOtherMachine": 2,    "From_Account": "lumotuwe1",    "To_Account": "lumotuwe2",    "MsgSeq": 93847636,    "MsgRandom": 1287657,    "MsgBody": [        {            "MsgType": "TIMTextElem",            "MsgContent": {                "Text": "hi, beauty"            }        }    ],    "CloudCustomData": "your cloud custom data",    "OfflinePushInfo": {        "PushFlag": 0,        "Desc": "Content to push offline",        "Ext": "Passthrough content",        "AndroidInfo": {            "Sound": "android.mp3"        },        "ApnsInfo": {            "Sound": "apns.mp3",            "BadgeMode": 1, // If this field is left as default or is set to `0`, the message is counted. If this field is set to `1`, the message is not counted, that is, the badge counter in the upper-right corner does not increase.            "Title":"apns title", // APNs title            "SubTitle":"apns subtitle", // APNs subtitle            "Image":"www.image.com" // Image URL        }    }}
```

#### Администратор указывает учётную запись для отправки сообщения другой учётной записи, и указывает, что сообщение будет отправлено только онлайн-пользователям, и исторические записи чата не будут сохранены (установите OnlineOnlyFlag в 1).

> **Внимание** Чтобы синхронизировать сообщение с `From_Account`, установите `SyncOtherMachine` в `1`.

```
{    "SyncOtherMachine": 1, // Synchronize the message to the sender.    "From_Account": "lumotuwe1",    "To_Account": "lumotuwe2",    "OnlineOnlyFlag": 1,    "MsgSeq": 93847636,    "MsgRandom": 1287657,    "MsgBody": [        {            "MsgType": "TIMTextElem",            "MsgContent": {                "Text": "hi, beauty"            }        }    ],    "CloudCustomData": "your cloud custom data"}
```

### Поля запроса

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| SyncOtherMachine | Integer | Нет | `1`: синхронизировать сообщение с онлайн-терминалом `From_Account` и сервером перемещаемости. `2`: не синхронизировать сообщение с `From_Account`. `3`: не синхронизировать сообщение с `To_Account`. Если это поле не указано, сообщение будет синхронизировано с сервером перемещаемости `From_Account`. |
| From_Account | String | Нет | `UserID` отправителя (используется для указания отправителя сообщения) |
| To_Account | String | Да | `UserID` получателя |
| OnlineOnlyFlag | Integer | Нет | Если для этого поля установлено значение `0`, сообщение будет отправлено только получателю онлайн и не будет сохранено в автономном режиме. `1` означает, что сообщения не будут сохранены в исторических записях чата, то есть если получатель онлайн, он получит сообщение; если получатель не онлайн, он не получит сообщение. Это подходит для реализации некоторых функций статуса в реальном времени, таких как функция «печать». |
| MsgSeq | Integer | Нет | Порядковый номер (32-битное беззнаковое целое число) сообщения. Бэкэнд будет использовать это поле для удаления дублирующихся сообщений и сортировки сообщений в одну секунду. Подробно см. **Описание функции**. Если это поле оставлено пустым, бэкэнд введёт случайное число. |
| MsgRandom | Integer | Да | Случайное число сообщения (32-битное беззнаковое целое число), используемое бэкэндом для дедупликации сообщений в одну секунду. Пожалуйста, убедитесь, что это поле заполнено случайным числом. |
| ForbidCallbackControl | Array | Нет | Поле запрета обратного вызова сообщения, которое действительно только для этого сообщения. `ForbidBeforeSendMsgCallback` запрещает обратный вызов перед отправкой сообщения. `ForbidAfterSendMsgCallback` запрещает обратный вызов после отправки сообщения. |
| SendMsgControl | Array | Нет | Параметры управления отправкой сообщений — это массив String, который действителен только для этого сообщения. «NoUnread» означает, что это сообщение не засчитывается в количество непрочитанных. «NoLastMsg» означает, что это сообщение не обновляет список бесед. «WithMuteNotifications» означает, что для этого сообщения вступает в силу параметр «Не беспокоить», установленный получателем для отправителя (по умолчанию не действует). «NoMsgCheck» означает, что при включённой облачной проверке это сообщение не будет отправлено на проверку. Пример: "SendMsgControl": ["NoUnread","NoLastMsg","WithMuteNotifications","NoMsgCheck"] |
| MsgBody | Array | Да | Тело сообщения. Подробную информацию о форматах см. в разделе [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). (Примечание: сообщение может содержать несколько элементов сообщения, в этом случае `MsgBody` — это массив.) |
| MsgType | String | Да | Тип объекта сообщения TIM. Допустимые значения: `TIMTextElem` (текстовое сообщение) `TIMLocationElem` (сообщение о местоположении) `TIMFaceElem` (сообщение с эмодзи) `TIMCustomElem` (пользовательское сообщение) `TIMSoundElem` (голосовое сообщение) `TIMImageElem` (сообщение с изображением) `TIMFileElem` (сообщение с файлом) `TIMVideoFileElem` (видеосообщение) |
| MsgContent | Object | Да | Различные типы объектов сообщений (`MsgType`) имеют разные форматы (`MsgContent`). Подробную информацию см. в разделе [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| CloudCustomData | String | Нет | Пользовательские данные сообщения. Сохраняется в облаке и будет отправлено другой стороне. Такие данные можно получить после удаления и переустановки приложения. |
| SupportMessageExtension | Integer | Нет | Поддерживает ли сообщение расширение сообщения. `0`: Нет. `1`: Да |
| OfflinePushInfo | Object | Нет | Информация, которая будет отправлена в автономном режиме. Дополнительную информацию см. в разделе [Описание OfflinePushInfo](https://www.tencentcloud.com/document/product/1047/77715). |
| IsNeedReadReceipt | Integer | Нет | Требует ли сообщение подтверждение прочтения. `0`: Нет. `1`: Да. Значение по умолчанию — `0`. |

### Пример ответа

- Ответ на успешный запрос

```
{  "ActionStatus": "OK",   "ErrorInfo": "",   "ErrorCode": 0,   "MsgTime": 1572870301,  "MsgKey": "89541_2574206_1572870301",  "MsgId": "144115233406643804-1727580296-4026038328"}
```

- Ответ на неудачный запрос

```
{  "ActionStatus": "FAIL",   "ErrorInfo": "Fail to Parse json data of body, Please check it",   "ErrorCode": 90001}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: Успешно; `FAIL`: Не удалось |
| ErrorCode | Integer | Код ошибки. `0`: Успешно; другие значения: Не удалось |
| ErrorInfo | String | Информация об ошибке |
| MsgTime | Integer | Временная метка сообщения в формате UNIX |
| MsgKey | String | Уникальный идентификатор сообщения. Это поле требуется для отзыва сообщения. Значение — это строка длиной не более 50 символов. |
| MsgId | String | Уникальный идентификатор сообщения на клиенте. |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда 200, если не возникает ошибка сети (например, ошибка 502). Конкретный код ошибки и подробности можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, характерные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 20001 | Неверный запрос. |
| 20002 | `UserSig` или `A2` истёк срок действия. |
| 20003 | `UserID` отправителя или получателя недействителен или не существует. Убедитесь, что `UserID` был импортирован в Chat. |
| 20004 | Ошибка сети. Повторите попытку. |
| 20005 | Внутренняя ошибка сервера. Повторите попытку. |
| 20006 | Был вызван обратный вызов перед отправкой персонального сообщения, и бэкэнд приложения вернул ответ, запрещающий доставку сообщения. |
| 90001 | Ошибка синтаксического анализа запроса JSON. Убедитесь, что формат правильный. |
| 90002 | `MsgBody` в запросе JSON не соответствует требованиям формата сообщения или `MsgBody` не является массивом. Дополнительную информацию см. в разделе **Элемент сообщения TIMMsgElement** в [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| 90003 | Запрос JSON не содержит поле `To_Account` или поле `To_Account` не является строкой. |
| 90005 | Запрос JSON не содержит поле `MsgRandom` или поле `MsgRandom` не является целым числом. |
| 90007 | Поле `MsgBody` в запросе JSON не является массивом. Измените его на массив. |
| 90009 | Запрос требует разрешений администратора приложения. |
| 90010 | Запрос JSON не соответствует требованиям формата сообщения. Дополнительную информацию см. в разделе **Элемент сообщения TIMMsgElement** в [Форматы сообщений](https://intl.cloud.tencent.com/document/product/1047/33527). |
| 90012 | Учётная запись, указанная в `To_Account`, не существует или не зарегистрирована. Убедитесь, что учётная запись была импортирована в Chat и правильна. |
| 90026 | Время автономного хранения сообщения неверно. Сообщения не могут храниться в автономном режиме более 7 дней. |
| 90031 | Поле `SyncOtherMachine` в запросе JSON не является целым числом. |
| 90044 | Поле `MsgLifeTime` в запросе JSON не является целым числом. |
| 91000 | Ошибка внутреннего сервиса. Повторите попытку. |
| 90992 | Ошибка внутреннего сервиса. Повторите попытку. Если этот код ошибки возвращается для всех запросов и сторонний обратный вызов включён, убедитесь, что сервер приложения нормально возвращает результаты обратного вызова бэкэнду Chat. |
| 93000 | Пакет JSON превышает максимальный размер 12 КБ. |
| 90048 | Запрашиваемая учётная запись не существует. |

# Получение квитанций о прочтении для групповых сообщений

## Обзор функции

Этот API используется для получения квитанций о прочтении групповых сообщений.

> **Примечание:** Эта функция доступна только в выпусках Pro, Pro Plus и Enterprise. Чтобы использовать её, необходимо [приобрести выпуск Pro, Pro Plus или Enterprise](https://www.tencentcloud.com/document/product/1047/34577), перейти в [консоль](https://console.trtc.io/chat), выбрать **Feature Configuration** > **Group configuration** > **Group Message Read Receipts** и включить эту функцию.

## Описание вызова API

### Применимые типы групп

| ID типа группы | Поддержка этого RESTful API |
| --- | --- |
| Private | Да. То же самое, что рабочие группы (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. То же самое, что группы встреч (Meeting) в новой версии. |
| AVChatRoom | Нет |
| Community | Нет |

Выше приведены встроенные типы групп Chat. Дополнительную информацию см. в разделе [Group System](https://intl.cloud.tencent.com/document/product/1047/33529).

### Пример URL запроса

```
https://xxxxxx/v4/group_open_http_svc/get_group_msg_receipt?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

| Параметр | Описание |
| --- | --- |

Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/group_open_http_svc/get_group_msg_receipt | API запроса |

| usersig | Подпись, созданная учётной записью администратора приложения. Подробности см. в разделе [Generating UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

Пакетное получение квитанций о прочтении групповых сообщений, включая количество прочитанных и непрочитанных.

- **Базовый формат**

```
{  "GroupId":"@TGS#2TTV7VSII",  "MsgSeqList":[        // Получить список значений `seq` сообщений      {"MsgSeq":1},      {"MsgSeq":2}  ]}
```

### Поля запроса

| Поле | Тип | Требуется | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | ID группы, для которой извлекаются квитанции о прочтении сообщений |
| MsgSeqList | Array | Да | Список значений `seq` сообщений |
| MsgSeq | Integer | Да | Seq сообщения для получения |

### Пример ответа

- **Базовый формат**

```
{  "ActionStatus": "OK",  "ErrorCode": 0,  "ErrorInfo": "",  "GroupMsgReceiptList": [   // Квитанции о прочтении      {          "Code": 0,          "MsgSeq": 1,                 "ReadNum": 1,      // Количество прочитанных группового сообщения          "UnreadNum": 6     // Количество непрочитанных группового сообщения      },      {          "Code": 0,          "MsgSeq": 2,          "ReadNum": 1,          "UnreadNum": 6      }  ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorInfo | String | Информация об ошибке |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: ошибка |
| GroupMsgReceiptList | Array | Квитанция о прочтении группового сообщения |
| Code | Integer | Код ошибки для квитанции о прочтении. Например, `10062` указывает, что квитанция о прочтении не существует. |
| MsgSeq | Integer | Seq сообщения для получения |
| ReadNum | Integer | Количество прочитанных группового сообщения |
| UnreadNum | Integer | Количество непрочитанных группового сообщения |

## Коды ошибок

Возвращаемый этим API код состояния HTTP всегда равен 200, если не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. раздел [Error Codes](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10003 | Неверное слово команды. |
| 10004 | Неверный параметр. Проверьте описание ошибки и устраните неполадку. |
| 10007 | Нет разрешения на операцию. Оператор должен иметь разрешения на выполнение соответствующих операций. |
| 10010 | Группа не существует или была удалена. |
| 10015 | Неверный ID группы. Используйте правильный ID группы. |
| 10062 | Квитанция о прочтении не существует. |

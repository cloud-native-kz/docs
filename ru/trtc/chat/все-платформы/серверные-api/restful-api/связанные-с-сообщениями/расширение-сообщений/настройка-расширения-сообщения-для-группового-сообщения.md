# Настройка расширения сообщения для группового сообщения

## Обзор функции

Администраторы приложений и члены группы могут настраивать расширение сообщения для обычных групповых сообщений. Расширение сообщения — это набор пользовательских пар «ключ-значение».

> **Примечание:** Для использования этой функции необходимо [приобрести выпуск Ultimate](https://www.tencentcloud.com/document/product/1047/34577) и включить функцию **Set message extension** на странице **Login and Message** в консоли Chat. Для каждого группового сообщения можно установить до 300 пар ключ-значение. Для групповых сообщений, для которых установлены пары ключ-значение, необходимо указать поддержку расширения сообщения при отправке. Дополнительные сведения об операциях см. в разделе [Отправка обычных сообщений в группе](https://intl.cloud.tencent.com/document/product/1047/34959).

## Описание вызова API

### Применимые типы групп

| Идентификатор типа группы | Поддержка RESTful API |
| --- | --- |
| Private | Да. Аналогично рабочим группам (Work) в новой версии. |
| Public | Да |
| ChatRoom | Да. Аналогично группе встреч (Meeting) в новой версии. |
| AVChatRoom | Нет |
| Community | Нет |

Это встроенные типы групп в Chat. Дополнительные сведения см. в разделе [Система групп](https://intl.cloud.tencent.com/document/product/1047/33529).

### Пример URL запроса

```
https://xxxxxx/v4/openim_msg_ext_http_svc/group_set_key_values?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/openim_msg_ext_http_svc/group_get_key_values | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Дополнительные сведения см. в разделе **App Admin** в статье [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Дополнительные сведения см. в разделе [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса. Значение фиксировано как `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Базовый формат**

Настройка расширений сообщения для группового сообщения с `MsgSeq` 158:

```
{    "GroupId": "@TGS#1YMVAB3IZ",    "MsgSeq": 158,    "OperateType": 1,    "ExtensionList": [        {"Key": "key1", "Value": "value1", "Seq": 0},        {"Key": "key2", "Value": "value2", "Seq": 0},    ]}
```

Удаление расширений сообщения для группового сообщения с `MsgSeq` 158:

```
{    "GroupId": "@TGS#1YMVAB3IZ",    "MsgSeq": 158,    "OperateType": 2,    "ExtensionList": [        {"Key": "key1", "Value": "", "Seq": 1}    ]}
```

Очистка всех расширений сообщения для группового сообщения с `MsgSeq` 158:

```
{    "GroupId": "@TGS#1YMVAB3IZ",    "MsgSeq": 158,    "OperateType": 3}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| GroupId | String | Да | Идентификатор группы |
| MsgSeq | Integer | Да | Seq группового сообщения, которое требуется настроить |
| OperateType | Integer | Да | `1`: установка пар ключ-значение для сообщения; `2`: удаление определенных пар ключ-значение для сообщения; `3`: очистка всех пар ключ-значение для сообщения |
| ExtensionList | Array | Да (когда `OperateType` равен `1` или `2`) | За один запрос можно установить или удалить до 20 пар ключ-значение |

Поля объектов в `ExtensionList` описаны следующим образом:

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| Key | String | Да | Ключ в паре ключ-значение, может быть до 100 байт |
| Value | String | Да | Значение в паре ключ-значение, может быть до 1000 байт |
| Seq | Integer | Да (когда API не вызывается администратором приложения) | Номер версии текущей пары ключ-значение. `Seq` равен `0`, когда член группы в первый раз вызывает API для настройки пары ключ-значение, и равен `1`, когда настройка выполнена успешно. При последующей настройке пары ключ-значение необходимо передать на серверную часть последний `Seq` пары ключ-значение, который затем проверяет, была ли пара изменена другими. Если `Seq` не совпадает с парой ключ-значение, настройка завершается сбоем и возвращается код ошибки. Серверная часть не проверяет `Seq` при вызове API администратором приложения. |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0,    "ExtensionList": [        {            "ErrorCode": 0,            "Extension": {                "Key": "key1",                "Value": "value1",                "Seq": 1            }        },        {            "ErrorCode": 23001,            "Extension": {                "Key": "key2",                "Value": "value1234",                "Seq": 2            }        }    ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: ошибка |
| ErrorInfo | String | Информация об ошибке |
| ExtensionList | Array | Результат настройки расширения сообщения |

## Коды ошибок

Возвращаемый HTTP код состояния для этого API всегда 200, если только не происходит сетевая ошибка (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10004 | Неверный параметр. Проверьте описание ошибки и устраните проблему. |
| 10008 | Неверный запрос: это не запрос плана Ultimate edition. |
| 23001 | Конфликт Seq: параметр ключ-значение был изменен другими. Необходимо получить последний Seq и повторить запрос. |
| 23002 | Настраиваемое групповое сообщение не поддерживает расширение сообщения. |
| 23003 | Слишком много попыток настройки пар ключ-значение (более 200 попыток в минуту на сообщение) |
| 23004 | Настраиваемое групповое сообщение не существует. |

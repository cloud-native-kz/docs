# Получение расширения сообщения для личного сообщения

## Обзор функции

Администраторы приложения и участники беседы могут получить расширение сообщения для личного сообщения. Расширение сообщения — это набор пользовательских пар ключ-значение.

> **Примечание** Для использования этой функции необходимо [приобрести издание Pro, Pro Plus или Enterprise](https://www.tencentcloud.com/document/product/1047/34577) и включить функцию **Set message extension** на странице **Login and Message** в консоли Chat. Для каждого личного сообщения можно установить до 300 пар ключ-значение. Для личных сообщений, для которых установлены пары ключ-значение, при отправке необходимо указать поддержку расширения сообщения. Подробнее см. [Отправка личных сообщений одному пользователю](https://intl.cloud.tencent.com/document/product/1047/34919).

### Пример URL запроса

```
https://xxxxxx/v4/openim_msg_ext_http_svc/get_key_values?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже описаны измененные параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/openim_msg_ext_http_svc/get_key_values | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Учетная запись администратора приложения. Для получения дополнительной информации см. раздел **App Admin** в разделе [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробнее см. [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса, который всегда должен быть `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Базовый формат**

Получить расширение сообщения личного сообщения с `MsgKey` равным `44739199_12_1665388280`:

```
{    "From_Account": "62768",    "To_Account": "116400",    "MsgKey": "44739199_12_1665388280"}
```

Получить расширение сообщения личного сообщения с `MsgKey` равным `44739199_12_1665388280` и `Seq` больше или равно 2:

```
{    "From_Account": "62768",    "To_Account": "116400",    "MsgKey": "44739199_12_1665388280",    "StartSeq": 2}
```

### Поля запроса

| Поле | Тип | Требуется | Описание |
| --- | --- | --- | --- |
| From_Account | String | Нет | `UserID` отправителя (используется для указания отправителя сообщения) |
| To_Account | String | Да | `UserID` получателя |
| MsgKey | String | Да | Уникальный идентификатор личного сообщения. Информацию о том, как получить `MsgKey` сообщения, см. в API отправки личных сообщений одному пользователю. |
| StartSeq | Integer | Нет | Расширение сообщения будет получено для сообщений, у которых `Seq` больше или равно `StartSeq`. |

### Пример ответа

```
{    "ErrorCode": 0,    "ErrorInfo": "",    "ActionStatus": "OK",    "CompleteFlag": 1,    "LatestSeq": 2,    "ClearSeq": 0,    "ExtensionList": [        {            "Key": "k1",            "Value": "v1",            "Seq": 1        },        {            "Key": "k2",            "Value": "v2",            "Seq": 1        },        {            "Key": "k3",            "Value": "v3",            "Seq": 2        }    ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно. `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки. `0`: успешно. Другие значения: ошибка |
| ErrorInfo | String | Информация об ошибке |
| ExtensionList | Array | Список возвращаемых пар ключ-значение, где пары ключ-значение отсортированы по `Seq`. За один запрос может быть возвращено до 200 пар ключ-значение. Если пар ключ-значение больше 200, они должны быть получены партиями на основе `StartSeq`. |
| CompleteFlag | Integer | `1`: все пары ключ-значение возвращены. `0`: возвращены не все пары ключ-значение. |
| LatestSeq | Integer | Последний `Seq` на сервере, который является максимальным значением `Seq` среди всех пар ключ-значение. |
| ClearSeq | Integer | `Seq` для операции очистки. Все пары ключ-значение, у которых `Seq` меньше или равно `ClearSeq`, должны быть удалены локально на клиенте. |

## Коды ошибок

HTTP код состояния, возвращаемый этим API, всегда 200, если только не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и подробности можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В таблице ниже описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10004 | Неправильный параметр. Проверьте описание ошибки и устраните проблему. |
| 10008 | Неправильный запрос: запрос не является запросом плана Pro, Pro Plus или Enterprise edition. |
| 23004 | Личное сообщение не существует. |

# Настройка расширения сообщения для одного-к-одному сообщения

## Обзор функции

Администраторы приложений и члены беседы могут настроить расширение сообщения для обычных одно-к-одному сообщений. Расширение сообщения — это набор пользовательских пар ключ-значение.

> **Примечание**Для использования этой функции необходимо [приобрести Pro edition, Pro Plus edition или Enterprise edition](https://www.tencentcloud.com/document/product/1047/34577) и включить функцию **Set message extension** на странице **Login and Message** в консоли Chat.Для каждого одно-к-одному сообщения можно установить до 300 пар ключ-значение.Для одно-к-одному сообщений, для которых установлены пары ключ-значение, при отправке необходимо указать поддержку расширения сообщения. Дополнительные сведения см. в разделе [Sending One-to-One Messages to One User](https://intl.cloud.tencent.com/document/product/1047/34919).

### Пример URL запроса

```
https://xxxxxx/v4/openim_msg_ext_http_svc/set_key_values?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Информацию о других параметрах см. в разделе [RESTful API Overview](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID.Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Сеул: `adminapikr.im.qcloud.com`Токио: `adminapijpn.im.qcloud.com`Франкфурт: `adminapiger.im.qcloud.com`Кремниевая долина: `adminapiusa.im.qcloud.com`Джакарта: `adminapiidn.im.qcloud.com` |
| v4/openim_msg_ext_http_svc/set_key_values | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Аккаунт администратора приложения. Дополнительную информацию см. в разделе **App Admin** в [Login Authentication](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, созданная аккаунтом администратора приложения. Дополнительные сведения см. в разделе [Generating UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса, который всегда должен быть `json`. |

### Максимальная частота вызовов

200 вызовов в секунду

### Пример запроса

- **Базовый формат**

Настройка расширения сообщения для одно-к-одному сообщения с `MsgKey` `44739199_12_1665388280`:

```
{    "From_Account": "62768",    "To_Account": "116400",    "MsgKey": "44739199_12_1665388280",    "OperateType": 1,    "ExtensionList": [        {"Key": "k1", "Value": "v1", "Seq": 0},        {"Key": "k2", "Value": "v2", "Seq": 0},        {"Key": "k3", "Value": "v3", "Seq": 0}    ]}
```

Удаление определенного расширения сообщения для одно-к-одному сообщения с `MsgKey` `44739199_12_1665388280`:

```
{    "From_Account": "62768",    "To_Account": "116400",    "MsgKey": "44739199_12_1665388280",    "OperateType": 2,    "ExtensionList": [        {"Key": "key1", "Value": "", "Seq": 1}    ]}
```

Очистка всех расширений сообщения для одно-к-одному сообщения с `MsgKey` `44739199_12_1665388280`:

```
{    "From_Account": "62768",    "To_Account": "116400",    "MsgKey": "44739199_12_1665388280",    "OperateType": 3}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| From_Account | String | Нет | `UserID` отправителя (используется для указания отправителя сообщения) |
| To_Account | String | Да | `UserID` получателя |
| MsgKey | String | Да | Уникальный идентификатор одно-к-одному сообщения. Информацию о том, как получить `MsgKey` сообщения, см. в API отправки одно-к-одному сообщений одному пользователю. |
| OperateType | Integer | Да | `1`: установка пар ключ-значение для сообщения; `2`: удаление определенных пар ключ-значение для сообщения; `3`: очистка всех пар ключ-значение для сообщения |
| ExtensionList | Array | Да (когда `OperateType` равно `1` или `2`) | Можно устанавливать или удалять до 20 пар ключ-значение за один запрос |

Поля объектов в `ExtensionList` описаны следующим образом:

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| Key | String | Да | Ключ в паре ключ-значение, который может быть до 100 байт |
| Value | String | Да | Значение в паре ключ-значение, которое может быть до 1000 байт |
| Seq | Integer | Да (когда API не вызывается администратором приложения) | Номер версии текущей пары ключ-значение. `Seq` равно `0`, когда API впервые вызывается членом беседы для настройки пары ключ-значение, и равно `1` когда настройка успешна. Для последующей настройки пары ключ-значение необходимо передать последний `Seq` пары ключ-значение на сервер, который затем проверяет, была ли пара ключ-значение изменена другими. Если `Seq` не совпадает с парой ключ-значение, настройка завершается ошибкой и возвращается код ошибки. Сервер не проверяет `Seq` при вызове API администратором приложения. |

### Пример ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0,    "ExtensionList": [        {            "ErrorCode": 0,            "Extension": {                "Key": "k1",                "Value": "v1",                "Seq": 1            }        },        {            "ErrorCode": 23001,            "Extension": {                "Key": "k2",                "Value": "v1234",                "Seq": 2            }        }    ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно. `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки. `0`: успешно. Другие значения: ошибка |
| ErrorInfo | String | Информация об ошибке |
| ExtensionList | Array | Результат настройки расширения сообщения |

## Коды ошибок

HTTP-код состояния, возвращаемый этим API, всегда равен 200, если не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Общие коды ошибок (60000–79999) см. в разделе [Error Codes](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 10002 | Внутренняя ошибка сервера. Повторите попытку. |
| 10004 | Неверный параметр. Проверьте описание ошибки и устраните проблему. |
| 10008 | Неверный запрос: запрос не является запросом плана Pro edition, Pro Plus edition или Enterprise edition. |
| 23001 | Конфликт Seq: параметр ключ-значение был изменен другими. Необходимо получить последний Seq и повторить запрос. |
| 23002 | Настроенное одно-к-одному сообщение не поддерживает расширение сообщения. |
| 23003 | Слишком много попыток установки ключ-значение (более 200 попыток в минуту на сообщение) |
| 23004 | Настроенное одно-к-одному сообщение не существует. |

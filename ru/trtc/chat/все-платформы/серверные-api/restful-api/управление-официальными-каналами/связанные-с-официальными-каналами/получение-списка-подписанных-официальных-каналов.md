# Получение списка подписанных официальных каналов

## Обзор функции

Этот API используется администратором приложения для получения списка информации всех официальных каналов, на которые подписан пользователь.

> **Примечание:** Функция доступна только для Enterprise. Чтобы включить её, [свяжитесь с нами](https://trtc.io/zh/contact). Вы можете [протестировать](https://trtc.io/document/34577?product=chat#.E5.89.8D.E6.8F.90.E6.9D.A1.E4.BB.B6) эту функцию в консоли. Пробная версия позволяет создавать до 5 открытых аккаунтов, действительна в течение одного месяца.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/official_account_open_http_svc/get_subscribed_official_account_list?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже перечислены только параметры, изменённые при вызове этого API, и их описания. Подробную информацию о параметрах см. в разделе [Обзор RESTful API](https://www.tencentcloud.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Выделенное доменное имя, соответствующее стране/региону SDKAppID: Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/official_account_open_http_svc/get_subscribed_official_account_list | API запроса. |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения. |
| identifier | Учётная запись администратора приложения. Подробнее см. в разделе [Администратор приложения](https://www.tencentcloud.com/document/product/1047/33517#app-.E7.AE.A1.E7.90.86.E5.91.98). |
| usersig | Подпись, созданная учётной записью администратора приложения. Подробнее см. в разделе [Генерирование UserSig](https://www.tencentcloud.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295. |
| contenttype | Значение всегда `json`. |

### Максимальная частота вызовов

200 раз/секунду

### Пример пакета запроса

```
{   "Subscriber_Account": "107867", 		//Subscriber   "Limit": 200,		   "Offset": 0,   "ResponseFilter":{		//Filter fields      "OfficialAccountBaseInfoFilter":[	//Filter fields for official channel information	            "CreateTime",            "Name",            "Owner_Account",            "LastMsgTime",            "SubscriberNum",            "Introduction",            "FaceUrl",            "Organization",            "CustomString"      ],      "SelfInfoFilter":[		//Filter fields for subscriber information            "SubscribeTime",            "CustomString"      ]   }}
```

### Поля запроса

| Поле | Тип | Требуется | Описание |
| --- | --- | --- | --- |
| Subscriber_Account | String | Да | ID подписчика, информацию которого необходимо изменить. |
| Limit | Integer | Нет | Количество официальных каналов, получаемых при одной выборке, в настоящее время поддерживает максимальное значение 200. Если не указано, по умолчанию 200. |
| Offset | Integer | Нет | Указывает, с какого официального канала начинать выборку. |
| ResponseFilter | Object | Нет | Включает два фильтра: OfficialAccountBaseInfoFilter и SelfInfoFilter. OfficialAccountBaseInfoFilter: указывает на основные поля информации официальных каналов для выборки, в основном включая: CreateTime: время создания официального канала Name: название официального канала Owner_Account: пользователь, владеющий официальным каналом LastMsgTime: время последнего сообщения в официальном канале SubscriberNum: текущее количество подписчиков официального канала Introduction: описание официального канала FaceUrl: URL аватара официального канала Organization: организация официального канала CustomString: пользовательское поле на уровне официального канала Если не указано, будут выбраны все поля. SelfInfoFilter: указывает на необходимость выборки собственной информации подписчика, в основном включая: SubscribeTime: время подписки подписчика на официальный канал CustomString: пользовательское поле на уровне подписчика Если не указано, будут выбраны все поля. |

### Пример пакета ответа

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0,    "TotalCount": 8,	//The total number of subscribed official channels, unrelated to the pull limit    "OfficialAccountInfoList": [		//List of official channels returned        {            "Name": "test_mill_subscriber_OA",			            "Official_Account": "@TOA#_test_mill_subscriber_OA",		            "Owner_Account": "107867",		            "LastMsgTime": 1698894911,		            "SubscriberNum": 499800,            "Introduction": "This is test Introduction",            "FaceUrl": "http://this.is.face.url",            "Organization": "Tencent",            "CustomString": "test_custom_string",            "SelfInfo": {                "SubscribeTime": 1700655351,                "CustomString": ""       }       //...There might be a lot more information to follow, but here information of only one official channel is displayed.            ],    "CompleteFlag": 1,		//Indicates whether the pull has ended. 1 means it has ended; 0 means it has not ended.    "Next": 0		//If the pull has ended, returns 0.}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки: OK: успешно. FAIL: не удалось. |
| ErrorCode | Integer | Код ошибки: 0: успешно. Другие значения: не удалось. |
| ErrorInfo | String | Информация об ошибке. |
| OfficialAccountInfoList | Array | Полученная информация об официальном канале. Результаты — это отфильтрованная информация на основе установленных полей фильтра. |
| TotalCount | Integer | Количество официальных каналов, на которые подписан пользователь. |
| CompleteFlag | Integer | Указывает, завершена ли выборка. 1: выборка завершена. 0: выборка не завершена. |
| Next | Integer | Указывает на следующую выборку. Если текущая выборка не завершена, можно использовать смещение следующей выборки в качестве смещения для разбиения на страницы. |

## Коды ошибок

Если не возникает сетевая ошибка (например, ошибка 502), код возврата HTTP для этого API всегда равен 200. ErrorCode и ErrorInfo в пакетах ответа представляют фактический код ошибки и информацию об ошибке.
Для распространённых кодов ошибок (от 60000 до 79999) см. раздел [Коды ошибок](https://www.tencentcloud.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, относящиеся к этому API:

| Код ошибки | Описание |
| --- | --- |
| 6017 | Неверный параметр запроса. Пожалуйста, измените и повторите попытку. |
| 10004 | Внутренняя ошибка сервера. Пожалуйста, попробуйте снова. |
| 20001 | Неверный пакет запроса. Пожалуйста, измените и повторите попытку. |
| 20005 | Внутренняя ошибка сервера. Пожалуйста, попробуйте снова. |
| 130002 | Недостаточно прав доступа. Пожалуйста, проверьте параметры запроса в соответствии с сообщением об ошибке. |
| 130004 | Пользователь официального канала не существует, или он существовал, но теперь был завершён. |
| 130007 | Неправильный ID пользователя официального канала. Пожалуйста, измените и повторите попытку. |
| 131000 | Для этого AppId не включена служба официального канала. |

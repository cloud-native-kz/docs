# Удаление пользователя с места

## Обзор функции

Администратор приложения может удалять членов с места через этот API.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/room_engine_http_mic/kick_user_off_seat?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В приведённой ниже таблице указаны только параметры, измененные при вызове этого интерфейса. Дополнительную информацию о других параметрах см. в разделе [Обзор RESTful API](https://www.tencentcloud.com/document/product/647/60703#).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Зарезервированное доменное имя для страны/региона, где находится SDKAppID: Сингапур: `adminapisgp.im.qcloud.com` |
| v4/room_engine_http_mic/kick_user_off_seat | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Должна быть учётной записью администратора приложения. Дополнительную информацию см. в разделе [Администратор приложения](https://www.tencentcloud.com/document/product/1047/33517#app-.E7.AE.A1.E7.90.86.E5.91.98). |
| usersig | Подпись, созданная учётной записью администратора приложения. Дополнительную информацию см. в разделе [Генерация UserSig](https://www.tencentcloud.com/document/product/1047/34385). |
| random | Введите случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса, фиксированное значение `json` |

### Максимальная частота вызовов

200 раз в секунду.

### Пример пакета запроса

**Основная форма**

```
{  "RoomId":"room-test",  "Member_Account":"user2",  "Data":"time over"}
```

### Описание полей пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| RoomId | String | Обязательно | ID комнаты |
| Member_Account | String | Обязательно | ID пользователя на месте |
| Data | String | Обязательно | Пользовательская информация, которая отправляется, когда пользователь удаляется с места и отправляется уведомление. |

### Пример пакета ответа

**Основная форма**

```
{  "ErrorCode": 0,  "ErrorInfo": "",  "ActionStatus": "OK",  "RequestId": "Id-7c1680be52734bdc8d6de398ab9505e7-O-Seq-57717"}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса. OK для успеха, FAIL для неудачи. |
| ErrorCode | Integer | Код ошибки. 0 для успеха, другие значения для неудачи. |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Уникальный ID запроса возвращается с каждым запросом и требуется предоставить этот RequestId при локализации проблем. |

## Коды ошибок

Если не возникает ошибка сети (например, ошибка 502), код статуса HTTP для этого интерфейса всегда будет 200. Фактические коды ошибок и сообщения передаются через ErrorCode и ErrorInfo в теле ответа.
Для общих кодов ошибок (60000-79999) см. документацию [Код ошибки](https://www.tencentcloud.com/document/product/647/57276#).
Приватные коды ошибок для этого API следующие:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, повторите попытку. |
| 100002 | Неверный параметр, проверьте правильность запроса на основе описания ошибки. |
| 100004 | Комната не существует, или она когда-то существовала, но теперь была уничтожена. |

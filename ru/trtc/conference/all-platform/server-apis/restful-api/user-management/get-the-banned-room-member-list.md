# Получить список забанленных членов комнаты

## Обзор функции

Администраторы приложений могут получить список забанленных членов через этот API.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/room_engine_http_srv/get_banned_member_list?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже указаны только параметры, измененные при вызове этого API. Для получения дополнительной информации см. [Обзор RESTful API](https://www.tencentcloud.com/document/product/647/60703#).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Зарезервированный домен для страны/региона, где расположен SDKAppID: Сингапур: `adminapisgp.im.qcloud.com` |
| v4/room_engine_http_srv/get_banned_member_list | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Должна быть учетная запись администратора приложения. Для получения дополнительной информации см. [Администратор приложения](https://www.tencentcloud.com/document/product/1047/33517#app-.E7.AE.A1.E7.90.86.E5.91.98). |
| usersig | Подпись, созданная учетной записью администратора приложения. Для получения дополнительной информации см. [Создание UserSig](https://www.tencentcloud.com/document/product/1047/34385). |
| random | Введите случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса, фиксированное значение `json` |

### Максимальная частота вызовов

200 раз в секунду.

### Пример пакета запроса

**Базовая форма**

```
{  "RoomId":"room-test",  "Limit":100,  "Offset":0}
```

### Описание полей пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| RoomId | String | Обязательный | ID комнаты |
| Limit | Integer | Необязательный | Получить количество забанленных членов за один раз, максимум 100 |
| Next | String | Необязательный | Смещение, начинается с 0 для первого запроса, последующие запросы основаны на значении NextOffset в пакете ответа |

### Пример пакета ответа

**Базовая форма**

```
{  "ErrorCode": 0,  "ErrorInfo": "",  "ActionStatus": "OK",  "RequestId": "Id-99d6df2213b54216a02bc18024a5c218-O-Seq-57681",  "Response": {      "BannedAccountList": [          {              "Member_Account": "user2",              "BannedUntil": 1703559417          }      ],      "NextOffset": 0  }}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат процесса запроса. OK для успеха, FAIL для неудачи. |
| ErrorCode | Integer | Код ошибки. 0 для успеха, остальные для неудачи. |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Уникальный ID запроса возвращается с каждым запросом и требуется предоставить этот RequestId при локализации проблем. |
| BannedAccountList | Array | Информация о забанленных членах, BannedUntil - время истечения бана участника, Member_Account - ID учетной записи забанленного члена. |
| NextOffset | Integer | Значение Offset для следующего запроса. Если это значение равно 0, это означает, что список забанленных членов полностью получен. |

## Коды ошибок

Если не произойдет сетевая ошибка (например, ошибка 502), код состояния HTTP для этого интерфейса всегда будет 200. Фактические коды ошибок и сообщения передаются через ErrorCode и ErrorInfo в теле ответа.
Для общих кодов ошибок (60000-79999) см. документацию [Код ошибки](https://www.tencentcloud.com/document/product/647/57276#).
Приватные коды ошибок для этого API следующие:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, повторите попытку. |
| 100002 | Неверный параметр, проверьте корректность запроса на основе описания ошибки. |
| 100004 | Комната не существует, или она существовала ранее, но теперь была удалена. |
| 100005 | Не членом комнаты |
| 100006 | Недостаточно прав на выполнение операции |

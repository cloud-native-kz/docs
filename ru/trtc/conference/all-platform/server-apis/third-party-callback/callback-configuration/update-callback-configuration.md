# Обновление конфигурации обратного вызова

## Обзор функции

Администраторы приложения могут создать обратный вызов через этот API.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/room_config/update_callback?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже перечислены только параметры, которые изменяются при вызове этого API. Для получения дополнительной информации см. [Обзор RESTful API](https://www.tencentcloud.com/document/product/647/60703#).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Зарезервированное доменное имя для страны/региона, где находится SDKAppID: Сингапур: `adminapisgp.im.qcloud.com` |
| v4/room_config/update_callback | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Должна быть учетная запись администратора приложения. Для получения дополнительной информации см. [Администратор приложения](https://www.tencentcloud.com/document/product/1047/33517#app-.E7.AE.A1.E7.90.86.E5.91.98). |
| usersig | Подпись, созданная учетной записью администратора приложения. Для получения дополнительной информации см. [Генерация UserSig](https://www.tencentcloud.com/document/product/1047/34385). |
| random | Введите случайное 32-битное беззнаковое целое число в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса — фиксированное значение `json` |

### Максимальная частота вызовов

200 раз в секунду.

### Пример пакетов запроса

**Базовая форма**

```
{  "Url":"http://www.example.com/callback",  "CallbackCommandList":[      "Room.CallbackAfterMemberLeave",      "Mic.CallbackAfterSeatInfoChanged"  ]}
```

### Описание полей пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| Url | String | Обязательно | Адрес обратного вызова должен начинаться с http/https, рекомендуется использовать более безопасный https. |
| CallbackCommandList | Array | Обязательно | Словоформы команд, которые запускают обратный вызов, см. [Список словоформ команд обратного вызова](https://www.tencentcloud.com/document/product/647/60720#). |

### Пример пакетов ответа

**Базовая форма**

```
{  "ActionStatus": "OK",  "ErrorInfo": "",  "ErrorCode": 0,  "RequestId": "Id-8c9858f01e954611ae2d4c1b1ed7d583-O-Seq-52720"}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат процесса запроса. OK для успеха, FAIL для отказа. |
| ErrorCode | Integer | Код ошибки. 0 для успеха, остальные для отказа. |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Возвращается уникальный идентификатор запроса с каждым запросом, необходимо предоставить этот RequestId при поиске проблем. |

## Коды ошибок

Если не происходит сетевая ошибка (например, ошибка 502), код состояния HTTP для этого интерфейса всегда будет 200. Фактические коды ошибок и сообщения передаются через ErrorCode и ErrorInfo в теле ответа.
Для общих кодов ошибок (60000–79999) см. документацию [Код ошибки](https://www.tencentcloud.com/document/product/647/57276#).
Приватные коды ошибок для этого API следующие:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, пожалуйста, повторите попытку. |
| 100002 | Параметр невалиден, пожалуйста, проверьте запрос на корректность на основе описания ошибки. |
| 100301 | Конфигурация обратного вызова не существует, вы можете создать её с помощью интерфейса Create Callback Configuration. |

# После уничтожения комнаты

## Обзор функции

Бэкэнд приложения может просматривать динамику уничтожения комнаты в реальном времени через этот обратный вызов, включая: запись в реальном времени уничтожения комнаты (например, запись логов или синхронизацию с другими системами).

## Примечания

- Чтобы включить обратный вызов, должен быть настроен URL обратного вызова и активирован переключатель, соответствующий этому протоколу обратного вызова. Методы конфигурации см. в документе [Конфигурация сторонних обратных вызовов](https://www.tencentcloud.com/document/product/1047/34520).
- Направление обратного вызова идёт от бэкэнда комнаты к бэкэнду приложения через HTTP POST запрос.
- После получения запроса обратного вызова бэкэнд приложения должен проверить, что SDKAppID, содержащийся в URL запроса, совпадает с его собственным SDKAppID.

## Сценарии

- Пользователи приложения успешно уничтожают комнату через клиент.
- Администраторы приложения успешно уничтожают комнату через RESTful API.

## Время срабатывания обратного вызова

После успешного уничтожения комнаты.

## Описание API

### Пример URL запроса

В следующем примере URL обратного вызова, настроенный в приложении, — это `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST |
| www.example.com | URL обратного вызова |
| SdkAppid | SDKAppID, назначенный консолью Chat при создании приложения |
| CallbackCommand | Зафиксирован как Room.CallbackAfterDestroyRoom |
| contenttype | Зафиксированное значение: JSON |
| ClientIP | IP адрес клиента, формат, например 127.0.0.1 |
| OptPlatform | Платформа клиента. Для значения см. описание параметра OptPlatform в [Обзоре Webhook: Протокол обратного вызова](https://www.tencentcloud.com/document/product/1047/34354#.E5.9B.9E.E8.B0.83.E5.8D.8F.E8.AE.AE). |

### Пример пакета запроса

```
{    "CallbackCommand":"Room.CallbackAfterDestroyRoom",    "Operator_Account":"admin",    "RoomId":"tandy-test-rest",    "EventTime":1703589922780}
```

### Описание полей пакета запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда обратного вызова |
| Operator_Account | String | UserID оператора, инициирующего запрос на уничтожение комнаты |
| RoomId | String | ID комнаты |
| EventTime | Integer | Временная метка срабатывания события в миллисекундах |

### Пример пакета ответа

Пакет ответа обратного вызова отправляется после того, как бэкэнд приложения синхронизирует данные.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // Игнорировать результат обратного вызова}
```

### Описание полей пакета ответа

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательное | Результат обработки запроса. OK для успеха, FAIL для ошибки. |
| ErrorCode | Integer | Обязательное | Код ошибки, 0 означает игнорировать результат ответа |
| ErrorInfo | String | Обязательное | Сообщение об ошибке |

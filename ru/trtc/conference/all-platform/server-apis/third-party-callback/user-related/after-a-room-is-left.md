# После выхода из комнаты

## Обзор функции

Серверная часть приложения может просматривать сообщения о членах комнаты в режиме реального времени через этот обратный вызов, включая: уведомление серверной части приложения о том, что член покинул комнату, что позволяет приложению выполнить необходимую синхронизацию данных.

## Примечания

- Для включения обратного вызова необходимо настроить URL-адрес обратного вызова и активировать переключатель, соответствующий протоколу этого обратного вызова. Методы конфигурации см. в документе [Конфигурация обратного вызова третьей стороны](https://www.tencentcloud.com/document/product/1047/34520).
- Направление обратного вызова — от серверной части комнаты к серверной части приложения через HTTP POST запрос.
- После получения запроса обратного вызова серверная часть приложения должна проверить, соответствует ли SDKAppID, содержащийся в URL-адресе запроса, своему собственному SDKAppID.

## Сценарии

- Пользователи приложения активно выходят из комнаты через клиент.
- Пользователи приложения исключены из комнаты.
- Пользователи приложения вынуждены покинуть комнату из-за истечения срока сердцебиения.

## Время срабатывания обратного вызова

После успешного выхода пользователя из комнаты.

## Описание API

### Пример URL-адреса запроса

В следующем примере URL-адрес обратного вызова, настроенный в приложении, — `https://www.example.com`.
**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST |
| www.example.com | URL-адрес обратного вызова |
| SdkAppid | SDKAppID, назначенный консолью Chat при создании приложения |
| CallbackCommand | Фиксированное значение: Room.CallbackAfterMemberLeave |
| contenttype | Фиксированное значение: JSON |
| ClientIP | IP-адрес клиента, формат, например 127.0.0.1 |
| OptPlatform | Платформа клиента. Информацию о значении см. в описании параметра OptPlatform в [Обзор Webhook: Протокол обратного вызова](https://www.tencentcloud.com/document/product/1047/34354#.E5.9B.9E.E8.B0.83.E5.8D.8F.E8.AE.AE). |

### Пример пакета запроса

```
{  "CallbackCommand":"Room.CallbackAfterMemberLeave",  "Operator_Account":"user1",  "RoomId":"rid-123",  "MemberCount":20,  "Type":"Leave",   "MemberList_Account":["user1"],  "Reason":"xxxx",  "EventTime":1670574414123// Миллисекундный уровень, временная метка срабатывания события}
```

### Описание полей пакета запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда обратного вызова |
| Operator_Account | String | UserID запрашивающего |
| RoomId | String | ID комнаты |
| MemberCount | Integer | Вместимость комнаты |
| Type | String | Способ выхода: Leave (самостоятельно покинул); Kicked (исключен); Expired (сердцебиение истекло) |
| MemberList_Account | Array | Список членов, вышедших из комнаты |
| Reason | String | Причина выхода |
| EventTime | Integer | Временная метка срабатывания события в миллисекундах |

### Пример пакета ответа

Пакет ответа обратного вызова отправляется после синхронизации данных серверной частью приложения.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // Игнорировать результат обратного вызова}
```

### Описание полей пакета ответа

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Обязательное | Результат обработки запроса. OK — успех, FAIL — ошибка. |
| ErrorCode | Integer | Обязательное | Код ошибки, 0 означает игнорировать результат ответа |
| ErrorInfo | String | Обязательное | Сообщение об ошибке |

# Совместное использование экрана

Этот документ подробно описывает функции совместного использования экрана в `TUIRoomKit` и помогает вам лучше освоить операции, связанные с совместным использованием экрана во время встреч `TUIRoomKit`. С помощью этого документа вы сможете в полной мере использовать возможности TUIRoomKit для достижения высокого качества видеоконференций. Если взаимодействие пользовательского интерфейса TUIRoomKit не соответствует требованиям вашего продукта и вам требуется настроить взаимодействие, связанное с совместным использованием экрана, вы можете получить доступ к соответствующему API TUIRoomEngineSDK для решения ваших задач.

## Функция

TUIRoomKit поддерживает совместное использование экрана, которое позволяет пользователям делиться своим экраном после входа в комнату, а другие члены комнаты могут просматривать передаваемый экран. В этой статье подробно описана функциональность этого средства и объясняется, как его использовать в компонентах TUIRoomKit.

## Инструкции по использованию

После входа в комнату пользователи на платформах `Android&iOS&Flutter` могут осуществлять совместное использование экрана, нажав кнопку совместного использования на нижней панели инструментов.

Из-за изменения политики конфиденциальности системы Android, если вы используете функции записи экрана и другие функции в приложениях Android 10 и выше, вы должны использовать Foreground Service, иначе система выдаст ошибку или приложение будет завершено системой при записи/совместном использовании экрана. В нашем компоненте этот Android Foreground Service будет автоматически включен для вас, вам просто нужно нажать кнопку совместного использования экрана и согласиться с соответствующей политикой конфиденциальности для совместного использования.

Код для Android Foreground Service можно найти в наших [примерах кода](https://github.com/Tencent-RTC/TUIRoomKit/blob/main/Android/tuiroomkit/src/main/java/com/tencent/cloud/tuikit/roomkit/common/KeepAliveService.java#L36).

Совместное использование экрана между приложениями на iOS требует добавления процесса расширения записи для работы с основным процессом приложения при потоковой передаче. Процесс расширения записи создается системой, когда она должна записать экран, и отвечает за получение изображений экрана, захваченных системой. Поэтому вам необходимо:

1. создать App Group и настроить его в XCode (дополнительно). Целью этого шага является разрешить процессу расширения записи экрана взаимодействовать между процессами с основным процессом приложения.
2. В вашем проекте создать новый Target для Broadcast Upload Extension и интегрировать [TXLiteAVSDK_ReplayKitExt.framework](https://trtc.io/sdkDownload?id=conference), который специально предназначен для расширения в архиве SDK.
3. Нажмите кнопку "Совместное использование экрана", чтобы начать совместное использование экрана.

> **Примечание:** Если вы пропустите [Шаг 1](https://www.tencentcloud.com/document/product/647/67520#8c55e9f6-8124-45e7-bb5c-a055950b78c8), то есть не настроите App Group (интерфейс передает null), совместное использование экрана все еще будет работать, но стабильность снизится, поэтому, хотя шагов больше, постарайтесь настроить правильный App Group, чтобы обеспечить стабильность функции совместного использования экрана.

#### Шаг 1:

#### Создание App Group (дополнительно)

[Войдите](https://developer.apple.com/) с помощью своей учетной записи и выполните следующие действия, обратите внимание, что вам потребуется повторно загрузить соответствующий Provisioning Profile после завершения процесса.

1. Нажмите Certificates, IDs & Profiles. 2.
2. Нажмите знак плюса на правой стороне экрана. 3.
3. Выберите App Groups и нажмите Continue. 4.
4. В появившейся форме заполните Description и Identifier, где Identifier необходимо передать в соответствующий параметр AppGroup в интерфейсе. Нажмите Continue после завершения.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/111ade23c12311efa28d525400329841.jpeg)

5. На странице Identifier выберите App IDs из меню в верхнем левом углу и нажмите на ваш App ID (AppIDs для основного приложения и расширения необходимо настроить одинаково).
6. Выберите App Groups и нажмите Edit.
7. В появившейся форме выберите созданную ранее App Group, нажмите Continue, чтобы вернуться на страницу Edit, и нажмите Save для сохранения.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/10ee7472c12311ef934f52540055f650.jpeg)

8. Повторно загрузите Provisioning Profile и настройте его в XCode.

#### **Шаг 2: Создание Broadcast Upload Extension**

1. В меню Xcode нажмите File > New > Target.... Выберите Broadcast Upload Extension. 2.
2. Заполните соответствующую информацию в появившемся диалоговом окне, не устанавливая флажок Include UI Extension, нажмите Finish для завершения создания.
3. Перетащите [TXLiteAVSDK_ReplayKitExt.framework](https://trtc.io/sdkDownload?id=conference) из загруженного пакета SDK zip в проект и выберите только что созданный Target. 4.
4. Если вы решили выполнить [Шаг 1](https://www.tencentcloud.com/document/product/647/67520#8c55e9f6-8124-45e7-bb5c-a055950b78c8), вам необходимо нажать + Capability и дважды щелкнуть App Groups в новом Target, как показано ниже:

![AddCapability](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/10ee4ed5c12311efb86b525400f69702.png)

После завершения операции в списке файлов будет создан файл с именем Target.entitlements, как показано на следующем рисунке. Выберите файл и нажмите знак +, чтобы заполнить App Group из приведенных выше шагов.

![AddGroup](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/110279abc12311efbb7252540075b605.png)

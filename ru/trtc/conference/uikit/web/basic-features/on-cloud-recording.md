# Облачная запись

В этой статье описывается, как быстро использовать функцию облачной записи TUIRoomKit для решения различных потребностей разработчиков, таких как архивирование и просмотр важного контента при видеоконференциях, онлайн-образовании, живых интерактивных сессиях и других сценариях. Мы предоставляем два решения на выбор: [Автоматическая запись](https://www.tencentcloud.com/document/product/647/60094#.E6.96.B9.E6.A1.88.E4.B8.80.EF.BC.9A.E8.87.AA.E5.8A.A8.E5.BD.95.E5.88.B6.E6.96.B9.E6.A1.88.EF.BC.88.E6.8E.A8.E8.8D.90.EF.BC.89) и [Запись на основе RESTful API](https://www.tencentcloud.com/document/product/647/60094#.E6.96.B9.E6.A1.88.E4.B8.80.EF.BC.9A.E8.87.AA.E5.8A.A8.E5.BD.95.E5.88.B6.E6.96.B9.E6.A1.88.EF.BC.88.E6.8E.A8.E8.8D.90.EF.BC.89#.E6.96.B9.E6.A1.88.E4.BA.8C.EF.BC.9Arest-api-.E5.BD.95.E5.88.B6.E6.96.B9.E6.A1.88).

## Способ 1. Автоматическая запись (рекомендуется)

Мы рекомендуем использовать решение автоматической записи, которое не требует от бизнеса запуска и остановки записи. Серверная часть Tencent Cloud Real-Time Audio and Video управляет задачами записи и автоматически записывает при загрузке аудио- и видеопотоков во время вызова. Процесс интеграции быстрый и простой. Вы можете выполнить его, следуя этим шагам:

1. Найдите приложение целевого `SDKAppId` в разделе [**Управление приложениями**](https://console.tencentcloud.com/trtc/app) в консоли TRTC и перейдите на страницу **Дополнительные функции**.
2. На странице конфигурации **Дополнительные функции** вы можете увидеть опцию конфигурации **Облачная запись**. Нажмите на **Глобальная автозапись**, чтобы открыть окно конфигурации.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/c2568e33fbda11eeac26525400ae4d13.png)

3. В окне конфигурации вы можете настроить шаблон записи по мере необходимости.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/c28eb414fbda11eeb7f5525400a7e516.png)

> **Примечание:** Глобальная автоматическая запись поддерживает только [одноканальную запись (то есть каждый ведущий записывает в отдельный файл)](https://www.tencentcloud.com/document/product/647/45169#single-stream-recording). После включения она применяется только к вновь созданным комнатам и не действует на комнаты, созданные до включения функции автоматической записи. Если вам требуется запись экрана после смешивания нескольких потоков, используйте [запись на основе RESTful API](https://www.tencentcloud.com/document/product/647/60094#.E6.96.B9.E6.A1.88.E4.BA.8C.EF.BC.9Arest-api-.E5.BD.95.E5.88.B6.E6.96.B9.E6.A1.88). Глобальная автоматическая запись поддерживает запись до 25 ведущих в одной комнате. Если ведущих более 25, они сортируются по времени входа в комнату, и записываются первые 25 ведущих (если вам нужно записать более 25 ведущих в один поток, см. [запись на основе RESTful API](https://www.tencentcloud.com/document/product/647/60094#.E6.96.B9.E6.A1.88.E4.BA.8C.EF.BC.9Arest-api-.E5.BD.95.E5.88.B6.E6.96.B9.E6.A1.88)).

После включения функции глобальной автоматической записи задача автоматической записи будет запущена после начала конференции и загрузки аудио и видео. По окончании конференции запись автоматически остановится. Если вы аварийно покидаете комнату из-за сетевых ошибок или других ситуаций, серверная часть записи автоматически остановит задачу записи согласно установленному вами значению MaxIdleTime (время ожидания без активности, по умолчанию 5 сек), чтобы избежать дополнительных расходов на платежи.

## Способ 2. Запись на основе RESTful API

Если решение автоматической записи не соответствует вашим потребностям, вы также можете использовать более гибкое решение записи на основе RESTful API. В этом решении вы можете записывать и подписываться на определённого якоря в комнате, настраивать макет смешанных потоков и обновлять макет и подписку во время записи. Однако **использование его функций требует использования вместе с сервисом бизнес-бэкенда и выполнения сложных операций интеграции**:

1. Найдите приложение целевого `SDKAppId` в разделе [**Управление приложениями**](https://console.tencentcloud.com/trtc/app) в консоли TRTC и перейдите на страницу **Дополнительные функции**.
2. На странице конфигурации дополнительных функций вы можете увидеть опции конфигурации облачной записи. Включите функцию облачной записи. Здесь рекомендуется отключить глобальную автоматическую запись, которая по умолчанию использует режим ручной пользовательской записи, то есть режим записи на основе RESTful API.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/c2270289fbda11ee894852540046dfed.png)

3. После этого вы можете вызвать REST API ([CreateCloudRecording](https://trtc.io/document/46960)), чтобы запустить облачную запись. Здесь рекомендуется слушать события уведомлений [TUIRoomObserver](https://www.tencentcloud.com/document/product/647/54863#) и запускать запись при начале совещания.

> **Примечание:** **В ручном режиме записи вы можете перейти в консоль и настроить адрес обратного вызова для получения событий обратного вызова записи**, см. [Описание обратного вызова записи](https://www.tencentcloud.com/document/product/647/54914#).

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/c234603bfbda11eeac26525400ae4d13.png)

## Часто задаваемые вопросы

### 1. Можно ли использовать оба решения записи одновременно?

[Решение автоматической записи](https://www.tencentcloud.com/document/product/647/60094#.E6.96.B9.E6.A1.88.E4.B8.80.EF.BC.9A.E8.87.AA.E5.8A.A8.E5.BD.95.E5.88.B6.E6.96.B9.E6.A1.88.EF.BC.88.E6.8E.A8.E8.8D.90.EF.BC.89) и [решение записи REST API](https://www.tencentcloud.com/document/product/647/60094#.E6.96.B9.E6.A1.88.E4.B8.80.EF.BC.9A.E8.87.AA.E5.8A.A8.E5.BD.95.E5.88.B6.E6.96.B9.E6.A1.88.EF.BC.88.E6.8E.A8.E8.8D.90.EF.BC.89#.E6.96.B9.E6.A1.88.E4.BA.8C.EF.BC.9Arest-api-.E5.BD.95.E5.88.B6.E6.96.B9.E6.A1.88) не конфликтуют и могут использоваться одновременно. Однако будут созданы два файла записи и будут рассчитаны [расходы](https://www.tencentcloud.com/document/product/647/45176#).

### 2. Как просмотреть записанные файлы?

Войдите в [консоль VOD](https://console.tencentcloud.com/vod), выберите **Управление видео/аудио** в левой боковой панели, нажмите **Поиск по префиксу** над списком, выберите **Поиск по префиксу** и введите ключевое слово в поле поиска. Имена файлов записей имеют следующие форматы:

- Формат имени файла MP4 одноканальной записи: `<SdkAppId>_<RoomId>_UserId_s_<UserId>_UserId_e_<MediaId>_<Index>.mp4`
- Формат имени файла MP4 смешанной записи: `<SdkAppId>_<RoomId>_<Index>.mp4`

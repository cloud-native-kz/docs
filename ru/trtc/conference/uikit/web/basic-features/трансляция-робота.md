# Трансляция робота

## Описание функции

TUIRoomKit поддерживает использование трансляции с помощью робота, позволяя пользователям передавать локальные или онлайн-файлы мультимедиа в комнату. В некоторых сценариях встречи могут требовать трансляцию предварительно записанных обучающих видео или онлайн-видеопотоков в онлайн-классы для просмотра и обучения пользователями. TUIRoomKit может удовлетворить эти требования, обеспечивая автоматизированное и эффективное воспроизведение видео.

Для использования этой функции необходимо подписаться на [пакет TUIRoomKit Monthly Package](https://www.tencentcloud.com/document/product/647/59973#.E8.B4.AD.E4.B9.B0.E6.AD.A3.E5.BC.8F.E7.89.88). Поток робота будет рассматриваться как пользователь в комнате, что повлечет расходы на время вызова. Дополнительную информацию см. в разделе [Выставление счетов за продолжительность аудио и видео](https://www.tencentcloud.com/document/product/647/42734#). Операции перекодирования будут выполняться в процессе подключения потоков робота, что повлечет расходы на перекодирование. Дополнительную информацию см. в разделе [Выставление счетов за микширование и перекодирование в облаке](https://trtc.io/document/45176?product=pricing).

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/ec99b45e174b11efb8ef5254002fd0a8.png)

### Сценарии применения

- Совместный просмотр, совместное прослушивание, совместное проведение и совместное обучение — опыт, который ранее требовал автономного, личного взаимодействия, все чаще переводится в онлайн. Возможность смотреть фильмы, слушать музыку, а затем обсуждать и делиться мнениями с друзьями на расстоянии тысяч миль — это волшебный опыт взаимодействия в реальном времени, который нравится молодым людям сегодня и становится основным направлением развития современных аудио- и видеопродуктов.
- Функция входящего медиапотока TUIRoomKit позволяет обмениваться внешними медиапотоками в комнате. Пользователи могут использовать эту функцию для обмена музыкой, фильмами, лекциями, курсами и другим мультимедийным контентом с другими пользователями в комнате, осуществляя взаимодействие с ними в реальном времени. Платформы могут использовать эту новую функцию TUIRoomKit для быстрого внедрения сценария **совместного просмотра**. Помимо совместного просмотра фильмов и прослушивания музыки, возможность входящего медиапотока TUIRoomKit может открыть больше инновационных возможностей для платформы в сценариях интерактивных классов, спортивных соревнований и веб-конференций.

# Использование: трансляция через RTMP

TUIRoomKit поддерживает стандартный протокол RTMP для трансляции. В зависимости от ситуации вы можете выбрать установку [OBS](https://obsproject.com/download), FFmpeg или других библиотек RTMP для трансляции.

## Использование FFmpeg для трансляции потока

FFmpeg требует определенной конфигурации параметров команды для различных сценариев, поэтому вы должны иметь некоторый опыт работы с FFmpeg. Ниже приведены часто используемые параметры командной строки FFmpeg; дополнительную информацию о параметрах FFmpeg см. на [официальном сайте FFmpeg](https://ffmpeg.org/ffmpeg.html).

#### Командная строка FFmpeg

```
ffmpeg [global_options] {[input_file_options] -i input_url} ... {[output_file_options] output_url}
```

#### Часто используемые параметры FFmpeg

| Параметр | Описание |
| --- | --- |
| -re | Читает входные данные с исходной частотой кадров, обычно используется только для чтения локальных файлов |

Среди них, настраиваемые параметры для **output_file_options** включают:

| Параметр | Описание |
| --- | --- |
| -c:v | Кодирование видео, рекомендуется использовать libx264 |
| -b:v | Битрейт видео, например, 1500k означает 1500 кбит/с |
| -r | Частота кадров видео |
| -profile:v | Профиль видео, указание baseline не будет кодировать B-кадры, бэкенд TUIRoomKit не поддерживает B-кадры |
| -g | Интервал GOP-кадра |
| -c:a | Кодирование аудио, рекомендуется использовать libfdk_aac |
| -ac | Количество каналов, введите 2 или 1 |
| -b:a | Битрейт аудио |
| -f | Укажите формат, зафиксируйте заполнение на `flv`, отправляйте в TUIRoomKit, используя контейнер FLV |

#### Использование

В следующем примере команда FFmpeg используется для чтения файлов и трансляции в TUIRoomKit, обратите внимание на кавычки вокруг URL.

```
ffmpeg -loglevel debug -re -i sample.flv -c:v libx264 -preset ultrafast -profile:v baseline -g 30 -sc_threshold 0 -b:v 1500k -c:a libfdk_aac -ac 2 -b:a 128k -f flv 'rtmp://rtmp.rtc.qq.com/push/yourRoomId?userid=yourUserId&sdkappid=xxxxxxxxx&usersig=xxxxxxxxxx'
```

Объяснения параметров в приведенной выше команде следующие:

| Параметр | Значение |
| --- | --- |
| -i sample.flv | Файл мультимедиа, который необходимо транслировать в TUIRoomKit. Вы можете заменить sample.flv на локальный или онлайн-файл мультимедиа, который вам нужно транслировать. |
| yourRoomId | ID комнаты, в которую вам нужно транслировать. Вам нужно заменить yourRoomId на ваш фактический RoomId. |
| userId | UserID, с которым вам нужно транслировать. Вам нужно изменить значение после "=" на фактический userId. |
| sdkappid | Ваш sdkappid, который вы ранее получили при [активации службы](https://www.tencentcloud.com/document/product/647/59973#). Вам нужно изменить значение после "=" на фактический sdkAppID. |
| usersig | Ваш usersig, который вы получили при входе в TUIRoomKit. Вам нужно изменить значение после "=" на фактический userSig. |

Когда вам нужно реализовать приведенную выше функцию в коде, вы можете использовать соответствующую библиотеку FFmpeg на вашей платформе или вызвать FFmpeg через командную строку для достижения приведенной выше команды.

## Использование OBS для трансляции потока

### Предварительные условия

Вы установили [OBS](https://obsproject.com/download?spm=a2c4g.11186623.2.15.6aac1445JPlKR8).

### Шаг 1. Выбор источника

В панели **Sources** внизу нажмите **+** и выберите источник в зависимости от ваших потребностей. Общие источники включают следующее:

| Источник | Примечание |
| --- | --- |
| Image | Транслирует одно изображение |
| Image Slide Show | Транслирует несколько изображений (вы можете определить порядок воспроизведения и повторение воспроизведения) |
| Scene | Вставляет целую сцену для достижения различных эффектов трансляции |
| Media Source | Загружает локальный файл и транслирует его как прямую трансляцию |
| Text | Добавляет текст в реальном времени в вашу трансляцию |
| Window Capture | Захватывает и транслирует выбранное окно в реальном времени |
| Video Capture Device | Захватывает и транслирует изображения, захватываемые камерой в реальном времени |
| Audio Input Capture | Аудиотрансляция (устройство аудиовхода) |
| Audio Output Capture | Аудиотрансляция (устройство аудиовыхода) |

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b54d9379346111ef9c9c525400d5f8ef.png)

### Шаг 2. Настройка параметров трансляции

1. В панели **Controls** внизу нажмите **Settings**.
![img](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b5307f94346111ef9c9c525400d5f8ef.png)
2. Нажмите **Stream** и выберите **Custom** для **Service**.
3. Введите `rtmp://intl-rtmp.rtc.qq.com/push/` для **Server**.
4. Введите ключ трансляции в следующем формате:

```
Room ID?sdkappid=Application&userid=User ID&usersig=Signature
```

Замените "Room ID", "Application ID", "User ID" и "Signature" на фактические значения, например:

```
yourRoomId?sdkappid=140*****66&userid=******rtmp2&usersig=eJw1jdE***************ZLgi5UAgOzoMhrayt*cjbmiCJ699T09juc833IMT94Ld7I0iHZqVDzvVAqkZsG-IKlzLiXOnEhswHu1iUyTc9pv*****D8MQwoA496Ke6U1ip4EAH4UMc5H9pSmv6MeTBWLamhwFnWRBZ8qKGRj8Yp-wVbv*mGMVZqS7w-mMDQL
```

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b61deb27346111ef9397525400fdb830.png)

### Шаг 3. Настройка вывода

Поскольку RTMP не поддерживает B-кадры, установите параметры кодирования видео следующим образом, чтобы удалить B-кадры.

1. Перейдите к **Controls** > **Settings** > **Output**.
2. Выберите **Advanced** для **Output Mode**. Рекомендуемый **Keyframe Interval** составляет **1** или **2**. Для **CPU Usage Preset** выберите **ultrafast**. Для **Profile** выберите **baseline**. Для **Tune** выберите **zerolatency**. И для **x264 Options** введите `threads=1`, затем нажмите **OK**.

> **Предупреждение:** Вам нужно удалить B-кадры из потоков RTMP, иначе соединение будет разорвано после трансляции. Чтобы достичь этого, выберите baseline для Profile.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b5f30c14346111ef9b60525400bdab9d.png)

### Шаг 4. Установка параметров видео

Вы можете установить разрешение видео и частоту кадров в разделе

**Video**

в

**Settings**

. Разрешение определяет четкость видео, показываемого зрителям. Чем выше разрешение, тем четче видео. Частота кадров (кадры в секунду) определяет плавность воспроизведения. Типичная частота кадров находится в диапазоне от 24 кадр/с до 30 кадр/с. Воспроизведение может заикаться, если частота кадров ниже 16 кадр/с. Видеоигры требуют более высокой частоты кадров и обычно заикаются при частоте кадров ниже 30 кадр/с.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b53c3d42346111ef9b60525400bdab9d.png)

### Шаг 5. Настройка расширенных параметров

- Чтобы уменьшить сквозную задержку, мы рекомендуем вам не включать **Stream Delay**.
- Держите **Automatically Reconnect** включенным и сделайте **Retry Delay** как можно короче, чтобы издатель мог быстро переподключиться после отключения, вызванного сетевыми помехами.
![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b6085c58346111efb958525400f69702.png)

### Шаг 6. Трансляция потока

1. В панели **Controls** внизу нажмите **Start Streaming**.
![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b5a1010c346111ef9397525400fdb830.png)
2. Если трансляция прошла успешно, нижняя панель будет показывать статистику трансляции, и вход пользователя будет записан в [панель мониторинга](https://www.tencentcloud.com/document/product/647/39070).
![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b68f8d58346111ef9c9c525400d5f8ef.png)

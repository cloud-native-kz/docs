# Часто задаваемые вопросы

## Проблемы, связанные с окружением

### Какие платформы поддерживает TUIRoomKit Web?

Пожалуйста, обратитесь к [поддерживаемым браузерам TUI Веб-SDK](/document/product/647/41664) для платформ, поддерживаемых TUIRoomEngine Web.

Для сред, не указанных выше, вы можете открыть [Тест возможностей TRTC](https://web.sdk.qcloud.com/trtc/webrtc/demo/detect/index.html) в вашем текущем браузере, чтобы проверить, полностью ли он поддерживает функции WebRTC.

### Почему TUIRoomKit работает нормально при локальном тестировании, но не работает при развертывании в онлайн?

Исходя из соображений безопасности и приватности пользователей, браузеры ограничивают веб-страницы возможностью захватывать микрофон и камеру только в защищенных окружениях (таких как https, localhost, протоколы file://). Протокол HTTP небезопасен, и браузеры запрещают захват устройств мультимедиа по протоколу HTTP.

Если все работает хорошо при локальном тестировании, но вы не можете захватывать камеру и микрофон после развертывания страницы, пожалуйста, проверьте, развернута ли ваша веб-страница по протоколу HTTP. Если да, пожалуйста, используйте HTTPS для развертывания веб-страницы и убедитесь, что у вас есть действительный сертификат безопасности HTTPS.

Для получения дополнительной информации обратитесь к [описанию ограничений по URL-домену и протоколу](/document/product/647/41664).

### Поддерживает ли TUIRoomKit Web интеграцию с iframe?

Да, поддерживает. Для интеграции TUIRoom Web в iframe необходимо добавить атрибуты к тегу iframe для включения соответствующих разрешений, как показано ниже.

```
// Включить разрешения для микрофона, камеры, общего доступа к экрану и полноэкранного режима<iframe allow="microphone; camera; display-capture; display; fullscreen;">
```

## Проблемы, связанные с компиляцией

### Ошибка импорта Webpack 5 при импорте TUIRoomEngine SDK: webpack < 5 по умолчанию включал полифилы для модулей основной библиотеки node.js. Это больше не так. Проверьте, нужен ли вам этот модуль, и настройте для него полифил.

Ошибка вызвана удалением автоматического включения полифилов модулей основной библиотеки nodejs в webpack5. Вы можете добавить конфигурацию configureWebpack в vue.config.js для решения этой проблемы.

```
module.exports = defineConfig({  // ...  configureWebpack: (config) => {    config.resolve.fallback = {      ...config.resolve.fallback,      url: false,      path: false,      fs: false,      crypto: false,    };  }});
```

## Проблемы, связанные с функциональностью

#### Можно ли импортировать tim-js-sdk и @tencentcloud/tuiroom-engine-js в проект одновременно?

Да, можно. Вы можете создать объект экземпляра tim через TIM в tim-js-sdk и передать его в интерфейс init при инициализации TUIRoomEngine.

```
await TUIRoomEngine.init({ sdkAppId: 0,   // Заполните ваш применяемый sdkAppId userId: '',    // Заполните userId, связанный с вашим бизнесом userSig: '',   // Заполните userSig, вычисленный сервером или локально tim,    // Передайте экземпляр tim});
```

Также вы можете получить экземпляр tim, используемый внутри TUIRoomEngine, через метод roomEngine.getTIM.

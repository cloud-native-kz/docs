# Управление профилем

Live и Instant Messaging (Chat) используют одну и ту же систему учётных записей, а данные представляют собой информацию, привязанную к пользователю в системе учётных записей.

## Обзор системы профилей

Instant Messaging (Chat) предоставляет полный набор решений для работы с профилями через свои возможности хостинга профилей пользователей. Сервис хостинга профилей Chat позволяет вашим пользователям легко устанавливать и получать профили.

- Chat может хранить информацию профиля, обеспечивая удалённое восстановление при сбое, развёртывание в нескольких регионах и автоматическое масштабирование ваших данных. Таким образом, вы полностью освобождаетесь от сложных процессов обработки, таких как отказ в обслуживании сервера, репликация с несколькими копиями первичного-вторичного уровня и масштабирование ёмкости.
- Chat предоставляет деловые процессы, часто используемые в отрасли, поэтому вам не нужно беспокоиться о деловой логике профиля пользователя.
- Chat предоставляет профессиональные процессы и команды операций, обеспечивая качество обслуживания 99,99% в год и помогая вам предоставлять услуги, известные своей стабильностью.
- Chat предоставляет удобные в использовании API-интерфейсы обслуживания и простые в доступе рекомендации с полной поддержкой на протяжении всего процесса.

Используя сервис хостинга профилей Chat, вы сможете:

- Хранить, читать и записывать стандартные поля профиля.
- Хранить, читать и записывать пользовательские поля профиля.

## Поля профиля

Профиль — это набор данных, описывающих свойства пользователя. Система профилей Chat поддерживает стандартные и пользовательские поля профиля. Поля профиля имеют следующие характеристики:

- Поля профиля выражены в формате ключ-значение.
- Ключ имеет строковый формат, и его название может содержать только заглавные и строчные буквы, цифры и подчёркивания.
- Значение имеет следующие типы:
- a. Целое число типа uint32_t (не поддерживается для пользовательских полей)
- b. Целое число типа uint64_t (не поддерживается для пользовательских полей)
- c. Строка типа string (длина строки не может превышать 500 байт)

## Пользовательские поля профиля

Пользовательские поля профиля — это данные пользователя, установленные каждым приложением в соответствии с его собственными деловыми потребностями. Используя пользовательские поля профиля, приложение может добавлять дополнительные данные к профилям пользователей и выполнять операции чтения и записи через существующие API.

### Подача заявки на пользовательские поля профиля

Чтобы подать заявку на пользовательские поля профиля, войдите в [консоль Chat](https://console.trtc.io/) и выберите **Application Configuration** > **Feature Configuration**. После отправки вашего приложения пользовательские поля профиля вступят в силу в течение пяти минут.

При подаче заявки на пользовательские поля профиля вам необходимо предоставить следующую информацию для каждого поля:

- Имя пользовательского поля профиля (Key). Для получения дополнительной информации см. [Правила именования пользовательских полей профиля](https://www.tencentcloud.com/document/product/647/69884#naming-rules-for-custom-profile-fields).
- Тип пользовательского поля профиля (Value). Для получения дополнительной информации см. [Поля профиля](https://www.tencentcloud.com/document/product/647/69884#profile-fields).

### Правила именования пользовательских полей профиля

Правила для именования пользовательских полей профиля выглядят следующим образом:

- Имя пользовательского поля профиля содержит части префикса и ключевого слова.
- Префикс пользовательского поля профиля: Tag_Profile_Custom.
- Ключевое слово: ключевое слово должно быть строкой букв с длиной не более 8 байт. Рекомендуем использовать английское слово или его аббревиатуру в качестве ключевого слова.
- Пример: если пользовательское поле, заявленное приложением, имеет ключевое слово **Test**, то имя пользовательского поля профиля: Tag_Profile_Custom_Test.

# Добавление связей категорий подарков

## Описание функции

Администратор приложения может использовать этот API для добавления связи между указанными категориями подарков и подарками.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_engine_http_srv/add_gift_category_relations?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице перечислены только параметры, задействованные при изменении, и их описания при вызове этого API. Дополнительные сведения о параметрах см. в разделе [Введение в REST API](https://www.tencentcloud.com/document/product/647/63398).

| Параметр | Описание |
| --- | --- |
| xxxxxx | SDKAppID расположен в стране/регионе с выделенным доменным именем: Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` США: `adminapiusa.im.qcloud.com` |
| v4/live_engine_http_srv/add_gift_category_relations | API запроса. |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения. |
| identifier | Должна быть учетная запись администратора приложения. Для получения дополнительной информации см. раздел [администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin). |
| usersig | Подпись, сгенерированная учетной записью администратора приложения. Подробнее см. в разделе [Генерирование UserSig](https://www.tencentcloud.com/document/product/647/69883). |
| random | Введите случайное 32-битное целое число без знака, диапазон значений от 0 до 4294967295. |
| contenttype | Формат запроса зафиксирован как `json`. |

### Максимальная частота вызовов

1 раз/сек.

### Пример пакета запроса

**Базовая форма**

```
{    "CategoryId":"id1",    "GiftIdList" :["id1","id2","id3"]}
```

### Поля пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| CategoryId | String | Обязательно | ID категории подарков, которую необходимо добавить. |
| GiftIdList | Array | Обязательно | ID подарка, который необходимо добавить в категорию. |

> **Примечание:** Эта операция добавляет новую связь поверх существующей, не перезаписывая исходную связь категории подарков.

### Пример пакета ответа

**Базовая форма**

```
{    "ErrorCode": 0,    "ErrorInfo": "",    "ActionStatus": "OK",    "RequestId": "Id-821589b588fb4df79e6a5fe8f0cb4519-O-Seq-289843"}
```

### Поля пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса. OK: обработка успешна; FAIL: обработка не удалась. |
| ErrorCode | Integer | Код ошибки: 0 указывает на успех, ненулевое значение указывает на ошибку. |
| ErrorInfo | String | Сообщение об ошибке. |
| RequestId | String | Уникальный ID запроса, возвращаемый для каждого запроса. Для устранения неполадок запроса его RequestId должен быть предоставлен. |
| InvalidGiftIdList | Array | Список неинформированных GiftId. Удалите эти GiftId из списка GiftIdList запроса. |

## Описание кода ошибки

Если не возникает сетевая ошибка (например, 502), код возврата HTTP для этого API составляет 200. Фактический код ошибки и информация об ошибке указаны в параметрах ErrorCode и ErrorInfo в полезной нагрузке ответа.

Часто встречающиеся коды ошибок (60000–79999) см. в документе [Код ошибки](https://www.tencentcloud.com/document/product/647/60027).

Приватные коды ошибок этого API приведены ниже:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, попробуйте еще раз. |
| 100002 | Недопустимые параметры запроса. Проверьте правильность запроса согласно описанию ошибки. |
| 100006 | Недостаточно прав для операции. Необходимо использовать операции учетной записи администратора. |
| 100012 | Частота вызовов слишком высока, максимум 1 вызов/сек. |
| 102001 | Отсутствует информация о платеже. Приобретите пакет в консоли. |

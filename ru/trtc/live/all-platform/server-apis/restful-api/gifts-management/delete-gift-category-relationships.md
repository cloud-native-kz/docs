# Удаление связей подарков и категорий

## Описание функции

Администратор приложения может использовать этот API для удаления указанных связей между подарками и категориями.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_engine_http_srv/del_gift_category_relations?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице перечислены только параметры, участвующие в изменении при вызове этого API, и их описание. Для получения дополнительной информации о параметрах см. [Введение в REST API](https://www.tencentcloud.com/document/product/647/63398).

| Параметр | Описание |
| --- | --- |
| xxxxxx | SDKAppID находится в стране/регионе специального доменного имени: Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Соединённые Штаты: `adminapiusa.im.qcloud.com` |
| v4/live_engine_http_srv/del_gift_category_relations | API запроса. |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения. |
| identifier | Должна быть учётная запись администратора приложения. Для получения дополнительной информации см. [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin). |
| usersig | Подпись, сгенерированная учётной записью администратора приложения. Для получения конкретных инструкций см. [Генерация UserSig](https://www.tencentcloud.com/document/product/647/69883). |
| random | Введите случайное 32-битное целое число без знака, значение варьируется от 0 до 4294967295. |
| contenttype | Формат запроса фиксирован как `json`. |

### Максимальная частота вызовов

1 раз в секунду.

### Пример пакета запроса

**Базовая форма**

```
{    "CategoryId":"id1",    "GiftIdList" :["id1","id2","id3"]}
```

### Поля пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| CategoryId | String | Обязательное | ID категории подарков, которую необходимо удалить. |
| GiftIdList | Array | Обязательное | ID подарка, который необходимо удалить под категорией. |

### Пример пакета ответа

**Базовая форма**

```
{    "ErrorCode": 0,    "ErrorInfo": "",    "ActionStatus": "OK",    "RequestId": "Id-821589b588fb4df79e6a5fe8f0cb4519-O-Seq-289843"}
```

### Поля пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса. OK: обработка успешна; FAIL: обработка не удалась. |
| ErrorCode | Integer | Код ошибки: 0 указывает на успех, ненулевое значение указывает на ошибку. |
| ErrorInfo | String | Сообщение об ошибке. |
| RequestId | String | Уникальный ID запроса, возвращаемый для каждого запроса. Для устранения неполадок запроса следует предоставить его RequestId. |

## Описание кодов ошибок

Если не возникает сетевая ошибка (например, 502), код возврата HTTP этого API равен 200. Фактический код ошибки и информация об ошибке указаны в полях ErrorCode и ErrorInfo в теле пакета ответа.

Общие коды ошибок (60000–79999) см. в документе [Код ошибки](https://www.tencentcloud.com/document/product/647/60027).

Коды ошибок этого API указаны ниже:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, пожалуйста, повторите попытку. |
| 100002 | Недопустимые параметры запроса. Проверьте, правильный ли запрос в соответствии с описанием ошибки. |
| 100006 | Недостаточно прав для выполнения операции. Требуются операции учётной записи администратора. |
| 100012 | Частые вызовы. Максимальная частота вызовов 1/с. |
| 102001 | Отсутствует информация об оплате. Приобретите пакет в консоли. |

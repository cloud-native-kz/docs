# Отправить пользовательское сообщение

## Описание функции

Администратор приложения может использовать этот API для отправки пользовательских сообщений через этот интерфейс.

## Описание API-вызова

### Пример URL запроса

```
https://xxxxxx/v4/live_engine_http_srv/send_custom_msg?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице указаны только параметры, задействованные при изменении при вызове этого API, и их описания. Для получения дополнительной информации о параметрах см. [Введение в REST API](https://www.tencentcloud.com/document/product/647/63398).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Выделенное доменное имя SDKAppID, где оно находится:Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Соединенные Штаты: `adminapiusa.im.qcloud.com` |
| v4/live_engine_http_srv/send_custom_msg | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения. Его можно получить из карточки приложения в [консоли](https://console.tencentcloud.com/im) Chat. |
| identifier | Должна быть учетная запись администратора приложения. Дополнительные сведения см. в документации [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin). |
| usersig | Подпись, созданная учетной записью администратора приложения. Для получения конкретных инструкций см. [Создание UserSig](https://www.tencentcloud.com/document/product/647/69883). |
| random | Введите случайное 32-битное целое число без знака, диапазон значений от 0 до 4294967295. |
| contenttype | Формат запроса, значение зафиксировано как `json`. |

### Максимальная частота вызовов

200 раз/секунду

### Пример пакета запроса

**Основная форма**

```
{    "RoomId":"room1",    "Sender_Account":"user1",    "BusinessId":"xxxxx",        "Data":"zzzzzzz"}
```

### Поля пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| RoomId | String | Обязательно | ID комнаты. |
| Sender_Account | String | Опционально | Отправитель сообщения. |
| BusinessId | String | Обязательно | Маркер сообщения. |
| Data | Integer | Обязательно | Содержание сообщения. |

### Описание формата сообщения

Этот эндпоинт является оболочкой для стандартных групповых сообщений Chat. `MsgBody` в теле запроса следует формату [пользовательского сообщения](https://www.tencentcloud.com/document/product/647/74355#TIMCustomElem). Структура обернутого запроса выглядит следующим образом:

```
{     "MsgBody" : [      {         "MsgContent" : {            "Data" : "{\\"BusinessId\\":\\"key\\",\\"Data\\":\\"value\\"}"         },         "MsgType" : "TIMCustomElem"      }}
```

- Data: строка JSON, составленная из полей `BusinessId` и `Data` исходного запроса сообщения.

## Интеграция обратных вызовов

Информация этого запроса используется в обратных вызовах сообщений. Для получения подробного описания см. документацию [До отправки группового сообщения](https://www.tencentcloud.com/document/product/647/74347) и [После отправки группового сообщения](https://www.tencentcloud.com/document/product/647/74348).

### Пример пакета ответа

```
{    "ErrorCode": 0,    "ErrorInfo": "",    "ActionStatus": "OK",    "RequestId": "Id-b6454678e8a647ffa476ec1fee1263d1-O-Seq-2433991",    "Response": {        "MsgTime": 1750074537,        "MsgSeq": 26,        "MsgDropReason" : "MsgFreqCtrl"    }}
```

### Поля пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса.OK: обработка успешна.FAIL: обработка не удалась. |
| ErrorCode | Integer | Код ошибки.0: успех.Ненулевое значение: указывает на ошибку. |
| ErrorInfo | String | Сообщение об ошибке. |
| RequestId | String | Уникальный ID запроса, возвращается для каждого запроса. Для определения проблемы следует предоставить RequestId запроса. |
| MsgTime | Integer | Временная метка отправки сообщения, соответствующая времени серверной части. |
| MsgSeq | Integer | Порядковый номер сообщения, уникальный идентификатор сообщения. |
| MsgDropReason | String | Причина отклонения сообщения. По умолчанию пусто. В настоящее время отклоняются только сообщения, превышающие предел частоты. |

## Описание кодов ошибок

Если не возникает ошибка сети (например, 502), код возврата HTTP этого API равен 200. Фактические код ошибки и информацию об ошибке обозначают ErrorCode и ErrorInfo в полезной нагрузке ответа.

Распространенные коды ошибок (от 60000 до 79999) см. в документе [Код ошибки](https://www.tencentcloud.com/document/product/647/60027).

Приватные коды ошибок этого API выглядят следующим образом:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, попробуйте снова. |
| 100002 | Неверный параметр. Проверьте правильность запроса согласно описанию ошибки. |
| 100004 | Комната не существует, возможно, она никогда не была создана или уже распущена. |
| 100006 | Указывает, что эта комната не является прямой трансляцией и не имеет разрешения на использование интерфейсов запроса прямой трансляции. |
| 100601 | Сообщение не может быть отправлено из-за отключения звука. Проверьте, отключен ли звук отправителя. |
| 100602 | Достигнут предел частоты отправки сообщений. Попробуйте позже. |
| 100603 | Содержание сообщения слишком длинное. В настоящее время максимальная поддерживаемая длина сообщения составляет 12 КБ. Отрегулируйте длину сообщения. |

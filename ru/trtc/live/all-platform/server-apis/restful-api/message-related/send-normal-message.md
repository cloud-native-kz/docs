# Отправить обычное сообщение

## Описание функции

Администратор приложения может использовать этот API для отправки обычных сообщений через данный интерфейс.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_engine_http_srv/send_text_msg?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице перечислены только параметры, задействованные при изменении при вызове этого API, и их описание. Для получения дополнительной информации о параметрах см. [введение REST API](https://www.tencentcloud.com/document/product/647/63398).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Выделенное доменное имя SDKAppID, в котором оно находится:Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Соединенные Штаты: `adminapiusa.im.qcloud.com` |
| v4/live_engine_http_srv/send_text_msg | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения, можно получить из карточки приложения в [Консоли](https://console.tencentcloud.com/im) Chat. |
| identifier | Должна быть учетная запись администратора приложения. Для получения дополнительной информации см. документацию [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin). |
| usersig | Подпись, генерируемая учетной записью администратора приложения. Для конкретных операций см. [Генерирование UserSig](https://www.tencentcloud.com/document/product/647/69883). |
| random | Введите случайное 32-битное целое число без знака, диапазон значений от 0 до 4294967295. |
| contenttype | Формат запроса, значение фиксировано как `json`. |

### Максимальная частота вызовов

200 раз/сек

### Пример пакета запроса

**Базовая форма**

```
{    "RoomId":"room1",    "Sender_Account":"user1",    "TextContent":"msg",                               // message context    "ExtensionInfo" :"{\\"data\\":1,\\"value\\":2}"        // Optional field; it must be in JSON format}
```

### Поля пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| RoomId | String | Обязательный | ID комнаты. |
| Sender_Account | String | Обязательный | Отправитель сообщения. |
| TextContent | String | Обязательный | Текстовое содержание сообщения. |
| ExtensionInfo | String | Опциональный | Пользовательские данные сообщения. Они сохраняются в облаке и отправляются получателю. Такие данные можно получить после удаления и переустановки приложения. |

### Описание формата сообщения

Этот интерфейс происходит от метода отправки обычного сообщения в группе в Chat. Это сообщение затем обернуто запросом `MsgBody` соответствует формату [текстового сообщения](https://www.tencentcloud.com/document/product/647/74355#.E6.B6.88.E6.81.AF.E5.85.83.E7.B4.A0-timmsgelement), и оно также содержит [пользовательские данные сообщения CloudCustomizaData](https://www.tencentcloud.com/document/product/647/74355#description-of-custom-message-data-cloudcustomdata).Упакованная информация запроса выглядит следующим образом:

Этот интерфейс представляет собой оболочку для стандартных групповых сообщений IM Chat. Тело запроса `MsgBody` следует формату [текстового сообщения](https://www.tencentcloud.com/document/product/647/74355#.E6.B6.88.E6.81.AF.E5.85.83.E7.B4.A0-timmsgelement) и включает поле [CloudCustomData](https://www.tencentcloud.com/document/product/647/74355#description-of-custom-message-data-cloudcustomdata) для пользовательских данных сообщения. Структура упакованного запроса выглядит следующим образом:

```
{     "CloudCustomData" : "{\\"data\\":1,\\"value\\":2}",   "MsgBody" : [      {         "MsgContent" : {            "Text" : "msg"         },         "MsgType" : "TIMTextElem"      }   ],}
```

- Сопоставление полей
  - `CloudCustomData`: Соответствует полю `ExtensionInfo` в исходном запросе.
  - `Text`: Соответствует текстовому содержанию отправляемого сообщения.
- Жизненный цикл и обратные вызовы

Эта информация запроса используется в обратных вызовах сообщений. Для подробного описания см. документацию [Обратный вызов перед доставкой](https://www.tencentcloud.com/document/product/647/74347) и [Обратный вызов после доставки](https://www.tencentcloud.com/document/product/647/74348).

### Пример пакета ответа

```
{    "ErrorCode": 0,    "ErrorInfo": "",    "ActionStatus": "OK",    "RequestId": "Id-ec3xxxxxxxxxxxxxxxxxxxxxxxxed-O-Seq-2433650",    "Response": {        "MsgTime": 1750074420,        "MsgSeq": 25,        "MsgDropReason" : "MsgFreqCtrl"    }}
```

### Поля пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса.OK: обработка успешна.FAIL: обработка не удалась. |
| ErrorCode | Integer | Код ошибки.0: успех.Ненулевое значение: указывает на ошибку. |
| ErrorInfo | String | Сообщение об ошибке. |
| RequestId | String | Уникальный идентификатор запроса, возвращается для каждого запроса. Для локализации проблемы следует предоставить RequestId запроса. |
| MsgTime | Integer | Временная метка отправки сообщения, соответствует времени серверной части. |
| MsgSeq | Integer | Порядковый номер сообщения, уникальный идентификатор сообщения. |
| MsgDropReason | String | Причина отбрасывания сообщения. По умолчанию пусто. В настоящее время отбрасываются только сообщения, превышающие ограничение по частоте. |

## Описание кодов ошибок

Если не возникает сетевая ошибка (например, 502), код возврата HTTP этого API составляет 200. Фактический код ошибки и информация об ошибке указаны в ErrorCode и ErrorInfo в полезной нагрузке ответа.

Обычные коды ошибок (от 60000 до 79999), см. документ [Код ошибки](https://www.tencentcloud.com/document/product/647/60027).

Приватные коды ошибок этого API указаны ниже:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, повторите попытку. |
| 100002 | Недопустимый параметр. Проверьте правильность запроса в соответствии с описанием ошибки. |
| 100004 | Комната не существует, возможно, она никогда не была создана или уже была распущена |
| 100006 | Указывает, что эта комната не является трансляционной и не имеет разрешения использовать интерфейсы прямого запроса |
| 100601 | Сообщение не может быть отправлено из-за отключения звука. Проверьте, отключен ли отправитель. |
| 100602 | Достигнут предел частоты отправки сообщений. Повторите попытку позже. |
| 100603 | Содержимое сообщения слишком длинное. В настоящее время максимальная длина сообщения составляет 12 КБ. Пожалуйста, отрегулируйте длину сообщения. |

# Добавление робота в комнату

## Описание функции

Администраторы приложений могут добавить чат-бота в комнату с помощью этого API.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_engine_http_srv/add_robot?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице приведены только параметры, участвующие в изменении при вызове этого API, и их описания. Для получения более подробной информации о параметрах см. [Обзор RESTful API](https://www.tencentcloud.com/document/product/647/63398).

| Параметры | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, в котором находится ваш SDKAppID: Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Соединённые Штаты: `adminapiusa.im.qcloud.com` |
| v4/live_engine_http_srv/add_robot | API запроса |
| sdkappid | Назначенный SDKAppID, созданный для приложения Live |
| identifier | Должен быть аккаунт администратора приложения. Для получения более подробной информации см. [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin) |
| usersig | Сгенерированная подпись аккаунта администратора приложения. Для конкретных операций см. [Генерирование UserSig](https://www.tencentcloud.com/document/product/647/69883) |
| random | Введите случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса имеет фиксированное значение `json` |

### Максимальная частота вызовов

100 раз/секунду.

### Пакет запроса

**Базовая форма**

```
{    "RoomId":"mix1",    "RobotList_Account":["brennanli"]    }
```

### Поля пакета запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| RoomId | String | Обязательно | Идентификатор комнаты, до 48 байт |
| RobotList_Account | Array | Обязательно | Список робота, аккаунт Bot, т.е. UserID пользователя, должен быть [импортированный аккаунт](https://www.tencentcloud.com/document/product/647/69887) |

### Пример пакета ответа

**Базовая форма**

```
{  "ActionStatus": "OK",  "ErrorInfo": "",  "ErrorCode": 0,  "RequestId": "Id-8c9858f01e954611ae2d4c1b1ed7d583-O-Seq-52720"}
```

### Поля пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса. OK: обработка успешна; FAIL: сбой |
| ErrorCode | Integer | Код ошибки. 0: успех; не 0: сбой |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Уникальный идентификатор запроса. Возвращается для каждого запроса. RequestId требуется для выявления проблемы |

## Описание кодов ошибок

Если не возникает сетевая ошибка (например, ошибка 502), код возврата HTTP этого API всегда равен 200. Фактический код ошибки и информация об ошибке представлены ErrorCode и ErrorInfo в теле пакета ответа.

Для распространённых кодов ошибок (60000 и 79999) см. документ [Код ошибки](https://www.tencentcloud.com/document/product/647/60027).

Приватные коды ошибок этого API приведены ниже:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, пожалуйста, повторите попытку. |
| 100002 | Неверный параметр. Проверьте, правильный ли запрос в соответствии с описанием ошибки. |
| 100006 | Требуемый тип комнаты — это тип Live-комнаты. |
| 100004 | Комната не существует или распущена. |

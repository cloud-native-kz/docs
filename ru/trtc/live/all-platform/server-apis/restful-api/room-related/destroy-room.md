# Удаление комнаты

## Обзор функции

Администратор приложения может распустить комнату, используя этот интерфейс.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_engine_http_srv/destroy_room?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже указаны только параметры, изменяемые при вызове этого интерфейса. Дополнительные сведения о других параметрах см. в разделе [Обзор RESTful API](https://www.tencentcloud.com/document/product/647/63398).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Зарезервированный домен для страны/региона, где находится SDKAppID: прочие регионы Китая: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`США: `adminapiusa.im.qcloud.com` |
| v4/live_engine_http_srv/destroy_room | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Должен быть учетной записью администратора приложения. Дополнительные сведения см. в разделе [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin) |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробные инструкции см. в разделе [Генерация UserSig](https://www.tencentcloud.com/document/product/647/69883) |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса, фиксированное значение: `json` |

### Максимальная частота вызовов

200 запросов/сек.

### Пример пакета запроса

**Базовая форма**

```
{    "RoomId":"live-room"      }
```

### Поля пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| RoomId | String | Обязательно | ID комнаты |

### Пример пакета ответа

**Базовая форма**

```
{    "ErrorCode": 0,    "ErrorInfo": "",    "ActionStatus": "OK",    "RequestId": "Id-1b30123ad8a149b0aa225d8596999714-O-Seq-616334",    "Response": {        "Statistic": {            "TotalViewers": 0,            "TotalGiftsSent": 0,            "TotalGiftCoins": 0,            "TotalUniqueGiftSenders": 0,            "TotalLikesReceived": 0        }    }}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса. OK: успешно обработан; FAIL: ошибка обработки. |
| ErrorCode | Integer | Код ошибки: 0 означает успех, ненулевое значение означает ошибку. |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Уникальный ID запроса, возвращаемый для каждого запроса. Для локализации проблемы предоставьте RequestId этого запроса. |
| TotalViewers | Integer | Общее количество входов в комнату. Повторный вход одного и того же пользователя N раз считается как N. |
| TotalGiftsSent | Integer | Общее количество отправленных подарков |
| TotalGiftCoins | Integer | Общая стоимость подарков |
| TotalUniqueGiftSenders | Integer | Общее количество отправителей подарков |
| TotalLikesReceived | Integer | Общее количество отметок «Нравится» |

## Механизм автоматического удаления комнаты

Автоматическое удаление комнаты срабатывает при выполнении следующих 3 условий. Эти политики можно настроить на уровне sdkappid. Чтобы изменить параметры, обратитесь к своему менеджеру по учетным записям или [отправьте заявку](https://console.tencentcloud.com/workorder/category).

1. Комната будет автоматически удалена, если она остается пустой в течение непрерывного периода (по умолчанию: 30 минут) без новых участников после входа пользователей в комнату. Sdkappids, созданные до 20 мая 2025 г., сохраняют исходный порог автоудаления в 6 часов. Временной диапазон можно настраивать в пределах [10 минут, 24 часа]. Комнаты с ботами освобождаются от автоматического удаления.
2. Если владелец комнаты выходит из комнаты через API и не входит в комнату в течение настроенного временного диапазона (рекомендуется: 10 минут), комната запустит правило автоудаления, которое по умолчанию отключено. Временной диапазон можно настраивать в пределах [2 минуты, 12 часов]. Механизм автоматического удаления отключен, когда владелец комнаты идентифицирован как бот.
3. Если владелец комнаты отключится за пределы периода сохранения пульса и не сможет отправить пульс на сервер в течение настроенного временного диапазона (рекомендуется: 10 минут), комната запустит правило автоудаления, которое по умолчанию отключено. Временной диапазон можно настраивать в пределах [2 минуты, 12 часов]. Механизм автоматического удаления отключен, когда владелец комнаты идентифицирован как бот.

## Коды ошибок

Если не происходит ошибка сети (например, ошибка 502), код возврата HTTP для этого интерфейса всегда будет 200. Фактический код ошибки и информация об ошибке указаны в полезной нагрузке ответа в ErrorCode и ErrorInfo. Общие коды ошибок (60000–79999) можно найти в документе [Код ошибки](https://www.tencentcloud.com/document/product/647/60027). Приватные коды ошибок для этого API следующие:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, повторите попытку |
| 100002 | Неверный параметр, проверьте корректность запроса на основе описания ошибки |
| 100004 | Комната не существует, возможно, она никогда не была создана или уже была удалена |
| 100006 | Нет разрешения на удаление, например оператор не является владельцем дома и администратором root, или удаляемая комната не является комнатой прямой трансляции и т. д. |

# Получить список комнат

## Обзор функции

Администратор приложения может использовать этот интерфейс для получения списка прямых трансляций.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_engine_http_srv/get_room_list?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже перечислены только параметры, которые изменяются при вызове этого интерфейса. Для получения более подробной информации о других параметрах см. [Обзор RESTful API](https://www.tencentcloud.com/document/product/647/63398).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Зарезервированный домен для страны/региона, где находится SDKAppID: Прочие регионы Китая: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Соединённые Штаты: `adminapiusa.im.qcloud.com` |
| v4/live_engine_http_srv/get_room_list | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Должен быть аккаунт администратора приложения. Подробнее см. [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin) |
| usersig | Подпись, созданная аккаунтом администратора приложения. Для подробного описания операций см. [Генерирование UserSig](https://www.tencentcloud.com/document/product/647/69883#) |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса с фиксированным значением: `json` |

### Максимальная частота вызовов

10 раз в секунду.

### Пример пакетов запроса

**Основная форма**

```
{    "Next":"",     "Count":20}
```

**Примечание: Список отсортирован в порядке убывания по полю ViewCount с использованием разбиения на страницы**

### Поля пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| Next | String | Обязательно | Параметры разбиения на страницы для запроса с разбиением на страницы. При первом запросе это пустая строка. Впоследствии используется поле Next из тела ответа. Когда поле Next в теле ответа является пустой строкой, это означает, что загрузка завершена |
| Count | Integer | Обязательно | Количество комнат для одной загрузки. По умолчанию 20, максимум 20 |

### Пример пакетов ответа

**Основная форма**

```
{  "ActionStatus": "OK",  "ErrorInfo": "",  "ErrorCode": 0,  "RequestId": "Id-8c9858f01e954611ae2d4c1b1ed7d583-O-Seq-52720",  "Response" : {    "Next":"1693271355-123123123123",    "RoomList":[      {        "RoomName":"name",        "RoomId":"room_id0",        "Owner_Account":"bren",        "CoverURL":"cover_url",        "Category":[1,2,3],        "CreateTime":1693271354,        "ActivityStatus":1,        "ViewCount":1000      },      ...      {        "RoomName":"name",        "RoomId":"room_id19",        "Owner_Account":"bren",        "CoverURL":"cover_url",        "Category":[1,2,3],        "CreateTime":1693271354,        "ActivityStatus":1,        "ViewCount":900      }    ]   }}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результаты обработки запроса. OK указывает на успех, FAIL указывает на ошибку |
| ErrorCode | Integer | Код ошибки. 0 указывает на успех, ненулевое значение указывает на ошибку |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Уникальный идентификатор запроса, каждый запрос будет возвращать идентификатор запроса, необходимый при локализации проблем |
| RoomList | Array | Список ID комнат, отсортированный по количеству просмотров в порядке убывания по умолчанию |
| RoomName | String | Имя комнаты |
| RoomId | String | ID комнаты |
| Owner_Account | String | ID владельца комнаты |
| CoverURL | String | Обложка комнаты |
| Category | Array | Теги категорий прямой трансляции, максимальный размер массива составляет 3 |
| CreateTime | Integer | Время создания комнаты |
| ActivityStatus | Integer | Статус активности прямой трансляции: определяемый пользователем тег определения |
| ViewCount | Integer | Общее количество входов пользователя в комнату |
| Next | String | Когда список комнат полностью загружен, будет возвращена пустая строка; в противном случае возвращается Next для следующего запроса |

## Коды ошибок

Если не возникает ошибка сетевого соединения (например, ошибка 502), код возврата HTTP для этого интерфейса всегда будет 200. Фактический код ошибки и информация об ошибке указаны в полезной нагрузке ответа с помощью ErrorCode и ErrorInfo. Распространённые коды ошибок (60000–79999) см. в документе [Код ошибки](https://www.tencentcloud.com/document/product/647/60027#). Коды ошибок, специфичные для этого API, следующие:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, пожалуйста повторите попытку |
| 100002 | Недопустимый параметр, пожалуйста проверьте правильность запроса на основе описания ошибки |

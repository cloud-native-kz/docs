# Блокировка членов комнаты

## Обзор функции

Администратор приложения может блокировать членов через этот интерфейс. Заблокированные пользователи не могут входить в комнату в течение периода действия блокировки.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_engine_http_srv/ban_member?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже перечислены только параметры, которые изменяются при вызове этого интерфейса. Для получения дополнительной информации о других параметрах обратитесь к [Обзору RESTful API](https://www.tencentcloud.com/document/product/647/63398).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Зарезервированный домен для страны/региона, в котором находится SDKAppID: Остальное в Китае: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Соединённые Штаты: `adminapiusa.im.qcloud.com` |
| v4/live_engine_http_srv/ban_member | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Должна быть учётная запись администратора приложения. Дополнительную информацию см. в разделе [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin) |
| usersig | Подпись, созданная учётной записью администратора приложения. Для получения подробных инструкций см. [Генерирование UserSig](https://www.tencentcloud.com/document/product/647/69883#) |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса, фиксированное значение: `json` |

### Максимальная частота вызовов

200 запросов/сек.

### Пример пакетов запроса

**Базовая форма**

```
{    "RoomId": "test",     "MemberList_Account": [            "user1",            "user2"            ],    "Reason":"",    "Duration":3600}
```

### Описание полей пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| RoomId | String | Обязательное | ID комнаты |
| MemberList_Account | Array | Обязательное | Список заблокированных членов. Максимум 20 за раз. |
| Reason | String | Опциональное | Причина блокировки |
| Duration | Integer | Обязательное | Время блокировки |

### Пример пакетов ответа

**Базовая форма**

```
{  "ActionStatus": "OK",  "ErrorInfo": "",  "ErrorCode": 0,  "RequestId": "Id-8c9858f01e954611ae2d4c1b1ed7d583-O-Seq-52720"}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат процесса запроса: OK указывает на успех; FAIL указывает на ошибку |
| ErrorCode | Integer | Код ошибки. 0: успех; другие значения: ошибка |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Уникальный ID запроса возвращается с каждым запросом и требуется предоставить этот RequestId при локализации проблем |

## Коды ошибок

Если не возникнет сетевая ошибка (например, ошибка 502), код возврата HTTP для этого интерфейса всегда будет 200. Фактический код ошибки и информация об ошибке указаны в полезной нагрузке ответа в полях ErrorCode и ErrorInfo. Общие коды ошибок (60000–79999) можно найти в документе [Код ошибки](https://www.tencentcloud.com/document/product/647/60027#). Приватные коды ошибок для этого API приведены ниже:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, пожалуйста, повторите попытку |
| 100002 | Неверный параметр, пожалуйста, проверьте корректность запроса на основе описания ошибки |
| 100004 | Комната не существует, возможно, она никогда не была создана или уже была удалена |
| 100006 | Нет прав на передачу владельца |
| 100016 | Новый владелец заблокирован, пожалуйста, разблокируйте сначала |

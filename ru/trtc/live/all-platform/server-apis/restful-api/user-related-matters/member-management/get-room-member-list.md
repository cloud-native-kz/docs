# Получение списка участников комнаты

## Обзор функции

Администраторы приложения могут получить список участников комнаты через этот интерфейс.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_engine_http_srv/get_member_list?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

> **Примечание:** Общая гранулярность обновления списка участников в сети составляет 40 сек. Когда количество участников комнаты превышает 1000, интерфейс возвращает только 1000 участников.

### Параметры запроса

В таблице ниже указаны только параметры, которые изменяются при вызове этого интерфейса. Для получения дополнительной информации о других параметрах обратитесь к документации [REST API](https://www.tencentcloud.com/document/product/647/63398#).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где находится ваш SDKAppID: Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Соединенные Штаты: `adminapiusa.im.qcloud.com` |
| v4/live_engine_http_srv/get_member_list | API запроса |
| sdkappid | SDKAppID, присвоенный при создании приложения |
| identifier | Должна быть учетная запись администратора приложения. Для получения дополнительной информации обратитесь к документации [App Admin](https://www.tencentcloud.com/document/product/647/69882#app-admin) |
| usersig | Подпись, созданная учетной записью администратора приложения; конкретные этапы см. в документации [Generate UserSig](https://www.tencentcloud.com/document/product/647/69883#) |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса — фиксированное значение: `json` |

### Максимальная частота вызовов

200 запросов/сек.

### Примеры пакетов запроса

**Базовая форма**

```
{    "RoomId":"room_id",    "Next": "" // Изначально пустая строка, для последующих запросов используйте значение Next из предыдущего ответа, пока пустая строка не указывает, что все данные получены}
```

### Поля пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| Next | String | Обязательно | Параметры постраничной навигации |
| RoomId | String | Обязательно | ID комнаты |

### Примеры пакетов ответа

**Базовая форма**

```
{  "ErrorCode": 0,   "ErrorInfo": "",  "ActionStatus": "OK",  "RequestId": "requestId-123-456",  "Response" : {    "Next": "",  //Пустой ответ указывает, что все участники получены    "MemberCount": 2, //Общее количество людей в комнате    "MemberList":[      {        "Member_Account" : "brennan1"      },      {        "Member_Account" : "tandy"      },   ]   }}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат процесса запроса: OK указывает на успех; FAIL указывает на ошибку |
| ErrorCode | Integer | Код ошибки. 0: успех; другие значения: ошибка |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Уникальный ID запроса возвращается с каждым запросом и требуется указать этот RequestId при локализации проблем |
| MemberList | Array | Список участников комнаты |
| Next | String | Значение Next в параметрах следующего запроса |
| MemberCount | Integer | Общее количество людей в комнате |

## Коды ошибок

Если не произойдет ошибка сети (например, ошибка 502), код возврата HTTP для этого интерфейса всегда равен 200. Фактический код ошибки и сообщение об ошибке находятся в поле ответа ErrorCode и ErrorInfo.

Общие коды ошибок (60000–79999) см. в документации [Error Code](https://www.tencentcloud.com/document/product/647/60027).

Частные коды ошибок для этого API:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Системная ошибка, повторите попытку |
| 100004 | Комната не существует, возможно потому, что она никогда не была создана или уже была закрыта |
| 100006 | Указывает, что эта комната не является прямой комнатой и не имеет разрешения на использование интерфейсов запроса прямых трансляций |

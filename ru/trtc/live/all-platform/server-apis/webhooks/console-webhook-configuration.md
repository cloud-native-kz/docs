# Конфигурация веб-хука консоли

Войдите в [консоль TencentRTC Live](https://console.trtc.io/), выберите целевое приложение для входа на страницу управления приложением. На левой панели навигации выберите **Live > Configuration > Live Webhook**. Вы можете настроить URL веб-хука и включить веб-хуки в соответствии с вашими фактическими потребностями в бизнесе.

## Настройка URL веб-хука

1. На странице **Live Webhook** выполните **редактирование** URL веб-хука в области конфигурации Live webhook.
2. В всплывающем диалоговом окне **Live webhook configuration** введите URL веб-хука.

> **Примечание:** Новый URL обратного вызова вступает в силу через 2 минуты после успешного изменения. URL обратного вызова должен начинаться с `http://` или `https://`. Если вы не зарегистрировали доменное имя, вы можете напрямую настроить IP-адрес, например `http://123.123.123.123/imcallback`. Допускается использование только английских букв (a-z, без учета регистра), цифр (0-9) и "-" (дефис). Не используйте пробелы или специальные символы, такие как !, $, &, ?. Дефис не может появляться подряд, регистрироваться отдельно или находиться в начале и в конце. Длина доменного имени не превышает 63 символов. URL обратного вызова в IM по умолчанию включает порты 80/443. При замене URL обратного вызова требуется настройка портов. Избегайте взаимно префиксных портов до и после замены, например изменение `https://xxx:443` на `https://xxx:4433` или `https://xxx` на `https://xxx:4433`.

3. Нажмите **Confirm** для сохранения конфигурации.

## Настройка веб-хука события

1. На странице **Live webhook** вы можете напрямую включить/отключить переключатель веб-хука в области конфигурации Live webhook.

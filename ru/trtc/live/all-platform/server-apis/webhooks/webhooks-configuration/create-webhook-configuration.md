# Создание конфигурации веб-хука

## Обзор функции

Администраторы приложения могут создавать конфигурации веб-хуков через этот интерфейс.

## Описание вызова API

### Пример URL запроса

```
https://console.tim.qq.com/v4/live_config/set_callback?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже приведены только параметры, изменяемые при вызове этого API, и их описание. Дополнительную информацию см. в [Обзоре RESTful API](https://www.tencentcloud.com/document/product/647/63398#).

| Параметр | Описание |
| --- | --- |
| v4/live_config/set_callback | API запроса |
| xxxxxx | Зарезервированный домен для страны/региона, где находится SDKAppID: Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` США: `adminapiusa.im.qcloud.com` |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Должен быть учетной записью администратора приложения. Дополнительную информацию см. в [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin) |
| usersig | Подпись, созданная учетной записью администратора приложения. Для получения подробной информации см. [Создание UserSig](https://www.tencentcloud.com/document/product/647/69883#) |
| random | Случайное 32-битное беззнаковое целое число в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса. Фиксированное значение: `json` |

### Максимальная частота вызовов

200 запросов/сек.

### Примеры пакетов запроса

- **Базовая форма**
Создание комнаты

```
{  "Url":"http://www.example.com/callback",  "CallbackCommandList":[      "Live.CallbackAfterCreateRoom",      "Live.CallbackAfterDestroyRoom",      "Live.CallbackAfterMemberStateChanged"  ]}
```

### Поля пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| Url | String | Обязательно | Адрес веб-хука, должен начинаться с http/https, рекомендуется использовать более безопасный https |
| CallbackCommandList | Array | Обязательно | Команды, которые срабатывают при отправке веб-хука. См. [Список команд веб-хука](https://www.tencentcloud.com/document/product/647/64413#) |

### Примеры пакетов ответа

- **Базовая форма**

```
{  "ActionStatus": "OK",  "ErrorInfo": "",  "ErrorCode": 0,  "RequestId": "Id-8c9858f01e954611ae2d4c1b1ed7d583-O-Seq-52720"}
```

### Описание поля пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат процесса запроса: OK указывает на успех; FAIL указывает на ошибку |
| ErrorCode | Integer | Код ошибки. 0: успех; другие значения: ошибка |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Уникальный идентификатор запроса возвращается с каждым запросом и требуется предоставить этот RequestId при поиске проблем |

## Коды ошибок

Если не происходит сетевая ошибка (например, ошибка 502), код состояния HTTP для этого интерфейса всегда будет 200. Фактические коды ошибок и сообщения передаются через ErrorCode и ErrorInfo в теле ответа.
Для общих кодов ошибок (60000–79999) см. документацию [Код ошибки](https://www.tencentcloud.com/document/product/647/60027#).
Приватные коды ошибок для этого API приведены ниже:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, пожалуйста, повторите попытку |
| 100002 | Неверный параметр. Пожалуйста, проверьте запрос в соответствии с описанием ошибки. |
| 100300 | Конфигурация веб-хука уже существует. Вы можете использовать интерфейс обновления конфигурации веб-хука для обновления |

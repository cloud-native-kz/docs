# Конфигурация вебхука запроса

## Обзор функции

Администраторы приложения могут создать вебхук через этот интерфейс.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_config/get_callback?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже приведены только параметры, измененные при вызове этого API, и их описание. Дополнительные сведения см. в разделе [Обзор RESTful API](https://www.tencentcloud.com/document/product/647/63398#).

| Параметр | Описание |
| --- | --- |
| xxxxxxx | Зарезервированный домен для страны/региона, в котором расположен SDKAppID: Китай: `console.tim.qq.com`, Сингапур: `adminapisgp.im.qcloud.com`, США: `adminapiusa.im.qcloud.com` |
| v4/livee_config/get_callback | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Должно быть учетной записью администратора приложения. Дополнительные сведения см. в разделе [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin) |
| usersig | Подпись, созданная учетной записью администратора приложения. Подробные инструкции см. в разделе [Генерирование UserSig](https://www.tencentcloud.com/document/product/647/69883#) |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса, фиксированное значение: `json` |

### Максимальная частота вызовов

200 запросов/сек.

### Поля пакета запроса

### Примеры пакетов ответа

**Основная форма**

```
{  "ErrorCode": 0,  "ErrorInfo": "",  "ActionStatus": "OK",  "RequestId": "Id-1cc8828fd3d84795ac866ced43b15b5c-O-Seq-61309",  "Response": {      "Url": "http://www.example.com/callback",      "CallbackCommandList": [        "Live.CallbackAfterCreateRoom",        "Live.CallbackAfterDestroyRoom"      ]  }}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат процесса запроса: OK указывает на успех; FAIL указывает на ошибку |
| ErrorCode | Integer | Код ошибки. 0: успех; другие значения: ошибка |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Уникальный идентификатор запроса возвращается с каждым запросом и требуется предоставить этот RequestId при локализации проблем |
| Url | String | Адрес вебхука, должен начинаться с http/https, рекомендуется использовать более безопасный https |
| CallbackCommandList | Array | Для слов команд, которые запускают вебхук, см. [Список слов команд вебхука](https://www.tencentcloud.com/document/product/647/64413#) |

## Коды ошибок

Если не возникает ошибка сети (например, ошибка 502), код состояния HTTP для этого интерфейса всегда будет 200. Фактические коды ошибок и сообщения передаются через ErrorCode и ErrorInfo в теле ответа.
Распространенные коды ошибок (60000–79999) см. в документации [Код ошибки](https://www.tencentcloud.com/document/product/647/60027#).
Приватные коды ошибок для этого API приведены ниже:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, пожалуйста, повторите попытку |
| 100002 | Недопустимый параметр. Проверьте запрос согласно описанию ошибки. |
| 100301 | Конфигурация вебхука не существует, вы можете создать ее с помощью интерфейса Create Webhook Configuration |

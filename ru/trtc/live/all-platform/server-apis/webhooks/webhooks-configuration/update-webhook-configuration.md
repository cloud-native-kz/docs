# Обновление конфигурации Webhook

## Обзор функции

Администраторы приложения могут создать webhook через этот интерфейс.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_config/update_callback?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже указаны только параметры, которые изменяются при вызове этого API, и их описание. Для получения дополнительной информации см. [Обзор RESTful API](https://www.tencentcloud.com/document/product/647/63398#).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Зарезервированный домен для страны/региона, где находится SDKAppID:Китай: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Соединённые Штаты: `adminapiusa.im.qcloud.com` |
| v4/live_config/update_callback | API запроса |
| sdkappid | SDKAppID, присвоенный консолью Chat при создании приложения |
| identifier | Должна быть учётная запись администратора приложения. Для получения дополнительной информации см. [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin) |
| usersig | Подпись, сгенерированная учётной записью администратора приложения. Для подробной процедуры см. [Генерирование UserSig](https://www.tencentcloud.com/document/product/647/69883#) |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса, фиксированное значение `json` |

### Максимальная частота вызовов

200 запросов/сек.

### Примеры пакетов запроса

- **Базовая форма**
Создание комнаты

```
{  "Url":"http://www.example.com/callback",  "CallbackCommandList":[    "Live.CallbackAfterCreateRoom",    "Live.CallbackAfterDestroyRoom"  ]}
```

### Поля пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| Url | String | Обязательное | Адрес webhook, должен начинаться с http/https, рекомендуется использовать более безопасный https |
| CallbackCommandList | Array | Обязательное | Команды, которые запускают webhook, см. [Список команд Webhook](https://www.tencentcloud.com/document/product/647/64413#) |

### Примеры пакетов ответа

- **Базовая форма**

```
{  "ActionStatus": "OK",  "ErrorInfo": "",  "ErrorCode": 0,  "RequestId": "Id-8c9858f01e954611ae2d4c1b1ed7d583-O-Seq-52720"}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат процесса запроса: OK указывает на успех; FAIL указывает на ошибку |
| ErrorCode | Integer | Код ошибки. 0: успех; другие значения: ошибка |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Уникальный ID запроса возвращается с каждым запросом и требуется предоставить этот RequestId при поиске проблем |

## Коды ошибок

Если не возникает ошибка сети (например, ошибка 502), код состояния HTTP для этого интерфейса всегда будет 200. Фактические коды ошибок и сообщения передаются через ErrorCode и ErrorInfo в теле ответа.
Распространённые коды ошибок (60000–79999) см. в документации [Код ошибки](https://www.tencentcloud.com/document/product/647/60027#).
Приватные коды ошибок для этого API следующие:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, повторите попытку |
| 100002 | Неверный параметр. Проверьте запрос в соответствии с описанием ошибки. |
| 100301 | Конфигурация webhook не существует. Вы можете создать её с помощью интерфейса создания конфигурации Webhook |

# Обзор вебхуков

Для обеспечения контроля над функциями приложения LiveKit предоставляет возможности вебхуков.

## Обзор функций

Пользователи могут настроить вебхук на указанный URL через REST API. Когда выполняемая WebhookCommand находится в списке конфигурации, это вызовет вебхук.

## Примечания

- За каждый sdkAppId можно установить только один вебхук.
- Убедитесь, что URL вебхука доступен.

## Протокол вебхука

Сторонние вебхуки основаны на протоколах HTTP/HTTPS. Бэкэнд приложения должен предоставить URL вебхука на живую платформу, и Instant Chat использует POST-запрос для инициирования вебхука на бэкэнд приложения. При инициировании вебхука Chat добавляет следующие параметры к URL, предоставленному приложением:

| Параметр | Значение |
| --- | --- |
| SdkAppId | SDKAppId, назначенный консолью Chat при создании приложения |
| WebhookCommand | Команда вебхука |
| contenttype | Необязательно, обычно установлено как JSON |
| ClientIP | IP-адрес клиента |
| OptPlatform | Платформа клиента, соответствующая различным типам платформ, возможные значения: RESTAPI (отправка запросов с помощью REST API), Web (отправка запросов с помощью Web SDK), Android, iOS, Windows, Mac, iPad, Unknown (отправка запросов с использованием неизвестного типа устройства) |

## Вопросы безопасности

Соответствующие проблемы безопасности следующие:

1. HTTP передает данные открытым текстом, и конфиденциальность данных не гарантируется. Поэтому рекомендуется использовать HTTPS.
2. Невозможно определить, действительно ли запрос вебхука поступает из Live.

Для безопасности источника запроса мы предоставляем решение **аутентификации вебхука**:

1. Настройте URL вебхука и включите использование вебхука [установить конфигурацию вебхука](https://www.tencentcloud.com/document/product/647/64420#).
2. В [настройке токена аутентификации вебхука](https://www.tencentcloud.com/document/product/647/69528#) включите аутентификацию и настройте токен аутентификации. Затем к URL запроса вебхука будут добавлены подпись (`Sign`) и временная метка подписания (`RequestTime`). Алгоритм подписи: **Sign=sha256(TokenRequestTime)**.
3. Бэкэнд приложения аутентифицирует запрос вебхука. Он использует SHA256 для вычисления и проверки подписи на основе локального токена аутентификации и временной метки подписания (`RequestTime`) в URL вебхука. Также проверяется своевременность RequestTime. Если разница между RequestTime и текущим временем превышает 1 минуту, это считается недействительным запросом для предотвращения атак воспроизведения.

```
Образец алгоритма подписи:Token=xxxxyyyyRequestTime=1669872112Sign=sha256(xxxxyyyy1669872112)=17773bc39a671d7b9aa835458704d2a6db81360a5940292b587d6d760d484061Webhook URL=URL&Sign=17773bc39a671d7b9aa835458704d2a6db81360a5940292b587d6d760d484061&RequestTime=1669872112
```

## Распространённые причины сбоев вебхуков

Если возникает сбой вебхука, проверьте, имеет ли настроенный сервис вебхука проблему в соответствии со следующим контрольным списком:

| Признак сбоя вебхука | Возможная причина |
| --- | --- |
| Время доступа к URL вебхука истекло | Live не может выполнить разрешение DNS. В этом случае проверьте, действителен ли доменное имя в общей сети. Например, если хост вебхука — http://notexist.com, Live не может выполнить разрешение DNS, потому что это доменное имя не существует. Live не может получить доступ к IP-адресу, настроенному в URL вебхука. В этом случае проверьте, доступен ли этот IP-адрес из общей сети. Например, если хост вебхука — http://10.0.0.1, Live не может получить доступ к этому IP-адресу, потому что доменное имя является частным IP-адресом приложения. Сбой происходит из-за политики брандмауэра сервиса вебхука приложения. В этом случае проверьте конфигурацию брандмауэра. Например, сбой вебхука происходит, если сервер вебхука приложения отклоняет все запросы, приходящие на порт 80. |
| Доступ запрещён сервисом вебхука | Live может получить доступ к хосту, но соединение не установлено. В этом случае проверьте, правильно ли запущен WebServer. Например, сбой вебхука произойдёт, если WebServer сервера вебхука приложения не запущен или конфигурация порта неверна. |
| Код возврата HTTP сервиса вебхука не равен 200 | Запрос вебхука успешен, но код возврата HTTP в пакете ответа не равен 200. |
| Пакет ответа вебхука не удалось разобрать | Пакет ответа вебхука имеет формат, отличный от JSON. |

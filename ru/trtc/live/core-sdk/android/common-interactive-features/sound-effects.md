# Звуковые эффекты

Этот документ содержит рекомендации по интеграции элементов управления звуковыми эффектами в приложение прямой трансляции с использованием модулей `AudioEffectStore` и `DeviceStore` из фреймворка **AtomicXCore**. Эти модули позволяют реализовать функции управления громкостью микрофона, внутриушного мониторинга и различные эффекты изменения голоса и реверберации.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/db216561c6bc11f0a7775254005ef0f7.png)

## Основные возможности

Благодаря комбинации `AudioEffectStore` и `DeviceStore` вы можете реализовать следующие ключевые функции:

- **Регулировка громкости микрофона**: управление громкостью локального микрофона (захвата), которая определяет громкость, которую слышит ваша аудитория.
- **Внутриушной мониторинг**: позволяет ведущим или сооведущим слышать собственный голос в реальном времени через наушники для мониторинга и корректировки произношения.
- **Эффекты голоса**: применяйте различные эффекты изменения голоса, такие как «маленькая девочка», «глубокий мужской голос», «непослушный ребенок» и другие.
- **Эффекты реверберации**: имитируйте различные акустические окружения, такие как KTV, металлический, глубокий, яркий и другие.

## Основные концепции

| **Основная концепция** | **Тип** | **Основная ответственность и описание** |
| --- | --- | --- |
| `AudioChangerType` | `enum` | Перечисляет доступные эффекты голоса, такие как «child», «man» и т. д. |
| `AudioReverbType` | `enum` | Перечисляет доступные эффекты реверберации, такие как «ktv», «hall» и т. д. |
| `AudioEffectState` | `data` | Представляет текущее состояние модуля звуковых эффектов, обычно используется для отрисовки пользовательского интерфейса. Включает состояние эффекта голоса, состояние реверберации, состояние включения внутриушного мониторинга и громкость внутриушного монитора. |
| `AudioEffectStore` | `abstract` | Класс управления данными-синглтон для звуковых эффектов. Используйте его для вызова API звуковых эффектов. После вызова API соответствующее свойство состояния автоматически обновляется. Подпишитесь на это реактивное состояние, чтобы получать и отслеживать обновления. |
| `DeviceState` | `data` | Представляет текущее состояние модуля устройства, обычно используется для отрисовки пользовательского интерфейса. Основные свойства включают статус камеры и микрофона. |
| `DeviceStore` | `abstract` | Класс управления данными-синглтон для операций с устройствами. Используйте его для управления камерой и микрофоном. После вызова API соответствующее свойство состояния автоматически обновляется. Подпишитесь на это реактивное состояние, чтобы получать и отслеживать обновления. |

## Реализация

### Шаг 1: Интеграция компонентов

- **Видео прямая трансляция**: обратитесь к [Быстрый старт](https://www.tencentcloud.com/document/product/647/74593) для интеграции **AtomicXCore**.
- **Голосовая комната чата**: обратитесь к [Быстрый старт](https://www.tencentcloud.com/document/product/647/74681) для интеграции **AtomicXCore**.

### Шаг 2: Получение экземпляров Store

`AudioEffectStore` и `DeviceStore` — это объекты-синглтоны. Получайте доступ к их экземплярам в любом месте вашего проекта через свойство `shared`. Полный пример реализации см. в [AudioEffectPanel.kt](https://github.com/Tencent-RTC/TUIKit_Android/blob/main/live/tuilivekit/src/main/java/com/trtc/uikit/livekit/component/audioeffect/AudioEffectPanel.kt) в демо-версии с открытым исходным кодом TUILiveKit.

```
import io.trtc.tuikit.atomicxcore.api.device.AudioEffectStoreimport io.trtc.tuikit.atomicxcore.api.device.DeviceStoreval audioEffectStore = AudioEffectStore.shared()val deviceStore = DeviceStore.shared()
```

### Шаг 3: Реализация внутриушного мониторинга

Внутриушной мониторинг позволяет пользователям слышать свой микрофон в реальном времени. Управляйте внутриушным мониторингом, используя следующие API.

> **Примечание:** Внутриушной мониторинг обычно требует наушников для правильной работы.

#### Реализация

1. **Переключение внутриушного мониторинга**: используйте элемент управления `Switch` для включения или отключения внутриушного мониторинга.
2. **Установка громкости внутриушного монитора**: используйте `SeekBar` (или аналогичный элемент управления) для регулировки громкости внутриушного монитора. Сопоставьте значение элемента управления пользовательского интерфейса с диапазоном [0, 150] и вызовите `setVoiceEarMonitorVolume(volume)`.
3. **Прослушивание обновлений состояния**: подпишитесь на изменения состояния внутриушного монитора, чтобы обновлять пользовательский интерфейс в реальном времени.

#### Пример кода

```
import io.trtc.tuikit.atomicxcore.api.device.AudioEffectStoreimport kotlinx.coroutines.CoroutineScopeimport kotlinx.coroutines.Dispatchersimport kotlinx.coroutines.launch// 1. Включить внутриушной мониторинг (попросить пользователя подключить наушники)audioEffectStore.setVoiceEarMonitorEnable(true)// 2. Установить громкость внутриушного монитораaudioEffectStore.setVoiceEarMonitorVolume(80)// 3. Отключить внутриушной мониторингaudioEffectStore.setVoiceEarMonitorEnable(false)// 4. Прослушивать обновления состоянияCoroutineScope(Dispatchers.Main).launch {  audioEffectStore.audioEffectState.earMonitorVolume.collect {    print("Ear monitor volume: $it")    // Обновить пользовательский интерфейс с помощью значения  }}
```

#### Параметры setVoiceEarMonitorEnable

| **Имя параметра** | **Тип** | **Описание** |
| --- | --- | --- |
| `enable` | `Boolean` | Включить ли внутриушной мониторинг.- true: включить- false: отключить |

#### Параметры setVoiceEarMonitorVolume

| **Имя параметра** | **Тип** | **Описание** |
| --- | --- | --- |
| `volume` | `Int` | Громкость внутриушного монитора.- Диапазон: [0, 150]- По умолчанию: 100 |

### Шаг 4: Реализация регулировки громкости микрофона

Чтобы отрегулировать громкость микрофона (захвата), вызовите `setCaptureVolume(volume)` на `DeviceStore` с требуемым значением.

#### Реализация

1. **Установка громкости захвата**: используйте `SeekBar` или аналогичный элемент управления для регулировки громкости микрофона. Сопоставьте значение элемента управления пользовательского интерфейса с диапазоном [0, 150] и вызовите `setCaptureVolume(volume)`.
2. **Прослушивание обновлений состояния**: подпишитесь на изменения состояния громкости захвата, чтобы обновлять пользовательский интерфейс в реальном времени.

#### Пример кода

```
import kotlinx.coroutines.CoroutineScopeimport kotlinx.coroutines.Dispatchersimport kotlinx.coroutines.launchimport io.trtc.tuikit.atomicxcore.api.device.DeviceStore// 1. Отрегулировать громкость микрофонаdeviceStore.setCaptureVolume(80)// 2. Прослушивать изменения громкостиCoroutineScope(Dispatchers.Main).launch {   deviceStore.deviceState.captureVolume.collect {     print("Capture volume: $it")     // Обновить пользовательский интерфейс с помощью значения   }}
```

#### Параметры setCaptureVolume

| **Имя параметра** | **Тип** | **Описание** |
| --- | --- | --- |
| `volume` | `Int` | Громкость микрофона (захвата).- Диапазон: [0, 150]- По умолчанию: 100 |

### Шаг 5: Реализация эффектов голоса

Чтобы применить эффект голоса, вызовите `setAudioChangerType(type)` с требуемым значением перечисления `AudioChangerType`.

#### Пример кода

```
import io.trtc.tuikit.atomicxcore.api.device.AudioEffectStoreimport io.trtc.tuikit.atomicxcore.api.device.AudioChangerType// Установить эффект голоса на «маленькая девочка»audioEffectStore.setAudioChangerType(AudioChangerType.LITTLE_GIRL)// Отключить эффект голосаaudioEffectStore.setAudioChangerType(AudioChangerType.NONE)
```

#### Параметры setAudioChangerType

| **Имя параметра** | **Тип** | **Описание** |
| --- | --- | --- |
| `type` | `AudioChangerType` | Значение перечисления эффекта голоса. |

### Шаг 6: Реализация эффектов реверберации

Чтобы применить эффект реверберации, вызовите `setAudioReverbType(type)` с требуемым значением перечисления `AudioReverbType`.

```
import io.trtc.tuikit.atomicxcore.api.device.AudioEffectStoreimport io.trtc.tuikit.atomicxcore.api.device.AudioReverbType// Установить реверберацию на «KTV»audioEffectStore.setAudioReverbType(AudioReverbType.KTV)// Отключить реверберациюaudioEffectStore.setAudioReverbType(AudioReverbType.NONE)
```

#### Параметры setAudioReverbType

| **Имя параметра** | **Тип** | **Описание** |
| --- | --- | --- |
| `type` | `AudioReverbType` | Значение перечисления эффекта реверберации. |

### Шаг 7: Сброс параметров звуковых эффектов

> **Примечание:** `AudioEffectStore` и `DeviceStore` — это синглтоны, поэтому параметры звуковых эффектов и устройств применяются глобально. Чтобы предотвратить влияние параметров одной комнаты трансляции на другие, сбросьте параметры звуковых эффектов и устройств по окончании текущей комнаты трансляции.

Сбросьте параметры звуковых эффектов в следующих сценариях:

1. Когда текущая комната трансляции завершается.
2. Чтобы предоставить функцию «одиночный щелчок для восстановления», вызовите метод `reset()` на каждом хранилище.

```
import io.trtc.tuikit.atomicxcore.api.device.AudioEffectStoreimport io.trtc.tuikit.atomicxcore.api.device.DeviceStore// Сброс всех эффектов голоса и реверберации одиночным щелчкомAudioEffectStore.shared().reset()DeviceStore.shared().reset()
```

## Часто задаваемые вопросы

### Есть ли какие-либо ограничения на время вызова API звуковых эффектов и устройств?

Нет. Параметры `AudioEffectStore` и `DeviceStore` действуют глобально. Вы можете вызывать соответствующие API (например, установку эффектов голоса, реверберации или внутриушного мониторинга) в любое время до или после входа в комнату трансляции. Параметры вступают в силу немедленно и сохраняются.

### Какая разница между «громкостью микрофона» и «громкостью внутриушного монитора»?

Это распространенный источник путаницы:

- **Громкость микрофона (громкость захвата)**: установка через `DeviceStore.setCaptureVolume()`. Управляет громкостью, которую слышит ваша аудитория.
- **Громкость внутриушного монитора**: установка через `AudioEffectStore.setVoiceEarMonitorVolume()`. Определяет только уровень громкости, который вы слышите в своем внутриушном мониторе, и не влияет на аудиторию.

### Почему в новой созданной комнате трансляции уже есть звуковые эффекты, или почему громкость слишком высокая или слишком низкая?

Потому что `AudioEffectStore` и `DeviceStore` — это синглтоны, параметры звуковых эффектов и устройств применяются глобально. Если вы ранее установили звуковые эффекты и не сбросили их, эти параметры сохранятся. Вызовите метод `reset()` в подходящее время, чтобы очистить их.

### Почему я не слышу свой голос после включения «внутриушного мониторинга»?

Проверьте следующее:

1. **Подключены ли наушники?** Внутриушной мониторинг обычно требует наушников. Убедитесь, что подключены проводные или беспроводные наушники.
2. **Установлена ли громкость внутриушного монитора на 0?** Проверьте, что громкость внутриушного монитора больше 0.
3. **Включен ли микрофон?** Убедитесь, что микрофон включен в DeviceStore (то есть вызван `openLocalMicrophone`).

### Можно ли одновременно включить эффекты голоса и реверберацию?

Да. `AudioChangerType` и `AudioReverbType` можно включать одновременно. Например, вы можете установить одновременно `AudioChangerType.LITTLE_GIRL` и `AudioReverbType.KTV`.

# Виртуальные подарки

`GiftStore` — это специализированный модуль в составе **AtomicXCore** для управления функциями дарения подарков в прямых трансляциях. Он позволяет создать полноценную систему дарения подарков для приложения прямой трансляции, поддерживающую надежную монетизацию и интерактивные возможности.

## Основные возможности

- **Получение списка подарков**: получение данных панели подарков с сервера, включая категории подарков и их детали.
- **Отправка подарка**: зрители могут отправлять выбранные подарки и указывать количество ведущему.
- **Трансляция событий подарков**: получение событий подарков в реальном времени в комнате для отображения анимаций подарков и уведомлений в барраде.

## Основные концепции

| **Основная концепция** | **Тип** | **Основные обязанности и описание** |
| --- | --- | --- |
| `Gift` | `data class` | Представляет модель данных подарка, включая ID, название, URL значка, URL ресурса анимации (resourceURL), цену (монеты) и многое другое. |
| `GiftCategory` | `data class` | Представляет категорию подарка (например, «Популярные», «Люкс»). Содержит название категории и список объектов Gift. |
| `GiftState` | `data class` | Представляет текущее состояние модуля подарков. Основное свойство usableGifts — это StateFlow, содержащий полный список подарков с сервера. |
| `GiftEvent` | `abstract class` | Определяет прослушиватель событий подарков в прямой трансляции. Сейчас доступен только onReceiveGift; все пользователи в комнате получают это событие, когда кто-либо отправляет подарок. |
| `GiftStore` | `abstract class` | Основной класс управления функциями подарков. Используйте его для получения списка подарков, отправки подарков и получения событий подарков через прослушиватели. |

### Шаг 2: инициализация и прослушивание событий подарков

Получите экземпляр `GiftStore` и установите прослушиватели для получения событий подарков и обновлений списка подарков.

#### Реализация

1. **Получение экземпляра**: используйте `GiftStore.create(liveID)` для получения экземпляра `GiftStore` для текущей прямой трансляции.
2. **Добавление прослушивателя**: добавьте `GiftListener` для получения событий `onReceiveGift`.
3. **Подписка на состояние**: подпишитесь на `giftStore.giftState.usableGifts` для получения обновлений списка подарков.

#### Пример кода

```
import io.trtc.tuikit.atomicxcore.api.gift.Giftimport io.trtc.tuikit.atomicxcore.api.gift.GiftCategoryimport io.trtc.tuikit.atomicxcore.api.gift.GiftListenerimport io.trtc.tuikit.atomicxcore.api.gift.GiftStoreimport io.trtc.tuikit.atomicxcore.api.live.LiveUserInfoimport kotlinx.coroutines.flow.StateFlowimport kotlinx.coroutines.flow.MutableStateFlowimport kotlinx.coroutines.flow.asStateFlowimport kotlinx.coroutines.CoroutineScopeimport kotlinx.coroutines.Dispatchersimport kotlinx.coroutines.launchclass GiftManager(    private val liveId: String) {    private val giftStore: GiftStore = GiftStore.create(liveId)    private val coroutineScope = CoroutineScope(Dispatchers.Main)    // Expose gift list externally    private val _giftList = MutableStateFlow<List<GiftCategory>>(emptyList())    val giftList: StateFlow<List<GiftCategory>> = _giftList.asStateFlow()    private val giftListener = object : GiftListener() {        override fun onReceiveGift(liveID: String, gift: Gift, count: Int, sender: LiveUserInfo) {            // Forward event to UI layer for processing        }    }    init {        setupGiftListener()        subscribeToGiftState()    }    // Subscribe to gift events    private fun setupGiftListener() {        giftStore.addGiftListener(giftListener)    }    // Subscribe to gift list state    private fun subscribeToGiftState() {        coroutineScope.launch(Dispatchers.Main) {            giftStore.giftState.usableGifts.collect { giftList ->                _giftList.value = giftList            }        }    }}
```

### Шаг 3: получение списка подарков

Вызовите метод `refreshUsableGifts` для получения данных подарков с сервера.

#### Реализация

1. **Вызов API**: вызовите `giftStore.refreshUsableGifts()` в нужный момент (например, после входа в прямую трансляцию).
2. **Обработка обратного вызова**: опционально обработайте обратный вызов завершения, чтобы определить результат.
3. **Получение данных**: после успешного получения обновленный список usableGifts доставляется автоматически через вашу подписку на `giftStore.giftState.usableGifts`.

#### Пример кода

```
class GiftManager(    private val liveId: String) {    private val giftStore: GiftStore = GiftStore.create(liveId)    // Refresh the gift list from the server    fun fetchGiftList() {        giftStore.refreshUsableGifts(object : CompletionHandler {            override fun onSuccess() {                println("Gift list fetched successfully")                // Data will be updated via the giftState subscription            }            override fun onFailure(code: Int, desc: String) {                println("Failed to fetch gift list: $desc")            }        })    }}
```

### Шаг 4: отправка подарка

Когда пользователь выбирает подарок и нажимает отправить, вызовите API `sendGift`.

#### Реализация

1. **Получение параметров**: получите выбранный `giftID` и `quantity (count)` из пользовательского интерфейса.
2. **Вызов API**: вызовите `giftStore.sendGift(giftID, count, completion)`.
3. **Обработка обратного вызова**: используйте обратный вызов `completion` для обработки ошибок (например, недостаточный баланс). Обновление пользовательского интерфейса (анимации, баррада) после успешной отправки должно быть инициировано событием `onReceiveGift`.

#### Пример кода

```
class GiftManager(    private val liveId: String) {    private val giftStore: GiftStore = GiftStore.create(liveId)    // Send a gift    fun sendGift(giftID: String, count: Int) {        giftStore.sendGift(giftID, count, object : CompletionHandler {            override fun onSuccess() {                println("Gift $giftID sent successfully")                // All users, including the sender, will receive the onReceiveGift event            }            override fun onFailure(code: Int, desc: String) {                println("Gift send failed: $desc")                // Prompt user if needed, e.g., "Insufficient balance" or "Network error"            }        })    }}
```

#### **Параметры API sendGift**

| **Имя параметра** | **Тип** | **Описание** |
| --- | --- | --- |
| `giftID` | `String` | Уникальный ID подарка для отправки. |
| `count` | `Int` | Количество подарков для отправки. |
| `completion` | `CompletionHandler?` | Обратный вызов, вызываемый после завершения отправки. |

## Расширенные возможности

### Конфигурация активов подарков

Настройте доступные типы подарков, категории, названия, значки, цены и эффекты анимации в соответствии с вашими операционными требованиями и требованиями брендинга.

#### Реализация

1. **Конфигурация на стороне сервера**: используйте REST API сервера `LiveKit` для управления информацией о подарках, категориями и поддержкой многоязычности. См. [Руководство по конфигурации подарков](https://www.tencentcloud.com/document/product/647/72844).
2. **Получение на клиенте**: на клиенте вызовите `refreshUsableGifts` для получения данных конфигурации.
3. **Отображение в пользовательском интерфейсе**: используйте полученный `List<GiftCategory>` для заполнения панели подарков.

### Процесс выставления счетов и вычитания подарков

Когда зритель отправляет подарок, убедитесь, что баланс его счета достаточен, и выполните вычитание перед воспроизведением эффекта подарка и его трансляцией.

#### Реализация

1. **Конфигурация обратного вызова на стороне сервера**: установите URL обратного вызова вашей системы выставления счетов в серверной части LiveKit. См. [Документацию по конфигурации обратного вызова](https://www.tencentcloud.com/document/product/647/64412).
2. **Отправка на клиенте**: клиент вызывает `sendGift`.
3. **Взаимодействие с сервером**: серверная часть LiveKit вызывает URL вашего обратного вызова; ваша система выставления счетов обрабатывает вычитание и возвращает результат.
4. **Синхронизация результата**: если успешно, **AtomicXCore** транслирует событие `onReceiveGift`; если неудачно, обратный вызов завершения sendGift получает ошибку.

### Реализация воспроизведения анимации подарка во весь экран

Когда пользователь отправляет роскошный подарок (например, «Ракету», «Карнавал»), воспроизведите анимацию во весь экран (например, SVGA) для повышения привлекательности.

#### Реализация

**AtomicXCore** не включает встроенный проигрыватель анимации подарков. Интегрируйте стороннюю библиотеку в зависимости от ваших требований. Ниже приведено сравнение двух вариантов:

| **Элемент сравнения** | **Базовое решение (SVGAPlayer)** | **Gift AR (TCEffectPlayerKit)** |
| --- | --- | --- |
| Выставление счетов | Бесплатно (открытый исходный код) | Платный (требуется лицензия). См. [Руководство по выставлению счетов](https://trtc.io/document/69949?product=beautyar&menulabel=core%20sdk&platform=android#X-Series-Capabilities) |
| Метод интеграции | Ручная интеграция через Gradle | Требует дополнительной интеграции и аутентификации |
| Поддерживаемые форматы анимации | Только SVGA | SVGA, PAG, WebP, Lottie, MP4 и другие |
| Производительность | Рекомендуемый размер файла SVGA ≤ 10 МБ | Поддерживает файлы большего размера, лучшая производительность для сложных/множественных/низкопроизводительных сценариев |
| Рекомендуемый сценарий | Единообразный формат SVGA, управляемый размер файла | Несколько форматов или более высокие требования к производительности/совместимости устройств |

В этом разделе демонстрируется базовое решение. Для **Gift AR** см. [Руководство по интеграции TCEffectPlayer](https://trtc.io/document/70531?product=beautyar&menulabel=core%20sdk&platform=android).

#### Базовое решение: использование SVGAPlayer

1. **Интеграция SVGAPlayer**: добавьте зависимость SVGAPlayer в build.gradle и синхронизируйте проект.

```
dependencies {    // ... other dependencies    implementation 'com.github.yyued:SVGAPlayer-Android:2.6.1'}
```

2. **Прослушивание событий подарков**: подпишитесь на `GiftListener` в `GiftStore`.
3. **Парсинг и воспроизведение**: на `onReceiveGift` проверьте, является ли `gift.resourceURL` корректным файлом SVGA. Используйте `SVGAParser` для парсинга и воспроизведения анимации.

### Отображение сообщений о подарках в области барады

Когда пользователь отправляет подарок, отобразите системное сообщение в общей области барады (например, «[Никнейм зрителя] отправил [Название подарка] x [Количество]») для всех зрителей.

#### Реализация

1. **Прослушивание событий**: подпишитесь на `GiftListener` в `giftStore`.
2. **Получение информации**: на `onReceiveGift` извлеките отправителя, подарок и количество.
3. **Получение хранилища барады**: используйте `BarrageStore.create(liveID)` для текущей комнаты.
4. **Создание сообщения**: создайте объект Barrage, установите `messageType = BarrageType.TEXT` и установите textContent на ваше сообщение.
5. **Вставка локально**: вызовите `barrageStore.appendLocalTip(message: giftTip)` для добавления сообщения в локальный список барады.

#### Пример кода

```
import io.trtc.tuikit.atomicxcore.api.barrage.Barrageimport io.trtc.tuikit.atomicxcore.api.barrage.BarrageStoreimport io.trtc.tuikit.atomicxcore.api.barrage.BarrageTypeimport io.trtc.tuikit.atomicxcore.api.gift.Giftimport io.trtc.tuikit.atomicxcore.api.gift.GiftListenerimport io.trtc.tuikit.atomicxcore.api.gift.GiftStoreimport io.trtc.tuikit.atomicxcore.api.live.LiveUserInfoclass LiveRoomManager(    private val liveId: String) {    private val giftStore: GiftStore = GiftStore.create(liveId)    private val barrageStore: BarrageStore = BarrageStore.create(liveId)            private val giftListener = object : GiftListener() {        override fun onReceiveGift(liveID: String, gift: Gift, count: Int, sender: LiveUserInfo) {                   val tipText = "${sender.userName} sent ${gift.name} x $count"            val giftTip = Barrage(                liveID = liveID,                messageType = BarrageType.TEXT,                textContent = tipText                sender = sender            )            barrageStore.appendLocalTip(giftTip)        }    }    private fun setupGiftToBarrageFlow() {        giftStore.addGiftListener(giftListener)    }}
```

## Часто задаваемые вопросы

### Список подарков в GiftStore пуст. Что делать?

Вы должны вызвать `refreshUsableGifts(completion)` для получения данных подарков с вашего сервера. Убедитесь, что ваш серверная часть настроена с данными подарков через REST API сервера.

### Как реализовать многоязычное отображение подарков (например, китайский, английский)?

Используйте API `GiftStore.setLanguage(language: String)` перед вызовом `refreshUsableGifts`, передав код целевого языка (например, "en" или "zh-CN"). Сервер вернет названия и описания подарков на указанном языке.

### Я вызвал sendGift, но анимация подарка воспроизводится дважды. Почему?

Событие `onReceiveGift` транслируется всем членам комнаты, включая отправителя. Если вы обновляете пользовательский интерфейс как в обратном вызове завершения sendGift, так и в обработчике события `onReceiveGift`, анимация будет воспроизведена дважды.

- **Лучшая практика**: обновляйте пользовательский интерфейс (например, воспроизведение анимаций, отображение барады) только в обработчике события `onReceiveGift`. Используйте обратный вызов завершения `sendGift` только для обработки ошибок (таких как "Отправка не удалась" или "Недостаточный баланс").

### Где реализована логика вычитания подарков?

Вычитание подарков полностью обрабатывается вашей системой выставления счетов. AtomicXCore интегрируется с вашей системой выставления счетов через обратный вызов на стороне сервера. Когда клиент вызывает sendGift, ваш сервер выполняет вычитание. После возврата результата на серверную часть AtomicX

# Список живых трансляций

`LiveListStore` — это центральный модуль в `AtomicXCore` для управления списками живых комнат и обработки всех аспектов жизненного цикла живой комнаты, включая создание, вход и поддержание состояния. Используйте `LiveListStore` для реализации надежного управления жизненным циклом прямой трансляции в приложении.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e2538748f6d111f09fbf525400370dda.png)

## Основные возможности

- **Получение списка живых комнат**: Получайте текущий список всех открытых живых комнат с поддержкой постраничной загрузки.
- **Управление жизненным циклом прямой трансляции**: Предоставляет полный API процесса для создания, начала трансляции, присоединения, выхода и завершения прямой трансляции.
- **Обновление информации прямой трансляции**: Ведущие могут обновлять открытую информацию в живой комнате в любой момент, такую как обложка и объявления.
- **Мониторинг событий в реальном времени**: Слушайте ключевые события, такие как завершение прямой трансляции и исключение пользователей.
- **Пользовательские данные бизнеса**: Мощная емкость пользовательских метаданных (`MetaData`) позволяет хранить и синхронизировать любую информацию, связанную с бизнесом, в комнате, такую как статус трансляции, информация о музыке и пользовательская роль.

## Основные концепции

Перед началом интеграции ознакомьтесь с несколькими основными концепциями, связанными с `LiveListStore`, в следующей таблице:

| **Основные концепции** | **Тип** | **Основные обязанности и описание** |
| --- | --- | --- |
| [LiveInfo](https://tencent-rtc.github.io/TUIKit_Flutter/api_live_live_list_store/LiveInfo-class.html) | `class` | Представляет модель информации о живой комнате. Содержит ID прямой трансляции (`liveID`), имя (`liveName`), информацию о владельце комнаты (`liveOwner`) и пользовательские метаданные (`metaData`). |
| [LiveListState](https://tencent-rtc.github.io/TUIKit_Flutter/api_live_live_list_store/LiveListState-class.html) | `class` | Представляет текущий статус модуля живого списка. Его основной атрибут `liveList` имеет тип данных `ValueListenable<List<LiveInfo>>` и хранит полученный список прямых трансляций. `currentLive` представляет информацию о живой комнате, в которой находится пользователь. |
| [LiveListListener](https://tencent-rtc.github.io/TUIKit_Flutter/api_live_live_list_store/LiveListListener-class.html) | `class` | Представляет глобальный прослушиватель событий для живой комнаты. Включает два обратных вызова: `onLiveEnded` (завершение прямой трансляции) и `onKickedOutOfLive` (исключение из прямой трансляции), используемых для обработки ключевых обновлений статуса комнаты. |
| [LiveListStore](https://tencent-rtc.github.io/TUIKit_Flutter/api_live_live_list_store/LiveListStore-class.html) | `class` | Это основной класс управления для взаимодействия со списком прямых трансляций и жизненным циклом комнаты. Это глобальный синглтон (`shared`), отвечающий за все операции с живыми комнатами, такие как создание, присоединение и обновление информации. |

## Расширенное использование

### Сценарий 1: Категоризированный список живых комнат

На странице живой площади приложения верхний раздел содержит теги категорий, такие как «Популярное», «Музыка» и «Игры». Когда пользователи нажимают на разные теги, список прямых трансляций ниже динамически фильтруется, чтобы показать только живые комнаты соответствующей категории, помогая пользователям быстро найти интересующий контент.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e254a211f6d111f09af3525400a31896.png)

#### Как это работает

Основная идея — использовать свойство `categoryList` в модели `LiveInfo`. Когда ведущий начинает трансляцию и устанавливает категорию, объект `LiveInfo`, возвращаемый `fetchLiveList`, будет содержать информацию о классификации. После того как приложение получит полный список трансляций, просто выполните простую фильтрацию на клиенте в соответствии с выбранной пользователем категорией, затем обновите пользовательский интерфейс.

#### Пример кода

В следующем примере показано, как создать `LiveListManager` для обработки данных и логики фильтрации.

```
import 'package:flutter/material.dart';import 'package:atomic_x_core/atomicxcore.dart';/// Менеджер данных списка прямых трансляцийclass LiveListManager extends ChangeNotifier {  final _liveListStore = LiveListStore.shared;  List<LiveInfo> _fullLiveList = [];  List<LiveInfo> _filteredLiveList = [];  int? _selectedCategoryId;  List<LiveInfo> get filteredLiveList => _filteredLiveList;  int? get selectedCategoryId => _selectedCategoryId;  late final VoidCallback _liveListChangedListener = _onLiveListChanged;  LiveListManager() {    _liveListStore.liveState.liveList.addListener(_liveListChangedListener);  }  void _onLiveListChanged() {    _fullLiveList = _liveListStore.liveState.liveList.value;    _applyFilter();  }  Future<void> fetchFirstPage() async {    _liveListStore.reset();    await _liveListStore.fetchLiveList(cursor: '', count: 20);  }  /// Фильтровать список прямых трансляций по категории  void filterLiveList({int? categoryId}) {    _selectedCategoryId = categoryId;    _applyFilter();  }  void _applyFilter() {    if (_selectedCategoryId == null) {      // Если categoryId равен null, отобразить полный список      _filteredLiveList = List.from(_fullLiveList);    } else {      // Фильтровать живые комнаты по указанной категории      _filteredLiveList = _fullLiveList.where((liveInfo) {        return liveInfo.categoryList.contains(_selectedCategoryId);      }).toList();    }    notifyListeners();  }  @override  void dispose() {    _liveListStore.liveState.liveList.removeListener(_liveListChangedListener);    super.dispose();  }}
```

### Сценарий 2: Показ информации о продукте в живом магазине

Во время прямой трансляции электронной коммерции, когда ведущий представляет продукт, карточка продукта появляется под видеотрансляцией для всех зрителей, показывая изображение продукта, имя и цену в реальном времени. Когда ведущий переключается на другие продукты, карточка продукта мгновенно синхронизируется на всех клиентах аудитории.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e24959b0f6d111f0b34b525400ecee81.png)

#### Как это работает

На стороне ведущего установите структурированную информацию текущего продукта (предпочтительно в формате JSON) на пользовательский ключ (например, "product_info") с помощью API `updateLiveMetaData`. AtomicXCore мгновенно синхронизирует эти данные со всеми зрителями. На стороне аудитории подпишитесь на `LiveListState.currentLive` и прослушивайте изменения в metaData. Когда product_info обновляется, распарсьте JSON и обновите пользовательский интерфейс карточки продукта.

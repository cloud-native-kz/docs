# Виртуальные подарки

`GiftStore` — это модуль в `AtomicXCore`, специализированный на управлении функцией подарков в комнатах живого потока. С его помощью вы можете построить полную систему подарков для приложения прямого эфира и реализовать различные сценарии монетизации и взаимодействия.

## Основные возможности

- **Получение списка подарков**: получение необходимых данных для панели подарков с сервера, включая категории подарков и детали подарков.
- **Отправка подарка**: аудитория может отправить выбранный подарок якорю с указанием количества.
- **Трансляция событий подарков**: получение событий раздачи подарков в комнате в реальном времени для отображения анимации подарков и уведомления в барраже.

## Основные концепции

Перед началом интеграции ознакомьтесь с несколькими основными концепциями, связанными с `GiftStore`, в следующей таблице:

| **Основные концепции** | **Тип** | **Основные обязанности и описание** |
| --- | --- | --- |
| [Gift](https://tencent-rtc.github.io/TUIKit_Flutter/api_gift_gift_store/Gift-class.html) | `class` | Представляет конкретную модель данных подарка. Содержит ID подарка, название, адрес значка, URL ресурса анимации и цену (монеты). |
| [GiftCategory](https://tencent-rtc.github.io/TUIKit_Flutter/api_gift_gift_store/GiftCategory-class.html) | `class` | Представляет категорию подарка, такую как "популярные" и "люкс". Содержит название категории, а также список **Gift** в этой категории. |
| [GiftState](https://tencent-rtc.github.io/TUIKit_Flutter/api_gift_gift_store/GiftState-class.html) | `class` | Представляет текущий статус модуля подарков. Его основной атрибут `usableGifts` — это `ValueListenable<List<GiftCategory>>`, содержащий полный список подарков, полученный с сервера. |
| [GiftListener](https://tencent-rtc.github.io/TUIKit_Flutter/api_gift_gift_store/GiftListener-class.html) | `class` | Слушатель события подарка, получает событие раздачи подарков через обратный вызов `onReceiveGift`. Когда кто-либо (включая себя) отправляет подарок, все люди в комнате получат это событие. |
| [GiftStore](https://tencent-rtc.github.io/TUIKit_Flutter/api_gift_gift_store/GiftStore-class.html) | `class` | Это основной класс управления для взаимодействия с функцией подарков. Через него можно получить список подарков, отправить подарок и получить события подарков через `addGiftListener`. |

## Этапы реализации

### Этап 1: Интеграция компонента

> **Примечание:** Для использования системы подарков активируйте либо **Free Trial**, либо **Pro** издание. Количество настраиваемых подарков зависит от выбранного пакета. Подробнее см. раздел **Gift System** в [Feature and Billing Description](https://www.tencentcloud.com/document/product/647/59407#658e2423-30d2-45e2-91b8-128b2730b072) и выберите пакет, соответствующий вашим потребностям.

- **Прямой эфир**: обратитесь к [Quick Integration](https://www.tencentcloud.com/document/product/647/77552) для безупречной интеграции с **AtomicXCore**.
- **Голосовая чат-комната**: обратитесь к [Quick Integration](https://www.tencentcloud.com/document/product/647/77561) для безупречной интеграции с **AtomicXCore**.

### Этап 2: Инициализация и прослушивание событий подарков

Получите экземпляр `GiftStore` и установите слушатель для получения событий подарков и обновлений списка.

1. **Получить экземпляр**: используйте `GiftStore.create(liveId)` для получения экземпляра `GiftStore`, привязанного к текущей комнате прямого эфира.
2. **Подписаться на события**: добавьте слушатель события подарка через `giftStore.addGiftListener(listener)`.
3. **Статус подписки**: прослушайте `giftStore.giftState.usableGifts` для получения обновлений списка подарков.

#### Пример кода:

```
import 'package:flutter/foundation.dart';import 'package:atomic_x_core/atomicxcore.dart';class GiftManager {  final String liveId;  late final GiftStore giftStore;  late final GiftListener _giftListener;  late final VoidCallback _giftListChangedListener = _onGiftListChanged;  // Expose gift event callback externally  void Function(String liveID, Gift gift, int count, LiveUserInfo sender)? onReceiveGift;  // Expose the gift list externally  ValueNotifier<List<GiftCategory>> giftListNotifier = ValueNotifier([]);  GiftManager({required this.liveId}) {    // 1. Get the instance of GiftStore by liveId    giftStore = GiftStore.create(liveId);    _subscribeToGiftState();    _subscribeToGiftEvents();  }  /// 2. Subscribe to gift events  void _subscribeToGiftEvents() {    _giftListener = GiftListener(      onReceiveGift: (liveID, gift, count, sender) {        // After receiving the event, forward it to external processing        onReceiveGift?.call(liveID, gift, count, sender);      },    );    giftStore.addGiftListener(_giftListener);  }  /// 3. Subscribe to gift list status  void _subscribeToGiftState() {    giftStore.giftState.usableGifts.addListener(_giftListChangedListener);  }  void _onGiftListChanged() {    giftListNotifier.value = giftStore.giftState.usableGifts.value;  }  // Call the dispose method to reclaim resources when exiting the live streaming room  void dispose() {    giftStore.removeGiftListener(_giftListener);    giftStore.giftState.usableGifts.removeListener(_giftListChangedListener);    giftListNotifier.dispose();  }}
```

### Этап 3: Получение списка подарков

Вызовите метод `refreshUsableGifts` для получения данных подарков с сервера.

1. **Вызов API**: вызовите `giftStore.refreshUsableGifts()` в подходящий момент (например, после входа в комнату прямого эфира).
2. **Обработка результата**: дополнительно обработайте возвращаемые результаты для получения результатов извлечения.
3. **Получение данных**: после обновления список подарков автоматически получается путем прослушивания `giftStore.giftState.usableGifts` на этапе 2.

#### Пример кода:

```
extension GiftManagerFetch on GiftManager {  /// Refresh the gift list from the server  Future<void> fetchGiftList() async {    final result = await giftStore.refreshUsableGifts();    if (result.isSuccess) {      debugPrint("Gift list success message");      // Upon success, the data is auto-updated via usableGifts listen    } else {      debugPrint("Gift list failed to pull: ${result.errorMessage}");    }  }}
```

### Этап 4: Отправка подарка

Когда пользователь выбирает подарок в панели подарков и нажимает отправить, вызовите API `sendGift` для отправки подарка.

1. **Получить параметр**: получите `giftID` выбранный пользователем и количество отправленных сообщений (`count`) из UI.
2. **Вызов API**: вызовите `giftStore.sendGift(giftID:, count:)`.
3. **Обработка результата**: обработайте случаи ошибок (такие как уведомление о недостаточном балансе счета) в возвращаемом результате. Обновления UI (анимация, барраж) после успешной отправки должны быть управляемы событием `onReceiveGift`.

#### Пример кода:

```
extension GiftManagerSend on GiftManager {  /// User sends a gift  Future<void> sendGift(String giftID, int count) async {    final result = await giftStore.sendGift(giftID: giftID, count: count);    if (result.isSuccess) {      debugPrint("Gift $giftID sent successfully");      // After the message is sent successfully, everyone including the sender will receive the onReceiveGift event    } else {      debugPrint("Gift sending failed: ${result.errorMessage}");      // You can prompt user here, such as "Insufficient balance" or "Network error"    }  }}
```

#### Параметры API `SendGift`

| **Имя параметра** | **Тип** | **Описание** |
| --- | --- | --- |
| `giftID` | `String` | Уникальный ID отправляемого подарка. |
| `count` | `int` | Количество отправленных. |

### Конфигурация ресурсов **подарков**

Настройте доступные подарки вашей комнаты прямого эфира, категории, названия, значки, цены и эффекты анимации в соответствии с вашими операционными потребностями и брендингом.

1. **Конфигурация на стороне сервера**: используйте REST API сервера LiveKit для управления информацией о подарках, категориями и многоязычной поддержкой. См. [Gift Configuration Guide](https://www.tencentcloud.com/document/product/647/72844).
2. **Получение на клиенте**: на клиенте вызовите `refreshUsableGifts` для получения данных конфигурации.
3. **Отображение UI**: используйте полученный `List<GiftCategory>` для отображения панели подарков.

### Рабочий процесс выставления счетов и вычитания подарков

Когда аудитория отправляет подарки, убедитесь в достаточном балансе счета и завершите фактическое вычитание перед запуском воспроизведения эффекта подарка и трансляции.

#### Рабочий процесс

1. **Конфигурация бэкэнд-вебхука**: настройте URL обратного вызова вашей системы выставления счетов в бэкэнде LiveKit. См. [Callback Configuration Documentation](https://www.tencentcloud.com/document/product/647/72210).
2. **Запрос клиента**: клиент вызывает `sendGift`.
3. **Проверка бэкэнда**: бэкэнд LiveKit вызывает URL вашего обратного вызова; ваша система выставления счетов обрабатывает вычитание и возвращает результат.
4. **Синхронизация результата**: если успешно, **AtomicXCore** трансляцирует событие `onReceiveGift`; если неудачно, обратный вызов завершения `sendGift` получает ошибку.

### Воспроизведение полноэкранных анимаций подарков

Когда пользователь отправляет "Ракету", "Карнавал" или другой люкс подарок, отобразите полноэкранную анимацию (например, SVGA) для повышения вовлеченности.

#### Этапы реализации

`AtomicXCore` сам по себе не включает плеер анимации подарков. Вам необходимо выбрать и интегрировать стороннюю библиотеку на основе бизнес-требований. Далее приведено сравнение двух решений:

| **Элемент сравнения** | **Базовое решение (Svgaplayer_flutter)** | **Продвинутый план (TCEffectPlayerKit)** |
| --- | --- | --- |
| Выставление счетов | Бесплатно (библиотека с открытым исходным кодом) | Платно (требуется покупка лицензии), см. [Payment Guide](https://trtc.io/document/69949?product=beautyar&menulabel=core%20sdk&platform=android#X-Series-Capabilities) |
| Метод интеграции | Требуется ручная интеграция библиотеки [svgaplayer_flutter](https://pub.dev/packages/svgaplayer_flutter). | Требуется дополнительная интеграция библиотеки [flutter_effect_player](https://github.com/Tencent-RTC/EffectPlayer_Flutter) и аутентификация. |
| Поддержка формата анимации | Только SVGA | SVGA, PAG, WebP, Lottie, MP4 и другие форматы |
| Производительность | Рекомендуется SVGA файлы ≤ 10MB | Поддержка более крупных файлов анимации, лучшая производительность для сложных эффектов, многоанимационная параллелизм и устройства низкого уровня |
| Рекомендуемые сценарии | Формат анимации подарка унифицирован как SVGA с управляемым размером файла | Поддержка нескольких форматов анимации или требования к более высокой производительности/совместимости |

В этом разделе демонстрируется базовое решение. Если вы выбрали продвинутый план, обратитесь к [EffectPlayer Flutter integration guide](https://trtc.io/document/73975?product=beautyar&menulabel=core%20sdk&platform=flutter).

#### Базовое решение: использование Svgaplayer_flutter

1. **Интегрировать svgaplayer_flutter**: в вашем файле `pubspec.yaml` добавьте зависимость и запустите `flutter pub get`.

```
dependencies:  svgaplayer_flutter: ^2.0.0
```

2. **Прослушивание событий подарков**: добавьте слушатель через `addGiftListener` в `GiftStore`.
3. **Анализ и воспроизведение анимации**: при получении события `onReceiveGift` проверьте, является ли `gift.resourceURL` допустимым и указывает на файл SVGA. Если да, используйте `SVGAAnimationController` для воспроизведения анимации.

#### Пример кода:

```
import 'package:flutter/material.dart';import 'package:atomic_x_core/atomicxcore.dart';import 'package:svgaplayer_flutter/svgaplayer_flutter.dart';class LiveRoomWidget extends StatefulWidget {  final String liveId;  const LiveRoomWidget({Key? key, required this.liveId}) : super(key: key);  @override  State<LiveRoomWidget> createState() => _LiveRoomWidgetState();}class _LiveRoomWidgetState extends State<LiveRoomWidget> with SingleTickerProviderStateMixin {  late GiftManager giftManager;  // SVGA player  late SVGAAnimationController _svgaController;  bool _showAnimation = false;  @override  void initState() {    super.initState();    _svgaController = SVGAAnimationController(vsync: this);    _setupGiftSubscription();  }  void _setupGiftSubscription() {    giftManager = GiftManager(liveId: widget.liveId);    giftManager.onReceiveGift = (liveID, gift, count, sender) {      if (gift.resourceURL.isNotEmpty) {        _playAnimation(gift.resourceURL);      }    };  }  Future<void> _playAnimation(String url) async {    try {      final videoItem = await SVGAParser.shared.decodeFromURL(url);      _svgaController.videoItem = videoItem;      setState(() {        _showAnimation = true;      });      _svgaController.forward().whenComplete(() {        setState(() {          _showAnimation = false;        });      });    } catch (e) {      debugPrint("SVGA animation parsing failure: $e");    }  }  @override  void dispose() {    _svgaController.dispose();    giftManager.dispose();    super.dispose();  }  @override  Widget build(BuildContext context) {    return Stack(      children: [        // Live stream        // ...        // Full-Screen Gift Animation        if (_showAnimation)          Positioned.fill(            child: SVGAImage(_svgaController),          ),      ],    );  }}
```

#### Отображение сообщений о подарках в области барража

Когда пользователь отправляет подарок, он не только воспроизводит анимацию, но и отображает системное сообщение в области комментариев на экране, такое как "[Никнейм] отправил [имя подарка] x [количество]", чтобы все зрители могли видеть.

#### Этапы реализации

1. **Прослушивание событий**: добавьте слушатель через `giftStore.addGiftListener`.
2. **Получение информации**: после получения события `onReceiveGift` извлеките отправителя, подарок и количество.
3. **Получить хранилище барража**: используйте `BarrageStore.create(liveId)` для получения экземпляра, привязанного к текущей комнате.
4. **Конкатенация сообщений**: создайте объект `Barrage`, установите `messageType = BarrageType.text` и `textContent` как конкатенированную строку (такую как "[`sender.userName`] отправил [`gift.name`] x [`count`]").
5. **Локальная вставка**: вызовите `barrageStore.appendLocalTip(giftTip)` для вставки сообщения в локальный список.

#### Пример кода:

```
// In LiveRoomManager or similar top-level managersvoid _setupGiftToBarrageFlow() {  final giftStore = GiftStore.create(liveId);  final barrageStore = BarrageStore.create(liveId);  final giftListener = GiftListener(    onReceiveGift: (liveID, gift, count, sender) {      // Concatenate the message      final tipText = "${sender.userName} sent ${gift.name} x $count";      final giftTip = Barrage(        messageType: BarrageType.text,        textContent: tipText,      );      // Optional: Set a special sender      // local insertion      barrageStore.appendLocalTip(giftTip);    },  );  giftStore.addGiftListener(giftListener);}
```

## Часто задаваемые вопросы

### Почему список подарков в GiftStore пуст?

Вы должны активно вызвать `refreshUsableGifts()` для получения данных подарков с вашего бэкэнда. Эти данные подарков должны быть настроены на вашем бэкэнде через REST API на стороне сервера.

### Как показать подарки на нескольких языках (например, китайский, английский)?

`GiftStore` предоставляет API `setLanguage(String language)`. Вы можете вызвать этот метод перед `refreshUsableGifts`, введите код целевого языка (такой как "en" или "zh-CN"). Сервер вернет название и описание подарка на соответствующем языке на основе этого кода языка.

### Почему анимация подарка воспроизводится дважды после отправки подарка?

`onReceiveGift` — это трансляция всем членам в комнате (включая отправителя). Если вы выполните операцию UI в обратном вызове успеха `sendGift` и одновременно выполните одну и ту же операцию UI в обратном вызове `onReceiveGift`, это может привести к дублированию.

- **Лучшая практика**: обновления UI (такие как воспроизведение анимации, уведомление барража) должны обрабатываться только в обратном вызове события `onReceiveGift`. Возвращаемые результаты `sendGift` используются только для обработки логики сбоя отправки (такой как приглашение пользователя "сбой отправки" или "недостаточный баланс").

### Где обрабатывается логика вычитания подарков?

Логика взимания платежей за подарки полностью обрабатывается вашей самостоятельно построенной системой выставления счетов. `AtomicXCore` соединяется с вашей системой выставления счетов через механизм обратного вызова бэкэнда. Клиент вызывает `sendGift` для запуска обратного вызова. После того как ваш бэкэнд-сервис завершит взимание платежей, он возвращает результат в бэкэнд `AtomicXCore`, тем самым определяя, транслировать ли событие подарка.

### На события уведомления о подарках влияют отключение звука или ограничение частоты сообщений?

Нет. Уведомления об отправке подарков (событие `onReceiveGift`) не затронуты отключением звука или контролем частоты сообщений, обеспечивая надежную доставку.

### Что делать, если воспроизведение анимации подарка происходит прерывисто?

Проверьте размер файла SVGA. SDK плеера рекомендует

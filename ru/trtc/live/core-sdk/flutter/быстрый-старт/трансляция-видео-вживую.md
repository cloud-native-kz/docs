# Трансляция видео вживую

`LiveListStore` — это центральный модуль в `AtomicXCore` для управления списками комнат прямой трансляции и обработки всех аспектов жизненного цикла комнаты прямой трансляции, включая создание, вход и поддержание состояния. Используйте `LiveListStore` для реализации надежного управления жизненным циклом трансляции в вашем приложении.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e2538748f6d111f09fbf525400370dda.png)

## Основные возможности

- **Получить список комнат прямой трансляции**: Получить текущий список всех общедоступных комнат прямой трансляции с поддержкой постраничной загрузки.
- **Управление жизненным циклом прямой трансляции**: Предоставляет полный API процесса для создания, перехода в прямой эфир, присоединения, выхода и завершения прямой трансляции.
- **Обновление информации о прямой трансляции**: Хосты могут обновлять общедоступную информацию в комнате прямой трансляции в любое время, например обложку и объявление.
- **Мониторинг событий в режиме реального времени**: Прослушивание ключевых событий, таких как завершение прямой трансляции и исключение пользователей.
- **Пользовательские бизнес-данные**: Мощная емкость пользовательских метаданных (`MetaData`) позволяет вам хранить и синхронизировать любую информацию, связанную с бизнесом, в комнате, такую как статус прямой трансляции, информацию о музыке и пользовательскую роль.

## Основные концепции

Перед началом интеграции давайте изучим несколько основных концепций, связанных с `LiveListStore`, в следующей таблице:

| **Основные концепции** | **Тип** | **Основные обязанности и описание** |
| --- | --- | --- |
| [LiveInfo](https://tencent-rtc.github.io/TUIKit_Flutter/api_live_live_list_store/LiveInfo-class.html) | `class` | Представляет информационную модель комнаты прямой трансляции. Содержит ID прямой трансляции (`liveID`), имя (`liveName`), информацию о владельце комнаты (`liveOwner`) и пользовательские метаданные (`metaData`). |
| [LiveListState](https://tencent-rtc.github.io/TUIKit_Flutter/api_live_live_list_store/LiveListState-class.html) | `class` | Представляет текущее состояние модуля списка прямых трансляций. Его основной атрибут `liveList` — это тип данных `ValueListenable<List<LiveInfo>>`, в котором хранится полученный список потоков прямой трансляции. `currentLive` представляет информацию комнаты прямой трансляции, в которой находится пользователь. |
| [LiveListListener](https://tencent-rtc.github.io/TUIKit_Flutter/api_live_live_list_store/LiveListListener-class.html) | `class` | Представляет глобальный слушатель событий для комнаты прямой трансляции. | [LiveListStore](https://tencent-rtc.github.io/TUIKit_Flutter/api_live_live_list_store/LiveListStore-class.html) | `class` | Это основной класс управления для взаимодействия со списком прямых трансляций и жизненным циклом комнаты. Это глобальный синглтон (`shared`), отвечающий за все операции комнаты прямой трансляции, такие как создание, присоединение и обновление информации. |

## Расширенное использование

### Сценарий 1: Список комнат прямой трансляции по категориям

На странице живого квадрата приложения в верхней части находятся теги категорий, такие как «Популярное», «Музыка» и «Игры». Когда пользователи нажимают на различные теги, список прямых трансляций ниже динамически фильтруется, чтобы отображать только комнаты прямой трансляции соответствующей категории, помогая пользователям быстро идентифицировать интересующий контент.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e254a211f6d111f09af3525400a31896.png)

#### Как это работает

Суть в использовании свойства `categoryList` в модели `LiveInfo`. Когда хост начинает прямую трансляцию и устанавливает категорию, объект `LiveInfo`, возвращаемый `fetchLiveList`, будет содержать эту информацию о классификации. После того как ваше приложение получит полный список прямых трансляций, просто выполните простую фильтрацию на клиенте в соответствии с выбранной пользователем категорией, затем обновите пользовательский интерфейс.

#### Пример кода

Следующий пример показывает, как создать `LiveListManager` для обработки логики данных и фильтрации.

```
import 'package:flutter/material.dart';import 'package:atomic_x_core/atomicxcore.dart';/// Менеджер данных списка прямых трансляцийclass LiveListManager extends ChangeNotifier {  final _liveListStore = LiveListStore.shared;  List<LiveInfo> _fullLiveList = [];  List<LiveInfo> _filteredLiveList = [];  int? _selectedCategoryId;  List<LiveInfo> get filteredLiveList => _filteredLiveList;  int? get selectedCategoryId => _selectedCategoryId;  late final VoidCallback _liveListChangedListener = _onLiveListChanged;  LiveListManager() {    _liveListStore.liveState.liveList.addListener(_liveListChangedListener);  }  void _onLiveListChanged() {    _fullLiveList = _liveListStore.liveState.liveList.value;    _applyFilter();  }  Future<void> fetchFirstPage() async {    _liveListStore.reset();    await _liveListStore.fetchLiveList(cursor: '', count: 20);  }  /// Фильтровать список прямых трансляций по категориям  void filterLiveList({int? categoryId}) {    _selectedCategoryId = categoryId;    _applyFilter();  }  void _applyFilter() {    if (_selectedCategoryId == null) {      // Если categoryId равно null, отобразить полный список      _filteredLiveList = List.from(_fullLiveList);    } else {      // Отфильтровать комнаты прямой трансляции по указанной категории      _filteredLiveList = _fullLiveList.where((liveInfo) {        return liveInfo.categoryList.contains(_selectedCategoryId);      }).toList();    }    notifyListeners();  }  @override  void dispose() {    _liveListStore.liveState.liveList.removeListener(_liveListChangedListener);    super.dispose();  }}
```

### Сценарий 2: Показ информации о продукте в прямой покупке

Во время трансляции электронной коммерции, когда хост представляет продукт, под видео прямой трансляции для всех зрителей появляется карточка продукта, отображающая изображение, название и цену продукта в реальном времени. Когда хост переключается на другой продукт, карточка продукта мгновенно синхронизируется на всех клиентах зрителей.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e24959b0f6d111f0b34b525400ecee81.png)

#### Как это работает

На стороне хоста установите структурированную информацию о текущем продукте (предпочтительно в формате JSON) на пользовательский ключ (например, "product_info") с помощью API `updateLiveMetaData`. AtomicXCore мгновенно синхронизирует эти данные всем зрителям. На стороне зрителя подпишитесь на `LiveListState.currentLive` и слушайте изменения в metaData. Когда product_info обновляется, разберите JSON и обновите пользовательский интерфейс карточки продукта.

# Введение в продукт

**AtomicXCore** — это SDK, разработанный для сценариев, таких как **видеотрансляция в прямом эфире**, **видеоконференции**, **голосовые чат-комнаты** и **звонки**. Он предоставляет подход к интеграции **без интерфейса пользователя**, позволяя вам сосредоточиться исключительно на разработке интерфейса и быстро создавать полнофункциональные, высоко настраиваемые приложения для реального взаимодействия.

## Сценарии использования

**AtomicXCore** инкапсулирует всю бизнес-логику, необходимую для создания широкого спектра сложных сценариев, позволяя вам сосредоточиться исключительно на дизайне пользовательского интерфейса.

## Основные концепции **AtomicXCore**

| **Концепция** | **Определение** | **Пример** |
| --- | --- | --- |
| `Store` (менеджер состояния) | `Store` — это логический и управляющий состояние компонент для модулей сценариев AtomicXCore (таких как вызовы, прямые трансляции, прямой чат). | • `CallListStore`: менеджер вызовов • `LiveListStore`: менеджер списка прямых трансляций • `ChatStore`: менеджер прямого чата |
| `State` (данные состояния) | Снимок данных, управляемых `Store`, обычно представляется как `struct`. Интерфейс автоматически обновляется путём подписки на изменения State. | • `CallParticipantState`: состояние участника вызова • `LiveSeatState`: состояние позиции в прямой комнате |

## Принцип работы **AtomicXCore**

**AtomicXCore** абстрагирует каждую бизнес-функцию как `Store` (менеджер состояния). Следуйте этим основным шагам для интеграции **AtomicXCore**:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/98cf7230cf7a11f0aecd525400454e06.png)

### 1. Получить экземпляр Store (менеджер состояния)

`Store` — это специализированный логический и управляющий состояние компонент для каждого бизнес-модуля (таких как прямой чат, подарки, вызовы). Выполняйте все операции через соответствующий `Store`.

### 2. Подписаться на State

`State` — это снимок текущих данных в `Store`. Ваш интерфейс должен быть подписан на изменения `State` и обновляться автоматически — не изменяйте `State` напрямую.

### 3. Вызвать Action (метод)

Методы `Action` — единственный способ обновить `State`. Когда пользователь взаимодействует с интерфейсом (например, нажимает кнопку лайка), вызовите соответствующий метод `Action` из `Store`. `Store` обработает бизнес-логику (которая может включать сетевые запросы) и обновит свой `State` при успехе. Все подписчики на предыдущий `State` уведомляются автоматически, что запускает обновление интерфейса.

### 4. Подписаться на Event

Уведомления `Event` отличаются от `State`. `State` отражает текущий статус (например, общее количество лайков), в то время как `Event` указывает на то, что только что произошло (например, был получен подарок).

Используйте уведомления `Event` для запуска одноразовых эффектов интерфейса, таких как анимации или всплывающие подсказки.

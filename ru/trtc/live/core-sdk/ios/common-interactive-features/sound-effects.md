# Звуковые эффекты

Этот документ содержит рекомендации по интеграции элементов управления аудиоэффектами в приложение для прямой трансляции с использованием модулей `AudioEffectStore` и `DeviceStore` из фреймворка **AtomicXCore**. Эти модули обеспечивают функции управления громкостью микрофона, мониторинга в наушниках и различные эффекты изменения голоса и реверберации.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b6b61ae4c90111f0b011525400bf7822.png)

## Основные возможности

Комбинируя `AudioEffectStore` и `DeviceStore`, вы можете реализовать следующие основные функции:

- **Регулировка громкости микрофона**: Управление громкостью локального микрофона (захват), которая определяет громкость, которую слышит ваша аудитория.
- **Мониторинг в наушниках**: Позволяет ведущим или соведущим услышать собственный голос в реальном времени через наушники для контроля и коррекции произношения.
- **Эффекты изменения голоса**: Применяйте различные эффекты изменения голоса, такие как "Девочка", "Глубокий голос мужчины", "Озорной ребенок" и другие.
- **Эффекты реверберации**: Имитируйте различные акустические окружения, такие как KTV, металлический, глубокий, яркий и другие.

## Основные понятия

| **Основное понятие** | **Тип** | **Основная ответственность и описание** |
| --- | --- | --- |
| `AudioChangerType` | `enum` | Перечисляет доступные эффекты изменения голоса, такие как "child", "man" и т.д. |
| `AudioReverbType` | `enum` | Перечисляет доступные эффекты реверберации, такие как "ktv", "hall" и т.д. |
| `AudioEffectState` | `struct` | Представляет текущее состояние модуля аудиоэффектов, как правило используется для отрисовки пользовательского интерфейса. Включает состояние эффекта изменения голоса, состояние реверберации, состояние включения мониторинга в наушниках и громкость мониторинга. |
| `AudioEffectStore` | `class` | Синглтон-класс управления данными для аудиоэффектов. Используйте его для вызова API аудиоэффектов. После вызова API соответствующее свойство состояния автоматически обновляется. Подпишитесь на это реактивное состояние для получения и мониторинга обновлений. |
| `DeviceState` | `struct` | Представляет текущее состояние модуля устройства, как правило используется для отрисовки пользовательского интерфейса. Основные свойства включают статус камеры и микрофона. |
| `DeviceStore` | `class` | Синглтон-класс управления данными для операций с устройствами. Используйте его для управления камерой и микрофоном. После вызова API соответствующее свойство состояния автоматически обновляется. Подпишитесь на это реактивное состояние для получения и мониторинга обновлений. |

## Реализация

### Шаг 1: Интеграция компонентов

- **Видеопрямая трансляция**: Обратитесь к [Быстрому старту](https://www.tencentcloud.com/document/product/647/74594) для интеграции **AtomicXCore**.
- **Голосовой чат-комната**: Обратитесь к [Быстрому старту](https://www.tencentcloud.com/document/product/647/74682) для интеграции **AtomicXCore**.

### Шаг 2: Получение экземпляров Store

`AudioEffectStore` и `DeviceStore` — это объекты-синглтоны. Получите доступ к их экземплярам через свойство shared в любом месте вашего проекта. Полный пример реализации см. в файле [AudioEffectView.swift](https://github.com/Tencent-RTC/TUIKit_iOS/blob/main/live/Sources/Component/AudioEffect/AudioEffectView.swift) в проекте демонстрационного интерфейса `TUILiveKit` с открытым исходным кодом.

```
import UIKitimport AtomicXCoreimport Combinelet audioEffectStore = AudioEffectStore.sharedlet deviceStore = DeviceStore.shared
```

### Шаг 3: Реализация мониторинга в наушниках

Мониторинг в наушниках позволяет пользователям слышать звук своего микрофона в реальном времени. Управляйте мониторингом в наушниках, вызывая интерфейсы, описанные ниже.

> **Примечание:** Мониторинг в наушниках обычно требует наушников для правильной работы.

#### Реализация

1. **Включение/отключение мониторинга в наушниках**: Используйте элемент управления `UISwitch` для включения или отключения мониторинга в наушниках.
2. **Установка громкости монитора в наушниках**: Используйте элемент управления `UISlider` для регулировки громкости. Сопоставьте значение `UISlider` с диапазоном `[0, 150]` и вызовите `setVoiceEarMonitorVolume(_:)`.
3. **Подписка на обновления состояния**: Прослушивайте изменения состояния монитора в наушниках в реальном времени для обновления интерфейса.

#### Пример кода

```
// 1. Включить мониторинг в наушникахaudioEffectStore.setVoiceEarMonitorEnable(enable: true)// 2. Отрегулировать громкость монитора в наушникахaudioEffectStore.setVoiceEarMonitorVolume(volume: 80)// 3. Отключить мониторинг в наушникахaudioEffectStore.setVoiceEarMonitorEnable(enable: false)// 4. Прослушивать изменения состояния audioEffectStore.state    .subscribe(StatePublisherSelector(keyPath: \\AudioEffectState.earMonitorVolume))    .removeDuplicates()    .sink { currentVolume in        print("Ear monitor volume: \\(currentVolume)")        valueLabel.text = "\\(currentVolume)" // Обновить UILabel для отображения значения    }    .store(in: &cancellables)
```

#### Параметры setVoiceEarMonitorEnable

| **Имя параметра** | **Тип** | **Описание** |
| --- | --- | --- |
| `enable` | `Bool` | Включить ли мониторинг в наушниках.- true: включить- false: отключить |

#### Параметры setVoiceEarMonitorVolume

| **Имя параметра** | **Тип** | **Описание** |
| --- | --- | --- |
| `volume` | `Int` | Громкость монитора в наушниках.- Диапазон значений: [0, 150]- По умолчанию: 100 |

### Шаг 4: Реализация регулировки громкости микрофона

Для регулировки громкости микрофона (захват) вызовите метод `setCaptureVolume(volume:)` DeviceStore с желаемым значением.

#### Реализация

1. **Установка громкости захвата**: Используйте элемент управления `UISlider` для регулировки громкости. Сопоставьте значение `UISlider` с диапазоном `[0, 150]` и вызовите `setCaptureVolume(volume:)`.
2. **Подписка на обновления состояния**: Прослушивайте изменения состояния громкости в реальном времени для обновления интерфейса.

#### Пример кода

```
// 1. Отрегулировать громкость захватаdeviceStore.setCaptureVolume(volume: 80)// 2. Прослушивать изменения состояния deviceStore.state    .subscribe(StatePublisherSelector(keyPath: \\DeviceState.captureVolume))    .removeDuplicates()    .sink { captureVolume in        print("Capture volume: \\(captureVolume)")        valueLabel.text = "\\(captureVolume)" // Обновить UILabel для отображения значения    }    .store(in: &cancellables)
```

#### Параметры setCaptureVolume

| **Имя параметра** | **Тип** | **Описание** |
| --- | --- | --- |
| `volume` | `Int` | Громкость микрофона (захват).- Диапазон значений: [0, 150]- По умолчанию: 100 |

### Шаг 5: Реализация эффектов изменения голоса

Чтобы применить эффект изменения голоса, вызовите метод `setAudioChangerType(type:)` и передайте желаемое значение перечисления.

#### Пример кода

```
// Установить эффект изменения голоса на "child"audioEffectStore.setAudioChangerType(type: .child)// Отключить эффект изменения голосаaudioEffectStore.setAudioChangerType(type: .none)
```

#### Параметры setAudioChangerType

| **Имя параметра** | **Тип** | **Описание** |
| --- | --- | --- |
| `type` | `AudioChangerType` | Перечисление эффектов изменения голоса. |

### Шаг 6: Реализация эффектов реверберации

Чтобы применить эффект реверберации, вызовите метод `setAudioReverbType(type:) ` и передайте желаемое значение перечисления.

```
// Установить эффект реверберации на "KTV"audioEffectStore.setAudioReverbType(type: .ktv)// Отключить эффект реверберацииaudioEffectStore.setAudioReverbType(type: .none)
```

#### Параметры setAudioReverbType

| **Имя параметра** | **Тип** | **Описание** |
| --- | --- | --- |
| `type` | `AudioReverbType` | Перечисление эффектов реверберации. |

### Шаг 7: Сброс параметров аудиоэффектов

> **Примечание:** AudioEffectStore и DeviceStore — это синглтоны, поэтому параметры аудиоэффектов и устройств применяются глобально. Чтобы избежать того, чтобы параметры из одной прямой трансляции влияли на другие, сбросьте параметры аудиоэффектов и устройств при завершении текущей прямой трансляции.

Сбросьте параметры аудиоэффектов в следующих сценариях:

1. При завершении текущей прямой трансляции.
2. Для предоставления функции "восстановление одним нажатием" вызовите метод `reset()` store.

```
class YourMainViewController: UIViewController {        func reset() {        AudioEffectStore.shared.reset()        DeviceStore.shared.reset()    }}
```

## Документация API

Для получения подробной информации о всех публичных интерфейсах и свойствах [AudioEffectStore](https://tencent-rtc.github.io/TUIKit_iOS/documentation/atomicxcore/audioeffectstore), [DeviceStore](https://tencent-rtc.github.io/TUIKit_iOS/documentation/atomicxcore/devicestore) и связанных классов обратитесь к официальной документации API фреймворка [AtomicXCore](https://tencent-rtc.github.io/TUIKit_iOS/documentation/atomicxcore).

| **Store/Компонент** | **Описание** | **Справочник API** |
| --- | --- | --- |
| DeviceStore | Управление аудио/видео устройствами: микрофон (включение/отключение, громкость), камера (включение/отключение, переключение, качество), совместное использование экрана и мониторинг статуса устройства в реальном времени. | [Документация API](https://tencent-rtc.github.io/TUIKit_iOS/documentation/atomicxcore/devicestore) |

## Часто задаваемые вопросы

### Есть ли какие-либо ограничения на вызов API аудиоэффектов и устройств?

Нет. Параметры `AudioEffectStore` и `DeviceStore` действуют глобально. Вы можете вызывать соответствующие API (такие как установка эффектов изменения голоса, реверберации или мониторинга в наушниках) в любой момент до или после входа в прямую трансляцию. Параметры вступают в силу немедленно и сохраняются.

### В чем разница между "громкостью микрофона" и "громкостью монитора в наушниках"?

Это распространенный источник путаницы:

- **Громкость микрофона (громкость захвата)**: Устанавливается через `DeviceStore.setCaptureVolume()`. Управляет громкостью, которую слышит ваша аудитория.
- **Громкость монитора в наушниках**: Устанавливается через `AudioEffectStore.setVoiceEarMonitorVolume()`. Определяет только уровень громкости, который вы слышите в своем мониторе в наушниках, и не влияет на аудиторию.

### Почему в только что созданной прямой трансляции уже есть аудиоэффекты, или почему громкость слишком высокая или слишком низкая?

Потому что `AudioEffectStore` и `DeviceStore` — это синглтоны, параметры аудиоэффектов и устройств применяются глобально. Если вы ранее установили аудиоэффекты и не сбросили их, эти параметры сохраняются. Вызовите метод `reset()` в подходящий момент, чтобы их очистить.

### Почему я не слышу свой голос после включения "Мониторинга в наушниках"?

Проверьте следующее:

1. **Подключены ли наушники?** Мониторинг в наушниках обычно требует наушников. Убедитесь, что проводные или беспроводные наушники подключены.
2. **Установлена ли громкость монитора в наушниках на 0?** Убедитесь, что громкость монитора в наушниках выше 0.
3. **Включен ли микрофон?** Подтвердите, что микрофон включен в DeviceStore (т.е. вызван openLocalMicrophone).

### Можно ли включить эффекты изменения голоса и реверберацию одновременно?

Да. `AudioChangerType` и `AudioReverbType` можно включать одновременно. Например, вы можете одновременно установить как `AudioChangerType.LITTLE_GIRL`, так и `AudioReverbType.KTV`.

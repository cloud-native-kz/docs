# Управление аудиторией

`LiveAudienceStore` — это модуль в `AtomicXCore`, предназначенный для управления информацией об аудитории в комнатах прямых трансляций. С помощью `LiveAudienceStore` вы можете реализовать комплексную систему списка и управления аудиторией для приложения прямой трансляции.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/9013fb14c6b611f087ad52540099c741.png)

## Основные возможности

- **Список аудитории в реальном времени**: получение и отображение списка пользователей, находящихся в комнате.
- **Статистика аудитории**: получение числа зрителей в реальном времени.
- **Мониторинг событий**: прослушивание событий входа и выхода зрителей из комнаты.
- **Управление ролями**: хост может предоставить или отозвать привилегии администратора зрителям.
- **Управление комнатой**: хост или администратор может удалить конкретного зрителя из комнаты трансляции.

## Основные концепции

| **Основная концепция** | **Тип** | **Основные обязанности и описание** |
| --- | --- | --- |
| `LiveUserInfo` | `class` | Модель данных, представляющая зрителя. Содержит основную информацию профиля, такую как userID, userName и avatarURL. |
| `LiveAudienceState` | `struct` | Контейнер состояния модуля аудитории. Предоставляет `audienceList` (StateFlow) для списка пользователей в реальном времени и `audienceCount` для подсчета зрителей. |
| `LiveAudienceEvent` | `enum` | Слушатель событий для изменений аудитории. Предоставляет обратные вызовы, такие как `.onAudienceJoined` и `.onAudienceLeft`, для обработки добавочного обновления списка. |
| `LiveAudienceStore` | `class` | Основной интерфейс для управления аудиторией. Используется для получения снимков списка, выполнения операций администратора (например, удаления пользователей) и подписки на события в реальном времени. |

### Шаг 2: инициализация и получение списка аудитории

Получите экземпляр `LiveAudienceStore`, связанный с `liveId`, и получите начальный снимок аудитории.

```
import Foundation
import AtomicXCore
import Combine

class AudienceManager {
    private let liveId: String
    private let audienceStore: LiveAudienceStore
    private var cancellables = Set<AnyCancellable>()
    
    // Expose the [full] audience list publisher for UI layer subscription
    let audienceListPublisher = CurrentValueSubject<[LiveUserInfo], Never>([])
    // Expose the audience count publisher
    let audienceCountPublisher = PassthroughSubject<UInt, Never>()
    
    init(liveId: String) {
        self.liveId = liveId
        // 1. Get the LiveAudienceStore instance by liveId
        self.audienceStore = LiveAudienceStore.create(liveID: liveId)
        
        // 2. Subscribe to state and events
        subscribeToAudienceState()
        subscribeToAudienceEvents()
        
        // 3. Proactively fetch initial data for the first screen
        fetchInitialAudienceList()
    }
    
    /// Proactively fetch a snapshot of the audience list
    func fetchInitialAudienceList() {
        audienceStore.fetchAudienceList { [weak self] result in
            switch result {
            case .success:
                print("Successfully fetched the audience list for the first time")
                // On success, data will be automatically updated via the state subscription channel below
            case .failure(let error):
                print("Failed to fetch the audience list for the first time: \(error.localizedDescription)")
            }
        }
    }
    
    // ... subsequent code
}
```

### Шаг 3: подписка на состояние и события

Наблюдайте за состоянием `audienceStore` и прослушивайте события для получения снимков списка и обновлений в реальном времени при присоединении/выходе.

```
extension AudienceManager {
    /// Subscribe to state to get audience count and list snapshot
    private func subscribeToAudienceState() {
        audienceStore.state
            .subscribe()
            .receive(on: DispatchQueue.main)
            .sink { [weak self] state in
                // state.audienceList is a full snapshot
                self?.audienceListPublisher.send(state.audienceList)
                // state.audienceCount is the real-time total
                self?.audienceCountPublisher.send(state.audienceCount)
            }
            .store(in: &cancellables)
    }
    
    /// Subscribe to events to handle real-time audience join/leave
    private func subscribeToAudienceEvents() {
        audienceStore.liveAudienceEventPublisher
            .receive(on: DispatchQueue.main)
            .sink { [weak self] event in
                guard let self = self else { return }
                
                var currentList = self.audienceListPublisher.value
                
                switch event {
                case .onAudienceJoined(let newAudience):
                    print("Audience \(newAudience.userName) joined the live room")
                    // Incremental update: add the new audience member to the end of the current list
                    if !currentList.contains(where: { $0.userID == newAudience.userID }) {
                        currentList.append(newAudience)
                        self.audienceListPublisher.send(currentList)
                    }
                
                case .onAudienceLeft(let departedAudience):
                    print("Audience \(departedAudience.userName) left the live room")
                    // Incremental update: remove the departed audience member from the current list
                    currentList.removeAll { $0.userID == departedAudience.userID }
                    self.audienceListPublisher.send(currentList)
                }
            }
            .store(in: &cancellables)
    }
}
```

### Шаг 4: управление аудиторией

В качестве хоста или администратора вы можете управлять зрителями в комнате трансляции.

#### 4.1 удаление зрителя из комнаты трансляции

Вызовите API `kickUserOutOfRoom`, чтобы удалить указанного пользователя из комнаты трансляции.

```
extension AudienceManager {
    func kick(userId: String) {
        audienceStore.kickUserOutOfRoom(userID: userId) { result in
            switch result {
            case .success:
                print("Successfully kicked user \(userId) out of the room")
                // After a successful kick, you will receive an onAudienceLeft event
            case .failure(let error):
                print("Failed to kick user \(userId) out: \(error.localizedDescription)")
            }
        }
    }
}
```

#### 4.2 управление ролями администратора

Используйте API `setAdministrator` и `revokeAdministrator` для предоставления или отозвания привилегий администратора пользователю.

```
extension AudienceManager {
    /// Promote user to administrator
    func promoteToAdmin(userId: String) {
        audienceStore.setAdministrator(userID: userId) { result in
            if case .success = result {
                print("Successfully promoted user \(userId) to administrator")
            }
        }
    }
    
    /// Revoke user's administrator status
    func revokeAdmin(userId: String) {
        audienceStore.revokeAdministrator(userID: userId) { result in
            if case .success = result {
                print("Successfully revoked administrator status for user \(userId)")
            }
        }
    }
}
```

## Расширенные функции

### отображение приветственного сообщения при входе

Когда новый участник аудитории входит в комнату трансляции, приветственное сообщение автоматически отображается локально в области барража/чата, например: `"Добро пожаловать [Имя пользователя] в комнату трансляции"`.

#### реализация

1. Прослушивание событий присоединения: наблюдайте обратный вызов `LiveAudienceEvent.onAudienceJoined` в `LiveAudienceStore` для получения уведомлений в реальном времени при входе нового пользователя.
2. Вставка локального сообщения: при срабатывании события извлеките никнейм пользователя и сразу же вызовите API `appendLocalTip` в BarrageStore для вставки системного сообщения локально.

```
import Foundation
import AtomicXCore
import Combine

class LiveRoomManager {
    private let liveId: String
    private let audienceManager: AudienceManager
    private let barrageManager: BarrageManager // Assume you have implemented BarrageManager as described in the barrage documentation
    private var cancellables = Set<AnyCancellable>()
    
    init(liveId: String) {
        self.liveId = liveId
        // Initialize two core managers
        self.audienceManager = AudienceManager(liveId: liveId)
        self.barrageManager = BarrageManager(liveId: liveId)
        // Set up linkage
        setupWelcomeMessageFlow()
    }
    
    private func setupWelcomeMessageFlow() {
        // 1. Get the LiveAudienceStore instance
        let audienceStore = LiveAudienceStore.create(liveID: self.liveId)
        // 2. Get the BarrageStore instance (thanks to internal mechanisms, this will be the same instance)
        let barrageStore = BarrageStore.create(liveID: self.liveId)
        // 3. Subscribe to audience events
        audienceStore.liveAudienceEventPublisher
            .receive(on: DispatchQueue.main)
            .sink { event in
                // 4. Only handle audience join events
                guard case .onAudienceJoined(let newAudience) = event else {
                    return
                }
                // 5. Create a local tip message
                var welcomeTip = Barrage()
                welcomeTip.messageType = .text
                welcomeTip.textContent = "Welcome \(newAudience.userName) to the live room!"
                // 6. Call BarrageStore's API to insert it into the barrage list
                barrageStore.appendLocalTip(message: welcomeTip)
            }
            .store(in: &cancellables)
    }
}
```

## Часто задаваемые вопросы

### как обновляется количество активной аудитории (audienceCount) в LiveAudienceState? каковы время и частота обновления?

`audienceCount` является высокоточной оценкой. Обновления происходят на основе конкретных триггеров и подлежат контролю частоты системой:

- **активный вход/выход (немедленно)**: когда пользователь явно входит или выходит из комнаты, счет обновляется немедленно в `LiveAudienceState`.
- **Непредвиденное отключение (с задержкой)**: если пользователь отключается ненормально (например, сбой приложения, потеря сетевого соединения), система использует механизм проверки жизненных признаков для проверки статуса. пользователь помечается как неактивный только после 90 секунд неактивности, что запускает отложенное обновление счета.
- **контроль частоты и ограничение скорости**: обновления счета аудитории передаются как сообщения в комнате. система принудительно ограничивает лимит в 40 сообщений в секунду на комнату.

> **примечание**: во время высокотрафичных всплесков (например, потопы в чате или быстрые отправки подарков), если скорость сообщений превышает этот порог, система приоритизирует основные сообщения (например, чат/барраж). следовательно, сообщения об обновлении счета аудитории могут быть отброшены для поддержания стабильности.

- **Что это означает для разработчиков?**

Рассматривайте `audienceCount` как метрику только для целей отображения интерфейса. Не полагайтесь на это для выставления счетов, критической статистики или сценариев, требующих 100% точности, так как во время событий с высокой параллельностью могут возникнуть краткие задержки или потеря данных.

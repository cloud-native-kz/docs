# Прямые сражения

`AtomicXCore` предоставляет два основных модуля: `CoHostStore` для **кросс-румовых соединений** и `BattleStore` для **PK сражений**. В этом руководстве объясняется, как интегрировать эти модули для включения функций соединения и PK в приложение голосовой комнаты.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/281d42a3c9fd11f09e745254007c27c5.png)

## Основные концепции

| **Ключевое понятие** | **Тип** | **Основные обязанности и описание** |
| --- | --- | --- |
| **CoHostStore** | `class` | Управляет жизненным циклом совместного хостинга между стримерами, предоставляя API для совместного хостинга (requestHostConnection, acceptHostConnection). |
| **BattleStore** | `class` | Обрабатывает сигнализацию и управление состоянием для кросс-румовых PK сражений, включая статус PK. |
| **LiveListStore** | `class` | Получает список активных комнат (`fetchLiveList`), позволяя обнаруживать другие доступные комнаты для совместного хостинга. |
| **LiveSeatStore** | `class` | Управляет статусом всех мест в комнате (включая кросс-румовые места при совместном хостинге), отслеживает присоединение/отключение пользователей к местам и статус микрофона. |
| **CoHostLayoutTemplate** | `enum` | Определяет шаблоны расположения мест для совместного хостинга, например `.hostStaticVoice6v6`. |
| **BattleConfig** | `struct` | Настраивает детали PK, включая продолжительность PK (`duration`) и требуется ли ответ (`needResponse`). |

## Предварительные условия

Интегрируйте [Быстрый старт](https://www.tencentcloud.com/document/product/647/74682) и убедитесь, что вы реализовали как функции **стример начинает трансляцию**, так и **аудитория смотрит**.

## Стример инициирует сеанс совместного хостинга

### Шаг 1: Получение активных стримеров

Для инициирования совместного хостинга или PK получите текущий список активных комнат и выберите целевого стримера.

**Реализация:** Вызовите `LiveListStore.shared.fetchLiveList`.

> **Примечание:** Если нужная вам комната не отображается в списке, проверьте, что поле isPublicVisible в TUILiveInfo было установлено в true при запуске трансляции. Этот API поддерживает постраничное разбиение. Используйте курсор из колбэка `.success` для последующих запросов. Если возвращаемый курсор пуст, все комнаты были получены. Для оптимальной производительности установите count в 20.

```
import AtomicXCorefunc fetchLiveListForCoHost() {    var cursor = "" // Initial fetch, cursor is empty    let count = 20  // Fetch 20 items per request    let liveListStore = LiveListStore.shared    liveListStore.fetchLiveList(cursor: cursor, count: count, completion: { [weak self] result in        guard let self = self else { return }        switch result {            case .success:                print("Live room list fetched successfully")                // The live room list is updated in LiveListStore.shared.state.value.liveList.                // Update the co-host invitation panel with the recommended users.                // inviteAdapter.submitList(LiveListStore.shared.state.value.liveList)            case .failure(let error):                print("Failed to fetch live room list: \\(error.code), \\(error.message)")        }    })}
```

### Шаг 2: Отправка приглашения на совместный хостинг

После выбора целевого стримера используйте `CoHostStore.requestHostConnection` для отправки приглашения на совместный хостинг.

> **Примечание:** AtomicXCore поддерживает до 9 комнат для кросс-румового совместного хостинга, с 50 пользователями на комнату. API поддерживает массовые приглашения. Чтобы проверить, было ли приглашение в одну комнату успешным, необходимо проверить, является ли `Result` `.success` в колбэке completion. API принимает поля расширения. После отправки приглашения это поле доставляется целевому стримеру через колбэк `onCoHostRequestReceived`. Используйте его для передачи пользовательских данных бизнеса, таких как "начать PK немедленно".

```
import AtomicXCorelet targetHostLiveID = "target_host_room_id"let layoutTemplate = .hostStaticVoice6v6let timeout: TimeInterval = 10 // Invitation timeout (seconds)let extraInfo: String = "" // Custom business info, e.g., for direct PKcoHostStore.requestHostConnection(    targetHost: targetHostLiveID,    layoutTemplate: layoutTemplate,    timeout: timeout,    extraInfo: extraInfo,    completion: { [weak self] result in        guard let self = self else { return }        switch result {            case .success():                print("Invitation sent successfully")            case .failure(let error):                print("Failed to send invitation: \\(error.code), \\(error.message)")        }    })
```

### Шаг 3: Целевой стример получает приглашение

После отправки приглашения целевой стример получает событие `onCoHostRequestReceived` из `CoHostEventPublisher`. Прослушивайте это событие для отображения UI подсказки, например "Стример приглашает вас присоединиться к совместному хостингу".

```
var coHostStore: CoHostStore {    return CoHostStore.create(liveID: liveID)}coHostStore.coHostEventPublisher    .receive(on: RunLoop.main)    .sink { [weak self] event in        guard let self = self else { return }        switch event {            case .onCoHostRequestReceived(let inviter, let extensionInfo):                // Handle co-hosting invitation            default:                break        }    }.store(in: &cancellableSet)
```

### Шаг 4: Целевой стример принимает или отклоняет приглашение

На UI целевой стример может принять или отклонить приглашение, используя `acceptHostConnection` или `rejectHostConnection`.

```
import AtomicXCoreimport Combineprivate var cancellableSet: Set<AnyCancellable> = []var coHostStore: CoHostStore {    return CoHostStore.create(liveID: liveID)}coHostStore.coHostEventPublisher    .receive(on: RunLoop.main)    .sink { [weak self] event in        guard let self = self else { return }        switch event {            case .onCoHostRequestReceived(let inviter, let extensionInfo):                // Accept co-hosting request                coHostStore.acceptHostConnection(fromHostLiveID: inviter.liveID) { [weak self] result in                    guard let self = self else { return }                    switch result {                        case .success():                            // Accepted                        case .failure(let error):                            // Handle error                    }                }                // To reject:                // coHostStore.rejectHostConnection(fromHostLiveID: inviter.liveID) { ... }            default:                break        }    }.store(in: &cancellableSet)
```

### Шаг 5: Отрисовка UI макета совместного хостинга

После установления совместного хостинга прослушивайте `CoHostStore.coHostState.connected` и `LiveSeatStore.liveSeatState.seatList` для динамического обновления расположения мест.

```
import AtomicXCoreimport Combineprivate var cancellableSet: Set<AnyCancellable> = []var liveSeatStore: LiveSeatStore {    return LiveSeatStore.create(liveID: liveID)}// Listen for co-hosting statecoHostStore.state.subscribe(StatePublisherSelector(keyPath: \\CoHostState.connected))    .receive(on: RunLoop.main)    .dropFirst()    .sink { [weak self] connected in        guard let self = self else { return }        // If connected > 0, host is in co-hosting state    }    .store(in: &cancellableSet)// Listen for seat info to refresh avatars, mute status, etc.liveSeatStore.state.subscribe(StatePublisherSelector(keyPath: \\LiveSeatState.seatList))    .removeDuplicates()    .receive(on: RunLoop.main)    .sink { [weak self] seatList in        guard let self = self else { return }        // Use seatInfo.userInfo.liveID to distinguish local/remote users        // Use seatInfo.userInfo.microphoneStatus for mute icon    }    .store(in: &cancellableSet)
```

### Шаг 6: Приглашение дополнительных стримеров при совместном хостинге

Вы можете приглашать больше стримеров присоединиться к совместному хостингу в любой момент, вызвав `requestHostConnection` снова.

```
import AtomicXCorelet targetHostLiveID = "target_host_room_id"let layoutTemplate = .hostStaticVoice6v6let timeout: TimeInterval = 10let extraInfo: String = ""coHostStore.requestHostConnection(    targetHost: targetHostLiveID,    layoutTemplate: layoutTemplate,    timeout: timeout,    extraInfo: extraInfo,    completion: { [weak self] result in        guard let self = self else { return }        switch result {            case .success():                print("Invitation sent successfully")            case .failure(let error):                print("Failed to send invitation: \\(error.code), \\(error.message)")        }    })
```

### Шаг 7: Выход из совместного хостинга

Чтобы выйти из совместного хостинга, вызовите `exitHostConnection`. Другие стримеры в сеансе получат обновленную информацию о местах и должны изменить свои макеты.

```
func onExitButtonClicked() {    coHostStore.exitHostConnection() { [weak self] result in        guard let self = self else { return }        switch result {            case .success():                // Exited co-hosting            case .failure(let error):                // Handle error        }    }}
```

## Стример инициирует PK сражение

Стримеры могут начать PK сражения, используя два основных потока, в зависимости от состояния комнаты до начала PK:

| **Режим PK** | **Состояние комнаты (до PK)** | **Состояние комнаты (после PK)** |
| --- | --- | --- |
| **Сценарий 1: PK после совместного хостинга** | Обе стороны **уже совместно хостят** | Возврат в состояние совместного хостинга |
| **Сценарий 2: Совместный хостинг с режимом PK** | Обе стороны **не совместно хостят** | Возврат в состояние совместного хостинга |

### Сценарий 1: PK после совместного хостинга

**AtomicXCore** позволяет проводить синхронизированные PK сражения между стримерами на совместном хостинге. Типичный поток:

1. Отправьте приглашение на совместный хостинг; целевой стример принимает.
2. Стримеры на совместном хостинге нажимают `PK` для начала синхронизированного сражения. Победитель определяется по полученным подаркам.
3. Целевой стример получает приглашение PK и может принять или отклонить.
4. При принятии начинается режим PK, и отображается полоса прогресса с метриками (подарки, лайки и т.д.).
5. Когда время PK заканчивается, отображаются результаты и может последовать фаза штрафа.
6. После PK сеанс возвращается в состояние совместного хостинга.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/27f2a816c9fd11f0a7775254005ef0f7.png)

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/27feada9c9fd11f0a93d52540044a08e.png)

#### Шаг 1: Инициирование приглашения PK

При совместном хостинге вызовите `BattleStore.requestBattle` для отправки приглашения PK. Настройте продолжительность PK, требуется ли принятие и любую дополнительную информацию.

> **Примечание:** Если продолжительность не установлена, PK по умолчанию составляет 5 минут. Чтобы пропустить принятие целевой стороной, установите needResponse в false в BattleConfig. PK начинается немедленно. Если needResponse равен false, установите timeout в 0. Если установлен timeout и не принято вовремя, как инициатор, так и приглашенный получают onBattleRequestTimeout. Стримеры на совместном хостинге могут инициировать еще один PK.

```
var battleStore: BattleStore {    return BattleStore.create(liveID: liveID)}let targetUserID = ""let config = BattleConfig(    duration: 30, // PK duration in seconds    needResponse: true, // Target must accept    extensionInfo: "")battleStore.requestBattle(config: config, userIDList: [targetUserID], timeout: 10) { [weak self] result in    guard let self = self else { return }    switch result {        case .success(let (battleInfo, _)):            // PK started        case .failure(_):            // Handle error    }}
```

#### Шаг 2: Целевой стример получает приглашение PK

Целевой стример получает событие `onBattleRequestReceived` из `BattleEventPublisher`. Используйте это событие для отправки приглашения целевому пользователю в UI.

```
battleStore.battleEventPublisher    .receive(on: RunLoop.main)    .sink { [weak self] event in        guard let self = self else { return }        switch event {            case .onBattleRequestReceived(let battleID, let inviter, let invitee):                // Handle PK invitation            default:                break        }    }    .store(in: &cancellableSet)
```

#### Шаг 3: Вход в состояние PK

При получении события `onBattleStarted` отобразите полосу прогресса PK в UI.

- Если `needResponse` равен `false`, PK начинается автоматически после отправки приглашения.
- Если `needResponse` равен `true`, PK начинается только после того, как все целевые пользователи вызовут `acceptBattle`.
- Когда целевой пользователь принимает, все участники PK получают `onUserJoinBattle`.
- Когда начинается PK, все участники получают `onBattleStarted`.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/27ff4d76c9fd11f0b011525400bf7822.png)

#### Шаг 4: Завершение состояния PK

PK может закончиться двумя способами:

1. **Истечение времени PK:** Когда настроенная продолжительность заканчивается, все участники получают `onBattleEnded`.
2. **Ручной выход:** Любой участник может вызвать `exitBattle` для выхода из PK. Остальные пользователи получают `onUserExitBattle`. Если все участники выйдут до истечения времени, PK завершится раньше.

```
var battleStore: BattleStore {    return BattleStore.create(liveID: liveID)}guard let battleID = battleStore.state.value.currentBattleInfo?.battleID else { return }battleStore.exitBattle(battleID: battleID, completion: { [weak self] result in    guard let self = self else { return }    switch result {        case .success():            // Exited PK        case .failure(let error):            // Handle error    }})
```

После завершения PK (`onBattleEnded`) отобразите результаты и определите победителя/проигравшего (см. раздел Определение победителя).

> **Примечание:** После завершения PK сеанс возвращается в состояние кросс-румового совместного хостинга. Для завершения совместного хостинга после PK вызовите `exitHostConnection`.

### Сценарий 2: Совместный хостинг с режимом PK

**AtomicXCore** поддерживает прямое инициирование PK во время прямой трансляции без предварительного совместного хостинга. После завершения PK сеанс возвращается в состояние прямой комнаты.

1. Стример нажимает `PK`, открывает панель с активными стримерами (см. Шаг 1 выше).
2. Выбирает стримера и отправляет приглашение PK.
3. Целевой стример получает приглашение и может принять или отклонить.
4. При принятии PK начинается автоматически. Очки накапливаются через подарки или лайки.
5. После завершения PK победитель определяется по очкам.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/280714c7c9fd11f0aedb525400454e06.png)

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/280558e0c9fd11f08942525400e889b2.png)

****

> **Примечание:** Этот поток требует как CoHostStore, так и BattleStore, с `BattleConfig.needResponse` установленным в `false`.

#### Шаг 1: Отправка запроса на совместный хостинг

Аналогично предыдущим шагам, но установите поле `extraInfo` в `requestHostConnection` для указания, что PK должен следовать совместному хостингу. Это позволяет использовать пользовательские UI приглашения для целевого стримера.

Например:

- `{"withPK:true"}` подсказывает "Стример приглашает вас на PK".
- `{"withPK:false"}` подсказывает "Стример приглашает вас присоединиться к кросс-румовому совместному хостингу".

```
import AtomicXCorelet targetHostLiveID = "target_host_room_id"let layoutTemplate = .hostStaticVoice6v6let timeout: TimeInterval = 10let extraInfo: String = "" // e.g., {"withPK:true"}coHostStore.requestHostConnection(    targetHost: targetHostLiveID,    layoutTemplate: layoutTemplate,    timeout: timeout,    extraInfo: extraInfo,    completion: { [weak self] result in        guard let self = self else { return }        switch result {            case .success():                print("Invitation sent successfully")            case .failure(let error):                print("Failed to send invitation: \\(error.code), \\(error.message)")        }    })
```

#### Шаг 2: Целевой стример получает запрос и подтверждает

Целевой стример получает `onCoHostRequestReceived` и может принять или отклонить приглашение.

```
var coHostStore: CoHostStore {    return CoHostStore.create(liveID: liveID)}coHostStore.coHostEventPublisher    .receive(on: RunLoop.main)    .sink { [weak self] event in        guard let self = self else { return }        switch event {            case .onCoHostRequestReceived(let inviter, let extensionInfo):                // Accept co-hosting request                coHostStore.acceptHostConnection(fromHostLiveID: inviter.liveID) { [weak self] result in                    guard let self = self else { return }                    switch result {                        case .success():                            // Accepted                        case .failure(let error):                            // Handle error                    }                }                // To reject:                // coHostStore.rejectHostConnection(fromHostLiveID: inviter.liveID) { ... }            default:                break        }    }.store(in: &cancellableSet)
```

#### Шаг 3: Автоматическое инициирование PK

Когда целевой стример принимает, немедленно вызовите `BattleStore.requestBattle` с `needResponse: false` для начала PK.

```
import AtomicXCorevar coHostStore: CoHostStore {    return CoHostStore.create(liveID: liveID)}coHostStore.coHostEventPublisher    .receive(on: RunLoop.main)    .sink { [weak self] event in        guard let self = self else { return }        switch event {            case .onCoHostRequestAccepted(let invitee):                let config = BattleConfig(duration: 30, needResponse: false, extensionInfo: "")                battleStore.requestBattle(config: config, userIDList: [invitee.userID], timeout: 0) { [weak self] result in                    guard let self = self else { return }                    switch result {                        case .success:                            // PK started                        case .failure(let error):                            // Handle error                    }                }            default:                break        }    }.store(in: &cancellableSet)
```

#### Шаг 4: Вход в состояние PK

Этот шаг аналогичен [Входу в состояние PK](#step-3-enter-pk-state).

## Обновление PK счета и определение победителя

### Обновление PK счета через REST API

В типичных **сценариях PK между стримерами** стоимость подарков, полученных хостом, связана со счетом PK (например, когда зритель отправляет подарок "Ракета", счет PK хоста увеличивается на 500 очков). Вы можете реализовать обновления счета PK в реальном времени, используя наш `REST API`.

> **Примечание:** Система счета PK в бэкенде LiveKit использует чистый численный расчет и накопление. Вы должны рассчитать счет PK в соответствии с вашей собственной бизнес-логикой перед вызовом API обновления. См. следующие примеры расчета счета PK: Тип подарка | Правило расчета счета | Пример
Базовый подарок | Стоимость подарка × 5 | Подарок 10 юаней → 50 очков
Промежуточный подарок | Стоимость подарка × 8 | Подарок 50 юаней → 400 очков
Продвинутый подарок | Стоимость подарка × 12 | Подарок 100 юаней → 1200 очков
Подарок со спецэффектом | Фиксированный высокий счет | Подарок 520 юаней → 1314 очков

#### Описание ключевого процесса

1. **Получение статуса PK:**
  - **Конфигурация колбэка**: Настройте [Колбэк статуса PK](https://www.tencentcloud.com/document/product/647/68260) так, чтобы бэкенд `LiveKit` активно уведомлял вашу систему при запуске или завершении PK.
  - **Активный запрос**: Ваш бэкенд может в любой момент вызвать API [Запрос статуса PK](https://www.tencentcloud.com/document/product/647/68256) для проверки текущего статуса PK.
2. **Расчет счета PK**: Ваш бэкенд рассчитывает прирост счета PK на основе ваших бизнес-правил.
3. **Обновление счета PK**: Ваш бэкенд вызывает API [Обновить счет PK](https://www.tencentcloud.com/document/product/647/68255) для обновления счета PK в бэкенде `LiveKit`.
4. **Синхронизация бэкенда LiveKit с клиентом**: Бэкенд автоматически синхронизирует обновленный счет PK со всеми клиентами.

В сценариях PK счета стримеров привязаны к полученным подаркам. Модуль PK в RoomEngine поддерживает обновления счета и трансляцию. Свяжите события подарков с подсчетом PK для обновления полосы прогресса в реальном времени.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/cb8a5a37cc3d11f091ab5254007c27c5.png)

Когда система выставления счетов обнаруживает успешную транзакцию подарка, вызовите [REST API изменения счета PK](https://www.tencentcloud.com/document/product/647/68255) для обновления счета PK стримера. Все пользователи в комнатах PK будут видеть обновленный `battleScore` в `BattleState`. Используйте это для обновления полосы прогресса PK.

```
battleStore.state.subscribe(StatePublisherSelector(keyPath: \\BattleState.battleScore))    .receive(on: RunLoop.main)    .sink { [weak self] battleScore in        guard let self = self else { return }        // Update PK progress bar here    }    .store(in: &cancellableSet)
```

# Живые комментарии

Это руководство показывает веб-разработчикам, как быстро добавить надежную высокопроизводительную систему живых комментариев в приложение прямой трансляции, используя модуль `BarrageState` из фреймворка `AtomicXCore`.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/131a16add18d11f0b1675254001c06ec.png)

## Основные возможности

`BarrageState` обеспечивает полное решение для живых комментариев в приложении прямой трансляции. Основные возможности включают:

- Получение и отображение сообщений живых комментариев в вашей трансляции.
- Отправку текстовых живых комментариев для взаимодействия с аудиторией.
- Отправку пользовательских живых комментариев для сценариев с подарками, лайками и другими расширенными взаимодействиями с аудиторией.
- Вставку системных уведомлений в локальный список сообщений (например, «Добро пожаловать XX в прямую трансляцию»).

## Пример проекта

Для полной реализации ознакомьтесь с компонентами [BarrageList](https://github.com/Tencent-RTC/TUIKit_Vue3/blob/main/packages/tuikit-atomicx-vue3/src/components/BarrageList/BarrageList.vue) и [BarrageInput](https://github.com/Tencent-RTC/TUIKit_Vue3/blob/main/packages/tuikit-atomicx-vue3/src/components/BarrageInput/BarrageInput.vue) на GitHub.

## Реализация

### Шаг 1: Интеграция компонента

Следуйте руководству [Быстрый старт](https://www.tencentcloud.com/document/product/647/74840) для интеграции **AtomicXCore** и завершения начальной настройки.

> **Примечание:** Перед тем как продолжить, необходимо создать или присоединиться к прямой трансляции, как описано в [Быстрый старт](https://www.tencentcloud.com/document/product/647/74840).

### Шаг 2: Инициализация и прослушивание живых комментариев

Получите экземпляр `BarrageState`, связанный с `liveId` текущей прямой трансляции, и отслеживайте `messageList` для получения последнего списка сообщений живых комментариев.

```
import { useBarrageState } from 'tuikit-atomicx-vue3';import { watch } from 'vue'const { messageList } = useBarrageState()// Watch for real-time updates to messageList and update your UI accordinglywatch(messageList, (newVal, oldVal) => {  console.log(newVal)});
```

### Шаг 3: Отправка текстовых живых комментариев

Используйте метод `sendTextMessage` для трансляции обычного текстового сообщения всем в прямой трансляции.

```
import { useBarrageState } from 'tuikit-atomicx-vue3'const { sendTextMessage } = useBarrageState();const sendMessage = async () => {  try {    await sendTextMessage({      text: 'Welcome to the live roomï¼',      extensionInfo: {        color: '#ff0000',        fontSize: '16px'      }    });  } catch (error) {    console.error('Failed to send message:', error);  }}
```

#### Параметры API

| **Параметр** | **Тип** | **Описание** |
| --- | --- | --- |
| `text` | `string` | Текстовое содержание для отправки. |
| `extensionInfo` | `Record<string, string>` | Дополнительная информация расширения для пользовательского использования в бизнес-логике. |

#### Отправка пользовательских живых комментариев

Отправляйте сообщения с пользовательской бизнес-логикой, такие как подарки, лайки или игровые команды. Формат сообщения определяется слоем вашей бизнес-логики, и получатель должен анализировать и обрабатывать его, используя `businessId` и `data`.

```
import { useBarrageState } from 'tuikit-atomicx-vue3'import { watch } from 'vue';const { sendCustomMessage } = useBarrageState();// Send a custom live comment, for example, a giftconst sendGiftMessage = async (giftId: string, giftCount: number) => {  // 1. Set a business-specific ID  const businessId = 'live_gift';  // 2. Encode business data as a JSON string  const giftData = { gift_id: giftId, gift_count: giftCount };  // 3. Call the API to send the custom message  try {    await sendCustomMessage({      businessId,      data: JSON.stringify(giftData)    });  } catch (error) {    console.error('Failed to send gift message:', error);  }};
```

#### Параметры API

| **Параметр** | **Тип** | **Описание** |
| --- | --- | --- |
| `businessId` | `string` | Уникальный идентификатор бизнеса, например «live_gift», используется получателем для различения разных пользовательских сообщений. |
| `data` | `string` | Бизнес-данные, обычно строка в формате JSON. |

### Шаг 4: Вставка локального уведомления

Добавьте локальное сообщение в список сообщений текущего пользователя. Это сообщение не отправляется другим пользователям в прямой трансляции. Используйте это для системных приветствий, предупреждений или советов по операциям.

```
import { useBarrageState } from 'tuikit-atomicx-vue3'const { appendLocalTip } = useBarrageState()const systemTip = {  liveId: 'live123',  sender: {    userId: 'system',    userName: 'jasper',    avatarUrl: '',  },  sequence: Date.now(), // Message sequence number for sorting  timestampInSecond: Math.floor(Date.now() / 1000), // 10-digit Unix timestamp  messageType: 0, // Message type enum value (e.g., 0 for plain text message; see API enum definition)  textContent: 'Welcome to the live room', // Text content to display  extensionInfo: { type: 'system' }, // Extension field for special UI rendering}appendLocalTip(systemTip)
```

### Шаг 5: Управление правами на отправку сообщений пользователем (Mute и Unmute)

Хосты или администраторы могут контролировать, какие пользователи могут отправлять сообщения в прямой трансляции.

#### Отключение/включение звука для отдельных пользователей

Используйте API `disableSendMessage` из `LiveAudienceState` для отключения или включения звука для конкретного пользователя. Это состояние сохраняется даже если пользователь повторно присоединится к прямой трансляции, его статус отключения звука остается прежним.

```
import { useLiveAudienceState } from 'tuikit-atomicx-vue3';const { disableSendMessage } = useLiveAudienceState()// If isDisable is true, the user is muted.// If isDisable is false, the user is unmuted.await disableSendMessage({ userId: 'user123', isDisable: true });
```

#### Включение/отключение глобального отключения звука

Для отключения звука всем пользователям в прямой трансляции (обычно кроме хоста) обновите информацию о прямой трансляции с помощью API `updateLiveInfo` из `LiveListState`.

```
import { useLiveListState } from 'tuikit-atomicx-vue3';// 1. Get a LiveListState instanceconst { updateLiveInfo } = useLiveListState();const setGlobalMute = async (shouldMute: boolean) => {  try {    // 2. Call the update API    // LiveListState will automatically merge and update the live room info based on the parameters    await updateLiveInfo({      // Maps to the isMessageDisable field in LiveInfo      // true: mute all audience members      // false: allow audience members to send messages      isMessageDisable: shouldMute     });    console.log(`Global mute status updated to: ${shouldMute}`);  } catch (error) {    console.error('Failed to update global mute:', error);  }};// Example usage: Enable global muteawait setGlobalMute(true);
```

## Часто задаваемые вопросы

### Как реализовать расширенные стили, такие как «цветные живые комментарии» или «подарки», помимо обычного текста?

Вы можете использовать пользовательские сообщения через `sendCustomMessage`. `BarrageState` поддерживает гибкие форматы пользовательских сообщений, поэтому вы можете определять структуры и стили сообщений в соответствии с вашими потребностями бизнеса.

**Подход к реализации**

1. **Определение структуры данных:** Работайте с командами клиента и сервера для разработки структуры JSON для пользовательских сообщений. Например, цветной живой комментарий может выглядеть следующим образом:

```
{ "type": "colored_text", "text": "This is a colored live comment!", "color": "#FF5733" }
```

2. **Отправитель:** Преобразуйте эту структуру JSON в строку и отправьте ее, используя параметр `data` метода `sendCustomMessage`. Установите `businessId` как уникальный идентификатор для вашего бизнеса, например "`barrage_style_v1`".
3. **Получатель:** Когда вы получаете сообщение живого комментария, проверьте, что его `messageType` имеет значение `'CUSTOM'` и что `businessId` совпадает. Если это так, разберите строку `data` (обычно как JSON) и отрендерите свой пользовательский стиль UI на основе разобранных данных (например, `color`, `text`).

### Почему отправленное сообщение не появляется в списке сообщений после вызова sendTextMessage?

**Шаги для устранения неполадок:**

1. **Проверьте наличие ошибок в catch:** Метод `sendTextMessage` включает обработку ошибок. Проверьте результат обратного вызова на успех или ошибку. Если это ошибка, сообщение об ошибке укажет причину (например, «Вы отключены», «Ошибка сети» и т. д.).
2. **Подтвердите время подписки:** Убедитесь, что вы подписываетесь на `barrageState` после того, как соответствующая прямая трансляция `liveId` началась. Если вы начнете прослушивание перед присоединением к прямой трансляции, вы можете потерять некоторые сообщения.
3. **Проверьте liveId:** Убедитесь, что `liveId`, используемый для создания экземпляра `BarrageState`, присоединения к прямой трансляции и отправки сообщений, совпадает полностью, включая чувствительность к регистру.
4. **Проблемы с сетью:** Убедитесь, что подключение к сети устройства стабильно. Отправка сообщений зависит от подключения к сети.

### Как новые зрители смогут увидеть исторические сообщения барража, отправленные до присоединения к прямой трансляции?

`AtomicXCore` поддерживает получение исторических сообщений чата, но вам необходимо включить эту функцию в **консоли сервера**. После настройки SDK автоматически обрабатывает все необходимое — дополнительный код клиента не требуется.

#### **Шаг 1: Настройка в консоли Live**

1. Войдите в [**Консоль > Live > Конфигурация**](https://console.trtc.io/live/configuration) и выберите целевое приложение в верхней части страницы.
2. На странице Live Configuration выберите **View Past Messages** (Просмотр предыдущих сообщений) и установите Previous Messages Viewable (Видимость предыдущих сообщений), чтобы указать количество сообщений (максимум 50), которые новые зрители смогут видеть.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/66f78e98e20c11f0ab0f5254005ef0f7.png)

#### **Шаг 2: Автоматическое получение на стороне клиента**

После завершения вышеуказанной конфигурации **в коде клиента не требуются никакие изменения**.

Когда новый пользователь присоединяется к прямой трансляции, `AtomicXCore` автоматически получает настроенное количество исторических сообщений чата в фоновом режиме. Эти сообщения доставляются на слой UI через канал подписки `BarrageState`, как и сообщения в реальном времени. Ваше приложение будет получать и отображать эти исторические сообщения чата так же, как живые сообщения.

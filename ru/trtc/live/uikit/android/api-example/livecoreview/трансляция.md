# Трансляция

Этот документ главным образом описывает, как использовать основной компонент видеотрансляции — `LiveCoreView` модуля `LiveStreamCore` для начала прямой трансляции. Только после начала трансляции другие зрители смогут войти в комнату трансляции для просмотра трансляции хоста.

## Предварительные требования

Перед использованием `LiveStreamCore` необходимо [интегрировать и войти](https://www.tencentcloud.com/document/product/647/67475#) в LiveStreamCore, чтобы убедиться, что последующие функции работают корректно.

## Руководство по использованию

### Шаг 1: Добавление LiveCoreView в представление

Сначала необходимо импортировать модуль `LiveStreamCore`, затем создать объект представления `LiveCoreView` и добавить его в представление.

iOS

Android

```
import RTCRoomEngineimport LiveStreamCore class BroadcastController: UIViewController {    private let liveCoreView: LiveCoreView = {         let view = LiveCoreView()      return view    }()        override func viewDidLoad() {        super.viewDidLoad()        // Add liveCoreView to the view and set layout constraints    }}
```

```
import com.trtc.uikit.livekit.livestreamcore.LiveCoreView;public class LiveActivity extends AppCompatActivity {    @Override    protected void onCreate(@Nullable Bundle savedInstanceState) {        super.onCreate(savedInstanceState);        LiveCoreView liveCoreView = new LiveCoreView(this);        addContentView(liveCoreView,                new ViewGroup.LayoutParams(ViewGroup.LayoutParams.MATCH_PARENT, ViewGroup.LayoutParams.MATCH_PARENT));    }}
```

### Шаг 2: Подготовка параметров прямой трансляции

При вызове API `startLiveStream` необходимо заполнить ключевые параметры `TUIRoomInfo`. Подробное описание приведено ниже:

#### Параметры: TUIRoomInfo

`TUIRoomInfo` состоит из множества полей, но обычно вам нужно сосредоточиться только на следующих полях:

| Имя параметра | Описание поля | Дополнительные примечания | Тип данных | Пример |
| --- | --- | --- | --- | --- |
| roomId | ID комнаты | Допускаются только буквы (a-z, A-Z), цифры (0-9), подчеркивания и дефисы. | Новая строка | "room_100001" |
| name | Имя комнаты | Имя комнаты строкового типа. | Новая строка | "denny`s room" |
| seatMode |  Режим микрофона | Это поле действует только при включении управления микрофоном. Оно разделено на режимы "freeToTake" и "applyToTake". В режиме freeToTake зрители могут свободно включить микрофон без запроса. В режиме applyToTake зрители нуждаются в одобрении владельца комнаты для включения микрофона. | Значение перечисления | TUISeatMode.applyToTake |
| maxSeatCount | Максимальное количество микрофонов | Максимальное количество позиций микрофона числового типа, означающее максимальное количество человек на микрофоне одновременно, maxSeatCount. Максимальное количество мест соответствует максимальному количеству «Нескольких гостей» в [пакете](https://trtc.io/document/59407), который вы приобрели. | Цифры | 10 |
| isSeatEnabled | Включить ли управление позициями микрофона | Флаг управления микрофоном логического типа. | Логическое значение | true |
| roomType | Тип комнаты | Существует два типа комнат: "conference" и "live". Для видеотрансляции используйте live. | Значение перечисления | TUIRoomType.live |

### Шаг 3: Запуск прямой трансляции

После подготовки параметров `TUIRoomInfo` на шаге 2 можно вызвать функцию API `startLiveStream` для начала прямой трансляции.

После запуска прямой трансляции также необходимо вызвать API `startCamera` и `startMicrophone`, чтобы включить локальную камеру и микрофон, гарантируя, что зрители смогут видеть видео и слышать ваш голос.

iOS

Android

```
// Assemble TRTC room entry parameters. Replace the field values in `TRTCParams` with your own parameter valueslet roomInfo = TUIRoomInfo()roomInfo.roomId = "voice_100001"  // Please replace it with your own room ID.roomInfo.name = "denny`s room"    // Please replace it with your own room name.roomInfo.maxSeatCount = 10        // Please replace it with the maximum number of seats in the Live package you purchased.roomInfo.isSeatEnabled = true     // If you do not need seat control, you can set isSeatEnabled to falseroomInfo.roomType = .live         // Ensure the room type is live.roomInfo.seatMode = .applyToTake  // The usual mode for video live streaming is applyToTake.self.liveCoreView.startLiveStream(roomInfo: roomInfo) { [weak self] roomInfo in    // Going live successfully    guard let self = self else { return }    // Turn the local camera on    self.liveCoreView.startCamera(useFrontCamera: true) { // This selects the front camera. You can also pass in false to select the rear camera.        // Successfully turned the local camera on    } onError: { code, message in         // Failed to turn the local camera on    }    // Turn the local mic on    self.liveCoreView.startMicrophone {       // Successfully turned the local mic on    } onError: { code, message in        // Failed to turn the local mic on    }} onError: { code, message in     // Failed to go live}
```

```

```

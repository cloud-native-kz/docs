### Активация сервиса

Перед использованием **TUILiveKit** обратитесь к разделу [Активация сервиса](https://www.tencentcloud.com/document/product/647/60033), чтобы получить пробную версию TUILiveKit или активировать платную версию.

## Интеграция кода

### **Шаг 1.** Загрузка компонента TUILiveKit

Клонируйте или загрузите код с [GitHub](https://github.com/Tencent-RTC/TUIKit_Android). Затем скопируйте подпапки `live` и `atomic_x` в ту же директорию, что и папка `app` вашего текущего проекта Android.

### Шаг 2. Конфигурация проекта

#### 1. Конфигурация репозитория JitPack

В корневой директории вашего проекта откройте файл `settings.gradle.kts` или `settings.gradle` и добавьте адрес репозитория JitPack. Сторонняя библиотека для воспроизведения SVG-анимаций подарков размещена на JitPack, поэтому вы должны включить этот репозиторий для разрешения зависимостей.

settings.gradle

```
dependencyResolutionManagement {    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)    repositories {        google()        mavenCentral()        // Add JitPack repository url        maven { url = uri("https://jitpack.io") }    }}
```

#### 2. Импорт компонента TUILiveKit

В корневой директории вашего проекта добавьте следующую строку в файл `settings.gradle.kts` или `settings.gradle`, чтобы импортировать компонент **TUILiveKit** (как модуль).

settings.gradle.kts

settings.gradle

include ':tuilivekit'project(':tuilivekit').projectDir = new File(settingsDir, "live/tuilivekit")include(":atomic")project(':atomic').projectDir = new File(settingsDir, "atomic_x")
```

```

#### 3. Добавление зависимости компонента

Перейдите в директорию app и найдите файл `build.gradle.kts` или `build.gradle`. Добавьте следующую зависимость, чтобы указать, что ваш модуль app зависит от недавно добавленного модуля компонента `TUILiveKit`.

build.gradle.kts

build.gradle

```
dependencies {    // Add tuilivekit dependency    api(project(":tuilivekit"))}
```

```
dependencies {    // Add tuilivekit dependency    api project(':tuilivekit')}
```

> **Примечание:** TUILiveKit имеет зависимости по умолчанию от публичных библиотек, таких как TRTC SDK и IM SDK, поэтому вам **не требуется** конфигурировать их отдельно.

#### 5. Изменение AndroidManifest.xml

Чтобы предотвратить конфликты во время процесса слияния **AndroidManifest**, вы должны добавить `tools:replace="android:allowBackup"` и `android:allowBackup="false"` к узлу **<application>** в файле `app/src/main/AndroidManifest.xml`. Это переопределяет параметры внутри компонента и отключает функцию резервного копирования.

```
 <application    ...      // Add the following configuration to override the settings in the dependent SDK    android:allowBackup="false"    tools:replace="android:allowBackup">
```

После завершения описанных выше шагов Android Studio обычно предложит вам кнопку **Sync Now**. Нажмите эту кнопку для завершения синхронизации проекта. Если IDE не предложит автоматически, вручную нажмите кнопку синхронизации на панели инструментов. После синхронизации IDE завершит конфигурацию и индексирование проекта, позволяя вам использовать компонент **TUILiveKit** в вашем проекте.

## Завершение входа

После интеграции кода следующий шаг — завершить вход. Все функции **TUILiveKit** требуют успешного входа для правильной работы, поэтому убедитесь, что ваши параметры конфигурированы правильно.

> **Примечание:** В примере кода API входа вызывается напрямую. Однако в реальном приложении настоятельно рекомендуется вызывать сервис входа **TUILiveKit** только после завершения вашей собственной аутентификации пользователя и других внутренних процессов входа. Это предотвращает потенциальную путаницу в бизнес-логике или несогласованность данных, вызванные слишком ранним вызовом сервиса входа, и лучше согласуется с вашей существующей системой управления пользователями.

Kotlin

Java

```
// 1. Import dependencyimport com.tencent.qcloud.tuicore.TUILogin// 2. Call the login API. It is recommended to call TUILogin after your own login business is completed.TUILogin.login(applicationContext,    1400000001,      // Please replace with the SDKAppID from the service activation console    "denny",         // Please replace with your UserID    "xxxxxxxxxxx",   // You can generate a UserSig in the console and fill it in here    object : TUICallback() {        override fun onSuccess() {            Log.i(TAG, "login success")        }        override fun onError(errorCode: Int, errorMessage: String) {            Log.e(TAG, "login failed, errorCode: $errorCode msg:$errorMessage")        }    })
```

**Описание параметров API входа**

| Параметр | Тип | Описание |
| --- | --- | --- |
| SDKAppID | Int | Получите это из [консоли TRTC > Управление приложениями](https://console.trtc.io/app). |
| UserID | String | Уникальный идентификатор текущего пользователя. Должен содержать только английские буквы, цифры, дефисы и подчеркивания. |
| userSig | String | Билет для аутентификации в Tencent Cloud. Обратите внимание: **Среда разработки**: Вы можете использовать локальную функцию `GenerateTestUserSig.genTestSig` для генерации UserSig или создать временный UserSig через [инструмент генерации UserSig](https://console.trtc.io/usersig). **Производственная среда**: Чтобы предотвратить утечку ключей, вы должны использовать метод на стороне сервера для генерации UserSig. Подробнее см. в разделе [Генерирование UserSig на сервере](https://www.tencentcloud.com/document/product/647/69883). Дополнительную информацию см. в разделе [Как вычислять и использовать UserSig](https://www.tencentcloud.com/document/product/647/35166). |

### Обработка состояния исключения входа [Опционально]

**TUILogin** предоставляет механизм обратного вызова состояния входа, чтобы помочь вам обработать возможные исключения входа, в основном включая обратные вызовы "**отключен в режиме онлайн**" и "**подпись истекла**":

- **Отключен в режиме онлайн:** Если пользователь отключается в режиме онлайна, **SDK** уведомит вас через обратный вызов `onKickedOffline`. В этом случае вы можете отобразить подсказку UI пользователю и вызвать `TUILogin.login` для входа снова.
- **Подпись истекла:** Если пользователь получает обратный вызов `onUserSigExpired` в режиме онлайна, это означает, что ранее выданный **userSig** для этого пользователя истек. Если сеанс входа пользователя на вашем бэкенде все еще действителен, вы можете попросить ваше приложение запросить новый **userSig** у вашего бэкенда и вызвать `TUILogin.login` для обновления сеанса входа.

kotlin

java

```
class YourLoginService : TUILoginListener() {    // Listen to login status callback    fun addObserver() {        TUILogin.addLoginListener(this)    }    // Cancel listening for login status callback    fun removeObserver() {        TUILogin.removeLoginListener(this)    }    // User kicked offline callback    override fun onKickedOffline() {        // Your business code: UI prompts the user, then log in again    }    // User signature expired callback    override fun onUserSigExpired() {        // Your business code: If the current user is still logged in on your backend, you can let your app request a new userSig from the backend and call TUILogin.login to renew the login status.    }}
```

## Часто задаваемые вопросы

### Нужно ли мне вызывать API входа каждый раз при входе в комнату?

Нет. Обычно вам нужно вызвать `TUILogin.login` только один раз. Мы рекомендуем связать `TUILogin.login` и `TUILogin.logout` с вашей собственной бизнес-логикой входа и выхода приложения.

### Как мне исправить ошибку компиляции `allowBackup`, показанную на рисунке ниже, после интеграции кода?

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/bc551d7b985a11f0af98525400454e06.png)

- **Причина**: Атрибут `allowBackup` конфигурирован в `AndroidManifest.xml` нескольких модулей, что вызывает конфликт во время процесса слияния манифеста.
- **Решение**: В файле `AndroidManifest.xml` вашего проекта вы можете либо удалить атрибут `allowBackup`, либо изменить его на false, чтобы отключить функцию резервного копирования и восстановления. Критически важно добавить `tools:replace="android:allowBackup"` к узлу **<application>**, чтобы переопределить параметры других модулей с помощью вашей конфигурации. Смотрите пример исправления:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/bc619de5985a11f09b0c525400bf7822.png)

### После интеграции кода, нужно ли мне явно объявлять разрешения на доступ к камере и микрофону?

Нет, **TUILiveKit** уже имеет встроенные объявления разрешений, таких как доступ к камере и микрофону, в своем файле манифеста. Вам не нужно беспокоиться об объявлении этих разрешений отдельно во время процесса интеграции.

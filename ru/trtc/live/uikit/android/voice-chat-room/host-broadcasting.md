# Трансляция хоста

`TUILiveKit` Voice Chat Room — это готовый к использованию интерфейс, предоставляющий все основные функциональные возможности для запуска прямой трансляции. Он позволяет интегрировать поток трансляции хоста в ваше приложение в несколько простых шагов.

## Обзор функций

- **Предпросмотр перед началом трансляции**: позволяет хостам настраивать персонализированные параметры, такие как название комнаты и изображение обложки, перед началом трансляции.
- **Управление местами**: поддерживает операции управления местами, включая занятие места, освобождение места, отключение звука и блокировку мест.
- **Взаимодействие с аудиторией**: обеспечивает интерактивные функции, такие как живой чат (плавающие комментарии), виртуальные подарки и лайки.
- **Управление живой комнатой**: отображает список онлайн-пользователей и предоставляет элементы управления модерацией, такие как отключение звука или удаление пользователей из живой комнаты.

### Создание и отображение Voice Chat Room Activity

Компонент `VoiceRoomActivity` включает полностью встроенный пользовательский интерфейс и бизнес-логику для стороны хоста в голосовой комнате. Чтобы включить трансляцию хоста, просто запустите это `Activity`. Мы рекомендуем запустить следующую логику при нажатии кнопки `Start Live` в вашем приложении:

```
import android.content.Intentimport androidx.appcompat.app.AppCompatActivityimport com.tencent.cloud.tuikit.engine.room.TUIRoomDefine.SeatModeimport com.trtc.uikit.livekit.voiceroom.view.TUIVoiceRoomFragment.RoomBehavior.PREPARE_CREATEimport com.trtc.uikit.livekit.voiceroom.view.TUIVoiceRoomFragment.RoomParamsimport com.trtc.uikit.livekit.voiceroom.view.VoiceRoomActivity// YourActivity представляет страницу, с которой вы начинаете трансляцииclass YourActivity : AppCompatActivity() {    // Обработка события нажатия кнопки "Start Live"    fun onStartVoiceRoomClicked() {        // 1. Настройте параметры комнаты (RoomParams)        // RoomParams должен реализовать интерфейс Parcelable        val params = RoomParams().apply {            maxSeatCount = 10 // Максимальное количество мест            seatMode = SeatMode.APPLY_TO_TAKE // Режим мест        }        // 2. Подготовьте Intent и передайте необходимые параметры        val roomId = "test_voice_room_id"        val intent = Intent(this, VoiceRoomActivity::class.java).apply {            putExtra(VoiceRoomActivity.INTENT_KEY_ROOM_ID, roomId)            // behavior: PREPARE_CREATE означает сначала перейти на страницу предпросмотра перед началом трансляции            putExtra(VoiceRoomActivity.INTENT_KEY_CREATE_ROOM_PARAMS, params)            putExtra(VoiceRoomActivity.INTENT_KEY_ROOM_BEHAVIOR, PREPARE_CREATE.ordinal)        }        // 3. Перейдите на страницу голосовой комнаты        startActivity(intent)    }}
```

- **Описание параметра Intent Extra:**

| **Параметр** |  | **Тип** | **Описание** |
| --- | --- | --- | --- |
| `VoiceRoomActivity.INTENT_KEY_ROOM_ID` |  | `string` | Глобально уникальный идентификатор живой комнаты. |
| `VoiceRoomActivity.INTENT_KEY_ROOM_BEHAVIOR` |  | `Int` | Поведение входа в комнату: `AUTO_CREATE`: автоматически создать живую комнату и войти. `PREPARE_CREATE`: сначала перейти на страницу предпросмотра перед началом трансляции; после нажатия пользователем кнопки "`Start Live`" создать живую комнату и войти. `JOIN:` аудитория присоединяется к комнате. |
| `roomParams` |  | `RoomParams` | Параметры трансляции хоста. Подробности см. в следующем разделе. |

- **Описание параметра RoomParams:**

| **Параметр** | **Тип** | **Описание** |
| --- | --- | --- |
| `maxSeatCount` | `Int` | Максимальное количество мест в живой комнате. |
| `seatMode` | `TUIRoomDefine.SeatMode` | Режим мест для аудитории: `APPLY_TO_TAKE`: аудитория должна подать заявку и получить одобрение хоста перед занятием места. `FREE_TO_TAKE`: аудитория может занять место без одобрения хоста. |

## Настройка интерфейса

`TUILiveKit` предоставляет гибкую настройку интерфейса для удовлетворения различных бизнес-требований. Вы можете выбрать различные шаблоны макетов и легко заменить текст и значки пользовательского интерфейса.

### Выбор шаблона макета трансляции

`TUILiveKit` Voice Chat предлагает **два стиля макета**. Выберите предпочитаемый стиль, используя кнопку **макета** на странице предпросмотра перед началом трансляции:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/a034e296c08011f08c0e52540044a08e.png)

| **Предпросмотр** | ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/4d07fbebc06e11f083155254001c06ec.png) | ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/c85b08ebc07311f0b35752540099c741.png) |

### Настройка текста (строковые ресурсы)

`TUILiveKit` использует стандартные файлы `Res-Common>Values>Strings.xml` для управления текстом, отображаемым в пользовательском интерфейсе. Вы можете напрямую изменить строки, которые требуют корректировки, через XML-файл:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/23515658c06511f0a5e052540099c741.png)

### Настройка значков (ресурсы Drawable)

`TUILiveKit` использует стандартную папку `Android drawable resource` для управления графическими ресурсами пользовательского интерфейса. Вы можете быстро изменить пользовательские значки, заменив файлы ресурсов. При замене убедитесь, что новые имена файлов совпадают с исходными именами файлов.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2344db82c06511f0a6c652540044a08e.png)

## Часто задаваемые вопросы

### Нет звука после начала трансляции?

Убедитесь, что приложение имеет разрешение на использование микрофона. Перейдите в `Информация о приложении > Разрешения > Микрофон` на телефоне и убедитесь, что доступ к микрофону включен.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/548602ebc06a11f0a6dd5254005ef0f7.png)

### Невозможно начать трансляцию при нажатии кнопки запуска с запросом "Not Logged In"?

Обратитесь к разделу [Завершение входа](https://www.tencentcloud.com/document/product/647/60335#CompleteLogin), чтобы убедиться, что функциональность входа правильно интегрирована.

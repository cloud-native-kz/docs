# Трансляция хоста

**Страница прямой трансляции хоста TUILiveKit** предоставляет **полнофункциональный пользовательский интерфейс** для сценариев прямой трансляции. Она поддерживает быстрое развертывание основных возможностей хоста, позволяя вам эффективно интегрировать рабочий процесс прямой трансляции без необходимости беспокоиться о сложной реализации пользовательского интерфейса и логики.

## Обзор функций

- **Настройка перед трансляцией:** поддерживает различные персонализированные конфигурации перед выходом хоста в эфир, включая имя комнаты, фон, просмотр видео, отладку фильтров красоты (эффектов красоты), отладку звуковых эффектов и шаблоны макетов.
- **Взаимодействие с соведущим:** поддерживает взаимодействие в реальном времени (совместное ведение) со зрителями или другими хостами во время прямой трансляции.
- **Взаимодействие со зрителями:** поддерживает богатые формы взаимодействия, такие как баррейд (экран пуль) и подарки.
- **Управление прямой комнатой:** поддерживает отображение списка онлайн-пользователей и различные операции управления в комнате, такие как отключение звука (блокировка) и исключение пользователей.

## Быстрый старт

### Шаг 1. Активируйте сервис

Обратитесь к документу [Активировать сервис](https://www.tencentcloud.com/document/product/647/60033) для включения **пробной версии** или активации пакета **Pro Edition**.

### Шаг 2. Установите TUILiveKit

Обратитесь к руководству [Подготовка](https://www.tencentcloud.com/document/product/647/63255) для интеграции SDK TUILiveKit.

### Шаг 3. Добавьте страницу хоста

`TUILiveRoomAnchorWidget` имеет встроенный полный пользовательский интерфейс и бизнес-логику для сценариев прямой трансляции. Этот компонент не поддерживает режим плавающего окна. Если вам нужна функциональность плавающего окна, перейдите к [Добавить плавающее окно (страница хоста)](#330378b2-2f2c-45ba-9a1a-743448cc3d19). Просто настройте точку входа для вызова `TUILiveRoomAnchorWidget` (на основе вашей бизнес-логики), а затем используйте один из следующих методов для перехода на страницу хоста или встраивания её в дерево Widget:

Прямая навигация

Встраивание в дерево Widget

```
import 'package:tencent_live_uikit/tencent_live_uikit.dart';// navigate to the host webpageNavigator.push(context, MaterialPageRoute(  builder: (BuildContext context) {     final roomId = "test_live_room_id";     return TUILiveRoomAnchorWidget(roomId: roomId);}));
```

```
// --- Select a way to integrate based on your Widget tree structure ---// [option one] as the only child Widget (single subtree)// Suitable for containers like Container, Padding that usually only contain one child WidgetContainer(       child: TUILiveRoomAnchorWidget(roomId: roomId) // integrate the host page here)// [option two] as one of multiple child Widgets (multi-subtree)// Suitable for layouts like Column, Row, Stack that can contain multiple child WidgetsStack(       children: [           YourOtherWidget(), // your other child Widget           TUILiveRoomAnchorWidget(roomId: roomId), // integrate the host page here        YourOtherWidget(), // your other child Widget   ])
```

После интеграции запуск этого кода отобразит страницу хоста, как показано на изображении просмотра прямой трансляции в разделе [Обзор функций](#77420663-dc80-46f2-857f-1cecc466c2fb).

### Шаг 4. (Опционально) Добавьте плавающее окно

`TUILiveRoomAudienceOverlay` обеспечивает режим плавающего окна для страницы просмотра аудитории. Во время прямой трансляции вы можете переключаться между **встроенным в приложение** и **вне приложения** (Picture-in-Picture) режимами плавающего окна. `TUILiveRoomAudienceOverlay` построен с использованием официального API `Overlay` Flutter и собственного Picture-in-Picture. Для интеграции:

#### 1. Включите системное Picture-in-Picture

Обратитесь к [Конфигурации проекта](https://www.tencentcloud.com/document/product/647/63255#62b3f143-07f5-416a-8057-fff865e35d46) для включения системной функции Picture-in-Picture.

#### 2. Добавьте вторичный навигатор

Обратитесь к [Конфигурация плавающей прямой комнаты](https://www.tencentcloud.com/document/product/647/63255#43bbc9ad-91f3-4216-8807-462e9df28f7a) для добавления нового вторичного навигатора в приложение. Этот навигатор размещает страницу прямой трансляции, отделяя её от основного стека навигации и обеспечивая эффекты плавающего окна.

#### 3. Перейдите к плавающему окну

В точке входа, где ваша аудитория входит в комнату (на основе вашей бизнес-логики), перейдите на страницу просмотра следующим образом:

```
import 'package:tencent_live_uikit/tencent_live_uikit.dart';// Navigate to the Anchor webpageNavigator.push(context, MaterialPageRoute(  builder: (BuildContext context) {     final roomId = "test_live_room_id";     return TUILiveRoomAnchorOverlay(roomId: roomId);}));
```

> **Примечание:** `TUILiveRoomAudienceOverlay` невозможно встроить в качестве дочернего виджета в контейнерные виджеты (такие как Container, Stack и т.д.); на него необходимо переходить как на отдельную страницу. Это связано с тем, что `Overlay` используется внутренне, и LiveKit должна контролировать всю страницу `Overlay` для переключения режимов плавающего окна. На iOS только обычные зрители могут входить в режим плавающего окна вне приложения.

#### Эффект режима плавающего окна

Нажмите кнопку плавающего окна в верхнем правом углу страницы хоста, чтобы войти в режим плавающего окна, как показано ниже.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/6c4937c1e22811f0a5a25254007c27c5.png)

#### Эффект плавающего окна вне приложения

Нажмите **Ещё > Picture-in-Picture** в нижнем правом углу страницы хоста, затем включите переключатель **PIP**. Когда ваше приложение переходит в фоновый режим, оно автоматически перейдёт в режим системного Picture-in-Picture, как показано ниже.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/911b6101e22811f08775525400bf7822.png)

### Настройка текста (ARB)

TUILiveKit использует **ARB-файлы** и **стандартное решение Flutter для интернационализации (i18n)** для управления отображением текста пользовательского интерфейса. Вы можете напрямую редактировать ARB-файлы в каталоге `livekit/lib/common/language/i10n/` для изменения текста:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2378e871991611f0aa4252540044a08e.png)

- `livekit_en.arb`: текст на английском языке.
- `livekit_zh.arb`: текст на упрощённом китайском языке.
- `livekit_zh_Hant.arb`: текст на традиционном китайском языке.

После внесения изменений выполните `flutter gen-l10n` в командной строке для повторного создания кода локализации.

### Настройка значков (ресурсы изображений)

Ресурсы изображений пользовательского интерфейса TUILiveKit хранятся в каталоге `livekit/assets/images/`. Чтобы обновить значки, просто замените файлы PNG в этой папке.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/25af402498f411f08bb25254005ef0f7.png)

Перестройте и запустите приложение, и вы увидите обновленные значки.

### Приглашение "Не авторизирован" при запуске прямой трансляции?

См. [Завершить вход](https://www.tencentcloud.com/document/product/647/63255#6b283542-df2f-4f16-b3b4-1fb0065d497a) для проверки того, что интеграция входа завершена.

### Почему приложение не может войти в режим плавающего окна вне приложения (Picture-in-Picture) при работе в фоне?

Обратитесь к [Режиму плавающего окна вне приложения](#c28b2363-6d3e-40e8-be0d-4e4dba28a665). После входа в прямую комнату включите переключатель Picture-in-Picture.

На Android после включения Picture-in-Picture приложение автоматически проверит системный параметр. Если Picture-in-Picture не включен, приложение предложит вам разрешить этот режим в системных параметрах:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/092d1165e22911f0bbb1525400454e06.png)

### Почему всплывающие окна не появляются или покрываются?

Если вы интегрировали `TUILiveRoomAudienceOverlay`, убедитесь, что компоненты всплывающих окон (такие как BottomSheet, Dialog и т.д.) используют контекст `rootNavigator` (рекомендуется: `Global.appContext()`).

**Причина:** Overlay страницы прямой трансляции отображается на `secondaryNavigator`. Всплывающие окна, использующие контекст `secondaryNavigator`, будут загораживаться Overlay, потому что на одном навигаторе слой Overlay находится выше обычных страниц, позволяя ему плавать над ними. Использование `rootNavigator` гарантирует, что всплывающие окна отображаются правильно над Overlay.

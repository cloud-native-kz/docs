# Просмотр аудиторией

**Страница просмотра аудиторией в TUILiveKit** предоставляет полный набор функций для трансляции и интерактивного взаимодействия, разработанных для беспрепятственной интеграции в ваше приложение. С помощью TUILiveKit вы можете быстро предоставить полнофункциональный опыт прямой трансляции, адаптированный к потребностям вашей аудитории.

## Обзор функций

- **Прямая трансляция:** Четкий и плавный просмотр прямой трансляции хоста в реальном времени.
- **Интерактивное соучастие:** Подайте запрос на подключение микрофона для взаимодействия с хостом через аудио и видео.
- **Информация о трансляции:** Просмотр названия комнаты прямой трансляции, описания, списка зрителей и т. д.
- **Интерактивность прямой трансляции:** Отправка подарков (с эффектами анимации и уведомлением хосту), лайки (с анимацией и статистикой в реальном времени) и взаимодействие через комментарии.

### Шаг 3: Добавить страницу просмотра аудиторией

`TUILiveRoomAudienceWidget` предоставляет полный пользовательский интерфейс и бизнес-логику со стороны аудитории для сценариев прямой трансляции. Этот виджет не поддерживает режим плавающего окна. Если вам требуется плавающее окно, см. [Добавить плавающее окно (страница аудитории)](#330378b2-2f2c-45ba-9a1a-743448cc3d19).

Чтобы запустить страницу просмотра аудиторией или встроить её в дерево виджетов, установите точку входа для вызова `TUILiveRoomAudienceWidget` в соответствии с вашей бизнес-логикой и используйте один из следующих подходов:

Прямая навигация

Встраивание в дерево виджетов

```
import 'package:tencent_live_uikit/tencent_live_uikit.dart';// Перейти на страницу просмотра аудиторией Navigator.push(context, MaterialPageRoute(  builder: (BuildContext context) {     final roomId = "test_live_room_id";     return TUILiveRoomAudienceWidget(roomId: roomId);}));
```

```
// --- Выберите один метод интеграции в зависимости от структуры дерева виджетов ---// [Вариант один] В качестве единственного дочернего виджета (одна подветвь)// Подходит для контейнеров типа Container, Padding, которые обычно содержат только один дочерний виджетContainer(       child: TUILiveRoomAudienceWidget(roomId: roomId) // Встроить просмотр аудитории здесь)// [Вариант два] В качестве одного из нескольких дочерних виджетов (несколько подветвей)// Подходит для макетов типа Column, Row, Stack, которые могут содержать несколько дочерних виджетовStack(       children: [           YourOtherWidget(), // Ваш другой дочерний виджет           TUILiveRoomAudienceWidget(roomId: roomId), // Встроить просмотр аудитории здесь        YourOtherWidget(), // Ваш другой дочерний виджет   ])
```

После интеграции выполнение этого кода откроет страницу аудитории и начнёт воспроизведение прямой трансляции, как показано на рисунке для просмотра прямой трансляции в [Обзоре функций](#ea46de7c-2609-4756-8514-ee368993168c).

### Шаг 4: (Опционально) Добавить плавающее окно

`TUILiveRoomAudienceOverlay` включает режим плавающего окна для страницы просмотра аудиторией. Во время прямой трансляции вы можете переключаться между режимами плавающего окна **внутри приложения** и **вне приложения** (Picture-in-Picture). `TUILiveRoomAudienceOverlay` построен с использованием официального `Overlay` API Flutter и собственного Picture-in-Picture. Для интеграции:

#### 1. Включить системный Picture-in-Picture

Обратитесь к [Конфигурация проекта](https://www.tencentcloud.com/document/product/647/63255#62b3f143-07f5-416a-8057-fff865e35d46), чтобы включить системную функцию Picture-in-Picture.

#### 2. Добавить вторичный навигатор

Обратитесь к [Конфигурация плавающей комнаты прямой трансляции](https://www.tencentcloud.com/document/product/647/63255#43bbc9ad-91f3-4216-8807-462e9df28f7a), чтобы добавить новый вторичный навигатор в приложение. Этот навигатор размещает страницу прямой трансляции, отделяя её от основного стека навигации и включая эффекты плавающего окна.

#### 3. Перейти в плавающее окно

В точке входа, где ваша аудитория входит в комнату (на основе вашей бизнес-логики), перейдите на страницу просмотра следующим образом:

```
import 'package:tencent_live_uikit/tencent_live_uikit.dart';// Перейти на страницу просмотраNavigator.push(context, MaterialPageRoute(  builder: (BuildContext context) {     final roomId = "test_live_room_id";     return TUILiveRoomAudienceOverlay(roomId: roomId);}));
```

> **Примечание:** `TUILiveRoomAudienceOverlay` не может быть встроен в качестве дочернего виджета в контейнерные виджеты (такие как Container, Stack и т. д.); он должен быть навигирован как отдельная страница. Это потому, что `Overlay` используется внутри, и LiveKit должен контролировать всю страницу `Overlay` для переключения режимов плавающего окна. На iOS только обычные члены аудитории могут входить в режим плавающего окна вне приложения.

#### Эффект режима плавающего окна

Нажмите кнопку плавающего окна в верхнем правом углу страницы, чтобы войти в режим плавающего окна, как показано ниже:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/0db067cde22c11f095ab5254001c06ec.png)

#### Эффект плавающего окна вне приложения

Нажмите **Ещё** > **PIP** внизу страницы просмотра, чтобы включить Picture-in-Picture. Когда ваше приложение переходит в фоновый режим, оно автоматически войдёт в системный режим Picture-in-Picture:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/602e0c95e22c11f08775525400bf7822.png)

## Настройка макета пользовательского интерфейса

### Настройка текста (ARB)

TUILiveKit использует **файлы ARB** и **стандартное решение для интернационализации (i18n) Flutter** для управления отображением текста пользовательского интерфейса. Вы можете напрямую отредактировать файлы ARB в каталоге `livekit/lib/common/language/i10n/` для изменения текста:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/f4575de09a0911f091df5254005ef0f7.png)

- `livekit_en.arb`: Английский текст.
- `livekit_zh.arb`: Упрощённый китайский текст.
- `livekit_zh_Hant.arb`: Традиционный китайский текст.

После редактирования запустите `flutter gen-l10n` в терминале, чтобы переработать код локализации.

### Настройка значков

Ресурсы изображений пользовательского интерфейса TUILiveKit хранятся в каталоге `livekit/assets/images/`. Чтобы обновить значки, просто замените файлы PNG в этой папке.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/fe57fa0798f511f0961e52540099c741.png)

Пересоберите и запустите приложение, и вы увидите обновленные значки.

## Часто задаваемые вопросы

### Почему видеоэкран становится чёрным после того, как аудитория выбирает видеовзаимодействие?

Перейдите в **Параметры телефона** > **Приложение** > **Камера** и проверьте, включено ли разрешение на доступ к камере. См. изображение ниже:

| iOS | Android |
| --- | --- |
| ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/327c4be8991f11f0a3b8525400e889b2.png) | ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/32793103991f11f08bb25254005ef0f7.png) |

### Почему другие члены аудитории не видят комментарии прямой трансляции, отправленные членом аудитории?

- **Причина 1**: Сначала проверьте подключение к сети, чтобы убедиться, что сетевое соединение устройства зрителя включено и стабильно.
- **Причина 2**: Зритель был **отключен (заблокирован)** хостом и не может отправлять комментарии прямой трансляции.
- **Причина 3**: Комментарий прямой трансляции может содержать запрещённые ключевые слова. Убедитесь, что комментарии соответствуют правилам комнаты прямой трансляции.

### Почему приложение не может входить в режим плавающего окна вне приложения (Picture-in-Picture) в фоновом режиме?

Обратитесь к [Входу в режим плавающего окна вне приложения](#c28b2363-6d3e-40e8-be0d-4e4dba28a665). После входа в комнату прямой трансляции включите переключатель Picture-in-Picture.

На Android после включения Picture-in-Picture приложение автоматически проверит системные параметры. Если Picture-in-Picture не включен, приложение предложит вам разрешить этот режим в системных параметрах:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/e64f5d4de22b11f08775525400bf7822.png)

### Почему всплывающие окна не отображаются или закрыты?

Если вы интегрировали `TUILiveRoomAudienceOverlay`, убедитесь, что всплывающие компоненты (такие как BottomSheet, Dialog и т. д.) используют контекст `rootNavigator` (рекомендуется: `Global.appContext()`).

**Причина:** Overlay страницы прямой трансляции отображается на `secondaryNavigator`. Всплывающие окна, использующие контекст `secondaryNavigator`, будут скрыты Overlay, потому что на одном и том же Navigator слой Overlay находится выше обычных страниц, позволяя ему плавать над ними. Использование `rootNavigator` гарантирует, что всплывающие окна отображаются правильно над Overlay.

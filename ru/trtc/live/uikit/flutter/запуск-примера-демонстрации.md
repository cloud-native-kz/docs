# Запуск примера демонстрации

Это руководство проведет вас через начало работы с демонстрацией `TUILiveKit` Flutter всего за 10 минут. Получите полнофункциональный пользовательский интерфейс для прямых видеотрансляций и сценариев голосовых чат-комнат.

## Предварительные требования

### Активация сервиса

Обратитесь к [Активация сервиса](https://www.tencentcloud.com/document/product/647/60033) для получения пробной версии TUILiveKit. Затем перейдите в [Консоль](https://console.trtc.io/app) для управления приложением и получите следующее:

- `SDKAppID`: идентификатор приложения (обязательно). Tencent Cloud использует `SDKAppId` для выставления счетов и сведений.
- `SDKSecretKey`: секретный ключ приложения, используется для инициализации файла конфигурации с конфиденциальной информацией.

### Подготовка окружения

Убедитесь, что ваша среда разработки соответствует следующим требованиям перед запуском демонстрации:

| **Платформа** | **Описание** |
| --- | --- |
| **Flutter** | Flutter: версия 3.29.3 или позжеDart: версия 3.7.2 или позже |
| **Android** | Android Studio: версия 3.5 или позжеAndroid OS: устройства под управлением Android 5.0 или позжеJava Version: требуется JDK 17 |
| **iOS** | Xcode: версия 15 или позжеiOS: устройства под управлением iOS 13.0 или позжеCocoaPods: установлена среда CocoaPods. Если не установлена, см. [Официальное руководство по установке CocoaPods](https://guides.cocoapods.org/using/getting-started.html). |

### Конфигурация среды IDE

Для эффективной разработки на Flutter рекомендуется использовать Visual Studio Code в качестве IDE, как рекомендовано в документации Flutter. См. [Visual Studio Code](https://docs.flutter.cn/tools/vs-code) для инструкций по установке.

## Инструкции

### Получение демонстрации

1. **Загрузка исходного кода**: загрузите исходный код [TUIKit_Flutter](https://github.com/Tencent-RTC/TUIKit_Flutter.git) с GitHub или выполните следующую команду в терминале:

```
git clone https://github.com/Tencent-RTC/TUIKit_Flutter.git
```

2. **Установка зависимостей**: используйте терминал для навигации в каталог, содержащий файл `pubspec.yaml` для примера проекта, затем запустите flutter pub get для установки зависимостей:

```
# Файл pubspec.yaml для примера проекта находится в TUIKit_Flutter/applicationcd TUIKit_Flutter/application# Установка зависимостейflutter pub get
```

3. **Установка зависимостей iOS (только iOS)**: используйте терминал для навигации в каталог, содержащий ваш `Podfile`, затем выполните следующую команду для установки требуемых зависимостей:

```
# Podfile для вашего проекта iOS находится в каталоге application/ios.cd application/ios# Установка зависимостейpod install --repo-update
```

4. **Очистка глобальной конфигурации Gradle (только Android)**: если у вас локально есть **~/.gradle/init.gradle**, выполните:

```
rm -f ~/.gradle/init.gradle
```

> **Примечание:** если у вас на локальной машине есть глобальный файл конфигурации `Gradle` в `~/.gradle/init.gradle`, который включает `allprojects.repositories`, следуйте этим шагам: **Вариант 1 (Рекомендуется)**: удалите любую конфигурацию, связанную с `allprojects.repositories`, из файла. **Вариант 2**: полностью удалите файл `~/.gradle/init.gradle`. **Причина**: эта конфигурация несовместима с параметром `RepositoriesMode.FAIL_ON_PROJECT_REPOS` в более новых версиях Gradle и приведет к [ошибкам выполнения](#709d982d-f012-46e8-9b21-b993ca4c591a).

5. **Конфигурация JDK 17 (только Android)**: в `gradle.properties` под `application/android` добавьте `org.gradle.java.home` и установите его на путь вашего **JDK** 17:

```
# Замените на ваш фактический путь JDK 17org.gradle.java.home=xx/Java/JavaVirtualMachines/jdk-17.0.6.jdk/Contents/Home
```

6. После завершения этих шагов откройте каталог `TUIKit_Flutter/application` в **Visual Studio Code**.

### Настройка

1. **Конфигурация `sdkAppId` и `secretKey`**: в проекте **livekit** введите ваши `sdkAppId` и `secretKey` в файл `application/lib/debug/generate_test_user_sig.dart`:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/9150ed56bf9a11f08863525400e889b2.png)

> **Примечание:** в этой демонстрации аутентификация осуществляется путем конфигурации `secretKey` в коде клиента. `secretKey` может быть декомпилирован и скомпрометирован. При утечке злоумышленники могут украсть ваш трафик Tencent Cloud. Этот подход подходит только для локального запуска демонстрации и отладки функций. Для продакшена генерируйте `UserSig` на вашем сервере. Когда требуется `UserSig`, ваше приложение должно запросить динамический `UserSig` на вашем бэкенде для аутентификации. См. [Генерация UserSig на сервере](https://www.tencentcloud.com/document/product/647/35166) для получения подробной информации.

2. **Конфигурация подписания Apple Developer (только iOS)**: в **TARGETS** проекта на вкладке **Signing & Capabilities** выберите Automatically manage signing и настройте вашу **учетную запись Apple Developer** и Bundle Identifier:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/79fdc843bf9a11f0a808525400bf7822.png)

### Сборка и запуск демонстрации

Прямая трансляция требует доступа к камере и микрофону. Рекомендуется запускать и отлаживать демонстрацию на физическом устройстве.

Если вы встретили ошибку выполнения `'Error resolving plugin [id: 'dev.flutter.flutter-plugin-loader', version: '1.0.0']'`, пожалуйста, обратитесь к [руководству по устранению ошибок выполнения](#709d982d-f012-46e8-9b21-b993ca4c591a) и попробуйте снова.

## Часто задаваемые вопросы

### Метод установки CocoaPods?

**Установите CocoaPods с помощью gem**: запустите `sudo gem install cocoapods` в терминале.

> **Совет:** во время `sudo gem install cocoapods` вас могут попросить пароль компьютера. Введите пароль администратора по указанию.

### Отладка на реальном устройстве?

Включите режим разработчика для запуска на реальном устройстве.

- **iOS**: подключите устройство к компьютеру, затем перейдите в `Settings > Privacy & Security > Developer Mode` и включите режим разработчика.
- **Android**: включите режим разработчика, нажав несколько раз на номер сборки в параметрах. Если вы не можете его найти, обратитесь на официальный веб-сайт вашего устройства или поищите инструкции в сети.

### Получение ошибки подписи или ошибки входа при запуске демонстрации?

Убедитесь, что `sdkAppId` и `secretKey`, введенные в `application/lib/debug/generate_test_user_sig.dart`, верны и совпадают со значениями со страницы [Консоль Tencent RTC > Управление приложениями](https://console.trtc.io/app).

### Ошибка выполнения: Error resolving plugin [id: 'dev.flutter.flutter-plugin-loader', version: '1.0.0']

Если вы встретили следующее сообщение об ошибке во время выполнения:

```
* What went wrong:Error resolving plugin [id: 'dev.flutter.flutter-plugin-loader', version: '1.0.0']> Build was configured to prefer settings repositories over project repositories but repository 'xxx' was added by settings file 'settings.gradle'
```

Обратитесь к шагу 4 в [Получение демонстрации](#f1199140-253f-43b6-a562-a0a9c7a9923d) для очистки глобальной конфигурации Gradle.

### Встреча кода ошибки "9519" или "9523" с сообщением "The request packets timed out while waiting to be sent..." при входе?

- Пожалуйста, сначала попробуйте изменить вашу текущую сетевую среду и попытайтесь войти снова.
- Если проблема сохраняется даже после изменения сетевой среды, [свяжитесь с нами](https://trtc.io/contact) для поддержки.

### Встреча кода ошибки "-3301" с сообщением "Services are not available in your region" после входа в комнату?

Если вы получили этот код ошибки, [свяжитесь с нами](https://trtc.io/contact) для поддержки.

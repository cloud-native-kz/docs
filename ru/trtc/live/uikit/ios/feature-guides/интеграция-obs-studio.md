# Интеграция OBS Studio

## Введение функции

Push Media Stream Into Room — это технология, которая проталкивает потоки **RTMP** в комнаты прямого эфира. Вы можете использовать инструменты такие как **OBS** или **FFMPEG** для передачи предварительно записанных видео, генерируемых в реальном времени визуальных элементов или других онлайн-видеопотоков (таких как видео с камер наблюдения, игровые экраны, сцены встреч и т. д.) в комнату прямого эфира, имитируя эффект трансляции в реальном времени.

> **Примечание:** Соответствующие сборы выглядят следующим образом: Разблокировка функции: функция **Push Media Stream Into Room** требует подписки на [ежемесячный пакет Live](https://www.tencentcloud.com/document/product/647/59407) **Standard** или **Pro** для разблокировки (пробная версия также позволяет использовать эту функцию). Плата за использование: использование функции потоковой передачи будет выполнять операции транскодирования, что приводит к расходам на транскодирование. Подробнее см. [Инструкции по выставлению счетов за облачное микширование и транскодирование](https://www.tencentcloud.com/document/product/647/47631). Плата за продолжительность звука будет взиматься за потоковые чат-боты в комнате (примечание: плата за чат-ботов в комнате, созданных функцией ввода онлайн-медиапотока, будет отменена до 15 августа 2024 г. и будет взиматься начиная с 16 августа 2024 г.). Примечание: платежи за аудио и видео вызовы будут взиматься, когда аудитория в комнате подписывается на аудио и видеоконтент, переданный в комнату. Подробнее см. [Инструкции по выставлению счетов за продолжительность аудио и видео](https://www.tencentcloud.com/document/product/647/42734).

## Варианты использования

Потоковая передача робота предназначена для разнообразных потребностей прямой трансляции. Вы можете реализовать следующие варианты использования:

| Тип сценария | Описание |
| --- | --- |
| Непрерывная 24/7 прямая трансляция | С потоковой передачей робота вы можете достичь 24-часовой непрерывной прямой трансляции, которая подходит для сценариев, таких как трансляция новостей, музыкальные радиостанции и видео наблюдения. |
| Воспроизведение контента | Повторно передавайте популярный контент прямой трансляции или записанные видео, чтобы привлечь больше зрителей. |
| Виртуальный стример | Объедините технологию аватаров и используйте потоковую передачу робота для реализации прямой трансляции виртуального стримера. |
| Корпоративная прямая трансляция | Используется для корпоративных встреч, мероприятий запуска продуктов, обучения и других сценариев для трансляции записанных видео на внутренние или внешние платформы. |
| Прямая трансляция игр | Передавайте видеопоток игры в комнату прямой трансляции. Объедините его с комментариями или фоновой музыкой, чтобы улучшить эффекты прямой трансляции. |
| Образовательная прямая трансляция | Передавайте записанное видео курса на платформу образования для реализации онлайн-обучения. |
| Электронная коммерция прямая трансляция | Показывайте информацию о продуктах и рекламные акции с помощью потоковой передачи робота для повышения коэффициента конверсии продаж. |
| Другие сценарии | Любой режим игры с реальным взаимодействием на основе медиапотока может быть реализован через потоковую передачу RTMP. Вас ждут новые режимы игры для исследования. |

## Руководство пользователя

Ниже в качестве примеров будут использованы **REST API** и инструмент **OBS** для быстрого ознакомления с функцией потоковой передачи робота.

### Потоковая передача робота

> **Примечание:** Вам необходимо реализовать потоковую передачу робота, следуя этапам **создание комнаты прямой трансляции > создание робота > использование OBS для передачи потока RTMP в комнату прямой трансляции > робот становится спикером**.

#### Этап 1: Создание комнаты прямой трансляции

Вы можете создать комнату прямой трансляции через REST API [Create a live streaming room](https://www.tencentcloud.com/document/product/647/63401).

Если ID владельца комнаты не существует, вам нужно будет использовать REST API [Import Multiple Accounts](https://www.tencentcloud.com/document/product/647/69888). (Рекомендуется установить Host ID как ID робота.)

#### Этап 2: Создание робота

Когда в комнате прямой трансляции нет людей в течение определенного времени, она автоматически растворится. Чтобы убедиться, что ваша комната не растворилась, вам нужно создать робота внутри комнаты.

Вы можете создать робота через REST API [create a robot](https://www.tencentcloud.com/document/product/647/69079). Если ID робота не существует, вам нужно будет использовать REST API [Import Multiple Accounts](https://www.tencentcloud.com/document/product/647/69888).

#### Этап 3: Использование OBS для передачи потока RTMP в комнату прямой трансляции

Здесь вы получите инструкции по использованию инструмента **OBS** для передачи потока **RTMP** в комнату прямой трансляции. Вы можете обратиться к документу [RTMP stream push to the live streaming room](https://www.tencentcloud.com/document/product/647/62716#OBS-.E6.8E.A8.E6.B5.81.E8.AE.BE.E7.BD.AE).

> **Примечание:** ID комнаты при передаче потока устанавливается на ID комнаты, который вы установили при создании комнаты прямой трансляции на **Этап 1**. userId при передаче потока устанавливается на userId робота, который вы создали на **Этап 2**. При создании бота, если userId бота не существует, вам необходимо сначала использовать REST API [Import Multiple Accounts](https://www.tencentcloud.com/document/product/647/69888) для импорта аккаунта.

#### Этап 4: Робот берет микрофон

Зрители прямой трансляции онлайн автоматически будут получать аудио и видеопоток спикера, который говорит в микрофон. Чтобы убедиться, что зрители в вашей комнате получают аудио и видеопоток, переданные через RTMP, вам нужно поставить робота в состояние говорения.

Вы можете позволить роботу взять микрофон через REST API [take the mic](https://www.tencentcloud.com/document/product/647/64428).

> **Примечание:** Если вы хотите настроить размер и позицию аудио и видеопотока, который зрители видят в комнате прямой трансляции, вы можете достичь это, используя REST API [custom layout API](https://www.tencentcloud.com/document/product/647/69077).

### Остановка потоковой передачи робота

> **Примечание:** Вам необходимо реализовать остановку потоковой передачи робота, следуя этапам **робот становится слушателем > удаление робота > закрытие комнаты прямой трансляции > остановка потоковой передачи RTMP**.

#### Этап 1: Робот становится слушателем

Вы можете позволить роботу стать слушателем через REST API [become a listener](https://www.tencentcloud.com/document/product/647/64429).

#### Этап 2: Удаление робота

Вы можете удалить робота через REST API [delete a robot](https://www.tencentcloud.com/document/product/647/69080).

#### Этап 3: Закрытие комнаты прямой трансляции

Когда вы хотите заранее закрыть комнату прямой трансляции, вы можете закрыть комнату прямой трансляции через REST API [dismiss a live streaming room](https://www.tencentcloud.com/document/product/647/63406).

#### Этап 4: Остановка потоковой передачи RTMP в комнату прямой трансляции

Нажмите кнопку остановки прямой трансляции, чтобы остановить передачу в комнату прямой трансляции.

## Часто задаваемые вопросы

### **Что делать, если передача не удалась?**

Распространенные причины: отсутствие приобретенного пакета или его истечение; ошибка подписи или истечение подписи; передача B-кадров (явление на приборной панели — "потоковая передача заканчивается через одну секунду"), может быть установлено базовое кодирование.

Другие причины: если это встроенное аппаратное устройство, передающее поток, URL может быть обрезан; если передается H.265, переключитесь на H.264; если набор chunk на терминале слишком большой, измените на 1360.

### **Что делать, если есть задержка или эффект мозаичного экрана?**

Посмотрите на [приборную панель консоли](https://www.tencentcloud.com/document/product/647/39070) Tencent Cloud Real-Time Communication, чтобы проверить, является ли частота кадров потоковой передачи стабильной. Если она стабильна, это, вероятно, проблема на стороне воспроизведения; проверьте сторону воспроизведения. **Если частота кадров нестабильна**, рассмотрите следующие моменты:

- Проверьте, находится ли локальный процессор и память клиента передачи под высокой нагрузкой. Если для потоковой передачи используется OBS, мониторьте панель состояния внизу программы на предмет информации о потере кадров, сети, процессоре и частоте кадров.
- Проверьте, достаточна ли полоса пропускания локальной сети. Выполните ping доменного имени потоковой передачи, чтобы наблюдать RTT; используйте [сетевой инструмент диагностики](https://itango.tencent.com/app/data/huatuo) для обнаружения доменного имени потоковой передачи и проверьте пропускную способность, которая в идеале должна достичь 10M.
- Сторона передачи может попытаться снизить битрейт и частоту кадров, чтобы снизить нагрузку на клиент. Для справки см. настройки OBS в тексте. Рекомендуемый битрейт для 720p составляет 1500 Кбит/с.

### **Что делать, если задержка высокая?**

Если сторона вытягивания использует роль якоря, задержка обычно ниже, чем у роли аудитории. Если это не роль якоря, вы можете попробовать сравнить, улучшится ли. 

Локальное кодирование и влияние сети на сторону потоковой передачи относительно велики. Вы можете попробовать тестирование на разных платформах. Если используется OBS, вы можете попробовать потоковую передачу в системах Windows; выполните ping доменного имени потоковой передачи, чтобы наблюдать RTT.

### **Что делать, если потоковая передача RTMP прерывается?**

Если вы возобновите потоковую передачу RTMP в течение 2 минут после прерывания, зрители в той же комнате снова будут получать аудио и видеопоток. Если пройдет более 2 минут, смешанный поток, просматриваемый зрителями, будет прерван. Вам нужно вызвать [resume mcu API](https://www.tencentcloud.com/document/product/647/72333) для восстановления смешанного потока.

### **Как установить макет для потоковой передачи?**

Вы можете использовать REST API для установки макета микширования потока комнаты для реализации этой функции. Когда LayoutMode установлен на 0, это макет сетки 3x3 по умолчанию. Когда LayoutMode установлен на 1000, это пользовательский макет. Вы можете установить размер видео и позицию для стороны зрителя, установив эти два поля: VideoEncode и LayoutInfo.

### **Что делать, если вы не видите соответствующую комнату робота в списке онлайн-прямых трансляций?**

**Ответ:** Когда вы используете REST API для создания комнаты прямой трансляции, установите поле IsPublicVisible на true.

### Прослушивание прерываний потока OBS?

- OBS входит в комнату TRTC при потоковой передаче и запускает событие обратного вызова входа в комнату TRTC.
- OBS покидает комнату TRTC при прерывании потока, запуская событие обратного вызова выхода из комнаты TRTC.

Вы можете мониторить потоковую передачу OBS и события прерывания потока, слушая [обратный вызов комнаты на стороне сервера](https://www.tencentcloud.com/document/product/647/39558#event-type).

| Поле | Значение | Описание |
| --- | --- | --- |
| EVENT_TYPE_ENTER_ROOM | 103 | Вход в комнату |
| EVENT_TYPE_EXIT_ROOM | 104 | Выход из комнаты |

> **Примечание:** Рекомендуется использовать унифицированный формат для userId робота. Деловая сторона должна определить, является ли это соответствующим роботом после получения обратного вызова входа/выхода из комнаты, прежде чем обрабатывать соответствующую логику.

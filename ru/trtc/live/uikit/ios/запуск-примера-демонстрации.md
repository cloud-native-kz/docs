# Запуск примера демонстрации

Это руководство проведет вас через начало работы с демонстрацией `TUILiveKit` для iOS всего за 10 минут. Испытайте полнофункциональный интерфейс для сценариев прямой трансляции видео и чат-комнат голосовой связи.

## Быстрый опыт

Вы можете напрямую испытать все функции **Live**, используя следующие методы.

## Предварительные требования

### Включение сервиса

Обратитесь к [Включение сервиса](https://www.tencentcloud.com/document/product/647/60033), чтобы получить пробную версию TUILiveKit. Затем перейдите в [Консоль](https://console.trtc.io/app) для управления приложением и получите следующее:

- `SDKAppID`: Идентификатор приложения (обязательно). Tencent Cloud использует `SDKAppId` для выставления счетов и деталей.
- `SDKSecretKey`: Ключ секрета приложения, используется для инициализации файла конфигурации с конфиденциальной информацией.

### Подготовка окружения

Убедитесь, что ваша среда разработки соответствует этим требованиям:

- **Xcode**: Xcode 15 или более поздняя версия
- **iOS System**: Устройства с iOS 13.0 или более поздней версией
- **CocoaPods**: CocoaPods должен быть установлен. Если нет, обратитесь к [Официальному руководству по установке CocoaPods](https://guides.cocoapods.org/using/getting-started.html) или выполните:
  - Установите CocoaPods через gem:

```
sudo gem install cocoapods
```

> **Совет:** Во время `sudo gem install cocoapods` вам может быть предложено ввести пароль компьютера. Введите пароль администратора в соответствии с инструкциями.

## Инструкции

### Получение демонстрации

1. **Загрузите исходный код**: Загрузите исходный код [TUIKit_iOS](https://github.com/Tencent-RTC/TUIKit_iOS) с GitHub или выполните:

```
git clone https://github.com/Tencent-RTC/TUIKit_iOS.git
```

2. **Установите зависимости**: В вашем `терминале` перейдите в директорию, содержащую `Podfile`, и выполните:

```
# Podfile проекта Demo находится в директории приложенияcd application/# Установите зависимостиpod install --repo-update
```

3. **После установки** вы увидите директорию `Pods` и файл `App-UIKit.xcworkspace` в директории вашего проекта. Дважды щелкните `App-UIKit.xcworkspace` для открытия проекта.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/2530c0c5bf7b11f0b4c35254001c06ec.png)

### Настройка

1. **Настройте** `SDKAppID` **и** `SDKSecretKey`: В проекте `App-UIKit.xcworkspace` введите ваши `SDKAppID` и `SDKSecretKey` в файле `Debug/GenerateTestUserSig.swift`.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/5acccb7fbf7b11f08863525400e889b2.png)

> **Примечание:** В этой демонстрации аутентификация выполняется путем настройки `SecretKey` в коде клиента. `SecretKey` можно подвергнуть обратной инженерии; если он будет скомпрометирован, злоумышленники смогут украсть ваш трафик Tencent Cloud. Этот метод предназначен только для локального тестирования и отладки демонстрации. Для производства генерируйте UserSig на вашем сервере. Когда потребуется, ваше приложение должно запросить динамический UserSig с вашего бизнес-сервера. Подробнее см. [Генерация UserSig на сервере](https://www.tencentcloud.com/document/product/647/35166).

2. **Настройте подпись разработчика Apple**: В **TARGETS** проекта на вкладке **Signing & Capabilities** включите **Automatically manage signing** и установите ваш **Apple Developer account** и Bundle Identifier.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/6efb9271bf9a11f0a5e052540099c741.png)

### Сборка и запуск демонстрации

Сценарии прямой трансляции требуют доступа к камере и микрофону. Используйте физическое устройство для отладки и запуска демонстрации.

1. Подключите ваше устройство iOS к компьютеру. На устройстве перейдите в `Settings > Privacy & Security > Developer Mode` и включите Developer Mode.
2. В Xcode выберите ваше устройство из выпадающего списка **iOS Device**.
3. Нажмите кнопку Run, чтобы развернуть приложение на вашем устройстве.

> **Примечание:** Для полного рабочего процесса прямой трансляции запустите демонстрацию на **двух устройствах** и войдите с **двумя разными пользователями** (например, один как хост, один как аудитория).

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/3c9e53e8bf9a11f08863525400e889b2.png)

Используйте отличительные UserID при отладке.

## Часто задаваемые вопросы

### При запуске демонстрации возникают ошибки подписи или сбои входа

Убедитесь, что `SDKAppID` и `SDKSecretKey` в `Debug/GenerateTestUserSig.swift` правильны и совпадают с теми, что на странице [Управление приложением Tencent Cloud Console](https://console.tencentcloud.com/trtc).

### После входа в комнату возникает код ошибки "-3301" с сообщением "Services are not available in your region"

Если вы получите этот код ошибки, [свяжитесь с нами](https://trtc.io/contact) для поддержки.

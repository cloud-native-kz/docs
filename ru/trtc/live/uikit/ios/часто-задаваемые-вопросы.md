# Часто задаваемые вопросы

### Ошибка компилятора Xcode 15?

1. **Sandbox: rsync** отображается.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/11f98d2f0f6111efb8f85254005a7266.png)

Вы можете установить User Script Sandboxing на **NO** в **Build Settings:**

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1142dcdb0f6111ef98aa525400493f3c.png)

2. Если **SDK does not contain**, снимок экрана ошибки компиляции:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/11abe9e10f6111efbab15254000ded98.png)

Добавьте следующий код в Podfile:

```
post_install do |installer|  installer.pods_project.targets.each do |target|    target.build_configurations.each do |config|      config.build_settings['IPHONEOS_DEPLOYMENT_TARGET'] = '13.0'    end  endend
```

3. Если вы запускаете эмулятор на компьютере с процессором M-series, может появиться **Linker command failed with exit code 1 (use-v to see invocation)**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1187403c0f6111ef9f885254000781d8.png)

Конфигурация xcode требует изменения. xcode open projects > **Product > Destination > Destination Architectures** позволяет выбрать, в каком режиме открывать эмулятор, и необходимо выбрать эмулятор с окончанием (**Rosetta**).

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/114df26e0f6111efbab15254000ded98.png)

### Есть ли конфликт между TUILiveKit и интегрированной библиотекой аудио и видео?

Библиотеки аудио и видео Tencent Cloud не могут быть интегрированы одновременно и могут возникнуть конфликты символов. Вы можете решить проблему в соответствии со следующими сценариями.

1. Если вы используете библиотеку `TXLiteAVSDK_TRTC`, конфликтов символов не будет. Вы можете напрямую добавить зависимости в файл Podfile.

```
pod 'TUILiveKit'
```

2. Если вы используете библиотеку `TXLiteAVSDK_Professional`, будут конфликты символов. Вы можете добавить зависимости в файл `Podfile`.

```
pod 'TUILiveKit/Professional'
```

Если вы используете библиотеку `TXLiteAVSDK_Enterprise`, будут конфликты символов. Рекомендуется обновиться до `TXLiteAVSDK_Professional`, а затем использовать TUILiveKit/Professional.

### Как просмотреть логи TRTC?

Логи TRTC по умолчанию сжимаются и шифруются с расширением .xlog. Возможность шифрования лога можно контролировать с помощью setLogCompressEnabled. Имя файла, содержащее C (сжатый), зашифровано и сжато, а имя файла, содержащее R (raw), открыто.

iOS: Sandbox's `Documents/log`

> **Примечание:** Для просмотра файла .xlog необходимо загрузить [инструмент расшифровки](https://dldir1.qq.com/hudongzhibo/log_tool/decode_mars_log_file.py) и запустить его непосредственно в среде Python 2.7 с файлом xlog в той же директории, используя python decode_mars_log_file.py. Для просмотра файла .clog (новый формат журнала после версии 9.6) необходимо загрузить [инструмент расшифровки](http://dldir1.qq.com/hudongzhibo/log_tool/decompress_clog.py) и запустить его непосредственно в среде Python 2.7 с файлом clog в той же директории, используя python decompress_clog.py.

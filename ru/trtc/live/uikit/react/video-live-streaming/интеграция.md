# Интеграция

## Обзор функций

TUILiveKit React — это многофункциональный веб-компонент для прямой трансляции, который предоставляет комплексные возможности прямой трансляции «из коробки»:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/100c509f116f11f18b07525400ecee81.png)

## **Подготовка**

### Активация сервиса

Перед использованием TUILiveKit необходимо сначала активировать сервис. Смотрите [Активация сервисов](https://trtc.io/document/60033?product=live&menulabel=uikit&platform=web), чтобы получить пробную версию или активировать платную версию.

### Требования к окружению

- **Устройства**: Камера, микрофон и динамики

### Требования к конфигурации

- **Node.js**: ≥ 18.19.1 (рекомендуется официальная версия LTS, проверьте текущую версию командой `node -v`)
- **react**: ≥ 18.2.0 и < 19.0.0
- **vite**: ≥ 5.4.0
- **typescript**: ≥ 5.8.3
- **sass**: ≥ 1.77.0

### Шаг 2: Завершение входа

Завершение входа — это основа для использования **TUILiveKit**. Функции TUILiveKit могут использоваться нормально только после успешного входа. Убедитесь, что все параметры настроены правильно перед продолжением.

> **Примечание:** Хотя в образце кода показаны прямые вызовы входа, **мы настоятельно рекомендуем вызывать сервис входа TUILiveKit только после завершения вашего собственного потока аутентификации пользователя**. Это предотвращает конфликты бизнес-логики или проблемы с несогласованностью данных и лучше интегрируется с вашей существующей системой управления пользователями и разрешениями.

```
// Best practice: Call this login logic immediately after your authentication flow completesimport { useLoginState } from 'tuikit-atomicx-react';const { login, logout } = useLoginState();async function initLogin() {  try {    await login({      SDKAppID: 0,        // SDKAppID - SDKAppID obtained when activating service      userID: '',         // UserID   - User ID      userSig: '',        // userSig  - User signature, see parameter description below for detailed instructions    });  } catch (error) {    console.error('Login failed:', error);  }}
```

**Описание параметров API входа**

| Параметр | Тип | Описание |
| --- | --- | --- |
| SDKAppID | Int | Получите значение в [консоли TRTC > Управление приложениями](https://console.trtc.io/app). |
| UserID | String | Уникальный идентификатор текущего пользователя. Может содержать только буквы (a-z, A-Z), цифры (0-9), дефисы (-) и подчеркивания (_). |
| userSig | String | Билет для аутентификации TRTC:**Среда разработки**: Используйте `GenerateTestUserSig.genTestSig` локально или [инструмент генерации UserSig](https://console.trtc.io/usersig) для генерации временных учетных данных.**Production**: Генерируйте UserSig на стороне сервера, чтобы предотвратить раскрытие ключей. Смотрите [генерацию UserSig на стороне сервера](https://www.tencentcloud.com/document/product/647/69883). |

## Часто задаваемые вопросы

### Нужно ли мне вызывать login каждый раз при входе в комнату?

Нет. Обычно нужно вызвать `login()` только один раз. Мы настоятельно рекомендуем интегрировать функции `login()` и `logout()`, возвращаемые `useLoginState()`, с системой аутентификации вашего приложения.

### Почему HTTPS требуется для production развертывания?

При развертывании приложения на production вы должны использовать **HTTPS домен**. TUILiveKit использует технологию WebRTC и требует доступа к камерам, микрофонам и динамикам пользователей. Браузеры требуют HTTPS для защиты конфиденциальности и безопасности пользователя при доступе к этим чувствительным устройствам.

# Интеграция

## Обзор функций

**TUILiveKit** — это комплексный компонент для прямой трансляции. После интеграции он позволяет быстро реализовать следующие функциональные модули:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1728c8fe978f11f0ad595254007c27c5.png)

## Подготовка

### Активация сервиса

Перед использованием TUILiveKit необходимо [активировать сервисы](https://trtc.io/document/60033?product=live&menulabel=uikit&platform=web) в консоли Tencent Cloud и получить пробную версию или активировать платную версию.

### Требования к окружению

- **Vue 3 + Composition API:** Используйте последние возможности Vue 3 для создания высокопроизводительных и поддерживаемых приложений.
- **TypeScript:** Повысьте качество кода и эффективность разработки с помощью статической проверки типов.
- **SCSS Module:** Реализуйте модульное управление стилями, чтобы эффективно избежать конфликтов стилей.

### Интеграция кода

#### Шаг 1: Установка зависимостей

Вы можете установить зависимости любым из следующих способов:

npm

pnpm

yarn

```
npm install tuikit-atomicx-vue3 @tencentcloud/uikit-base-component-vue3 --save
```

```
pnpm install tuikit-atomicx-vue3 @tencentcloud/uikit-base-component-vue3
```

```
yarn install tuikit-atomicx-vue3 @tencentcloud/uikit-base-component-vue3
```

#### Шаг 2: Завершение входа

```
import { useLoginState } from 'tuikit-atomicx-vue3';const { login } = useLoginState();async function initLogin() {  try {    await login({      sdkAppId: 0,        // SDKAppId, см. шаг 1 для получения      userId: '',         // UserID, см. шаг 1 для получения      userSig: '',        // userSig, см. шаг 1 для получения    });  } catch (error) {    console.error('Login Error:', error);  }}
```

**Описание параметров API входа**

| Параметр | Тип | Описание |
| --- | --- | --- |
| SDKAppID | Int | Получите из [консоли TRTC > Управление приложениями](https://console.trtc.io/app). |
| UserID | String | Уникальный идентификатор текущего пользователя. Может содержать только буквы (a-z, A-Z), цифры (0-9), дефисы (-) и подчеркивания (_). |
| userSig | String | Билет для аутентификации TRTC:**Среда разработки**: Используйте `GenerateTestUserSig.genTestSig` локально или [инструмент генерации UserSig](https://console.trtc.io/usersig) для создания временных учетных данных.**Production**: Генерируйте UserSig на стороне сервера, чтобы предотвратить утечку ключа. См. [генерацию UserSig на стороне сервера](https://www.tencentcloud.com/document/product/647/69883). |

#### Шаг 3: Функциональный опыт

Поздравляем! Вы успешно интегрировали компонент видеотрансляции и выполнили вход. Далее вы можете начать трансляцию, позволить зрителям смотреть или реализовать другие функции прямой трансляции. Подробнее см. таблицу ниже.

| **Функция** | **Описание** | **Ссылка на опыт** |
| --- | --- | --- |
| **Трансляция хоста** | Поддерживается весь процесс трансляции ведущего прямой трансляции, включая подготовку перед трансляцией и взаимодействие после трансляции. | [Трансляция хоста](https://www.tencentcloud.com/document/product/647/73741) |
| **Просмотр аудиторией** | Зрители могут войти в комнату ведущего прямой трансляции для просмотра, подключиться к микрофону, просмотреть информацию о комнате, посмотреть активных зрителей и отображать комментарии. | [Просмотр аудиторией](https://www.tencentcloud.com/document/product/647/73747) |
| **Набор инструментов для прямой трансляции** | Предоставляется комплексный набор готовых к использованию интерфейсов и функций прямой трансляции, включая управление источником видео, подключение микрофона, взаимодействие с аудиторией, прямую трансляцию в альбомной и портретной ориентации, а также управление зрителями. | [Набор инструментов для прямой трансляции](https://www.tencentcloud.com/document/product/647/73792) |
| **Список прямых трансляций** | Интерфейс списка прямых трансляций и функции включают список прямых трансляций и отображение информации о комнате. | [Список прямых трансляций](https://www.tencentcloud.com/document/product/647/73760) |

### Часто задаваемые вопросы

Если вам необходимо развернуть упакованный файл dist проекта, в производственной среде необходимо использовать доменное имя HTTPS.

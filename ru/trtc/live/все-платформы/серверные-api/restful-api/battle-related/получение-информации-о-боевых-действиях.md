# Получение информации о боевых действиях

## Описание функции

Администратор приложения может получить сведения о боевых действиях через этот API.

## Описание вызова API

### Пример URL запроса:

```
https://xxxxxx/v4/live_engine_http_pk/get_battle_info?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Описание параметров запроса

В следующей таблице перечислены только параметры, задействованные при вызове этого API, и их описания.

| Параметры | Описание |
| --- | --- |
| xxxxxx | Эксклюзивное доменное имя, соответствующее стране/региону SDKAppID: Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` США: `adminapiusa.im.qcloud.com` |
| v4/live_engine_http_pk/get_battle_info | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Должен быть аккаунтом администратора приложения. Для получения подробной информации см. [App Admin](https://www.tencentcloud.com/document/product/647/69882#app-admin) |
| usersig | Подписи, созданные аккаунтом администратора приложения. Для получения конкретных инструкций см. [Generate UserSig](https://www.tencentcloud.com/document/product/647/69883#) |
| random | Введите случайное 32-битное целое число без знака, диапазон значений 0 - 4294967295 |
| contenttype | Формат запроса имеет фиксированное значение `json`. |

### Максимальная частота вызовов

200 раз/секунду.

### Пример пакета запроса:

**Базовая форма**

```
{     "BattleId": "HuYM036AMk2JJeSUgnv5H0KDJpc+HXfqTAtGuYWzRPs=",      "RoomId": "room_pk1" //When the business side does not know the specific BattleId, you can fill in the RoomId, so that you can obtain the BattleId information of the ongoing RoomId.}
```

### Описание полей пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| BattleId | String | Обязательно | ID сеанса PK |
| RoomId | String | Опционально | ID комнаты. Когда бизнес-сторона не знает конкретный BattleId, вы можете заполнить RoomId, чтобы получить информацию BattleId текущего RoomId. |

### Пример тела пакета ответа

**Базовая форма**

```
{    "ErrorCode": 0,    "ErrorInfo": "",    "ActionStatus": "OK",    "RequestId": "Id-e1fffbc751dc44bc95ec960d1fdfbefe-O-Seq-126652",    "Response": {        "BattleId": "HuYM036AMk2JJeSUgnv5H0KDJpc+HXfqTAtGuYWzRPs=",        "Duration": 120000,        "StartTime": 1739508625,        "EndTime": 1739508746,        "Status": 3,        "FromRoomInfo": {            "RoomId": "pk-3",            "Score": 100,            "Owner_Account": "brennanli1"        },        "ToRoomList": [            {                "RoomId": "pk-4",                "Score": 100,                "Owner_Account": "brennanli2"            }        ]    }}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ErrorCode | Integer | Код ошибки, 0 указывает на успех, ненулевое значение указывает на ошибку. |
| ErrorInfo | String | Сообщение об ошибке. |
| ActionStatus | String | Результат обработки запроса, OK указывает на успех, FAIL указывает на ошибку. |
| RequestId | String | Уникальный ID запроса, возвращаемый с каждым запросом, необходимый для устранения неполадок. |
| BattleId | String | ID боевых действий |
| Duration | Integer | Продолжительность боевых действий |
| StartTime | Integer | Время начала боевых действий |
| EndTime | Integer | Время окончания боевых действий |
| Status | Integer | Статус боевых действий, 1 указывает на не начатые, 2 указывает на идущие, 3 указывает на завершенные, 4 указывает на завершенные до начала. |
| FromRoomInfo | Object | Информация о комнате инициатора боевых действий |
| ToRoomList | Array | Информация о комнате получателя боевых действий |
| RoomId | String | ID комнаты |
| Score | Integer | Счет комнаты |
| Owner_Account | String | ID владельца комнаты |

## Описание кодов ошибок

Если не возникает ошибка сети (например, ошибка 502), код возврата HTTP этого API составляет 200. Реальный код ошибки и сообщение об ошибке находятся в теле пакета ответа. Представлены `ErrorCode` и `ErrorInfo`. Распространенные коды ошибок (60000–79999) см. в документации [Error Code](https://www.tencentcloud.com/document/product/647/60027#).

Приватные коды ошибок этого API выглядят следующим образом:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, пожалуйста, повторите попытку. |
| 100002 | Параметр неверен. Проверьте правильность запроса в соответствии с описанием ошибки. |
| 100411 | Текущие боевые действия не существуют или информация была очищена из-за слишком длительного времени завершения. |

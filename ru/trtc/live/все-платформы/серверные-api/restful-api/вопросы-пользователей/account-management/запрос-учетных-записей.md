# Запрос учетных записей

## Обзор функции

Этот API используется для проверки того, были ли учетные записи, принадлежащие приложению, импортированы в Chat. Поддерживается пакетный запрос.

## Описание вызова API

### Образец URL запроса

```
https://xxxxxx/v4/im_open_login_svc/account_check?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны измененные параметры при вызове этого API. Для других параметров см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Доменное имя, соответствующее стране/региону, где расположен ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/im_open_login_svc/account_check | API запроса |
| sdkappid | `SDKAppID`, назначенный консолью при создании приложения |
| identifier | Учетная запись администратора приложения. Для получения дополнительной информации см. раздел **Администратор приложения** в разделе [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, сгенерированная учетной записью администратора приложения. Для получения подробной информации см. [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное целое число без знака в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса, должен всегда быть `json`. |

### Максимальная частота вызовов

100 раз в секунду

### Образец запроса

```
{  "CheckItem":  [      {          "UserID":"UserID_1"      },      {          "UserID":"UserID_2"      }  ]}
```

### Поля запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| CheckItem | Array | Да | Массив объектов учетных записей для проверки. Один запрос может содержать до 100 учетных записей. |
| UserID | String | Да | UserID учетной записи для проверки |

### Образец ответа

```
{    "ActionStatus": "OK",    "ErrorCode": 0,    "ErrorInfo": "",    "ResultItem": [        {            "UserID": "UserID_1",            "ResultCode": 0,            "ResultInfo": "",            "AccountStatus": "Imported"        },        {            "UserID": "UserID_2",            "ResultCode": 0,            "ResultInfo": "",            "AccountStatus": "NotImported"        }    ]}
```

### Поля ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: Успешно. `FAIL`: Ошибка |
| ErrorCode | Integer | Код ошибки. `0`: Успешно. Другие значения: Ошибка |
| ErrorInfo | String | Информация об ошибке при сбое запроса |
| ResultItem | Array | Массив результатов для различных учетных записей |
| UserID | String | UserID учетной записи для проверки |
| ResultCode | Integer | Результат проверки учетной записи. `0`: Успешно. Другие значения: Ошибка |
| ResultInfo | String | Информация об ошибке при сбое проверки учетной записи |
| AccountStatus | String | Статус импорта учетной записи. Допустимые значения: `Imported` и `NotImported` |

## Коды ошибок

Код состояния HTTP, возвращаемый этим API, всегда равен 200, если только не произойдет ошибка сети (например, ошибка 502). Конкретный код ошибки и подробности можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, характерные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 70169 | Истечение времени ожидания сервера. Повторите попытку позже. |
| 70202 | Истечение времени ожидания сервера. Повторите попытку позже. |
| 70402 | Недопустимые параметры. Убедитесь, что все требуемые поля заполнены и параметры соответствуют требованиям протокола. |
| 70403 | Ошибка запроса. Для выполнения этой операции требуются разрешения администратора приложения. |
| 70500 | Внутренняя ошибка сервера. Повторите попытку позже. |

# Удаление учетных записей

## Описание функции

Приложения Chat Trial Edition и Chat Free Trial поддерживают удаление учетных записей по умолчанию. Для других типов приложений (таких как: TRTC, доска, Pro Edition, Pro Plus Edition, Enterprise Edition) необходимо включить эту функцию в консоли.

> **Примечание:** Для конкретных типов пакетов вы можете перейти на страницу приложения в [Консоль](https://console.trtc.io/chat), а затем нажать **Application** > **Application Details** > **Plan** для просмотра.

> **Примечание:** После удаления учетной записи **данные пользователя будут невозможно восстановить**, используйте этот интерфейс с осторожностью. При удалении учетной записи будут также удалены контакты, профиль, группы, сеансы и другие данные пользователя. Chat Trial Edition и Chat Free Trial Edition поддерживают повторное импортирование удаленных UserID. Для других типов приложений (таких как: TRTC, доска, Pro Edition, Pro Plus Edition, Enterprise Edition) **удаленные UserID нельзя переимпортировать в течение 3 месяцев**.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/im_open_login_svc/account_delete?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В следующей таблице описаны только измененные параметры при вызове этого API. Для получения дополнительной информации о других параметрах см. [Обзор RESTful API](https://intl.cloud.tencent.com/document/product/1047/34620).

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| xxxxxx | Страна/регион, где находится ваш SDKAppID. Китай: `console.tim.qq.com` Сингапур: `adminapisgp.im.qcloud.com` Сеул: `adminapikr.im.qcloud.com` Токио: `adminapijpn.im.qcloud.com` Франкфурт: `adminapiger.im.qcloud.com` Кремниевая долина: `adminapiusa.im.qcloud.com` Джакарта: `adminapiidn.im.qcloud.com` |
| v4/im_open_login_svc/account_delete | API запроса |
| sdkappid | `SDKAppID`, назначенный консолью при создании приложения |
| identifier | Учетная запись администратора приложения. Дополнительную информацию см. в разделе **App Admin** в [Аутентификация входа](https://intl.cloud.tencent.com/document/product/1047/33517). |
| usersig | Подпись, сгенерированная в учетной записи администратора приложения. Дополнительную информацию о создании подписи см. в разделе [Генерирование UserSig](https://intl.cloud.tencent.com/document/product/1047/34385). |
| random | Случайное 32-битное неподписанное целое число в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса. Значение всегда `json`. |

### Максимальная частота вызовов

100 вызовов в секунду

### Пример пакета запроса

```
{  "DeleteItem":  [      {          "UserID":"UserID_1"      },      {          "UserID":"UserID_2"      }  ]}
```

### Поля пакета запроса

| Поле | Тип | Обязательно | Описание |
| --- | --- | --- | --- |
| DeleteItem | Array | Да | Массив учетных записей для удаления. Один запрос может содержать до 100 учетных записей. |
| UserID | String | Да | `UserID` удаляемой учетной записи |

### Пример пакета ответа

```
{    "ActionStatus": "OK",    "ErrorCode": 0,    "ErrorInfo": "",    "ResultItem": [        {            "ResultCode": 0,            "ResultInfo": "",            "UserID": "UserID_1"        },        {            "ResultCode": 70107,            "ResultInfo": "Err_TLS_PT_Open_Login_Account_Not_Exist",            "UserID": "UserID_2"        }    ]}
```

### Поля пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат запроса. `OK`: успешно; `FAIL`: ошибка |
| ErrorCode | Integer | Код ошибки. `0`: успешно; другие значения: ошибка |
| ErrorInfo | String | Информация об ошибке при неудачном выполнении запроса |
| ResultItem | Array | Массив результатов для различных учетных записей |
| ResultCode | Integer | Код ошибки для учетной записи. `0`: успешно; другие значения: ошибка |
| ResultInfo | String | Информация об ошибке при неудачном удалении учетной записи |
| UserID | String | `UserID` удаляемой учетной записи |

## Коды ошибок

HTTP-код состояния, возвращаемый этим API, всегда равен 200, если не возникает сетевая ошибка (например, ошибка 502). Конкретный код ошибки и детали можно найти в полях ответа `ErrorCode` и `ErrorInfo` соответственно.
Для общих кодов ошибок (60000–79999) см. [Коды ошибок](https://intl.cloud.tencent.com/document/product/1047/34348).
В следующей таблице описаны коды ошибок, специфичные для этого API:

| Код ошибки | Описание |
| --- | --- |
| 30006 | При очистке данных контактов произошла внутренняя ошибка сервера. Повторите попытку позже. |
| 30007 | При очистке данных контактов произошло внутреннее истечение времени ожидания сервера. Повторите попытку позже. |
| 30008 | При записи данных контактов произошел конфликт записи. Рекомендуется использовать пакетный режим. |
| 40006 | При очистке профилей произошла внутренняя ошибка сервера. Повторите попытку позже. |
| 70107 | Удаляемый `UserID` не существует. Убедитесь, что `UserID` действителен. |
| 70169 | Истечение времени ожидания сервера. Повторите попытку позже. |
| 70202 | Истечение времени ожидания сервера. Повторите попытку позже. |
| 70402 | Недействительные параметры. Убедитесь, что все обязательные поля заполнены и параметры соответствуют требованиям протокола. |
| 70403 | Ошибка запроса. Для выполнения этой операции требуются разрешения администратора приложения. |
| 70500 | Внутренняя ошибка сервера. Повторите попытку позже. |
| 71000 | Ошибка удаления учетных записей. Удаление учетных записей поддерживается только для приложений, использующих план Chat Trial. Ваше приложение использует план Pro и поэтому не поддерживает удаление учетных записей. |

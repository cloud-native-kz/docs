# Получить список заблокированных участников

## Обзор функции

Администратор приложения может использовать этот интерфейс для получения списка заблокированных участников.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_engine_http_srv/get_banned_member_list?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже указаны только параметры, изменяемые при вызове этого интерфейса. Для получения дополнительной информации о других параметрах обратитесь к [Обзору RESTful API](https://www.tencentcloud.com/document/product/647/63398).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Зарезервированный домен для страны/региона, где находится SDKAppID: прочие в Китае: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Соединённые Штаты: `adminapiusa.im.qcloud.com` |
| v4/live_engine_http_srv/get_banned_member_list | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Должна быть учётная запись администратора приложения. Дополнительные сведения см. в разделе [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin) |
| usersig | Подпись, созданная учётной записью администратора приложения. Дополнительные сведения см. в разделе [Создание UserSig](https://www.tencentcloud.com/document/product/647/69883#) |
| random | Случайное 32-битное целое число без знака от 0 до 4294967295 |
| contenttype | Формат запроса фиксированное значение: `json` |

### Максимальная частота вызовов

200 запросов/сек.

### Пример пакета запроса

**Основная форма**

```
{    "RoomId": "test",     "Offset":0,    "Limit":10}
```

### Описание полей пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| RoomId | String | Обязательное | ID комнаты |
| Offset | Integer | Опциональное | Смещение, начиная с 0 при первом запросе, последующие запросы основаны на значении NextOffset в пакете ответа |
| Limit | Integer | Опциональное | Количество записей при одиночной выборке, максимальное значение: 100 |

### Пример пакета ответа

**Основная форма**

```
{    "ErrorCode": 0,    "ErrorInfo": "",    "ActionStatus": "OK",    "RequestId": "Id-138c94122acc4ca485c8c7bf270686ac-O-Seq-135511",    "Response": { //Response Body        "BannedAccountList": [            {                "Member_Account": "user2",                "BannedUntil": 1719300627            },            {                "Member_Account": "user1",                "BannedUntil": 1719300627            }        ],        "NextOffset": 0 // Pagination Identifier    }}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результаты обработки запроса. OK указывает на успех, FAIL указывает на ошибку |
| ErrorCode | Integer | Код ошибки. 0: успешно; другие значения: ошибка |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Уникальный RequestId, каждый запрос вернёт RequestId, который требуется при локализации проблем |
| BannedAccountList | Array | Список заблокированных участников |
| Member_Account | String | ID учётной записи заблокированного участника |
| BannedUntil | Integer | Время окончания блокировки |

## Коды ошибок

Если не возникает сетевая ошибка (например, ошибка 502), код возврата HTTP для этого интерфейса всегда будет 200. Фактический код ошибки и информация об ошибке указаны в полезной нагрузке ответа с помощью ErrorCode и ErrorInfo. Общие коды ошибок (60000–79999) можно найти в документе [Коды ошибок](https://www.tencentcloud.com/document/product/647/60027#). Приватные коды ошибок для этого API приведены ниже:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, повторите попытку |
| 100002 | Недопустимый параметр, проверьте корректность запроса на основе описания ошибки |
| 100004 | Комната не существует, возможно, она никогда не была создана или уже удалена |

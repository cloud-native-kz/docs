# Обновление информации комнаты

## Обзор функции

Администраторы приложения могут обновлять информацию о комнате через этот интерфейс.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_engine_http_srv/update_room_info?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже перечислены только параметры, которые изменяются при вызове этого интерфейса. Для получения более подробной информации о других параметрах см. [Обзор RESTful API](https://www.tencentcloud.com/document/product/647/63398).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Зарезервированный домен для страны/региона, где находится SDKAppID: Другие в Китае: `console.tim.qq.com`. Сингапур: `adminapisgp.im.qcloud.com`. Соединенные Штаты: `adminapiusa.im.qcloud.com`. |
| v4/live_engine_http_srv/update_room_info | API запроса. |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения. |
| identifier | Должен быть аккаунтом администратора приложения. Для получения более подробной информации см. [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin). |
| usersig | Подпись, созданная аккаунтом администратора приложения, для подробных инструкций см. [Создание UserSig](https://www.tencentcloud.com/document/product/647/69883). |
| random | Случайное 32-битное беззнаковое целое число в диапазоне от 0 до 4294967295. |
| contenttype | Формат запроса, фиксированное значение: `json`. |

### Максимальная частота вызовов

200 запросов/сек.

### Примеры пакетов запроса

**Базовая форма**

```
{    "RoomInfo" : {      "RoomId":"live-room",      "RoomName" : "rname-123",       "MaxMemberCount" : 300,       "IsMessageDisabled" : false,       "CoverURL": "https://xxxx.png",       "Category": [1,2,3],       "ActivityStatus":1,       "IsPublicVisible": false,      "TakeSeatMode": "ApplyToTake"  }}
```

### Поля пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| RoomId | String | Обязательно | ID комнаты. |
| RoomName | String | Опционально | Название комнаты. |
| MaxMemberCount | Integer | Опционально | Максимальная емкость комнаты. |
| IsMessageDisabled | Bool | Опционально | Переключатель полного отключения звука в комнате, за исключением администратора и владельца. |
| CoverURL | String | Опционально | Обложка комнаты. |
| Category | Array | Опционально | Тег категории комнаты, максимальная длина массива — 3. |
| ActivityStatus | Integer | Опционально | Статус самоопределения комнаты. |
| IsPublicVisible | Bool | Опционально | Переключатель открытия комнаты, в основном используется для получения списка комнат. |
| TakeSeatMode | String | Опционально | Режим место, свободное присоединение к подиуму FreeToTake, применение для присоединения к микрофону ApplyToTake. |
| Notice | String | Опционально | Уведомление комнаты, максимум 100 байт. |

### Примеры пакетов ответа

**Базовая форма**

```
{  "ActionStatus": "OK",  "ErrorInfo": "",  "ErrorCode": 0,  "RequestId": "Id-8c9858f01e954611ae2d4c1b1ed7d583-O-Seq-52720"}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат обработки запроса: OK указывает на успех; FAIL указывает на сбой. |
| ErrorCode | Integer | Код ошибки. 0: успех; другие значения: сбой. |
| ErrorInfo | String | Сообщение об ошибке. |
| RequestId | String | Уникальный ID запроса возвращается с каждым запросом и требуется предоставить этот RequestId при выявлении проблем. |

## Коды ошибок

Если не произойдет сетевая ошибка (например, ошибка 502), код возврата HTTP для этого интерфейса всегда будет 200. Фактический код ошибки и информация об ошибке указаны в полезной нагрузке ответа с помощью ErrorCode и ErrorInfo. Распространенные коды ошибок (60000–79999) можно найти в документе [Код ошибки](https://www.tencentcloud.com/document/product/647/60027). Частные коды ошибок для этого API приведены ниже:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, пожалуйста, повторите попытку. |
| 100002 | Неверный параметр, проверьте корректность запроса на основе описания ошибки. |
| 100004 | Комната не существует, возможно, она никогда не была создана или уже была удалена. |
| 100006 | Нет разрешения на удаление, например, оператор не является ни владельцем, ни администратором root, или удаляемая комната не является живой комнатой и т. д. |

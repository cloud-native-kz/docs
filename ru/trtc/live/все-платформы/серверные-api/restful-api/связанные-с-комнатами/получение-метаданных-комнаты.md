# Получение метаданных комнаты

## Обзор функции

Администратор приложения может получать метаданные комнаты через этот интерфейс.

## Описание вызова API

### Пример URL запроса

```
https://xxxxxx/v4/live_engine_http_srv/get_room_metadata?sdkappid=88888888&identifier=admin&usersig=xxx&random=99999999&contenttype=json
```

### Параметры запроса

В таблице ниже указаны только параметры, которые изменяются при вызове этого интерфейса. Для получения подробной информации о других параметрах обратитесь к [обзору RESTful API](https://www.tencentcloud.com/document/product/647/63398).

| Параметр | Описание |
| --- | --- |
| xxxxxx | Зарезервированный домен для страны/региона, где расположен SDKAppID: Другие в Китае: `console.tim.qq.com`Сингапур: `adminapisgp.im.qcloud.com`Соединённые Штаты: `adminapiusa.im.qcloud.com` |
| v4/live_engine_http_srv/get_room_metadata | API запроса |
| sdkappid | SDKAppID, назначенный консолью Chat при создании приложения |
| identifier | Должна быть учётная запись администратора приложения. Для получения подробной информации см. [Администратор приложения](https://www.tencentcloud.com/document/product/647/69882#app-admin) |
| usersig | Подпись, сгенерированная учётной записью администратора приложения. Для получения подробной информации об операциях обратитесь к [Генерирование UserSig](https://www.tencentcloud.com/document/product/647/69883#) |
| random | Случайное 32-разрядное целое число без знака в диапазоне от 0 до 4294967295 |
| contenttype | Формат запроса фиксированное значение: `json` |

### Максимальная частота вызовов

200 запросов/сек.

### Пример пакетов запроса

**Основной формат**

```
{  "RoomId": "live-room",  "Keys":["key1", "key2"] // Если массив пуст, это означает получение всей пользовательской информации комнаты, в противном случае это означает получение пользовательской информации из указанного списка ключей}
```

### Описание полей пакета запроса

| Поле | Тип | Атрибут | Описание |
| --- | --- | --- | --- |
| RoomId | String | Обязательное | ID комнаты |
| Keys | Array | Обязательное | Указанный список ключей. Если пуст, это означает получение всех метаданных комнаты в комнате |

### Пример пакетов ответа

**Основной формат**

```
{  "ActionStatus": "OK",  "ErrorInfo": "",  "ErrorCode": 0,  "RequestId": "Id-8c9858f01e954611ae2d4c1b1ed7d583-O-Seq-52720",  "Response": {        "Metadata": [            {                "Key": "key1",                "Value": "value1"            },            {                "Key": "key2",                "Value": "value2"            }        ]  }}
```

### Описание полей пакета ответа

| Поле | Тип | Описание |
| --- | --- | --- |
| ActionStatus | String | Результат процесса запроса: OK указывает на успех; FAIL указывает на ошибку |
| ErrorCode | Integer | Код ошибки. 0: успех; другие значения: ошибка |
| ErrorInfo | String | Сообщение об ошибке |
| RequestId | String | Уникальный ID запроса возвращается с каждым запросом и требуется предоставить этот RequestId при локализации проблем |
| Response | Object | Метаданные комнаты |
| Metadata | Array | Допустимые метаданные комнаты, возвращённые для комнаты |

## Коды ошибок

Если не возникнет ошибка сети (например, ошибка 502), HTTP код возврата для этого интерфейса всегда будет 200. Фактический код ошибки и информация об ошибке указываются в полезной нагрузке ответа с помощью ErrorCode и ErrorInfo. Распространённые коды ошибок (60000–79999) можно найти в документе [Коды ошибок](https://www.tencentcloud.com/document/product/647/60027#). Приватные коды ошибок для этого API выглядят следующим образом:

| Код ошибки | Описание |
| --- | --- |
| 100001 | Внутренняя ошибка сервера, пожалуйста, повторите попытку |
| 100002 | Неверный параметр. Пожалуйста, проверьте правильность запроса на основе описания ошибки |
| 100003 | ID комнаты уже существует. Пожалуйста, выберите другой ID комнаты |
| 100006 | Нет прав на работу с пользовательской информацией комнаты |
| 100007 | Отсутствует информация об оплате. Необходимо приобрести пакет из консоли |
| 100500 | Количество ключей для пользовательской информации комнаты превышает верхний предел в 10 |
| 100501 | Значение одного ключа для пользовательской информации комнаты превышает верхний предел в 2 КБ |
| 100502 | Общий размер значений для всех ключей пользовательской информации комнаты превышает верхний предел в 16 КБ |
| 100503 | Ни один из указанных ключей не является допустимым при удалении пользовательской информации комнаты |

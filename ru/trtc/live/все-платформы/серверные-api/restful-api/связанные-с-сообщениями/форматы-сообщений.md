# Форматы сообщений

## MsgBody

Содержимое сообщения вводится в поля MsgBody. Tencent Cloud Chat поддерживает несколько элементов сообщения в одном сообщении, например, сообщение может содержать и текст, и эмодзи. Поэтому MsgBody определяется как массив, который может включать столько элементов сообщения, сколько необходимо. Название элемента сообщения — TIMMsgElement. Примеры TIMMsgElements, составляющих MsgBody, см. в разделе [Примеры содержимого сообщения MsgBody](#msgbody-examples).

Формат TIMMsgElement определен следующим образом:

```
{    "MsgType": "",    "MsgContent": {}}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| MsgType | String | Тип элемента сообщения. В настоящее время поддерживаются следующие объекты сообщений: TIMTextElem (текстовое сообщение), TIMLocationElem (сообщение о местоположении), TIMFaceElem (сообщение с эмодзи), TIMCustomElem (пользовательское сообщение), TIMSoundElem (аудиосообщение), TIMImageElem (сообщение с изображением), TIMFileElem (сообщение с файлом) и TIMVideoFileElem (видеосообщение). |
| MsgContent | Object | Содержимое элемента сообщения. Каждый MsgType имеет свой формат MsgContent. Дополнительную информацию см. ниже. |

В следующей таблице перечислены типы сообщений (MsgType), которые в настоящее время поддерживаются:

| Значение MsgType | Тип |
| --- | --- |
| TIMTextElem | Текстовое сообщение |
| TIMLocationElem | Географическое местоположение |
| TIMFaceElem | Эмодзи |
| TIMCustomElem | Пользовательское сообщение. Когда получателем является устройство iOS и приложение работает в фоне, этот тип сообщения предоставляет поля помимо текста в APNs. Объединенное сообщение может содержать только один элемент пользовательского сообщения TIMCustomElem. |
| TIMSoundElem | Голосовое сообщение |
| TIMImageElem | Сообщение с изображением |
| TIMFileElem | Сообщение с файлом |
| TIMVideoFileElem | Видеосообщение |

> **Внимание:** сообщения указанных выше типов могут отправляться серверными интегрированными REST API.

## TIMMsgElement

### Элемент текстового сообщения

```
{    "MsgType": "TIMTextElem",    "MsgContent": {        "Text": "hello world"    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Text | String | Содержимое сообщения. Когда получателем является устройство iOS или Android и приложение работает в фоне, содержимое сообщения отображается как текст автономной отправки. |

Когда получателем является устройство iOS или Android и приложение работает в фоне, поле Text в теле JSON-запроса отображается как текст автономной отправки.

### Элемент сообщения о местоположении

```
{    "MsgType": "TIMLocationElem",     "MsgContent": {        "Desc": "someinfo",         "Latitude": 29.340656774469956,         "Longitude": 116.77497920478824    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Desc | String | Описание местоположения |
| Latitude | Number | Широта |
| Longitude | Number | Долгота |

Когда получателем является устройство iOS или Android и приложение работает в фоне, текст автономной отправки — это [Position] для китайской версии и [Location] для английской версии.

### Элемент сообщения с эмодзи

```
{    "MsgType": "TIMFaceElem",     "MsgContent": {        "Index": 1,         "Data": "content"    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Index | Number | Индекс эмодзи, определяемый пользователем |
| Data | String | Дополнительные данные |

Когда получателем является устройство iOS или Android и приложение находится в фоне, текст автономной отправки для английской версии — это [Expression].

> **Описание:** когда сообщение содержит только один элемент пользовательского сообщения `TIMCustomElem`, если не введены оба поля `Desc` и `OfflinePushInfo.Desc`, сообщение не получит автономную отправку, если только не введено поле `OfflinePushInfo.Desc`.

### Элемент пользовательского сообщения

> **Примечание:** когда сообщение содержит только один элемент пользовательского сообщения TIMCustomElem, для получения автономной отправки сообщения необходимо заполнить поле OfflinePushInfo.Desc.

```
{    "MsgType": "TIMCustomElem",     "MsgContent": {        "Data": "message"    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Data | String | Данные пользовательского сообщения. |

### Элемент голосового сообщения

> **Внимание:** для отправки голосового сообщения через серверный REST API необходимо ввести поля `Url`, `UUID` и `Download_Flag`. Убедитесь, что голосовое сообщение можно загрузить через `URL` и что `UUID` — это глобально уникальная строковая значение, обычно значение MD5 голосового файла. Получатель сообщения может получить указанный `UUID` через `V2TIMSoundElem.getUUID()`, и приложение может использовать `UUID` для идентификации голосового сообщения. `Download_Flag` должен быть установлен в `2`.

Версии 4.X и более поздние Chat SDK (для Android, iOS, macOS и Windows) отправляют элементы голосового сообщения в следующем формате:

```
{    "MsgType": "TIMSoundElem",    "MsgContent": {        "Url": "https://1234-5678187359-1253735226.cos.ap-shanghai.myqcloud.com/abc123/c9be9d32c05bfb77b3edafa4312c6c7d",        "UUID": "1053D4B3D61040894AC3DE44CDF28B3EC7EB7C0F",        "Size": 62351,        "Second": 1,        "Download_Flag": 2    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Url | String | URL загрузки голосового сообщения, через который содержимое голосового сообщения можно загрузить напрямую. |
| UUID | String | Уникальный идентификатор голосового сообщения, ключевое значение, используемое клиентом для индексации голосового сообщения. |
| Size | Number | Размер данных голосового сообщения в байтах. |
| Second | Number | Длительность голосового сообщения в секундах. |
| Download_Flag | Number | Флаг метода загрузки голосового сообщения. В настоящее время значение `Download_Flag` должно быть `2`, что означает, что содержимое голосового сообщения можно загрузить из URL, указанного в поле `Url`. |

> **Примечание:** версии 2.X и 3.X Chat SDK (для Android, iOS, macOS и Windows) отправляют элементы голосового сообщения в следующем формате:

```
{    "MsgType": "TIMSoundElem",    "MsgContent": {        "UUID": "305c0201", //Уникальный идентификатор голосового сообщения строкового типа. Это ключевое значение, которое клиент использует для индексации голосового сообщения. Голосовое сообщение не может быть загружено через это поле. Для получения голосового сообщения обновите Chat SDK до версии 4.X.        "Size": 62351,      //Размер данных голосового сообщения в байтах, числового типа.        "Second": 1         //Длительность голосового сообщения в секундах, числового типа.    }}
```

### Элемент сообщения с изображением

> **Внимание:** для отправки сообщения с изображением через серверный REST API необходимо ввести поля `URL`, `UUID`, `Width` и `Height`. Убедитесь, что изображение можно загрузить через `URL`, чтобы вы могли [получить основную информацию об изображении](https://intl.cloud.tencent.com/document/product/1045/33722) и [обработать изображение](https://intl.cloud.tencent.com/document/product/1045/33726). `Width` и `Height` — это ширина и высота (в пикселях) изображения. `UUID` должен быть установлен в глобально уникальное строковое значение, обычно значение MD5 изображения. Получатель сообщения может получить объект `V2TIMImage`, вызвав `V2TIMImageElem.getImageList()`, а затем получить указанный `UUID`, вызвав `V2TIMImage.getUUID()`. Приложение может использовать `UUID` для идентификации изображения.

```
{    "MsgType": "TIMImageElem",    "MsgContent": {        "UUID": "1853095_D61040894AC3DE44CDFFFB3EC7EB720F",        "ImageFormat": 1,        "ImageInfoArray": [            {                "Type": 1,           //Исходное изображение                "Size": 1853095,                "Width": 2448,                "Height": 3264,                "URL": "http://xxx/3200490432214177468_144115198371610486_D61040894AC3DE44CDFFFB3EC7EB720F/0"            },            {                "Type": 2,      //Большое изображение                "Size": 2565240,                "Width": 0,                "Height": 0,                "URL": "http://xxx/3200490432214177468_144115198371610486_D61040894AC3DE44CDFFFB3EC7EB720F/720"            },            {                "Type": 3,   //Миниатюра                "Size": 12535,                "Width": 0,                "Height": 0,                "URL": "http://xxx/3200490432214177468_144115198371610486_D61040894AC3DE44CDFFFB3EC7EB720F/198"            }        ]    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| UUID | String | Уникальный идентификатор изображения, ключевое значение, используемое клиентом для индексации изображения |
| ImageFormat | Number | Формат изображения. JPG = 1, GIF = 2, PNG = 3, BMP = 4, Others = 255. |
| ImageInfoArray | Array | Информация загрузки исходного изображения, миниатюры или большого изображения. |
| Type | Number | Тип изображения. 1: исходное изображение, 2: большое изображение, 3: миниатюра. |
| Size | Number | Размер данных изображения в байтах. |
| Width | Number | Ширина изображения в пикселях. |
| Height | Number | Высота изображения в пикселях. |
| URL | String | URL загрузки изображения. |

### Элемент сообщения с файлом

> **Внимание:** для отправки сообщения с файлом через серверный REST API необходимо ввести поля `URL`, `UUID` и `Download_Flag`. Убедитесь, что файл можно загрузить через `URL` и что `UUID` — это глобально уникальная строковая значение, обычно значение MD5 файла. Получатель сообщения может получить указанный `UUID`, вызвав `V2TIMFileElem.getUUID()`, и бизнес-приложение может использовать `UUID` для идентификации файла. `Download_Flag` должен быть установлен в `2`.

Версии 4.X и более поздние Chat SDK (для Android, iOS, macOS и Windows) отправляют элементы сообщения с файлом в следующем формате:

```
{    "MsgType": "TIMFileElem",    "MsgContent": {        "Url": "https://7492-5678539059-1253735326.cos.ap-shanghai.myqcloud.com/abc123/49be9d32c0fbfba7b31dafa4312c6c7d",        "UUID": "1053D4B3D61040894AC3DE44CDF28B3EC7EB7C0F",        "FileSize": 1773552,        "FileName": "file:///private/var/Application/tmp/trim.B75D5F9B-1426-4913-8845-90DD46797FCD.MOV",        "Download_Flag": 2    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Url | String | URL загрузки файла, через который файл можно загрузить напрямую. |
| UUID | String | Уникальный идентификатор файла, ключевое значение, используемое клиентом для индексации файла. |
| FileSize | Number | Размер данных файла в байтах. |
| fileName | String | Имя файла. |
| Download_Flag | Number | Флаг метода загрузки файла. В настоящее время значение `Download_Flag` должно быть 2, что означает, что файл можно загрузить из URL, указанного в поле `Url`. |

> **Примечание:** версии 2.X и 3.X Chat SDK (для Android, iOS, macOS и Windows) отправляют элементы сообщения с файлом в следующем формате:{    "MsgType": "TIMFileElem",    "MsgContent": {        "UUID": "305c02010", //Уникальный идентификатор файла строкового типа. Это ключевое значение, которое клиент использует для индексации файла. Файл не может быть загружен через это поле. Для получения файла обновите Chat SDK до версии 4.X.        "FileSize": 1773552, //Размер данных файла в байтах, числового типа.        "FileName": "file:///private/var/Application/tmp/trim.B75D5F9B-1426-4913-8845-90DD46797FCD.MOV" //Имя файла строкового типа.    }}

### Элемент видеосообщения

> **Внимание:** для отправки видеосообщения через серверный REST API необходимо ввести поля `VideoUrl`, `VideoUUID`, `ThumbUrl`, `ThumbUUID`, `ThumbWidth`, `ThumbHeight`, `VideoDownloadFlag` и `ThumbDownloadFlag`. Убедитесь, что видео и миниатюра видео можно загрузить через `VideoUrl` и `ThumbUrl` соответственно. `VideoUUID` и `ThumbUUID` должны быть глобально уникальными строковыми значениями, обычно значениями MD5 видео и миниатюры видео. Получатель сообщения может получить указанные `VideoUUID` и `ThumbUUID`, вызвав `V2TIMVideoElem.getVideoUUID()` и `V2TIMVideoElem.getSnapshotUUID()` соответственно. Приложение может использовать `VideoUUID` для идентификации видео. `VideoDownloadFlag` и `ThumbDownloadFlag` должны быть установлены в `2`.

Версии 4.X и более поздние Chat SDK (для Android, iOS, macOS и Windows) отправляют элементы видеосообщения в следующем формате:

```
{    "MsgType": "TIMVideoFileElem",    "MsgContent": {        "VideoUrl": "https://0345-1400187352-1256635546.cos.ap-shanghai.myqcloud.com/abcd/f7c6ad3c50af7d83e23efe0a208b90c9",        "VideoUUID": "5da38ba89d6521011e1f6f3fd6692e35",        "VideoSize": 1194603,        "VideoSecond": 5,        "VideoFormat": "mp4",        "VideoDownloadFlag":2,        "ThumbUrl": "https://0345-1400187352-1256635546.cos.ap-shanghai.myqcloud.com/abcd/a6c170c9c599280cb06e0523d7a1f37b",        "ThumbUUID": "6edaffedef5150684510cf97957b7bc8",        "ThumbSize": 13907,        "ThumbWidth": 720,        "ThumbHeight": 1280,        "ThumbFormat": "JPG",        "ThumbDownloadFlag":2    }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| VideoUrl | String | URL загрузки видео, через который видео можно загрузить напрямую. |
| VideoUUID | String | Уникальный идентификатор видео, ключевое значение, используемое клиентом для индексации видео. |
| VideoSize | Number | Размер данных видео в байтах. |
| VideoSecond | Number | Длительность видео в секундах. Для веб-клиентов это поле недоступно и по умолчанию имеет значение 0. |
| VideoFormat | String | Формат видео, например MP4. |
| VideoDownloadFlag | Number | Флаг, указывающий на метод загрузки видео. В настоящее время значение `VideoDownloadFlag` должно быть 2, что означает, что видео можно загрузить из URL, указанного в поле `VideoUrl`. |
| ThumbUrl | String | URL загрузки миниатюры видео, через который миниатюру видео можно загрузить напрямую. |
| ThumbUUID | String | Уникальный идентификатор миниатюры видео, ключевое значение, используемое клиентом для индексации миниатюры видео. |
| ThumbSize | Number | Размер данных миниатюры в байтах. |
| ThumbWidth | Number | Ширина миниатюры в пикселях. |
| ThumbHeight | Number | Высота миниатюры в пикселях. |
| ThumbFormat | String | Формат миниатюры видео, например JPG или BMP. |
| ThumbDownloadFlag | Number | Флаг метода загрузки миниатюры видео. В настоящее время значение `ThumbDownloadFlag` должно быть 2, что означает, что миниатюру видео можно загрузить из URL, указанного в поле `ThumbUrl`. |

> **Примечание:** версии 2.X и 3.X Chat SDK (для Android, iOS, macOS и Windows) отправляют элементы видеосообщения в следующем формате:{    "MsgType": "TIMVideoFileElem",    "MsgContent": {        "VideoUUID": "1400123456_dramon_34ca36be7dd214dc50a49238ef80a6b5",//Уникальный идентификатор видео строкового типа. Это ключевое значение, которое клиент использует для индексации миниатюры видео. Миниатюра видео не может быть загружена через это поле. ### Элементы объединенного сообщения

> **Внимание:** отправку и получение объединенных сообщений поддерживают только собственный SDK версии 5.2.210 или более поздней и веб SDK версии 2.10.1 или более поздней.

```
{  "MsgType": "TIMRelayElem",  "MsgContent": {    "Title": "Group chat history",    "MsgNum": 2,    "CompatibleText": "The SDK version does not support combined messages. Please upgrade to the latest version.",    "AbstractList": [      "A: What do you think of this?",      "B: I think it's great."    ],    "MsgList": [      {        "From_Account": "A",        "GroupId": "group1",        "MsgSeq": 85,        "MsgRandom": 3998651049,        "MsgTimeStamp": 1664437702,        "MsgBody": [          {            "MsgContent": {              "Text": " What do you think of this?"            },            "MsgType": "TIMTextElem"          }        ]      },      {        "From_Account": "B",        "GroupId": "group1",        "MsgSeq": 86,        "MsgRandom": 965790,        "MsgTimeStamp": 1664437703,        "MsgBody": [          {            "MsgContent": {              "Text": "I think it's great."            },            "MsgType": "TIMTextElem"          }        ]      }    ]  }}
```

| Поле | Тип | Описание |
| --- | --- | --- |
| Title | String | Название объединенного сообщения. |
| MsgNum | Integer | Количество перенаправленных сообщений. |
| CompatibleText | String | Совместимый текст. Когда серверная часть Chat преобразует объединенное сообщение в совместимый текст и доставляет его при получении объединенного сообщения SDK более ранней версии, который не поддерживает объединенные сообщения. |
| AbstractList | Array | Массив строк, содержащих краткие описания объединенного сообщения. |
| MsgList | Array | Список сообщений. Это поле доступно только если общая длина объединенных сообщений меньше или равна 12 КБ, в этом случае поле `JsonMsgKey` недоступно. |
| JsonMsgKey | String | Ключ списка объединенного сообщения. Это поле доступно только если общая длина объединенных сообщений превышает 12 КБ, в этом случае поле `MsgList` недоступно. |

Структура каждого сообщения в `MsgList` выглядит следующим образом:

| Поле | Тип | Описание |
| --- | --- | --- |
| From_Account | String | UserID отправителя сообщения. Это поле доступно, когда перенаправленное сообщение является сообщением C2C или группы. |
| To_Account | String | UserID получателя сообщения. Это поле доступно только когда перенаправленное сообщение является сообщением C2C. |
| GroupId | String | Идентификатор группы. Это поле доступно только когда перенаправленное сообщение является сообщением группы. |
| MsgSeq | Integer | Порядковый номер сообщения (32-битное целое число без знака). |
| MsgRandom | Integer | Случайное число сообщения (32-битное целое число без знака). |
| MsgTimeStamp | Integer | Временная метка сообщения в секундах. |
| MsgBody | Array | Тело сообщения. Подробную информацию о форматах см. в разделе [Форматы сообщений](#TIMMsgElement). (Примечание: сообщение может содержать несколько элементов сообщения, в этом случае `MsgBody` — это массив.) |
| CloudCustomData | String | Данные пользовательского сообщения. Они сохраняются в облаке и отправляются противоположной стороне. Такие данные можно получить после удаления и переустановки приложения. |

## Примеры MsgBody

### Сообщение с одним элементом текста

Одно сообщение содержит только один элемент текстового сообщения. Текстовое содержимое — **hello world**.

```
{    "MsgBody": [        {            "MsgType": "TIMTextElem",             "MsgContent": {                "Text": "hello world"            }        }    ]}
```

### Объединенные сообщения

Следующее одно сообщение содержит два элемента текстового сообщения и один элемент сообщения с эмодзи в последовательности текст + эмодзи + текст.

```
{    "MsgBody": [        {            "MsgType": "TIMTextElem",             "MsgContent": {                "Text": "hello"            }        },         {            "MsgType": "TIMFaceElem",             "MsgContent": {                "Index": 1,                 "Data": "content"            }        },         {            "MsgType": "TIMTextElem",             "MsgContent": {                "Text": "world"            }        }    ]}
```

> **Внимание:** объединенное сообщение может содержать только один элемент пользовательского сообщения `TIMCustomElem` и неограниченное количество других элементов сообщения.

## Описание пользовательских данных сообщения CloudCustomData

Каждое сообщение может содержать пользовательские данные `CloudCustomData`.

`CloudCustomData` сохраняется в облаке вместе с `MsgBody` сообщения. Это будет отправлено получателю и может быть получено даже после удаления и переустановки приложения.

В следующем примере показаны форматы `CloudCustomData` и `MsgBody`:

```
{    "MsgBody": [        {            "MsgType": "TIMTextElem",             "MsgContent": {                "Text": "hello"            }        }    ],    "CloudCustomData": "your cloud custom data"}
```

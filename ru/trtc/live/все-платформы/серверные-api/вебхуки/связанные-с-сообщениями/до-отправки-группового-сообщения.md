# До отправки группового сообщения

## Обзор функции

Это событие webhook используется серверной частью приложения для проверки сообщений комнаты пользователей в реальном времени, включая:

- Запись сообщений комнаты в реальном времени, например, путем записи журнала или синхронизации сообщений с другими системами.
- Блокировка запросов пользователей на отправку сообщений в комнату.

> **Примечание:** Период ожидания webhook перед отправкой сообщения по умолчанию составляет две секунды, и мы рекомендуем вам его не изменять. Этот webhook может истечь при использовании для модерации текста. Если вам нужно модерировать текст, используйте webhook содержимого.

## Ограничения

- Чтобы включить этот webhook, вы должны настроить URL webhook и включить соответствующий переключатель для этого webhook. Дополнительные сведения о методе конфигурации см. в разделе [Конфигурация webhook](https://www.tencentcloud.com/document/product/647/74157).
- Во время этого события webhook серверная часть Chat инициирует HTTP POST запрос к серверной части приложения.
- После получения запроса webhook серверная часть приложения должна проверить, является ли `SDKAppID`, содержащийся в URL запроса, `SDKAppID` приложения.
- Дополнительные соображения безопасности см. в разделе **Соображения безопасности** в [Обзор webhook](https://www.tencentcloud.com/document/product/647/64412#security-considerations).

## Сценарии срабатывания webhook

- Пользователь приложения отправляет сообщение комнаты на клиенте.
- Администратор приложения отправляет сообщение комнаты через REST API.

## Время срабатывания webhook

Webhook срабатывает до того, как серверная часть Live отправит сообщение комнаты участникам комнаты.

## Описание вызова API

### Пример URL запроса

В следующем примере настроенный в приложении URL webhook — это `https://www.example.com`.

**Пример:**

```
https://www.example.com?SdkAppid=$SDKAppID&CallbackCommand=$CallbackCommand&contenttype=json&ClientIP=$ClientIP&OptPlatform=$OptPlatform
```

### Параметры запроса

| Параметр | Описание |
| --- | --- |
| https | Протокол запроса — HTTPS, метод запроса — POST. |
| www.example.com | URL webhook |
| SdkAppid | `SDKAppID`, назначенный консолью Chat при создании приложения |
| CallbackCommand | Фиксированное значение: `Group.CallbackBeforeSendMsg`. |
| contenttype | Фиксированное значение: `json`. |
| ClientIP | IP клиента, например 127.0.0.1 |
| OptPlatform | Платформа клиента. Допустимые значения см. в описании `OptPlatform` в разделе **Протоколы webhook** в [Обзор обратного вызова webhook](https://www.tencentcloud.com/document/product/647/64412#d8e83f91-15ef-46e9-b370-9ba6c93a6ada). |

### Примеры запросов

```
{    "CallbackCommand": "Group.CallbackBeforeSendMsg", // Команда webhook    "GroupId": "@TGS#2J4SZEAEL", // ID комнаты    "Type": "Live", // Тип комнаты    "From_Account": "jared", // Отправитель    "Operator_Account":"admin", // Инициатор запроса    "Random": 123456, // Случайное число    "MsgId": "144115233406643804-1727580296-4026038328", // Уникальный идентификатор сообщения на клиенте    "MsgBody": [ // Тело сообщения. Дополнительные сведения см. в объекте сообщения `TIMMessage`.        {            "MsgType": "TIMTextElem", // Текст            "MsgContent":{                "Text": "red packet"            }        }    ],    "CloudCustomData": "your cloud custom data",    "EventTime":"1670574414123"// Временная метка события срабатывания в миллисекундах		}
```

### Поля запроса

| Поле | Тип | Описание |
| --- | --- | --- |
| CallbackCommand | String | Команда webhook. |
| GroupId | String | ID комнаты, которая генерирует сообщения комнаты. |
| Type | String | Тип комнаты, которая генерирует сообщения комнаты. Фиксированное значение `Live`. |
| From_Account | String | `UserID` отправителя сообщения. |
| Operator_Account | String | `UserID` инициатора запроса, на основе которого система может определить, инициирован ли запрос администратором. |
| Random | Integer | 32-битное случайное число в запросе. |
| MsgId | String | Уникальный идентификатор сообщения на клиенте. |
| MsgBody | Array | Тело сообщения. Содержание связано с запросом на отправку сообщения, как описано в [Отправка обычного сообщения](https://www.tencentcloud.com/document/product/647/74353#f1db57c4-02dd-47ff-ae52-19e4e91d9c25) и [Отправка пользовательского сообщения](https://www.tencentcloud.com/document/product/647/74354#f1db57c4-02dd-47ff-ae52-19e4e91d9c25). |
| CloudCustomData | String | Данные пользовательского сообщения. Они сохраняются в облаке и отправляются получателю. Такие данные можно получить после удаления и переустановки приложения. Содержание связано с запросом на отправку сообщения, как описано в [Отправка обычного сообщения](https://www.tencentcloud.com/document/product/647/74353#f1db57c4-02dd-47ff-ae52-19e4e91d9c25). |
| EventTime | Integer | Временная метка события срабатывания в миллисекундах. |

> **Примечание:** Перед отправкой сообщения в комнату повторно используются возможности обратного вызова сообщений группы IM. В обратном вызове `MsgBody` и `CloudCustomData` являются результатами упаковки запроса на отправку сообщения. Дополнительные сведения см. в [Отправка обычного сообщения](https://www.tencentcloud.com/document/product/647/74353#f1db57c4-02dd-47ff-ae52-19e4e91d9c25) и [Отправка пользовательского сообщения](https://www.tencentcloud.com/document/product/647/74354#f1db57c4-02dd-47ff-ae52-19e4e91d9c25).

### Примеры ответов

#### Разрешение отправки сообщения комнаты

Пользователю разрешено отправлять сообщения комнаты, и содержание сообщения не изменяется.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0 // `0` указывает, что пользователю разрешено отправлять сообщения.}
```

#### Запрет отправки сообщения комнаты

Пользователю не разрешено отправлять сообщения комнаты. В этом случае сообщение не отправляется, и код ошибки `10016` возвращается вызывающей стороне.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 1 // `1` указывает, что пользователю не разрешено отправлять сообщения.}
```

#### Молчаливое отбрасывание сообщений

Пользователю не разрешено отправлять сообщения комнаты. В этом случае сообщение не отправляется, но вызывающей стороне будет возвращено сообщение об успехе.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 2 // Значение `2` указывает, что сообщение молча отбрасывается.}
```

#### Изменение содержимого сообщения

В следующем примере ответа сообщение комнаты, отправленное пользователем, изменяется (добавляется пользовательское сообщение), и серверная часть Chat отправит измененное сообщение. С помощью этой функции серверная часть приложения может добавлять специальное содержание, например уровень и звание пользователя, к сообщению, отправленному пользователем.

```
{    "ActionStatus": "OK",    "ErrorInfo": "",    "ErrorCode": 0, // Это поле должно быть установлено на `0`, чтобы измененное сообщение было отправлено нормально.    "MsgBody": [ // Сообщение, измененное серверной частью приложения. Если серверная часть приложения не изменяет сообщение, отправляется сообщение, отправленное пользователем.        {            "MsgType": "TIMTextElem", // Текст            "MsgContent":{                "Text": "red packet"            }        },        {            "MsgType": "TIMCustomElem", // Пользовательское сообщение            "MsgContent":{                "Desc": "CustomElement.MemberLevel", // Описание                "Data": "LV1" // Данные            }        }    ],    "CloudCustomData": "your cloud custom data"}
```

### Поля ответа

| Поле | Тип | Обязательное | Описание |
| --- | --- | --- | --- |
| ActionStatus | String | Да | Результат запроса. `OK`: Успешно; `FAIL`: Ошибка. |
| ErrorCode | Integer | Да | Возвращаемый код ошибки. `0`: разрешает отправку сообщения; `1`: запрещает отправку сообщения; `2`: молча отбрасывает сообщение. Если деловая сторона хочет запретить пользователю отправлять сообщения и отправить `ErrorCode` и `ErrorInfo` клиенту, убедитесь, что значение `ErrorCode` находится в диапазоне [10100, 10200]. |
| ErrorInfo | String | Да | Информация об ошибке. |
| MsgBody | Array | Нет | Тело сообщения, измененное серверной частью приложения. Серверная часть Live отправляет измененное сообщение в комнату. Дополнительные сведения о формате см. в [Форматы сообщений](https://www.tencentcloud.com/document/product/647/74355). |
| CloudCustomData | String | Нет | Данные пользовательского сообщения. Они сохраняются в облаке и отправляются другой стороне. Такие данные можно получить после удаления и переустановки приложения. |

# Технический документ по безопасности

## 1. Обзор

Используя многолетний опыт Tencent в области сетевых технологий и технологий аудио/видео, TRTC предоставляет унифицированные и стандартизированные интерфейсы прикладного программирования (API) для решений группового аудио/видео связи и потоковой трансляции с низкой задержкой. Он также предоставляет решения по комплектам разработки программного обеспечения (SDK), совместимые с основными операционными системами и платформами для различных отраслей и сценариев. С помощью TRTC вы можете быстро разработать услуги интерактивного аудио/видео с низкой задержкой и высоким качеством при низких затратах.

Как лидер отрасли в области облачных услуг PaaS в реальном времени для аудио/видео, TRTC придает большое значение безопасности данных и конфиденциальности пользователей. TRTC всегда уделяет первостепенное внимание безопасности данных и конфиденциальности пользователей и включает их в ежедневную разработку возможностей безопасности. Чтобы помочь вам понять возможности защиты безопасности TRTC, далее описывается разработка безопасности и аудит соответствия безопасности услуг PaaS TRTC.

## 3. Безопасность данных

Безопасность данных является одной из главных забот TRTC. TRTC обрабатывает ваши данные законным и соответствующим образом, необходимым для обеспечения безопасности данных. В этом разделе описаны технические средства управления безопасностью данных и политика управления, реализованные Tencent Cloud и TRTC.

### 3.1 Политика безопасности данных

TRTC сосредоточена на конфиденциальности, целостности и высокой доступности как предпосылках развития безопасности данных и включает управление безопасностью данных и разработку в свои практики разработки. Tencent Cloud всегда будет привержена обеспечению доступности, конфиденциальности и целостности ваших данных следующим образом:

- **Доступность**: гарантирует высокую доступность данных через приватный сетевой протокол передачи Tencent Cloud.
- **Конфиденциальность**: предотвращает несанкционированный доступ и подслушивание.
- **Целостность**: обеспечивает целостность ваших данных и защищает данные от подделки.

TRTC предоставляет всем своим сотрудникам регулярное обучение по безопасности данных, соответствию приватности и защите шифрования данных и заключает соглашения о конфиденциальности с сотрудниками для обеспечения доступности, конфиденциальности и целостности данных при ежедневных операциях.

### 3.2 Высокая доступность данных

TRTC стремится предоставлять высокодоступные услуги данных PaaS для аудио/видео:

- **Глобально распределенные IDC**: TRTC имеет множество IDC, предоставляющих услуги глобально. Любая атака на один IDC не может повлиять на другие или общие услуги, реализуя защиту на основе изоляции.
- **Изоляция сбоев и восстановление**: если IDC подвергается вредоносным атакам, которые трудно предотвратить, таким как атака отказа в обслуживании (DoS), TRTC разумно обработает неисправный сервер, чтобы обеспечить общую стабильность и доступность услуг.
- **Смягчение DDoS**: TRTC развернула брандмауэры защиты от DDoS в каждом IDC и имеет достаточные возможности и ресурсы для контроля риска атак DDoS.

### 3.3 Сбор данных

TRTC собирает только поля данных с согласия пользователя и только те, которые необходимы для услуг и в минимальной степени детализации. Данные пользователя, собранные вами, такие как информация входа в приложение, идентификация, пароли, информация об оплате, имя и адрес, хранятся у вас, а не на платформе TRTC.

### 3.4 Маскирование данных

Для защиты вашей конфиденциальности данных TRTC маскирует информацию об предприятии и личные данные в консоли перед отображением. Эта политика также применяется к внутренним системам TRTC и другим продуктам, таким как платформа внутреннего управления, печать журнала и каналы мониторинга и тревог.

### 3.5 Использование и хранение данных

- Ваши личные или корпоративные данные пользователя, данные конечного пользователя, данные аудио/видео вызовов и данные операций и безопасности системы классифицируются перед сохранением для обеспечения соответствия и безопасности ваших данных.
- Среды разработки, тестирования и производства строго изолированы во время разработки TRTC, чтобы обеспечить, что ваши фактические данные не будут напрямую использоваться для разработки и тестирования. Кроме того, конфиденциальная информация вас и пользователей, такая как пароли, будет храниться в зашифрованном виде.
- Разработчики и пользователи, которые используют SDK записи на локальном сервере и функцию облачной записи, предоставляемую TRTC, могут записывать все или часть содержимого вызова, и все записи видео/аудио напрямую записываются на сервер хранения, предоставленный разработчиками и пользователями, а не на сервер TRTC для хранения.

## 4. Безопасность услуг PaaS TRTC

Решение для низкоселективного и высокого качества реального времени имеет строгие требования к TRTC. При разработке услуг PaaS для аудио/видео TRTC полностью оценивает технические и риски безопасности для своей архитектуры, в максимальной степени следует системе управления рисками безопасности в стандарте соответствия и реализует его во время разработки услуг PaaS для аудио/видео, чтобы предоставить набор высокого качества, стабильные и защищенные решения PaaS для аудио/видео для разработчиков и пользователей.

### 4.1 Безопасность сетевой передачи

На основе приватной сети передачи Tencent Cloud TRTC разработала платформу для аудио/видео, характеризующуюся ультра низкой задержкой, высокой передачей качества и поддерживающую реальное взаимодействие для миллионов пользователей. Приватная сеть передачи является одной из основных услуг TRTC PaaS. Она предоставляет совместимую и безопасную поддержку услуг для подключения сигнализации, аутентификации, планирования в реальном времени и передачи данных аудио/видео в реальном времени на терминале TRTC. Кроме того, для предоставления безопасных и стабильных услуг разработчикам и пользователям с учетом факторов безопасности, с которыми сталкивается текущая интернет-среда, дизайн архитектуры приватной сети передачи Tencent Cloud включает следующие средства управления.

| Управление безопасностью сетевой передачи | Описание |
| --- | --- |
| Зашифрованная передача | Чтобы обеспечить конфиденциальность данных аудио/видео во время передачи, TRTC предоставляет встроенное шифрование и пользовательское шифрование для линии передачи. По умолчанию встроенное шифрование включено глобально для TRTC PaaS, охватывая всю цепочку данных. Это обеспечивает безопасность шифрования передачи данных. |
| Изоляция ресурсов | TRTC выделяет выделенные ресурсы для каждого приложения TRTC (SdkAppId), чтобы обеспечить его независимость от других проектов и предоставить безопасную и надежную гарантию вычислительных ресурсов. После регистрации в консоли TRTC разработчикам и пользователям нужно только выполнить простые операции в консоли, чтобы создать приложения TRTC (SdkAppId) и выделить соответствующие ресурсы. |
| Изоляция комнат | TRTC создает независимый канал изоляции (Roomid) для передачи данных каждого типа аудио, видео или сообщений. Все комнаты логически отделены, и пользователь может присоединиться к одному и тому же каналу, только если использует приложение TRTC с одинаковыми `SdkAppid` и одним и тем же названием комнаты. Комната создается при запуске сеанса и завершается при окончании сеанса (когда последний пользователь уходит). Таким образом, изоляция передачи реализуется на уровне комнаты. |
| Проверка идентичности | Когда пользователь использует приложение TRTC и подключается к услугам TRTC PaaS, TRTC будет использовать информацию аутентификации, сгенерированную на основе `SdkAppid` и ключа, для выполнения аутентификации входа в комнату, чтобы помочь разработчикам и пользователям аутентифицировать своих пользователей посредством надежной аутентификации. |

TRTC также стремится создавать совместимые и безопасные услуги PaaS для аудио/видео для разработчиков и пользователей, с целью снижения их усилий по борьбе с нормативными требованиями соответствия и угрозами безопасности данных и информации.

Эта спецификация определяет детальные меры управления и стандарты реализации услуг, которые полностью отражаются в физической безопасности окружающей среды, плановых проверках, мониторинге исключений и отчетности, а также в поддержке энергоресурсов в IDC и соответствуют требованиям безопасности соответствия и основного развития безопасности TRTC. |

Эта схема может обнаруживать и защищаться от атак DDoS из сетевого слоя и слоя передачи в реальном времени. Она отслеживает сетевой трафик в реальном времени, оперативно очищает трафик, как только обнаруживается атака, и обеспечивает защиту за считанные секунды для TRTC. |

### 4.4 Безопасность веб-API

Чтобы облегчить разработчикам эффективную разработку и управление собственным аудио/видео бизнесом, TRTC предоставляет некоторые функции в консоли в виде RESTful API для вызова разработчиками. RESTful API предоставляют следующую защиту безопасности:

| Защита безопасности | Описание |
| --- | --- |
| Аутентификация | Перед использованием TRTC RESTful API разработчикам сначала необходимо войти в консоль Tencent Cloud и создать свой выделенный `SecretId` и `SecretKey`, чтобы обеспечить уникальность идентичности поставщика услуг. |
| Проверка входных данных | Действительность параметров запроса разработчика будет проверена на сервере TRTC для фильтрации недействительных параметров, чтобы избежать уязвимостей, уязвимых для атак. |
| Безопасность передачи | RESTful API поддерживают только протокол HTTPS, чтобы гарантировать шифрование всех коммуникаций API с помощью SSL/TLS. Это помогает защитить учетные данные API и передаваемые данные. |
| Ограничение частоты запросов API | На сервере существует ограничение на частоту запросов API, ограничивающее запросы API от вредоносных пользователей при обеспечении ответов на запросы обычных пользователей. |

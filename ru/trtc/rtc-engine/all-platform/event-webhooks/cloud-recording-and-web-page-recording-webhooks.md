# Вебхуки облачной записи и записи веб-страниц

Этот документ описывает события обратного вызова [обновленной функции облачной записи](https://www.tencentcloud.com/document/product/647/45169) и [функции записи веб-страниц](https://www.tencentcloud.com/document/product/647/72325).

## Информация о конфигурации

На [консоли Tencent RTC](https://console.trtc.io/app) вы можете настроить информацию обратного вызова. После завершения конфигурации вы сможете получать уведомления о событиях обратного вызова.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/6e9ee85c9c0f11efaaca525400fdb830.jpeg)

> **Примечание:** Вам необходимо заранее подготовить следующую информацию и завершить [конфигурацию обратного вызова](https://www.tencentcloud.com/document/product/647/39559) в консоли. **Обязательно:** адрес сервера HTTP/HTTPS для получения уведомлений обратного вызова. **Опционально:** [ключ для вычисления подписи](https://www.tencentcloud.com/document/product/647/54914#.E8.AE.A1.E7.AE.97.E7.AD.BE.E5.90.8D). Вы можете настроить ключ из максимум 32 символов, состоящий из прописных и строчных букв и цифр.

## Повторная попытка при истечении времени ожидания

Если ваш сервер не отвечает в течение 5 секунд после отправки уведомления сервером обратного вызова события, это считается неудачным уведомлением. При неудаче первого уведомления выполняется немедленная повторная попытка. Если уведомление снова не удается, повторные попытки будут выполняться с интервалом **10 секунд** до тех пор, пока сообщение хранится в течение 1 минуты, после чего повторные попытки не будут выполняться.

## API обратного вызова

Вы можете назначить шлюз услуги HTTP/HTTPS для подписки на сообщения обратного вызова. Когда происходит связанное событие, система облачной записи отправляет уведомление о событии на ваш сервер получения сообщений.

### Формат сообщения обратного вызова события

Сообщения обратного вызова события отправляются на ваш сервер через HTTP/HTTPS POST-запросы, в которых:

- **Формат кодирования символов**: UTF-8.
- **Запрос**: тело находится в формате JSON.
- **Ответ**: HTTP STATUS CODE = 200. Сервер игнорирует конкретное содержимое пакета ответа. Для обеспечения совместимости с протоколом рекомендуется, чтобы клиент поместил следующее в содержимое ответа: JSON: {"code":0}.

### Описание параметров облачной записи

#### Заголовок сообщения обратного вызова события содержит следующие поля:

| Имя поля | Значение |
| --- | --- |
| Content-Type | application/json |
| Sign | Значение подписи |
| SdkAppId | идентификатор приложения sdk |

#### Тело сообщения обратного вызова события включает следующие поля:

| Имя поля | Тип | Описание |
| --- | --- | --- |
| EventGroupId | Число | Идентификатор группы события, для облачной записи зафиксирован на 3 |
| EventType | Число | Тип события для уведомления об обратном вызове |
| CallbackTs | Число | Временная метка Unix (в миллисекундах) при отправке запроса обратного вызова сервером обратного вызова события на ваш сервер |
| EventInfo | JSON-объект | Информация о событии |

#### Описание типов событий:

| Имя поля | Тип | Описание |
| --- | --- | --- |
| EVENT_TYPE_CLOUD_RECORDING_RECORDER_START | [301](https://www.tencentcloud.com/document/product/647/54914#301) | Модуль облачной записи запускается. |
| EVENT_TYPE_CLOUD_RECORDING_RECORDER_STOP | [302](https://www.tencentcloud.com/document/product/647/54914#302) | Модуль облачной записи завершает работу. |
| EVENT_TYPE_CLOUD_RECORDING_UPLOAD_START | [303](https://www.tencentcloud.com/document/product/647/54914#303) | Задача загрузки файла облачной записи начинается, обратный вызов осуществляется только при выборе COS. |
| EVENT_TYPE_CLOUD_RECORDING_FILE_INFO | [304](https://www.tencentcloud.com/document/product/647/54914#304) | Облачная запись: создание файла индекса M3U8. После первого успешного создания и загрузки обратный вызов осуществляется только при записи API с выбранным COS. |
| EVENT_TYPE_CLOUD_RECORDING_UPLOAD_STOP | [305](https://www.tencentcloud.com/document/product/647/54914#305) | Загрузка файла облачной записи завершена. Обратный вызов осуществляется только при выборе COS. |
| EVENT_TYPE_CLOUD_RECORDING_FAILOVER | [306](https://www.tencentcloud.com/document/product/647/54914#306) | Происходит миграция облачной записи. Срабатывает при миграции существующей задачи записи на новую нагрузку. |
| EVENT_TYPE_CLOUD_RECORDING_FILE_SLICE | [307](https://www.tencentcloud.com/document/product/647/54914#307) | Облачная запись: создание файла индекса M3U8 (создание первого среза ts). После создания обратный вызов осуществляется только при записи API с выбранным COS. |
| EVENT_TYPE_CLOUD_RECORDING_DOWNLOAD_IMAGE_ERROR | [309](https://www.tencentcloud.com/document/product/647/54914#309) | Ошибка при попытке облачной записи загрузить и декодировать файл изображения. |
| EVENT_TYPE_CLOUD_RECORDING_MP4_STOP | [310](https://www.tencentcloud.com/document/product/647/54914#310) | Задача записи MP4 облачной записи останавливается, обратный вызов осуществляется только при записи API с выбранным COS (когда автоматическая запись включена в консоли и выбранное авторизованное VOD COS выбрано в качестве хранилища, обратите внимание на событие 311). |
| EVENT_TYPE_CLOUD_RECORDING_VOD_COMMIT | [311](https://www.tencentcloud.com/document/product/647/54914#311) | Задача записи VOD облачной записи завершила загрузку медиаресурсов, обратный вызов осуществляется при выборе **Video on Demand** и **автоматической записи в COS** через консоль (после окончания записи файла передается информация об индексе VOD. Подпишитесь на этот тип события обратного вызова). |
| EVENT_TYPE_CLOUD_RECORDING_VOD_STOP | [312](https://www.tencentcloud.com/document/product/647/54914#312) | Задача VOD облачной записи останавливается, обратный вызов осуществляется только при выборе VOD. |

> **Примечание:** Статусы обратного вызова типов событий 301–309 являются промежуточными состояниями записи в реальном времени, которые помогают вам лучше понять процесс записи и отслеживать статус. Успешная загрузка фактического файла записи на видео по запросу вызовет обратный вызов события 311, и общая задача завершена, обратный вызов события 312.

Запись страницы автоматически обрабатывает эти два статуса, поэтому клиентам не требуется выполнять никаких действий.

> **Примечание:** При получении обратного вызова с кодом статуса 5 задача записи будет прервана и больше не будет повторных попыток. Пожалуйста, обратитесь в деловую отдел для устранения неполадок.

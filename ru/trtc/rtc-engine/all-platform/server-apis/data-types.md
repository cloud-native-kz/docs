# Типы данных

## AbnormalEvent

Информация о событии ошибки (возможная причина аномального взаимодействия с пользователем).

Используется действиями: DescribeUnusualEvent.

| Имя | Тип | Описание |
| --- | --- | --- |
| AbnormalEventId | Integer | ID события ошибки. Подробнее см. https://www.tencentcloud.com/document/product/647/37906?has_map=1 |
| PeerId | String | ID удаленного пользователя. Если этот параметр пуст, это означает, что событие ошибки не связано с удаленным пользователем. Примечание: это поле может возвращать null, что указывает на отсутствие допустимых значений. |

## AbnormalExperience

Информация об аномальном взаимодействии с пользователем и возможных причинах.

Используется действиями: DescribeUnusualEvent.

| Имя | Тип | Описание |
| --- | --- | --- |
| UserId | String | ID пользователя. |
| ExperienceId | Integer | ID аномального опыта. |
| RoomId | String | ID комнаты (строка). |
| AbnormalEventList | Array of [AbnormalEvent](#AbnormalEvent) | Возможные события ошибок. |
| EventTime | Integer | Время отчета. |

## AgentConfig

Параметры робота

Используется действиями: StartAIConversation.

| Имя | Тип | Требуется | Описание |
| --- | --- | --- | --- |
| UserId | String | Да | UserId робота используется для входа в комнату и инициации задач. [Примечание] Этот UserId не может повторяться с UserId хоста-зрителя [UserId](https://intl.cloud.tencent.com/document/product/647/37714) в текущей комнате. При инициации нескольких задач в комнате UserId робота не может повторяться, иначе предыдущая задача будет прервана. UserId робота должен быть уникален в комнате. |
| UserSig | String | Да | Сигнатура проверки, соответствующая UserId робота, то есть UserId и UserSig эквивалентны паролю входа робота для входа в комнату. О методе расчета см. решение по расчету [UserSig](https://cloud.tencent.com/document/product/647/45910#UserSig) TRTC. |
| TargetUserId | String | Да | UserId робота, извлекающего медиапоток. После заполнения робот будет извлекать медиапоток UserId для обработки в реальном времени |
| MaxIdleTime | Integer | Нет | Если в комнате нет потока более MaxIdleTime, служба автоматически закроет задачу. Значение по умолчанию — 60 сек. |
| WelcomeMessage | String | Нет | Приветственное сообщение робота |
| InterruptMode | Integer | Нет | Режим интеллектуального прерывания, значение по умолчанию — 0, 0 означает, что сервер прерывает автоматически, 1 означает, что сервер не прерывает, а клиент отправляет сигнал прерывания для прерывания |
| InterruptSpeechDuration | Integer | Нет | Используется, когда InterruptMode равен 0, в миллисекундах, значение по умолчанию — 500 мс. Это означает, что сервер прерывает, когда обнаруживает человеческий голос, длящийся InterruptSpeechDuration миллисекунд. |

Используется действиями: StartPublishCdnStream.

| Имя | Тип | Требуется | Описание |
| --- | --- | --- | --- |
| UserId | String | Да | [ID пользователя](https://intl.cloud.tencent.com/document/product/647/37714) робота ретрансляции в комнате TRTC, который не может совпадать с уже используемым ID пользователя. Рекомендуется включить ID комнаты в этот ID пользователя. |
| UserSig | String | Нет | Сигнатура (похожа на пароль входа), требуемая для робота ретрансляции для входа в комнату. Информацию о расчете сигнатуры см. в [Что такое UserSig?](https://intl.cloud.tencent.com/document/product/647/38104). \| |
| MaxIdleTime | Integer | Нет | Период истечения (секунды) для остановки ретрансляции автоматически после того, как все пользователи, чьи потоки смешиваются, покидают комнату. Значение не может быть меньше 5 или больше 86400 (24 часа). Значение по умолчанию: 30. |

## AudioEncode

Параметры кодирования звука.

Используется действиями: StartPublishCdnStream, UpdatePublishCdnStream.

| Имя | Тип | Требуется | Описание |
| --- | --- | --- | --- |
| SampleRate | Integer | Да | Частота дискретизации звука (Гц). Допустимые значения: 48000, 44100, 32000, 24000, 16000, 8000. |
| Channel | Integer | Да | Количество звуковых каналов. Допустимые значения: 1 (моно), 2 (стерео). |
| BitRate | Integer | Да | Битрейт звука (Кбит/с). Диапазон значений: 8-500. |
| Codec | Integer | Нет | Кодек звука. Допустимые значения: 0 (LC-AAC), 1 (HE-AAC), 2 (HE-AACv2). Значение по умолчанию — 0. Если этот параметр установлен на 2, `Channel` должен быть 2. Если он установлен на 1 или 2, `SampleRate` может быть только 48000, 44100, 32000, 24000 или 16000. |

## McuAudioParams

Параметры звука для ретрансляции.

Используется действиями: StartPublishCdnStream, UpdatePublishCdnStream.

| Имя | Тип | Требуется | Описание |
| --- | --- | --- | --- |
| AudioEncode | [AudioEncode](#AudioEncode) | Нет | Параметры кодирования звука. |
| SubscribeAudioList | Array of [McuUserInfoParams](#McuUserInfoParams) | Нет | Список разрешений микширования звука. Для API `StartPublishCdnStream`, если вы не передадите этот параметр или оставите его пустым, звуки всех якорей будут смешаны. Для API `UpdatePublishCdnStream`, если вы не передадите этот параметр, изменений в текущий список разрешений не будет; если вы передадите пустую строку, звуки всех якорей будут смешаны. Если вы не передадите ни `SubscribeAudioList`, ни `UnSubscribeAudioList`, изменений не будет. Если пользователь включен в оба параметра, звук пользователя не будет смешан. |
| UnSubscribeAudioList | Array of [McuUserInfoParams](#McuUserInfoParams) | Нет | Список блокировки микширования звука. Если вы не передадите этот параметр или оставите его пустым, списка блокировки не будет. Для API `UpdatePublishCdnStream`, если вы не передадите этот параметр, изменений в текущий список блокировки не будет; если вы передадите пустую строку, список блокировки будет сброшен. В случаях, когда `SubscribeAudioList` и `UnSubscribeAudioList` используются одновременно, необходимо указать оба параметра. Если вы не передадите ни `SubscribeAudioList`, ни `UnSubscribeAudioList`, изменений не будет. Если пользователь включен в оба параметра, звук пользователя не будет смешан. |

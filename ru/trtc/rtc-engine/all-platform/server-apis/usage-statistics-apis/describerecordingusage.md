# DescribeRecordingUsage

## 1. Описание API

Доменное имя для запроса API: trtc.intl.tencentcloudapi.com.

Этот API используется для получения информации об использовании записи TRTC.

Если период запроса составляет один день или меньше, возвращаемая статистика рассчитывается на основе пятиминутных интервалов. Если период запроса превышает один день, возвращаемая статистика рассчитывается на основе дневных интервалов.
Период запроса за один запрос не может быть больше 31 дня.
Если вы запрашиваете статистику текущего дня, возвращаемые данные могут быть неточными из-за задержки в сборке данных.
Вы можете использовать этот API для запроса исторических данных об использовании или для проверки данных, но мы не рекомендуем использовать его для критически важной бизнес-логики.
Лимит частоты запросов для этого API составляет пять вызовов в секунду.

Максимум 5 запросов можно инициировать в секунду для этого API.

Мы рекомендуем вам использовать API Explorer

## 2. Входные параметры

Следующий список параметров запроса содержит только параметры запроса API и некоторые общие параметры. Полный список общих параметров см. в разделе [Общие параметры запроса](https://www.tencentcloud.com/document/api/647/34263).

| Имя параметра | Обязательный | Тип | Описание |
| --- | --- | --- | --- |
| Action | Да | String | [Общие параметры](https://www.tencentcloud.com/document/api/647/34263). Значение, используемое для этого API: DescribeRecordingUsage. |
| Version | Да | String | [Общие параметры](https://www.tencentcloud.com/document/api/647/34263). Значение, используемое для этого API: 2019-07-22. |
| Region | Да | String | [Общие параметры](https://www.tencentcloud.com/document/api/647/34263). Дополнительную информацию см. в разделе [список поддерживаемых регионов](https://www.tencentcloud.com/document/api/647/34263#region-list) для продукта. Этот API поддерживает только: ap-guangzhou, ap-singapore. |
| StartTime | Да | String | Дата начала в формате YYYY-MM-DD. |
| EndTime | Да | String | Дата окончания в формате YYYY-MM-DD. Период запроса за один запрос не может быть больше 31 дня. |
| MixType | Да | String | Нужно ли запрашивать статистику однопоточной или смешанной записи. Допустимые значения: `single`, `multi`. |
| SdkAppId | Нет | Integer | `SDKAppID` приложения TRTC, к которому принадлежит целевая комната. Если вы не укажете этот параметр, будет возвращена статистика использования всех приложений TRTC под текущей учетной записью. |

## 3. Выходные параметры

| Имя параметра | Тип | Описание |
| --- | --- | --- |
| UsageKey | Array of String | Тип использования. Каждый элемент этого параметра соответствует элементу `UsageValue` в том порядке, в котором они указаны. |
| UsageList | Array of [TrtcUsage](https://www.tencentcloud.com/document/api/647/36760#TrtcUsage) | Данные об использовании в каждой единице времени. |
| RequestId | String | Уникальный идентификатор запроса, генерируемый сервером, будет возвращен для каждого запроса (если запрос не достигнет сервера по другим причинам, запрос не получит RequestId). RequestId требуется для определения проблемы. |

## 4. Пример

### Пример 1. Запрос статистики однопоточной записи приложения с `SDKAppID` `1400123456` с 1 января 2022 г. по 2 января 2022 г.

#### Пример входных данных

```
POST / HTTP/1.1
Host: trtc.intl.tencentcloudapi.com
Content-Type: application/json
X-TC-Action: DescribeRecordingUsage
<Common request parameters>

{
    "EndTime": "2022-01-01",
    "StartTime": "2022-01-02",
    "SdkAppId": 1400123456,
    "MixType": "single"
}
```

#### Пример выходных данных

```json
{
    "Response": {
        "UsageKey": [

            "4K"

            {
                "TimeKey": "2022-01-01 00:00:00",
                "UsageValue": [
```

# Callback субтитров разговорного AI

Conversational AI Tencent RTC предоставляет возможность отображения субтитров в реальном времени. Субтитры в реальном времени передаются через пользовательские сообщения Tencent RTC, обеспечивая синхронизацию на уровне миллисекунд с аудиопотоком разговора.

Вы можете использовать [функцию приёма пользовательских сообщений](https://www.tencentcloud.com/document/product/647/47866), предоставляемую SDK Engine RTC, для прослушивания callbacks в клиенте и получения данных субтитров AI разговора в реальном времени. **Значение cmdID фиксировано и равно 1.**

## Возможности

1. Синхронизация в реальном времени: Субтитры синхронизируются с аудиопотоком разговора с задержкой на уровне миллисекунд.
2. Гибкость: Используется формат пользовательского сообщения, что облегчает интеграцию и расширение.

## Формат сообщения

Сообщения субтитров в реальном времени имеют формат JSON. Конкретные поля следующие:

| Поле | Тип | Описание |
| --- | --- | --- |
| type | Number | Тип сообщения. 10000 указывает на субтитры в реальном времени. |
| sender | String | ID пользователя отправителя. |
| receiver | Array | Список ID пользователей получателей. Сообщение фактически транслируется в комнате. |
| payload | Object | Содержимое сообщения. Содержит детальную информацию о субтитрах. |

Объект payload содержит следующие поля:

| Поле | Тип | Описание |
| --- | --- | --- |
| text | String | Исходный текст, распознанный ASR (STT). |
| start_time | String | Время начала предложения. Формат: "HH:MM:SS". |
| end_time | String | Время окончания предложения. Формат: "HH:MM:SS". |
| roundid | String | Уникальный ID, идентифицирующий раунд разговора. |
| end | Boolean | Если значение true, предложение является полным. |

## Пример сообщения

```
{  "type": 10000,  "sender": "user_a",  "receiver": [],  "payload": {    "text": "Hello. Nice to meet you."    "start_time": "00:00:01",    "end_time": "00:00:03",    "roundid": "conversation_123456",    "end": true  }}
```

## Замечания по реализации

1. Обработка сообщений: Получатели должны корректно разбирать JSON сообщение и распознавать субтитры в реальном времени на основе поля type.
2. Синхронизация времени: Вы можете использовать start_time и end_time для обеспечения правильной синхронизации субтитров с аудио.
3. Сегментация разговора: Вы можете использовать поле end для определения того, завершено ли предложение. Это может использоваться для отображения субтитров или сохранения полного разговора.

## Разбор пользовательского сообщения с помощью Web SDK

```
    trtcClient.on(Tencent RTC.EVENT.CUSTOM_MESSAGE, (event) => {        let data = new TextDecoder().decode(event.data);        let jsonData = JSON.parse(data);        console.log(`receive custom msg from ${event.userId} cmdId: ${event.cmdId} seq: ${event.seq} data: ${data}`);                if (jsonData.type == 10000 && jsonData.payload.end == false) {            // The subtitle is not complete.        } else if (jsonData.type == 10000 && jsonData.payload.end == true) {           // The sentence is finished.         }    });
```

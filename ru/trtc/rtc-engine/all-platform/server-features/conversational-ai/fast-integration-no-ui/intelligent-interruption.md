# Интеллектуальное прерывание

## Обзор

Интеллектуальное прерывание — это ключевая функция в режиме реального времени голосовых разговоров, которая напрямую влияет на пользовательский опыт. Опираясь на богатый набор схем прерывания, AI-диалог Tencent Real-Time Communication (Tencent RTC) предоставляет клиентам гибкие и мощные решения для удовлетворения различных потребностей сценариев. Наша функция интеллектуального прерывания в основном разделена на режимы автоматического и ручного прерывания, что эффективно повышает плавность и естественность разговора.

### Автоматическое прерывание

Автоматическое прерывание означает, что система будет автоматически инициировать прерывание на основе звукового ввода пользователя при обнаружении определённых условий. Оно поддерживает два основных правила: прерывание на основе длительности звука и прерывание на основе семантики.

1. **Прерывание на основе длительности звука**
- Пользователи могут динамически настраивать длительность прерывания, которая по умолчанию установлена на 500 мс.
- Регулируемый диапазон: от 300 мс (более чувствительный) до 5000 мс (для избежания ложных срабатываний).
- Рекомендация:
  - Для сценариев с высокой интерактивностью, таких как разговоры в центре обслуживания клиентов, рекомендуется установить более короткую длительность (например, 300 мс или 500 мс) для повышения скорости отклика.
  - Для сценариев с речью или лекциями рекомендуется установить более длительное время (например, 1000 мс) для избежания ложных прерываний.
2. **Прерывание на основе семантики**

На основе технологии определения активности голоса (VAD), когда сервер обнаруживает, что пользователь ввёл семантически полное предложение, он автоматически инициирует прерывание.

3. **Взаимодействие между автоматическим и ручным прерываниями**

В режиме автоматического прерывания всё ещё возможно использовать SDK на стороне клиента для отправки пользовательских сигналов прерывания для специфического ручного управления прерыванием, обеспечивая более тонкий интерактивный опыт.

### Ручное прерывание

Ручное прерывание — это действие, инициированное пользователем, которое достигается путём отправки пользовательского сообщения через SDK на стороне клиента. Это обеспечивает пользователям более прямой метод управления. Обычно оно зависит от кнопок или горячих клавиш в пользовательском интерфейсе.

1. **Описание сигнала прерывания**

Ручное прерывание использует пользовательские сообщения в следующем формате JSON:

```
{
  "type": , // Fixed type
  "sender": "user_a", // Sender's userid. The server will verify its validity.
  "receiver": ["user_bot"], // List of receiver userids. The robot's userid is entered.
  "payload": {
    "id": "uuid", // Message id, for debugging purposes
    "timestamp": 123 // Timestamp, for debugging purposes
  }
}
```

2. **Использование**

Вы можете выполнить ручное прерывание, отправив вышеуказанное пользовательское сообщение через метод `sendCustomCmdMessage` SDK.

## Варианты использования

1. Разговор в центре обслуживания клиентов: используется более короткая длительность автоматического прерывания вместе с ручным прерыванием для достижения быстрого отклика и точного управления.
2. Онлайн-образование: используется более длительное время автоматического прерывания для избежания ошибочного прерывания речи студентов. Преподаватели могут использовать ручное прерывание для необходимых вмешательств.
3. Групповое совещание: автоматическое и ручное прерывания комбинируются для гибкого управления порядком и длительностью выступлений.

# Интеллектуальный голосовой сервис клиентов

## Введение в сценарий

Интеллектуальный голосовой сервис клиентов использует Искусственный интеллект (ИИ) и Автоматическое распознавание речи (ASR) для автоматизации взаимодействия с клиентами и эффективного решения проблем.

Традиционно такие системы полагались на обработку естественного языка и алгоритмы машинного обучения для понимания намерений клиентов, в сочетании с предопределенными правилами и базами знаний для предоставления ответов. С развитием больших языковых моделей (LLM) современные системы интеллектуального обслуживания клиентов теперь могут глубже понимать контекст беседы, обеспечивая связные и контекстно релевантные диалоги, которые тесно имитируют человеческую беседу.

Интегрируя технологию Real-Time Communication (RTC), вы можете дополнительно улучшить решение для интеллектуального обслуживания клиентов:

- Включение возможности аудио и видеосвязи в реальном времени для бесперебойного взаимодействия с клиентами
- Предоставление мгновенных ответов на запросы клиентов с немедленной обратной связью и решениями
- Поддержка многостороннего вызова и совместного использования экрана для повышения эффективности и качества поддержки клиентов

## Решение для реализации

Комплексное решение для интеллектуального голосового сервиса клиентов состоит из нескольких основных модулей: Real-Time Audio/Video, AI Real-Time Conversation, Large Language Models (LLM) и Text-to-Speech (TTS). В таблице ниже указаны ключевые возможности каждого модуля:

| **Функция** | **Приложение AI интеллектуального голосового сервиса клиентов** |
| --- | --- |
| Real-Time Audio/Video | Обеспечивает непрерывную, стабильную потоковую передачу аудио и видео с минимизированной задержкой и дрожанием, обеспечивая высокое качество, сравнимое с вызовами человеческого агента. Это позволяет естественному взаимодействию, которое улучшает удовлетворенность пользователей. |
| AI Real-Time Conversation | Обеспечивает гибкую интеграцию с несколькими сервисами LLM для поддержки взаимодействия в реальном времени между ИИ-агентами и пользователями в аудио и видеоформате. Благодаря глобальной низколатентной сети Tencent RTC, задержка голосовой беседы может быть снижена до 1 секунды, обеспечивая естественный, похожий на человека диалог с беспрепятственной интеграцией. |
| Large Language Model (LLM) | Позволяет системе понимать контекст беседы и поддерживать связные, контекстно релевантные диалоги. LLM захватывают семантическую информацию, распознают намерения пользователя и связывают предыдущий диалог с текущим взаимодействием для более интеллектуальных ответов. |
| Text-to-Speech (TTS) | Поддерживает интеграцию с решениями TTS третьих сторон и позволяет настройку через данные обучения или корректировку параметров модели. Сервис TTS может генерировать голосовой выход в соответствии с конкретными требованиями и предлагать различные стили голоса на основе предпочтений пользователя или потребностей сценария. |

### Предварительные требования

#### Подготовка LLM

AI Real-Time Conversation поддерживает любую модель LLM, совместимую с протоколом OpenAI, а также платформы, такие как Tencent Cloud Agent Development Platform, Dify и Coze. Полный список поддерживаемых платформ см. в [Руководстве по конфигурации LLMConfig](https://trtc.io/document/68338?product=conversationalai).

**Использование Retrieval-Augmented Generation (RAG)**

Для сценариев интеллектуального голосового сервиса клиентов организациям обычно необходимо интегрировать собственные базы знаний, включая собственные документы и материалы Q&A. Это требует расширенных возможностей поиска через LLM+RAG. Разработчики могут реализовать интерфейс, совместимый с OpenAI API, в своем бэкенде, чтобы отправлять обогащенные контекстом запросы моделям третьих сторон.

Для руководства по реализации см. демонстрацию: [LLM RAG Service](https://github.com/Tencent-RTC/conversational-ai-cloudbase/tree/main/llm-rag).

> **Примечание:** Использование функций LLM, таких как RAG или Function Call, может увеличить время начального ответа токена, что приведет к более высокой задержке ответа ИИ. Если ваше приложение чувствительно к задержкам, мы рекомендуем использовать SystemPrompt вместо RAG.

#### Подготовка Text-to-Speech (TTS)

**Использование Tencent Cloud TTS**

1. [Активируйте сервис TTS](https://console.tencentcloud.com/tts) для вашего приложения, чтобы включить синтез речи
2. Получите `APPID` из [Информация об учетной записи](https://console.tencentcloud.com/developer)
3. Получите `SecretId` и `SecretKey` из [Управление API ключами](https://console.tencentcloud.com/cam/capi). Обратите внимание, что `SecretKey` отображается только один раз при создании, поэтому немедленно сохраните его
4. Просмотрите доступные стили голоса в [Списке голосов](https://www.tencentcloud.com/document/product/1154/48916)

**Использование TTS третьих сторон или пользовательского TTS:**

Для поддерживаемых конфигураций см. [Конфигурация Text-to-Speech (TTSConfig)](https://trtc.io/document/68340?product=conversationalai).

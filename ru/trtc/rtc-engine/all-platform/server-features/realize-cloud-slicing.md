# Реализация облачного срезания

## Описание сценария

В сценариях дистанционного образования, трансляции прямых эфиров, видеоконференций, удаленной оценки убытков, финансовой аудиовизуальной записи и онлайн-консультаций врачей, учитывая потребности в сборе доказательств, контроле качества, просмотре, архивировании и воспроизведении, часто необходимо сохранять весь процесс видеозвонка или прямой трансляции через облачное хранилище срезанных данных.

## Стоимость облачного срезания

Для облачной модерации, инициированной через TRTC, TRTC взимает с вас плату только за аудиосрезы и скриншоты видео, в то время как аудит третьих сторон будет взимать плату за услуги аудита в соответствии с их собственными правилами выставления счетов. Подробную информацию о стоимости аудиосрезов и скриншотов видео см. в разделе [Описание выставления счетов за аудиосрезы и скриншоты видео.](https://www.tencentcloud.com/document/product/647/64803).

## Описание функции

Функция облачного срезания TRTC позволяет выполнять облачное срезание аудио или обработку скриншотов видео потока каждого пользователя в комнате без необходимости доступа клиента.

### Определения

**Аудиосрез:** срезание аудиопотока пользователя в комнате через указанные интервалы времени, в результате чего получаются аудиосегменты.

**Скриншот видео:** захват скриншотов видеопотока пользователя в комнате через определенные интервалы времени, в результате чего получаются изображения.

**Хранилище файлов:** поддерживает сохранение облачных срезанных файлов в COS, AWS S3 или Alibaba Cloud OSS.

**Уведомление обратного вызова:** мы поддерживаем возможность уведомления обратного вызова. Путем настройки домена обратного вызова статус события облачного срезания будет уведомлять ваш сервер обратного вызова.

### Процесс аудиосреза и скриншота видео

Как показано ниже, внутри комнаты якорь 1 и якорь 2 загрузили аудио и видеопотоки. Если вы подписаны на их потоки через робота и одновременно установлены скриншот видео и срез аудио, внутренний сервис выведет потоки от обоих якорей, разбив их на несколько отдельных аудиофайлов и файлов изображений скриншотов.

- Несколько файлов аудиосреза от якоря 1.
- Несколько файлов скриншота видео от якоря 1.
- Несколько файлов аудиосреза от якоря 2.
- Несколько файлов скриншота видео от якоря 2.

Внутренний сервис загрузит эти файлы в вашу выбранную облачную платформу хранилища (COS, AWS S3 или Alibaba Cloud OSS). Конкретный процесс срезания выглядит следующим образом:

## Интерфейс API и ограничение параллельности срезания

Лимит частоты вызовов API срезания составляет 20 qps (при необходимости [отправьте запрос](https://console.tencentcloud.com/workorder/category) для увеличения qps).

Время ожидания API составляет 6 секунд.

Одно приложение поддерживает **500 потоков срезания по умолчанию** (всего задач API срезания). Задачи, превышающие лимит параллельности, будут отклонены. Если вам требуется больше параллельных потоков, пожалуйста, [отправьте запрос](https://console.tencentcloud.com/workorder/category) для связи с нами.

Одна задача срезания поддерживает до 25 параллельных подписанных хостов в комнате. Хосты, передающие только восходящее аудио, будут занимать отдельный поток.

## Процесс выполнения задачи срезания

### Запуск задачи срезания ([CreateCloudSlice](https://trtc.io/document/72331?product=rtcengine&menulabel=core%20sdk&platform=flutter))

Для запуска облачного срезания вызовите REST API ([CreateCloudSlice](https://trtc.io/document/72331?product=rtcengine&menulabel=core%20sdk&platform=flutter)) через вашу внутреннюю службу. Особое внимание обратите на параметр Task ID (TaskId); этот параметр является уникальным идентификатором этой задачи срезания. Вам необходимо сохранить этот Task ID, так как он будет требоваться для последующих операций, направленных на интерфейс этой задачи срезания.

> **Примечание:** Инициирование задачи ([CreateCloudSlice](https://trtc.io/document/72331?product=rtcengine&menulabel=core%20sdk&platform=flutter)) Для инициирования задачи облачного срезания через API CreateCloudSlice необходимо указать параметры входа в комнату UserId и UserSig ([как получить UserSig](https://www.tencentcloud.com/document/product/647/35166?from_search=1&lang=en&pg=)) для робота срезания. Убедитесь, что UserId не совпадает с идентификаторами обычных якорей в комнате или аудитории, и не должен совпадать с выбранным UserId робота срезания в любой текущей задаче срезания внутри комнаты, иначе это приведет к отказу задачи. Указание пользователей срезания ([SubscribeStreamUserIds](https://trtc.io/document/36760?product=rtcengine&menulabel=core%20sdk&platform=flutter#subscribestreamuserids)) Вы также можете указать информацию черного и белого списков якорей для срезания через параметр SubscribeStreamUserIds. Конечно, мы также поддерживаем операции обновления во время процесса срезания. Указание места хранения ([SliceStorageParams](https://trtc.io/document/36760?product=rtcengine&menulabel=core%20sdk&platform=flutter#slicestorageparams)) Место хранения: поддерживает сохранение в AWS S3 или COS. Укажите параметры хранилища в параметре [SliceStorageParams](https://trtc.io/document/36760?product=rtcengine&menulabel=core%20sdk&platform=flutter#slicestorageparams). Формат срезания: файлы срезов изображений имеют расширение png, а файлы аудиосреза имеют расширение ogg.

### Запрос статуса задачи срезания ([DescribeCloudSlice](https://trtc.io/document/72329?product=rtcengine&menulabel=core%20sdk&platform=flutter))

При необходимости вы можете вызвать этот API для запроса статуса услуг.

### Изменение параметров задачи срезания ([ModifyCloudSlice](https://trtc.io/document/72328?product=rtcengine&menulabel=core%20sdk&platform=flutter))

При необходимости вы можете вызвать этот API для изменения параметров услуг срезания, например для указания пользователей срезания ([SubscribeStreamUserIds](https://trtc.io/document/36760?product=rtcengine&menulabel=core%20sdk&platform=flutter#subscribestreamuserids)).

### Остановка задачи срезания ([DeleteCloudSliceTask](https://trtc.io/document/72330?product=rtcengine&menulabel=core%20sdk&platform=flutter))

После успешной активации задачи срезания вы можете использовать этот API для остановки задачи.

## Управление файлами срезания

### Спецификация наименования файлов срезания

**Аудиосегменты по умолчанию названы как:**

{bucket name}/{taskId}/{userId}/audios/{sdkappid}_{roomId}_{userid}_{UTC time}.ogg

**Изображения скриншотов видео по умолчанию названы как:**

{bucket name}/{taskId}/{userId}/images/{sdkappid}_{roomId}_{userid}_{UTC time}.png

**Описание значений полей:**

| **Поле** | **Значение** |
| --- | --- |
| <taskId> | ID задачи срезания |
| <sdkappid> | SdkAppId задачи срезания |
| <roomId> | Номер комнаты задачи срезания |
| <userid> | ID пользователя задачи срезания |
| UTC time | Текущая строка времени, например: 20250106143143 |

### Поиск файлов срезания

Войдите в [консоль COS](https://console.tencentcloud.com/cos?language=en), выберите ваше заданное хранилище (Bucket) для поиска:

Путь аудиосегмента: {bucket name}/{taskId}/{userId}/audios/

Путь скриншота видео: {bucket name}/{taskId}/{userId}/images/

## События обратного вызова срезания

Мы предоставляем несколько событий обратного вызова для функции облачного срезания, чтобы помочь вам своевременно узнать о статусе обработки и завершения задач срезания.

### Настройка адреса обратного вызова срезания

Консоль Tencent Real-Time Communication (TRTC) поддерживает самостоятельную настройку информации обратного вызова. После завершения настройки вы сможете получать уведомления об событиях обратного вызова. Подробное руководство см. в разделе [Настройка обратного вызова](https://trtc.io/document/39559?product=rtcengine&menulabel=core%20sdk&platform=flutter).

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/acc367bf824f11f0b1df52540044a08e.png)

### API обратного вызова

Вы можете предоставить шлюз сервиса HTTP/HTTPS для получения обратного вызова для подписки на сообщения обратного вызова. При возникновении связанных событий система облачного срезания отправит уведомления о событиях обратного вызова на ваш сервер получения сообщений.

#### Формат сообщения события обратного вызова

Сообщения события обратного вызова отправляются на ваш сервер через POST-запрос HTTP/HTTPS, где:

Формат кодирования символов: UTF-8.

Запрос: формат body - JSON.

Ответ: HTTP STATUS CODE = 200. Сервер игнорирует содержимое пакета ответа. Для совместимости протокола рекомендуется клиенту включить JSON в содержимое ответа: {"code":0}.

### События обратного вызова срезания

#### Описание параметров

Заголовок сообщения события обратного вызова содержит следующие поля:

| Имя поля | Значение |
| --- | --- |
| Content-Type | application/json |
| Sign | значение сигнатуры |
| SdkAppId | значение ID приложения SDK |

#### Body сообщения события обратного вызова содержит следующие поля

| Имя поля | Тип | Описание |
| --- | --- | --- |
| EventGroupId | Number | ID группы событий, облачное срезание зафиксировано как 10 |
| EventType | Number | Тип события уведомления обратного вызова |
| CallbackTs | Number | Unix Timestamp запроса обратного вызова, отправленного сервером событий обратного вызова на ваш сервер, в миллисекундах |
| EventInfo | JSON Object | информация о событии |

#### Описание типа события

| Имя поля | Тип | Описание |
| --- | --- | --- |
| EVENT_TYPE_CLOUD_SLICE_START | 1001 | Запуск модуля облачного срезания |
| EVENT_TYPE_CLOUD_SLICE_STOP | 1002 | Выход из модуля облачного срезания |
| EVENT_TYPE_CLOUD_SLICE_UPLOAD_START | 1003 | Начало задачи загрузки файла облачного срезания, обратный вызов только при выборе COS |
| EVENT_TYPE_CLOUD_SLICE_FILE_INFO | 1004 | Модуль облачного срезания генерирует файлы скриншотов видео или аудиофрагментов, обратный вызов после успешной загрузки, только при выборе COS |
| EVENT_TYPE_CLOUD_SLICE_UPLOAD_STOP | 1005 | Завершение загрузки файлов облачного срезания, обратный вызов только при выборе COS |
| EVENT_TYPE_CLOUD_SLICE_UPLOAD_ERROR | 1006 | Возникла ошибка в модуле доставки облачного срезания |

#### Описание информации о событии

| Имя поля | Тип | Описание |
| --- | --- | --- |
| RoomId | String/Number | имя комнаты (тип совпадает с типом ID комнаты клиента) |
| EventTs | Number | Unix Timestamp возникновения события, единица: секунды (не рекомендуется к использованию, рекомендуется использовать EventMsTs) |
| EventMsTs | Number | Unix Timestamp возникновения события в миллисекундах |
| UserId | String | ID пользователя робота срезания |
| TaskId | String | ID задачи срезания, уникальный ID задачи облачного срезания |
| Payload | JsonObject | определение различных типов событий |

# Ротация видео

## Обзор

Мобильная трансляция в реальном времени в основном использует портретный режим, но TRTC поддерживает как альбомный, так и портретный режимы, что требует реализации различных логик ориентации страницы. В этом документе представлена информация о следующем:

- Как реализовать портретный режим, например, для видеозвонков, как в WeChat
- Как реализовать альбомный режим, например, для групповых приложений связи, таких как Zoom
- Как настроить параметры ротации и режима отображения локального изображения и удаленных изображений.

## Портретный режим

Чтобы обеспечить впечатление, аналогичное видеозвонкам в WeChat, необходимо выполнить два действия.

### 1. Установите ориентацию приложения в портретный режим.

iOS

Android

Установите ориентацию страницы в Xcode > **General** > **Deployment Info** > **Device Orientation**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/6ef3d0163a7b11ed90fd525400c56988.png)

Также можно использовать метод `supportedInterfaceOrientationsForWindow` в `Appdelegate`.

```
- (UIInterfaceOrientationMask)application:(UIApplication *)application     supportedInterfaceOrientationsForWindow:(UIWindow *)window {    return  UIInterfaceOrientationMaskPortrait ;}
```

> **примечание** Эта [статья на CSDN](https://blog.csdn.net/DreamcoffeeZS/article/details/79037207) содержит подробное руководство по ориентации страницы и адаптации на iOS для разработчиков.

Установите атрибут `screenOrientation` элемента activity на `portrait`:

```
<activity android:name=".trtc.TRTCMainActivity"  android:launchMode="singleTask" android:windowSoftInputMode="adjustPan"          android:screenOrientation="portrait" />
```

### 2. Установите ориентацию SDK в портретный режим.

iOS

Android

```
TRTCVideoEncParam* encParam = [TRTCVideoEncParam new];encParam.videoResolution = TRTCVideoResolution_640_360;encParam.videoBitrate = 600;encParam.videoFps = 15;encParam.resMode = TRTCVideoResolutionModePortrait; // Установите режим разрешения в портретный.[trtc setVideoEncoderParam: encParam];
```

```
TRTCCloudDef.TRTCVideoEncParam encParam = new TRTCCloudDef.TRTCVideoEncParam();encParam.videoResolution = TRTCCloudDef.TRTC_VIDEO_RESOLUTION_640_360;encParam.videoBitrate = 600;encParam.videoFps = 15;encParam.videoResolutionMode = TRTCCloudDef.TRTC_VIDEO_RESOLUTION_MODE_PORTRAIT; //Установите режим разрешения в портретный.trtc.setVideoEncoderParam(encParam);
```

## Альбомный режим

Шаги для реализации альбомного режима в приложении аналогичны шагам реализации портретного режима, за исключением того, что для параметров на шаге 1 и шаге 2 используются разные значения.
В частности, относительно значения `resMode` в `TRTCVideoEncParam` на [шаге 2](#step2),

- на iOS установите его на `TRTCVideoResolutionModeLandscape`.
- на Android установите его на `TRTC_VIDEO_RESOLUTION_MODE_LANDSCAPE`.

## Пользовательские настройки

SDK TRTC предоставляет различные API для установки режима ротации и отображения локального изображения и удаленных изображений.

| API | Описание | Примечания |
| --- | --- | --- |
| setLocalViewRotation | Установить поворот локального предпросмотра изображения по часовой стрелке | Повернуть на 90, 180 или 270 градусов по часовой стрелке |
| setLocalViewFillMode | Установить режим отображения локального предпросмотра изображения | Обрезать изображение или заполнить пустое пространство черными полосами |
| setRemoteViewRotation | Установить поворот удаленного видеоизображения по часовой стрелке | Повернуть на 90, 180 или 270 градусов по часовой стрелке |
| setRemoteViewFillMode | Установить режим отображения удаленного видеоизображения | Обрезать изображение или заполнить пустое пространство черными полосами |
| setVideoEncoderRotation | Установить поворот закодированного изображения по часовой стрелке | Повернуть на 90, 180 или 270 градусов по часовой стрелке |

## GSensorMode

Для адаптации при видеозаписи и трансляции CDN SDK TRTC предоставляет простую функцию адаптации на основе датчика ориентации, которую можно включить с помощью API `setGSensorMode` класса `TRTCCloud`.

Эта функция поддерживает адаптивный поворот на 90, 180 и 270 градусов. Это означает, что когда пользователь повернет телефон, ориентация изображения пользователя, видимая удаленными пользователями, остается неизменной. Поскольку эта функция реализуется путем поворота на основе кодера, адаптивный поворот также возможен для записанных видео и видео, воспроизводимых в проигрывателях HTML5.

> **внимание** Еще один способ достичь адаптивного поворота — это встроить направление гравитации видео в информацию каждого видеокадра и отрегулировать степень поворота видео на конце зрителя. Эта схема требует введения дополнительных ресурсов транскодирования для регулировки ориентации записанных видео в соответствии с ожиданиями и поэтому не рекомендуется.

## enableVoiceEarMonitor

**enableVoiceEarMonitor**

| void enableVoiceEarMonitor | (boolean enable) |
| --- | --- |

**Включение мониторинга в наушниках.**

После включения мониторинга в наушниках ведущие смогут слышать в наушниках свой собственный голос, захватываемый микрофоном. Это предназначено для сценариев пения.

Мониторинг в наушниках не может быть включен для беспроводных наушников. Это объясняется тем, что беспроводные наушники имеют высокую задержку. Пожалуйста, попросите ведущих использовать проводные наушники через напоминание в интерфейсе.

Учитывая, что не все телефоны обеспечивают отличные эффекты мониторинга в наушниках, мы отключили эту функцию на некоторых телефонах.

| Param | DESC |
| --- | --- |
| enable | ` true: ` включить; ` false `: отключить |

> **Примечание** Мониторинг в наушниках можно включить только при использовании наушников. Пожалуйста, напомните ведущим использовать проводные наушники.

## setVoiceEarMonitorVolume

**setVoiceEarMonitorVolume**

| void setVoiceEarMonitorVolume | (int volume) |
| --- | --- |

**Установка громкости мониторинга в наушниках.**

Этот API используется для установки громкости мониторинга в наушниках.

| Param | DESC |
| --- | --- |
| volume | Громкость. Диапазон значений: [0, 150]; по умолчанию: 100 |

> **Примечание** Громкость мониторинга в наушниках — один из наиболее важных факторов, влияющих на пользовательский опыт. Учитывая, что разные пользователи по-разному чувствительны к громкости, настоятельно рекомендуется предоставить ползунок регулировки громкости мониторинга в наушниках в интерфейсе пользователя. Рекомендуемый диапазон регулировки находится между 0 и 150. Эффект усиления 100-150 очень полезен для некоторых телефонов Android с низким уровнем записи.

## seekMusicToPosInMS

**seekMusicToPosInMS**

| void seekMusicToPosInMS | (int id |
| --- | --- |
|  | int pts) |

**Установка прогресса воспроизведения (мс) фоновой музыки.**

| Param | DESC |
| --- | --- |
| id | ID музыки |
| pts | Единица: миллисекунда |

> **Примечание** Не вызывайте этот API часто, так как музыкальный файл может быть прочитан и записан каждый раз при вызове API, что может быть времени заинтересованным. Дождитесь завершения пользователем перетаскивания полосы прогресса, прежде чем вызывать этот API. Контроллер полосы прогресса в интерфейсе пользователя имеет тенденцию обновлять прогресс с высокой частотой, когда пользователи перетаскивают полосу прогресса. Это приведет к плохому пользовательскому опыту, если вы не ограничите частоту.

## TRTCVideoStreamType

**TRTCVideoStreamType**

**Тип видеопотока.**

TRTC предоставляет три различных видеопотока, включая:

- HD большое изображение: обычно используется для передачи видеоданных с камеры.
- Плавное малое изображение: имеет то же содержание, что и большое изображение, но с более низким разрешением и битрейтом, а следовательно, и более низким качеством.
- Изображение подпотока: обычно используется для общего доступа к экрану. Только один пользователь в комнате может публиковать видеоизображение подпотока одновременно, в то время как другие пользователи должны дождаться закрытия подпотока этим пользователем перед тем, как они смогут опубликовать свой собственный подпоток.

> **Примечание**: SDK не поддерживает включение только плавного малого изображения, оно должно быть включено вместе с большим изображением. Разрешение и битрейт малого изображения будут установлены автоматически.

| Enum | Value | DESC |
| --- | --- | --- |
| TRTC_VIDEO_STREAM_TYPE_BIG | 0 | HD большое изображение: обычно используется для передачи видеоданных с камеры. |
| TRTC_VIDEO_STREAM_TYPE_SMALL | 1 | Плавное малое изображение: имеет то же содержание, что и большое изображение, но с более низким разрешением и битрейтом и таким образом более низким качеством. |
| TRTC_VIDEO_STREAM_TYPE_SUB | 2 | Изображение подпотока: обычно используется для общего доступа к экрану. Только один пользователь в комнате может публиковать видеоизображение подпотока одновременно, в то время как другие пользователи должны дождаться закрытия подпотока этим пользователем перед тем, как они смогут опубликовать свой собственный подпоток. |

## TRTCVideoFillMode

**TRTCVideoFillMode**

**Режим заполнения видеоизображения.**

Если соотношение сторон области отображения видео не равно соотношению сторон видеоизображения, необходимо указать режим заполнения:

| Enum | Value | DESC |

## TRTCAudioQuality

**TRTCAudioQuality**

**Качество звука.**

TRTC предоставляет три тщательно настроенных режима для удовлетворения дифференцированных требований к качеству звука в различных областях:

- Режим речи (Speech): подходит для сценариев использования, ориентированных на общение между людьми. В этом режиме передача аудиосигнала более устойчива к помехам, и TRTC использует различные технологии обработки голоса для обеспечения оптимальной плавности даже в условиях слабого сетевого соединения.
- Режим музыки (Music): подходит для сценариев с высокими требованиями к качеству музыки. В этом режиме объем передаваемых данных аудиосигнала очень велик, и TRTC использует различные технологии для обеспечения того, чтобы высокочастотные детали музыкальных сигналов могли быть восстановлены во всех частотных диапазонах.
- Режим по умолчанию (Default): находится между режимами `Speech` и `Music`. В этом режиме воспроизведение музыки лучше, чем в режиме `Speech`, а объем передаваемых данных намного меньше, чем в режиме `Music`. Поэтому этот режим хорошо адаптирован к различным сценариям.

| TRTC_AUDIO_QUALITY_DEFAULT | 2 | Режим по умолчанию: моноканал; битрейт: 50 Кбит/с. Этот режим находится между режимом речи и режимом музыки как режим по умолчанию в SDK и рекомендуется. |

## TRTCSystemVolumeType

**TRTCSystemVolumeType**

**Тип системного уровня громкости (только для мобильных устройств).**

Смартфоны обычно имеют два типа системного уровня громкости: уровень громкости для звонков и уровень громкости для мультимедиа.

- Уровень громкости для звонков предназначен для сценариев звонков. Он поставляется с технологией подавления акустических эхо (AEC) и поддерживает захват аудиосигнала через наушники Bluetooth, но его качество звука в среднем.

Если вы не можете снизить громкость до 0 (то есть отключить звук на телефоне) с помощью кнопок громкости, то ваш телефон использует уровень громкости для звонков.

- Уровень громкости для мультимедиа предназначен для сценариев мультимедиа, таких как воспроизведение музыки. При использовании уровня громкости для мультимедиа AEC не работает, и наушники Bluetooth не могут использоваться для захвата аудиосигнала. Однако уровень громкости для мультимедиа обеспечивает лучший опыт прослушивания музыки.

Если вы можете отключить звук на телефоне с помощью кнопок громкости, то ваш телефон использует уровень громкости для мультимедиа.

SDK предлагает три режима управления системной громкостью: автоматический, уровень громкости для звонков и уровень громкости для мультимедиа.

| Enum | Value | DESC |
| --- | --- | --- |
| TRTCSystemVolumeTypeAuto | 0 | Автоматический: в автоматическом режиме для трансляторов используется уровень громкости для звонков, а для зрителей используется уровень громкости для мультимедиа. Этот режим подходит для сценариев потокового вещания. Если сценарий, выбранный во время `enterRoom`, это `TRTCAppSceneLIVE` или `TRTCAppSceneVoiceChatRoom`, SDK автоматически использует этот режим. |
| TRTCSystemVolumeTypeMedia | 1 | Уровень громкости для мультимедиа: в этом режиме уровень громкости для мультимедиа используется во всех сценариях. Используется редко, в основном подходит для сценариев музыки с высокими требованиями к качеству звука. Используйте этот режим, если большинство ваших пользователей используют периферийные устройства, такие как звуковые карты. В противном случае это не рекомендуется. |
| TRTCSystemVolumeTypeVOIP | 2 | Уровень громкости для звонков: в этом режиме модуль аудиосигнала не меняет режим работы, когда пользователи переключаются между трансляторами и зрителями, обеспечивая бесперебойное включение/выключение микрофона. Этот режим подходит для сценариев, где пользователи часто переключаются между трансляторами и зрителями. Если сценарий, выбранный во время `enterRoom`, это `TRTCAppSceneVideoCall` или `TRTCAppSceneAudioCall`, SDK автоматически использует этот режим. |

Это самый популярный режим макета, потому что он позволяет вам установить положение каждого видеоизображения заранее через заполнители, а затем SDK автоматически динамически корректирует его в соответствии с количеством видеоизображений в комнате. В этом режиме вам все еще нужно установить параметр `mixUsers`, но вы можете установить `userId` как «заполнитель». Значения заполнителя включают: "$PLACE_HOLDER_REMOTE$": изображение удаленного пользователя. Можно установить несколько изображений. "$PLACE_HOLDER_LOCAL_MAIN$": изображение локальной камеры. Можно установить только одно изображение. "$PLACE_HOLDER_LOCAL_SUB$": изображение локального общего доступа к экрану. Можно установить только одно изображение. В этом режиме вам не нужно прослушивать обратные вызовы [onUserVideoAvailable](https://www.tencentcloud.com/document/product/647/50763#448623ba3ddafa44cdb425bea100c2d8) и [onUserAudioAvailable](https://www.tencentcloud.com/document/product/647/50763#cb979bbb36c24acc891ce2115ff2b6c6) в [TRTCCloudListener](https://www.tencentcloud.com/document/product/647/50763#3ac99d5f5509a822ae68d6d0fff9bde0) для внесения изменений в реальном времени. Вместо этого вам нужно только вызвать setMixTranscodingConfig один раз после успешного входа в комнату. Затем SDK автоматически заполнит заполнители, которые вы установили, фактическими значениями `userId`. |
| TRTC_TranscodingConfigMode_Template_ScreenSharing | 4 | Режим общего доступа к экрану. Этот режим подходит для случаев использования на основе общего доступа к экрану, таких как онлайн-образование, и поддерживается только SDK для Windows и macOS. В этом режиме SDK сначала создает холст в соответствии с целевым разрешением, которое вы установили (через параметры `videoWidth` и `videoHeight`). До того как учитель включит общий доступ к экрану, SDK масштабирует изображение камеры учителя и рисует его на холст. После того как учитель включит общий доступ к экрану, SDK рисует изображение видео, общего на экране, на тот же холст. Цель этого режима макета — обеспечить согласованность выходного разрешения модуля микширования и транскодирования и избежать проблем с размытым экраном во время повторного воспроизведения курса и воспроизведения на веб-странице (веб-плееры не поддерживают корректируемое разрешение). Между тем, аудиосигнал учеников с включенным микрофоном по умолчанию микшируется в аудио/видеопоток учителя. Видеоконтент в основном является общим экраном в режиме обучения, и передача изображения камеры и изображения экрана одновременно — это пустая трата пропускной способности. Поэтому рекомендуемый способ — прямое рисование изображения камеры на текущий экран через API setLocalVideoRenderCallback. В этом режиме вам не нужно устанавливать параметр `mixUsers` в [TRTCTranscodingConfig](https://www.tencentcloud.com/document/product/647/50768#28fdad3591324ef129b20463677f1399), и SDK не будет микшировать изображения учеников, чтобы не мешать эффекту общего доступа к экрану. Вы можете установить width x height в [TRTCTranscodingConfig](https://www.tencentcloud.com/document/product/647/50768#28fdad3591324ef129b20463677f1399) на 0 px x 0 px, и SDK автоматически рассчитает подходящее разрешение на основе соотношения сторон текущего экрана пользователя. Если текущая ширина экрана учителя меньше или равна 1920 px, SDK использует фактическое разрешение текущего экрана учителя. Если текущая ширина экрана учителя больше 1920 px, SDK выберет одно из трех разрешений 1920x1080 (16:9), 1920x1200 (16:10) и 1920x1440 (4:3) в соответствии с текущим соотношением сторон экрана. |

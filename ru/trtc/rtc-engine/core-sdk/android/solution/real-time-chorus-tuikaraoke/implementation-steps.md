# Этапы реализации

## Введение

Для реализации полного сценария онлайн-караоке требуется несколько функциональных модулей, включая управление комнатой, управление местами, управление выбором песен и управление караоке. Ключевые действия и функции каждого функционального модуля показаны в таблице ниже. В следующих разделах каждый функциональный модуль будет подробно описан для полного понимания необходимых функций при создании комнаты караоке.

| Управление комнатой | Управление местами | Управление выбором песен | Управление караоке |
| --- | --- | --- | --- |
| Список комнат | Подняться/сойти с места | Отображение списка песен | Режим воспроизведения караоке |
| Создать комнату | Управление местом | Поиск песен | Переключение песен |
| Присоединиться к комнате | Заблокировать место | Выбор песни | Регулировка громкости вокала |
| Покинуть комнату | Занять место | Топ песен | Ревербер/Звуковые эффекты |
| Удалить комнату | Отключить звук места | Список выбранных песен | Синхронизация текста |

Владелец комнаты создает комнату караоке, и пользователи могут выбрать присоединиться к интересующей их комнате. После входа в комнату пользователи могут подняться на место для участия во взаимодействии и голосовому общению с владельцем комнаты. Конечно, пользователи также могут выбрать прямое поднятие на место для участия в хоре. Это два разных режима воспроизведения караоке. Общий бизнес-процесс сценария онлайн-караоке показан на рисунке ниже.

## Управление комнатой

Управление комнатой в основном отвечает за ведение списка комнат. Основные функции включают создание комнаты, присоединение к комнате, удаление комнаты и выход из комнаты. Кроме того, комнаты караоке отличаются от обычных комнат и требуют отдельного идентификатора комнаты караоке для запуска управления связанными компонентами, такими как управление выбором песен и управление караоке.

- **Создать комнату**: После входа в бизнес-систему пользователи могут создать комнату. После создания комнаты список комнат необходимо обновить новой комнатой.
- **Удалить комнату**: После того как все пользователи покинут комнату, комнату необходимо удалить. После удаления комнаты список комнат необходимо обновить с удалением комнаты.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/9254b9845b6f11ee974d5254005f490f.png)

> **Примечание**: Управление комнатой — это необходимый модуль для реализации онлайн-караоке, но он не является основным функциональным модулем. Конкретная реализация может быть объединена с бизнес-системой и TRTC SDK; для получения дополнительной информации см. решение для доступа к сценарию голосового чата.

## Управление местами

Места в комнате караоке обычно упорядочены и ограничены. Управление местами в основном отвечает за определение количества мест в комнате и управление состоянием всех мест в текущей комнате в соответствии со сценарием бизнеса. Управление местами в основном включает следующие функции: поднятие/сход с места, блокировку места, приглашение подняться на место и отключение звука места.

- После входа в комнату пользователи могут подать заявку на подъем на место только для мест, которые находятся в свободном состоянии.
- После согласия хозяина позволить пользователю подняться на место, состояние места необходимо изменить на не свободное состояние.
- После того как пользователь прекратит трансляцию и сойдет с места, состояние места необходимо сбросить.
- Хозяин имеет право блокировать место, приглашать подняться на место, принудительно сбивать со сцены и отключать звук на месте.

> **Примечание**: Управление местами — это необходимый модуль для реализации онлайн-караоке, но он не является основным функциональным модулем. Конкретная реализация может быть объединена с бизнес-системой и IM SDK; для получения дополнительной информации см. решение для доступа к сценарию голосового чата.

## Управление выбором песен

### Базовое введение

Управление выбором песен — это важная часть сценария онлайн-караоке, которая в основном включает следующие функции: отображение списка песен, поиск песен, выбор песен и управление очередью, а также список выбранных песен. Кроме того, каждая комната караоке должна поддерживать список выбранных песен и функцию автоматического управления очередью, что требует реализации на бизнес-бэкенде. Отображение списка песен и поиск песен необходимо объединить с авторизованной музыкой Yinsuda для прямого вещания.

### Процесс реализации

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b8f46a305b6f11eeabd75254005810a4.png)

Все управление выбором песен в основном включает бизнес-приложение, бизнес-бэкенд и бэкенд Yinsuda, каждый с его собственными функциями:

- **Бизнес-приложение**:
  - Вызов API выбора песен для отчета информации о песне.
  - Вызов API обрезки песни для уведомления бизнес-бэкенда об обновлении списка выбранных песен.
  - Вызов API подтверждения пения для уведомления бизнес-бэкенда.
- **Бизнес-бэкенд**:
  - Ведение списка выбранных песен.
  - Отправка уведомлений бизнес-приложению для обновления текущего списка выбранных песен.
- **Бэкенд Yinsuda**:
  - Предоставление API для получения рекомендуемого списка песен и деталей списка песен для интерактивной музыки в прямом эфире «Список песен/Детали списка песен».
  - Предоставление API для получения деталей интерактивной музыки в прямом эфире «Получить детали интерактивной музыки в прямом эфире (playToken, URL загрузки текста)».
  - Предоставление API для поиска интерактивной музыки в прямом эфире «Поиск интерактивной музыки в прямом эфире».

## Управление караоке

Система караоке в основном включает следующие функции: геймплей пения, запуск/остановка/обрезка песни, регулировка громкости вокала, ревербер/звуковые эффекты и синхронизация текста. Ниже мы подробно рассмотрим процесс реализации модуля управления караоке через два типичных геймплея караоке: сольное пение и пение в реальном времени.

### Сольное пение

Это в основном многопользовательский интерактивный сценарий Karaoke. После того как хост подняться на место, он может выбрать песни для пения. После того как хост успешно выберет песню, вся информация о выборе песни будет отображена на платформе выбора песен. Хост может затем выбрать начать пение.

#### **(1) Архитектура решения**

Архитектура общего решения в основном использует VOD SDK для загрузки песен, VOD бэкенд для получения playToken и адреса загрузки текста песни, и TRTC SDK для реализации потоковой передачи голоса певца, воспроизведения песни и потоковой передачи. | Певец | Певец в комнате караоке развивается из хоста, который выбирает песни и поет после поднятия на место. | Аудитория | Аудитория в комнате караоке воспроизводит поток певца. | Роль — аудитория, но может также стать хостом, подняться на местоПотоковая передача аудио и видео вниз по течениюПолучение информации SEI (получение информации текста) |

Базовые процессы реализации для разных ролей выглядят следующим образом:

**Хост**

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/d156cbb45b6f11ee94c3525400d793d0.png)

- Хост создает и присоединяется к комнате TRTC, автоматически поднимается на место и становится певцом после выбора песни.
- После выбора песни песня/текст загружается, а затем песня воспроизводится через интерфейс воспроизведения BGM.
- Если певец не поднимает потоковую передачу видео, необходимо включить потоковую передачу видео.
- Синхронизируйте прогресс текста для всех через информацию SEI.
- Певец может обрезать песню в любой момент процесса пения, а затем загрузить и петь песню/текст снова после завершения загрузки.
- После того как хост покидает комнату, комната TRTC автоматически распускается.

**Аудитория**

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/8a0e920a5c2e11ee94c3525400d793d0.png)

- Аудитория присоединяется к комнате TRTC.
- Слушайте изменения в песне комнаты и загружайте текст.
- Вытяните поток певца.
- Разберите информацию SEI, отправленную певцом, и синхронизируйте текст.

Основная задача — слушать информацию SEI песни и обновлять соответствующее управление песней.

### Пение в реальном времени

Пение в реальном времени относится к одновременному воспроизведению песен на различных концах при подключении, а затем совместному пению на месте. В многопользовательском режиме певцы могут слышать голоса друг друга почти без задержки, достигая истинного пения в реальном времени.

#### **(1) Архитектура решения**

С точки зрения медиа-потоков певцы отправляют и получают потоки друг другу, один ***ведущий певец отправляет музыку***, в то время как другие ***певцы воспроизводят музыку локально***, с синхронизацией времени через NTP. Кроме того, песня и голоса всех певцов смешиваются и обрабатываются в один поток робот-микшером, а затем отправляются обратно в комнату TRTC. Аудитории нужно только вытянуть один поток, чтобы услышать синхронизированные голоса со всех концов, идеально достигая эффекта пения нескольких человек. Архитектура решения для пения в реальном времени показана на следующем рисунке.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/22243bde5b7111eeabd75254005810a4.png)

Преимущества этого решения:

- Снижает задержку от конца к концу.
- Обеспечивает решение для пользователей присоединиться к хору на полпути.
- Точно синхронизирует музыку, текст и вокал между разными концами.
- Улучшает производительность устройств на разных концах и точность локального времени, а также снижает влияние задержки сетевой среды.

> **Примечание**: В зависимости от бизнес-потребностей вы можете выбрать решение для пения в реальном времени либо для чистого аудио, либо для аудио и видео сценариев. Если это чистый аудио сценарий, черные кадры должны быть добавлены для отправки SEI сообщений для синхронизации текста. Ведущий певец должен использовать субэкземпляр для одновременной потоковой передачи музыки и вокала; другие певцы только должны вытянуть потоки вокала друг друга и воспроизвести музыку локально; аудитория только нужно вытянуть один смешанный поток. На рисунке показано решение для просмотра RTC, где робот-микшер отправляет смешанный поток обратно в комнату RTC; в решении для просмотра CDN робот-микшер отправляет смешанный поток в CDN прямого вещания, и аудитория вытягивает поток CDN для просмотра.

#### **(2) Конкретная реализация**

Мы можем разделить пользователей в комнате онлайн-караоке на три роли: ведущий певец, хор и аудитория, как показано в таблице ниже.

| Роль | Описание | Различия |
| --- | --- | --- |
| Ведущий певец | Ведущий певец отвечает за выбор песен, отправку сигналов хора и отправку сообщений SEI. | Роль должна быть AnchorПотоковая передача музыки и вокалаВыбор песни и инициирование хораОтправка смешанного потока обратноОтправка SEI сообщений |
| Хор | Хор может получать и обрабатывать сигналы хора и участвовать в хоре на месте. | Роль должна быть AnchorПотоковая передача вокалаВоспроизведение музыки локальноПолучение сигналов хора |
| Аудитория | После входа в комнату караоке аудитория может вытянуть поток с места и также может участвовать в хоре на месте. | Роль должна быть AudienceПотоковая передача смешанного потока вниз по течениюПолучение SEI сообщений Подайте заявку на то, чтобы стать Anchor для поднятия на место |

Базовые процессы реализации для разных ролей показаны на следующем рисунке:

**Ведущий певец**

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/065b6a775b7011ee94c3525400d793d0.png)

- Ведущий певец должен выбрать песню и отправить сигналы хора.
- Ведущий певец создает субэкземпляр для отправки вокала и музыки, и вытягивает вокал других певцов.
- После отправки потока ведущий певец отвечает за инициирование задачи отправки смешанного потока.
- После начала выступления воспроизведите музыку и синхронизируйте текст через обратный вызов прогресса воспроизведения.
- SEI сообщения должны быть отправлены для синхронизации прогресса песни на конце аудитории.
- Все певцы должны калибровать локальный прогресс воспроизведения песни в соответствии с NTP.

**Хор**

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/1a334ad85b7011ee9ff8525400d917da.png)

- Хор отправляет один поток вокала и вытягивает поток вокала пользователя на месте.
- Хор должен слушать и получать сигналы хора, а также предварительно загружать музыкальные ресурсы.
- После начала выступления воспроизведите музыку локально, и хор синхронизирует текст через обратный вызов прогресса воспроизведения.
- Все певцы должны калибровать локальный прогресс воспроизведения песни в соответствии с NTP.

**Аудитория**

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/b2b307e35b7011ee9ff8525400d917da.png)

- Вытяните смешанный поток для прослушивания хора.
- Разберите информацию о прогрессе песни в SEI смешанного потока для синхронизации текста.
- После поднятия на место остановите вытягивание смешанного потока, переключитесь на вытягивание потока вокала на месте и начните режим хора.

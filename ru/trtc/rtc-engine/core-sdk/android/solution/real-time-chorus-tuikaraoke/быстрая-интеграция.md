# Быстрая интеграция

## Обзор компонента

TUIKaraoke — это компонент пользовательского интерфейса для аудио/видео с открытым исходным кодом, который вы можете интегрировать в свой проект для добавления функций онлайн-караоке, управления местами, отправки/получения подарков, текстового чата и других возможностей TRTC в ваше приложение. TUIKaraoke требует всего несколько строк кода и также поддерживает платформу iOS. Его основные функции показаны ниже:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/a4afb1df3a7b11ed90fd525400c56988.png)

## Интеграция компонента

### Шаг 1. Загрузка и импорт компонента TUIKaraoke

Перейдите на [GitHub](https://github.com/tencentyun/TUIKaraoke), клонируйте или загрузите код, скопируйте директории `Source` и `Debug` из директории `Android` в ваш проект и выполните следующие операции импорта:

- Выполните импорт в `setting.gradle` как показано ниже:

```
include ':Source'include ':Debug'
```

- Добавьте зависимости на `TUIKaraoke` в файл `build.gradle` в `app`:

```
api project(':Source')
```

- Добавьте зависимости на `TRTC SDK` и `IM SDK` в файл `build.gradle` в корневую директорию:

```
ext {  liteavSdk = "com.tencent.liteav:LiteAVSDK_TRTCl:latest.release"  imSdk = "com.tencent.imsdk:imsdk-plus:latest.release"}
```

### Шаг 2. Настройка запросов разрешений и правил обфускации

Настройте запросы разрешений для вашего приложения в `AndroidManifest.xml`. SDK требуют следующих разрешений (на Android 6.0 и более поздних версиях разрешения на доступ к микрофону и чтение хранилища должны запрашиваться во время выполнения):

```
<uses-permission android:name="android.permission.SYSTEM_ALERT_WINDOW" />        // Use case: This permission is required for the floating window<uses-permission android:name="android.permission.INTERNET" /><uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" /><uses-permission android:name="android.permission.ACCESS_WIFI_STATE" /><uses-permission android:name="android.permission.RECORD_AUDIO" /><uses-permission android:name="android.permission.MODIFY_AUDIO_SETTINGS" /><uses-permission android:name="android.permission.BLUETOOTH" />                  // Use case: This permission is required when a Bluetooth headset is used
```

В файле `proguard-rules.pro` добавьте классы SDK в список «не обфусцировать».

```
-keep class com.tencent.** { *;}
```

### Шаг 3. Инициализация и вход в компонент

Дополнительная информация о соответствующих API см. в [TUIKaraoke](https://intl.cloud.tencent.com/document/product/647/41943).

```
  // 1. Initialize  TRTCKaraokeRoom mTRTCKaraokeRoom = TRTCKaraokeRoom.sharedInstance(this);  mTRTCKaraokeRoom.setDelegate(this);  // 2. Log in  mTRTCKaraokeRoom.login(SDKAppID, UserID, UserSig, new TRTCKaraokeRoomCallback.ActionCallback() {      @Override      public void onCallback(int code, String msg) {          if (code == 0) {          // Logged in          }      }  });
```

**Описание параметров:**

- **SDKAppID**: **TRTC application ID** (идентификатор приложения TRTC). Если вы еще не активировали службу TRTC, войдите в [консоль TRTC](https://console.tencentcloud.com/trtc/app), создайте приложение TRTC и нажмите **Application Info**. Информация `SDKAppID` показана на рисунке ниже:
![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/5bf13347b5d011eeb2a1525400170219.png)
- **Secretkey**: **TRTC application key** (ключ приложения TRTC), который соответствует `SDKAppID`. На странице [Application Management](https://console.tencentcloud.com/trtc/app) в консоли TRTC `SecretKey` показан ниже:
- **userId**: текущий идентификатор пользователя, представляющий собой строку, которая может содержать до 32 байт букв и цифр (специальные символы не поддерживаются). Вы можете настроить его на основе вашей фактической системы учетных записей.
- **userSig**: сигнатура защиты безопасности, вычисляемая на основе `SDKAppID`, `userId` и `Secretkey`. Вы можете нажать [здесь](https://console.trtc.io/usersig), чтобы напрямую сгенерировать отладку `userSig` онлайн. Для получения дополнительной информации см. [UserSig](https://intl.cloud.tencent.com/document/product/647/35166).

### Шаг 4. Реализация сценария онлайн-караоке

1. **Якорь создает комнату через**[TUIKaraoke.createRoom](https://intl.cloud.tencent.com/document/product/647/41943).

```
int roomId = "Room ID";TRTCKaraokeRoomDef.RoomParam roomParam = new TRTCKaraokeRoomDef.RoomParam();roomParam.roomName = "Room name";roomParam.needRequest = false; // Whether your consent is required for listeners to speakroomParam.seatCount = 8;       // Number of seats in the room. Set it to `8`roomParam.coverUrl = "URL of room cover image";mTRTCKaraokeRoom.createRoom(roomId, roomParam, new TRTCKaraokeRoomCallback.ActionCallback() { @Override public void onCallback(int code, String msg) {   if (code == 0) {     // Room created successfully   } }});
```

2. **Слушатель входит в комнату через**[TUIKaraoke.enterRoom](https://intl.cloud.tencent.com/document/product/647/41943).

```
mTRTCKaraokeRoom.enterRoom(roomId, new TRTCKaraokeRoomCallback.ActionCallback() { @Override public void onCallback(int code, String msg) {     if (code == 0) {     // Entered room successfully     } }});
```

3. **Слушатель включает микрофон через**[TUIKaraoke.enterSeat](https://intl.cloud.tencent.com/document/product/647/41943).

```
// 1. A listener calls an API to mic onint seatIndex = 1;mTRTCKaraokeRoom.enterSeat(seatIndex, new TRTCKaraokeRoomCallback.ActionCallback() { @Override public void onCallback(int code, String msg) {     if (code == 0) {     // Mic turned on successfully     } }});// 2. The listener receives the `onSeatListChange` callback and refreshes the seat list@Overridepublic void onSeatListChange(final List<TRTCKaraokeRoomDef.SeatInfo> seatInfoList) {}
```

> **Примечание:** Вы можете реализовать другие операции управления местами в соответствии с инструкциями в [TRTCKaraoke (Android)](https://intl.cloud.tencent.com/document/product/647/41943) или посмотрев [демо-проект TUIKaraoke](https://github.com/tencentyun/TUIKaraoke/).

4. **Воспроизведение песен и апробация сценария караоке**
Вы можете получить идентификатор музыки и URL для воспроизведения песни в зависимости от вашего бизнеса. Дополнительную информацию см. в [API воспроизведения музыки](https://www.tencentcloud.com/document/product/647/41943#music-playback-apis2).

```
// Play back the musicmTRTCKaraokeRoom.startPlayMusic(musicID,url);// Stop the musicmTRTCKaraokeRoom.stopPlayMusic();
```

После выполнения предыдущих шагов вы можете реализовать базовые функции караоке. Если ваш бизнес требует дополнительных функций, таких как текстовый чат и отправка подарков, вы можете интегрировать следующие возможности:

### Шаг 5. Добавление функции текстового чата (необязательно)

Если вы хотите получить функцию текстового чата между якорем и слушателями, реализуйте отправку/получение сообщений следующим образом:
Дополнительная информация о соответствующих API см. в [sendRoomTextMsg](https://intl.cloud.tencent.com/document/product/647/41943#sendroomtextmsg).

```
// Sender: Sends text messagesmTRTCKaraokeRoom.sendRoomTextMsg("Hello Word!", new TRTCKaraokeRoomCallback.ActionCallback() {    @Override    public void onCallback(int code, String msg) {        if (code == 0) {            // Sent successfully        }    }});// Receiver: Listens for text messagesmTRTCKaraokeRoom.setDelegate(new TRTCKaraokeRoomDelegate() {  @Override  public void onRecvRoomTextMsg(String message, TRTCKaraokeRoomDef.UserInfo userInfo) {      Log.d(TAG, "Received a message from" + userInfo.userName + ": " + message);  }});
```

### Шаг 6. Добавление функции отправки подарков (необязательно)

Если вы хотите получить функции отправки и получения подарков, реализуйте отправку, получение и отображение подарков следующим образом:

```
// Sender: Customize `CMD_GIFT` to distinguish between gift messagesmTRTCKaraokeRoom.sendRoomCustomMsg("CMD_GIFT",date, new TRTCKaraokeRoomCallback.ActionCallback() {    @Override    public void onCallback(int code, String msg) {        if (code == 0) {            // Sent successfully        }    }});// Receiver: Listens for gift messagesmTRTCKaraokeRoom.setDelegate(new TRTCKaraokeRoomDelegate() {    @Override    public void onRecvRoomCustomMsg(String cmd, String message, TRTCKaraokeRoomDef.UserInfo userInfo) {        if ("CMD_GIFT".equals(cmd)) {            // Received a gift message            Log.d(TAG, "Received a gift from" + userInfo.userName + ": " + message);        }    }});
```

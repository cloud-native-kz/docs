# Синхронизация вокала

## Введение в синхронизацию вокала и песни

Из-за джиттерного буфера локального сбора вокала, джиттерного буфера воспроизведения песни, а также определённого промежутка между воспроизведением звука и восприятием человеком, когда певец поёт под текст и фонограмму (BGM), удалённая аудитория ощущает определённую задержку при воспроизведении фонограммы, вокала и текста. Схема хорового режима использует низкозадержное сбор данных AAudio внутри TRTC SDK. В частности, вам нужно только включить режим хора и режим низкой задержки после входа в комнату.

## Конкретная реализация кода

### Включение режима хора

```
// Основной экземпляр (экземпляр вокала) включает режим хора (уменьшение интервала буфера и защита от избыточности аудиоданных)mTRTCCloud.callExperimentalAPI("{\\"api\\":\\"enableChorus\\",\\"params\\":{\\"enable\\":1,\\"audioSource\\":0}}");// Подчиненный экземпляр (экземпляр аккомпанемента) включает режим хора (уменьшение интервала буфера и защита от избыточности аудиоданных)subCloud.callExperimentalAPI("{\\"api\\":\\"enableChorus\\",\\"params\\":{\\"enable\\":1,\\"audioSource\\":1}}");
```

> **Примечание:** Параметры для экспериментального интерфейса enableChorus для включения режима хора: audioSource=0 (вокал), audioSource=1 (аккомпанемент).

### Включение режима низкой задержки (высокопроизводительный аудиокодек AAudio)

```
// Основной экземпляр (экземпляр вокала) включает высокопроизводительный аудиокодек AAudiomTRTCCloud.callExperimentalAPI("{\\"api\\":\\"setLowLatencyModeEnabled\\",\\"params\\":{\\"enable\\":1}}");// Подчиненный экземпляр (экземпляр аккомпанемента) включает высокопроизводительный аудиокодек AAudiosubCloud.callExperimentalAPI("{\\"api\\":\\"setLowLatencyModeEnabled\\",\\"params\\":{\\"enable\\":1}}");
```

# Синхронизация текстов песен

## Процесс реализации

В схеме синхронизации текстов песен действия трех различных ролей выглядят следующим образом:

| Певец (лидер) | Хор | Аудитория |
| --- | --- | --- |
| Калибровка времени NTPВключение компенсации черных кадровОтправка SEI сообщенияЛокальная синхронизация текстовОбновление виджета текстов | Калибровка времени NTPЛокальная синхронизация текстовОбновление виджета текстов | Калибровка времени NTPПолучение SEI сообщенияОбновление виджета текстов |

Певец и хор обновляют локально прогресс воспроизведения текстов в соответствии с синхронизированным прогрессом воспроизведения песни; аудитория должна получить SEI сообщение, отправленное певцом, которое содержит последний прогресс текстов, для обновления локального прогресса текстов.

## Диаграмма синхронизации

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/78a659b75ce211ee974d5254005f490f.png)

Синхронизация текстов песен может быть разделена в основном на три части: калибровка времени NTP, включение компенсации черных кадров и локальная и удаленная синхронизация текстов. Реализация кода для калибровки времени NTP приведена в разделе [синхронизация песен](https://www.tencentcloud.com/document/product/647/57026), а конкретная реализация кода для последних двух частей будет предоставлена ниже.

## Ключевые реализации кода

### **1.** Включение компенсации черных кадров

```
// В режиме чистого аудио основной экземпляр (экземпляр вокала) должен включить компенсацию черных кадров для передачи SEI сообщения.mTRTCCloud.callExperimentalAPI("{\\"api\\":\\"enableBlackStream\\",\\"params\\": {\\"enable\\":true}}");
```

> **Примечание:** Экспериментальный интерфейс enableBlackStream должен быть вызван после входа в комнату. На Android тип значения параметра enable является boolean, а на iOS это integer. Получатель должен вызвать startRemoteView(userId, null) при получении onUserVideoAvailable(userId, true).

### **2.** Отправка прогресса песни через SEI сообщение

```
mAudioEffectManager.setMusicObserver(mCurPlayMusicId, new TXAudioEffectManager.TXMusicPlayObserver() {    @Override    public void onPlayProgress(int id, long curPtsMS, long durationMS) {        JSONObject jsonObject = new JSONObject();        // Текущее время NTP        long ntpTime = TXLiveBase.getNetworkTimestamp();        jsonObject.put("bgmProgressTime", curTime);        jsonObject.put("ntpTime", ntpTime);        jsonObject.put("musicId", musicId);        jsonObject.put("duration", duration);        jsonObject.toString().getBytes();        mTRTCCloud.sendSEIMsg(jsonObject.toString().getBytes(), 1);    }}
```

> **Примечание:** Частота отправки певцом SEI сообщений определяется частотой обратных вызовов событий воспроизведения фоновой музыки, обычно 200 мс; Причина **неиспользования CMD сообщений для отправки прогресса песни**: Сигналы, передаваемые через канал SEI, могут передаваться в прямую трансляцию CDN вместе с видеокадрами, обеспечивая лучшую совместимость для аудитории, получающей поток CDN.

### **3.** Локальная и удаленная синхронизация текстов

```
// Локальная синхронизация текстовmAudioEffectManager.setMusicObserver(mCurPlayMusicId, new TXAudioEffectManager.TXMusicPlayObserver() {    @Override    public void onPlayProgress(int id, long curPtsMS, long durationMS) {        ...        // TODO Логика обновления виджета текстов:        // Определите, нужно ли перемещать виджет текстов на основе последнего прогресса и ошибки локального прогресса текстов        ...    }}// Удаленная синхронизация текстов@Overridepublic void onRecvSEIMsg(String userId, byte[] data) {    String result = new String(data);    JSONObject jsonObject = new JSONObject(result);    long bgmProgressTime = jsonObject.getLong("bgmProgressTime");    long ntpTime = jsonObject.getLong("ntpTime");    String musicId = jsonObject.getString("musicId");    long duration = jsonObject.getLong("duration");    ...    // TODO Логика обновления виджета текстов:    // Если вы переиспользуете виджет текстов компонента TUIKaraoke,     // пожалуйста, обратитесь к логике кода раздела TUIKaraoke LyricsView для синхронизации прогресса виджета текстов.    ...}
```

> **Примечание:** Если вы переиспользуете виджет текстов компонента TUIKaraoke, пожалуйста, обратитесь к логике кода раздела [TUIKaraoke LyricsView](https://github.com/tencentyun/TUIKaraoke/blob/main/Android/tuikaraoke/src/main/java/com/tencent/liteav/tuikaraoke/ui/lyric/LyricView.java) для синхронизации прогресса виджета текстов.

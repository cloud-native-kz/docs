# Часто задаваемые вопросы

## Проблемы мониторинга звука в наушниках

### В сценариях караоке обычно используется мониторинг звука в наушниках. Как включить эту функцию?

```
mTRTCCloud.getAudioEffectManager().enableVoiceEarMonitor(true)
```

### Что делать, если после включения функции мониторинга звука в наушниках эффект отсутствует?

Из-за высокой аппаратной задержки беспроводных наушников рекомендуется предложить пользователю использовать проводные наушники. Кроме того, обратите внимание, что не все телефоны обеспечивают хороший эффект мониторинга звука после включения этой функции. SDK TRTC уже отключил эту функцию на некоторых телефонах с плохим качеством мониторинга.

### Задержка мониторинга звука слишком высокая?

Пожалуйста, проверьте, используете ли вы беспроводные наушники. Из-за высокой аппаратной задержки беспроводных наушников рекомендуется использовать проводные наушники.

Кроме того, вы можете попытаться снизить высокую задержку мониторинга звука, включив аппаратный мониторинг звука через экспериментальный интерфейс setSystemAudioKitEnabled. В настоящее время для устройств Huawei и Vivo SDK использует аппаратный мониторинг звука по умолчанию, а для других устройств используется программный мониторинг по умолчанию.

```
// Включить аппаратный мониторинг звука в наушникахmTRTCCloud.callExperimentalAPI("{\\"api\\":\\"setSystemAudioKitEnabled\\", \\"params\\": {\\"enable\\":1}}");
```

## Проблемы синхронизации времени NTP

### Напоминание: **«NTP time sync finished, but result maybe inaccurate»**

Синхронизация времени NTP успешна, но отклонение может быть более 30 мс, что указывает на плохое состояние сети клиента и непрерывный jitter RTT.

### Напоминание: **«Error in AddressResolver: No address associated with hostname»**

Ошибка синхронизации времени NTP может быть вызвана временными сбоями в разрешении DNS локального провайдера в текущей сетевой среде. Пожалуйста, попробуйте позже.

## Рекомендации по тестированию скорости сети

Сценарии онлайн-караоке предъявляют высокие требования к сетевому подключению пользователей, особенно при синхронном пении. Для хорошего качества караоке необходима стабильная и высокоскоростная сетевая среда. Поэтому рекомендуется выполнить тест скорости сети перед входом пользователя в комнату и показать предупреждение на уровне UI пользователям, которые не соответствуют требованиям к сети, запретив им присоединяться к комнате караоке или участвовать в синхронном пении.

Инициирование теста скорости сети с помощью SDK TRTC:

```
TRTCCloudDef.TRTCSpeedTestParams speedTestParams = new TRTCCloudDef.TRTCSpeedTestParams();speedTestParams.sdkAppId = SDK_APP_ID;speedTestParams.userId = userId;speedTestParams.userSig = userSig;// Если фактическая пропускная способность выше ожидаемого значения, результат теста является ожидаемым значением;// если фактическая пропускная способность ниже ожидаемого значения, результат теста является фактическим значением.speedTestParams.expectedDownBandwidth = 3000; // Ожидаемая пропускная способность по нисходящему каналу, от 10 до 5000 кбит/сspeedTestParams.expectedUpBandwidth = 3000; // Ожидаемая пропускная способность по восходящему каналу, от 10 до 5000 кбит/сmTRTCCloud.startSpeedTest(speedTestParams);
```

> **Примечание:** Ожидаемая пропускная способность по восходящему каналу, от 10 до 5000 кбит/с. Выполняйте тест скорости сети перед входом в комнату. Тестирование скорости сети в комнате будет негативно влиять на нормальную передачу видео и аудио, а из-за слишком большого количества помех результаты теста скорости сети также будут неточными.

Обратный вызов результата теста скорости сети SDK TRTC:

```
@Overridepublic void onSpeedTestResult(TRTCCloudDef.TRTCSpeedTestResult result) {    String tquality = "Unknown";    switch (result.quality) {        case 0:            tquality = "Unknown";            break;        case 1:            tquality = "The current network is very good";            break;        case 2:            tquality = "The current network is good";            break;        case 3:            tquality = "The current network is average";            break;        case 4:            tquality = "The current network is poor";            break;        case 5:            tquality = "The current network is very poor";            break;        case 6:            tquality = "The current network does not meet TRTC's minimum requirements";            break;    }    if (result.success) {        mTextTestResult.append("Speed test successful!" + "\\n");        mTextTestResult.append("IP address: " + result.ip + "\\n");        mTextTestResult.append("Uplink packet loss rate: " + result.upLostRate + "\\n");        mTextTestResult.append("Downlink packet loss rate: " + result.downLostRate + "\\n");        mTextTestResult.append("Network latency: " + result.rtt + "ms\\n");        mTextTestResult.append("Downlink bandwidth: " + result.availableDownBandwidth + "kbps\\n");        mTextTestResult.append("Uplink bandwidth: " + result.availableUpBandwidth + "kbps\\n");        mTextTestResult.append("Network quality: " + tquality + "\\n");    } else {        mTextTestResult.append("Speed test failed!" + "\\n");        mTextTestResult.append("Error message: " + result.errMsg + "\\n");    }}
```

## Присоединение к синхронному пению в процессе

Решение для синхронного пения в реальном времени теоретически не имеет ограничений на количество участников пения, поддерживает одновременное участие нескольких человек в пении, а также присоединение в процессе.

Ниже приведены ключевые действия для присоединения к синхронному пению в процессе:

- Синхронизация времени NTP
- Включение экспериментального интерфейса синхронного пения
- Вход в комнату и начало трансляции с микрофона
- Получение сигналов синхронного пения, получение ресурсов аккомпанемента и согласованного времени
- Расчет разницы между согласованным временем и текущим временем, предварительная загрузка и поиск аккомпанемента
- Начало участия в синхронном пении и синхронизация прогресса аккомпанемента и прогресса текста в реальном времени

Для получения конкретного процесса реализации и примеров кода вышеуказанных ключевых действий см. документацию по [синхронизации песен](https://www.tencentcloud.com/zh/document/product/647/57026), [синхронизации текста](https://www.tencentcloud.com/zh/document/product/647/57028), [синхронизации вокала](https://www.tencentcloud.com/zh/document/product/647/57033).

# Запуск примера кода

В этом документе приводится пошаговое руководство по началу работы с TRTC Android SDK через [наш пример проекта на Github.](https://github.com/Tencent-RTC/TRTC_Android)

## Предварительные требования

- **Минимально поддерживаемая версия**: Android 4.4 (SDK API уровень 19); **Рекомендуется**: Android 5.0 (SDK API уровень 21) или позже.
- Android Studio 4.0 или позже.
- Устройства с Android 4.4 или позже.

## Настройка проекта

### Шаг 1. Загрузите проект

Загрузите наш SDK и демонстрационный код с [GitHub](https://github.com/Tencent-RTC/TRTC_Android) или выполните следующую команду для клонирования репозитория.

```
git clone https://github.com/Tencent-RTC/TRTC_Android.git
```

### Шаг 2. Настройте проект

1. Войдите в [консоль TRTC](https://console.trtc.io/), нажмите **Create Application** в разделе **Applications**. Вы можете пропустить этот шаг, если у вас уже есть приложение.
2. Получите `SDKAppID` и `SDKSecretKey` вашего приложения в разделе **Basic Information**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/a22576de39ac11ef9b60525400bdab9d.png)

3. Замените `SDKAppID` и `SDKSecretKey` в `GenerateTestUserSig.java`, находящемся в директории `TRTC-API-Example/Debug/src/main/java/com.tencent.trtc.debug`.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/a25a5b0f39ac11ef9c9c525400d5f8ef.png)

> **Примечание:** В приведённой выше демонстрации мы использовали **SDKSecretKey** для локального создания **UserSig**, чтобы облегчить вам работу с демонстрацией. Однако в рабочей среде вам не следует создавать userSig таким образом, так как это может привести к утечке **SDKSecretKey** и создать возможность для злоумышленников украсть ваш трафик TRTC. **Правильный способ создания UserSig — это интеграция** [**Server-Side Generation of UserSig**](https://trtc.io/document/35166) **на вашем сервере.** Когда пользователь входит в комнату: отправьте HTTP запрос на ваш сервер; создайте UserSig на вашем сервере; верните его пользователю для входа в комнату. При развёртывании вашей страницы в рабочей среде вам необходимо получить доступ к странице через HTTPS (например, `https://domain/xxx`). Для получения дополнительной информации см. документ [Page Access Protocol Restriction Description](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/tutorial-05-info-browser.html#h2-3).

### Шаг 3. Запустите проект

Откройте корневую директорию проекта (`/TRTC-API-Example/`) в Android Studio, затем запустите проект.

### Шаг 4. Ознакомьтесь с функциями TRTC

Вы можете выбрать и изучить функции, соответствующие вашим интересам:

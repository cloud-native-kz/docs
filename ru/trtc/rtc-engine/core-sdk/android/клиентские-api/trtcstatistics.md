# TRTCStatistics

Copyright (c) 2021 Tencent. All rights reserved.

Модуль: метрики аудио/видео TRTC (только для чтения)

Функция: TRTC SDK передает вам текущие метрики аудио/видео в реальном времени (частота кадров, битрейт, задержку и т. д.) один раз в две секунды

| [TRTCRemoteStatistics](https://www.tencentcloud.com/document/product/647/50764#31083ca966a8ee092affeab44b65fe9a) | Метрики удаленного аудио/видео. |

## TRTCLocalStatistics

**TRTCLocalStatistics**

**Метрики локального аудио/видео.**

| EnumType | DESC |
| --- | --- |
| audioBitrate | Описание поля: локальный битрейт аудио в Кбит/с, т. е. объем данных аудио, генерируемых в секунду |
| audioCaptureState | Описание поля: статус сбора аудиооборудования (0 — нормально, 1 — обнаружена долгая тишина, 2 — обнаружено прерывистое звучание, 3 — обнаружена аномальная прерывистость звука) |
| audioSampleRate | Описание поля: локальная частота дискретизации аудио (Гц) |
| frameRate | Описание поля: локальная частота кадров видео в fps, т. е. количество видеокадров в секунду |
| height | Описание поля: высота локального видео в px |
| streamType | Описание поля: тип видеопотока (большое изображение высокого качества \| плавное маленькое изображение \| вспомогательный поток) |
| videoBitrate | Описание поля: локальный битрейт видео в Кбит/с, т. е. объем данных видео, генерируемых в секунду |
| width | Описание поля: ширина локального видео в px |

## TRTCRemoteStatistics

**TRTCRemoteStatistics**

**Метрики удаленного аудио/видео.**

| EnumType | DESC |
| --- | --- |
| audioBitrate | Описание поля: локальный битрейт аудио (Кбит/с) |
| audioBlockRate | Описание поля: частота задержки воспроизведения аудио (%)Частота задержки воспроизведения аудио (audioBlockRate) = кумулятивная продолжительность задержки воспроизведения аудио (audioTotalBlockTime)/общая продолжительность воспроизведения аудио |
| audioPacketLoss | Описание поля: общая частота потери пакетов (%) аудиопотока` audioPacketLoss ` представляет частоту потери пакетов, в конечном итоге вычисленную на стороне зрителя после того, как аудио/видеопоток проходит полный путь передачи "якорь -> облако -> зритель".Чем меньше ` audioPacketLoss `, тем лучше. Частота потери пакетов 0 указывает на то, что все данные аудиопотока полностью достигли зрителя.Если ` downLoss ` равна ` 0 `, но ` audioPacketLoss ` нет, то на канале "облако -> зритель" для аудиопотока нет потерь пакетов, но на канале "якорь -> облако" есть безвозвратные потери пакетов. |
| audioSampleRate | Описание поля: локальная частота дискретизации аудио (Гц) |
| audioTotalBlockTime | Описание поля: кумулятивная продолжительность задержки воспроизведения аудио (мс) |
| finalLoss | Описание поля: общая частота потери пакетов (%) аудио/видеопотока Не рекомендуется использовать, вместо этого используйте audioPacketLoss и videoPacketLoss. |
| frameRate | Описание поля: частота кадров удаленного видео (fps) |
| height | Описание поля: высота удаленного видео в px |
| jitterBufferDelay | Описание поля: задержка воспроизведения (мс)Чтобы избежать задержек аудио/видео, вызванных сетевыми колебаниями и нарушением порядка пакетов, TRTC поддерживает буфер воспроизведения на стороне воспроизведения для организации полученных сетевых пакетов.Размер буфера адаптивно регулируется в соответствии с текущим качеством сети и преобразуется в длительность в миллисекундах, т. е. ` jitterBufferDelay `. |
| point2PointDelay | Описание поля: сквозная задержка (мс)` point2PointDelay ` представляет задержку "якорь -> облако -> зритель". Более точно, это задержка полного пути "сбор -> кодирование -> передача по сети -> получение -> буферизация -> декодирование -> воспроизведение".` point2PointDelay ` работает только в том случае, если оба локальный и удаленный SDK имеют версию 8.5 или выше. Если удаленный SDK имеет версию ниже 8.5, это значение всегда будет 0 и, таким образом, бессмысленно. |
| remoteNetworkRTT | Описание поля: время в пути туда и обратно (мс) от SDK к облакуЭто значение представляет общее время, необходимое для отправки сетевого пакета от SDK в облако и обратно от облака к SDK, т. е. общее время для прохождения сетевого пакета через путь "SDK -> облако -> SDK".Чем меньше значение, тем лучше. Если ` remoteNetworkRTT ` ниже 50 мс, это означает короткую задержку аудио/видеовызова; если ` remoteNetworkRTT ` выше 200 мс, это означает длительную задержку аудио/видеовызова.Следует пояснить, что ` remoteNetworkRTT ` представляет общее время, затраченное на путь "SDK -> облако -> SDK"; поэтому нет необходимости различать ` remoteNetworkUpRTT ` и ` remoteNetworkDownRTT `. |
| remoteNetworkUplinkLoss | Описание поля: частота потери пакетов восходящего потока (%) от SDK к облакуЧем меньше значение, тем лучше. Если ` remoteNetworkUplinkLoss ` равна ` 0% `, качество восходящей сети очень хорошее, и пакеты данных, загруженные в облако, практически не теряются.Если ` remoteNetworkUplinkLoss ` равна ` 30% `, то 30% пакетов аудио/видео, отправленных в облако SDK, теряются при передаче. |
| streamType | Описание поля: тип видеопотока (большое изображение высокого качества \| плавное маленькое изображение \| вспомогательный поток) |
| userId | Описание поля: ID пользователя |
| videoBitrate | Описание поля: битрейт удаленного видео (Кбит/с) |
| videoBlockRate | Описание поля: частота задержки воспроизведения видео (%)Частота задержки воспроизведения видео (videoBlockRate) = кумулятивная продолжительность задержки воспроизведения видео (videoTotalBlockTime)/общая продолжительность воспроизведения видео |
| videoPacketLoss | Описание поля: общая частота потери пакетов (%) видеопотока` videoPacketLoss ` представляет частоту потери пакетов, в конечном итоге вычисленную на стороне зрителя после того, как аудио/видеопоток проходит полный путь передачи "якорь -> облако -> зритель".Чем меньше ` videoPacketLoss `, тем лучше. Частота потери пакетов 0 указывает на то, что все данные видеопотока полностью достигли зрителя.Если ` downLoss ` равна ` 0 `, но ` videoPacketLoss ` нет, то на канале "облако -> зритель" для видеопотока нет потерь пакетов, но на канале "якорь -> облако" есть безвозвратные потери пакетов. |
| videoTotalBlockTime | Описание поля: кумулятивная продолжительность задержки воспроизведения видео (мс) |
| width | Описание поля: ширина удаленного видео в px |

## TRTCStatistics

**TRTCStatistics**

**Метрики сети и производительности.**

| EnumType | DESC |
| --- | --- |
| appCpu | Описание поля: загрузка ЦПУ (%) текущего приложения, системы Android 8.0 и выше не поддерживаются |
| downLoss | Описание поля: частота потери пакетов нисходящего потока (%) от облака к SDKЧем меньше значение, тем лучше. Если ` downLoss ` равна ` 0% `, качество нисходящей сети очень хорошее, и пакеты данных, полученные из облака, практически не теряются.Если ` downLoss ` равна ` 30% `, то 30% пакетов аудио/видео, отправленных в SDK облаком, теряются при передаче. |
| gatewayRtt | Описание поля: время в пути туда и обратно (мс) от SDK к шлюзуЭто значение представляет общее время, необходимое для отправки сетевого пакета от SDK к шлюзу и обратно от шлюза к SDK, т. е. общее время для прохождения сетевого пакета через путь "SDK -> шлюз -> SDK".Чем меньше значение, тем лучше. Если ` gatewayRtt ` ниже 50 мс, это означает короткую задержку аудио/видеовызова; если ` gatewayRtt ` выше 200 мс, это означает длительную задержку аудио/видеовызова.Следует пояснить, что ` gatewayRtt ` недействителен для сотовой сети. |
| localArray | Описание поля: статистика локального аудио/видеоПоскольку может быть три локальных аудио/видеопотока (т. е. большое изображение высокого качества, плавное маленькое изображение и вспомогательный поток), статистика локального аудио/видео представляет собой массив. |
| receiveBytes | Описание поля: общее количество полученных байтов (включая данные сигнализации и аудио/видео) |
| remoteArray | Описание поля: статистика удаленного аудио/видеоПоскольку может быть несколько одновременных удаленных пользователей, и каждый из них может иметь несколько одновременных аудио/видеопотоков (т. е. большое изображение высокого качества, плавное маленькое изображение и вспомогательный поток), статистика удаленного аудио/видео представляет собой массив. |
| rtt | Описание поля: время в пути туда и обратно (мс) от SDK к облакуЭто значение представляет общее время, необходимое для отправки сетевого пакета от SDK в облако и обратно от облака к SDK, т. е. общее время для прохождения сетевого пакета через путь "SDK -> облако -> SDK".Чем меньше значение, тем лучше. Если ` rtt ` ниже 50 мс, это означает короткую задержку аудио/видеовызова; если ` rtt ` выше 200 мс, это означает длительную задержку аудио/видеовызова.Следует пояснить, что ` rtt ` представляет общее время, затраченное на путь "SDK -> облако -> SDK"; поэтому нет необходимости различать ` upRtt ` и ` downRtt `. |
| sendBytes | Описание поля: общее количество отправленных байтов (включая данные сигнализации и аудио/видео) |
| systemCpu | Описание поля: загрузка ЦПУ (%) текущей системы, системы Android 8.0 и выше не поддерживаются |
| upLoss | Описание поля: частота потери пакетов восходящего потока (%) от SDK к облакуЧем меньше значение, тем лучше. Если ` upLoss ` равна ` 0% `, качество восходящей сети очень хорошее, и пакеты данных, загруженные в облако, практически не теряются.Если ` upLoss ` равна ` 30% `, то 30% пакетов аудио/видео, отправленных в облако SDK, теряются при передаче. |

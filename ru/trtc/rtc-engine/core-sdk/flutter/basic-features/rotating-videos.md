# Поворот видео

## Обзор

В отличие от повсеместного вертикального режима экрана при мобильной трансляции в реальном времени, Tencent Real-Time Communication (TRTC) должна поддерживать как ландшафтный, так и портретный режимы просмотра, что требует обширной обработки ориентации экрана. В этой статье основное внимание уделяется:

- Реализации паттерна портретного режима, например: видеовызовы WeChat являются типичным примером паттерна портретного режима.
- Выполнению паттерна ландшафтного режима, например: многопользовательские приложения аудио-видео комнат (такие как Little Fish Easy Connection) обычно принимают паттерн ландшафтного режима.
- Способам настройки управления направлением поворота и паттерном заполнения для локальных и удалённых изображений.

## Портретный паттерн

Для реализации паттерна опыта видеовызова, подобного WeChat, необходимо выполнить две задачи:

### 1. Настройка пользовательского интерфейса приложения на отображение в портретном режиме

Платформа iOS

Платформа Android

Вы можете установить это непосредственно в **Deployment Info** > **Device Orientation** Xcode в **General**:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/c85c94d8be5d11ee8c36525400bb593a.png)

Установив атрибут `screenOrientation` activity в portrait, можно указать, что этот интерфейс находится в портретном режиме:

```
<activity android:name=".trtc.TRTCMainActivity"  android:launchMode="singleTask" android:windowSoftInputMode="adjustPan"          android:screenOrientation="portrait" />
```

### 2. Настройка SDK на использование разрешения вертикального экрана

При использовании параметра setVideoEncoderParam облака TRTCCloud для установки параметров кодирования видео достаточно указать videoResolutionMode как `TRTCVideoResolutionMode.portrait`.
Ниже приведен пример кода:

```
_trtcCloud.setVideoEncoderParam(    TRTCVideoEncParam(      videoFps: 15,      videoResolution: TRTCVideoResolution.res_640_360,      videoBitrate: 600,      videoResolutionMode: TRTCVideoResolutionMode.portrait,    ));
```

## Ландшафтный режим

Если вы хотите, чтобы приложение имело ландшафтную ориентацию, работа, которую вам необходимо выполнить, аналогична работе портретного паттерна. Вам просто нужно соответственно отрегулировать параметры на первом и втором этапах.

В частности, на [втором этапе](#2.-.E9.85.8D.E7.BD.AE-SDK-.E4.BD.BF.E7.94.A8.E7.AB.96.E5.B1.8F.E5.88.86.E8.BE.A8.E7.8E.87), значение videoResolutionMode в TRTCVideoEncParam должно быть: `TRTCVideoResolutionMode.landscape`.

## GSensorMode

Учитывая различные проблемы совместимости, связанные с поворотом экрана, записью и трансляцией CDN, TRTC SDK предоставляет только простую функцию адаптации гравитационного датчика. Вы можете включить это через интерфейс [setGravitySensorAdaptiveMode](https://liteav.sdk.qcloud.com/doc/product/trtc/dart/api/v3/en/trtc_cloud/TRTCCloud/setGravitySensorAdaptiveMode.html) облака TRTCCloud.

Эта функция поддерживает адаптацию при повороте на 90, 180 и 270 градусов. То есть, когда телефон пользователя поворачивается, ориентация изображения, видимого другими, остаётся неизменной. Кроме того, эта адаптация основана на корректировках направления кодировщика. Поэтому записанные видео, а также изображения, просматриваемые в мини-программах и на конечных точках H5, могут сохранять исходную ориентацию.

> **Примечание:** Другая реализация адаптации гравитационного датчика включает кодирование направления гравитации каждого видеокадра, а затем адаптивную регулировку направления отрисовки на стороне удалённого пользователя. Однако эта реализация требует внедрения дополнительных ресурсов транскодирования для решения проблемы сохранения направления записанного видео в соответствии с ожидаемым направлением видео. Поэтому это не рекомендуется.

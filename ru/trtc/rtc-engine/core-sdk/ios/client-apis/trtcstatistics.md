# TRTCStatistics

Авторские права (c) 2021 Tencent. Все права защищены.

Модуль: метрики аудио/видео TRTC (только для чтения)

Функция: SDK TRTC сообщает вам текущие метрики аудио/видео в реальном времени (частота кадров, битрейт, задержка и т. д.) один раз в две секунды

## TRTCLocalStatistics

**TRTCLocalStatistics**

**Локальные метрики аудио/видео.**

| EnumType | DESC |
| --- | --- |
| audioBitrate | Описание поля: локальный битрейт аудио в Кбит/с, т. е. сколько аудиоданных генерируется в секунду |
| audioCaptureState | Описание поля: статус сбора аудиооборудования (0 — нормально, 1 — обнаружена длительная тишина, 2 — обнаружены прерывистые звуки, 3 — обнаружены аномальные перемежающиеся звуки) |
| audioSampleRate | Описание поля: локальная частота дискретизации аудио (Гц) |
| frameRate | Описание поля: локальная частота кадров видео в fps, т. е. сколько видеокадров в секунду |
| height | Описание поля: локальная высота видео в пикселях |
| streamType | Описание поля: тип видеопотока (HD большое изображение \| плавное маленькое изображение \| дополнительный видеопоток) |
| videoBitrate | Описание поля: локальный битрейт видео в Кбит/с, т. е. сколько видеоданных генерируется в секунду |
| width | Описание поля: локальная ширина видео в пикселях |

## TRTCRemoteStatistics

**TRTCRemoteStatistics**

**Метрики удаленного аудио/видео.**

| EnumType | DESC |
| --- | --- |
| audioBitrate | Описание поля: локальный битрейт аудио (Кбит/с) |
| audioBlockRate | Описание поля: доля задержки воспроизведения аудио (%) Доля задержки воспроизведения аудио (audioBlockRate) = совокупная длительность задержки воспроизведения аудио (audioTotalBlockTime) / общая длительность воспроизведения аудио |
| audioPacketLoss | Описание поля: общий процент потери пакетов (%) аудиопотока ` audioPacketLoss ` представляет процент потери пакетов, в конечном итоге рассчитанный на стороне аудитории после того, как аудио/видеопоток проходит по полной цепочке передачи «якорь -> облако -> аудитория». Чем меньше ` audioPacketLoss `, тем лучше. Процент потери пакетов 0 указывает на то, что все данные аудиопотока полностью достигли аудитории. Если ` downLoss ` равен ` 0 `, но ` audioPacketLoss ` нет, то на канале «облако -> аудитория» для аудиопотока нет потери пакетов, но есть невосстанавливаемые потери пакетов на канале «якорь -> облако». |
| audioSampleRate | Описание поля: локальная частота дискретизации аудио (Гц) |
| audioTotalBlockTime | Описание поля: совокупная длительность задержки воспроизведения аудио (мс) |
| finalLoss | Описание поля: общий процент потери пакетов (%) аудио/видеопотока Устарело, используйте вместо этого audioPacketLoss и videoPacketLoss. |
| frameRate | Описание поля: частота кадров удаленного видео (fps) |
| height | Описание поля: высота удаленного видео в пикселях |
| jitterBufferDelay | Описание поля: задержка воспроизведения (мс) Чтобы избежать задержек аудио/видео, вызванных сетевыми помехами и нарушением порядка пакетов, TRTC поддерживает буфер воспроизведения на стороне воспроизведения для организации полученных сетевых пакетов данных. Размер буфера адаптивно регулируется в соответствии с текущим качеством сети и преобразуется в длину времени в миллисекундах, т. е. ` jitterBufferDelay `. |
| point2PointDelay | Описание поля: сквозная задержка (мс) ` point2PointDelay ` представляет задержку «якорь -> облако -> аудитория». Более точно говоря, это задержка всей цепочки «сбор -> кодирование -> передача в сети -> прием -> буферизация -> декодирование -> воспроизведение». ` point2PointDelay ` работает только в том случае, если оба локальный и удаленный SDK имеют версию 8.5 или выше. Если удаленный SDK имеет версию ниже 8.5, это значение всегда будет 0 и, таким образом, будет бессмысленным. |
| remoteNetworkRTT | Описание поля: время приема-передачи в оба конца (мс) от SDK к облаку Это значение представляет общее время, необходимое для отправки сетевого пакета от SDK к облаку, а затем отправки сетевого пакета от облака обратно к SDK, т. е. общее время для сетевого пакета, чтобы пройти по цепочке «SDK -> облако -> SDK». Чем меньше значение, тем лучше. Если ` remoteNetworkRTT ` ниже 50 мс, это означает короткую задержку аудио/видео вызова; если ` remoteNetworkRTT ` выше 200 мс, это означает длительную задержку аудио/видео вызова. Следует уточнить, что ` remoteNetworkRTT ` представляет общее время на цепочке «SDK -> облако -> SDK»; поэтому нет необходимости различать ` remoteNetworkUpRTT ` и ` remoteNetworkDownRTT `. |
| remoteNetworkUplinkLoss | Описание поля: процент потери пакетов восходящего канала (%) от SDK к облаку Чем меньше значение, тем лучше. Если ` remoteNetworkUplinkLoss ` равен ` 0% `, качество сети восходящего канала очень хорошее, и загруженные в облако пакеты данных практически не теряются. Если ` remoteNetworkUplinkLoss ` равен ` 30% `, 30% пакетов аудио/видеоданных, отправляемых SDK в облако, теряются на пути передачи. |
| streamType | Описание поля: тип видеопотока (HD большое изображение \| плавное маленькое изображение \| дополнительный видеопоток) |
| userId | Описание поля: идентификатор пользователя |
| videoBitrate | Описание поля: битрейт удаленного видео (Кбит/с) |
| videoBlockRate | Описание поля: доля задержки воспроизведения видео (%) Доля задержки воспроизведения видео (videoBlockRate) = совокупная длительность задержки воспроизведения видео (videoTotalBlockTime) / общая длительность воспроизведения видео |
| videoPacketLoss | Описание поля: общий процент потери пакетов (%) видеопотока ` videoPacketLoss ` представляет процент потери пакетов, в конечном итоге рассчитанный на стороне аудитории после того, как аудио/видеопоток проходит по полной цепочке передачи «якорь -> облако -> аудитория». Чем меньше ` videoPacketLoss `, тем лучше. Процент потери пакетов 0 указывает на то, что все данные видеопотока полностью достигли аудитории. Если ` downLoss ` равен ` 0 `, но ` videoPacketLoss ` нет, то на канале «облако -> аудитория» для видеопотока нет потери пакетов, но есть невосстанавливаемые потери пакетов на канале «якорь -> облако». |
| videoTotalBlockTime | Описание поля: совокупная длительность задержки воспроизведения видео (мс) |
| width | Описание поля: ширина удаленного видео в пикселях |

## TRTCStatistics

**TRTCStatistics**

**Метрики сети и производительности.**

| EnumType | DESC |
| --- | --- |
| appCpu | Описание поля: использование CPU (%) текущего приложения, системы Android 8.0 и выше не поддерживаются |
| downLoss | Описание поля: процент потери пакетов нисходящего канала (%) от облака к SDK Чем меньше значение, тем лучше. Если ` downLoss ` равен ` 0% `, качество сети нисходящего канала очень хорошее, и полученные от облака пакеты данных практически не теряются. Если ` downLoss ` равен ` 30% `, 30% пакетов аудио/видеоданных, отправляемых облаком в SDK, теряются на пути передачи. |
| gatewayRtt | Описание поля: время приема-передачи в оба конца (мс) от SDK к шлюзу Это значение представляет общее время, необходимое для отправки сетевого пакета от SDK к шлюзу, а затем отправки сетевого пакета от шлюза обратно к SDK, т. е. общее время для сетевого пакета, чтобы пройти по цепочке «SDK -> шлюз -> SDK». Чем меньше значение, тем лучше. Если ` gatewayRtt ` ниже 50 мс, это означает короткую задержку аудио/видео вызова; если ` gatewayRtt ` выше 200 мс, это означает длительную задержку аудио/видео вызова. Следует уточнить, что ` gatewayRtt ` недействителен для мобильной сети. |
| localStatistics | Описание поля: локальная статистика аудио/видео Поскольку может быть три локальных аудио/видеопотока (т. е. HD большое изображение, плавное маленькое изображение и дополнительный видеопоток), локальная статистика аудио/видео представляет собой массив. |
| receivedBytes | Описание поля: общее количество полученных байтов (включая данные сигнализации и аудио/видеоданные) |
| remoteStatistics | Описание поля: статистика удаленного аудио/видео Поскольку может быть несколько одновременно подключенных удаленных пользователей, и каждый из них может иметь несколько одновременно активных аудио/видеопотоков (т. е. HD большое изображение, плавное маленькое изображение и дополнительный видеопоток), удаленная статистика аудио/видео представляет собой массив. |
| rtt | Описание поля: время приема-передачи в оба конца (мс) от SDK к облаку Это значение представляет общее время, необходимое для отправки сетевого пакета от SDK к облаку, а затем отправки сетевого пакета от облака обратно к SDK, т. е. общее время для сетевого пакета, чтобы пройти по цепочке «SDK -> облако -> SDK». Чем меньше значение, тем лучше. Если ` rtt ` ниже 50 мс, это означает короткую задержку аудио/видео вызова; если ` rtt ` выше 200 мс, это означает длительную задержку аудио/видео вызова. Следует уточнить, что ` rtt ` представляет общее время на цепочке «SDK -> облако -> SDK»; поэтому нет необходимости различать ` upRtt ` и ` downRtt `. |
| sentBytes | Описание поля: общее количество отправленных байтов (включая данные сигнализации и аудио/видеоданные) |
| systemCpu | Описание поля: использование CPU (%) текущей системы, системы Android 8.0 и выше не поддерживаются |
| upLoss | Описание поля: процент потери пакетов восходящего канала (%) от SDK к облаку Чем меньше значение, тем лучше. Если ` upLoss ` равен ` 0% `, качество сети восходящего канала очень хорошее, и загруженные в облако пакеты данных практически не теряются. Если ` upLoss ` равен ` 30% `, 30% пакетов аудио/видеоданных, отправляемых SDK в облако, теряются на пути передачи. |

## TRTCVideoStreamType

**TRTCVideoStreamType**

**Тип видеопотока.**

TRTC предоставляет три различных видеопотока:

- Высокое качество большого изображения: обычно используется для передачи видеоданных с камеры.
- Плавное малое изображение: имеет то же содержимое, что и большое изображение, но с более низким разрешением и битрейтом, следовательно, с более низким качеством.
- Доп. поток изображения: обычно используется для совместного использования экрана. Только один пользователь в комнате может одновременно публиковать видеоизображение доп. потока, остальные пользователи должны дождаться, пока этот пользователь закроет доп. поток, прежде чем они смогут публиковать свой собственный доп. поток.

> **Примечание** SDK не поддерживает включение плавного малого изображения отдельно, оно должно быть включено вместе с большим изображением. Разрешение и битрейт малого изображения будут установлены автоматически.

| Enum | Value | DESC |
| --- | --- | --- |
| TRTCVideoStreamTypeBig | 0 | Высокое качество большого изображения: обычно используется для передачи видеоданных с камеры. |
| TRTCVideoStreamTypeSmall | 1 | Плавное малое изображение: имеет то же содержимое, что и большое изображение, но с более низким разрешением и битрейтом, следовательно, с более низким качеством. |
| TRTCVideoStreamTypeSub | 2 | Доп. поток изображения: обычно используется для совместного использования экрана. Только один пользователь в комнате может одновременно публиковать видеоизображение доп. потока, остальные пользователи должны дождаться, пока этот пользователь закроет доп. поток, прежде чем они смогут публиковать свой собственный доп. поток. |

## TRTCVideoFillMode

**TRTCVideoFillMode**

**Режим заполнения видеоизображения.**

Если соотношение сторон области отображения видео не равно соотношению сторон видеоизображения, необходимо указать режим заполнения:

| Enum | Value | DESC |
| --- | --- | --- |
| TRTCVideoFillMode_Fill | 0 | Режим заполнения: видеоизображение будет отцентрировано и масштабировано для заполнения всей области отображения, при этом части, выходящие за границы области, будут обрезаны. В этом режиме отображаемое изображение может быть неполным. |
| TRTCVideoFillMode_Fit | 1 | Режим соответствия: видеоизображение будет масштабировано по его длинной стороне для соответствия области отображения, при этом короткая сторона будет заполнена черными полосами. В этом режиме отображаемое изображение полное, но могут быть черные полосы. |
| TRTCVideoFillMode_ScaleFill | 2 | Режим масштабирования до заполнения: независимо от соотношения сторон изображения оно будет растянуто или сжато для полного заполнения области отображения. В этом режиме соотношение сторон изображения может быть изменено, что приведет к искажению отрисованного изображения. |

TRTC предоставляет API-интерфейсы установки угла поворота для локальных и удаленных изображений. Все углы поворота по часовой стрелке.

| TRTCVideoRotation_180 | 2 | Поворот по часовой стрелке на 180 градусов |
| TRTCVideoRotation_270 | 3 | Поворот по часовой стрелке на 270 градусов |

## TRTCAudioRoute

**TRTCAudioRoute**

**Маршрут звука (т.е. режим воспроизведения звука).**

"Маршрут звука" определяет, воспроизводится ли звук из динамика или приемника мобильного устройства; поэтому этот API применим только к мобильным устройствам, таким как телефоны.

## TRTCSystemVolumeType

**TRTCSystemVolumeType**

**Тип системной громкости (только для мобильных устройств).**

Смартфоны обычно имеют два типа системной громкости: громкость вызова и громкость медиа.

- Громкость вызова разработана для сценариев вызовов. Она поставляется с подавлением акустического эха (AEC) и поддерживает захват звука беспроводными наушниками, но качество звука среднее.

Если вы не можете снизить громкость до 0 (т.е. отключить звук телефона) с помощью кнопок громкости, то ваш телефон использует громкость вызова.

- Громкость медиа разработана для сценариев медиа, таких как воспроизведение музыки. AEC не работает при использовании громкости медиа, и беспроводные наушники не могут использоваться для захвата звука. Однако громкость медиа обеспечивает лучший опыт прослушивания музыки.

Если вы можете отключить звук телефона с помощью кнопок громкости, то ваш телефон использует громкость медиа.

SDK предлагает три режима управления системной громкостью: автоматический, громкость вызова и громкость медиа.

| Enum | Value | DESC |
| --- | --- | --- |
| TRTCSystemVolumeTypeAuto | 0 | Автоматический: в автоматическом режиме для ведущих используется громкость вызова, а для аудитории — громкость медиа. Этот режим подходит для сценариев прямой трансляции. Если сценарий, выбранный во время `enterRoom`, является `TRTCAppSceneLIVE` или `TRTCAppSceneVoiceChatRoom`, SDK автоматически будет использовать этот режим. |
| TRTCSystemVolumeTypeMedia | 1 | Громкость медиа: в этом режиме громкость медиа используется во всех сценариях. Это редко используется, в основном подходит для музыкальных сценариев с высокими требованиями к качеству звука. Используйте этот режим, если большинство ваших пользователей используют периферийные устройства, такие как звуковые карты. В противном случае это не рекомендуется. |
| TRTCSystemVolumeTypeVOIP | 2 | Громкость вызова: в этом режиме аудиомодуль не изменяет свой режим работы, когда пользователи переключаются между ведущими и аудиторией, обеспечивая беспроблемное включение/отключение микрофона. Этот режим подходит для сценариев, где пользователям необходимо часто переключаться между ведущими и аудиторией. Если сценарий, выбранный во время `enterRoom`, является `TRTCAppSceneVideoCall` или `TRTCAppSceneAudioCall`, SDK автоматически будет использовать этот режим. |

## TRTCTranscodingConfigMode

**TRTCTranscodingConfigMode**

**Режим компоновки облачного смешивания и транскодирования.**

Служба облачного смешивания и транскодирования TRTC может смешивать несколько аудио/видеопотоков в комнате в один поток. Поэтому необходимо указать схему компоновки видеоизображений. Предусмотрены следующие режимы компоновки:

| Enum | Value | DESC |
| --- | --- | --- |
| TRTCTranscodingConfigMode_Unknown | 0 | Не определено |
| TRTCTranscodingConfigMode_Manual | 1 | Режим ручной компоновки. В этом режиме необходимо указать точное положение каждого видеоизображения. Этот режим имеет наибольшую степень свободы, но его удобство использования наихудшее: необходимо ввести все параметры в [TRTCTranscodingConfig](https://www.tencentcloud.com/document/product/647/50760#c3eef0088da62b5dd27fb0c08db18906), включая координаты положения каждого видеоизображения (TRTCMixUser). Необходимо слушать обратные вызовы событий [onUserVideoAvailable](https://www.tencentcloud.com/document/product/647/50755#36daf607f51a906ea0b48b33fc628161) и [onUserAudioAvailable](https://www.tencentcloud.com/document/product/647/50755#e9535d2e80eb01b4d671dcbd7dfa8c8f) в [TRTCCloudDelegate](https://www.tencentcloud.com/document/product/647/50755#d8f7bddf1dbd4490d7801cb74808ed04) и постоянно корректировать параметр `mixUsers` в соответствии со статусом аудио/видео каждого пользователя с включенным микрофоном в текущей комнате. |
| TRTCTranscodingConfigMode_Template_PureAudio | 2 | Режим чистого звука. Этот режим подходит для чисто аудиосценариев, таких как звуковой вызов (AudioCall) и комната для голосового чата (VoiceChatRoom). Необходимо установить его один раз через API setMixTranscodingConfig после входа в комнату, и затем SDK автоматически смешает звук всех пользователей с включенным микрофоном в комнате в прямой поток текущего пользователя. Не требуется устанавливать параметр `mixUsers` в [TRTCTranscodingConfig](https://www.tencentcloud.com/document/product/647/50760#c3eef0088da62b5dd27fb0c08db18906); вместо этого требуется установить только параметры `audioSampleRate`, `audioBitrate` и `audioChannels`. |
| TRTCTranscodingConfigMode_Template_PresetLayout | 3 | Режим предустановленной компоновки. Это наиболее популярный режим компоновки, поскольку он позволяет предварительно установить положение каждого видеоизображения через заполнители, и затем SDK автоматически динамически адаптирует его в соответствии с количеством видеоизображений в комнате. В этом режиме необходимо установить параметр `mixUsers`, но можно установить `userId` как "заполнитель". Значения заполнителей включают: "$PLACE_HOLDER_REMOTE$": видеоизображение удаленного пользователя. Можно установить несколько изображений. "$PLACE_HOLDER_LOCAL_MAIN$": видеоизображение локальной камеры. Можно установить только одно изображение. "$PLACE_HOLDER_LOCAL_SUB$": видеоизображение локального общего доступа к экрану. Можно установить только одно изображение. В этом режиме не требуется слушать обратные вызовы [onUserVideoAvailable](https://www.tencentcloud.com/document/product/647/50755#36daf607f51a906ea0b48b33fc628161) и [onUserAudioAvailable](https://www.tencentcloud.com/document/product/647/50755#e9535d2e80eb01b4d671dcbd7dfa8c8f) в [TRTCCloudDelegate](https://www.tencentcloud.com/document/product/647/50755#d8f7bddf1dbd4490d7801cb74808ed04) для внесения поправок в режиме реального времени. Вместо этого требуется вызвать setMixTranscodingConfig один раз после успешного входа в комнату. Затем SDK автоматически заполнит установленные вами заполнители реальными значениями `userId`. |
| TRTCTranscodingConfigMode_Template_ScreenSharing | 4 | Режим совместного доступа к экрану. Этот режим подходит для вариантов использования на основе совместного доступа к экрану, таких как онлайн-образование, и поддерживается только SDK для Windows и macOS. В этом режиме SDK сначала построит холст в соответствии с целевым разрешением, которое вы устанавливаете (через параметры `videoWidth` и `videoHeight`). До того как учитель включит совместный доступ к экрану, SDK увеличит видеоизображение учителя и нарисует его на холсте. После того как учитель включит совместный доступ к экрану, SDK нарисует видеоизображение, совместно используемое на экране, на том же холсте. Цель этого режима компоновки — обеспечить согласованность выходного разрешения модуля смешивания транскодирования и избежать проблем с размытым экраном при воспроизведении записи курса и воспроизведении веб-страницы (веб-плееры не поддерживают регулируемое разрешение). Между тем звук студентов с включенным микрофоном по умолчанию будет смешан с аудио/видеопотоком учителя. Видеоконтент в режиме обучения в основном представляет собой совместно используемый экран, и передача изображения камеры и изображения экрана одновременно является пустой тратой полосы пропускания. Поэтому рекомендуемой практикой является прямое рисование видеоизображения камеры на текущем экране через API setLocalVideoRenderCallback. В этом режиме не требуется устанавливать параметр `mixUsers` в [TRTCTranscodingConfig](https://www.tencentcloud.com/document/product/647/50760#c3eef0088da62b5dd27fb0c08db18906), и SDK не будет смешивать видеоизображения студентов, чтобы не помешать эффекту совместного доступа к экрану. Можно установить ширину x высоту в [TRTCTranscodingConfig](https://www.tencentcloud.com/document/product/647/50760#c3eef0088da62b5dd27fb0c08db18906) на 0 px x 0 px, и SDK автоматически рассчитает подходящее разрешение на основе соотношения сторон текущего экрана пользователя. Если текущая ширина экрана учителя меньше или равна 1920 px, SDK будет использовать фактическое разрешение текущего экрана учителя. Если текущая ширина экрана учителя больше 1920 px, SDK выберет одно из трех разрешений 1920x1080 (16:9), 1920x1200 (16:10) и 1920x1440 (4:3) в соответствии с текущим соотношением сторон экрана. |

` Примечание: ` Этот параметр требуется, если вы установили режим публикации на ` TRTCPublish_MixStream_ToCdn ` или ` TRTCPublish_MixStream_ToRoom ` в [TRTCPublishTarget](https://www.tencentcloud.com/document/product/647/50760#11c06c485af4d4bd3b60bc0c883a9a32).

` Примечание: ` Если вы используете функцию релеирования на CDN (режим публикации установлен на ` RTCPublish_BigStream_ToCdn ` или ` TRTCPublish_SubStream_ToCdn `), чтобы улучшить стабильность релеирования и совместимость воспроизведения, мы также рекомендуем вам установить этот параметр.

| EnumType | DESC |
| --- | --- |
| audioEncodedChannelNum | ` Описание: ` Звуковые каналы потока для публикации.` Значение: ` Допустимые значения: 1 (моно); 2 (двойной канал). По умолчанию: 1. |
| audioEncodedCodecType | ` Описание: ` Аудиокодек потока для публикации.` Значение: ` Допустимые значения: 0 (LC-AAC); 1 (HE-AAC); 2 (HE-AACv2). По умолчанию: 0.**Примечание** Частоты дискретизации звука, поддерживаемые HE-AAC и HE-AACv2: 48000, 44100, 32000, 24000 и 16000. При использовании HE-AACv2 выходной поток может быть только двойной канал. |
| audioEncodedKbps | ` Описание: ` Аудиобитрейт (Kbps) потока для публикации.` Значение: ` Диапазон значений: [32,192]. По умолчанию: 50. |
| audioEncodedSampleRate | ` Описание: ` Частота дискретизации звука потока для публикации.` Значение: ` Допустимые значения: [48000, 44100, 32000, 24000, 16000, 8000]. По умолчанию: 48000 (Гц). |
| videoEncodedCodecType | ` Описание: ` Видеокодек потока для публикации.` Значение: ` Допустимые значения: 0 (H264); 1 (H265). По умолчанию: 0. |
| videoEncodedFPS | ` Описание: ` Частота кадров (fps) потока для публикации.` Значение: ` Диапазон значений: (0,30]. По умолчанию: 20. |
| videoEncodedGOP | ` Описание: ` Интервал ключевого кадра (GOP) потока для публикации.` Значение: ` Диапазон значений: [1,5]. По умолчанию: 3 (секунды). |
| videoEncodedHeight | ` Описание: ` Разрешение (высота) потока для публикации.` Значение: ` Рекомендуемое значение: 640. Если вы смешиваете только аудиопотоки, чтобы избежать отображения черного видео в транскодированном потоке, установите как ` width `, так и ` height ` в ` 0 `. |
| videoEncodedKbps | ` Описание: ` Видеобитрейт (Kbps) потока для публикации.` Значение: ` Если вы установите этот параметр в ` 0 `, TRTC рассчитает битрейт на основе ` videoWidth ` и ` videoHeight `. Для деталей см. рекомендуемые битрейты констант типа enum разрешения (см. комментарий). |
| videoEncodedWidth | ` Описание: ` Разрешение (ширина) потока для публикации.` Значение: ` Рекомендуемое значение: 368. Если вы смешиваете только аудиопотоки, чтобы избежать отображения черного видео в транскодированном потоке, установите как ` width `, так и ` height ` в ` 0 `. |
| videoSeiParams | ` Описание: ` Параметры SEI. По умолчанию: null` Примечание: ` параметр передается в виде JSON-строки. Вот пример его использования:{  "payLoadContent":"xxx",  "payloadType":5,  "payloadUuid":"1234567890abcdef1234567890abcdef",  "interval":1000,  "followIdr":false}Поддерживаемые поля и их значения следующие: payloadContent: Обязательно. Содержимое полезной нагрузки SEI для транспортировки, которая не может быть пустой. payloadType: Обязательно. Тип сообщения SEI, с диапазоном значений 5 или целое число в диапазоне [100, 254] (исключая 244, которое является внутренним временным меткой SEI). payloadUuid: Обязательно при payloadType равно 5, игнорируется в других случаях. Значение должно быть 32-значным шестнадцатеричным числом. interval: Опционально, по умолчанию 1000. Интервал отправки SEI в миллисекундах. followIdr: Опционально, по умолчанию false. Когда это значение true, SEI будет гарантировано отправлено при отправке ключевого кадра, иначе это не гарантируется. |

## TRTCStreamMixingConfig

**TRTCStreamMixingConfig**

**Параметры транскодирования.**

Этот тип enum используется API публикации [startPublishMediaStream](https://www.tencentcloud.com/document/product/647/50754#9cea9ae34a50a44c0a7023295313bf2e).

Вы можете использовать этот параметр для указания видеомакета и информации о входящем звуке для On-Cloud MixTranscoding.

| EnumType | DESC |
| --- | --- |
| audioMixUserList | ` Описание: ` Информация о каждом аудиопотоке для смешивания.` Значение: ` Этот параметр является массивом. Каждый элемент ` TRTCUser ` в массиве указывает информацию об аудиопотоке.**Примечание**Если вы не указываете этот массив, серверная часть TRTC будет автоматически смешивать все потоки якорей, отправляющих аудио в комнате, в соответствии с параметром кодирования звука [TRTCStreamEncoderParam](https://www.tencentcloud.com/document/product/647/50760#22718fe81d94d21ec895cbc11820c726), который вы указываете (в настоящее время поддерживает до 16 входов аудио и видео). |
| backgroundColor | ` Описание: ` Цвет фона смешанного потока.` Значение: ` Значение должно быть шестнадцатеричным числом. Например, "0x61B9F1" представляет значение цвета RGB (97,158,241). Значение по умолчанию: 0x000000 (черный). |
| backgroundImage | ` Описание: ` URL-адрес фонового изображения смешанного потока. Изображение, указанное URL-адресом, будет смешано во время On-Cloud MixTranscoding.` Значение: ` Этот параметр по умолчанию оставляется пустым, что означает, что никакое фоновое изображение не будет использоваться.**Примечание** URL-адрес может быть длиной не более 512 байт, а изображение не должно превышать 2 МБ. Изображение может быть в формате PNG, JPG, JPEG или BMP. Рекомендуем использовать полупрозрачное изображение в формате PNG. |
| videoLayoutList | ` Описание: ` Позиция, размер, слой и тип потока каждого видео в On-Cloud MixTranscoding.` Значение: ` Этот параметр является массивом. Каждый элемент ` TRTCVideoLayout ` в массиве указывает информацию о видео в On-Cloud MixTranscoding. |
| watermarkList | ` Описание: ` Позиция, размер и слой каждого изображения водяного знака в On-Cloud MixTranscoding.` Значение: ` Этот параметр является массивом. Каждый элемент ` TRTCWatermark ` в массиве указывает информацию о водяном знаке. |

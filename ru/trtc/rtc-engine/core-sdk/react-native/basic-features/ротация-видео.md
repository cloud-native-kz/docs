# Ротация видео

## Введение

В отличие от единообразного вертикального интерфейса мобильной прямой трансляции, Tencent Real-Time Communication (TRTC) должна учитывать как альбомную, так и портретную ориентацию. Поэтому существует большой объем логики обработки для работы с альбомной/портретной ориентацией. Этот документ описывает:

- как достичь портретного режима. Например, видеозвонок в WeChat — это типичный пример портретного режима.
- Как достичь альбомного режима. Например, приложения для многопользовательских аудио-видео комнат (аналогично Xiaoyu Yilian) часто используют альбомный режим.
- Как настроить направление вращения и режим заполнения локального экрана и удалённых видеоизображений.

## Портретный режим

Для достижения режима, аналогичного видеозвонку WeChat, требуется выполнить две задачи:

### Шаг 1: Настройте интерфейс приложения в портретном режиме.

Платформа iOS

Платформа Android

Вы можете напрямую установить в XCode **General** > **Deployment Info** > **Device Orientation**:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/c85c94d8be5d11ee8c36525400bb593a.png)

Укажите атрибут `screenOrientation` действия (activity) как портрет для установки интерфейса в портретный режим.

```
<activity android:name=".trtc.TRTCMainActivity"  android:launchMode="singleTask" android:windowSoftInputMode="adjustPan"          android:screenOrientation="portrait" />
```

### Шаг 2: Настройте SDK для использования портретного разрешения.

При использовании `setVideoEncoderParam` TRTCCloud для установки параметров видеокодирования просто укажите `videoResolutionMode` как `TRTC_VIDEO_RESOLUTION_MODE_PORTRAIT`.

Ниже приведен пример кода:

```
const trtcCloud = TRTCCloud.sharedInstance();trtcCloud.setVideoEncoderParam({  videoFps: settings.fps,  videoResolution: settings.resolution,  minVideoBitrate: 0,  enableAdjustRes: false,  videoResolutionMode: TRTCCloudDef.TRTC_VIDEO_RESOLUTION_MODE_PORTRAIT,  videoBitrate,});
```

## Альбомный режим

Если вы хотите получить альбомную ориентацию для приложения, работа, которую вам нужно выполнить, аналогична портретному режиму. Вам просто нужно внести коррективы в параметры на шаге 1 и шаге 2.

В частности, на [Шаге 2](https://www.tencentcloud.com/document/product/647/69706#2.-.E9.85.8D.E7.BD.AE-SDK-.E4.BD.BF.E7.94.A8.E7.AB.96.E5.B1.8F.E5.88.86.E8.BE.A8.E7.8E.87) значение `videoResolutionMode` в TRTCVideoEncParam: `TRTC_VIDEO_RESOLUTION_MODE_LANDSCAPE`.

## Пользовательские настройки

Сам TRTC SDK предоставляет функции API для управления направлением вращения и режимом заполнения локальных и удалённых видеоизображений.

| Функция API | Функциональная роль | Примечания |
| --- | --- | --- |
| setVideoEncoderRotation | Установить угол поворота по часовой стрелке для выходного кадра кодировщика. | В настоящее время поддерживаются только 0 градусов и 180 градусов. |

## GSensorMode

Учитывая, что ротация экрана включает различные проблемы совместимости с записью и CDN обходом прямой трансляции, TRTC SDK предоставляет только простую функцию адаптации гравитационного датчика. Вы можете включить её через API `setGSensorMode` TRTCCloud.

Эта функция поддерживает адаптивный поворот на 90 градусов, 180 градусов и 270 градусов. То есть, когда мобильный телефон пользователя вращается, ориентация видео, видимого для другой стороны, остаётся неизменной. Более того, эта адаптация достигается на основе ориентационной корректировки кодировщика. Поэтому записанное видео, а также видеоматериалы, видимые на конце мини-программы и H5, также могут сохранить исходную ориентацию неизменной.

> **Примечания:** Другая схема реализации адаптации гравитационного датчика — это передача направления гравитации текущего видео в информации видео каждого кадра, а затем адаптивная настройка направления отрисовки на стороне удалённого пользователя. Однако эта схема требует введения дополнительных ресурсов транскодирования для решения проблемы сохранения ориентации записанного видео в соответствии с ожидаемой ориентацией видео. Поэтому это не рекомендуется.

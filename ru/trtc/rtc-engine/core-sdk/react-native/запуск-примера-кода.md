# Запуск примера кода

Этот документ содержит пошаговое руководство по началу работы с TRTC React Native SDK через

[наш пример проекта.](https://github.com/Tencent-RTC/TRTC_ReactNative)

## Предварительные требования

- React Native 0.63 или позже
- Node (версия позже v12) и Watchman
- **Если вы нацеливаетесь на платформу Android:**
  - Android Studio 3.5 или позже
  - Устройства с Android 4.1 или позже
- **Если вы нацеливаетесь на платформы iOS и macOS:**
  - Xcode 11.0 или позже
  - OS X 10.11 или позже
  - Действительная подпись разработчика для вашего проекта
- Инструкции по настройке окружения см. в [официальной документации](https://reactnative.dev/docs/environment-setup) React Native.

## Настройка аккаунта

Вы [зарегистрировали учетную запись Tencent Cloud](https://trtc.io/register?s_url=https://console.trtc.io).

## Инструкции

### Шаг 1. Создайте приложение в консоли TRTC

1. Войдите в [консоль TRTC](https://console.trtc.io/), нажмите **Create Application** в разделе **Applications**.
2. На открывшейся странице введите имя приложения, выберите **RTC Engine** и нажмите **Create**.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/6b59d04b19f311f08ac55254007c27c5.png)

> **Примечание:** Имя приложения может содержать до 15 символов. Допускаются только цифры, буквы, китайские символы и подчеркивания. Теги используются для идентификации и организации ресурсов Tencent Cloud. Например, компания может иметь несколько бизнес-подразделений, каждое из которых имеет одно или несколько приложений TRTC. В этом случае компания может использовать теги для приложений TRTC, чтобы отметить информацию подразделения. Теги являются необязательными и могут быть добавлены или отредактированы в соответствии с вашими потребностями.

### Шаг 2. Получите `SDKAppId` и `SecretKey` вашего приложения

Выберите только что созданное приложение и получите его `SDKAppID` и `SDKSecretKey` в разделе **Basic Information**. Эта информация будет использована позже при [конфигурации проекта](https://www.tencentcloud.com/document/product/647/59412#step-3.-configure-the-project).

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/a67ef93819cd11f0b5c65254001c06ec.png)

### Шаг 3. Загрузите проект

1. Загрузите наш SDK и код демонстрации с [GitHub](https://github.com/LiteAVSDK/TRTC_ReactNative/tree/main/TRTC-Simple-Demo), или выполните следующую команду для клонирования репозитория.

```
git clone https://github.com/LiteAVSDK/TRTC_ReactNative.git
```

2. Перейдите в каталог `TRTC_ReactNative/TRTC-Simple-Demo` и выполните следующие команды для установки зависимостей.

```
yarn install
```

### Шаг 4. Конфигурация проекта

Откройте каталог `/TRTC-Simple-Demo/debug/config.js` и заполните следующие параметры:

- `SDKAPPID`: `SDKAppID` вашего приложения из шага 2.
- `SDKSECRETKEY`: `SDKSecretKey` вашего приложения из шага 2.

> **Примечание:** Метод генерации `UserSig`, описанный в этом документе, включает конфигурацию `SDKSECRETKEY` в коде клиента. При таком методе `SDKSECRETKEY` может быть легко декомпилирован и обращен, и если ваш ключ будет раскрыт, злоумышленники могут украсть ваш трафик Tencent Cloud. Поэтому **этот метод подходит только для локального выполнения и отладки TRTC-Simple-Demo**. Лучшей практикой является интеграция кода расчета `UserSig` на ваш сервер и предоставление API, ориентированного на приложение. Когда требуется `UserSig`, ваше приложение может отправить запрос на ваш сервер для получения динамического `UserSig`. Для получения дополнительной информации см. [Как рассчитать UserSig в production?](https://www.tencentcloud.com/zh/document/product/1047/34385)

### Шаг 5. Конфигурация запросов прав доступа

Перед построением и запуском проекта необходимо сконфигурировать следующие запросы прав доступа:

Android

iOS

1. Добавьте следующие разрешения в `AndroidManifest.xml`:

```
<uses-permission android:name="android.permission.INTERNET" /><uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" /><uses-permission android:name="android.permission.ACCESS_WIFI_STATE" /><uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" /><uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" /><uses-permission android:name="android.permission.RECORD_AUDIO" /><uses-permission android:name="android.permission.MODIFY_AUDIO_SETTINGS" /><uses-permission android:name="android.permission.BLUETOOTH" /><uses-permission android:name="android.permission.CAMERA" /><uses-permission android:name="android.permission.READ_PHONE_STATE" /><uses-feature android:name="android.hardware.camera" /><uses-feature android:name="android.hardware.camera.autofocus" />
```

> **Примечание:** Отключение аппаратного ускорения приведет к ошибке при рендеринге удаленных видео. Убедитесь, что **аппаратное ускорение не отключено** (не используйте `android:hardwareAccelerated="false"`).

2. На платформе Android разрешения на аудио и видео должны запрашиваться вручную, например:

```
if (Platform.OS === 'android') {  await PermissionsAndroid.requestMultiple([    PermissionsAndroid.PERMISSIONS.RECORD_AUDIO, //For audio calls    PermissionsAndroid.PERMISSIONS.CAMERA, // For video calls  ]);}
```

Добавьте следующие разрешения в `Info.plist`:

```
<key>NSCameraUsageDescription</key><string>You can make video calls only if you grant the app camera permission.</string><key>NSMicrophoneUsageDescription</key><string>You can make audio calls only if you grant the app mic permission.</string>
```

### Шаг 6. Построение и запуск проекта

Запустите Metro в каталоге демонстрации.

Android

iOS

1. Запустите Metro в каталоге демонстрации.

```
npx react-native run-android
```

2. Откройте новое окно в каталоге демонстрации и начните отладку.

```
npx react-native run-android
```

1. Запустите `pod install` в каталоге iOS демонстрации для установки зависимостей.
2. Запустите Metro в каталоге демонстрации.

```
npx react-native start
```

3. Откройте новое окно в каталоге демонстрации и начните отладку. Если произойдет ошибка, используйте Xcode для отладки вашего проекта.

```
npx react-native run-ios
```

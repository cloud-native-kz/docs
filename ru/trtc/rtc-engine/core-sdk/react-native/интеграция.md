# Интеграция

## Подготовка окружения

1. React Native 0.60+
2. Android 5.0+
3. iOS 12.0+
4. Node.js 14+

Дополнительные рекомендации по настройке среды разработки см. в разделе [Настройка среды разработки](https://reactnative.dev/docs/environment-setup)

## Шаг 1. Импорт SDK

Установите SDK через npm или yarn:

```
npm install trtc-react-native --save# или yarn add trtc-react-native
```

## Шаг 2. Конфигурация проекта

Android

iOS

1. Откройте файл `/android/app/src/main/AndroidManifest.xml` и добавьте следующие разрешения:

```
<uses-permission android:name="android.permission.INTERNET" /><uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" /><uses-permission android:name="android.permission.ACCESS_WIFI_STATE" /><uses-permission android:name="android.permission.RECORD_AUDIO" /><uses-permission android:name="android.permission.MODIFY_AUDIO_SETTINGS" /><uses-permission android:name="android.permission.BLUETOOTH" /><uses-permission android:name="android.permission.CAMERA" /><uses-feature android:name="android.hardware.camera.autofocus" />
```

2. Создайте файл `proguard-rules.pro` в директории `android/app` вашего проекта и добавьте в него следующий код:

```
-keep class com.tencent.** { *; }
```

1. iOS требует добавления приложений для разрешений камеры и микрофона в директорию первого уровня `<dict>` в файле `Info.plist`:

```
<key>NSCameraUsageDescription</key><string>Authorize camera permissions for a normal video cal</string><key>NSMicrophoneUsageDescription</key><string>Authorize microphone permissions for a normal voice call</string>
```

2. Выполните `pod install` для установки нативных зависимостей (необходимо выполнить в директории `ios`).

> **Примечание:** Если во время процесса интеграции возникают проблемы, см. [пример проекта](https://github.com/Tencent-RTC/TRTC_ReactNative/tree/main/TRTC-Simple-Demo).

## Шаг 3. Создание экземпляра TRTC

```
import TRTCCloud from 'trtc-react-native';// Create a TRTC singleton instance.const trtcCloud = TRTCCloud.sharedInstance();
```

## Шаг 4. Вход в комнату

```
import { TRTCParams, TRTCCloudDef } from 'trtc-react-native';const params = new TRTCParams({  sdkAppId: SDKAPPID,  // SDKAppID applied in the Tencent Cloud console  userId: 'user123',   // user id  userSig: 'xxxxxx',   // user signature (generated on the server side)  roomId: 2366,        // room numb});// Enter the room in audio and video call scenariostrtcCloud.enterRoom(params, TRTCCloudDef.TRTC_APP_SCENE_VIDEOCALL);
```

> **Примечание:** Вы можете получить sdkAppId вашего приложения в [консоли TRTC](https://console.trtc.io/), а также см. раздел [аутентификация пользователя](https://www.tencentcloud.com/document/product/647/35166?lang=en) для получения userSig.

## Шаг 5. Включение микрофона

```
trtcCloud.startLocalAudio(); // Start local audio capture and uplink
```

## Шаг 6. Воспроизведение/остановка видеопотока

**Воспроизведение локального видео**:

```
import { TXVideoView } from 'trtc-react-native';// Render local video in the component<TXVideoView.LocalView style={{ width: 1080, height: 1080 }} />
```

**Воспроизведение удаленного видео**:

```
<TXVideoView.RemoteView   userId={remoteUserId}   streamType={TRTCCloudDef.TRTC_VIDEO_STREAM_TYPE_BIG}  style={{ width: 1080, height: 1080 }}/>
```

**Остановка воспроизведения**:

```
trtcCloud.stopRemoteView(userId, TRTCCloudDef.TRTC_VIDEO_STREAM_TYPE_BIG);
```

## Шаг 7. Воспроизведение/остановка аудиопотока

**Воспроизведение удаленного аудио**:

```
trtcCloud.muteRemoteAudio(userId, false); // `false` means unmute
```

**Остановка воспроизведения**:

```
trtcCloud.muteRemoteAudio(userId, true);  // `true` means mute
```

## Шаг 8. Выход из комнаты

```
trtcCloud.exitRoom(); // Exit the current room
```

## Часто задаваемые вопросы

### **Как решить проблему чёрного экрана?**

• Убедитесь, что разрешение на доступ к камере авторизировано.

• Убедитесь, что компонент `TXVideoView` отображается корректно и его размер не равен нулю.

### **Как решить проблему невозможности входа в комнату?**

• Проверьте, что `sdkAppId`, `userSig` и `roomId` действительны.

• Убедитесь в нормальном подключении к сети.

### Возникла аномалия аудио. Как её решить?

• Проверьте, что разрешение на доступ к микрофону авторизировано.

• Подтвердите статус микрофона после вызова `startLocalAudio()`.

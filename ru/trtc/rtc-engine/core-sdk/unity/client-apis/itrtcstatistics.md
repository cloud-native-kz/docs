# ITRTCStatistics

Copyright (c) 2021 Tencent. All rights reserved.

Модуль: метрики аудио/видео TRTC (только для чтения)

Функция: SDK TRTC сообщает вам текущие метрики аудио/видео в режиме реального времени (частота кадров, битрейт, задержка и т. д.) один раз в две секунды

| [TRTCRemoteStatistics](https://www.tencentcloud.com/document/product/647/72272#31083ca966a8ee092affeab44b65fe9a) | Метрики удаленного аудио/видео. |

## TRTCLocalStatistics

**TRTCLocalStatistics**

#### Метрики локального аудио/видео.

| EnumType | DESC |
| --- | --- |
| audioBitrate | Описание поля: локальный битрейт аудио в Кбит/с, т. е. объем аудиоданных, генерируемых в секунду |
| audioCaptureState | Описание поля: статус сбора оборудованием аудио (0 — нормально, 1 — обнаружена длительная тишина, 2 — обнаружены помехи, 3 — обнаружены прерывания звука) |
| audioSampleRate | Описание поля: локальная частота дискретизации аудио (Гц) |
| frameRate | Описание поля: локальная частота кадров видео в кадр/с, т. е. количество видеокадров в секунду |
| height | Описание поля: локальная высота видео в пикс. |
| streamType | Описание поля: тип видеопотока (HD-видео большого размера \| сглаженное видео малого размера \| побочный поток) |
| videoBitrate | Описание поля: локальный битрейт видео в Кбит/с, т. е. объем видеоданных, генерируемых в секунду |
| width | Описание поля: локальная ширина видео в пикс. |

## TRTCRemoteStatistics

**TRTCRemoteStatistics**

#### Метрики удаленного аудио/видео.

| EnumType | DESC |
| --- | --- |
| audioBitrate | Описание поля: локальный битрейт аудио (Кбит/с) |
| audioBlockRate | Описание поля: коэффициент задержки воспроизведения аудио (%) Коэффициент задержки воспроизведения аудио (audioBlockRate) = кумулятивная продолжительность задержки воспроизведения аудио (audioTotalBlockTime) / общая продолжительность воспроизведения аудио |
| audioPacketLoss | Описание поля: общий коэффициент потери пакетов (%) аудиопотока ` audioPacketLoss ` представляет коэффициент потери пакетов, в конечном итоге рассчитанный на стороне аудитории после прохождения аудио/видеопотока по полной цепи передачи «якорь -> облако -> аудитория». Чем меньше ` audioPacketLoss `, тем лучше. Коэффициент потери пакетов 0 указывает на то, что все данные аудиопотока полностью достигли аудитории. Если ` downLoss ` равен ` 0 `, но ` audioPacketLoss ` нет, на цепи «облако -> аудитория» для аудиопотока нет потери пакетов, но на цепи «якорь -> облако» есть безвозвратные потери пакетов. |
| audioSampleRate | Описание поля: локальная частота дискретизации аудио (Гц) |
| audioTotalBlockTime | Описание поля: кумулятивная продолжительность задержки воспроизведения аудио (мс) |
| finalLoss | Описание поля: общий коэффициент потери пакетов (%) аудио/видеопотока Устарело, вместо этого используйте audioPacketLoss и videoPacketLoss. |
| frameRate | Описание поля: частота кадров удаленного видео (кадр/с) |
| height | Описание поля: высота удаленного видео в пикс. |
| jitterBufferDelay | Описание поля: задержка воспроизведения (мс) Для избежания задержек аудио/видео, вызванных колебаниями сети и беспорядком сетевых пакетов, TRTC поддерживает буфер воспроизведения на стороне воспроизведения для организации полученных сетевых пакетов данных. Размер буфера адаптивно изменяется в зависимости от текущего качества сети и преобразуется в продолжительность в миллисекундах, т. е. ` jitterBufferDelay `. |
| point2PointDelay | Описание поля: задержка от конца к концу (мс) ` point2PointDelay ` представляет задержку «якорь -> облако -> аудитория». Точнее говоря, она представляет задержку всей цепи «сбор -> кодирование -> сетевая передача -> получение -> буферизация -> декодирование -> воспроизведение». ` point2PointDelay ` работает только если оба локальный и удаленный SDK имеют версию 8.5 или выше. Если удаленный SDK имеет версию ниже 8.5, это значение всегда будет равно 0 и таким образом бессмысленно. |
| remoteNetworkRTT | Описание поля: время приема-передачи (мс) от SDK к облаку Это значение представляет общее время, необходимое для отправки сетевого пакета из SDK в облако и последующей отправки сетевого пакета из облака обратно в SDK, т. е. общее время, необходимое сетевому пакету для прохождения цепи «SDK -> облако -> SDK». Чем меньше значение, тем лучше. Если ` remoteNetworkRTT ` менее 50 мс, это означает короткую задержку при аудио/видеовызове; если ` remoteNetworkRTT ` превышает 200 мс, это означает длительную задержку при аудио/видеовызове. Следует уточнить, что ` remoteNetworkRTT ` представляет общее время, потраченное на цепь «SDK -> облако -> SDK»; поэтому нет необходимости различать ` remoteNetworkUpRTT ` и ` remoteNetworkDownRTT `. |
| remoteNetworkUplinkLoss | Описание поля: коэффициент потери пакетов при передаче (%) от SDK к облаку Чем меньше значение, тем лучше. Если ` remoteNetworkUplinkLoss ` равен ` 0% `, качество восходящей сети очень хорошее, и пакеты данных, загруженные в облако, практически не теряются. Если ` remoteNetworkUplinkLoss ` равен ` 30% `, 30% пакетов аудио/видеоданных, отправленных в облако SDK, теряются на цепи передачи. |
| streamType | Описание поля: тип видеопотока (HD-видео большого размера \| сглаженное видео малого размера \| побочный поток) |
| userId | Описание поля: идентификатор пользователя |
| videoBitrate | Описание поля: битрейт удаленного видео (Кбит/с) |
| videoBlockRate | Описание поля: коэффициент задержки воспроизведения видео (%) Коэффициент задержки воспроизведения видео (videoBlockRate) = кумулятивная продолжительность задержки воспроизведения видео (videoTotalBlockTime) / общая продолжительность воспроизведения видео |
| videoPacketLoss | Описание поля: общий коэффициент потери пакетов (%) видеопотока ` videoPacketLoss ` представляет коэффициент потери пакетов, в конечном итоге рассчитанный на стороне аудитории после прохождения аудио/видеопотока по полной цепи передачи «якорь -> облако -> аудитория». Чем меньше ` videoPacketLoss `, тем лучше. Коэффициент потери пакетов 0 указывает на то, что все данные видеопотока полностью достигли аудитории. Если ` downLoss ` равен ` 0 `, но ` videoPacketLoss ` нет, на цепи «облако -> аудитория» для видеопотока нет потери пакетов, но на цепи «якорь -> облако» есть безвозвратные потери пакетов. |
| videoTotalBlockTime | Описание поля: кумулятивная продолжительность задержки воспроизведения видео (мс) |
| width | Описание поля: ширина удаленного видео в пикс. |

## TRTCStatistics

**TRTCStatistics**

#### Метрики сети и производительности.

| EnumType | DESC |
| --- | --- |
| appCpu | Описание поля: использование ЦПУ (%) текущего приложения, системы Android 8.0 и выше не поддерживаются |
| appMemoryUsageInMB | Описание поля: размер использования памяти (МБ) текущего приложения |
| downLoss | Описание поля: коэффициент потери пакетов при приеме (%) от облака к SDK Чем меньше значение, тем лучше. Если ` downLoss ` равен ` 0% `, качество нисходящей сети очень хорошее, и пакеты данных, полученные из облака, практически не теряются. Если ` downLoss ` равен ` 30% `, 30% пакетов аудио/видеоданных, отправленных в SDK облаком, теряются на цепи передачи. |
| gatewayRtt | Описание поля: время приема-передачи (мс) от SDK к шлюзу Это значение представляет общее время, необходимое для отправки сетевого пакета из SDK на шлюз и последующей отправки сетевого пакета со шлюза обратно в SDK, т. е. общее время, необходимое сетевому пакету для прохождения цепи «SDK -> шлюз -> SDK». Чем меньше значение, тем лучше. Если ` gatewayRtt ` менее 50 мс, это означает короткую задержку при аудио/видеовызове; если ` gatewayRtt ` превышает 200 мс, это означает длительную задержку при аудио/видеовызове. Следует уточнить, что ` gatewayRtt ` неприменим для сотовых сетей. |
| localStatisticsArray | Описание поля: статистика локального аудио/видео Поскольку может быть три локальных аудио/видеопотока (т. е. HD-видео большого размера, сглаженное видео малого размера и побочный поток), статистика локального аудио/видео представляет собой массив. |
| localStatisticsArraySize | Описание поля: размер массива ` localStatisticsArray ` |
| receivedBytes | Описание поля: общее количество полученных байтов (включая данные сигнализации и аудио/видеоданные) |
| remoteStatisticsArray | Описание поля: статистика удаленного аудио/видео Поскольку может быть несколько одновременных удаленных пользователей, и каждый из них может иметь несколько одновременных аудио/видеопотоков (т. е. HD-видео большого размера, сглаженное видео малого размера и побочный поток), статистика удаленного аудио/видео представляет собой массив. |
| remoteStatisticsArraySize | Описание поля: размер массива ` remoteStatisticsArray ` |
| rtt | Описание поля: время приема-передачи (мс) от SDK к облаку Это значение представляет общее время, необходимое для отправки сетевого пакета из SDK в облако и последующей отправки сетевого пакета из облака обратно в SDK, т. е. общее время, необходимое сетевому пакету для прохождения цепи «SDK -> облако -> SDK». Чем меньше значение, тем лучше. Если ` rtt ` менее 50 мс, это означает короткую задержку при аудио/видеовызове; если ` rtt ` превышает 200 мс, это означает длительную задержку при аудио/видеовызове. Следует уточнить, что ` rtt ` представляет общее время, потраченное на цепь «SDK -> облако -> SDK»; поэтому нет необходимости различать ` upRtt ` и ` downRtt `. |
| sentBytes | Описание поля: общее количество отправленных байтов (включая данные сигнализации и аудио/видеоданные) |
| systemCpu | Описание поля: использование ЦПУ (%) текущей системы, системы Android 8.0 и выше не поддерживаются |
| systemMemoryInMB | Описание поля: размер памяти (МБ) текущей системы |
| systemMemoryUsageInMB | Описание поля: размер использования памяти (МБ) текущей системы, iOS и MAC не поддерживаются |
| upLoss | Описание поля: коэффициент потери пакетов при передаче (%) от SDK к облаку Чем меньше значение, тем лучше. Если ` upLoss ` равен ` 0% `, качество восходящей сети очень хорошее, и пакеты данных, загруженные в облако, практически не теряются. Если ` upLoss ` равен ` 30% `, 30% пакетов аудио/видеоданных, отправленных в облако SDK, теряются на цепи передачи. |

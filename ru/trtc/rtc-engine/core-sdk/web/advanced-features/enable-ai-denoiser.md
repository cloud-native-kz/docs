# Включение AI Denoiser

Плагин AI Denoiser уменьшает шум во время вызовов и минимизирует влияние окружающего шума на качество коммуникации. Он эффективно подавляет звуки клавиатуры, стуки и другие шумы, что особенно полезно в сценариях, чувствительных к импульсному шуму, таких как конференции.

В этой статье описано, как применить AI Denoiser к локальному аудиопотоку в приложениях TRTC. [Нажмите здесь, чтобы попробовать демо](/https://trtc.io/demo/homepage/#/detail?scene=trtc&active=ai) с деноизером.

## Предварительные требования

- Подробные сведения об использовании и стоимости см. в TRTC [Характеристики выпуска и цены](https://trtc.io/document/56025#f10b65d1-6e8d-41e3-8686-84909b00a1a2).
- Поддерживаемые браузеры
  - Chrome 66+, Edge 79+, Safari 14.1+, Firefox 76+.
  - Для систем Android версия Chrome должна быть выше 92.

Для оптимального использования AI Denoiser рекомендуется использовать последнюю версию браузера Chrome.

> **Примечание:** Если в входном сигнале микрофона присутствует фоновая музыка, плагин деноизера может рассматривать её как шум и устранить.

## Процесс реализации

#### Развертывание статических ресурсов

Этот плагин зависит от файлов статических ресурсов. Чтобы убедиться, что браузер может правильно загружать и запускать эти файлы, выполните следующие шаги:

1. Опубликуйте каталог `node_modules/trtc-sdk-v5/assets` на CDN или сервере статических ресурсов.
2. Передайте адрес вашей CDN параметру `assetsPath` при создании экземпляра TRTC, например: `TRTC.create({ assetsPath: 'https://xxx/assets' })`. SDK будет загружать необходимые файлы ресурсов по мере необходимости.

> **Примечание:** Если ваша версия ниже `v5.10.0`, необходимо опубликовать `node_modules/trtc-sdk-v5/plugins/ai-denoiser/denoiser-wasm.js` на CDN и поместить его в тот же публичный каталог, например `https://xxx/assets`. Если URL-адрес хоста файлов в каталоге отличается от URL-адреса хоста веб-приложения, необходимо включить политику CORS для домена файлов. Не размещайте файлы каталога в HTTP-сервисе, так как загрузка HTTP-ресурсов в домене HTTPS будет заблокирована политиками безопасности браузера.

#### Включение деноизера

```
const trtc = TRTC.create({ assets: 'https://xxx/assets' });await trtc.startLocalAudio();await trtc.startPlugin('AIDenoiser', {  sdkAppId: 123456,  userId: 'user_123',  userSig: 'XXXXXXXX'});
```

#### Отключение деноизера

```
await trtc.stopPlugin('AIDenoiser');
```

## Описание API

### trtc.startPlugin('AIDenoiser', options)

Используется для включения деноизера.

#### options:

| Имя | Тип | Описание |
| --- | --- | --- |
| assetsPath | `string` | Расположение, где хранится файл `denoiser-wasm.js`. Если ваша версия ниже `v5.5.2`, необходимо передать каталог ресурсов assets через этот параметр. |
| sdkAppId | `number` | sdkAppId текущего приложения |
| userId | `string` | userId текущего пользователя |
| userSig | `string` | userSig текущего пользователя |

### trtc.stopPlugin('AIDenoiser')

Используется для отключения деноизера.

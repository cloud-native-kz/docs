# Включение кроссрумного вызова

По умолчанию аудио и видеопотоки в разных комнатах изолированы, и только пользователи в одной комнате могут общаться через аудио и видео. Функция кроссрумного соединения позволяет пользователям в разных комнатах обмениваться аудио и видеопотоками.

В сценариях прямой трансляции эта функция поддерживает общение между ведущими и позволяет зрителям в обеих комнатах смотреть другого ведущего. В этом документе объясняется, как использовать плагин CrossRoom для реализации кроссрумного соединения.

Кроссрумное соединение поддерживает два режима:

- **Соединение на уровне комнаты**: подключение со всеми ведущими в целевой комнате.
- **Соединение на уровне пользователя**: подключение с определённым ведущим в целевой комнате.

## Принцип работы

- **Соединение на уровне комнаты**: передача аудио и видеопотоков всех ведущих из текущей комнаты в целевую комнату и передача аудио и видеопотоков всех ведущих из целевой комнаты в текущую комнату.
- **Соединение на уровне пользователя**: передача собственных аудио и видеопотоков из текущей комнаты в целевую комнату и передача аудио и видеопотоков указанного ведущего из целевой комнаты в текущую комнату.

## Этапы использования

> **Примечание:** убедитесь, что версия вашего SDK ≥ v5.8.0.

### 1. Начало кроссрумного соединения

```
import { CrossRoom } from 'trtc-sdk-v5/plugins/cross-room';const trtc = TRTC.create({ plugins: [CrossRoom] });await trtc.enterRoom({ sdkAppId, userId: 'user1', userSig, roomId: 7777 });// Connect user1 in room 7777 with user2 in room 8888. The stream of user2 will be pushed to room 7777, and the stream of user1 will be pushed to room 8888.await trtc.startPlugin('CrossRoom', {  roomId: 8888,  // Choose either roomId or strRoomId  // strRoomId: '8888'  userId: 'user2' // If not provided, it indicates a room-level cross-room connection});
```

#### Описание параметров

| Имя параметра | Тип | Обязательный | Описание |
| --- | --- | --- | --- |
| roomId | Number | Нет | ID целевой комнаты. |
| strRoomId | String | Нет | Строковый ID целевой комнаты, выберите либо `roomId`, либо `strRoomId`. |
| userId | String | Нет | ID ведущего в целевой комнате. Если не указано, это указывает на кроссрумное соединение на уровне комнаты. |

### 2. Обновление статуса отключения звука ведущего кроссрумного соединения

После начала кроссрумного соединения потоки ведущих из целевой комнаты будут передаваться в текущую комнату, и все пользователи в текущей комнате смогут получать аудио и видеопотоки ведущего.

Вы можете использовать следующий интерфейс для ограничения восходящих возможностей (аудио/основное видео/подпоток) ведущего кроссрумного соединения в текущей комнате, что влияет на всех пользователей в комнате.

Если определённая восходящая возможность отключена, пользователи в текущей комнате не смогут получать или подписываться на соответствующие аудио или видеопотоки.

```
await trtc.updatePlugin('CrossRoom', {  updateList: [{    roomId: 8888,    // strRoomId: '8888'    userId: 'user2', // If not provided, it indicates a room-level cross-room connection    muteAudio: true,    muteVideo: false,    muteSubStream: false,  }]});
```

#### Описание параметров

| Имя параметра | Тип | Обязательный | Описание |
| --- | --- | --- | --- |
| updateList | Array | Да | Список ведущих кроссрумного соединения, статус отключения звука которых необходимо обновить. |
| updateList[0].roomId | Number | Нет | ID целевой комнаты. |
| updateList[0].strRoomId | String | Нет | Строковый ID целевой комнаты, выберите либо roomId, либо strRoomId. |
| updateList[0].userId | String | Нет | ID ведущего в целевой комнате. Если не указано, это указывает на кроссрумное соединение на уровне комнаты. |
| updateList[0].muteAudio | Boolean | Нет | Следует ли остановить аудиопоток ведущего кроссрумного соединения из целевой комнаты. |
| updateList[0].muteVideo | Boolean | Нет | Следует ли остановить основной видеопоток ведущего кроссрумного соединения из целевой комнаты. |
| updateList[0].muteSubStream | Boolean | Нет | Следует ли остановить видеопоток подпотока ведущего кроссрумного соединения из целевой комнаты. |

### 3. Остановка кроссрумного соединения

```
await trtc.stopPlugin('CrossRoom', { roomId: 8888 }); // Stop the cross-room connection with the specified roomawait trtc.stopPlugin('CrossRoom'); // Stop all cross-room connections initiated by the current user
```

#### Описание параметров

| Имя параметра | Тип | Обязательный | Описание |
| --- | --- | --- | --- |
| roomId | Number | Нет | ID целевой комнаты. |
| strRoomId | String | Нет | Строковый ID целевой комнаты, выберите либо `roomId`, либо `strRoomId`. |

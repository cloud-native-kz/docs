## Демонстрация

## Включение/выключение микрофона и камеры

Существует 3 варианта реализации этой функции. Рекомендуется [первый вариант](#436632d8-e65d-4ef4-b3cf-63bb807e1a15).

| Варианты | Программное или аппаратное обеспечение | Загораются ли индикаторы захвата | Продолжить отправку пакетов отключения звука | Возможность предпросмотра локальной камеры |
| --- | --- | --- | --- | --- |
| Использование параметра Mute | Программное | Да | Да | Нет |
| Использование stopLocalVideo() & stopLocalAudio() | Аппаратное | Нет | Нет | Нет |
| Использование параметра Publish | Программное | Да | Нет | Да |

Для удалённых пользователей все три плана работают одинаково:

- После выключения камеры остальные пользователи в комнате получат событие [TRTC.EVENT.REMOTE_VIDEO_UNAVAILABLE](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.REMOTE_VIDEO_UNAVAILABLE).
- После включения камеры остальные пользователи в комнате получат событие [TRTC.EVENT.REMOTE_VIDEO_AVAILABLE](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.REMOTE_VIDEO_AVAILABLE).
- После выключения микрофона остальные пользователи в комнате получат событие [TRTC.EVENT.REMOTE_AUDIO_UNAVAILABLE](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.REMOTE_AUDIO_UNAVAILABLE).
- После включения микрофона остальные пользователи в комнате получат событие [TRTC.EVENT.REMOTE_AUDIO_AVAILABLE](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.REMOTE_AUDIO_AVAILABLE).

### Использование параметра Mute

Вы можете использовать параметр **mute** методов [trtc.updateLocalVideo()](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#updateLocalVideo) и [trtc.updateLocalAudio()](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#updateLocalAudio) для управления состоянием включения/выключения камеры и микрофона после их включения. В этом варианте камера и микрофон отключаются на уровне программного обеспечения. Фактически устройство остаётся в активном состоянии, и физический индикатор захвата устройства всё ещё включён.

С физической точки зрения, поскольку запуск микрофона занимает некоторое время, голос пользователя может быть потерян в этот период. Поэтому **рекомендуется** использовать этот вариант. Преимущество этого варианта в том, что повторное включение камеры или микрофона будет происходить быстрее:

```
await trtc.startLocalVideo();// отключить звук камеры// После выключения камеры предпросмотр камеры станет чёрным, // в этот момент вы можете отобразить собственную маску пользовательского интерфейса.await trtc.updateLocalVideo({ mute: true });// включить звук камерыawait trtc.updateLocalVideo({ mute: false });
```

```
await trtc.startLocalAudio();// отключить звук микрофонаawait trtc.updateLocalAudio({ mute: true });// включить звук микрофонаawait trtc.updateLocalAudio({ mute: false });
```

> **Примечание:** Кроме того, после отключения звука микрофона или камеры будет отправляться пакет с очень низким битрейтом (около 1 кбит/с).

### Использование [stopLocalVideo()](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#stopLocalVideo) & [stopLocalAudio()](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#stopLocalAudio)

Вы можете вызвать методы [trtc.stopLocalVideo()](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#stopLocalVideo) & [trtc.stopLocalAudio()](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#stopLocalAudio) для отключения камеры и микрофона.

В этом варианте камера и микрофон отключаются физически, и физический индикатор захвата устройства будет отключён.

```
// Выключить камеруawait trtc.stopLocalVideo();// Включить камеру// 'local-video' — это идентификатор элемента в DOM, используемый для воспроизведения контейнера видео локальной камеры.await trtc.startLocalVideo({ view: 'local-video' });
```

```
// Выключить микрофонawait trtc.stopLocalAudio();// Включить микрофонawait trtc.startLocalAudio();
```

### Использование параметра Publish

Вы можете использовать параметр publish методов [updateLocalVideo()](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#updateLocalVideo) и [updateLocalAudio()](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#updateLocalAudio) для управления публикацией видео и аудиопотока.

В этом варианте вы управляете состоянием публикации аудио и видеопотока на уровне программного обеспечения. Фактически устройство остаётся в активном состоянии, и физический индикатор захвата устройства всё ещё включён.

```
await trtc.startLocalVideo();// не публиковать поток камеры. await trtc.updateLocalVideo({ publish: false });// опубликовать поток камеры.await trtc.updateLocalVideo({ publish: true });
```

```
await trtc.startLocalAudio();
// не публиковать поток микрофона
await trtc.updateLocalAudio({ publish: false });
// опубликовать поток микрофона
await trtc.updateLocalAudio({ publish: true });
```

## Переключение микрофона, камеры и динамика

### Переключение камеры

```
// Открыть камеру, по умолчанию первая камера в списке камер.await trtc.startLocalVideo();const cameraList = await TRTC.getCameraList();// Переключиться на вторую камеруif (cameraList[1]) {  await trtc.updateLocalVideo({ option: { cameraId: cameraList[1].deviceId }});}// На мобильном устройстве.// переключиться на фронтальную камеру.await trtc.updateLocalVideo({ option: { useFrontCamera: true }});// переключиться на тыльную камеру.await trtc.updateLocalVideo({ option: { useFrontCamera: false }});
```

### Переключение микрофона

```
// Открыть микрофон, по умолчанию первый микрофон в списке микрофонов.await trtc.startLocalAudio();const microphoneList = await TRTC.getMicrophoneList();// Переключиться на второй микрофонif (microphoneList[1]) {  await trtc.updateLocalAudio({ option: { microphoneId: microphoneList[1].deviceId }});}
```

### Переключение динамика

```
// Динамик по умолчанию — первый динамик в списке динамиков.const speakerList = await TRTC.getSpeakerList();if (speakerList[1]) {    await TRTC.setCurrentSpeaker(speakerList[1].deviceId);}
```

## Определение состояния медиа-устройства

### Событие DEVICE_CHANGED

Вы можете прослушивать событие [DEVICE_CHANGED](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.DEVICE_CHANGED) для определения поведения при подключении устройства, отключении устройства и запуске устройства.

```
trtc.on(TRTC.EVENT.DEVICE_CHANGED, (event) => {  // Подключение устройства  if (event.action === 'add') {    // Подключение камеры    if (event.type === 'camera') {}  } else if (event.action === 'remove') {    // Отключение устройства  } else if (event.action === 'active') {    // Запуск устройства  }  console.log(`${event.type}(${event.device.label}) ${event.action}`);});
```

#### Проверка отключения используемого устройства

```
trtc.on(TRTC.EVENT.DEVICE_CHANGED, (event) => {  if (event.action === 'remove') {    if (event.type === 'camera') {      const currentCameraId = trtc.getVideoTrack()?.getSettings()?.deviceId;      if (event.device.deviceId === currentCameraId) {        // Используемая камера отключена.      }    } else if (event.type === 'microphone') {      const currentMicId = trtc.getAudioTrack()?.getSettings()?.deviceId;      if (event.device.deviceId === currentMicId) {        // Используемый микрофон отключен.      }    }  }});
```

#### Переключение на новое устройство при подключении нового устройства пользователем

```
trtc.on(TRTC.EVENT.DEVICE_CHANGED, (event) => {  if (event.action === 'add') {    if (event.type === 'camera') {      // После подключения камеры переключиться на новую камеру      trtc.updateLocalVideo({ option: { cameraId: event.device.deviceId }});    }  }});
```

### Автоматический повторный захват

SDK будет определять поведение при подключении и отключении камеры и микрофона, а также автоматически повторно захватывать медиа-устройство в подходящее время. Конкретная логика выглядит следующим образом:

1. Если текущая используемая камера/микрофон отключена, SDK попытается повторно захватить доступное медиа-устройство.
2. Если камера/микрофон отключены и нет доступного медиа-устройства, SDK будет определять поведение при подключении устройства. Если будет подключено новое доступное медиа-устройство, SDK попытается повторно захватить новое подключённое медиа-устройство.
3. Если повторный захват не удался, будет выброшена ошибка [TRTC.ERROR_CODE.DEVICE_ERROR](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-ERROR_CODE.html#.DEVICE_ERROR), а extraCode будет 5308 и 5309.

Прослушивание событий исключения при захвате устройства:

```
trtc.on(TRTC.EVENT.VIDEO_PLAY_STATE_CHANGED, event => {  // Исключение при захвате локальной камеры, в этот момент SDK попытается автоматически восстановить захват, вы можете направить пользователя на проверку нормальности работы камеры.  if (event.userId === '' && event.streamType === TRTC.TYPE.STREAM_TYPE_MAIN && (event.reason === 'mute' || event.reason === 'ended')) {}});trtc.on(TRTC.EVENT.AUDIO_PLAY_STATE_CHANGED, event => {  // Исключение при захвате локального микрофона, в этот момент SDK попытается автоматически восстановить захват, вы можете направить пользователя на проверку нормальности работы микрофона.  if (event.userId === '' && (event.reason === 'mute' || event.reason === 'ended')) {}});
```

Прослушивание событий неудачного повторного захвата SDK:

```
trtc.on(TRTC.EVENT.ERROR, error => {  if (error.code === TRTC.ERROR_CODE.DEVICE_ERROR) {    // Повторный захват камеры не удался    if (error.extraCode === 5308) {      // После направления пользователя на проверку устройства вызовите updateLocalVideo и передайте cameraId для повторного захвата.      trtc.updateLocalVideo({ option: { cameraId: '' }});    }    // Повторный захват микрофона не удался    if (error.extraCode === 5309) {      // После направления пользователя на проверку устройства вызовите updateLocalAudio и передайте microphoneId для повторного захвата.      trtc.updateLocalAudio({ option: { microphoneId: '' }});    }  }})
```

## Определение того, говорит ли пользователь после отключения микрофона

```
const trtc = TRTC.create();await trtc.startLocalAudio();let isAudioMuted = false;await trtc.updateLocalAudio({ mute: true });isAudioMuted = true;// создать новый экземпляр trtc для определения уровня громкости микрофона.const trtcA = TRTC.create();trtcA.enableAudioVolumeEvaluation();trtcA.on(TRTC.EVENT.AUDIO_VOLUME, event => {  event.result.forEach(item => {      // 1. userId === '' — это локальный уровень громкости.      // 2. Обычно считается, что когда уровень громкости больше 10, пользователь говорит, вы также можете настроить этот порог.      if (item.userId === '' && item.volume > 10 && isAudioMuted) {        // Пользователь говорит после отключения микрофона.      }    })})await trtcA.startLocalAudio();
```

## Определение состояния включения/выключения микрофона и камеры удалённого пользователя

Это часто используется для подтверждения того, как отображать состояние включения/выключения микрофона удалённого пользователя.

Предположим, что есть два пользователя A и B.

1. После того как A успешно входит в комнату, если в комнате есть ещё один якорь B, A получит событие [REMOTE_USER_ENTER](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.REMOTE_USER_ENTER).
2. В этот момент A может предположить, что B по умолчанию не включал микрофон или камеру.
3. Если A получит событие [REMOTE_AUDIO_AVAILABLE](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.REMOTE_AUDIO_AVAILABLE) или [REMOTE_VIDEO_AVAILABLE](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.REMOTE_VIDEO_AVAILABLE) от B, это означает, что B включил микрофон или камеру.
4. Если A получит событие [REMOTE_AUDIO_UNAVAILABLE](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.REMOTE_AUDIO_UNAVAILABLE) или [REMOTE_VIDEO_UNAVAILABLE](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.REMOTE_VIDEO_UNAVAILABLE) от B, это означает, что B выключил микрофон или камеру.

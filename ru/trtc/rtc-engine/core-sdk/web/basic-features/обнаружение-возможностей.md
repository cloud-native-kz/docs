# Обнаружение возможностей

Чтобы обеспечить хороший опыт использования SDK TRTC, рекомендуется протестировать медиа-устройства пользователя и качество сети перед входом в комнату TRTC.

## Поддерживаемые платформы

См. [Поддерживаемые платформы](https://www.tencentcloud.com/document/product/647/59733).

## Обнаружение возможностей браузера

Перед входом в комнату рекомендуется использовать API [TRTC.isSupported](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#.isSupported) для проверки поддержки SDK в текущем браузере. Если браузер не поддерживается, направьте пользователя на использование поддерживаемого браузера (**Chrome 56+, Edge 80+, Firefox 56+, Safari 11+**).

```
TRTC.isSupported().then(checkResult => {  // Not supported, guide the user to use a supported browser(Chrome 56+, Edge 80+, Firefox 56+, Safari 11+).  if (!checkResult.result) {}  // Not support to publish video  if (!checkResult.detail.isH264EncodeSupported) {}  // Not support to subscribe video  if (!checkResult.detail.isH264DecodeSupported) {}});
```

Если результат проверки, возвращаемый **TRTC.isSupported**, равен **false**, причина может быть следующей:

1. Веб-страница использует протокол http. Браузеры не разрешают сайтам с протоколом http захватывать камеры и микрофоны, необходимо развернуть страницу с использованием протокола https.
2. Браузер Firefox должен динамически загружать кодек H264 после установки, поэтому результат обнаружения будет false в течение короткого времени. Пожалуйста, дождитесь и повторите попытку или направьте пользователя на использование других браузеров.

## Тестирование медиа-устройства с помощью плагина

Плагин детектора устройств используется для тестирования медиа-устройств пользователя (камера, микрофон, динамик) и сети, что рекомендуется использовать перед входом пользователя в комнату.

> **Примечание:** Поддержка версии TRTC Web SDK ≥ v5.8.0. Z-индекс плагина обнаружения устройств установлен на 9999, необходимо убедиться, что ваш z-индекс ниже 9999.

### Использование

```
import { DeviceDetector } from 'trtc-sdk-v5/plugins/device-detector';
const trtc = TRTC.create({ plugins: [DeviceDetector] });

// 1. Test Media Device Only
const result = await trtc.startPlugin('DeviceDetector');

// 2. Test Media Device & Network Quality
const options = { 
    networkDetect: { sdkAppId, userId, userSig }
}
const resultWithNetwork = await trtc.startPlugin('DeviceDetector', options);
```

### Описание API

#### trtc.startPlugin('DeviceDetector', options)

##### options.networkDetect (опционально)

| Имя | Тип | Атрибуты | Описание |
| --- | --- | --- | --- |
| sdkAppId | `number` | обязательно | Ваш sdkAppId |
| userId | `string` | обязательно | userId для тестирования качества восходящей сети. Должен отличаться от downlinkUserId. |
| userSig | `string` | обязательно | [UserSig](https://trtc.io/document/35166) для userId. |
| downlinkUserId | `string` | опционально | userId для тестирования качества нисходящей сети. Должен отличаться от userId. Заполнение downlinkUserId и downlinkUserSig выполнит тест нисходящей сети. |
| downlinkUserSig | `string` | опционально | [UserSig](https://trtc.io/document/35166) для downlinkUserId. |
| roomId | `number` | опционально | Опционально. Значение по умолчанию — 8080. Значение должно быть целым числом в диапазоне [1, 4294967294]. |

##### Результат startPlugin

> **Примечание:** Если пользователь нажимает кнопку пропуска обнаружения на странице тестирования, возвращается `undefined`.

| Имя | Тип | Результат |
| --- | --- | --- |
| **camera** | `object` | { isSuccess: boolean, device: MediaDeviceInfo} |
| **microphone** | `object` | { isSuccess: boolean, device: MediaDeviceInfo} |
| **speaker** | `object` | { isSuccess: boolean, device: MediaDeviceInfo} |
| **network** | `object` | { isSuccess: boolean, result: { quality: number , rtt: number }} Качество — это качество восходящей сети. Когда передаются downlinkUserId и downlinkUserSig, качество является худшим из качеств восходящей и нисходящей сети. **Значения качества сети** 0: качество сети неизвестно, указывает, что текущий экземпляр TRTC не установил медиа-соединение. 1: качество сети отличное. 2: качество сети хорошее. 3: качество сети среднее. 4: качество сети плохое. 5: качество сети крайне плохое. 6: медиа-соединение разорвано. |

## Тестирование медиа-устройства вручную

### Тестирование камеры

1. Получение списка камер

Как правило, сначала необходимо проверить список камер, чтобы подтвердить, что пользователь имеет камеру. Затем вы можете показать список камер, чтобы пользователь мог выбрать нужную ему камеру.

```
const cameraList = await TRTC.getCameraList();const hasCameraDevice = cameraList.length > 0;if (!hasCameraDevice) {    // User has not any camera.}
```

2. Захват и воспроизведение видео с камеры

На этом шаге необходимо захватить и воспроизвести видео с камеры без необходимости публиковать поток.

  - Если захват камеры выполнен успешно, необходимо направить пользователя на подтверждение того, что он видит видеопоток камеры в норме.
  - Если захват камеры не удался, это обычно вызвано ошибками NotAllowedError или NotReadableError, см. [DEVICE_ERROR](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-ERROR_CODE.html#.DEVICE_ERROR).

```
try {    // Capture and play camera, not need to publish stream    await trtc.startLocalVideo({ view: 'camera-video-view-id', publish: false });} catch(error) {    // Common reasons: NotAllowedError or NotReadableError}
```

3. Переключение камеры

SDK будет захватывать первую камеру из списка камер по умолчанию, если вы не указали cameraId при вызове trtc.startLocalVideo.

Пользователь может захотеть переключиться на другую камеру перед входом в комнату, если у него есть несколько камер.

```
// You can get the cameraId from the camera list you got in fisrt step.if (cameraList[1]) {    try {        await trtc.updateLocalVideo({ option: { cameraId: cameraList[1].deviceId } });    } catch(error) {      // Common reasons: NotAllowedError or NotReadableError, same with step 2.    }}
```

4. Остановка камеры

Необходимо остановить камеру по завершении тестирования.

```
await trtc.stopLocalVideo();
```

### Тестирование микрофона

1. Получение списка микрофонов

Как правило, сначала необходимо проверить список микрофонов, чтобы подтвердить, что пользователь имеет микрофон. Затем вы можете показать список микрофонов, чтобы пользователь мог выбрать нужный ему микрофон.

```
const microphoneList = await TRTC.getMicrophoneList();const hasMicrophoneDevice = micList.length > 0;if (!hasMicrophoneDevice) {    // User has not any microphone.}
```

2. Захват микрофона и получение громкости

На этом шаге необходимо захватить микрофон, получить громкость звука микрофона для отображения полосы громкости и не нужно публиковать поток.

  - Если захват успешен, необходимо направить пользователя на подтверждение того, что полоса громкости изменилась.
  - Если захват не удался, это обычно вызвано ошибками NotAllowedError или NotReadableError, см. [DEVICE_ERROR](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-ERROR_CODE.html#.DEVICE_ERROR).

```
try {    // Capture microphone    await trtc.startLocalAudio({ publish: false });    trtc.on(TRTC.EVENT.AUDIO_VOLUME, event => {        event.result.forEach(({ userId, volume }) => {            // When userId is an empty string, it represents the volume of the local microphone.            const isMe = userId === '';             if (isMe) {               // show a volume bar based on the value of volume.            }        })    });    trtc.enableAudioVolumeEvaluation(500);} catch(error) {    // Common reasons: NotAllowedError or NotReadableError}
```

3. Переключение микрофона

SDK будет захватывать первый микрофон из списка микрофонов по умолчанию, если вы не указали microphoneId при вызове trtc.startLocalAudio.

Пользователь может захотеть переключиться на другой микрофон перед входом в комнату, если у него есть несколько микрофонов.

```
// You can get the microphoneId from the microphone list you got in fisrt step.if (microphoneList[1]) {    try {        await trtc.updateLocalAudio({ option: { microphoneId: microphoneList[1].deviceId } });    } catch(error) {      // Common reasons: NotAllowedError or NotReadableError, same with step 2.    }}
```

4. Остановка микрофона

```
// You need to stop microphone when the testing finished.await trtc.stopLocalAudio();// disable audio-volume eventtrtc.enableAudioVolumeEvaluation(-1);
```

### Тестирование динамика

1. Получение списка динамиков

```
const speakerList = await TRTC.getSpeakerList();if (speakerList.length === 0) {    // User has not speaker.}
```

2. Воспроизведение mp3-файла через элемент audio.

Необходимо направить пользователя на воспроизведение mp3-файла и подтверждение того, что он слышит звук от динамика.

```
<audio id="audio-player" src="xxxxx" controls></audio>
```

3. Переключение динамика

Вы не можете использовать [TRTC.setCurrentSpeaker](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#.setCurrentSpeaker) для переключения динамика, потому что вы воспроизводите mp3 через элемент audio, а не через SDK TRTC. Есть свойство [HTMLMediaElement.sinkId](https://developer.mozilla.org/en-US/docs/Web/API/HTMLMediaElement/sinkId), которое можно использовать для установки динамика.

```
const audioElement = document.getElementById('audio-player');if (speakerList[1]) {    audioElement.sinkId = speakerList[1].deviceId;}
```

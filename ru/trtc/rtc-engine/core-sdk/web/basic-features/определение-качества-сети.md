# Определение качества сети

Перед входом в комнату или во время звонка вы можете проверить качество сети пользователя для оценки текущего качества сети. Если качество сети пользователя низкое, рекомендуется направить пользователей на проверку своей сети, чтобы обеспечить нормальное качество звонка.

В этом руководстве описано, как реализовать определение качества сети до или во время звонка на основе события [NETWORK_QUALITY](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.NETWORK_QUALITY).

## Определение качества сети во время звонка

Обратитесь к [NETWORK_QUALITY](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.NETWORK_QUALITY).

```
const trtc = TRTC.create();trtc.on(TRTC.EVENT.NETWORK_QUALITY, event => {   console.log(`network-quality, uplinkNetworkQuality:${event.uplinkNetworkQuality}, downlinkNetworkQuality: ${event.downlinkNetworkQuality}`)   console.log(`uplink rtt:${event.uplinkRTT} loss:${event.uplinkLoss}`)   console.log(`downlink rtt:${event.downlinkRTT} loss:${event.downlinkLoss}`)})
```

## Определение качества сети перед звонком

### Процесс реализации

1. Вызовите [TRTC.create()](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#.createTRTC) для создания двух экземпляров TRTC, именуемых uplinkTRTC и downlinkTRTC.
2. Оба экземпляра TRTC входят в одну и ту же комнату.
3. Используйте uplinkTRTC для startLocalVideo и прослушивайте событие [NETWORK_QUALITY](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.NETWORK_QUALITY) для определения качества сети восходящего канала.
4. Используйте downlinkTRTC для startRemoteVideo и прослушивайте событие [NETWORK_QUALITY](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.NETWORK_QUALITY) для определения качества сети нисходящего канала.
5. Весь процесс длится около 15 секунд, после чего берется среднее качество сети для грубого определения условий сети восходящего и нисходящего каналов.

> **Примечание**
> Процесс определения будет генерировать небольшое количество платежей за [Биллинг](https://www.tencentcloud.com/document/product/647/34610). Если разрешение опубликованного потока не указано, по умолчанию опубликуется видеопоток с разрешением 640*480.

### Пример кода

```
let uplinkTRTC = null; // Used to detect uplink network qualitylet downlinkTRTC = null; // Used to detect downlink network qualitylet localStream = null; // Stream used for testinglet testResult = {  // Record uplink network quality data  uplinkNetworkQualities: [],  // Record downlink network quality data  downlinkNetworkQualities: [],  average: {    uplinkNetworkQuality: 0,    downlinkNetworkQuality: 0  }}// 1. Test uplink network qualityasync function testUplinkNetworkQuality() {  uplinkTRTC = TRTC.create();  uplinkTRTC.enterRoom({    roomId: 8080,    sdkAppId: 0, // Fill in sdkAppId    userId: 'user_uplink_test',    userSig: '', // userSig of uplink_test    scene: 'rtc'  })  uplinkTRTC.on(TRTC.EVENT.NETWORK_QUALITY, event => {    const { uplinkNetworkQuality } = event;    testResult.uplinkNetworkQualities.push(uplinkNetworkQuality);  });}// 2. Detect downlink network qualityasync function testDownlinkNetworkQuality() {  downlinkTRTC = TRTC.create();  downlinkTRTC.enterRoom({    roomId: 8080,    sdkAppId: 0, // Fill in sdkAppId    userId: 'user_downlink_test',    userSig: '', // userSig    scene: 'rtc'  });  downlinkTRTC.on(TRTC.EVENT.NETWORK_QUALITY, event => {      const { downlinkNetworkQuality } = event;      testResult.downlinkNetworkQualities.push(downlinkNetworkQuality);  })}// 3. Start detectiontestUplinkNetworkQuality();testDownlinkNetworkQuality();// 4. Stop detection after 15s and calculate the average network qualitysetTimeout(() => {  // Calculate the average uplink network quality  if (testResult.uplinkNetworkQualities.length > 0) {    testResult.average.uplinkNetworkQuality = Math.ceil(      testResult.uplinkNetworkQualities.reduce((value, current) => value + current, 0) / testResult.uplinkNetworkQualities.length    );  }  if (testResult.downlinkNetworkQualities.length > 0) {    // Calculate the average downlink network quality    testResult.average.downlinkNetworkQuality = Math.ceil(      testResult.downlinkNetworkQualities.reduce((value, current) => value + current, 0) / testResult.downlinkNetworkQualities.length    );  }      // Detection is over, clean up related states.  uplinkTRTC.exitRoom();  downlinkTRTC.exitRoom();}, 15 * 1000);
```

## Анализ результатов

После выполнения описанных выше шагов вы получите среднее качество сети восходящего канала и среднее качество сети нисходящего канала. Значения перечисления качества сети следующие:

| Значение | Значение |
| --- | --- |
| 0 | Качество сети неизвестно, указывает, что текущий экземпляр TRTC не установил медиа-соединение |
| 1 | Качество сети отличное |
| 2 | Качество сети хорошее |
| 3 | Качество сети среднее |
| 4 | Качество сети низкое |
| 5 | Качество сети крайне низкое |
| 6 | Медиа-соединение разорвано. Примечание: Если качество сети нисходящего канала имеет это значение, это означает, что все соединения нисходящего канала разорваны. |

> **Рекомендация** Когда качество сети больше 3, рекомендуется направить пользователя на проверку сети и попытаться изменить сеть, в противном случае будет сложно обеспечить нормальное качество аудио и видео звонка.
> Вы также можете снизить потребление полосы пропускания, используя следующие стратегии: Если качество сети восходящего канала больше 3, вы можете снизить битрейт с помощью интерфейса [trtc.updateLocalVideo()](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#updateLocalVideo) или отключить видео с помощью метода [trtc.stopLocalVideo()](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#stopLocalVideo) для снижения потребления полосы пропускания восходящего канала. Если качество сети нисходящего канала больше 3, вы можете снизить потребление полосы пропускания нисходящего канала, подписываясь на поток меньшего размера (см.: [Оптимизация видеозвонков между несколькими пользователями](https://www.tencentcloud.com/document/product/647/59665)) или подписываясь только на аудио.

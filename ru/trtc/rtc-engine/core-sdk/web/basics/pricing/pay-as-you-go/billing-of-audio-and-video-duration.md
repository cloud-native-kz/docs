# Тарификация аудио и видео длительности

Аудио и видео длительность — это базовый элемент тарификации TRTC. Использование аудио и видео длительности, генерируемой при использовании компонентов RTC Engine, Call, Conference и Live Audio Room, будет тарифицироваться в соответствии с расценками в этом документе. Использование, генерируемое компонентом Video Live, будет тарифицироваться в соответствии с [Тарификацией компонента Video Live](https://www.tencentcloud.com/document/product/647/66078).

Расценки основаны на длительности каждого участника в аудиовызове, видеовызове или комнате интерактивной трансляции в прямом эфире. Цена видео длительности варьируется в зависимости от разрешения.

Если у вас есть контракт с TRTC, будут применяться сведения о тарификации из контракта.

> **Примечание:** [Совокупное разрешение видео](https://www.tencentcloud.com/document/product/647/42734#.E7.AC.AC.E4.B8.80.E6.AD.A5.EF.BC.8C.E8.AE.A1.E7.AE.97.E5.90.84.E7.94.A8.E6.88.B7.E8.AE.A2.E9.98.85.E7.9A.84.E8.A7.86.E9.A2.91.E6.80.BB.E5.88.86.E8.BE.A8.E7.8E.87.EF.BC.8C.E4.BB.A5.E5.8F.8A.E6.97.B6.E9.95.BF.E7.94.A8.E9.87.8F2) применяется к учетным записям, первое приложение TRTC которых [создано](https://www.tencentcloud.com/document/product/647/37714) после 27 октября 2021 года. Компонент голосовой комнаты Live: когда вы используете компонент управления [Room core control (SeatGridView)](https://trtc.io/document/60334?product=live) Live Voice или интегрируетесь, ссылаясь на [API RTCRoomEngine](https://trtc.io/document/61639?product=live), это рассматривается как использование данного компонента. Компонент видео трансляции Live: когда ваше приложение создано после 18 декабря 2024 года (SDKAppID) и интегрируется с версией Live 2.8.0 или более поздней [основного управления для Video Live (LiveStreamCoreView)](https://trtc.io/document/60036?product=live), это рассматривается как использование данного компонента. Если вышеуказанные условия не выполнены, это рассматривается как использование компонента голосовой комнаты Live.

## Инструкции по тарификации

- Режим тарификации: ежедневная оплата по факту использования.
- Цикл тарификации: тарификация происходит ежедневно, плата за аудио и видео длительность вычитается при формировании платежного отчета на следующий день. Точные сведения о тарификации и время создания отчета см. в [Платежном отчете](https://console.tencentcloud.com/expense/bill/overview).
- Вы можете купить [предоплаченные пакеты](https://www.tencentcloud.com/document/product/647/56025) для вычета аудио и видео длительности. Предоплаченные пакеты будут использованы в первую очередь. Если вы не приобрели пакет или исчерпали свои пакеты, аудио и видео длительность будут тарифицироваться по ставкам оплаты по факту использования. Коэффициенты вычета для предоплаченных пакетов составляют **1:1, 1:4, 1:9, 1:16 и 1:36** для аудио, HD, FHD, 2K и 4K длительности соответственно. Например, для 1 минуты генерируемой HD видео длительности из предоплаченного пакета будет вычтено 4 минуты.
- Также доступно 10 000 бесплатных минут для использования аудио и видео длительности. Подробности см. в разделе [Бесплатные минуты](https://www.tencentcloud.com/document/product/647/42735).
- Вы можете использовать [TRTC Price Calculator](https://trtc.io/pricing/calculator) для оценки использования и расходов на аудио и видео длительность вашего сервиса.
- Если аудио/видео потоки в комнате TRTC транслируются в CSS для воспроизведения через CDN, будут взиматься дополнительные сборы. Подробности см. в разделе [Тарификация MixTranscoding и трансляции в CDN](https://www.tencentcloud.com/document/product/647/47631).

> **Примечание:** Если для комнаты инициирована задача обработки аудио/видео потока (запись, проверка, трансляция и т. д.), система назначит робота для комнаты (если инициировано несколько задач, будет несколько роботов). Робот войдет в комнату как пользователь и будет подписан на потоки комнаты. Это приведет к возникновению сборов за аудио/видео длительность. При трансляции в комнаты RTC робот не подписывается на потоки в комнате, поэтому плата за аудио длительность будет рассчитываться в зависимости от времени пребывания робота в комнате. В других сценариях (трансляция потоков из одной комнаты RTC в другую комнату или на стороннюю платформу и т. д.) плата за аудио/видео длительность будет рассчитываться в зависимости от медиа, полученного роботом. Если вы используете другие сервисы TRTC, которые не являются аудио/видео вызовами или интерактивной трансляцией в прямом эфире, такие как [облачная запись](https://www.tencentcloud.com/document/product/647/45176), будут взиматься дополнительные сборы. Подробности см. в инструкциях по тарификации для соответствующих сервисов TRTC. Если вы хотите, чтобы ваше использование по оплате по факту было тарифицировано ежемесячно, [свяжитесь с нами](https://console.tencentcloud.com/workorder/category).

## Формула тарификации

**Ежемесячная стоимость** = **Аудио длительность** x **Цена за аудио длительность** + **Видео длительность разных категорий** x **Цена за соответствующую видео категорию**

## Тарифы

| Категория | Цена за единицу (USD/1000 минут) | Совокупное разрешение видео |
| --- | --- | --- |
| Аудио | 0.99 | - |
| HD | 3.99 | Разрешение ≤ 921,600 (1280 x 720) |
| FHD | 8.99 | 921,600 (1280 x 720) < Разрешение ≤ 2,073,600 (1920 x 1080) |
| 2K | 15.99 | 2,073,600 (1920 x 1080) < Разрешение ≤ 3,686,400 (2560 x 1440) |
| 4K | 35.99 | 3,686,400 (2560 x 1440) < Разрешение ≤ 8,847,360 (4096 x 2160) |

> **Примечание:** Вы можете использовать [Price Calculator](https://trtc.io/pricing/calculator) TRTC для оценки ваших расходов. TRTC будет рассчитывать общую длительность, генерируемую вашей [учетной записью Tencent RTC](https://console.trtc.io/statistics), для каждой категории каждый день. Видео длительность классифицируется на четыре категории на основе **совокупного разрешения видео** и тарифицируется по-разному. Система сначала вычтет ваше использование с помощью [10,000 бесплатных минут](https://www.tencentcloud.com/document/product/647/42735), предложенных TRTC для каждой учетной записи разработчика, и минут приобретенных [ежемесячных пакетов](https://www.tencentcloud.com/document/product/647/56025). Оставшаяся аудио и видео длительность умножаются на их соответствующие цены за единицу и складываются. Сумма этих расходов составляет вашу ежедневную плату за аудио/видео длительность.

### Разрешение двойного потока

В режиме двойного потока разрешение пользователя рассчитывается следующим образом:

- Если пользователь подписан на поток высокого качества, для расчета совокупного разрешения будет использоваться разрешение потока высокого качества, установленное отправителем.
- Если пользователь подписан на поток низкого качества, для расчета совокупного разрешения будет использоваться разрешение, полученное пользователем.

### Разрешение потока общего доступа к экрану

Для потоков общего доступа к экрану плата за аудио/видео длительность рассчитывается на основе разрешения, установленного в TRTCVideoEncParam. Подробности о том, как установить разрешение, см. в следующих документах:

- Все платформы (C++): [startScreenCapture](https://www.tencentcloud.com/document/product/647/50770#94be1579f497befa5e6450725b4f1a5c)
- iOS: [startScreenCaptureInApp](https://liteav.sdk.qcloud.com/doc/api/zh-cn/group__TRTCCloud__ios.html#abf51acf26b2212192f7145468886b791), [startScreenCaptureByReplaykit](https://www.tencentcloud.com/document/product/647/50754#a40e37e782a049150b3314810516bf44)
- macOS: [startScreenCapture](https://www.tencentcloud.com/document/product/647/50754#a5556ab15765655b288ad33f8f49176d)
- Android: [startScreenCapture](https://www.tencentcloud.com/document/product/647/50762#9b55d2e9cd4e32eae74a9eb3555f6c8b)

В веб-версии, когда целевое разрешение не может быть достигнуто из-за ограничений устройства или браузера, браузер автоматически отрегулирует разрешение на ближайшее возможное совпадение целевому. Фактическое используемое разрешение будет основанием для расчета платежей. Более подробную информацию об этом процессе см. в разделе [setScreenProfile](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/global.html#ScreenShareProfile).

## Расчет использования

### Видео длительность

Для пользователей, которые успешно вошли в [комнату](https://www.tencentcloud.com/document/product/647/37714) TRTC:

- Видео длительность определяется как **длительность видео потоков, на которые подписаны пользователи** (длительность полученного видео).
- Если пользователь подписан на аудио и видео потоки другого пользователя, будет тарифицироваться только видео длительность.
- Пользователи могут входить и выходить из комнаты несколько раз. TRTC будет накапливать длительности нескольких видео сегментов в реальном времени.
- Если пользователь подписан на несколько видео потоков одновременно, разрешения каждого видео потока будут суммированы. Результат — это совокупное разрешение пользователя.
- TRTC будет классифицировать видео на основе **фактически полученного разрешения видео** и затем тарифицировать видео длительность разных категорий соответственно.

### Аудио длительность

Для пользователей, которые успешно вошли в [комнату](https://www.tencentcloud.com/document/product/647/37714) TRTC:

- Аудио длительность определяется как **длительность пребывания пользователя в комнате, когда видео потоки не получаются**, включая когда пользователь подписан только на аудио потоки, когда пользователь не подписан ни на какие потоки вообще, и когда пользователь только отправляет, но не получает аудио и видео в комнате.
- Если пользователь не подписан ни на один видео поток в комнате, плата за аудио длительность будет рассчитываться на основе времени пребывания пользователя в комнате.
- Пользователи могут входить и выходить из комнаты несколько раз. TRTC будет накапливать длительности нескольких аудио сегментов в реальном времени.
- Плата за аудио длительность рассчитывается на основе времени пребывания пользователя в комнате, независимо от того, является ли он единственным пользователем или публикует какие-либо потоки.

> **Примечание:** Длительность за каждый день рассчитывается в секундах для каждого приложения (SDKAppID), а затем преобразуется в минуты (округляется в большую сторону до ближайшей минуты) для тарификации. Когда в комнате есть поток общего доступа к экрану, общий доступ к экрану рассматривается как опубликованный виртуальным якорем, и плата за аудио/видео длительность будет рассчитываться отдельно.

## Примеры тарификации

### Пример 1

Предположим, что шесть пользователей вошли в комнату одновременно и общались по видео в течение 60 минут.

В комнате были три якоря (A, B и C). Разрешение видео A составляло 960 x 720, а B и C — 640 x 480. Двое из зрителей подписались на видео потоки всех якорей, а третий подписался только на аудио. Якорь A также делился экраном с разрешением 1920 x 1080.

Таблица ниже показывает совокупное разрешение видео, полученного каждым пользователем, категории, в которые это разрешение входит, и их цены.

| Пользователь | Полученные потоки | Разрешения видео | Совокупное разрешение | Категория | Длительность (минуты) |
| --- | --- | --- | --- | --- | --- |
| A (делился экраном) | Видео потоки B и C | 640 x 480 x 2 | 614,400 | HD | 60 |
| B | Видео и поток общего доступа экрана A + видео поток C | (960 x 720) + (640 x 480) + (1920 x 1080) | 3,072,000 | 2K | 60 |
| C | Видео и поток общего доступа экрана A + видео поток B | (960 x 720) + (640 x 480) + (1920 x 1080) | 3,072,000 | 2K | 60 |
| Зритель 1 | Видео и поток общего доступа экрана A + видео потоки B и C | (960 x 720) + (640 x 480 x 2) + (1920 x 1080) | 3,379,200 | 2K | 60 |
| Зритель 2 | Видео и поток общего доступа экрана A + видео потоки B и C | (960 x 720) + (640 x 480 x 2) + (1920 x 1080) | 3,379,200 | 2K | 60 |
| Зритель 3 | Только аудио потоки | - | - | Аудио | 60 |

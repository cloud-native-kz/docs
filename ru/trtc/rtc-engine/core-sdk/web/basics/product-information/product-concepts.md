# Концепции продукта

Этот документ объясняет некоторые базовые концепции, которые вы можете встретить при использовании сервисов TRTC.

### Приложение

В TRTC бизнес и проекты управляются с помощью **приложений**. Вы можете создавать различные приложения для своих проектов в [консоли TRTC](https://console.tencentcloud.com/trtc/app) для разделения их данных. Каждый аккаунт Tencent Cloud может создать до 100 приложений TRTC.

### SDKAppID

Tencent Cloud использует `SDKAppID` (ID приложения) для уникальной идентификации приложений TRTC. Он генерируется автоматически при создании приложения в [консоли TRTC](https://console.tencentcloud.com/trtc/app). Приложения, которые не имеют одинакового `SDKAppID`, не могут обмениваться данными друг с другом.

### UserID

UserID (ID пользователя) уникально идентифицирует пользователя в приложении TRTC.

- `UserID` — это отображение учетных записей пользователей вашего проекта в Tencent Cloud. Обычно вы можете использовать имя пользователя в качестве `UserID`.
- `UserID` должен быть длиной предпочтительно 32 байта или меньше. Может содержать цифры, подчеркивания и буквы (чувствителен к регистру).

### Комната

Комната — это пространство, в котором пользователи могут получать друг у друга данные аудио и видео в реальном времени.

- Комнаты — это виртуальные пространства, которые TRTC использует для разделения одной группы пользователей от другой.
- Только пользователи в одной комнате могут получать друг у друга данные аудио и видео.
- Пользователь может находиться только в одной комнате одновременно. Пользователь, который уже находится в комнате, должен выйти из нее, прежде чем он сможет войти в другую.

> **Уведомление**Первый пользователь, который входит в комнату, является владельцем комнаты. Владельцы комнат не могут закрывать комнаты вручную. В режимах вызовов TRTC закрывает комнату, когда все пользователи выходят из нее. В режимах прямого вещания, если последний пользователь, который выходит из комнаты, является вещателем, TRTC немедленно закроет комнату; если пользователь является зрителем, TRTC закроет комнату через 10 минут. Пользователь удаляется из комнаты через 90 секунд после неожиданного отключения. Если все пользователи отключены, комната закроется через 90 секунд. **Время ожидания после отключения также подлежит оплате**. Если пользователь пытается войти в комнату, которая не существует, TRTC автоматически создает комнату.

### ID комнаты

`RoomID` (номер/ID комнаты) уникально идентифицирует комнату в приложении TRTC. Это число UInt32, которое вы назначаете комнате и сохраняете. Его диапазон значений: 1-4294967295.

### UserSig

`UserSig` (подпись пользователя) — это подпись безопасности, разработанная Tencent Cloud для аутентификации входов пользователей и проверки того, что пользователь является реальным. Это помогает предотвратить доступ злоумышленников к вашему аккаунту Tencent Cloud. Для получения дополнительной информации см. [FAQ > UserSig](https://intl.cloud.tencent.com/document/product/647/35166).

### Передача

Передача — это операция, в которой пользователь загружает локальные данные аудио/видео на сервер TRTC.

### Подписка

Подписка — это операция, в которой пользователь отправляет запрос на сервер TRTC для получения данных аудио/видео указанного пользователя.

### Роль

В TRTC пользователи могут иметь одну из двух ролей: **вещатель** (`TRTCRoleAnchor`) и **зритель** (`TRTCRoleAudience`).

- Вещатель может передавать локальные данные аудио/видео на сервер. Они также могут подписываться на данные аудио/видео других вещателей и воспроизводить их.
- Зрители **могут только** подписываться на данные аудио/видео вещателей и воспроизводить их.

В режимах вызовов все пользователи в комнате имеют роль вещателя. В режимах прямого вещания пользователи в комнате могут быть либо вещателями, либо зрителями. Пользователь может переключать роли по мере необходимости.

### CDN Live Watching

CDN Live Watching также известен как CDN релейное прямое вещание. TRTC использует кластеры трансляции с переносом для преобразования своих потоков аудио/видео UDP в потоки RTMP в облаке. Затем потоки передаются в стандартную систему прямого вещания и распределяются через CDN аудитории. Для получения подробной информации см. [Relay to CDN](https://www.tencentcloud.com/document/product/647/47858#).

### Облачная запись

Функция облачной записи TRTC позволяет записывать поток аудио/видео в реальном времени с помощью RESTful API. Это гибко, легко и просто в использовании, избавляя вас от необходимости развертывания серверов и модулей записи. Для получения подробной информации см. [On-Cloud Recording](https://www.tencentcloud.com/document/product/647/45169#).

### On-Cloud MixTranscoding

В сценариях, таких как **CDN Live Watching** и **облачная запись**, вам может потребоваться смешивание нескольких потоков аудио/видео в комнате TRTC в один поток. Это может быть достигнуто с помощью кластера MCU смешивания и трансляции потоков TRTC. Кластер MCU может смешивать несколько потоков аудио/видео по мере необходимости и распределять смешанный поток на CDN прямого вещания и систему облачной записи. Для получения подробной информации см. [Relay to CDN](https://www.tencentcloud.com/document/product/647/47858#).

### Простой терминал

Пользователь, входящий в комнату через простой терминал, не будет обнаружен SDK, и удаленные пользователи не будут получать уведомления о входе или выходе пользователя из комнаты.

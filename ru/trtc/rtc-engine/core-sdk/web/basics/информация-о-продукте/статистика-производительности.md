# Статистика производительности

Этот документ анализирует производительность TRTC с точки зрения качества аудио/видео, задержки, плавности, стабильности, использования CPU, использования памяти, расхода батареи, нагрева и других ключевых показателей в тестах при **нормальных** и **плохих** сетевых условиях и в **различных сценариях приложений** (один-к-одному, один-ко-многим и т. д.).

- **Сценарий**: один-к-одному, один-к-двум, один-к-четырем, один-к-восьми

## Приложение 1: Сетевые метрики

| Метрика | Описание | Пример |
| --- | --- | --- |
| Loss | Процент потери пакетов | 50%: из каждых 10 отправленных пакетов теряется 5. |
| Delay | Сетевая задержка | 200: пакеты данных доставляются сетью с задержкой в 200 мс после отправки SDK. |
| Jitter | Сетевой джиттер | 300: отправка пакета может быть задержана на 20 мс, 50 мс, 250 мс, 280 мс или на любое значение до 300 мс. Средняя задержка составляет 150 мс. |

## Приложение 2: Метрики производительности при плохих сетевых условиях

| Метрика производительности | Описание |
| --- | --- |
| MOS | Важный показатель качества звука в системах телекоммуникаций. MOS генерируется Spirent Nomad с использованием стандарта [POLQA](#POLQA). Чем выше оценка, тем выше качество звука. |
| Сквозная задержка | Время от момента захвата звука на стороне отправителя до момента его воспроизведения на стороне получателя |
| Тест устойчивости к плохой сети | Spirent Nomad используется для оценки SDK при различных плохих сетевых условиях с использованием стандарта [POLQA](#POLQA). Видеопоследовательности Foreman используются для отправки данных, и интервалы кадров отслеживаются на стороне получателя. Данные собираются в 30 точках в течение 10 мин или дольше. Если есть заметные аномалии продолжительностью 3 мин в более чем 3 точках данных или SDK недоступен в течение относительно длительного периода времени, SDK считается неустойчивым к сетевым условиям. |

> **Примечание** Perceptual Objective Listening Quality Analysis (POLQA) — это стандарт ITU-T P.863. Это глобально применимый стандарт, используемый для оценки качества речи при различных сетевых условиях.

## Приложение 3: Показатели производительности SDK

| Показатель | Описание |  |
| --- | --- | --- |
| Использование CPU приложением | Android | Ненормализованное использование CPU приложением, которое совпадает с результатами, полученными Android Studio Profiler |
|  | iOS | Использование CPU приложением, которое совпадает с результатами, полученными XcodePerfDog usage = Xcode usage / Количество ядер |
| Использование CPU системой | Android | Ненормализованное использование CPU устройством, которое совпадает с результатами, полученными Android Studio Profiler |
|  | iOS | Использование CPU устройством, которое совпадает с результатами, полученными XcodePerfDog usage = Xcode usage / количество ядер |
| Использование памяти | Android | Proportional Set Size (PSS), которое совпадает с результатами, полученными Android Java API и Meminfo |
|  | iOS | Память Xcode, полученная через debug gauges |
| Расход батареи | Уменьшение процента заряда батареи за 30 мин (расчет начинается в момент, когда процент заряда батареи падает со 100% на 99%.) |  |
| Увеличение нагрева | Температура измеряется термометром, когда приложение не запущено. Затем приложение запускается на 30 мин при различных сценариях. Увеличение нагрева = Температура через 30 мин − Температура при неработающем приложении |  |

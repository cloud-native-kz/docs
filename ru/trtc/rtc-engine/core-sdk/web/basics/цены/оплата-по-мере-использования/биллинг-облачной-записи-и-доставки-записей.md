# Биллинг облачной записи и доставки записей

Функция записи TRTC состоит из трёх компонентов: [Облачная запись](https://www.tencentcloud.com/document/product/647/45176#On-CloudRecording), которая включает запись аудио- и видеопотоков в комнате TRTC в облаке, [Запись веб-страниц](https://www.tencentcloud.com/document/product/647/45176#402d1803-472e-44f5-8744-9b180ccffb66), которая объединяет содержимое страницы и звук из любого браузера, и [Доставка записей](https://www.tencentcloud.com/document/product/647/45176#RecordingDelivery), которая доставляет файлы записей в указанное хранилище.

Если у вас есть договор с TRTC, применяются сведения о биллинге, указанные в контракте.

## Облачная запись

### Структура биллинга

При использовании облачной записи могут взиматься два типа платежей:

#### **Платежи за подписку робота записи**

Для запуска задачи записи в комнате TRTC система назначает роботу записи, который действует как виртуальный зритель и подписывается на аудио- и видеопотоки, которые необходимо записать. Платежи рассчитываются на основе продолжительности аудио- и видеопотоков, полученных роботом. Подробнее см. в [Биллинг продолжительности аудио и видео](https://www.tencentcloud.com/document/product/647/42734).

#### **Платежи за облачную запись**

После того как робот записи подписывается на потоки, начинается процесс записи. Платежи за запись рассчитываются на основе продолжительности записи, а цена варьируется в зависимости от совокупного разрешения видео.

### Подробнее о биллинге

- Способ биллинга: Использование сначала будет вычтено из [10 000 бесплатных минут](https://www.tencentcloud.com/document/product/647/42735), предоставляемых TRTC для каждой учётной записи разработчика в месяц. Использование сверх ежемесячного количества бесплатных минут будет взиматься ежедневно по схеме оплаты по мере использования, при условии что ваша учётная запись не приостановлена.
- Цикл биллинга: Биллинг производится ежедневно. Платежи за облачную запись, произведённые в каждый день, будут вычтены в следующий день в момент биллинга. Точное время биллинга и выставления счётов будет зависеть от фактической [Выписки по счёту](https://console.tencentcloud.com/expense/bill/overview).
- В рамках одной задачи облачной записи при создании файлов записей в нескольких форматах для каждого формата будут взиматься отдельные платежи за продолжительность записи.

### Формула биллинга

**Платежи за облачную запись** = **Продолжительность аудио** x **Единица цены продолжительности аудио** + **Продолжительность видео разных типов** x **Единица цены соответствующего типа**

### Ценовая политика биллинга

Использование видеозаписи разделяется на следующие четыре типа в зависимости от совокупного разрешения каждого процесса записи. Платежи за каждый тип рассчитываются отдельно.

| Категория | Тип использования видео | Диапазон совокупного разрешения | Единица цены (USD/1000 мин) |
| --- | --- | --- | --- |
| Запись аудио | - | - | 1.49 |
| Запись видео | HD | Совокупное разрешение ≤ 921 600 (1280 x 720) | 5.99 |
|  | FHD | 921 600 (1280 x 720) < Совокупное разрешение ≤ 2 073 600 (1920 x 1080) | 13.49 |
|  | 2K | 2 073 600 (1920 x 1080) < Совокупное разрешение ≤ 3 686 400 (2560 x 1440) | 23.99 |
|  | 2K+ | 3 686 400 (2560 x 1440) < Совокупное разрешение ≤ 8 847 360 (4096 x 2160) | 53.99 |

### Расчёт использования

> **Примечание:** Продолжительность рассчитывается в секундах, а затем по дням конвертируется в минуты (с округлением вверх до ближайшей минуты). Если записывается только один поток, и этот поток имеет сегмент только с аудио, за которым следует видеосегмент, сегмент только с аудио будет взиматься как запись аудио, а видеосегмент — как запись видео. Если сегмент продолжительности записи включает как потоки только с аудио, так и видеопотоки, он будет взиматься только как запись видео. Процесс записи — это задача записи, которая запускается и останавливается с помощью API.

- Продолжительность записи: Время от начала процесса записи до его остановки.
  - Использование видеозаписи: Общая продолжительность сегментов, в которых видео записывается во всех процессах записи в комнате.
  - Использование записи аудио: Сумма продолжительности записи всех процессов записи в комнате минус продолжительность видеозаписи. Если сегмент продолжительности записи не содержит ни видео, ни аудио, он по-прежнему будет взиматься как использование записи аудио.
- Факторы, влияющие на расчёт продолжительности
  - Расчёт продолжительности зависит от количества процессов записи. Если в комнате запускается несколько процессов записи, продолжительность каждого процесса необходимо рассчитать и включить в общее использование услуги записи за этот месяц.
- Расчёт продолжительности не зависит от количества записываемых аудио- и видеопотоков. Продолжительность потоков, записываемых одним и тем же процессом записи, не будет суммироваться. Например, если процесс записи одновременно записывает два видеопотока в течение 5 минут, биллируемая продолжительность записи составит 5 минут, и платежи будут рассчитаны на основе совокупного разрешения двух потоков.
- Во время процесса записи робот записи подписывается на аудио- и видеопотоки, которые необходимо записать, что влечёт платежи за продолжительность аудио/видео TRTC.

### Примеры биллинга

В этом разделе описывается, как TRTC рассчитывает совокупное разрешение, продолжительность записи и платежи за запись.

#### Пример 1

Если процесс записи одновременно записывает два видеопотока с разрешением 960 x 720, совокупное разрешение этого процесса записи составило бы 960 x 720 + 960 x 720 = 1 382 400. Процесс записи был бы выставлен счётом как использование видеозаписи FHD.

#### Пример 2

Предположим, что у вас есть проект testRTC в вашей учётной записи Tencent Cloud. Помимо использования услуг аудио/видео, проект также записывает и сохраняет взаимодействия аудио/видео.

**Запись 1**

11 февраля 2022 года четыре пользователя провели видеозвонок продолжительностью 5000 секунд. Был запущен один процесс облачной записи для записи четырёх аудиопотоков.

Использование: Всего 5000 секунд продолжительности записи аудио.

**Запись 2**

12 февраля 2022 года четыре пользователя провели видеозвонок продолжительностью 5000 секунд. Были запущены два процесса облачной записи. Один был одиночной записью потока, а другой — смешанной записью потока. Четыре аудиопотока были записаны в каждом процессе.

Использование: Всего 5000 + 5000 = 10 000 секунд продолжительности записи аудио.

**Запись 3**

13 февраля 2022 года четыре пользователя провели видеозвонок продолжительностью 3500 секунд. Был запущен один процесс облачной записи для записи четырёх аудио- и видеопотоков. Каждый видеопоток имел разрешение 640 x 360.

Использование: Всего 3500 секунд продолжительности видеозаписи HD.

| Полученные потоки | Расчёт разрешения | Совокупное разрешение | Тип продолжительности записи |
| --- | --- | --- | --- |
| Аудио/видеопотоки 4 авторов | 640 x 360 x 4 | 921 600 | HD |

**Запись 4**

14 февраля 2022 года три пользователя взаимодействовали по видео в течение 1800 секунд, и их разрешения видео составили 640 x 360, 1280 x 720 и 960 x 720 соответственно. Через 1800 секунд в комнату вошёл четвёртый пользователь и взаимодействовал в течение 540 секунд, а разрешение видео этого пользователя составило 1920 x 1080. Был запущен один процесс облачной записи для записи аудио- и видеопотоков всех пользователей на протяжении всего процесса.

Использование: 1800 секунд продолжительности видеозаписи FHD и 540 секунд продолжительности видеозаписи 2K+.

| Полученные потоки | Расчёт разрешения | Совокупное разрешение | Тип продолжительности записи |
| --- | --- | --- | --- |
| Аудио/видеопотоки 3 авторов | 640 x 360 + 1280 x 720 + 960 x 720 | 1 843 200 | FHD |
| Аудио/видеопотоки 4 авторов | 640 x 360 + 1280 x 720 + 960 x 720 + 1920 x 1080 | 3 916 800 | 2K+ |

**Расчёт платежей (общие платежи за четыре вышеуказанные записи)**

| Биллируемый элемент (тип записи) | Продолжительность (мин) | Платежи (USD) |
| --- | --- | --- |
| Аудио | 250 | (250/1000) x 1.49 = 0.3725 |
| HD | 59 | (59/1000) x 5.99 = 0.35341 |
| FHD | 30 | (30/1000) x 13.49 = 0.4047 |
| 2K+ | 9 | (9/1000) x 53.99 = 0.48591 |
| Итого | - | 1.61652 ≈ 1.62 |

## Запись веб-страниц

### Структура биллинга

Услуга записи страниц объединяет содержимое указанной веб-страницы URL с аудио и видео в один файл. Во время записи страницы выберите разрешение на выходе. Биллинг основывается на продолжительности выбранного разрешения на выходе файла.

### Подробнее о биллинге

Способ биллинга: Постоплата после ежедневного расчёта.

Цикл биллинга: Ежедневно. Платёж за запись страницы за каждый день будет вычтен при выдаче выписки по счёту на следующий день. Точное время биллинга и выставления счётов будет зависеть от фактической [Выписки по счёту](https://console.tencentcloud.com/expense/bill/overview).

### Формула биллинга

Платёж за запись страницы = Объём выходной продолжительности x Цена уровня разрешения видео на выходе

### Ценовая политика биллинга

Путём вызова функции [Запись веб-страниц](https://www.tencentcloud.com/document/product/647/72325) вы можете записать содержимое любой веб-страницы браузера со смешиванием аудио, а биллинг основывается на продолжительности итогового сгенерированного аудио- и видеофайла.

Прайс-лист выглядит следующим образом:

| Тип биллинга | Тип разрешения | Разрешение на выходе | Единица цены (USD/1000 мин) |
| --- | --- | --- | --- |
| Запись веб-страниц (разрешение на выходе) | HD | Не более 1280 x 720 (включительно) | 14 |
|  | Full HD | 1280 x 720 — 1920 x 1080 (включительно) | 28 |

### Расчёт использования

Продолжительность биллинга для записи страницы рассчитывается на основе выходной продолжительности записей в рамках одной учётной записи Tencent Cloud, используемой как статистика использования услуги записи.

> **Примечание:** Точность статистики продолжительности в секундах. Накопленные секунды конвертируются в количество минут для биллинга, при этом менее одной минуты округляется вверх до одной минуты. При инициации запроса задачи записи страницы отсчёт продолжительности записи начинается после завершения загрузки url. Этап загрузки url не включается в расчёт продолжительности.

### Пример биллинга

Предположим, пользователь запускает задачу записи страницы, выполняет запись страницы в течение 10 минут и устанавливает разрешение на выходе 1280 x 720 (HD).

Общая стоимость записи страницы, созданная внутри комнаты TRTC = Продолжительность записи страницы x Единица цены сверхвысокой чёткости для записи веб-страниц = 14 USD/1000 минут x (10 минут / 1000) = 0.14 USD.

## Доставка записей

Услуга доставки записей — это вспомогательная услуга, которая доставляет и сохраняет сгенерированные файлы записей в указанном бакете во время использования услуги облачной записи аудио/видео в реальном времени (TRTC).

> **Примечание:** Корректировка биллинга: Правило биллинга для функции доставки записей было скорректировано, начиная с 00:00 1 сентября 2025 года. После корректировки эта функция преобразована в бесплатную услугу, и платежи за эту услугу больше не взиматься. Обратите внимание, что платежи за доставку записей, сгенерированные до 00:00 1 сентября 2025 года, не могут быть возвращены. Спасибо за ваше понимание и поддержку.

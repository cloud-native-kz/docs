# Качество видео и аудио

## 1. Видео

### Как удалить черные полосы на видеоизображении TRTC?

- TRTCVideoFillMode_Fill: изображение заполняет весь экран, лишние части обрезаются. Изображение может отображаться не полностью.
- TRTCVideoFillMode_Fit: изображение растягивается как можно больше по длинной стороне, пустая область заполняется черными полосами. Изображение отображается полностью.

### Как исправить рывки при воспроизведении?

- Проверьте статистику отправки и получения с точки зрения получателя.
- Проверьте потерю пакетов при отправке и получении. Высокая потеря пакетов указывает на то, что рывки могут быть вызваны нестабильным сетевым соединением.
- Проверьте частоту кадров и использование ЦПУ. Как низкая частота кадров, так и высокое использование ЦПУ могут вызвать рывки.

### Как исправить видео низкого качества, размытое и пиксельное видео?

- С помощью облачной политики контроля качества обслуживания TRTC динамически регулирует битрейт и разрешение на основе условий сети. Он снижает битрейт в случае плохого сетевого соединения, что приводит к снижению разрешения.
- Проверьте, используется ли режим `VideoCall` или `Live` при входе в комнату. Поскольку режим `VideoCall` предназначен для вызовов и характеризуется низкой латентностью и плавностью, он имеет тенденцию жертвовать качеством видео ради плавности при плохом сетевом соединении. Мы рекомендуем использовать режим `Live` для сценариев приложений с высокими требованиями к качеству видео.

### Что мне делать, если изображения предпросмотра/воспроизведения повернуты?

  - Обновите TRTC SDK до последней версии.
  - Если вы используете специальные устройства, вы можете вызвать API поворота локального просмотра `setLocalViewRotation`, API поворота удаленного изображения `setRemoteViewRotation` и API поворота закодированного изображения `setVideoEncoderRotation` для регулировки поворота. Подробные инструкции см. в разделе [Поворот видео](https://intl.cloud.tencent.com/document/product/647/35154).
- **Если вы самостоятельно захватываете видеоданные**:
  - Обновите TRTC SDK до последней версии.
  - Проверьте исходный поворот захватываемого вами видео.
  - Отправьте видеоданные в TRTC SDK и проверьте, указан ли угол поворота для `TRTCCloudDef.TRTCVideoFrame`.
  - Если вы используете специальные устройства, вы можете вызвать API поворота локального просмотра `setLocalViewRotation`, API поворота удаленного изображения `setRemoteViewRotation` и API поворота закодированного изображения `setVideoEncoderRotation` для регулировки поворота. Подробные инструкции см. в разделе [Поворот видео](https://intl.cloud.tencent.com/document/product/647/35154).

### Каковы причины черного экрана при прямой трансляции?

- Проблема с метаданными. Например, метаданные содержат только информацию об аудиопотоке, но в фактических данных также есть видео, или в начале есть только аудио, но позже добавляется видео. В этом случае мы рекомендуем вам изменить метаданные исходного потока.
- Закодированные видеоданные содержат только кадры, такие как SEI, но нет информации об изображении. В результате нет изображений для декодирования, следовательно, черный экран. Это обычно происходит с пользовательскими видеоданными.

### Каковы причины размытого или зеленого экрана при прямой трансляции?

- Изменение метаданных. Большинство проигрывателей анализируют метаданные только перед декодированием для настройки параметров декодирования. В случае изменения видео, например, если разрешение изменяется, но проигрыватель не обновляет параметры декодирования, может возникнуть проблема с размытым или зеленым экраном. Лучшее решение этой проблемы — сохранять параметры кодирования неизменными во время прямой трансляции, чтобы метаданные не изменялись.
- Проблемы совместимости аппаратных кодировщиков/декодировщиков. Эта проблема обычно обнаруживается на устройствах Android. Аппаратные кодировщики/декодировщики некоторых устройств Android реализованы плохо и имеют низкую совместимость. В этом случае мы рекомендуем вам переключиться на программные кодировщики/декодировщики.
- Использование различных цветовых форматов для публикации и воспроизведения. Например, если для публикации используется NV12, а для воспроизведения используется I420, проблема размытого или зеленого экрана может возникнуть при декодировании. Чтобы решить эту проблему, убедитесь, что для публикации и воспроизведения используется одинаковый цветовой формат.

## 2. Аудио

### Что мне делать, если громкость звука низкая, когда я использую TXVodPlayer для воспроизведения видео во время вызова TRTC?

Вызовите API `setSystemVolumeTyp` для установки типа громкости системы, используемой во время вызова, на `TRTCSystemVolumeTypeMedia` (режим медиагромкости).

### Как выбрать тип громкости медиа или вызова?

- TRTCAudioVolumeTypeAuto (по умолчанию): тип громкости вызова используется, когда микрофон включен, и тип громкости медиа используется, когда микрофон выключен.
- TRTCAudioVolumeTypeVOIP: тип громкости вызова всегда используется.
- TRTCAudioVolumeTypeMedia: тип громкости медиа всегда используется.

### Что мне делать, если громкость звука низкая?

  - Проверьте, не установлена ли `volume` ниже 50 в API [setCurrentDeviceVolume](https://liteav.sdk.qcloud.com/doc/api/zh-cn/group__ITXDeviceManager__csharp.html#ae07615c5d80ac1f7f32a07c41654a5b0) для Windows и macOS или API [setAudioCaptureVolume](https://liteav.sdk.qcloud.com/doc/api/zh-cn/group__TRTCCloud__cplusplus.html#a8677a812326511ef92f963bbe049d42e) для всех платформ. Если это так, установите большее значение.
  - Проверьте, включена ли автоматическая регулировка усиления (AGC).
  - Проверьте, не стал ли причиной проблемы использование наушников Bluetooth.
- Если **громкость низка только для некоторых пользователей**, то это проблема нисходящего потока.
  - Проверьте, не установлена ли `volume` ниже 50 в API [setAudioPlayoutVolume](https://liteav.sdk.qcloud.com/doc/api/zh-cn/group__TRTCCloud__cplusplus.html#a338984f5503d59ae06d67f55bd8f0766) или [setCurrentDeviceVolume](https://liteav.sdk.qcloud.com/doc/api/zh-cn/group__TXDeviceManager__cplusplus.html#ae06c4a3c51e1a6e9db48219f996c0177). Если это так, установите большее значение.
  - На мобильных телефонах проверьте, был ли вызван API `setAudioRoute` для переключения на приемник для воспроизведения.

### Что мне делать, если аудио рывкает?

- В **Состояние устройства**, если использование ЦПУ приемником и отправителем превышает 90%, закройте другие процессы, работающие в фоне.
- Если наблюдается значительная потеря пакетов в восходящем и нисходящем потоках и значительные колебания RTT, это указывает на плохие условия сети. Перейдите на другую сеть.

### Почему я слышу эхо?

Проблема с эхом является частой, если участники звонка находятся близко друг к другу. Пожалуйста, обеспечьте определенное расстояние между участниками звонка при тестировании. Также проверьте, не отключили ли вы непреднамеренно подавление акустического эха (AEC).

### Что мне делать, если аудио низкого качества и громкость постоянно меняется?

Эта проблема возникает, если вы используете внешнюю звуковую карту и одновременно включаете мониторинг в наушниках. Это потому, что звуковые карты часто встроены с мониторингом в наушниках. Пожалуйста, отключите мониторинг в наушниках при использовании внешней звуковой карты.

### Что мне делать, если во время звонка слышно эхо или шум, или громкость звонка низкая?

- `echoCancellation`: подавление эха
- `noiseSuppression`: подавление шума
- `autoGainControl`: автоматическая регулировка усиления

Если вы используете API [TRTC.createStream](https://web.sdk.qcloud.com/trtc/webrtc/doc/zh-cn/TRTC.html#createStream) для захвата, вам не нужно устанавливать параметры 3A вручную. TRTC SDK включает 3A по умолчанию.

## 3. Прочее

### Как я могу контролировать статус сети и отображать силу сигнала в TRTC?

Вы можете использовать `onNetworkQuality()` для контроля текущего качества сети в восходящем и нисходящем потоках. Для отображения силы сигнала, например на Android, см. [TRTC-API-Example](https://github.com/LiteAVSDK/TRTC_Android/tree/main/TRTC-API-Example/Advanced/SpeedTest).

### Почему моя камера или микрофон заняты?

Когда вызывается API `exitRoom()`, выполняется логика, такая как освобождение устройств аудио/видео и кодеков. Освобождение устройств является асинхронной операцией. После освобождения SDK будет использовать обратный вызов `onExitRoom()` в `TRTCCloudListener` для уведомления верхнего уровня. Пожалуйста, дождитесь получения обратного вызова `onExitRoom()`, прежде чем вызывать `enterRoom()` снова или переключаться на другой SDK аудио/видео.

### Как узнать, успешно ли включена камера?

Если получен обратный вызов `onCameraDidReady`, камера готова.

### Как узнать, успешно ли включен микрофон?

Если получен обратный вызов `onMicDidReady`, микрофон готов.

### Что мне делать, если камера не включается?

- TRTC SDK поддерживает внешние камеры, которые необходимы, если вы используете телевизоры или приставки. Проверьте, правильно ли подключена внешняя камера к вашему устройству.

### Какие технические метрики использует TRTC?

> **уведомление** Следующее применимо к iOS, macOS, Android и Windows.

TRTC SDK предлагает обратный вызов `onStatistics (TRTCStatistics statics)`. Каждые 2 секунды обратный вызов возвращает статистику по техническим метрикам, включая `appCpu` (использование ЦПУ приложения), `systemCpu` (использование ЦПУ системой), `rtt` (латентность), `upLoss` (потеря пакетов в восходящем потоке), `downLoss` (потеря пакетов в нисходящем потоке) и статистику видео/аудио локального пользователя и удаленных пользователей. Подробности см. в [TRTCStatistics](https://liteav.sdk.qcloud.com/doc/api/zh-cn/group__TRTCStatisic__ios.html#interfaceTRTCStatistics).

# Функции

### Что такое Tencent Real-Time Communication?

Используя многолетний опыт Tencent в области сетевых технологий и аудио-видео коммуникаций, Tencent Real-Time Communication (TRTC) предлагает решения для групповых аудио-видео звонков и низколатентного интерактивного потокового вещания. С помощью TRTC вы можете быстро разработать экономичные, низколатентные и высококачественные приложения для интерактивной аудио-видео коммуникации с широким спектром функций. Дополнительные сведения см. в разделе [Введение в продукт > Обзор](https://www.tencentcloud.com/document/product/647/35078).

### Как я могу попробовать демо TRTC?

Вы можете перейти на страницу [Бесплатное демо](https://www.tencentcloud.com/document/product/647/35076), чтобы найти демонстрации TRTC для различных платформ.

### Как быстро начать работу с TRTC?

TRTC предоставляет исходный код демонстрационных приложений для различных платформ, который вы можете использовать для быстрого создания собственных приложений. Дополнительные сведения см. в разделе [Учебное пособие для пользователя](https://www.tencentcloud.com/document/product/647/39386).

### Что такое `RoomID` в TRTC? Каков его диапазон значений?

`RoomID` уникально идентифицирует комнату и может принимать значения от 1 до 4294967295. Вы несете ответственность за управление и назначение идентификаторов комнат вашего приложения.

### Что такое `UserID` в TRTC? Каков его диапазон значений?

`UserID` уникально идентифицирует пользователя в приложении TRTC. Может содержать буквы (с учетом регистра), цифры и подчеркивания, предпочтительно не более 32 байт.

### Каков жизненный цикл комнаты TRTC?

- **В режимах вызовов** TRTC закрывает комнату, когда все пользователи выходят из нее.
- **В режимах потокового вещания** если последний пользователь, который выходит из комнаты, является ведущим, TRTC немедленно закроет комнату; если пользователь является зрителем, TRTC закроет комнату через 10 минут.
- Пользователь удаляется из комнаты через 90 секунд после неожиданного отключения. Если все пользователи отключены, комната закрывается через 90 секунд. **Время ожидания после отключения также тарифицируется**.
- Если пользователь попытается войти в несуществующую комнату, TRTC автоматически создаст комнату с указанным идентификатором.

### Могут ли пользователи не подписываться на аудио-видео потоки?

Чтобы включить мгновенную трансляцию, TRTC по умолчанию подписывает пользователей на аудио-видео потоки при входе в комнату. Вы можете вызвать API `setDefaultStreamRecvMode` для переключения в режим ручной подписки.

### Как сделать так, чтобы пользователи не были автоматически подписаны на аудио-видео потоки при входе в комнату?

Да. Вы можете указать `streamId` через `TRTCParams` из `enterRoom` или вызвать API `startPublishing`, передав `streamId`.

### Какие роли поддерживаются во время потокового вещания в TRTC? Чем они отличаются друг от друга?

Сценарии потокового вещания (`TRTCAppSceneLIVE` и `TRTCAppSceneVoiceChatRoom`) поддерживают две роли: `TRTCRoleAnchor` (ведущий) и `TRTCRoleAudience` (зритель). Ведущий может как отправлять, так и получать аудио-видео данные, но зрители могут только получать и воспроизводить данные других. Вы можете вызвать `switchRole()` для переключения ролей.

### Какие сценарии приложений поддерживаются в комнатах TRTC?

- TRTCAppSceneVideoCall: Сценарий видео вызова подходит для видеозвонков один-на-один, видеоконференций с до 300 участников, онлайн консультаций, видео чатов и видео интервью.
- TRTCAppSceneLIVE: Сценарий интерактивного видео потокового вещания подходит для низколатентного потокового видео вещания, интерактивных классов для до 100 000 участников, видео соревнований, видео знакомств, удаленного обучения и мега-конференций.
- TRTCAppSceneAudioCall: Сценарий аудио вызова подходит для аудио звонков один-на-один, аудио конференций с до 300 участников, аудио чатов и онлайн вечеринок с играми.
- TRTCAppSceneVoiceChatRoom: Сценарий интерактивного аудио потокового вещания подходит для низколатентного аудио потокового вещания, интерактивного аудио с соведущим, аудио чат-комнат, караоке и радио.

Нет, это невозможно.

### Возможна ли коммуникация между TRTC и MLVB?

TRTC и MLVB имеют разные архитектуры бэкэнда и поэтому не могут взаимодействовать друг с другом. Однако вы можете транслировать потоки из TRTC на CDN.

### Чем отличаются различные режимы входа в комнату (`AppScene`) в TRTC друг от друга?

TRTC имеет четыре режима входа в комнату. Видеозвонок (`VideoCall`) и аудиозвонок (`VoiceCall`) — это режимы вызовов, а интерактивное видео потоковое вещание (`Live`) и интерактивное аудио потоковое вещание (`VoiceChatRoom`) — это режимы потокового вещания.

- Режимы вызовов позволяют максимум 300 пользователям в каждой комнате TRTC, и до 50 из них могут говорить одновременно. Режимы вызовов подходят для таких сценариев, как видеозвонки один-на-один, видеоконференции с до 300 участников, онлайн консультации, видео интервью, видео служба поддержки и онлайн игры на вечеринке.
- Режимы потокового вещания поддерживают максимум 100 000 одновременных пользователей в каждой комнате и позволяют плавное включение/отключение микрофона. Задержка совместного вещания поддерживается ниже 300 миллисекунд, а задержка просмотра ниже 1000 миллисекунд. Режимы потокового вещания подходят для таких сценариев, как низколатентное интерактивное потоковое вещание, интерактивные классы для до 100 000 участников, видео знакомства, онлайн образование, удаленное обучение и мега-конференции.

### Могу ли я использовать режим свободного общения во время видеовызовов в TRTC?

Да. Вы можете включить режим свободного общения, установив маршруты аудио. В собственном SDK используйте API `setAudioRoute` для переключения маршрутов.

### Поддерживает ли TRTC напоминания об уровне громкости?

Да. Вы можете вызвать API `enableAudioVolumeEvaluation` для включения напоминаний об уровне громкости.

### Поддерживает ли TRTC зеркальные изображения?

Да. Вы можете вызвать API `setLocalViewMirror` для установки режима зеркального отображения для предварительного просмотра локальной камеры или вызвать `setVideoEncoderMirror` для установки режима зеркального отображения для закодированных изображений.

### Могу ли я записать аудио вызова TRTC и сохранить файл записи локально?

Да. Вы можете вызвать `startAudioRecording` для записи всех аудио вызова, включая аудио локального пользователя, удаленных пользователей и фоновой музыки, в один файл в формате PCM, WAV или AAC.

### Могу ли я записать видео вызова TRTC в файл?

TRTC поддерживает аудио-видео запись на локальном сервере. Чтобы использовать эту функцию, пожалуйста [отправьте запрос](https://console.tencentcloud.com/workorder/category) для SDK и инструкций.
Вы также можете использовать функцию облачной записи для записи видео.

### Поддерживает ли TRTC плавающие окна (как в WeChat) или переключение между большим и малым окном?

Эти функции являются частью дизайна пользовательского интерфейса, для которого SDK TRTC не устанавливает ограничений. Наше официальное демонстрационное приложение предоставляет пример кода для наложения изображений и сеточного макета и поддерживает плавающие окна, переключение большого/малого окна и перетаскивание окна. Дополнительные сведения см. в [демонстрации TUICalling](https://github.com/tencentyun/TUICalling).

### Как я могу совершить только аудиозвонок в TRTC?

TRTC не использует отдельные каналы для аудио и видео. Вы можете совершить только аудиозвонок, вызвав только `startLocalAudio` и не вызывая `startLocalPreview`.

### Как включить трансляцию на CDN и запись для только аудиозвонка в TRTC?

- В SDK TRTC версии 6.9 или более поздней просто установите параметр сцены на `TRTCAppSceneAudioCall` или `TRTCAppSceneVoiceChatRoom` при входе в комнату.

### Могу ли я исключить пользователя, запретить пользователю говорить или отключить звук пользователя в комнате TRTC?

- Чтобы включить функции через простые операции сигнализации, используйте `sendCustomCmdMsg`, API пользовательской сигнализации TRTC, для определения собственной сигнализации управления, и пользователи, которые получат сообщение, выполнят ожидаемое действие. Например, чтобы исключить пользователя, просто определите сигнализацию исключения, и пользователь, который ее получит, выйдет из комнаты.
- Если вы хотите реализовать более комплексную логику операций, мы рекомендуем использовать [Tencent Cloud Chat](https://www.tencentcloud.com/document/product/1047) для соответствия комнаты TRTC группе Chat и включение функций через отправку/получение пользовательских сообщений в группе.

### Может ли TRTC захватывать и воспроизводить потоки RTMP/FLV?

Да. SDK TRTC интегрировал TXLivePlayer. Если вам нужны больше функций плеера, рассмотрите использование полнофункционального LiteAVSDK_Professional.

### Сколько человек может быть в вызове TRTC?

- В сценариях потокового вещания каждая комната может вместить до 100 000 одновременных пользователей, и до 50 из них могут быть назначены роль ведущего и одновременно включить камеры или микрофоны.

### Как мне начать сеанс потокового вещания в TRTC?

TRTC предлагает специализированное решение для низколатентного интерактивного потокового вещания, которое позволяет до 100 000 участников. Задержка совместного вещания поддерживается в пределах 200 миллисекунд, а задержка просмотра составляет менее одной секунды. Он отлично адаптируется к плохим условиям сети и оптимизирован для сложных мобильных сетевых сред.

Дополнительные сведения см. в разделе [Режим потокового вещания](https://intl.cloud.tencent.com/document/product/647/35107).

### Как я могу прослушивать отключение и переподключение сети в TRTC?

- `onConnectionLost`: SDK отключена от сервера.
- `onTryToReconnect`: SDK переподключается к серверу.
- `onConnectionRecovery`: SDK переподключена к серверу.

### Поддерживает ли SDK TRTC автоматическое переподключение?

SDK автоматически переподключает пользователя после отключения. Если ему не удается переподключить пользователя в течение 30 минут, он удалит пользователя из комнаты и вернет ошибку -3301. На рисунке ниже показаны обратные вызовы, которые триггер при входе `Userid1` в комнату, отключении от SDK и повторном входе в комнату:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/333bbc3f2c5211ee818e525400088f3a.png)

**Описание**:

- T1: Пользователь вызывает API `enterRoom` для входа в комнату.
- T2: `Userid1` получает обратный вызов `onEnterRoom`, и `Userid2` получает обратный вызов `onRemoteUserEnterRoom` примерно через 300 миллисекунд (задержка).
- T3: `Userid1` отключена из-за проблемы с сетью, и SDK пытается переподключить пользователя.
- T4: Если `Userid1` не переподключена в течение первых восьми секунд, пользователь получит обратный вызов `onConnectionLost`.
- T5: Если пройдет еще три секунды и `Userid1` по-прежнему не переподключена, пользователь получит обратный вызов `onTryToReconnect`.
- T6: `Userid1` затем получит обратный вызов `onTryToReconnect` каждые 24 секунды.
- T7: Через 90 секунд после обратного вызова `onConnectionLost`, `Userid2` получит обратный вызов `onRemoteUserLeaveRoom`, который указывает, что `Userid1` находится в автономном режиме.
- T8: Если переподключение происходит в любой момент в течение 90 секунд, `Userid1` получит обратный вызов `onConnectionRecovery`.

### Есть ли обратный вызов для рендеринга первого кадра? Могу ли я прослушивать начало рендеринга изображения или воспроизведение аудио?

Да. Вы можете использовать `onFirstVideoFrame` и `onFirstAudioFrame` для прослушивания событий.

### Могу ли я сделать скриншот видео в TRTC?

В настоящее время вы можете вызвать `snapshotVideo()` на iOS и Android для создания скриншотов локальных и удаленных видео.

### Почему я не могу подключить периферийные устройства, такие как беспроводные наушники, к TRTC?

В настоящее время TRTC поддерживает основные беспроводные наушники и периферийные устройства, но для некоторых устройств все еще есть проблемы совместимости. Мы рекомендуем использовать наши официальные демонстрационные приложения и аудио-видео звонки QQ для тестирования совместимости устройства.

### Как мне получить информацию, такую как битрейт восходящей/нисходящей линии, разрешение, процент потери пакетов и частота дискретизации аудио вызова аудио-видео TRTC?

Вы можете вызвать API `onStatistics()` SDK для получения статистики.

### Поддерживает ли API фоновой музыки TRTC `playBGM()` онлайн музыку?

Нет. В настоящее время он поддерживает только локальную музыку. Вы можете загрузить файл онлайн музыки и затем вызвать `playBGM()` для его воспроизведения.

### Могу ли я установить громкость локального захвата аудио или громкость воспроизведения каждого удаленного пользователя?

Да. Вы можете вызвать `setAudioCaptureVolume()` для установки громкости захвата аудио SDK и `setRemoteAudioVolume()` для установки громкости воспроизведения удаленного пользователя.

### В чем отличия между `stopLocalPreview` и `muteLocalVideo`?

- `muteLocalVideo` используется для остановки отправки локальных видеоизображений. Если вы вызовете этот API, другие пользователи не будут видеть ваше изображение, но вы все равно сможете предварительно просмотреть ваше собственное изображение.

### В чем отличия между `stopLocalAudio` и `muteLocalAudio`?

- Когда вызывается `muteLocalAudio`, TRTC не прекращает отправку аудио-видео данных. Она продолжает отправлять отключенные пакеты данных с чрезвычайно низким битрейтом.

### Как мне установить битрейт восходящей видео, разрешение и частоту кадров в SDK TRTC?

Вызовите API `setVideoEncoderParam()` из `TRTCCloud` и установите `videoResolution` (разрешение), `videoFps` (частота кадров) и `videoBitrate` (битрейт) в `TRTCVideoEncParam`.

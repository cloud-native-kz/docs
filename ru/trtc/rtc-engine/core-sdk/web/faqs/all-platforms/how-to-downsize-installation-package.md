# Как уменьшить размер пакета установки

### Насколько увеличится размер файла после интеграции TRTC SDK?

Увеличение размера файла зависит от версии TRTC SDK. Дополнительную информацию см. в разделе [SDK Download](https://intl.cloud.tencent.com/document/product/647/34615).

### Как уменьшить размер пакета установки для iOS?

Способ 1. Упаковать только архитектуру ARM64 (рекомендуется)

Способ 2. Включить Bitcode

Для моделей iPhone после 5s можно упаковать только архитектуру x64, установив в Xcode в разделе "Build Settings" параметр "Build Active Architecture Only" на "Yes" и добавив только

`arm64`

в "Valid Architectures". Прирост размера IPA с одной архитектурой для TRTC SDK составит всего 1,9 МБ.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/126ccefa388d11edb1de525400c56988.png)

Для iPhone 5s и более старых моделей,

**если все сторонние библиотеки в проекте поддерживают Bitcode,**

вы можете включить Bitcode для уменьшения размера пакета установки. Включите переключатель "Enable Bitcode" в "Build Settings" > "Build Options", чтобы активировать bitcode.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/126835ea388d11edb1de525400c56988.png)

Начиная с 2016 года Apple начала поддерживать компиляцию Bitcode в среде разработки Xcode. После включения Bitcode компилятор будет генерировать промежуточный код приложения вместо фактического кода ассемблера, а пользователи будут загружать и устанавливать машинный код, сгенерированный для конкретной архитектуры CPU мобильного устройства из App Store, что значительно уменьшает размер пакета установки.

### Как уменьшить размер пакета установки для Android?

Способ 1. Упаковать только определённые файлы .so

Способ 2. Упаковать только JAR файлы (т.е. файлы .so будут загружены после установки)

Если ваше приложение используется только в материковой части Китая, вы можете упаковать только файлы .so для архитектуры `armeabi-v7a`, чтобы сократить прирост размера пакета установки до менее 5 МБ. Если вы хотите предложить своё приложение в Google Play, вы можете упаковать файлы .so для архитектур `armeabi-v7a` и `arm64-v8a`.
**Инструкции:** добавьте `abiFilters "armeabi-v7a"` в `build.gradle` текущего проекта, чтобы указать упаковку файлов .so только для одной архитектуры, или добавьте `abiFilters "armeabi-v7a","arm64-v8a"` для упаковки файлов .so для двух архитектур.

- Если упакованы только файлы .so архитектуры `armeabi-v7a` (т.е. ваше приложение не предлагается в Google Play):
  ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/126a91f1388d11ed8088525400463ef7.png)
- Если упакованы файлы .so архитектур `armeabi-v7a` и `arm64-v8a` (т.е. ваше приложение предлагается в Google Play):
  ![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/125e8d2b388d11ed8088525400463ef7.png)

> **примечание** Если вы хотите предложить своё приложение в Google Play, пожалуйста, не используйте этот метод, так как это может привести к невозможности предложить приложение.

Размер файлов .so занимает наибольшую часть от общего размера SDK для Android. Если вы хотите сократить размер пакета установки до менее 1 МБ, можно использовать метод загрузки файлов .so после установки:

2. Загрузите файлы .so, загруженные на [шаге 1](#step1), на ваш сервер (или в Tencent Cloud [COS](https://intl.cloud.tencent.com/product/cos)) и запишите адрес загрузки, такой как `http://xxx.com/so_files.zip`.
3. Перед запуском функции SDK, такой как воспроизведение видео, используйте анимацию загрузки, чтобы уведомить пользователя о загрузке модуля соответствующей функции.
  Пока пользователь ждёт, приложение может загрузить файлы .so с адреса `http://xxx.com/so_files.zip` и сохранить файлы в каталог приложения (например, в папку `files` в корневом каталоге приложения). Чтобы убедиться, что этот процесс не затронут перехватом ISP DNS, пожалуйста, проверьте целостность файлов .so после загрузки и убедитесь, что zip-пакет не был изменён ISP.
4. После того как все файлы .so готовы, вызовите API `setLibraryPath()` в классе `TXLiveBase` (самый ранний основной модуль `LiteAVSDK`), чтобы установить целевые пути загруженных файлов .so на пути в SDK, чтобы SDK мог загружать требуемые файлы .so по этим путям и запускать соответствующие функции.

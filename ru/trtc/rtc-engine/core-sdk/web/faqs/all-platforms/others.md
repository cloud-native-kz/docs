# Прочее

### Чем отличаются и как связаны между собой прямая трансляция, интерактивная прямая трансляция, TRTC и транслирование с реле?

- **Интерактивная прямая трансляция** (ключевые слова: совместное якорение, кросс-комнатная коммуникация)
 В интерактивной прямой трансляции зрители могут совместно якориться с якорями, а якоря из разных комнат могут конкурировать друг с другом.
- **Двусторонняя коммуникация Tencent в реальном времени** (ключевые слова: многопользовательское взаимодействие, проприетарный протокол на основе UDP, низкая задержка)
 Основные возможности TRTC — это аудио/видео взаимодействие и низколатентная прямая трансляция. Она использует проприетарный протокол на основе UDP и может поддерживать задержку на уровне 100 мс. Типичные приложения включают вызовы QQ, VooV Meeting и онлайн-занятия в группах. TRTC предлагает решения для основных платформ, включая iOS, Android и Windows, обеспечивает совместимость с WebRTC и поддерживает микширование потоков в облаке и их трансляцию на CDN.
- **Транслирование с реле** (ключевые слова: микширование потоков в облаке, трансляция RTC с реле, CDN)
 Технология трансляции с реле реплицирует несколько потоков в низколатентной комнате совместного якорения и микширует их в один поток в облаке перед отправкой на CDN прямой трансляции для распространения и воспроизведения.

### Как просмотреть логи TRTC?

- iOS/macOS: `Documents/log` в изолированной папке приложения
- Android:
  - v6.7 и более ранние версии: `/sdcard/log/tencent/liteav`
  - v6.8-8.5: `/sdcard/Android/data/имя пакета/files/log/tencent/liteav/`
  - Более поздние, чем v8.5: `/sdcard/Android/data/имя пакета/files/log/liteav/`
- Windows:
  - Более ранние, чем v8.8: `%appdata%/tencent/liteav/log`
  - v8.8 и более поздние версии: `%appdata%/liteav/log`
- Web: откройте консоль браузера или используйте vConsole для логирования информации SDK.

> **Примечание** Для просмотра файла XLOG необходимо скачать инструмент расшифровки. Поместите инструмент в ту же папку, что и файл XLOG, в Python 2.7 и выполните `python decode_mars_log_file.py`. Инструмент расшифровки логов можно скачать на `dldir1.qq.com/hudongzhibo/log_tool/decode_mars_log_file.py`.

### Почему при входе в комнату возвращается код ошибки -100018?

- Переданный параметр `SDKAppID` неправильный. Можно войти в консоль TRTC и нажать [Application Management](https://console.tencentcloud.com/trtc/app) для просмотра `SDKAppID`.
- Переданный `UserSig`, который должен соответствовать `UserID`, неправильный. Для проверки вашего `UserSig` войдите в консоль TRTC и нажмите **Development Assistance** > [UserSig Generation & Verification](https://console.tencentcloud.com/trtc/usersigtool).

### Как осуществить кросс-комнатный вызов?

Можно осуществить кросс-комнатный вызов, используя API `connectOtherRoom`. Якорь A вызывает `connectOtherRoom()` для подключения к якорю B и получает результат через callback `onConnectOtherRoom`. Все пользователи в комнате якоря A будут уведомлены через callback `onUserEnter` о том, что якорь B вошел в комнату, и все пользователи в комнате якоря B будут уведомлены через callback `onUserEnter` о том, что якорь A вошел в комнату.

### Можно ли вручную включить запись для вызова?

1. В консоли TRTC нажмите [Application Management](https://console.tencentcloud.com/trtc/app) > **Function Configuration**, включите **Relay to CDN** и отключите **On-Cloud Recording**.
2. После входа пользователя (`userid`) в комнату составьте `streamid` пользователя согласно правилу генерации ID потока.
3. Используйте API [CreateRecordTask](https://intl.cloud.tencent.com/document/product/267/37309) CSS для запуска задачи записи для `streamid`.
- Установите `DomainName` на `[bizid].livepush.myqcloud.com`.
- Установите `AppName` на `trtc_[sdkappid]`.
- Установите `StreamName` на `streamid`.
4. После завершения задачи записи CSS сохранит файл в VOD и уведомит вас через [callback записи](https://intl.cloud.tencent.com/document/product/267/38080).

### Как вести список пользователей и подсчитать количество пользователей в комнате TRTC?

Если вы интегрировали [Chat](https://intl.cloud.tencent.com/document/product/1047) в свой проект, вы можете использовать API подсчета пользователей в группе Chat для расчета количества пользователей в комнате. Однако такой расчет не всегда точен. Вы можете использовать этот метод, если у вас нет высоких требований к точности.
Если у вас есть высокие требования к точности расчета, мы рекомендуем реализовать следующую логику расчета:

1. Увеличить количество пользователей (клиент -> сервер): каждый раз, когда пользователь входит в комнату, увеличьте количество пользователей на 1. Для этого вы можете заставить клиент пользователя отправить запрос на увеличение количества серверу при входе в комнату.
2. Уменьшить количество пользователей (клиент -> сервер): когда пользователь выходит из комнаты, уменьшьте количество пользователей на 1. Для этого вы можете заставить клиент пользователя отправить запрос на уменьшение количества серверу при выходе из комнаты.

### Как реализовать функцию, которая позволяет якорям приглашать членов аудитории совместно якориться в сценариях конференций в Windows?

Необходимо использовать другой продукт Tencent Cloud, [Chat](https://intl.cloud.tencent.com/document/product/1047/33996), для реализации этой функции.

Вот как это работает: A отправляет пользовательское сообщение X (вы можете определить способ отображения сообщения) пользователю B, и отображается страница вызова; B получает X и отображается вызванная страница; B использует [enterRoom](https://liteav.sdk.qcloud.com/doc/api/zh-cn/group__TRTCCloud__cplusplus.html#a0fab3ea6c23c6267112bd1c0b64aa50b) для входа в комнату и отправляет пользовательское сообщение X1 пользователю A; A получает X1 (вы можете определить, отображать ли сообщение) и использует `enterRoom` для входа в комнату. Сообщения отправляются через Chat.

### Как зрители смотрят видео совместных якорей в комнате?

В сценариях прямой трансляции зрители получают `userid` якорей в комнате через callback onUserVideoAvailable в `TRTCCloudDelegate` (пользователи совместного якорения входят в комнату, вызывая [enterRoom](https://liteav.sdk.qcloud.com/doc/api/zh-cn/group__TRTCCloud__cplusplus.html#a0fab3ea6c23c6267112bd1c0b64aa50b), и также являются якорями для аудитории). Затем они вызывают [startRemoteView](https://liteav.sdk.qcloud.com/doc/api/zh-cn/group__TRTCCloud__cplusplus.html#a5c5ea936418b106c2e801db57938dde9) для воспроизведения видео якорей.
Для получения дополнительной информации см. [Live Streaming Mode > Windows](https://intl.cloud.tencent.com/document/product/647/35109).

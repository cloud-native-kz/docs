# Обработка ограничения автозапуска

## Введение

Для предотвращения автоматического воспроизведения аудио и видео на веб-страницах и создания помех для пользователей браузеры ввели ограничения на автоматическое воспроизведение: **до взаимодействия пользователя с веб-страницей (например, щелчка, касания и т. д.) воспроизведение медиа со звуком запрещено.** **В iOS 10+ при включённом режиме низкого энергопотребления также запрещено воспроизведение медиа без звука.**

Из-за ограничений политики автозапуска браузера пользователи могут не услышать звук при воспроизведении аудио. Наиболее распространённый сценарий, когда это происходит — пользователи входят в комнату и автоматически запускают удалённое аудио без взаимодействия с пользователем (например, щелчка) после обновления страницы.

В этом руководстве основное внимание уделяется **решению проблемы отказа при воспроизведении, вызванной ограничением политики автозапуска**.

## Решения

### 1. Использование всплывающего окна автозапуска SDK

По умолчанию при сбое автозапуска SDK отображает всплывающее окно, побуждающее пользователя взаимодействовать со страницей. После взаимодействия SDK автоматически вызывает интерфейс для возобновления воспроизведения.

- **Преимущество**: бизнес-сторона не должна ничего делать, просто и эффективно.
- **Недостаток**: всплывающее окно, предоставляемое SDK, может не соответствовать требованиям дизайна вашего продукта. В этом случае рассмотрите использование Решения 2.

Всплывающее окно, предоставляемое SDK, адаптировано для настольного и мобильного браузера со следующим стилем:

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/f2b91ca2e51611eeadff525400e97a5f.png)

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/f60fa586e51611ee9f745254008eb8a8.png)

- SDK переключает английские и китайские подсказки во всплывающем окне в соответствии со значением [navigator.language](https://developer.mozilla.org/en-US/docs/Web/API/Navigator/language).
- Когда пользователь нажимает «OK», SDK автоматически вызывает соответствующий интерфейс для возобновления воспроизведения.
- Чтобы обеспечить отображение всплывающего окна на верхнем слое, его значение z-index установлено на 1500.

### 2. Обработка всплывающего окна со стороны бизнеса

Если Решение 1 не соответствует требованиям дизайна вашего продукта, вы можете решить проблему автозапуска следующим образом.

2.1 Установите параметр `enableAutoPlayDialog` в интерфейсе [trtc.enterRoom](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#enterRoom) в значение `false`, чтобы отключить всплывающее окно SDK.

2.2 Прослушивайте событие [AUTOPLAY_FAILED](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.AUTOPLAY_FAILED) и побуждайте пользователя щёлкнуть на странице при возникновении события. SDK автоматически возобновит воспроизведение аудио после щелчка пользователя.

2.3 В дизайне продукта рекомендуется побуждать пользователей взаимодействовать со страницей перед входом в комнату (например, щелкнув кнопку входа в комнату), что может избежать ошибки автозапуска.

```
trtc.on(TRTC.EVENT.AUTOPLAY_FAILED, event => {  // Guide the user to click the page, you can pop up a prompt.});await trtc.enterRoom({ enableAutoPlayDialog: false });
```

Большинство браузеров снимают ограничение на автозапуск после взаимодействия пользователя, и этот шаг может эффективно избежать ошибки автозапуска. Однако из-за различий в реализации политик автозапуска различными производителями браузеров даже при побуждении пользователей взаимодействовать со страницей заранее могут всё ещё возникать ошибки автозапуска. Поэтому вы должны продолжить прослушивание события [AUTOPLAY_FAILED](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/module-EVENT.html#.AUTOPLAY_FAILED) и побудить пользователя щёлкнуть на странице.

Например: В браузере iOS WeChat и его webview мини-программы при воспроизведении удалённого потока в первый раз на странице **вы должны вызвать интерфейс воспроизведения в функции обратного вызова события щелчка пользователя**, в противном случае воспроизведение будет неудачным. Если вы используете `setTimeout` и другие интерфейсы для асинхронного вызова интерфейса воспроизведения в обратном вызове события щелчка пользователя, вы также столкнётесь с ошибками автозапуска.

## Отключение политики автозапуска в Webview

Политика автозапуска по умолчанию в Android и iOS Webview может отличаться от политики браузера. Вы можете обратиться к следующей документации, чтобы отключить ограничения автозапуска в вашем приложении.

- Android Webview: вызовите [setMediaPlaybackRequiresUserGesture(false)](https://developer.android.com/reference/android/webkit/WebSettings#setMediaPlaybackRequiresUserGesture(boolean)).
- iOS Webview: установите [mediaTypesRequiringUserActionForPlayback](https://developer.apple.com/documentation/webkit/wkwebviewconfiguration/1851524-mediatypesrequiringuseractionfor) в WKAudiovisualMediaTypeNone.

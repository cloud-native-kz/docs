# Оптимизация видеовызовов с несколькими участниками

В сценарии видеовызова с несколькими участниками по мере увеличения количества пользователей, передающих потоки, потребление пропускной способности и использование ресурсов устройства увеличиваются для пользователей, которые получают потоки. Если оптимизация не выполняется, качество работы на устройствах с низкими характеристиками значительно снижается.

В этом учебнике представлены лучшие практики для сценария видеовызова с несколькими участниками. Благодаря следующим двум решениям вы можете снизить потребление пропускной способности и обеспечить лучший опыт вызова.

- **Включение малого потока**
- **Включение "receiveWhenViewVisible"**

## Включение малого потока

Малый поток — это кодирование двух видеопотоков одновременно при включенной камере. Один — это видео с нормальным разрешением (называемое основным потоком), а другой — видео с низким разрешением (называемое малым потоком). На конце приёма вы можете выбрать получение основного потока или малого потока для сэкономления пропускной способности.

Применимые сценарии: Малый поток подходит для сценария видеовызова с несколькими участниками. Обычно в макете страницы видеовызова с несколькими участниками используется "одно большое представление и много малых представлений". Видео в большом представлении — это основной поток, а видео в малом представлении — это малый поток. Эта функция немного увеличит пропускную способность передачи, но может значительно сэкономить пропускную способность приёма.

### Передача малого потока издателем

При вызове [trtc.startLocalVideo](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#startLocalVideo) вы можете установить параметр **small** для включения кодирования малого потока.

```
await trtc.startLocalVideo({ option: { small: '120p' }});
```

Начиная с версии 5.3.0+, вы можете динамически включать и отключать малый поток.

```
await trtc.startLocalVideo();// включить малый поток
await trtc.updateLocalVideo({ option: { small: '120p' }});// отключить малый поток
await trtc.updateLocalVideo({ option: { small: false }});
```

> **Примечание:** В среде, которая не поддерживает включение малых потоков, параметр small не будет действовать и не будет сообщать об ошибке.

### Получение малого потока подписчиком

Вы можете установить параметр **small** [trtc.startRemoteVideo](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#startRemoteVideo) для получения малого потока. В противном случае по умолчанию будет получен основной поток.

```
trtc.on(TRTC.EVENT.REMOTE_VIDEO_AVAILABLE, ({ userId, streamType }) => {  trtc.startRemoteVideo({     view: '', // Передайте elementId или объект экземпляра элемента контейнера видео.    userId,     streamType,     option: { small: true }  });})
```

> **Примечание:** При получении малого потока нет необходимости предварительно проверять, опубликовал ли удалённый пользователь малый поток. Если удалённая сторона не опубликовала малый поток, SDK автоматически получит основной поток.

Вы также можете установить параметр small [trtc.updateRemoteVideo](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#updateRemoteVideo) для выбора получения основного потока или малого потока. Обычно используется для реализации следующей функции:

- Воспроизведение удалённого видео в "большом представлении" при нажатии пользователем на "малое представление"
- Воспроизведение удалённого видео в "малом представлении" при нажатии пользователем на "большое представление"

```
// Переключение на воспроизведение в "большом представлении" при нажатии пользователем на "малое представление".
await trtc.updateRemoteVideo({   view: '', // Передайте elementId или объект экземпляра элемента контейнера видео "большого представления".  userId: '', // Передайте userId удалённого пользователя.  streamType: TRTC.TYPE.STREAM_TYPE_MAIN,   option: { small: false }})// Переключение на воспроизведение в "малом представлении" при нажатии пользователем на "большое представление".
await trtc.updateRemoteVideo({   view: '', // Передайте elementId или объект экземпляра элемента контейнера видео "малого представления".  userId: '', // Передайте userId удалённого пользователя.  streamType: TRTC.TYPE.STREAM_TYPE_MAIN,   option: { small: true }})
```

> **Примечание:** Видеопоток TRTC имеет основной поток и дополнительный поток. Основной поток включает основной поток и малый поток, а дополнительный поток обычно используется для общего доступа к экрану. При получении видеопотока основной поток по умолчанию получает основной поток, а параметр small можно использовать для получения малого потока. TRTC поддерживает получение **основной поток + дополнительный поток**, **малый поток + дополнительный поток** и не поддерживает одновременное получение **основного потока + малого потока**. Основной поток и малый поток воспроизводятся в одном видеопредставлении. Только когда издатель включает малый поток, удалённый основной поток будет переключен на малый поток при указании параметра small. Если нет малого потока, SDK автоматически получит основной поток. В настоящее время нет уведомления о событии успешного переключения между основным и малым потоками.

## Включение "receiveWhenViewVisible"

Как правило, в многопользовательском видеосценарии из-за ограничений размера веб-страницы подписчик не будет отображать все видео на странице. Поэтому для удалённых видео, которые не нужно отображать на странице, они не должны быть получены. Видеопоток должен быть получен только когда представление видимо. Это позволяет сэкономить трафик и уменьшить затраты на производительность.

Однако реализация этой функции для вас не очень просто. Начиная с версии 5.4.0, SDK имеет встроенную поддержку этой функции. Вам нужно только установить параметры **receiveWhenViewVisible** и **viewRoot** при вызове [trtc.startRemoteVideo](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#startRemoteVideo) для включения этой функции, что очень удобно и быстро.

SDK автоматически проверяет состояние видимости **view**, переданного в [trtc.startRemoteVideo](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#startRemoteVideo). Передача видео начнётся, когда **view** видима, и остановится, когда **view** не видима.

### Пример кода

Предположим, что ваша структура DOM для воспроизведения видео выглядит следующим образом:

```
<div id='view-list'>  <div id='view1_main'></div>  <div id='view2_main'></div>  <div id='view3_main'></div></div>
```

```
trtc.on(TRTC.EVENT.REMOTE_VIDEO_AVAILABLE, ({ userId, streamType }) => {  trtc.startRemoteVideo({     view: 'view1_main',    userId,     streamType,     option: {       small: true,      // Включить "receiveWhenViewVisible"      receiveWhenViewVisible: true,       // Передайте первый уровень родительского элемента всего списка представлений,       // который используется для расчёта соотношения между view и viewRoot       // и проверки видимости представления. Значение по умолчанию — document.body.      viewRoot: document.getElementById('view-list')     }  });})
```

### Добавление эффекта загрузки для лучшего пользовательского опыта

После включения этой функции, когда пользователь прокручивает список видео:

- Когда представление изменяется с видимого на невидимое, SDK остановит передачу.
- Когда представление изменяется с невидимого на видимое, SDK возобновит передачу.

Поскольку возобновление передачи до успешного воспроизведения видео является асинхронным процессом, в этот период может появиться чёрный экран. Для обеспечения лучшего пользовательского опыта рекомендуется добавить эффект загрузки в область видео во время этого асинхронного процесса.

Вы можете обратиться к следующему коду, чтобы проверить, следует ли отображать эффект загрузки.

```
let isLoading = true;trtc.on(TRTC.EVENT.VIDEO_PLAY_STATE_CHANGED, ({ userId, streamType, state }) => {  // '' — это локальное видео  if (userId !== '') return;  // установить isLoading на false при воспроизведении удалённого видео.  isLoading = state !== 'PLAYING';  // Вы можете определить, чьего удалённого пользователя изменилось состояние воспроизведения основного/дополнительного потока  // по userId и streamType.});await trtc.stopRemoteVideo({ userId, streamType })// установить isLoading на false после остановки удалённого видео.
isLoading = false;
```

## Часто задаваемые вопросы

### Как SDK определяет, видимо ли представление?

Когда любая точка пиксела представления появляется в области **viewRoot**, представление считается видимым. И наоборот, когда все пиксели представления не находятся в области **viewRoot**, представление считается невидимым.

### Нужно ли мне учитывать совместимость браузера для основного и малого потоков?

Хотя некоторые браузеры не поддерживают включение основного и малого потоков (такие как все браузеры в системах iOS и Chrome версии ниже 63), включение малого потока не имеет побочных эффектов. Если окружение не поддерживает это, SDK будет использовать только основной поток, и вызов может продолжаться нормально. Вам не нужно писать дополнительные операторы if-else.

Вы можете вызвать [TRTC.isSupported](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#.isSupported), чтобы получить checkResult.detail.isSmallStreamSupported, чтобы подтвердить, поддерживает ли текущее окружение включение малого потока.

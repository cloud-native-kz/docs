### Я могу использовать веб-SDK TRTC в локальной среде разработки, но не после развертывания в Интернете. Что мне делать?

Если вы можете использовать SDK в локальной среде разработки, но не можете захватывать данные с камеры или микрофона после развертывания в веб, проверьте, развернута ли ваша веб-страница с использованием HTTP. Если это так, используйте HTTPS и убедитесь, что у вас есть действительный HTTPS-сертификат.

Дополнительные сведения см. в разделе [Поддержка доменов и протоколов](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/tutorial-05-info-browser.html#h2-2).

См. [Известные проблемы WebRTC и решения](https://web.sdk.qcloud.com/trtc/webrtc/doc/en/tutorial-02-info-webrtc-issues.html).

### **Что такое основной/дополнительный видеопоток?**

1. TRTC имеет основной видеопоток (основной поток) и дополнительный видеопоток (дополнительный поток).
2. Камера публикуется через основной поток, а совместное использование экрана публикуется через дополнительный поток.
3. Основной видеопоток включает: большой поток и малый поток. По умолчанию [TRTC.startRemoteVideo](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/TRTC.html#startRemoteVideo) воспроизводит большой поток, а малый поток можно воспроизводить через параметр small. См.: [Оптимизация видеозвонков с несколькими участниками](https://www.tencentcloud.com/document/product/647/59665).

### Почему камера поддерживает захват 1080p, но захватанное разрешение меньше 1080p?

Эта ситуация обычно возникает из-за того, что камера предварительно занята низким разрешением, в результате чего SDK не может захватить целевое разрешение.

Решение: Проверьте логику на стороне бизнеса, если камера захвачена заранее, пожалуйста, освободите камеру вовремя, когда она больше не используется.

### Могу ли я изменить стиль всплывающего окна совместного использования экрана в веб-SDK TRTC?

Нет. Стиль совместного использования экрана контролируется браузером и не может быть изменен.

### Что мне делать, если при использовании веб-SDK TRTC в iframes возникает ошибка "Permission denied"?

Для использования WebRTC в iframes вам необходимо добавить следующий атрибут к тегу iframe, чтобы получить необходимые разрешения.
Разрешения микрофона, камеры и совместного использования экрана:

```
<iframe allow="microphone; camera; display-capture;">
```

## 3. Воспроизведение

### Почему во время звонка есть видео, но нет звука?

Это может быть связано с политикой автоматического воспроизведения браузера, которая вызывает ошибку "PLAY_NOT_ALLOWED". Чтобы решить эту проблему, см. [Обработка ограничения автоматического воспроизведения](https://www.tencentcloud.com/document/product/647/59666).

### Почему звук переходит с динамика на трубку после захвата микрофона?

На Android обычно имеется несколько микрофонов, и список меток имеет вид: ['default', 'Speakerphone', 'Headset earpiece'].

Если вы не указываете микрофон в trtc.startLocalAudio, микрофон по умолчанию браузера может быть Headset earpiece и звук будет выходить из трубки.

Если вам нужно воспроизводить звук через динамик, вам необходимо указать микрофон с меткой 'Speakerphone'.

### Что вызывает событие trtc.on(TRTC.EVENT.KICKED_OUT)?

Событие срабатывает, когда пользователь удаляется из комнаты, например, когда один и тот же ID пользователя используется для входа с разных устройств или когда вызывается REST API [RemoveUser](https://intl.cloud.tencent.com/document/product/647/34268) для удаления пользователя. Дополнительные сведения см. в разделе [KICKED_OUT](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/zh-cn/module-EVENT.html#.KICKED_OUT).

> **Примечание:** Повторный вход не разрешен SDK (это может вызвать исключения звонков) и должен быть避避 на уровне бизнеса.

### Предостережение при использовании Vue 3

При использовании фреймворка Vue3 необходимо использовать интерфейс markRaw для маркировки экземпляра trtc, чтобы избежать преобразования trtc в объект Proxy с помощью Vue, что может вызвать некоторые неожиданные проблемы.

```
import { markRaw } from 'vue';const trtc = markRaw(TRTC.create());
```

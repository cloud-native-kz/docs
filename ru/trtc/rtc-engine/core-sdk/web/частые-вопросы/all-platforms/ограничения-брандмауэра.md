# Ограничения брандмауэра

### Какие порты и доменные имена следует добавить в список разрешений брандмауэра для нативного SDK?

Добавьте следующие порты в список разрешений:

| TRTC SDK (Native) | Порты |
| --- | --- |
| TCP | 443, 20166, 10443, 10444, 10445, 10446, 10447, 10448, 10449, 10450, 10451, 13275, 23275, 33000, 37528 |
| UDP | 8000, 8080, 8001, 8002, 8003, 8004, 8005, 8006, 8007, 8008, 8009, 16285, 9000 |

Добавьте следующие доменные имена в список разрешений:

```
intl-query.trtc.tencent-cloud.comintl-accelerate.trtc.tencent-cloud.comtrtc-client-log-overseas-1258344699.cos.ap-singapore.myqcloud.comintl-sdklog.trtc.tencent-cloud.comsdkdc.live.qcloud.comspeedtestint.trtc.tencent-cloud.comintl-query.trtc.tencent-cloud.comhwapi.im.qcloud.comvideoapi-sgp.im.qcloud.comtrtc-sdk-config-1258344699.file.myqcloud.com
```

> Примечание: Поскольку IP-адреса/доменные имена на стороне сервиса Tencent Cloud являются динамическими и не привязаны к фиксированному набору IP-адресов/доменных имен, мы не можем предоставить вам фиксированный список.

### Какие порты и доменные имена следует добавить в список разрешений брандмауэра для WebRTC?

Добавьте следующие порты в список разрешений:

| WebRTC (H5) | Порты |
| --- | --- |
| TCP | 443 |
| UDP | 8000, 8080, 8800, 843, 443, 16285 |

Добавьте следующие доменные имена в список разрешений:

```
signailing.rtc.tencentcloud.comschedule.rtc.tencentcloud.com*.rtc.tencentcloud.com*.rtc-web.com*.rtc-web.io
```

### Как настроить прокси для доступа клиентов к TRTC SDK для веб-приложений из частной сети?

Вы можете использовать схему Nginx + Coturn. Подробные инструкции см. в разделе [Handle Firewall Restriction](https://web.sdk.qcloud.com/trtc/webrtc/v5/doc/en/tutorial-34-advanced-proxy.html).

| Решение | Сценарий применения | Требования к сети |
| --- | --- | --- |
| Решение 1 | Разрешить клиентам доступ к конкретным прокси-серверам в общественной сети | Разрешить клиентам доступ к прокси-серверам, настроенным в общественной сети |
| Решение 2 | Разрешить клиентам доступ в общественную сеть через прокси-серверы, настроенные в частной сети | Разрешить прокси-серверам доступ в общественную сеть |

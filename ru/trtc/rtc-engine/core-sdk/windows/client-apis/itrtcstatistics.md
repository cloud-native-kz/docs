# ITRTCStatistics

Copyright (c) 2021 Tencent. All rights reserved.

Модуль: метрики аудио/видео TRTC (только для чтения)

Функция: TRTC SDK сообщает вам текущие метрики аудио/видео в реальном времени (частота кадров, битрейт, задержку и т. д.) один раз в две секунды

| [TRTCRemoteStatistics](https://www.tencentcloud.com/document/product/647/50772#31083ca966a8ee092affeab44b65fe9a) | Удаленные метрики аудио/видео. |

## TRTCLocalStatistics

**TRTCLocalStatistics**

**Локальные метрики аудио/видео.**

| EnumType | DESC |
| --- | --- |
| audioBitrate | Описание поля: локальный битрейт аудио в Кбит/с, т. е. объем генерируемых аудиоданных в секунду |
| audioCaptureState | Описание поля: статус сбора аудиооборудования (0 — нормально, 1 — обнаружена длительная тишина, 2 — обнаружены помехи, 3 — обнаружены аномальные перерывы) |
| audioSampleRate | Описание поля: локальная частота дискретизации аудио (Гц) |
| frameRate | Описание поля: локальная частота кадров видео в fps, т. е. количество видеокадров в секунду |
| height | Описание поля: высота локального видео в пикселях |
| streamType | Описание поля: тип видеопотока (HD большое изображение \| плавное малое изображение \| дополнительный поток) |
| videoBitrate | Описание поля: локальный битрейт видео в Кбит/с, т. е. объем генерируемых видеоданных в секунду |
| width | Описание поля: ширина локального видео в пикселях |

## TRTCRemoteStatistics

**TRTCRemoteStatistics**

**Удаленные метрики аудио/видео.**

| EnumType | DESC |
| --- | --- |
| audioBitrate | Описание поля: локальный битрейт аудио (Кбит/с) |
| audioBlockRate | Описание поля: скорость задержки воспроизведения аудио (%) Скорость задержки воспроизведения аудио (audioBlockRate) = совокупная продолжительность задержки воспроизведения аудио (audioTotalBlockTime) / общая продолжительность воспроизведения аудио |
| audioPacketLoss | Описание поля: общая скорость потери пакетов (%) в аудиопотоке ` audioPacketLoss ` представляет скорость потери пакетов, окончательно рассчитанную на стороне зрителя после прохождения аудио/видео потока через всю цепочку передачи "вещатель -> облако -> зритель". Чем меньше ` audioPacketLoss `, тем лучше. Скорость потери пакетов 0 означает, что все данные аудиопотока полностью доставлены зрителю. Если ` downLoss ` равен ` 0 `, но ` audioPacketLoss ` не равен, то на канале "облако -> зритель" потерь пакетов нет, но на канале "вещатель -> облако" имеются неуправляемые потери пакетов. |
| audioSampleRate | Описание поля: локальная частота дискретизации аудио (Гц) |
| audioTotalBlockTime | Описание поля: совокупная продолжительность задержки воспроизведения аудио (мс) |
| finalLoss | Описание поля: общая скорость потери пакетов (%) в аудио/видео потоке Устарело, используйте вместо этого audioPacketLoss и videoPacketLoss. |
| frameRate | Описание поля: удаленная частота кадров видео (fps) |
| height | Описание поля: высота удаленного видео в пикселях |
| jitterBufferDelay | Описание поля: задержка воспроизведения (мс) Чтобы избежать задержек аудио/видео, вызванных сетевыми колебаниями и беспорядком пакетов в сети, TRTC поддерживает буфер воспроизведения на стороне воспроизведения для организации полученных сетевых пакетов данных. Размер буфера адаптивно регулируется в соответствии с текущим качеством сети и преобразуется в продолжительность в миллисекундах, т. е. ` jitterBufferDelay `. |
| point2PointDelay | Описание поля: сквозная задержка (мс) ` point2PointDelay ` представляет задержку канала "вещатель -> облако -> зритель". Точнее говоря, она представляет задержку всей цепочки "захват -> кодирование -> передача по сети -> получение -> буферизация -> декодирование -> воспроизведение". ` point2PointDelay ` работает только если оба локальный и удаленный SDK имеют версию 8.5 или выше. Если удаленный SDK имеет версию ниже 8.5, это значение всегда будет 0 и, таким образом, бессмысленно. |
| remoteNetworkRTT | Описание поля: время приема-передачи (мс) от SDK до облака Это значение представляет общее время, необходимое для отправки сетевого пакета от SDK к облаку и затем отправки сетевого пакета обратно от облака к SDK, т. е. общее время для прохождения сетевого пакета через цепочку "SDK -> облако -> SDK". Чем меньше значение, тем лучше. Если ` remoteNetworkRTT ` ниже 50 мс, это означает короткую задержку аудио/видео вызова; если ` remoteNetworkRTT ` выше 200 мс, это означает долгую задержку аудио/видео вызова. Следует отметить, что ` remoteNetworkRTT ` представляет общее время, затраченное на цепочку "SDK -> облако -> SDK"; поэтому нет необходимости различать ` remoteNetworkUpRTT ` и ` remoteNetworkDownRTT `. |
| remoteNetworkUplinkLoss | Описание поля: скорость потери пакетов (%) в восходящем канале от SDK к облаку Чем меньше значение, тем лучше. Если ` remoteNetworkUplinkLoss ` равен ` 0% `, качество восходящей сети очень хорошее, и пакеты данных, загружаемые в облако, практически не теряются. Если ` remoteNetworkUplinkLoss ` равен ` 30% `, 30% пакетов аудио/видео данных, отправленных в облако SDK, теряются на канале передачи. |
| streamType | Описание поля: тип видеопотока (HD большое изображение \| плавное малое изображение \| дополнительный поток) |
| userId | Описание поля: ID пользователя |
| videoBitrate | Описание поля: удаленный битрейт видео (Кбит/с) |
| videoBlockRate | Описание поля: скорость задержки воспроизведения видео (%) Скорость задержки воспроизведения видео (videoBlockRate) = совокупная продолжительность задержки воспроизведения видео (videoTotalBlockTime) / общая продолжительность воспроизведения видео |
| videoPacketLoss | Описание поля: общая скорость потери пакетов (%) в видеопотоке ` videoPacketLoss ` представляет скорость потери пакетов, окончательно рассчитанную на стороне зрителя после прохождения аудио/видео потока через всю цепочку передачи "вещатель -> облако -> зритель". Чем меньше ` videoPacketLoss `, тем лучше. Скорость потери пакетов 0 означает, что все данные видеопотока полностью доставлены зрителю. Если ` downLoss ` равен ` 0 `, но ` videoPacketLoss ` не равен, то на канале "облако -> зритель" потерь пакетов нет, но на канале "вещатель -> облако" имеются неуправляемые потери пакетов. |
| videoTotalBlockTime | Описание поля: совокупная продолжительность задержки воспроизведения видео (мс) |
| width | Описание поля: ширина удаленного видео в пикселях |

## TRTCStatistics

**TRTCStatistics**

**Метрики сети и производительности.**

| EnumType | DESC |
| --- | --- |
| appCpu | Описание поля: использование ЦПУ (%) текущего приложения, системы Android 8.0 и выше не поддерживаются |
| appMemoryUsageInMB | Описание поля: размер использования памяти (МБ) текущего приложения |
| downLoss | Описание поля: скорость потери пакетов (%) в нисходящем канале от облака к SDK Чем меньше значение, тем лучше. Если ` downLoss ` равен ` 0% `, качество нисходящей сети очень хорошее, и пакеты данных, полученные из облака, практически не теряются. Если ` downLoss ` равен ` 30% `, 30% пакетов аудио/видео данных, отправленных SDK из облака, теряются на канале передачи. |
| gatewayRtt | Описание поля: время приема-передачи (мс) от SDK к шлюзу Это значение представляет общее время, необходимое для отправки сетевого пакета от SDK к шлюзу и затем отправки сетевого пакета обратно от шлюза к SDK, т. е. общее время для прохождения сетевого пакета через цепочку "SDK -> шлюз -> SDK". Чем меньше значение, тем лучше. Если ` gatewayRtt ` ниже 50 мс, это означает короткую задержку аудио/видео вызова; если ` gatewayRtt ` выше 200 мс, это означает долгую задержку аудио/видео вызова. Следует отметить, что ` gatewayRtt ` не действителен для мобильной сети. |
| localStatisticsArray | Описание поля: локальная статистика аудио/видео Поскольку может быть три локальных потока аудио/видео (т. е. HD большое изображение, плавное малое изображение и дополнительный поток), локальная статистика аудио/видео является массивом. |
| localStatisticsArraySize | Описание поля: размер массива ` localStatisticsArray ` |
| receivedBytes | Описание поля: общее количество полученных байтов (включая данные сигнализации и аудио/видео) |
| remoteStatisticsArray | Описание поля: удаленная статистика аудио/видео Поскольку может быть несколько одновременных удаленных пользователей, и каждый из них может иметь несколько одновременных потоков аудио/видео (т. е. HD большое изображение, плавное малое изображение и дополнительный поток), удаленная статистика аудио/видео является массивом. |
| remoteStatisticsArraySize | Описание поля: размер массива ` remoteStatisticsArray ` |
| rtt | Описание поля: время приема-передачи (мс) от SDK к облаку Это значение представляет общее время, необходимое для отправки сетевого пакета от SDK к облаку и затем отправки сетевого пакета обратно от облака к SDK, т. е. общее время для прохождения сетевого пакета через цепочку "SDK -> облако -> SDK". Чем меньше значение, тем лучше. Если ` rtt ` ниже 50 мс, это означает короткую задержку аудио/видео вызова; если ` rtt ` выше 200 мс, это означает долгую задержку аудио/видео вызова. Следует отметить, что ` rtt ` представляет общее время, затраченное на цепочку "SDK -> облако -> SDK"; поэтому нет необходимости различать ` upRtt ` и ` downRtt `. |
| sentBytes | Описание поля: общее количество отправленных байтов (включая данные сигнализации и аудио/видео) |
| systemCpu | Описание поля: использование ЦПУ (%) текущей системы, системы Android 8.0 и выше не поддерживаются |
| systemMemoryInMB | Описание поля: размер памяти (МБ) текущей системы |
| systemMemoryUsageInMB | Описание поля: размер использования памяти (МБ) текущей системы, iOS и MAC не поддерживаются |
| upLoss | Описание поля: скорость потери пакетов (%) в восходящем канале от SDK к облаку Чем меньше значение, тем лучше. Если ` upLoss ` равен ` 0% `, качество восходящей сети очень хорошее, и пакеты данных, загружаемые в облако, практически не теряются. Если ` upLoss ` равен ` 30% `, 30% пакетов аудио/видео данных, отправленных в облако SDK, теряются на канале передачи. |

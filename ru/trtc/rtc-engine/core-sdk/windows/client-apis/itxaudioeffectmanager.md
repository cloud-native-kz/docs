## enableVoiceEarMonitor

**enableVoiceEarMonitor**

| void enableVoiceEarMonitor | (bool enable) |
| --- | --- |

**Включение внутриушного мониторинга.**

После включения внутриушного мониторинга авторы могут слышать в наушниках свой собственный голос, захватываемый микрофоном. Эта функция предназначена для сценариев пения.

Внутриушный мониторинг не может быть включен для беспроводных наушников Bluetooth. Это обусловлено высокой задержкой беспроводных наушников. Пожалуйста, попросите авторов использовать проводные наушники через напоминание в пользовательском интерфейсе.

Поскольку не все телефоны обеспечивают отличные эффекты внутриушного мониторинга, эта функция заблокирована на некоторых телефонах.

| Param | DESC |
| --- | --- |
| enable | ` true: ` включить; ` false `: отключить |

> **Примечание** Внутриушный мониторинг можно включить только при использовании наушников. Пожалуйста, напомните авторам использовать проводные наушники.

## setVoiceEarMonitorVolume

**setVoiceEarMonitorVolume**

| void setVoiceEarMonitorVolume | (int volume) |
| --- | --- |

**Установка громкости внутриушного мониторинга.**

Этот API используется для установки громкости внутриушного мониторинга.

| Param | DESC |
| --- | --- |
| volume | Громкость. Диапазон значений: [0, 150]; по умолчанию: 100 |

> **Примечание** Громкость наушников — один из наиболее важных факторов, влияющих на пользовательский опыт. Учитывая, что разные пользователи имеют разную чувствительность к громкости, настоятельно рекомендуется предоставить ползунок регулировки громкости внутриушного мониторинга в интерфейсе пользователя. Рекомендуемый диапазон регулировки находится между 0 и 150. Эффект усиления 100-150 очень полезен для некоторых телефонов Android с низким уровнем записи.

## seekMusicToPosInTime

**seekMusicToPosInTime**

| void seekMusicToPosInTime | (int id |
| --- | --- |
|  | int pts) |

**Установка прогресса воспроизведения (мс) фоновой музыки.**

| Param | DESC |
| --- | --- |
| id | Идентификатор музыки |
| pts | Единица: миллисекунда |

> **Примечание** Не вызывайте этот API часто, так как музыкальный файл может быть прочитан и записан каждый раз при вызове API, что может быть затратно по времени. Дождитесь, пока пользователь закончит перетаскивание полосы воспроизведения, прежде чем вызывать этот API. Контроллер полосы воспроизведения в интерфейсе пользователя обычно обновляет прогресс с высокой частотой по мере того, как пользователи перетаскивают полосу воспроизведения. Это приведет к плохому пользовательскому опыту, если вы не ограничите частоту.

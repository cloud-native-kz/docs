## TRTCVideoStreamType

**TRTCVideoStreamType**

**Тип видеопотока.**

TRTC предоставляет три различных видеопотока, включая:

- HD большое изображение: обычно используется для передачи видеоданных с камеры.
- Гладкое маленькое изображение: имеет то же содержимое, что и большое изображение, но с более низким разрешением и битрейтом, следовательно, более низкой четкостью.
- Субпоток изображение: обычно используется для совместного использования экрана. В любой момент времени в комнате может быть разрешено только одному пользователю опубликовать субпоток видеоизображения, остальные пользователи должны дождаться, пока этот пользователь закроет субпоток, прежде чем они смогут опубликовать свой собственный субпоток.

> **Примечание** SDK не поддерживает включение гладкого маленького изображения отдельно, оно должно быть включено вместе с большим изображением. Он автоматически установит разрешение и битрейт маленького изображения.

| Enum | Value | DESC |
| --- | --- | --- |
| TRTCVideoStreamTypeBig | 0 | HD большое изображение: обычно используется для передачи видеоданных с камеры. |
| TRTCVideoStreamTypeSmall | 1 | Гладкое маленькое изображение: имеет то же содержимое, что и большое изображение, но с более низким разрешением и битрейтом, следовательно, более низкой четкостью. |
| TRTCVideoStreamTypeSub | 2 | Субпоток изображение: обычно используется для совместного использования экрана. В любой момент времени в комнате может быть разрешено только одному пользователю опубликовать субпоток видеоизображения, остальные пользователи должны дождаться, пока этот пользователь закроет субпоток, прежде чем они смогут опубликовать свой собственный субпоток. |

## TRTCVideoFillMode

**TRTCVideoFillMode**

**Режим заполнения видеоизображения.**

Если соотношение сторон области отображения видео не равно соотношению сторон видеоизображения, необходимо указать режим заполнения:

| Enum | Value | DESC |
| --- | --- | --- |
| TRTCVideoFillMode_Fill | 0 | Режим заполнения: видеоизображение будет центрировано и масштабировано для заполнения всей области отображения, части, выходящие за границы области, будут обрезаны. Отображаемое изображение может быть неполным в этом режиме. |
| TRTCVideoFillMode_Fit | 1 | Режим подгонки: видеоизображение будет масштабировано на основе его длинной стороны для подгонки к области отображения, короткая сторона будет заполнена черными полосами. Отображаемое изображение является полным в этом режиме, но могут быть черные полосы. |
| TRTCVideoFillMode_ScaleFill | 2 | Режим масштабирования до полного заполнения: это означает, что независимо от соотношения сторон изображения оно будет растянуто или сжато для полного заполнения области отображения. В этом режиме соотношение сторон изображения может быть изменено, что приведет к искажению отображаемого изображения. |

| TRTCVideoRotation270 | 3 | Поворот по часовой стрелке на 270 градусов |

## TRTCTranscodingConfigMode

**TRTCTranscodingConfigMode**

**Режим макета облачного микширования-транскодирования.**

Облачный сервис микширования-транскодирования TRTC может смешивать несколько аудио/видеопотоков в комнате в один поток. Поэтому вам необходимо указать схему макета видеоизображений. Предоставляются следующие режимы макета:

| Enum | Value | DESC |
| --- | --- | --- |
| TRTCTranscodingConfigMode_Unknown | 0 | Не определено |
| TRTCTranscodingConfigMode_Manual | 1 | Режим ручного макета. В этом режиме вам нужно указать точное положение каждого видеоизображения. Этот режим имеет наивысшую степень свободы, но его удобство использования наихудшее: вам нужно ввести все параметры в [TRTCTranscodingConfig](https://www.tencentcloud.com/document/product/647/50775#c3eef0088da62b5dd27fb0c08db18906), включая координаты положения каждого видеоизображения (TRTCMixUser). Вам нужно прослушивать обратные вызовы событий [onUserVideoAvailable](https://www.tencentcloud.com/document/product/647/50771#6d1cb6b1fcf292ac6f34cd8baa647937) и [onUserAudioAvailable](https://www.tencentcloud.com/document/product/647/50771#2efaaf0cd5c69f4857c4c40f6cec038f) в [ITRTCCloudCallback](https://www.tencentcloud.com/document/product/647/50771#c334ca01f4134afa7b339a9da12fbb33) и постоянно корректировать параметр ` mixUsers ` в соответствии со статусом аудио/видео каждого пользователя с включенным микрофоном в текущей комнате. |
| TRTCTranscodingConfigMode_Template_PureAudio | 2 | Режим чистого аудио. Этот режим подходит для чистых аудиосценариев, таких как аудиозвонок (AudioCall) и аудиочат (VoiceChatRoom). Вам нужно установить это один раз через API setMixTranscodingConfig после входа в комнату, а затем SDK автоматически смешает аудио всех пользователей с включенным микрофоном в комнате в прямую трансляцию текущего пользователя. Вам не нужно устанавливать параметр ` mixUsers ` в [TRTCTranscodingConfig](https://www.tencentcloud.com/document/product/647/50775#c3eef0088da62b5dd27fb0c08db18906); вместо этого вам нужно установить только параметры ` audioSampleRate `, ` audioBitrate ` и ` audioChannels `. |
| TRTCTranscodingConfigMode_Template_PresetLayout | 3 | Режим предустановленного макета. Это наиболее популярный режим макета, поскольку он позволяет вам заранее установить положение каждого видеоизображения через заполнители, а затем SDK автоматически динамически корректирует его в зависимости от количества видеоизображений в комнате. В этом режиме вам все еще нужно установить параметр ` mixUsers `, но вы можете установить ` userId ` как "заполнитель". Значения заполнителей включают: "$PLACE_HOLDER_REMOTE$": изображение удаленного пользователя. Можно установить несколько изображений. "$PLACE_HOLDER_LOCAL_MAIN$": изображение локальной камеры. Можно установить только одно изображение. "$PLACE_HOLDER_LOCAL_SUB$": изображение локального общего доступа к экрану. Можно установить только одно изображение. В этом режиме вам не нужно прослушивать обратные вызовы [onUserVideoAvailable](https://www.tencentcloud.com/document/product/647/50771#6d1cb6b1fcf292ac6f34cd8baa647937) и [onUserAudioAvailable](https://www.tencentcloud.com/document/product/647/50771#2efaaf0cd5c69f4857c4c40f6cec038f) в [ITRTCCloudCallback](https://www.tencentcloud.com/document/product/647/50771#c334ca01f4134afa7b339a9da12fbb33) для внесения реальных изменений. Вместо этого вам нужно вызвать setMixTranscodingConfig один раз после успешного входа в комнату. Затем SDK автоматически заполнит установленные вами заполнители реальными значениями ` userId `. |
| TRTCTranscodingConfigMode_Template_ScreenSharing | 4 | Режим общего доступа к экрану. Этот режим подходит для сценариев использования на основе общего доступа к экрану, таких как онлайн-образование, и поддерживается только SDK для Windows и macOS. В этом режиме SDK сначала построит холст в соответствии с целевым разрешением, которое вы установили (через параметры ` videoWidth ` и ` videoHeight `). До того как учитель включит общий доступ к экрану, SDK увеличит масштаб изображения с камеры учителя и нарисует его на холсте. После того как учитель включит общий доступ к экрану, SDK нарисует видеоизображение, которое используется совместно на экране, на этом же холсте. Цель этого режима макета — обеспечить согласованность выходного разрешения модуля микширования-транскодирования и избежать проблем с нечетким экраном при воспроизведении курса и воспроизведении на веб-странице (веб-плееры не поддерживают регулируемое разрешение). Тем временем аудио студентов с включенным микрофоном будет автоматически смешано с аудио/видеопотоком учителя. Видеосодержание в режиме преподавания в основном представляет собой общий экран, и передача изображения с камеры и изображения экрана одновременно — это пустая трата полосы пропускания. Поэтому рекомендуемая практика — напрямую нарисовать изображение с камеры на текущий экран через API [setLocalVideoRenderCallback](https://www.tencentcloud.com/document/product/647/50770#58ead030937b17839c952192ba2810fb). В этом режиме вам не нужно устанавливать параметр ` mixUsers ` в [TRTCTranscodingConfig](https://www.tencentcloud.com/document/product/647/50775#c3eef0088da62b5dd27fb0c08db18906), и SDK не будет смешивать изображения студентов, чтобы не мешать эффекту общего доступа к экрану. Вы можете установить ширину x высоту в [TRTCTranscodingConfig](https://www.tencentcloud.com/document/product/647/50775#c3eef0088da62b5dd27fb0c08db18906) на 0 пиксель x 0 пиксель, и SDK автоматически рассчитает подходящее разрешение на основе соотношения сторон текущего экрана пользователя. Если текущая ширина экрана учителя меньше или равна 1920 пиксель, SDK использует фактическое разрешение текущего экрана учителя. Если текущая ширина экрана учителя больше 1920 пиксель, SDK выберет одно из трех разрешений 1920x1080 (16:9), 1920x1200 (16:10) и 1920x1440 (4:3) в соответствии с текущим соотношением сторон экрана. |

## TRTCParams

## TRTCPayloadPrivateEncryptionConfig

**TRTCPayloadPrivateEncryptionConfig**

**Конфигурация приватного шифрования медиапотока.**

Эта конфигурация используется для установки алгоритма и ключа для приватного шифрования медиапотока.

| EnumType | DESC |
| --- | --- |
| encryptionAlgorithm | ` Описание: ` Алгоритм шифрования, по умолчанию [TRTCEncryptionAlgorithmAes128Gcm](https://www.tencentcloud.com/document/product/647/50775#d013615ce328f9be9b01ee3f147651c0). |
| encryptionKey | ` Описание: ` Ключ шифрования, тип string.` Значение: ` Если алгоритм шифрования [TRTCEncryptionAlgorithmAes128Gcm](https://www.tencentcloud.com/document/product/647/50775#d013615ce328f9be9b01ee3f147651c0), длина ключа должна быть 16 байт; если алгоритм шифрования [TRTCEncryptionAlgorithmAes256Gcm](https://www.tencentcloud.com/document/product/647/50775#d013615ce328f9be9b01ee3f147651c0), длина ключа должна быть 32 байта. |
| encryptionSalt[32] | ` Описание: ` Соль, вектор инициализации для шифрования.` Значение: ` Необходимо убедиться, что массив, заполненный в этом параметре, не пуст, не все нули и длина данных составляет 32 байта. |

## TRTCAudioVolumeEvaluateParams

**TRTCAudioVolumeEvaluateParams**

**Параметры оценки громкости и другие связанные параметры.**

Эта настройка используется для включения обнаружения вокала и вычисления звукового спектра.

| EnumType | DESC |
| --- | --- |
| enablePitchCalculation | ` Описание: ` Следует ли включить расчёт локальной частоты вокала. |
| enableSpectrumCalculation | ` Описание: ` Следует ли включить расчёт звукового спектра. |
| enableVadDetection | ` Описание: ` Следует ли включить локальное обнаружение голоса.**Примечание**Вызовите перед startLocalAudio. |
| interval | ` Описание: ` Установите интервал срабатывания обратного вызова [onUserVoiceVolume](https://www.tencentcloud.com/document/product/647/50771#454923f2ebbf576be5048238c2908e1f), единица измерения — миллисекунды, минимальный интервал — 100 мс, если значение меньше или равно 0, обратный вызов будет закрыт.` Значение: ` Рекомендуемое значение: 300, в миллисекундах.**Примечание**Когда интервал больше 0, уведомление о громкости будет включено по умолчанию, дополнительная настройка не требуется. |

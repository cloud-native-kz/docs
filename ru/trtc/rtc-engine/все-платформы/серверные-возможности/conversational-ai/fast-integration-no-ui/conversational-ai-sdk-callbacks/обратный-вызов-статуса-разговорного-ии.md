# Обратный вызов статуса разговорного ИИ

Разговорный ИИ Tencent RTC предоставляет богатые обратные вызовы статуса. Эти обратные вызовы статуса отправляются через пользовательские сообщения RTC Engine, что позволяет легко переключать состояния на клиенте, включая "Прослушивание", "Обдумывание", "Речь" и другие состояния.

Вы можете использовать функцию [Receive Custom Messages](https://www.tencentcloud.com/document/product/647/47866), предоставляемую SDK RTC Engine, для прослушивания обратных вызовов на клиенте и получения данных о статусе разговора ИИ. **Значение cmdID фиксировано на уровне 1.**

## Возможности

1. Синхронизация в реальном времени: Изменения статуса могут быть синхронизированы для всех участников в реальном времени.
2. Гибкость: Используется формат пользовательского сообщения для облегчения интеграции и расширения.
3. Поддержка нескольких состояний: Поддерживаются несколько состояний, такие как "Прослушивание", "Обдумывание", "Речь" и "Прерывание".

## Формат сообщения

Сообщения обратного вызова статуса находятся в формате JSON. Конкретные поля выглядят следующим образом:

| Поле | Тип | Описание |
| --- | --- | --- |
| type | Number | Тип сообщения. 10001 указывает на статус чат-бота. |
| sender | String | Идентификатор пользователя отправителя, который является ID чат-бота. |
| receiver | Array | Список идентификаторов пользователей получателей. Сообщение фактически транслируется в комнате. |
| payload | Object | Полезная нагрузка сообщения. Содержит подробную информацию о статусе. |

Объект payload содержит следующие поля:

| Поле | Тип | Описание |
| --- | --- | --- |
| roundid | String | Уникальный ID, определяющий раунд разговора. |
| timestamp | Number | Временная метка, указывающая конкретное время изменения статуса. |
| state | Number | Текущий код статуса чат-бота. |

Описание кодов статуса:

| Код статуса | Описание |
| --- | --- |
| 1 | Прослушивание |
| 2 | Обдумывание |
| 3 | Речь |
| 4 | Прерывание |

## Пример сообщения

```
{  "type": 10001,  "sender": "ai_assistant_001",  "receiver": [],  "payload": {    "roundid": "conversation_789012",    "timestamp": 1629384755,    "state": 2  }}
```

## Сценарии использования

1. Обратная связь UI: Отображение соответствующих элементов интерфейса на основе статуса, таких как анимация статуса "Прослушивание".
2. Управление взаимодействием: Отключение ввода определенными пользователями, когда чат-бот ИИ думает или говорит, чтобы избежать прерываний.
3. Отладка и анализ: Использование информации о статусе для выполнения анализа разговора для улучшения взаимодействия с пользователем.

# Обзор

Для облегчения интеграции TRTC мы поддерживаем публикацию и воспроизведение по стандартному протоколу RTMP. Вы можете установить [OBS](https://obsproject.com/download), FFmpeg или другие библиотеки RTMP для публикации потоков с помощью TRTC. OBS — это сторонний инструмент с открытым исходным кодом для прямого эфира. Он прост в использовании, бесплатен и поддерживает OS X, Windows и Linux. OBS можно использовать в широком диапазоне сценариев и он способен удовлетворить большинство потребностей прямого эфира без необходимости в дополнительных плагинах. Вы можете загрузить его последнюю версию на [сайте OBS](https://obsproject.com/download?spm=a2c4g.11186623.2.15.6aac1445JPlKR8).

**Эта функция завершит бета-тестирование с 15 февраля 2023 г. Для её использования необходимо приобрести [**ежемесячные пакеты**](https://www.tencentcloud.com/document/product/647/56025#) TRTC Standard или Pro. Приложения (SdkAppID), включённые до этого времени, могут использоваться бесплатно до 1 апреля 2023 г. (переходный период).** Поток RTMP будет рассматриваться как пользователь в комнате TRTC, что будет влечь за собой расходы на время вызова. Для получения дополнительной информации см. [Биллинг аудио и видеопотока](https://www.tencentcloud.com/document/product/647/42734#). Операции перекодирования будут выполняться в процессе подключения потоков RTMP, что будет влечь за собой расходы на перекодирование. Для получения дополнительной информации см. [Биллинг облачного микширования и перекодирования](https://www.tencentcloud.com/document/product/647/47631#).

# Сценарии использования

| Сценарий | Описание |
| --- | --- |
| Онлайн-образование | Используйте настольную версию OBS или FFmpeg для публикации учебных материалов (поддерживаются большинство форматов мультимедиа) через RTMP в комнату TRTC. Студенты в комнате могут воспроизводить поток через SDK TRTC и видеть те же учебные материалы, что и преподаватель, управляющий прогрессом воспроизведения/скоростью или переключением между главами. Отличная синхронизация на нескольких устройствах обеспечивает лучшее качество обучения. |
| Просмотр спортивных событий | Организаторы спортивных событий предоставляют контент в виде потоков RTMP. Вы можете публиковать потоки в комнаты TRTC, чтобы пользователи в комнатах могли смотреть событие с минимальной задержкой. С возможностью взаимодействия TRTC пользователи также могут общаться в аудио/видео формате друг с другом на протяжении всего события. |
| Другое | Вы также можете использовать функцию публикации RTMP для реализации других приложений в реальном времени на основе потокового контента. |

# URL-адреса публикации и воспроизведения

## URL-адреса публикации

```
rtmp://intl-rtmp.rtc.qq.com/push/Room ID?sdkappid=Application ID&userid=User ID&usersig=Signature
```

- Для публикации RTMP `appName` — это `push`.
- Замените «Room ID», «Application ID», «User ID» и «Signature» их фактическими значениями.
- Для простоты мы поддерживаем только строковые идентификаторы комнат. Идентификатор комнаты может содержать цифры, буквы и подчёркивания и не может превышать 64 символа.

> **Предупреждение:** Для воспроизведения потока RTMP на других клиентах TRTC при входе в комнату убедитесь, что вы используете строковый идентификатор комнаты.

- Информацию о способе создания `UserSig` см. в разделе [UserSig](https://www.tencentcloud.com/document/product/647/35166). **Убедитесь, что ваша подпись находится в пределах периода действительности**.

**Пример:**

```
rtmp://intl-rtmp.rtc.qq.com/push/hello-string-room?sdkappid=140*****66&userid=******rtmp2&usersig=eJw1jdE********RBZ8qKGRj8Yp-wVbv*mGMVZqS7w-mMDQL
```

# Пример

Вы можете использовать программное обеспечение или библиотеку программирования, поддерживающую RTMP, для публикации потоков RTMP. В разделе ниже показано, как это делать.

## Использование OBS для публикации потоков

### Предварительные требования

Вы установили [OBS](https://obsproject.com/download?spm=a2c4g.11186623.2.15.6aac1445JPlKR8).

### Шаг 1. Выбор источника

На панели **Sources** (Источники) в нижней части нажмите **+** и выберите источник в соответствии с вашими потребностями. Распространённые источники включают следующее:

| Источник | Примечание |
| --- | --- |
| Image | Публикует одно изображение |
| Image Slide Show | Публикует несколько изображений (вы можете определить порядок воспроизведения и будет ли оно циклическим) |
| Scene | Вставляет целую сцену для достижения различных эффектов потокового видео |
| Media Source | Загружает локальный файл и публикует его в виде прямого эфира |
| Text | Добавляет текст в реальном времени в ваш поток |
| Window Capture | Захватывает и публикует выбранное вами окно в реальном времени |
| Video Capture Device | Захватывает и публикует изображения, снятые камерой в реальном времени |
| Audio Input Capture | Прямая трансляция звука (устройство входного аудио) |
| Audio Output Capture | Прямая трансляция звука (устройство выходного аудио) |

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/00d504ed7b7c11ed8e7e525400463ef7.png)

### Шаг 2. Установка параметров публикации

1. На панели **Controls** (Элементы управления) внизу нажмите **Settings** (Параметры).
![img](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/00e684b27b7c11ed8e7e525400463ef7.png)
2. Нажмите **Stream** (Поток) и выберите **Custom** (Пользовательский) для **Service** (Сервис).
3. Введите `rtmp://intl-rtmp.rtc.qq.com/push/` для **Server** (Сервер).
4. Введите ключ потока в следующем формате:

```
Room ID?sdkappid=Application&userid=User ID&usersig=Signature
```

Замените «Room ID», «Application ID», «User ID» и «Signature» на фактические значения, например:

```
hello-string-room?sdkappid=140*****66&userid=******rtmp2&usersig=eJw1jdE***************ZLgi5UAgOzoMhrayt*cjbmiCJ699T09juc833IMT94Ld7I0iHZqVDzvVAqkZsG-IKlzLiXOnEhswHu1iUyTc9pv*****D8MQwoA496Ke6U1ip4EAH4UMc5H9pSmv6MeTBWLamhwFnWRBZ8qKGRj8Yp-wVbv*mGMVZqS7w-mMDQL
```

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/4f6b507aa2ae11edbb45525400c56988.png)

### Шаг 3. Настройка выходных данных

Так как RTMP не поддерживает B-кадры, установите параметры видеокодирования следующим образом для удаления B-кадров.

1. Перейдите в **Controls** (Элементы управления) > **Settings** (Параметры) > **Output** (Вывод).
2. Выберите **Advanced** (Дополнительно) для **Output Mode** (Режим вывода). Рекомендуемый **Keyframe Interval** (Интервал ключевых кадров) — это **1** или **2**. Для **CPU Usage Preset** (Предустановка использования ЦП) выберите **ultrafast** (Очень быстро). Для **Profile** (Профиль) выберите **baseline** (Базовый). Для **Tune** (Настройка) выберите **zerolatency** (Нулевая задержка). И для **x264 Options** (Параметры x264) введите `threads=1`, затем нажмите **OK** (ОК).

> **Предупреждение:** Вам необходимо удалить B-кадры в потоках RTMP, в противном случае соединение будет разорвано после отправки. Чтобы добиться этого, выберите baseline для Profile.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/685ee210c6fe11edbe95525400088f3a.png)

### Шаг 4. Установка параметров видео

Вы можете установить разрешение видео и частоту кадров в разделе **Video** (Видео) параметров **Settings** (Параметры). Разрешение определяет чёткость видео, показываемого зрителям. Чем выше разрешение, тем чётче видео. Частота кадров (кадров в секунду) определяет плавность воспроизведения. Типичная частота кадров находится в диапазоне от 24 кадров в секунду до 30 кадров в секунду. Воспроизведение может заикаться, если частота кадров ниже 16 кадров в секунду. Видеоигры требуют более высокой частоты кадров и, как правило, заикаются при частоте кадров ниже 30 кадров в секунду.

![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/00d7736d7b7c11ed8e7e525400463ef7.png)

### Шаг 5. Настройка дополнительных параметров

- Чтобы снизить сквозную задержку, мы рекомендуем не включать **Stream Delay** (Задержка потока).
- Держите **Automatically Reconnect** (Автоматическое переподключение) включённым и установите **Retry Delay** (Задержка повтора) как можно короче, чтобы издатель мог быстро переподключиться после отключения из-за колебаний сети.
![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/00fb5f967b7c11edab50525400c56988.png)

### Шаг 6. Публикация потока

1. На панели **Controls** (Элементы управления) в нижней части нажмите **Start Streaming** (Начать потоковую передачу).
![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/00ff974d7b7c11ed8e7e525400463ef7.png)
2. Если потоковая передача успешна, в нижней строке будет показана статистика потоковой передачи, и вход пользователя будет записан [панелью мониторинга](https://www.tencentcloud.com/document/product/647/39070) TRTC.
![](https://cloudcache.intl.tencent-cloud.com/cms/backend-cms/00f6df817b7c11edab50525400c56988.png)

### Шаг 7. Воспроизведение потока на других клиентах

Как упоминалось выше в разделе **Установка параметров публикации**, для воспроизведения потока RTMP на других клиентах TRTC необходимо использовать строковый идентификатор комнаты при входе в комнату. ## Использование FFmpeg для публикации потоков

Для публикации потоков RMTP с использованием команд FFmpeg или других библиотек RTMP необходимо использовать полный URL, видеокодек H.264 и аудиокодек AAC. Для формата контейнера рекомендуется FLV. Для GOP рекомендуется одна или две секунды.
Конфигурация параметров FFmpeg варьируется в зависимости от разных сценариев, поэтому вам необходимо иметь определённые знания о FFmpeg, чтобы использовать его для публикации потоков. В таблице ниже указаны некоторые распространённые команды FFmpeg. Для получения дополнительных параметров см. [документацию FFmpeg](https://ffmpeg.org/ffmpeg.html).

### Команды FFmpeg

```
ffmpeg [global_options] {[input_file_options] -i input_url} ... {[output_file_options] output_url}
```

### Распространённые параметры FFmpeg

| Параметр | Примечание |
| --- | --- |
| -re | Читает входные данные с нативной частотой кадров. Обычно используется для чтения локальных файлов. |

Параметры **output_file_options** включают:

| Параметр | Примечание |
| --- | --- |
| -c:v | Библиотека видеокодирования. Рекомендуется `libx264`. |
| -b:v | Битрейт видео. Например, `1500k` означает 1500 кбит/с. |
| -r | Частота кадров видео. |
| -profile:v | Профиль видео. Если установить его в `baseline`, B-кадры не будут кодироваться. Серверная часть TRTC не поддерживает B-кадры. |
| -g | GOP (интервал ключевого кадра). |
| -c:a | Библиотека аудиокодирования. Рекомендуется `libfdk_aac`. |
| -ac | Количество звуковых каналов. Допустимые значения: `1`, `2`. |
| -b:a | Битрейт звука. |
| -f | Формат контейнера. Установите его в `flv`. Для публикации в TRTC требуется формат контейнера FLV. |

Ниже приведён пример чтения локального файла и его публикации в TRTC (обратите внимание, что для URL требуются кавычки):

```
ffmpeg -loglevel debug -re -i sample.flv -c:v libx264 -preset ultrafast -profile:v baseline -g 30 -sc_threshold 0 -b:v 1500k -c:a libfdk_aac -ac 2 -b:a 128k -f flv 'rtmp://intl-rtmp.rtc.qq.com/push/hello-string-room?userid=rtmpForFfmpeg&sdkappid=140xxxxxx&usersig=xxxxxxxxxx'
```

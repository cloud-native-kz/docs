# Основные концепции Tencent RTC

В этом документе объясняются некоторые из основных концепций, с которыми вы можете столкнуться при использовании сервисов Tencent RTC.

### Приложение

Продукты Call, Conference, Live, RTC Engine и Chat компании Tencent RTC управляют различными бизнес-процессами или проектами через использование **приложений**. Вы можете создавать различные приложения для своих проектов в [консоли Tencent RTC](https://console.trtc.io/), чтобы разделить их данные. Подробнее см. в [руководстве по созданию приложения](https://www.tencentcloud.com/document/product/647/39077).

### SDKAppID

Tencent Cloud использует `SDKAppID` (ID приложения) для уникальной идентификации приложений TRTC. Он генерируется автоматически при создании приложения в [консоли Tencent RTC](https://console.trtc.io/). Приложения, не имеющие одинаковый `SDKAppID`, не могут обмениваться данными друг с другом.

### SDKSecretKey

`SDKSecretKey` — это ключ шифрования, связанный с `SDKAppID`, используемый для создания маркера аутентификации пользователя UserSig. Он обеспечивает законность и безопасность вызовов сервиса и автоматически генерируется при создании приложения в [консоли Tencent RTC](https://console.trtc.io/).

### UserID

UserID (ID пользователя) уникально идентифицирует пользователя в приложении TRTC.

- `UserID` является отображением учетных записей пользователей вашего проекта в Tencent Cloud. Обычно вы можете использовать имя пользователя в качестве `UserID`.
- `UserID` предпочтительно должен быть длиной не более 32 байт. Может содержать цифры, подчеркивания и буквы (с учетом регистра).

### TUICallKit и CallEngine

- TUICallKit: компонент для проведения вызовов в реальном времени с **полным пользовательским интерфейсом**, выпущенный компанией Tencent Cloud, также известный как продукт Call. Он предоставляет готовые компоненты пользовательского интерфейса для аудио и видео вызовов, поддерживающие вызовы 1v1, групповые вызовы, пользовательские рингтоны, плавающие окна и другие функции, что позволяет быстро интегрировать его в приложения.
- CallEngine: **не имеющий пользовательского интерфейса** базовый механизм аудио и видео вызовов. Разработчики могут полностью настраивать интерфейс взаимодействия вызовов на основе этого API. Он служит в качестве фундаментальной зависимости для компонентов пользовательского интерфейса (TUICallKit) и поддерживает более сложную настройку функций вызовов.

### TUIRoomKit, TUILiveKit и RoomEngine

- TUIRoomKit: компонент для проведения многопользовательских аудио и видео встреч в реальном времени с **полным пользовательским интерфейсом**, выпущенный компанией Tencent Cloud, также известный как продукт Conference. Он предоставляет готовые к использованию функции, такие как управление комнатой, совместное использование экрана, управление участниками и чат во время встречи. Поддерживает быструю интеграцию на несколько платформ.
- TUILiveKit: компонент для интерактивной потоковой трансляции в реальном времени с **полным пользовательским интерфейсом**, выпущенный компанией Tencent Cloud, также известный как продукт Live. Он предоставляет компоненты основной трансляции, поддерживающие начало трансляции, несколько гостей, подключение и батл якорей, чат в прямой трансляции, подарки, управление аудиторией и прочее, и может быть быстро интегрирован в приложение.
- RoomEngine: **не имеющий пользовательского интерфейса** базовый механизм аудио и видео комнаты. Он служит в качестве фундаментальной зависимости для компонентов пользовательского интерфейса (TUIRoomKit и TUILiveKit) и поддерживает настройку более продвинутых функций конференций и потоковой трансляции.

### MAU

MAU (Monthly Active Users — ежемесячно активные пользователи) — это общее количество уникальных пользователей, использующих функции Chat (такие как отправка/получение сообщений, управление группами) каждый месяц. Помимо продукта Chat, продукты Live, Call и Conference также генерируют MAU при использовании. Различные версии пакетов соответствуют различным бесплатным квотам; превышение квоты требует обновления или оплаты.

### Лицензия

Учетные данные авторизации, используемые для активации SDK Tencent Effects (такие как продукт Beauty AR), содержащие информацию о периоде действия услуги и версии пакета.

- Официальная лицензия: генерируется после покупки пакета красоты в [консоли Tencent RTC](https://console.trtc.io/subscription/buy/beauty), динамически получается через LicenseUrl (ссылка на скачивание файла авторизации) и LicenseKey (ключ авторизации).
- Пробная лицензия: в некоторых сценариях (например, локальная отладка) можно настроить временный файл (такой как v_cube.license), но первая сетевая аутентификация должна быть успешной.

### License Key

Ключ авторизации: используется вместе с LicenseUrl для проверки законности файла авторизации. Хранение его в открытом виде на фронтенде запрещено; в рабочей среде он должен быть динамически получен через API на стороне сервера.

### License Token

Ключ подписи: используется для создания временных подписей (Signature) для проверки законности запросов фронтенда. Token должен быть перенесен на сторону сервера, и подписи должны быть получены через API, чтобы избежать рисков утечки.

### Комната

Комната — это пространство, где пользователи могут получать друг друга в реальном времени аудио и видео данные.

- Комнаты — это виртуальные пространства, которые TRTC использует для разделения одной группы пользователей от другой.
- Только пользователи в одной комнате могут получать аудио и видео друг друга.
- Пользователь может находиться только в одной комнате одновременно. Пользователь, уже находящийся в комнате, должен выйти из комнаты, прежде чем сможет войти в другую.

> **Примечание:** Первый пользователь, входящий в комнату, является владельцем комнаты. Владельцы комнат не могут закрывать комнаты вручную. В режимах вызовов TRTC закрывает комнату, когда все пользователи выходят из комнаты. В режимах потоковой трансляции, если последний пользователь, выходящий из комнаты, является якорем, TRTC немедленно закроет комнату; если пользователь является зрителем, TRTC закроет комнату через 10 минут. Пользователь удаляется из комнаты через 90 секунд после неожиданного отключения. Если все пользователи отключены, комната закрывается через 90 секунд. **Время ожидания после разъединения также учитывается при выставлении счета**. Если пользователь пытается войти в несуществующую комнату, TRTC автоматически создаст комнату.

### Room ID

`RoomID` (номер комнаты/ID) уникально идентифицирует комнату в приложении Tencent RTC. Это число UInt32, которое вы назначаете комнате и поддерживаете. Диапазон его значений составляет 1-4294967295.

### UserSig

`UserSig` (подпись пользователя) — это подпись безопасности, разработанная Tencent Cloud для аутентификации входа пользователя и проверки того, что пользователь является реальным. Это помогает предотвратить доступ злоумышленников к вашему аккаунту Tencent Cloud. Подробнее см. в разделе [Создание UserSig](https://www.tencentcloud.com/document/product/647/35166).

### Push

Push — это операция, при которой пользователь загружает локальные аудио/видео данные на сервер Tencent RTC.

### Subscription

Subscription (подписка) — это операция, при которой пользователь отправляет запрос на сервер Tencent RTC для получения аудио/видео данных указанного пользователя.

### Роль

В Tencent RTC пользователи могут иметь одну из двух ролей: **якорь** (`TRTCRoleAnchor`) и **зритель** (`TRTCRoleAudience`).

- Якорь может загружать локальные аудио/видео данные на сервер. Они также могут подписываться на воспроизведение аудио/видео данных других якорей.
- Зрители **только** могут подписываться на воспроизведение аудио/видео данных якорей.

В режимах вызовов все пользователи в комнате имеют роль якоря. В режимах потоковой трансляции пользователи в комнате могут быть либо якорями, либо зрителями. Пользователь может переключать роли по мере необходимости.

### CDN Live Watching

CDN Live Watching (просмотр прямой трансляции через CDN), также известный как CDN relayed live streaming (ретрансляция прямой трансляции через CDN). Tencent RTC использует кластеры ретрансляционной перекодирования для преобразования своих UDP аудио/видео потоков в RTMP потоки в облаке. Затем потоки отправляются в стандартную систему потоковой трансляции и распространяются через CDN аудитории. Подробнее см. в разделе [Relay to CDN](https://www.tencentcloud.com/document/product/647/47858#).

### On-cloud recording

Tencent RTC использует возможности [CSS](https://intl.cloud.tencent.com/document/product/267) для записи полных аудио/видео вызовов в облаке. Файлы записи безопасно сохраняются в реальном времени в [VOD](https://intl.cloud.tencent.com/document/product/266). Подробнее см. в разделе [On-Cloud Recording](https://www.tencentcloud.com/document/product/647/45169#).

### On-Cloud MixTranscoding

В сценариях, таких как **CDN Live Watching** и **on-cloud recording**, вам может потребоваться смешать несколько аудио/видео потоков в комнате Tencent RTC в один поток. Это можно достичь, используя кластер MCU перемешивания и перекодирования потоков Tencent RTC. Кластер MCU может смешивать несколько аудио/видео потоков по мере необходимости и распространять смешанный поток на CDN потоковой трансляции и систему облачной записи. Подробнее см. в разделе [Relay to CDN](https://www.tencentcloud.com/document/product/647/47858#).

### Dumb terminal

Пользователь, входящий в комнату на немом терминале, не будет обнаружен SDK, и удаленные пользователи не получат уведомлений при входе или выходе пользователя из комнаты.
